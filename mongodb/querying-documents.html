Introduction

Querying documents is a fundamental aspect of working with MongoDB. This section will guide you through the basics of querying documents, advancing to more complex queries. By the end of this section, you will be able to retrieve and manipulate data stored in MongoDB collections efficiently.

Basic Queries

Finding All Documents

To retrieve all documents in a collection, use the find method without any parameters.

db.collection.find()

Finding Documents with a Filter

To find documents that match specific criteria, pass a query document to the find method.

db.collection.find({ "field": "value" })

Example:

db.users.find({ "age": 25 })

This query retrieves all documents from the users collection where the age field is 25.

Query Operators

MongoDB provides various operators to build complex queries.

Comparison Operators

  • $eq: Matches values that are equal to a specified value.
  • $ne: Matches all values that are not equal to a specified value.
  • $gt: Matches values that are greater than a specified value.
  • $gte: Matches values that are greater than or equal to a specified value.
  • $lt: Matches values that are less than a specified value.
  • $lte: Matches values that are less than or equal to a specified value.

Example:

db.users.find({ "age": { "$gt": 20, "$lt": 30 } })

This query retrieves all documents where the age field is greater than 20 and less than 30.

Logical Operators

  • $and: Joins query clauses with a logical AND.
  • $or: Joins query clauses with a logical OR.
  • $not: Inverts the effect of a query expression.
  • $nor: Joins query clauses with a logical NOR.

Example:

db.users.find({
  "$or": [
    { "age": { "$lt": 20 } },
    { "age": { "$gt": 30 } }
  ]
})

This query retrieves all documents where the age field is either less than 20 or greater than 30.

Projection

Projection is used to specify or restrict the fields to return in the query results.

Including Fields

To include specific fields in the result, use a projection document with the find method.

db.collection.find({}, { "field1": 1, "field2": 1 })

Example:

db.users.find({}, { "name": 1, "age": 1 })

This query retrieves all documents from the users collection, but only includes the name and age fields in the results.

Excluding Fields

To exclude specific fields, set the field to 0 in the projection document.

db.collection.find({}, { "field": 0 })

Example:

db.users.find({}, { "password": 0 })

This query retrieves all documents from the users collection, excluding the password field from the results.

Sorting

To sort the results of a query, use the sort method.

db.collection.find().sort({ "field": 1 }) // Ascending order
db.collection.find().sort({ "field": -1 }) // Descending order

Example:

db.users.find().sort({ "age": 1 })

This query retrieves all documents from the users collection, sorted by the age field in ascending order.

Limiting and Skipping

To limit the number of documents returned by a query, use the limit method. To skip a number of documents, use the skip method.

db.collection.find().limit(number)
db.collection.find().skip(number)

Example:

db.users.find().limit(5)
db.users.find().skip(5)

The first query retrieves the first 5 documents from the users collection. The second query skips the first 5 documents and retrieves the rest.

Advanced Queries

Nested Documents

To query nested documents, use dot notation.

db.collection.find({ "field.subfield": "value" })

Example:

db.users.find({ "address.city": "New York" })

This query retrieves all documents from the users collection where the city field in the address sub-document is "New York".

Array Queries

MongoDB provides operators to query arrays.

  • $elemMatch: Matches documents that contain an array field with at least one element that matches all the specified query criteria.
  • $size: Matches any array with the specified number of elements.

Example:

db.users.find({ "hobbies": { "$elemMatch": { "name": "reading", "frequency": "daily" } } })

This query retrieves all documents where the hobbies array contains an element with name "reading" and frequency "daily".

Conclusion

Querying documents in MongoDB is a powerful feature that allows you to retrieve and manipulate data efficiently. By understanding and utilizing various query operators, projection, sorting, and advanced query techniques, you can perform complex data retrieval tasks with ease. Practice these concepts with real-world data to become proficient in querying MongoDB collections.
