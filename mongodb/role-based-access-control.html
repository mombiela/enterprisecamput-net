Introduction to RBAC

Role-Based Access Control (RBAC) is a method of regulating access to a computer or network resources based on the roles of individual users within an enterprise. In MongoDB, RBAC is used to manage permissions and ensure that users have the appropriate access levels to perform their tasks.

Key Concepts

  • Role: A collection of privileges that can be assigned to a user.
  • Privilege: A specific action that a user can perform on a resource.
  • User: An individual who is assigned one or more roles.

Setting Up RBAC in MongoDB

Creating Users

To create a user in MongoDB, you use the db.createUser() method. Each user must be assigned at least one role.

db.createUser({
  user: "username",
  pwd: "password",
  roles: [
    { role: "readWrite", db: "exampleDB" }
  ]
});
  • user: The username.
  • pwd: The password for the user.
  • roles: An array of roles assigned to the user.

Built-in Roles

MongoDB provides several built-in roles that cover common use cases. Some of the key built-in roles include:

| Role | Description | |---------------|------------------------------------------------| | read | Grants read-only access to the database. | | readWrite | Grants read and write access to the database. | | dbAdmin | Provides administrative privileges on the database. | | userAdmin | Allows user management within the database. | | clusterAdmin| Grants administrative privileges on the cluster. |

Custom Roles

You can also create custom roles to meet specific requirements.

db.createRole({
  role: "customRole",
  privileges: [
    { resource: { db: "exampleDB", collection: "" }, actions: ["find", "insert"] }
  ],
  roles: []
});
  • role: The name of the custom role.
  • privileges: An array of privileges, where each privilege specifies the resource and the actions allowed.
  • roles: An array of roles that this role inherits.

Managing User Roles

Assigning Roles to Users

You can assign additional roles to an existing user using the db.grantRolesToUser() method.

db.grantRolesToUser("username", [{ role: "dbAdmin", db: "exampleDB" }]);

Revoking Roles from Users

To revoke roles from a user, use the db.revokeRolesFromUser() method.

db.revokeRolesFromUser("username", [{ role: "readWrite", db: "exampleDB" }]);

Practical Examples

Example 1: Creating a Read-Only User

db.createUser({
  user: "readonlyUser",
  pwd: "readonlyPass",
  roles: [
    { role: "read", db: "exampleDB" }
  ]
});

This creates a user readonlyUser with read-only access to exampleDB.

Example 2: Creating a User with Custom Role

db.createRole({
  role: "dataAnalyst",
  privileges: [
    { resource: { db: "exampleDB", collection: "" }, actions: ["find"] }
  ],
  roles: []
});

db.createUser({
  user: "analystUser",
  pwd: "analystPass",
  roles: [
    { role: "dataAnalyst", db: "exampleDB" }
  ]
});

This creates a custom role dataAnalyst with read-only access and assigns it to analystUser.

Conclusion

RBAC in MongoDB is a powerful feature that allows you to manage user permissions effectively. By understanding and utilizing roles, privileges, and users, you can ensure that your MongoDB deployment is secure and that users have the appropriate access levels to perform their tasks. Whether using built-in roles or creating custom roles, RBAC provides the flexibility needed to handle various access control scenarios.
