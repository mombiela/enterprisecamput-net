Introduction to `$lookup`

MongoDB is a NoSQL database that doesn't support traditional SQL joins. However, MongoDB provides the $lookup aggregation stage to perform join operations between collections.

  • Purpose: $lookup is used to join documents from different collections.
  • Syntax: 
    {
  $lookup: {
    from: <collection to join>,
    localField: <field from the input documents>,
    foreignField: <field from the documents of the "from" collection>,
    as: <output array field>
  }
}

Basic Usage of `$lookup`

Let's start with a simple example to understand how $lookup works.

Example Scenario

We have two collections: orders and customers.

orders Collection

[
  { "_id": 1, "product": "Laptop", "customerId": 101 },
  { "_id": 2, "product": "Phone", "customerId": 102 },
  { "_id": 3, "product": "Tablet", "customerId": 101 }
]

customers Collection

[
  { "_id": 101, "name": "John Doe", "email": "john@example.com" },
  { "_id": 102, "name": "Jane Smith", "email": "jane@example.com" }
]

Performing a Join

We want to join the orders collection with the customers collection to include customer details in each order.

Aggregation Pipeline

db.orders.aggregate([
  {
    $lookup: {
      from: "customers",
      localField: "customerId",
      foreignField: "_id",
      as: "customerDetails"
    }
  }
])

Result

[
  {
    "_id": 1,
    "product": "Laptop",
    "customerId": 101,
    "customerDetails": [
      { "_id": 101, "name": "John Doe", "email": "john@example.com" }
    ]
  },
  {
    "_id": 2,
    "product": "Phone",
    "customerId": 102,
    "customerDetails": [
      { "_id": 102, "name": "Jane Smith", "email": "jane@example.com" }
    ]
  },
  {
    "_id": 3,
    "product": "Tablet",
    "customerId": 101,
    "customerDetails": [
      { "_id": 101, "name": "John Doe", "email": "john@example.com" }
    ]
  }
]

Advanced Usage of `$lookup`

Unwinding the Joined Array

By default, $lookup creates an array of matched documents. To flatten this array, we use the $unwind stage.

Example

db.orders.aggregate([
  {
    $lookup: {
      from: "customers",
      localField: "customerId",
      foreignField: "_id",
      as: "customerDetails"
    }
  },
  { $unwind: "$customerDetails" }
])

Result

[
  {
    "_id": 1,
    "product": "Laptop",
    "customerId": 101,
    "customerDetails": { "_id": 101, "name": "John Doe", "email": "john@example.com" }
  },
  {
    "_id": 2,
    "product": "Phone",
    "customerId": 102,
    "customerDetails": { "_id": 102, "name": "Jane Smith", "email": "jane@example.com" }
  },
  {
    "_id": 3,
    "product": "Tablet",
    "customerId": 101,
    "customerDetails": { "_id": 101, "name": "John Doe", "email": "john@example.com" }
  }
]

Handling Unmatched Documents

To include documents that do not have a match in the joined collection, use the preserveNullAndEmptyArrays option with $unwind.

Example

db.orders.aggregate([
  {
    $lookup: {
      from: "customers",
      localField: "customerId",
      foreignField: "_id",
      as: "customerDetails"
    }
  },
  { $unwind: { path: "$customerDetails", preserveNullAndEmptyArrays: true } }
])

Result

If there were orders with customerId not present in the customers collection, those orders would still appear in the result with customerDetails set to null.

Summary

  • Basic Join: Use $lookup to join documents from different collections.
  • Flattening Results: Use $unwind to flatten the array created by $lookup.
  • Handling Unmatched Documents: Use preserveNullAndEmptyArrays with $unwind to include documents without matches.

By mastering $lookup, you can effectively perform join operations in MongoDB, enabling you to combine data from multiple collections in a flexible and powerful way.
