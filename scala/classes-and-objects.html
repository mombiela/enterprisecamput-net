Introduction

In Scala, classes and objects are fundamental building blocks for creating structured and reusable code. This section will guide you through the basics of defining classes and objects, and how to use them effectively in your Scala programs.

Classes in Scala

A class in Scala is a blueprint for creating objects. It can contain fields (variables) and methods (functions) to define the behavior and state of the objects.

Defining a Class

To define a class in Scala, you use the class keyword followed by the class name and an optional parameter list.

class Person(val name: String, val age: Int) {
  def greet(): String = s"Hello, my name is $name and I am $age years old."
}
  • class Person(val name: String, val age: Int): This defines a class named Person with two parameters, name and age.
  • def greet(): String = ...: This defines a method named greet that returns a greeting string.

Creating Objects from a Class

To create an object from a class, you use the new keyword.

val person = new Person("Alice", 30)
println(person.greet()) // Output: Hello, my name is Alice and I am 30 years old.

Objects in Scala

In Scala, an object is a singleton instance of a class. Objects are defined using the object keyword and can contain methods and values.

Defining an Object

An object is similar to a class but it cannot take parameters.

object MathUtils {
  def add(a: Int, b: Int): Int = a + b
  def subtract(a: Int, b: Int): Int = a - b
}
  • object MathUtils: This defines a singleton object named MathUtils.
  • def add(a: Int, b: Int): Int = ...: This defines a method named add that returns the sum of two integers.

Using an Object

You can call methods on an object directly without creating an instance.

val sum = MathUtils.add(5, 3)
println(sum) // Output: 8

Companion Objects

In Scala, a companion object is an object that shares the same name as a class and is defined in the same file. Companion objects are used to define methods and values that are related to the class but not tied to a specific instance.

Example of Companion Object

class Circle(val radius: Double) {
  def area: Double = Circle.pi * radius * radius
}

object Circle {
  val pi: Double = 3.14159
}
  • class Circle(val radius: Double): This defines a class named Circle.
  • object Circle: This defines a companion object for the Circle class.
  • val pi: Double = 3.14159: This defines a constant value pi in the companion object.

Using Companion Objects

You can access the members of a companion object using the class name.

val circle = new Circle(5.0)
println(circle.area) // Output: 78.53975

Case Classes

Case classes are a special type of class in Scala that are immutable and come with built-in methods for comparison and pattern matching.

Defining a Case Class

case class Point(x: Int, y: Int)
  • case class Point(x: Int, y: Int): This defines a case class named Point with two parameters, x and y.

Creating and Using Case Classes

You can create instances of case classes without using the new keyword.

val point1 = Point(1, 2)
val point2 = Point(1, 2)
println(point1 == point2) // Output: true

Summary

In this section, we covered the basics of classes and objects in Scala, including:

  • How to define and instantiate classes.
  • How to define and use singleton objects.
  • The concept of companion objects and their usage.
  • The special features of case classes.

Understanding these concepts is crucial for writing organized and reusable code in Scala. As you progress, you'll see how these building blocks fit into more complex Scala applications.
