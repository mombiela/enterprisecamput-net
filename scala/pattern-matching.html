Pattern Matching in Scala
Exception Handling ►

Pattern matching is a powerful feature in Scala that allows you to match values against patterns and decompose data structures. It is similar to switch-case statements in other languages but much more powerful and expressive.

Introduction to Pattern Matching

  • Definition: Pattern matching is a mechanism for checking a value against a pattern. It is a more powerful version of the switch statement found in many other languages.
  • Use Cases: It is commonly used for:
    • Decomposing data structures
    • Handling different types of data
    • Implementing control flow based on pattern matching

Basic Syntax

The basic syntax of pattern matching in Scala uses the match keyword followed by case statements.

val number = 3

number match {
  case 1 => println("One")
  case 2 => println("Two")
  case 3 => println("Three")
  case _ => println("Other number")
}
  • Explanation:
    • number match { ... }: The match keyword is used to start the pattern matching.
    • case 1 => println("One"): Each case keyword introduces a pattern to match against the value.
    • case _ => println("Other number"): The underscore _ is a wildcard pattern that matches any value.

Matching on Types

Pattern matching can also be used to match on types, which is useful for handling different types of data in a type-safe manner.

def process(value: Any): String = value match {
  case i: Int => s"Integer: $i"
  case s: String => s"String: $s"
  case _ => "Unknown type"
}

println(process(42))        // Output: Integer: 42
println(process("Scala"))   // Output: String: Scala
println(process(3.14))      // Output: Unknown type
  • Explanation:
    • case i: Int => ...: Matches if value is of type Int.
    • case s: String => ...: Matches if value is of type String.
    • case _ => ...: Matches any other type.

Decomposing Data Structures

Pattern matching is particularly powerful when working with complex data structures like tuples, lists, and case classes.

Tuples

val tuple = (1, "Scala")

tuple match {
  case (1, lang) => println(s"Language: $lang")
  case (2, lang) => println(s"Second language: $lang")
  case _ => println("Other tuple")
}
  • Explanation:
    • case (1, lang) => ...: Matches a tuple where the first element is 1 and binds the second element to lang.

Lists

val list = List(1, 2, 3)

list match {
  case List(1, 2, 3) => println("Exact match")
  case List(1, _*) => println("Starts with 1")
  case _ => println("Other list")
}
  • Explanation:
    • case List(1, 2, 3) => ...: Matches a list with exactly three elements: 1, 2, and 3.
    • case List(1, _*) => ...: Matches a list that starts with 1 followed by any number of elements.

Case Classes

case class Person(name: String, age: Int)

val person = Person("Alice", 25)

person match {
  case Person("Alice", age) => println(s"Alice is $age years old")
  case Person(name, age) => println(s"$name is $age years old")
  case _ => println("Unknown person")
}
  • Explanation:
    • case Person("Alice", age) => ...: Matches a Person object with the name "Alice" and binds the age to age.
    • case Person(name, age) => ...: Matches any Person object and binds the name and age to name and age.

Advanced Pattern Matching

Nested Patterns

Pattern matching can be nested to handle more complex data structures.

val nestedList = List(List(1, 2), List(3, 4))

nestedList match {
  case List(List(1, 2), List(a, b)) => println(s"Nested match: $a, $b")
  case _ => println("Other nested list")
}
  • Explanation:
    • case List(List(1, 2), List(a, b)) => ...: Matches a list containing two lists, where the first list is exactly List(1, 2) and the second list has two elements that are bound to a and b.

Guards

Guards add additional conditions to patterns.

val number = 5

number match {
  case x if x % 2 == 0 => println("Even number")
  case x if x % 2 != 0 => println("Odd number")
  case _ => println("Unknown number")
}
  • Explanation:
    • case x if x % 2 == 0 => ...: Matches if x is an even number.
    • case x if x % 2 != 0 => ...: Matches if x is an odd number.

Summary

Pattern matching in Scala is a versatile and powerful feature that allows you to:

  • Match values against patterns.
  • Decompose complex data structures.
  • Handle different types of data safely.
  • Implement control flow based on pattern matching.

By mastering pattern matching, you can write more expressive and concise Scala code, making it easier to handle a wide range of programming scenarios.
