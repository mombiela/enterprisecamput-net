Introduction

Futures and Promises are powerful constructs in Scala that allow you to work with concurrent and asynchronous computations. They are essential for writing non-blocking code and handling tasks that may complete at some point in the future.

Futures

A Future represents a value that may not yet be available. It is a placeholder for a result that will eventually be computed.

Creating Futures

You can create a Future using the Future companion object and an implicit ExecutionContext.

import scala.concurrent._
import ExecutionContext.Implicits.global

val future: Future[Int] = Future {
  // Simulate a long-running computation
  Thread.sleep(1000)
  42
}

Working with Futures

Futures provide several methods to work with their results once they are available.

onComplete

The onComplete method allows you to specify a callback that will be executed when the future completes.

future.onComplete {
  case Success(value) => println(s"The result is $value")
  case Failure(exception) => println(s"An error occurred: ${exception.getMessage}")
}

map and flatMap

You can transform the result of a future using map and flatMap.

val transformedFuture: Future[String] = future.map(value => s"The answer is $value")

for-comprehensions

For-comprehensions provide a more readable way to chain multiple futures.

val future1 = Future { 10 }
val future2 = Future { 20 }

val combinedFuture: Future[Int] = for {
  value1 <- future1
  value2 <- future2
} yield value1 + value2

Promises

A Promise is a writable, single-assignment container that completes a Future. You can think of a Promise as a way to manually control the completion of a Future.

Creating Promises

You can create a Promise using the Promise companion object.

val promise: Promise[Int] = Promise[Int]()
val futureFromPromise: Future[Int] = promise.future

Completing Promises

You can complete a Promise by calling success or failure.

promise.success(42)
// or
promise.failure(new Exception("Something went wrong"))

Example: Using Promises to Complete Futures

Here is an example that demonstrates how to use a Promise to complete a Future.

import scala.concurrent._
import ExecutionContext.Implicits.global

val promise = Promise[Int]()
val future = promise.future

future.onComplete {
  case Success(value) => println(s"The result is $value")
  case Failure(exception) => println(s"An error occurred: ${exception.getMessage}")
}

// Simulate some asynchronous computation
Future {
  Thread.sleep(1000)
  promise.success(42)
}

Comparison: Futures vs Promises

| Feature | Future | Promise | |------------------|---------------------------------|---------------------------------| | Creation | Created with Future object | Created with Promise object | | Completion | Completed by the computation | Manually completed by the user | | Readability | Read-only | Read-write | | Use Case | Asynchronous computation result | Manual control over completion |

Conclusion

Futures and Promises are essential tools for handling asynchronous and concurrent programming in Scala. Futures allow you to work with values that will be available in the future, while Promises give you control over when and how those values are provided. By understanding and using these constructs, you can write more efficient and non-blocking code.
