Introduction to Queues

Queues are fundamental data structures that follow the FIFO (First In, First Out) principle, where the first element to enter is the first to exit. They are widely used in scenarios where tasks need to be managed in sequential order, such as in printing systems, process management in operating systems, and graph search algorithms.

Key Concepts

  • FIFO (First In, First Out): The first element inserted into the queue is the first to be removed.
  • Basic Operations:
    • enqueue: Add an element to the end of the queue.
    • dequeue: Remove the first element from the queue.
    • peek or front: Get the first element of the queue without removing it.
    • isEmpty: Check if the queue is empty.
  • Common Implementations:
    • Queues using linked lists.
    • Queues using arrays.

Code Examples

Queue Implementation Using Arrays

class Queue:
    def __init__(self):
        self.queue = []

    def enqueue(self, item):
        self.queue.append(item)
        print(f"Enqueued: {item}")

    def dequeue(self):
        if not self.isEmpty():
            item = self.queue.pop(0)
            print(f"Dequeued: {item}")
            return item
        else:
            print("Queue is empty")
            return None

    def peek(self):
        if not self.isEmpty():
            return self.queue[0]
        else:
            print("Queue is empty")
            return None

    def isEmpty(self):
        return len(self.queue) == 0

# Usage example
q = Queue()
q.enqueue(1)
q.enqueue(2)
q.enqueue(3)
print(f"Front element: {q.peek()}")
q.dequeue()
q.dequeue()
q.dequeue()
q.dequeue()  # Attempt to dequeue when the queue is empty

Queue Implementation Using Linked Lists

class Node:
    def __init__(self, value):
        self.value = value
        self.next = None

class LinkedListQueue:
    def __init__(self):
        self.front = None
        self.rear = None

    def enqueue(self, item):
        new_node = Node(item)
        if self.rear is None:
            self.front = self.rear = new_node
        else:
            self.rear.next = new_node
            self.rear = new_node
        print(f"Enqueued: {item}")

    def dequeue(self):
        if self.isEmpty():
            print("Queue is empty")
            return None
        temp = self.front
        self.front = temp.next
        if self.front is None:
            self.rear = None
        print(f"Dequeued: {temp.value}")
        return temp.value

    def peek(self):
        if not self.isEmpty():
            return self.front.value
        else:
            print("Queue is empty")
            return None

    def isEmpty(self):
        return self.front is None

# Usage example
llq = LinkedListQueue()
llq.enqueue(1)
llq.enqueue(2)
llq.enqueue(3)
print(f"Front element: {llq.peek()}")
llq.dequeue()
llq.dequeue()
llq.dequeue()
llq.dequeue()  # Attempt to dequeue when the queue is empty

Practical Exercises

  1. Implement a Circular Queue:

    • Description: Implement a circular queue using arrays. Ensure proper handling of cases where the queue is full or empty.
    • Key points:
      • Use of front and rear pointers.
      • Handling wrap-around in the array.

  2. Simulation of a Printing System:

    • Description: Simulate a printing system where print jobs are queued and processed in order.
    • Key points:
      • Each print job has a unique identifier and duration.
      • Implement a function to add jobs and another to process them.

  3. Basic Priority Queue:

    • Description: Implement a basic priority queue where each element has an associated priority and elements with higher priority are processed first.
    • Key points:
      • Use an appropriate data structure to handle priorities (can be a heap or a sorted list).
      • Implementation of enqueue and dequeue operations with priorities.

Conclusion

Through the provided examples and exercises, students can strengthen their understanding and skills in handling queues, better preparing themselves to tackle real-world problems that require this data structure.
