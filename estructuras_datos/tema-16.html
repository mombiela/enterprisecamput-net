Introduction to Queues

Queues are linear data structures that follow the FIFO (First In, First Out) principle, meaning the first element to enter is the first to leave. They are useful in situations where elements need to be processed in the order they were received.

Key Concepts

  • FIFO (First In, First Out): The first element added to the queue will be the first to be removed.
  • Enqueue: Operation to add an element to the end of the queue.
  • Dequeue: Operation to remove the first element from the queue.
  • Front: Operation to get the first element of the queue without removing it.
  • Rear: Operation to get the last element of the queue.

Types of Queues

Simple Queue

A simple queue allows insertion operations at one end (rear) and deletion operations at the other end (front).

Circular Queue

A circular queue is an improvement over the simple queue that connects the last element of the queue with the first, forming a circle. This allows efficient use of space.

Priority Queue

A priority queue is a structure where each element has an associated priority. Elements with higher priority are processed before those with lower priority.

Queue Implementation in Python

Simple Queue

Below is a basic implementation of a simple queue using a list in Python:

class Queue:
    def __init__(self):
        self.items = []

    def is_empty(self):
        return len(self.items) == 0

    def enqueue(self, item):
        self.items.append(item)

    def dequeue(self):
        if not self.is_empty():
            return self.items.pop(0)
        else:
            raise IndexError("Dequeue from empty queue")

    def front(self):
        if not self.is_empty():
            return self.items[0]
        else:
            raise IndexError("Front from empty queue")

    def rear(self):
        if not self.is_empty():
            return self.items[-1]
        else:
            raise IndexError("Rear from empty queue")

    def size(self):
        return len(self.items)

# Usage example
q = Queue()
q.enqueue(1)
q.enqueue(2)
q.enqueue(3)
print(q.dequeue())  # Output: 1
print(q.front())    # Output: 2
print(q.rear())     # Output: 3
print(q.size())     # Output: 2

Circular Queue

The implementation of a circular queue can be more efficient in terms of memory usage:

class CircularQueue:
    def __init__(self, max_size):
        self.max_size = max_size
        self.queue = [None] * max_size
        self.front = -1
        self.rear = -1

    def is_empty(self):
        return self.front == -1

    def is_full(self):
        return (self.rear + 1) % self.max_size == self.front

    def enqueue(self, item):
        if self.is_full():
            raise OverflowError("Enqueue on full queue")
        elif self.is_empty():
            self.front = self.rear = 0
        else:
            self.rear = (self.rear + 1) % self.max_size
        self.queue[self.rear] = item

    def dequeue(self):
        if self.is_empty():
            raise IndexError("Dequeue from empty queue")
        elif self.front == self.rear:
            item = self.queue[self.front]
            self.front = self.rear = -1
        else:
            item = self.queue[self.front]
            self.front = (self.front + 1) % self.max_size
        return item

    def front_item(self):
        if self.is_empty():
            raise IndexError("Front from empty queue")
        return self.queue[self.front]

    def rear_item(self):
        if self.is_empty():
            raise IndexError("Rear from empty queue")
        return self.queue[self.rear]

# Usage example
cq = CircularQueue(3)
cq.enqueue(1)
cq.enqueue(2)
cq.enqueue(3)
print(cq.dequeue())  # Output: 1
cq.enqueue(4)
print(cq.front_item())  # Output: 2
print(cq.rear_item())   # Output: 4

Conclusion

Understanding how to implement and use queues is essential for any IT professional.
