Introduction

In this section of the CI/CD course, you will learn how to set up a continuous deployment server. This process is essential for automating the integration and deployment of applications, improving efficiency, and reducing human errors. We will use popular tools like Jenkins, GitLab CI/CD, and CircleCI to illustrate the concepts.

Prerequisites

Before starting, make sure you have the following prerequisites:

  • Basic knowledge of CI/CD.
  • Familiarity with Unix/Linux-based operating systems.
  • Access to a server or virtual machine with administrator permissions.
  • Docker installation (optional but recommended).

Choosing the CI/CD Tool

There are several tools to set up a continuous deployment server. Here we compare some of the most popular ones:

| Tool | Advantages | Disadvantages | |---------------|-------------------------------------------------|----------------------------------------------| | Jenkins | Open-source, highly configurable | Requires extensive manual configuration | | GitLab CI/CD | Integrated with GitLab, easy to use | Can be costly in premium versions | | CircleCI | Easy to set up, good integration with GitHub | Limitations in the free plan |

Installing Jenkins

Step 1: Installing Java

Jenkins requires Java to run. Install Java with the following commands:

sudo apt update
sudo apt install openjdk-11-jdk -y

Step 2: Installing Jenkins

Add the Jenkins repository and then install Jenkins:

wget -q -O - https://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -
sudo sh -c 'echo deb http://pkg.jenkins.io/debian-stable binary/ > /etc/apt/sources.list.d/jenkins.list'
sudo apt update
sudo apt install jenkins -y

Step 3: Starting Jenkins

Start the Jenkins service and check its status:

sudo systemctl start jenkins
sudo systemctl status jenkins

Step 4: Initial Configuration

Access Jenkins in your web browser at http://<your-server>:8080. Follow the on-screen instructions to complete the initial setup.

Setting Up a Basic Pipeline in Jenkins

Step 1: Create a New Job

  1. In the Jenkins dashboard, select "New Item".
  2. Choose "Pipeline" and provide a name for your job.

Step 2: Define the Pipeline

In the job configuration section, define your pipeline using the following example script:

pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                echo 'Building...'
                // Build commands
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
                // Test commands
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
                // Deployment commands
            }
        }
    }
}

Step 3: Run the Pipeline

Save the configuration and run the pipeline to verify that everything works correctly.

Setting Up GitLab CI/CD

Step 1: Create a `.gitlab-ci.yml` File

In the root directory of your project, create a file called .gitlab-ci.yml with the following content:

stages:
  - build
  - test
  - deploy

build:
  stage: build
  script:
    - echo "Building the project..."

test:
  stage: test
  script:
    - echo "Running tests..."

deploy:
  stage: deploy
  script:
    - echo "Deploying the project..."

Step 2: Configure Runners

Make sure you have runners configured in your GitLab instance. You can do this from the GitLab administration section.

Step 3: Verify the Pipeline

Commit and push your .gitlab-ci.yml file to your repository. GitLab will automatically run the defined pipeline.

Conclusion

Setting up a continuous deployment server is a crucial step for any development team looking to automate and improve their workflow. In this topic, we have covered the basic installation and configuration of Jenkins and GitLab CI/CD, two of the most popular tools in the CI/CD field. With this foundation, you will be prepared to explore more advanced and customized configurations according to your project's needs.

Keep practicing and experimenting with different configurations to master the art of continuous deployment!
