Introduction

In this exercise, you will learn to automate tests within a CI/CD pipeline. Test automation is crucial to ensure that new code does not break existing functionalities and that the software remains stable and of high quality.

Objectives

  • Understand the importance of test automation in CI/CD.
  • Set up an automated testing environment.
  • Integrate automated tests into a CI/CD pipeline.

Key Concepts

  • Unit Tests: Tests that verify the functionality of individual components of the code.
  • Integration Tests: Tests that verify the interaction between different components of the system.
  • Functional Tests: Tests that ensure the system meets functional requirements.
  • CI/CD Pipeline: A set of automated processes that allow continuous integration and continuous delivery of software.

Setting Up the Testing Environment

Necessary Tools

  • Testing Framework: JUnit (for Java), pytest (for Python), etc.
  • CI/CD Tool: Jenkins, GitLab CI, Travis CI, etc.

Example Configuration with pytest and GitLab CI

Step 1: Set up pytest

  1. Install pytest:

    pip install pytest

  2. Create a test file test_sample.py:

    def test_example():
    assert 1 + 1 == 2

Step 2: Set up GitLab CI

  1. Create a .gitlab-ci.yml file at the root of your project:

    stages:
  - test

test:
  stage: test
  script:
    - pip install pytest
    - pytest

  2. This file defines a test stage that installs pytest and runs the tests.

Integrating Tests into the CI/CD Pipeline

Example Pipeline in Jenkins

  1. Install the Git plugin in Jenkins.
  2. Set up a new Freestyle project in Jenkins.
  3. In the "Source Code Management" section, select Git and provide your repository URL.
  4. In the "Build Triggers" section, select "GitHub hook trigger for GITScm polling".
  5. In the "Build" section, add a shell execution step: 
    pip install pytest
pytest

Example Pipeline in GitLab CI

  1. Ensure your .gitlab-ci.yml file is correctly configured as shown in the previous section.
  2. Every time you push to your repository, GitLab CI will automatically run the tests.

Practical Exercise

  1. Set up pytest in your project: Follow the steps mentioned to install and set up pytest.
  2. Write Unit Tests: Create at least three unit tests for different functions in your code.
  3. Set up GitLab CI: Create a .gitlab-ci.yml file and ensure the tests run automatically on each push.
  4. Verify Results: Ensure the tests run correctly and verify the results in the GitLab CI interface.

Conclusion

Test automation is an essential component in any CI/CD pipeline. By integrating automated tests, you can detect errors early and ensure software quality. In this exercise, you have learned to set up a testing environment, write unit tests, and automate their execution in a CI/CD pipeline.
