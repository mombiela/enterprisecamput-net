Introduction

In this exercise, you will learn how to set up a basic CI/CD (Continuous Integration and Continuous Delivery) pipeline using a popular tool like GitHub Actions. This exercise is designed to help you understand the fundamental concepts and to practice a basic setup that you can expand on in future projects.

Exercise Objectives

  • Set up a GitHub repository for CI/CD.
  • Create a GitHub Actions configuration file.
  • Run automated tests on each push to the repository.
  • Automatically deploy to a staging environment.

Prerequisites

  • Basic knowledge of Git and GitHub.
  • Familiarity with YAML.
  • A simple project in any programming language (we will use a Python project for this example).

Step 1: Create a Repository on GitHub

  1. Go to GitHub and create a new account if you don't have one.
  2. Create a new repository:
    • Name: ci-cd-demo
    • Description: Demo repository for basic CI/CD setup
    • Visibility: Public or private, according to your preference.

Step 2: Set Up GitHub Actions

  1. Navigate to the "Actions" tab in your new repository.
  2. Select "Set up a workflow yourself" to create a configuration file from scratch.
  3. Create a file named ci.yml in the .github/workflows/ directory.

Configuration File Example

name: CI/CD Pipeline

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Python
      uses: actions/setup-python@v2
      with:
        python-version: '3.8'

    - name: Install dependencies
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt

    - name: Run tests
      run: |
        pytest

File Explanation

  • name: Name of the workflow.
  • on: Defines the event that triggers the workflow (in this case, a push to the main branch).
  • jobs: Defines the jobs that will run.
    • build: Name of the job.
    • runs-on: Specifies the operating system of the runner.
    • steps: List of steps to execute.
      • Checkout code: Uses the actions/checkout@v2 action to clone the repository.
      • Set up Python: Uses the actions/setup-python@v2 action to set up Python.
      • Install dependencies: Installs the project dependencies.
      • Run tests: Runs the tests using pytest.

Step 3: Run the Pipeline

  1. Commit and push your ci.yml file to the repository.
  2. Navigate to the "Actions" tab in your repository to see the pipeline's progress.
  3. Verify that all stages (checkout, setup, install, test) run correctly.

Conclusion

In this exercise, you have set up a basic CI/CD pipeline using GitHub Actions. You have learned to:

  • Set up a GitHub repository for CI/CD.
  • Create and understand a GitHub Actions configuration file.
  • Run automated tests on each push to the repository.

This is just the beginning. As you become more familiar with these concepts, you will be able to expand your pipeline to include deployments to production, static code analysis, and much more.
