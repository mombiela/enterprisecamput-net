Final Project and Assessment

Overview

The final project and assessment for this NGINX course are designed to consolidate your learning from beginner to advanced levels. This project will test your understanding of NGINX concepts, configurations, and best practices. You will be required to set up a comprehensive NGINX server environment that includes various features and optimizations covered throughout the course.

Project Requirements

Your final project will involve setting up an NGINX server that meets the following requirements:

  • Basic Configuration:

    • Install NGINX on a Linux server.
    • Configure a basic HTTP server block to serve static content.

  • Reverse Proxy:

    • Set up NGINX as a reverse proxy for a backend application server (e.g., Node.js, Python Flask).

  • Load Balancing:

    • Implement load balancing with multiple backend servers.

  • SSL/TLS:

    • Configure SSL/TLS for secure connections using a self-signed certificate or a certificate from a trusted CA.

  • Caching:

    • Implement caching to improve performance.

  • Security:

    • Set up basic security measures, such as rate limiting and access control.

  • Logging and Monitoring:

    • Configure logging and set up basic monitoring using tools like nginx_status.

Step-by-Step Instructions

1. Basic Configuration

  • Install NGINX:

    sudo apt update
sudo apt install nginx

  • Configure HTTP Server Block:

    server {
    listen 80;
    server_name example.com;
    root /var/www/html;

    location / {
        try_files $uri $uri/ =404;
    }
}

2. Reverse Proxy

  • Set Up Reverse Proxy: 
    server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

3. Load Balancing

  • Implement Load Balancing: 
    upstream backend {
    server backend1.example.com;
    server backend2.example.com;
}

server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

4. SSL/TLS

  • Configure SSL/TLS: 
    server {
    listen 443 ssl;
    server_name example.com;

    ssl_certificate /etc/nginx/ssl/nginx.crt;
    ssl_certificate_key /etc/nginx/ssl/nginx.key;

    location / {
        try_files $uri $uri/ =404;
    }
}

5. Caching

  • Implement Caching: 
    server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://backend;
        proxy_cache my_cache;
        proxy_cache_valid 200 1h;
    }
}

proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m max_size=1g;

6. Security

  • Set Up Rate Limiting: 
    http {
    limit_req_zone $binary_remote_addr zone=mylimit:10m rate=1r/s;

    server {
        listen 80;
        server_name example.com;

        location / {
            limit_req zone=mylimit burst=5;
            proxy_pass http://backend;
        }
    }
}

7. Logging and Monitoring

  • Configure Logging:

    server {
    listen 80;
    server_name example.com;

    access_log /var/log/nginx/access.log;
    error_log /var/log/nginx/error.log;

    location / {
        try_files $uri $uri/ =404;
    }
}

  • Set Up Monitoring:

    server {
    listen 80;
    server_name example.com;

    location /nginx_status {
        stub_status;
        allow 127.0.0.1;
        deny all;
    }
}

Submission Guidelines

  • Documentation:

    • Provide a README file explaining your setup and configurations.
    • Include comments in your configuration files to explain each section.

  • Code Repository:

    • Push your project to a Git repository (e.g., GitHub, GitLab) and share the link.

  • Demo:

    • Provide a short video or live demo showcasing your NGINX setup and its features.

Conclusion

This final project is an opportunity to apply everything you've learned about NGINX, from basic configurations to advanced features. By completing this project, you will have a comprehensive understanding of how to set up and manage an NGINX server in a real-world scenario. Good luck!
