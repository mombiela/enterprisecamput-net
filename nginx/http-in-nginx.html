Overview

NGINX is a powerful web server that can also be used as a reverse proxy, load balancer, and HTTP cache. Understanding how NGINX handles HTTP requests is fundamental for configuring and optimizing your web server.

Key Concepts

  • HTTP Protocol: The foundation of data communication on the web.
  • NGINX Configuration File: The main file where you define how NGINX handles HTTP requests.
  • Directives: Instructions in the configuration file that control NGINX's behavior.
  • Contexts: Sections in the configuration file that group related directives.

NGINX Configuration File Structure

The NGINX configuration file is typically located at /etc/nginx/nginx.conf. It consists of directives and contexts.

Example Configuration File

user www-data;
worker_processes auto;
pid /run/nginx.pid;

events {
    worker_connections 1024;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    sendfile on;
    keepalive_timeout 65;

    server {
        listen 80;
        server_name example.com;

        location / {
            root /var/www/html;
            index index.html index.htm;
        }
    }
}

Explanation

  • Global Directives: user, worker_processes, and pid are global settings.
  • events Context: Contains directives related to network connections.
  • http Context: Contains directives for handling HTTP traffic.
  • server Block: Defines a virtual server.
  • location Block: Specifies how to process requests for a specific location.

Handling HTTP Requests

NGINX processes HTTP requests in a series of phases, including:

  1. Connection Phase: Establishing a connection with the client.
  2. Request Phase: Receiving and parsing the HTTP request.
  3. Response Phase: Generating and sending the HTTP response.

Example: Basic HTTP Server Configuration

server {
    listen 80;
    server_name mywebsite.com;

    location / {
        root /usr/share/nginx/html;
        index index.html;
    }

    location /images/ {
        root /data;
    }
}

Explanation

  • listen: Specifies the port NGINX listens on.
  • server_name: Defines the domain name.
  • location /: Handles requests to the root URL.
  • location /images/: Handles requests to the /images/ URL path.

Common Directives

  • listen: Specifies the IP address and port.
  • server_name: Defines the domain names.
  • root: Sets the root directory for requests.
  • index: Specifies the index file.

Example: Using Directives

server {
    listen 8080;
    server_name example.org;

    location / {
        root /var/www/example;
        index index.html index.htm;
    }
}

Explanation

  • listen 8080: NGINX listens on port 8080.
  • server_name example.org: The server responds to requests for example.org.
  • root /var/www/example: The root directory for requests is /var/www/example.
  • index index.html index.htm: The index files are index.html and index.htm.

Conclusion

Understanding how NGINX handles HTTP requests and how to configure it using directives and contexts is crucial for setting up a reliable and efficient web server. By mastering these basics, you can move on to more advanced configurations and optimizations.
