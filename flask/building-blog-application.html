Introduction

In this course, we will build a complete blog application using Flask, a popular micro web framework for Python. This course is designed to take you from a beginner to an advanced level, covering all the essential aspects of Flask development.

Setting Up the Environment

  • Install Flask: Use pip to install Flask. 
    pip install Flask
  • Project Structure: Organize your project files. 
    blog/
├── app.py
├── templates/
│   ├── base.html
│   ├── index.html
│   ├── post.html
├── static/
│   ├── style.css
└── models.py

Creating the Flask Application

  • Basic Flask App: Create a simple Flask application. 
    from flask import Flask

app = Flask(__name__)

@app.route('/')
def home():
    return "Hello, Blog!"

if __name__ == '__main__':
    app.run(debug=True)

Templates and Static Files

  • HTML Templates: Use Jinja2 templating engine. 
    <!-- templates/base.html -->
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>{% block title %}Blog{% endblock %}</title>
    <link rel="stylesheet" href="{{ url_for('static', filename='style.css') }}">
</head>
<body>
    {% block content %}{% endblock %}
</body>
</html>
  • Static Files: Serve CSS and JavaScript files. 
    /* static/style.css */
body {
    font-family: Arial, sans-serif;
}

Routing and Views

  • Dynamic Routing: Create routes with dynamic segments. 
    @app.route('/post/<int:post_id>')
def post(post_id):
    return f"Post {post_id}"

Models and Database Integration

  • SQLAlchemy Setup: Integrate SQLAlchemy for database management.

    from flask_sqlalchemy import SQLAlchemy

app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///blog.db'
db = SQLAlchemy(app)

  • Creating Models: Define models for the blog posts.

    # models.py
from app import db

class Post(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    title = db.Column(db.String(100), nullable=False)
    content = db.Column(db.Text, nullable=False)
    date_posted = db.Column(db.DateTime, nullable=False, default=datetime.utcnow)

Forms and User Input

  • WTForms: Use Flask-WTF for form handling. 
    from flask_wtf import FlaskForm
from wtforms import StringField, TextAreaField, SubmitField
from wtforms.validators import DataRequired

class PostForm(FlaskForm):
    title = StringField('Title', validators=[DataRequired()])
    content = TextAreaField('Content', validators=[DataRequired()])
    submit = SubmitField('Post')

User Authentication

  • Flask-Login: Implement user authentication. 
    from flask_login import LoginManager, UserMixin, login_user, login_required, logout_user, current_user

login_manager = LoginManager()
login_manager.init_app(app)

class User(UserMixin, db.Model):
    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(150), unique=True, nullable=False)
    password = db.Column(db.String(150), nullable=False)

Advanced Features

  • Pagination: Implement pagination for blog posts.

    @app.route('/posts')
def posts():
    page = request.args.get('page', 1, type=int)
    posts = Post.query.paginate(page=page, per_page=5)
    return render_template('posts.html', posts=posts)

  • Search Functionality: Add search functionality to the blog.

    @app.route('/search')
def search():
    query = request.args.get('query')
    results = Post.query.filter(Post.title.contains(query) | Post.content.contains(query)).all()
    return render_template('search_results.html', results=results)

Deployment

  • Deploying to Heroku: Steps to deploy your Flask app to Heroku. 
    heroku create
git push heroku master
heroku open

Conclusion

In this course, we have built a fully functional blog application using Flask. We covered the basics of setting up a Flask environment, creating routes and views, integrating a database, handling user input, implementing authentication, and adding advanced features like pagination and search. Finally, we deployed our application to Heroku. This comprehensive guide should give you a solid foundation to build and expand your own Flask applications.
