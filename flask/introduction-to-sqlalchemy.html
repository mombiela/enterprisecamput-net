Overview

SQLAlchemy is a powerful SQL toolkit and Object-Relational Mapping (ORM) library for Python. It provides a full suite of well-known enterprise-level persistence patterns, designed for efficient and high-performing database access. In this section, we will cover the basics of SQLAlchemy and how to integrate it with a Flask application.

Key Concepts

  • SQLAlchemy Core: The foundational layer that provides a SQL expression language and a schema-centric approach to database interaction.
  • SQLAlchemy ORM: An optional layer that provides a higher-level, object-oriented interface for database operations.
  • Engine: The starting point for any SQLAlchemy application, representing the database connection.
  • Session: A workspace for all the objects loaded from the database during a transaction.
  • Model: A Python class that maps to a database table.

Setting Up SQLAlchemy with Flask

Installation

To get started, you need to install Flask and SQLAlchemy. You can do this using pip:

pip install Flask SQLAlchemy

Configuration

First, configure your Flask application to use SQLAlchemy. Here is a basic example:

from flask import Flask
from flask_sqlalchemy import SQLAlchemy

app = Flask(__name__)
app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///example.db'
app.config['SQLALCHEMY_TRACK_MODIFICATIONS'] = False

db = SQLAlchemy(app)
  • SQLALCHEMY_DATABASE_URI: Specifies the database URL.
  • SQLALCHEMY_TRACK_MODIFICATIONS: Disables a feature that signals the application every time a change is about to be made in the database, which can be useful for debugging but is not recommended for production.

Defining Models

Models are Python classes that represent database tables. Here is an example of a simple model:

class User(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(80), unique=True, nullable=False)
    email = db.Column(db.String(120), unique=True, nullable=False)

    def __repr__(self):
        return f'<User {self.username}>'
  • db.Column: Defines a column in the database table.
  • db.Integer, db.String: Specify the data type of the column.
  • primary_key=True: Indicates that this column is the primary key.
  • unique=True: Ensures that all values in this column are unique.
  • nullable=False: Ensures that this column cannot be null.

Creating the Database

To create the database and tables, you need to run the following commands in a Python shell:

from your_application import db
db.create_all()

This will create the tables defined by your models in the database specified by SQLALCHEMY_DATABASE_URI.

CRUD Operations

Create

To add a new record to the database:

new_user = User(username='john_doe', email='john@example.com')
db.session.add(new_user)
db.session.commit()

Read

To query records from the database:

users = User.query.all()  # Get all users
user = User.query.filter_by(username='john_doe').first()  # Get a specific user

Update

To update an existing record:

user = User.query.filter_by(username='john_doe').first()
user.email = 'john_doe@example.com'
db.session.commit()

Delete

To delete a record:

user = User.query.filter_by(username='john_doe').first()
db.session.delete(user)
db.session.commit()

Conclusion

In this section, we introduced SQLAlchemy and demonstrated how to integrate it with a Flask application. We covered the basics of setting up SQLAlchemy, defining models, and performing CRUD operations. SQLAlchemy is a powerful tool that can greatly simplify database interactions in your Flask applications. As you progress, you will learn more advanced features and best practices for using SQLAlchemy effectively.
