Introduction

Git is a distributed version control system (VCS) that allows multiple people to work on a project simultaneously without overwriting each other's changes. It helps track changes, revert to previous stages, and collaborate efficiently. Created by Linus Torvalds in 2005, Git has become the most widely used version control system in the world.

Key Concepts

  • Version Control System (VCS): A tool that helps manage changes to source code over time.
  • Distributed: Unlike centralized VCS, Git allows every user to have a complete copy of the repository.
  • Repository (Repo): A storage location for software packages, containing all the project files and the history of changes.
  • Commit: A snapshot of your repository at a specific point in time.
  • Branch: A separate line of development, allowing you to work on different features or fixes independently.
  • Merge: The process of combining changes from different branches.
  • Clone: Creating a copy of a repository from a remote source.
  • Pull: Fetching changes from a remote repository and merging them into your local repository.
  • Push: Sending your local changes to a remote repository.

Basic Git Workflow

  1. Initialize a Repository: Create a new Git repository or clone an existing one.
  2. Make Changes: Modify files in your working directory.
  3. Stage Changes: Add changes to the staging area.
  4. Commit Changes: Save a snapshot of the staged changes.
  5. Push Changes: Send your commits to a remote repository.

Basic Git Commands

Initializing a Repository

To start a new Git repository, navigate to your project directory and run:

git init

This command creates a new .git directory in your project, which contains all the necessary metadata for the repository.

Cloning a Repository

To clone an existing repository, use:

git clone <repository_url>

This command copies the entire repository, including its history, to your local machine.

Staging Changes

To add changes to the staging area, use:

git add <file_name>

You can also add all changes at once:

git add .

Committing Changes

To commit the staged changes, use:

git commit -m "Your commit message"

The -m flag allows you to add a commit message inline.

Pushing Changes

To push your commits to a remote repository, use:

git push origin <branch_name>

This command sends your local commits to the specified branch on the remote repository.

Pulling Changes

To fetch and merge changes from a remote repository, use:

git pull origin <branch_name>

This command updates your local repository with the latest changes from the remote branch.

Example Workflow

  1. Initialize a Repository:

    git init

  2. Create a File:

    echo "Hello, Git!" > hello.txt

  3. Stage the File:

    git add hello.txt

  4. Commit the File:

    git commit -m "Add hello.txt"

  5. Create a Remote Repository (e.g., on GitHub) and link it:

    git remote add origin <repository_url>

  6. Push the Commit:

    git push -u origin master

Summary

Git is a powerful and flexible version control system that supports distributed development. Understanding its basic concepts and commands is essential for efficient collaboration and project management. By mastering Git, you can track changes, collaborate with others, and maintain a history of your project's development.
