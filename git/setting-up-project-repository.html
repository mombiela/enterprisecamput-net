Setting Up the Project Repository
Implementing Features and Collaboration ►

Setting up a project repository is a fundamental skill for any developer. This section will guide you through the process of initializing a Git repository, adding files, and making your first commit. By the end of this section, you'll have a solid understanding of how to set up and manage a Git repository.

Initializing a Git Repository

To start using Git, you need to initialize a repository in your project directory. This process creates a hidden .git directory that tracks all changes to your files.

Steps to Initialize a Repository

  1. Navigate to your project directory: 
    cd path/to/your/project
  2. Initialize the repository: 
    git init
    This command sets up the necessary files and directories for Git to start tracking your project.

Adding Files to the Repository

Once your repository is initialized, you need to add files to it. This process involves staging files and then committing them to the repository.

Staging Files

Staging files means preparing them for a commit. You can stage files individually or all at once.

  • Stage a single file: 
    git add filename
  • Stage all files: 
    git add .

Committing Files

After staging files, the next step is to commit them. A commit is a snapshot of your project at a specific point in time.

  • Commit staged files: 
    git commit -m "Initial commit"

Creating a .gitignore File

A .gitignore file specifies which files and directories Git should ignore. This is useful for excluding temporary files, build artifacts, and other files that do not need to be tracked.

Example .gitignore File

# Ignore node_modules directory
node_modules/

# Ignore log files
*.log

# Ignore build artifacts
dist/

To create a .gitignore file:

  1. Create the file: 
    touch .gitignore
  2. Edit the file: 
    nano .gitignore
  3. Add patterns to ignore (as shown in the example above).

Viewing the Repository Status

To see the current status of your repository, use the git status command. This command shows which files are staged, unstaged, and untracked.

  • Check repository status: 
    git status

Summary

In this section, you learned how to set up a Git repository from scratch. You now know how to:

  • Initialize a Git repository.
  • Stage and commit files.
  • Create and use a .gitignore file.
  • Check the status of your repository.

These foundational skills are essential for managing any project with Git. In the next sections, we will delve deeper into more advanced Git features and workflows.
