Introduction

Branches in Git allow you to diverge from the main line of development and continue to work without affecting the main codebase. This is particularly useful for developing new features, fixing bugs, or experimenting with new ideas.

Key Concepts

  • Branch: A pointer to a specific commit in the repository.
  • Main Branch: The default branch in a Git repository, typically named main or master.
  • Feature Branch: A branch created to develop a new feature.
  • Switching Branches: Moving from one branch to another to work on different parts of the project.

Creating a Branch

To create a new branch, you use the git branch command followed by the name of the new branch.

Example

git branch feature-branch

This command creates a new branch named feature-branch but does not switch to it.

Explanation

  • git branch: The command to create a new branch.
  • feature-branch: The name of the new branch.

Switching Branches

To switch to a different branch, you use the git checkout command followed by the name of the branch you want to switch to.

Example

git checkout feature-branch

This command switches your working directory to the feature-branch.

Explanation

  • git checkout: The command to switch branches.
  • feature-branch: The name of the branch you want to switch to.

Creating and Switching in One Step

You can create and switch to a new branch in one step using the -b flag with the git checkout command.

Example

git checkout -b new-feature

This command creates a new branch named new-feature and switches to it immediately.

Explanation

  • -b: A flag that tells Git to create a new branch and switch to it.
  • new-feature: The name of the new branch.

Viewing Branches

To see a list of all branches in your repository, use the git branch command without any arguments.

Example

git branch

This command lists all branches and highlights the current branch with an asterisk (*).

Explanation

  • git branch: The command to list all branches.

Deleting a Branch

Once a branch is no longer needed, you can delete it using the git branch -d command.

Example

git branch -d feature-branch

This command deletes the feature-branch.

Explanation

  • -d: A flag that tells Git to delete the specified branch.
  • feature-branch: The name of the branch to delete.

Summary

  • Creating a Branch: Use git branch <branch-name>.
  • Switching Branches: Use git checkout <branch-name>.
  • Creating and Switching in One Step: Use git checkout -b <branch-name>.
  • Viewing Branches: Use git branch.
  • Deleting a Branch: Use git branch -d <branch-name>.

Understanding how to create and switch branches in Git is fundamental for managing different lines of development in your projects. Practice these commands to become proficient in handling branches in Git.
