Introduction

Git is a powerful version control system that helps developers manage changes to their codebase. In this section, we will cover three fundamental Git commands: add, commit, and status. Understanding these commands is essential for anyone starting with Git.

Git `add` Command

The add command is used to add changes in the working directory to the staging area. This is the first step in the process of committing changes to the repository.

Key Concepts

  • Working Directory: The directory where your project files are located.
  • Staging Area: A temporary area where changes are gathered before committing.
  • Tracked Files: Files that Git is aware of and is tracking changes for.

Usage

# Add a single file to the staging area
git add <file_name>

# Add all changes in the current directory to the staging area
git add .

Example

# Adding a single file
git add index.html

# Adding all changes in the current directory
git add .

Git `commit` Command

The commit command is used to save changes from the staging area to the repository. Each commit is a snapshot of your project at a specific point in time.

Key Concepts

  • Commit Message: A brief description of the changes being committed.
  • SHA-1 Hash: A unique identifier for each commit.

Usage

# Commit changes with a message
git commit -m "Your commit message here"

Example

# Committing changes with a descriptive message
git commit -m "Added new feature to the homepage"

Git `status` Command

The status command displays the state of the working directory and the staging area. It shows which changes have been staged, which haven't, and which files aren't being tracked by Git.

Key Concepts

  • Untracked Files: Files that are not being tracked by Git.
  • Modified Files: Files that have been changed but not yet staged.
  • Staged Files: Files that have been added to the staging area.

Usage

# Check the status of the working directory and staging area
git status

Example

# Displaying the status of the repository
git status

Summary

Understanding the add, commit, and status commands is crucial for effective version control with Git. Here’s a quick recap:

  • Use git add to move changes to the staging area.
  • Use git commit to save staged changes to the repository with a descriptive message.
  • Use git status to check the state of your working directory and staging area.

By mastering these commands, you will be well on your way to efficiently managing your codebase with Git.
