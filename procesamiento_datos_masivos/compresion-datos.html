Introduction to Data Compression

Data compression is an essential technique in massive data processing, as it allows reducing the size of data to facilitate its storage and transmission. In this topic, we will explore the fundamental concepts of data compression, the most common algorithms, and how to apply them in the context of large volumes of data.

Key Concepts

  • Lossless Compression: Compression technique that allows recovering the original data exactly after decompression. Examples include Huffman Coding and Lempel-Ziv-Welch (LZW).
  • Lossy Compression: Compression technique that reduces data size by eliminating part of the information, resulting in a loss of quality. Examples include JPEG and MP3.
  • Compression Ratio: Measure of the effectiveness of a compression algorithm, calculated as the original size divided by the compressed size.
  • Entropy: Measure of the amount of information contained in a data set, which influences the compression capability.

Compression Algorithms

Lossless Compression

  1. Huffman Coding

    • Compression algorithm based on the frequency of character occurrences.
    • Builds a binary tree where the most frequent characters have shorter codes.
    import heapq
from collections import defaultdict, Counter

def huffman_encoding(data):
    frequency = Counter(data)
    heap = [[weight, [symbol, ""]] for symbol, weight in frequency.items()]
    heapq.heapify(heap)
    while len(heap) > 1:
        lo = heapq.heappop(heap)
        hi = heapq.heappop(heap)
        for pair in lo[1:]:
            pair[1] = '0' + pair[1]
        for pair in hi[1:]:
            pair[1] = '1' + pair[1]
        heapq.heappush(heap, [lo[0] + hi[0]] + lo[1:] + hi[1:])
    return sorted(heapq.heappop(heap)[1:], key=lambda p: (len(p[-1]), p))

data = "this is an example for huffman encoding"
huff = huffman_encoding(data)
print("Symbol\tWeight\tHuffman Code")
for p in huff:
    print(f"{p[0]}\t{frequency[p[0]]}\t{p[1]}")

  2. Lempel-Ziv-Welch (LZW)

    • Dictionary-based compression algorithm that replaces repeated substrings with references to a dictionary.
    def lzw_compress(uncompressed):
    dict_size = 256
    dictionary = {chr(i): i for i in range(dict_size)}
    w = ""
    result = []
    for c in uncompressed:
        wc = w + c
        if wc in dictionary:
            w = wc
        else:
            result.append(dictionary[w])
            dictionary[wc] = dict_size
            dict_size += 1
            w = c
    if w:
        result.append(dictionary[w])
    return result

data = "TOBEORNOTTOBEORTOBEORNOT"
compressed = lzw_compress(data)
print("Compressed data:", compressed)

Lossy Compression

  1. JPEG (Joint Photographic Experts Group)

    • Image compression algorithm that reduces size by eliminating details that are less perceptible to the human eye.
    • Uses discrete cosine transforms (DCT) and quantization.

  2. MP3 (MPEG-1 Audio Layer III)

    • Audio compression algorithm that eliminates inaudible sounds to the human ear.
    • Uses Fourier transforms and perceptual quantization.

Algorithm Comparison

| Algorithm | Type | Common Use | Advantages | Disadvantages | |-----------------|---------------|------------------|-------------------------------------|------------------------------------| | Huffman Coding | Lossless | Texts, files | Simple and efficient | Does not always achieve high compression | | LZW | Lossless | Texts, images | Good compression for repetitive data | Can be inefficient for non-repetitive data | | JPEG | Lossy | Images | High compression, good visual quality | Irreversible quality loss | | MP3 | Lossy | Audio | High compression, good auditory quality | Irreversible quality loss |

Conclusion

Data compression is a powerful tool in massive data processing, allowing significant reduction in data size for storage and transmission. Understanding the different compression algorithms and their applications is crucial for selecting the appropriate technique based on the type of data and specific project requirements.
