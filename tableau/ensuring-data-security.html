Ensuring Data Security
Using Tableau Extensions ►

In this topic, we will cover the essential aspects of ensuring data security in Tableau. This includes understanding the importance of data security, implementing user permissions, and utilizing Tableau's built-in security features to protect sensitive information.

Importance of Data Security

Data security is crucial in any data visualization tool, including Tableau. Ensuring that sensitive information is protected from unauthorized access and breaches is essential for maintaining trust and compliance with regulations.

  • Confidentiality: Ensuring that data is accessible only to those who are authorized.
  • Integrity: Maintaining the accuracy and consistency of data over its lifecycle.
  • Availability: Ensuring that data is available to authorized users when needed.

User Permissions

Tableau provides robust user permission settings that allow administrators to control who can access and interact with data.

Types of Permissions

  1. Workbook Permissions: Control access to workbooks.
  2. Data Source Permissions: Control access to data sources.
  3. Project Permissions: Control access to projects, which can contain multiple workbooks and data sources.

Setting Permissions

Permissions can be set at different levels to ensure that only authorized users have access to specific data.

Example: Setting Workbook Permissions

  1. Navigate to the Workbook: Go to the workbook you want to set permissions for.
  2. Open Permissions: Click on the 'Permissions' tab.
  3. Add Users/Groups: Add the users or groups you want to set permissions for.
  4. Set Permissions: Choose the appropriate permissions (e.g., View, Edit, Delete).
| Permission Type | Description                  |
|-----------------|------------------------------|
| View            | Allows users to view the workbook. |
| Edit            | Allows users to edit the workbook. |
| Delete          | Allows users to delete the workbook. |

Data Source Security

Securing data sources is critical to ensure that sensitive data is not exposed.

Embedded Credentials

When publishing a data source, you can choose to embed credentials. This ensures that the data source can be accessed without requiring users to enter credentials each time.

Example: Embedding Credentials

  1. Publish Data Source: When publishing a data source, select the 'Embedded Credentials' option.
  2. Enter Credentials: Enter the necessary credentials for the data source.
  3. Save: Save the data source with the embedded credentials.

Row-Level Security

Row-level security allows you to restrict access to specific rows in a data source based on user roles.

Example: Implementing Row-Level Security

  1. Create a User Filter: Create a calculated field that filters data based on the user's role.
  2. Apply the Filter: Apply the user filter to the data source.
IF [User Role] = 'Manager' THEN [Data]
ELSEIF [User Role] = 'Employee' THEN [Data] END

Tableau Server Security Features

Tableau Server offers additional security features to protect your data.

SSL Encryption

SSL (Secure Sockets Layer) encryption ensures that data transmitted between Tableau Server and clients is secure.

Enabling SSL Encryption

  1. Obtain an SSL Certificate: Purchase or generate an SSL certificate.
  2. Configure Tableau Server: Configure Tableau Server to use the SSL certificate.
  3. Restart Tableau Server: Restart Tableau Server to apply the changes.

Data Encryption at Rest

Tableau Server can encrypt data at rest to protect it from unauthorized access.

Enabling Data Encryption at Rest

  1. Navigate to Settings: Go to the Tableau Server settings.
  2. Enable Encryption: Enable the data encryption at rest option.
  3. Save Settings: Save the settings to apply the changes.

Conclusion

Ensuring data security in Tableau involves a combination of user permissions, data source security, and leveraging Tableau Server's built-in security features. By understanding and implementing these security measures, you can protect sensitive information and maintain the integrity, confidentiality, and availability of your data.

In summary:

  • Understand the importance of data security.
  • Set appropriate user permissions for workbooks, data sources, and projects.
  • Use embedded credentials and row-level security to protect data sources.
  • Utilize Tableau Server's SSL encryption and data encryption at rest features.

By following these best practices, you can ensure that your data remains secure while using Tableau.
