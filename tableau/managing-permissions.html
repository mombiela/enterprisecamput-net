Managing Permissions
Scheduling Extracts ►

Managing permissions in Tableau is crucial for ensuring that the right users have the appropriate access to data and functionalities. This topic will guide you through the basics of permissions, how to set them up, and advanced techniques for managing permissions effectively.

Introduction to Permissions

Permissions in Tableau control what users can see and do with content such as workbooks, views, and data sources. Understanding how to manage permissions is essential for maintaining data security and ensuring that users have the access they need.

Key Concepts

  • Users and Groups: Individual users and groups of users can be assigned permissions.
  • Content: Permissions can be set on various content types, including projects, workbooks, views, and data sources.
  • Permission Rules: Define what actions users can perform, such as viewing, editing, or deleting content.
  • Inheriting Permissions: Content can inherit permissions from parent objects, like projects.

Setting Up Permissions

Creating Users and Groups

Before setting permissions, you need to create users and groups.

  1. Creating Users:

    • Navigate to the 'Users' tab in Tableau Server or Tableau Online.
    • Click 'Add User' and fill in the required details.

  2. Creating Groups:

    • Navigate to the 'Groups' tab.
    • Click 'New Group' and provide a name for the group.
    • Add users to the group by selecting them from the list.

Assigning Permissions to Content

Permissions can be assigned at different levels, from broad project-level permissions to specific view-level permissions.

  1. Project-Level Permissions:

    • Navigate to the 'Projects' tab.
    • Select a project and click 'Permissions'.
    • Add users or groups and set their permissions using the permission grid.

  2. Workbook-Level Permissions:

    • Navigate to the 'Workbooks' tab.
    • Select a workbook and click 'Permissions'.
    • Add users or groups and define their permissions.

  3. View-Level Permissions:

    • Navigate to the specific view within a workbook.
    • Click 'Permissions' and set the desired permissions for users or groups.

Permission Rules

Permission rules define what actions users can perform. The main actions include:

  • View: Allows users to see the content.
  • Interact: Allows users to interact with the content, such as filtering or drilling down.
  • Edit: Allows users to modify the content.
  • Delete: Allows users to delete the content.

Example: Setting Permissions for a Workbook

1. Navigate to the 'Workbooks' tab.
2. Select the workbook 'Sales Analysis'.
3. Click 'Permissions'.
4. Add the group 'Sales Team'.
5. Set the following permissions:
    - View: Allow
    - Interact: Allow
    - Edit: Deny
    - Delete: Deny
6. Save the changes.

Advanced Permission Management

Inheriting Permissions

Content can inherit permissions from parent objects, simplifying permission management.

  • Project-Level Inheritance: All workbooks and views within a project can inherit the project's permissions.
  • Overriding Inherited Permissions: Specific content can override inherited permissions if necessary.

Permission Hierarchy

Understanding the hierarchy of permissions helps in managing complex permission structures.

| Level | Description | |-------------|--------------------------------------------------| | Project | Broad permissions applied to all content within. | | Workbook | Specific permissions for individual workbooks. | | View | Granular permissions for specific views. |

Best Practices

  • Use Groups: Assign permissions to groups rather than individual users to simplify management.
  • Least Privilege Principle: Grant the minimum permissions necessary for users to perform their tasks.
  • Regular Audits: Periodically review permissions to ensure they are still appropriate.

Conclusion

Managing permissions in Tableau is a critical skill for ensuring data security and proper access control. By understanding the basics of users, groups, and permission rules, and by leveraging advanced techniques like inheritance and hierarchy, you can effectively manage permissions in Tableau. Always follow best practices to maintain a secure and efficient Tableau environment.
