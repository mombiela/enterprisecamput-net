Introduction to Assertions

Assertions are a fundamental part of testing in JUnit. They allow you to verify that your code behaves as expected. If an assertion fails, it means there is a discrepancy between the expected and actual outcomes, indicating a potential bug in the code.

Key Concepts of Assertions

  • Assertions: Statements that check if a condition is true. If the condition is false, the test fails.
  • Expected vs. Actual: Assertions typically compare an expected value with an actual value.
  • Common Assertions: JUnit provides a variety of assertions to handle different types of checks.

Common Assertions in JUnit

Here are some of the most commonly used assertions in JUnit:

| Assertion Method | Description | |-------------------------|------------------------------------------------------------| | assertEquals | Checks if two values are equal | | assertNotEquals | Checks if two values are not equal | | assertTrue | Checks if a condition is true | | assertFalse | Checks if a condition is false | | assertNull | Checks if an object is null | | assertNotNull | Checks if an object is not null | | assertArrayEquals | Checks if two arrays are equal | | assertSame | Checks if two references point to the same object | | assertNotSame | Checks if two references do not point to the same object | | assertThrows | Checks if a specific exception is thrown |

Basic Usage of Assertions

Example: Using `assertEquals`

import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    @Test
    public void testAddition() {
        Calculator calculator = new Calculator();
        int result = calculator.add(2, 3);
        assertEquals(5, result, "2 + 3 should equal 5");
    }
}

In this example, assertEquals checks if the result of calculator.add(2, 3) is equal to 5. If not, the test will fail with the message "2 + 3 should equal 5".

Example: Using `assertTrue` and `assertFalse`

import static org.junit.jupiter.api.Assertions.assertTrue;
import static org.junit.jupiter.api.Assertions.assertFalse;
import org.junit.jupiter.api.Test;

public class NumberTest {

    @Test
    public void testIsEven() {
        NumberUtils numberUtils = new NumberUtils();
        assertTrue(numberUtils.isEven(4), "4 should be even");
        assertFalse(numberUtils.isEven(5), "5 should not be even");
    }
}

Here, assertTrue checks if numberUtils.isEven(4) returns true, and assertFalse checks if numberUtils.isEven(5) returns false.

Advanced Usage of Assertions

Example: Using `assertThrows`

import static org.junit.jupiter.api.Assertions.assertThrows;
import org.junit.jupiter.api.Test;

public class ExceptionTest {

    @Test
    public void testException() {
        ExceptionThrower thrower = new ExceptionThrower();
        assertThrows(IllegalArgumentException.class, () -> {
            thrower.throwException();
        }, "Expected IllegalArgumentException to be thrown");
    }
}

In this example, assertThrows checks if thrower.throwException() throws an IllegalArgumentException. If the exception is not thrown, the test fails.

Example: Using `assertArrayEquals`

import static org.junit.jupiter.api.Assertions.assertArrayEquals;
import org.junit.jupiter.api.Test;

public class ArrayTest {

    @Test
    public void testArrayEquality() {
        int[] expected = {1, 2, 3};
        int[] actual = {1, 2, 3};
        assertArrayEquals(expected, actual, "Arrays should be equal");
    }
}

assertArrayEquals checks if the expected and actual arrays are equal. If they are not, the test fails.

Conclusion

Assertions are a powerful tool in JUnit for verifying that your code behaves as expected. By using assertions, you can catch bugs early in the development process. Understanding and effectively using assertions is crucial for writing robust and reliable tests.
