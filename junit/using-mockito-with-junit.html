Introduction

Mockito is a popular mocking framework for Java that is used in conjunction with JUnit to create unit tests. It allows developers to create mock objects and define their behavior, making it easier to isolate the code being tested. This section will cover the basics of using Mockito with JUnit, including setup, creating mocks, and verifying interactions.

Setting Up Mockito with JUnit

To use Mockito with JUnit, you need to add the Mockito library to your project. This can be done using a build tool like Maven or Gradle.

Maven Dependency

<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-core</artifactId>
    <version>4.0.0</version>
    <scope>test</scope>
</dependency>

Gradle Dependency

testImplementation 'org.mockito:mockito-core:4.0.0'

Creating Mocks

Mocks are objects that simulate the behavior of real objects. In Mockito, you can create mocks using the @Mock annotation or the Mockito.mock() method.

Using `@Mock` Annotation

import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.Test;
import org.mockito.Mock;
import org.mockito.MockitoAnnotations;

public class ExampleTest {

    @Mock
    private MyService myService;

    @BeforeEach
    public void setUp() {
        MockitoAnnotations.openMocks(this);
    }

    @Test
    public void testMyService() {
        // Define behavior and test
    }
}

Using `Mockito.mock()` Method

import org.junit.jupiter.api.Test;
import static org.mockito.Mockito.*;

public class ExampleTest {

    @Test
    public void testMyService() {
        MyService myService = mock(MyService.class);
        // Define behavior and test
    }
}

Defining Behavior

Once you have created a mock, you can define its behavior using the when and thenReturn methods.

Example

import static org.mockito.Mockito.*;
import static org.junit.jupiter.api.Assertions.*;

public class ExampleTest {

    @Test
    public void testMyService() {
        MyService myService = mock(MyService.class);
        when(myService.performAction()).thenReturn("Success");

        String result = myService.performAction();
        assertEquals("Success", result);
    }
}

Verifying Interactions

Mockito allows you to verify that certain methods were called on your mock objects. This is useful for ensuring that your code interacts with dependencies as expected.

Example

import static org.mockito.Mockito.*;

public class ExampleTest {

    @Test
    public void testMyService() {
        MyService myService = mock(MyService.class);
        myService.performAction();

        verify(myService).performAction();
    }
}

Advanced Features

Mockito provides several advanced features that can be useful in more complex testing scenarios.

Argument Matchers

Argument matchers allow you to specify flexible criteria for method arguments.

import static org.mockito.Mockito.*;
import static org.mockito.ArgumentMatchers.*;

public class ExampleTest {

    @Test
    public void testMyService() {
        MyService myService = mock(MyService.class);
        when(myService.performAction(anyString())).thenReturn("Success");

        String result = myService.performAction("Test");
        assertEquals("Success", result);
    }
}

Stubbing Void Methods

You can stub void methods to throw exceptions or perform other actions.

import static org.mockito.Mockito.*;

public class ExampleTest {

    @Test
    public void testMyService() {
        MyService myService = mock(MyService.class);
        doThrow(new RuntimeException()).when(myService).performVoidAction();

        assertThrows(RuntimeException.class, () -> {
            myService.performVoidAction();
        });
    }
}

Conclusion

Using Mockito with JUnit allows you to create robust unit tests by isolating the code under test and simulating the behavior of its dependencies. By understanding how to create mocks, define their behavior, and verify interactions, you can write more effective and reliable tests. As you become more comfortable with these basics, you can explore advanced features like argument matchers and stubbing void methods to handle more complex testing scenarios.
