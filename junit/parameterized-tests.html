Parameterized Tests
Creating Test Suites ►

Parameterized tests allow you to run the same test multiple times with different inputs. This is particularly useful for testing functions that should behave the same way for a variety of inputs.

Introduction to Parameterized Tests

  • Definition: Parameterized tests enable a developer to run the same test with different sets of parameters.
  • Purpose: They help in reducing code duplication and improving test coverage.

Setting Up Parameterized Tests in JUnit

To use parameterized tests in JUnit, you need to include the junit-jupiter-params dependency in your project. Here is an example for a Maven project:

<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter-params</artifactId>
    <version>5.8.1</version>
    <scope>test</scope>
</dependency>

Writing a Parameterized Test

Step 1: Import Necessary Annotations

You need to import the following annotations:

import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.ValueSource;

Step 2: Create a Parameterized Test Method

Here is a simple example that tests if a number is even:

import static org.junit.jupiter.api.Assertions.assertTrue;
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.ValueSource;

public class EvenNumberTest {

    @ParameterizedTest
    @ValueSource(ints = {2, 4, 6, 8, 10})
    void testIsEven(int number) {
        assertTrue(number % 2 == 0);
    }
}
  • Explanation:
    • @ParameterizedTest: Marks the method as a parameterized test.
    • @ValueSource(ints = {2, 4, 6, 8, 10}): Provides the parameters for the test method. In this case, it provides an array of integers.

Advanced Parameterized Tests

Using `@CsvSource`

You can use @CsvSource to pass multiple arguments to your test method:

import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.CsvSource;

public class StringUtilsTest {

    @ParameterizedTest
    @CsvSource({
        "hello, HELLO",
        "world, WORLD",
        "junit, JUNIT"
    })
    void testToUpperCase(String input, String expected) {
        assertEquals(expected, input.toUpperCase());
    }
}
  • Explanation:
    • @CsvSource: Allows you to pass multiple sets of arguments to the test method.
    • The test method testToUpperCase takes two parameters: input and expected.

Using `@MethodSource`

For more complex scenarios, you can use @MethodSource to provide parameters from a method:

import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.MethodSource;
import java.util.stream.Stream;

public class MathUtilsTest {

    @ParameterizedTest
    @MethodSource("provideNumbersForAddition")
    void testAddition(int a, int b, int expected) {
        assertEquals(expected, a + b);
    }

    static Stream<Arguments> provideNumbersForAddition() {
        return Stream.of(
            Arguments.of(1, 1, 2),
            Arguments.of(2, 3, 5),
            Arguments.of(3, 5, 8)
        );
    }
}
  • Explanation:
    • @MethodSource("provideNumbersForAddition"): Specifies the method that provides the parameters.
    • The method provideNumbersForAddition returns a Stream of Arguments.

Conclusion

Parameterized tests in JUnit are a powerful feature that can help you write more efficient and comprehensive tests. By using annotations like @ValueSource, @CsvSource, and @MethodSource, you can easily run the same test with different inputs, reducing code duplication and improving test coverage.

  • Key Takeaways:
    • Parameterized tests allow for running the same test with different inputs.
    • They help in reducing code duplication and improving test coverage.
    • JUnit provides several annotations (@ValueSource, @CsvSource, @MethodSource) to facilitate parameterized testing.

By mastering parameterized tests, you can significantly enhance the robustness and maintainability of your test suites.
