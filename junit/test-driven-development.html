Introduction to TDD

Test-Driven Development (TDD) is a software development process where you write tests before writing the actual code. This approach helps ensure that your code is reliable, maintainable, and bug-free.

  • Red-Green-Refactor Cycle:
    1. Red: Write a failing test.
    2. Green: Write the minimum code necessary to pass the test.
    3. Refactor: Improve the code while keeping the test passing.

Benefits of TDD

  • Improved Code Quality: Writing tests first forces you to think about the requirements and design before implementation.
  • Reduced Bugs: Tests catch bugs early in the development process.
  • Documentation: Tests serve as documentation for how the code is supposed to work.
  • Refactoring Confidence: With a suite of tests, you can refactor code with confidence that you won't break existing functionality.

Setting Up JUnit

Before diving into TDD, you need to set up JUnit in your project.

Maven Setup

Add the following dependency to your pom.xml file:

<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter-api</artifactId>
    <version>5.7.0</version>
    <scope>test</scope>
</dependency>

Gradle Setup

Add the following to your build.gradle file:

testImplementation 'org.junit.jupiter:junit-jupiter-api:5.7.0'
testRuntimeOnly 'org.junit.jupiter:junit-jupiter-engine:5.7.0'

Writing Your First Test

Let's start with a simple example to understand the TDD process.

Example: Calculator Addition

  1. Red: Write a failing test.
import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.api.Test;

public class CalculatorTest {
    @Test
    public void testAddition() {
        Calculator calculator = new Calculator();
        assertEquals(5, calculator.add(2, 3));
    }
}
  • This test will fail because the Calculator class and the add method do not exist yet.
  1. Green: Write the minimum code necessary to pass the test.
public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
}
  • Now, the test should pass because the add method returns the correct sum.
  1. Refactor: Improve the code while keeping the test passing.
  • In this simple example, there might not be much to refactor, but always look for opportunities to improve code readability and maintainability.

Advanced TDD Concepts

Mocking and Stubbing

When writing tests, you might need to isolate the unit of code being tested. This is where mocking and stubbing come in.

  • Mocking: Creating a fake version of an object to control its behavior.
  • Stubbing: Providing predefined responses to method calls.

Example: Using Mockito

Add Mockito dependency to your project:

<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-core</artifactId>
    <version>3.6.0</version>
    <scope>test</scope>
</dependency>

Example of mocking a service:

import static org.mockito.Mockito.*;
import org.junit.jupiter.api.Test;

public class UserServiceTest {
    @Test
    public void testGetUser() {
        UserRepository mockRepo = mock(UserRepository.class);
        when(mockRepo.findById(1)).thenReturn(new User(1, "John Doe"));

        UserService userService = new UserService(mockRepo);
        User user = userService.getUser(1);

        assertEquals("John Doe", user.getName());
    }
}

Parameterized Tests

JUnit allows you to run the same test with different parameters.

Example:

import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.ValueSource;
import static org.junit.jupiter.api.Assertions.assertTrue;

public class ParameterizedTestExample {
    @ParameterizedTest
    @ValueSource(strings = {"racecar", "radar", "level"})
    public void testIsPalindrome(String word) {
        assertTrue(isPalindrome(word));
    }

    public boolean isPalindrome(String word) {
        return new StringBuilder(word).reverse().toString().equals(word);
    }
}

Conclusion

Test-Driven Development (TDD) is a powerful methodology that can significantly improve the quality and maintainability of your code. By writing tests first, you ensure that your code meets the requirements and is free of bugs. Using JUnit, you can easily implement TDD in your Java projects, and with advanced techniques like mocking and parameterized tests, you can handle more complex scenarios. Remember the Red-Green-Refactor cycle and apply it consistently to reap the benefits of TDD.
