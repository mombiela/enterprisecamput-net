Introduction to File Handling

File handling in MUMPS (also known as M) involves reading from and writing to files. This is essential for data storage, retrieval, and manipulation. In this section, we'll cover the basics of file handling, including opening, reading, writing, and closing files.

Basic File Operations

Opening a File

To work with a file, you first need to open it. In MUMPS, you use the OPEN command to open a file.

OPEN "filename":("R"):1
IF '$TEST WRITE "Error opening file",! QUIT
  • filename: The name of the file you want to open.
  • ("R"): The mode in which to open the file. "R" stands for read mode.
  • 1: The device number. This is optional but recommended for clarity.

Closing a File

After you are done with a file, you should close it using the CLOSE command.

CLOSE "filename"

Reading from a File

To read from a file, you use the READ command. You must first open the file in read mode.

OPEN "filename":("R"):1
IF '$TEST WRITE "Error opening file",! QUIT

USE "filename"
READ line
WRITE "Read line: ", line,!

CLOSE "filename"
  • USE "filename": Directs subsequent I/O operations to the specified file.
  • READ line: Reads a line from the file into the variable line.

Writing to a File

To write to a file, you use the WRITE command. You must first open the file in write mode.

OPEN "filename":("W"):1
IF '$TEST WRITE "Error opening file",! QUIT

USE "filename"
WRITE "This is a line of text",!

CLOSE "filename"
  • ("W"): The mode in which to open the file. "W" stands for write mode.
  • WRITE "This is a line of text": Writes the specified text to the file.

Advanced File Operations

Appending to a File

To append data to an existing file, you open the file in append mode using ("A").

OPEN "filename":("A"):1
IF '$TEST WRITE "Error opening file",! QUIT

USE "filename"
WRITE "Appending this line",!

CLOSE "filename"
  • ("A"): The mode in which to open the file. "A" stands for append mode.

Error Handling

Proper error handling is crucial for robust file operations. Use the $TEST variable to check if file operations succeed.

OPEN "filename":("R"):1
IF '$TEST DO  QUIT
. WRITE "Error opening file",!
. ; Additional error handling code

USE "filename"
READ line
IF '$TEST DO  QUIT
. WRITE "Error reading file",!
. ; Additional error handling code

CLOSE "filename"

Working with Multiple Files

You can work with multiple files simultaneously by using different device numbers.

OPEN "file1":("R"):1
IF '$TEST WRITE "Error opening file1",! QUIT

OPEN "file2":("W"):2
IF '$TEST WRITE "Error opening file2",! QUIT

USE "file1"
READ line
USE "file2"
WRITE line,!

CLOSE "file1"
CLOSE "file2"

Summary

In this topic, we covered the essentials of file handling in MUMPS, including:

  • Opening and closing files.
  • Reading from and writing to files.
  • Appending data to files.
  • Handling errors during file operations.
  • Working with multiple files simultaneously.

Understanding these concepts is crucial for effective data management in MUMPS. Practice these operations with different files to become proficient in file handling.
