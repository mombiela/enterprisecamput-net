Setting Up the MUMPS Environment

Setting up the MUMPS (Massachusetts General Hospital Utility Multi-Programming System) environment is the first step to start programming in MUMPS. This section will guide you through the process of installing and configuring the MUMPS environment on your system.

Installing MUMPS

Prerequisites

Before installing MUMPS, ensure that your system meets the following requirements:

  • A compatible operating system (Linux, Windows, macOS).
  • Basic knowledge of command-line interface (CLI).

Installation Steps

On Linux

  1. Update your package list: 
    sudo apt-get update
  2. Install GT.M (a popular MUMPS implementation): 
    sudo apt-get install -y fis-gtm
  3. Verify the installation: 
    gtm
    You should see the GT.M prompt (GTM>).

On Windows

  1. Download the GT.M installer from the official website.
  2. Run the installer and follow the on-screen instructions.
  3. Set environment variables:
    • Add the GT.M binary directory to your PATH.
    • Set the gtm_dist environment variable to the GT.M installation directory.
  4. Verify the installation by opening a command prompt and typing: 
    gtm
    You should see the GT.M prompt (GTM>).

On macOS

  1. Install Homebrew if you haven't already: 
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  2. Install GT.M using Homebrew: 
    brew install fis-gtm
  3. Verify the installation: 
    gtm
    You should see the GT.M prompt (GTM>).

Configuring the MUMPS Environment

Setting Up Directories

Create a directory structure for your MUMPS projects. For example:

mkdir -p ~/mumps/projects
mkdir -p ~/mumps/data

Environment Variables

Set the necessary environment variables to ensure GT.M can find your projects and data files.

On Linux/macOS

Add the following lines to your .bashrc or .zshrc file:

export gtm_dist=/usr/lib/fis-gtm/V6.3-003
export gtmroutines="~/mumps/projects"
export gtmgbldir="~/mumps/data/global.gld"

Then, source the file to apply the changes:

source ~/.bashrc

On Windows

Set the environment variables through the System Properties:

  1. Open System Properties.
  2. Go to the Advanced tab and click on Environment Variables.
  3. Add new variables:
    • gtm_dist with the path to the GT.M installation directory.
    • gtmroutines with the path to your projects directory.
    • gtmgbldir with the path to your global directory.

Running Your First MUMPS Program

Creating a Simple Program

  1. Create a new file named hello.m in your projects directory: 
    nano ~/mumps/projects/hello.m
  2. Write a simple MUMPS program: 
    hello
write "Hello, MUMPS!",!
quit
  3. Save and exit the editor.

Executing the Program

  1. Start the GT.M environment: 
    gtm
  2. Run your program: 
    GTM>do ^hello
Hello, MUMPS!
GTM>

Conclusion

Setting up the MUMPS environment involves installing the GT.M implementation, configuring the necessary directories and environment variables, and running your first MUMPS program. With the environment set up, you are now ready to dive deeper into MUMPS programming. This foundational setup ensures that you have a stable and consistent environment to develop and test your MUMPS applications.
