Introduction

In this section, we will cover the basics of variables and data types in MUMPS (M). Understanding these fundamental concepts is crucial for writing effective and efficient MUMPS code. We will start with an introduction to variables, followed by an exploration of the different data types available in MUMPS.

Variables

Variables in MUMPS are used to store data that can be manipulated and retrieved throughout the execution of a program. Unlike many other programming languages, MUMPS does not require explicit declaration of variables before use.

Key Concepts

  • Implicit Declaration: Variables are created when they are first assigned a value.
  • Case Sensitivity: Variable names in MUMPS are case-sensitive.
  • Naming Conventions: Variable names must start with a letter and can be followed by letters, numbers, or the underscore character.

Examples

SET name="John Doe"
SET age=30
SET _score=95

In the above examples:

  • name is a variable storing a string value.
  • age is a variable storing an integer value.
  • _score is a variable storing an integer value.

Data Types

MUMPS supports several data types, but it primarily deals with strings and numbers. The language is designed to handle data dynamically, so the type of data a variable holds can change during execution.

Key Data Types

  • String: A sequence of characters.
  • Number: Can be an integer or a floating-point number.

String Data Type

Strings in MUMPS are enclosed in double quotes. They can contain any character, including numbers and special characters.

Examples

SET greeting="Hello, World!"
SET address="123 Main St."

Number Data Type

Numbers in MUMPS can be integers or floating-point numbers. They are not enclosed in quotes.

Examples

SET count=100
SET price=19.99

Dynamic Typing

MUMPS variables can change types dynamically based on the assigned value.

Example

SET var="123"  ; var is a string
SET var=123    ; var is now a number

Operations on Variables

MUMPS allows various operations on variables, including arithmetic operations, string concatenation, and logical operations.

Arithmetic Operations

You can perform standard arithmetic operations on numeric variables.

Examples

SET a=10
SET b=5
SET sum=a+b       ; sum is 15
SET difference=a-b ; difference is 5
SET product=a*b   ; product is 50
SET quotient=a/b  ; quotient is 2

String Concatenation

Strings can be concatenated using the underscore (_) operator.

Example

SET firstName="John"
SET lastName="Doe"
SET fullName=firstName_" "_lastName  ; fullName is "John Doe"

Logical Operations

Logical operations can be performed using standard logical operators.

Examples

SET x=1
SET y=0
SET andResult=x&y  ; andResult is 0 (false)
SET orResult=x!y   ; orResult is 1 (true)

Summary

In this section, we have covered the basics of variables and data types in MUMPS. We learned that variables in MUMPS are implicitly declared and can dynamically change types. We also explored the primary data types in MUMPS: strings and numbers, and saw how to perform various operations on these variables.

Understanding these fundamental concepts is essential for progressing to more advanced topics in MUMPS programming. Practice creating and manipulating variables to become more comfortable with these concepts.
