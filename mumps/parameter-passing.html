Parameter Passing in MUMPS (M)

Introduction

Parameter passing is a fundamental concept in programming that allows you to pass data to subroutines or functions. In MUMPS (M), understanding how to pass parameters effectively can help you write more modular and reusable code.

Key Concepts

  • Subroutines: Blocks of code that perform specific tasks and can be called from other parts of the program.
  • Parameters: Data passed to subroutines to be used within their execution.
  • By Value: Passing a copy of the data.
  • By Reference: Passing a reference to the actual data.

Passing Parameters to Subroutines

In MUMPS, you can pass parameters to subroutines using the DO command. Parameters can be passed by value or by reference.

Passing by Value

When you pass parameters by value, a copy of the data is passed to the subroutine. Any changes made to the parameters within the subroutine do not affect the original data.

Example

Main
    NEW X
    SET X=10
    DO Subroutine(X)
    WRITE "Value of X after subroutine call: ", X, !
    QUIT

Subroutine(Value)
    SET Value=20
    WRITE "Value inside subroutine: ", Value, !
    QUIT

Explanation:

  • Main sets X to 10 and calls Subroutine with X as a parameter.
  • Inside Subroutine, Value is set to 20, but this does not affect X in Main.
  • Output: 
    Value inside subroutine: 20
Value of X after subroutine call: 10

Passing by Reference

When you pass parameters by reference, a reference to the actual data is passed to the subroutine. Changes made to the parameters within the subroutine affect the original data.

Example

Main
    NEW X
    SET X=10
    DO Subroutine(.X)
    WRITE "Value of X after subroutine call: ", X, !
    QUIT

Subroutine(.Value)
    SET Value=20
    WRITE "Value inside subroutine: ", Value, !
    QUIT

Explanation:

  • Main sets X to 10 and calls Subroutine with X as a reference.
  • Inside Subroutine, Value is set to 20, which directly affects X in Main.
  • Output: 
    Value inside subroutine: 20
Value of X after subroutine call: 20

Advanced Parameter Passing

Advanced techniques involve passing multiple parameters and using arrays.

Passing Multiple Parameters

You can pass multiple parameters to a subroutine by separating them with commas.

Example

Main
    NEW A,B
    SET A=5, B=10
    DO Subroutine(A,B)
    WRITE "Values after subroutine call: A=", A, ", B=", B, !
    QUIT

Subroutine(Value1, Value2)
    SET Value1=Value1*2
    SET Value2=Value2+5
    WRITE "Values inside subroutine: Value1=", Value1, ", Value2=", Value2, !
    QUIT

Explanation:

  • Main sets A to 5 and B to 10 and calls Subroutine with both as parameters.
  • Inside Subroutine, Value1 and Value2 are modified.
  • Output: 
    Values inside subroutine: Value1=10, Value2=15
Values after subroutine call: A=5, B=10

Passing Arrays

Arrays can be passed by reference to subroutines, allowing you to manipulate complex data structures.

Example

Main
    NEW ARR
    SET ARR(1)=1, ARR(2)=2, ARR(3)=3
    DO Subroutine(.ARR)
    WRITE "Array after subroutine call: ", !
    ZWRITE ARR
    QUIT

Subroutine(.Array)
    SET Array(1)=Array(1)+10
    SET Array(2)=Array(2)+20
    SET Array(3)=Array(3)+30
    WRITE "Array inside subroutine: ", !
    ZWRITE Array
    QUIT

Explanation:

  • Main initializes an array ARR and calls Subroutine with ARR as a reference.
  • Inside Subroutine, the array elements are modified.
  • Output: 
    Array inside subroutine: 
ARR(1)=11
ARR(2)=22
ARR(3)=33
Array after subroutine call: 
ARR(1)=11
ARR(2)=22
ARR(3)=33

Conclusion

Understanding parameter passing in MUMPS is crucial for writing efficient and modular code. By mastering both value and reference passing, as well as handling multiple parameters and arrays, you can create more flexible and powerful subroutines. Practice these concepts with various examples to solidify your understanding and improve your MUMPS programming skills.
