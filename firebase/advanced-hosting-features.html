Introduction

Firebase Hosting is a powerful tool for deploying web applications. While basic hosting features are sufficient for many projects, advanced features can significantly enhance your application's performance, security, and user experience. This section will cover these advanced features, including custom domains, URL rewrites and redirects, custom headers, and deploying multiple sites.

Custom Domains

Custom domains allow you to use your own domain name instead of the default Firebase subdomain.

  • Setting Up a Custom Domain:

    1. Go to the Firebase Console.
    2. Select your project and navigate to the Hosting section.
    3. Click on "Add Custom Domain".
    4. Follow the instructions to verify domain ownership and configure DNS records.

  • Example:

    Type: CNAME
Name: www
Value: your-project-id.firebaseapp.com

URL Rewrites and Redirects

URL rewrites and redirects are essential for managing your application's URL structure and ensuring a smooth user experience.

  • Rewrites: Map a URL pattern to a specific resource, such as a Cloud Function or a static file.

  • Redirects: Automatically forward one URL to another, useful for maintaining SEO and user bookmarks.

  • Example Configuration:

    {
  "hosting": {
    "rewrites": [
      {
        "source": "/api/**",
        "function": "api"
      }
    ],
    "redirects": [
      {
        "source": "/old-page",
        "destination": "/new-page",
        "type": 301
      }
    ]
  }
}

Custom Headers

Custom headers can be used to enhance security, control caching, and manage content types.

  • Common Headers:

    • Content-Security-Policy: Prevents XSS attacks.
    • Cache-Control: Manages caching behavior.
    • Strict-Transport-Security: Enforces HTTPS.

  • Example Configuration:

    {
  "hosting": {
    "headers": [
      {
        "source": "**/*.@(js|css)",
        "headers": [
          {
            "key": "Cache-Control",
            "value": "max-age=31536000"
          }
        ]
      },
      {
        "source": "/**",
        "headers": [
          {
            "key": "Strict-Transport-Security",
            "value": "max-age=31536000; includeSubDomains"
          }
        ]
      }
    ]
  }
}

Deploying Multiple Sites

Firebase Hosting allows you to deploy multiple sites within a single Firebase project. This is useful for managing different environments (e.g., staging and production) or hosting multiple applications.

  • Steps to Deploy Multiple Sites:

    1. Define each site in your firebase.json file.
    2. Use the firebase use --add command to set up multiple site configurations.
    3. Deploy each site using the firebase deploy --only hosting:<site-name> command.

  • Example Configuration:

    {
  "hosting": [
    {
      "target": "site1",
      "public": "public-site1",
      "ignore": ["firebase.json", "**/.*", "**/node_modules/**"]
    },
    {
      "target": "site2",
      "public": "public-site2",
      "ignore": ["firebase.json", "**/.*", "**/node_modules/**"]
    }
  ]
}

Conclusion

Advanced hosting features in Firebase provide powerful tools to enhance your web applications. Custom domains, URL rewrites and redirects, custom headers, and the ability to deploy multiple sites offer flexibility and control over your hosting environment. By mastering these features, you can optimize your application's performance, security, and user experience.
