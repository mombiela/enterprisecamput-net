Introduction to Firebase Hosting

Firebase Hosting is a production-grade web content hosting platform for developers. It allows you to deploy web apps and static content quickly and securely.

Key Concepts

  • Static and Dynamic Content: Firebase Hosting supports both static files (HTML, CSS, JavaScript) and dynamic content through Cloud Functions.
  • SSL by Default: All content is served over HTTPS, ensuring security.
  • Global CDN: Firebase Hosting uses a global content delivery network (CDN) to ensure fast content delivery.

Getting Started with Firebase Hosting

  1. Install Firebase CLI: 
    npm install -g firebase-tools
  2. Initialize Your Project: 
    firebase init hosting
    • Follow the prompts to set up your project.
  3. Deploy Your Site: 
    firebase deploy

Example: Deploying a Simple HTML Page

  1. Create an index.html file in your project directory: 
    <!DOCTYPE html>
<html>
<head>
    <title>My Firebase Hosted Site</title>
</head>
<body>
    <h1>Hello, Firebase Hosting!</h1>
</body>
</html>
  2. Initialize Firebase Hosting: 
    firebase init hosting
  3. Deploy your site: 
    firebase deploy

Introduction to Firebase Cloud Functions

Firebase Cloud Functions allow you to run backend code in response to events triggered by Firebase features and HTTPS requests.

Key Concepts

  • Event-driven Functions: Functions that are triggered by Firebase events (e.g., database changes, authentication events).
  • HTTPS Functions: Functions that can be triggered via HTTP requests.
  • Scalability: Automatically scales up and down based on the number of incoming requests.

Setting Up Cloud Functions

  1. Initialize Cloud Functions: 
    firebase init functions
    • Follow the prompts to set up your project.
  2. Write Your First Function: 
    const functions = require('firebase-functions');

// Create and Deploy Your First Cloud Function
exports.helloWorld = functions.https.onRequest((request, response) => {
    response.send("Hello from Firebase!");
});
  3. Deploy Your Functions: 
    firebase deploy --only functions

Example: Creating an HTTPS Function

  1. Initialize Cloud Functions: 
    firebase init functions
  2. Write a simple HTTPS function in functions/index.js: 
    const functions = require('firebase-functions');

exports.helloWorld = functions.https.onRequest((request, response) => {
    response.send("Hello from Firebase!");
});
  3. Deploy your function: 
    firebase deploy --only functions

Advanced Topics in Hosting and Cloud Functions

Custom Domains

  • Setting Up Custom Domains: Firebase Hosting allows you to connect your custom domain to your Firebase project.
    • Go to the Firebase Console > Hosting > Connect Domain.
    • Follow the instructions to verify your domain and update DNS records.

Using Cloud Functions with Firestore

  • Triggering Functions on Firestore Events: 
    const functions = require('firebase-functions');
const admin = require('firebase-admin');
admin.initializeApp();

exports.createUserProfile = functions.firestore
    .document('users/{userId}')
    .onCreate((snap, context) => {
        const newValue = snap.data();
        // Perform operations with newValue
        return admin.firestore().collection('profiles').doc(context.params.userId).set(newValue);
    });

Combining Hosting and Cloud Functions

  • Serving Dynamic Content: Use Cloud Functions to serve dynamic content alongside static files. 
    const functions = require('firebase-functions');
const express = require('express');
const app = express();

app.get('/timestamp', (req, res) => {
    res.send(`Timestamp: ${Date.now()}`);
});

exports.app = functions.https.onRequest(app);

Conclusion

Firebase Hosting and Cloud Functions provide a powerful combination for deploying both static and dynamic web applications. With Firebase Hosting, you can quickly deploy secure, fast-loading static content, while Cloud Functions allow you to add backend logic that scales automatically. By mastering these tools, you can build robust, scalable web applications with ease.
