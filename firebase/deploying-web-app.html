Introduction to Firebase Hosting

Firebase Hosting is a production-grade web content hosting service for developers. It is designed to host your web app, static and dynamic content, and microservices.

Key Features

  • Fast and secure hosting: Global CDN, SSL by default, and powerful caching.
  • One-click deployment: Deploy your app with a single command.
  • Custom domains: Easily connect your custom domain.

Setting Up Firebase Hosting

Prerequisites

  • Node.js and npm installed on your machine.
  • Firebase CLI installed globally via npm.
  • A Firebase project created in the Firebase Console.

Installing Firebase CLI

To install the Firebase CLI, run the following command in your terminal:

npm install -g firebase-tools

Initializing Firebase in Your Project

Navigate to your project directory and run:

firebase init
  • Select "Hosting" from the list of Firebase features.
  • Choose an existing Firebase project or create a new one.
  • Specify the public directory (e.g., build or public).
  • Configure as a single-page app if applicable (e.g., for React or Angular apps).

Example: Initializing Firebase Hosting

cd your-project-directory
firebase init

Follow the prompts:

  • Select "Hosting".
  • Choose your Firebase project.
  • Set the public directory (e.g., build).
  • Configure as a single-page app if using a framework like React.

Deploying Your Web App

Building Your Web App

Before deploying, ensure your web app is built and ready for production. For example, if you are using React:

npm run build

Deploying to Firebase Hosting

Run the following command to deploy your app:

firebase deploy

This command will:

  • Upload your files to Firebase Hosting.
  • Provide a URL where your app is hosted.

Example: Deploying a React App

  1. Build your app: 
    npm run build
  2. Deploy to Firebase: 
    firebase deploy

Advanced Configuration

Custom Domains

You can add a custom domain to your Firebase project:

  1. Go to the Firebase Console.
  2. Navigate to Hosting.
  3. Click "Add Custom Domain" and follow the instructions.

Redirects and Rewrites

You can configure redirects and rewrites in your firebase.json file:

{
  "hosting": {
    "public": "build",
    "rewrites": [
      {
        "source": "**",
        "destination": "/index.html"
      }
    ],
    "redirects": [
      {
        "source": "/old-page",
        "destination": "/new-page",
        "type": 301
      }
    ]
  }
}

Summary

In this topic, you learned how to deploy a web app using Firebase Hosting. We covered:

  • Setting up Firebase Hosting.
  • Building and deploying your web app.
  • Advanced configurations like custom domains, redirects, and rewrites.

Firebase Hosting provides a fast, secure, and easy way to deploy your web applications, making it an excellent choice for developers of all levels.
