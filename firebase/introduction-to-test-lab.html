Introduction to Firebase Test Lab
Setting Up Test Lab ►

Firebase Test Lab is a cloud-based app-testing infrastructure that allows you to test your app on a wide range of devices and configurations. This ensures that your app works well across different environments, helping you catch issues before they reach your users.

Key Concepts

  • Device Matrix: A combination of different devices, OS versions, and locales on which you can test your app.
  • Instrumentation Tests: Tests that run on actual devices and can interact with the app's UI.
  • Robo Tests: Automated tests that explore the app's UI without needing any test scripts.
  • Game Loop Tests: Specifically designed for testing games, allowing you to simulate user interactions.
  • Test Results: Detailed reports including logs, screenshots, and videos of the test runs.

Setting Up Firebase Test Lab

Prerequisites

Before you start using Firebase Test Lab, ensure you have the following:

  • A Firebase project set up.
  • An Android or iOS app connected to your Firebase project.
  • Basic knowledge of writing tests for your app.

Adding Firebase Test Lab to Your Project

  1. Navigate to Firebase Console:

    • Go to the Firebase Console and select your project.
    • In the left-hand menu, click on "Test Lab".

  2. Upload Your App:

    • Click on "Run a test".
    • Upload your app's APK (for Android) or IPA (for iOS) file.

  3. Select Test Type:

    • Choose between Instrumentation Tests, Robo Tests, or Game Loop Tests.

  4. Configure Device Matrix:

    • Select the devices, OS versions, and locales you want to test on.

  5. Run the Test:

    • Click on "Start Test" to begin the testing process.

Writing Tests for Firebase Test Lab

Instrumentation Tests

Instrumentation tests are written using testing frameworks like Espresso for Android or XCTest for iOS. Below is an example of an Espresso test for an Android app:

@RunWith(AndroidJUnit4.class)
public class ExampleInstrumentedTest {
    @Rule
    public ActivityTestRule<MainActivity> activityRule =
        new ActivityTestRule<>(MainActivity.class);

    @Test
    public void useAppContext() {
        // Context of the app under test.
        Context appContext = InstrumentationRegistry.getInstrumentation().getTargetContext();
        assertEquals("com.example.myapp", appContext.getPackageName());
    }

    @Test
    public void checkButtonClick() {
        // Perform a click on the button with ID 'button'
        onView(withId(R.id.button)).perform(click());
        // Check if the text view with ID 'textView' displays the expected text
        onView(withId(R.id.textView)).check(matches(withText("Button Clicked")));
    }
}

Robo Tests

Robo tests do not require any test scripts. Firebase Test Lab will automatically explore your app's UI and perform actions like clicking buttons and entering text.

Game Loop Tests

Game Loop tests are designed for games and require you to define specific scenarios. Below is an example of a simple game loop test:

public class GameLoopTest {
    public static void main(String[] args) {
        // Initialize the game
        Game game = new Game();
        game.start();

        // Simulate user interactions
        game.tapScreen();
        game.swipeLeft();
        game.swipeRight();

        // End the game
        game.stop();
    }
}

Viewing Test Results

After your tests have run, you can view the results in the Firebase Console:

  • Logs: Detailed logs of the test execution.
  • Screenshots: Screenshots taken during the test.
  • Videos: Videos of the test run.
  • Performance Metrics: Metrics like CPU usage, memory usage, and network activity.

Conclusion

Firebase Test Lab is a powerful tool for ensuring your app works well across a variety of devices and configurations. By using Instrumentation Tests, Robo Tests, and Game Loop Tests, you can catch issues early and deliver a more robust app to your users. Make sure to regularly run tests and review the results to maintain high app quality.
