Passing Data Between Pages
Lazy Loading Modules ►

In this section, we will explore how to pass data between pages in an Ionic application. This is a crucial skill for creating dynamic and interactive mobile applications. We will cover various methods, including using Angular's Router, Ionic's NavController, and shared services.

Key Concepts

  • Routing and Navigation: Understanding how Ionic handles page navigation.
  • Passing Data via URL Parameters: Using Angular's Router to pass data.
  • Using Ionic's NavController: Leveraging Ionic's built-in navigation controller.
  • Shared Services: Using Angular services to share data between components.

Routing and Navigation

Ionic uses Angular's routing system to navigate between pages. Understanding how to configure routes and navigate between them is the first step in passing data between pages.

Configuring Routes

In Ionic, routes are defined in the app-routing.module.ts file. Here is an example of how to set up routes:

import { NgModule } from '@angular/core';
import { PreloadAllModules, RouterModule, Routes } from '@angular/router';

const routes: Routes = [
  {
    path: '',
    redirectTo: 'home',
    pathMatch: 'full'
  },
  {
    path: 'home',
    loadChildren: () => import('./home/home.module').then(m => m.HomePageModule)
  },
  {
    path: 'details/:id',
    loadChildren: () => import('./details/details.module').then(m => m.DetailsPageModule)
  }
];

@NgModule({
  imports: [
    RouterModule.forRoot(routes, { preloadingStrategy: PreloadAllModules })
  ],
  exports: [RouterModule]
})
export class AppRoutingModule { }

In this example, we have two routes: home and details/:id. The :id part is a route parameter that we can use to pass data to the details page.

Passing Data via URL Parameters

One common method to pass data between pages is by using URL parameters. This is done using Angular's Router.

Sending Data

To navigate to the details page and pass an id parameter, you can use the Router service:

import { Component } from '@angular/core';
import { Router } from '@angular/router';

@Component({
  selector: 'app-home',
  templateUrl: 'home.page.html',
  styleUrls: ['home.page.scss'],
})
export class HomePage {

  constructor(private router: Router) {}

  goToDetails(id: number) {
    this.router.navigate(['/details', id]);
  }
}

Receiving Data

On the details page, you can retrieve the id parameter using the ActivatedRoute service:

import { Component, OnInit } from '@angular/core';
import { ActivatedRoute } from '@angular/router';

@Component({
  selector: 'app-details',
  templateUrl: './details.page.html',
  styleUrls: ['./details.page.scss'],
})
export class DetailsPage implements OnInit {
  id: number;

  constructor(private route: ActivatedRoute) {}

  ngOnInit() {
    this.id = +this.route.snapshot.paramMap.get('id');
  }
}

Using Ionic's NavController

Ionic provides its own navigation controller, NavController, which can also be used to pass data between pages.

Sending Data

To navigate and pass data using NavController, you can use the navigateForward method with a state object:

import { Component } from '@angular/core';
import { NavController } from '@ionic/angular';

@Component({
  selector: 'app-home',
  templateUrl: 'home.page.html',
  styleUrls: ['home.page.scss'],
})
export class HomePage {

  constructor(private navCtrl: NavController) {}

  goToDetails(id: number) {
    this.navCtrl.navigateForward('/details', {
      state: {
        id: id
      }
    });
  }
}

Receiving Data

On the details page, you can retrieve the data from the navigation state:

import { Component, OnInit } from '@angular/core';
import { Router } from '@angular/router';

@Component({
  selector: 'app-details',
  templateUrl: './details.page.html',
  styleUrls: ['./details.page.scss'],
})
export class DetailsPage implements OnInit {
  id: number;

  constructor(private router: Router) {}

  ngOnInit() {
    if (this.router.getCurrentNavigation().extras.state) {
      this.id = this.router.getCurrentNavigation().extras.state.id;
    }
  }
}

Shared Services

Another method to pass data between pages is by using Angular services. This is particularly useful for sharing complex data or state.

Creating a Service

First, create a service to hold the shared data:

import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class DataService {
  private data: any;

  setData(data: any) {
    this.data = data;
  }

  getData() {
    return this.data;
  }
}

Using the Service

You can then use this service in your components to set and get data:

import { Component } from '@angular/core';
import { Router } from '@angular/router';
import { DataService } from '../services/data.service';

@Component({
  selector: 'app-home',
  templateUrl: 'home.page.html',
  styleUrls: ['home.page.scss'],
})
export class HomePage {

  constructor(private router: Router, private dataService: DataService) {}

  goToDetails(data: any) {
    this.dataService.setData(data);
    this.router.navigate(['/details']);
  }
}
import { Component, OnInit } from '@angular/core';
import { DataService } from '../services/data.service';

@Component({
  selector: 'app-details',
  templateUrl: './details.page.html',
  styleUrls: ['./details.page.scss'],
})
export class DetailsPage implements OnInit {
  data: any;

  constructor(private dataService: DataService) {}

  ngOnInit() {
    this.data = this.dataService.getData();
  }
}

Conclusion

Passing data between pages in an Ionic application can be achieved through various methods, each with its own use cases and advantages. Whether you choose to use URL parameters, Ionic's NavController, or shared services, understanding these techniques will help you create more dynamic and interactive applications. Experiment with each method to see which one best fits your application's needs.
