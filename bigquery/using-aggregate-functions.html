Introduction

Aggregate functions are essential tools in SQL and BigQuery for summarizing and analyzing data. They allow you to perform calculations on multiple rows of data and return a single value. This topic will guide you through the basics to advanced usage of aggregate functions in BigQuery.

Key Aggregate Functions

  • COUNT(): Counts the number of rows.
  • SUM(): Calculates the total sum of a numeric column.
  • AVG(): Computes the average value of a numeric column.
  • MIN(): Finds the minimum value in a column.
  • MAX(): Finds the maximum value in a column.

Basic Usage

COUNT()

The COUNT() function returns the number of rows that match a specified condition.

SELECT COUNT(*) AS total_rows
FROM `project.dataset.table`;

SUM()

The SUM() function adds up all the values in a numeric column.

SELECT SUM(sales) AS total_sales
FROM `project.dataset.table`;

AVG()

The AVG() function calculates the average value of a numeric column.

SELECT AVG(sales) AS average_sales
FROM `project.dataset.table`;

MIN() and MAX()

The MIN() and MAX() functions return the smallest and largest values in a column, respectively.

SELECT MIN(sales) AS minimum_sales, MAX(sales) AS maximum_sales
FROM `project.dataset.table`;

Grouping Data with Aggregate Functions

GROUP BY Clause

To apply aggregate functions to subsets of data, use the GROUP BY clause.

SELECT region, SUM(sales) AS total_sales
FROM `project.dataset.table`
GROUP BY region;

HAVING Clause

The HAVING clause filters groups based on aggregate function results.

SELECT region, SUM(sales) AS total_sales
FROM `project.dataset.table`
GROUP BY region
HAVING total_sales > 10000;

Advanced Usage

Combining Multiple Aggregate Functions

You can combine multiple aggregate functions in a single query.

SELECT region, 
       COUNT(*) AS total_orders, 
       SUM(sales) AS total_sales, 
       AVG(sales) AS average_sales
FROM `project.dataset.table`
GROUP BY region;

Window Functions

Window functions allow you to perform aggregate calculations across a set of table rows that are somehow related to the current row.

SELECT region, 
       sales, 
       SUM(sales) OVER (PARTITION BY region) AS total_sales_per_region
FROM `project.dataset.table`;

Handling NULL Values

Aggregate functions generally ignore NULL values. However, you can use COALESCE() to handle NULL values.

SELECT region, 
       SUM(COALESCE(sales, 0)) AS total_sales
FROM `project.dataset.table`
GROUP BY region;

Summary

Aggregate functions in BigQuery are powerful tools for summarizing and analyzing data. By mastering functions like COUNT(), SUM(), AVG(), MIN(), and MAX(), and understanding how to use them with GROUP BY and HAVING clauses, you can perform complex data analysis efficiently. Advanced techniques like window functions and handling NULL values further enhance your ability to derive insights from your data.
