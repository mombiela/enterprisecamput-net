Introduction

In this section, we will explore how to use functions in BigQuery to transform data. Data transformation is a crucial step in data analysis and preparation, allowing you to clean, manipulate, and prepare your data for further analysis or reporting.

Key Concepts

  • Functions: Predefined operations that can be applied to data to perform specific tasks.
  • Data Transformation: The process of converting data from one format or structure into another.
  • BigQuery SQL: The SQL dialect used in Google BigQuery for querying and manipulating data.

Common Functions in BigQuery

BigQuery provides a wide range of functions that can be used for data transformation. Here are some of the most commonly used ones:

String Functions

  • UPPER(): Converts a string to uppercase.
  • LOWER(): Converts a string to lowercase.
  • CONCAT(): Concatenates two or more strings.
  • SUBSTR(): Extracts a substring from a string.
SELECT 
  UPPER('hello world') AS upper_case,
  LOWER('HELLO WORLD') AS lower_case,
  CONCAT('Hello', ' ', 'World') AS concatenated_string,
  SUBSTR('Hello World', 1, 5) AS substring;

Numeric Functions

  • ROUND(): Rounds a number to a specified number of decimal places.
  • CEIL(): Rounds a number up to the nearest integer.
  • FLOOR(): Rounds a number down to the nearest integer.
  • ABS(): Returns the absolute value of a number.
SELECT 
  ROUND(3.14159, 2) AS rounded_value,
  CEIL(3.14159) AS ceiling_value,
  FLOOR(3.14159) AS floor_value,
  ABS(-42) AS absolute_value;

Date Functions

  • CURRENT_DATE(): Returns the current date.
  • DATE_ADD(): Adds a specified time interval to a date.
  • DATE_SUB(): Subtracts a specified time interval from a date.
  • FORMAT_DATE(): Formats a date according to a specified format.
SELECT 
  CURRENT_DATE() AS current_date,
  DATE_ADD(CURRENT_DATE(), INTERVAL 7 DAY) AS next_week,
  DATE_SUB(CURRENT_DATE(), INTERVAL 1 MONTH) AS last_month,
  FORMAT_DATE('%Y-%m-%d', CURRENT_DATE()) AS formatted_date;

Conditional Functions

  • IF(): Returns a value based on a condition.
  • CASE: Evaluates a list of conditions and returns a corresponding result.
SELECT 
  IF(1 > 0, 'True', 'False') AS if_example,
  CASE 
    WHEN 1 > 0 THEN 'Positive'
    WHEN 1 < 0 THEN 'Negative'
    ELSE 'Zero'
  END AS case_example;

Advanced Data Transformation

As you become more comfortable with basic functions, you can start combining them to perform more complex transformations.

Combining Functions

You can nest functions within each other to perform multiple transformations in a single query.

SELECT 
  UPPER(CONCAT('Hello', ' ', 'World')) AS upper_concatenated_string,
  ROUND(ABS(-3.14159), 2) AS rounded_absolute_value;

Using Window Functions

Window functions allow you to perform calculations across a set of table rows that are related to the current row.

SELECT 
  name,
  score,
  RANK() OVER (ORDER BY score DESC) AS rank
FROM 
  students;

Pivoting Data

Pivoting data is a common transformation that involves turning rows into columns.

SELECT 
  name,
  MAX(CASE WHEN subject = 'Math' THEN score END) AS math_score,
  MAX(CASE WHEN subject = 'Science' THEN score END) AS science_score
FROM 
  student_scores
GROUP BY 
  name;

Conclusion

Using functions for data transformation in BigQuery is a powerful way to manipulate and prepare your data for analysis. By mastering these functions, you can clean, format, and transform your data efficiently. Start with basic functions and gradually move to more complex transformations as you become more comfortable with BigQuery SQL.
