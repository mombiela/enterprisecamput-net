Introduction to Multiplayer in Unity

Overview

In this section, we will explore the fundamentals of creating multiplayer games in Unity. We will cover the basic concepts, tools, and techniques required to get started with multiplayer game development. By the end of this section, you will have a solid understanding of how to set up a simple multiplayer game in Unity.

Key Concepts

  • Networking: The process of connecting multiple devices to share data.
  • Client-Server Model: A common architecture where clients (players) connect to a central server.
  • Peer-to-Peer Model: An architecture where each player connects directly to other players.
  • Synchronization: Ensuring that all players see the same game state.
  • Latency: The delay between an action and its effect in the game, often due to network transmission times.

Tools and Frameworks

  • Unity Networking (UNet): Unity's built-in networking solution (deprecated but still used in legacy projects).
  • Mirror: A community-driven networking library for Unity, based on UNet.
  • Photon Unity Networking (PUN): A popular third-party networking solution for Unity.

Setting Up a Basic Multiplayer Game

Step 1: Setting Up the Project

  1. Create a New Unity Project:

    • Open Unity Hub and create a new 3D project.
    • Name your project "MultiplayerGame".

  2. Install Networking Package:

    • For this example, we will use Mirror.
    • Go to the Unity Asset Store and search for "Mirror".
    • Import the Mirror package into your project.

Step 2: Creating the Network Manager

  1. Add Network Manager:
    • Create an empty GameObject in your scene and name it "NetworkManager".
    • Add the NetworkManager and NetworkManagerHUD components to this GameObject.
using Mirror;

public class MyNetworkManager : NetworkManager
{
    public override void OnServerAddPlayer(NetworkConnection conn)
    {
        // Call the base functionality (adds the player)
        base.OnServerAddPlayer(conn);

        // Additional setup for the player can be done here
    }
}
  1. Configure Network Manager:
    • Set the Player Prefab to your player GameObject.
    • Ensure the NetworkManagerHUD component is enabled for a simple UI to start/stop the server and client.

Step 3: Creating the Player Prefab

  1. Create Player GameObject:

    • Create a new GameObject and name it "Player".
    • Add a NetworkIdentity component to the Player GameObject.
    • Add a NetworkTransform component to synchronize the player's position across the network.

  2. Create Player Script:

    • Create a new script named PlayerController and attach it to the Player GameObject.
using UnityEngine;
using Mirror;

public class PlayerController : NetworkBehaviour
{
    public float moveSpeed = 5f;

    void Update()
    {
        if (!isLocalPlayer) return;

        float moveX = Input.GetAxis("Horizontal") * moveSpeed * Time.deltaTime;
        float moveZ = Input.GetAxis("Vertical") * moveSpeed * Time.deltaTime;

        transform.Translate(new Vector3(moveX, 0, moveZ));
    }
}
  1. Convert Player to Prefab:
    • Drag the Player GameObject into the Project window to create a prefab.
    • Delete the Player GameObject from the scene.
    • Assign the Player prefab to the Player Prefab field in the NetworkManager.

Step 4: Testing the Multiplayer Game

  1. Run the Game:
    • Click the Play button in Unity.
    • Use the NetworkManagerHUD to start a Host (which acts as both server and client).
    • Open a second instance of the game and connect as a client.
    • Move both players and observe the synchronization.

Conclusion

In this section, we covered the basics of setting up a multiplayer game in Unity using the Mirror networking library. We discussed key networking concepts, set up a NetworkManager, created a player prefab, and tested the multiplayer functionality. This foundational knowledge will serve as a stepping stone for more advanced multiplayer features and optimizations in Unity.
