Procedure Division

The Procedure Division in COBOL is where the actual processing of data occurs. This is the section of the program where you write the instructions that tell the computer what to do with the data defined in the Data Division. Understanding the Procedure Division is crucial for writing effective COBOL programs.

Key Concepts

  • Paragraphs and Sections: The Procedure Division is organized into paragraphs and sections. A section is a collection of paragraphs, and a paragraph is a collection of statements.
  • Statements: These are the individual instructions that perform operations such as moving data, performing calculations, and controlling program flow.
  • Control Flow: COBOL provides various constructs for controlling the flow of the program, including IF statements, PERFORM loops, and GO TO statements.

Structure of the Procedure Division

The Procedure Division starts with the PROCEDURE DIVISION header and is followed by sections, paragraphs, and statements.

Basic Structure

PROCEDURE DIVISION.
    SECTION-1.
    PARAGRAPH-1.
        DISPLAY 'Hello, World!'.
        STOP RUN.

Sections and Paragraphs

  • Sections: Group related paragraphs together.
  • Paragraphs: Contain the executable statements.
PROCEDURE DIVISION.
    INITIALIZE-SECTION.
    INITIALIZE-PARAGRAPH.
        DISPLAY 'Initializing Program'.
    
    PROCESS-SECTION.
    PROCESS-PARAGRAPH.
        DISPLAY 'Processing Data'.
    
    TERMINATE-SECTION.
    TERMINATE-PARAGRAPH.
        DISPLAY 'Terminating Program'.
        STOP RUN.

Common Statements

DISPLAY Statement

The DISPLAY statement is used to output text to the console.

PROCEDURE DIVISION.
    DISPLAY 'Hello, World!'.
    STOP RUN.

MOVE Statement

The MOVE statement assigns values to variables.

DATA DIVISION.
WORKING-STORAGE SECTION.
01  WS-NAME PIC X(20).

PROCEDURE DIVISION.
    MOVE 'John Doe' TO WS-NAME.
    DISPLAY WS-NAME.
    STOP RUN.

IF Statement

The IF statement is used for conditional processing.

DATA DIVISION.
WORKING-STORAGE SECTION.
01  WS-AGE PIC 99.

PROCEDURE DIVISION.
    MOVE 25 TO WS-AGE.
    IF WS-AGE > 18
        DISPLAY 'Adult'
    ELSE
        DISPLAY 'Minor'.
    STOP RUN.

PERFORM Statement

The PERFORM statement is used to execute a paragraph or section.

PROCEDURE DIVISION.
    PERFORM INITIALIZE-PARAGRAPH.
    PERFORM PROCESS-PARAGRAPH.
    PERFORM TERMINATE-PARAGRAPH.
    STOP RUN.

INITIALIZE-PARAGRAPH.
    DISPLAY 'Initializing Program'.

PROCESS-PARAGRAPH.
    DISPLAY 'Processing Data'.

TERMINATE-PARAGRAPH.
    DISPLAY 'Terminating Program'.

GO TO Statement

The GO TO statement transfers control to another part of the program.

PROCEDURE DIVISION.
    DISPLAY 'Start'.
    GO TO END-PARAGRAPH.
    DISPLAY 'This will not be displayed'.
    
END-PARAGRAPH.
    DISPLAY 'End'.
    STOP RUN.

Control Flow Constructs

Nested IF Statements

DATA DIVISION.
WORKING-STORAGE SECTION.
01  WS-SCORE PIC 99.

PROCEDURE DIVISION.
    MOVE 85 TO WS-SCORE.
    IF WS-SCORE >= 90
        DISPLAY 'Grade: A'
    ELSE
        IF WS-SCORE >= 80
            DISPLAY 'Grade: B'
        ELSE
            DISPLAY 'Grade: C'.
    STOP RUN.

PERFORM UNTIL Loop

DATA DIVISION.
WORKING-STORAGE SECTION.
01  WS-COUNTER PIC 99 VALUE 1.

PROCEDURE DIVISION.
    PERFORM UNTIL WS-COUNTER > 5
        DISPLAY 'Counter: ' WS-COUNTER
        ADD 1 TO WS-COUNTER
    END-PERFORM.
    STOP RUN.

Conclusion

The Procedure Division is the heart of a COBOL program, where the actual logic and data processing occur. By understanding how to structure sections, paragraphs, and statements, and by mastering control flow constructs like IF statements and PERFORM loops, you can write efficient and effective COBOL programs. Practice writing different types of statements and control structures to become proficient in COBOL programming.
