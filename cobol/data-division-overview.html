Introduction

The Data Division in COBOL is one of the four main divisions and is crucial for defining the variables and data structures that your program will use. This division is responsible for describing the format, size, and type of data that will be processed. Understanding the Data Division is essential for both beginners and advanced COBOL programmers.

Sections of the Data Division

The Data Division is divided into several sections, each serving a specific purpose:

  1. File Section
  2. Working-Storage Section
  3. Local-Storage Section
  4. Linkage Section
  5. Report Section
  6. Screen Section

File Section

The File Section is used to define the structure of files that the program will read from or write to.

  • FD (File Description) Entry: Describes the file and its attributes.
  • 01 Level Entries: Define the structure of records within the file.

Example:

DATA DIVISION.
FILE SECTION.
FD  INPUT-FILE.
01  INPUT-RECORD.
    05  EMP-ID        PIC 9(5).
    05  EMP-NAME      PIC X(20).
    05  EMP-SALARY    PIC 9(7)V99.

Explanation:

  • FD INPUT-FILE declares a file named INPUT-FILE.
  • 01 INPUT-RECORD defines the structure of each record in the file.
  • 05 level entries define individual fields within the record.

Working-Storage Section

The Working-Storage Section is used to declare variables that will be used throughout the program.

  • 01 Level Entries: Define complex data structures.
  • 77 Level Entries: Define standalone variables.

Example:

DATA DIVISION.
WORKING-STORAGE SECTION.
01  WS-EMPLOYEE.
    05  WS-EMP-ID        PIC 9(5).
    05  WS-EMP-NAME      PIC X(20).
    05  WS-EMP-SALARY    PIC 9(7)V99.
77  WS-TOTAL-SALARY      PIC 9(9)V99 VALUE 0.

Explanation:

  • 01 WS-EMPLOYEE defines a group item with multiple fields.
  • 77 WS-TOTAL-SALARY defines a standalone variable initialized to 0.

Local-Storage Section

The Local-Storage Section is similar to the Working-Storage Section but is used for variables that are local to a specific program or subprogram.

Example:

DATA DIVISION.
LOCAL-STORAGE SECTION.
01  LS-COUNTER        PIC 9(4) VALUE 0.

Explanation:

  • 01 LS-COUNTER defines a local variable initialized to 0.

Linkage Section

The Linkage Section is used to define data items that are passed between programs or from a calling program to a subprogram.

Example:

DATA DIVISION.
LINKAGE SECTION.
01  LK-EMP-ID        PIC 9(5).
01  LK-EMP-NAME      PIC X(20).
01  LK-EMP-SALARY    PIC 9(7)V99.

Explanation:

  • 01 level entries define variables that will be linked to the calling program's data.

Report Section

The Report Section is used for defining report items and their layout. This section is less commonly used in modern COBOL programming.

Screen Section

The Screen Section is used for defining screen layouts for interactive programs. This section is also less commonly used today.

Summary

The Data Division in COBOL is a fundamental part of any COBOL program. It is divided into several sections, each with a specific purpose:

  • File Section: Defines file structures.
  • Working-Storage Section: Declares global variables.
  • Local-Storage Section: Declares local variables.
  • Linkage Section: Defines data passed between programs.
  • Report Section: Defines report layouts.
  • Screen Section: Defines screen layouts.

Understanding these sections and their purposes is crucial for writing effective COBOL programs. Each section allows you to manage and manipulate data in a structured and organized manner, making your programs more readable and maintainable.
