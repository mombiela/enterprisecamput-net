Introduction

COBOL (Common Business-Oriented Language) is one of the oldest high-level programming languages, primarily used in business, finance, and administrative systems for companies and governments. Understanding the structure of a COBOL program is fundamental for writing efficient and error-free code. This section will guide you through the basic structure of a COBOL program, including its divisions, sections, and paragraphs.

Basic Structure of a COBOL Program

A COBOL program is divided into four main divisions:

  1. Identification Division
  2. Environment Division
  3. Data Division
  4. Procedure Division

Each division serves a specific purpose and contains various sections and paragraphs.

Identification Division

The Identification Division is the first division in a COBOL program. It provides metadata about the program.

IDENTIFICATION DIVISION.
PROGRAM-ID. HelloWorld.
AUTHOR. John Doe.
  • PROGRAM-ID: Specifies the name of the program.
  • AUTHOR: Indicates the author of the program.

Environment Division

The Environment Division specifies the environment in which the program will run. It contains two sections: Configuration Section and Input-Output Section.

ENVIRONMENT DIVISION.
CONFIGURATION SECTION.
SOURCE-COMPUTER. IBM-370.
OBJECT-COMPUTER. IBM-370.

INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT INPUT-FILE ASSIGN TO 'input.txt'.
    SELECT OUTPUT-FILE ASSIGN TO 'output.txt'.
  • SOURCE-COMPUTER: Specifies the computer on which the source program is compiled.
  • OBJECT-COMPUTER: Specifies the computer on which the program will run.
  • FILE-CONTROL: Manages file assignments.

Data Division

The Data Division is used to define the variables and data structures. It contains several sections, but the most commonly used are the File Section and the Working-Storage Section.

DATA DIVISION.
FILE SECTION.
FD  INPUT-FILE.
01  INPUT-RECORD PIC X(100).

FD  OUTPUT-FILE.
01  OUTPUT-RECORD PIC X(100).

WORKING-STORAGE SECTION.
01  WS-NAME PIC X(20).
01  WS-AGE  PIC 99.
  • FILE SECTION: Defines the structure of files.
  • WORKING-STORAGE SECTION: Defines variables and constants used in the program.

Procedure Division

The Procedure Division contains the logic of the program. It consists of a series of statements that manipulate the data defined in the Data Division.

PROCEDURE DIVISION.
MAIN-PARA.
    OPEN INPUT INPUT-FILE.
    OPEN OUTPUT OUTPUT-FILE.
    PERFORM READ-INPUT UNTIL END-OF-FILE.
    CLOSE INPUT-FILE.
    CLOSE OUTPUT-FILE.
    STOP RUN.

READ-INPUT.
    READ INPUT-FILE INTO INPUT-RECORD
        AT END
            MOVE 'YES' TO END-OF-FILE
        NOT AT END
            MOVE INPUT-RECORD TO OUTPUT-RECORD
            WRITE OUTPUT-RECORD.
  • MAIN-PARA: The main paragraph where the execution starts.
  • READ-INPUT: A paragraph that reads records from the input file and writes them to the output file.

Conclusion

Understanding the structure of a COBOL program is essential for writing effective and maintainable code. The four main divisions—Identification, Environment, Data, and Procedure—each serve a specific purpose and must be included in every COBOL program. By mastering these divisions and their respective sections, you will be well-equipped to tackle more complex COBOL programming tasks.
