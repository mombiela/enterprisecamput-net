Indexed File Processing in COBOL
Relative File Processing ►

Indexed file processing is a crucial concept in COBOL, especially for applications that require efficient data retrieval and manipulation. This topic will guide you through the basics to advanced techniques of handling indexed files in COBOL.

Introduction to Indexed Files

Indexed files in COBOL allow for fast access to records using keys. Each record in an indexed file has one or more keys that can be used to quickly locate the record.

  • Primary Key: Unique identifier for each record.
  • Alternate Key: Additional keys that can be used to access records.

Defining Indexed Files

To define an indexed file in COBOL, you need to specify it in the FILE-CONTROL and FILE SECTION.

FILE-CONTROL

The FILE-CONTROL paragraph specifies the file organization and access mode.

SELECT EMPLOYEE-FILE ASSIGN TO 'employee.dat'
    ORGANIZATION IS INDEXED
    ACCESS MODE IS DYNAMIC
    RECORD KEY IS EMP-ID
    ALTERNATE RECORD KEY IS EMP-NAME WITH DUPLICATES.

FILE SECTION

The FILE SECTION describes the structure of the records in the file.

FD EMPLOYEE-FILE.
01 EMPLOYEE-RECORD.
   05 EMP-ID      PIC X(5).
   05 EMP-NAME    PIC X(20).
   05 EMP-ADDRESS PIC X(50).
   05 EMP-SALARY  PIC 9(7)V99.

Opening and Closing Indexed Files

Before processing an indexed file, it must be opened. Similarly, it should be closed after processing.

Opening a File

OPEN INPUT EMPLOYEE-FILE.

Closing a File

CLOSE EMPLOYEE-FILE.

Reading Indexed Files

Reading records from an indexed file can be done using the READ statement. You can read sequentially or randomly based on the key.

Sequential Read

READ EMPLOYEE-FILE NEXT
    AT END
        DISPLAY 'End of File Reached'
    NOT AT END
        DISPLAY EMPLOYEE-RECORD.

Random Read

MOVE '12345' TO EMP-ID.
READ EMPLOYEE-FILE
    INVALID KEY
        DISPLAY 'Record Not Found'
    NOT INVALID KEY
        DISPLAY EMPLOYEE-RECORD.

Writing to Indexed Files

To add new records to an indexed file, use the WRITE statement.

MOVE '67890' TO EMP-ID.
MOVE 'John Doe' TO EMP-NAME.
MOVE '123 Elm Street' TO EMP-ADDRESS.
MOVE 50000.00 TO EMP-SALARY.
WRITE EMPLOYEE-RECORD.

Updating Indexed Files

Updating records involves reading the record, modifying it, and then rewriting it.

MOVE '12345' TO EMP-ID.
READ EMPLOYEE-FILE
    INVALID KEY
        DISPLAY 'Record Not Found'
    NOT INVALID KEY
        MOVE 'Jane Doe' TO EMP-NAME
        REWRITE EMPLOYEE-RECORD.

Deleting Records

To delete a record, use the DELETE statement.

MOVE '12345' TO EMP-ID.
READ EMPLOYEE-FILE
    INVALID KEY
        DISPLAY 'Record Not Found'
    NOT INVALID KEY
        DELETE EMPLOYEE-RECORD.

Error Handling

Proper error handling is crucial for robust file processing.

  • INVALID KEY: Used when a key is not found.
  • AT END: Used when the end of the file is reached during sequential processing.
READ EMPLOYEE-FILE NEXT
    AT END
        DISPLAY 'End of File Reached'
    NOT AT END
        DISPLAY EMPLOYEE-RECORD.

Summary

Indexed file processing in COBOL allows for efficient data retrieval and manipulation using keys. Understanding how to define, open, read, write, update, and delete records in indexed files is essential for developing robust COBOL applications. Proper error handling ensures that your application can gracefully handle unexpected situations.

By mastering these concepts, you will be well-equipped to handle complex data processing tasks in COBOL.
