Relative File Processing in COBOL
Procedure Division ►

Introduction

Relative file processing in COBOL involves handling files where records are accessed based on their relative position within the file. This method is useful for applications that require quick access to records without sequentially reading through the entire file.

Key Concepts

  • Relative File: A file where records are identified by their relative record number (RRN).
  • RRN (Relative Record Number): A unique number that represents the position of a record within a relative file.
  • Access Modes: Methods to access records in a relative file, including sequential and random access.

Defining a Relative File

To work with relative files in COBOL, you need to define the file in the FILE SECTION of the DATA DIVISION and specify its organization in the ENVIRONMENT DIVISION.

Example

IDENTIFICATION DIVISION.
PROGRAM-ID. RelativeFileExample.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT REL-FILE ASSIGN TO 'relative.dat'
    ORGANIZATION IS RELATIVE
    ACCESS MODE IS RANDOM
    RELATIVE KEY IS REL-KEY.

DATA DIVISION.
FILE SECTION.
FD  REL-FILE.
01  REL-RECORD.
    05  REL-KEY        PIC 9(4).
    05  REL-DATA       PIC X(20).

WORKING-STORAGE SECTION.
01  WS-REL-KEY         PIC 9(4).
01  WS-REL-DATA        PIC X(20).

PROCEDURE DIVISION.

Opening and Closing Relative Files

Relative files must be opened before they can be read or written to, and closed after operations are complete.

Example

OPEN INPUT REL-FILE.
CLOSE REL-FILE.

Reading Records

To read a record from a relative file, you need to set the relative key and use the READ statement.

Example

MOVE 1 TO WS-REL-KEY.
READ REL-FILE
    INVALID KEY DISPLAY 'Record not found.'
    NOT INVALID KEY DISPLAY 'Record found: ' REL-DATA.

Writing Records

To write a record to a relative file, you need to set the relative key and data, then use the WRITE statement.

Example

MOVE 2 TO WS-REL-KEY.
MOVE 'Sample Data' TO WS-REL-DATA.
WRITE REL-RECORD INVALID KEY DISPLAY 'Error writing record.'

Updating Records

To update a record, read it first, modify the data, and then rewrite it.

Example

MOVE 1 TO WS-REL-KEY.
READ REL-FILE
    INVALID KEY DISPLAY 'Record not found.'
    NOT INVALID KEY
        MOVE 'Updated Data' TO REL-DATA
        REWRITE REL-RECORD INVALID KEY DISPLAY 'Error updating record.'

Deleting Records

To delete a record, use the DELETE statement after setting the relative key.

Example

MOVE 1 TO WS-REL-KEY.
DELETE REL-FILE INVALID KEY DISPLAY 'Error deleting record.'

Summary

Relative file processing in COBOL allows for efficient access to records based on their relative positions. Key operations include reading, writing, updating, and deleting records using the relative record number. Understanding how to define, open, close, and manipulate relative files is essential for effective COBOL programming.

By mastering these concepts, you can handle data more efficiently in applications that require quick and direct access to records.
