Introduction to GraphQL and WebSockets

GraphQL is a query language for APIs and a runtime for executing those queries by using a type system you define for your data. WebSockets provide a full-duplex communication channel over a single TCP connection, allowing for real-time data transfer between client and server.

Combining GraphQL with WebSockets enables real-time updates and subscriptions, making it possible to build highly interactive and dynamic applications.

Key Concepts

  • GraphQL: A query language for your API, and a server-side runtime for executing queries by using a type system you define for your data.
  • WebSockets: A protocol providing full-duplex communication channels over a single TCP connection.
  • Subscriptions: A GraphQL feature that allows clients to subscribe to specific events and receive real-time updates.

Setting Up GraphQL with WebSockets

To use GraphQL with WebSockets, you need to set up a GraphQL server that supports subscriptions. One popular choice is Apollo Server.

Installing Dependencies

First, install the necessary packages:

npm install apollo-server graphql subscriptions-transport-ws

Setting Up Apollo Server with WebSocket Support

Here's a basic setup for an Apollo Server with WebSocket support:

const { ApolloServer, gql } = require('apollo-server');
const { createServer } = require('http');
const { execute, subscribe } = require('graphql');
const { SubscriptionServer } = require('subscriptions-transport-ws');

// Define your schema
const typeDefs = gql`
  type Query {
    hello: String
  }

  type Subscription {
    time: String
  }
`;

// Define your resolvers
const resolvers = {
  Query: {
    hello: () => 'Hello world!',
  },
  Subscription: {
    time: {
      subscribe: (_, __, { pubsub }) => pubsub.asyncIterator('TIME'),
    },
  },
};

// Create an instance of ApolloServer
const server = new ApolloServer({
  typeDefs,
  resolvers,
  context: ({ req }) => ({
    pubsub,
  }),
});

// Create an HTTP server
const httpServer = createServer(server);

// Set up WebSocket for subscriptions
server.installSubscriptionHandlers(httpServer);

// Start the server
httpServer.listen(4000, () => {
  console.log(`Server is running on http://localhost:4000${server.graphqlPath}`);
  console.log(`Subscriptions ready at ws://localhost:4000${server.subscriptionsPath}`);
});

Publishing Events

To publish events to subscribers, you can use a PubSub instance from the graphql-subscriptions package:

const { PubSub } = require('graphql-subscriptions');
const pubsub = new PubSub();

// Publish an event every second
setInterval(() => {
  const time = new Date().toISOString();
  pubsub.publish('TIME', { time });
}, 1000);

Example: Real-Time Chat Application

Let's build a simple real-time chat application using GraphQL and WebSockets.

Schema Definition

Define the schema for messages and subscriptions:

const typeDefs = gql`
  type Message {
    id: ID!
    content: String!
    user: String!
  }

  type Query {
    messages: [Message!]
  }

  type Mutation {
    postMessage(user: String!, content: String!): ID!
  }

  type Subscription {
    messages: [Message!]
  }
`;

Resolvers

Implement the resolvers for queries, mutations, and subscriptions:

const messages = [];
const resolvers = {
  Query: {
    messages: () => messages,
  },
  Mutation: {
    postMessage: (parent, { user, content }) => {
      const id = messages.length;
      messages.push({ id, user, content });
      pubsub.publish('MESSAGES', { messages });
      return id;
    },
  },
  Subscription: {
    messages: {
      subscribe: () => pubsub.asyncIterator('MESSAGES'),
    },
  },
};

Client-Side Implementation

On the client side, you can use Apollo Client to connect to the server and handle subscriptions:

import { ApolloClient, InMemoryCache, gql } from '@apollo/client';
import { WebSocketLink } from '@apollo/client/link/ws';
import { split } from '@apollo/client';
import { getMainDefinition } from '@apollo/client/utilities';
import { HttpLink } from '@apollo/client';

// Create an HTTP link
const httpLink = new HttpLink({
  uri: 'http://localhost:4000/graphql',
});

// Create a WebSocket link
const wsLink = new WebSocketLink({
  uri: 'ws://localhost:4000/graphql',
  options: {
    reconnect: true,
  },
});

// Split links based on operation type
const link = split(
  ({ query }) => {
    const definition = getMainDefinition(query);
    return (
      definition.kind === 'OperationDefinition' &&
      definition.operation === 'subscription'
    );
  },
  wsLink,
  httpLink,
);

// Create an Apollo Client instance
const client = new ApolloClient({
  link,
  cache: new InMemoryCache(),
});

// Subscribe to messages
client
  .subscribe({
    query: gql`
      subscription {
        messages {
          id
          content
          user
        }
      }
    `,
  })
  .subscribe({
    next({ data }) {
      console.log(data);
    },
  });

Conclusion

GraphQL and WebSockets together provide a powerful toolset for building real-time applications. By leveraging GraphQL subscriptions, you can create dynamic and interactive user experiences that update in real-time. This combination is particularly useful for applications like chat apps, live sports updates, and collaborative tools.

In this section, you learned how to set up a GraphQL server with WebSocket support, implement subscriptions, and build a real-time chat application. With these skills, you can now explore more advanced use cases and optimize your real-time data handling.
