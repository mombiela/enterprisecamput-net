Introduction to GraphQL

  • What is GraphQL?

    • A query language for your API.
    • Provides a complete and understandable description of the data in your API.
    • Allows clients to request exactly what they need and nothing more.

  • Why use GraphQL?

    • Efficient data fetching.
    • Strongly typed schema.
    • Real-time data with subscriptions.
    • Better developer experience.

Setting Up the Environment

  • Prerequisites

    • Basic knowledge of JavaScript/TypeScript.
    • Familiarity with Node.js and Express.
    • Understanding of front-end frameworks like React.

  • Tools and Libraries

    • Node.js

Creating the Backend with GraphQL

  • Setting Up the Server

    mkdir graphql-fullstack
cd graphql-fullstack
npm init -y
npm install express apollo-server-express graphql

  • Creating the Express Server

    const express = require('express');
const { ApolloServer, gql } = require('apollo-server-express');

const app = express();

const typeDefs = gql`
  type Query {
    hello: String
  }
`;

const resolvers = {
  Query: {
    hello: () => 'Hello world!',
  },
};

const server = new ApolloServer({ typeDefs, resolvers });
server.applyMiddleware({ app });

app.listen({ port: 4000 }, () =>
  console.log(`Server ready at http://localhost:4000${server.graphqlPath}`)
);

  • Defining the Schema

    type Query {
  hello: String
  users: [User]
}

type User {
  id: ID!
  name: String!
  email: String!
}

  • Creating Resolvers

    const resolvers = {
  Query: {
    hello: () => 'Hello world!',
    users: () => [
      { id: 1, name: 'John Doe', email: 'john@example.com' },
      { id: 2, name: 'Jane Doe', email: 'jane@example.com' },
    ],
  },
};

Connecting to a Database

  • Using MongoDB with Mongoose

    npm install mongoose

  • Setting Up Mongoose Models

    const mongoose = require('mongoose');

mongoose.connect('mongodb://localhost:27017/graphql', {
  useNewUrlParser: true,
  useUnifiedTopology: true,
});

const UserSchema = new mongoose.Schema({
  name: String,
  email: String,
});

const User = mongoose.model('User', UserSchema);

  • Updating Resolvers to Use Database

    const resolvers = {
  Query: {
    hello: () => 'Hello world!',
    users: async () => await User.find(),
  },
};

Building the Frontend with React and Apollo Client

  • Setting Up React Application

    npx create-react-app client
cd client
npm install @apollo/client graphql

  • Creating Apollo Client

    import React from 'react';
import { ApolloClient, InMemoryCache, ApolloProvider } from '@apollo/client';
import App from './App';

const client = new ApolloClient({
  uri: 'http://localhost:4000/graphql',
  cache: new InMemoryCache(),
});

const Root = () => (
  <ApolloProvider client={client}>
    <App />
  </ApolloProvider>
);

export default Root;

  • Fetching Data with Apollo Client

    import React from 'react';
import { useQuery, gql } from '@apollo/client';

const GET_USERS = gql`
  query GetUsers {
    users {
      id
      name
      email
    }
  }
`;

const Users = () => {
  const { loading, error, data } = useQuery(GET_USERS);

  if (loading) return <p>Loading...</p>;
  if (error) return <p>Error :(</p>;

  return (
    <ul>
      {data.users.map(({ id, name, email }) => (
        <li key={id}>
          {name} - {email}
        </li>
      ))}
    </ul>
  );
};

export default Users;

Advanced Features

  • Mutations

    type Mutation {
  addUser(name: String!, email: String!): User
}

    const resolvers = {
  Mutation: {
    addUser: async (_, { name, email }) => {
      const user = new User({ name, email });
      await user.save();
      return user;
    },
  },
};

    import React, { useState } from 'react';
import { useMutation, gql } from '@apollo/client';

const ADD_USER = gql`
  mutation AddUser($name: String!, $email: String!) {
    addUser(name: $name, email: $email) {
      id
      name
      email
    }
  }
`;

const AddUser = () => {
  let input;
  const [addUser, { data }] = useMutation(ADD_USER);

  return (
    <div>
      <form
        onSubmit={e => {
          e.preventDefault();
          addUser({ variables: { name: input.value, email: input.value } });
          input.value = '';
        }}
      >
        <input
          ref={node => {
            input = node;
          }}
        />
        <button type="submit">Add User</button>
      </form>
    </div>
  );
};

export default AddUser;

  • Subscriptions

    type Subscription {
  userAdded: User
}

    const { PubSub } = require('apollo-server');
const pubsub = new PubSub();

const resolvers = {
  Subscription: {
    userAdded: {
      subscribe: () => pubsub.asyncIterator(['USER_ADDED']),
    },
  },
  Mutation: {
    addUser: async (_, { name, email }) => {
      const user = new User({ name, email });
      await user.save();
      pubsub.publish('USER_ADDED', { userAdded: user });
      return user;
    },
  },
};

    import React from 'react';
import { useSubscription, gql } from '@apollo/client';

const USER_ADDED = gql`
  subscription OnUserAdded {
    userAdded {
      id
      name
      email
    }
  }
`;

const UserAdded = () => {
  const { data, loading } = useSubscription(USER_ADDED);

  if (loading) return <p>Loading...</p>;

  return (
    <div>
      <p>New User Added:</p>
      <p>{data.userAdded.name} - {data.userAdded.email}</p>
    </div>
  );
};

export default UserAdded;

Conclusion

Building a full-stack application with GraphQL involves setting up a backend server with a well-defined schema and resolvers, connecting to a database, and creating a frontend application that interacts with the GraphQL API. By leveraging the power of GraphQL, developers can create efficient, scalable, and flexible applications that provide a great user experience. This course has covered the basics to advanced features, providing a solid foundation for building robust full-stack applications with GraphQL.
