Introduction to GraphQL Schema

A GraphQL schema is a central piece of any GraphQL server implementation. It defines the types of data that can be queried and the relationships between those types. Understanding the schema is crucial for both designing and querying a GraphQL API.

Key Concepts

  • Types: The building blocks of a GraphQL schema. Types define the shape of the data.
  • Queries: The read operations that clients can perform to fetch data.
  • Mutations: The write operations that clients can perform to modify data.
  • Resolvers: Functions that resolve the values for the fields in the schema.

Defining Types

In GraphQL, you define types using the type keyword. Each type has fields, and each field has a type.

type Book {
  id: ID!
  title: String!
  author: Author!
}

type Author {
  id: ID!
  name: String!
  books: [Book!]!
}

Explanation:

  • Book and Author are custom types.
  • ID! is a non-nullable unique identifier.
  • String! is a non-nullable string.
  • Author! indicates that the author field in Book cannot be null.
  • [Book!]! indicates a non-nullable list of non-nullable Book objects.

Queries

Queries are defined using the type Query keyword. They specify what data can be fetched and how.

type Query {
  books: [Book]
  book(id: ID!): Book
  authors: [Author]
  author(id: ID!): Author
}

Explanation:

  • books returns a list of Book objects.
  • book(id: ID!) returns a single Book object based on the provided id.
  • authors returns a list of Author objects.
  • author(id: ID!) returns a single Author object based on the provided id.

Mutations

Mutations are defined using the type Mutation keyword. They specify what data can be modified and how.

type Mutation {
  addBook(title: String!, authorId: ID!): Book
  updateBook(id: ID!, title: String, authorId: ID): Book
  deleteBook(id: ID!): Book
}

Explanation:

  • addBook(title: String!, authorId: ID!) adds a new Book and returns the added Book.
  • updateBook(id: ID!, title: String, authorId: ID) updates an existing Book and returns the updated Book.
  • deleteBook(id: ID!) deletes an existing Book and returns the deleted Book.

Resolvers

Resolvers are functions that provide the logic to fetch or modify the data for each field in the schema.

const resolvers = {
  Query: {
    books: () => getAllBooks(),
    book: (parent, args) => getBookById(args.id),
    authors: () => getAllAuthors(),
    author: (parent, args) => getAuthorById(args.id),
  },
  Mutation: {
    addBook: (parent, args) => addNewBook(args.title, args.authorId),
    updateBook: (parent, args) => updateExistingBook(args.id, args.title, args.authorId),
    deleteBook: (parent, args) => deleteExistingBook(args.id),
  },
  Book: {
    author: (book) => getAuthorById(book.authorId),
  },
  Author: {
    books: (author) => getBooksByAuthorId(author.id),
  },
};

Explanation:

  • Query resolvers handle fetching data.
  • Mutation resolvers handle modifying data.
  • Field-specific resolvers like Book.author and Author.books handle nested data fetching.

Summary

In this section, we covered the basics of GraphQL schemas, including:

  • Defining types to shape the data.
  • Creating queries to fetch data.
  • Creating mutations to modify data.
  • Implementing resolvers to provide the logic for fetching and modifying data.

Understanding these fundamentals is crucial for building and querying a GraphQL API effectively. As you progress, you'll learn more advanced concepts like interfaces, unions, and custom scalars to further enhance your GraphQL schemas.
