Input Types in GraphQL
Enums and Unions ►

Introduction

In GraphQL, input types are used to define the structure of the data that can be sent to the server. They are crucial for creating mutations and queries that require complex input. Understanding input types is essential for building robust and flexible GraphQL APIs.

Key Concepts

  • Input Types: Special types used to define the structure of input data.
  • Scalars: Basic data types like Int, Float, String, Boolean, and ID.
  • Custom Input Types: User-defined types that can be used to encapsulate complex input structures.
  • Non-Null and List Modifiers: Enhancements to input types to specify required fields and arrays.

Scalars

Scalars are the simplest form of input types. They represent primitive data types.

Example

type Query {
  getUser(id: ID!): User
}

type User {
  id: ID!
  name: String!
  age: Int
}

In this example, ID, String, and Int are scalar types.

Custom Input Types

Custom input types allow you to define more complex structures for your inputs. They are defined using the input keyword.

Example

input UserInput {
  name: String!
  age: Int
}

type Mutation {
  createUser(input: UserInput!): User
}

Here, UserInput is a custom input type that encapsulates the name and age fields.

Non-Null and List Modifiers

Non-null (!) and list ([]) modifiers can be used with input types to enforce required fields and arrays of inputs.

Example

input UserInput {
  name: String!
  age: Int
  hobbies: [String!]
}

type Mutation {
  createUser(input: UserInput!): User
}

In this example:

  • name is a required field.
  • age is optional.
  • hobbies is an optional list of required strings.

Using Input Types in Mutations

Input types are most commonly used in mutations to pass structured data to the server.

Example

input PostInput {
  title: String!
  content: String!
  tags: [String!]
}

type Mutation {
  createPost(input: PostInput!): Post
}

type Post {
  id: ID!
  title: String!
  content: String!
  tags: [String!]
}

Here, PostInput is used to encapsulate the data required to create a new post.

Comparison of Scalars and Custom Input Types

| Feature | Scalars | Custom Input Types | |------------------|------------------------|--------------------------| | Definition | Built-in primitive types | User-defined structures | | Complexity | Simple | Can be complex | | Use Cases | Basic fields | Complex inputs for mutations and queries |

Conclusion

Input types in GraphQL are essential for defining the structure of the data that can be sent to the server. Scalars provide basic data types, while custom input types allow for more complex structures. Understanding how to use non-null and list modifiers can help enforce data integrity and flexibility in your GraphQL API. By mastering input types, you can create more robust and flexible GraphQL schemas.
