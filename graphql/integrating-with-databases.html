Integrating with Databases
Microservices with GraphQL ►

Integrating GraphQL with databases is a crucial skill for building efficient and scalable applications. This topic will guide you through the process of connecting GraphQL to various types of databases, from setting up the connection to performing CRUD operations.

Introduction to Database Integration

  • Understanding the Role of Databases: Databases store and manage data for applications. Integrating GraphQL with databases allows you to query and manipulate this data efficiently.
  • Types of Databases:
    • SQL Databases: Relational databases like PostgreSQL, MySQL.
    • NoSQL Databases: Non-relational databases like MongoDB, Redis.

Setting Up the Environment

Prerequisites

  • Basic knowledge of GraphQL.
  • Familiarity with JavaScript (Node.js) or another backend language.
  • Installed database (e.g., PostgreSQL, MongoDB).

Installing Necessary Packages

For Node.js, you will typically use packages like graphql, express, apollo-server-express, and a database client (e.g., pg for PostgreSQL, mongoose for MongoDB).

npm install graphql express apollo-server-express pg mongoose

Connecting to a SQL Database

Setting Up PostgreSQL

  1. Install PostgreSQL: Follow the installation guide for your operating system.
  2. Create a Database: Use the PostgreSQL command line or a GUI tool like pgAdmin.
CREATE DATABASE mydatabase;

Connecting GraphQL to PostgreSQL

  1. Create a Connection: Use the pg package to connect to PostgreSQL.
const { Pool } = require('pg');
const pool = new Pool({
  user: 'yourusername',
  host: 'localhost',
  database: 'mydatabase',
  password: 'yourpassword',
  port: 5432,
});
  1. Define GraphQL Schema and Resolvers:
const { ApolloServer, gql } = require('apollo-server-express');
const express = require('express');

const typeDefs = gql`
  type Query {
    users: [User]
  }

  type User {
    id: ID!
    name: String!
    email: String!
  }
`;

const resolvers = {
  Query: {
    users: async () => {
      const res = await pool.query('SELECT * FROM users');
      return res.rows;
    },
  },
};

const server = new ApolloServer({ typeDefs, resolvers });
const app = express();
server.applyMiddleware({ app });

app.listen({ port: 4000 }, () =>
  console.log(`Server ready at http://localhost:4000${server.graphqlPath}`)
);

Connecting to a NoSQL Database

Setting Up MongoDB

  1. Install MongoDB: Follow the installation guide for your operating system.
  2. Create a Database: Use the MongoDB shell or a GUI tool like MongoDB Compass.

Connecting GraphQL to MongoDB

  1. Create a Connection: Use the mongoose package to connect to MongoDB.
const mongoose = require('mongoose');

mongoose.connect('mongodb://localhost:27017/mydatabase', {
  useNewUrlParser: true,
  useUnifiedTopology: true,
});
  1. Define GraphQL Schema and Resolvers:
const { ApolloServer, gql } = require('apollo-server-express');
const express = require('express');

const UserSchema = new mongoose.Schema({
  name: String,
  email: String,
});

const User = mongoose.model('User', UserSchema);

const typeDefs = gql`
  type Query {
    users: [User]
  }

  type User {
    id: ID!
    name: String!
    email: String!
  }
`;

const resolvers = {
  Query: {
    users: async () => {
      return await User.find();
    },
  },
};

const server = new ApolloServer({ typeDefs, resolvers });
const app = express();
server.applyMiddleware({ app });

app.listen({ port: 4000 }, () =>
  console.log(`Server ready at http://localhost:4000${server.graphqlPath}`)
);

Performing CRUD Operations

Create (Insert)

  • SQL Example:
const resolvers = {
  Mutation: {
    addUser: async (_, { name, email }) => {
      const res = await pool.query(
        'INSERT INTO users (name, email) VALUES ($1, $2) RETURNING *',
        [name, email]
      );
      return res.rows[0];
    },
  },
};
  • NoSQL Example:
const resolvers = {
  Mutation: {
    addUser: async (_, { name, email }) => {
      const user = new User({ name, email });
      await user.save();
      return user;
    },
  },
};

Read (Select)

  • SQL Example:
const resolvers = {
  Query: {
    users: async () => {
      const res = await pool.query('SELECT * FROM users');
      return res.rows;
    },
  },
};
  • NoSQL Example:
const resolvers = {
  Query: {
    users: async () => {
      return await User.find();
    },
  },
};

Update

  • SQL Example:
const resolvers = {
  Mutation: {
    updateUser: async (_, { id, name, email }) => {
      const res = await pool.query(
        'UPDATE users SET name = $1, email = $2 WHERE id = $3 RETURNING *',
        [name, email, id]
      );
      return res.rows[0];
    },
  },
};
  • NoSQL Example:
const resolvers = {
  Mutation: {
    updateUser: async (_, { id, name, email }) => {
      return await User.findByIdAndUpdate(id, { name, email }, { new: true });
    },
  },
};

Delete

  • SQL Example:
const resolvers = {
  Mutation: {
    deleteUser: async (_, { id }) => {
      const res = await pool.query('DELETE FROM users WHERE id = $1 RETURNING *', [id]);
      return res.rows[0];
    },
  },
};
  • NoSQL Example:
const resolvers = {
  Mutation: {
    deleteUser: async (_, { id }) => {
      return await User.findByIdAndDelete(id);
    },
  },
};

Conclusion

Integrating GraphQL with databases allows you to leverage the power of GraphQL queries and mutations to interact with your data efficiently. Whether you are using SQL or NoSQL databases, the principles remain the same: establish a connection, define your schema and resolvers, and perform CRUD operations. By mastering these skills, you can build robust and scalable applications that can handle complex data interactions.
