Introduction to GraphQL Variables

GraphQL variables allow you to dynamically pass arguments to your queries and mutations, making your GraphQL operations more flexible and reusable. Instead of hardcoding values directly into your queries, you can use variables to pass in different values at runtime.

Key Concepts

  • Variables: Placeholders in your GraphQL queries and mutations that can be replaced with actual values at runtime.
  • Variable Definitions: The part of the GraphQL query where you declare the variables you intend to use.
  • Variable Usage: The part of the GraphQL query where you use the declared variables.

Defining and Using Variables

To use variables in a GraphQL query or mutation, follow these steps:

  1. Declare the variables in the query or mutation.
  2. Pass the variables when executing the query or mutation.

Example: Query with Variables

Step 1: Declare the Variables

In the query, declare the variables with a $ prefix and specify their types.

query GetUser($id: ID!) {
  user(id: $id) {
    id
    name
    email
  }
}
  • $id: ID! declares a variable named id of type ID that is non-nullable (!).

Step 2: Pass the Variables

When executing the query, pass the variables in a separate variables object.

{
  "query": "query GetUser($id: ID!) { user(id: $id) { id name email } }",
  "variables": {
    "id": "123"
  }
}

Example: Mutation with Variables

Step 1: Declare the Variables

In the mutation, declare the variables with a $ prefix and specify their types.

mutation CreateUser($name: String!, $email: String!) {
  createUser(name: $name, email: $email) {
    id
    name
    email
  }
}
  • $name: String! and $email: String! declare variables named name and email of type String that are non-nullable.

Step 2: Pass the Variables

When executing the mutation, pass the variables in a separate variables object.

{
  "query": "mutation CreateUser($name: String!, $email: String!) { createUser(name: $name, email: $email) { id name email } }",
  "variables": {
    "name": "John Doe",
    "email": "john.doe@example.com"
  }
}

Benefits of Using Variables

  • Reusability: You can reuse the same query or mutation with different variable values.
  • Security: Helps prevent injection attacks by separating query structure from data.
  • Readability: Makes queries and mutations cleaner and easier to read.

Common Variable Types

  • Scalars: Basic data types like Int, Float, String, Boolean, and ID.
  • Enums: Custom scalar types that represent a set of named values.
  • Input Objects: Complex objects that can be passed as a single variable.

Example: Input Object

Step 1: Define the Input Object Type

In your GraphQL schema, define an input object type.

input UserInput {
  name: String!
  email: String!
}

Step 2: Use the Input Object in a Mutation

Declare the input object variable and use it in the mutation.

mutation CreateUser($input: UserInput!) {
  createUser(input: $input) {
    id
    name
    email
  }
}

Step 3: Pass the Input Object

When executing the mutation, pass the input object in the variables object.

{
  "query": "mutation CreateUser($input: UserInput!) { createUser(input: $input) { id name email } }",
  "variables": {
    "input": {
      "name": "Jane Doe",
      "email": "jane.doe@example.com"
    }
  }
}

Conclusion

GraphQL variables are a powerful feature that enhances the flexibility, security, and readability of your GraphQL operations. By understanding how to declare, use, and pass variables, you can create more dynamic and reusable queries and mutations. Practice using variables in different scenarios to become proficient in leveraging this feature in your GraphQL applications.
