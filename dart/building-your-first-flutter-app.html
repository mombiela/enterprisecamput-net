Introduction to Flutter

Flutter is an open-source UI software development toolkit created by Google. It is used to develop applications for Android, iOS, Linux, Mac, Windows, Google Fuchsia, and the web from a single codebase.

  • Why Flutter?
    • Single codebase for multiple platforms.
    • Rich set of pre-designed widgets.
    • High performance due to direct compilation to native code.
    • Strong community and extensive documentation.

Setting Up Your Environment

Before you start building your first Flutter app, you need to set up your development environment.

  • Install Flutter SDK

    • Download the Flutter SDK from the official website.
    • Extract the zip file and add the flutter/bin directory to your system's PATH.

  • Install an IDE

    • Recommended: Visual Studio Code or Android Studio.
    • Install the Flutter and Dart plugins for your chosen IDE.

  • Verify Installation

    • Run flutter doctor in your terminal to check for any missing dependencies.

Creating a New Flutter Project

Once your environment is set up, you can create a new Flutter project.

  • Using the Command Line

    flutter create my_first_app
cd my_first_app

  • Using an IDE

    • Open your IDE and select "Create New Flutter Project".
    • Follow the prompts to set up your project.

Understanding the Project Structure

A Flutter project has a specific structure that you should understand.

  • Key Directories and Files
    • lib/: Contains the main Dart code for your application.
    • pubspec.yaml: Manages the project's dependencies.
    • android/ and ios/: Platform-specific code for Android and iOS.
    • test/: Contains unit and widget tests.

Writing Your First Flutter App

Let's start by writing a simple Flutter app that displays "Hello, World!".

  • main.dart

    import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(
          title: Text('Hello, World!'),
        ),
        body: Center(
          child: Text('Hello, World!'),
        ),
      ),
    );
  }
}

  • Explanation

    • main(): The entry point of the application.
    • runApp(): Inflates the given widget and attaches it to the screen.
    • MyApp: A stateless widget that builds the app's UI.
    • MaterialApp: A convenience widget that wraps several widgets commonly required for material design applications.
    • Scaffold: Implements the basic material design visual layout structure.
    • AppBar: A material design app bar.
    • Center: Centers its child widget.
    • Text: A widget that displays a string of text.

Running Your App

You can run your Flutter app on an emulator or a physical device.

  • Using the Command Line

    flutter run

  • Using an IDE

    • Click on the "Run" button in your IDE.

Adding Interactivity

Let's add a button that changes the text when pressed.

  • Updated main.dart

    import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatefulWidget {
  @override
  _MyAppState createState() => _MyAppState();
}

class _MyAppState extends State<MyApp> {
  String _displayText = 'Hello, World!';

  void _updateText() {
    setState(() {
      _displayText = 'You pressed the button!';
    });
  }

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(
          title: Text('Interactive App'),
        ),
        body: Center(
          child: Column(
            mainAxisAlignment: MainAxisAlignment.center,
            children: <Widget>[
              Text(_displayText),
              SizedBox(height: 20),
              ElevatedButton(
                onPressed: _updateText,
                child: Text('Press me'),
              ),
            ],
          ),
        ),
      ),
    );
  }
}

  • Explanation

    • StatefulWidget: A widget that has mutable state.
    • _MyAppState: The state class for MyApp.
    • _displayText: A string that holds the text to be displayed.
    • _updateText(): A method that updates _displayText and calls setState() to rebuild the UI.
    • Column: A widget that arranges its children in a vertical column.
    • ElevatedButton: A material design elevated button.

Conclusion

In this topic, you learned how to set up your Flutter development environment, create a new Flutter project, understand the project structure, write a simple Flutter app, run the app, and add interactivity. This foundational knowledge will help you as you continue to explore and build more complex Flutter applications.
