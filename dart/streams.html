Streams in Dart
Generators ►

Streams are a fundamental part of asynchronous programming in Dart. They allow you to work with sequences of asynchronous events, such as data received from a web server or user input.

What are Streams?

  • Definition: A stream is a sequence of asynchronous events. Each event is either a data event or an error event.
  • Use Cases: Streams are useful for handling multiple asynchronous events over time, such as:
    • Receiving data from a web server.
    • Handling user input.
    • Listening to real-time updates.

Types of Streams

Dart provides two main types of streams:

  • Single-Subscription Streams: These streams can only be listened to once. They are suitable for events that occur only once or need to be processed in a specific order.
  • Broadcast Streams: These streams can be listened to multiple times. They are suitable for events that can be processed by multiple listeners independently.

Creating Streams

Single-Subscription Stream

You can create a single-subscription stream using the Stream class.

Stream<int> countStream(int max) async* {
  for (int i = 1; i <= max; i++) {
    yield i;
  }
}

void main() async {
  await for (var value in countStream(5)) {
    print(value); // Prints 1, 2, 3, 4, 5
  }
}

Broadcast Stream

You can create a broadcast stream using the StreamController class.

import 'dart:async';

void main() {
  final controller = StreamController<int>.broadcast();

  controller.stream.listen((data) {
    print('Listener 1: $data');
  });

  controller.stream.listen((data) {
    print('Listener 2: $data');
  });

  for (int i = 1; i <= 3; i++) {
    controller.add(i); // Adds data to the stream
  }

  controller.close();
}

Listening to Streams

  • await for: Use this syntax to listen to a stream asynchronously.
  • listen(): Use this method to handle stream events with callbacks.

Using `await for`

Stream<int> countStream(int max) async* {
  for (int i = 1; i <= max; i++) {
    yield i;
  }
}

void main() async {
  await for (var value in countStream(5)) {
    print(value); // Prints 1, 2, 3, 4, 5
  }
}

Using `listen()`

Stream<int> countStream(int max) async* {
  for (int i = 1; i <= max; i++) {
    yield i;
  }
}

void main() {
  countStream(5).listen((value) {
    print(value); // Prints 1, 2, 3, 4, 5
  });
}

Transforming Streams

Streams can be transformed using various methods such as map(), where(), expand(), and reduce().

Example: Using `map()`

Stream<int> countStream(int max) async* {
  for (int i = 1; i <= max; i++) {
    yield i;
  }
}

void main() {
  countStream(5).map((value) => value * 2).listen((value) {
    print(value); // Prints 2, 4, 6, 8, 10
  });
}

Handling Errors in Streams

Streams can emit error events. You can handle these errors using the onError parameter in the listen() method.

Stream<int> errorStream() async* {
  yield 1;
  throw Exception('An error occurred');
  yield 2;
}

void main() {
  errorStream().listen(
    (value) {
      print(value); // Prints 1
    },
    onError: (error) {
      print('Error: $error'); // Prints 'Error: Exception: An error occurred'
    },
  );
}

Summary

Streams in Dart are a powerful tool for handling asynchronous sequences of events. They come in two main types: single-subscription and broadcast streams. You can create streams using the Stream class or StreamController. Listening to streams can be done using await for or listen(). Streams can be transformed and errors can be handled gracefully. Understanding streams is essential for effective asynchronous programming in Dart.
