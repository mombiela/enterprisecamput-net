Networking in Flutter

Networking is a crucial aspect of modern mobile applications, enabling them to fetch data from the internet, communicate with servers, and interact with various web services. In this section, we will explore how to handle networking in Flutter using Dart.

Introduction to Networking in Flutter

  • HTTP Requests: Learn how to make HTTP requests to fetch data from the internet.
  • JSON Parsing: Understand how to parse JSON data received from web services.
  • Error Handling: Implement error handling for network requests.
  • Asynchronous Programming: Utilize Dart's async and await for handling asynchronous operations.

Making HTTP Requests

To make HTTP requests in Flutter, we use the http package. First, add the http package to your pubspec.yaml file:

dependencies:
  http: ^0.13.3

GET Request

A GET request is used to fetch data from a server. Here’s an example of making a GET request:

import 'package:http/http.dart' as http;
import 'dart:convert';

void fetchData() async {
  final response = await http.get(Uri.parse('https://jsonplaceholder.typicode.com/posts/1'));

  if (response.statusCode == 200) {
    var data = jsonDecode(response.body);
    print(data);
  } else {
    throw Exception('Failed to load data');
  }
}
  • Explanation:
    • http.get sends a GET request to the specified URL.
    • response.statusCode checks the status of the response.
    • jsonDecode parses the JSON response into a Dart object.

POST Request

A POST request is used to send data to a server. Here’s an example:

import 'package:http/http.dart' as http;
import 'dart:convert';

void sendData() async {
  final response = await http.post(
    Uri.parse('https://jsonplaceholder.typicode.com/posts'),
    headers: <String, String>{
      'Content-Type': 'application/json; charset=UTF-8',
    },
    body: jsonEncode(<String, String>{
      'title': 'foo',
      'body': 'bar',
      'userId': '1',
    }),
  );

  if (response.statusCode == 201) {
    var data = jsonDecode(response.body);
    print(data);
  } else {
    throw Exception('Failed to send data');
  }
}
  • Explanation:
    • http.post sends a POST request to the specified URL.
    • headers specify the content type.
    • jsonEncode converts the Dart object to a JSON string.

JSON Parsing

When working with APIs, data is often returned in JSON format. Here’s how to parse JSON data:

Example JSON Data

{
  "userId": 1,
  "id": 1,
  "title": "Sample Title",
  "body": "This is a sample body."
}

Parsing JSON in Dart

import 'dart:convert';

class Post {
  final int userId;
  final int id;
  final String title;
  final String body;

  Post({required this.userId, required this.id, required this.title, required this.body});

  factory Post.fromJson(Map<String, dynamic> json) {
    return Post(
      userId: json['userId'],
      id: json['id'],
      title: json['title'],
      body: json['body'],
    );
  }
}

void parseJson() {
  String jsonString = '{"userId": 1, "id": 1, "title": "Sample Title", "body": "This is a sample body."}';
  Map<String, dynamic> jsonMap = jsonDecode(jsonString);

  Post post = Post.fromJson(jsonMap);
  print('Title: ${post.title}');
}
  • Explanation:
    • Post class represents the JSON structure.
    • Post.fromJson is a factory constructor that creates a Post instance from a JSON map.
    • jsonDecode converts the JSON string into a Dart map.

Error Handling

Handling errors is essential for robust networking. Here’s how to handle errors in HTTP requests:

void fetchDataWithErrorHandling() async {
  try {
    final response = await http.get(Uri.parse('https://jsonplaceholder.typicode.com/posts/1'));

    if (response.statusCode == 200) {
      var data = jsonDecode(response.body);
      print(data);
    } else {
      print('Server error: ${response.statusCode}');
    }
  } catch (e) {
    print('Network error: $e');
  }
}
  • Explanation:
    • try-catch block is used to handle exceptions.
    • response.statusCode is checked to handle server errors.

Asynchronous Programming

Dart uses async and await to handle asynchronous operations. Here’s a brief overview:

  • async: Marks a function as asynchronous.
  • await: Waits for an asynchronous operation to complete.

Example

Future<void> fetchData() async {
  final response = await http.get(Uri.parse('https://jsonplaceholder.typicode.com/posts/1'));

  if (response.statusCode == 200) {
    var data = jsonDecode(response.body);
    print(data);
  } else {
    throw Exception('Failed to load data');
  }
}
  • Explanation:
    • Future<void> indicates that the function returns a Future.
    • await is used to wait for the HTTP request to complete.

Conclusion

In this section, we covered the basics of networking in Flutter using Dart. We learned how to make GET and POST requests, parse JSON data, handle errors, and use asynchronous programming. These skills are essential for building modern, data-driven mobile applications.

By mastering these concepts, you will be able to create Flutter applications that can interact with web services and provide dynamic content to users.
