Introduction to Interfaces

Interfaces in Dart are a way to define a contract that classes can implement. Unlike some other languages, Dart does not have a special syntax for interfaces. Instead, any class can be used as an interface.

Key Concepts

  • Interface Definition: Any class can act as an interface.
  • Implementation: Classes can implement multiple interfaces.
  • Abstract Methods: Methods in an interface that must be implemented by the implementing class.

Defining an Interface

In Dart, you define an interface by simply creating a class with method signatures (abstract methods) that you want other classes to implement.

class Animal {
  void makeSound();
}

In this example, Animal acts as an interface with a single method makeSound.

Implementing an Interface

A class implements an interface using the implements keyword. The implementing class must provide concrete implementations for all the methods defined in the interface.

class Dog implements Animal {
  @override
  void makeSound() {
    print('Bark');
  }
}

class Cat implements Animal {
  @override
  void makeSound() {
    print('Meow');
  }
}

Here, Dog and Cat classes implement the Animal interface and provide their own versions of the makeSound method.

Multiple Interfaces

A class in Dart can implement multiple interfaces. This allows for more flexible and reusable code.

class Flyable {
  void fly();
}

class Bird implements Animal, Flyable {
  @override
  void makeSound() {
    print('Chirp');
  }

  @override
  void fly() {
    print('Flap Flap');
  }
}

In this example, the Bird class implements both Animal and Flyable interfaces.

Abstract Classes vs Interfaces

Abstract classes can also be used to define interfaces, but they can contain method implementations as well. This is useful when you want to provide some default behavior.

Comparison Table

| Feature | Interface (Class) | Abstract Class | |------------------------|-------------------|----------------------| | Method Implementations | No | Yes | | Multiple Inheritance | Yes | No | | Instantiation | No | No |

abstract class Vehicle {
  void startEngine();

  void stopEngine() {
    print('Engine stopped');
  }
}

class Car implements Vehicle {
  @override
  void startEngine() {
    print('Car engine started');
  }
}

void main() {
  Car myCar = Car();
  myCar.startEngine(); // Car engine started
  myCar.stopEngine();  // Engine stopped
}

In this example, Vehicle is an abstract class with a concrete method stopEngine and an abstract method startEngine. The Car class implements the Vehicle interface and provides an implementation for startEngine.

Conclusion

Interfaces in Dart provide a powerful way to define contracts for your classes. By using interfaces, you can ensure that your classes adhere to specific behaviors and can be easily extended or modified. Remember:

  • Any class can act as an interface.
  • Use the implements keyword to implement an interface.
  • Classes can implement multiple interfaces.
  • Abstract classes can also be used to define interfaces with some default behavior.

Understanding interfaces will help you write more modular, reusable, and maintainable code.
