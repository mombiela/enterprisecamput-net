Introduction to Child Routes

Child routes in Angular allow you to create nested routes within a parent route. This is useful for building complex applications with multiple levels of navigation. Understanding child routes is essential for creating modular and maintainable Angular applications.

Key Concepts

  • Parent Route: The main route that contains one or more child routes.
  • Child Route: A route nested within a parent route.
  • Router Module: Angular module that provides navigation and URL manipulation capabilities.
  • Route Configuration: An array of route definitions that specify the paths and components.

Setting Up Child Routes

Step 1: Define Parent and Child Components

First, create the components that will serve as the parent and child in your routing structure.

// parent.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-parent',
  template: `
    <h2>Parent Component</h2>
    <router-outlet></router-outlet>
  `
})
export class ParentComponent {}

// child.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-child',
  template: `<h3>Child Component</h3>`
})
export class ChildComponent {}

Step 2: Configure Routes

Next, configure the routes in your AppRoutingModule.

// app-routing.module.ts
import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { ParentComponent } from './parent.component';
import { ChildComponent } from './child.component';

const routes: Routes = [
  {
    path: 'parent',
    component: ParentComponent,
    children: [
      {
        path: 'child',
        component: ChildComponent
      }
    ]
  }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule {}

Step 3: Update App Module

Ensure that your components and routing module are declared in your AppModule.

// app.module.ts
import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AppRoutingModule } from './app-routing.module';
import { AppComponent } from './app.component';
import { ParentComponent } from './parent.component';
import { ChildComponent } from './child.component';

@NgModule({
  declarations: [
    AppComponent,
    ParentComponent,
    ChildComponent
  ],
  imports: [
    BrowserModule,
    AppRoutingModule
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule {}

Step 4: Navigate to Child Routes

To navigate to the child route, you can use Angular's router link directive.

<!-- app.component.html -->
<nav>
  <a routerLink="/parent">Parent</a>
  <a routerLink="/parent/child">Child</a>
</nav>
<router-outlet></router-outlet>

Summary

Child routes in Angular allow you to create nested navigation structures, making your application more modular and easier to manage. By defining parent and child components, configuring routes, and using the router link directive, you can effectively implement child routes in your Angular application. Understanding and utilizing child routes is crucial for building scalable and maintainable Angular applications.
