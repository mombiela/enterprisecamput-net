Testing and Deployment

Introduction

In this section, we will cover the essential aspects of testing and deploying Angular applications. Testing ensures that your application works as expected, while deployment is the process of making your application available to users. This section is divided into two main parts: Testing and Deployment.

Testing

Why Testing is Important

  • Ensures Code Quality: Helps catch bugs and errors early.
  • Facilitates Refactoring: Makes it easier to change code without breaking functionality.
  • Improves Maintainability: Well-tested code is easier to maintain and extend.

Types of Tests

  • Unit Tests: Test individual components or services.
  • Integration Tests: Test the interaction between different parts of the application.
  • End-to-End (E2E) Tests: Test the entire application flow from start to finish.

Setting Up Testing Environment

Angular comes with built-in support for testing using Jasmine and Karma.

Installing Necessary Packages

npm install --save-dev jasmine-core karma karma-chrome-launcher karma-jasmine karma-jasmine-html-reporter

Writing Unit Tests

Unit tests focus on testing individual components or services.

Example: Testing a Component

import { ComponentFixture, TestBed } from '@angular/core/testing';
import { MyComponent } from './my-component.component';

describe('MyComponent', () => {
  let component: MyComponent;
  let fixture: ComponentFixture<MyComponent>;

  beforeEach(async () => {
    await TestBed.configureTestingModule({
      declarations: [ MyComponent ]
    })
    .compileComponents();
  });

  beforeEach(() => {
    fixture = TestBed.createComponent(MyComponent);
    component = fixture.componentInstance;
    fixture.detectChanges();
  });

  it('should create', () => {
    expect(component).toBeTruthy();
  });

  it('should have a default title', () => {
    expect(component.title).toBe('Default Title');
  });
});

Running Tests

To run your tests, use the following command:

ng test

Writing E2E Tests

E2E tests are written using Protractor.

Example: E2E Test

import { browser, by, element } from 'protractor';

describe('My App', () => {
  it('should display the welcome message', () => {
    browser.get('/');
    expect(element(by.css('h1')).getText()).toEqual('Welcome to My App!');
  });
});

Running E2E Tests

To run your E2E tests, use the following command:

ng e2e

Deployment

Preparing for Deployment

  • Build Optimization: Use Angular CLI to build an optimized version of your application.
  • Environment Configuration: Set up different environments (development, production).

Building the Application

To build your application for production, use the following command:

ng build --prod

Deployment Options

  • Static Hosting: Deploy your Angular app to static hosting services like GitHub Pages, Firebase Hosting, or Netlify.
  • Server Hosting: Deploy your app to a server using services like AWS, Azure, or Heroku.

Deploying to GitHub Pages

  1. Install Angular CLI GitHub Pages Deploy: 
    npm install -g angular-cli-ghpages
  2. Build the Project: 
    ng build --prod --base-href "https://<username>.github.io/<repository-name>/"
  3. Deploy: 
    ngh --dir=dist/<project-name>

Deploying to Firebase Hosting

  1. Install Firebase Tools: 
    npm install -g firebase-tools
  2. Login to Firebase: 
    firebase login
  3. Initialize Firebase in Your Project: 
    firebase init
  4. Build the Project: 
    ng build --prod
  5. Deploy: 
    firebase deploy

Conclusion

Testing and deployment are critical aspects of developing robust and reliable Angular applications. By writing comprehensive tests, you can ensure that your application behaves as expected and is maintainable. Deploying your application correctly ensures that it is accessible to your users. With the tools and techniques covered in this section, you are well-equipped to test and deploy your Angular applications effectively.
