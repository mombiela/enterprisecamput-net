Setting up a project is the first step in developing an Angular application. This section will guide you through the process of setting up an Angular project from scratch, ensuring you have all the necessary tools and configurations to start building your application.

Prerequisites

Before setting up an Angular project, ensure you have the following installed on your machine:

  • Node.js: Angular requires Node.js for its build and development tools.
  • npm (Node Package Manager): Comes with Node.js and is used to install Angular CLI and other dependencies.
  • Angular CLI: A command-line interface tool that helps in creating, managing, and building Angular applications.

Installing Node.js and npm

  1. Download and Install Node.js:

    • Visit the Node.js official website.
    • Download the LTS (Long Term Support) version for your operating system.
    • Follow the installation instructions.

  2. Verify Installation:

    • Open your terminal or command prompt.
    • Run the following commands to check the installed versions: 
      node -v
npm -v

Installing Angular CLI

  1. Install Angular CLI Globally:

    • Open your terminal or command prompt.
    • Run the following command to install Angular CLI globally: 
      npm install -g @angular/cli

  2. Verify Installation:

    • Run the following command to check the installed version of Angular CLI: 
      ng version

Creating a New Angular Project

With Angular CLI installed, you can now create a new Angular project.

  1. Create a New Project:

    • Navigate to the directory where you want to create your project.
    • Run the following command, replacing my-angular-app with your desired project name: 
      ng new my-angular-app

  2. Project Setup Prompts:

    • You will be prompted to choose some configuration options:
      • Would you like to add Angular routing?: Choose Yes or No depending on whether you need routing.
      • Which stylesheet format would you like to use?: Choose your preferred stylesheet format (CSS, SCSS, SASS, LESS, Stylus).

  3. Navigate to Project Directory:

    • After the project is created, navigate into the project directory: 
      cd my-angular-app

  4. Serve the Application:

    • Run the following command to start the development server: 
      ng serve
    • Open your browser and navigate to http://localhost:4200/ to see your new Angular application running.

Project Structure

Understanding the structure of an Angular project is crucial for effective development. Here is an overview of the key files and directories:

  • e2e/: End-to-end tests for the application.
  • node_modules/: Contains all the npm packages installed for the project.
  • src/: Main source folder containing the application code.
    • app/: Contains the main application modules and components.
    • assets/: Contains static assets like images and styles.
    • environments/: Contains environment-specific configuration files.
    • index.html: The main HTML file.
    • main.ts: The main entry point for the application.
    • styles.css: Global styles for the application.
  • angular.json: Configuration file for Angular CLI.
  • package.json: Lists the project's dependencies and scripts.
  • tsconfig.json: TypeScript configuration file.

Conclusion

Setting up an Angular project involves installing the necessary tools, creating a new project using Angular CLI, and understanding the project structure. With these steps, you are now ready to start developing your Angular application. In the next sections, we will dive deeper into building components, services, and other essential parts of an Angular application.
