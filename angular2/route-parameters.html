Introduction to Route Parameters

Route parameters are a way to pass information to a route in Angular. They are essential for creating dynamic and flexible applications where the same component can display different data based on the route parameters.

Setting Up Route Parameters

To use route parameters, you need to define them in your route configuration.

Defining Route Parameters

Here's how you define route parameters in your app-routing.module.ts:

import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { UserComponent } from './user/user.component';

const routes: Routes = [
  { path: 'user/:id', component: UserComponent }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

In this example:

  • path: 'user/:id' defines a route with a parameter id.

Accessing Route Parameters

Once you have defined route parameters, you can access them in your component.

Using ActivatedRoute

To access route parameters, you use the ActivatedRoute service.

import { Component, OnInit } from '@angular/core';
import { ActivatedRoute } from '@angular/router';

@Component({
  selector: 'app-user',
  template: `<h1>User ID: {{ userId }}</h1>`
})
export class UserComponent implements OnInit {
  userId: string;

  constructor(private route: ActivatedRoute) {}

  ngOnInit(): void {
    this.userId = this.route.snapshot.paramMap.get('id');
  }
}

In this example:

  • ActivatedRoute is injected into the component.
  • this.route.snapshot.paramMap.get('id') retrieves the id parameter from the route.

Using Route Parameters in Templates

You can use the retrieved route parameters directly in your templates.

<h1>User ID: {{ userId }}</h1>

This will display the user ID on the page.

Reacting to Parameter Changes

Sometimes, route parameters can change while the component is still active. To handle such cases, you need to subscribe to the parameter changes.

Subscribing to Parameter Changes

Here's how you can subscribe to route parameter changes:

import { Component, OnInit, OnDestroy } from '@angular/core';
import { ActivatedRoute } from '@angular/router';
import { Subscription } from 'rxjs';

@Component({
  selector: 'app-user',
  template: `<h1>User ID: {{ userId }}</h1>`
})
export class UserComponent implements OnInit, OnDestroy {
  userId: string;
  private routeSub: Subscription;

  constructor(private route: ActivatedRoute) {}

  ngOnInit(): void {
    this.routeSub = this.route.paramMap.subscribe(params => {
      this.userId = params.get('id');
    });
  }

  ngOnDestroy(): void {
    this.routeSub.unsubscribe();
  }
}

In this example:

  • this.route.paramMap.subscribe is used to react to parameter changes.
  • ngOnDestroy ensures that the subscription is cleaned up when the component is destroyed.

Optional Route Parameters

Angular also supports optional route parameters.

Defining Optional Route Parameters

Optional parameters are defined using the queryParams property.

import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { UserComponent } from './user/user.component';

const routes: Routes = [
  { path: 'user', component: UserComponent }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

Accessing Optional Route Parameters

You can access optional parameters using the queryParams property of ActivatedRoute.

import { Component, OnInit } from '@angular/core';
import { ActivatedRoute } from '@angular/router';

@Component({
  selector: 'app-user',
  template: `<h1>User ID: {{ userId }}</h1>`
})
export class UserComponent implements OnInit {
  userId: string;

  constructor(private route: ActivatedRoute) {}

  ngOnInit(): void {
    this.route.queryParams.subscribe(params => {
      this.userId = params['id'];
    });
  }
}

In this example:

  • this.route.queryParams.subscribe is used to react to optional parameter changes.

Summary

  • Route Parameters: Allow passing dynamic data to routes.
  • Defining Parameters: Use :param in the route path.
  • Accessing Parameters: Use ActivatedRoute service.
  • Reacting to Changes: Subscribe to paramMap or queryParams.
  • Optional Parameters: Accessed via queryParams.

Understanding and utilizing route parameters effectively can significantly enhance the flexibility and dynamism of your Angular applications.
