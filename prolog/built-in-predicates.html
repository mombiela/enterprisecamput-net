Introduction

Prolog, a logic programming language, comes with a rich set of built-in predicates that facilitate various operations such as arithmetic, list manipulation, input/output, and more. Understanding these built-in predicates is crucial for writing efficient and effective Prolog programs.

Key Concepts

What are Built-in Predicates?

  • Definition: Built-in predicates are predefined functions in Prolog that perform specific operations.
  • Purpose: They simplify common tasks and enhance the language's functionality without requiring users to define these operations from scratch.

Categories of Built-in Predicates

  • Arithmetic Predicates: Perform mathematical operations.
  • List Manipulation Predicates: Handle operations on lists.
  • Input/Output Predicates: Manage reading from and writing to streams.
  • Control Predicates: Control the flow of the program.
  • Comparison Predicates: Compare terms and values.

Arithmetic Predicates

Arithmetic predicates allow you to perform mathematical operations. Here are some common ones:

  • is/2: Evaluates an arithmetic expression.
  • =/2: Checks if two terms are equal.
  • >/2, >=/2, </2, =</2: Comparison operators.

Examples

% Example of `is/2`
X is 3 + 4. % X = 7

% Example of comparison
7 > 5. % true
3 =< 4. % true

List Manipulation Predicates

Prolog provides several predicates to work with lists:

  • member/2: Checks if an element is a member of a list.
  • append/3: Concatenates two lists.
  • length/2: Finds the length of a list.
  • reverse/2: Reverses a list.

Examples

% Example of `member/2`
member(3, [1, 2, 3, 4]). % true

% Example of `append/3`
append([1, 2], [3, 4], X). % X = [1, 2, 3, 4]

% Example of `length/2`
length([1, 2, 3, 4], L). % L = 4

% Example of `reverse/2`
reverse([1, 2, 3, 4], R). % R = [4, 3, 2, 1]

Input/Output Predicates

These predicates handle reading from and writing to streams:

  • write/1: Writes a term to the current output stream.
  • read/1: Reads a term from the current input stream.
  • nl/0: Writes a newline character.

Examples

% Example of `write/1` and `nl/0`
write('Hello, World!'), nl. % Hello, World!

% Example of `read/1`
% Assuming user inputs: hello.
read(X). % X = hello

Control Predicates

Control predicates manage the flow of the program:

  • fail/0: Always fails.
  • true/0: Always succeeds.
  • repeat/0: Always succeeds and can be used to create loops.
  • cut (!)/0: Prunes choice points to control backtracking.

Examples

% Example of `fail/0` and `true/0`
true. % true
fail. % false

% Example of `repeat/0`
repeat, write('Hello'), nl, fail. % Infinite loop printing 'Hello'

% Example of `cut (!)/0`
member(X, [1, 2, 3]), !, write(X). % Prints the first element and stops backtracking

Comparison Predicates

Comparison predicates are used to compare terms:

  • ==/2: Checks if two terms are identical.
  • \==/2: Checks if two terms are not identical.
  • @</2, @>/2, @=</2, @>=/2: Lexicographical comparison of terms.

Examples

% Example of `==/2` and `\==/2`
1 == 1. % true
1 \== 2. % true

% Example of lexicographical comparison
apple @< banana. % true
banana @> apple. % true

Conclusion

Built-in predicates in Prolog are powerful tools that simplify many common programming tasks. By mastering these predicates, you can write more efficient and readable Prolog code. This topic covered the essential built-in predicates across various categories, providing examples to illustrate their usage. Practice using these predicates in different scenarios to become proficient in Prolog programming.
