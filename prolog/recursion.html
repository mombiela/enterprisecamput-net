Introduction to Recursion

Recursion is a fundamental concept in Prolog and many other programming languages. It involves a function calling itself to solve smaller instances of the same problem. In Prolog, recursion is often used to traverse lists, trees, and other data structures.

Key Concepts

  • Base Case: The condition under which the recursive calls stop.
  • Recursive Case: The part of the function that includes the recursive call.

Basic Recursion

Let's start with a simple example: calculating the factorial of a number.

Factorial Example

The factorial of a number \( n \) is defined as:

  • \( 0! = 1 \) (Base Case)
  • \( n! = n \times (n-1)! \) (Recursive Case)

Here's how you can implement this in Prolog:

% Base Case
factorial(0, 1).

% Recursive Case
factorial(N, Result) :-
    N > 0,
    N1 is N - 1,
    factorial(N1, Result1),
    Result is N * Result1.

Explanation

  • The base case factorial(0, 1) states that the factorial of 0 is 1.
  • The recursive case factorial(N, Result) checks if \( N \) is greater than 0, then calculates \( N-1 \) and recursively calls factorial with \( N-1 \). Finally, it multiplies \( N \) by the result of the recursive call.

Recursion with Lists

Prolog excels at handling lists through recursion. Let's look at an example of summing all elements in a list.

Sum of List Elements

To sum the elements of a list, we define:

  • The sum of an empty list is 0 (Base Case).
  • The sum of a list is the head of the list plus the sum of the tail (Recursive Case).
% Base Case
sum_list([], 0).

% Recursive Case
sum_list([Head|Tail], Sum) :-
    sum_list(Tail, SumTail),
    Sum is Head + SumTail.

Explanation

  • The base case sum_list([], 0) states that the sum of an empty list is 0.
  • The recursive case sum_list([Head|Tail], Sum) splits the list into its head and tail, recursively sums the tail, and adds the head to this sum.

Advanced Recursion

Fibonacci Sequence

The Fibonacci sequence is a classic example of recursion, where each number is the sum of the two preceding ones.

% Base Cases
fibonacci(0, 0).
fibonacci(1, 1).

% Recursive Case
fibonacci(N, Result) :-
    N > 1,
    N1 is N - 1,
    N2 is N - 2,
    fibonacci(N1, Result1),
    fibonacci(N2, Result2),
    Result is Result1 + Result2.

Explanation

  • The base cases fibonacci(0, 0) and fibonacci(1, 1) define the first two numbers of the sequence.
  • The recursive case fibonacci(N, Result) calculates the \( N-1 \) and \( N-2 \) Fibonacci numbers and sums them.

Optimizing Recursion with Accumulators

Recursion can be optimized using accumulators to avoid redundant calculations. Let's optimize the Fibonacci example.

% Helper predicate with accumulator
fibonacci_acc(N, Result) :-
    fibonacci_acc(N, 0, 1, Result).

% Base Case
fibonacci_acc(0, A, _, A).

% Recursive Case
fibonacci_acc(N, A, B, Result) :-
    N > 0,
    N1 is N - 1,
    Sum is A + B,
    fibonacci_acc(N1, B, Sum, Result).

Explanation

  • The helper predicate fibonacci_acc/2 initializes the accumulators.
  • The base case fibonacci_acc(0, A, _, A) returns the accumulator when \( N \) is 0.
  • The recursive case fibonacci_acc(N, A, B, Result) updates the accumulators and calls itself with \( N-1 \).

Conclusion

Recursion is a powerful tool in Prolog for solving problems involving repetitive structures like lists and trees. By understanding the base and recursive cases, you can implement efficient recursive solutions. Optimizing recursion with techniques like accumulators can further enhance performance. Practice these concepts with various problems to master recursion in Prolog.
