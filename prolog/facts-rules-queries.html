Introduction to Prolog

Prolog is a logic programming language that is widely used for solving problems that involve objects and the relationships between them. The three fundamental components of Prolog are facts, rules, and queries. Understanding these components is essential for writing and interpreting Prolog programs.

Facts

Facts are the basic assertions about some world or domain. They state things that are unconditionally true.

Key Concepts

  • Syntax: Facts are written as simple predicates followed by a period.
  • Predicate: A predicate is a statement that may be true or false depending on the values of its variables.

Examples

% Facts about family relationships
parent(john, mary).
parent(mary, susan).
parent(mary, tom).

% Facts about colors
color(apple, red).
color(banana, yellow).
color(grape, purple).

In these examples:

  • parent(john, mary). states that John is a parent of Mary.
  • color(apple, red). states that the color of an apple is red.

Rules

Rules define relationships between facts and are used to infer new information from known facts. They are conditional statements.

Key Concepts

  • Syntax: Rules are written with a head and a body, separated by :-.
  • Head: The part before :-, representing the conclusion.
  • Body: The part after :-, representing the conditions.

Examples

% Rule to determine if someone is a grandparent
grandparent(X, Y) :- parent(X, Z), parent(Z, Y).

% Rule to determine if something is a fruit
is_fruit(X) :- color(X, _).

In these examples:

  • grandparent(X, Y) :- parent(X, Z), parent(Z, Y). states that X is a grandparent of Y if X is a parent of Z and Z is a parent of Y.
  • is_fruit(X) :- color(X, _). states that X is a fruit if X has any color.

Queries

Queries are questions asked to the Prolog system to retrieve information based on the facts and rules defined.

Key Concepts

  • Syntax: Queries are written as predicates followed by a question mark.
  • Response: Prolog responds with true, false, or variable bindings.

Examples

% Query to find out if John is a parent of Mary
?- parent(john, mary).

% Query to find all children of Mary
?- parent(mary, Child).

% Query to find all grandparents
?- grandparent(Grandparent, Grandchild).

In these examples:

  • ?- parent(john, mary). asks if John is a parent of Mary. Prolog will respond with true.
  • ?- parent(mary, Child). asks for all children of Mary. Prolog will respond with Child = susan and Child = tom.
  • ?- grandparent(Grandparent, Grandchild). asks for all grandparent-grandchild pairs. Prolog will provide the relevant bindings.

Conclusion

Understanding facts, rules, and queries is fundamental to programming in Prolog. Facts represent basic truths, rules define relationships and infer new information, and queries retrieve information based on the defined facts and rules. Mastering these components will enable you to write and understand Prolog programs effectively.
