Arithmetic Operations in Prolog
Control Structures ►

Introduction

Arithmetic operations are fundamental in any programming language, including Prolog. This section will cover the basics of arithmetic operations in Prolog, including how to perform calculations, use arithmetic operators, and handle arithmetic expressions.

Basic Arithmetic Operators

Prolog supports a variety of arithmetic operators. Here are the most commonly used ones:

  • + : Addition
  • - : Subtraction
  • * : Multiplication
  • / : Division (floating-point)
  • // : Integer division
  • mod : Modulus (remainder)
  • ^ : Exponentiation

Examples

Let's look at some basic examples of arithmetic operations in Prolog:

% Addition
Result is 3 + 2. % Result = 5

% Subtraction
Result is 5 - 3. % Result = 2

% Multiplication
Result is 4 * 2. % Result = 8

% Division (floating-point)
Result is 7 / 2. % Result = 3.5

% Integer Division
Result is 7 // 2. % Result = 3

% Modulus
Result is 7 mod 2. % Result = 1

% Exponentiation
Result is 2 ^ 3. % Result = 8

Using Arithmetic Expressions

In Prolog, arithmetic expressions are evaluated using the is operator. The is operator is used to assign the result of an arithmetic expression to a variable.

Example

% Assigning the result of an arithmetic expression to a variable
X is 5 + 3. % X = 8

Important Note

The is operator requires that the right-hand side of the expression is fully instantiated (i.e., all variables must have values) before evaluation.

% This will work
A = 5, B = 3, X is A + B. % X = 8

% This will not work and will result in an error
X is A + B, A = 5, B = 3. % Error: Arguments are not sufficiently instantiated

Comparison Operators

Prolog also provides comparison operators for arithmetic expressions:

  • =:= : Equal to
  • =\= : Not equal to
  • < : Less than
  • > : Greater than
  • =< : Less than or equal to
  • >= : Greater than or equal to

Examples

% Equal to
5 =:= 5. % true

% Not equal to
5 =\= 3. % true

% Less than
3 < 5. % true

% Greater than
5 > 3. % true

% Less than or equal to
3 =< 5. % true

% Greater than or equal to
5 >= 3. % true

Practical Exercises

Here are some exercises to practice arithmetic operations in Prolog:

  1. Write a Prolog predicate sum/3 that takes two numbers and returns their sum.

    sum(A, B, Sum) :-
    Sum is A + B.

  2. Write a Prolog predicate factorial/2 that calculates the factorial of a given number.

    factorial(0, 1).
factorial(N, Result) :-
    N > 0,
    N1 is N - 1,
    factorial(N1, R1),
    Result is N * R1.

  3. Write a Prolog predicate max/3 that takes three numbers and returns the maximum of the three.

    max(A, B, C, Max) :-
    Max1 is max(A, B),
    Max is max(Max1, C).

Conclusion

Arithmetic operations in Prolog are straightforward once you understand the basic operators and how to use the is operator for evaluating expressions. Remember to ensure that all variables in an arithmetic expression are instantiated before evaluation. Practice with the provided exercises to solidify your understanding of arithmetic operations in Prolog.
