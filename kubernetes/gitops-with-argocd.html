Introduction to GitOps

  • Definition: GitOps is a set of practices to manage infrastructure and application configurations using Git.
  • Core Principles:
    • Declarative Descriptions: All configurations are described declaratively.
    • Version Controlled: Configurations are stored in a version control system (e.g., Git).
    • Automated Application: Changes to configurations are automatically applied to the system.
    • Continuous Reconciliation: The system continuously ensures the actual state matches the desired state.

Introduction to ArgoCD

  • Definition: ArgoCD is a declarative, GitOps continuous delivery tool for Kubernetes.
  • Key Features:
    • Declarative GitOps: Uses Git repositories as the source of truth for the desired state of applications.
    • Automated Syncing: Automatically syncs the state of the cluster with the state defined in Git.
    • Self-Healing: Automatically corrects drift between the desired and actual state.
    • Multi-Cluster Support: Manages applications across multiple Kubernetes clusters.

Setting Up ArgoCD

  • Prerequisites:

    • A running Kubernetes cluster.
    • kubectl command-line tool configured to interact with your cluster.
    • A Git repository with Kubernetes manifests.

  • Installation Steps:

    1. Install ArgoCD in Kubernetes: 
      kubectl create namespace argocd
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
    2. Access the ArgoCD API Server: 
      kubectl port-forward svc/argocd-server -n argocd 8080:443
    3. Login to ArgoCD: 
      # Get the initial admin password
kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d
# Open the ArgoCD UI in your browser at https://localhost:8080

Configuring ArgoCD

  • Connecting a Git Repository:

    • Create an Application: 
      argocd app create my-app 
            --repo https://github.com/my-org/my-repo.git 
            --path path/to/manifests 
            --dest-server https://kubernetes.default.svc 
            --dest-namespace default
    • Sync the Application: 
      argocd app sync my-app

  • Application Configuration:

    • Example Application Manifest: 
      apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: guestbook
  namespace: argocd
spec:
  project: default
  source:
    repoURL: 'https://github.com/argoproj/argocd-example-apps'
    targetRevision: HEAD
    path: guestbook
  destination:
    server: 'https://kubernetes.default.svc'
    namespace: default
  syncPolicy:
    automated:
      prune: true
      selfHeal: true

Advanced ArgoCD Features

  • Automated Sync Policies:

    • Self-Healing: Automatically corrects any drift between the desired and actual state.
    • Pruning: Automatically removes resources that are no longer defined in the Git repository.

  • Multi-Cluster Management:

    • Add a Cluster: 
      argocd cluster add <context-name>

  • Custom Health Checks:

    • Define Custom Health Checks: 
      apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: guestbook
  namespace: argocd
spec:
  project: default
  source:
    repoURL: 'https://github.com/argoproj/argocd-example-apps'
    targetRevision: HEAD
    path: guestbook
  destination:
    server: 'https://kubernetes.default.svc'
    namespace: default
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
  healthChecks:
    - apiVersion: apps/v1
      kind: Deployment
      name: guestbook-ui
      health:
        status: Healthy
        message: 'Guestbook UI is healthy'

Conclusion

  • Summary:
    • GitOps with ArgoCD provides a powerful way to manage Kubernetes applications declaratively using Git as the source of truth.
    • ArgoCD automates the deployment process, ensuring that the actual state of the cluster matches the desired state defined in Git.
    • Advanced features like automated sync policies, multi-cluster management, and custom health checks make ArgoCD a versatile tool for Kubernetes continuous delivery.

By mastering GitOps with ArgoCD, you can achieve more reliable and consistent deployments, reduce manual intervention, and enhance the overall stability of your Kubernetes environments.
