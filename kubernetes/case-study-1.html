Introduction

Migrating an application to Kubernetes can be a complex but rewarding process. This case study will walk you through the steps of migrating a simple web application to Kubernetes, highlighting key concepts, best practices, and common pitfalls.

Key Concepts

  • Containers: Lightweight, standalone, executable packages that include everything needed to run a piece of software.
  • Kubernetes: An open-source platform designed to automate deploying, scaling, and operating application containers.
  • Pods: The smallest deployable units in Kubernetes, which can contain one or more containers.
  • Services: Abstractions that define a logical set of Pods and a policy by which to access them.
  • Deployments: Provide declarative updates to applications, ensuring the desired state is maintained.

Prerequisites

Before starting the migration, ensure you have:

  • A basic understanding of Docker and containerization.
  • A Kubernetes cluster up and running (e.g., Minikube for local development or a cloud provider like GKE, EKS, or AKS).
  • The kubectl command-line tool installed and configured to interact with your cluster.

Step-by-Step Migration

Step 1: Containerize the Application

First, we need to containerize our web application using Docker.

Dockerfile Example

# Use an official Node.js runtime as a parent image
FROM node:14

# Set the working directory
WORKDIR /usr/src/app

# Copy package.json and install dependencies
COPY package*.json ./
RUN npm install

# Copy the rest of the application code
COPY . .

# Expose the port the app runs on
EXPOSE 8080

# Define the command to run the app
CMD ["node", "app.js"]
  • FROM: Specifies the base image.
  • WORKDIR: Sets the working directory inside the container.
  • COPY: Copies files from the host to the container.
  • RUN: Executes commands in the container.
  • EXPOSE: Informs Docker that the container listens on the specified network ports.
  • CMD: Provides the default command to run the application.

Step 2: Build and Push the Docker Image

Build the Docker image and push it to a container registry (e.g., Docker Hub).

# Build the Docker image
docker build -t yourusername/yourapp:latest .

# Push the image to Docker Hub
docker push yourusername/yourapp:latest

Step 3: Create Kubernetes Manifests

Create Kubernetes manifests for the Deployment and Service.

Deployment Manifest (deployment.yaml)

apiVersion: apps/v1
kind: Deployment
metadata:
  name: webapp-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: webapp
  template:
    metadata:
      labels:
        app: webapp
    spec:
      containers:
      - name: webapp
        image: yourusername/yourapp:latest
        ports:
        - containerPort: 8080
  • replicas: Number of pod replicas.
  • selector: Identifies the pods managed by the deployment.
  • template: Defines the pod template.

Service Manifest (service.yaml)

apiVersion: v1
kind: Service
metadata:
  name: webapp-service
spec:
  selector:
    app: webapp
  ports:
    - protocol: TCP
      port: 80
      targetPort: 8080
  type: LoadBalancer
  • selector: Matches the pods to expose.
  • ports: Defines the port mappings.
  • type: Specifies the service type (e.g., LoadBalancer for external access).

Step 4: Deploy to Kubernetes

Apply the manifests to deploy the application to the Kubernetes cluster.

# Apply the deployment
kubectl apply -f deployment.yaml

# Apply the service
kubectl apply -f service.yaml

Step 5: Verify the Deployment

Check the status of the deployment and service.

# Get the status of the deployment
kubectl get deployments

# Get the status of the pods
kubectl get pods

# Get the status of the service
kubectl get services

Conclusion

Migrating to Kubernetes involves several steps, including containerizing the application, creating Kubernetes manifests, and deploying to the cluster. By following this structured approach, you can ensure a smooth migration process. Kubernetes provides powerful tools for managing containerized applications, making it easier to scale and maintain your services.

In this case study, we covered:

  • Containerizing a web application with Docker.
  • Creating Kubernetes manifests for deployment and service.
  • Deploying and verifying the application in a Kubernetes cluster.

This foundational knowledge will help you tackle more complex Kubernetes scenarios in the future.
