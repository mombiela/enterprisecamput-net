Introduction to CTEs

Common Table Expressions (CTEs) are a powerful feature in SQL that allow you to create temporary result sets that can be referenced within a SELECT, INSERT, UPDATE, or DELETE statement. They can make complex queries easier to read and maintain.

  • Definition: A CTE is a temporary result set that is defined within the execution scope of a single SELECT, INSERT, UPDATE, or DELETE statement.
  • Syntax: The basic syntax for a CTE is as follows: 
    WITH cte_name AS (
    -- CTE query
    SELECT column1, column2
    FROM table_name
    WHERE condition
)
-- Main query
SELECT column1, column2
FROM cte_name;

Benefits of Using CTEs

  • Improved Readability: CTEs can break down complex queries into simpler parts.
  • Reusability: You can reference the CTE multiple times within the main query.
  • Recursion: CTEs support recursive queries, which are useful for hierarchical data.

Basic Example of a CTE

Let's start with a simple example to understand how CTEs work.

Example: Selecting Employees from a Department

Suppose we have an employees table and we want to select employees from the 'Sales' department.

WITH SalesEmployees AS (
    SELECT employee_id, first_name, last_name
    FROM employees
    WHERE department = 'Sales'
)
SELECT employee_id, first_name, last_name
FROM SalesEmployees;
  • Explanation:
    • The CTE SalesEmployees is defined to select employees from the 'Sales' department.
    • The main query then selects from the SalesEmployees CTE.

Advanced Usage of CTEs

Multiple CTEs

You can define multiple CTEs in a single query by separating them with commas.

WITH SalesEmployees AS (
    SELECT employee_id, first_name, last_name
    FROM employees
    WHERE department = 'Sales'
),
MarketingEmployees AS (
    SELECT employee_id, first_name, last_name
    FROM employees
    WHERE department = 'Marketing'
)
SELECT * FROM SalesEmployees
UNION ALL
SELECT * FROM MarketingEmployees;

Recursive CTEs

Recursive CTEs are useful for hierarchical data, such as organizational charts or tree structures.

Example: Organizational Hierarchy

Suppose we have an employees table with a manager_id column that references the employee's manager.

WITH RECURSIVE OrgChart AS (
    SELECT employee_id, first_name, last_name, manager_id
    FROM employees
    WHERE manager_id IS NULL
    UNION ALL
    SELECT e.employee_id, e.first_name, e.last_name, e.manager_id
    FROM employees e
    INNER JOIN OrgChart o ON e.manager_id = o.employee_id
)
SELECT * FROM OrgChart;
  • Explanation:
    • The base case selects employees with no manager (manager_id IS NULL).
    • The recursive part joins the employees table with the OrgChart CTE to find employees managed by those in the OrgChart.

Practical Exercises

  1. Basic CTE: Create a CTE to select products with a price greater than $100 from a products table.
  2. Multiple CTEs: Create two CTEs to select employees from 'HR' and 'IT' departments, then combine the results.
  3. Recursive CTE: Create a recursive CTE to display a category hierarchy from a categories table with category_id and parent_category_id.

Conclusion

CTEs are a versatile tool in SQL that can simplify complex queries, improve readability, and handle hierarchical data efficiently. By mastering CTEs, you can write more maintainable and efficient SQL queries.

  • Key Takeaways:
    • CTEs provide a way to create temporary result sets.
    • They improve query readability and maintainability.
    • Recursive CTEs are powerful for working with hierarchical data.

Practice using CTEs in your SQL queries to become more proficient and take advantage of their full potential.
