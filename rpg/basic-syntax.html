Introduction

RPG (Report Program Generator) is a high-level programming language primarily used for business applications. Understanding the basic syntax and structure of RPG is crucial for writing efficient and effective programs. This section will cover the fundamental elements of RPG syntax and structure, providing a solid foundation for further learning.

Key Concepts

  • Specifications: RPG programs are divided into different specification types, each serving a specific purpose.

    • H-Spec: Header specification, used for global settings.
    • F-Spec: File specification, used to define files.
    • D-Spec: Definition specification, used for defining variables, constants, and data structures.
    • C-Spec: Calculation specification, used for executable code.
    • P-Spec: Procedure specification, used for defining subroutines and procedures.

  • Fixed Format vs. Free Format: RPG IV introduced free-format coding, which is more flexible and readable compared to the traditional fixed-format.

Specifications in Detail

H-Spec (Header Specification)

The H-spec is used to set global parameters for the program.

H dftactgrp(*no) actgrp('MYACTGRP')
  • dftactgrp(*no): Specifies that the program does not run in the default activation group.
  • actgrp('MYACTGRP'): Defines the activation group for the program.

F-Spec (File Specification)

The F-spec is used to define files that the program will use.

FMYFILE   IF   E           K DISK
  • FMYFILE: The name of the file.
  • I: Input file.
  • F: Full procedural file.
  • E: External file.
  • K: Keyed file.
  • DISK: File type.

D-Spec (Definition Specification)

The D-spec is used to define variables, constants, and data structures.

D CustomerName   S             50A
D OrderTotal     S              9P 2
  • CustomerName: A variable of type character with a length of 50.
  • OrderTotal: A packed decimal variable with 9 digits and 2 decimal places.

C-Spec (Calculation Specification)

The C-spec is where the executable code resides.

C     Eval      OrderTotal = OrderTotal + 100.00
C     If        OrderTotal > 500.00
C     Eval      Discount = OrderTotal * 0.1
C     EndIf
  • Eval: Used to assign values to variables.
  • If and EndIf: Used for conditional logic.

P-Spec (Procedure Specification)

The P-spec is used to define subroutines and procedures.

P MyProcedure   B
D MyProcedure   PI
D InputValue    10I 0
D OutputValue   10I 0

C     Eval      OutputValue = InputValue * 2
C     Return
P MyProcedure   E
  • B and E: Begin and End of the procedure.
  • PI: Procedure Interface.
  • Eval: Calculation within the procedure.
  • Return: End of the procedure logic.

Fixed Format vs. Free Format

Fixed Format Example

FMYFILE   IF   E           K DISK
D CustomerName   S             50A
C     Eval      CustomerName = 'John Doe'
C     If        CustomerName = 'John Doe'
C     Eval      Greeting = 'Hello, John!'
C     EndIf

Free Format Example

dcl-f MYFILE DISK(*EXT) USAGE(*INPUT);

dcl-s CustomerName char(50);
dcl-s Greeting char(50);

CustomerName = 'John Doe';

if CustomerName = 'John Doe';
  Greeting = 'Hello, John!';
endif;
  • Free Format: More readable and flexible, allowing for modern coding practices.

Conclusion

Understanding the basic syntax and structure of RPG is essential for writing effective programs. By mastering the different specifications and the differences between fixed and free format, you will be well-equipped to tackle more complex programming tasks in RPG. This foundational knowledge will serve as a stepping stone to more advanced topics in RPG programming.
