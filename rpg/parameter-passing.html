Introduction

Parameter passing is a fundamental concept in RPG (Report Program Generator) programming. It involves sending data to and from subroutines, procedures, and functions. Understanding how to pass parameters effectively can greatly enhance the modularity and reusability of your code.

Types of Parameter Passing

In RPG, parameters can be passed in several ways:

  • By Value
  • By Reference

By Value

When parameters are passed by value, a copy of the data is sent to the subroutine or procedure. Changes made to the parameter within the subroutine do not affect the original data.

Example:

DCL-PROC MyProcedure;
  DCL-PI MyProcedure;
    num INT(10);
  END-PI;

  num = num + 10;
END-PROC;

DCL-S myNum INT(10) INZ(5);
CALLP MyProcedure(myNum);
DSPLY myNum; // Output will be 5, as the original value is unchanged

By Reference

When parameters are passed by reference, the subroutine or procedure receives a reference to the original data. Changes made to the parameter within the subroutine will affect the original data.

Example:

DCL-PROC MyProcedure;
  DCL-PI MyProcedure;
    num POINTER;
  END-PI;

  num = num + 10;
END-PROC;

DCL-S myNum INT(10) INZ(5);
CALLP MyProcedure(%ADDR(myNum));
DSPLY myNum; // Output will be 15, as the original value is changed

Parameter Passing in Subroutines

Subroutines in RPG can also accept parameters. The parameters are defined in the BEGSR and ENDSR blocks.

Example:

DCL-S myNum INT(10) INZ(5);

CALLP MySubroutine(myNum);
DSPLY myNum; // Output will be 15

BEGSR MySubroutine;
  DCL-PARM num INT(10);
  num = num + 10;
ENDSR;

Parameter Passing in Procedures

Procedures in RPG are more flexible and can return values. Parameters are defined in the DCL-PI block.

Example:

DCL-PROC AddTen;
  DCL-PI AddTen INT(10);
    num INT(10);
  END-PI;

  RETURN num + 10;
END-PROC;

DCL-S myNum INT(10) INZ(5);
myNum = AddTen(myNum);
DSPLY myNum; // Output will be 15

Best Practices

  • Use By Value for Read-Only Data: If the subroutine or procedure does not need to modify the data, pass parameters by value to avoid unintended side effects.
  • Use By Reference for Large Data Structures: Passing large data structures by reference can improve performance by avoiding the overhead of copying data.
  • Clear Documentation: Always document the expected input and output of your subroutines and procedures to make your code easier to understand and maintain.

Conclusion

Parameter passing is a crucial aspect of RPG programming that allows for the creation of modular and reusable code. By understanding the differences between passing by value and by reference, and knowing when to use each method, you can write more efficient and maintainable programs. Practice with various examples and scenarios to become proficient in parameter passing in RPG.
