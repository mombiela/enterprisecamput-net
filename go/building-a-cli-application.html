Introduction

Command Line Interface (CLI) applications are powerful tools that allow users to interact with software through text commands. In this section, we will learn how to build a CLI application using Go. We will cover the basics, explore popular libraries, and build a simple CLI tool from scratch.

Key Concepts

  • CLI Application: A program that processes commands entered by users in a terminal or command prompt.
  • Flags: Command-line arguments that modify the behavior of the CLI application.
  • Subcommands: Commands within a CLI application that perform specific tasks.
  • Libraries: Packages that provide utilities to simplify the creation of CLI applications.

Setting Up Your Environment

Before we start building our CLI application, ensure you have Go installed on your machine. You can download it from the official Go website.

Basic CLI Application

Let's start with a simple CLI application that accepts a name as an argument and prints a greeting.

Example: Hello World CLI

package main

import (
    "flag"
    "fmt"
)

func main() {
    name := flag.String("name", "World", "a name to say hello to")
    flag.Parse()
    fmt.Printf("Hello, %s!\n", *name)
}

Explanation

  • flag.String: Defines a string flag with a default value ("World") and a description.
  • flag.Parse: Parses the command-line flags.
  • fmt.Printf: Prints the formatted string to the console.

Using a CLI Library

While the standard library's flag package is useful, there are more powerful libraries like cobra and urfave/cli that provide additional features and better structure for complex CLI applications.

Example: Using Cobra

First, install the cobra package:

go get -u github.com/spf13/cobra/cobra

Now, let's create a simple CLI application using cobra.

package main

import (
    "fmt"
    "github.com/spf13/cobra"
)

func main() {
    var rootCmd = &cobra.Command{
        Use:   "greet",
        Short: "Greet is a CLI tool for greeting people",
        Run: func(cmd *cobra.Command, args []string) {
            fmt.Println("Hello, World!")
        },
    }

    var name string
    var greetCmd = &cobra.Command{
        Use:   "hello",
        Short: "Prints a greeting message",
        Run: func(cmd *cobra.Command, args []string) {
            fmt.Printf("Hello, %s!\n", name)
        },
    }

    greetCmd.Flags().StringVarP(&name, "name", "n", "World", "name to greet")
    rootCmd.AddCommand(greetCmd)
    rootCmd.Execute()
}

Explanation

  • cobra.Command: Defines a command with usage, description, and a function to execute.
  • Flags().StringVarP: Adds a string flag to the command.
  • AddCommand: Adds a subcommand to the root command.
  • Execute: Runs the root command.

Building a More Complex CLI Application

Let's build a more complex CLI application with multiple subcommands and flags.

Example: Task Manager CLI

package main

import (
    "fmt"
    "github.com/spf13/cobra"
)

var tasks = []string{}

func main() {
    var rootCmd = &cobra.Command{Use: "task"}

    var addCmd = &cobra.Command{
        Use:   "add [task]",
        Short: "Add a new task",
        Args:  cobra.MinimumNArgs(1),
        Run: func(cmd *cobra.Command, args []string) {
            task := args[0]
            tasks = append(tasks, task)
            fmt.Printf("Added task: %s\n", task)
        },
    }

    var listCmd = &cobra.Command{
        Use:   "list",
        Short: "List all tasks",
        Run: func(cmd *cobra.Command, args []string) {
            fmt.Println("Tasks:")
            for i, task := range tasks {
                fmt.Printf("%d: %s\n", i+1, task)
            }
        },
    }

    rootCmd.AddCommand(addCmd, listCmd)
    rootCmd.Execute()
}

Explanation

  • Args: cobra.MinimumNArgs(1): Ensures that the command receives at least one argument.
  • tasks: A slice to store tasks.
  • addCmd: Adds a new task to the list.
  • listCmd: Lists all tasks.

Conclusion

Building CLI applications in Go can be straightforward with the right tools and libraries. We started with a basic example using the standard library and then explored the cobra library to build more complex applications. By understanding flags, subcommands, and how to structure your application, you can create powerful and user-friendly CLI tools.

In the next sections, we will dive deeper into advanced topics such as testing CLI applications, handling errors, and packaging your CLI tool for distribution.
