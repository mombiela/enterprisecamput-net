Introduction to Case Statements

Case statements in Ada are a control structure used to execute one block of code from multiple possible blocks based on the value of an expression. They are similar to switch statements in other programming languages like C or Java. Case statements improve code readability and maintainability by clearly delineating different execution paths.

Basic Syntax

The basic syntax of a case statement in Ada is as follows:

case expression is
   when choice1 =>
      -- Statements for choice1
   when choice2 =>
      -- Statements for choice2
   when others =>
      -- Statements for all other choices
end case;
  • expression: The value being evaluated.
  • choice1, choice2, etc.: Possible values of the expression.
  • others: A catch-all for any values not explicitly listed.

Example

Here is a simple example of a case statement in Ada:

with Ada.Text_IO; use Ada.Text_IO;

procedure Case_Example is
   Grade : Character := 'B';
begin
   case Grade is
      when 'A' =>
         Put_Line("Excellent!");
      when 'B' =>
         Put_Line("Good job!");
      when 'C' =>
         Put_Line("Well done!");
      when 'D' =>
         Put_Line("You passed.");
      when 'F' =>
         Put_Line("Better try again.");
      when others =>
         Put_Line("Invalid grade.");
   end case;
end Case_Example;

In this example, the program prints a message based on the value of the Grade variable.

Key Concepts

  • Expression: The value that the case statement evaluates.
  • Choices: The possible values that the expression can take.
  • Others: A default case that handles any values not explicitly listed.

Advanced Usage

Ranges and Multiple Choices

Ada allows you to specify ranges and multiple choices in a case statement. This can be useful for grouping similar cases together.

Example with Ranges

with Ada.Text_IO; use Ada.Text_IO;

procedure Case_Range_Example is
   Score : Integer := 85;
begin
   case Score is
      when 90 .. 100 =>
         Put_Line("Excellent!");
      when 80 .. 89 =>
         Put_Line("Very Good!");
      when 70 .. 79 =>
         Put_Line("Good!");
      when 60 .. 69 =>
         Put_Line("Pass.");
      when others =>
         Put_Line("Fail.");
   end case;
end Case_Range_Example;

In this example, the program prints a message based on the range in which the Score variable falls.

Nested Case Statements

You can nest case statements within each other to handle more complex decision-making processes.

Example of Nested Case Statements

with Ada.Text_IO; use Ada.Text_IO;

procedure Nested_Case_Example is
   Day : Integer := 3;
   Weather : Character := 'S';
begin
   case Day is
      when 1 | 2 | 3 | 4 | 5 =>
         Put_Line("Weekday");
         case Weather is
            when 'S' =>
               Put_Line("Sunny");
            when 'R' =>
               Put_Line("Rainy");
            when others =>
               Put_Line("Unknown weather");
         end case;
      when 6 | 7 =>
         Put_Line("Weekend");
         case Weather is
            when 'S' =>
               Put_Line("Sunny");
            when 'R' =>
               Put_Line("Rainy");
            when others =>
               Put_Line("Unknown weather");
         end case;
      when others =>
         Put_Line("Invalid day");
   end case;
end Nested_Case_Example;

In this example, the program first determines if the day is a weekday or weekend and then checks the weather condition.

Conclusion

Case statements in Ada are a powerful tool for controlling the flow of a program based on the value of an expression. They enhance code readability and maintainability by clearly separating different execution paths. By understanding the basic syntax, using ranges and multiple choices, and nesting case statements, you can handle complex decision-making processes effectively. Practice with the provided examples and exercises to become proficient in using case statements in Ada.
