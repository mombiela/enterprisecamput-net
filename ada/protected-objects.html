Protected Objects in Ada
Introduction to OOP ►

Protected objects in Ada provide a mechanism for safe and efficient concurrent programming. They are used to protect shared data from concurrent access, ensuring that only one task can access the protected data at a time. This is crucial in real-time and embedded systems where data integrity and synchronization are paramount.

Key Concepts

  • Protected Object: A construct that encapsulates data and operations, ensuring mutual exclusion and synchronization.
  • Protected Type: The definition of a protected object, specifying its data and operations.
  • Protected Procedures: Operations that modify the protected data.
  • Protected Functions: Operations that read the protected data.
  • Protected Entries: Operations that include synchronization mechanisms, such as waiting for a condition to be true.

Defining a Protected Type

A protected type is defined using the protected keyword. It includes the declaration of data and operations (procedures, functions, and entries).

protected type Shared_Counter is
   procedure Increment;
   function Value return Integer;
private
   Counter : Integer := 0;
end Shared_Counter;

Explanation

  • Protected Type Declaration: The Shared_Counter type includes a procedure Increment and a function Value.
  • Private Section: The Counter variable is private, meaning it can only be accessed by the procedures and functions within the protected type.

Implementing a Protected Type

The implementation of the protected type specifies the behavior of its operations.

protected body Shared_Counter is
   procedure Increment is
   begin
      Counter := Counter + 1;
   end Increment;

   function Value return Integer is
   begin
      return Counter;
   end Value;
end Shared_Counter;

Explanation

  • Increment Procedure: Increases the Counter by 1.
  • Value Function: Returns the current value of Counter.

Using Protected Objects

To use a protected object, you declare an instance of the protected type and call its operations.

procedure Main is
   Counter : Shared_Counter;
begin
   Counter.Increment;
   Ada.Text_IO.Put_Line("Counter Value: " & Integer'Image(Counter.Value));
end Main;

Explanation

  • Counter Declaration: An instance of Shared_Counter is declared.
  • Operation Calls: The Increment procedure is called to increase the counter, and the Value function is called to retrieve and print the counter's value.

Protected Entries

Protected entries are used for operations that require synchronization, such as waiting for a condition to be true.

protected type Synchronized_Buffer is
   entry Write(Item : in Integer);
   entry Read(Item : out Integer);
private
   Buffer : Integer;
   Full : Boolean := False;
end Synchronized_Buffer;

protected body Synchronized_Buffer is
   entry Write(Item : in Integer) when not Full is
   begin
      Buffer := Item;
      Full := True;
   end Write;

   entry Read(Item : out Integer) when Full is
   begin
      Item := Buffer;
      Full := False;
   end Read;
end Synchronized_Buffer;

Explanation

  • Write Entry: Waits until the buffer is not full before writing an item.
  • Read Entry: Waits until the buffer is full before reading an item.
  • Synchronization Conditions: The when clause specifies the condition that must be true for the entry to proceed.

Conclusion

Protected objects in Ada are a powerful tool for managing concurrent access to shared data. They provide a structured way to ensure data integrity and synchronization, which is essential in real-time and embedded systems. By encapsulating data and operations within a protected type, Ada ensures that only one task can access the protected data at a time, preventing race conditions and ensuring safe concurrent programming.
