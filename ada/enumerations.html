Enumerations in Ada
Access Types (Pointers) ►

Enumerations in Ada are a fundamental data type that allows you to define a set of named values. These values are called enumeration literals and are useful for representing a collection of related constants in a readable and maintainable way.

Key Concepts

  • Enumeration Type: A user-defined type that consists of a set of named values.
  • Enumeration Literals: The named values that belong to an enumeration type.
  • Type Safety: Enumerations in Ada provide type safety by ensuring that only valid values are assigned to variables of the enumeration type.

Defining Enumerations

To define an enumeration type in Ada, you use the type keyword followed by the type name and the list of enumeration literals enclosed in parentheses.

Example

type Day is (Monday, Tuesday, Wednesday, Thursday, Friday, Saturday, Sunday);

In this example, Day is an enumeration type with seven literals: Monday, Tuesday, Wednesday, Thursday, Friday, Saturday, and Sunday.

Using Enumerations

Once you have defined an enumeration type, you can declare variables of that type and assign them enumeration literals.

Example

procedure Enumerations_Example is
   type Day is (Monday, Tuesday, Wednesday, Thursday, Friday, Saturday, Sunday);
   Today : Day;
begin
   Today := Monday;
   if Today = Monday then
      Put_Line("It's Monday!");
   end if;
end Enumerations_Example;

In this example, the variable Today is of type Day, and it is assigned the value Monday. The program then checks if Today is Monday and prints a message accordingly.

Enumeration Attributes

Ada provides several attributes for enumeration types that can be useful in various situations.

  • 'Pos: Returns the position number of an enumeration literal.
  • 'Val: Converts a position number to the corresponding enumeration literal.
  • 'Succ: Returns the successor of an enumeration literal.
  • 'Pred: Returns the predecessor of an enumeration literal.

Example

procedure Enumeration_Attributes is
   type Day is (Monday, Tuesday, Wednesday, Thursday, Friday, Saturday, Sunday);
   Today : Day := Monday;
begin
   Put_Line("Position of Monday: " & Integer'Image(Day'Pos(Today)));
   Put_Line("Literal at position 2: " & Day'Image(Day'Val(2)));
   Put_Line("Successor of Monday: " & Day'Image(Day'Succ(Today)));
   Put_Line("Predecessor of Wednesday: " & Day'Image(Day'Pred(Wednesday)));
end Enumeration_Attributes;

In this example, various attributes are used to get information about the enumeration literals.

Advanced Usage

Enumeration Representation Clauses

You can specify the internal representation of enumeration literals using representation clauses. This can be useful for interfacing with hardware or other programming languages.

Example

type Color is (Red, Green, Blue);
for Color use (Red => 1, Green => 2, Blue => 3);

In this example, the enumeration literals Red, Green, and Blue are assigned the internal values 1, 2, and 3, respectively.

Enumeration and Case Statements

Enumerations are often used with case statements to handle different values in a structured way.

Example

procedure Case_Statement_Example is
   type Traffic_Light is (Red, Yellow, Green);
   Signal : Traffic_Light := Red;
begin
   case Signal is
      when Red =>
         Put_Line("Stop!");
      when Yellow =>
         Put_Line("Caution!");
      when Green =>
         Put_Line("Go!");
   end case;
end Case_Statement_Example;

In this example, a case statement is used to print different messages based on the value of the Signal variable.

Conclusion

Enumerations in Ada provide a powerful way to define and work with a set of related constants. They enhance code readability and maintainability while ensuring type safety. By understanding how to define, use, and manipulate enumerations, you can write more robust and clear Ada programs.
