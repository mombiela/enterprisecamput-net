Introduction to Web Storage

Web Storage is a feature in HTML5 that allows web applications to store data locally within the user's browser. This is useful for saving user preferences, maintaining session information, and more, without the need for server-side storage.

Key Concepts

  • Local Storage: Stores data with no expiration date. Data persists even when the browser is closed and reopened.
  • Session Storage: Stores data for the duration of the page session. Data is cleared when the page session ends (i.e., when the browser tab is closed).

Local Storage

Local Storage provides a way to store key-value pairs in a web browser with no expiration date.

Basic Operations

  • Set Item: Store data.
  • Get Item: Retrieve data.
  • Remove Item: Remove data.
  • Clear: Clear all stored data.

Code Examples

Setting an Item

// Store data in local storage
localStorage.setItem('username', 'JohnDoe');

Getting an Item

// Retrieve data from local storage
let username = localStorage.getItem('username');
console.log(username); // Output: JohnDoe

Removing an Item

// Remove data from local storage
localStorage.removeItem('username');

Clearing All Items

// Clear all data from local storage
localStorage.clear();

Practical Example

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Local Storage Example</title>
</head>
<body>
    <input type="text" id="nameInput" placeholder="Enter your name">
    <button onclick="saveName()">Save Name</button>
    <button onclick="showName()">Show Name</button>
    <p id="displayName"></p>

    <script>
        function saveName() {
            let name = document.getElementById('nameInput').value;
            localStorage.setItem('name', name);
        }

        function showName() {
            let name = localStorage.getItem('name');
            document.getElementById('displayName').innerText = name ? `Hello, ${name}!` : 'No name found.';
        }
    </script>
</body>
</html>

Session Storage

Session Storage works similarly to Local Storage but with a shorter lifespan. Data is only available for the duration of the page session.

Basic Operations

  • Set Item: Store data.
  • Get Item: Retrieve data.
  • Remove Item: Remove data.
  • Clear: Clear all stored data.

Code Examples

Setting an Item

// Store data in session storage
sessionStorage.setItem('sessionUser', 'JaneDoe');

Getting an Item

// Retrieve data from session storage
let sessionUser = sessionStorage.getItem('sessionUser');
console.log(sessionUser); // Output: JaneDoe

Removing an Item

// Remove data from session storage
sessionStorage.removeItem('sessionUser');

Clearing All Items

// Clear all data from session storage
sessionStorage.clear();

Practical Example

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Session Storage Example</title>
</head>
<body>
    <input type="text" id="sessionInput" placeholder="Enter session data">
    <button onclick="saveSessionData()">Save Session Data</button>
    <button onclick="showSessionData()">Show Session Data</button>
    <p id="displaySessionData"></p>

    <script>
        function saveSessionData() {
            let data = document.getElementById('sessionInput').value;
            sessionStorage.setItem('sessionData', data);
        }

        function showSessionData() {
            let data = sessionStorage.getItem('sessionData');
            document.getElementById('displaySessionData').innerText = data ? `Session Data: ${data}` : 'No session data found.';
        }
    </script>
</body>
</html>

Comparison of Local Storage and Session Storage

| Feature | Local Storage | Session Storage | |-------------------|----------------------------------|----------------------------------| | Lifespan | No expiration | Duration of the page session | | Data Persistence | Persistent across browser sessions | Cleared when the tab is closed | | Storage Capacity | Typically 5-10 MB per origin | Typically 5-10 MB per origin |

Conclusion

Web Storage provides a powerful way to store data on the client side, offering both Local Storage for persistent data and Session Storage for temporary data. Understanding how to use these storage mechanisms can greatly enhance the functionality and user experience of web applications.
