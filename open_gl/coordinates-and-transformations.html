Introduction

Understanding coordinates and transformations is fundamental to working with OpenGL. This topic will cover the basics of coordinate systems, transformations, and how to apply them in OpenGL. By the end of this section, you will be able to manipulate objects in 3D space effectively.

Coordinate Systems

In OpenGL, several coordinate systems are used to transform vertices from object space to screen space.

  • Object Coordinates: The local coordinates of an object.
  • World Coordinates: The coordinates of an object in the world space.
  • View Coordinates: The coordinates from the camera's perspective.
  • Clip Coordinates: The coordinates after applying the projection matrix.
  • Normalized Device Coordinates (NDC): The coordinates after perspective division.
  • Window Coordinates: The final coordinates on the screen.

Example

// Vertex in object coordinates
vec3 objectVertex = vec3(1.0, 1.0, 1.0);

// Model matrix transforms object coordinates to world coordinates
mat4 model = mat4(1.0);
vec4 worldVertex = model * vec4(objectVertex, 1.0);

// View matrix transforms world coordinates to view coordinates
mat4 view = lookAt(cameraPos, cameraTarget, upVector);
vec4 viewVertex = view * worldVertex;

// Projection matrix transforms view coordinates to clip coordinates
mat4 projection = perspective(radians(45.0f), aspectRatio, 0.1f, 100.0f);
vec4 clipVertex = projection * viewVertex;

// Perspective division to get NDC
vec3 ndcVertex = clipVertex.xyz / clipVertex.w;

// Viewport transformation to get window coordinates
vec2 windowVertex = (ndcVertex.xy + 1.0) / 2.0 * vec2(windowWidth, windowHeight);

Transformations

Transformations are used to move, rotate, and scale objects in OpenGL. The primary transformations are:

  • Translation: Moving an object in 3D space.
  • Rotation: Rotating an object around an axis.
  • Scaling: Changing the size of an object.

Translation

Translation moves an object by a certain offset.

mat4 translation = mat4(1.0);
translation = translate(translation, vec3(1.0, 2.0, 3.0));

Rotation

Rotation rotates an object around a specified axis by a certain angle.

mat4 rotation = mat4(1.0);
rotation = rotate(rotation, radians(45.0f), vec3(0.0, 1.0, 0.0));

Scaling

Scaling changes the size of an object.

mat4 scaling = mat4(1.0);
scaling = scale(scaling, vec3(2.0, 2.0, 2.0));

Combining Transformations

Transformations can be combined by multiplying their matrices. The order of multiplication is crucial.

Example

mat4 model = mat4(1.0);
model = translate(model, vec3(1.0, 2.0, 3.0)); // Translation
model = rotate(model, radians(45.0f), vec3(0.0, 1.0, 0.0)); // Rotation
model = scale(model, vec3(2.0, 2.0, 2.0)); // Scaling

Practical Example

Let's create a simple example where we apply these transformations to a cube.

Vertex Shader

#version 330 core
layout (location = 0) in vec3 aPos;

uniform mat4 model;
uniform mat4 view;
uniform mat4 projection;

void main()
{
    gl_Position = projection * view * model * vec4(aPos, 1.0);
}

Fragment Shader

#version 330 core
out vec4 FragColor;

void main()
{
    FragColor = vec4(1.0, 0.5, 0.2, 1.0);
}

Main Program

// Set up vertex data (and buffer(s)) and configure vertex attributes
float vertices[] = {
    // positions          
    -0.5f, -0.5f, -0.5f,  
     0.5f, -0.5f, -0.5f,  
     0.5f,  0.5f, -0.5f,  
     0.5f,  0.5f, -0.5f,  
    -0.5f,  0.5f, -0.5f,  
    -0.5f, -0.5f, -0.5f,  

    // ... (other vertices)
};

// Initialize buffers and configure vertex attributes
unsigned int VBO, VAO;
glGenVertexArrays(1, &VAO);
glGenBuffers(1, &VBO);

glBindVertexArray(VAO);

glBindBuffer(GL_ARRAY_BUFFER, VBO);
glBufferData(GL_ARRAY_BUFFER, sizeof(vertices), vertices, GL_STATIC_DRAW);

glVertexAttribPointer(0, 3, GL_FLOAT, GL_FALSE, 3 * sizeof(float), (void*)0);
glEnableVertexAttribArray(0);

// Main render loop
while (!glfwWindowShouldClose(window))
{
    // Input handling
    processInput(window);

    // Render
    glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);

    // Activate shader
    ourShader.use();

    // Create transformations
    mat4 model = mat4(1.0f);
    model = translate(model, vec3(0.0f, 0.0f, -3.0f));
    model = rotate(model, (float)glfwGetTime(), vec3(0.5f, 1.0f, 0.0f));
    ourShader.setMat4("model", model);

    mat4 view = mat4(1.0f);
    view = translate(view, vec3(0.0f, 0.0f, -5.0f));
    ourShader.setMat4("view", view);

    mat4 projection = perspective(radians(45.0f), (float)SCR_WIDTH / (float)SCR_HEIGHT, 0.1f, 100.0f);
    ourShader.setMat4("projection", projection);

    // Render container
    glBindVertexArray(VAO);
    glDrawArrays(GL_TRIANGLES, 0, 36);

    // Swap buffers and poll IO events
    glfwSwapBuffers(window);
    glfwPollEvents();
}

// Deallocate resources
glDeleteVertexArrays(1, &VAO);
glDeleteBuffers(1, &VBO);

glfwTerminate();

Conclusion

Understanding coordinates and transformations is essential for 3D graphics programming in OpenGL. By mastering these concepts, you can control the positioning, orientation, and scaling of objects in your scene. Practice combining different transformations to see their effects and solidify your understanding.
