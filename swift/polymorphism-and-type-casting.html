Polymorphism and Type Casting in Swift
Protocols and Delegates ►

Introduction

Polymorphism and type casting are fundamental concepts in object-oriented programming that allow for more flexible and reusable code. In Swift, these concepts are implemented in a way that is both powerful and easy to use. This section will cover the basics of polymorphism and type casting, provide examples, and include exercises to solidify your understanding.

Polymorphism

Polymorphism allows objects to be treated as instances of their parent class rather than their actual class. This enables a single function to operate on different types of objects.

Key Concepts

  • Polymorphism: The ability of different classes to respond to the same method call in different ways.
  • Method Overriding: Subclasses can provide a specific implementation of a method that is already defined in its superclass.
  • Dynamic Dispatch: The process of selecting which implementation of a polymorphic operation (method) to call at runtime.

Example

class Animal {
    func makeSound() {
        print("Some generic animal sound")
    }
}

class Dog: Animal {
    override func makeSound() {
        print("Bark")
    }
}

class Cat: Animal {
    override func makeSound() {
        print("Meow")
    }
}

let animals: [Animal] = [Dog(), Cat(), Animal()]

for animal in animals {
    animal.makeSound()
}

In this example:

  • We have a base class Animal with a method makeSound.
  • Dog and Cat are subclasses of Animal that override the makeSound method.
  • An array of Animal type holds instances of Dog, Cat, and Animal.
  • When iterating through the array, the overridden makeSound method is called based on the actual object type.

Type Casting

Type casting in Swift is used to check the type of an instance or to treat that instance as a different superclass or subclass from somewhere else in its class hierarchy.

Key Concepts

  • is Operator: Checks if an instance is of a certain subclass type.
  • as? Operator: Attempts to downcast to a subclass type and returns an optional.
  • as! Operator: Forcefully downcasts to a subclass type and can cause a runtime error if the downcast fails.
  • as Operator: Used for upcasting and type conversion.

Example

class Vehicle {
    var name: String
    init(name: String) {
        self.name = name
    }
}

class Car: Vehicle {
    var numberOfDoors: Int
    init(name: String, numberOfDoors: Int) {
        self.numberOfDoors = numberOfDoors
        super.init(name: name)
    }
}

class Bicycle: Vehicle {
    var hasBasket: Bool
    init(name: String, hasBasket: Bool) {
        self.hasBasket = hasBasket
        super.init(name: name)
    }
}

let vehicles: [Vehicle] = [Car(name: "Sedan", numberOfDoors: 4), Bicycle(name: "Mountain Bike", hasBasket: true)]

for vehicle in vehicles {
    if let car = vehicle as? Car {
        print("Car: \(car.name), Doors: \(car.numberOfDoors)")
    } else if let bicycle = vehicle as? Bicycle {
        print("Bicycle: \(bicycle.name), Has Basket: \(bicycle.hasBasket)")
    }
}

In this example:

  • We have a base class Vehicle with subclasses Car and Bicycle.
  • An array of Vehicle type holds instances of Car and Bicycle.
  • We use as? to safely downcast each Vehicle to its specific subclass type and print specific properties.

Exercises

  1. Polymorphism Exercise: Create a base class Shape with a method area(). Create subclasses Circle and Rectangle that override the area() method. Create an array of Shape objects and print the area of each shape.
  2. Type Casting Exercise: Create a base class Employee with subclasses Manager and Developer. Create an array of Employee objects and use type casting to print specific properties of Manager and Developer.

Conclusion

Polymorphism and type casting are essential tools in Swift that allow for more flexible and reusable code. Polymorphism enables objects to be treated as instances of their parent class, while type casting allows you to check and convert types at runtime. By mastering these concepts, you can write more dynamic and efficient Swift programs.
