Control Transfer Statements in Swift
Optionals and Unwrapping ►

Control transfer statements are essential in programming as they allow you to change the flow of execution in your code. In Swift, these statements help you manage loops, conditionals, and other control structures effectively. This section will cover the various control transfer statements available in Swift, from basic to advanced usage.

Break Statement

The break statement is used to exit a loop or switch statement immediately.

Usage in Loops

for number in 1...10 {
    if number == 5 {
        break
    }
    print(number)
}
  • Explanation: This loop prints numbers from 1 to 4. When number equals 5, the break statement exits the loop.

Usage in Switch Statements

let fruit = "Apple"

switch fruit {
case "Apple":
    print("It's an apple.")
    break
case "Banana":
    print("It's a banana.")
default:
    print("Unknown fruit.")
}
  • Explanation: The break statement here is redundant because Swift automatically exits the switch case after executing the matched case. However, it can be useful for readability or future modifications.

Continue Statement

The continue statement skips the current iteration of a loop and proceeds to the next iteration.

Usage in Loops

for number in 1...10 {
    if number % 2 == 0 {
        continue
    }
    print(number)
}
  • Explanation: This loop prints only odd numbers from 1 to 10. When number is even, the continue statement skips the current iteration.

Fallthrough Statement

The fallthrough statement is used in switch cases to continue execution to the next case.

Usage in Switch Statements

let number = 3

switch number {
case 1:
    print("One")
case 2:
    print("Two")
case 3:
    print("Three")
    fallthrough
case 4:
    print("Four")
default:
    print("Other number")
}
  • Explanation: When number is 3, the switch case prints "Three" and then "Four" due to the fallthrough statement.

Return Statement

The return statement exits a function and optionally returns a value.

Usage in Functions

func greet(name: String) -> String {
    return "Hello, \(name)!"
}

let greeting = greet(name: "Alice")
print(greeting)
  • Explanation: The return statement exits the greet function and returns the greeting string.

Throw Statement

The throw statement is used to throw an error in Swift.

Usage in Functions

enum CustomError: Error {
    case invalidInput
}

func validate(input: Int) throws {
    if input < 0 {
        throw CustomError.invalidInput
    }
}

do {
    try validate(input: -1)
} catch {
    print("Caught an error: \(error)")
}
  • Explanation: The throw statement throws a CustomError.invalidInput error if the input is negative. The do-catch block handles the error.

Guard Statement

The guard statement is used to transfer control out of a scope if a condition is not met.

Usage in Functions

func process(input: Int?) {
    guard let validInput = input else {
        print("Invalid input")
        return
    }
    print("Processing \(validInput)")
}

process(input: nil)
process(input: 10)
  • Explanation: The guard statement checks if input is non-nil. If it is nil, it prints "Invalid input" and exits the function. Otherwise, it continues to process the valid input.

Summary

Control transfer statements in Swift are powerful tools for managing the flow of your code. They allow you to:

  • Exit loops and switch statements (break).
  • Skip iterations in loops (continue).
  • Continue execution to the next switch case (fallthrough).
  • Exit functions and return values (return).
  • Throw and handle errors (throw).
  • Ensure conditions are met before proceeding (guard).

Understanding and using these statements effectively will help you write more robust and readable Swift code.
