Introduction to Extensions

Extensions in Swift are a powerful feature that allows you to add new functionality to existing classes, structures, enumerations, or protocols. This can be done without modifying the original source code, making your code more modular and organized.

Key Concepts

  • Adding Computed Properties: Extensions can add new computed properties to existing types.
  • Adding Methods: You can add new instance methods and type methods.
  • Adding Initializers: Extensions can add new initializers to existing types.
  • Adding Subscripts: You can add new subscripts to existing types.
  • Adding Nested Types: Extensions can add new nested types to existing classes, structures, or enumerations.
  • Conforming to Protocols: Extensions can make an existing type conform to a protocol.

Adding Computed Properties

Extensions can add computed properties to existing types. However, they cannot add stored properties or property observers.

extension Double {
    var km: Double { return self * 1_000.0 }
    var m: Double { return self }
    var cm: Double { return self / 100.0 }
    var mm: Double { return self / 1_000.0 }
}

let oneMeter = 1.0.m
print("One meter is \(oneMeter) meters") // One meter is 1.0 meters

Adding Methods

Extensions can add new instance methods and type methods to existing types.

extension Int {
    func repetitions(task: () -> Void) {
        for _ in 0..<self {
            task()
        }
    }
}

3.repetitions {
    print("Hello!")
}
// Prints "Hello!" three times

Adding Initializers

Extensions can add new initializers to existing types. This is particularly useful for adding convenience initializers.

struct Size {
    var width = 0.0, height = 0.0
}

extension Size {
    init(side: Double) {
        self.init(width: side, height: side)
    }
}

let square = Size(side: 10.0)
print("Square size: \(square.width) x \(square.height)") // Square size: 10.0 x 10.0

Adding Subscripts

Extensions can add new subscripts to existing types.

extension String {
    subscript(index: Int) -> Character {
        return self[self.index(self.startIndex, offsetBy: index)]
    }
}

let greeting = "Hello"
print(greeting[1]) // e

Adding Nested Types

Extensions can add new nested types to existing classes, structures, or enumerations.

extension Int {
    enum Kind {
        case negative, zero, positive
    }

    var kind: Kind {
        switch self {
        case 0:
            return .zero
        case let x where x > 0:
            return .positive
        default:
            return .negative
        }
    }
}

func printIntegerKinds(_ numbers: [Int]) {
    for number in numbers {
        switch number.kind {
        case .negative:
            print("- ", terminator: "")
        case .zero:
            print("0 ", terminator: "")
        case .positive:
            print("+ ", terminator: "")
        }
    }
    print("")
}

printIntegerKinds([3, 19, -27, 0, -6, 0, 7])
// Prints "+ + - 0 - 0 + "

Conforming to Protocols

Extensions can make an existing type conform to a protocol, which is a common way to add protocol conformance to a type that you do not own.

protocol TextRepresentable {
    var textualDescription: String { get }
}

extension Int: TextRepresentable {
    var textualDescription: String {
        return String(self)
    }
}

let number = 42
print(number.textualDescription) // 42

Conclusion

Extensions are a versatile feature in Swift that enable you to add new functionality to existing types in a clean and modular way. They allow you to extend types you do not own, add computed properties, methods, initializers, subscripts, nested types, and make types conform to protocols. By using extensions, you can keep your code organized and maintainable while enhancing the capabilities of existing types.
