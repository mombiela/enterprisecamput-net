Introduction

In Swift, properties and methods are fundamental concepts that allow you to store and manipulate data within your classes, structures, and enumerations. Understanding these concepts is crucial for writing efficient and effective Swift code.

Properties

Properties associate values with a particular class, structure, or enumeration. They can be categorized into stored properties, computed properties, and property observers.

Stored Properties

Stored properties store constant or variable values as part of an instance. They can be either variable properties (declared with var) or constant properties (declared with let).

struct Person {
    var name: String
    let birthYear: Int
}

var person = Person(name: "Alice", birthYear: 1990)
person.name = "Bob" // Allowed because 'name' is a variable property
// person.birthYear = 1991 // Error: Cannot assign to 'birthYear' because it is a 'let' constant

Computed Properties

Computed properties do not store a value directly. Instead, they provide a getter and an optional setter to retrieve and set other properties or values indirectly.

struct Rectangle {
    var width: Double
    var height: Double
    
    var area: Double {
        return width * height
    }
}

let rect = Rectangle(width: 5.0, height: 10.0)
print("Area: \(rect.area)") // Output: Area: 50.0

Property Observers

Property observers observe and respond to changes in a property's value. They can be added to stored properties (except for lazy properties) and are called every time a property's value is set.

class StepCounter {
    var totalSteps: Int = 0 {
        willSet(newTotalSteps) {
            print("About to set totalSteps to \(newTotalSteps)")
        }
        didSet {
            if totalSteps > oldValue {
                print("Added \(totalSteps - oldValue) steps")
            }
        }
    }
}

let stepCounter = StepCounter()
stepCounter.totalSteps = 200
// Output:
// About to set totalSteps to 200
// Added 200 steps

Methods

Methods are functions that are associated with a particular type. They can be instance methods or type methods.

Instance Methods

Instance methods are functions that belong to instances of a particular class, structure, or enumeration.

class Counter {
    var count = 0
    
    func increment() {
        count += 1
    }
    
    func increment(by amount: Int) {
        count += amount
    }
    
    func reset() {
        count = 0
    }
}

let counter = Counter()
counter.increment()
print(counter.count) // Output: 1
counter.increment(by: 5)
print(counter.count) // Output: 6
counter.reset()
print(counter.count) // Output: 0

Type Methods

Type methods are methods that are called on the type itself, rather than on an instance of the type. They are defined with the static keyword for structures and enumerations, and with the class keyword for classes to allow for method overriding.

class SomeClass {
    class func someTypeMethod() {
        print("Type method called")
    }
}

SomeClass.someTypeMethod() // Output: Type method called

Conclusion

Properties and methods are essential building blocks in Swift programming. Properties allow you to store and compute values, while methods enable you to define behaviors for your types. By mastering these concepts, you can create more robust and maintainable Swift applications. Practice using stored properties, computed properties, property observers, instance methods, and type methods to deepen your understanding and improve your coding skills.
