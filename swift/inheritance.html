Inheritance in Swift
Polymorphism and Type Casting ►

Inheritance is a fundamental concept in object-oriented programming that allows a class to inherit properties and methods from another class. This helps in reusing code and creating a hierarchical relationship between classes.

Key Concepts

  • Superclass: The class whose properties and methods are inherited.
  • Subclass: The class that inherits properties and methods from the superclass.
  • Overriding: The process of providing a new implementation for an inherited method or property.
  • Final Classes: Classes that cannot be inherited.

Creating a Superclass

To create a superclass in Swift, you define a class with properties and methods that can be inherited by other classes.

class Vehicle {
    var currentSpeed = 0.0
    
    func description() -> String {
        return "Traveling at \(currentSpeed) miles per hour"
    }
}

In this example, Vehicle is a superclass with a property currentSpeed and a method description().

Creating a Subclass

A subclass inherits properties and methods from its superclass. You create a subclass by specifying the superclass after the subclass name, separated by a colon.

class Bicycle: Vehicle {
    var hasBasket = false
}

Here, Bicycle is a subclass of Vehicle. It inherits the currentSpeed property and the description() method from Vehicle.

Overriding Methods

A subclass can override methods and properties of its superclass to provide a specific implementation.

class Bicycle: Vehicle {
    var hasBasket = false
    
    override func description() -> String {
        return "Traveling at \(currentSpeed) miles per hour on a bicycle"
    }
}

In this example, the description() method is overridden in the Bicycle class to provide a more specific description.

Using Superclass Methods and Properties

A subclass can use the super keyword to access methods and properties of its superclass.

class Tandem: Bicycle {
    var currentNumberOfPassengers = 0
    
    override func description() -> String {
        return super.description() + " with \(currentNumberOfPassengers) passengers"
    }
}

Here, the Tandem class, which is a subclass of Bicycle, uses super.description() to call the description() method of its superclass and then adds additional information.

Final Classes

If you want to prevent a class from being inherited, you can mark it as final.

final class Train: Vehicle {
    var numberOfCarriages = 0
}

The Train class cannot be subclassed because it is marked as final.

Exercises

  1. Create a Superclass and Subclass:

    • Define a superclass Animal with a property name and a method makeSound().
    • Create a subclass Dog that overrides the makeSound() method to print "Woof!".

  2. Override Methods:

    • Create a superclass Shape with a method area().
    • Create subclasses Circle and Rectangle that override the area() method to calculate the area of the shape.

  3. Use Superclass Methods:

    • Create a superclass Employee with a method work().
    • Create a subclass Manager that overrides the work() method and uses super.work() to call the superclass method.

Summary

Inheritance allows you to create a new class based on an existing class, promoting code reuse and establishing a hierarchical relationship between classes. Key concepts include superclasses, subclasses, overriding methods, and final classes. By understanding and utilizing inheritance, you can create more organized and maintainable code in Swift.
