Inheritance in C#
Polymorphism ►

Inheritance is a fundamental concept in object-oriented programming (OOP) that allows a class to inherit properties and methods from another class. This promotes code reusability and establishes a natural hierarchy between classes.

Key Concepts

  • Base Class (Parent Class): The class whose properties and methods are inherited.
  • Derived Class (Child Class): The class that inherits from the base class.
  • protected Access Modifier: Allows a member to be accessible within its own class and by derived class instances.
  • virtual Keyword: Indicates that a method or property can be overridden in a derived class.
  • override Keyword: Used to extend or modify the virtual/abstract implementation of an inherited method, property, indexer, or event.
  • sealed Keyword: Prevents a class from being inherited or a method from being overridden.

Basic Inheritance

Example

// Base class
public class Animal
{
    public string Name { get; set; }

    public void Eat()
    {
        Console.WriteLine($"{Name} is eating.");
    }
}

// Derived class
public class Dog : Animal
{
    public void Bark()
    {
        Console.WriteLine($"{Name} is barking.");
    }
}

class Program
{
    static void Main()
    {
        Dog myDog = new Dog();
        myDog.Name = "Buddy";
        myDog.Eat();  // Inherited method
        myDog.Bark(); // Derived class method
    }
}

Explanation

  • The Animal class is the base class with a property Name and a method Eat.
  • The Dog class is the derived class that inherits from Animal and adds a new method Bark.
  • In the Main method, an instance of Dog is created, and both inherited and new methods are called.

Access Modifiers in Inheritance

Example

public class Animal
{
    protected string Name { get; set; }

    public void Eat()
    {
        Console.WriteLine($"{Name} is eating.");
    }
}

public class Dog : Animal
{
    public void SetName(string name)
    {
        Name = name; // Accessing protected member
    }

    public void Bark()
    {
        Console.WriteLine($"{Name} is barking.");
    }
}

class Program
{
    static void Main()
    {
        Dog myDog = new Dog();
        myDog.SetName("Buddy");
        myDog.Eat();
        myDog.Bark();
    }
}

Explanation

  • The Name property in the Animal class is marked as protected, making it accessible within the Dog class.
  • The SetName method in the Dog class sets the Name property, demonstrating the use of the protected access modifier.

Overriding Methods

Example

public class Animal
{
    public virtual void MakeSound()
    {
        Console.WriteLine("Some generic animal sound.");
    }
}

public class Dog : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("Bark!");
    }
}

class Program
{
    static void Main()
    {
        Animal myAnimal = new Animal();
        myAnimal.MakeSound(); // Output: Some generic animal sound.

        Dog myDog = new Dog();
        myDog.MakeSound(); // Output: Bark!
    }
}

Explanation

  • The MakeSound method in the Animal class is marked as virtual, allowing it to be overridden.
  • The Dog class overrides the MakeSound method to provide a specific implementation.

Sealed Classes and Methods

Example

public class Animal
{
    public virtual void MakeSound()
    {
        Console.WriteLine("Some generic animal sound.");
    }
}

public class Dog : Animal
{
    public sealed override void MakeSound()
    {
        Console.WriteLine("Bark!");
    }
}

// This will cause a compile-time error
public class Bulldog : Dog
{
    // Cannot override the sealed method
    // public override void MakeSound()
    // {
    //     Console.WriteLine("Bulldog bark!");
    // }
}

class Program
{
    static void Main()
    {
        Dog myDog = new Dog();
        myDog.MakeSound(); // Output: Bark!
    }
}

Explanation

  • The MakeSound method in the Dog class is marked as sealed, preventing further overriding in derived classes like Bulldog.

Conclusion

Inheritance in C# allows for the creation of a class hierarchy where derived classes inherit and extend the functionality of base classes. Key concepts include the use of access modifiers, virtual and override keywords, and the sealed keyword to control inheritance and method overriding. Understanding these concepts is crucial for designing robust and reusable object-oriented software.
