Introduction to Interfaces

Interfaces in C# are a fundamental concept that allows you to define a contract that classes or structs can implement. They are used to specify what methods and properties a class should have, without providing the implementation details.

Key Concepts

  • Definition: An interface is defined using the interface keyword.
  • Implementation: A class or struct implements an interface by providing the code for each method and property defined in the interface.
  • Multiple Inheritance: C# supports multiple inheritance of interfaces, meaning a class can implement multiple interfaces.
  • Polymorphism: Interfaces enable polymorphic behavior, allowing you to use different classes interchangeably if they implement the same interface.

Defining an Interface

To define an interface in C#, use the interface keyword followed by the interface name. Interface names typically start with an uppercase 'I'.

public interface IAnimal
{
    void MakeSound();
    void Move();
}

Implementing an Interface

A class or struct implements an interface by using the : symbol followed by the interface name. The class must provide concrete implementations for all the methods and properties defined in the interface.

public class Dog : IAnimal
{
    public void MakeSound()
    {
        Console.WriteLine("Bark");
    }

    public void Move()
    {
        Console.WriteLine("Run");
    }
}

Using Interfaces

Interfaces can be used to achieve polymorphism. You can write code that works with objects of any class that implements a particular interface.

public class Program
{
    public static void Main()
    {
        IAnimal myDog = new Dog();
        myDog.MakeSound(); // Output: Bark
        myDog.Move();      // Output: Run
    }
}

Interface Inheritance

Interfaces can inherit from other interfaces. This allows you to create more specific interfaces based on more general ones.

public interface IAnimal
{
    void MakeSound();
    void Move();
}

public interface IPet : IAnimal
{
    void Play();
}

public class Cat : IPet
{
    public void MakeSound()
    {
        Console.WriteLine("Meow");
    }

    public void Move()
    {
        Console.WriteLine("Jump");
    }

    public void Play()
    {
        Console.WriteLine("Chase laser pointer");
    }
}

Explicit Interface Implementation

Sometimes, a class might implement multiple interfaces that have methods with the same name. In such cases, you can use explicit interface implementation to avoid ambiguity.

public interface IPrinter
{
    void Print();
}

public interface IScanner
{
    void Print();
}

public class MultiFunctionDevice : IPrinter, IScanner
{
    void IPrinter.Print()
    {
        Console.WriteLine("Printing document");
    }

    void IScanner.Print()
    {
        Console.WriteLine("Scanning document");
    }
}

Advanced Concepts

Default Interface Methods (C# 8.0 and later)

C# 8.0 introduced default interface methods, allowing you to provide a default implementation for methods in an interface.

public interface IAnimal
{
    void MakeSound();
    void Move()
    {
        Console.WriteLine("Moving...");
    }
}

public class Bird : IAnimal
{
    public void MakeSound()
    {
        Console.WriteLine("Chirp");
    }
}

Interface vs Abstract Class

Conclusion

Interfaces in C# are a powerful tool for defining contracts and achieving polymorphism. They allow you to write flexible and reusable code by specifying what methods and properties a class should implement without dictating how they should be implemented. Understanding interfaces and their advanced features, such as default methods and explicit implementation, is crucial for writing robust and maintainable C# applications.
