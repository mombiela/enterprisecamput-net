What is a Functor?

In Haskell, a Functor is a type class that represents a type that can be mapped over. It is a fundamental concept in functional programming and is used to apply a function to values wrapped in a context, such as a list, Maybe, or other data structures.

Key Concepts

  • Type Class: A type class defines a set of functions that can be implemented by different types.
  • Mapping: Applying a function to each element in a data structure.
  • Context: The structure that holds the values, such as a list or Maybe.

Functor Type Class

The Functor type class is defined in the Prelude as follows:

class Functor f where
    fmap :: (a -> b) -> f a -> f b
  • fmap is the primary function of the Functor type class.
  • It takes a function (a -> b) and a functor f a, and returns a new functor f b.

Example: Functor Instance for List

The list type [] is a functor. Here’s how it works:

instance Functor [] where
    fmap = map
  • The map function applies a given function to each element of a list.

Code Example

-- Using fmap with a list
fmap (+1) [1, 2, 3]  -- Result: [2, 3, 4]

In this example, fmap applies the function (+1) to each element of the list [1, 2, 3], resulting in [2, 3, 4].

Functor Laws

Functors must obey two laws to ensure consistent behavior:

  1. Identity Law: fmap id == id
  2. Composition Law: fmap (f . g) == fmap f . fmap g

Identity Law

Applying the identity function id using fmap should return the original functor.

Code Example

fmap id [1, 2, 3]  -- Result: [1, 2, 3]

Composition Law

Mapping the composition of two functions should be the same as mapping one function and then the other.

Code Example

fmap ((+1) . (*2)) [1, 2, 3]  -- Result: [3, 5, 7]
-- Equivalent to
fmap (+1) (fmap (*2) [1, 2, 3])  -- Result: [3, 5, 7]

Functor Instances for Other Types

Maybe Functor

The Maybe type is also a functor. It represents computations that might fail.

instance Functor Maybe where
    fmap _ Nothing  = Nothing
    fmap f (Just x) = Just (f x)

Code Example

fmap (+1) (Just 5)  -- Result: Just 6
fmap (+1) Nothing   -- Result: Nothing

Either Functor

The Either type represents computations that might result in an error.

instance Functor (Either a) where
    fmap _ (Left x)  = Left x
    fmap f (Right y) = Right (f y)

Code Example

fmap (+1) (Right 5)  -- Result: Right 6
fmap (+1) (Left "Error")  -- Result: Left "Error"

Practical Use Cases

Transforming Data in a List

-- Doubling each element in a list
fmap (*2) [1, 2, 3, 4]  -- Result: [2, 4, 6, 8]

Handling Optional Values

-- Incrementing a value inside a Maybe
fmap (+1) (Just 10)  -- Result: Just 11
fmap (+1) Nothing    -- Result: Nothing

Conclusion

Functors are a powerful abstraction in Haskell that allow you to apply functions to values wrapped in a context. By understanding and using the Functor type class, you can write more generic and reusable code. Remember the Functor laws to ensure your instances behave consistently. Functors are just the beginning; they pave the way to more advanced concepts like Applicatives and Monads.
