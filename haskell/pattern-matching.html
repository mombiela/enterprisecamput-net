Pattern Matching in Haskell
Immutability ►

Pattern matching is a powerful feature in Haskell that allows you to check a value against a pattern. It is a fundamental concept that is used extensively in Haskell programming for deconstructing data structures, controlling flow, and more.

Introduction to Pattern Matching

Pattern matching in Haskell is used to:

  • Deconstruct data structures.
  • Simplify code by handling different cases.
  • Improve readability and maintainability.

Basic Pattern Matching

In its simplest form, pattern matching can be used in function definitions to match specific values.

-- Function to check if a number is zero
isZero :: Int -> Bool
isZero 0 = True
isZero _ = False

In this example:

  • isZero 0 = True matches the input 0 and returns True.
  • isZero _ = False uses the wildcard pattern _ to match any other value and returns False.

Pattern Matching with Lists

Lists are a common data structure in Haskell, and pattern matching can be used to deconstruct them.

-- Function to get the head of a list
head' :: [a] -> a
head' [] = error "Empty list"
head' (x:_) = x

In this example:

  • head' [] = error "Empty list" matches an empty list and throws an error.
  • head' (x:_) = x matches a non-empty list, binds the head of the list to x, and returns it.

Pattern Matching with Tuples

Tuples can also be deconstructed using pattern matching.

-- Function to add two pairs of integers
addPairs :: (Int, Int) -> (Int, Int) -> (Int, Int)
addPairs (a, b) (c, d) = (a + c, b + d)

In this example:

  • addPairs (a, b) (c, d) = (a + c, b + d) matches two pairs of integers and returns a new pair with their sums.

Advanced Pattern Matching

Guards

Guards provide a way to add more complex conditions to pattern matching.

-- Function to classify a number
classifyNumber :: Int -> String
classifyNumber n
  | n < 0     = "Negative"
  | n == 0    = "Zero"
  | otherwise = "Positive"

In this example:

  • Guards (|) are used to check multiple conditions.
  • otherwise is a catch-all guard that matches any value not caught by previous guards.

As-Patterns

As-patterns allow you to keep a reference to the entire value while pattern matching on its components.

-- Function to duplicate the first element of a list
duplicateHead :: [a] -> [a]
duplicateHead xs@(x:_) = x : xs
duplicateHead [] = []

In this example:

  • xs@(x:_) matches a non-empty list, binds the entire list to xs, and the head to x.
  • The function returns a new list with the head duplicated.

Pattern Matching in Case Expressions

Case expressions provide a way to use pattern matching within expressions.

-- Function to describe a list
describeList :: [a] -> String
describeList xs = case xs of
  []      -> "Empty list"
  [x]     -> "Single element list"
  (x:xs)  -> "Longer list"

In this example:

  • case xs of is used to match the list xs against different patterns.

Summary

Pattern matching is a versatile and powerful feature in Haskell that simplifies code and enhances readability. By mastering pattern matching, you can write more concise and expressive Haskell programs. Key concepts include:

  • Basic pattern matching in function definitions.
  • Deconstructing lists and tuples.
  • Using guards for complex conditions.
  • As-patterns for referencing entire values.
  • Case expressions for inline pattern matching.

Understanding and effectively using pattern matching will significantly improve your Haskell programming skills.
