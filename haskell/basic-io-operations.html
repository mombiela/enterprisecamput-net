Introduction

In Haskell, Input/Output (I/O) operations are handled differently compared to imperative languages. Haskell is a purely functional language, meaning functions have no side effects. However, I/O operations inherently have side effects, so Haskell uses a special type called IO to manage these operations. This section will cover the basics of I/O in Haskell, including reading from and writing to the console, working with files, and understanding the IO type.

Key Concepts

  • Pure Functions vs. I/O Actions: Understanding the difference between pure functions and I/O actions.
  • The IO Type: Introduction to the IO type and how it encapsulates side effects.
  • Basic I/O Functions: Using putStrLn, getLine, readFile, and writeFile.
  • Combining I/O Actions: Using do notation to sequence I/O actions.

Pure Functions vs. I/O Actions

In Haskell, pure functions always produce the same output for the same input and have no side effects. I/O actions, on the other hand, interact with the outside world and can produce different results each time they are executed.

-- Pure function example
add :: Int -> Int -> Int
add x y = x + y

-- I/O action example
main :: IO ()
main = do
    putStrLn "Enter your name:"
    name <- getLine
    putStrLn ("Hello, " ++ name ++ "!")

The `IO` Type

The IO type is used to represent actions that perform I/O. An IO action is a value that, when executed, interacts with the outside world and produces a result.

-- Example of an IO action
main :: IO ()
main = putStrLn "Hello, World!"

In the above example, putStrLn "Hello, World!" is an IO action that prints a string to the console.

Basic I/O Functions

`putStrLn`

putStrLn is used to print a string followed by a newline to the console.

main :: IO ()
main = putStrLn "Hello, World!"

`getLine`

getLine reads a line of input from the console.

main :: IO ()
main = do
    putStrLn "Enter your name:"
    name <- getLine
    putStrLn ("Hello, " ++ name ++ "!")

`readFile` and `writeFile`

readFile reads the contents of a file as a string, and writeFile writes a string to a file.

main :: IO ()
main = do
    contents <- readFile "example.txt"
    putStrLn contents

writeExample :: IO ()
writeExample = do
    writeFile "example.txt" "This is an example file."

Combining I/O Actions

To perform multiple I/O actions in sequence, we use do notation. This allows us to write a series of actions that are executed in order.

main :: IO ()
main = do
    putStrLn "Enter the first number:"
    num1 <- getLine
    putStrLn "Enter the second number:"
    num2 <- getLine
    let sum = (read num1 :: Int) + (read num2 :: Int)
    putStrLn ("The sum is: " ++ show sum)

In this example, we read two numbers from the user, calculate their sum, and print the result.

Conclusion

Understanding basic I/O operations in Haskell is crucial for interacting with the outside world. The IO type encapsulates side effects, allowing Haskell to maintain its functional purity. By using functions like putStrLn, getLine, readFile, and writeFile, and combining them with do notation, you can perform complex I/O operations in a structured and readable way.
