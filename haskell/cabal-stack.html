Introduction to Build Tools in Haskell

Haskell, like many programming languages, has its own set of tools to manage dependencies, build projects, and handle package distribution. The two primary build tools in the Haskell ecosystem are Cabal and Stack. Understanding these tools is crucial for efficient Haskell development.

Cabal

Cabal is a system for building and packaging Haskell libraries and programs. It is part of the Haskell Platform and has been around for a long time.

Key Concepts of Cabal

  • Package Description File: Each Haskell package has a .cabal file that describes the package, its dependencies, and other metadata.
  • Hackage: The central package repository for Haskell libraries.
  • Sandboxing: A way to isolate dependencies for a project to avoid conflicts.

Basic Cabal Commands

  • Initialize a new project:

    cabal init

    This command creates a new .cabal file and sets up the project structure.

  • Build a project:

    cabal build

    This command compiles the project and its dependencies.

  • Run a project:

    cabal run

    This command builds and runs the project.

  • Install dependencies:

    cabal install --only-dependencies

    This command installs all the dependencies listed in the .cabal file.

Example: Creating a Simple Cabal Project

  1. Initialize the project:

    cabal init

    Follow the prompts to set up the project.

  2. Edit the .cabal file:

    name:                my-project
version:             0.1.0.0
build-type:          Simple
cabal-version:       >=1.10

library
  exposed-modules:     MyLib
  build-depends:       base >=4.7 && <5
  hs-source-dirs:      src
  default-language:    Haskell2010

  3. Create the source directory and a simple module:

    mkdir src
echo "module MyLib where\n\nhello :: String\nhello = \"Hello, World!\"" > src/MyLib.hs

  4. Build and run the project:

    cabal build
cabal repl

Stack

Stack is a newer build tool that aims to provide a more user-friendly experience and better dependency management.

Key Concepts of Stack

  • Stackage: A curated set of packages that are known to work well together.
  • Snapshots: Specific sets of package versions that are tested together.
  • Project Configuration: Stack uses a stack.yaml file to configure the project.

Basic Stack Commands

  • Initialize a new project:

    stack new my-project

    This command creates a new project with a default template.

  • Build a project:

    stack build

    This command compiles the project and its dependencies.

  • Run a project:

    stack run

    This command builds and runs the project.

  • Install dependencies:

    stack setup

    This command sets up the compiler and installs dependencies.

Example: Creating a Simple Stack Project

  1. Initialize the project:

    stack new my-project
cd my-project

  2. Edit the stack.yaml file:

    resolver: lts-18.18

packages:
- .
extra-deps: []

  3. Edit the package.yaml file:

    name: my-project
version: 0.1.0.0
dependencies:
- base >= 4.7 && < 5

library:
  source-dirs: src
  exposed-modules: MyLib

  4. Create the source directory and a simple module:

    mkdir src
echo "module MyLib where\n\nhello :: String\nhello = \"Hello, World!\"" > src/MyLib.hs

  5. Build and run the project:

    stack build
stack ghci

Comparison of Cabal and Stack

| Feature | Cabal | Stack | |------------------------|--------------------------------------------|--------------------------------------------| | Package Repository | Hackage | Stackage | | Dependency Management | Manual, with sandboxing | Automatic, with curated snapshots | | Configuration File | .cabal file | stack.yaml and package.yaml | | Ease of Use | Requires more manual setup | More user-friendly, especially for beginners| | Isolation | Sandboxing (optional) | Always isolated |

Conclusion

Both Cabal and Stack are powerful tools for managing Haskell projects, each with its own strengths. Cabal offers flexibility and has been a staple in the Haskell community for years, while Stack provides a more streamlined and user-friendly experience, especially for beginners. Understanding both tools will give you a solid foundation for managing Haskell projects effectively.
