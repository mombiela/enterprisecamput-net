Introduction to Currying and Partial Application

Currying and partial application are fundamental concepts in Haskell that allow for more flexible and modular code. They enable functions to be applied in stages, which can lead to more readable and reusable code.

Key Concepts

  • Currying: The process of transforming a function that takes multiple arguments into a sequence of functions, each taking a single argument.
  • Partial Application: The process of fixing a few arguments of a function, producing another function of smaller arity.

Understanding Currying

Currying is named after the mathematician Haskell Curry. In Haskell, all functions are curried by default.

Example of Currying

Consider a simple function that adds two numbers:

add :: Int -> Int -> Int
add x y = x + y

This function can be curried:

addCurried :: Int -> (Int -> Int)
addCurried x = \y -> x + y

In Haskell, add and addCurried are essentially the same due to Haskell's inherent currying.

Using Curried Functions

You can apply arguments one at a time:

let addFive = add 5
addFive 3  -- Result: 8

Here, add 5 returns a new function that adds 5 to its argument.

Partial Application

Partial application is closely related to currying. It involves applying a function to some of its arguments, producing another function that takes the remaining arguments.

Example of Partial Application

Using the same add function:

let addTen = add 10
addTen 7  -- Result: 17

Here, add 10 is a partially applied function that adds 10 to any given number.

Practical Use Cases

Partial application is useful for creating specialized functions from more general ones:

multiply :: Int -> Int -> Int
multiply x y = x * y

let double = multiply 2
double 8  -- Result: 16

let triple = multiply 3
triple 5  -- Result: 15

Comparison of Currying and Partial Application

| Concept | Description | Example | |--------------------|-----------------------------------------------------------------------------|----------------------------------------------| | Currying | Transforming a function with multiple arguments into a sequence of functions | addCurried x = \y -> x + y | | Partial Application| Applying a function to some of its arguments to get a new function | let addFive = add 5 |

Advanced Usage

Higher-Order Functions

Currying and partial application shine when used with higher-order functions:

map (add 3) [1, 2, 3]  -- Result: [4, 5, 6]

Here, add 3 is partially applied and then mapped over a list.

Function Composition

You can compose curried functions to build more complex operations:

let increment = add 1
let double = multiply 2
let incrementThenDouble = double . increment

incrementThenDouble 3  -- Result: 8

Exercises

  1. Currying Practice: Write a curried function that concatenates three strings. 
    concatThree :: String -> String -> String -> String
concatThree a b c = a ++ b ++ c
  2. Partial Application Practice: Create a partially applied function from a curried function that multiplies three numbers. 
    multiplyThree :: Int -> Int -> Int -> Int
multiplyThree x y z = x * y * z

let multiplyByTwoAndThree = multiplyThree 2 3
multiplyByTwoAndThree 4  -- Result: 24

Conclusion

Currying and partial application are powerful techniques in Haskell that enhance code modularity and reusability. By understanding and utilizing these concepts, you can write more concise and flexible Haskell programs. Practice these techniques with various functions to become proficient in their use.
