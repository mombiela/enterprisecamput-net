Introduction to Exception Handling

Exception handling is a crucial aspect of programming that allows developers to manage errors and exceptional conditions gracefully. In Haskell, exception handling is approached differently compared to imperative languages due to its functional nature.

Basic Concepts

  • Exception: An unexpected event that disrupts the normal flow of a program.
  • Exception Handling: Mechanisms to catch and manage exceptions to prevent program crashes.

Types of Exceptions

Haskell categorizes exceptions into:

  • Synchronous Exceptions: Errors that occur during the execution of a program, such as division by zero or file not found.
  • Asynchronous Exceptions: External events that interrupt the program, such as user interrupts or timeouts.

Handling Exceptions with `Maybe` and `Either`

Haskell provides two primary types for handling errors in a functional way:

  • Maybe: Represents an optional value that can either be Just a value or Nothing.
  • Either: Represents a value that can be one of two types, typically used for error handling where Left represents an error and Right represents a success.

Example: Using `Maybe`

safeDivide :: Double -> Double -> Maybe Double
safeDivide _ 0 = Nothing
safeDivide x y = Just (x / y)

main :: IO ()
main = do
    let result = safeDivide 10 2
    case result of
        Nothing -> putStrLn "Division by zero!"
        Just value -> putStrLn $ "Result: " ++ show value

Example: Using `Either`

safeDivide :: Double -> Double -> Either String Double
safeDivide _ 0 = Left "Division by zero!"
safeDivide x y = Right (x / y)

main :: IO ()
main = do
    let result = safeDivide 10 0
    case result of
        Left err -> putStrLn err
        Right value -> putStrLn $ "Result: " ++ show value

Advanced Exception Handling with `Control.Exception`

For more complex scenarios, Haskell provides the Control.Exception module which allows for more fine-grained control over exceptions.

Key Functions

  • throw: Throws an exception.
  • catch: Catches an exception.
  • try: Attempts an action and returns either an exception or a result.

Example: Using `Control.Exception`

import Control.Exception

main :: IO ()
main = do
    result <- try (evaluate (5 `div` 0)) :: IO (Either SomeException Int)
    case result of
        Left ex -> putStrLn $ "Caught exception: " ++ show ex
        Right value -> putStrLn $ "Result: " ++ show value

Custom Exceptions

You can define your own exceptions by creating a new data type and making it an instance of the Exception class.

Example: Defining and Using Custom Exceptions

import Control.Exception

data MyException = MyException String deriving (Show)

instance Exception MyException

main :: IO ()
main = do
    result <- try (throwIO (MyException "Something went wrong")) :: IO (Either MyException ())
    case result of
        Left ex -> putStrLn $ "Caught custom exception: " ++ show ex
        Right _ -> putStrLn "No exception"

Summary

Exception handling in Haskell can be approached using simple types like Maybe and Either for basic error management, or more advanced techniques using the Control.Exception module for complex scenarios. Understanding these mechanisms allows you to write robust and fault-tolerant Haskell programs.
