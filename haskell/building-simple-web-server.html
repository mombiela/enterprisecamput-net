Building a Simple Web Server in Haskell
Database Access ►

Introduction

In this topic, we will explore how to build a simple web server using Haskell. This will cover the basics of setting up a Haskell environment, understanding HTTP, and using the wai and warp libraries to create a functional web server.

Prerequisites

Before starting, ensure you have:

  • Basic understanding of Haskell syntax and functions.
  • Haskell installed on your machine (GHC and Cabal or Stack).

Setting Up the Environment

To begin, we need to set up our Haskell environment and install necessary libraries.

Installing Haskell

  1. GHC (Glasgow Haskell Compiler): The main compiler for Haskell.
  2. Cabal or Stack: Build tools for managing Haskell projects.

Installing Libraries

We will use the wai (Web Application Interface) and warp (a fast HTTP server library) libraries.

cabal update
cabal install wai warp

Or, if using Stack:

stack update
stack install wai warp

Understanding HTTP Basics

Before diving into code, let's briefly understand HTTP, the protocol used by web servers to communicate with clients (browsers).

  • Request: Sent by the client to the server, includes methods like GET, POST, etc.
  • Response: Sent by the server to the client, includes status codes (200, 404, etc.) and data.

Building the Web Server

Creating a Basic Server

Let's start by creating a simple web server that responds with "Hello, World!" to any request.

  1. Create a new Haskell file:

    touch Main.hs

  2. Write the following code in Main.hs:

    {-# LANGUAGE OverloadedStrings #-}
import Network.Wai (responseLBS, Application)
import Network.Wai.Handler.Warp (run)
import Network.HTTP.Types (status200)

main :: IO ()
main = do
    putStrLn "Starting server on port 8080..."
    run 8080 app

app :: Application
app _ respond = respond $ responseLBS status200 [("Content-Type", "text/plain")] "Hello, World!"

Explanation

  • Imports: We import necessary modules from wai, warp, and http-types.
  • Main Function: Starts the server on port 8080 and uses the app function to handle requests.
  • App Function: Defines the application logic, which responds with a status 200 and "Hello, World!" message.

Running the Server

To run the server, use the following command:

runhaskell Main.hs

Open your browser and navigate to http://localhost:8080 to see "Hello, World!".

Adding Routes

Next, let's add some routes to handle different paths.

Code Example

Update Main.hs to include routing:

{-# LANGUAGE OverloadedStrings #-}
import Network.Wai (responseLBS, Application, pathInfo)
import Network.Wai.Handler.Warp (run)
import Network.HTTP.Types (status200, status404)

main :: IO ()
main = do
    putStrLn "Starting server on port 8080..."
    run 8080 app

app :: Application
app req respond = case pathInfo req of
    ["hello"] -> respond $ responseLBS status200 [("Content-Type", "text/plain")] "Hello, World!"
    ["goodbye"] -> respond $ responseLBS status200 [("Content-Type", "text/plain")] "Goodbye, World!"
    _ -> respond $ responseLBS status404 [("Content-Type", "text/plain")] "404 - Not Found"

Explanation

  • pathInfo: Extracts the path from the request.
  • Case Matching: Matches the path to predefined routes and responds accordingly.

Running the Server

Run the server again and navigate to:

  • http://localhost:8080/hello to see "Hello, World!"
  • http://localhost:8080/goodbye to see "Goodbye, World!"
  • Any other path will return "404 - Not Found".

Handling Query Parameters

Let's handle query parameters to make our server more dynamic.

Code Example

Update Main.hs to handle query parameters:

{-# LANGUAGE OverloadedStrings #-}
import Network.Wai (responseLBS, Application, queryString)
import Network.Wai.Handler.Warp (run)
import Network.HTTP.Types (status200, status400)
import Data.Maybe (fromMaybe)
import Data.ByteString.Lazy.Char8 (pack)

main :: IO ()
main = do
    putStrLn "Starting server on port 8080..."
    run 8080 app

app :: Application
app req respond = case pathInfo req of
    ["greet"] -> do
        let name = fromMaybe "World" $ lookup "name" $ queryString req
        respond $ responseLBS status200 [("Content-Type", "text/plain")] (pack $ "Hello, " ++ show name ++ "!")
    _ -> respond $ responseLBS status400 [("Content-Type", "text/plain")] "400 - Bad Request"

Explanation

  • queryString: Extracts query parameters from the request.
  • fromMaybe: Provides a default value if the parameter is not found.
  • pack: Converts a string to a ByteString.

Running the Server

Run the server and navigate to:

  • http://localhost:8080/greet?name=Haskell to see "Hello, Haskell!"

Conclusion

In this topic, we covered the basics of building a simple web server in Haskell using the wai and warp libraries. We started with a basic server, added routing, and handled query parameters. This foundation allows you to build more complex web applications in Haskell.

Summary

  • Set up the Haskell environment and installed necessary libraries.
  • Built a basic web server responding with "Hello, World!".
  • Added routing to handle different paths.
  • Handled query parameters to make the server dynamic.

With these skills, you can now explore more advanced topics such as middleware, database integration, and more complex routing.
