Higher-Order Functions in Haskell
Lambda Expressions ►

Higher-order functions are a fundamental concept in Haskell and functional programming in general. They are functions that can take other functions as arguments and/or return functions as results. This topic will guide you from the basics to more advanced uses of higher-order functions in Haskell.

Introduction to Higher-Order Functions

  • Definition: A higher-order function is a function that:

    • Takes one or more functions as arguments.
    • Returns a function as its result.

  • Importance: Higher-order functions allow for more abstract and flexible code. They enable powerful patterns like function composition, currying, and more.

Basic Examples

Let's start with a simple example to illustrate the concept:

-- A function that takes a function and a value, and applies the function to the value
applyTwice :: (a -> a) -> a -> a
applyTwice f x = f (f x)

-- Example usage
double :: Int -> Int
double x = x * 2

main = print (applyTwice double 3)  -- Output: 12

In this example:

  • applyTwice is a higher-order function because it takes a function f as an argument.
  • double is a simple function that multiplies its input by 2.
  • applyTwice double 3 applies double to 3 twice, resulting in 12.

Common Higher-Order Functions

`map`

  • Description: Applies a function to each element of a list.
  • Type Signature: (a -> b) -> [a] -> [b]
-- Example usage of map
increment :: Int -> Int
increment x = x + 1

main = print (map increment [1, 2, 3, 4])  -- Output: [2, 3, 4, 5]

`filter`

  • Description: Selects elements of a list that satisfy a predicate.
  • Type Signature: (a -> Bool) -> [a] -> [a]
-- Example usage of filter
isEven :: Int -> Bool
isEven x = x `mod` 2 == 0

main = print (filter isEven [1, 2, 3, 4, 5, 6])  -- Output: [2, 4, 6]

`foldl` and `foldr`

  • Description: Reduces a list to a single value by combining elements using a binary function.
  • Type Signature:
    • foldl: (b -> a -> b) -> b -> [a] -> b
    • foldr: (a -> b -> b) -> b -> [a] -> b
-- Example usage of foldl
sumList :: [Int] -> Int
sumList = foldl (+) 0

main = print (sumList [1, 2, 3, 4])  -- Output: 10

Function Composition

  • Description: Combining two or more functions to form a new function.
  • Operator: (.)
  • Type Signature: (b -> c) -> (a -> b) -> a -> c
-- Example usage of function composition
addOne :: Int -> Int
addOne x = x + 1

multiplyByTwo :: Int -> Int
multiplyByTwo x = x * 2

-- Compose addOne and multiplyByTwo
composedFunction = addOne . multiplyByTwo

main = print (composedFunction 3)  -- Output: 7

Currying

  • Description: Transforming a function that takes multiple arguments into a series of functions that each take a single argument.
  • Example:
-- A function that adds two numbers
add :: Int -> Int -> Int
add x y = x + y

-- Curried version
addFive :: Int -> Int
addFive = add 5

main = print (addFive 3)  -- Output: 8

Advanced Usage

Creating Custom Higher-Order Functions

  • Example: A function that takes a function and applies it three times.
-- A function that applies a given function three times
applyThrice :: (a -> a) -> a -> a
applyThrice f x = f (f (f x))

-- Example usage
main = print (applyThrice double 2)  -- Output: 16

Using Higher-Order Functions with Data Structures

  • Example: Using map with a custom data structure.
-- Define a custom data structure
data Tree a = Leaf a | Node (Tree a) (Tree a) deriving Show

-- Define a map function for the Tree
treeMap :: (a -> b) -> Tree a -> Tree b
treeMap f (Leaf x) = Leaf (f x)
treeMap f (Node left right) = Node (treeMap f left) (treeMap f right)

-- Example usage
main = print (treeMap increment (Node (Leaf 1) (Leaf 2)))  -- Output: Node (Leaf 2) (Leaf 3)

Conclusion

Higher-order functions are a powerful feature of Haskell that enable more abstract and flexible programming. By understanding and utilizing higher-order functions, you can write more concise, reusable, and expressive code. This topic covered the basics of higher-order functions, common higher-order functions like map, filter, and fold, as well as more advanced concepts like function composition and currying. Practice using these concepts to become proficient in functional programming with Haskell.
