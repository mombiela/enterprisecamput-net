Introduction

Hackage and Stackage are two essential components of the Haskell ecosystem that help manage and distribute Haskell libraries and packages. Understanding these tools is crucial for Haskell developers, from beginners to advanced users.

Hackage

Hackage is the central package archive for Haskell libraries. It hosts a vast collection of Haskell packages contributed by the community.

Key Concepts

  • Package Repository: Hackage serves as a repository where developers can upload and share their Haskell packages.
  • Cabal: The build system and package manager for Haskell, which interacts with Hackage to download and manage dependencies.
  • Metadata: Each package on Hackage includes metadata such as version, dependencies, and documentation.

Using Hackage

To use Hackage, you typically interact with it through the cabal command-line tool.

Example: Installing a Package from Hackage

cabal update
cabal install <package-name>

Explanation:

  1. cabal update updates the local list of available packages from Hackage.
  2. cabal install <package-name> installs the specified package.

Publishing a Package to Hackage

To publish your own package to Hackage, you need to follow these steps:

  1. Prepare your package: Ensure your package has a .cabal file with all necessary metadata.
  2. Build and test: Use cabal build and cabal test to ensure your package builds and works correctly.
  3. Upload: Use cabal upload to upload your package to Hackage.

Example: Uploading a Package

cabal sdist
cabal upload dist/<package-name-version>.tar.gz

Explanation:

  1. cabal sdist creates a source distribution of your package.
  2. cabal upload uploads the generated tarball to Hackage.

Stackage

Stackage is a stable source of Haskell packages that provides curated sets of packages known to work well together.

Key Concepts

  • LTS (Long Term Support): Stackage provides LTS releases, which are stable snapshots of Haskell packages.
  • Stability: Stackage ensures that packages in a snapshot are compatible with each other, reducing dependency issues.
  • Stack: A build tool that works with Stackage to manage project dependencies.

Using Stackage

To use Stackage, you typically interact with it through the stack command-line tool.

Example: Creating a New Project with Stack

stack new my-project
cd my-project
stack setup
stack build

Explanation:

  1. stack new my-project creates a new Haskell project using a template.
  2. cd my-project changes the directory to the new project.
  3. stack setup installs the appropriate GHC (Glasgow Haskell Compiler) version.
  4. stack build builds the project using the Stackage snapshot.

Configuring Stackage Snapshots

You can specify which Stackage snapshot to use in your stack.yaml file.

Example: Specifying a Snapshot

resolver: lts-18.18

Explanation:

  • The resolver field specifies the Stackage snapshot version to use for your project.

Conclusion

Hackage and Stackage are fundamental tools in the Haskell ecosystem that facilitate package management and ensure stability in Haskell projects. Hackage serves as a comprehensive package repository, while Stackage provides curated, stable sets of packages. By mastering these tools, Haskell developers can efficiently manage dependencies and maintain robust projects.
