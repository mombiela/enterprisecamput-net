Introduction to Algebraic Data Types (ADTs)

Algebraic Data Types (ADTs) are a fundamental concept in Haskell and functional programming. They allow you to define custom data types that can represent a combination of different values. ADTs are composed of two main types: product types and sum types.

Key Concepts

  • Product Types: Combine multiple values into a single data structure (e.g., tuples, records).
  • Sum Types: Represent a value that can be one of several different types (e.g., Either, Maybe).

Product Types

Product types are used to group multiple values together. In Haskell, the most common product types are tuples and records.

Tuples

Tuples are simple product types that group a fixed number of values together.

-- A tuple containing an Int and a String
exampleTuple :: (Int, String)
exampleTuple = (42, "Hello")

Records

Records are more descriptive product types that use named fields.

-- Defining a record type for a Person
data Person = Person {
    name :: String,
    age :: Int
}

-- Creating an instance of Person
examplePerson :: Person
examplePerson = Person { name = "Alice", age = 30 }

Sum Types

Sum types allow a value to be one of several different types. They are defined using the data keyword with multiple constructors.

Either

The Either type represents a value that can be one of two types.

-- Either can be Left or Right
exampleEither :: Either String Int
exampleEither = Right 42

Maybe

The Maybe type represents an optional value that can either be Just a value or Nothing.

-- Maybe can be Just a value or Nothing
exampleMaybe :: Maybe Int
exampleMaybe = Just 42

Defining Custom ADTs

You can define your own ADTs using the data keyword.

Example: Shape

Let's define a custom ADT for geometric shapes.

-- Defining a Shape type with Circle and Rectangle constructors
data Shape = Circle Float
           | Rectangle Float Float

-- Function to calculate the area of a Shape
area :: Shape -> Float
area (Circle radius) = pi * radius * radius
area (Rectangle width height) = width * height

-- Examples of using the Shape type
exampleCircle :: Shape
exampleCircle = Circle 5.0

exampleRectangle :: Shape
exampleRectangle = Rectangle 4.0 6.0

Pattern Matching

Pattern matching is a powerful feature in Haskell that allows you to deconstruct ADTs and work with their components.

Example: Pattern Matching with Shape

Using the Shape type defined earlier, we can pattern match to handle different shapes.

-- Function to describe a Shape
describeShape :: Shape -> String
describeShape (Circle radius) = "A circle with radius " ++ show radius
describeShape (Rectangle width height) = "A rectangle with width " ++ show width ++ " and height " ++ show height

-- Examples of using describeShape
exampleDescription1 :: String
exampleDescription1 = describeShape exampleCircle

exampleDescription2 :: String
exampleDescription2 = describeShape exampleRectangle

Advanced Topics

As you become more comfortable with ADTs, you can explore more advanced topics such as:

  • Recursive Data Types: ADTs that reference themselves.
  • Parameterized Types: ADTs that take type parameters.
  • Type Classes and ADTs: Using type classes with ADTs for polymorphism.

Recursive Data Types

Recursive data types are used to define structures like lists and trees.

-- Defining a recursive List type
data List a = Empty
            | Cons a (List a)

-- Function to calculate the length of a List
listLength :: List a -> Int
listLength Empty = 0
listLength (Cons _ tail) = 1 + listLength(tail)

-- Example of using the List type
exampleList :: List Int
exampleList = Cons 1 (Cons 2 (Cons 3 Empty))

Parameterized Types

Parameterized types allow ADTs to be more flexible and reusable.

-- Defining a parameterized Pair type
data Pair a b = Pair a b

-- Example of using the Pair type
examplePair :: Pair Int String
examplePair = Pair 42 "Answer"

Conclusion

Algebraic Data Types are a powerful feature in Haskell that allow you to define complex data structures in a clear and concise way. By understanding product types, sum types, and pattern matching, you can create robust and expressive programs. As you advance, exploring recursive and parameterized types will further enhance your ability to model and solve problems effectively in Haskell.
