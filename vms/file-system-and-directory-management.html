Introduction

OpenVMS is a multi-user, multiprocessing virtual memory-based operating system designed for use in time-sharing, batch processing, and transaction processing. Understanding how to manage files and directories is crucial for effective system administration and user operations.

Key Concepts

  • File System Structure: OpenVMS uses a hierarchical file system structure.
  • Directories: Directories are used to organize files.
  • File Naming Conventions: OpenVMS has specific rules for naming files.
  • File Operations: Basic operations include creating, reading, writing, and deleting files.
  • Directory Operations: Basic operations include creating, listing, and deleting directories.

File System Structure

OpenVMS uses a hierarchical file system where files are organized in directories. The root directory is the top-level directory, and all other directories branch off from it.

Directories

Directories in OpenVMS are used to organize files logically. Each directory can contain files and subdirectories.

Creating a Directory

To create a directory, use the CREATE/DIRECTORY command:

$ CREATE/DIRECTORY [MYDIR]

This command creates a directory named MYDIR.

Listing Directory Contents

To list the contents of a directory, use the DIRECTORY command:

$ DIRECTORY [MYDIR]

This command lists all files and subdirectories within MYDIR.

Deleting a Directory

To delete a directory, use the DELETE command:

$ DELETE [MYDIR]

Note: The directory must be empty before it can be deleted.

File Naming Conventions

OpenVMS file names have the following format:

filename.extension;version
  • Filename: Up to 39 characters.
  • Extension: Up to 39 characters.
  • Version: A number that allows multiple versions of the same file.

Example

$ CREATE MYFILE.TXT;1

This command creates a file named MYFILE.TXT with version 1.

File Operations

Creating a File

To create a file, use the CREATE command:

$ CREATE MYFILE.TXT

This command opens a text editor to create MYFILE.TXT.

Reading a File

To read a file, use the TYPE command:

$ TYPE MYFILE.TXT

This command displays the contents of MYFILE.TXT.

Writing to a File

To write to a file, use the EDIT command:

$ EDIT MYFILE.TXT

This command opens a text editor to modify MYFILE.TXT.

Deleting a File

To delete a file, use the DELETE command:

$ DELETE MYFILE.TXT;1

This command deletes version 1 of MYFILE.TXT.

Directory Operations

Creating a Subdirectory

To create a subdirectory within an existing directory:

$ CREATE/DIRECTORY [MYDIR.SUBDIR]

This command creates a subdirectory named SUBDIR within MYDIR.

Navigating Directories

To change the current directory, use the SET DEFAULT command:

$ SET DEFAULT [MYDIR]

This command sets MYDIR as the current working directory.

Listing All Directories

To list all directories from the root:

$ DIRECTORY [000000...]

This command lists all directories starting from the root.

Conclusion

Understanding file system and directory management in OpenVMS is essential for efficient system use. By mastering these basic commands and concepts, users can effectively organize and manage their files and directories. Practice these commands to become proficient in navigating and manipulating the OpenVMS file system.
