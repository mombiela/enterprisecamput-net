User Account Management in OpenVMS

Introduction

User account management is a critical aspect of system administration in OpenVMS. This topic will guide you through the essentials of creating, managing, and securing user accounts from a beginner to an advanced level.

Beginner Level

Understanding User Accounts

  • User Account: A user account in OpenVMS is a unique identifier that allows a person to log in and use the system.
  • Username: The name assigned to a user account.
  • Password: A secret word or phrase used to authenticate the user.

Creating a User Account

To create a user account, you need to use the AUTHORIZE utility. Below is a step-by-step guide:

  1. Access the AUTHORIZE Utility:

    $ RUN SYS$SYSTEM:AUTHORIZE

  2. Add a New User:

    UAF> ADD username /PASSWORD=password /UIC=[group,member] /DEVICE=device /DIRECTORY=directory
    • username: The name of the new user.
    • password: The initial password for the user.
    • UIC: User Identification Code, which is unique for each user.
    • device: The disk device where the user's directory will reside.
    • directory: The directory path for the user.

  3. Example:

    UAF> ADD JOHNDOE /PASSWORD=welcome123 /UIC=[200,100] /DEVICE=SYS$SYSDEVICE /DIRECTORY=[JOHNDOE]

Modifying a User Account

To modify an existing user account, use the MODIFY command in the AUTHORIZE utility:

UAF> MODIFY username /parameter=value
  • Example: Change the password for user JOHNDOE: 
    UAF> MODIFY JOHNDOE /PASSWORD=newpassword

Deleting a User Account

To delete a user account, use the REMOVE command:

UAF> REMOVE username
  • Example: Remove user JOHNDOE: 
    UAF> REMOVE JOHNDOE

Intermediate Level

User Account Attributes

  • Privileges: Special rights assigned to a user account to perform specific tasks.
  • Quotas: Limits on the resources a user can consume.
  • Login Flags: Control various aspects of the login process.

Managing Privileges

To assign or modify privileges, use the MODIFY command:

UAF> MODIFY username /PRIVILEGES=(priv1,priv2,...)
  • Example: Grant SYSPRV and OPER privileges to JOHNDOE: 
    UAF> MODIFY JOHNDOE /PRIVILEGES=(SYSPRV,OPER)

Setting Quotas

Quotas control the amount of system resources a user can use:

UAF> MODIFY username /WSQUOTA=value /PGFLQUOTA=value /BYTLM=value
  • Example: Set quotas for JOHNDOE: 
    UAF> MODIFY JOHNDOE /WSQUOTA=1024 /PGFLQUOTA=50000 /BYTLM=100000

Login Flags

Login flags can be set to control user login behavior:

UAF> MODIFY username /FLAGS=(flag1,flag2,...)
  • Example: Set DISUSER flag to disable JOHNDOE's account: 
    UAF> MODIFY JOHNDOE /FLAGS=DISUSER

Advanced Level

Advanced Security Features

  • ACLs (Access Control Lists): Define fine-grained access permissions.
  • Auditing: Monitor and log user activities for security purposes.

Managing ACLs

To manage ACLs, use the SET ACL command:

$ SET ACL /ACL=(IDENTIFIER=username,ACCESS=access) object
  • Example: Grant JOHNDOE read and write access to a file: 
    $ SET ACL /ACL=(IDENTIFIER=JOHNDOE,ACCESS=READ+WRITE) MYFILE.TXT

Configuring Auditing

To enable auditing, modify the system's audit settings:

  1. Edit the Audit Server Configuration:

    $ RUN SYS$SYSTEM:SYSMAN
SYSMAN> SET PROFILE /AUDIT

  2. Enable Auditing for Specific Events:

    $ SET AUDIT /ENABLE=SECURITY /CLASS=LOGIN

Automating User Account Management

  • DCL Scripts: Use Digital Command Language (DCL) scripts to automate repetitive tasks.

  • Example Script: Create a script to add multiple users:

    $! Add multiple users
$ OPEN/READ userfile USERS.DAT
$ LOOP:
$   READ/END=EOF userfile line
$   username = F$ELEMENT(0, " ", line)
$   password = F$ELEMENT(1, " ", line)
$   UIC = F$ELEMENT(2, " ", line)
$   device = F$ELEMENT(3, " ", line)
$   directory = F$ELEMENT(4, " ", line)
$   RUN SYS$SYSTEM:AUTHORIZE
UAF> ADD 'username' /PASSWORD='password' /UIC='UIC' /DEVICE='device' /DIRECTORY='directory'
UAF> EXIT
$   GOTO LOOP
$ EOF:
$ CLOSE userfile
$ EXIT

Conclusion

User account management in OpenVMS involves creating, modifying, and deleting user accounts, managing privileges and quotas, and implementing advanced security features like ACLs and auditing. By mastering these tasks, you can ensure that your OpenVMS system remains secure and efficient.
