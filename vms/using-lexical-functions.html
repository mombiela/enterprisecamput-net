Using Lexical Functions

Lexical functions in OpenVMS are built-in functions that provide a way to retrieve system information, manipulate strings, and perform various other tasks. They are an essential part of DCL (Digital Command Language) scripting and can greatly enhance the functionality and efficiency of your scripts.

Introduction to Lexical Functions

  • Definition: Lexical functions are special functions in OpenVMS that return information about the system, environment, or perform specific operations.
  • Syntax: Lexical functions are typically prefixed with F$ and are used within DCL scripts.

Basic Lexical Functions

Let's start with some basic lexical functions that are commonly used:

  1. F$GETSYI: Retrieves system information.
  2. F$GETJPI: Retrieves job/process information.
  3. F$TRNLNM: Translates logical names.
  4. F$FAO: Formats output.

Example: Using F$GETSYI

The F$GETSYI function is used to get system information. Here's a simple example:

$ SYSTEM_NAME = F$GETSYI("NODENAME")
$ WRITE SYS$OUTPUT "System Name: ''SYSTEM_NAME'"
  • Explanation:
    • F$GETSYI("NODENAME"): Retrieves the node name of the system.
    • WRITE SYS$OUTPUT "System Name: ''SYSTEM_NAME'": Outputs the system name to the console.

String Manipulation Functions

Lexical functions can also be used to manipulate strings. Some common string manipulation functions include:

  1. F$EXTRACT: Extracts a substring from a string.
  2. F$LENGTH: Returns the length of a string.
  3. F$LOCATE: Finds the position of a substring within a string.
  4. F$EDIT: Edits a string according to specified rules.

Example: Using F$EXTRACT and F$LENGTH

$ FULL_NAME = "OpenVMS User"
$ FIRST_NAME = F$EXTRACT(0, 7, FULL_NAME)
$ NAME_LENGTH = F$LENGTH(FULL_NAME)
$ WRITE SYS$OUTPUT "First Name: ''FIRST_NAME'"
$ WRITE SYS$OUTPUT "Name Length: ''NAME_LENGTH'"
  • Explanation:
    • F$EXTRACT(0, 7, FULL_NAME): Extracts the first 7 characters from FULL_NAME.
    • F$LENGTH(FULL_NAME): Returns the length of the string FULL_NAME.
    • WRITE SYS$OUTPUT "First Name: ''FIRST_NAME'": Outputs the first name.
    • WRITE SYS$OUTPUT "Name Length: ''NAME_LENGTH'": Outputs the length of the full name.

Advanced Lexical Functions

For more advanced usage, lexical functions can interact with system processes and logical names.

Example: Using F$GETJPI and F$TRNLNM

$ PROCESS_ID = F$GETJPI("","PID")
$ LOGICAL_NAME = F$TRNLNM("SYS$LOGIN")
$ WRITE SYS$OUTPUT "Process ID: ''PROCESS_ID'"
$ WRITE SYS$OUTPUT "Logical Name SYS$LOGIN: ''LOGICAL_NAME'"
  • Explanation:
    • F$GETJPI("","PID"): Retrieves the process ID of the current process.
    • F$TRNLNM("SYS$LOGIN"): Translates the logical name SYS$LOGIN to its equivalence.
    • WRITE SYS$OUTPUT "Process ID: ''PROCESS_ID'": Outputs the process ID.
    • WRITE SYS$OUTPUT "Logical Name SYS$LOGIN: ''LOGICAL_NAME'": Outputs the translated logical name.

Practical Exercises

  1. Retrieve and Display System Uptime:

    • Use F$GETSYI to retrieve the system uptime and display it.
    $ UPTIME = F$GETSYI("BOOTTIME")
$ WRITE SYS$OUTPUT "System Uptime: ''UPTIME'"

  2. Extract and Display User Information:

    • Use F$GETJPI to retrieve the current username and display it.
    $ USERNAME = F$GETJPI("","USERNAME")
$ WRITE SYS$OUTPUT "Current User: ''USERNAME'"

  3. Translate and Display Logical Names:

    • Use F$TRNLNM to translate a logical name of your choice and display it.
    $ LOGICAL = F$TRNLNM("SYS$SCRATCH")
$ WRITE SYS$OUTPUT "SYS$SCRATCH: ''LOGICAL'"

Conclusion

Lexical functions in OpenVMS are powerful tools that can greatly enhance your DCL scripts. They allow you to retrieve system and process information, manipulate strings, and work with logical names. By mastering these functions, you can write more efficient and effective scripts to automate tasks and manage your OpenVMS environment. Practice using these functions with the provided exercises to solidify your understanding and improve your scripting skills.
