Introduction to Exception Handling

Exception handling in PL/SQL is a critical aspect of writing robust and error-free code. It allows developers to manage errors gracefully and ensure that the program can handle unexpected situations without crashing.

Key Concepts

  • Exception: An error condition during a program execution.
  • Exception Handling: The process of responding to the occurrence of exceptions.
  • Predefined Exceptions: Standard exceptions provided by PL/SQL.
  • User-defined Exceptions: Custom exceptions defined by the programmer.

Types of Exceptions

PL/SQL supports two types of exceptions:

  1. Predefined Exceptions
  2. User-defined Exceptions

Predefined Exceptions

PL/SQL provides several predefined exceptions that are automatically raised by the runtime engine. Some common predefined exceptions include:

| Exception Name | Description | |----------------|-------------| | NO_DATA_FOUND | Raised when a SELECT INTO statement returns no rows. | | TOO_MANY_ROWS | Raised when a SELECT INTO statement returns more than one row. | | ZERO_DIVIDE | Raised when an attempt is made to divide a number by zero. | | INVALID_CURSOR | Raised when an illegal cursor operation is attempted. |

User-defined Exceptions

User-defined exceptions are custom exceptions that you can define in your PL/SQL code. These exceptions are useful for handling specific error conditions that are not covered by predefined exceptions.

Declaring and Raising Exceptions

Declaring User-defined Exceptions

To declare a user-defined exception, you use the EXCEPTION keyword in the declarative section of a PL/SQL block.

DECLARE
  my_custom_exception EXCEPTION;
BEGIN
  -- PL/SQL block
END;

Raising Exceptions

To raise an exception, you use the RAISE statement.

DECLARE
  my_custom_exception EXCEPTION;
BEGIN
  -- Some condition to check
  IF some_condition THEN
    RAISE my_custom_exception;
  END IF;
EXCEPTION
  WHEN my_custom_exception THEN
    DBMS_OUTPUT.PUT_LINE('Custom exception raised.');
END;

Handling Exceptions

Exception Handling Structure

Exception handling in PL/SQL is done using the EXCEPTION block, which follows the BEGIN block.

BEGIN
  -- PL/SQL block
EXCEPTION
  WHEN exception_name1 THEN
    -- Exception handling code
  WHEN exception_name2 THEN
    -- Exception handling code
  WHEN OTHERS THEN
    -- Handling all other exceptions
END;

Example: Handling Predefined Exceptions

DECLARE
  v_number NUMBER;
BEGIN
  v_number := 10 / 0; -- This will raise ZERO_DIVIDE exception
EXCEPTION
  WHEN ZERO_DIVIDE THEN
    DBMS_OUTPUT.PUT_LINE('Cannot divide by zero.');
END;

Example: Handling User-defined Exceptions

DECLARE
  my_custom_exception EXCEPTION;
  v_number NUMBER := 10;
BEGIN
  IF v_number > 5 THEN
    RAISE my_custom_exception;
  END IF;
EXCEPTION
  WHEN my_custom_exception THEN
    DBMS_OUTPUT.PUT_LINE('Custom exception raised: Number is greater than 5.');
END;

Advanced Exception Handling

Propagating Exceptions

Exceptions can be propagated to the calling environment if they are not handled in the current block.

DECLARE
  my_custom_exception EXCEPTION;
  PROCEDURE test_proc IS
  BEGIN
    RAISE my_custom_exception;
  END;
BEGIN
  test_proc;
EXCEPTION
  WHEN my_custom_exception THEN
    DBMS_OUTPUT.PUT_LINE('Exception propagated from procedure.');
END;

Using PRAGMA EXCEPTION_INIT

The PRAGMA EXCEPTION_INIT directive associates a user-defined exception with an Oracle error number.

DECLARE
  my_custom_exception EXCEPTION;
  PRAGMA EXCEPTION_INIT(my_custom_exception, -20001);
BEGIN
  RAISE_APPLICATION_ERROR(-20001, 'Custom error message');
EXCEPTION
  WHEN my_custom_exception THEN
    DBMS_OUTPUT.PUT_LINE('Caught custom exception with error number -20001.');
END;

Conclusion

Exception handling in PL/SQL is essential for creating robust and reliable applications. By understanding and utilizing predefined and user-defined exceptions, you can manage errors effectively and ensure your programs can handle unexpected situations gracefully. Remember to always handle exceptions where appropriate and propagate them when necessary to maintain clean and maintainable code.
