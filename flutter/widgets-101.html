Introduction to Widgets

Widgets are the building blocks of a Flutter application. They describe what their view should look like given their current configuration and state. Widgets are immutable and are used to construct the user interface.

Types of Widgets

Flutter provides a wide range of widgets, but they can be broadly categorized into two types:

  • Stateless Widgets: These widgets do not have any internal state. They are immutable and are rebuilt when their configuration changes.
  • Stateful Widgets: These widgets maintain an internal state that can change over time. They are mutable and can rebuild themselves when their state changes.

Creating a Stateless Widget

A stateless widget is a widget that does not require mutable state. Here is an example of a simple stateless widget:

import 'package:flutter/material.dart';

class MyStatelessWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Stateless Widget Example'),
      ),
      body: Center(
        child: Text('Hello, Flutter!'),
      ),
    );
  }
}

void main() => runApp(MaterialApp(
  home: MyStatelessWidget(),
));

Explanation

  • MyStatelessWidget extends StatelessWidget.
  • The build method describes the part of the user interface represented by the widget.
  • Scaffold provides a framework for the basic material design visual layout structure.
  • AppBar is a material design app bar.
  • Center centers its child widget.
  • Text displays a string of text.

Creating a Stateful Widget

A stateful widget is a widget that has mutable state. Here is an example of a simple stateful widget:

import 'package:flutter/material.dart';

class MyStatefulWidget extends StatefulWidget {
  @override
  _MyStatefulWidgetState createState() => _MyStatefulWidgetState();
}

class _MyStatefulWidgetState extends State<MyStatefulWidget> {
  int _counter = 0;

  void _incrementCounter() {
    setState(() {
      _counter++;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Stateful Widget Example'),
      ),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: <Widget>[
            Text('You have pushed the button this many times:'),
            Text(
              '$_counter',
              style: Theme.of(context).textTheme.headline4,
            ),
          ],
        ),
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: _incrementCounter,
        tooltip: 'Increment',
        child: Icon(Icons.add),
      ),
    );
  }
}

void main() => runApp(MaterialApp(
  home: MyStatefulWidget(),
));

Explanation

  • MyStatefulWidget extends StatefulWidget.
  • _MyStatefulWidgetState is the state class that extends State<MyStatefulWidget>.
  • _counter is a private integer variable that holds the current count.
  • _incrementCounter is a method that increments _counter and calls setState to update the UI.
  • The build method describes the part of the user interface represented by the widget.
  • Column arranges its children in a vertical array.
  • FloatingActionButton is a circular button that floats above the content.

Commonly Used Widgets

Here are some commonly used widgets in Flutter:

  • Container: A convenience widget that combines common painting, positioning, and sizing widgets.
  • Row: Lays out its children in a horizontal array.
  • Column: Lays out its children in a vertical array.
  • Stack: Allows for the overlaying of widgets on top of each other.
  • ListView: A scrollable list of widgets arranged linearly.
  • GridView: A scrollable, 2D array of widgets.

Example of a Container Widget

Container(
  height: 100.0,
  width: 100.0,
  color: Colors.blue,
  child: Center(
    child: Text('Hello'),
  ),
)

Example of a Row Widget

Row(
  mainAxisAlignment: MainAxisAlignment.spaceEvenly,
  children: <Widget>[
    Icon(Icons.star, color: Colors.red),
    Icon(Icons.star, color: Colors.green),
    Icon(Icons.star, color: Colors.blue),
  ],
)

Widget Lifecycle

Understanding the lifecycle of a widget is crucial for managing state and resources effectively.

Stateful Widget Lifecycle

  1. createState: Called when the framework creates a stateful widget.
  2. initState: Called when this object is inserted into the tree.
  3. didChangeDependencies: Called when a dependency of this State object changes.
  4. build: Describes the part of the user interface represented by this widget.
  5. didUpdateWidget: Called whenever the widget configuration changes.
  6. setState: Notifies the framework that the internal state has changed.
  7. deactivate: Called when this object is removed from the tree.
  8. dispose: Called when this object is permanently removed from the tree.

Conclusion

Widgets are the core building blocks of a Flutter application. Understanding the difference between stateless and stateful widgets, as well as the lifecycle of stateful widgets, is essential for building efficient and effective Flutter applications. By mastering these concepts, you can create dynamic and responsive user interfaces with ease.
