Introduction

In this section, we will explore how to handle networking and HTTP requests in Flutter. This is crucial for building apps that need to communicate with web services or APIs. We will cover the basics of making HTTP requests, handling responses, and parsing JSON data. By the end of this section, you will be able to integrate network calls into your Flutter applications.

Key Concepts

  • HTTP Methods: GET, POST, PUT, DELETE
  • Asynchronous Programming: Futures, async, await
  • JSON Parsing: Converting JSON data to Dart objects and vice versa
  • Error Handling: Managing network errors and exceptions

Making HTTP Requests

Using the `http` Package

Flutter provides the http package to handle HTTP requests. First, add the http package to your pubspec.yaml file:

dependencies:
  http: ^0.13.3

Then, import the package in your Dart file:

import 'package:http/http.dart' as http;

GET Request

A GET request is used to retrieve data from a server. Here is an example of making a GET request:

import 'dart:convert';
import 'package:http/http.dart' as http;

Future<void> fetchData() async {
  final response = await http.get(Uri.parse('https://jsonplaceholder.typicode.com/posts'));

  if (response.statusCode == 200) {
    List<dynamic> data = jsonDecode(response.body);
    print(data);
  } else {
    throw Exception('Failed to load data');
  }
}

POST Request

A POST request is used to send data to a server. Here is an example of making a POST request:

import 'dart:convert';
import 'package:http/http.dart' as http;

Future<void> sendData() async {
  final response = await http.post(
    Uri.parse('https://jsonplaceholder.typicode.com/posts'),
    headers: <String, String>{
      'Content-Type': 'application/json; charset=UTF-8',
    },
    body: jsonEncode(<String, String>{
      'title': 'foo',
      'body': 'bar',
      'userId': '1',
    }),
  );

  if (response.statusCode == 201) {
    print('Data sent successfully');
  } else {
    throw Exception('Failed to send data');
  }
}

Parsing JSON Data

When you receive data from a server, it is often in JSON format. You need to parse this JSON data into Dart objects.

Example JSON Data

[
  {
    "userId": 1,
    "id": 1,
    "title": "Sample Title",
    "body": "Sample Body"
  }
]

Parsing JSON in Dart

First, create a Dart class to represent the data:

class Post {
  final int userId;
  final int id;
  final String title;
  final String body;

  Post({required this.userId, required this.id, required this.title, required this.body});

  factory Post.fromJson(Map<String, dynamic> json) {
    return Post(
      userId: json['userId'],
      id: json['id'],
      title: json['title'],
      body: json['body'],
    );
  }
}

Then, parse the JSON data:

Future<List<Post>> fetchPosts() async {
  final response = await http.get(Uri.parse('https://jsonplaceholder.typicode.com/posts'));

  if (response.statusCode == 200) {
    List<dynamic> jsonData = jsonDecode(response.body);
    return jsonData.map((json) => Post.fromJson(json)).toList();
  } else {
    throw Exception('Failed to load posts');
  }
}

Error Handling

Handling errors is crucial for a smooth user experience. Here is an example of how to handle errors in your network requests:

Future<void> fetchDataWithErrorHandling() async {
  try {
    final response = await http.get(Uri.parse('https://jsonplaceholder.typicode.com/posts'));

    if (response.statusCode == 200) {
      List<dynamic> data = jsonDecode(response.body);
      print(data);
    } else {
      print('Server error: ${response.statusCode}');
    }
  } catch (e) {
    print('Network error: $e');
  }
}

Conclusion

In this section, we covered the basics of networking and HTTP requests in Flutter. You learned how to make GET and POST requests, parse JSON data, and handle errors. These skills are essential for building robust Flutter applications that interact with web services. Practice these concepts by integrating them into your projects to solidify your understanding.
