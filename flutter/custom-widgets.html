In this section, we will explore how to create custom widgets in Flutter. Custom widgets are essential for building reusable and maintainable code. By the end of this section, you will understand how to create and use custom widgets, and how to make them flexible and adaptable to different use cases.

Introduction to Custom Widgets

Custom widgets are user-defined widgets that encapsulate a piece of UI and its behavior. They help in breaking down complex UIs into smaller, manageable pieces.

Key Concepts

  • StatelessWidget: A widget that does not maintain any state.
  • StatefulWidget: A widget that maintains state and can change over time.
  • Composition: Building complex widgets by combining simpler ones.

Creating a Stateless Custom Widget

A StatelessWidget is a widget that does not change its state once it is built. It is ideal for static content.

Example: Simple Custom Button

import 'package:flutter/material.dart';

class CustomButton extends StatelessWidget {
  final String text;
  final VoidCallback onPressed;

  CustomButton({required this.text, required this.onPressed});

  @override
  Widget build(BuildContext context) {
    return ElevatedButton(
      onPressed: onPressed,
      child: Text(text),
    );
  }
}

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: Text('Custom Button Example')),
        body: Center(
          child: CustomButton(
            text: 'Click Me',
            onPressed: () {
              print('Button Pressed');
            },
          ),
        ),
      ),
    );
  }
}

Explanation

  • CustomButton Class: This class extends StatelessWidget and takes two parameters: text and onPressed.
  • build Method: Returns an ElevatedButton with the provided text and onPressed callback.

Creating a Stateful Custom Widget

A StatefulWidget is a widget that can change its state during its lifetime. It is useful for dynamic content.

Example: Custom Counter Widget

import 'package:flutter/material.dart';

class CustomCounter extends StatefulWidget {
  @override
  _CustomCounterState createState() => _CustomCounterState();
}

class _CustomCounterState extends State<CustomCounter> {
  int _count = 0;

  void _increment() {
    setState(() {
      _count++;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Column(
      mainAxisAlignment: MainAxisAlignment.center,
      children: [
        Text('Count: $_count', style: TextStyle(fontSize: 24)),
        SizedBox(height: 20),
        ElevatedButton(
          onPressed: _increment,
          child: Text('Increment'),
        ),
      ],
    );
  }
}

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: Text('Custom Counter Example')),
        body: Center(
          child: CustomCounter(),
        ),
      ),
    );
  }
}

Explanation

  • CustomCounter Class: Extends StatefulWidget and creates its state using _CustomCounterState.
  • _CustomCounterState Class: Manages the state of CustomCounter. It includes a _count variable and an _increment method that updates the state.
  • build Method: Returns a Column with a Text widget displaying the count and an ElevatedButton to increment the count.

Making Custom Widgets Flexible

Custom widgets should be flexible and reusable. This can be achieved by using parameters and default values.

Example: Flexible Custom Button

import 'package:flutter/material.dart';

class FlexibleButton extends StatelessWidget {
  final String text;
  final VoidCallback onPressed;
  final Color color;
  final double fontSize;

  FlexibleButton({
    required this.text,
    required this.onPressed,
    this.color = Colors.blue,
    this.fontSize = 16.0,
  });

  @override
  Widget build(BuildContext context) {
    return ElevatedButton(
      style: ElevatedButton.styleFrom(primary: color),
      onPressed: onPressed,
      child: Text(
        text,
        style: TextStyle(fontSize: fontSize),
      ),
    );
  }
}

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: Text('Flexible Button Example')),
        body: Center(
          child: FlexibleButton(
            text: 'Click Me',
            onPressed: () {
              print('Button Pressed');
            },
            color: Colors.red,
            fontSize: 20.0,
          ),
        ),
      ),
    );
  }
}

Explanation

  • FlexibleButton Class: Extends StatelessWidget and includes additional parameters for color and fontSize with default values.
  • build Method: Uses the provided parameters to style the ElevatedButton.

Conclusion

Custom widgets are a powerful feature in Flutter that allows you to create reusable and maintainable UI components. By understanding the difference between StatelessWidget and StatefulWidget, and learning how to make your widgets flexible, you can build complex UIs more efficiently. Practice creating your own custom widgets to become proficient in Flutter development.
