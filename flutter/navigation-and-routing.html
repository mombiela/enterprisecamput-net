Introduction

Navigation and routing are fundamental concepts in mobile app development. In Flutter, these concepts allow you to move between different screens (or pages) in your application. This section will cover the basics of navigation and routing, and gradually move towards more advanced topics.

Basic Navigation

Using Navigator

The Navigator widget is the primary way to manage a stack of routes in Flutter. Each route is a screen or page in your app.

  • Pushing a Route: Adding a new route to the stack.
  • Popping a Route: Removing the current route from the stack.

Example: Pushing and Popping Routes

import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: FirstScreen(),
    );
  }
}

class FirstScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('First Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go to Second Screen'),
          onPressed: () {
            Navigator.push(
              context,
              MaterialPageRoute(builder: (context) => SecondScreen()),
            );
          },
        ),
      ),
    );
  }
}

class SecondScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Second Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go Back'),
          onPressed: () {
            Navigator.pop(context);
          },
        ),
      ),
    );
  }
}

In this example:

  • Navigator.push is used to navigate to the SecondScreen.
  • Navigator.pop is used to return to the FirstScreen.

Named Routes

Named routes provide a way to navigate using string identifiers. This can make your code cleaner and easier to manage, especially in larger applications.

Defining Named Routes

import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      initialRoute: '/',
      routes: {
        '/': (context) => FirstScreen(),
        '/second': (context) => SecondScreen(),
      },
    );
  }
}

class FirstScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('First Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go to Second Screen'),
          onPressed: () {
            Navigator.pushNamed(context, '/second');
          },
        ),
      ),
    );
  }
}

class SecondScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Second Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go Back'),
          onPressed: () {
            Navigator.pop(context);
          },
        ),
      ),
    );
  }
}

In this example:

  • Named routes are defined in the MaterialApp widget.
  • Navigator.pushNamed is used to navigate to the SecondScreen.

Passing Data Between Screens

Often, you need to pass data between screens. This can be done using the Navigator methods.

Example: Passing Data

import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: FirstScreen(),
    );
  }
}

class FirstScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('First Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go to Second Screen'),
          onPressed: () {
            Navigator.push(
              context,
              MaterialPageRoute(
                builder: (context) => SecondScreen(data: 'Hello from FirstScreen'),
              ),
            );
          },
        ),
      ),
    );
  }
}

class SecondScreen extends StatelessWidget {
  final String data;

  SecondScreen({required this.data});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Second Screen'),
      ),
      body: Center(
        child: Text(data),
      ),
    );
  }
}

In this example:

  • Data is passed from FirstScreen to SecondScreen via the constructor.

Advanced Navigation

Using Navigator 2.0

Flutter's Navigator 2.0 provides more control over the navigation stack and is suitable for complex navigation scenarios.

  • Page-based API: Allows for more flexible and declarative navigation.
  • Router: Manages the app's navigation state.

Example: Basic Navigator 2.0 Setup

import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp.router(
      routerDelegate: MyRouterDelegate(),
      routeInformationParser: MyRouteInformationParser(),
    );
  }
}

class MyRouterDelegate extends RouterDelegate with ChangeNotifier, PopNavigatorRouterDelegateMixin {
  final GlobalKey<NavigatorState> navigatorKey;

  MyRouterDelegate() : navigatorKey = GlobalKey<NavigatorState>();

  @override
  Widget build(BuildContext context) {
    return Navigator(
      key: navigatorKey,
      pages: [
        MaterialPage(child: FirstScreen()),
        if (showSecondScreen) MaterialPage(child: SecondScreen()),
      ],
      onPopPage: (route, result) {
        if (!route.didPop(result)) {
          return false;
        }
        showSecondScreen = false;
        notifyListeners();
        return true;
      },
    );
  }

  bool showSecondScreen = false;

  @override
  Future<void> setNewRoutePath(configuration) async {
    if (configuration == '/second') {
      showSecondScreen = true;
    } else {
      showSecondScreen = false;
    }
  }
}

class MyRouteInformationParser extends RouteInformationParser<String> {
  @override
  Future<String> parseRouteInformation(RouteInformation routeInformation) async {
    return routeInformation.location ?? '/';
  }

  @override
  RouteInformation restoreRouteInformation(String configuration) {
    return RouteInformation(location: configuration);
  }
}

class FirstScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('First Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go to Second Screen'),
          onPressed: () {
            (Router.of(context).routerDelegate as MyRouterDelegate).showSecondScreen = true;
            (Router.of(context).routerDelegate as MyRouterDelegate).notifyListeners();
          },
        ),
      ),
    );
  }
}

class SecondScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Second Screen'),
      ),
      body: Center(
        child: ElevatedButton(
          child: Text('Go Back'),
          onPressed: () {
            (Router.of(context).routerDelegate as MyRouterDelegate).showSecondScreen = false;
            (Router.of(context).routerDelegate as MyRouterDelegate).notifyListeners();
          },
        ),
      ),
    );
  }
}

In this example:

  • RouterDelegate and RouteInformationParser are used to manage the navigation state.
  • The Navigator widget is used to build the navigation stack based on the state.

Conclusion

Navigation and routing are essential for building multi-screen applications in Flutter. This topic covered:

  • Basic navigation using Navigator.
  • Named routes for cleaner navigation.
  • Passing data between screens.
  • Advanced navigation using Navigator 2.0.

Understanding these concepts will help you build robust and user-friendly Flutter applications. Practice by implementing different navigation scenarios in your projects to solidify your understanding.
