Introduction

In this exercise, we will learn to implement security policies in AWS using IAM (Identity and Access Management). Security policies are essential for controlling access to cloud resources and ensuring that only authorized users can perform specific actions.

Exercise Objectives

  • Understand the concept of security policies in AWS.
  • Create and assign IAM policies.
  • Apply policies to users, groups, and roles.
  • Verify the effectiveness of the implemented policies.

Prerequisites

  • Active AWS account.
  • Basic knowledge of AWS IAM.
  • Access to the AWS management console.

Steps to Implement Security Policies in AWS

1. Create an IAM Policy

To create an IAM policy, follow these steps:

  1. Log in to the AWS management console.
  2. Navigate to IAM.
  3. In the navigation pane, select Policies.
  4. Click on Create policy.
  5. Use the visual editor or JSON to define the policy.

Example of JSON Policy

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": "s3:ListBucket",
            "Resource": "arn:aws:s3:::example_bucket"
        },
        {
            "Effect": "Allow",
            "Action": [
                "s3:GetObject",
                "s3:PutObject"
            ],
            "Resource": "arn:aws:s3:::example_bucket/*"
        }
    ]
}

This policy allows listing the contents of the example_bucket bucket and getting and putting objects within that bucket.

2. Assign the Policy to a User

  1. In the IAM navigation pane, select Users.
  2. Choose the user to whom you want to assign the policy.
  3. In the Permissions tab, click on Add permissions.
  4. Select Attach policies directly.
  5. Search for and select the policy you created earlier.
  6. Click on Next: Review and then on Add permissions.

3. Verify the Policy

To verify that the policy has been applied correctly:

  1. Log in with the user to whom you assigned the policy.
  2. Try to perform actions allowed and not allowed by the policy.
  3. Verify that only the actions permitted in the policy can be performed.

Practical Exercise

Task 1: Create a Read-Only Policy for S3

  1. Create a policy that allows only read actions (s3:GetObject) on a specific bucket.
  2. Assign this policy to a new user.

Task 2: Create an Administration Policy for EC2

  1. Create a policy that allows all actions (ec2:*) on EC2 resources.
  2. Assign this policy to an administrators group.

Task 3: Verify Policies

  1. Log in with the corresponding users and verify that the applied policies work as expected.
  2. Document any issues encountered and how you resolved them.

Conclusion

In this exercise, we learned to create and assign security policies in AWS using IAM. Security policies are fundamental to protecting cloud resources and ensuring that only authorized users can access them. Practicing the creation and assignment of policies will help you better manage security in your AWS environments.
