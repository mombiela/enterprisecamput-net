Introduction

In this exercise, you will learn how to create a development environment on Heroku, a platform as a service (PaaS) that allows developers to build, run, and operate applications entirely in the cloud. Heroku simplifies the deployment and management process of applications, allowing developers to focus on writing code.

Exercise Objectives

  • Set up an account on Heroku.
  • Install and configure the Heroku CLI.
  • Create and deploy a basic application on Heroku.
  • Manage the development environment on Heroku.

Prerequisites

Before starting, make sure you have:

  • An account on Heroku (you can sign up at Heroku).
  • Git installed on your machine.
  • Node.js and npm installed (for the application example).

Steps to Create a Development Environment on Heroku

1. Setting Up the Account on Heroku

  1. Register on Heroku:

    • Visit Heroku.
    • Click on "Sign Up" and follow the instructions to create an account.

  2. Account Verification:

    • Verify your account through the email sent by Heroku.

2. Installing the Heroku CLI

  1. Download and Installation:

    • Download the Heroku CLI from Heroku CLI.
    • Follow the installation instructions for your operating system.

  2. Installation Verification:

    • Open a terminal and run the following command to verify the installation: 
      heroku --version
    • You should see the installed version of the Heroku CLI.

3. Creating and Deploying a Basic Application

  1. Login to Heroku:

    • In the terminal, run: 
      heroku login
    • Follow the instructions to log in to your Heroku account.

  2. Creating a Node.js Application:

    • Create a new directory for your application and navigate to it: 
      mkdir my-heroku-app
cd my-heroku-app
    • Initialize a new Node.js project: 
      npm init -y
    • Install Express.js: 
      npm install express
    • Create an index.js file with the following content: 
      const express = require('express');
const app = express();
const PORT = process.env.PORT || 3000;

app.get('/', (req, res) => {
  res.send('Hello, Heroku!');
});

app.listen(PORT, () => {
  console.log(`Server is running on port ${PORT}`);
});

  3. Preparing for Deployment:

    • Create a Procfile in the root directory with the following content: 
      web: node index.js
    • Initialize a Git repository and commit: 
      git init
git add .
git commit -m "Initial commit"

  4. Deploying to Heroku:

    • Create a new application on Heroku: 
      heroku create
    • Deploy the application: 
      git push heroku master

  5. Deployment Verification:

    • Once the deployment is complete, open the application in your browser: 
      heroku open

4. Managing the Development Environment

  1. Viewing Application Logs:

    • To view your application's logs, run: 
      heroku logs --tail

  2. Scaling Dynos:

    • To scale the number of dynos (instances of your application), use: 
      heroku ps:scale web=1

  3. Setting Environment Variables:

    • To set environment variables, use: 
      heroku config:set MY_VAR=value

Conclusion

In this exercise, you have learned how to set up a development environment on Heroku, from creating an account to deploying and managing a basic application. Heroku simplifies the deployment and management process of applications, allowing developers to focus on writing code and improving their applications.
