Introduction to Debugging in CL

Debugging is an essential skill for any programmer. In Control Language (CL), understanding how to identify and fix errors can save you a lot of time and frustration. This section will cover various debugging techniques from beginner to advanced levels.

Basic Debugging Techniques

Understanding Error Messages

  • Read Error Messages Carefully: Error messages provide clues about what went wrong. Pay attention to the message text and any codes provided.
  • Common Error Codes: Familiarize yourself with common error codes in CL.

Using Display Commands

  • DSPJOBLOG: Use the DSPJOBLOG command to display the job log, which contains detailed information about the job's execution.
  • DSPMSG: The DSPMSG command displays messages sent to a message queue, which can help identify issues.

Example: Using DSPJOBLOG

DSPJOBLOG JOB(123456/USER/JOBNAME)

Explanation: This command displays the job log for the specified job, helping you identify where the error occurred.

Intermediate Debugging Techniques

Using Breakpoints

  • STRDBG: The STRDBG command starts a debug session.
  • ADDBKP: Use the ADDBKP command to add breakpoints in your CL program.

Example: Setting a Breakpoint

STRDBG PGM(MYPGM)
ADDBKP STMT(10)

Explanation: This starts a debug session for the program MYPGM and adds a breakpoint at statement number 10.

Analyzing Variable Values

  • DSPPGMVAR: The DSPPGMVAR command displays the values of variables in the program.
  • CHGVAR: Use the CHGVAR command to change the value of a variable during debugging.

Example: Displaying and Changing Variable Values

DSPPGMVAR PGM(MYPGM) VAR(&MYVAR)
CHGVAR VAR(&MYVAR) VALUE('NEWVALUE')

Explanation: This displays the value of the variable &MYVAR in the program MYPGM and changes its value to 'NEWVALUE'.

Advanced Debugging Techniques

Using Trace Commands

  • TRCJOB: The TRCJOB command traces the job's execution, providing detailed information about each step.
  • ENDTRC: Use the ENDTRC command to end the trace session.

Example: Tracing a Job

TRCJOB JOB(123456/USER/JOBNAME)
ENDTRC JOB(123456/USER/JOBNAME)

Explanation: This starts and ends a trace session for the specified job, allowing you to analyze its execution in detail.

Debugging Batch Jobs

  • SBMJOB: Submit a job with debugging enabled using the SBMJOB command.
  • WRKSBMJOB: Use the WRKSBMJOB command to work with submitted jobs and monitor their status.

Example: Submitting a Job with Debugging

SBMJOB CMD(CALL PGM(MYPGM)) JOB(MYJOB) JOBD(MYJOBD) JOBQ(MYJOBQ) LOG(4 00 *SECLVL)
WRKSBMJOB JOB(MYJOB)

Explanation: This submits a job with debugging enabled and allows you to monitor its status using the WRKSBMJOB command.

Conclusion

Debugging in CL involves a range of techniques from reading error messages and using display commands to setting breakpoints and tracing job execution. By mastering these techniques, you can efficiently identify and resolve issues in your CL programs, improving your productivity and the reliability of your code.
