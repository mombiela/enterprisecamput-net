Overview

Control Language (CL) is a scripting language used on IBM's midrange systems, such as the AS/400 (now IBM i). CL commands are used to control system operations, manage jobs, and handle data. This section will introduce you to the basics of CL commands, their structure, and how to use them effectively.

Key Concepts

  • Command Structure: CL commands follow a specific syntax and structure.
  • Parameters: Commands often require parameters to specify details.
  • Execution: How to execute CL commands in different environments.
  • Error Handling: Basic error handling in CL commands.

Command Structure

CL commands are structured in a specific way to ensure clarity and consistency. A typical CL command looks like this:

COMMAND PARAMETER1(VALUE1) PARAMETER2(VALUE2)

Example

CRTUSRPRF USRPRF(USER1) PASSWORD(PASS123)
  • CRTUSRPRF: Command to create a user profile.
  • USRPRF: Parameter specifying the user profile name.
  • PASSWORD: Parameter specifying the password for the user profile.

Commonly Used CL Commands

Here are some commonly used CL commands and their purposes:

| Command | Description | |-------------|--------------------------------------------| | CRTUSRPRF | Create a user profile | | DLTUSRPRF | Delete a user profile | | WRKJOB | Work with jobs | | DSPMSG | Display messages | | SBMJOB | Submit a job | | ENDJOB | End a job | | CHGJOB | Change job attributes | | SNDMSG | Send a message |

Example: Creating a User Profile

CRTUSRPRF USRPRF(USER1) PASSWORD(PASS123) TEXT('New User Profile')
  • TEXT: An optional parameter to add a description.

Parameters

Parameters are essential components of CL commands. They provide the necessary details for the command to execute correctly.

Example: Submitting a Job

SBMJOB CMD(CALL PGM(MYPGM)) JOB(MYJOB) JOBQ(QBATCH)
  • CMD: Specifies the command to be executed.
  • JOB: Names the job.
  • JOBQ: Specifies the job queue.

Execution

CL commands can be executed in various environments:

  • Command Line: Directly enter commands in the command line interface.
  • CL Programs: Embed commands within CL programs for batch processing.
  • Job Scheduler: Schedule commands to run at specific times.

Example: Running a Command in a CL Program

PGM
    CRTUSRPRF USRPRF(USER1) PASSWORD(PASS123)
    SBMJOB CMD(CALL PGM(MYPGM)) JOB(MYJOB) JOBQ(QBATCH)
ENDPGM

Error Handling

Error handling in CL commands is crucial for robust script execution. Common techniques include:

  • Monitor Message (MONMSG): Capture and handle errors.
  • Condition Handling: Use conditions to manage different scenarios.

Example: Using MONMSG

PGM
    CRTUSRPRF USRPRF(USER1) PASSWORD(PASS123)
    MONMSG MSGID(CPF0000) EXEC(DO)
        /* Error handling code */
    ENDDO
ENDPGM
  • MONMSG MSGID(CPF0000): Captures any error message starting with CPF.

Conclusion

This introduction to CL commands provides a foundation for understanding how to use CL to manage and control IBM i systems. By mastering the structure, parameters, execution methods, and error handling techniques, you can effectively automate and streamline system operations. As you progress through this course, you will delve deeper into more advanced topics and complex scenarios, enhancing your proficiency in CL commands.
