Interacting with Other Languages
Performance Optimization ►

In this section, we will explore how Control Language (CL) can interact with other programming languages. This is a crucial skill for leveraging the strengths of different languages within a single application or system.

Key Concepts

  • Interoperability: The ability of CL to work with other programming languages.
  • Calling Programs: How to call programs written in other languages from a CL program.
  • Data Exchange: Methods for exchanging data between CL and other languages.
  • Error Handling: Managing errors when interacting with other languages.

Calling Programs from CL

One of the primary ways CL interacts with other languages is by calling programs written in those languages. This is often done using the CALL command.

Example: Calling an RPG Program

PGM
    DCL VAR(&RESULT) TYPE(*CHAR) LEN(10)
    CALL PGM(MYLIB/MYPGM) PARM(&RESULT)
    /* Process the result */
    SNDPGMMSG MSG('Result from RPG program: ' *CAT &RESULT)
ENDPGM
  • Explanation:
    • DCL VAR(&RESULT) TYPE(*CHAR) LEN(10): Declares a variable to hold the result.
    • CALL PGM(MYLIB/MYPGM) PARM(&RESULT): Calls an RPG program and passes the &RESULT variable.
    • SNDPGMMSG MSG('Result from RPG program: ' *CAT &RESULT): Sends a program message with the result.

Example: Calling a COBOL Program

PGM
    DCL VAR(&STATUS) TYPE(*CHAR) LEN(1)
    CALL PGM(MYLIB/COBOLPGM) PARM(&STATUS)
    IF COND(&STATUS *EQ '0') THEN(DO)
        SNDPGMMSG MSG('COBOL program completed successfully.')
    ENDDO
    ELSE DO
        SNDPGMMSG MSG('COBOL program failed.')
    ENDDO
ENDPGM
  • Explanation:
    • DCL VAR(&STATUS) TYPE(*CHAR) LEN(1): Declares a variable to hold the status.
    • CALL PGM(MYLIB/COBOLPGM) PARM(&STATUS): Calls a COBOL program and passes the &STATUS variable.
    • IF COND(&STATUS *EQ '0') THEN(DO): Checks if the status is '0' (success).
    • SNDPGMMSG MSG('COBOL program completed successfully.'): Sends a success message.
    • ELSE DO: Handles the failure case.
    • SNDPGMMSG MSG('COBOL program failed.'): Sends a failure message.

Data Exchange

Exchanging data between CL and other languages can be done through parameters, data areas, or files.

Using Parameters

Parameters are the most straightforward way to exchange data.

PGM
    DCL VAR(&INPUT) TYPE(*CHAR) LEN(20)
    DCL VAR(&OUTPUT) TYPE(*CHAR) LEN(20)
    CHGVAR VAR(&INPUT) VALUE('Hello, World!')
    CALL PGM(MYLIB/OTHERPGM) PARM(&INPUT &OUTPUT)
    SNDPGMMSG MSG('Output from other program: ' *CAT &OUTPUT)
ENDPGM
  • Explanation:
    • DCL VAR(&INPUT) TYPE(*CHAR) LEN(20): Declares an input variable.
    • DCL VAR(&OUTPUT) TYPE(*CHAR) LEN(20): Declares an output variable.
    • CHGVAR VAR(&INPUT) VALUE('Hello, World!'): Sets the input variable.
    • CALL PGM(MYLIB/OTHERPGM) PARM(&INPUT &OUTPUT): Calls another program with input and output parameters.
    • SNDPGMMSG MSG('Output from other program: ' *CAT &OUTPUT): Sends a message with the output.

Using Data Areas

Data areas can be used for more persistent data exchange.

PGM
    DCL VAR(&DATA) TYPE(*CHAR) LEN(50)
    RTVDTAARA DTAARA(MYLIB/MYDATA) RTNVAR(&DATA)
    CALL PGM(MYLIB/OTHERPGM)
    CHGDTAARA DTAARA(MYLIB/MYDATA) VALUE(&DATA)
ENDPGM
  • Explanation:
    • DCL VAR(&DATA) TYPE(*CHAR) LEN(50): Declares a variable for data area content.
    • RTVDTAARA DTAARA(MYLIB/MYDATA) RTNVAR(&DATA): Retrieves data from a data area.
    • CALL PGM(MYLIB/OTHERPGM): Calls another program.
    • CHGDTAARA DTAARA(MYLIB/MYDATA) VALUE(&DATA): Changes the data area content.

Error Handling

Proper error handling is essential when interacting with other languages to ensure robustness.

PGM
    DCL VAR(&STATUS) TYPE(*CHAR) LEN(1)
    MONMSG MSGID(CPF0000) EXEC(DO)
        SNDPGMMSG MSG('An error occurred while calling the program.')
        RETURN
    ENDDO
    CALL PGM(MYLIB/OTHERPGM) PARM(&STATUS)
    IF COND(&STATUS *NE '0') THEN(DO)
        SNDPGMMSG MSG('Other program returned an error.')
    ENDDO
ENDPGM
  • Explanation:
    • MONMSG MSGID(CPF0000) EXEC(DO): Monitors for any error messages.
    • SNDPGMMSG MSG('An error occurred while calling the program.'): Sends an error message.
    • RETURN: Exits the program.
    • IF COND(&STATUS *NE '0') THEN(DO): Checks if the status indicates an error.
    • SNDPGMMSG MSG('Other program returned an error.'): Sends an error message.

Conclusion

Interacting with other languages from CL is a powerful feature that allows you to leverage the strengths of different programming environments. By understanding how to call programs, exchange data, and handle errors, you can create robust and flexible applications. Practice these techniques with various languages to become proficient in multi-language interoperability.
