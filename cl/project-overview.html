Introduction to Control Language (CL)

Control Language (CL) is a scripting language used on IBM's midrange systems, such as the AS/400 (now known as IBM i). It is primarily used for system administration tasks, job control, and automation of repetitive tasks. This course will take you from a beginner to an advanced level in CL, covering essential concepts, practical examples, and advanced techniques.

Key Concepts

  • Commands: The basic building blocks of CL, used to perform specific tasks.
  • Variables: Used to store data that can be manipulated within a CL program.
  • Control Structures: Includes loops and conditional statements to control the flow of the program.
  • File Handling: Techniques to read from and write to files.
  • Error Handling: Methods to manage and respond to errors in CL programs.
  • Subroutines: Reusable blocks of code that can be called from multiple places within a program.

Beginner Level

Basic Commands

CL commands are used to perform a wide range of tasks. Here are some basic commands:

DSPJOB
  • DSPJOB: Displays the status of a job.
WRKACTJOB
  • WRKACTJOB: Displays a list of active jobs.

Variables

Variables in CL are defined using the DCL command:

DCL VAR(&MYVAR) TYPE(*CHAR) LEN(10)
  • DCL: Declares a variable.
  • VAR: Specifies the variable name.
  • TYPE: Specifies the data type (e.g., *CHAR for character).
  • LEN: Specifies the length of the variable.

Control Structures

Conditional Statements

IF COND(&MYVAR *EQ 'HELLO') THEN(DO)
    /* Commands to execute if condition is true */
ENDDO
  • IF: Starts a conditional statement.
  • COND: Specifies the condition to evaluate.
  • THEN: Specifies the commands to execute if the condition is true.
  • DO/ENDDO: Marks the beginning and end of a block of commands.

Loops

DOFOR VAR(&I) FROM(1) TO(10)
    /* Commands to execute in the loop */
ENDDO
  • DOFOR: Starts a loop that iterates a specified number of times.
  • VAR: Specifies the loop variable.
  • FROM/TO: Specifies the range for the loop variable.

Intermediate Level

File Handling

Reading from a File

DCLF FILE(MYFILE)
RCVF
  • DCLF: Declares a file for use in the program.
  • RCVF: Reads a record from the file into the declared variables.

Writing to a File

SNDF FILE(MYFILE) RCDFMT(MYRCDFMT)
  • SNDF: Sends a record to the file.
  • FILE: Specifies the file name.
  • RCDFMT: Specifies the record format.

Error Handling

MONMSG MSGID(CPF0000) EXEC(DO)
    /* Commands to execute if an error occurs */
ENDDO
  • MONMSG: Monitors for specific messages (errors).
  • MSGID: Specifies the message ID to monitor.
  • EXEC: Specifies the commands to execute if the message is received.

Advanced Level

Subroutines

Subroutines allow you to create reusable blocks of code:

CALLSUBR SUBR(MYSUBR)
SUBR MYSUBR
    /* Commands in the subroutine */
ENDSUBR
  • CALLSUBR: Calls a subroutine.
  • SUBR: Defines the start of a subroutine.
  • ENDSUBR: Marks the end of a subroutine.

Advanced File Handling

Using Data Structures

DCLF FILE(MYFILE)
DCL DS MYDS
RCVF
  • DS: Declares a data structure.
  • MYDS: Specifies the data structure name.

Conclusion

This course has provided a comprehensive overview of Control Language (CL) from basic commands and variables to advanced file handling and subroutines. By mastering these concepts, you will be well-equipped to automate tasks, manage jobs, and handle files efficiently on IBM's midrange systems. Continue practicing and exploring more advanced topics to deepen your understanding and proficiency in CL.
