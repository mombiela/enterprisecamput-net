Introduction

In this section, we will explore how design patterns are applied in real-world situations. We will analyze specific cases and see how design patterns can solve common problems in software development. This practical approach will help you better understand the utility and implementation of these patterns.

Case 1: Order Management System

Context

An e-commerce company needs a system to manage customer orders. The system must be flexible and scalable to handle different types of orders and payment methods.

Problem

The current system is monolithic and difficult to maintain. Each time a new type of order or payment method is added, significant code rewriting is required.

Solution: Strategy Design Pattern

The Strategy design pattern allows you to define a family of algorithms, encapsulate each one, and make them interchangeable. This pattern allows the algorithm to vary independently of the clients that use it.

Implementation

  1. Define the strategy interface:

    public interface PaymentStrategy {
    void pay(int amount);
}

  2. Implement different payment strategies:

    public class CreditCardPayment implements PaymentStrategy {
    @Override
    public void pay(int amount) {
        System.out.println("Paid " + amount + " using Credit Card.");
    }
}

public class PayPalPayment implements PaymentStrategy {
    @Override
    public void pay(int amount) {
        System.out.println("Paid " + amount + " using PayPal.");
    }
}

  3. Context that uses the strategy:

    public class Order {
    private PaymentStrategy paymentStrategy;

    public void setPaymentStrategy(PaymentStrategy paymentStrategy) {
        this.paymentStrategy = paymentStrategy;
    }

    public void processOrder(int amount) {
        paymentStrategy.pay(amount);
    }
}

  4. Using the Strategy pattern:

    public class Main {
    public static void main(String[] args) {
        Order order = new Order();

        // Pay with credit card
        order.setPaymentStrategy(new CreditCardPayment());
        order.processOrder(100);

        // Pay with PayPal
        order.setPaymentStrategy(new PayPalPayment());
        order.processOrder(200);
    }
}

Benefits

  • Flexibility: Easy to add new payment methods without modifying existing code.
  • Maintainability: Clear separation of responsibilities.

Case 2: Notification System

Context

A messaging application needs to send notifications to users through different channels such as email, SMS, and push notifications.

Problem

The code to send notifications is scattered and duplicated in various parts of the application, making it difficult to maintain and scale.

Solution: Observer Design Pattern

The Observer pattern defines a one-to-many dependency between objects so that when one object changes state, all its dependents are notified and updated automatically.

Implementation

  1. Define the observer interface:

    public interface Observer {
    void update(String message);
}

  2. Implement different observers:

    public class EmailNotifier implements Observer {
    @Override
    public void update(String message) {
        System.out.println("Email notification: " + message);
    }
}

public class SMSNotifier implements Observer {
    @Override
    public void update(String message) {
        System.out.println("SMS notification: " + message);
    }
}

public class PushNotifier implements Observer {
    @Override
    public void update(String message) {
        System.out.println("Push notification: " + message);
    }
}

  3. Define the subject that notifies observers:

    import java.util.ArrayList;
import java.util.List;

public class NotificationService {
    private List<Observer> observers = new ArrayList<>();

    public void addObserver(Observer observer) {
        observers.add(observer);
    }

    public void removeObserver(Observer observer) {
        observers.remove(observer);
    }

    public void notifyObservers(String message) {
        for (Observer observer : observers) {
            observer.update(message);
        }
    }
}

  4. Using the Observer pattern:

    public class Main {
    public static void main(String[] args) {
        NotificationService notificationService = new NotificationService();

        Observer emailNotifier = new EmailNotifier();
        Observer smsNotifier = new SMSNotifier();
        Observer pushNotifier = new PushNotifier();

        notificationService.addObserver(emailNotifier);
        notificationService.addObserver(smsNotifier);
        notificationService.addObserver(pushNotifier);

        notificationService.notifyObservers("New message received!");
    }
}

Benefits

  • Decoupling: Observers are decoupled from the subject.
  • Scalability: Easy to add new types of notifications.

Conclusion

The study of real cases shows us how design patterns can solve common problems in software development. By applying patterns like Strategy and Observer, we can create more flexible, maintainable, and scalable systems. These practical examples illustrate the importance of design patterns and how they can be implemented in real-world situations.
