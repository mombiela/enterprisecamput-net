Introduction

The Bridge Pattern is a structural design pattern that decouples an abstraction from its implementation, allowing both to vary independently. This pattern is useful when you want to avoid an exponential growth of classes due to the combination of multiple abstraction and implementation variants.

Key Concepts

  • Abstraction: The high-level interface that defines the operations.
  • Implementation: The interface that defines the primitive operations.
  • Refined Abstraction: An extension of the abstraction that adds more operations.
  • Concrete Implementation: A specific implementation of the implementation interface.

Structure of the Bridge Pattern

The Bridge Pattern consists of the following components:

  • Abstraction: Defines the interface and maintains a reference to an object of type Implementation.
  • Implementation: Defines the interface for all concrete implementations.
  • Refined Abstraction: Extends the Abstraction interface.
  • Concrete Implementation: Implements the Implementation interface.

UML Diagram

| Component | Description | |--------------------------|-----------------------------------------------------------------------------| | Abstraction | Defines the interface and maintains a reference to an object of type Implementor. | | Implementor | Defines the interface for all concrete implementations. | | RefinedAbstraction | Extends the Abstraction interface. | | ConcreteImplementor | Implements the Implementor interface. |

Code Example

Below is a Python example to illustrate the Bridge Pattern:

# Implementor
class DrawingAPI:
    def draw_circle(self, x, y, radius):
        pass

# ConcreteImplementor 1
class DrawingAPI1(DrawingAPI):
    def draw_circle(self, x, y, radius):
        print(f"API1.circle at {x}:{y} radius {radius}")

# ConcreteImplementor 2
class DrawingAPI2(DrawingAPI):
    def draw_circle(self, x, y, radius):
        print(f"API2.circle at {x}:{y} radius {radius}")

# Abstraction
class Shape:
    def __init__(self, drawing_api):
        self.drawing_api = drawing_api

    def draw(self):
        pass

    def resize_by_percentage(self, pct):
        pass

# RefinedAbstraction
class CircleShape(Shape):
    def __init__(self, x, y, radius, drawing_api):
        super().__init__(drawing_api)
        self.x = x
        self.y = y
        self.radius = radius

    def draw(self):
        self.drawing_api.draw_circle(self.x, self.y, self.radius)

    def resize_by_percentage(self, pct):
        self.radius *= (1 + pct / 100)

# Using the Bridge Pattern
circle1 = CircleShape(1, 2, 3, DrawingAPI1())
circle2 = CircleShape(5, 7, 11, DrawingAPI2())

circle1.draw()
circle2.draw()

circle1.resize_by_percentage(50)
circle1.draw()

Code Explanation

  • DrawingAPI: Is the implementation interface that defines the draw_circle method.
  • DrawingAPI1 and DrawingAPI2: Are concrete implementations of DrawingAPI that provide different ways to draw a circle.
  • Shape: Is the abstraction that maintains a reference to an object of type DrawingAPI.
  • CircleShape: Is an extension of Shape that adds specific operations for a circle, such as draw and resize_by_percentage.

Conclusion

The Bridge Pattern is an effective solution to decouple an abstraction from its implementation, allowing both to evolve independently. This is especially useful in complex systems where the combinations of abstraction and implementation can grow exponentially. By applying this pattern, the flexibility and maintainability of the code are improved.
