Introduction

The Factory Method pattern is one of the most important and widely used creational design patterns in software development. Its main objective is to define an interface for creating an object, but it allows subclasses to alter the type of objects that will be created.

Key Concepts

  • Purpose: Provide an interface for creating objects in a superclass, while allowing subclasses to modify the type of objects that will be created.
  • Applicability: It is used when a class cannot anticipate the type of objects it must create or when a class wants its subclasses to specify the objects it creates.
  • Participants:
    • Product: Defines the interface of the objects that the factory method creates.
    • ConcreteProduct: Implements the Product interface.
    • Creator: Declares the factory method, which returns an object of type Product.
    • ConcreteCreator: Overrides the factory method to return an instance of ConcreteProduct.

Code Example

Below is an example in Python to illustrate the Factory Method pattern:

from abc import ABC, abstractmethod

# Product
class Button(ABC):
    @abstractmethod
    def render(self):
        pass

# ConcreteProduct
class WindowsButton(Button):
    def render(self):
        return "Render a button in Windows style"

class HTMLButton(Button):
    def render(self):
        return "Render a button in HTML style"

# Creator
class Dialog(ABC):
    @abstractmethod
    def create_button(self) -> Button:
        pass

    def render(self):
        button = self.create_button()
        return button.render()

# ConcreteCreator
class WindowsDialog(Dialog):
    def create_button(self) -> Button:
        return WindowsButton()

class WebDialog(Dialog):
    def create_button(self) -> Button:
        return HTMLButton()

# Client
def client_code(dialog: Dialog):
    print(dialog.render())

# Usage
if __name__ == "__main__":
    dialog = WindowsDialog()
    client_code(dialog)

    dialog = WebDialog()
    client_code(dialog)

Code Explanation

  • Product: Button is an abstract interface that defines the render method.
  • ConcreteProduct: WindowsButton and HTMLButton are concrete implementations of Button.
  • Creator: Dialog is an abstract class that declares the factory method create_button.
  • ConcreteCreator: WindowsDialog and WebDialog are subclasses of Dialog that implement the factory method to create instances of WindowsButton and HTMLButton, respectively.
  • Client: The client_code function uses the render method of Dialog to display the created button.

Comparison with Other Creational Patterns

| Pattern | Purpose | Usage Example | |-----------------------|---------------------------------------------------------------------------|-------------------------------------------------------------------------------| | Factory Method | Define an interface for creating an object, but let subclasses decide which class to instantiate. | Create different types of buttons (Windows, HTML) without changing client code. | | Abstract Factory | Provide an interface for creating families of related or dependent objects without specifying their concrete classes. | Create families of products (buttons, checkboxes) for different platforms (Windows, Mac). | | Singleton | Ensure that a class has only one instance and provide a global point of access to it. | Manage a single instance of a configuration class throughout the application. |

Conclusion

The Factory Method pattern is a powerful tool for object creation in a software system. It allows subclasses to decide which objects to create, promoting code flexibility and extensibility. By understanding and applying this pattern, developers can design more robust and maintainable systems.
