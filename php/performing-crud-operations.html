Performing CRUD Operations
Using Prepared Statements ►

CRUD stands for Create, Read, Update, and Delete. These are the four basic operations for managing data in a database. In this section, we will learn how to perform these operations using PHP and MySQL.

Prerequisites

Before diving into CRUD operations, ensure you have:

  • A basic understanding of PHP syntax.
  • A local server environment (e.g., XAMPP, WAMP, or MAMP).
  • MySQL installed and running.
  • A database and table to work with.

Setting Up the Database

First, let's create a database and a table to perform our CRUD operations.

CREATE DATABASE crud_example;
USE crud_example;

CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) NOT NULL UNIQUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Connecting to the Database

To interact with the database, we need to establish a connection using PHP.

<?php
$servername = "localhost";
$username = "root";
$password = "";
$dbname = "crud_example";

// Create connection
$conn = new mysqli($servername, $username, $password, $dbname);

// Check connection
if ($conn->connect_error) {
    die("Connection failed: " . $conn->connect_error);
}
echo "Connected successfully";
?>

Create Operation

To insert data into the database, we use the INSERT INTO SQL statement.

<?php
$name = "John Doe";
$email = "john.doe@example.com";

$sql = "INSERT INTO users (name, email) VALUES ('$name', '$email')";

if ($conn->query($sql) === TRUE) {
    echo "New record created successfully";
} else {
    echo "Error: " . $sql . "<br>" . $conn->error;
}
?>

Explanation

  • $name and $email are the data we want to insert.
  • The INSERT INTO statement specifies the table (users) and the columns (name, email) where the data will be inserted.
  • $conn->query($sql) executes the SQL statement.

Read Operation

To retrieve data from the database, we use the SELECT SQL statement.

<?php
$sql = "SELECT id, name, email, created_at FROM users";
$result = $conn->query($sql);

if ($result->num_rows > 0) {
    // Output data of each row
    while($row = $result->fetch_assoc()) {
        echo "id: " . $row["id"]. " - Name: " . $row["name"]. " - Email: " . $row["email"]. " - Created At: " . $row["created_at"]. "<br>";
    }
} else {
    echo "0 results";
}
?>

Explanation

  • The SELECT statement specifies the columns we want to retrieve.
  • $result = $conn->query($sql) executes the SQL statement.
  • $result->num_rows checks if there are any rows returned.
  • while($row = $result->fetch_assoc()) loops through the result set and fetches each row as an associative array.

Update Operation

To update existing data, we use the UPDATE SQL statement.

<?php
$id = 1;
$newEmail = "new.email@example.com";

$sql = "UPDATE users SET email='$newEmail' WHERE id=$id";

if ($conn->query($sql) === TRUE) {
    echo "Record updated successfully";
} else {
    echo "Error updating record: " . $conn->error;
}
?>

Explanation

  • $id is the identifier of the record we want to update.
  • $newEmail is the new value for the email column.
  • The UPDATE statement specifies the table (users), the new value (email='$newEmail'), and the condition (WHERE id=$id).

Delete Operation

To delete data from the database, we use the DELETE SQL statement.

<?php
$id = 1;

$sql = "DELETE FROM users WHERE id=$id";

if ($conn->query($sql) === TRUE) {
    echo "Record deleted successfully";
} else {
    echo "Error deleting record: " . $conn->error;
}
?>

Explanation

  • $id is the identifier of the record we want to delete.
  • The DELETE statement specifies the table (users) and the condition (WHERE id=$id).

Conclusion

In this section, we covered the basics of performing CRUD operations using PHP and MySQL:

  • Create: Inserting new records into the database.
  • Read: Retrieving records from the database.
  • Update: Modifying existing records in the database.
  • Delete: Removing records from the database.

Understanding these operations is fundamental for managing data in any web application. Practice these examples and experiment with different queries to deepen your understanding.
