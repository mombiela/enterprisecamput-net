Overview

Object-Oriented Programming (OOP) is a programming paradigm that uses objects and classes to structure software in a way that is modular, reusable, and easy to maintain. In this section, we will cover the fundamental concepts of OOP in C++, including classes, objects, inheritance, polymorphism, encapsulation, and abstraction.

Key Concepts

Classes and Objects

  • Class: A blueprint for creating objects. It defines a datatype by bundling data and methods that work on the data.
  • Object: An instance of a class. It is a self-contained component that contains properties and methods needed to make a certain type of data useful.

Example:

#include <iostream>
using namespace std;

class Car {
public:
    string brand;
    string model;
    int year;

    void displayInfo() {
        cout << "Brand: " << brand << ", Model: " << model << ", Year: " << year << endl;
    }
};

int main() {
    Car car1;
    car1.brand = "Toyota";
    car1.model = "Corolla";
    car1.year = 2020;
    car1.displayInfo();

    return 0;
}

Explanation:

  • We define a class Car with three public attributes: brand, model, and year.
  • We also define a method displayInfo to print the car's details.
  • In the main function, we create an object car1 of the class Car and set its attributes.

Encapsulation

  • Encapsulation: The concept of wrapping data and methods that operate on the data within a single unit (class). It restricts direct access to some of the object's components, which is a means of preventing accidental interference and misuse of the data.

Example:

#include <iostream>
using namespace std;

class Person {
private:
    string name;
    int age;

public:
    void setName(string n) {
        name = n;
    }

    string getName() {
        return name;
    }

    void setAge(int a) {
        if (a > 0) {
            age = a;
        }
    }

    int getAge() {
        return age;
    }
};

int main() {
    Person person1;
    person1.setName("Alice");
    person1.setAge(30);
    cout << "Name: " << person1.getName() << ", Age: " << person1.getAge() << endl;

    return 0;
}

Explanation:

  • The class Person has private attributes name and age.
  • Public methods setName, getName, setAge, and getAge are used to access and modify the private attributes.

Inheritance

  • Inheritance: A mechanism by which one class can inherit properties and methods from another class. It promotes code reusability.

Example:

#include <iostream>
using namespace std;

class Animal {
public:
    void eat() {
        cout << "This animal is eating." << endl;
    }
};

class Dog : public Animal {
public:
    void bark() {
        cout << "The dog is barking." << endl;
    }
};

int main() {
    Dog dog1;
    dog1.eat();  // Inherited method
    dog1.bark(); // Dog's own method

    return 0;
}

Explanation:

  • The class Animal has a method eat.
  • The class Dog inherits from Animal and adds its own method bark.
  • The Dog object dog1 can use both the inherited eat method and its own bark method.

Polymorphism

  • Polymorphism: The ability of different classes to be treated as instances of the same class through inheritance. It allows methods to do different things based on the object it is acting upon.

Example:

#include <iostream>
using namespace std;

class Animal {
public:
    virtual void makeSound() {
        cout << "Some generic animal sound." << endl;
    }
};

class Dog : public Animal {
public:
    void makeSound() override {
        cout << "Woof!" << endl;
    }
};

class Cat : public Animal {
public:
    void makeSound() override {
        cout << "Meow!" << endl;
    }
};

int main() {
    Animal* animal1 = new Dog();
    Animal* animal2 = new Cat();

    animal1->makeSound(); // Outputs: Woof!
    animal2->makeSound(); // Outputs: Meow!

    delete animal1;
    delete animal2;

    return 0;
}

Explanation:

  • The base class Animal has a virtual method makeSound.
  • The derived classes Dog and Cat override the makeSound method.
  • Pointers to Animal can point to Dog or Cat objects, and the correct makeSound method is called based on the actual object type.

Abstraction

  • Abstraction: The concept of hiding the complex implementation details and showing only the necessary features of an object. It helps in reducing programming complexity and effort.

Example:

#include <iostream>
using namespace std;

class AbstractDevice {
public:
    virtual void turnOn() = 0; // Pure virtual function
    virtual void turnOff() = 0; // Pure virtual function
};

class Fan : public AbstractDevice {
public:
    void turnOn() override {
        cout << "Fan is now ON." << endl;
    }

    void turnOff() override {
        cout << "Fan is now OFF." << endl;
    }
};

int main() {
    Fan fan1;
    fan1.turnOn();
    fan1.turnOff();

    return 0;
}

Explanation:

  • AbstractDevice is an abstract class with pure virtual functions turnOn and turnOff.
  • The Fan class inherits from AbstractDevice and provides implementations for the pure virtual functions.

Summary

In this section, we introduced the fundamental concepts of Object-Oriented Programming (OOP) in C++. We covered:

  • Classes and Objects: The basic building blocks of OOP.
  • Encapsulation: Protecting data by restricting access to it.
  • Inheritance: Reusing code by inheriting properties and methods from existing classes.
  • Polymorphism: Allowing methods to behave differently based on the object they are acting upon.
  • Abstraction: Simplifying complex systems by showing only the necessary details.

Understanding these concepts is crucial for writing modular, reusable, and maintainable code in C++. As you progress through the course, you will see how these principles are applied in more complex scenarios.
