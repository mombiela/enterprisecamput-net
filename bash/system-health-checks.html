System Health Checks
Custom Command Line Tools ►

System health checks are essential for maintaining the performance and reliability of your systems. In this topic, we will explore how to perform system health checks using Bash scripts. This will include checking CPU usage, memory usage, disk space, and network status.

Key Concepts

  • CPU Usage: Monitoring the percentage of CPU being used.
  • Memory Usage: Checking the amount of RAM being used and available.
  • Disk Space: Ensuring there is sufficient disk space available.
  • Network Status: Verifying network connectivity and performance.

CPU Usage

To monitor CPU usage, you can use the top or mpstat commands. Here, we will use mpstat as it provides a more straightforward output for scripting.

Example

#!/bin/bash

# Check CPU usage
cpu_usage=$(mpstat | grep 'all' | awk '{print 100 - $12}')
echo "CPU Usage: $cpu_usage%"

Explanation

  • mpstat: Displays CPU usage.
  • grep 'all': Filters the output to show the line with overall CPU usage.
  • awk '{print 100 - $12}': Calculates the CPU usage percentage by subtracting the idle percentage from 100.

Memory Usage

To check memory usage, you can use the free command.

Example

#!/bin/bash

# Check memory usage
memory_usage=$(free | grep Mem | awk '{print $3/$2 * 100.0}')
echo "Memory Usage: $memory_usage%"

Explanation

  • free: Displays memory usage.
  • grep Mem: Filters the output to show the line with memory information.
  • awk '{print $3/$2 * 100.0}': Calculates the memory usage percentage.

Disk Space

To check disk space, you can use the df command.

Example

#!/bin/bash

# Check disk space
disk_usage=$(df -h / | grep / | awk '{print $5}')
echo "Disk Usage: $disk_usage"

Explanation

  • df -h /: Displays disk space usage for the root filesystem in a human-readable format.
  • grep /: Filters the output to show the line with root filesystem information.
  • awk '{print $5}': Extracts the disk usage percentage.

Network Status

To verify network connectivity, you can use the ping command.

Example

#!/bin/bash

# Check network status
ping -c 4 google.com > /dev/null 2>&1

if [ $? -eq 0 ]; then
  echo "Network Status: Connected"
else
  echo "Network Status: Disconnected"
fi

Explanation

  • ping -c 4 google.com: Sends 4 ping requests to google.com.
  • > /dev/null 2>&1: Redirects the output to /dev/null to suppress it.
  • if [ $? -eq 0 ]; then: Checks the exit status of the ping command. If it is 0, the network is connected; otherwise, it is disconnected.

Summary

In this topic, we covered how to perform basic system health checks using Bash scripts. We learned how to monitor CPU usage, memory usage, disk space, and network status. These checks are crucial for maintaining the health and performance of your systems. By automating these checks with Bash scripts, you can ensure that your systems are running smoothly and efficiently.
