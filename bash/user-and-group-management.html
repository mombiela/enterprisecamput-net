Introduction

User and group management is a fundamental aspect of system administration in Unix-like operating systems, including those that use Bash. Proper management ensures that users have the appropriate permissions and access to resources, enhancing both security and efficiency.

Key Concepts

  • User: An individual account that can log in and perform tasks on the system.
  • Group: A collection of users that can share permissions and access to resources.
  • UID (User ID): A unique identifier assigned to each user.
  • GID (Group ID): A unique identifier assigned to each group.
  • Home Directory: A personal directory for each user where they can store files.
  • Shell: The command-line interface used by the user.

Managing Users

Adding a User

To add a new user, use the useradd command. This command creates a new user account and sets up the home directory and shell.

sudo useradd -m -s /bin/bash newuser
  • -m: Creates a home directory for the user.
  • -s: Specifies the shell for the user.

Setting a Password

After creating a user, you need to set a password using the passwd command.

sudo passwd newuser

You will be prompted to enter and confirm the new password.

Modifying a User

To modify an existing user, use the usermod command. For example, to change the user's shell:

sudo usermod -s /bin/zsh newuser

Deleting a User

To delete a user and their home directory, use the userdel command with the -r option.

sudo userdel -r newuser

Managing Groups

Adding a Group

To add a new group, use the groupadd command.

sudo groupadd newgroup

Adding a User to a Group

To add a user to a group, use the usermod command with the -aG option.

sudo usermod -aG newgroup newuser
  • -aG: Appends the user to the specified group without removing them from other groups.

Viewing Group Membership

To view the groups a user belongs to, use the groups command.

groups newuser

Deleting a Group

To delete a group, use the groupdel command.

sudo groupdel newgroup

File Permissions and Ownership

Changing Ownership

To change the ownership of a file or directory, use the chown command.

sudo chown newuser:newgroup filename
  • newuser:newgroup: Specifies the new owner and group.

Changing Permissions

To change the permissions of a file or directory, use the chmod command.

chmod 755 filename
  • 755: Sets the permissions (rwxr-xr-x).

Understanding Permissions

Permissions are represented by a three-digit number, where each digit can be a value from 0 to 7:

  • r (read) = 4
  • w (write) = 2
  • x (execute) = 1

For example, 755 means:

  • Owner: rwx (4+2+1 = 7)
  • Group: r-x (4+0+1 = 5)
  • Others: r-x (4+0+1 = 5)

Practical Exercises

Exercise 1: Create a New User

  1. Create a new user named student.
  2. Set a password for student.
  3. Verify the creation by listing all users.

Exercise 2: Manage Groups

  1. Create a new group named developers.
  2. Add the student user to the developers group.
  3. Verify the group membership.

Exercise 3: File Permissions

  1. Create a file named project.txt.
  2. Change the ownership of project.txt to student:developers.
  3. Set the permissions of project.txt to 644.

Conclusion

User and group management in Bash is essential for maintaining a secure and efficient system. By understanding how to add, modify, and delete users and groups, as well as manage file permissions and ownership, you can effectively control access to system resources. Practice these commands and concepts to become proficient in managing users and groups in a Unix-like environment.
