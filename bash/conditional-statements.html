Conditional Statements in Bash
Loops: for, while, until ►

Introduction

Conditional statements are fundamental in any programming language, allowing you to execute different code blocks based on certain conditions. In Bash, conditional statements help control the flow of your scripts, making them more dynamic and responsive to different inputs or states.

Key Concepts

  • if-else statements: Basic conditional structure.
  • elif statements: Adding multiple conditions.
  • case statements: Handling multiple conditions more efficiently.
  • test command: Used to evaluate conditions.
  • Logical operators: Combine multiple conditions.

if-else Statements

The if-else statement allows you to execute a block of code if a condition is true, and another block if it is false.

Syntax

if [ condition ]; then
    # Code to execute if condition is true
else
    # Code to execute if condition is false
fi

Example

#!/bin/bash

read -p "Enter a number: " number

if [ $number -gt 10 ]; then
    echo "The number is greater than 10."
else
    echo "The number is 10 or less."
fi

In this example, the script reads a number from the user and checks if it is greater than 10.

elif Statements

The elif statement allows you to check multiple conditions sequentially.

Syntax

if [ condition1 ]; then
    # Code to execute if condition1 is true
elif [ condition2 ]; then
    # Code to execute if condition2 is true
else
    # Code to execute if none of the conditions are true
fi

Example

#!/bin/bash

read -p "Enter a number: " number

if [ $number -gt 10 ]; then
    echo "The number is greater than 10."
elif [ $number -eq 10 ]; then
    echo "The number is exactly 10."
else
    echo "The number is less than 10."
fi

This script checks if the number is greater than, equal to, or less than 10.

case Statements

The case statement is useful for handling multiple conditions more efficiently, especially when you have many possible values to check.

Syntax

case $variable in
    pattern1)
        # Code to execute if variable matches pattern1
        ;;
    pattern2)
        # Code to execute if variable matches pattern2
        ;;
    *)
        # Code to execute if variable doesn't match any pattern
        ;;
esac

Example

#!/bin/bash

read -p "Enter a character: " char

case $char in
    [a-z])
        echo "You entered a lowercase letter."
        ;;
    [A-Z])
        echo "You entered an uppercase letter."
        ;;
    [0-9])
        echo "You entered a digit."
        ;;
    *)
        echo "You entered a special character."
        ;;
esac

This script categorizes the input character as a lowercase letter, uppercase letter, digit, or special character.

test Command

The test command (or [ ] syntax) is used to evaluate conditions.

Common Test Expressions

  • String comparisons:
    • [ "$str1" = "$str2" ]: True if strings are equal.
    • [ "$str1" != "$str2" ]: True if strings are not equal.
  • Numeric comparisons:
    • [ $num1 -eq $num2 ]: True if numbers are equal.
    • [ $num1 -ne $num2 ]: True if numbers are not equal.
    • [ $num1 -gt $num2 ]: True if num1 is greater than num2.
    • [ $num1 -lt $num2 ]: True if num1 is less than num2.
  • File tests:
    • [ -e $file ]: True if file exists.
    • [ -d $directory ]: True if directory exists.
    • [ -f $file ]: True if file is a regular file.

Example

#!/bin/bash

read -p "Enter a filename: " filename

if [ -e $filename ]; then
    echo "The file exists."
else
    echo "The file does not exist."
fi

This script checks if a file exists.

Logical Operators

Logical operators allow you to combine multiple conditions.

Common Logical Operators

  • AND (&&): Both conditions must be true.
  • OR (||): At least one condition must be true.
  • NOT (!): Negates the condition.

Example

#!/bin/bash

read -p "Enter a number: " number

if [ $number -gt 0 ] && [ $number -lt 100 ]; then
    echo "The number is between 1 and 99."
else
    echo "The number is not in the range 1-99."
fi

This script checks if the number is between 1 and 99.

Conclusion

Conditional statements in Bash are essential for controlling the flow of your scripts. By mastering if-else, elif, case, and the test command, along with logical operators, you can create more dynamic and responsive scripts. Practice these concepts with various examples to become proficient in writing Bash scripts that handle different conditions effectively.
