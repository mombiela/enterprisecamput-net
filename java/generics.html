Generics in Java
Collections Framework ►

Generics in Java allow you to create classes, interfaces, and methods with a placeholder for the type of data they operate on. This provides stronger type checks at compile time and eliminates the need for type casting.

Introduction to Generics

  • Definition: Generics enable types (classes and interfaces) to be parameters when defining classes, interfaces, and methods.
  • Purpose: They provide a way to re-use the same code with different inputs, and they provide stronger type checks at compile time.

Basic Syntax

// A generic class example
public class Box<T> {
    private T t;

    public void set(T t) {
        this.t = t;
    }

    public T get() {
        return t;
    }
}

In this example, T is a type parameter that will be replaced with a concrete type when an instance of Box is created.

Benefits of Using Generics

  • Type Safety: Ensures that you get compile-time checking and avoid ClassCastException.
  • Code Reusability: Write a method or class once and use it for any type.
  • Elimination of Type Casting: Reduces the need for explicit type casting.

Example Without Generics

List list = new ArrayList();
list.add("Hello");
String s = (String) list.get(0); // Needs casting

Example With Generics

List<String> list = new ArrayList<>();
list.add("Hello");
String s = list.get(0); // No casting needed

Generic Methods

You can also define generic methods in Java.

Syntax

public static <T> void printArray(T[] array) {
    for (T element : array) {
        System.out.println(element);
    }
}

Here, <T> before the return type void indicates that the method is generic.

Example Usage

Integer[] intArray = {1, 2, 3, 4, 5};
String[] stringArray = {"Hello", "World"};

printArray(intArray); // Prints integers
printArray(stringArray); // Prints strings

Bounded Type Parameters

Sometimes you want to restrict the types that can be used as type arguments. This is done using bounded type parameters.

Upper Bound

public <T extends Number> void printNumbers(T[] numbers) {
    for (T number : numbers) {
        System.out.println(number);
    }
}

In this example, T can only be a type that extends Number.

Lower Bound

public static void addNumbers(List<? super Integer> list) {
    list.add(1);
    list.add(2);
}

Here, ? super Integer means any type that is a supertype of Integer.

Wildcards in Generics

Wildcards are used to specify an unknown type.

Unbounded Wildcard

public void printList(List<?> list) {
    for (Object element : list) {
        System.out.println(element);
    }
}

<?> represents an unknown type.

Bounded Wildcards

  • Upper Bounded Wildcard: <? extends Number>
  • Lower Bounded Wildcard: <? super Integer>

Example

public void processElements(List<? extends Number> list) {
    for (Number number : list) {
        System.out.println(number);
    }
}

Generic Classes

You can create classes that can work with any type.

Example

public class Pair<K, V> {
    private K key;
    private V value;

    public Pair(K key, V value) {
        this.key = key;
        this.value = value;
    }

    public K getKey() {
        return key;
    }

    public V getValue() {
        return value;
    }
}

Here, K and V are type parameters.

Conclusion

Generics in Java provide a powerful and flexible way to write type-safe and reusable code. By understanding and utilizing generics, you can create more robust and maintainable applications. Remember to use bounded type parameters and wildcards to further enhance the flexibility and safety of your generic code.
