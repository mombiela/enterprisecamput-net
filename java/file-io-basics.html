Introduction

File I/O (Input/Output) is a fundamental concept in Java programming that allows programs to read from and write to files. This section will cover the basics of file handling in Java, including reading from files, writing to files, and handling exceptions.

Key Concepts

  • File Class: Represents a file or directory path.
  • FileReader and FileWriter: Classes for reading and writing character files.
  • BufferedReader and BufferedWriter: Classes for efficient reading and writing.
  • FileInputStream and FileOutputStream: Classes for reading and writing binary files.
  • Exception Handling: Managing errors that occur during file operations.

Working with the File Class

The File class in Java is used to represent file and directory pathnames. It provides methods to create, delete, and check properties of files and directories.

Example: Creating a File Object

import java.io.File;

public class FileExample {
    public static void main(String[] args) {
        File file = new File("example.txt");
        
        if (file.exists()) {
            System.out.println("File exists.");
        } else {
            System.out.println("File does not exist.");
        }
    }
}

In this example, we create a File object representing a file named "example.txt" and check if it exists.

Reading from a File

To read from a file, you can use FileReader and BufferedReader for character files, or FileInputStream for binary files.

Example: Reading a Text File

import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;

public class ReadFileExample {
    public static void main(String[] args) {
        try (BufferedReader br = new BufferedReader(new FileReader("example.txt"))) {
            String line;
            while ((line = br.readLine()) != null) {
                System.out.println(line);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

In this example, BufferedReader is used to read the file line by line. The try-with-resources statement ensures that the file is closed automatically.

Writing to a File

To write to a file, you can use FileWriter and BufferedWriter for character files, or FileOutputStream for binary files.

Example: Writing to a Text File

import java.io.BufferedWriter;
import java.io.FileWriter;
import java.io.IOException;

public class WriteFileExample {
    public static void main(String[] args) {
        try (BufferedWriter bw = new BufferedWriter(new FileWriter("example.txt"))) {
            bw.write("Hello, World!");
            bw.newLine();
            bw.write("This is a test.");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

In this example, BufferedWriter is used to write text to the file. The try-with-resources statement ensures that the file is closed automatically.

Handling Exceptions

File I/O operations can throw various exceptions, such as FileNotFoundException and IOException. Proper exception handling is crucial to ensure that your program can handle errors gracefully.

Example: Handling Exceptions

import java.io.File;
import java.io.FileNotFoundException;
import java.io.FileReader;
import java.io.IOException;

public class ExceptionHandlingExample {
    public static void main(String[] args) {
        File file = new File("nonexistent.txt");
        try (FileReader fr = new FileReader(file)) {
            // Read file content
        } catch (FileNotFoundException e) {
            System.out.println("File not found: " + file.getName());
        } catch (IOException e) {
            System.out.println("I/O error occurred: " + e.getMessage());
        }
    }
}

In this example, we handle FileNotFoundException and IOException separately to provide specific error messages.

Summary

In this section, we covered the basics of File I/O in Java, including:

  • Creating and using the File class.
  • Reading from and writing to text files using FileReader, FileWriter, BufferedReader, and BufferedWriter.
  • Handling exceptions that may occur during file operations.

Understanding these concepts is essential for working with files in Java, and they form the foundation for more advanced file handling techniques.
