Introduction to Concurrency in Java

Concurrency in Java allows multiple threads to run simultaneously, making programs more efficient and responsive. Java provides a rich set of concurrency utilities in the java.util.concurrent package to help developers manage threads and synchronization.

Key Concepts

  • Thread: The smallest unit of processing. Java provides the Thread class and the Runnable interface to create and manage threads.
  • Executor Framework: A higher-level replacement for working with threads directly. It provides a pool of threads to execute tasks.
  • Synchronization: Mechanisms to control the access of multiple threads to shared resources.
  • Locks: More flexible and powerful than synchronized blocks for controlling access to shared resources.
  • Atomic Variables: Variables that support lock-free thread-safe programming on single variables.
  • Concurrent Collections: Thread-safe versions of standard collections like List, Map, and Queue.

Creating and Managing Threads

Basic Thread Creation

public class BasicThreadExample {
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            System.out.println("Hello from a thread!");
        });
        thread.start();
    }
}
  • Explanation: This example creates a new thread using a lambda expression and starts it. The start() method begins the execution of the thread.

Using Runnable Interface

public class RunnableExample implements Runnable {
    @Override
    public void run() {
        System.out.println("Hello from a Runnable!");
    }

    public static void main(String[] args) {
        Thread thread = new Thread(new RunnableExample());
        thread.start();
    }
}
  • Explanation: This example implements the Runnable interface and passes an instance of RunnableExample to a new Thread object.

Executor Framework

Using Executors

import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class ExecutorExample {
    public static void main(String[] args) {
        ExecutorService executor = Executors.newFixedThreadPool(2);
        executor.submit(() -> System.out.println("Task 1"));
        executor.submit(() -> System.out.println("Task 2"));
        executor.shutdown();
    }
}
  • Explanation: This example uses the ExecutorService to manage a pool of threads. The submit method is used to execute tasks, and shutdown is called to stop the executor from accepting new tasks.

Synchronization

Synchronized Methods and Blocks

public class SynchronizedExample {
    private int count = 0;

    public synchronized void increment() {
        count++;
    }

    public static void main(String[] args) {
        SynchronizedExample example = new SynchronizedExample();
        Thread t1 = new Thread(example::increment);
        Thread t2 = new Thread(example::increment);
        t1.start();
        t2.start();
    }
}
  • Explanation: The increment method is synchronized, ensuring that only one thread can execute it at a time.

Locks

Using ReentrantLock

import java.util.concurrent.locks.Lock;
import java.util.concurrent.locks.ReentrantLock;

public class LockExample {
    private final Lock lock = new ReentrantLock();
    private int count = 0;

    public void increment() {
        lock.lock();
        try {
            count++;
        } finally {
            lock.unlock();
        }
    }

    public static void main(String[] args) {
        LockExample example = new LockExample();
        Thread t1 = new Thread(example::increment);
        Thread t2 = new Thread(example::increment);
        t1.start();
        t2.start();
    }
}
  • Explanation: This example uses ReentrantLock to control access to the increment method. The lock method is called before the critical section, and unlock is called in a finally block to ensure it is always executed.

Atomic Variables

Using AtomicInteger

import java.util.concurrent.atomic.AtomicInteger;

public class AtomicExample {
    private final AtomicInteger count = new AtomicInteger(0);

    public void increment() {
        count.incrementAndGet();
    }

    public static void main(String[] args) {
        AtomicExample example = new AtomicExample();
        Thread t1 = new Thread(example::increment);
        Thread t2 = new Thread(example::increment);
        t1.start();
        t2.start();
    }
}
  • Explanation: This example uses AtomicInteger to perform atomic operations on an integer variable without using explicit synchronization.

Concurrent Collections

Using ConcurrentHashMap

import java.util.concurrent.ConcurrentHashMap;
import java.util.Map;

public class ConcurrentMapExample {
    public static void main(String[] args) {
        Map<String, Integer> map = new ConcurrentHashMap<>();
        map.put("key1", 1);
        map.put("key2", 2);

        Thread t1 = new Thread(() -> map.put("key3", 3));
        Thread t2 = new Thread(() -> map.put("key4", 4));
        t1.start();
        t2.start();
    }
}
  • Explanation: This example uses ConcurrentHashMap to perform thread-safe operations on a map.

Conclusion

Concurrency in Java is a powerful feature that allows developers to write efficient and responsive applications. By understanding and utilizing the concurrency utilities provided by Java, such as threads, the Executor framework, synchronization, locks, atomic variables, and concurrent collections, developers can effectively manage concurrent tasks and ensure thread safety in their applications.
