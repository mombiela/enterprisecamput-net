Introduction to Servlets

Servlets are Java programs that run on a web server, handling client requests and generating dynamic web content. They are a key component of Java EE (Enterprise Edition) and are used to create web applications.

Key Concepts

  • Servlet Container: The environment where servlets run, managing their lifecycle and providing necessary services.
  • HTTP Protocol: The protocol used by servlets to communicate with clients.
  • Request and Response: The mechanism by which clients send requests to the server and receive responses.

Setting Up Your Environment

Before diving into servlets, ensure you have the following set up:

  • Java Development Kit (JDK): Required to compile and run Java programs.
  • Apache Tomcat: A popular servlet container.
  • IDE: Integrated Development Environment like Eclipse or IntelliJ IDEA for easier development.

Creating Your First Servlet

Step-by-Step Guide

  1. Create a Dynamic Web Project in your IDE.
  2. Add a Servlet Class: 
    import java.io.IOException;
import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

@WebServlet("/hello")
public class HelloWorldServlet extends HttpServlet {
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        response.setContentType("text/html");
        response.getWriter().println("<h1>Hello, World!</h1>");
    }
}
  3. Deploy and Run: Deploy your project to the servlet container (e.g., Tomcat) and run it. Access the servlet via http://localhost:8080/your_project_name/hello.

Explanation

  • Annotations: @WebServlet("/hello") maps the servlet to the URL pattern /hello.
  • doGet Method: Handles GET requests from the client. response.setContentType("text/html") sets the response type, and response.getWriter().println("<h1>Hello, World!</h1>") sends an HTML response.

Handling HTTP Requests

Servlets can handle various types of HTTP requests, such as GET, POST, PUT, DELETE, etc.

Handling POST Requests

protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
    String name = request.getParameter("name");
    response.setContentType("text/html");
    response.getWriter().println("<h1>Hello, " + name + "!</h1>");
}
  • doPost Method: Handles POST requests. request.getParameter("name") retrieves form data sent by the client.

Session Management

Sessions allow servlets to maintain state across multiple requests from the same client.

Using HttpSession

protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
    HttpSession session = request.getSession();
    session.setAttribute("username", "JohnDoe");
    response.getWriter().println("Session created with username: JohnDoe");
}
  • HttpSession: request.getSession() retrieves the current session or creates a new one. session.setAttribute("username", "JohnDoe") stores data in the session.

Advanced Topics

Filters

Filters are used to preprocess or postprocess requests and responses.

import java.io.IOException;
import javax.servlet.Filter;
import javax.servlet.FilterChain;
import javax.servlet.FilterConfig;
import javax.servlet.ServletException;
import javax.servlet.annotation.WebFilter;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

@WebFilter("/hello")
public class LoggingFilter implements Filter {
    public void doFilter(HttpServletRequest request, HttpServletResponse response, FilterChain chain) throws IOException, ServletException {
        System.out.println("Request received at " + new java.util.Date());
        chain.doFilter(request, response);
    }
}
  • doFilter Method: Intercepts the request and response. chain.doFilter(request, response) passes control to the next filter or servlet.

Listeners

Listeners are used to monitor events in a web application.

import javax.servlet.ServletContextEvent;
import javax.servlet.ServletContextListener;
import javax.servlet.annotation.WebListener;

@WebListener
public class AppContextListener implements ServletContextListener {
    public void contextInitialized(ServletContextEvent sce) {
        System.out.println("Application started");
    }

    public void contextDestroyed(ServletContextEvent sce) {
        System.out.println("Application stopped");
    }
}
  • ServletContextListener: contextInitialized and contextDestroyed methods handle application startup and shutdown events.

Conclusion

Servlets are a powerful tool for building dynamic web applications in Java. By understanding the basics of servlets, handling HTTP requests, managing sessions, and utilizing advanced features like filters and listeners, you can create robust and scalable web applications. Practice by creating various servlets and experimenting with different features to deepen your understanding.
