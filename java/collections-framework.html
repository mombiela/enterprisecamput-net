Introduction to Collections Framework

The Java Collections Framework provides a set of interfaces and classes to handle a group of objects as a single unit. It is a part of the java.util package and includes various data structures like lists, sets, and maps.

  • Key Concepts:
    • Collection Interface: The root interface for all collection classes.
    • List Interface: An ordered collection (also known as a sequence).
    • Set Interface: A collection that does not allow duplicate elements.
    • Map Interface: An object that maps keys to values, with no duplicate keys allowed.

Collection Interface

The Collection interface is the root interface of the Java Collections Framework. It provides basic methods that all collections must implement.

  • Common Methods:
    • add(E e): Adds an element to the collection.
    • remove(Object o): Removes a specified element from the collection.
    • size(): Returns the number of elements in the collection.
    • clear(): Removes all elements from the collection.
    • isEmpty(): Checks if the collection is empty.
Collection<String> collection = new ArrayList<>();
collection.add("Hello");
collection.add("World");
System.out.println(collection.size()); // Output: 2

List Interface

The List interface extends the Collection interface and represents an ordered collection. Elements can be accessed by their integer index.

  • Implementations:

    • ArrayList: A resizable array implementation.
    • LinkedList: A doubly-linked list implementation.

  • Common Methods:

    • get(int index): Returns the element at the specified position.
    • set(int index, E element): Replaces the element at the specified position.
    • add(int index, E element): Inserts the element at the specified position.
    • remove(int index): Removes the element at the specified position.
List<String> list = new ArrayList<>();
list.add("Apple");
list.add("Banana");
list.add(1, "Cherry");
System.out.println(list.get(1)); // Output: Cherry

Set Interface

The Set interface extends the Collection interface and represents a collection that does not allow duplicate elements.

  • Implementations:

    • HashSet: A hash table implementation.
    • LinkedHashSet: A hash table and linked list implementation.
    • TreeSet: A Red-Black tree implementation.

  • Common Methods:

    • add(E e): Adds the specified element if it is not already present.
    • remove(Object o): Removes the specified element if it is present.
Set<String> set = new HashSet<>();
set.add("One");
set.add("Two");
set.add("One"); // Duplicate element, will not be added
System.out.println(set.size()); // Output: 2

Map Interface

The Map interface does not extend the Collection interface. It represents a collection of key-value pairs.

  • Implementations:

    • HashMap: A hash table implementation.
    • LinkedHashMap: A hash table and linked list implementation.
    • TreeMap: A Red-Black tree implementation.

  • Common Methods:

    • put(K key, V value): Associates the specified value with the specified key.
    • get(Object key): Returns the value associated with the specified key.
    • remove(Object key): Removes the mapping for the specified key.
    • keySet(): Returns a set view of the keys.
    • values(): Returns a collection view of the values.
Map<String, Integer> map = new HashMap<>();
map.put("A", 1);
map.put("B", 2);
System.out.println(map.get("A")); // Output: 1
map.remove("A");
System.out.println(map.size()); // Output: 1

Comparison of List, Set, and Map

| Feature | List | Set | Map | |------------------|--------------------|--------------------|--------------------| | Allows Duplicates| Yes | No | Keys: No, Values: Yes | | Ordered | Yes | No (except LinkedHashSet) | No (except LinkedHashMap) | | Access by Index | Yes | No | No | | Key-Value Pair | No | No | Yes |

Conclusion

The Java Collections Framework is a powerful and flexible tool for managing groups of objects. Understanding the different interfaces and their implementations allows you to choose the right data structure for your needs. Practice using these collections to become proficient in handling data in Java.
