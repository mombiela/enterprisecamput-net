Introduction to CI/CD

Continuous Integration (CI) and Continuous Deployment (CD) are essential practices in modern software development. CI/CD pipelines automate the process of integrating code changes, running tests, and deploying applications, ensuring faster and more reliable software delivery.

Key Concepts of CI/CD

  • Continuous Integration (CI): The practice of automatically integrating code changes from multiple contributors into a shared repository several times a day.
  • Continuous Deployment (CD): The practice of automatically deploying every code change that passes the automated tests to production.
  • Pipeline: A set of automated processes that drive software development through various stages, from code integration to deployment.
  • Build: The process of compiling source code into executable code.
  • Test: Automated tests to verify the correctness of the code.
  • Deploy: The process of delivering the application to a production environment.

Setting Up CI/CD on Azure

Azure provides several tools and services to implement CI/CD pipelines, including Azure DevOps and GitHub Actions.

Azure DevOps

Azure DevOps is a suite of development tools that support the entire software development lifecycle.

Creating a CI Pipeline

  1. Create a new project in Azure DevOps.
  2. Set up a repository:
    • Navigate to Repos > Files.
    • Initialize the repository with your code.
  3. Define a build pipeline:
    • Navigate to Pipelines > Builds.
    • Click on "New pipeline" and select your repository.
    • Choose a template or start with an empty job.
    • Define the build steps in the YAML file.
# azure-pipelines.yml
trigger:
- main

pool:
  vmImage: 'ubuntu-latest'

steps:
- task: UseDotNet@2
  inputs:
    packageType: 'sdk'
    version: '5.x'
    installationPath: $(Agent.ToolsDirectory)/dotnet

- script: dotnet build --configuration Release
  displayName: 'Build project'

- script: dotnet test --no-build --verbosity normal
  displayName: 'Run tests'

Creating a CD Pipeline

  1. Define a release pipeline:
    • Navigate to Pipelines > Releases.
    • Click on "New pipeline" and select a template.
    • Define the stages (e.g., Dev, QA, Production).
  2. Add artifacts:
    • Link the build pipeline as an artifact source.
  3. Define deployment tasks:
    • Add tasks to deploy the application to the desired environment (e.g., Azure App Service).
# Example deployment task in YAML
- task: AzureRmWebAppDeployment@4
  inputs:
    azureSubscription: 'your-azure-subscription'
    appType: 'webApp'
    WebAppName: 'your-app-service-name'
    packageForLinux: '$(System.DefaultWorkingDirectory)/**/*.zip'

GitHub Actions

GitHub Actions is an automation tool integrated with GitHub that allows you to create CI/CD workflows.

Creating a CI Workflow

  1. Create a .github/workflows directory in your repository.
  2. Add a workflow file:
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2
    - name: Set up .NET
      uses: actions/setup-dotnet@v1
      with:
        dotnet-version: '5.x'
    - name: Build with dotnet
      run: dotnet build --configuration Release
    - name: Test with dotnet
      run: dotnet test --no-build --verbosity normal

Creating a CD Workflow

  1. Add a deployment workflow file:
# .github/workflows/cd.yml
name: CD

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2
    - name: Set up .NET
      uses: actions/setup-dotnet@v1
      with:
        dotnet-version: '5.x'
    - name: Deploy to Azure Web App
      uses: azure/webapps-deploy@v2
      with:
        app-name: 'your-app-service-name'
        publish-profile: ${{ secrets.AZURE_WEBAPP_PUBLISH_PROFILE }}
        package: .

Advanced CI/CD Practices

  • Environment Variables and Secrets: Securely manage sensitive information using Azure Key Vault or GitHub Secrets.
  • Multi-Stage Pipelines: Implement pipelines with multiple stages for different environments (e.g., Dev, QA, Production).
  • Monitoring and Alerts: Integrate monitoring tools like Azure Monitor to track the health of your deployments and set up alerts for failures.
  • Rollback Strategies: Implement strategies to rollback deployments in case of failures.

Conclusion

Implementing CI/CD pipelines on Azure using Azure DevOps or GitHub Actions can significantly enhance your software development process by automating code integration, testing, and deployment. By following best practices and leveraging Azure's robust toolset, you can achieve faster and more reliable software delivery.
