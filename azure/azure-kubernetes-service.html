Introduction to Azure Kubernetes Service (AKS)

Azure Kubernetes Service (AKS) is a managed container orchestration service based on the open-source Kubernetes system. AKS simplifies deploying a managed Kubernetes cluster in Azure by offloading the operational overhead to Azure. This section will cover the basics of AKS, its benefits, and key concepts.

  • What is Kubernetes?

    • An open-source platform for automating deployment, scaling, and operations of application containers.
    • Manages containerized applications across multiple hosts.
    • Provides container-centric infrastructure.

  • What is AKS?

    • A managed Kubernetes service provided by Azure.
    • Simplifies Kubernetes cluster management.
    • Integrates with Azure services for monitoring, scaling, and security.

  • Benefits of AKS:

    • Managed Service: Azure handles the Kubernetes control plane.
    • Scalability: Easily scale applications up or down.
    • Integration: Seamless integration with other Azure services.
    • Security: Built-in security features and compliance with Azure standards.

Setting Up AKS

This section will guide you through setting up an AKS cluster from scratch.

Prerequisites

  • An active Azure subscription.
  • Azure CLI installed on your local machine.
  • Basic understanding of Kubernetes concepts.

Steps to Create an AKS Cluster

  1. Install Azure CLI:

    curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash

  2. Login to Azure:

    az login

  3. Create a Resource Group:

    az group create --name myResourceGroup --location eastus

  4. Create an AKS Cluster:

    az aks create --resource-group myResourceGroup --name myAKSCluster --node-count 1 --enable-addons monitoring --generate-ssh-keys

  5. Get AKS Credentials:

    az aks get-credentials --resource-group myResourceGroup --name myAKSCluster

  6. Verify the Cluster:

    kubectl get nodes

Deploying Applications on AKS

This section will cover how to deploy applications on your AKS cluster using Kubernetes manifests.

Creating a Deployment

  • Sample Deployment YAML:

    apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 2
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.14.2
        ports:
        - containerPort: 80

  • Apply the Deployment:

    kubectl apply -f nginx-deployment.yaml

  • Verify the Deployment:

    kubectl get deployments
kubectl get pods

Exposing the Application

  • Service YAML:

    apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  type: LoadBalancer
  selector:
    app: nginx
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80

  • Apply the Service:

    kubectl apply -f nginx-service.yaml

  • Get the External IP:

    kubectl get services

Advanced AKS Features

This section will delve into more advanced features of AKS, such as scaling, monitoring, and security.

Scaling Applications

  • Horizontal Pod Autoscaler (HPA):

    apiVersion: autoscaling/v1
kind: HorizontalPodAutoscaler
metadata:
  name: nginx-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: nginx-deployment
  minReplicas: 1
  maxReplicas: 10
  targetCPUUtilizationPercentage: 50

  • Apply HPA:

    kubectl apply -f nginx-hpa.yaml

Monitoring and Logging

  • Azure Monitor for Containers:

    • Integrates with AKS to provide monitoring and logging.
    • Automatically enabled if specified during cluster creation.

  • Viewing Metrics:

    • Use Azure Portal to view metrics and logs.
    • Use kubectl top to view resource usage.

Security Best Practices

  • Role-Based Access Control (RBAC):

    • Use RBAC to control access to the Kubernetes API.
    • Define roles and role bindings.

  • Network Policies:

    • Use network policies to control traffic between pods.
    • Example Network Policy YAML: 
      apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-nginx
spec:
  podSelector:
    matchLabels:
      app: nginx
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          app: nginx
  egress:
  - to:
    - podSelector:
        matchLabels:
          app: nginx

Conclusion

Azure Kubernetes Service (AKS) is a powerful tool for managing containerized applications at scale. By leveraging AKS, you can simplify the deployment, management, and scaling of Kubernetes clusters. This course has covered the basics of AKS, setting up a cluster, deploying applications, and advanced features like scaling, monitoring, and security. With this knowledge, you are well-equipped to start using AKS for your container orchestration needs.
