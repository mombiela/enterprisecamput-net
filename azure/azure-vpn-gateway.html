Introduction

Azure VPN Gateway is a critical component in Azure networking that allows you to securely connect your on-premises networks to Azure resources. This section will cover the basics of Azure VPN Gateway, its types, configurations, and advanced features.

Key Concepts

  • Virtual Network Gateway: A virtual network gateway is composed of two or more VMs that are automatically configured and deployed to a specific subnet called the gateway subnet.
  • Gateway Subnet: A specific subnet in your virtual network that contains the virtual network gateway resources.
  • VPN Gateway Types: There are two types of VPN gateways: Route-based and Policy-based.
  • Connection Types: Site-to-Site (S2S), Point-to-Site (P2S), and VNet-to-VNet connections.

Setting Up a VPN Gateway

Prerequisites

  • An Azure subscription.
  • A virtual network with a gateway subnet.
  • A public IP address for the VPN gateway.

Steps to Create a VPN Gateway

  1. Create a Virtual Network:

    az network vnet create --name MyVNet --resource-group MyResourceGroup --address-prefix 10.1.0.0/16 --subnet-name MySubnet --subnet-prefix 10.1.0.0/24

  2. Create a Gateway Subnet:

    az network vnet subnet create --address-prefix 10.1.255.0/27 --name GatewaySubnet --resource-group MyResourceGroup --vnet-name MyVNet

  3. Create a Public IP Address:

    az network public-ip create --name MyGatewayIP --resource-group MyResourceGroup --allocation-method Dynamic

  4. Create the VPN Gateway:

    az network vnet-gateway create --name MyVPNGateway --public-ip-address MyGatewayIP --resource-group MyResourceGroup --vnet MyVNet --gateway-type Vpn --vpn-type RouteBased --sku VpnGw1 --no-wait

Configuring VPN Connections

Site-to-Site (S2S) VPN

  • Definition: Connects your on-premises network to an Azure VNet over an IPsec/IKE VPN tunnel.
  • Use Case: Ideal for extending your on-premises network to Azure.

Point-to-Site (P2S) VPN

  • Definition: Allows individual clients to connect to your Azure VNet from anywhere over the internet.
  • Use Case: Suitable for remote workers who need secure access to Azure resources.

VNet-to-VNet VPN

  • Definition: Connects two Azure VNets.
  • Use Case: Useful for connecting VNets across different regions or subscriptions.

Example: Configuring a Site-to-Site VPN

  1. Create a Local Network Gateway:

    az network local-gateway create --gateway-ip-address <OnPremisesPublicIP> --name MyLocalGateway --resource-group MyResourceGroup --local-address-prefixes 10.2.0.0/16

  2. Create the VPN Connection:

    az network vpn-connection create --name MyS2SConnection --resource-group MyResourceGroup --vnet-gateway1 MyVPNGateway --local-gateway2 MyLocalGateway --shared-key 'MySharedKey'

Advanced Features

High Availability

  • Active-Active Mode: Deploys two VPN gateways in an active-active configuration for high availability.
  • Active-Passive Mode: Default configuration where one gateway is active and the other is passive.

BGP (Border Gateway Protocol)

  • Definition: BGP is used to exchange routing information between networks.
  • Use Case: Useful for complex network topologies and dynamic routing.

ExpressRoute Integration

  • Definition: ExpressRoute provides a private connection to Azure.
  • Use Case: For scenarios requiring higher bandwidth and lower latency than typical VPN connections.

Conclusion

Azure VPN Gateway is a versatile and secure way to connect your on-premises networks, remote clients, and other Azure VNets. By understanding the different types of VPN gateways and connections, you can design a robust and scalable network architecture in Azure. Whether you are setting up a simple site-to-site VPN or a complex multi-region VNet-to-VNet connection, Azure VPN Gateway provides the necessary tools and features to meet your networking needs.
