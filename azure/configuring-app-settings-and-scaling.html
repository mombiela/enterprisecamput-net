Introduction

In this module, we will cover how to configure app settings and scale applications in Microsoft Azure. This is crucial for ensuring your applications run efficiently and can handle varying loads.

Configuring App Settings

What are App Settings?

  • App settings in Azure allow you to manage configuration settings for your applications.
  • They can be used to store configuration values such as database connection strings, API keys, and other settings that your application needs to function.

Setting Up App Settings

  1. Navigate to the Azure Portal:
    • Go to the Azure portal and select your App Service.
  2. Access Configuration:
    • In the left-hand menu, select "Configuration" under the "Settings" section.
  3. Add New Settings:
    • Click on the "Application settings" tab.
    • Click on "+ New application setting" to add a new setting.
    • Enter the name and value for your setting.
    • Click "OK" to save.

Example: Adding a Database Connection String

{
  "name": "ConnectionStrings:DefaultConnection",
  "value": "Server=tcp:yourserver.database.windows.net,1433;Initial Catalog=yourdb;Persist Security Info=False;User ID=yourusername;Password=yourpassword;MultipleActiveResultSets=False;Encrypt=True;TrustServerCertificate=False;Connection Timeout=30;"
}
  • This example shows how to add a connection string for an SQL database.

Accessing App Settings in Code

  • In .NET applications, you can access app settings using the ConfigurationManager class.
using System.Configuration;

string connectionString = ConfigurationManager.ConnectionStrings["DefaultConnection"].ConnectionString;

Scaling Applications

What is Scaling?

  • Scaling refers to adjusting the resources allocated to your application to handle varying loads.
  • There are two types of scaling:
    • Vertical Scaling: Increasing the capacity of existing resources (e.g., upgrading to a more powerful server).
    • Horizontal Scaling: Adding more instances of a resource (e.g., adding more servers).

Scaling Options in Azure

  • Manual Scaling: Manually adjust the number of instances or the size of the resources.
  • Auto-Scaling: Automatically adjust resources based on predefined rules and metrics.

Configuring Manual Scaling

  1. Navigate to the Azure Portal:
    • Go to the Azure portal and select your App Service.
  2. Access Scale Out (App Service Plan):
    • In the left-hand menu, select "Scale out (App Service plan)".
  3. Adjust Instance Count:
    • Use the slider to adjust the number of instances.
    • Click "Save" to apply the changes.

Configuring Auto-Scaling

  1. Navigate to the Azure Portal:
    • Go to the Azure portal and select your App Service.
  2. Access Scale Out (App Service Plan):
    • In the left-hand menu, select "Scale out (App Service plan)".
  3. Add a Scale Condition:
    • Click on "Add a rule".
    • Define the metric (e.g., CPU usage) and the threshold.
    • Set the action (e.g., increase by 1 instance).
    • Click "Add" to save the rule.

Example: Auto-Scaling Rule

{
  "metricName": "CpuPercentage",
  "operator": "GreaterThan",
  "threshold": 70,
  "direction": "Increase",
  "changeCount": 1,
  "cooldown": "5m"
}
  • This example shows an auto-scaling rule that increases the instance count by 1 when CPU usage exceeds 70%.

Conclusion

Configuring app settings and scaling are essential aspects of managing applications in Azure. Proper configuration ensures that your application can securely access necessary resources, while effective scaling ensures that your application can handle varying loads efficiently. By mastering these skills, you can optimize the performance and reliability of your applications in the cloud.
