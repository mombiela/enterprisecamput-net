Introduction

Welcome to the AZ-104: Microsoft Azure Administrator course. This course is designed to take you from a beginner to an advanced level in managing and administering Azure resources. By the end of this course, you will be well-prepared to take the AZ-104 certification exam and excel in your role as an Azure Administrator.

Course Outline

  1. Introduction to Azure
  2. Managing Azure Subscriptions and Resources
  3. Implementing and Managing Storage
  4. Deploying and Managing Virtual Machines (VMs)
  5. Configuring and Managing Virtual Networks
  6. Managing Identities
  7. Monitoring and Backing Up Azure Resources

1. Introduction to Azure

What is Azure?

  • Azure is Microsoft's cloud computing platform.
  • It provides a range of services including computing, analytics, storage, and networking.

Key Concepts

  • Resource Groups: Logical containers for resources deployed on Azure.
  • Regions: Geographical areas where Azure data centers are located.
  • Availability Zones: Physically separate locations within an Azure region.

Example: Creating a Resource Group

# Create a resource group in the East US region
az group create --name MyResourceGroup --location eastus

Explanation: This command creates a new resource group named MyResourceGroup in the eastus region.

2. Managing Azure Subscriptions and Resources

Azure Subscriptions

  • A subscription is a logical container used to provision resources in Azure.
  • Each subscription has a unique ID and is linked to a billing account.

Resource Management

  • Azure Resource Manager (ARM): The deployment and management service for Azure.
  • Tags: Key-value pairs that help organize resources.

Example: Applying Tags to a Resource

# Apply a tag to a resource
az resource tag --tags Environment=Production --resource-id /subscriptions/{subscription-id}/resourceGroups/{resource-group}/providers/Microsoft.Compute/virtualMachines/{vm-name}

Explanation: This command applies the tag Environment=Production to a specified virtual machine.

3. Implementing and Managing Storage

Types of Storage Accounts

  • Blob Storage: Optimized for storing massive amounts of unstructured data.
  • File Storage: Managed file shares that can be accessed via SMB protocol.
  • Queue Storage: Provides message queuing for large workloads.
  • Table Storage: NoSQL key-value store for rapid development.

Example: Creating a Storage Account

# Create a storage account
az storage account create --name mystorageaccount --resource-group MyResourceGroup --location eastus --sku Standard_LRS

Explanation: This command creates a new storage account named mystorageaccount in the MyResourceGroup resource group with the Standard_LRS (Locally Redundant Storage) SKU.

4. Deploying and Managing Virtual Machines (VMs)

Creating VMs

  • Azure Marketplace: Offers a wide range of pre-configured VM images.
  • Custom Images: Create VMs from custom images.

Example: Creating a Virtual Machine

# Create a virtual machine
az vm create --resource-group MyResourceGroup --name MyVM --image UbuntuLTS --admin-username azureuser --generate-ssh-keys

Explanation: This command creates a new virtual machine named MyVM in the MyResourceGroup resource group using the UbuntuLTS image.

5. Configuring and Managing Virtual Networks

Virtual Networks (VNet)

  • Subnets: Segments within a VNet.
  • Network Security Groups (NSGs): Control inbound and outbound traffic.

Example: Creating a Virtual Network and Subnet

# Create a virtual network
az network vnet create --resource-group MyResourceGroup --name MyVNet --address-prefix 10.0.0.0/16

# Create a subnet
az network vnet subnet create --resource-group MyResourceGroup --vnet-name MyVNet --name MySubnet --address-prefix 10.0.1.0/24

Explanation: These commands create a virtual network named MyVNet with an address space of 10.0.0.0/16 and a subnet named MySubnet with an address prefix of 10.0.1.0/24.

6. Managing Identities

Azure Active Directory (AAD)

  • Users and Groups: Manage access to resources.
  • Roles and Role Assignments: Define permissions.

Example: Creating a User in Azure AD

# Create a user in Azure AD
az ad user create --display-name JohnDoe --user-principal-name johndoe@mydomain.com --password P@ssw0rd!

Explanation: This command creates a new user named JohnDoe with the user principal name johndoe@mydomain.com.

7. Monitoring and Backing Up Azure Resources

Monitoring

  • Azure Monitor: Provides full-stack monitoring.
  • Log Analytics: Collects and analyzes log data.

Example: Setting Up Azure Monitor

# Create a Log Analytics workspace
az monitor log-analytics workspace create --resource-group MyResourceGroup --workspace-name MyWorkspace

# Enable monitoring for a VM
az monitor diagnostic-settings create --resource /subscriptions/{subscription-id}/resourceGroups/{resource-group}/providers/Microsoft.Compute/virtualMachines/{vm-name} --workspace MyWorkspace --name MyDiagnosticSetting --metrics '[{"category": "AllMetrics", "enabled": true}]' --logs '[{"category": "AuditLogs", "enabled": true}]'

Explanation: These commands create a Log Analytics workspace named MyWorkspace and enable monitoring for a specified virtual machine.

Conclusion

In this course, we covered the essential topics required to become a proficient Azure Administrator. From understanding the basics of Azure to managing complex resources and monitoring solutions, you now have the knowledge and skills to manage Azure environments effectively. Continue practicing and exploring Azure services to deepen your understanding and prepare for the AZ-104 certification exam. Good luck!
