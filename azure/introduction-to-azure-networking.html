Overview

Azure Networking is a critical component of Microsoft Azure, providing the infrastructure and services needed to connect and manage your cloud resources. This section will introduce you to the fundamental concepts and services within Azure Networking, helping you understand how to design and implement a robust network architecture in Azure.

Key Concepts

  • Virtual Networks (VNets): The foundation of Azure networking, allowing you to create isolated networks within the Azure cloud.
  • Subnets: Segments within a VNet that help organize and secure resources.
  • Network Security Groups (NSGs): Rules that control inbound and outbound traffic to network interfaces (NICs), VMs, and subnets.
  • Azure Load Balancer: Distributes incoming network traffic across multiple VMs to ensure high availability and reliability.
  • Azure VPN Gateway: Establishes secure, cross-premises connectivity between your Azure VNets and on-premises networks.
  • Azure ExpressRoute: Provides a dedicated, private connection between your on-premises infrastructure and Azure data centers.
  • Azure DNS: A hosting service for DNS domains, providing name resolution using Microsoft Azure infrastructure.
  • Azure Traffic Manager: A DNS-based traffic load balancer that enables you to distribute traffic optimally to services across global Azure regions.

Virtual Networks (VNets)

Creating a Virtual Network

A Virtual Network (VNet) is a representation of your own network in the cloud. It is a logical isolation of the Azure cloud dedicated to your subscription.

# Create a resource group
az group create --name MyResourceGroup --location eastus

# Create a virtual network
az network vnet create \
  --resource-group MyResourceGroup \
  --name MyVNet \
  --address-prefix 10.0.0.0/16 \
  --subnet-name MySubnet \
  --subnet-prefix 10.0.1.0/24

Key Points

  • VNets can be segmented into subnets.
  • VNets can be connected to other VNets or on-premises networks.
  • VNets support various Azure services like VMs, Azure Kubernetes Service (AKS), and more.

Subnets

Subnets allow you to divide your VNet into smaller, manageable sections. Each subnet can host different services and apply different security rules.

# Add a subnet to an existing VNet
az network vnet subnet create \
  --resource-group MyResourceGroup \
  --vnet-name MyVNet \
  --name MyAdditionalSubnet \
  --address-prefix 10.0.2.0/24

Key Points

  • Subnets help in organizing and securing resources.
  • Each subnet can have its own Network Security Groups (NSGs).

Network Security Groups (NSGs)

NSGs contain security rules that allow or deny inbound and outbound traffic to and from resources.

# Create a Network Security Group
az network nsg create \
  --resource-group MyResourceGroup \
  --name MyNSG

# Create a security rule
az network nsg rule create \
  --resource-group MyResourceGroup \
  --nsg-name MyNSG \
  --name AllowSSH \
  --protocol Tcp \
  --direction Inbound \
  --source-address-prefix '*' \
  --source-port-range '*' \
  --destination-address-prefix '*' \
  --destination-port-range 22 \
  --access Allow \
  --priority 1000

Key Points

  • NSGs can be associated with subnets or individual network interfaces.
  • Rules are evaluated in priority order, with lower numbers having higher priority.

Azure Load Balancer

Azure Load Balancer distributes incoming traffic among healthy instances of services in your VNet.

# Create a public IP address
az network public-ip create \
  --resource-group MyResourceGroup \
  --name MyPublicIP

# Create a load balancer
az network lb create \
  --resource-group MyResourceGroup \
  --name MyLoadBalancer \
  --public-ip-address MyPublicIP \
  --frontend-ip-name MyFrontEndPool \
  --backend-pool-name MyBackEndPool

Key Points

  • Supports both public and internal load balancing.
  • Ensures high availability and reliability by distributing traffic.

Azure VPN Gateway

Azure VPN Gateway provides secure cross-premises connectivity.

# Create a virtual network gateway
az network vnet-gateway create \
  --resource-group MyResourceGroup \
  --name MyVPNGateway \
  --public-ip-address MyPublicIP \
  --vnet MyVNet \
  --gateway-type Vpn \
  --vpn-type RouteBased \
  --sku VpnGw1 \
  --no-wait

Key Points

  • Supports site-to-site, point-to-site, and VNet-to-VNet connections.
  • Provides secure and encrypted connections.

Azure ExpressRoute

Azure ExpressRoute offers a private connection to Azure, bypassing the public internet.

Key Points

  • Provides higher security, reliability, and speed compared to VPN.
  • Ideal for scenarios requiring consistent and high-throughput connectivity.

Azure DNS

Azure DNS allows you to host your DNS domains in Azure.

# Create a DNS zone
az network dns zone create \
  --resource-group MyResourceGroup \
  --name mydomain.com

# Create a DNS record
az network dns record-set a add-record \
  --resource-group MyResourceGroup \
  --zone-name mydomain.com \
  --record-set-name www \
  --ipv4-address 10.0.0.4

Key Points

  • Provides fast and reliable DNS services.
  • Integrated with other Azure services.

Azure Traffic Manager

Azure Traffic Manager is a DNS-based traffic load balancer.

# Create a Traffic Manager profile
az network traffic-manager profile create \
  --resource-group MyResourceGroup \
  --name MyTrafficManagerProfile \
  --routing-method Performance \
  --unique-dns-name myapptrafficmanager

# Add an endpoint to the Traffic Manager profile
az network traffic-manager endpoint create \
  --resource-group MyResourceGroup \
  --profile-name MyTrafficManagerProfile \
  --name MyEndpoint \
  --type azureEndpoints \
  --target-resource-id /subscriptions/{subscription-id}/resourceGroups/MyResourceGroup/providers/Microsoft.Network/publicIPAddresses/MyPublicIP

Key Points

  • Distributes traffic across multiple regions.
  • Improves availability and responsiveness of applications.

Conclusion

Azure Networking provides a comprehensive suite of services to build, manage, and secure your network infrastructure in the cloud. By understanding and utilizing these services, you can create scalable, reliable, and secure network architectures that meet your business needs. Whether you are connecting VNets, securing resources with NSGs, or optimizing traffic with Load Balancers and Traffic Manager, Azure Networking has the tools you need to succeed.
