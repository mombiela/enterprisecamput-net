Introduction to Version Control

  • Definition: Version control is a system that records changes to a file or set of files over time so that you can recall specific versions later.
  • Importance: Helps in tracking changes, collaborating with others, and maintaining a history of your project.

What is Git?

  • Definition: Git is a distributed version control system designed to handle everything from small to very large projects with speed and efficiency.
  • Key Features:
    • Distributed: Every developer has a full copy of the repository.
    • Performance: Fast and efficient handling of large projects.
    • Branching and Merging: Easy to create, manage, and merge branches.

Setting Up Git

Installation

  • Windows: Download from git-scm.com and follow the installation instructions.
  • Mac: Use Homebrew: brew install git
  • Linux: Use the package manager: sudo apt-get install git (Debian-based) or sudo yum install git (Red Hat-based)

Configuration

  • Set your username and email: 
    git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

Basic Git Commands

Initializing a Repository

  • Command: git init
  • Explanation: Initializes a new Git repository in the current directory.

Cloning a Repository

  • Command: git clone <repository-url>
  • Explanation: Creates a copy of an existing repository.

Checking Status

  • Command: git status
  • Explanation: Shows the status of changes as untracked, modified, or staged.

Adding Changes

  • Command: git add <file>
  • Explanation: Adds changes in the specified file to the staging area.
  • Example: 
    git add script.py

Committing Changes

  • Command: git commit -m "commit message"
  • Explanation: Records the changes in the repository with a message.
  • Example: 
    git commit -m "Initial commit"

Viewing Commit History

  • Command: git log
  • Explanation: Shows the commit history for the repository.

Branching and Merging

Creating a Branch

  • Command: git branch <branch-name>
  • Explanation: Creates a new branch.
  • Example: 
    git branch feature-xyz

Switching Branches

  • Command: git checkout <branch-name>
  • Explanation: Switches to the specified branch.
  • Example: 
    git checkout feature-xyz

Merging Branches

  • Command: git merge <branch-name>
  • Explanation: Merges the specified branch into the current branch.
  • Example: 
    git merge feature-xyz

Working with Remote Repositories

Adding a Remote

  • Command: git remote add <name> <url>
  • Explanation: Adds a remote repository.
  • Example: 
    git remote add origin https://github.com/user/repo.git

Pushing Changes

  • Command: git push <remote> <branch>
  • Explanation: Pushes changes to the specified remote repository and branch.
  • Example: 
    git push origin main

Pulling Changes

  • Command: git pull <remote> <branch>
  • Explanation: Fetches and merges changes from the specified remote repository and branch.
  • Example: 
    git pull origin main

Handling Conflicts

  • Definition: Conflicts occur when changes in different branches contradict each other.
  • Resolution:
    1. Identify the conflict using git status.
    2. Open the conflicting file and manually resolve the conflict.
    3. Add the resolved file using git add <file>.
    4. Commit the changes using git commit.

Advanced Git Features

Stashing Changes

  • Command: git stash
  • Explanation: Temporarily saves changes that are not ready to be committed.
  • Example: 
    git stash

Rebasing

  • Command: git rebase <branch>
  • Explanation: Reapplies commits on top of another base tip.
  • Example: 
    git rebase main

Conclusion

Version control with Git is an essential skill for any programmer. It allows you to track changes, collaborate with others, and manage your project history efficiently. By mastering Git commands and understanding its features, you can significantly improve your workflow and productivity.
