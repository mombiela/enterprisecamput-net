Introduction to Code Quality

  • Definition: Code quality refers to the degree to which code adheres to agreed-upon standards and best practices, making it readable, maintainable, and efficient.
  • Importance:
    • Readability: Easier for others (and yourself) to read and understand.
    • Maintainability: Simplifies the process of updating and fixing code.
    • Efficiency: Ensures the code runs optimally.

What is PEP 8?

  • Definition: PEP 8 is the Python Enhancement Proposal which outlines the conventions for writing readable and consistent Python code.
  • Purpose: To improve the readability and consistency of Python code.

Key PEP 8 Guidelines

Indentation

  • Use 4 spaces per indentation level.
def example_function():
    if True:
        print("Follow PEP 8!")

Line Length

  • Limit all lines to a maximum of 79 characters.
# This is a comment that is exactly 79 characters long to demonstrate the guideline.

Blank Lines

  • Use blank lines to separate top-level function and class definitions, and within functions to separate logical sections.
class MyClass:
    def __init__(self):
        self.value = 0

    def increment(self):
        self.value += 1

Imports

  • Imports should usually be on separate lines.
  • Order imports: standard library, third-party libraries, local application/library-specific imports.
import os
import sys

import requests

from mymodule import myfunction

Naming Conventions

  • Variables and Functions: Use lower_case_with_underscores.
  • Classes: Use CapWords.
  • Constants: Use ALL_CAPS_WITH_UNDERSCORES.
# Variable and function names
my_variable = 10

def my_function():
    pass

# Class name
class MyClass:
    pass

# Constant name
MAX_LIMIT = 100

Comments

  • Use comments to explain why something is done, not what is done.
  • Use # for inline comments and block comments.
# This is a block comment
x = 42  # This is an inline comment

Docstrings

  • Use triple quotes for docstrings in functions, classes, and modules.
def my_function():
    """
    This is a docstring.
    It explains what the function does.
    """
    pass

Tools for Enforcing PEP 8

  • pycodestyle: A tool to check your Python code against some of the PEP 8 style conventions.
  • autopep8: A tool that automatically formats Python code to conform to the PEP 8 style guide.
  • flake8: A tool that combines pycodestyle, pyflakes, and other plugins to check the style and quality of Python code.

Example of PEP 8 Compliant Code

import os
import sys

class ExampleClass:
    """This is an example class following PEP 8 guidelines."""

    def __init__(self, value):
        self.value = value

    def increment(self):
        """Increments the value by one."""
        self.value += 1

def main():
    """Main function to run the example."""
    example = ExampleClass(0)
    example.increment()
    print(example.value)

if __name__ == "__main__":
    main()

Conclusion

  • Summary: Adhering to PEP 8 guidelines ensures that your Python code is readable, maintainable, and consistent. Utilizing tools like pycodestyle, autopep8, and flake8 can help enforce these standards.
  • Next Steps: Practice writing PEP 8 compliant code and use the mentioned tools to check and improve your code quality.
