Generics in Rust
Threads ►

Generics are a powerful feature in Rust that allows you to write flexible and reusable code. By using generics, you can create functions, structs, enums, and traits that can operate on many different types without sacrificing performance or safety.

Key Concepts

  • Generic Types: Placeholder types that can be specified when the function, struct, or enum is used.
  • Type Parameters: The actual types that replace the generic types when the code is compiled.
  • Traits and Bounds: Constraints that specify what capabilities a type must have to be used as a generic type.

Generic Functions

Generic functions allow you to write a function that can operate on different types without duplicating code.

Example

fn largest<T: PartialOrd>(list: &[T]) -> &T {
    let mut largest = &list[0];
    for item in list.iter() {
        if item > largest {
            largest = item;
        }
    }
    largest
}

In this example:

  • T is a generic type.
  • PartialOrd is a trait that ensures the type T can be compared using the > operator.

Explanation

  • fn largest<T: PartialOrd>(list: &[T]) -> &T declares a function named largest that takes a slice of type T and returns a reference to an element of type T.
  • T: PartialOrd is a trait bound that ensures the elements can be compared.

Generic Structs

You can define structs with generic types to create data structures that can store different types of data.

Example

struct Point<T> {
    x: T,
    y: T,
}

impl<T> Point<T> {
    fn new(x: T, y: T) -> Self {
        Point { x, y }
    }
}

Explanation

  • struct Point<T> defines a struct with a generic type T.
  • impl<T> Point<T> implements methods for the Point struct.
  • fn new(x: T, y: T) -> Self is a constructor method that initializes a Point with values of type T.

Generic Enums

Enums can also be generic, allowing you to define variants that can hold different types.

Example

enum Option<T> {
    Some(T),
    None,
}

Explanation

  • enum Option<T> defines an enum with a generic type T.
  • Some(T) and None are the variants of the enum. Some holds a value of type T, while None does not hold any value.

Traits and Bounds

Traits define shared behavior that types can implement. You can use trait bounds to specify that a generic type must implement a particular trait.

Example

fn print_area<T: Area>(shape: &T) {
    println!("The area is {}", shape.area());
}

trait Area {
    fn area(&self) -> f64;
}

struct Circle {
    radius: f64,
}

impl Area for Circle {
    fn area(&self) -> f64 {
        std::f64::consts::PI * self.radius * self.radius
    }
}

Explanation

  • fn print_area<T: Area>(shape: &T) defines a function that takes a reference to a generic type T that implements the Area trait.
  • trait Area defines a trait with a method area.
  • struct Circle defines a struct representing a circle.
  • impl Area for Circle implements the Area trait for the Circle struct.

Summary

Generics in Rust allow you to write flexible, reusable, and type-safe code. By using generic types, you can create functions, structs, enums, and traits that work with many different types while maintaining performance and safety. Understanding generics is crucial for writing idiomatic and efficient Rust code.
