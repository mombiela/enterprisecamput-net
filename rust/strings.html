Introduction to Strings

Strings are a fundamental part of programming, allowing you to work with text. In Rust, strings are more complex than in some other languages due to Rust's focus on safety and performance.

Key Concepts

  • String Types: Rust has two main string types: String and &str.
  • Ownership and Borrowing: Understanding how Rust handles memory with strings.
  • UTF-8 Encoding: Rust strings are UTF-8 encoded, which affects how you work with them.

String Types

`String`

The String type is a growable, heap-allocated data structure. It is used when you need an owned string that you can modify.

Example

fn main() {
    let mut s = String::from("Hello");
    s.push_str(", world!");
    println!("{}", s); // Outputs: Hello, world!
}
  • String::from("Hello"): Creates a new String from a string literal.
  • s.push_str(", world!"): Appends a string slice to the String.

`&str`

The &str type, also known as a string slice, is a reference to a string. It is used when you need an immutable view into a string.

Example

fn main() {
    let s = "Hello, world!";
    println!("{}", s); // Outputs: Hello, world!
}
  • "Hello, world!": A string literal, which is of type &str.

Ownership and Borrowing with Strings

Ownership

In Rust, String owns its data, meaning it is responsible for managing the memory where the string is stored.

Example

fn main() {
    let s1 = String::from("hello");
    let s2 = s1; // s1 is moved to s2
    // println!("{}", s1); // This would cause a compile-time error
}
  • s1 is moved to s2, and s1 is no longer valid.

Borrowing

You can borrow a string using a reference, allowing you to use the string without taking ownership.

Example

fn main() {
    let s1 = String::from("hello");
    let len = calculate_length(&s1);
    println!("The length of '{}' is {}.", s1, len);
}

fn calculate_length(s: &String) -> usize {
    s.len()
}
  • &s1: A reference to s1 is passed to the function.
  • s.len(): Returns the length of the string.

UTF-8 Encoding

Rust strings are UTF-8 encoded, which means they can contain any valid UTF-8 sequence. This affects how you can index and slice strings.

Example

fn main() {
    let hello = "Здравствуйте";
    let s = &hello[0..4];
    println!("{}", s); // Outputs: Зд
}
  • &hello[0..4]: Slices the string, but you must be careful to slice on valid UTF-8 boundaries.

Common String Operations

Concatenation

You can concatenate strings using the + operator or the format! macro.

Example

fn main() {
    let s1 = String::from("Hello, ");
    let s2 = String::from("world!");
    let s3 = s1 + &s2; // s1 is moved here and can no longer be used
    println!("{}", s3); // Outputs: Hello, world!
}

Iteration

You can iterate over strings by characters or bytes.

Example

fn main() {
    let s = String::from("hello");
    for c in s.chars() {
        println!("{}", c);
    }
}
  • s.chars(): Returns an iterator over the characters of the string.

Conclusion

Strings in Rust are a powerful and flexible way to handle text, but they come with complexities due to Rust's ownership and borrowing rules and UTF-8 encoding. Understanding the differences between String and &str, how to manage ownership, and how to perform common operations will help you effectively work with strings in Rust.
