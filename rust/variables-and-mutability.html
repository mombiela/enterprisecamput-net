Variables and Mutability in Rust
Data Types ►

Introduction

In Rust, variables are fundamental building blocks that store data. Understanding how to declare, use, and modify variables is crucial for writing effective Rust programs. This section will cover the basics of variables, the concept of mutability, and how to work with both mutable and immutable variables.

Declaring Variables

In Rust, variables are declared using the let keyword. By default, variables are immutable, meaning their values cannot be changed once set.

fn main() {
    let x = 5;
    println!("The value of x is: {}", x);
}
  • let: Keyword used to declare a variable.
  • x: Name of the variable.
  • 5: Initial value assigned to the variable x.

Mutability

To allow a variable's value to be changed, you need to declare it as mutable using the mut keyword.

fn main() {
    let mut y = 10;
    println!("The initial value of y is: {}", y);
    y = 15;
    println!("The new value of y is: {}", y);
}
  • mut: Keyword indicating that the variable is mutable.
  • y = 15;: Changing the value of y after its initial declaration.

Shadowing

Rust allows you to declare a new variable with the same name as a previous variable. This is known as shadowing and it allows you to reuse variable names while transforming their values.

fn main() {
    let z = 6;
    let z = z + 1;
    println!("The value of z after shadowing is: {}", z);
}
  • Shadowing allows for transformations without needing to declare new variable names.
  • The new z shadows the old z, but they are different variables.

Constants

Constants are similar to immutable variables but are declared using the const keyword and must have their type specified. Constants are always immutable and can be declared in any scope, including the global scope.

const MAX_POINTS: u32 = 100_000;
  • const: Keyword used to declare a constant.
  • MAX_POINTS: Name of the constant, conventionally written in uppercase.
  • u32: Type of the constant.
  • 100_000: Value assigned to the constant.

Variable Types

Rust is a statically typed language, meaning that the type of a variable must be known at compile time. Rust can often infer the type based on the value, but you can also explicitly specify the type.

fn main() {
    let a: i32 = 42; // Explicit type annotation
    let b = 3.14;    // Type inferred as f64
}
  • i32: 32-bit signed integer.
  • f64: 64-bit floating-point number.

Examples and Exercises

Example 1: Immutable Variable

fn main() {
    let name = "Alice";
    println!("Hello, {}!", name);
}
  • This example demonstrates an immutable variable name that holds a string.

Example 2: Mutable Variable

fn main() {
    let mut counter = 1;
    println!("Counter: {}", counter);
    counter += 1;
    println!("Counter: {}", counter);
}
  • This example shows a mutable variable counter whose value is incremented.

Exercise 1: Declare and Modify a Mutable Variable

  1. Declare a mutable variable score and set its initial value to 50.
  2. Print the value of score.
  3. Change the value of score to 75.
  4. Print the new value of score.

Exercise 2: Use Shadowing

  1. Declare an immutable variable temperature and set its value to 20.
  2. Shadow temperature by adding 5 to its value.
  3. Print the new value of temperature.

Summary

  • Variables in Rust are immutable by default.
  • Use the mut keyword to make a variable mutable.
  • Shadowing allows you to reuse variable names for transformations.
  • Constants are always immutable and must have their type specified.
  • Rust is statically typed, and types can be inferred or explicitly specified.

Understanding variables and mutability is essential for managing data and state in your Rust programs. Practice declaring, modifying, and shadowing variables to become proficient in handling data in Rust.
