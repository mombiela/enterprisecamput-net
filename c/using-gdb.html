The GNU Debugger (GDB) is a powerful tool for debugging C programs. This section will guide you through the basics of using GDB, from setting it up to using its advanced features.

Introduction to GDB

GDB allows you to:

  • Inspect what the program is doing at a specific point in time.
  • Change variables and execute code line by line.
  • Set breakpoints to pause execution at specific lines.
  • Trace the execution flow to understand how the program reaches a certain state.

Setting Up GDB

To use GDB, you need to compile your C program with debugging information. This is done using the -g flag with gcc.

gcc -g -o myprogram myprogram.c

This command compiles myprogram.c into an executable named myprogram with debugging information included.

Basic GDB Commands

Starting GDB

To start GDB with your compiled program:

gdb myprogram

Running the Program

To run the program within GDB:

(gdb) run

Setting Breakpoints

Breakpoints allow you to pause the execution of your program at specific lines or functions.

  • To set a breakpoint at a specific line:
(gdb) break 10
  • To set a breakpoint at the start of a function:
(gdb) break main

Inspecting Variables

To inspect the value of a variable:

(gdb) print variable_name

Stepping Through Code

  • To execute the next line of code:
(gdb) next
  • To step into a function call:
(gdb) step

Continuing Execution

To continue running the program after hitting a breakpoint:

(gdb) continue

Exiting GDB

To exit GDB:

(gdb) quit

Advanced GDB Features

Conditional Breakpoints

You can set breakpoints that only trigger when a certain condition is true.

(gdb) break 10 if x == 5

Watchpoints

Watchpoints pause execution when the value of a variable changes.

(gdb) watch variable_name

Backtrace

To see the call stack and understand how the program reached the current point:

(gdb) backtrace

Examining Memory

To examine memory at a specific address:

(gdb) x /nfu address
  • n is the number of units to display.
  • f is the format (e.g., x for hexadecimal, d for decimal).
  • u is the unit size (e.g., b for bytes, h for half-words).

Example Session

Here is an example GDB session to illustrate the commands:

$ gcc -g -o example example.c
$ gdb example
(gdb) break main
Breakpoint 1 at 0x400526: file example.c, line 5.
(gdb) run
Starting program: /path/to/example 

Breakpoint 1, main () at example.c:5
5       int x = 0;
(gdb) print x
$1 = 0
(gdb) next
6       x = 5;
(gdb) print x
$2 = 0
(gdb) next
7       printf("%d\n", x);
(gdb) print x
$3 = 5
(gdb) continue
5
[Inferior 1 (process 1234) exited normally]
(gdb) quit

Conclusion

GDB is an essential tool for debugging C programs. By mastering its basic and advanced features, you can significantly improve your ability to diagnose and fix bugs. Practice using GDB with different programs to become more comfortable with its commands and capabilities.
