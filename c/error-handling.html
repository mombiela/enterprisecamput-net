Introduction to Error Handling

Error handling is a critical aspect of programming that ensures your code can gracefully handle unexpected situations and errors. In C, error handling is not as straightforward as in some higher-level languages, but it is equally important. This section will guide you from basic to advanced error handling techniques in C.

Basic Error Handling

Using Return Values

One of the simplest ways to handle errors in C is by using return values from functions.

  • Standard Library Functions: Many standard library functions return specific values to indicate success or failure.
  • Custom Functions: You can design your own functions to return error codes.

Example: Checking Return Values

#include <stdio.h>

int divide(int a, int b, int *result) {
    if (b == 0) {
        return -1; // Error code for division by zero
    }
    *result = a / b;
    return 0; // Success
}

int main() {
    int result;
    int status = divide(10, 0, &result);
    
    if (status != 0) {
        printf("Error: Division by zero\n");
    } else {
        printf("Result: %d\n", result);
    }
    
    return 0;
}

Using `errno`

The errno variable is used in C to indicate error conditions for many standard library functions.

  • Header File: Include <errno.h> to use errno.
  • Error Codes: Standard error codes like EINVAL, ENOMEM, etc.

Example: Using errno

#include <stdio.h>
#include <errno.h>
#include <string.h>

int main() {
    FILE *file = fopen("nonexistent.txt", "r");
    
    if (file == NULL) {
        printf("Error opening file: %s\n", strerror(errno));
    } else {
        fclose(file);
    }
    
    return 0;
}

Intermediate Error Handling

Custom Error Codes

Creating custom error codes can help in making your error handling more descriptive and manageable.

  • Define Error Codes: Use #define to create custom error codes.
  • Error Handling Functions: Create functions to handle and display errors.

Example: Custom Error Codes

#include <stdio.h>

#define SUCCESS 0
#define ERROR_DIVISION_BY_ZERO 1
#define ERROR_NEGATIVE_NUMBER 2

int divide(int a, int b, int *result) {
    if (b == 0) {
        return ERROR_DIVISION_BY_ZERO;
    }
    *result = a / b;
    return SUCCESS;
}

void handle_error(int error_code) {
    switch (error_code) {
        case ERROR_DIVISION_BY_ZERO:
            printf("Error: Division by zero\n");
            break;
        case ERROR_NEGATIVE_NUMBER:
            printf("Error: Negative number\n");
            break;
        default:
            printf("Unknown error\n");
    }
}

int main() {
    int result;
    int status = divide(10, 0, &result);
    
    if (status != SUCCESS) {
        handle_error(status);
    } else {
        printf("Result: %d\n", result);
    }
    
    return 0;
}

Advanced Error Handling

Using `setjmp` and `longjmp`

For more complex error handling, C provides setjmp and longjmp for non-local jumps.

  • setjmp: Saves the current environment (stack context).
  • longjmp: Restores the saved environment, effectively jumping back to the point where setjmp was called.

Example: setjmp and longjmp

#include <stdio.h>
#include <setjmp.h>

jmp_buf buf;

void second() {
    printf("second\n"); // prints
    longjmp(buf, 1);    // jumps back to where setjmp was called - making setjmp now return 1
}

void first() {
    second();
    printf("first\n"); // does not print
}

int main() {
    if (setjmp(buf)) {
        printf("main\n"); // prints
    } else {
        first();          // does not return
    }
    
    return 0;
}

Error Handling in Large Projects

In large projects, error handling can become complex. Here are some best practices:

  • Centralized Error Handling: Use a centralized error handling mechanism.
  • Error Logging: Implement error logging to keep track of errors.
  • Consistent Error Codes: Maintain consistency in error codes across the project.

Summary

Error handling in C ranges from simple return value checks to more complex mechanisms like setjmp and longjmp. Understanding and implementing effective error handling strategies is crucial for building robust and reliable applications. By using return values, errno, custom error codes, and advanced techniques, you can ensure your programs handle errors gracefully and provide meaningful feedback to users and developers.
