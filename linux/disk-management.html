Introduction

Disk management is a crucial aspect of system administration in Linux. It involves tasks such as partitioning, formatting, mounting, and monitoring disk usage. This section will guide you from basic to advanced disk management techniques in Linux.

Basic Concepts

Understanding Partitions

  • Partition: A division of a hard disk drive (HDD) or solid-state drive (SSD) into separate, independent sections.
  • Types of Partitions:
    • Primary Partition: Can be used to boot the operating system.
    • Extended Partition: A special type of primary partition that can hold multiple logical partitions.
    • Logical Partition: Subdivisions within an extended partition.

Filesystems

  • Filesystem: A method for storing and organizing files on a disk.
  • Common filesystems in Linux:
    • ext4: Fourth extended filesystem, widely used.
    • XFS: High-performance filesystem.
    • Btrfs: Filesystem with advanced features like snapshots.

Disk Devices

  • Disk devices are represented in Linux as files in the /dev directory.
    • Example: /dev/sda (first SATA drive), /dev/sdb (second SATA drive).

Basic Disk Management Commands

Viewing Disk Information

  • lsblk: Lists information about all available or specified block devices. 
    lsblk
  • fdisk -l: Lists all partitions on all disks. 
    sudo fdisk -l

Creating Partitions

  • fdisk: A command-line utility to create and manipulate partition tables. 
    sudo fdisk /dev/sda
    • Follow the interactive prompts to create a new partition.

Formatting Partitions

  • mkfs: Used to create a filesystem on a partition. 
    sudo mkfs.ext4 /dev/sda1

Mounting and Unmounting Partitions

  • mount: Mounts a filesystem. 
    sudo mount /dev/sda1 /mnt
  • umount: Unmounts a filesystem. 
    sudo umount /mnt

Intermediate Disk Management

Persistent Mounts

  • To ensure a filesystem is mounted at boot, add an entry to /etc/fstab. 
    sudo nano /etc/fstab
    • Example entry: 
      /dev/sda1  /mnt  ext4  defaults  0  2

Disk Quotas

  • Quota: Limits the amount of disk space and number of files a user or group can use.
  • Enable quotas on a filesystem: 
    sudo mount -o remount,usrquota,grpquota /mnt
sudo quotacheck -cug /mnt
sudo quotaon /mnt

LVM (Logical Volume Manager)

  • LVM: Allows for flexible disk management.
    • Create a physical volume: 
      sudo pvcreate /dev/sda1
    • Create a volume group: 
      sudo vgcreate myvg /dev/sda1
    • Create a logical volume: 
      sudo lvcreate -L 10G -n mylv myvg
    • Format and mount the logical volume: 
      sudo mkfs.ext4 /dev/myvg/mylv
sudo mount /dev/myvg/mylv /mnt

Advanced Disk Management

RAID (Redundant Array of Independent Disks)

  • RAID: Combines multiple disk drives into a single unit for redundancy or performance.
  • Create a RAID 1 array using mdadm: 
    sudo mdadm --create --verbose /dev/md0 --level=1 --raid-devices=2 /dev/sda1 /dev/sdb1

Filesystem Tuning

  • tune2fs: Adjusts filesystem parameters on ext2/ext3/ext4 filesystems. 
    sudo tune2fs -m 1 /dev/sda1  # Set reserved blocks percentage to 1%

Monitoring Disk Usage

  • df: Reports file system disk space usage. 
    df -h
  • du: Estimates file space usage. 
    du -sh /path/to/directory

Conclusion

Disk management in Linux ranges from basic tasks like partitioning and formatting to advanced techniques like LVM and RAID. Mastering these skills ensures efficient and reliable storage management, which is crucial for both personal and enterprise environments. By understanding and utilizing the tools and commands discussed, you can effectively manage disk resources on a Linux system.
