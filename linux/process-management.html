Introduction to Processes

  • Definition: A process is an instance of a running program.
  • Process Lifecycle: Creation, execution, and termination.
  • Process ID (PID): Unique identifier for each process.

Viewing Processes

  • ps Command: Displays information about active processes.

    ps aux
    • a: Shows processes for all users.
    • u: Displays the user/owner of the processes.
    • x: Lists processes without a controlling terminal.

  • top Command: Provides a dynamic, real-time view of running processes.

    top
    • Use q to quit.
    • Use h for help within top.

Managing Processes

  • Foreground and Background Processes:

    • Foreground: Runs directly in the terminal.
    • Background: Runs independently of the terminal.
    ./script.sh &  # Runs script.sh in the background

  • jobs Command: Lists current jobs.

    jobs

  • fg and bg Commands:

    • fg %1: Brings job 1 to the foreground.
    • bg %1: Resumes job 1 in the background.

Process Control

  • Killing Processes:

    • kill Command: Sends a signal to a process. 
      kill -9 1234  # Forcefully kills process with PID 1234
      • -9: SIGKILL signal, forcefully terminates the process.

  • killall Command: Kills all processes with a given name.

    killall firefox

  • pkill Command: Sends a signal to processes based on name and other attributes.

    pkill -u username  # Kills all processes owned by 'username'

Process Prioritization

  • Nice Value: Determines the priority of a process.

    • nice Command: Starts a process with a specified nice value. 
      nice -n 10 ./script.sh  # Starts script.sh with a nice value of 10

  • renice Command: Changes the priority of an existing process.

    renice 15 -p 1234  # Changes the nice value of process 1234 to 15

Nice Value Table

| Nice Value | Priority Level | |------------|----------------| | -20 | Highest | | 0 | Default | | 19 | Lowest |

Advanced Process Management

  • htop Command: An interactive process viewer.

    htop
    • Use arrow keys to navigate.
    • Use F9 to kill a process.

  • Process States:

    • Running (R): Actively using CPU.
    • Sleeping (S): Waiting for an event.
    • Stopped (T): Halted by a signal.
    • Zombie (Z): Terminated but not reaped by parent.

Conclusion

Understanding process management is crucial for effective system administration and performance tuning in Linux. By mastering commands like ps, top, kill, and nice, you can monitor and control processes efficiently, ensuring optimal system performance and stability.
