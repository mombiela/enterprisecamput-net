Introduction

In Linux, wildcards and regular expressions are powerful tools for searching, matching, and manipulating text. This section will guide you from the basics to advanced usage of these tools, providing clear explanations, examples, and exercises.

Wildcards

Wildcards are special characters used to match patterns in filenames and directories. They are commonly used in shell commands to perform operations on multiple files at once.

Basic Wildcards

  • * (asterisk): Matches zero or more characters.
  • ? (question mark): Matches exactly one character.
  • [] (square brackets): Matches any one of the enclosed characters.

Examples

# List all files in the current directory
ls *

# List all files starting with 'a'
ls a*

# List all files with a single character name
ls ?

# List all files starting with 'a', 'b', or 'c'
ls [abc]*

Advanced Wildcards

  • [!abc]: Matches any character except 'a', 'b', or 'c'.
  • {} (curly braces): Matches a comma-separated list of patterns.

Examples

# List all files except those starting with 'a', 'b', or 'c'
ls [!abc]*

# List files with extensions .txt or .md
ls *.{txt,md}

Regular Expressions

Regular expressions (regex) are sequences of characters that define search patterns. They are more powerful and flexible than wildcards and are used in many Linux tools like grep, sed, and awk.

Basic Regular Expressions

  • . (dot): Matches any single character.
  • * (asterisk): Matches zero or more of the preceding element.
  • ^ (caret): Matches the start of a line.
  • $ (dollar sign): Matches the end of a line.
  • [] (square brackets): Matches any one of the enclosed characters.

Examples

# Match lines containing 'cat'
grep 'cat' file.txt

# Match lines starting with 'cat'
grep '^cat' file.txt

# Match lines ending with 'cat'
grep 'cat$' file.txt

# Match lines containing 'c', 'a', or 't'
grep '[cat]' file.txt

Advanced Regular Expressions

  • + (plus): Matches one or more of the preceding element.
  • ? (question mark): Matches zero or one of the preceding element.
  • {n,m}: Matches between n and m occurrences of the preceding element.
  • () (parentheses): Groups patterns.
  • | (pipe): Matches either the pattern before or the pattern after the pipe.

Examples

# Match lines containing 'cat' one or more times
grep 'cat+' file.txt

# Match lines containing 'cat' zero or one time
grep 'cat?' file.txt

# Match lines containing 'cat' between 2 and 4 times
grep 'cat{2,4}' file.txt

# Match lines containing 'cat' or 'dog'
grep 'cat|dog' file.txt

# Match lines containing 'cat' followed by 'dog'
grep '(cat)(dog)' file.txt

Exercises

Wildcards

  1. List all files in your home directory that start with 'd'.
  2. List all files in your home directory that have a '.sh' extension.
  3. List all files in your home directory that do not start with 'a', 'b', or 'c'.

Regular Expressions

  1. Find all lines in a file that start with 'Hello'.
  2. Find all lines in a file that end with 'world'.
  3. Find all lines in a file that contain either 'foo' or 'bar'.
  4. Find all lines in a file that contain a digit.

Conclusion

Wildcards and regular expressions are essential tools for efficiently managing files and processing text in Linux. Mastering these tools will significantly enhance your ability to perform complex searches and text manipulations. Practice using the examples and exercises provided to become proficient in using wildcards and regular expressions.
