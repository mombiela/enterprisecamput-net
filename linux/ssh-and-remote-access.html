Introduction to SSH

Secure Shell (SSH) is a protocol used to securely connect to remote systems over a network. It provides a secure channel over an unsecured network by using encryption.

  • What is SSH?

    • SSH stands for Secure Shell.
    • It is used for secure remote login and other secure network services over an insecure network.
    • SSH uses public-key cryptography to authenticate the remote computer and allow it to authenticate the user.

  • Why Use SSH?

    • Secure communication between client and server.
    • Encrypted data transfer.
    • Authentication of users and devices.
    • Secure file transfer (SCP, SFTP).

Setting Up SSH

To use SSH, you need to have an SSH client on your local machine and an SSH server on the remote machine.

Installing SSH

  • On Ubuntu/Debian:

    sudo apt update
sudo apt install openssh-server

  • On CentOS/RHEL:

    sudo yum install openssh-server
sudo systemctl start sshd
sudo systemctl enable sshd

  • On Windows:

    • Use an SSH client like PuTTY or enable the built-in OpenSSH client in Windows 10.

Starting and Enabling SSH Service

  • On Ubuntu/Debian:

    sudo systemctl start ssh
sudo systemctl enable ssh

  • On CentOS/RHEL:

    sudo systemctl start sshd
sudo systemctl enable sshd

Connecting to a Remote Server

To connect to a remote server using SSH, you need the server's IP address or hostname and the username.

Basic SSH Command

ssh username@hostname_or_ip
  • Example: 
    ssh user@192.168.1.10

Using SSH Keys for Authentication

SSH keys provide a more secure way of logging into an SSH server.

Generating SSH Keys

ssh-keygen -t rsa -b 2048
  • This command generates a pair of keys: a private key and a public key.

Copying the Public Key to the Server

ssh-copy-id username@hostname_or_ip
  • Example: 
    ssh-copy-id user@192.168.1.10

Advanced SSH Usage

Port Forwarding

SSH can be used to forward ports, allowing you to securely access services on a remote network.

  • Local Port Forwarding:

    ssh -L local_port:remote_address:remote_port username@hostname_or_ip
    • Example: 
      ssh -L 8080:localhost:80 user@192.168.1.10

  • Remote Port Forwarding:

    ssh -R remote_port:local_address:local_port username@hostname_or_ip
    • Example: 
      ssh -R 8080:localhost:80 user@192.168.1.10

SSH Tunneling

SSH tunneling allows you to create a secure connection between your local machine and a remote machine through an SSH server.

  • Dynamic Port Forwarding (SOCKS Proxy): 
    ssh -D local_port username@hostname_or_ip
    • Example: 
      ssh -D 8080 user@192.168.1.10

SSH Config File

You can simplify SSH connections by using the SSH config file (~/.ssh/config).

  • Example Config File:

    Host myserver
    HostName 192.168.1.10
    User user
    Port 22
    IdentityFile ~/.ssh/id_rsa

  • Connecting Using Config File:

    ssh myserver

Summary

SSH is a powerful tool for secure remote access and management of servers. By understanding the basics of SSH, setting up SSH keys, and utilizing advanced features like port forwarding and tunneling, you can enhance the security and efficiency of your remote operations. Always ensure your SSH server is properly configured and secured to prevent unauthorized access.
