Overview

Shell scripting is a powerful way to automate tasks in a Linux environment. This topic will guide you through the basics of shell scripting, from understanding what a shell is to writing your first script and beyond.

What is a Shell?

  • Definition: A shell is a command-line interpreter that provides a user interface for the Unix/Linux operating system.
  • Types of Shells:
    • Bourne Shell (sh)
    • Bourne Again Shell (bash)
    • C Shell (csh)
    • Korn Shell (ksh)
    • Z Shell (zsh)

Why Use Shell Scripting?

  • Automation: Automate repetitive tasks.
  • Efficiency: Perform complex tasks with simple commands.
  • Customization: Tailor your environment to your needs.
  • Integration: Combine multiple commands and utilities.

Basic Shell Commands

  • Navigation: cd, ls, pwd
  • File Operations: cp, mv, rm, touch
  • Viewing Files: cat, more, less, head, tail
  • Permissions: chmod, chown

Writing Your First Shell Script

  1. Create a Script File: 
    touch myscript.sh
  2. Make the Script Executable: 
    chmod +x myscript.sh
  3. Edit the Script: 
    nano myscript.sh
  4. Add Shebang Line: The shebang line tells the system which interpreter to use. 
    #!/bin/bash
  5. Add Commands: 
    #!/bin/bash
echo "Hello, World!"
  6. Run the Script: 
    ./myscript.sh

Variables

  • Definition: Variables store data that can be used and manipulated within a script.
  • Syntax: 
    VARIABLE_NAME="value"
echo $VARIABLE_NAME

Conditional Statements

  • If-Else: 
    #!/bin/bash
if [ $1 -gt 10 ]; then
    echo "The number is greater than 10"
else
    echo "The number is 10 or less"
fi
  • Case: 
    #!/bin/bash
case $1 in
    start)
        echo "Starting..."
        ;;
    stop)
        echo "Stopping..."
        ;;
    *)
        echo "Usage: $0 {start|stop}"
        ;;
esac

Loops

  • For Loop: 
    #!/bin/bash
for i in {1..5}; do
    echo "Welcome $i times"
done
  • While Loop: 
    #!/bin/bash
COUNTER=0
while [ $COUNTER -lt 5 ]; do
    echo "Counter: $COUNTER"
    COUNTER=$((COUNTER + 1))
done

Functions

  • Definition: Functions are reusable blocks of code.
  • Syntax: 
    #!/bin/bash
my_function() {
    echo "This is a function"
}
my_function

Input and Output

  • Reading User Input: 
    #!/bin/bash
echo "Enter your name:"
read NAME
echo "Hello, $NAME"
  • Redirecting Output: 
    #!/bin/bash
echo "This is a test" > output.txt

Error Handling

  • Exit Status: Every command returns an exit status (0 for success, non-zero for failure). 
    #!/bin/bash
cp file1.txt file2.txt
if [ $? -eq 0 ]; then
    echo "Copy successful"
else
    echo "Copy failed"
fi

Advanced Topics

  • Arrays: 
    #!/bin/bash
NAMES=("Alice" "Bob" "Charlie")
echo "First name: ${NAMES[0]}"
  • Regular Expressions: 
    #!/bin/bash
if [[ "hello" =~ ^h.*o$ ]]; then
    echo "Pattern matches"
else
    echo "Pattern does not match"
fi
  • Debugging: Use set -x to enable debugging. 
    #!/bin/bash
set -x
echo "Debugging enabled"

Conclusion

Shell scripting is an essential skill for anyone working in a Linux environment. It allows you to automate tasks, manage system operations, and customize your workflow. By understanding the basics and practicing with examples, you can become proficient in writing and using shell scripts to enhance your productivity.
