Introduction to Cron

Cron is a time-based job scheduler in Unix-like operating systems. Users can schedule jobs (commands or scripts) to run periodically at fixed times, dates, or intervals. It is a powerful tool for automating repetitive tasks.

Understanding Cron Syntax

Cron jobs are defined in a file called a crontab. Each line in a crontab file represents a scheduled task and follows a specific syntax.

Crontab Syntax

A typical crontab line consists of five time-and-date fields followed by the command to be executed:

* * * * * command_to_be_executed

The five fields represent:

  1. Minute (0-59)
  2. Hour (0-23)
  3. Day of the month (1-31)
  4. Month (1-12)
  5. Day of the week (0-7, where both 0 and 7 represent Sunday)

Example

To run a script every day at 2:30 AM:

30 2 * * * /path/to/script.sh

Managing Crontab Files

Viewing Crontab Entries

To view your current crontab entries, use:

crontab -l

Editing Crontab Entries

To edit your crontab file, use:

crontab -e

This opens the crontab file in the default text editor, allowing you to add, modify, or delete cron jobs.

Removing Crontab Entries

To remove all crontab entries for the current user, use:

crontab -r

Special Strings in Cron

Cron also supports special strings to simplify scheduling:

  • @reboot – Run once, at startup.
  • @yearly or @annually – Run once a year, i.e., 0 0 1 1 *.
  • @monthly – Run once a month, i.e., 0 0 1 * *.
  • @weekly – Run once a week, i.e., 0 0 * * 0.
  • @daily or @midnight – Run once a day, i.e., 0 0 * * *.
  • @hourly – Run once an hour, i.e., 0 * * * *.

Example

To run a script at every system reboot:

@reboot /path/to/script.sh

Advanced Cron Usage

Redirecting Output

Cron jobs can produce output, which can be redirected to a file or sent via email.

Redirecting to a File

To log the output of a cron job to a file:

* * * * * /path/to/script.sh >> /path/to/logfile 2>&1

This appends both standard output and standard error to logfile.

Sending Output via Email

By default, cron sends the output to the user's email. Ensure the MAILTO environment variable is set:

MAILTO="user@example.com"
* * * * * /path/to/script.sh

Using Environment Variables

You can set environment variables directly in the crontab file:

SHELL=/bin/bash
PATH=/usr/local/sbin:/usr/local/bin:/sbin:/bin:/usr/sbin:/usr/bin

Running Multiple Commands

To run multiple commands in a single cron job, use the && operator:

* * * * * /path/to/first_script.sh && /path/to/second_script.sh

Common Pitfalls and Troubleshooting

  • Path Issues: Ensure the full path to the command or script is specified.
  • Permissions: Verify that the script has executable permissions.
  • Environment Differences: Cron jobs run in a minimal shell environment. Set necessary environment variables within the script or crontab.

Example of a Troubleshooting Script

To debug a cron job, you can redirect the output to a log file:

* * * * * /path/to/script.sh >> /path/to/logfile 2>&1

Check logfile for errors or output.

Conclusion

Cron is an essential tool for automating tasks in Linux. By understanding its syntax, managing crontab files, and utilizing advanced features, you can efficiently schedule and manage repetitive tasks. Always remember to troubleshoot common issues by checking paths, permissions, and environment variables.
