Introduction to Addressing Modes

Addressing modes are techniques used in assembly language to specify the operand(s) for the instructions. Understanding addressing modes is crucial for writing efficient and effective assembly code.

Key Concepts

  • Operand: The data on which the instruction operates.
  • Addressing Mode: The way in which the operand is specified.

Types of Addressing Modes

Immediate Addressing Mode

In immediate addressing mode, the operand is specified directly in the instruction.

Example

MOV AL, 5  ; Move the immediate value 5 into register AL
  • Explanation: The value 5 is directly provided in the instruction.

Register Addressing Mode

In register addressing mode, the operand is located in a register.

Example

MOV AX, BX  ; Move the value in register BX into register AX
  • Explanation: The value in register BX is moved to register AX.

Direct Addressing Mode

In direct addressing mode, the address of the operand is specified in the instruction.

Example

MOV AX, [1234h]  ; Move the value at memory address 1234h into register AX
  • Explanation: The value at memory address 1234h is moved to register AX.

Indirect Addressing Mode

In indirect addressing mode, the address of the operand is held in a register.

Example

MOV AX, [BX]  ; Move the value at the address contained in register BX into register AX
  • Explanation: The value at the memory address contained in BX is moved to AX.

Indexed Addressing Mode

In indexed addressing mode, the operand's address is determined using a base address and an index.

Example

MOV AX, [BX+SI]  ; Move the value at the address (BX + SI) into register AX
  • Explanation: The value at the address calculated by adding BX and SI is moved to AX.

Based Indexed Addressing Mode

In based indexed addressing mode, the operand's address is determined using a base address, an index, and an optional displacement.

Example

MOV AX, [BX+SI+10h]  ; Move the value at the address (BX + SI + 10h) into register AX
  • Explanation: The value at the address calculated by adding BX, SI, and 10h is moved to AX.

Relative Addressing Mode

In relative addressing mode, the operand's address is determined relative to the current instruction address.

Example

JMP SHORT LABEL  ; Jump to the address relative to the current instruction
LABEL:
  • Explanation: The jump is made to the address specified by LABEL, relative to the current instruction.

Comparison of Addressing Modes

| Addressing Mode | Operand Specification | Example | |-------------------------|------------------------------------------------|--------------------------| | Immediate | Operand is directly specified | MOV AL, 5 | | Register | Operand is in a register | MOV AX, BX | | Direct | Address of operand is specified | MOV AX, [1234h] | | Indirect | Address of operand is in a register | MOV AX, [BX] | | Indexed | Address is base address + index | MOV AX, [BX+SI] | | Based Indexed | Address is base address + index + displacement | MOV AX, [BX+SI+10h] | | Relative | Address is relative to current instruction | JMP SHORT LABEL |

Exercises

Exercise 1: Immediate Addressing

Write an assembly instruction to move the value 10 into register CL.

Exercise 2: Register Addressing

Write an assembly instruction to move the value from register DX to register CX.

Exercise 3: Direct Addressing

Write an assembly instruction to move the value at memory address 2000h into register AX.

Exercise 4: Indirect Addressing

Write an assembly instruction to move the value at the address contained in register SI into register BX.

Exercise 5: Indexed Addressing

Write an assembly instruction to move the value at the address calculated by adding DI to BP into register DX.

Exercise 6: Based Indexed Addressing

Write an assembly instruction to move the value at the address calculated by adding BX, DI, and 20h into register AX.

Exercise 7: Relative Addressing

Write an assembly instruction to jump to a label NEXT which is 5 bytes ahead of the current instruction.

Conclusion

Addressing modes are fundamental to understanding and writing assembly language programs. Each mode provides a different way to specify the operands, allowing for flexibility and efficiency in coding. By mastering these modes, you can write more effective and optimized assembly code.
