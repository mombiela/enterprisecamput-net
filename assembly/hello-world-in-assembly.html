Introduction

Assembly language is a low-level programming language that is closely related to machine code. It provides a way to write programs that are highly efficient and optimized for specific hardware. In this section, we will cover how to write a simple "Hello World" program in Assembly language.

Key Concepts

  • Registers: Small storage locations within the CPU used to perform operations.
  • Instructions: Commands that tell the CPU what to do.
  • System Calls: Requests made by a program to the operating system to perform specific tasks.

Writing "Hello World" in Assembly

To write a "Hello World" program in Assembly, we need to understand the basic structure of an Assembly program and how to use system calls to print text to the screen.

Basic Structure of an Assembly Program

An Assembly program typically consists of the following sections:

  1. Data Section: Defines constants and initialized data.
  2. BSS Section: Defines uninitialized data.
  3. Text Section: Contains the actual code.

Example: "Hello World" Program

Below is an example of a "Hello World" program written in NASM (Netwide Assembler) for Linux:

section .data
    hello db 'Hello, World!', 0x0A  ; Define the string with a newline character

section .text
    global _start                  ; Entry point for the program

_start:
    ; Write the string to stdout
    mov eax, 4                     ; System call number for sys_write
    mov ebx, 1                     ; File descriptor 1 (stdout)
    mov ecx, hello                 ; Pointer to the string
    mov edx, 13                    ; Length of the string
    int 0x80                       ; Call the kernel

    ; Exit the program
    mov eax, 1                     ; System call number for sys_exit
    xor ebx, ebx                   ; Exit code 0
    int 0x80                       ; Call the kernel

Explanation of the Code

  • Data Section:

    • hello db 'Hello, World!', 0x0A: Defines a string "Hello, World!" followed by a newline character (0x0A).

  • Text Section:

    • global _start: Defines the entry point of the program.
    • _start:: Label marking the beginning of the program.
    • mov eax, 4: Moves the system call number for sys_write into the eax register.
    • mov ebx, 1: Moves the file descriptor for stdout into the ebx register.
    • mov ecx, hello: Moves the address of the string into the ecx register.
    • mov edx, 13: Moves the length of the string into the edx register.
    • int 0x80: Interrupt to call the kernel and execute the system call.
    • mov eax, 1: Moves the system call number for sys_exit into the eax register.
    • xor ebx, ebx: Sets the exit code to 0 by XORing ebx with itself.
    • int 0x80: Interrupt to call the kernel and exit the program.

Conclusion

Writing a "Hello World" program in Assembly language provides a fundamental understanding of how low-level programming works. By learning how to manipulate registers and use system calls, you gain insight into the inner workings of the CPU and the operating system. This knowledge is crucial for writing efficient and optimized code in Assembly.
