Introduction to HDFS

Hadoop Distributed File System (HDFS) is the primary storage system used by Hadoop applications. It provides high-throughput access to application data and is designed to be highly fault-tolerant. Understanding HDFS commands is crucial for managing and interacting with the Hadoop file system.

Basic HDFS Commands

These commands are essential for beginners to get started with HDFS.

Listing Files and Directories

  • hdfs dfs -ls [path]: Lists files and directories in the specified path.
hdfs dfs -ls /user/hadoop

This command lists all files and directories under /user/hadoop.

Creating Directories

  • hdfs dfs -mkdir [path]: Creates a new directory at the specified path.
hdfs dfs -mkdir /user/hadoop/newdir

This command creates a new directory named newdir under /user/hadoop.

Copying Files to HDFS

  • hdfs dfs -put [local_path] [hdfs_path]: Copies files from the local file system to HDFS.
hdfs dfs -put /local/path/to/file.txt /user/hadoop/

This command copies file.txt from the local file system to the HDFS directory /user/hadoop/.

Viewing File Contents

  • hdfs dfs -cat [path]: Displays the contents of a file.
hdfs dfs -cat /user/hadoop/file.txt

This command displays the contents of file.txt located in /user/hadoop/.

Deleting Files and Directories

  • hdfs dfs -rm [path]: Deletes a file.
hdfs dfs -rm /user/hadoop/file.txt

This command deletes file.txt from /user/hadoop/.

  • hdfs dfs -rm -r [path]: Deletes a directory and its contents recursively.
hdfs dfs -rm -r /user/hadoop/newdir

This command deletes the directory newdir and all its contents from /user/hadoop/.

Intermediate HDFS Commands

These commands provide more control and are useful for managing HDFS more effectively.

Moving and Renaming Files

  • hdfs dfs -mv [source_path] [dest_path]: Moves or renames files and directories.
hdfs dfs -mv /user/hadoop/file.txt /user/hadoop/renamed_file.txt

This command renames file.txt to renamed_file.txt.

Copying Files within HDFS

  • hdfs dfs -cp [source_path] [dest_path]: Copies files from one HDFS location to another.
hdfs dfs -cp /user/hadoop/file.txt /user/hadoop/copy_of_file.txt

This command copies file.txt to copy_of_file.txt within the same directory.

Checking File Status

  • hdfs dfs -stat [format] [path]: Displays the status of a file or directory.
hdfs dfs -stat %b /user/hadoop/file.txt

This command displays the size in bytes of file.txt.

Changing File Permissions

  • hdfs dfs -chmod [permissions] [path]: Changes the permissions of a file or directory.
hdfs dfs -chmod 755 /user/hadoop/file.txt

This command sets the permissions of file.txt to 755.

Advanced HDFS Commands

These commands are for advanced users who need to perform complex operations and optimizations.

Checking Disk Usage

  • hdfs dfs -du [path]: Displays the disk usage of files and directories.
hdfs dfs -du /user/hadoop

This command shows the disk usage of all files and directories under /user/hadoop.

Balancing the HDFS Cluster

  • hdfs balancer: Balances the data across the DataNodes in the HDFS cluster.
hdfs balancer

This command initiates the balancing process to ensure even distribution of data across DataNodes.

Safemode Operations

  • hdfs dfsadmin -safemode [enter|leave|get]: Manages the safemode state of the HDFS.
hdfs dfsadmin -safemode enter

This command puts the HDFS into safemode, where no modifications are allowed.

Snapshot Management

  • hdfs dfs -createSnapshot [path] [snapshotName]: Creates a snapshot of a directory.
hdfs dfs -createSnapshot /user/hadoop mySnapshot

This command creates a snapshot named mySnapshot of the directory /user/hadoop.

  • hdfs dfs -deleteSnapshot [path] [snapshotName]: Deletes a snapshot.
hdfs dfs -deleteSnapshot /user/hadoop mySnapshot

This command deletes the snapshot named mySnapshot.

Conclusion

Understanding and effectively using HDFS commands is crucial for managing data within a Hadoop ecosystem. From basic file operations to advanced cluster management, these commands provide the necessary tools to interact with HDFS efficiently. Practice these commands regularly to become proficient in handling HDFS and leveraging its full potential.
