Running Applications on YARN
YARN Schedulers ►

In this section, we will explore how to run applications on YARN (Yet Another Resource Negotiator), which is a core component of Hadoop responsible for managing resources and scheduling jobs. This topic will cover the basics of YARN, how to submit applications, and advanced configurations for optimizing performance.

Introduction to YARN

YARN is a resource management layer for Hadoop that allows multiple data processing engines to handle data stored in a single platform, unlocking the potential for a wide range of data analytics.

  • Resource Management: YARN manages resources across the cluster.
  • Job Scheduling: YARN schedules jobs and allocates resources based on the requirements.

Key Components of YARN

Understanding the key components of YARN is crucial for running applications effectively.

  • ResourceManager (RM): Manages resources and schedules applications.
  • NodeManager (NM): Manages resources on individual nodes and reports to the RM.
  • ApplicationMaster (AM): Manages the lifecycle of an application.
  • Container: A unit of resource allocation (CPU, memory) on a node.

Running a Simple YARN Application

To run an application on YARN, you need to follow these steps:

  1. Prepare the Application: Package your application into a JAR file.
  2. Submit the Application: Use the yarn command to submit your application.
  3. Monitor the Application: Use the YARN web UI or command-line tools to monitor the application's progress.

Example: Running a WordCount Application on YARN

Let's run a simple WordCount application on YARN.

  1. Prepare the Application:

    • Ensure you have a JAR file for the WordCount application.
    • Example JAR: wordcount.jar

  2. Submit the Application:

    yarn jar wordcount.jar org.apache.hadoop.examples.WordCount /input /output
    • yarn jar: Command to run a JAR file on YARN.
    • wordcount.jar: The JAR file containing the WordCount application.
    • org.apache.hadoop.examples.WordCount: The main class of the application.
    • /input: Input directory in HDFS.
    • /output: Output directory in HDFS.

  3. Monitor the Application:

    • Open the YARN ResourceManager web UI (usually at http://<ResourceManager-Host>:8088).
    • Check the status of your application under the "Applications" tab.

Advanced YARN Configurations

To optimize the performance of your applications on YARN, you can tweak various configurations.

Resource Allocation

  • Memory Configuration:

    • yarn.nodemanager.resource.memory-mb: Total memory available for containers on a node.
    • yarn.scheduler.maximum-allocation-mb: Maximum memory allocation for a single container.

  • CPU Configuration:

    • yarn.nodemanager.resource.cpu-vcores: Total CPU cores available for containers on a node.
    • yarn.scheduler.maximum-allocation-vcores: Maximum CPU cores allocation for a single container.

Example Configuration in `yarn-site.xml`

<configuration>
    <property>
        <name>yarn.nodemanager.resource.memory-mb</name>
        <value>8192</value>
    </property>
    <property>
        <name>yarn.scheduler.maximum-allocation-mb</name>
        <value>4096</value>
    </property>
    <property>
        <name>yarn.nodemanager.resource.cpu-vcores</name>
        <value>8</value>
    </property>
    <property>
        <name>yarn.scheduler.maximum-allocation-vcores</name>
        <value>4</value>
    </property>
</configuration>

Conclusion

Running applications on YARN involves understanding its architecture, preparing and submitting applications, and monitoring their progress. By configuring YARN's resource allocation settings, you can optimize the performance of your applications. With these basics, you are now equipped to run and manage applications on YARN effectively.
