Hadoop Cluster Setup
Running a Hadoop Cluster ►

Setting up a Hadoop cluster is a crucial step in leveraging the power of distributed computing for big data processing. This section will guide you through the process of setting up a Hadoop cluster from scratch, covering both single-node and multi-node configurations.

Prerequisites

Before setting up a Hadoop cluster, ensure you have the following:

  • A basic understanding of Linux commands and file system.
  • Java Development Kit (JDK) installed on all nodes.
  • SSH access between nodes without a password.

Single-Node Hadoop Cluster Setup

A single-node setup is ideal for learning and development purposes. It allows you to run Hadoop on a single machine.

Step 1: Install Java

Hadoop requires Java to run. Ensure Java is installed and configured correctly.

sudo apt-get update
sudo apt-get install openjdk-8-jdk
java -version

Step 2: Download and Extract Hadoop

Download the latest stable version of Hadoop from the official website and extract it.

wget https://downloads.apache.org/hadoop/common/hadoop-3.3.1/hadoop-3.3.1.tar.gz
tar -xzvf hadoop-3.3.1.tar.gz
mv hadoop-3.3.1 /usr/local/hadoop

Step 3: Configure Hadoop Environment Variables

Add Hadoop environment variables to the .bashrc file.

export HADOOP_HOME=/usr/local/hadoop
export HADOOP_INSTALL=$HADOOP_HOME
export HADOOP_MAPRED_HOME=$HADOOP_HOME
export HADOOP_COMMON_HOME=$HADOOP_HOME
export HADOOP_HDFS_HOME=$HADOOP_HOME
export YARN_HOME=$HADOOP_HOME
export HADOOP_COMMON_LIB_NATIVE_DIR=$HADOOP_HOME/lib/native
export PATH=$PATH:$HADOOP_HOME/sbin:$HADOOP_HOME/bin

Apply the changes:

source ~/.bashrc

Step 4: Configure Hadoop

Edit the following configuration files located in the $HADOOP_HOME/etc/hadoop directory:

  • core-site.xml
<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://localhost:9000</value>
    </property>
</configuration>
  • hdfs-site.xml
<configuration>
    <property>
        <name>dfs.replication</name>
        <value>1</value>
    </property>
</configuration>
  • mapred-site.xml
<configuration>
    <property>
        <name>mapreduce.framework.name</name>
        <value>yarn</value>
    </property>
</configuration>
  • yarn-site.xml
<configuration>
    <property>
        <name>yarn.nodemanager.aux-services</name>
        <value>mapreduce_shuffle</value>
    </property>
</configuration>

Step 5: Format the Namenode

Format the Hadoop filesystem (HDFS) before starting the cluster.

hdfs namenode -format

Step 6: Start Hadoop Daemons

Start the Hadoop daemons (Namenode, Datanode, ResourceManager, and NodeManager).

start-dfs.sh
start-yarn.sh

Step 7: Verify the Setup

Check the Hadoop web interfaces to ensure everything is running correctly:

  • HDFS: http://localhost:9870
  • YARN: http://localhost:8088

Multi-Node Hadoop Cluster Setup

A multi-node setup is used for production environments to handle large-scale data processing.

Step 1: Setup SSH Key-Based Authentication

Generate an SSH key on the master node and copy it to all slave nodes.

ssh-keygen -t rsa
ssh-copy-id user@slave1
ssh-copy-id user@slave2

Step 2: Configure Hadoop on Master and Slave Nodes

Repeat the single-node setup steps on all nodes. Additionally, configure the slaves file on the master node with the IP addresses or hostnames of all slave nodes.

slave1
slave2

Step 3: Configure Core Hadoop Files

Ensure the core-site.xml, hdfs-site.xml, mapred-site.xml, and yarn-site.xml files are correctly configured on all nodes, similar to the single-node setup.

Step 4: Format the Namenode

Format the Namenode on the master node only.

hdfs namenode -format

Step 5: Start Hadoop Daemons

Start the Hadoop daemons on the master node.

start-dfs.sh
start-yarn.sh

Step 6: Verify the Cluster

Check the Hadoop web interfaces on the master node to ensure all nodes are connected and running correctly.

Conclusion

Setting up a Hadoop cluster involves several steps, from installing prerequisites to configuring various Hadoop components. A single-node setup is suitable for learning and development, while a multi-node setup is essential for production environments. By following the steps outlined in this section, you can successfully set up and verify a Hadoop cluster, enabling you to harness the power of distributed computing for big data processing.
