Overview

The certification exam for REXX (Restructured Extended Executor) is designed to test your knowledge and skills across various levels of proficiency, from beginner to advanced. This section will guide you through the key areas you need to focus on, provide examples, and offer tips for preparation.

Key Areas of Focus

1. Basic Syntax and Structure

  • Understanding REXX Syntax: Learn the basic syntax rules, including how to write and execute simple REXX programs.
  • Variables and Data Types: Understand how to declare and use variables, and the different data types available in REXX.
  • Control Structures: Get familiar with loops (DO, DO WHILE, DO UNTIL) and conditional statements (IF, THEN, ELSE).
/* Simple REXX Program */
say "Hello, World!"

/* Variables */
name = "John"
say "Hello, " name

/* Conditional Statement */
if name = "John" then
  say "Welcome, John!"
else
  say "Who are you?"

2. Functions and Procedures

  • Built-in Functions: Learn the most commonly used built-in functions like LENGTH, SUBSTR, DATE, and TIME.
  • User-defined Functions: Understand how to create and use your own functions and procedures.
/* Built-in Function Example */
name = "John"
say "The length of the name is: " LENGTH(name)

/* User-defined Function */
say greet("John")

greet: procedure
  parse arg name
  return "Hello, " name

3. String Manipulation

  • Concatenation: Learn how to concatenate strings using the || operator.
  • Substrings: Extract parts of strings using the SUBSTR function.
  • Pattern Matching: Use the POS function to find patterns within strings.
/* String Concatenation */
firstName = "John"
lastName = "Doe"
fullName = firstName || " " || lastName
say fullName

/* Substring Extraction */
text = "Hello, World!"
say SUBSTR(text, 1, 5)  /* Output: Hello */

/* Pattern Matching */
text = "Hello, World!"
position = POS("World", text)
say "Position of 'World': " position

4. File I/O Operations

  • Reading Files: Learn how to open, read, and close files.
  • Writing Files: Understand how to write data to files.
/* Reading from a File */
fileName = "example.txt"
call stream fileName, "c", "open read"
do while lines(fileName) > 0
  line = linein(fileName)
  say line
end
call stream fileName, "c", "close"

/* Writing to a File */
fileName = "output.txt"
call stream fileName, "c", "open write"
call lineout fileName, "This is a test."
call stream fileName, "c", "close"

5. Error Handling and Debugging

  • Error Trapping: Use the SIGNAL instruction to handle errors.
  • Debugging Techniques: Learn how to use TRACE to debug your REXX programs.
/* Error Trapping */
signal on syntax name errorHandler
say "This will cause an error: " 1 / 0
exit

errorHandler:
  say "An error occurred!"
  exit

/* Debugging with TRACE */
trace ?r
say "This is a debug message."

Preparation Tips

  • Practice Regularly: Write and execute REXX programs to get comfortable with the syntax and features.
  • Review Documentation: Go through the official REXX documentation and any additional resources provided in the course.
  • Mock Exams: Take practice exams to familiarize yourself with the format and types of questions you might encounter.

Conclusion

Preparing for the REXX certification exam requires a thorough understanding of the language's syntax, functions, and features. By focusing on the key areas outlined above and practicing regularly, you can build the confidence and skills needed to succeed. Good luck!
