Overview

The final project for this REXX course is designed to consolidate your learning from beginner to advanced levels. You will apply the concepts, techniques, and best practices you've learned throughout the course to create a comprehensive REXX script. This project will test your understanding of REXX programming and your ability to solve real-world problems using REXX.

Project Requirements

Your final project should meet the following requirements:

  • Script Length: The script should be at least 100 lines of code.
  • Functionality: The script should perform a meaningful task, such as data processing, file manipulation, or automation of a repetitive task.
  • Use of REXX Features: The script should demonstrate the use of various REXX features, including:
    • Variables and data types
    • Control structures (loops, conditionals)
    • Functions and subroutines
    • File I/O operations
    • Error handling
  • Documentation: The script should be well-documented with comments explaining the purpose of each section of code.
  • User Interaction: The script should include some form of user interaction, such as input prompts or command-line arguments.

Example Project Ideas

Here are a few example project ideas to inspire you:

  • Log File Analyzer: Create a script that reads a log file, extracts relevant information, and generates a summary report.
  • Task Automation: Develop a script that automates a repetitive task, such as renaming files in a directory based on a specific pattern.
  • Data Processing: Write a script that processes a CSV file, performs data transformations, and outputs the results to a new file.
  • Interactive Quiz: Build an interactive quiz application that asks the user a series of questions and provides feedback based on their answers.

Example Code

Below is an example of a simple REXX script that reads a text file and counts the number of lines:

/* Simple REXX script to count lines in a file */
parse arg filename

if filename = '' then do
  say 'Please provide a filename as an argument.'
  exit 1
end

lines = 0

/* Open the file for reading */
call stream filename, 'c', 'open read'

if stream(filename, 's') = 'READY:' then do
  do while lines = stream(filename, 'c', 'read')
    lines = lines + 1
  end
  call stream filename, 'c', 'close'
  say 'Number of lines in the file:' lines
end
else do
  say 'Failed to open the file.'
  exit 1
end

Explanation

  • parse arg filename: This line retrieves the filename from the command-line arguments.
  • if filename = '' then do: This block checks if a filename was provided and exits if not.
  • lines = 0: Initializes the line counter.
  • call stream filename, 'c', 'open read': Opens the file for reading.
  • if stream(filename, 's') = 'READY:' then do: Checks if the file was successfully opened.
  • do while lines = stream(filename, 'c', 'read'): Reads each line of the file and increments the line counter.
  • call stream filename, 'c', 'close': Closes the file after reading.
  • say 'Number of lines in the file:' lines: Outputs the total number of lines.

Submission Guidelines

  • Documentation: Include a README file that explains the purpose of your script, how to run it, and any dependencies or requirements.
