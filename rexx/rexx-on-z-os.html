Introduction to REXX

  • What is REXX?

    • REXX (Restructured Extended Executor) is a high-level programming language designed for ease of learning and reading.
    • It is widely used for scripting and automation on IBM's z/OS mainframe operating system.

  • Why use REXX on z/OS?

    • Simplifies complex tasks with fewer lines of code.
    • Integrates seamlessly with other z/OS utilities and programs.
    • Ideal for both beginners and experienced programmers due to its straightforward syntax.

Getting Started with REXX on z/OS

  • Setting up the Environment

    • Ensure you have access to a z/OS system.
    • Familiarize yourself with TSO (Time Sharing Option) and ISPF (Interactive System Productivity Facility).

  • Writing Your First REXX Script

    • Create a new member in a PDS (Partitioned Data Set) with a .REXX extension.
    • Example: 
      /* Hello World in REXX */
SAY 'Hello, World!'
    • Save and execute the script using the EXEC command in TSO.

Basic REXX Syntax and Commands

  • Comments

    • Single-line comments: /* This is a comment */

  • Variables

    • No need to declare variables explicitly.
    • Example: 
      name = 'John Doe'
SAY 'Hello, ' name

  • Basic Operations

    • Arithmetic: +, -, *, /
    • Example: 
      a = 5
b = 3
c = a + b
SAY 'The sum of' a 'and' b 'is' c

  • Control Structures

    • IF-THEN-ELSE 
      age = 18
IF age >= 18 THEN
  SAY 'You are an adult.'
ELSE
  SAY 'You are a minor.'
    • DO Loops 
      DO i = 1 TO 5
  SAY 'Iteration' i
END

Advanced REXX Features

  • Built-in Functions

    • String Manipulation 
      str = 'Hello, World!'
SAY LENGTH(str)  /* Outputs: 13 */
SAY SUBSTR(str, 1, 5)  /* Outputs: Hello */

  • File I/O Operations

    • Reading from a file: 
      /* Read a file line by line */
"ALLOCATE FILE(INPUT) DA('your.dataset.name') SHR"
DO WHILE LINES('INPUT') > 0
  LINEIN('INPUT')
  SAY LINE
END
"FREE FILE(INPUT)"
    • Writing to a file: 
      /* Write to a file */
"ALLOCATE FILE(OUTPUT) DA('your.dataset.name') SHR"
CALL LINEOUT 'OUTPUT', 'This is a line of text.'
"FREE FILE(OUTPUT)"

  • Interfacing with z/OS Utilities

    • TSO Commands

      ADDRESS TSO "LISTCAT ENTRIES('your.dataset.name')"

    • ISPF Panels

      ADDRESS ISPEXEC "DISPLAY PANEL(your.panel.name)"

Error Handling and Debugging

  • Error Trapping

    • Example: 
      SIGNAL ON ERROR
/* Code that might cause an error */
ERROR:
SAY 'An error occurred.'

  • Debugging Techniques

    • Use TRACE to follow the execution flow. 
      TRACE R

Best Practices for REXX Programming

  • Code Readability

    • Use meaningful variable names.
    • Comment your code extensively.

  • Modular Programming

    • Break down large scripts into smaller, reusable procedures.

  • Performance Optimization

    • Minimize the use of external commands and I/O operations.

Conclusion

  • Summary

    • REXX is a powerful scripting language on z/OS, offering simplicity and integration with mainframe utilities.
    • From basic syntax to advanced features, REXX can handle a variety of tasks efficiently.

  • Next Steps

    • Practice writing and executing REXX scripts.
    • Explore more advanced topics and real-world applications of REXX on z/OS.
