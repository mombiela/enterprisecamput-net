Introduction to REXX Extensions and Libraries

REXX (Restructured Extended Executor) is a versatile scripting language that can be extended with various libraries and extensions to enhance its functionality. This section will cover the most commonly used extensions and libraries, how to integrate them into your REXX programs, and practical examples to illustrate their usage.

Key Concepts

  • Extensions: Additional functionalities that can be added to the REXX language.
  • Libraries: Pre-written code modules that can be included in REXX programs to perform specific tasks.
  • Integration: The process of incorporating extensions and libraries into your REXX scripts.

Common REXX Extensions

1. REXXUTIL

REXXUTIL is a library that provides a variety of utility functions for REXX programs, especially on IBM mainframes and OS/2 systems.

Key Functions

  • SysFileTree: Recursively lists files in a directory.
  • SysSleep: Pauses execution for a specified number of seconds.
  • SysGetEnv: Retrieves the value of an environment variable.

Example

/* Example using REXXUTIL */
call RxFuncAdd 'SysLoadFuncs', 'REXXUTIL', 'SysLoadFuncs'
call SysLoadFuncs

/* List all files in the current directory */
call SysFileTree '*', 'fileList.', 'FO'

do i = 1 to fileList.0
    say fileList.i
end

Explanation:

  • RxFuncAdd is used to register the REXXUTIL functions.
  • SysFileTree lists all files in the current directory and stores them in the fileList. array.

2. REXXLIB

REXXLIB is another popular library that provides additional functions for file handling, string manipulation, and system operations.

Key Functions

  • RxFileCopy: Copies a file from one location to another.
  • RxFileDelete: Deletes a specified file.
  • RxStringReverse: Reverses a given string.

Example

/* Example using REXXLIB */
call RxFuncAdd 'RxFileCopy', 'REXXLIB', 'RxFileCopy'
call RxFuncAdd 'RxFileDelete', 'REXXLIB', 'RxFileDelete'

/* Copy a file */
call RxFileCopy 'source.txt', 'destination.txt'

/* Delete the original file */
call RxFileDelete 'source.txt'

Explanation:

  • RxFuncAdd registers the REXXLIB functions.
  • RxFileCopy copies source.txt to destination.txt.
  • RxFileDelete deletes source.txt.

Integrating Extensions and Libraries

To use extensions and libraries in your REXX programs, you typically need to:

  1. Load the library using RxFuncAdd.
  2. Call the library's initialization function if required.
  3. Use the provided functions as needed in your script.

Example: Combining REXXUTIL and REXXLIB

/* Load REXXUTIL and REXXLIB */
call RxFuncAdd 'SysLoadFuncs', 'REXXUTIL', 'SysLoadFuncs'
call SysLoadFuncs
call RxFuncAdd 'RxFileCopy', 'REXXLIB', 'RxFileCopy'
call RxFuncAdd 'RxFileDelete', 'REXXLIB', 'RxFileDelete'

/* List all files in the current directory */
call SysFileTree '*', 'fileList.', 'FO'

do i = 1 to fileList.0
    say 'Copying file:' fileList.i
    call RxFileCopy fileList.i, fileList.i || '.bak'
end

say 'All files have been backed up.'

Explanation:

  • Both REXXUTIL and REXXLIB are loaded and their functions registered.
  • SysFileTree lists all files in the current directory.
  • Each file is copied to a new file with a .bak extension using RxFileCopy.

Conclusion

REXX extensions and libraries significantly enhance the capabilities of the REXX language, allowing for more complex and powerful scripts. By understanding how to integrate and use these extensions, you can leverage their functionalities to streamline your programming tasks. Practice using these libraries in your projects to become proficient in extending REXX's capabilities.
