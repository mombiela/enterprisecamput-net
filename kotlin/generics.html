Generics in Kotlin
Extensions ►

Generics in Kotlin allow you to write flexible and reusable code. They enable you to create classes, interfaces, and functions that work with any data type while providing type safety. This topic will guide you from the basics of generics to more advanced concepts.

Introduction to Generics

Generics are a powerful feature in Kotlin that allows you to define classes, interfaces, and functions with type parameters. This means you can write code that works with different types without sacrificing type safety.

Why Use Generics?

  • Type Safety: Generics ensure that you catch type errors at compile time rather than at runtime.
  • Reusability: You can write more flexible and reusable code.
  • Performance: Generics can help avoid the need for type casting, which can improve performance.

Basic Syntax

To define a generic class or function, you use angle brackets (<>) to specify the type parameter.

// Generic class
class Box<T>(val value: T)

// Generic function
fun <T> printItem(item: T) {
    println(item)
}

In the above examples, T is a type parameter that can be replaced with any type.

Generic Classes

Generic classes allow you to create classes that can handle different types of data.

Example: Generic Box Class

class Box<T>(val value: T)

fun main() {
    val intBox = Box(1)
    val stringBox = Box("Hello")

    println(intBox.value)  // Output: 1
    println(stringBox.value)  // Output: Hello
}

In this example, Box can hold any type of value, making it a versatile container.

Multiple Type Parameters

You can define a class with multiple type parameters.

class Pair<A, B>(val first: A, val second: B)

fun main() {
    val pair = Pair(1, "One")
    println(pair.first)  // Output: 1
    println(pair.second)  // Output: One
}

Generic Functions

Generic functions allow you to write functions that can operate on different types.

Example: Generic Print Function

fun <T> printItem(item: T) {
    println(item)
}

fun main() {
    printItem(123)  // Output: 123
    printItem("Hello")  // Output: Hello
}

In this example, printItem can print any type of item.

Constraints on Type Parameters

You can restrict the types that can be used as type parameters using constraints.

Upper Bounds

You can specify an upper bound for a type parameter using the : symbol.

fun <T : Number> sum(a: T, b: T): Double {
    return a.toDouble() + b.toDouble()
}

fun main() {
    println(sum(1, 2))  // Output: 3.0
    println(sum(1.5, 2.5))  // Output: 4.0
    // println(sum("1", "2"))  // Error: Type mismatch
}

In this example, T is constrained to be a subtype of Number.

Variance

Variance allows you to define how subtyping between more complex types relates to subtyping between their components.

Covariance

Covariance allows a type to be a subtype of another type. You use the out keyword to declare a type parameter as covariant.

class Box<out T>(val value: T)

fun main() {
    val intBox: Box<Int> = Box(1)
    val numberBox: Box<Number> = intBox  // Covariance allows this
    println(numberBox.value)  // Output: 1
}

Contravariance

Contravariance allows a type to be a supertype of another type. You use the in keyword to declare a type parameter as contravariant.

class Box<in T> {
    fun put(value: T) { /*...*/ }
}

fun main() {
    val numberBox: Box<Number> = Box()
    val intBox: Box<Int> = numberBox  // Contravariance allows this
    intBox.put(1)  // This is allowed
}

Type Projections

Type projections allow you to use generic types in a more flexible way.

Star Projections

Star projections (*) allow you to use a generic type without specifying the exact type parameter.

fun printBox(box: Box<*>) {
    println(box.value)
}

fun main() {
    val intBox = Box(1)
    val stringBox = Box("Hello")

    printBox(intBox)  // Output: 1
    printBox(stringBox)  // Output: Hello
}

Summary

Generics in Kotlin provide a powerful way to write flexible, reusable, and type-safe code. By understanding the basics of generic classes and functions, constraints, variance, and type projections, you can create more robust and versatile applications. Practice using generics in your projects to fully grasp their potential and benefits.
