Function Overloads in TypeScript
Arrow Functions ►

Function overloading is a feature in TypeScript that allows you to define multiple signatures for a single function. This can be particularly useful when you want a function to handle different types of parameters or return different types of results based on the input.

Key Concepts

  • Function Signature: The combination of a function's name and its parameter types.
  • Overload Signature: Different versions of a function signature that a function can accept.
  • Implementation Signature: The actual function implementation that handles all the overloads.

Why Use Function Overloads?

  • Flexibility: Allows a single function to handle different types of inputs.
  • Readability: Makes the code easier to understand by clearly defining what types of inputs a function can accept.
  • Type Safety: Ensures that the function is used correctly according to its defined signatures.

Basic Syntax

In TypeScript, function overloads are defined by writing multiple function signatures followed by a single implementation.

// Overload signatures
function add(a: number, b: number): number;
function add(a: string, b: string): string;

// Implementation signature
function add(a: any, b: any): any {
    return a + b;
}

// Usage
console.log(add(1, 2)); // Output: 3
console.log(add("Hello, ", "world!")); // Output: Hello, world!

Detailed Example

Let's create a more detailed example to understand function overloads better. Suppose we want to create a function that can either concatenate two strings or add two numbers.

// Overload signatures
function combine(a: number, b: number): number;
function combine(a: string, b: string): string;

// Implementation signature
function combine(a: any, b: any): any {
    if (typeof a === "number" && typeof b === "number") {
        return a + b;
    } else if (typeof a === "string" && typeof b === "string") {
        return a + b;
    }
    throw new Error("Invalid arguments. Both arguments must be of the same type.");
}

// Usage
console.log(combine(10, 20)); // Output: 30
console.log(combine("foo", "bar")); // Output: foobar

Advanced Example with Optional Parameters

Function overloads can also handle optional parameters. Let's create a function that can either return the full name of a person or just the first name if the last name is not provided.

// Overload signatures
function getName(firstName: string, lastName: string): string;
function getName(firstName: string): string;

// Implementation signature
function getName(firstName: string, lastName?: string): string {
    if (lastName) {
        return `${firstName} ${lastName}`;
    } else {
        return firstName;
    }
}

// Usage
console.log(getName("John", "Doe")); // Output: John Doe
console.log(getName("John")); // Output: John

Conclusion

Function overloads in TypeScript provide a powerful way to create flexible and type-safe functions. By defining multiple signatures for a single function, you can handle different types of inputs and ensure that your function is used correctly. This feature enhances code readability and maintainability, making it easier to work with complex functions.

By understanding and utilizing function overloads, you can write more robust and versatile TypeScript code, catering to a variety of use cases with a single, well-defined function.
