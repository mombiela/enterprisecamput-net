Generics in TypeScript allow you to create reusable components that work with a variety of types rather than a single one. This topic will cover the basics of generic functions, their syntax, and practical examples to help you understand how to use them effectively.

Introduction to Generics

Generics provide a way to create flexible and reusable components. They allow you to define functions, classes, and interfaces that work with a variety of types without sacrificing type safety.

Key Concepts

  • Type Variables: Placeholders for types that you can specify when you use the generic function.
  • Type Inference: TypeScript can often infer the type arguments you pass to a generic function.
  • Constraints: You can restrict the types that can be used with generics.

Basic Syntax of Generic Functions

A generic function uses type variables to define the types it can work with. These type variables are specified within angle brackets (<>) after the function name.

function identity<T>(arg: T): T {
    return arg;
}

In this example:

  • T is a type variable.
  • identity is a generic function that takes an argument arg of type T and returns a value of type T.

Using Generic Functions

You can call a generic function with different types:

let output1 = identity<string>("Hello, TypeScript!");
let output2 = identity<number>(42);

TypeScript can also infer the type argument:

let output3 = identity("Hello, TypeScript!"); // inferred as string
let output4 = identity(42); // inferred as number

Generic Functions with Multiple Type Variables

You can define generic functions with multiple type variables:

function pair<T, U>(first: T, second: U): [T, U] {
    return [first, second];
}

let pair1 = pair<string, number>("Age", 30);
let pair2 = pair<boolean, string>(true, "Success");

Constraints in Generics

Sometimes, you need to restrict the types that can be used with a generic function. You can achieve this using constraints.

interface Lengthwise {
    length: number;
}

function logLength<T extends Lengthwise>(arg: T): T {
    console.log(arg.length);
    return arg;
}

logLength("Hello, TypeScript!"); // Works, string has a length property
logLength([1, 2, 3]); // Works, array has a length property
// logLength(42); // Error: number doesn't have a length property

Practical Examples

Generic Function for Array Operations

A common use case for generics is creating utility functions for arrays.

function getFirstElement<T>(arr: T[]): T | undefined {
    return arr[0];
}

let firstString = getFirstElement<string>(["apple", "banana", "cherry"]);
let firstNumber = getFirstElement<number>([1, 2, 3, 4, 5]);

Generic Function with Default Type

You can provide a default type for a generic function.

function createArray<T = string>(length: number, value: T): T[] {
    return Array(length).fill(value);
}

let stringArray = createArray(3, "default"); // Inferred as string[]
let numberArray = createArray<number>(3, 42); // Explicitly number[]

Conclusion

Generic functions in TypeScript are a powerful feature that allows you to write flexible and reusable code. By understanding the basic syntax, how to use multiple type variables, and how to apply constraints, you can create robust functions that work with a variety of types. Practice using generics in different scenarios to become more comfortable with this concept and enhance your TypeScript programming skills.
