Introduction

Setting up the TypeScript environment is the first step to start developing applications using TypeScript. This section will guide you through the process of installing TypeScript, configuring your development environment, and writing your first TypeScript program.

Prerequisites

Before you begin, ensure you have the following:

  • A computer with an operating system (Windows, macOS, or Linux).
  • Node.js installed on your machine. You can download it from nodejs.org.

Installing TypeScript

To install TypeScript, you need to use npm (Node Package Manager), which comes with Node.js. Open your terminal or command prompt and run the following command:

npm install -g typescript

This command installs TypeScript globally on your system, making the tsc (TypeScript Compiler) command available from any directory.

Verifying the Installation

To verify that TypeScript has been installed correctly, run the following command in your terminal:

tsc --version

You should see the version number of TypeScript printed on the screen, indicating that the installation was successful.

Setting Up a TypeScript Project

  1. Create a Project Directory: Start by creating a new directory for your TypeScript project.

    mkdir my-typescript-project
cd my-typescript-project

  2. Initialize a Node.js Project: Initialize a new Node.js project by running:

    npm init -y

    This command creates a package.json file with default settings.

  3. Install TypeScript Locally: Although we installed TypeScript globally, it's a good practice to install it locally in your project as well.

    npm install typescript --save-dev

  4. Create a tsconfig.json File: The tsconfig.json file is used to configure the TypeScript compiler options. Create this file in the root of your project directory:

    tsc --init

    This command generates a tsconfig.json file with default settings.

Configuring tsconfig.json

The tsconfig.json file contains various options to customize the TypeScript compiler. Here are some key options:

  • target: Specifies the JavaScript version to which TypeScript code will be compiled.
  • module: Defines the module system to be used (e.g., CommonJS, ES6).
  • outDir: Specifies the output directory for compiled JavaScript files.
  • rootDir: Specifies the root directory of TypeScript source files.

Example tsconfig.json:

{
  "compilerOptions": {
    "target": "es6",
    "module": "commonjs",
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true
  }
}

Writing Your First TypeScript Program

  1. Create a Source Directory: Create a directory named src to store your TypeScript files.

    mkdir src

  2. Create a TypeScript File: Inside the src directory, create a file named index.ts.

    // src/index.ts
let message: string = "Hello, TypeScript!";
console.log(message);

  3. Compile the TypeScript File: Use the TypeScript compiler to compile your TypeScript code to JavaScript.

    tsc

    This command reads the tsconfig.json file and compiles the TypeScript files in the src directory, outputting the JavaScript files to the dist directory.

  4. Run the Compiled JavaScript: Use Node.js to run the compiled JavaScript file.

    node dist/index.js

    You should see the message "Hello, TypeScript!" printed on the screen.

Conclusion

Setting up the TypeScript environment involves installing TypeScript, configuring the compiler options, and writing your first TypeScript program. By following these steps, you have laid the foundation for developing applications using TypeScript. In the next sections, we will dive deeper into TypeScript's features and capabilities, building on this initial setup.
