Error Handling in TypeScript
Refactoring TypeScript Code ►

Error handling is a crucial aspect of programming that ensures your application can gracefully handle unexpected situations. In TypeScript, error handling is similar to JavaScript but with added type safety features. This section will guide you through the basics to advanced concepts of error handling in TypeScript.

Basics of Error Handling

The `try...catch` Statement

The try...catch statement allows you to catch errors that occur during runtime and handle them appropriately.

try {
    // Code that may throw an error
    let result = riskyOperation();
    console.log(result);
} catch (error) {
    // Handle the error
    console.error("An error occurred:", error);
}
  • try block: Contains code that might throw an error.
  • catch block: Contains code to handle the error.

Throwing Errors

You can throw errors using the throw statement. This is useful for custom error handling.

function riskyOperation() {
    if (Math.random() > 0.5) {
        throw new Error("Something went wrong!");
    }
    return "Success!";
}

try {
    let result = riskyOperation();
    console.log(result);
} catch (error) {
    console.error("Caught an error:", error.message);
}

Advanced Error Handling

Custom Error Classes

Creating custom error classes can help you handle different types of errors more effectively.

class CustomError extends Error {
    constructor(message: string) {
        super(message);
        this.name = "CustomError";
    }
}

function riskyOperation() {
    throw new CustomError("This is a custom error!");
}

try {
    riskyOperation();
} catch (error) {
    if (error instanceof CustomError) {
        console.error("Caught a custom error:", error.message);
    } else {
        console.error("Caught an unknown error:", error);
    }
}

Using `finally` Block

The finally block contains code that will run regardless of whether an error was thrown or not.

try {
    let result = riskyOperation();
    console.log(result);
} catch (error) {
    console.error("An error occurred:", error);
} finally {
    console.log("This will always run.");
}

Error Handling with Promises

Handling errors in asynchronous code is crucial. Promises provide a catch method to handle errors.

function asyncOperation(): Promise<string> {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            if (Math.random() > 0.5) {
                resolve("Success!");
            } else {
                reject(new Error("Async operation failed!"));
            }
        }, 1000);
    });
}

asyncOperation()
    .then(result => {
        console.log(result);
    })
    .catch(error => {
        console.error("Caught an error:", error.message);
    });

Error Handling with `async/await`

async/await syntax makes it easier to work with asynchronous code and handle errors.

async function performAsyncOperation() {
    try {
        let result = await asyncOperation();
        console.log(result);
    } catch (error) {
        console.error("Caught an error:", error.message);
    }
}

performAsyncOperation();

Conclusion

Error handling in TypeScript involves using try...catch statements, creating custom error classes, and handling errors in asynchronous code using Promises and async/await. By understanding and implementing these techniques, you can create robust applications that can gracefully handle unexpected situations.
