Introduction

In TypeScript, classes and interfaces are fundamental building blocks for creating robust and maintainable code. This section will guide you from the basics to advanced concepts, ensuring you understand how to define, implement, and utilize classes and interfaces effectively.

Classes in TypeScript

Basic Class Structure

A class in TypeScript is a blueprint for creating objects with predefined properties and methods.

class Person {
    name: string;
    age: number;

    constructor(name: string, age: number) {
        this.name = name;
        this.age = age;
    }

    greet() {
        console.log(`Hello, my name is ${this.name} and I am ${this.age} years old.`);
    }
}

const person1 = new Person('Alice', 30);
person1.greet(); // Output: Hello, my name is Alice and I am 30 years old.
  • Properties: Variables that hold data specific to each instance of the class.
  • Constructor: A special method for initializing new objects.
  • Methods: Functions defined within a class that operate on the class's properties.

Inheritance

Inheritance allows a class to extend another class, inheriting its properties and methods.

class Employee extends Person {
    employeeId: number;

    constructor(name: string, age: number, employeeId: number) {
        super(name, age);
        this.employeeId = employeeId;
    }

    displayEmployeeId() {
        console.log(`Employee ID: ${this.employeeId}`);
    }
}

const employee1 = new Employee('Bob', 25, 12345);
employee1.greet(); // Output: Hello, my name is Bob and I am 25 years old.
employee1.displayEmployeeId(); // Output: Employee ID: 12345
  • extends keyword: Used to create a subclass.
  • super keyword: Calls the constructor of the base class.

Access Modifiers

TypeScript provides access modifiers to control the visibility of class members.

  • public: Accessible from anywhere.
  • private: Accessible only within the class.
  • protected: Accessible within the class and its subclasses.
class Animal {
    public name: string;
    private age: number;
    protected species: string;

    constructor(name: string, age: number, species: string) {
        this.name = name;
        this.age = age;
        this.species = species;
    }

    public getAge() {
        return this.age;
    }
}

class Dog extends Animal {
    constructor(name: string, age: number, species: string) {
        super(name, age, species);
    }

    public getSpecies() {
        return this.species;
    }
}

const dog1 = new Dog('Buddy', 5, 'Canine');
console.log(dog1.name); // Output: Buddy
console.log(dog1.getAge()); // Output: 5
console.log(dog1.getSpecies()); // Output: Canine

Interfaces in TypeScript

Defining Interfaces

Interfaces define the structure that an object should adhere to.

interface Person {
    name: string;
    age: number;
    greet(): void;
}

const person: Person = {
    name: 'Charlie',
    age: 40,
    greet() {
        console.log(`Hello, my name is ${this.name} and I am ${this.age} years old.`);
    }
};

person.greet(); // Output: Hello, my name is Charlie and I am 40 years old.
  • Properties: Define the required properties and their types.
  • Methods: Define the required methods and their signatures.

Implementing Interfaces in Classes

Classes can implement interfaces to ensure they adhere to a specific structure.

interface Drivable {
    startEngine(): void;
    stopEngine(): void;
}

class Car implements Drivable {
    startEngine() {
        console.log('Engine started.');
    }

    stopEngine() {
        console.log('Engine stopped.');
    }
}

const myCar = new Car();
myCar.startEngine(); // Output: Engine started.
myCar.stopEngine(); // Output: Engine stopped.
  • implements keyword: Ensures the class adheres to the interface structure.

Extending Interfaces

Interfaces can extend other interfaces, allowing for more complex structures.

interface Animal {
    name: string;
    age: number;
}

interface Mammal extends Animal {
    hasFur: boolean;
}

const mammal: Mammal = {
    name: 'Elephant',
    age: 10,
    hasFur: false
};

console.log(mammal); // Output: { name: 'Elephant', age: 10, hasFur: false }

Advanced Concepts

Abstract Classes

Abstract classes cannot be instantiated directly and are meant to be subclassed.

abstract class Shape {
    abstract area(): number;

    displayArea() {
        console.log(`Area: ${this.area()}`);
    }
}

class Circle extends Shape {
    radius: number;

    constructor(radius: number) {
        super();
        this.radius = radius;
    }

    area(): number {
        return Math.PI * this.radius * this.radius;
    }
}

const circle = new Circle(5);
circle.displayArea(); // Output: Area: 78.53981633974483
  • abstract keyword: Used to define abstract classes and methods.

Intersection Types

Combining multiple types into one using intersection types.

interface CanFly {
    fly(): void;
}

interface CanSwim {
    swim(): void;
}

type AmphibiousVehicle = CanFly & CanSwim;

const vehicle: AmphibiousVehicle = {
    fly() {
        console.log('Flying.');
    },
    swim() {
        console.log('Swimming.');
    }
};

vehicle.fly(); // Output: Flying.
vehicle.swim(); // Output: Swimming.

Conclusion

Understanding classes and interfaces in TypeScript is crucial for building scalable and maintainable applications. Classes provide a blueprint for creating objects, while interfaces define the structure that objects should adhere to. By mastering these concepts, you can write more robust and type-safe code.
