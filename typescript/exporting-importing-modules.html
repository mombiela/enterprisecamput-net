Exporting and Importing Modules
Namespaces ►

In this section, we will explore how to export and import modules in TypeScript. Modules are a way to organize and encapsulate code, making it easier to manage and reuse. Understanding how to work with modules is crucial for building scalable and maintainable applications.

Key Concepts

  • Modules: A module is a file containing TypeScript code. Modules can export classes, functions, variables, and interfaces to be used in other modules.
  • Exporting: Making code available to be imported into other modules.
  • Importing: Bringing in code from other modules to use in the current module.
  • Default Exports: A single export from a module that can be imported without curly braces.
  • Named Exports: Multiple exports from a module that must be imported using curly braces.

Exporting Modules

Named Exports

Named exports allow you to export multiple items from a module. Each item must be imported using its exact name.

// mathUtils.ts
export const add = (a: number, b: number): number => {
    return a + b;
};

export const subtract = (a: number, b: number): number => {
    return a - b;
};

Default Exports

Default exports allow you to export a single item from a module. This item can be imported without using curly braces.

// logger.ts
const log = (message: string): void => {
    console.log(message);
};

export default log;

Importing Modules

Importing Named Exports

To import named exports, you use curly braces and the exact names of the exports.

// app.ts
import { add, subtract } from './mathUtils';

console.log(add(5, 3)); // Output: 8
console.log(subtract(5, 3)); // Output: 2

Importing Default Exports

To import default exports, you simply use the name you want to assign to the imported module.

// app.ts
import log from './logger';

log('This is a log message'); // Output: This is a log message

Importing All Exports

You can import all exports from a module using the * as syntax. This is useful when you want to group all exports under a single namespace.

// app.ts
import * as MathUtils from './mathUtils';

console.log(MathUtils.add(5, 3)); // Output: 8
console.log(MathUtils.subtract(5, 3)); // Output: 2

Combining Default and Named Exports

A module can have both default and named exports. You can import them together as shown below:

// utils.ts
export const multiply = (a: number, b: number): number => {
    return a * b;
};

const divide = (a: number, b: number): number => {
    return a / b;
};

export default divide;
// app.ts
import divide, { multiply } from './utils';

console.log(multiply(6, 3)); // Output: 18
console.log(divide(6, 3)); // Output: 2

Conclusion

Understanding how to export and import modules in TypeScript is essential for writing modular, maintainable, and reusable code. By using named exports, default exports, and importing techniques, you can effectively manage dependencies and organize your codebase. Practice these concepts with the provided examples and exercises to solidify your understanding.
