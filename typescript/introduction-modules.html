Exporting and Importing Modules ►

Modules are a fundamental part of TypeScript and modern JavaScript, allowing you to organize and manage your code more effectively. This section will introduce you to the concept of modules, how to create them, and how to use them in your TypeScript projects.

What are Modules?

Modules are reusable pieces of code that can be imported and exported between different parts of your application. They help in organizing code, avoiding global scope pollution, and managing dependencies.

Key Concepts

  • Exporting: Making variables, functions, classes, or interfaces available to other modules.
  • Importing: Bringing in variables, functions, classes, or interfaces from other modules into the current module.
  • Default Export: A single export from a module that can be imported without specifying a name.
  • Named Export: Multiple exports from a module that must be imported using their specific names.

Exporting and Importing

Exporting

You can export variables, functions, classes, or interfaces from a module using the export keyword.

Example: Named Export

// mathUtils.ts
export function add(a: number, b: number): number {
    return a + b;
}

export function subtract(a: number, b: number): number {
    return a - b;
}

Example: Default Export

// logger.ts
export default function log(message: string): void {
    console.log(message);
}

Importing

You can import variables, functions, classes, or interfaces from other modules using the import keyword.

Example: Importing Named Exports

// main.ts
import { add, subtract } from './mathUtils';

console.log(add(5, 3)); // Output: 8
console.log(subtract(5, 3)); // Output: 2

Example: Importing Default Export

// main.ts
import log from './logger';

log('This is a log message'); // Output: This is a log message

Combining Named and Default Exports

You can combine named and default exports in a single module.

// utils.ts
export function multiply(a: number, b: number): number {
    return a * b;
}

export default function divide(a: number, b: number): number {
    return a / b;
}
// main.ts
import divide, { multiply } from './utils';

console.log(multiply(6, 3)); // Output: 18
console.log(divide(6, 3)); // Output: 2

Re-exporting

You can re-export modules to create a single entry point for multiple modules.

// mathOperations.ts
export { add, subtract } from './mathUtils';
export { multiply, default as divide } from './utils';
// main.ts
import { add, subtract, multiply, divide } from './mathOperations';

console.log(add(5, 3)); // Output: 8
console.log(subtract(5, 3)); // Output: 2
console.log(multiply(6, 3)); // Output: 18
console.log(divide(6, 3)); // Output: 2

Conclusion

Modules are a powerful feature in TypeScript that help you organize and manage your code more effectively. By understanding how to export and import modules, you can create more modular, maintainable, and scalable applications. Practice creating and using modules in your projects to become more comfortable with this essential concept.
