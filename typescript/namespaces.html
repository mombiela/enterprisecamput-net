Namespaces in TypeScript
Module Resolution ►

Namespaces in TypeScript provide a way to organize and group related code, making it easier to manage and avoid naming conflicts. They are particularly useful in large codebases where multiple developers might be working on different parts of the application.

Introduction to Namespaces

Namespaces are a TypeScript-specific feature that allows you to group related code together. They are similar to modules but are used to organize code within a single file or across multiple files.

Key Concepts

  • Namespace Declaration: Use the namespace keyword to declare a namespace.
  • Exporting Members: Use the export keyword to make variables, functions, classes, or interfaces available outside the namespace.
  • Nested Namespaces: Namespaces can be nested within other namespaces.
  • Referencing Namespaces: Use the /// <reference path="..."/> directive to reference namespaces across different files.

Creating and Using Namespaces

Basic Namespace Declaration

To declare a namespace, use the namespace keyword followed by the namespace name and a block of code.

namespace Utility {
    export function log(message: string): void {
        console.log(message);
    }

    export function error(message: string): void {
        console.error(message);
    }
}

In this example, we have created a namespace called Utility with two exported functions: log and error.

Accessing Namespace Members

To access the members of a namespace, use the dot notation.

Utility.log("This is a log message.");
Utility.error("This is an error message.");

Nested Namespaces

Namespaces can be nested to further organize code.

namespace App {
    export namespace Models {
        export class User {
            constructor(public name: string, public age: number) {}
        }
    }

    export namespace Services {
        export function getUserInfo(user: Models.User): string {
            return `${user.name} is ${user.age} years old.`;
        }
    }
}

Accessing nested namespace members:

const user = new App.Models.User("Alice", 30);
console.log(App.Services.getUserInfo(user));

Namespaces Across Multiple Files

To use namespaces across multiple files, you need to reference them using the /// <reference path="..."/> directive.

File: `models.ts`

namespace App.Models {
    export class User {
        constructor(public name: string, public age: number) {}
    }
}

File: `services.ts`

/// <reference path="models.ts" />

namespace App.Services {
    export function getUserInfo(user: App.Models.User): string {
        return `${user.name} is ${user.age} years old.`;
    }
}

File: `main.ts`

/// <reference path="models.ts" />
/// <reference path="services.ts" />

const user = new App.Models.User("Alice", 30);
console.log(App.Services.getUserInfo(user));

Conclusion

Namespaces in TypeScript are a powerful tool for organizing and managing code, especially in large projects. They help avoid naming conflicts and make the codebase more maintainable. By understanding how to declare, access, and reference namespaces, you can effectively structure your TypeScript applications.

Summary

  • Namespaces: Group related code together using the namespace keyword.
  • Exporting Members: Use the export keyword to make members accessible outside the namespace.
  • Nested Namespaces: Organize code further by nesting namespaces.
  • Referencing Across Files: Use the /// <reference path="..."/> directive to reference namespaces in different files.

By mastering these concepts, you'll be well-equipped to manage complex TypeScript projects with ease.
