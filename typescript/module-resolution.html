Module Resolution

Module resolution is a critical concept in TypeScript that determines how the compiler finds and connects modules in your code. Understanding module resolution helps in organizing your codebase efficiently and avoiding common pitfalls related to module imports.

Key Concepts

  • Module: A file containing TypeScript code. Modules can export and import functionalities.
  • Module Resolution: The process the TypeScript compiler uses to locate and load modules.
  • Module Specifier: The string in the import statement that tells the compiler which module to load.
  • Base URL: A compiler option that sets the base directory for non-relative module names.
  • Paths: A compiler option that allows you to define module aliasing.

Module Resolution Strategies

TypeScript uses two primary strategies for module resolution:

  1. Classic: The default strategy used in TypeScript 1.6 and earlier.
  2. Node: The strategy that mimics Node.js module resolution, suitable for Node.js applications.

Classic Module Resolution

In the classic strategy, TypeScript tries to locate the module by looking in the following order:

  1. Relative Path: If the module specifier starts with ./ or ../, TypeScript treats it as a relative path.
  2. Non-relative Path: If the module specifier does not start with ./ or ../, TypeScript treats it as a non-relative path and searches in the baseUrl and paths.

Node Module Resolution

The Node strategy is more complex and follows these steps:

  1. Relative Path: Similar to the classic strategy, if the module specifier starts with ./ or ../, TypeScript treats it as a relative path.
  2. Non-relative Path: For non-relative paths, TypeScript searches in node_modules directories, starting from the current directory and moving up the directory tree.
  3. File Extensions: TypeScript tries to resolve the module by appending .ts, .tsx, and .d.ts extensions.
  4. Index Files: If the module specifier points to a directory, TypeScript looks for an index.ts, index.tsx, or index.d.ts file in that directory.

Configuring Module Resolution

You can configure module resolution in the tsconfig.json file using the moduleResolution option.

{
  "compilerOptions": {
    "moduleResolution": "node", // or "classic"
    "baseUrl": "./src",
    "paths": {
      "@app/*": ["app/*"]
    }
  }
}
  • moduleResolution: Specifies the module resolution strategy (node or classic).
  • baseUrl: Sets the base directory for resolving non-relative module names.
  • paths: Defines module aliasing, allowing you to map module specifiers to specific paths.

Examples

Relative Module Import

// File: src/utils/math.ts
export function add(a: number, b: number): number {
  return a + b;
}

// File: src/index.ts
import { add } from './utils/math';

console.log(add(2, 3)); // Output: 5

Non-relative Module Import with Base URL and Paths

// File: src/app/services/logger.ts
export function log(message: string): void {
  console.log(message);
}

// File: src/index.ts
import { log } from '@app/services/logger';

log('Hello, World!'); // Output: Hello, World!

Node Module Resolution

// File: src/index.ts
import * as fs from 'fs';

fs.writeFileSync('test.txt', 'Hello, World!');

Conclusion

Understanding module resolution in TypeScript is essential for managing dependencies and organizing your codebase effectively. By configuring the tsconfig.json file appropriately and knowing the differences between classic and node module resolution strategies, you can avoid common pitfalls and ensure that your modules are correctly located and loaded by the TypeScript compiler.
