Mapped Types in TypeScript
Code Style and Formatting ►

Mapped types are a powerful feature in TypeScript that allow you to create new types by transforming existing ones. They are particularly useful for creating types that are variations of other types, such as making all properties optional or readonly.

Key Concepts

  • Mapped Types: Types that transform properties of an existing type.
  • TypeScript Utility Types: Predefined mapped types provided by TypeScript, such as Partial, Readonly, Record, and Pick.

Basic Mapped Types

Mapped types use the keyof operator and square brackets to iterate over the properties of an existing type and transform them.

Example: Making All Properties Optional

type Person = {
  name: string;
  age: number;
};

type OptionalPerson = {
  [P in keyof Person]?: Person[P];
};

// Equivalent to:
// type OptionalPerson = {
//   name?: string;
//   age?: number;
// };

In this example, OptionalPerson is a mapped type that makes all properties of Person optional.

TypeScript Utility Types

TypeScript provides several utility types that are built using mapped types. These utility types simplify common type transformations.

Partial

The Partial utility type makes all properties of a type optional.

type Person = {
  name: string;
  age: number;
};

type PartialPerson = Partial<Person>;

// Equivalent to:
// type PartialPerson = {
//   name?: string;
//   age?: number;
// };

Readonly

The Readonly utility type makes all properties of a type readonly.

type Person = {
  name: string;
  age: number;
};

type ReadonlyPerson = Readonly<Person>;

// Equivalent to:
// type ReadonlyPerson = {
//   readonly name: string;
//   readonly age: number;
// };

Record

The Record utility type constructs a type with a set of properties K of type T.

type PageInfo = {
  title: string;
};

type Page = "home" | "about" | "contact";

type Pages = Record<Page, PageInfo>;

// Equivalent to:
// type Pages = {
//   home: PageInfo;
//   about: PageInfo;
//   contact: PageInfo;
// };

Pick

The Pick utility type constructs a type by picking a set of properties K from type T.

type Person = {
  name: string;
  age: number;
  address: string;
};

type PersonNameAndAge = Pick<Person, "name" | "age">;

// Equivalent to:
// type PersonNameAndAge = {
//   name: string;
//   age: number;
// };

Custom Mapped Types

You can create your own custom mapped types to fit specific needs.

Example: Making All Properties Nullable

type Person = {
  name: string;
  age: number;
};

type Nullable<T> = {
  [P in keyof T]: T[P] | null;
};

type NullablePerson = Nullable<Person>;

// Equivalent to:
// type NullablePerson = {
//   name: string | null;
//   age: number | null;
// };

Conclusion

Mapped types in TypeScript provide a flexible way to transform existing types into new ones. By understanding and utilizing mapped types, you can create more dynamic and reusable type definitions. TypeScript's built-in utility types like Partial, Readonly, Record, and Pick are practical examples of how mapped types can simplify type transformations. Custom mapped types further extend this capability, allowing you to tailor type transformations to your specific needs.
