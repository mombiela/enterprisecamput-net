Type Annotations in TypeScript
Compiling TypeScript ►

Type annotations are a powerful feature in TypeScript that allow you to specify the types of variables, function parameters, and return values. This helps catch errors early and makes your code more readable and maintainable.

What are Type Annotations?

Type annotations are a way to explicitly declare the type of a variable, function parameter, or return value. This helps TypeScript understand what kind of data is being used and can prevent many common programming errors.

Key Concepts

  • Type Inference: TypeScript can often infer the type of a variable based on its value.
  • Explicit Annotations: You can explicitly declare the type of a variable, function parameter, or return value.
  • Basic Types: Common types include string, number, boolean, array, tuple, enum, any, void, null, and undefined.

Basic Type Annotations

Variables

You can annotate variables with a specific type using a colon followed by the type name.

let username: string = "JohnDoe";
let age: number = 30;
let isAdmin: boolean = true;

Functions

Annotate function parameters and return types to ensure they are used correctly.

function add(a: number, b: number): number {
    return a + b;
}

let result: number = add(5, 3);

Arrays

Annotate arrays by specifying the type of elements they contain.

let numbers: number[] = [1, 2, 3, 4, 5];
let names: string[] = ["Alice", "Bob", "Charlie"];

Advanced Type Annotations

Tuples

Tuples are arrays with a fixed number of elements of specific types.

let person: [string, number] = ["Alice", 25];

Enums

Enums allow you to define a set of named constants.

enum Color {
    Red,
    Green,
    Blue
}

let favoriteColor: Color = Color.Green;

Any

The any type can hold any value, but it should be used sparingly as it bypasses type checking.

let randomValue: any = 42;
randomValue = "Hello";
randomValue = true;

Void

The void type is used for functions that do not return a value.

function logMessage(message: string): void {
    console.log(message);
}

Null and Undefined

You can explicitly annotate variables as null or undefined.

let emptyValue: null = null;
let notAssigned: undefined = undefined;

Type Annotations in Practice

Example: User Profile

Let's create a simple user profile with type annotations.

type UserProfile = {
    username: string;
    age: number;
    isAdmin: boolean;
};

let user: UserProfile = {
    username: "JohnDoe",
    age: 30,
    isAdmin: true
};

function printUserProfile(user: UserProfile): void {
    console.log(`Username: ${user.username}, Age: ${user.age}, Admin: ${user.isAdmin}`);
}

printUserProfile(user);

Example: Shopping Cart

A more complex example involving arrays and functions.

type Product = {
    id: number;
    name: string;
    price: number;
};

let cart: Product[] = [];

function addToCart(product: Product): void {
    cart.push(product);
}

function calculateTotal(cart: Product[]): number {
    return cart.reduce((total, product) => total + product.price, 0);
}

addToCart({ id: 1, name: "Laptop", price: 999.99 });
addToCart({ id: 2, name: "Mouse", price: 19.99 });

console.log(`Total: $${calculateTotal(cart)}`);

Conclusion

Type annotations in TypeScript provide a robust way to ensure your code is type-safe and easier to understand. By explicitly declaring the types of variables, function parameters, and return values, you can catch errors early and make your code more maintainable. As you become more comfortable with TypeScript, you'll find that type annotations are an invaluable tool in your programming toolkit.
