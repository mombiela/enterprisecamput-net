Conditional Types in TypeScript
Mapped Types ►

Introduction

Conditional types in TypeScript allow you to create types that depend on a condition. They are a powerful feature that can make your type definitions more flexible and expressive. This topic will guide you from the basics to advanced usage of conditional types.

Basic Syntax

Conditional types follow a simple syntax:

T extends U ? X : Y
  • T extends U is the condition.
  • If the condition is true, the type resolves to X.
  • If the condition is false, the type resolves to Y.

Example

Let's start with a basic example:

type IsString<T> = T extends string ? "Yes" : "No";

type A = IsString<string>; // "Yes"
type B = IsString<number>; // "No"

In this example:

  • IsString<string> evaluates to "Yes" because string extends string.
  • IsString<number> evaluates to "No" because number does not extend string.

Conditional Types with Generics

Conditional types become more powerful when used with generics.

Example

Consider a function that returns different types based on a condition:

type ReturnType<T> = T extends () => infer R ? R : never;

type Func = () => string;
type Result = ReturnType<Func>; // string

Here:

  • ReturnType<T> checks if T is a function type.
  • If true, it extracts the return type R using infer.
  • If false, it resolves to never.

Distributive Conditional Types

Conditional types distribute over unions. This means that if you apply a conditional type to a union type, TypeScript will apply the conditional type to each member of the union individually.

Example

type ToArray<T> = T extends any ? T[] : never;

type StrArray = ToArray<string>; // string[]
type NumArray = ToArray<number>; // number[]
type UnionArray = ToArray<string | number>; // string[] | number[]

In this example:

  • ToArray<string | number> distributes over the union, resulting in string[] | number[].

Advanced Usage

Conditional types can be combined with other TypeScript features to create complex and powerful type definitions.

Example: Filtering Types

You can use conditional types to filter out certain types from a union.

type Exclude<T, U> = T extends U ? never : T;

type T1 = Exclude<"a" | "b" | "c", "a">; // "b" | "c"
type T2 = Exclude<string | number | (() => void), Function>; // string | number

Here:

  • Exclude<T, U> removes types from T that are assignable to U.

Practical Application

Conditional types are often used in real-world TypeScript projects to create more precise and flexible types.

Example: Deep Partial

A common utility type is DeepPartial, which makes all properties of a type optional, recursively.

type DeepPartial<T> = {
  [P in keyof T]?: T[P] extends object ? DeepPartial<T[P]> : T[P];
};

interface User {
  name: string;
  address: {
    street: string;
    city: string;
  };
}

type PartialUser = DeepPartial<User>;
/*
PartialUser is:
{
  name?: string;
  address?: {
    street?: string;
    city?: string;
  };
}
*/

In this example:

  • DeepPartial<T> uses a conditional type to check if a property is an object.
  • If it is, it applies DeepPartial recursively.

Conclusion

Conditional types in TypeScript provide a powerful way to create flexible and expressive type definitions. They allow you to define types that depend on conditions, work with generics, and even distribute over unions. By mastering conditional types, you can write more robust and maintainable TypeScript code.
