Static properties and methods are an essential concept in TypeScript and object-oriented programming. They belong to the class itself rather than to instances of the class. This topic will cover the basics of static properties and methods, their usage, and provide practical examples to solidify your understanding.

What are Static Properties and Methods?

  • Static Properties: These are properties that belong to the class itself, not to any specific instance of the class.
  • Static Methods: These are methods that can be called on the class itself, not on instances of the class.

Why Use Static Properties and Methods?

  • Shared Data: Static properties can be used to store data that is shared among all instances of a class.
  • Utility Functions: Static methods are often used for utility functions that do not require any data from instances of the class.

Defining Static Properties and Methods

In TypeScript, static properties and methods are defined using the static keyword.

Example: Static Properties

class MyClass {
    static staticProperty: string = "I am a static property";

    static showStaticProperty() {
        console.log(MyClass.staticProperty);
    }
}

// Accessing static property without creating an instance
console.log(MyClass.staticProperty); // Output: I am a static property

Example: Static Methods

class MathUtils {
    static add(a: number, b: number): number {
        return a + b;
    }
}

// Calling static method without creating an instance
console.log(MathUtils.add(5, 3)); // Output: 8

Practical Use Cases

Singleton Pattern

The Singleton pattern ensures that a class has only one instance and provides a global point of access to it. Static properties and methods are often used to implement this pattern.

class Singleton {
    private static instance: Singleton;

    private constructor() {
        // Private constructor to prevent instantiation
    }

    static getInstance(): Singleton {
        if (!Singleton.instance) {
            Singleton.instance = new Singleton();
        }
        return Singleton.instance;
    }

    showMessage(): void {
        console.log("Hello, I am a singleton instance!");
    }
}

const singleton1 = Singleton.getInstance();
const singleton2 = Singleton.getInstance();

singleton1.showMessage(); // Output: Hello, I am a singleton instance!
console.log(singleton1 === singleton2); // Output: true

Utility Classes

Utility classes often contain static methods that perform common tasks.

class StringUtils {
    static toUpperCase(str: string): string {
        return str.toUpperCase();
    }

    static toLowerCase(str: string): string {
        return str.toLowerCase();
    }
}

console.log(StringUtils.toUpperCase("hello")); // Output: HELLO
console.log(StringUtils.toLowerCase("WORLD")); // Output: world

Key Points to Remember

  • Static properties and methods belong to the class itself, not to instances of the class.
  • Use the static keyword to define static properties and methods.
  • Static methods are often used for utility functions and shared data.
  • Static properties and methods can be accessed directly using the class name.

Conclusion

Static properties and methods are powerful tools in TypeScript that allow you to define properties and methods that belong to the class itself rather than to instances of the class. They are particularly useful for shared data, utility functions, and design patterns like the Singleton pattern. Understanding how to use static properties and methods effectively can help you write more organized and efficient code.
