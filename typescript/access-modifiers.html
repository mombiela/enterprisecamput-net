Access Modifiers in TypeScript
Abstract Classes ►

Access modifiers in TypeScript are keywords that set the accessibility of properties and methods in classes. They help in encapsulating the data and controlling its visibility from outside the class.

Key Access Modifiers

TypeScript provides three main access modifiers:

  • public
  • private
  • protected

Public

  • The public modifier is the default access level.
  • Members marked as public can be accessed from anywhere, both inside and outside the class.
class Animal {
    public name: string;

    constructor(name: string) {
        this.name = name;
    }

    public makeSound() {
        console.log(`${this.name} makes a sound.`);
    }
}

const dog = new Animal('Dog');
console.log(dog.name); // Accessible
dog.makeSound(); // Accessible

Private

  • The private modifier restricts access to the members within the same class.
  • Members marked as private cannot be accessed from outside the class.
class Animal {
    private name: string;

    constructor(name: string) {
        this.name = name;
    }

    private makeSound() {
        console.log(`${this.name} makes a sound.`);
    }

    public revealName() {
        console.log(`The animal's name is ${this.name}`);
    }
}

const cat = new Animal('Cat');
// console.log(cat.name); // Error: Property 'name' is private and only accessible within class 'Animal'.
// cat.makeSound(); // Error: Method 'makeSound' is private and only accessible within class 'Animal'.
cat.revealName(); // Accessible

Protected

  • The protected modifier allows access to the members within the class and its subclasses.
  • Members marked as protected cannot be accessed from outside the class or its subclasses.
class Animal {
    protected name: string;

    constructor(name: string) {
        this.name = name;
    }

    protected makeSound() {
        console.log(`${this.name} makes a sound.`);
    }
}

class Dog extends Animal {
    constructor(name: string) {
        super(name);
    }

    public bark() {
        console.log(`${this.name} barks.`);
        this.makeSound(); // Accessible within subclass
    }
}

const dog = new Dog('Dog');
// console.log(dog.name); // Error: Property 'name' is protected and only accessible within class 'Animal' and its subclasses.
// dog.makeSound(); // Error: Method 'makeSound' is protected and only accessible within class 'Animal' and its subclasses.
dog.bark(); // Accessible

Comparison Table

| Modifier | Accessible within class | Accessible within subclass | Accessible outside class | |------------|--------------------------|----------------------------|--------------------------| | public | Yes | Yes | Yes | | private | Yes | No | No | | protected| Yes | Yes | No |

Conclusion

Access modifiers in TypeScript are essential for controlling the visibility and accessibility of class members. They help in encapsulating data and ensuring that only the intended parts of your code can interact with certain properties and methods. Understanding and using public, private, and protected modifiers effectively can lead to more robust and maintainable code.
