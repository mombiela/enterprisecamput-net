Introduction to Parameter Decorators

Parameter decorators are a special kind of decorator in TypeScript that can be used to annotate parameters within a class constructor or method. They allow you to modify the behavior of the parameters, providing a way to add metadata or perform validation.

Key Concepts

  • Decorator: A special kind of declaration that can be attached to a class, method, accessor, property, or parameter.
  • Parameter Decorator: A decorator that is applied to a specific parameter of a method or constructor.

Syntax and Usage

Parameter decorators are declared just before the parameter they are decorating. They receive three arguments:

  1. target: The prototype of the class for instance members, or the constructor function for static members.
  2. propertyKey: The name of the method where the parameter is declared.
  3. parameterIndex: The index of the parameter in the function's parameter list.

Basic Example

function logParameter(target: Object, propertyKey: string | symbol, parameterIndex: number) {
    console.log(`Parameter at index ${parameterIndex} in method ${String(propertyKey)} of class ${target.constructor.name}`);
}

class ExampleClass {
    method(@logParameter param1: string, @logParameter param2: number) {
        console.log(param1, param2);
    }
}

const example = new ExampleClass();
example.method('test', 42);

In this example:

  • logParameter is a parameter decorator.
  • It logs the index and name of the parameter it decorates.

Practical Applications

Parameter decorators can be used for various purposes, such as:

  • Validation: Ensuring that parameters meet certain criteria.
  • Dependency Injection: Automatically injecting dependencies into class constructors or methods.
  • Metadata: Adding metadata to parameters for later use.

Validation Example

function required(target: Object, propertyKey: string | symbol, parameterIndex: number) {
    const existingRequiredParameters: number[] = Reflect.getOwnMetadata('required', target, propertyKey) || [];
    existingRequiredParameters.push(parameterIndex);
    Reflect.defineMetadata('required', existingRequiredParameters, target, propertyKey);
}

function validate(target: any, propertyName: string, descriptor: TypedPropertyDescriptor<Function>) {
    const method = descriptor.value!;
    descriptor.value = function (...args: any[]) {
        const requiredParameters: number[] = Reflect.getOwnMetadata('required', target, propertyName) || [];
        for (const parameterIndex of requiredParameters) {
            if (args[parameterIndex] === undefined) {
                throw new Error(`Missing required argument at index ${parameterIndex}`);
            }
        }
        return method.apply(this, args);
    };
}

class User {
    @validate
    setName(@required name: string) {
        console.log(`User name is ${name}`);
    }
}

const user = new User();
user.setName('John'); // Works fine
user.setName(); // Throws an error: Missing required argument at index 0

In this example:

  • required is a parameter decorator that marks a parameter as required.
  • validate is a method decorator that checks if all required parameters are provided.

Conclusion

Parameter decorators in TypeScript provide a powerful way to add metadata, perform validation, and manage dependencies for method and constructor parameters. By understanding and utilizing parameter decorators, you can write more robust and maintainable code. Practice using parameter decorators in different scenarios to fully grasp their potential and applications.
