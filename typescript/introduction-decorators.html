What are Decorators?

Decorators are a special kind of declaration that can be attached to a class, method, accessor, property, or parameter. Decorators are a stage 2 proposal for JavaScript and are available as an experimental feature in TypeScript. They provide a way to add both annotations and a meta-programming syntax for class declarations and members.

Why Use Decorators?

  • Code Reusability: Decorators allow you to reuse common logic across multiple classes or methods.
  • Separation of Concerns: They help in separating cross-cutting concerns like logging, validation, and error handling from the main business logic.
  • Enhanced Readability: By using decorators, you can make your code more readable and maintainable.

Types of Decorators

  • Class Decorators: Used to modify or annotate a class.
  • Method Decorators: Used to modify or annotate a method.
  • Accessor Decorators: Used to modify or annotate a getter or setter.
  • Property Decorators: Used to modify or annotate a property.
  • Parameter Decorators: Used to modify or annotate a parameter.

Basic Syntax

The basic syntax for a decorator in TypeScript is as follows:

function decoratorName(target: any, propertyKey?: string, descriptor?: PropertyDescriptor) {
    // Decorator logic here
}

Examples

Class Decorator

A class decorator is applied to the constructor of the class. It can be used to modify or replace the class definition.

function sealed(constructor: Function) {
    Object.seal(constructor);
    Object.seal(constructor.prototype);
}

@sealed
class Greeter {
    greeting: string;
    constructor(message: string) {
        this.greeting = message;
    }
    greet() {
        return "Hello, " + this.greeting;
    }
}

In this example, the sealed decorator seals the Greeter class, preventing new properties from being added to it.

Method Decorator

A method decorator is applied to a method of a class. It can be used to modify the method's behavior.

function log(target: any, propertyKey: string, descriptor: PropertyDescriptor) {
    const originalMethod = descriptor.value;

    descriptor.value = function (...args: any[]) {
        console.log(`Calling ${propertyKey} with arguments: ${JSON.stringify(args)}`);
        const result = originalMethod.apply(this, args);
        console.log(`Result: ${result}`);
        return result;
    };

    return descriptor;
}

class Calculator {
    @log
    add(a: number, b: number): number {
        return a + b;
    }
}

const calculator = new Calculator();
calculator.add(2, 3);

In this example, the log decorator logs the method name and arguments before and after the method execution.

Property Decorator

A property decorator is applied to a property of a class. It can be used to modify the property.

function readonly(target: any, propertyKey: string) {
    Object.defineProperty(target, propertyKey, {
        writable: false
    });
}

class Person {
    @readonly
    name: string;

    constructor(name: string) {
        this.name = name;
    }
}

const person = new Person("John");
person.name = "Doe"; // Error: Cannot assign to 'name' because it is a read-only property.

In this example, the readonly decorator makes the name property read-only.

Conclusion

Decorators in TypeScript provide a powerful way to add annotations and meta-programming capabilities to your classes and their members. They help in enhancing code reusability, separation of concerns, and readability. By understanding and utilizing decorators, you can write more maintainable and scalable TypeScript applications.
