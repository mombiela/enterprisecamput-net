Optional and Default Parameters in TypeScript

Introduction

In TypeScript, functions can be defined with optional and default parameters. These features enhance the flexibility and robustness of your code by allowing functions to handle cases where some arguments might be missing or have default values.

Optional Parameters

Optional parameters are parameters that may or may not be provided when a function is called. They are defined using a question mark (?) after the parameter name.

Key Concepts

  • Optional parameters must come after required parameters in the function signature.
  • If an optional parameter is not provided, its value is undefined.

Example

function greet(name: string, greeting?: string): string {
    if (greeting) {
        return `${greeting}, ${name}!`;
    } else {
        return `Hello, ${name}!`;
    }
}

console.log(greet("Alice")); // Output: Hello, Alice!
console.log(greet("Bob", "Good morning")); // Output: Good morning, Bob!

In this example:

  • The greeting parameter is optional.
  • If greeting is not provided, the function defaults to saying "Hello".

Default Parameters

Default parameters allow you to specify a default value for a parameter. If the argument is not provided, the default value is used.

Key Concepts

  • Default parameters can be used in any position in the parameter list.
  • If a value is provided for a default parameter, it overrides the default value.

Example

function greet(name: string, greeting: string = "Hello"): string {
    return `${greeting}, ${name}!`;
}

console.log(greet("Alice")); // Output: Hello, Alice!
console.log(greet("Bob", "Good morning")); // Output: Good morning, Bob!

In this example:

  • The greeting parameter has a default value of "Hello".
  • If greeting is not provided, "Hello" is used.

Combining Optional and Default Parameters

You can combine optional and default parameters in a function. However, keep in mind that optional parameters must come after required parameters.

Example

function greet(name: string, greeting?: string, punctuation: string = "!"): string {
    if (greeting) {
        return `${greeting}, ${name}${punctuation}`;
    } else {
        return `Hello, ${name}${punctuation}`;
    }
}

console.log(greet("Alice")); // Output: Hello, Alice!
console.log(greet("Bob", "Good morning")); // Output: Good morning, Bob!
console.log(greet("Charlie", undefined, ".")); // Output: Hello, Charlie.

In this example:

  • The greeting parameter is optional.
  • The punctuation parameter has a default value of "!".
  • If greeting is not provided, the function defaults to saying "Hello".

Conclusion

Understanding optional and default parameters in TypeScript allows you to write more flexible and maintainable functions. Optional parameters provide a way to handle cases where some arguments might be missing, while default parameters ensure that your functions have sensible defaults when arguments are not provided. By combining these features, you can create robust functions that handle a variety of input scenarios gracefully.
