Introduction to Static Site Generation (SSG)

Static Site Generation (SSG) is a method of pre-rendering web pages at build time, which results in static HTML files. This can significantly improve performance and SEO.

  • Pre-rendering: Generating HTML at build time.
  • Performance: Faster page loads as HTML is pre-generated.
  • SEO: Better search engine indexing due to static content.

Why Use Next.js for SSG?

Next.js is a popular React framework that provides built-in support for SSG, making it easier to create static websites.

  • Ease of Use: Simplifies the setup and configuration.
  • Built-in Routing: Automatic routing based on the file system.
  • Data Fetching: Supports various data fetching methods for SSG.

Setting Up a Next.js Project

To get started with Next.js, you need to set up a new project.

  1. Install Next.js: 
    npx create-next-app@latest my-nextjs-site
cd my-nextjs-site
npm run dev
  2. Project Structure:
    • pages/: Contains all the page components.
    • public/: Static assets like images.
    • styles/: CSS files.

Creating a Static Page

Next.js allows you to create static pages by simply exporting a React component from a file in the pages/ directory.

  1. Create a new page:

    // pages/index.js
import React from 'react';

const HomePage = () => {
  return (
    <div>
      <h1>Welcome to My Static Site</h1>
      <p>This page is statically generated.</p>
    </div>
  );
};

export default HomePage;

  2. Run the development server:

    npm run dev

    Open http://localhost:3000 to see your static page.

Data Fetching with `getStaticProps`

To fetch data at build time, Next.js provides the getStaticProps function.

  1. Example with getStaticProps: 
    // pages/index.js
import React from 'react';

export async function getStaticProps() {
  // Fetch data from an API or file system
  const data = await fetch('https://api.example.com/data').then(res => res.json());

  return {
    props: {
      data,
    },
  };
}

const HomePage = ({ data }) => {
  return (
    <div>
      <h1>Welcome to My Static Site</h1>
      <p>Data fetched at build time:</p>
      <pre>{JSON.stringify(data, null, 2)}</pre>
    </div>
  );
};

export default HomePage;

Dynamic Routes with `getStaticPaths`

For pages that require dynamic routing, you can use getStaticPaths along with getStaticProps.

  1. Example with dynamic routes: 
    // pages/posts/[id].js
import React from 'react';

export async function getStaticPaths() {
  // Fetch list of IDs from an API or file system
  const posts = await fetch('https://api.example.com/posts').then(res => res.json());
  const paths = posts.map(post => ({
    params: { id: post.id.toString() },
  }));

  return { paths, fallback: false };
}

export async function getStaticProps({ params }) {
  // Fetch data for a single post
  const post = await fetch(`https://api.example.com/posts/${params.id}`).then(res => res.json());

  return {
    props: {
      post,
    },
  };
}

const PostPage = ({ post }) => {
  return (
    <div>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
    </div>
  );
};

export default PostPage;

Comparing SSG with Other Data Fetching Methods

| Method | Description | Use Case | |-----------------|----------------------------------------------|-----------------------------------------------| | Static Site Generation (SSG) | Pre-renders pages at build time. | Static content that doesn't change often. | | Server-Side Rendering (SSR) | Renders pages on each request. | Dynamic content that changes frequently. | | Client-Side Rendering (CSR) | Fetches data on the client side after load. | Content that is highly interactive or user-specific. |

Conclusion

Static Site Generation with Next.js is a powerful way to create fast, SEO-friendly websites. By leveraging getStaticProps and getStaticPaths, you can fetch data at build time and generate static pages efficiently. This method is ideal for content that doesn't change frequently, providing a great balance between performance and ease of use.

By mastering SSG in Next.js, you can build robust, high-performance static websites that are easy to maintain and scale.
