Introduction to Class Components

Class components are one of the fundamental building blocks in React, especially in older codebases. They allow you to create components with more complex logic and state management compared to functional components. Although functional components with hooks are more common in modern React development, understanding class components is crucial for maintaining and understanding legacy code.

Creating a Class Component

To create a class component in React, you need to extend the React.Component class and define a render method that returns the JSX to be rendered.

Example

import React, { Component } from 'react';

class Welcome extends Component {
  render() {
    return <h1>Hello, {this.props.name}</h1>;
  }
}

export default Welcome;

Explanation

  • import React, { Component } from 'react';: Import React and the Component class from the React library.
  • class Welcome extends Component { ... }: Define a new class Welcome that extends React.Component.
  • render() { ... }: Every class component must have a render method that returns JSX.
  • this.props.name: Access props passed to the component using this.props.

State in Class Components

Class components can hold and manage their own state using the this.state object.

Example

import React, { Component } from 'react';

class Counter extends Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
  }

  increment = () => {
    this.setState({ count: this.state.count + 1 });
  };

  render() {
    return (
      <div>
        <p>Count: {this.state.count}</p>
        <button onClick={this.increment}>Increment</button>
      </div>
    );
  }
}

export default Counter;

Explanation

  • constructor(props) { ... }: Initialize the component's state in the constructor.
  • this.state = { count: 0 };: Set the initial state.
  • this.setState({ count: this.state.count + 1 });: Update the state using setState.
  • onClick={this.increment}: Bind the increment method to the button's onClick event.

Lifecycle Methods

Class components come with several lifecycle methods that allow you to run code at specific points in a component's lifecycle.

Common Lifecycle Methods

  • componentDidMount(): Called after the component is mounted.
  • componentDidUpdate(prevProps, prevState): Called after the component updates.
  • componentWillUnmount(): Called before the component is unmounted and destroyed.

Example

import React, { Component } from 'react';

class Timer extends Component {
  constructor(props) {
    super(props);
    this.state = { seconds: 0 };
  }

  componentDidMount() {
    this.interval = setInterval(() => this.tick(), 1000);
  }

  componentDidUpdate(prevProps, prevState) {
    if (this.state.seconds !== prevState.seconds) {
      console.log(`Seconds updated: ${this.state.seconds}`);
    }
  }

  componentWillUnmount() {
    clearInterval(this.interval);
  }

  tick() {
    this.setState(state => ({
      seconds: state.seconds + 1
    }));
  }

  render() {
    return (
      <div>
        <p>Seconds: {this.state.seconds}</p>
      </div>
    );
  }
}

export default Timer;

Explanation

  • componentDidMount(): Starts a timer when the component mounts.
  • componentDidUpdate(prevProps, prevState): Logs a message whenever the seconds state updates.
  • componentWillUnmount(): Clears the timer when the component unmounts.
  • tick(): Increments the seconds state every second.

Props vs State

Understanding the difference between props and state is crucial in React.

| Feature | Props | State | |---------|-------|-------| | Definition | Data passed to the component | Data managed within the component | | Mutability | Immutable | Mutable | | Usage | Read-only | Can be updated with setState | | Source | Parent component | Within the component itself |

Example

import React, { Component } from 'react';

class Display extends Component {
  render() {
    return <h1>{this.props.message}</h1>;
  }
}

class App extends Component {
  constructor(props) {
    super(props);
    this.state = { message: 'Hello, World!' };
  }

  changeMessage = () => {
    this.setState({ message: 'Hello, React!' });
  };

  render() {
    return (
      <div>
        <Display message={this.state.message} />
        <button onClick={this.changeMessage}>Change Message</button>
      </div>
    );
  }
}

export default App;

Explanation

  • Display component receives message as a prop and displays it.
  • App component manages the message state and passes it to Display as a prop.
  • changeMessage method updates the message state when the button is clicked.

Conclusion

Class components are a powerful feature in React, allowing for complex state management and lifecycle methods. While functional components with hooks are more prevalent in modern React development, understanding class components is essential for working with legacy code and gaining a deeper understanding of React's capabilities. By mastering class components, you will be well-equipped to handle a wide range of scenarios in React development.
