Introduction

Docker is a powerful platform for developing, shipping, and running applications inside containers. Understanding basic Docker commands is essential for anyone starting with Docker. This section will cover the fundamental commands you need to get started with Docker.

Key Concepts

  • Docker Image: A lightweight, standalone, and executable software package that includes everything needed to run a piece of software.
  • Docker Container: A runtime instance of a Docker image.
  • Docker Hub: A cloud-based repository where Docker users can create, test, store, and distribute container images.

Docker Installation

Before diving into Docker commands, ensure Docker is installed on your system. You can download and install Docker from Docker's official website.

Basic Docker Commands

1. `docker --version`

  • Purpose: Check the installed Docker version.
  • Usage: 
    docker --version
  • Example: 
    $ docker --version
Docker version 20.10.7, build f0df350

2. `docker pull`

  • Purpose: Download a Docker image from Docker Hub.
  • Usage: 
    docker pull <image_name>
  • Example: 
    $ docker pull nginx
Using default tag: latest
latest: Pulling from library/nginx
...

3. `docker images`

  • Purpose: List all Docker images on your local machine.
  • Usage: 
    docker images
  • Example: 
    $ docker images
REPOSITORY          TAG                 IMAGE ID            CREATED             SIZE
nginx               latest              4bb46517cac3        2 weeks ago         133MB

4. `docker run`

  • Purpose: Create and start a new container from an image.
  • Usage: 
    docker run <image_name>
  • Example: 
    $ docker run nginx

5. `docker ps`

  • Purpose: List all running containers.
  • Usage: 
    docker ps
  • Example: 
    $ docker ps
CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS              PORTS               NAMES
1a2b3c4d5e6f        nginx               "nginx -g 'daemon of…"   5 seconds ago       Up 4 seconds        80/tcp              loving_turing

6. `docker stop`

  • Purpose: Stop a running container.
  • Usage: 
    docker stop <container_id>
  • Example: 
    $ docker stop 1a2b3c4d5e6f

7. `docker rm`

  • Purpose: Remove a stopped container.
  • Usage: 
    docker rm <container_id>
  • Example: 
    $ docker rm 1a2b3c4d5e6f

8. `docker rmi`

  • Purpose: Remove a Docker image.
  • Usage: 
    docker rmi <image_id>
  • Example: 
    $ docker rmi 4bb46517cac3

Summary

In this section, we covered the basic Docker commands essential for beginners. These commands include checking the Docker version, pulling images, running containers, listing images and containers, stopping and removing containers, and removing images. Mastering these commands will provide a solid foundation for working with Docker. As you progress, you will learn more advanced commands and concepts to manage Docker environments effectively.
