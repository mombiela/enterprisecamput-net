Introduction

Task planning is a crucial component in the development of artificial intelligence (AI) for video games. It allows non-player characters (NPCs) to make complex decisions and perform actions in an organized and efficient manner. In this section, we will explore key concepts, techniques, and practical examples for implementing task planning in video games.

Key Concepts

  • Tasks and Subtasks: Tasks are actions that an NPC must perform, and they can be broken down into smaller subtasks.
  • Behavior Trees: Hierarchical structures that logically organize tasks and subtasks.
  • Goal-Oriented Action Planning (GOAP): A technique that allows NPCs to select and plan actions based on specific goals.
  • Finite State Machines (FSM): Models that represent states and transitions between them, useful for sequential tasks.

Behavior Trees

Behavior trees are a powerful tool for organizing and managing NPC tasks. They consist of nodes that represent actions and conditions.

Behavior Tree Example

class Node:
    def __init__(self, name):
        self.name = name
        self.children = []

    def add_child(self, child):
        self.children.append(child)

class BehaviorTree:
    def __init__(self, root):
        self.root = root

    def execute(self):
        return self.root.execute()

class TaskNode(Node):
    def execute(self):
        print(f"Executing task: {self.name}")
        for child in self.children:
            child.execute()

# Create task nodes
root = TaskNode("Root Task")
subtask1 = TaskNode("Subtask 1")
subtask2 = TaskNode("Subtask 2")

# Build the behavior tree
root.add_child(subtask1)
root.add_child(subtask2)

# Execute the behavior tree
tree = BehaviorTree(root)
tree.execute()

In this example, we have created a simple behavior tree with a root task and two subtasks. When executing the tree, the tasks are printed in hierarchical order.

Goal-Oriented Action Planning (GOAP)

GOAP is an advanced technique that allows NPCs to plan their actions based on specific goals. NPCs evaluate their goals and select the most appropriate actions to achieve them.

GOAP Example

class Action:
    def __init__(self, name, cost):
        self.name = name
        self.cost = cost

    def execute(self):
        print(f"Executing action: {self.name}")

class Goal:
    def __init__(self, name):
        self.name = name
        self.actions = []

    def add_action(self, action):
        self.actions.append(action)

    def plan(self):
        # Sort actions by cost
        self.actions.sort(key=lambda x: x.cost)
        for action in self.actions:
            action.execute()

# Create actions
action1 = Action("Action 1", 10)
action2 = Action("Action 2", 5)

# Create goal and add actions
goal = Goal("Reach Goal")
goal.add_action(action1)
goal.add_action(action2)

# Plan and execute actions
goal.plan()

In this example, we have created a goal with two actions. The actions are sorted by cost and executed accordingly.

Finite State Machines (FSM)

FSMs are useful for modeling sequential behaviors and NPC states. Each state represents a specific task or action, and transitions define how to move between states.

FSM Example

class State:
    def __init__(self, name):
        self.name = name

    def execute(self):
        print(f"Executing state: {self.name}")

class FSM:
    def __init__(self):
        self.states = {}
        self.current_state = None

    def add_state(self, state):
        self.states[state.name] = state

    def set_state(self, state_name):
        self.current_state = self.states[state_name]

    def execute(self):
        if self.current_state:
            self.current_state.execute()

# Create states
state1 = State("State 1")
state2 = State("State 2")

# Create FSM and add states
fsm = FSM()
fsm.add_state(state1)
fsm.add_state(state2)

# Set initial state and execute
fsm.set_state("State 1")
fsm.execute()
fsm.set_state("State 2")
fsm.execute()

In this example, we have created an FSM with two states and demonstrated how to change and execute states.

Conclusion

Task planning is essential for creating intelligent and realistic NPCs in video games. Techniques such as behavior trees, GOAP, and FSM provide different approaches for organizing and managing NPC actions. With a solid understanding of these techniques, developers can create more immersive and challenging gaming experiences.

Remember to practice with the provided examples and experiment with different configurations to better understand how these techniques work in practice.
