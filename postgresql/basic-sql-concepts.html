Introduction to SQL and PostgreSQL

  • SQL (Structured Query Language): A standard language for managing and manipulating databases.
  • PostgreSQL: An advanced, open-source relational database management system (RDBMS) known for its robustness and standards compliance.

Setting Up PostgreSQL

  • Installation: Instructions for installing PostgreSQL on various operating systems.
  • Connecting to PostgreSQL: Using psql command-line tool or graphical tools like pgAdmin.

Database and Table Creation

  • Creating a Database:

    CREATE DATABASE mydatabase;
    • This command creates a new database named mydatabase.

  • Creating a Table:

    CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100),
    email VARCHAR(100) UNIQUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
    • id SERIAL PRIMARY KEY: Auto-incrementing primary key.
    • name VARCHAR(100): A variable character field with a maximum length of 100.
    • email VARCHAR(100) UNIQUE: A unique field to store email addresses.
    • created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP: A timestamp field with a default value of the current time.

Basic Data Manipulation

  • Inserting Data:

    INSERT INTO users (name, email) VALUES ('John Doe', 'john.doe@example.com');
    • This command inserts a new row into the users table.

  • Updating Data:

    UPDATE users SET email = 'john.new@example.com' WHERE name = 'John Doe';
    • This command updates the email of the user named 'John Doe'.

  • Deleting Data:

    DELETE FROM users WHERE name = 'John Doe';
    • This command deletes the user named 'John Doe' from the users table.

Querying Data

  • Basic SELECT Statement:

    SELECT * FROM users;
    • This command retrieves all columns from the users table.

  • Filtering Results with WHERE:

    SELECT * FROM users WHERE email = 'john.doe@example.com';
    • This command retrieves rows where the email is 'john.doe@example.com'.

  • Sorting Results with ORDER BY:

    SELECT * FROM users ORDER BY created_at DESC;
    • This command retrieves all users sorted by the created_at field in descending order.

Aggregating Data

  • COUNT, AVG, MAX, MIN, SUM: 
    SELECT COUNT(*) FROM users;
SELECT AVG(salary) FROM employees;
SELECT MAX(age) FROM users;
SELECT MIN(age) FROM users;
SELECT SUM(salary) FROM employees;
    • These commands perform various aggregate functions on the specified columns.

Joining Tables

  • INNER JOIN:

    SELECT users.name, orders.amount 
FROM users 
INNER JOIN orders ON users.id = orders.user_id;
    • This command retrieves data from users and orders tables where there is a matching user_id.

  • LEFT JOIN:

    SELECT users.name, orders.amount 
FROM users 
LEFT JOIN orders ON users.id = orders.user_id;
    • This command retrieves all users and their orders, including users with no orders.

Conclusion

In this topic, we covered the fundamental concepts of SQL and PostgreSQL, including database and table creation, basic data manipulation, querying, aggregating data, and joining tables. These basics form the foundation for more advanced SQL operations and database management tasks. Practice these commands to become comfortable with SQL syntax and PostgreSQL's capabilities.
