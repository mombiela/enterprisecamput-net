Introduction to PL/pgSQL

PL/pgSQL (Procedural Language/PostgreSQL) is a procedural language supported by PostgreSQL. It allows you to write complex functions and triggers in PostgreSQL, enhancing the capabilities of SQL.

  • Key Concepts:
    • Procedural Language: Extends SQL with procedural constructs.
    • Functions: Reusable blocks of code that can take parameters and return a value.
    • Triggers: Special kind of stored procedure that automatically executes when certain events occur.

Setting Up PL/pgSQL

Before you start writing PL/pgSQL code, ensure that the language is enabled in your PostgreSQL database.

CREATE EXTENSION IF NOT EXISTS plpgsql;

Writing PL/pgSQL Functions

Functions in PL/pgSQL are defined using the CREATE FUNCTION statement. They can take parameters, execute SQL commands, and return results.

Basic Function Example

Here's a simple function that takes two integers and returns their sum:

CREATE OR REPLACE FUNCTION add_numbers(a INTEGER, b INTEGER)
RETURNS INTEGER AS $$
BEGIN
    RETURN a + b;
END;
$$ LANGUAGE plpgsql;
  • Explanation:
    • CREATE OR REPLACE FUNCTION: Defines a new function or replaces an existing one.
    • RETURNS INTEGER: Specifies the return type of the function.
    • $$ ... $$: Delimiters for the function body.
    • BEGIN ... END;: Block where the function's logic is written.
    • RETURN a + b;: Returns the sum of the two input parameters.

Calling a Function

You can call the function using a SELECT statement:

SELECT add_numbers(5, 3);

Control Structures

PL/pgSQL supports various control structures such as conditional statements and loops.

Conditional Statements

Use IF, ELSIF, and ELSE to execute code conditionally.

CREATE OR REPLACE FUNCTION check_number(n INTEGER)
RETURNS TEXT AS $$
BEGIN
    IF n > 0 THEN
        RETURN 'Positive';
    ELSIF n < 0 THEN
        RETURN 'Negative';
    ELSE
        RETURN 'Zero';
    END IF;
END;
$$ LANGUAGE plpgsql;

Loops

PL/pgSQL provides several types of loops: LOOP, WHILE, FOR.

Example: FOR Loop

CREATE OR REPLACE FUNCTION sum_series(n INTEGER)
RETURNS INTEGER AS $$
DECLARE
    total INTEGER := 0;
BEGIN
    FOR i IN 1..n LOOP
        total := total + i;
    END LOOP;
    RETURN total;
END;
$$ LANGUAGE plpgsql;
  • Explanation:
    • DECLARE: Section to declare variables.
    • total INTEGER := 0;: Initializes the total variable.
    • FOR i IN 1..n LOOP: Loops from 1 to n.
    • total := total + i;: Adds the current value of i to total.

Error Handling

PL/pgSQL provides mechanisms to handle errors using EXCEPTION blocks.

Example: Error Handling

CREATE OR REPLACE FUNCTION divide_numbers(a INTEGER, b INTEGER)
RETURNS FLOAT AS $$
BEGIN
    RETURN a / b;
EXCEPTION
    WHEN division_by_zero THEN
        RETURN NULL;
END;
$$ LANGUAGE plpgsql;
  • Explanation:
    • EXCEPTION: Block to catch exceptions.
    • WHEN division_by_zero THEN: Catches division by zero errors.
    • RETURN NULL;: Returns NULL if an error occurs.

Triggers

Triggers are special procedures that are automatically executed in response to certain events on a table.

Example: Trigger to Log Changes

CREATE OR REPLACE FUNCTION log_changes()
RETURNS TRIGGER AS $$
BEGIN
    INSERT INTO changes_log(table_name, operation, changed_at)
    VALUES (TG_TABLE_NAME, TG_OP, NOW());
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER after_update
AFTER UPDATE ON your_table
FOR EACH ROW
EXECUTE FUNCTION log_changes();
  • Explanation:
    • TG_TABLE_NAME: Special variable holding the name of the table.
    • TG_OP: Special variable holding the type of operation (INSERT, UPDATE, DELETE).
    • AFTER UPDATE ON your_table: Specifies that the trigger fires after an update on your_table.

Conclusion

PL/pgSQL extends the capabilities of PostgreSQL by allowing you to write procedural code. This includes defining functions, using control structures, handling errors, and creating triggers. Mastering PL/pgSQL can significantly enhance your ability to manage and manipulate data within PostgreSQL.

By understanding and practicing the concepts covered in this topic, you will be well-equipped to write efficient and robust PL/pgSQL code for various applications.
