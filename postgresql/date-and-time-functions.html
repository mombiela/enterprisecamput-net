Date and Time Functions in PostgreSQL
String Functions ►

Introduction

Date and time functions are essential for managing and manipulating temporal data in PostgreSQL. This section will cover various functions and operations that you can perform on date and time data types, ranging from basic to advanced levels.

Basic Date and Time Functions

Current Date and Time

  • CURRENT_DATE: Returns the current date.
  • CURRENT_TIME: Returns the current time.
  • CURRENT_TIMESTAMP: Returns the current date and time (timestamp).
SELECT CURRENT_DATE;       -- Returns the current date
SELECT CURRENT_TIME;       -- Returns the current time
SELECT CURRENT_TIMESTAMP;  -- Returns the current date and time

Extracting Parts of a Date/Time

  • EXTRACT(field FROM source): Extracts a specific part of a date/time value.
SELECT EXTRACT(YEAR FROM CURRENT_DATE);  -- Extracts the year from the current date
SELECT EXTRACT(MONTH FROM CURRENT_DATE); -- Extracts the month from the current date
SELECT EXTRACT(DAY FROM CURRENT_DATE);   -- Extracts the day from the current date

Date and Time Arithmetic

  • Addition/Subtraction: You can add or subtract intervals to/from dates.
SELECT CURRENT_DATE + INTERVAL '1 day';  -- Adds one day to the current date
SELECT CURRENT_DATE - INTERVAL '1 month'; -- Subtracts one month from the current date

Intermediate Date and Time Functions

Formatting Date and Time

  • TO_CHAR: Converts a date/time value to a string with a specified format.
SELECT TO_CHAR(CURRENT_TIMESTAMP, 'YYYY-MM-DD HH24:MI:SS'); -- Formats the current timestamp

Parsing Date and Time

  • TO_DATE: Converts a string to a date.
  • TO_TIMESTAMP: Converts a string to a timestamp.
SELECT TO_DATE('2023-10-01', 'YYYY-MM-DD'); -- Converts a string to a date
SELECT TO_TIMESTAMP('2023-10-01 12:30:00', 'YYYY-MM-DD HH24:MI:SS'); -- Converts a string to a timestamp

Age Calculation

  • AGE: Calculates the age between two dates.
SELECT AGE('2023-10-01', '2000-01-01'); -- Calculates the age between two dates

Advanced Date and Time Functions

Time Zones

  • AT TIME ZONE: Converts a timestamp to a different time zone.
SELECT '2023-10-01 12:00:00'::timestamp AT TIME ZONE 'UTC'; -- Converts to UTC time zone
SELECT '2023-10-01 12:00:00'::timestamp AT TIME ZONE 'America/New_York'; -- Converts to New York time zone

Interval Arithmetic

  • JUSTIFY_DAYS: Adjusts intervals to be more human-readable.
  • JUSTIFY_HOURS: Adjusts intervals to be more human-readable.
SELECT JUSTIFY_DAYS(INTERVAL '30 days'); -- Adjusts interval to be more human-readable
SELECT JUSTIFY_HOURS(INTERVAL '25 hours'); -- Adjusts interval to be more human-readable

Date and Time Aggregation

  • Grouping by Date/Time: Useful for summarizing data over time periods.
SELECT DATE_TRUNC('month', order_date) AS month, COUNT(*) 
FROM orders 
GROUP BY DATE_TRUNC('month', order_date); -- Groups orders by month

Summary

In this section, we covered a wide range of date and time functions in PostgreSQL, from basic retrieval of the current date and time to advanced manipulation and formatting. Understanding these functions will enable you to handle temporal data more effectively, whether you're performing simple date arithmetic or complex time zone conversions. Practice these functions with real-world data to solidify your understanding and improve your PostgreSQL skills.
