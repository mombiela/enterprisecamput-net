Understanding Data Types
Using Arrays and JSON ►

Introduction

Data types are fundamental to any database system, including PostgreSQL. They define the kind of data that can be stored in a table column. Understanding data types is crucial for designing efficient databases and writing effective queries.

Key Concepts

  • Data Type: A classification that specifies which type of value a column can hold.
  • Type Categories: PostgreSQL supports several categories of data types, including numeric, character, date/time, and more.
  • Type Constraints: Rules that limit the type of data that can be stored in a column.

Numeric Types

Numeric data types are used to store numbers. PostgreSQL supports several numeric types:

| Data Type | Description | Storage Size | |-----------|-------------|--------------| | smallint | Small-range integer | 2 bytes | | integer or int | Typical choice for integer | 4 bytes | | bigint | Large-range integer | 8 bytes | | decimal or numeric | User-specified precision | Variable | | real | Single precision floating-point | 4 bytes | | double precision | Double precision floating-point | 8 bytes | | serial | Auto-incrementing integer | 4 bytes | | bigserial | Large auto-incrementing integer | 8 bytes |

Example

CREATE TABLE numbers_example (
    small_num smallint,
    medium_num integer,
    large_num bigint,
    precise_num numeric(10, 2),
    float_num real,
    double_num double precision
);

Explanation:

  • small_num can store small integers.
  • medium_num is suitable for typical integer values.
  • large_num can store very large integers.
  • precise_num stores numbers with up to 10 digits, 2 of which are after the decimal point.
  • float_num and double_num store floating-point numbers with single and double precision, respectively.

Character Types

Character data types are used to store text. PostgreSQL provides several options:

| Data Type | Description | Storage Size | |-----------|-------------|--------------| | char(n) | Fixed-length character type | n bytes | | varchar(n) | Variable-length character type | Up to n bytes | | text | Variable-length character type with no limit | Variable |

Example

CREATE TABLE text_example (
    fixed_char char(10),
    variable_char varchar(50),
    unlimited_text text
);

Explanation:

  • fixed_char stores a fixed-length string of 10 characters.
  • variable_char can store a string up to 50 characters long.
  • unlimited_text can store a string of any length.

Date/Time Types

Date and time data types are used to store dates, times, and timestamps.

| Data Type | Description | Storage Size | |-----------|-------------|--------------| | date | Calendar date (year, month, day) | 4 bytes | | time | Time of day (no time zone) | 8 bytes | | timestamp | Date and time (no time zone) | 8 bytes | | timestamptz | Date and time with time zone | 8 bytes | | interval | Time span | 16 bytes |

Example

CREATE TABLE datetime_example (
    event_date date,
    event_time time,
    event_timestamp timestamp,
    event_timestamptz timestamptz,
    duration interval
);

Explanation:

  • event_date stores a date.
  • event_time stores a time of day.
  • event_timestamp stores a date and time.
  • event_timestamptz stores a date and time with time zone information.
  • duration stores a time span.

Boolean Type

The boolean data type is used to store true/false values.

| Data Type | Description | Storage Size | |-----------|-------------|--------------| | boolean | Logical Boolean (true/false) | 1 byte |

Example

CREATE TABLE boolean_example (
    is_active boolean
);

Explanation:

  • is_active can store a true or false value.

Array Types

PostgreSQL allows columns to be defined as arrays of variable length.

Example

CREATE TABLE array_example (
    integer_array integer[],
    text_array text[]
);

Explanation:

  • integer_array can store an array of integers.
  • text_array can store an array of text strings.

JSON Types

PostgreSQL supports JSON data types for storing JSON data.

| Data Type | Description | Storage Size | |-----------|-------------|--------------| | json | Textual JSON data | Variable | | jsonb | Binary JSON data, more efficient for indexing | Variable |

Example

CREATE TABLE json_example (
    json_data json,
    jsonb_data jsonb
);

Explanation:

  • json_data stores JSON data in text format.
  • jsonb_data stores JSON data in a binary format, which is more efficient for indexing and querying.

Conclusion

Understanding data types in PostgreSQL is essential for designing efficient and effective databases. By choosing the appropriate data type for each column, you can ensure data integrity, optimize storage, and improve query performance. This topic covered the most commonly used data types, including numeric, character, date/time, boolean, array, and JSON types. As you progress in your PostgreSQL journey, you'll encounter more specialized data types and learn how to use them effectively.
