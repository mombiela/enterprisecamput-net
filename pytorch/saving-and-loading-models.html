Saving and Loading Models
Convolutional Neural Networks (CNNs) ►

In this section, we will cover how to save and load models in PyTorch. This is a crucial skill for any machine learning practitioner as it allows you to persist your models and reuse them later without retraining.

Key Concepts

  • State Dictionary: A Python dictionary object that maps each layer to its parameter tensor.
  • Saving a Model: Storing the model's state dictionary to a file.
  • Loading a Model: Restoring the model's state dictionary from a file.
  • Checkpointing: Saving additional information such as epoch number, optimizer state, etc.

Saving a Model

To save a model in PyTorch, you typically save the state dictionary of the model. This is done using the torch.save function.

Example: Saving a Model

import torch
import torch.nn as nn

# Define a simple model
class SimpleModel(nn.Module):
    def __init__(self):
        super(SimpleModel, self).__init__()
        self.fc = nn.Linear(10, 1)

    def forward(self, x):
        return self.fc(x)

# Instantiate the model
model = SimpleModel()

# Save the model's state dictionary
torch.save(model.state_dict(), 'model.pth')

In this example:

  • We define a simple neural network with one linear layer.
  • We instantiate the model.
  • We save the model's state dictionary to a file named model.pth.

Loading a Model

To load a model, you need to create an instance of the model and then load the state dictionary into it using the load_state_dict method.

Example: Loading a Model

import torch

# Define the same model architecture
class SimpleModel(nn.Module):
    def __init__(self):
        super(SimpleModel, self).__init__()
        self.fc = nn.Linear(10, 1)

    def forward(self, x):
        return self.fc(x)

# Instantiate the model
model = SimpleModel()

# Load the state dictionary
model.load_state_dict(torch.load('model.pth'))

# Set the model to evaluation mode
model.eval()

In this example:

  • We define the same model architecture as before.
  • We instantiate the model.
  • We load the state dictionary from the file model.pth.
  • We set the model to evaluation mode using model.eval(). This is important as it sets the model to inference mode, affecting layers like dropout and batch normalization.

Checkpointing

Checkpointing involves saving not just the model's state dictionary but also other information like the optimizer state, epoch number, etc. This is useful for resuming training from a specific point.

Example: Saving a Checkpoint

import torch.optim as optim

# Define the model and optimizer
model = SimpleModel()
optimizer = optim.SGD(model.parameters(), lr=0.01)

# Save a checkpoint
checkpoint = {
    'epoch': 10,
    'model_state_dict': model.state_dict(),
    'optimizer_state_dict': optimizer.state_dict(),
    'loss': 0.4,
}
torch.save(checkpoint, 'checkpoint.pth')

Example: Loading a Checkpoint

# Instantiate the model and optimizer
model = SimpleModel()
optimizer = optim.SGD(model.parameters(), lr=0.01)

# Load the checkpoint
checkpoint = torch.load('checkpoint.pth')
model.load_state_dict(checkpoint['model_state_dict'])
optimizer.load_state_dict(checkpoint['optimizer_state_dict'])
epoch = checkpoint['epoch']
loss = checkpoint['loss']

# Set the model to evaluation mode
model.eval()

In these examples:

  • We save a checkpoint that includes the epoch number, model state dictionary, optimizer state dictionary, and loss.
  • We load the checkpoint and restore the model and optimizer states, as well as the epoch number and loss.

Summary

  • State Dictionary: The key component for saving and loading models.
  • Saving a Model: Use torch.save(model.state_dict(), 'file.pth').
  • Loading a Model: Use model.load_state_dict(torch.load('file.pth')) and set the model to evaluation mode with model.eval().
  • Checkpointing: Save additional information like epoch number and optimizer state to resume training later.

By mastering these techniques, you can efficiently manage your models and training processes, ensuring that you can save your work and pick up right where you left off.
