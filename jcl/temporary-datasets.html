Introduction to Temporary Datasets

Temporary datasets in JCL are datasets that exist only for the duration of the job. They are automatically deleted once the job completes. These datasets are useful for intermediate storage during job execution and help in managing disk space efficiently.

Key Concepts

  • Temporary Dataset Naming: Temporary datasets are identified by names that start with two ampersands (&&).
  • Scope: They are only accessible within the job step or the entire job, depending on how they are defined.
  • Automatic Deletion: They are deleted automatically at the end of the job, reducing the need for manual cleanup.

Defining Temporary Datasets

To define a temporary dataset in JCL, you use the DSN parameter with a name that starts with &&.

Example

//STEP1    EXEC PGM=MYPROG
//TEMPDS   DD   DSN=&&TEMPFILE,DISP=(NEW,PASS),
//         SPACE=(TRK,(5,5)),UNIT=SYSDA
  • DSN=&&TEMPFILE: Defines a temporary dataset named TEMPFILE.
  • DISP=(NEW,PASS): Specifies that the dataset is new and should be passed to the next step.
  • SPACE=(TRK,(5,5)): Allocates space for the dataset.
  • UNIT=SYSDA: Specifies the device type.

Using Temporary Datasets Across Steps

Temporary datasets can be passed between steps using the DISP=PASS parameter.

Example

//STEP1    EXEC PGM=MYPROG1
//TEMPDS   DD   DSN=&&TEMPFILE,DISP=(NEW,PASS),
//         SPACE=(TRK,(5,5)),UNIT=SYSDA
//STEP2    EXEC PGM=MYPROG2
//TEMPDS   DD   DSN=&&TEMPFILE,DISP=(OLD,DELETE)
  • STEP1: Creates a temporary dataset TEMPFILE and passes it to the next step.
  • STEP2: Uses the temporary dataset TEMPFILE created in STEP1 and deletes it after use.

Advanced Usage of Temporary Datasets

Temporary datasets can also be used with more complex parameters and in conjunction with other JCL features like GDGs (Generation Data Groups).

Example with GDG

//STEP1    EXEC PGM=MYPROG1
//TEMPDS   DD   DSN=&&TEMPFILE,DISP=(NEW,PASS),
//         SPACE=(TRK,(5,5)),UNIT=SYSDA
//STEP2    EXEC PGM=MYPROG2
//TEMPDS   DD   DSN=&&TEMPFILE,DISP=(OLD,DELETE)
//GDGOUT   DD   DSN=MYGDG(+1),DISP=(NEW,CATLG),
//         SPACE=(TRK,(5,5)),UNIT=SYSDA
  • GDGOUT: Defines a new generation of a GDG using the temporary dataset.

Conclusion

Temporary datasets in JCL are a powerful feature for managing intermediate data within a job. They help in efficient disk space management and simplify the job cleanup process. By understanding how to define, use, and pass temporary datasets between steps, you can write more efficient and maintainable JCL scripts.
