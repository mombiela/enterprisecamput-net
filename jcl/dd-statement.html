Introduction to DD Statement

The DD (Data Definition) statement is a crucial component in JCL (Job Control Language) used to describe the data sets that a job will use. It specifies the input and output files, their attributes, and how they should be handled during job execution.

Key Concepts

  • DDNAME: The name used to refer to the data set within the JCL.
  • DSNAME (DSN): The actual name of the data set.
  • DISP: The disposition of the data set, indicating its status before and after the job runs.
  • UNIT: The device type on which the data set resides.
  • SPACE: The amount of space to allocate for the data set.
  • DCB: Data Control Block parameters, defining the data set's attributes like record format and length.

Basic Structure of a DD Statement

A simple DD statement might look like this:

//MYDATA  DD  DSN=MY.DATA.SET, DISP=SHR

Explanation

  • //MYDATA: The DDNAME, used to reference this data set in the JCL.
  • DD: Indicates this is a DD statement.
  • DSN=MY.DATA.SET: Specifies the data set name.
  • DISP=SHR: Specifies the disposition, in this case, shared access.

Detailed Explanation of DD Parameters

DSNAME (DSN)

The DSNAME parameter specifies the name of the data set. It can be a fully qualified name or a relative name.

//INPUT  DD  DSN=MY.INPUT.DATA, DISP=OLD

DISP (Disposition)

The DISP parameter has three sub-parameters:

  • Status: NEW, OLD, MOD, or SHR.
  • Normal Disposition: What happens to the data set if the job completes successfully.
  • Abnormal Disposition: What happens to the data set if the job fails.
//OUTPUT  DD  DSN=MY.OUTPUT.DATA, DISP=(NEW,CATLG,DELETE)

UNIT

The UNIT parameter specifies the type of device on which the data set resides.

//TEMP  DD  UNIT=SYSDA, SPACE=(CYL,(5,5))

SPACE

The SPACE parameter specifies the amount of space to allocate for a new data set.

//TEMP  DD  DSN=MY.TEMP.DATA, DISP=(NEW,DELETE), UNIT=SYSDA, SPACE=(CYL,(5,5))

DCB (Data Control Block)

The DCB parameter defines the data set's attributes, such as record format (RECFM), logical record length (LRECL), and block size (BLKSIZE).

//MYFILE  DD  DSN=MY.FILE.DATA, DISP=SHR, DCB=(RECFM=FB,LRECL=80,BLKSIZE=800)

Advanced DD Statement Usage

Concatenation of Data Sets

Multiple data sets can be concatenated in a single DD statement.

//MULTI  DD  DSN=DATA.SET1, DISP=SHR
//       DD  DSN=DATA.SET2, DISP=SHR

Referencing Temporary Data Sets

Temporary data sets can be created and referenced within the same job.

//STEP1  DD  DSN=&&TEMP, DISP=(NEW,PASS), UNIT=SYSDA, SPACE=(CYL,(1,1))
//STEP2  DD  DSN=&&TEMP, DISP=(OLD,DELETE)

Using SYSOUT

The SYSOUT parameter directs output to a system output class, typically for printing or logging.

//PRINT  DD  SYSOUT=A

Examples and Exercises

Example 1: Basic DD Statement

//STEP1  EXEC PGM=MYPROG
//INFILE  DD  DSN=MY.INPUT.DATA, DISP=SHR
//OUTFILE DD  DSN=MY.OUTPUT.DATA, DISP=(NEW,CATLG,DELETE), UNIT=SYSDA, SPACE=(CYL,(10,5))

Exercise 1

Write a DD statement to create a new data set USER.TEST.DATA with the following attributes:

  • Disposition: New, cataloged if successful, deleted if failed.
  • Unit: SYSDA
  • Space: 5 cylinders primary, 2 cylinders secondary.
  • Record format: Fixed block (FB)
  • Logical record length: 80
  • Block size: 800

Solution

//TESTDATA  DD  DSN=USER.TEST.DATA, DISP=(NEW,CATLG,DELETE), UNIT=SYSDA, SPACE=(CYL,(5,2)), DCB=(RECFM=FB,LRECL=80,BLKSIZE=800)

Conclusion

The DD statement is a fundamental part of JCL, allowing you to define and manage data sets used by your jobs. Understanding its parameters and how to use them effectively is crucial for efficient job control and data management in a mainframe environment. Practice writing and modifying DD statements to become proficient in JCL.
