Introduction to Procedures in JCL

Procedures, also known as PROCs, are reusable sets of JCL statements that can be invoked by a single EXEC statement. They help in reducing redundancy and improving maintainability of JCL scripts.

Key Concepts

  • Procedure (PROC): A set of JCL statements that can be reused.
  • EXEC Statement: Used to call a procedure.
  • Symbolic Parameters: Variables within a PROC that can be replaced with actual values when the PROC is called.

Creating a Basic Procedure

To create a procedure, you define it in a JCL member or inline within a JCL job.

Example of a Simple PROC

//MYPROC PROC
//STEP1 EXEC PGM=IEFBR14
//DD1   DD   DSN=MY.DATASET,DISP=(NEW,CATLG,DELETE),
//       SPACE=(TRK,(1,1)),UNIT=SYSDA
  • MYPROC: The name of the procedure.
  • STEP1: A step within the procedure.
  • IEFBR14: A dummy program often used for testing.

Using the Procedure

To use the procedure in a JCL job, you invoke it using the EXEC statement.

//MYJOB  JOB  (ACCT),'MY JOB'
//STEP1  EXEC MYPROC
  • MYJOB: The name of the job.
  • STEP1: The step that calls the procedure MYPROC.

Using Symbolic Parameters

Symbolic parameters allow you to pass values to a procedure, making it more flexible.

Defining Symbolic Parameters in a PROC

//MYPROC PROC DSN=MY.DATASET
//STEP1 EXEC PGM=IEFBR14
//DD1   DD   DSN=&DSN,DISP=(NEW,CATLG,DELETE),
//       SPACE=(TRK,(1,1)),UNIT=SYSDA
  • DSN=MY.DATASET: Default value for the symbolic parameter DSN.
  • &DSN: Symbolic parameter used in the DD statement.

Overriding Symbolic Parameters

When invoking the procedure, you can override the symbolic parameters.

//MYJOB  JOB  (ACCT),'MY JOB'
//STEP1  EXEC MYPROC,DSN=YOUR.DATASET
  • DSN=YOUR.DATASET: Overrides the default value of DSN in the procedure.

Advanced Procedure Concepts

Nested Procedures

You can call a procedure from within another procedure.

//OUTERPROC PROC
//STEP1 EXEC INNERPROC
//INNERPROC PROC
//STEP2 EXEC PGM=IEFBR14
  • OUTERPROC: Calls INNERPROC.
  • INNERPROC: Contains its own steps.

Conditional Execution

You can use conditional execution to control the flow based on the return codes of previous steps.

//MYPROC PROC
//STEP1 EXEC PGM=IEFBR14
//STEP2 EXEC PGM=IEFBR14,COND=(0,NE,STEP1)
  • COND=(0,NE,STEP1): STEP2 will execute only if STEP1's return code is not equal to 0.

Best Practices

  • Modularity: Break down complex jobs into smaller, reusable procedures.
  • Documentation: Comment your procedures to explain their purpose and usage.
  • Error Handling: Use conditional execution to handle errors gracefully.

Conclusion

Procedures in JCL are powerful tools that help in creating modular, maintainable, and reusable job scripts. By understanding how to create and use procedures, including the use of symbolic parameters and conditional execution, you can significantly enhance your JCL scripting capabilities. Practice creating and invoking procedures to become proficient in their usage.
