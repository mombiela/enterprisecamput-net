Defining Datasets in JCL
Dataset Disposition ►

Introduction

Defining datasets is a fundamental aspect of working with Job Control Language (JCL). Datasets are collections of data that can be used for various purposes, such as input, output, or intermediate storage during job execution. This section will cover the basics of defining datasets in JCL, including syntax, parameters, and examples.

Key Concepts

  • Dataset: A collection of data, often stored on disk or tape, that can be accessed and manipulated by JCL jobs.
  • DD Statement: The Data Definition (DD) statement is used in JCL to define datasets.
  • DSN (Dataset Name): The name assigned to a dataset.
  • DISP (Disposition): Specifies the status of the dataset (e.g., NEW, OLD, MOD) and what to do with it after the job step completes.

DD Statement Syntax

The DD statement is used to define datasets in JCL. Here is the basic syntax:

//DDNAME DD DSN=dataset.name,DISP=(status,normal-termination,abnormal-termination)
  • DDNAME: The name used to reference the dataset within the JCL.
  • DSN: The dataset name.
  • DISP: The disposition parameter, which includes the status and actions to take upon normal and abnormal termination.

DISP Parameter

The DISP parameter is crucial for managing datasets. It has three sub-parameters:

  1. Status: Indicates whether the dataset is new, old, or to be modified.
    • NEW: Create a new dataset.
    • OLD: Use an existing dataset.
    • MOD: Modify an existing dataset or create it if it does not exist.
  2. Normal Termination: Action to take if the job step completes successfully.
    • CATLG: Catalog the dataset.
    • KEEP: Keep the dataset.
    • DELETE: Delete the dataset.
  3. Abnormal Termination: Action to take if the job step fails.
    • CATLG: Catalog the dataset.
    • KEEP: Keep the dataset.
    • DELETE: Delete the dataset.

Example DISP Parameter Usage

//MYDATA DD DSN=MY.DATASET,DISP=(NEW,CATLG,DELETE)

In this example:

  • The dataset MY.DATASET is created new.
  • If the job step completes successfully, the dataset is cataloged.
  • If the job step fails, the dataset is deleted.

Defining Temporary Datasets

Temporary datasets are used for intermediate storage and are automatically deleted after the job completes. They do not require a dataset name.

Example of Temporary Dataset

//TEMP DD DSN=&&TEMPDS,DISP=(NEW,PASS,DELETE)

In this example:

  • &&TEMPDS indicates a temporary dataset.
  • The dataset is created new.
  • If the job step completes successfully, the dataset is passed to the next step.
  • If the job step fails, the dataset is deleted.

Defining VSAM Datasets

VSAM (Virtual Storage Access Method) datasets are used for high-performance access to data. Defining a VSAM dataset requires additional parameters.

Example of VSAM Dataset Definition

//VSAMDD DD DSN=MY.VSAM.DATASET,DISP=(NEW,CATLG,DELETE),
//       SPACE=(CYL,(5,1)),UNIT=SYSDA,
//       DCB=(RECFM=FB,LRECL=80,BLKSIZE=800)

In this example:

  • MY.VSAM.DATASET is the name of the VSAM dataset.
  • The dataset is created new, cataloged on successful completion, and deleted on failure.
  • SPACE specifies the primary and secondary space allocation.
  • UNIT specifies the device type.
  • DCB (Data Control Block) specifies the dataset characteristics.

Conclusion

Defining datasets in JCL is essential for managing data during job execution. Understanding the DD statement, DISP parameter, and how to define both temporary and VSAM datasets will enable you to effectively control data flow in your JCL jobs. Practice with different dataset definitions to become proficient in handling various data scenarios in JCL.
