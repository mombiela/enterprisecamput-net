Overview

In this section, we will introduce you to Procedures in Job Control Language (JCL). Procedures, often referred to as PROCs, are a powerful feature in JCL that allow for the reuse of JCL code. This helps in reducing redundancy and improving maintainability of JCL scripts.

Key Concepts

  • Procedure (PROC): A set of JCL statements that can be reused in multiple jobs.
  • Cataloged Procedure: A PROC that is stored in a procedure library and can be invoked by any job.
  • In-stream Procedure: A PROC that is defined within the same JCL job stream.
  • Symbolic Parameters: Variables that can be used within PROCs to make them more flexible and reusable.

Cataloged Procedures

Cataloged procedures are stored in a procedure library and can be invoked by any job. This makes them very useful for standardizing job steps across multiple jobs.

Example of a Cataloged Procedure

//MYPROC PROC
//STEP1 EXEC PGM=MYPROGRAM
//DD1   DD   DSN=MY.DATASET,DISP=SHR

In this example, MYPROC is a cataloged procedure that executes MYPROGRAM and uses a dataset MY.DATASET.

Invoking a Cataloged Procedure

//MYJOB JOB (ACCT),'MY JOB'
//STEP1 EXEC MYPROC

Here, MYJOB is a job that invokes the cataloged procedure MYPROC.

In-stream Procedures

In-stream procedures are defined within the same JCL job stream. They are useful for jobs that require a one-time use of a procedure.

Example of an In-stream Procedure

//MYJOB JOB (ACCT),'MY JOB'
//MYPROC PROC
//STEP1 EXEC PGM=MYPROGRAM
//DD1   DD   DSN=MY.DATASET,DISP=SHR
//MYPROC PEND
//STEP1 EXEC MYPROC

In this example, MYPROC is an in-stream procedure defined within the job MYJOB.

Symbolic Parameters

Symbolic parameters are placeholders within a PROC that can be replaced with actual values when the PROC is invoked. This makes PROCs more flexible and reusable.

Example of a Procedure with Symbolic Parameters

//MYPROC PROC DSN=
//STEP1 EXEC PGM=MYPROGRAM
//DD1   DD   DSN=&DSN,DISP=SHR

Invoking a Procedure with Symbolic Parameters

//MYJOB JOB (ACCT),'MY JOB'
//STEP1 EXEC MYPROC,DSN=MY.DATASET

In this example, the symbolic parameter &DSN in MYPROC is replaced with MY.DATASET when the procedure is invoked.

Summary

Procedures in JCL are a powerful way to reuse and standardize JCL code. They can be cataloged for use across multiple jobs or defined in-stream for one-time use. Symbolic parameters add flexibility, allowing the same PROC to be used with different datasets or parameters. Understanding and using procedures effectively can greatly enhance the maintainability and efficiency of your JCL scripts.
