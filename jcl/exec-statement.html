EXEC Statement in JCL
DD Statement ►

The EXEC statement is one of the most critical components in Job Control Language (JCL). It is used to specify the execution of a program or a procedure within a job. This topic will cover the EXEC statement from a beginner to an advanced level, providing clear explanations, examples, and exercises.

Introduction to the EXEC Statement

  • Purpose: The EXEC statement is used to identify the program or procedure that a job step will execute.
  • Syntax: The basic syntax of the EXEC statement is as follows: 
    //stepname EXEC PGM=program-name
    • stepname: A unique name for the job step.
    • PGM=program-name: Specifies the name of the program to be executed.

Basic Example

//STEP1    EXEC PGM=IEFBR14
  • Explanation:
    • STEP1: The name of the job step.
    • PGM=IEFBR14: Specifies that the program IEFBR14 (a dummy program) will be executed.

Using Parameters in the EXEC Statement

Common Parameters

  • PARM: Used to pass parameters to the program. 
    //STEP1    EXEC PGM=MYPROG,PARM='parameter-string'
  • ACCT: Specifies accounting information. 
    //STEP1    EXEC PGM=MYPROG,ACCT='account-info'
  • REGION: Specifies the amount of memory allocated to the job step. 
    //STEP1    EXEC PGM=MYPROG,REGION=4M

Example with Parameters

//STEP1    EXEC PGM=MYPROG,PARM='TEST',REGION=4M
  • Explanation:
    • PARM='TEST': Passes the string 'TEST' to the program MYPROG.
    • REGION=4M: Allocates 4 megabytes of memory to the job step.

Advanced Usage of the EXEC Statement

Executing Procedures

  • Syntax: To execute a procedure, use the following syntax: 
    //stepname EXEC procname
    • procname: The name of the procedure to be executed.

Example of Executing a Procedure

//STEP1    EXEC PROC=MYPROC
  • Explanation:
    • PROC=MYPROC: Specifies that the procedure MYPROC will be executed.

Overriding Procedure Parameters

  • Syntax: To override parameters in a procedure, use the following syntax: 
    //stepname EXEC procname
//stepname.ddname DD parameter-override
    • ddname: The name of the DD statement within the procedure.
    • parameter-override: The new parameter value.

Example of Overriding Procedure Parameters

//STEP1    EXEC PROC=MYPROC
//STEP1.INPUT DD DSN=NEW.DATASET,DISP=SHR
  • Explanation:
    • INPUT: The DD statement within the procedure MYPROC.
    • DSN=NEW.DATASET,DISP=SHR: Overrides the dataset name and disposition.

Exercises

  1. Basic EXEC Statement:

    • Write an EXEC statement to execute a program named MYPROG with a step name STEP1.

  2. Using Parameters:

    • Modify the EXEC statement to pass the parameter string 'RUN' and allocate 2 megabytes of memory.

  3. Executing a Procedure:

    • Write an EXEC statement to execute a procedure named MYPROC with a step name STEP2.

  4. Overriding Procedure Parameters:

    • Override the DD statement OUTPUT in the procedure MYPROC to use the dataset NEW.OUTPUT.DATA.

Conclusion

The EXEC statement is a fundamental part of JCL, allowing you to specify the execution of programs and procedures within a job. Understanding its syntax and parameters is crucial for effectively managing job steps. By mastering the EXEC statement, you can control the execution flow and resource allocation of your JCL jobs, making your job streams more efficient and manageable.
