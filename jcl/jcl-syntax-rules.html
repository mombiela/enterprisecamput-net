Introduction

Job Control Language (JCL) is used on IBM mainframe systems to instruct the system on how to run a batch job or start a subsystem. Understanding the syntax rules of JCL is crucial for writing effective and error-free job control statements.

Basic Syntax Rules

  • Statements: Each JCL statement starts with two slashes (//) in columns 1 and 2.
  • Statement Length: A JCL statement can be up to 80 characters long.
  • Continuation: If a statement exceeds 80 characters, it can be continued on the next line by placing a comma at the end of the line and starting the next line with // followed by at least one space.
  • Comments: Comments in JCL are written by placing //* at the beginning of the line.

Example

//JOBNAME  JOB (ACCT),'PROGRAMMER',CLASS=A,MSGCLASS=X
//* This is a comment line
//STEP1    EXEC PGM=MYPROG
//DD1      DD DSN=MY.DATASET,DISP=SHR

Statement Types

JCL consists of three main types of statements:

  1. JOB Statement: Defines the start of a job.
  2. EXEC Statement: Specifies the program or procedure to execute.
  3. DD Statement: Describes the data sets to be used in the job.

JOB Statement

  • Purpose: Identifies the job and provides job-level parameters.
  • Syntax: //JOBNAME JOB (accounting_info),'programmer_name',parameters

Example

//MYJOB    JOB (12345),'JOHN DOE',CLASS=A,MSGCLASS=X

EXEC Statement

  • Purpose: Specifies the program or procedure to be executed in a job step.
  • Syntax: //STEPNAME EXEC PGM=program_name,parameters

Example

//STEP1    EXEC PGM=MYPROG

DD Statement

  • Purpose: Defines the data sets to be used in the job step.
  • Syntax: //DDNAME DD parameters

Example

//DD1      DD DSN=MY.DATASET,DISP=SHR

Common Parameters

JOB Statement Parameters

  • CLASS: Specifies the job class.
  • MSGCLASS: Specifies the output class for messages.
  • MSGLEVEL: Controls the level of messages produced.

EXEC Statement Parameters

  • PGM: Specifies the program to execute.
  • PARM: Passes parameters to the program.

DD Statement Parameters

  • DSN: Data set name.
  • DISP: Disposition of the data set (e.g., NEW, OLD, SHR).
  • UNIT: Device type.
  • SPACE: Space allocation for new data sets.

Example of a Complete JCL Job

//MYJOB    JOB (12345),'JOHN DOE',CLASS=A,MSGCLASS=X
//STEP1    EXEC PGM=MYPROG
//DD1      DD DSN=MY.DATASET,DISP=SHR
//DD2      DD DSN=NEW.DATASET,DISP=(NEW,CATLG,DELETE),
//             UNIT=SYSDA,SPACE=(CYL,(5,5))
//* This is a comment line

Conclusion

Understanding the syntax rules of JCL is essential for writing effective job control scripts. By following the basic structure and parameters outlined in this topic, you can create JCL jobs that are both functional and efficient. Practice writing different types of JCL statements and using various parameters to become proficient in JCL syntax.
