Deploying to Heroku
Deploying to AWS ►

Deploying your Django application to Heroku is a straightforward process that allows you to make your web application accessible to the world. This section will guide you through the steps required to deploy your Django project to Heroku, from setting up your environment to pushing your code live.

Prerequisites

Before you start, ensure you have the following:

  • A Heroku account: Sign up at Heroku.
  • Heroku CLI installed: Follow the installation instructions at Heroku CLI.
  • Git installed: You can download it from Git.

Step 1: Prepare Your Django Project

Install Required Packages

Heroku requires some additional packages to run your Django application. Install these packages using pip:

pip install gunicorn dj-database-url psycopg2-binary
  • gunicorn: A Python WSGI HTTP Server for UNIX.
  • dj-database-url: Allows you to use the database URL provided by Heroku.
  • psycopg2-binary: PostgreSQL database adapter for Python.

Update `requirements.txt`

Ensure all your dependencies are listed in requirements.txt:

pip freeze > requirements.txt

Create a `Procfile`

Heroku uses a Procfile to determine how to run your application. Create a file named Procfile in your project's root directory with the following content:

web: gunicorn your_project_name.wsgi

Replace your_project_name with the name of your Django project.

Configure Database Settings

Modify your settings.py to use dj-database-url for database configuration. Add the following code:

import dj_database_url

DATABASES = {
    'default': dj_database_url.config(default='postgres://localhost/your_db_name')
}

Replace your_db_name with the name of your local database.

Static Files Configuration

Heroku requires some additional settings for serving static files. Add the following to your settings.py:

import os

BASE_DIR = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))

STATIC_ROOT = os.path.join(BASE_DIR, 'staticfiles')
STATIC_URL = '/static/'

# Extra places for collectstatic to find static files.
STATICFILES_DIRS = (
    os.path.join(BASE_DIR, 'static'),
)

# Simplified static file serving.
# https://warehouse.python.org/project/whitenoise/
MIDDLEWARE = [
    'whitenoise.middleware.WhiteNoiseMiddleware',
    # ... other middleware ...
]

STATICFILES_STORAGE = 'whitenoise.storage.CompressedManifestStaticFilesStorage'

Install whitenoise:

pip install whitenoise

Update requirements.txt again:

pip freeze > requirements.txt

Step 2: Deploy to Heroku

Initialize a Git Repository

If you haven't already, initialize a Git repository in your project directory:

git init

Add all your files and commit:

git add .
git commit -m "Initial commit"

Create a Heroku App

Log in to Heroku using the CLI:

heroku login

Create a new Heroku app:

heroku create your-app-name

Replace your-app-name with a unique name for your application.

Set Up Environment Variables

Set the SECRET_KEY and DEBUG environment variables:

heroku config:set SECRET_KEY='your_secret_key'
heroku config:set DEBUG=False

Replace your_secret_key with a secure key for your application.

Push Your Code to Heroku

Deploy your code to Heroku:

git push heroku master

Run Migrations

Run the database migrations on Heroku:

heroku run python manage.py migrate

Collect Static Files

Collect static files:

heroku run python manage.py collectstatic --noinput

Conclusion

Congratulations! You have successfully deployed your Django application to Heroku. Your app should now be live and accessible via the URL provided by Heroku. This process involves setting up your Django project for production, configuring necessary packages, and using Git and Heroku CLI to deploy your code. With these steps, you can easily deploy and manage your Django applications on Heroku.
