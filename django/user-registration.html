Introduction

User registration is a fundamental feature in web applications, allowing users to create accounts and access personalized content. In this section, we will cover how to implement user registration in Django, from basic setup to advanced customization.

Setting Up the Project

Before we start with user registration, ensure you have Django installed and a project created.

pip install django
django-admin startproject myproject
cd myproject
python manage.py startapp users

Basic User Registration

Creating the User Model

Django comes with a built-in user model, but you can extend it if needed. For simplicity, we'll use Django's default user model.

Creating the Registration Form

We'll use Django's built-in forms to create a user registration form.

# users/forms.py
from django import forms
from django.contrib.auth.models import User
from django.contrib.auth.forms import UserCreationForm

class UserRegisterForm(UserCreationForm):
    email = forms.EmailField()

    class Meta:
        model = User
        fields = ['username', 'email', 'password1', 'password2']

Creating the Registration View

Create a view to handle the registration form.

# users/views.py
from django.shortcuts import render, redirect
from django.contrib import messages
from .forms import UserRegisterForm

def register(request):
    if request.method == 'POST':
        form = UserRegisterForm(request.POST)
        if form.is_valid():
            form.save()
            username = form.cleaned_data.get('username')
            messages.success(request, f'Account created for {username}!')
            return redirect('login')
    else:
        form = UserRegisterForm()
    return render(request, 'users/register.html', {'form': form})

Creating the Registration Template

Create a template to display the registration form.

<!-- users/templates/users/register.html -->
{% extends "base_generic.html" %}

{% block content %}
  <div class="container">
    <h2>Register</h2>
    <form method="POST">
      {% csrf_token %}
      {{ form.as_p }}
      <button type="submit" class="btn btn-primary">Register</button>
    </form>
  </div>
{% endblock %}

Adding URL Patterns

Add the URL pattern for the registration view.

# users/urls.py
from django.urls import path
from . import views

urlpatterns = [
    path('register/', views.register, name='register'),
]

Include the users app URLs in the main project URLs.

# myproject/urls.py
from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('users/', include('users.urls')),
]

Advanced User Registration

Customizing the User Model

If you need to store additional information, you can extend the user model.

# users/models.py
from django.contrib.auth.models import AbstractUser
from django.db import models

class CustomUser(AbstractUser):
    age = models.PositiveIntegerField(null=True, blank=True)

Update the settings to use the custom user model.

# myproject/settings.py
AUTH_USER_MODEL = 'users.CustomUser'

Adding Email Verification

To enhance security, you can add email verification.

  1. Send Verification Email: Use Django's email backend to send a verification email.
  2. Confirm Email: Create a view to handle email confirmation.

Handling User Profiles

Create a profile model to store additional user information.

# users/models.py
from django.db import models
from django.contrib.auth.models import User

class Profile(models.Model):
    user = models.OneToOneField(User, on_delete=models.CASCADE)
    image = models.ImageField(default='default.jpg', upload_to='profile_pics')

    def __str__(self):
        return f'{self.user.username} Profile'

Signals for Profile Creation

Use Django signals to automatically create a profile when a user registers.

# users/signals.py
from django.db.models.signals import post_save
from django.contrib.auth.models import User
from django.dispatch import receiver
from .models import Profile

@receiver(post_save, sender=User)
def create_profile(sender, instance, created, **kwargs):
    if created:
        Profile.objects.create(user=instance)

@receiver(post_save, sender=User)
def save_profile(sender, instance, **kwargs):
    instance.profile.save()

Register the signals in the apps.py.

# users/apps.py
from django.apps import AppConfig

class UsersConfig(AppConfig):
    name = 'users'

    def ready(self):
        import users.signals

Conclusion

User registration is a crucial feature for many web applications. In this section, we covered the basics of setting up user registration in Django, including creating forms, views, and templates. We also explored advanced topics like customizing the user model, adding email verification, and handling user profiles. With these tools, you can create a robust user registration system tailored to your application's needs.
