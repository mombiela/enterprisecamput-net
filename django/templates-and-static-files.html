Templates and Static Files

In this section, we will explore how Django handles templates and static files. Templates are used to generate dynamic HTML content, while static files are used for serving CSS, JavaScript, and images. Understanding these concepts is crucial for building a fully functional web application.

Templates

Django templates are a powerful way to generate HTML dynamically. They allow you to separate the presentation layer from the business logic, making your code cleaner and more maintainable.

Key Concepts

  • Template Engine: Django uses its own template engine by default, but it can be configured to use others like Jinja2.
  • Template Tags: Special syntax within templates to perform logic like loops and conditionals.
  • Template Filters: Used to modify variables for display.
  • Template Inheritance: Allows you to create a base template and extend it in other templates.

Creating and Using Templates

  1. Create a Template Directory: By default, Django looks for templates in a directory named templates within each app.

    myapp/
├── templates/
│   └── myapp/
│       └── index.html

  2. Configure Template Settings: Ensure your settings.py file is configured to find your templates.

    TEMPLATES = [
    {
        'BACKEND': 'django.template.backends.django.DjangoTemplates',
        'DIRS': [os.path.join(BASE_DIR, 'templates')],
        'APP_DIRS': True,
        'OPTIONS': {
            'context_processors': [
                'django.template.context_processors.debug',
                'django.template.context_processors.request',
                'django.contrib.auth.context_processors.auth',
                'django.contrib.messages.context_processors.messages',
            ],
        },
    },
]

  3. Create a View to Render the Template: Use Django views to render templates.

    from django.shortcuts import render

def index(request):
    return render(request, 'myapp/index.html', {'message': 'Hello, World!'})

  4. Using Template Tags and Filters: Example of a template using tags and filters.

    <!-- templates/myapp/index.html -->
<!DOCTYPE html>
<html>
<head>
    <title>My Django App</title>
</head>
<body>
    <h1>{{ message|upper }}</h1>
    {% if user.is_authenticated %}
        <p>Welcome, {{ user.username }}!</p>
    {% else %}
        <p>Please log in.</p>
    {% endif %}
</body>
</html>

Template Inheritance

Template inheritance allows you to create a base template that other templates can extend.

  1. Create a Base Template:

    <!-- templates/base.html -->
<!DOCTYPE html>
<html>
<head>
    <title>{% block title %}My Site{% endblock %}</title>
</head>
<body>
    <header>
        <h1>My Site</h1>
    </header>
    <main>
        {% block content %}{% endblock %}
    </main>
    <footer>
        <p>&copy; 2023 My Site</p>
    </footer>
</body>
</html>

  2. Extend the Base Template:

    <!-- templates/myapp/index.html -->
{% extends "base.html" %}

{% block title %}Home{% endblock %}

{% block content %}
    <h2>Welcome to My Site</h2>
    <p>{{ message }}</p>
{% endblock %}

Static Files

Static files include CSS, JavaScript, and images that are served to the client. Django provides a way to manage these files efficiently.

Key Concepts

  • STATIC_URL: The URL to use when referring to static files.
  • STATICFILES_DIRS: A list of directories where Django will also look for static files.
  • collectstatic: A management command to collect static files into a single location for deployment.

Configuring Static Files

  1. Set Up Static Files in settings.py:

    STATIC_URL = '/static/'
STATICFILES_DIRS = [os.path.join(BASE_DIR, 'static')]

  2. Organize Static Files:

    myapp/
├── static/
│   └── myapp/
│       ├── css/
│       │   └── styles.css
│       ├── js/
│       │   └── scripts.js
│       └── images/
│           └── logo.png

  3. Using Static Files in Templates:

    <!-- templates/myapp/index.html -->
<!DOCTYPE html>
<html>
<head>
    <title>My Django App</title>
    <link rel="stylesheet" type="text/css" href="{% static 'myapp/css/styles.css' %}">
</head>
<body>
    <h1>{{ message }}</h1>
    <img src="{% static 'myapp/images/logo.png' %}" alt="Logo">
    <script src="{% static 'myapp/js/scripts.js' %}"></script>
</body>
</html>

  4. Collecting Static Files for Deployment:

    python manage.py collectstatic

Conclusion

In this section, we covered the basics of Django templates and static files. You learned how to create and use templates, including template inheritance, and how to manage static files for your web application. Mastering these concepts will allow you to build dynamic and visually appealing web applications with Django.
