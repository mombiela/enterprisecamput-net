Polymorphism in Objective-C
Categories and Extensions ►

Polymorphism is a core concept in object-oriented programming that allows objects to be treated as instances of their parent class rather than their actual class. This enables a single interface to represent different underlying forms (data types). In Objective-C, polymorphism is achieved through method overriding and dynamic typing.

Key Concepts of Polymorphism

  • Method Overriding: Subclasses can provide a specific implementation of a method that is already defined in its superclass.
  • Dynamic Typing: Objective-C allows you to send messages to objects without knowing their exact type at compile time.
  • id Type: A generic object type that can hold any object.

Method Overriding

Method overriding allows a subclass to provide a specific implementation of a method that is already defined in its superclass. This is a fundamental aspect of polymorphism.

Example

#import <Foundation/Foundation.h>

@interface Animal : NSObject
- (void)speak;
@end

@implementation Animal
- (void)speak {
    NSLog(@"Animal speaks");
}
@end

@interface Dog : Animal
@end

@implementation Dog
- (void)speak {
    NSLog(@"Dog barks");
}
@end

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        Animal *myAnimal = [[Animal alloc] init];
        [myAnimal speak]; // Output: Animal speaks
        
        Dog *myDog = [[Dog alloc] init];
        [myDog speak]; // Output: Dog barks
    }
    return 0;
}

In this example:

  • The Animal class has a method speak.
  • The Dog class overrides the speak method to provide its own implementation.
  • When speak is called on an Animal object, it uses the Animal implementation.
  • When speak is called on a Dog object, it uses the Dog implementation.

Dynamic Typing

Objective-C supports dynamic typing, which allows you to send messages to objects without knowing their exact type at compile time. This is particularly useful for polymorphism.

Example

#import <Foundation/Foundation.h>

@interface Animal : NSObject
- (void)speak;
@end

@implementation Animal
- (void)speak {
    NSLog(@"Animal speaks");
}
@end

@interface Dog : Animal
@end

@implementation Dog
- (void)speak {
    NSLog(@"Dog barks");
}
@end

@interface Cat : Animal
@end

@implementation Cat
- (void)speak {
    NSLog(@"Cat meows");
}
@end

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        NSArray *animals = @[[[Dog alloc] init], [[Cat alloc] init], [[Animal alloc] init]];
        
        for (Animal *animal in animals) {
            [animal speak];
        }
    }
    return 0;
}

In this example:

  • An array of Animal objects is created, containing instances of Dog, Cat, and Animal.
  • The speak method is called on each object in the array.
  • The correct implementation of speak is called for each object, demonstrating polymorphism.

id Type

The id type is a generic object type in Objective-C that can hold any object. This is useful for polymorphism because it allows you to write code that can work with objects of any type.

Example

#import <Foundation/Foundation.h>

@interface Animal : NSObject
- (void)speak;
@end

@implementation Animal
- (void)speak {
    NSLog(@"Animal speaks");
}
@end

@interface Dog : Animal
@end

@implementation Dog
- (void)speak {
    NSLog(@"Dog barks");
}
@end

@interface Cat : Animal
@end

@implementation Cat
- (void)speak {
    NSLog(@"Cat meows");
}
@end

void makeAnimalSpeak(id animal) {
    [animal speak];
}

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        id myDog = [[Dog alloc] init];
        id myCat = [[Cat alloc] init];
        
        makeAnimalSpeak(myDog); // Output: Dog barks
        makeAnimalSpeak(myCat); // Output: Cat meows
    }
    return 0;
}

In this example:

  • The makeAnimalSpeak function takes an id type parameter, allowing it to accept any object.
  • The correct implementation of speak is called for each object passed to makeAnimalSpeak.

Conclusion

Polymorphism in Objective-C allows for flexible and reusable code by enabling objects to be treated as instances of their parent class. Through method overriding, dynamic typing, and the use of the id type, Objective-C provides powerful tools for implementing polymorphism. Understanding and utilizing these concepts will help you write more efficient and maintainable code.
