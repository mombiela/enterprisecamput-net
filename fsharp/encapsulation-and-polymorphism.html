Encapsulation and Polymorphism in F#
Mixing Functional and Object-Oriented Programming ►

Introduction

In this section, we will explore two fundamental concepts of object-oriented programming (OOP) in F#: Encapsulation and Polymorphism. These concepts help in creating robust, maintainable, and scalable software.

Encapsulation

Encapsulation is the concept of bundling data and methods that operate on that data within a single unit, typically a class. It restricts direct access to some of an object's components, which can prevent the accidental modification of data.

Key Concepts of Encapsulation

  • Data Hiding: Restricting access to the internal state of the object.
  • Public Interface: Providing methods to interact with the object's data.

Example of Encapsulation in F#

type Person(name: string, age: int) =
    // Private fields
    let mutable privateName = name
    let mutable privateAge = age

    // Public methods
    member this.GetName() = privateName
    member this.GetAge() = privateAge
    member this.SetAge(newAge: int) =
        if newAge > 0 then
            privateAge <- newAge
        else
            printfn "Age must be positive"

// Usage
let person = Person("John", 30)
printfn "Name: %s" (person.GetName())
printfn "Age: %d" (person.GetAge())
person.SetAge(35)
printfn "Updated Age: %d" (person.GetAge())

In this example:

  • The Person class encapsulates the name and age fields.
  • The fields are private and can only be accessed or modified through public methods (GetName, GetAge, SetAge).

Polymorphism

Polymorphism allows objects of different classes to be treated as objects of a common super class. It is a way to perform a single action in different forms.

Key Concepts of Polymorphism

  • Method Overriding: A subclass provides a specific implementation of a method that is already defined in its superclass.
  • Interface Implementation: Different classes can implement the same interface in different ways.

Example of Polymorphism in F#

Method Overriding

type Animal() =
    abstract member Speak: unit -> unit

type Dog() =
    inherit Animal()
    override this.Speak() = printfn "Woof!"

type Cat() =
    inherit Animal()
    override this.Speak() = printfn "Meow!"

// Usage
let animals: Animal list = [ Dog(); Cat() ]
animals |> List.iter (fun animal -> animal.Speak())

In this example:

  • Animal is an abstract class with an abstract method Speak.
  • Dog and Cat classes inherit from Animal and provide their own implementation of the Speak method.
  • A list of Animal objects is created, containing both Dog and Cat objects. The Speak method is called on each object, demonstrating polymorphism.

Interface Implementation

type IShape =
    abstract member Area: unit -> float

type Circle(radius: float) =
    interface IShape with
        member this.Area() = System.Math.PI * radius * radius

type Rectangle(width: float, height: float) =
    interface IShape with
        member this.Area() = width * height

// Usage
let shapes: IShape list = [ Circle(5.0); Rectangle(4.0, 6.0) ]
shapes |> List.iter (fun shape -> printfn "Area: %f" (shape.Area()))

In this example:

  • IShape is an interface with an Area method.
  • Circle and Rectangle classes implement the IShape interface.
  • A list of IShape objects is created, containing both Circle and Rectangle objects. The Area method is called on each object, demonstrating polymorphism.

Conclusion

Encapsulation and Polymorphism are powerful concepts in F# that help in creating well-structured and flexible code. Encapsulation ensures that an object's internal state is protected, while Polymorphism allows for flexible and reusable code by enabling objects to be treated as instances of their parent class or interface. Understanding and applying these concepts will significantly enhance your ability to write robust and maintainable F# applications.
