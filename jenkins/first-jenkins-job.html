Introduction

In this section, we will guide you through creating your first Jenkins job. Jenkins jobs are the fundamental units of work in Jenkins, allowing you to automate various tasks such as building, testing, and deploying software.

Key Concepts

  • Jenkins Job: A task or a set of tasks that Jenkins executes.
  • Freestyle Project: A flexible and easy-to-configure job type in Jenkins.
  • Build Triggers: Conditions that determine when a job should be executed.
  • Build Steps: Actions that Jenkins performs during the job execution.
  • Post-build Actions: Steps that Jenkins performs after the build steps are completed.

Creating Your First Jenkins Job

Step 1: Access Jenkins Dashboard

  1. Open your web browser and navigate to your Jenkins URL (e.g., http://localhost:8080).
  2. Log in with your credentials.

Step 2: Create a New Job

  1. On the Jenkins dashboard, click on the "New Item" link.
  2. Enter a name for your job (e.g., FirstJenkinsJob).
  3. Select "Freestyle project" and click "OK".

Step 3: Configure the Job

  1. General Section:

    • Optionally, add a description for your job to explain its purpose.

  2. Source Code Management:

    • If your project uses version control, select the appropriate option (e.g., Git) and provide the repository URL.
    • Example: 
      Repository URL: https://github.com/your-repo/your-project.git

  3. Build Triggers:

    • Choose how you want to trigger the build. Common options include:
      • Build periodically: Schedule builds at regular intervals.
      • Poll SCM: Check for changes in the source code repository.
      • Build when a change is pushed to GitHub: Trigger builds on code changes.

  4. Build Environment:

    • Configure any necessary build environment settings, such as setting environment variables or preparing the workspace.

  5. Build Steps:

    • Add build steps to define the actions Jenkins should perform.
    • Example: Execute a shell command to compile code. 
      echo "Compiling code..."
gcc -o myapp main.c

  6. Post-build Actions:

    • Define actions to perform after the build steps are completed.
    • Example: Archive the build artifacts. 
      Archive the artifacts: myapp

Step 4: Save and Build

  1. Click the "Save" button to save your job configuration.
  2. To manually trigger a build, click the "Build Now" link on the job's main page.

Step 5: Monitor the Build

  1. Navigate to the job's main page and click on the build number under the "Build History" section.
  2. View the console output to monitor the build process and check for any errors.

Example Configuration

Here is an example configuration for a simple Jenkins job that compiles a C program:

Job Name: CompileCProgram

General:
  Description: This job compiles a simple C program.

Source Code Management:
  Git:
    Repository URL: https://github.com/your-repo/your-c-program.git

Build Triggers:
  Poll SCM: H/5 * * * *

Build Steps:
  Execute Shell:
    Command:
      echo "Compiling code..."
      gcc -o myapp main.c

Post-build Actions:
  Archive the artifacts: myapp

Conclusion

Creating your first Jenkins job involves setting up a new job, configuring its source code management, build triggers, build steps, and post-build actions. By following the steps outlined in this section, you can automate various tasks and streamline your development workflow. As you become more familiar with Jenkins, you can explore advanced job configurations and plugins to further enhance your CI/CD pipeline.
