Introduction to Continuous Integration (CI)

  • Definition: Continuous Integration (CI) is a development practice where developers integrate code into a shared repository frequently, ideally several times a day.
  • Benefits:
    • Early detection of bugs.
    • Improved software quality.
    • Faster delivery of updates.

Introduction to Jenkins

  • What is Jenkins?: Jenkins is an open-source automation server that helps automate parts of the software development process, including building, testing, and deploying code.
  • Key Features:
    • Extensible with plugins.
    • Supports distributed builds.
    • Easy to install and configure.

Setting Up Jenkins

  • Installation:
    • Windows: Download the Jenkins installer from the official website and follow the setup wizard.
    • Linux: Use package managers like apt or yum to install Jenkins.
    • Docker: Run Jenkins in a Docker container using the command: 
      docker run -p 8080:8080 -p 50000:50000 jenkins/jenkins:lts
  • Initial Configuration:
    • Access Jenkins via http://localhost:8080.
    • Unlock Jenkins using the initial admin password.
    • Install suggested plugins.
    • Create the first admin user.

Creating Your First Jenkins Job

  • Freestyle Project:
    • Navigate to New Item > Freestyle project.
    • Enter a name for the job and click OK.
    • Configure the job:
      • Source Code Management: Connect to a Git repository.
      • Build Triggers: Set up triggers like Poll SCM or Build periodically.
      • Build Steps: Add build steps like Execute shell or Invoke Ant.
      • Post-build Actions: Configure actions like Archive the artifacts or Email notifications.

Example: Simple Shell Script Build

echo "Building the project..."
# Simulate build process
sleep 5
echo "Build completed!"

Setting Up a CI Pipeline with Jenkins

  • Pipeline as Code: Use Jenkins Pipeline (Jenkinsfile) to define your CI pipeline as code.
  • Creating a Pipeline Job:
    • Navigate to New Item > Pipeline.
    • Enter a name for the job and click OK.
    • Configure the pipeline:
      • Pipeline Definition: Choose Pipeline script from SCM if your Jenkinsfile is in a repository, or Pipeline script to write it directly.

Example Jenkinsfile

pipeline {
    agent any
    stages {
        stage('Checkout') {
            steps {
                git 'https://github.com/your-repo/your-project.git'
            }
        }
        stage('Build') {
            steps {
                sh './build.sh'
            }
        }
        stage('Test') {
            steps {
                sh './test.sh'
            }
        }
        stage('Deploy') {
            steps {
                sh './deploy.sh'
            }
        }
    }
    post {
        success {
            echo 'Pipeline completed successfully!'
        }
        failure {
            echo 'Pipeline failed!'
        }
    }
}

Advanced CI Pipeline Concepts

  • Parallel Stages: Run multiple stages in parallel to speed up the pipeline.
  • Conditional Execution: Use conditions to execute stages based on certain criteria.
  • Shared Libraries: Reuse common pipeline code across multiple Jenkinsfiles.

Example: Parallel Stages

pipeline {
    agent any
    stages {
        stage('Parallel Stage') {
            parallel {
                stage('Unit Tests') {
                    steps {
                        sh './run-unit-tests.sh'
                    }
                }
                stage('Integration Tests') {
                    steps {
                        sh './run-integration-tests.sh'
                    }
                }
            }
        }
    }
}

Conclusion

  • Summary:
    • Jenkins is a powerful tool for setting up CI pipelines.
    • Start with basic job configurations and gradually move to more complex pipelines.
    • Use Jenkinsfile to define your CI pipeline as code for better version control and reusability.
  • Next Steps:
    • Explore Jenkins plugins to extend functionality.
    • Integrate Jenkins with other tools like Docker, Kubernetes, and cloud services.
    • Continuously improve your CI pipeline to adapt to your project's needs.
