Introduction to Jenkins and Docker

In this section, we'll explore the basics of Jenkins and Docker, and how they can be used together to streamline your CI/CD pipeline.

What is Jenkins?

  • Jenkins is an open-source automation server.
  • It helps automate parts of the software development process, including building, testing, and deploying code.
  • Jenkins supports continuous integration (CI) and continuous delivery (CD).

What is Docker?

  • Docker is a platform for developing, shipping, and running applications inside containers.
  • Containers are lightweight, portable, and ensure consistency across different environments.

Setting Up Jenkins with Docker

In this section, we'll cover how to set up Jenkins using Docker.

Prerequisites

  • Basic understanding of Docker and Jenkins.
  • Docker installed on your machine.

Running Jenkins in a Docker Container

  1. Pull the Jenkins Docker Image: 
    docker pull jenkins/jenkins:lts
  2. Run the Jenkins Container: 
    docker run -d -p 8080:8080 -p 50000:50000 --name jenkins jenkins/jenkins:lts
    • -d: Run container in detached mode.
    • -p 8080:8080: Map port 8080 of the host to port 8080 of the container.
    • -p 50000:50000: Map port 50000 of the host to port 50000 of the container.
    • --name jenkins: Name the container "jenkins".

Accessing Jenkins

  • Open a web browser and navigate to http://localhost:8080.
  • Follow the on-screen instructions to complete the setup.

Integrating Docker with Jenkins

In this section, we'll learn how to integrate Docker with Jenkins to build and deploy applications.

Installing Docker Plugin in Jenkins

  1. Navigate to Jenkins Dashboard:
    • Go to Manage Jenkins > Manage Plugins.
  2. Install Docker Plugin:
    • Search for "Docker" and install the Docker plugin.

Configuring Docker in Jenkins

  1. Navigate to Jenkins Dashboard:
    • Go to Manage Jenkins > Configure System.
  2. Add Docker Host:
    • Under Cloud, click Add a new cloud and select Docker.
    • Configure Docker host settings (e.g., Docker URL, credentials).

Creating a Jenkins Pipeline with Docker

  1. Create a New Pipeline Job:
    • Go to New Item, enter a name, and select Pipeline.
  2. Define Pipeline Script:
    • Use the following example script to build and run a Docker container: 
      pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                script {
                    docker.build('my-app-image')
                }
            }
        }
        stage('Run') {
            steps {
                script {
                    docker.image('my-app-image').inside {
                        sh 'echo "Running inside Docker container"'
                    }
                }
            }
        }
    }
}

Advanced Jenkins and Docker Usage

In this section, we'll cover more advanced topics, such as using Docker Compose with Jenkins and setting up multi-container applications.

Using Docker Compose in Jenkins Pipelines

  1. Install Docker Compose:
    • Ensure Docker Compose is installed on your Jenkins server.
  2. Define Docker Compose File:
    • Create a docker-compose.yml file for your application. 
      version: '3'
services:
  web:
    image: nginx
    ports:
      - "80:80"
  db:
    image: mysql
    environment:
      MYSQL_ROOT_PASSWORD: example
  3. Update Jenkins Pipeline Script:
    • Use the following example script to run Docker Compose: 
      pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                script {
                    sh 'docker-compose build'
                }
            }
        }
        stage('Run') {
            steps {
                script {
                    sh 'docker-compose up -d'
                }
            }
        }
    }
}

Conclusion

In this topic, we covered the integration of Jenkins and Docker, starting from basic setup to advanced usage. By leveraging Docker within Jenkins, you can create a robust and flexible CI/CD pipeline that ensures consistency across different environments. This integration allows for efficient building, testing, and deployment of applications, ultimately speeding up the development process and improving software quality.
