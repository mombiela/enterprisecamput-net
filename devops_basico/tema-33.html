Introduction

The final project of the Basic DevOps course aims to integrate and apply the knowledge acquired throughout the course. Students will work on a practical project that simulates a real DevOps environment, allowing them to experiment with essential DevOps tools and practices.

Project Objectives

  • Implement a CI/CD pipeline: Configure a Continuous Integration and Continuous Deployment pipeline using tools like Jenkins, GitLab CI, or GitHub Actions.
  • Infrastructure Automation: Use infrastructure as code (IaC) tools like Terraform or Ansible to automate resource provisioning.
  • Monitoring and Logging: Implement monitoring and logging solutions to ensure system observability.
  • Container Management: Use Docker for container creation and management, and Kubernetes for container orchestration.

Project Requirements

CI/CD Pipeline

  • Code Repository: Set up a repository on GitHub or GitLab.
  • Continuous Integration: Configure a pipeline that runs automated tests whenever a commit is made.
  • Continuous Deployment: Configure the pipeline to automatically deploy the application to a staging environment and subsequently to production.
# Example of a pipeline in GitLab CI
stages:
  - build
  - test
  - deploy

build:
  stage: build
  script:
    - echo "Building the application..."
    - ./build.sh

test:
  stage: test
  script:
    - echo "Running tests..."
    - ./run_tests.sh

deploy:
  stage: deploy
  script:
    - echo "Deploying to production..."
    - ./deploy.sh
  only:
    - master

Infrastructure Automation

  • Resource Provisioning: Use Terraform to create and manage cloud resources (AWS, Azure, GCP).
  • Server Configuration: Use Ansible to configure servers and deploy applications.
# Example of Terraform configuration
provider "aws" {
  region = "us-west-2"
}

resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"

  tags = {
    Name = "WebServer"
  }
}

Monitoring and Logging

  • Monitoring: Implement Prometheus for metrics collection and Grafana for visualization.
  • Logging: Configure ELK Stack (Elasticsearch, Logstash, Kibana) for log management.
# Example of Prometheus configuration
global:
  scrape_interval: 15s

scrape_configs:
  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9090']

Container Management

  • Docker: Create Dockerfiles for the application and its dependencies.
  • Kubernetes: Deploy the application in a Kubernetes cluster.
# Example of Dockerfile
FROM node:14
WORKDIR /app
COPY package.json .
RUN npm install
COPY . .
CMD ["npm", "start"]
# Example of Kubernetes configuration
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: web
  template:
    metadata:
      labels:
        app: web
    spec:
      containers:
      - name: web
        image: myapp:latest
        ports:
        - containerPort: 80

Conclusion

The final project is an opportunity to practically apply the concepts and tools learned during the Basic DevOps course. By completing this project, students will have gained experience in configuring CI/CD pipelines, infrastructure automation, monitoring and logging, and container management. This project will not only consolidate their knowledge but also provide them with a solid foundation to face real-world challenges in the DevOps field.
