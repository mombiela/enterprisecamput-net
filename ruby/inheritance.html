Inheritance in Ruby
Modules and Mixins ►

Inheritance is a fundamental concept in object-oriented programming (OOP) that allows a class to inherit attributes and methods from another class. This helps in code reusability and establishing a natural hierarchy between classes.

What is Inheritance?

  • Inheritance allows a new class (subclass or derived class) to inherit properties and behaviors (methods) from an existing class (superclass or base class).
  • The subclass can add its own properties and methods or override the inherited ones.

Basic Syntax

In Ruby, inheritance is implemented using the < symbol.

class Animal
  def speak
    puts "Generic animal sound"
  end
end

class Dog < Animal
  def speak
    puts "Woof!"
  end
end

dog = Dog.new
dog.speak  # Outputs: Woof!

In this example:

  • Animal is the superclass.
  • Dog is the subclass that inherits from Animal.
  • The Dog class overrides the speak method.

Key Concepts

  • Superclass and Subclass: The superclass is the class being inherited from, and the subclass is the class that inherits.
  • Method Overriding: Subclasses can provide a specific implementation of a method that is already defined in its superclass.
  • super Keyword: Used to call methods from the superclass within the subclass.

Method Overriding

When a subclass defines a method with the same name as a method in the superclass, the subclass's method overrides the superclass's method.

class Animal
  def speak
    puts "Generic animal sound"
  end
end

class Cat < Animal
  def speak
    puts "Meow!"
  end
end

cat = Cat.new
cat.speak  # Outputs: Meow!

Using `super`

The super keyword is used to call the same method from the superclass.

class Animal
  def speak
    puts "Generic animal sound"
  end
end

class Dog < Animal
  def speak
    super
    puts "Woof!"
  end
end

dog = Dog.new
dog.speak
# Outputs:
# Generic animal sound
# Woof!

Inheritance and Initialization

When a subclass is initialized, it can call the superclass's initializer using super.

class Animal
  def initialize(name)
    @name = name
  end

  def display_name
    puts "Name: #{@name}"
  end
end

class Dog < Animal
  def initialize(name, breed)
    super(name)
    @breed = breed
  end

  def display_breed
    puts "Breed: #{@breed}"
  end
end

dog = Dog.new("Buddy", "Golden Retriever")
dog.display_name   # Outputs: Name: Buddy
dog.display_breed  # Outputs: Breed: Golden Retriever

Inheritance and Access Control

Ruby provides three levels of access control for methods:

  • Public: Methods are accessible from anywhere.
  • Protected: Methods are accessible within the class and its subclasses.
  • Private: Methods are accessible only within the class.
class Animal
  def public_method
    puts "Public method"
  end

  protected
  def protected_method
    puts "Protected method"
  end

  private
  def private_method
    puts "Private method"
  end
end

class Dog < Animal
  def access_methods
    public_method
    protected_method
    # private_method  # This will raise an error
  end
end

dog = Dog.new
dog.access_methods
# Outputs:
# Public method
# Protected method

Conclusion

Inheritance is a powerful feature in Ruby that promotes code reuse and logical class hierarchy. By understanding and utilizing inheritance, you can create more organized and maintainable code. Remember to use method overriding and the super keyword effectively to extend and customize the behavior of inherited methods. Access control helps in encapsulating and protecting the internal state of objects, ensuring that methods are used appropriately within the class hierarchy.
