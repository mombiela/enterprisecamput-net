Introduction to Classes and Objects

In Ruby, everything is an object. Objects are instances of classes, which define the blueprint for the objects. Understanding classes and objects is fundamental to mastering Ruby.

Key Concepts

  • Class: A blueprint for creating objects.
  • Object: An instance of a class.
  • Instance Variables: Variables that belong to an instance of a class.
  • Methods: Functions defined within a class to perform actions on objects.

Defining a Class

To define a class in Ruby, use the class keyword followed by the class name. Class names are written in CamelCase.

class Person
  # Class body goes here
end

Example: Creating a Simple Class

Let's create a simple Person class with instance variables and methods.

class Person
  def initialize(name, age)
    @name = name
    @age = age
  end

  def display_details
    puts "Name: #{@name}, Age: #{@age}"
  end
end

# Creating an object of the Person class
person1 = Person.new("Alice", 30)
person1.display_details  # Output: Name: Alice, Age: 30

Explanation

  • initialize method: A special method called when a new object is created. It initializes the object's instance variables.
  • @name and @age: Instance variables that store the state of the object.
  • display_details method: A method to display the object's details.

Instance Variables and Methods

Instance variables are prefixed with @ and are used to store the state of an object. Methods are used to define the behavior of the objects.

Example: Adding More Methods

Let's add more methods to our Person class.

class Person
  def initialize(name, age)
    @name = name
    @age = age
  end

  def display_details
    puts "Name: #{@name}, Age: #{@age}"
  end

  def birthday
    @age += 1
  end
end

person1 = Person.new("Alice", 30)
person1.display_details  # Output: Name: Alice, Age: 30
person1.birthday
person1.display_details  # Output: Name: Alice, Age: 31

Explanation

  • birthday method: Increases the age of the person by 1.

Class Methods and Variables

Class methods and variables are associated with the class itself rather than any instance of the class.

Example: Class Methods and Variables

Let's add a class variable and a class method to our Person class.

class Person
  @@count = 0  # Class variable

  def initialize(name, age)
    @name = name
    @age = age
    @@count += 1
  end

  def display_details
    puts "Name: #{@name}, Age: #{@age}"
  end

  def self.total_count
    @@count
  end
end

person1 = Person.new("Alice", 30)
person2 = Person.new("Bob", 25)
puts Person.total_count  # Output: 2

Explanation

  • @@count: A class variable that keeps track of the number of Person objects created.
  • self.total_count: A class method that returns the value of the class variable @@count.

Inheritance

Inheritance allows a class to inherit features from another class.

Example: Inheritance

Let's create a Student class that inherits from the Person class.

class Student < Person
  def initialize(name, age, grade)
    super(name, age)
    @grade = grade
  end

  def display_details
    super
    puts "Grade: #{@grade}"
  end
end

student1 = Student.new("Charlie", 20, "A")
student1.display_details  # Output: Name: Charlie, Age: 20, Grade: A

Explanation

  • < Person: Indicates that Student inherits from Person.
  • super: Calls the initialize method of the parent class to initialize name and age.

Conclusion

Understanding classes and objects is crucial for Ruby programming. Classes define the blueprint for objects, and objects are instances of classes. Instance variables store the state of an object, while methods define the behavior. Class methods and variables are associated with the class itself. Inheritance allows a class to inherit features from another class, promoting code reuse and organization.

By mastering these concepts, you can create complex and efficient Ruby programs.
