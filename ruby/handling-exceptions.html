Introduction to Exceptions

In Ruby, exceptions are a way to handle errors that occur during the execution of a program. Instead of crashing the program, Ruby provides mechanisms to catch and handle these errors gracefully.

  • Exception: An event that disrupts the normal flow of a program.
  • Exception Handling: The process of responding to exceptions when they occur.

Basic Exception Handling

Ruby uses the begin, rescue, and end keywords to handle exceptions.

Syntax

begin
  # Code that might raise an exception
rescue
  # Code that runs if an exception occurs
end

Example

begin
  puts "Enter a number:"
  number = gets.to_i
  result = 10 / number
  puts "Result is #{result}"
rescue
  puts "An error occurred: Division by zero is not allowed."
end

In this example:

  • The begin block contains code that might raise an exception.
  • The rescue block contains code that runs if an exception occurs.

Handling Specific Exceptions

You can rescue specific types of exceptions by specifying the exception class.

Syntax

begin
  # Code that might raise an exception
rescue SpecificExceptionClass
  # Code that runs if a specific exception occurs
end

Example

begin
  puts "Enter a number:"
  number = gets.to_i
  result = 10 / number
  puts "Result is #{result}"
rescue ZeroDivisionError
  puts "An error occurred: Division by zero is not allowed."
end

In this example:

  • The rescue ZeroDivisionError block only catches ZeroDivisionError exceptions.

Ensuring Cleanup with `ensure`

The ensure block runs whether an exception occurs or not, making it useful for cleanup tasks.

Syntax

begin
  # Code that might raise an exception
rescue
  # Code that runs if an exception occurs
ensure
  # Code that always runs
end

Example

begin
  file = File.open("example.txt", "r")
  # Perform file operations
rescue
  puts "An error occurred while opening the file."
ensure
  file.close if file
  puts "File closed."
end

In this example:

  • The ensure block ensures that the file is closed whether an exception occurs or not.

Raising Exceptions

You can raise your own exceptions using the raise keyword.

Syntax

raise "Error message"

Example

def divide(a, b)
  raise "Division by zero is not allowed" if b == 0
  a / b
end

begin
  result = divide(10, 0)
  puts "Result is #{result}"
rescue => e
  puts "An error occurred: #{e.message}"
end

In this example:

  • The raise keyword is used to raise an exception if b is zero.
  • The rescue block catches the exception and prints the error message.

Custom Exception Classes

You can define your own exception classes by inheriting from StandardError.

Example

class MyCustomError < StandardError; end

def risky_method
  raise MyCustomError, "Something went wrong!"
end

begin
  risky_method
rescue MyCustomError => e
  puts "Caught a custom error: #{e.message}"
end

In this example:

  • MyCustomError is a custom exception class.
  • The rescue MyCustomError block catches exceptions of type MyCustomError.

Conclusion

Handling exceptions is a crucial part of writing robust and error-resistant Ruby programs. By using begin, rescue, ensure, and custom exception classes, you can manage errors gracefully and ensure your program can handle unexpected situations without crashing. Practice these concepts with various scenarios to become proficient in exception handling in Ruby.
