Introduction

In Ruby, understanding class variables and methods is crucial for mastering object-oriented programming. This section will cover the basics of class variables and methods, their usage, and how they differ from instance variables and methods.

Class Variables

Class variables are shared among all instances of a class. They are prefixed with @@ and are used to store information that is common to all objects of a class.

Key Concepts

  • Class Variable Declaration: Class variables are declared within the class body, prefixed with @@.
  • Shared State: All instances of a class share the same class variable.
  • Initialization: Class variables can be initialized directly within the class body or within class methods.

Example

class Car
  @@number_of_cars = 0

  def initialize(model)
    @model = model
    @@number_of_cars += 1
  end

  def self.total_cars
    @@number_of_cars
  end
end

car1 = Car.new("Toyota")
car2 = Car.new("Honda")

puts Car.total_cars # Output: 2

Explanation:

  • @@number_of_cars is a class variable.
  • It is incremented each time a new Car object is created.
  • self.total_cars is a class method that returns the value of @@number_of_cars.

Class Methods

Class methods are methods that are called on the class itself, rather than on instances of the class. They are defined using self. before the method name.

Key Concepts

  • Class Method Declaration: Use self. before the method name.
  • Accessing Class Variables: Class methods can access and modify class variables.
  • Utility Functions: Often used for utility functions that are related to the class but not to any specific instance.

Example

class Calculator
  def self.add(a, b)
    a + b
  end

  def self.subtract(a, b)
    a - b
  end
end

puts Calculator.add(5, 3)      # Output: 8
puts Calculator.subtract(5, 3) # Output: 2

Explanation:

  • self.add and self.subtract are class methods.
  • They perform operations that do not depend on instance-specific data.

Comparison: Class Variables vs Instance Variables

| Aspect | Class Variables (@@) | Instance Variables (@) | |-----------------------|------------------------|--------------------------| | Scope | Shared across all instances | Unique to each instance | | Initialization | Within class body or class methods | Within instance methods (e.g., initialize) | | Access | Accessible by class methods and instance methods | Accessible only by instance methods | | Use Case | Common data for all instances | Data specific to each instance |

Exercises

Exercise 1: Class Variable

Create a class Library that keeps track of the total number of books across all instances.

class Library
  @@total_books = 0

  def initialize(books)
    @books = books
    @@total_books += books
  end

  def self.total_books
    @@total_books
  end
end

# Create instances and test the class variable
library1 = Library.new(100)
library2 = Library.new(200)

puts Library.total_books # Expected output: 300

Exercise 2: Class Method

Create a class TemperatureConverter with class methods to convert temperatures between Celsius and Fahrenheit.

class TemperatureConverter
  def self.celsius_to_fahrenheit(celsius)
    (celsius * 9.0 / 5) + 32
  end

  def self.fahrenheit_to_celsius(fahrenheit)
    (fahrenheit - 32) * 5.0 / 9
  end
end

# Test the class methods
puts TemperatureConverter.celsius_to_fahrenheit(0)    # Expected output: 32
puts TemperatureConverter.fahrenheit_to_celsius(32)   # Expected output: 0

Conclusion

Class variables and methods are powerful tools in Ruby that allow you to manage shared data and functionality at the class level. Understanding when and how to use them will help you write more efficient and organized code. Remember:

  • Use class variables for data shared across all instances.
  • Use class methods for functionality that pertains to the class itself rather than any specific instance.

By mastering these concepts, you'll be well-equipped to handle more complex object-oriented programming tasks in Ruby.
