Introduction to Constants

In Ruby, constants are used to store values that are not meant to change throughout the execution of a program. They are a fundamental part of Ruby programming and help in maintaining the integrity of data that should remain constant.

  • Definition: Constants are variables whose values are intended to remain unchanged.
  • Naming Convention: Constants are written in uppercase letters, with words separated by underscores (_).

Declaring Constants

To declare a constant in Ruby, simply assign a value to a variable name that starts with an uppercase letter.

PI = 3.14159
MAX_USERS = 100
WELCOME_MESSAGE = "Welcome to Ruby Programming!"
  • PI, MAX_USERS, and WELCOME_MESSAGE are constants.
  • By convention, constants are written in all uppercase letters.

Accessing Constants

Constants can be accessed directly by their names within the same scope.

puts PI  # Output: 3.14159
puts MAX_USERS  # Output: 100
puts WELCOME_MESSAGE  # Output: Welcome to Ruby Programming!

Constants in Classes and Modules

Constants can also be defined within classes and modules. They are accessed using the scope resolution operator ::.

class Circle
  PI = 3.14159
end

module Config
  MAX_USERS = 100
end

puts Circle::PI  # Output: 3.14159
puts Config::MAX_USERS  # Output: 100
  • Circle::PI accesses the PI constant within the Circle class.
  • Config::MAX_USERS accesses the MAX_USERS constant within the Config module.

Modifying Constants

Although constants are meant to remain unchanged, Ruby does allow you to modify them. However, doing so will generate a warning.

PI = 3.14159
PI = 3.14  # Warning: already initialized constant PI
  • Modifying a constant is generally discouraged as it goes against the purpose of using constants.

Constants and Inheritance

When a class inherits from another class, it also inherits its constants.

class Parent
  CONSTANT = "Parent Constant"
end

class Child < Parent
end

puts Child::CONSTANT  # Output: Parent Constant
  • Child class inherits the CONSTANT from the Parent class.

Exercises

  1. Declare and Access Constants: Create a constant GRAVITY with a value of 9.8 and print it.
  2. Constants in Classes: Define a class MathConstants with a constant E (Euler's number) set to 2.718. Access and print this constant.
  3. Modify a Constant: Declare a constant SPEED_OF_LIGHT with a value of 299792458. Attempt to change its value and observe the warning.

Example Solutions

  1. Declare and Access Constants

    GRAVITY = 9.8
puts GRAVITY  # Output: 9.8

  2. Constants in Classes

    class MathConstants
  E = 2.718
end

puts MathConstants::E  # Output: 2.718

  3. Modify a Constant

    SPEED_OF_LIGHT = 299792458
SPEED_OF_LIGHT = 300000000  # Warning: already initialized constant SPEED_OF_LIGHT

Conclusion

Constants in Ruby are a powerful feature that helps maintain the integrity of values that should not change throughout the execution of a program. By following the conventions and understanding the scope and inheritance of constants, you can write more robust and maintainable Ruby code. Remember, while Ruby allows modifying constants, it is best practice to avoid doing so to preserve the intended immutability of these values.
