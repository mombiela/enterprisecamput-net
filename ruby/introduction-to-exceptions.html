What are Exceptions?

Exceptions are events that can alter the normal flow of a program. They are used to handle errors and other exceptional conditions in a controlled manner. In Ruby, exceptions are objects that represent these error conditions.

Key Concepts

  • Exception Handling: The process of responding to the occurrence of exceptions.
  • Raise: The action of creating an exception.
  • Rescue: The action of handling an exception.
  • Ensure: A block that will always execute, regardless of whether an exception was raised or not.

Raising Exceptions

In Ruby, you can raise an exception using the raise keyword. This can be done with or without an argument.

Example: Raising an Exception

# Raising an exception without an argument
raise "An error has occurred!"

# Raising an exception with a specific type
raise ArgumentError, "Invalid argument!"

Handling Exceptions

To handle exceptions, Ruby provides the begin-rescue block. This allows you to catch and manage exceptions gracefully.

Example: Handling an Exception

begin
  # Code that may raise an exception
  result = 10 / 0
rescue ZeroDivisionError => e
  # Code to handle the exception
  puts "Error: #{e.message}"
end

Explanation

  • begin: Starts the block of code where exceptions might occur.
  • rescue: Defines the block of code to execute if an exception is raised.
  • ZeroDivisionError: The specific type of exception being rescued.
  • e: The variable that holds the exception object.

Ensuring Execution

The ensure block is used to guarantee that certain code will always run, regardless of whether an exception was raised or not.

Example: Using Ensure

begin
  # Code that may raise an exception
  file = File.open("example.txt", "r")
  # Perform operations on the file
rescue Errno::ENOENT => e
  # Handle file not found exception
  puts "File not found: #{e.message}"
ensure
  # Ensure the file is closed
  file.close if file
end

Common Exception Types

Here are some common exceptions you might encounter in Ruby:

| Exception Type | Description | |----------------------|---------------------------------------| | StandardError | The default for most errors | | ArgumentError | Raised when arguments are wrong | | NoMethodError | Raised when a method is not found | | RuntimeError | A generic error raised by raise | | ZeroDivisionError | Raised when dividing by zero | | TypeError | Raised when an object is of the wrong type |

Custom Exceptions

You can define your own exceptions by creating a new class that inherits from StandardError.

Example: Custom Exception

class MyCustomError < StandardError
end

begin
  # Code that may raise a custom exception
  raise MyCustomError, "Something went wrong!"
rescue MyCustomError => e
  # Handle the custom exception
  puts "Custom error: #{e.message}"
end

Summary

  • Exceptions are used to handle errors and exceptional conditions in Ruby.
  • Use raise to create exceptions and rescue to handle them.
  • The ensure block guarantees that certain code will always run.
  • Ruby provides a variety of built-in exception types, and you can also create custom exceptions.

By understanding and utilizing exceptions, you can make your Ruby programs more robust and error-resistant.
