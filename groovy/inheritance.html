Introduction to Inheritance

Inheritance is a fundamental concept in object-oriented programming (OOP) that allows a class to inherit properties and methods from another class. This promotes code reusability and establishes a natural hierarchy between classes.

  • Superclass (Parent Class): The class whose properties and methods are inherited.
  • Subclass (Child Class): The class that inherits from the superclass.

Basic Inheritance in Groovy

In Groovy, inheritance is implemented using the extends keyword. Let's start with a simple example to illustrate basic inheritance.

Example: Basic Inheritance

// Superclass
class Animal {
    String name

    void eat() {
        println("${name} is eating")
    }
}

// Subclass
class Dog extends Animal {
    void bark() {
        println("${name} is barking")
    }
}

// Creating an instance of Dog
Dog dog = new Dog()
dog.name = "Buddy"
dog.eat()  // Inherited method
dog.bark() // Subclass method

Explanation

  • The Animal class is the superclass with a property name and a method eat().
  • The Dog class is the subclass that extends Animal and adds a new method bark().
  • An instance of Dog can access both the eat() method from Animal and the bark() method defined in Dog.

Overriding Methods

A subclass can override methods of its superclass to provide a specific implementation.

Example: Overriding Methods

class Animal {
    String name

    void eat() {
        println("${name} is eating")
    }
}

class Dog extends Animal {
    @Override
    void eat() {
        println("${name} is eating dog food")
    }
}

Dog dog = new Dog()
dog.name = "Buddy"
dog.eat()  // Outputs: Buddy is eating dog food

Explanation

  • The Dog class overrides the eat() method of the Animal class.
  • The @Override annotation is used to indicate that the method is being overridden.

Using `super` Keyword

The super keyword is used to call the superclass's methods and constructors from the subclass.

Example: Using `super` Keyword

class Animal {
    String name

    Animal(String name) {
        this.name = name
    }

    void eat() {
        println("${name} is eating")
    }
}

class Dog extends Animal {
    Dog(String name) {
        super(name)  // Calling the superclass constructor
    }

    @Override
    void eat() {
        super.eat()  // Calling the superclass method
        println("${name} is eating dog food")
    }
}

Dog dog = new Dog("Buddy")
dog.eat()
// Outputs:
// Buddy is eating
// Buddy is eating dog food

Explanation

  • The Dog constructor calls the Animal constructor using super(name).
  • The eat() method in Dog calls the eat() method of Animal using super.eat() before adding its own behavior.

Abstract Classes

Abstract classes cannot be instantiated and are meant to be subclassed. They can contain abstract methods that must be implemented by subclasses.

Example: Abstract Classes

abstract class Animal {
    String name

    Animal(String name) {
        this.name = name
    }

    abstract void makeSound()  // Abstract method
}

class Dog extends Animal {
    Dog(String name) {
        super(name)
    }

    @Override
    void makeSound() {
        println("${name} is barking")
    }
}

Dog dog = new Dog("Buddy")
dog.makeSound()  // Outputs: Buddy is barking

Explanation

  • The Animal class is abstract and contains an abstract method makeSound().
  • The Dog class extends Animal and provides an implementation for the makeSound() method.

Summary

Inheritance in Groovy allows classes to inherit properties and methods from other classes, promoting code reuse and establishing a hierarchy. Key concepts include:

  • Basic inheritance using the extends keyword.
  • Overriding methods to provide specific implementations.
  • Using the super keyword to call superclass methods and constructors.
  • Abstract classes that define abstract methods to be implemented by subclasses.

By understanding and utilizing inheritance, you can create more modular, maintainable, and reusable code in Groovy.
