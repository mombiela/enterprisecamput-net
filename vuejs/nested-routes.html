Introduction to Nested Routes

Nested routes in Vue.js allow you to create complex and hierarchical navigation structures within your application. This is particularly useful for applications with multiple layers of views, such as dashboards, user profiles, or e-commerce sites with product categories and subcategories.

Key Concepts

  • Nested Routes: Routes within routes, allowing for a hierarchical structure.
  • Router View: A component that renders the matched component for a given route.
  • Child Routes: Routes that are nested within a parent route.

Setting Up Nested Routes

To set up nested routes, you need to define them in your Vue Router configuration. Here’s a step-by-step guide:

  1. Install Vue Router: If you haven't already, install Vue Router in your Vue.js project.

    npm install vue-router

  2. Define Routes: Create a router.js file (or modify your existing router configuration) to define your nested routes.

Example: Basic Nested Routes

Consider a simple application with a User component that has nested routes for Profile and Posts.

// router.js
import Vue from 'vue';
import Router from 'vue-router';
import Home from './components/Home.vue';
import User from './components/User.vue';
import UserProfile from './components/UserProfile.vue';
import UserPosts from './components/UserPosts.vue';

Vue.use(Router);

const routes = [
  { path: '/', component: Home },
  {
    path: '/user/:id',
    component: User,
    children: [
      {
        path: 'profile',
        component: UserProfile
      },
      {
        path: 'posts',
        component: UserPosts
      }
    ]
  }
];

const router = new Router({
  routes
});

export default router;

Explanation

  • Parent Route: /user/:id - This route will render the User component.
  • Child Routes: /user/:id/profile and /user/:id/posts - These routes will render the UserProfile and UserPosts components respectively, within the User component.

Router View in Parent Component

To render the child components, you need to include a <router-view> in the parent component.

<!-- User.vue -->
<template>
  <div>
    <h2>User {{ $route.params.id }}</h2>
    <router-view></router-view> <!-- This is where child components will be rendered -->
  </div>
</template>

<script>
export default {
  name: 'User'
};
</script>

Advanced Nested Routes

For more complex applications, you might need deeper nested routes or dynamic nested routes.

Example: Deeper Nested Routes

Imagine an application where a user has multiple projects, and each project has multiple tasks.

// router.js
import Vue from 'vue';
import Router from 'vue-router';
import Home from './components/Home.vue';
import User from './components/User.vue';
import UserProfile from './components/UserProfile.vue';
import UserPosts from './components/UserPosts.vue';
import Project from './components/Project.vue';
import Task from './components/Task.vue';

Vue.use(Router);

const routes = [
  { path: '/', component: Home },
  {
    path: '/user/:id',
    component: User,
    children: [
      {
        path: 'profile',
        component: UserProfile
      },
      {
        path: 'posts',
        component: UserPosts
      },
      {
        path: 'project/:projectId',
        component: Project,
        children: [
          {
            path: 'task/:taskId',
            component: Task
          }
        ]
      }
    ]
  }
];

const router = new Router({
  routes
});

export default router;

Explanation

  • Deeper Nested Routes: The Project component is nested within the User component, and the Task component is nested within the Project component.
  • Dynamic Segments: Both projectId and taskId are dynamic segments, allowing for dynamic routing based on the project and task IDs.

Router View in Nested Components

Ensure each parent component includes a <router-view> to render its child components.

<!-- Project.vue -->
<template>
  <div>
    <h3>Project {{ $route.params.projectId }}</h3>
    <router-view></router-view> <!-- This is where Task component will be rendered -->
  </div>
</template>

<script>
export default {
  name: 'Project'
};
</script>

Summary

Nested routes in Vue.js provide a powerful way to structure your application’s navigation. By defining routes within routes, you can create complex, hierarchical views that are easy to manage and navigate. Remember to:

  • Define nested routes in your router configuration.
  • Use <router-view> in parent components to render child components.
  • Utilize dynamic segments for flexible and dynamic routing.

By mastering nested routes, you can build sophisticated and user-friendly applications with Vue.js.
