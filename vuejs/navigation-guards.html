Introduction to Navigation Guards

Navigation Guards are an essential feature in Vue.js that allow you to control the navigation of your application. They provide hooks that can be used to guard routes, perform actions before or after navigation, and even cancel navigation based on certain conditions.

Types of Navigation Guards

Vue.js offers several types of navigation guards:

  • Global Guards: Apply to every route in the application.
  • Per-Route Guards: Apply to specific routes.
  • In-Component Guards: Apply to specific components.

Global Guards

Global guards are defined at the router level and apply to all routes.

`beforeEach` Guard

The beforeEach guard is called before every navigation. It can be used to perform checks or fetch data before allowing the navigation to proceed.

const router = new VueRouter({
  routes: [
    // your routes here
  ]
});

router.beforeEach((to, from, next) => {
  // Perform some logic
  if (to.meta.requiresAuth && !isAuthenticated()) {
    next('/login');
  } else {
    next();
  }
});
  • to: The target Route Object being navigated to.
  • from: The current route being navigated away from.
  • next: A function that must be called to resolve the hook.

`beforeResolve` Guard

The beforeResolve guard is similar to beforeEach but is called right before the navigation is confirmed, after all beforeEach guards and async route components are resolved.

router.beforeResolve((to, from, next) => {
  // Perform some logic
  next();
});

`afterEach` Guard

The afterEach guard is called after every navigation. It does not affect the navigation itself and is typically used for analytics or cleanup tasks.

router.afterEach((to, from) => {
  // Perform some logic
  console.log(`Navigated to ${to.path} from ${from.path}`);
});

Per-Route Guards

Per-route guards are defined directly in the route configuration and apply only to specific routes.

const routes = [
  {
    path: '/dashboard',
    component: Dashboard,
    beforeEnter: (to, from, next) => {
      // Perform some logic
      if (!isAuthenticated()) {
        next('/login');
      } else {
        next();
      }
    }
  }
];

In-Component Guards

In-component guards are defined within the component itself and apply only when the component is being navigated to or from.

`beforeRouteEnter`

Called before the route that renders this component is confirmed. It does not have access to this because the component instance has not been created yet.

export default {
  beforeRouteEnter(to, from, next) {
    // Perform some logic
    next(vm => {
      // Access component instance via `vm`
    });
  }
};

`beforeRouteUpdate`

Called when the route that renders this component has changed but the component is reused in the new route.

export default {
  beforeRouteUpdate(to, from, next) {
    // Perform some logic
    next();
  }
};

`beforeRouteLeave`

Called when the route that renders this component is about to be navigated away from.

export default {
  beforeRouteLeave(to, from, next) {
    // Perform some logic
    next();
  }
};

Summary

Navigation Guards in Vue.js provide a powerful way to control the flow of your application. By using global, per-route, and in-component guards, you can ensure that your application behaves as expected during navigation. Whether you need to check for authentication, fetch data, or perform cleanup tasks, navigation guards offer a flexible solution to handle these scenarios effectively.
