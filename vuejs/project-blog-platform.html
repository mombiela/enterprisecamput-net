Introduction

In this project, we will build a fully functional blog platform using Vue.js. This project will help you understand how to create a complex application by breaking it down into manageable components. By the end of this project, you will have a solid understanding of Vue.js and how to use it to build dynamic web applications.

Key Concepts

  • Component-based architecture
  • Vue Router for navigation
  • Vuex for state management
  • API integration
  • Form handling and validation
  • Dynamic rendering

Setting Up the Project

First, let's set up our Vue.js project using Vue CLI.

# Install Vue CLI if you haven't already
npm install -g @vue/cli

# Create a new project
vue create blog-platform

# Navigate to the project directory
cd blog-platform

# Start the development server
npm run serve

Project Structure

Setting Up Vue Router

We need to set up Vue Router to handle navigation between different pages.

// src/router/index.js
import Vue from 'vue'
import VueRouter from 'vue-router'
import Home from '../views/Home.vue'
import About from '../views/About.vue'

Vue.use(VueRouter)

const routes = [
  {
    path: '/',
    name: 'Home',
    component: Home
  },
  {
    path: '/about',
    name: 'About',
    component: About
  }
]

const router = new VueRouter({
  mode: 'history',
  base: process.env.BASE_URL,
  routes
})

export default router

State Management with Vuex

To manage the state of our application, we will use Vuex.

// src/store/index.js
import Vue from 'vue'
import Vuex from 'vuex'

Vue.use(Vuex)

export default new Vuex.Store({
  state: {
    posts: []
  },
  mutations: {
    setPosts(state, posts) {
      state.posts = posts
    }
  },
  actions: {
    fetchPosts({ commit }) {
      // Simulate an API call
      const posts = [
        { id: 1, title: 'First Post', content: 'This is the first post.' },
        { id: 2, title: 'Second Post', content: 'This is the second post.' }
      ]
      commit('setPosts', posts)
    }
  },
  getters: {
    allPosts: state => state.posts
  }
})

Integrating API

Let's simulate fetching posts from an API.

// src/views/Home.vue
<template>
  <div>
    <Header />
    <div v-for="post in posts" :key="post.id">
      <Post :post="post" />
    </div>
  </div>
</template>

<script>
import Header from '@/components/Header.vue'
import Post from '@/components/Post.vue'
import { mapGetters, mapActions } from 'vuex'

export default {
  name: 'Home',
  components: {
    Header,
    Post
  },
  computed: {
    ...mapGetters(['allPosts'])
  },
  methods: {
    ...mapActions(['fetchPosts'])
  },
  created() {
    this.fetchPosts()
  }
}
</script>

Form Handling and Validation

Let's add a form to create new blog posts.

<!-- src/components/NewPost.vue -->
<template>
  <div>
    <h2>Create New Post</h2>
    <form @submit.prevent="createPost">
      <div>
        <label for="title">Title</label>
        <input type="text" v-model="title" required />
      </div>
      <div>
        <label for="content">Content</label>
        <textarea v-model="content" required></textarea>
      </div>
      <button type="submit">Create</button>
    </form>
  </div>
</template>

<script>
export default {
  name: 'NewPost',
  data() {
    return {
      title: '',
      content: ''
    }
  },
  methods: {
    createPost() {
      // Simulate API call to create a new post
      const newPost = {
        id: Date.now(),
        title: this.title,
        content: this.content
      }
      this.$store.commit('setPosts', [...this.$store.state.posts, newPost])
      this.title = ''
      this.content = ''
    }
  }
}
</script>

}
</style>
