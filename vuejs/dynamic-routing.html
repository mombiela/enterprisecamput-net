Dynamic Routing in Vue.js
Nested Routes ►

Dynamic routing is a powerful feature in Vue.js that allows you to create routes that can change based on user input or other dynamic data. This is essential for building applications that need to handle a variety of paths and parameters.

Key Concepts

  • Dynamic Segments: Parts of the URL that can change, often represented with a colon (:) in the route path.
  • Route Parameters: Variables that are passed through the URL and can be accessed within the component.
  • Nested Routes: Routes that are nested within other routes, allowing for more complex and hierarchical navigation structures.
  • Programmatic Navigation: Using Vue Router's API to navigate to different routes programmatically.

Setting Up Vue Router

First, ensure you have Vue Router installed in your Vue.js project. If not, you can install it using npm:

npm install vue-router

Then, set up your router in src/router/index.js:

import Vue from 'vue';
import VueRouter from 'vue-router';
import Home from '../views/Home.vue';
import User from '../views/User.vue';

Vue.use(VueRouter);

const routes = [
  {
    path: '/',
    name: 'Home',
    component: Home
  },
  {
    path: '/user/:id',
    name: 'User',
    component: User
  }
];

const router = new VueRouter({
  mode: 'history',
  base: process.env.BASE_URL,
  routes
});

export default router;

Dynamic Segments and Route Parameters

Dynamic segments in routes are defined using a colon (:) followed by the parameter name. For example, /user/:id defines a route with a dynamic segment id.

Example

In the User.vue component, you can access the route parameter id using this.$route.params.id:

<template>
  <div>
    <h1>User Profile</h1>
    <p>User ID: {{ userId }}</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      userId: this.$route.params.id
    };
  },
  watch: {
    '$route.params.id'(newId) {
      this.userId = newId;
    }
  }
};
</script>

Nested Routes

Nested routes allow you to create more complex route structures. For example, you might have a user profile route with nested routes for different sections of the profile.

Example

Define nested routes in src/router/index.js:

const routes = [
  {
    path: '/user/:id',
    component: User,
    children: [
      {
        path: 'profile',
        component: UserProfile
      },
      {
        path: 'posts',
        component: UserPosts
      }
    ]
  }
];

In the User.vue component, use <router-view> to display the nested routes:

<template>
  <div>
    <h1>User Profile</h1>
    <router-link :to="{ name: 'UserProfile', params: { id: userId } }">Profile</router-link>
    <router-link :to="{ name: 'UserPosts', params: { id: userId } }">Posts</router-link>
    <router-view></router-view>
  </div>
</template>

<script>
export default {
  data() {
    return {
      userId: this.$route.params.id
    };
  }
};
</script>

Programmatic Navigation

You can navigate to different routes programmatically using the Vue Router API. This is useful for navigation based on user actions or other dynamic conditions.

Example

In a method within your component, use this.$router.push to navigate to a different route:

<template>
  <div>
    <button @click="goToUserProfile">Go to User Profile</button>
  </div>
</template>

<script>
export default {
  methods: {
    goToUserProfile() {
      this.$router.push({ name: 'User', params: { id: 123 } });
    }
  }
};
</script>

Conclusion

Dynamic routing in Vue.js allows you to create flexible and powerful navigation structures in your applications. By understanding dynamic segments, route parameters, nested routes, and programmatic navigation, you can build complex and user-friendly interfaces. Practice these concepts with different scenarios to become proficient in dynamic routing with Vue.js.
