Introduction to Vue Router

Vue Router is the official router for Vue.js, enabling developers to create single-page applications with multiple views and navigation. This section will introduce you to the basics of Vue Router, its installation, and how to use it to manage routes in your Vue.js applications.

What is Vue Router?

Vue Router is a library that integrates with Vue.js to provide routing capabilities. It allows you to:

  • Define routes and map them to components.
  • Navigate between different views.
  • Handle dynamic routing and nested routes.
  • Manage navigation guards and route transitions.

Installing Vue Router

To get started with Vue Router, you need to install it in your Vue.js project. You can do this using npm or yarn.

Using npm:

npm install vue-router

Using yarn:

yarn add vue-router

Setting Up Vue Router

Once installed, you need to set up Vue Router in your Vue.js application. This involves creating a router instance and defining your routes.

Basic Setup

  1. Create a router.js file:
// src/router.js
import Vue from 'vue';
import Router from 'vue-router';
import Home from './components/Home.vue';
import About from './components/About.vue';

Vue.use(Router);

const routes = [
  { path: '/', component: Home },
  { path: '/about', component: About }
];

const router = new Router({
  routes
});

export default router;
  1. Integrate the router with your Vue instance:
// src/main.js
import Vue from 'vue';
import App from './App.vue';
import router from './router';

new Vue({
  render: h => h(App),
  router
}).$mount('#app');

Defining Routes

Routes are defined as objects within an array. Each route object typically contains:

  • path: The URL path.
  • component: The component to render when the path is matched.

Example:

const routes = [
  { path: '/', component: Home },
  { path: '/about', component: About },
  { path: '/contact', component: Contact }
];

Navigating Between Routes

Vue Router provides several ways to navigate between routes:

  • Using <router-link> component:
<template>
  <div>
    <router-link to="/">Home</router-link>
    <router-link to="/about">About</router-link>
  </div>
</template>
  • Programmatically using this.$router.push:
methods: {
  goToAbout() {
    this.$router.push('/about');
  }
}

Dynamic Routing

Dynamic routing allows you to define routes with dynamic segments that can match multiple paths.

Example:

const routes = [
  { path: '/user/:id', component: User }
];

In the User component, you can access the dynamic segment using this.$route.params.id.

Example Component:

<template>
  <div>User ID: {{ $route.params.id }}</div>
</template>

<script>
export default {
  name: 'User'
};
</script>

Nested Routes

Nested routes allow you to create routes within routes, which is useful for creating complex layouts.

Example:

const routes = [
  {
    path: '/user/:id',
    component: User,
    children: [
      {
        path: 'profile',
        component: UserProfile
      },
      {
        path: 'posts',
        component: UserPosts
      }
    ]
  }
];

Navigation Guards

Navigation guards are used to control access to certain routes. They can be defined globally, per-route, or in-component.

Global Guard Example:

router.beforeEach((to, from, next) => {
  if (to.path === '/protected' && !isAuthenticated()) {
    next('/login');
  } else {
    next();
  }
});

Summary

In this section, you learned:

  • What Vue Router is and its purpose.
  • How to install and set up Vue Router in a Vue.js project.
  • How to define and navigate between routes.
  • The basics of dynamic routing and nested routes.
  • How to use navigation guards to control access to routes.

Vue Router is a powerful tool that enhances the capabilities of Vue.js applications by providing seamless navigation and routing. Understanding these basics will help you build more complex and user-friendly single-page applications.
