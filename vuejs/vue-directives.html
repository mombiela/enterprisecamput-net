Vue.js Directives
Data Binding ►

Vue.js directives are special tokens in the markup that tell the library to do something to a DOM element. They are prefixed with v- to indicate that they are special attributes provided by Vue.js.

Key Directives

v-bind

  • Purpose: Dynamically bind one or more attributes, or a component prop, to an expression.
  • Syntax: v-bind:attribute="expression"
<!-- Binding an attribute -->
<img v-bind:src="imageSrc" alt="Dynamic Image">

<!-- Shorthand -->
<img :src="imageSrc" alt="Dynamic Image">

v-model

  • Purpose: Create a two-way binding on an input, textarea, or select element.
  • Syntax: v-model="data"
<!-- Two-way binding with an input element -->
<input v-model="message" placeholder="Enter a message">

<p>Message is: {{ message }}</p>

v-if, v-else-if, v-else

  • Purpose: Conditionally render elements based on the truthiness of an expression.
  • Syntax: v-if="condition", v-else-if="condition", v-else
<!-- Conditional rendering -->
<p v-if="seen">Now you see me</p>
<p v-else>Now you don't</p>

v-show

  • Purpose: Toggle the visibility of an element based on the truthiness of an expression.
  • Syntax: v-show="condition"
<!-- Toggle visibility -->
<p v-show="isVisible">I am visible</p>

v-for

  • Purpose: Render a list of items by iterating over an array or object.
  • Syntax: v-for="item in items"
<!-- Iterating over an array -->
<ul>
  <li v-for="item in items" :key="item.id">{{ item.name }}</li>
</ul>

v-on

  • Purpose: Attach event listeners to elements.
  • Syntax: v-on:event="method"
<!-- Event handling -->
<button v-on:click="doSomething">Click me</button>

<!-- Shorthand -->
<button @click="doSomething">Click me</button>

v-pre

  • Purpose: Skip compilation for this element and all its children.
  • Syntax: v-pre
<!-- Skipping compilation -->
<span v-pre>{{ this will not be compiled }}</span>

v-cloak

  • Purpose: Keep the element and its children hidden until Vue's compilation is done.
  • Syntax: v-cloak
<!-- Preventing flash of uncompiled content -->
<style>
[v-cloak] { display: none; }
</style>

<div v-cloak>
  {{ message }}
</div>

v-once

  • Purpose: Render the element and component once and skip future updates.
  • Syntax: v-once
<!-- One-time rendering -->
<span v-once>This will never change: {{ message }}</span>

Custom Directives

Vue.js also allows you to create your own custom directives. Custom directives can be useful for encapsulating reusable DOM manipulations.

Registering a Custom Directive

Vue.directive('focus', {
  // When the bound element is inserted into the DOM...
  inserted: function (el) {
    // Focus the element
    el.focus()
  }
})

Using a Custom Directive

<!-- Applying the custom directive -->
<input v-focus>

Summary

Vue.js directives are powerful tools that allow you to manipulate the DOM in a declarative way. They cover a wide range of functionalities, from binding attributes and event listeners to conditional rendering and list rendering. Understanding and effectively using these directives is crucial for building dynamic and responsive applications with Vue.js. Custom directives further extend Vue's capabilities, allowing you to encapsulate and reuse DOM manipulations across your application.
