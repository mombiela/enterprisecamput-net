Introduction to Vuex

Vuex is a state management pattern + library for Vue.js applications. It serves as a centralized store for all the components in an application, with rules ensuring that the state can only be mutated in a predictable fashion. Vuex is often used in medium to large-scale applications to manage state more efficiently.

Setting Up Vuex

Before using Vuex in your components, you need to set it up in your Vue.js project.

  1. Install Vuex:

    npm install vuex

  2. Create a Vuex Store:

    // store.js
import Vue from 'vue';
import Vuex from 'vuex';

Vue.use(Vuex);

export default new Vuex.Store({
    state: {
        count: 0
    },
    mutations: {
        increment(state) {
            state.count++;
        }
    },
    actions: {
        increment({ commit }) {
            commit('increment');
        }
    },
    getters: {
        count: state => state.count
    }
});

  3. Integrate the Store with Vue Instance:

    // main.js
import Vue from 'vue';
import App from './App.vue';
import store from './store';

new Vue({
    store,
    render: h => h(App)
}).$mount('#app');

Accessing State in Components

To access the state in your Vue components, you can use the mapState helper from Vuex.

  1. Using mapState:

    // MyComponent.vue
<template>
    <div>
        <p>Count: {{ count }}</p>
    </div>
</template>

<script>
import { mapState } from 'vuex';

export default {
    computed: {
        ...mapState(['count'])
    }
};
</script>

  2. Direct Access:

    // MyComponent.vue
<template>
    <div>
        <p>Count: {{ $store.state.count }}</p>
    </div>
</template>

<script>
export default {
    computed: {
        count() {
            return this.$store.state.count;
        }
    }
};
</script>

Committing Mutations

To change the state, you need to commit mutations. This ensures that state changes are trackable and predictable.

  1. Using mapMutations:

    // MyComponent.vue
<template>
    <div>
        <p>Count: {{ count }}</p>
        <button @click="increment">Increment</button>
    </div>
</template>

<script>
import { mapState, mapMutations } from 'vuex';

export default {
    computed: {
        ...mapState(['count'])
    },
    methods: {
        ...mapMutations(['increment'])
    }
};
</script>

  2. Direct Commit:

    // MyComponent.vue
<template>
    <div>
        <p>Count: {{ count }}</p>
        <button @click="increment">Increment</button>
    </div>
</template>

<script>
export default {
    computed: {
        count() {
            return this.$store.state.count;
        }
    },
    methods: {
        increment() {
            this.$store.commit('increment');
        }
    }
};
</script>

Dispatching Actions

Actions are similar to mutations, but instead of mutating the state directly, actions commit mutations. Actions can contain asynchronous operations.

  1. Using mapActions:

    // MyComponent.vue
<template>
    <div>
        <p>Count: {{ count }}</p>
        <button @click="increment">Increment</button>
    </div>
</template>

<script>
import { mapState, mapActions } from 'vuex';

export default {
    computed: {
        ...mapState(['count'])
    },
    methods: {
        ...mapActions(['increment'])
    }
};
</script>

  2. Direct Dispatch:

    // MyComponent.vue
<template>
    <div>
        <p>Count: {{ count }}</p>
        <button @click="increment">Increment</button>
    </div>
</template>

<script>
export default {
    computed: {
        count() {
            return this.$store.state.count;
        }
    },
    methods: {
        increment() {
            this.$store.dispatch('increment');
        }
    }
};
</script>

Using Getters

Getters are used to compute derived state based on store state. They are similar to computed properties for stores.

  1. Using mapGetters:

    // MyComponent.vue
<template>
    <div>
        <p>Count: {{ count }}</p>
    </div>
</template>

<script>
import { mapGetters } from 'vuex';

export default {
    computed: {
        ...mapGetters(['count'])
    }
};
</script>

  2. Direct Access:

    // MyComponent.vue
<template>
    <div>
        <p>Count: {{ count }}</p>
    </div>
</template>

<script>
export default {
    computed: {
        count() {
            return this.$store.getters.count;
        }
    }
};
</script>

Conclusion

Using Vuex in your Vue.js components allows you to manage state in a more structured and predictable way. By leveraging Vuex's state, mutations, actions, and getters, you can create applications that are easier to debug and maintain. Remember to use the provided helpers (mapState, mapMutations, mapActions, and mapGetters) to keep your components clean and concise.
