Introduction

Text search algorithms are fundamental in the field of computer science and have applications in various areas such as information retrieval, bioinformatics, and data compression. This topic will cover the most important and efficient algorithms for searching patterns in texts.

Classic Text Search Algorithms

Naive Search

The naive search algorithm is the simplest and most straightforward, but also the least efficient. It is based on comparing the pattern with all possible positions in the text.

  • Advantages:

    • Easy to implement.
    • Works well for small texts and patterns.

  • Disadvantages:

    • Inefficient for long texts.
    • Time complexity: O((n-m+1) * m), where n is the length of the text and m is the length of the pattern.
def naive_search(text, pattern):
    n = len(text)
    m = len(pattern)
    for i in range(n - m + 1):
        j = 0
        while j < m and text[i + j] == pattern[j]:
            j += 1
        if j == m:
            print(f"Pattern found at index {i}")

text = "ABABDABACDABABCABAB"
pattern = "ABABCABAB"
naive_search(text, pattern)

Knuth-Morris-Pratt (KMP) Algorithm

The KMP algorithm improves efficiency by avoiding redundant comparisons through the use of a prefix array.

  • Advantages:

    • More efficient than naive search.
    • Time complexity: O(n + m).

  • Disadvantages:

    • More complex to implement.
def kmp_search(text, pattern):
    def compute_lps(pattern):
        lps = [0] * len(pattern)
        length = 0
        i = 1
        while i < len(pattern):
            if pattern[i] == pattern[length]:
                length += 1
                lps[i] = length
                i += 1
            else:
                if length != 0:
                    length = lps[length - 1]
                else:
                    lps[i] = 0
                    i += 1
        return lps

    n = len(text)
    m = len(pattern)
    lps = compute_lps(pattern)
    i = 0
    j = 0
    while i < n:
        if pattern[j] == text[i]:
            i += 1
            j += 1
        if j == m:
            print(f"Pattern found at index {i - j}")
            j = lps[j - 1]
        elif i < n and pattern[j] != text[i]:
            if j != 0:
                j = lps[j - 1]
            else:
                i += 1

text = "ABABDABACDABABCABAB"
pattern = "ABABCABAB"
kmp_search(text, pattern)

Boyer-Moore Algorithm

The Boyer-Moore algorithm is one of the most efficient for pattern searching in text, using two heuristics: the bad character heuristic and the good suffix heuristic.

  • Advantages:

    • Very efficient in practice.
    • Average time complexity: O(n/m).

  • Disadvantages:

    • Complex to implement.
def boyer_moore_search(text, pattern):
    def bad_char_heuristic(pattern):
        bad_char = [-1] * 256
        for i in range(len(pattern)):
            bad_char[ord(pattern[i])] = i
        return bad_char

    def good_suffix_heuristic(pattern):
        m = len(pattern)
        good_suffix = [0] * (m + 1)
        border_pos = [0] * (m + 1)
        i = m
        j = m + 1
        border_pos[i] = j
        while i > 0:
            while j <= m and pattern[i - 1] != pattern[j - 1]:
                if good_suffix[j] == 0:
                    good_suffix[j] = j - i
                j = border_pos[j]
            i -= 1
            j -= 1
            border_pos[i] = j
        j = border_pos[0]
        for i in range(m + 1):
            if good_suffix[i] == 0:
                good_suffix[i] = j
            if i == j:
                j = border_pos[j]
        return good_suffix

    bad_char = bad_char_heuristic(pattern)
    good_suffix = good_suffix_heuristic(pattern)
    n = len(text)
    m = len(pattern)
    s = 0
    while s <= n - m:
        j = m - 1
        while j >= 0 and pattern[j] == text[s + j]:
            j -= 1
        if j < 0:
            print(f"Pattern found at index {s}")
            s += good_suffix[0]
        else:
            s += max(good_suffix[j + 1], j - bad_char[ord(text[s + j])])

text = "ABABDABACDABABCABAB"
pattern = "ABABCABAB"
boyer_moore_search(text, pattern)

Algorithm Comparison

| Algorithm | Time Complexity | Advantages | Disadvantages | |-------------------|-----------------------|-----------------------------------|----------------------------------| | Naive Search | O((n-m+1) * m) | Easy to implement | Inefficient for long texts | | KMP | O(n + m) | Avoids redundant comparisons | More complex to implement | | Boyer-Moore | O(n/m) (average) | Very efficient in practice | Complex to implement |

Conclusion

Text search algorithms are essential tools in computer science. The choice of the appropriate algorithm depends on the context and specific needs of the application. Naive search is simple but inefficient, while KMP and Boyer-Moore offer significant improvements in efficiency at the cost of greater implementation complexity. With a solid understanding of these algorithms, professionals can effectively tackle a wide range of text search problems.
