Introduction

Synchronization and communication between processes are fundamental aspects of operating systems. These concepts allow multiple processes to work in a coordinated manner, avoiding conflicts and ensuring data integrity. In this section, we will explore the mechanisms and techniques used to achieve effective synchronization and communication between processes.

Process Synchronization

Process synchronization is crucial to avoid race conditions and ensure that processes share resources safely.

Key Concepts

  • Race condition: Occurs when two or more processes access and manipulate shared data simultaneously, which can lead to inconsistent results.
  • Critical section: Part of the code where the process accesses shared resources.
  • Mutex (Mutual Exclusion): Mechanism that ensures only one process can access the critical section at a time.
  • Semaphores: Variables used to control access to shared resources.

Mutex Example in C

#include <pthread.h>
#include <stdio.h>

pthread_mutex_t lock;

void *thread_function(void *arg) {
    pthread_mutex_lock(&lock);
    // Critical section
    printf("Thread %d is in the critical section\n", *(int *)arg);
    pthread_mutex_unlock(&lock);
    return NULL;
}

int main() {
    pthread_t threads[2];
    int thread_ids[2] = {1, 2};

    pthread_mutex_init(&lock, NULL);

    for (int i = 0; i < 2; i++) {
        pthread_create(&threads[i], NULL, thread_function, &thread_ids[i]);
    }

    for (int i = 0; i < 2; i++) {
        pthread_join(threads[i], NULL);
    }

    pthread_mutex_destroy(&lock);
    return 0;
}

Semaphore Example in C

#include <pthread.h>
#include <semaphore.h>
#include <stdio.h>

sem_t sem;

void *thread_function(void *arg) {
    sem_wait(&sem);
    // Critical section
    printf("Thread %d is in the critical section\n", *(int *)arg);
    sem_post(&sem);
    return NULL;
}

int main() {
    pthread_t threads[2];
    int thread_ids[2] = {1, 2};

    sem_init(&sem, 0, 1);

    for (int i = 0; i < 2; i++) {
        pthread_create(&threads[i], NULL, thread_function, &thread_ids[i]);
    }

    for (int i = 0; i < 2; i++) {
        pthread_join(threads[i], NULL);
    }

    sem_destroy(&sem);
    return 0;
}

Inter-Process Communication (IPC)

Inter-process communication (IPC) allows processes to exchange data and signals. There are several IPC mechanisms, each with its own advantages and disadvantages.

IPC Mechanisms

  • Pipes: Allow unidirectional communication between related processes.
  • FIFOs (Named Pipes): Similar to pipes, but allow bidirectional communication and between unrelated processes.
  • Sockets: Allow communication between processes on different machines over a network.
  • Shared memory: Allows multiple processes to access a common memory region.
  • Messages: Allow the exchange of messages between processes.

Pipe Example in C

#include <stdio.h>
#include <unistd.h>

int main() {
    int fd[2];
    pipe(fd);
    pid_t pid = fork();

    if (pid == 0) {
        // Child process
        close(fd[0]);
        char message[] = "Hello from the child process";
        write(fd[1], message, sizeof(message));
        close(fd[1]);
    } else {
        // Parent process
        close(fd[1]);
        char buffer[100];
        read(fd[0], buffer, sizeof(buffer));
        printf("Message received: %s\n", buffer);
        close(fd[0]);
    }

    return 0;
}

Shared Memory Example in C

#include <stdio.h>
#include <stdlib.h>
#include <sys/ipc.h>
#include <sys/shm.h>
#include <sys/types.h>

int main() {
    key_t key = ftok("shmfile", 65);
    int shmid = shmget(key, 1024, 0666 | IPC_CREAT);
    char *str = (char *)shmat(shmid, (void *)0, 0);

    if (fork() == 0) {
        // Child process
        printf("Writing to shared memory\n");
        sprintf(str, "Hello from the child process");
        shmdt(str);
    } else {
        // Parent process
        wait(NULL);
        printf("Reading from shared memory: %s\n", str);
        shmdt(str);
        shmctl(shmid, IPC_RMID, NULL);
    }

    return 0;
}

Conclusion

Synchronization and communication between processes are essential components in modern operating systems. Synchronization ensures that processes do not interfere with each other when accessing shared resources, while inter-process communication allows the exchange of data and signals. Understanding these concepts and their practical implementations is crucial for developing robust and efficient applications in multitasking environments.
