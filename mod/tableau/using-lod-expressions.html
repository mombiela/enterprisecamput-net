Using LOD Expressions

Level of Detail (LOD) Expressions in Tableau allow you to control the granularity of your data. They enable you to perform complex calculations and aggregations at different levels of detail, independent of the view's level of detail. This topic will guide you from the basics to advanced usage of LOD Expressions in Tableau.

Introduction to LOD Expressions

  • Definition: LOD Expressions are a powerful feature in Tableau that allow you to compute values at the data source level and the visualization level.
  • Types of LOD Expressions:
    • FIXED: Computes values using the specified dimensions, ignoring the view's level of detail.
    • INCLUDE: Computes values using the specified dimensions in addition to whatever dimensions are in the view.
    • EXCLUDE: Computes values using the specified dimensions, excluding certain dimensions from the view.

Basic LOD Expressions

FIXED LOD Expressions

  • Syntax: { FIXED [Dimension1], [Dimension2], ... : Aggregate_Expression }
  • Example: Calculate the total sales per region, regardless of the view's level of detail.
{ FIXED [Region] : SUM([Sales]) }
  • Explanation: This expression calculates the sum of sales for each region, ignoring any other dimensions in the view.

INCLUDE LOD Expressions

  • Syntax: { INCLUDE [Dimension1], [Dimension2], ... : Aggregate_Expression }
  • Example: Calculate the average sales per customer within each region.
{ INCLUDE [Customer Name] : AVG([Sales]) }
  • Explanation: This expression calculates the average sales per customer, but it includes the customer dimension in addition to the dimensions already in the view.

EXCLUDE LOD Expressions

  • Syntax: { EXCLUDE [Dimension1], [Dimension2], ... : Aggregate_Expression }
  • Example: Calculate the total sales excluding the product category.
{ EXCLUDE [Category] : SUM([Sales]) }
  • Explanation: This expression calculates the sum of sales while excluding the product category dimension from the calculation.

Intermediate LOD Expressions

Combining LOD Expressions

  • Example: Calculate the difference between total sales per region and average sales per region.
{ FIXED [Region] : SUM([Sales]) } - { INCLUDE [Region] : AVG([Sales]) }
  • Explanation: This expression first calculates the total sales per region and then subtracts the average sales per region, combining both FIXED and INCLUDE LOD expressions.

Nested LOD Expressions

  • Example: Calculate the maximum sales per customer within each region.
{ FIXED [Region] : MAX({ INCLUDE [Customer Name] : SUM([Sales]) }) }
  • Explanation: This nested LOD expression first calculates the sum of sales per customer and then finds the maximum sales per customer within each region.

Advanced LOD Expressions

Using LOD Expressions with Filters

  • Example: Calculate the total sales per region, excluding certain categories.
{ FIXED [Region] : SUM(IF [Category] != 'Furniture' THEN [Sales] END) }
  • Explanation: This expression calculates the sum of sales per region, excluding sales from the 'Furniture' category.

Dynamic LOD Expressions

  • Example: Calculate the total sales per region, dynamically adjusting based on user-selected dimensions.
{ FIXED [Region], [User Selected Dimension] : SUM([Sales]) }
  • Explanation: This expression allows users to dynamically select dimensions that will be included in the LOD calculation.

Conclusion

LOD Expressions in Tableau are a versatile tool that can help you perform complex calculations and aggregations at different levels of detail. By mastering FIXED, INCLUDE, and EXCLUDE LOD expressions, you can gain deeper insights into your data and create more sophisticated visualizations. Practice using these expressions with various datasets to understand their behavior and applications fully.
