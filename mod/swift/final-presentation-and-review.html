Key Concepts Review

Let's revisit some of the most important topics covered in this course:

  • Variables and Constants
  • Data Types
  • Control Flow
  • Functions
  • Closures
  • Object-Oriented Programming (OOP)
  • Protocols and Extensions
  • Error Handling

Variables and Constants

Variables and constants are the building blocks of any Swift program.

var variableName = "Hello, Swift!"
let constantName = "Hello, World!"
  • var is used to declare a variable that can be changed.
  • let is used to declare a constant that cannot be changed.

Data Types

Swift has various data types including Int, Double, String, and Bool.

let age: Int = 25
let pi: Double = 3.14159
let name: String = "Alice"
let isSwiftFun: Bool = true

Control Flow

Control flow includes conditional statements and loops.

if isSwiftFun {
    print("Swift is fun!")
} else {
    print("Swift is not fun.")
}

for i in 1...5 {
    print(i)
}

Functions

Functions are reusable blocks of code.

func greet(name: String) -> String {
    return "Hello, \(name)!"
}

let greeting = greet(name: "Alice")
print(greeting)

Closures

Closures are self-contained blocks of functionality that can be passed around and used in your code.

let closure = { (name: String) -> String in
    return "Hello, \(name)!"
}

print(closure("Bob"))

Object-Oriented Programming (OOP)

OOP involves creating classes and objects.

class Person {
    var name: String
    var age: Int
    
    init(name: String, age: Int) {
        self.name = name
        self.age = age
    }
    
    func greet() -> String {
        return "Hello, my name is \(name)."
    }
}

let person = Person(name: "Alice", age: 25)
print(person.greet())

Protocols and Extensions

Protocols define a blueprint of methods, properties, and other requirements.

protocol Greetable {
    func greet() -> String
}

extension Person: Greetable {}

Error Handling

Swift provides robust error handling using do, try, and catch.

enum CustomError: Error {
    case invalidInput
}

func checkInput(input: Int) throws {
    if input < 0 {
        throw CustomError.invalidInput
    }
}

do {
    try checkInput(input: -1)

}

Concurrency

Concurrency in Swift is managed using Grand Central Dispatch (GCD) and Operation Queues.

DispatchQueue.global().async {
    // Background task
    DispatchQueue.main.async {
        // Update UI on the main thread
    }
}

Example Project: To-Do List App

Here's a simple example of what your final project might look like:

import UIKit

class ToDoItem {
    var title: String
    var isCompleted: Bool
    
    init(title: String, isCompleted: Bool = false) {
        self.title = title
        self.isCompleted = isCompleted
    }
}

class ToDoList {
    var items: [ToDoItem] = []
    
    func addItem(title: String) {
        let newItem = ToDoItem(title: title)
        items.append(newItem)
    }
    
    func completeItem(at index: Int) {
        items[index].isCompleted = true
    }
}

let toDoList = ToDoList()
toDoList.addItem(title: "Finish Swift course")
toDoList.completeItem(at: 0)

for item in toDoList.items {
    print("\(item.title) - Completed: \(item.isCompleted)")
}
