Introduction to GHCi

GHCi (Glasgow Haskell Compiler interactive) is the interactive environment for Haskell. It allows you to interactively load and run Haskell code, making it an invaluable tool for learning, debugging, and experimenting with Haskell programs.

  • REPL: Read-Eval-Print Loop, an interactive programming environment.
  • GHCi: The REPL for Haskell provided by the Glasgow Haskell Compiler.

Starting GHCi

To start GHCi, open your terminal and type:

ghci

You should see a prompt like this:

GHCi, version 8.10.4: https://www.haskell.org/ghc/  :? for help
Prelude>

Basic Commands

Here are some basic commands you can use in GHCi:

  • :load or :l: Load a Haskell file.
  • :reload or :r: Reload the current file.
  • :type or :t: Show the type of an expression.
  • :info or :i: Show information about a name.
  • :quit or :q: Exit GHCi.

Example Usage

Prelude> :load MyModule.hs
[1 of 1] Compiling MyModule        ( MyModule.hs, interpreted )
Ok, one module loaded.

Prelude> :type 5
5 :: Num p => p

Prelude> :info Maybe
data Maybe a = Nothing | Just a
    -- Defined in ‘GHC.Base’
instance Eq a => Eq (Maybe a) -- Defined in ‘GHC.Base’
instance Monad Maybe -- Defined in ‘GHC.Base’
...

Evaluating Expressions

You can evaluate Haskell expressions directly in GHCi:

Prelude> 2 + 3
5

Prelude> map (+1) [1, 2, 3]
[2, 3, 4]

Defining Functions

You can define functions directly in GHCi using the let keyword:

Prelude> let square x = x * x
Prelude> square 5
25

Loading and Reloading Modules

When working with larger projects, you will often load and reload modules:

Prelude> :load MyModule.hs
Prelude> :reload

Inspecting Types and Information

Understanding types and available functions is crucial in Haskell:

Prelude> :type map
map :: (a -> b) -> [a] -> [b]

Prelude> :info map
map :: (a -> b) -> [a] -> [b] 	-- Defined in ‘GHC.Base’

Using GHCi for Debugging

GHCi can be a powerful debugging tool. You can inspect intermediate values and test small pieces of code interactively.

Example Debugging Session

Suppose you have a function that isn't working as expected:

-- MyModule.hs
module MyModule where

factorial :: Integer -> Integer
factorial 0 = 1
factorial n = n * factorial (n - 1)

You can load this module and test the function:

Prelude> :load MyModule.hs
Prelude MyModule> factorial 5
120

If the function doesn't work as expected, you can inspect intermediate values:

Prelude MyModule> let factorial n = if n == 0 then 1 else n * factorial (n - 1)
Prelude MyModule> factorial 5
120

Conclusion

GHCi is an essential tool for Haskell programmers, providing an interactive environment to test, debug, and explore Haskell code. By mastering GHCi commands and features, you can significantly enhance your productivity and understanding of Haskell.

  • Practice: Regularly use GHCi to test small code snippets and debug your programs.
  • Explore: Use commands like :type and :info to learn more about Haskell's types and functions.
  • Experiment: Don't hesitate to experiment with new ideas in GHCi before integrating them into your main codebase.

By incorporating GHCi into your workflow, you'll find learning and working with Haskell to be a more interactive and enjoyable experience.
