Introduction to Applicative Functors

Applicative Functors are a powerful feature in Haskell that allow for function application lifted over a computational context. They are more powerful than Functors but less powerful than Monads. Understanding Applicative Functors is crucial for writing concise and expressive Haskell code.

Key Concepts

  • Functor: A type class that allows for a function to be applied to values wrapped in a context.
  • Applicative Functor: Extends Functor by allowing functions that are themselves wrapped in a context to be applied to values wrapped in a context.
  • Pure: A function that takes a value and returns it wrapped in a context.
  • <*> (Apply): An operator that applies a function wrapped in a context to a value wrapped in a context.

Functor Recap

Before diving into Applicative Functors, let's quickly recap Functors.

class Functor f where
    fmap :: (a -> b) -> f a -> f b

Example with Maybe:

fmap (+1) (Just 2) -- Result: Just 3
fmap (+1) Nothing  -- Result: Nothing

Applicative Functor Basics

The Applicative type class is defined as follows:

class Functor f => Applicative f where
    pure :: a -> f a
    (<*>) :: f (a -> b) -> f a -> f b

Pure

The pure function takes a value and wraps it in a context.

pure 3 :: Maybe Int -- Result: Just 3
pure 3 :: [Int]     -- Result: [3]

Apply (<*>)

The <*> operator applies a function wrapped in a context to a value wrapped in a context.

(Just (+1)) <*> (Just 2) -- Result: Just 3
Nothing <*> (Just 2)     -- Result: Nothing
(Just (+1)) <*> Nothing  -- Result: Nothing

Example with Lists

Lists are also Applicative Functors.

pure (+1) <*> [1, 2, 3] -- Result: [2, 3, 4]
[(+1), (*2)] <*> [1, 2, 3] -- Result: [2, 3, 4, 2, 4, 6]

Combining Applicative Functors

Applicative Functors can be combined to apply functions to multiple wrapped values.

pure (+) <*> Just 3 <*> Just 5 -- Result: Just 8
pure (+) <*> Just 3 <*> Nothing -- Result: Nothing

Example with Functions

Functions themselves can be Applicative Functors.

pure (+) <*> (+3) <*> (*2) $ 5 -- Result: 16
-- Explanation: ((+3) 5) + ((*2) 5) = 8 + 10 = 18

Practical Example: Validation

Applicative Functors are useful in scenarios like validation where multiple computations need to be combined.

import Control.Applicative

data Validation e a = Failure e | Success a deriving (Show)

instance Functor (Validation e) where
    fmap _ (Failure e) = Failure e
    fmap f (Success a) = Success (f a)

instance Monoid e => Applicative (Validation e) where
    pure = Success
    (Failure e1) <*> (Failure e2) = Failure (e1 <> e2)
    (Failure e) <*> _ = Failure e
    _ <*> (Failure e) = Failure e
    (Success f) <*> (Success a) = Success (f a)

validateName :: String -> Validation [String] String
validateName name
    | null name = Failure ["Name cannot be empty"]
    | otherwise = Success name

validateAge :: Int -> Validation [String] Int
validateAge age
    | age <= 0 = Failure ["Age must be positive"]
    | otherwise = Success age

validatePerson :: String -> Int -> Validation [String] (String, Int)
validatePerson name age = (,) <$> validateName name <*> validateAge age

-- Example usage:
-- validatePerson "" 0
-- Result: Failure ["Name cannot be empty","Age must be positive"]
-- validatePerson "Alice" 30
-- Result: Success ("Alice", 30)

Conclusion

Applicative Functors provide a middle ground between Functors and Monads, allowing for more flexible and expressive code. They are particularly useful for scenarios where multiple independent computations need to be combined, such as validation. Understanding Applicative Functors will enhance your ability to write concise and powerful Haskell code.
