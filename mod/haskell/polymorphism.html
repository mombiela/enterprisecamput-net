Polymorphism in Haskell
Recursive Data Structures ►

Polymorphism is a core concept in Haskell that allows functions to operate on different types. It enhances code reusability and flexibility. This topic will guide you through understanding polymorphism in Haskell, from basic to advanced levels.

Introduction to Polymorphism

Polymorphism in Haskell can be broadly categorized into two types:

  • Parametric Polymorphism: Functions or data types that can operate on any type.
  • Ad-hoc Polymorphism: Functions that can operate on different types but behave differently based on the type.

Parametric Polymorphism

Parametric polymorphism allows you to write generic functions that can handle values uniformly without depending on their type.

Example: A Generic Identity Function

identity :: a -> a
identity x = x
  • identity is a function that takes a value of any type a and returns a value of the same type.
  • The type variable a can represent any type, making identity a polymorphic function.

Example: A Generic Pair Function

makePair :: a -> b -> (a, b)
makePair x y = (x, y)
  • makePair takes two values of potentially different types a and b and returns a tuple containing both values.
  • This function is polymorphic in both of its arguments.

Ad-hoc Polymorphism

Ad-hoc polymorphism is achieved through type classes, allowing functions to operate differently based on the type of their arguments.

Example: The `Eq` Type Class

The Eq type class is used for types that support equality testing.

class Eq a where
    (==) :: a -> a -> Bool
    (/=) :: a -> a -> Bool
  • Types that are instances of Eq must implement the (==) and (/=) functions.

Example: Making a Type an Instance of `Eq`

data Color = Red | Green | Blue

instance Eq Color where
    Red == Red = True
    Green == Green = True
    Blue == Blue = True
    _ == _ = False
  • Here, Color is made an instance of Eq, defining how equality is determined for Color values.

Example: The `Show` Type Class

The Show type class is used for types whose values can be converted to strings.

class Show a where
    show :: a -> String
  • Types that are instances of Show must implement the show function.

Example: Making a Type an Instance of `Show`

data Color = Red | Green | Blue

instance Show Color where
    show Red = "Red"
    show Green = "Green"
    show Blue = "Blue"
  • Here, Color is made an instance of Show, defining how Color values are converted to strings.

Advanced Polymorphism

Higher-Kinded Types

Higher-kinded types allow you to abstract over type constructors. This is a more advanced form of polymorphism.

Example: Functor Type Class

The Functor type class is for types that can be mapped over.

class Functor f where
    fmap :: (a -> b) -> f a -> f b
  • f is a type constructor that takes one type argument.
  • fmap applies a function to every element inside a functor.

Example: Making a Type an Instance of `Functor`

instance Functor Maybe where
    fmap _ Nothing = Nothing
    fmap f (Just x) = Just (f x)
  • Here, Maybe is made an instance of Functor, defining how fmap works for Maybe.

Conclusion

Polymorphism in Haskell allows you to write flexible and reusable code. By understanding parametric polymorphism and ad-hoc polymorphism, you can create functions and types that work across a wide range of scenarios. As you advance, higher-kinded types and type classes like Functor will enable you to abstract even more complex patterns, making your Haskell code both powerful and elegant.
