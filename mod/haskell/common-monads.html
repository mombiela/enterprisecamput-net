Introduction to Monads

Monads are a powerful and flexible way to handle computations in Haskell. They provide a way to structure programs generically. This section will cover three common monads: Maybe, Either, and List.

Maybe Monad

The Maybe monad is used to represent computations that might fail. It encapsulates an optional value.

Definition

data Maybe a = Nothing | Just a

Key Concepts

  • Nothing: Represents a failure or absence of value.
  • Just a: Represents a successful computation with a value a.

Example Usage

safeDivide :: Double -> Double -> Maybe Double
safeDivide _ 0 = Nothing
safeDivide x y = Just (x / y)

Monad Instance

instance Monad Maybe where
    return = Just
    Nothing >>= _ = Nothing
    Just x >>= f = f x

Example with `do` Notation

maybeExample :: Maybe Double
maybeExample = do
    x <- Just 10
    y <- Just 2
    safeDivide x y

Either Monad

The Either monad is used for computations that can fail with an error message. It encapsulates a value that can be one of two types.

Definition

data Either a b = Left a | Right b

Key Concepts

  • Left a: Represents an error with a value a.
  • Right b: Represents a successful computation with a value b.

Example Usage

safeDivide :: Double -> Double -> Either String Double
safeDivide _ 0 = Left "Division by zero"
safeDivide x y = Right (x / y)

Monad Instance

instance Monad (Either e) where
    return = Right
    Left l >>= _ = Left l
    Right r >>= f = f r

Example with `do` Notation

eitherExample :: Either String Double
eitherExample = do
    x <- Right 10
    y <- Right 2
    safeDivide x y

List Monad

The List monad is used for computations that can return multiple results. It encapsulates a list of values.

Definition

data [] a = [] | a : [a]

Key Concepts

  • []: Represents an empty list.
  • a : [a]: Represents a list with head a and tail [a].

Example Usage

pairs :: [a] -> [b] -> [(a, b)]
pairs xs ys = [(x, y) | x <- xs, y <- ys]

Monad Instance

instance Monad [] where
    return x = [x]
    xs >>= f = concat (map f xs)

Example with `do` Notation

listExample :: [(Int, Char)]
listExample = do
    x <- [1, 2, 3]
    y <- ['a', 'b']
    return (x, y)

Comparison of Monads

| Monad | Purpose | Success Type | Failure Type | |--------|--------------------------------|--------------|--------------------| | Maybe | Optional value | Just a | Nothing | | Either | Computation with error message | Right b | Left a | | List | Multiple results | [a] | [] (empty list) |

Conclusion

Monads like Maybe, Either, and List provide powerful abstractions for handling different kinds of computations in Haskell. Understanding these monads and their usage can greatly enhance your ability to write robust and flexible Haskell programs. Practice using these monads in various scenarios to get comfortable with their patterns and idioms.
