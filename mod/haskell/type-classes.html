Introduction to Type Classes

Type classes in Haskell are a powerful feature that allows you to define functions that can operate on a variety of different types. They are similar to interfaces in other languages but more flexible and powerful.

  • Type Class: A collection of types that support certain operations.
  • Instance: A specific implementation of a type class for a particular type.

Basic Example: The `Eq` Type Class

The Eq type class is used for types that support equality testing.

class Eq a where
    (==) :: a -> a -> Bool
    (/=) :: a -> a -> Bool
  • class Eq a where: Defines a type class Eq with a type variable a.
  • (==) and (/=): Functions that must be implemented for any type that is an instance of Eq.

Creating Instances

To make a type an instance of a type class, you provide implementations for the required functions.

data Color = Red | Green | Blue

instance Eq Color where
    Red == Red = True
    Green == Green = True
    Blue == Blue = True
    _ == _ = False
  • data Color = Red | Green | Blue: Defines a new data type Color.
  • instance Eq Color where: Makes Color an instance of Eq by defining how == works for Color.

Common Type Classes

Here are some of the most commonly used type classes in Haskell:

| Type Class | Description | |------------|--------------| | Eq | Equality testing | | Ord | Comparison operations | | Show | Conversion to a string | | Read | Parsing from a string | | Num | Numeric operations |

The `Ord` Type Class

The Ord type class is for types that have an ordering.

class Eq a => Ord a where
    compare :: a -> a -> Ordering
    (<) :: a -> a -> Bool
    (<=) :: a -> a -> Bool
    (>) :: a -> a -> Bool
    (>=) :: a -> a -> Bool
  • class Eq a => Ord a where: Ord is a subclass of Eq, meaning any type that is an instance of Ord must also be an instance of Eq.

Example: `Ord` Instance for `Color`

instance Ord Color where
    compare Red Red = EQ
    compare Red _ = LT
    compare _ Red = GT
    compare Green Green = EQ
    compare Green _ = LT
    compare _ Green = GT
    compare Blue Blue = EQ
  • compare: Defines how to compare two Color values.

Advanced Type Classes

Functor

The Functor type class is used for types that can be mapped over.

class Functor f where
    fmap :: (a -> b) -> f a -> f b
  • class Functor f where: Defines a type class Functor with a type variable f.
  • fmap: A function that applies a function to every element in a structure.

Example: `Functor` Instance for `Maybe`

instance Functor Maybe where
    fmap _ Nothing = Nothing
    fmap f (Just x) = Just (f x)
  • instance Functor Maybe where: Makes Maybe an instance of Functor.
  • fmap _ Nothing = Nothing: If the value is Nothing, fmap returns Nothing.
  • fmap f (Just x) = Just (f x): If the value is Just x, fmap applies f to x.

Exercises

  1. Create an instance of Eq for a custom data type Shape with constructors Circle, Square, and Triangle.
  2. Implement the Ord type class for the Shape type.
  3. Define a Functor instance for a custom data type Box that wraps a single value.

Conclusion

Type classes in Haskell provide a way to define generic interfaces that can be implemented by different types. They are a cornerstone of Haskell's type system, enabling polymorphism and code reuse. By understanding and using type classes, you can write more flexible and reusable code.
