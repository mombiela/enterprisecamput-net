Introduction

In this section, we will present an overview of the course on Technological Architecture, highlighting the objectives, content, and evaluation methodology. This course is designed to provide IT professionals with a deep understanding of the principles and practices of technological architecture, covering everything from basic concepts to advanced techniques.

Course Objectives

  • Understand the fundamentals of technological architecture: Know the key components and how they integrate to form robust and scalable systems.
  • Develop practical skills: Apply theoretical concepts through exercises and practical projects.
  • Evaluate and select technologies: Learn to evaluate different technologies and architectures to select the most suitable ones according to project requirements.
  • Improve decision-making: Develop the ability to make informed decisions about technological architecture in business environments.

Course Content

The course is structured in modules, each addressing a specific aspect of technological architecture:

  1. Introduction to Technological Architecture
    • Definition and basic concepts
    • Importance in software development
  2. Architecture Patterns
    • Monolithic
    • Microservices
    • Event-driven
  3. Technologies and Tools
    • Comparison of frameworks and platforms
    • Design and modeling tools
  4. Design and Documentation
    • Architectural design techniques
    • Effective documentation
  5. Evaluation and Continuous Improvement
    • Architecture evaluation methods
    • Continuous improvement strategies

Evaluation Methodology

Evaluation in this course will be carried out through a combination of methods to ensure a comprehensive understanding of the topics covered:

  • Theoretical Exams: Periodic assessments to measure the understanding of theoretical concepts.
  • Practical Projects: Practical work that requires the application of learned concepts in real situations.
  • Class Participation: Active participation in class discussions and activities.
  • Self-assessments and Feedback: Self-assessment tools and continuous feedback to identify areas for improvement.

Example of Practical Evaluation

Below is an example of a practical exercise that could be part of the evaluation:

Exercise: Design a Microservices Architecture

  1. Project Description: Design a microservices architecture for an e-commerce application.
  2. Requirements:
    • The application must include services for user management, product catalog, payment processing, and order management.
    • Each service must be independent and communicate through REST APIs.
  3. Tasks:
    • Create an architecture diagram showing the services and their interactions.
    • Implement a basic service in a programming language of your choice (e.g., Node.js, Python).
    • Document the design and decisions made.

Conclusion

Through a combination of theory and practice, participants will develop the necessary skills to design and evaluate effective technological architectures.
