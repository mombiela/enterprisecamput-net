Introduction

Code style and formatting are crucial aspects of writing clean, readable, and maintainable code. In Rust, adhering to a consistent style helps ensure that your code is understandable not only to you but also to other developers who may work on your codebase. This section will cover the key principles of Rust code style and formatting, including naming conventions, indentation, and the use of Rust's built-in formatting tool, rustfmt.

Naming Conventions

Naming conventions in Rust help make the code more readable and consistent. Here are the key conventions:

  • Variables and Functions: Use snake_case.

    let my_variable = 10;
fn my_function() {
    // function body
}

  • Constants and Statics: Use SCREAMING_SNAKE_CASE.

    const MAX_SIZE: u32 = 100;
static GLOBAL_COUNT: u32 = 0;

  • Structs, Enums, and Traits: Use CamelCase.

    struct MyStruct {
    field: u32,
}

enum MyEnum {
    VariantOne,
    VariantTwo,
}

trait MyTrait {
    fn do_something(&self);
}

Indentation and Spacing

Proper indentation and spacing make your code more readable. Rust uses 4 spaces per indentation level.

  • Indentation: Use 4 spaces, not tabs.

    fn main() {
    let x = 5;
    if x > 0 {
        println!("x is positive");
    }
}

  • Spacing: Use spaces around operators and after commas.

    let sum = a + b;
let array = [1, 2, 3, 4];

Line Length

Keep lines to a maximum of 100 characters. This helps ensure that code is readable on various screen sizes and in different environments.

  • Example: 
    // This line is exactly 100 characters long
let long_variable_name = "This is a very long string that should be split into multiple lines if it exceeds 100 characters.";

Comments

Comments are essential for explaining the purpose of code. Rust supports two types of comments:

  • Line Comments: Use // for single-line comments.

    // This is a single-line comment
let x = 5; // This is an inline comment

  • Block Comments: Use /* ... */ for multi-line comments.

    /*
 * This is a block comment.
 * It can span multiple lines.
 */
let y = 10;

Using `rustfmt`

Rust provides a built-in tool called rustfmt to automatically format your code according to the official Rust style guidelines. This tool helps maintain consistency and readability across your codebase.

  • Installation:

    rustup component add rustfmt

  • Usage:

    cargo fmt

Example of Well-Formatted Code

Here is an example of a well-formatted Rust program:

// This is a simple Rust program that demonstrates proper code style and formatting.

fn main() {
    let numbers = vec![1, 2, 3, 4, 5];

    for number in &numbers {
        if is_even(*number) {
            println!("{} is even", number);
        } else {
            println!("{} is odd", number);
        }
    }
}

fn is_even(num: i32) -> bool {
    num % 2 == 0
}

Conclusion

Adhering to consistent code style and formatting guidelines is essential for writing clean, readable, and maintainable Rust code. By following the naming conventions, indentation rules, and using tools like rustfmt, you can ensure that your code is easy to understand and work with. Remember, well-formatted code not only benefits you but also your collaborators and future maintainers of the codebase.
