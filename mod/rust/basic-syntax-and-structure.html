Introduction to Rust

Rust is a systems programming language that focuses on speed, memory safety, and parallelism. It is syntactically similar to C++, but it guarantees memory safety by using a borrow checker to validate references.

Key Concepts

  • Variables and Mutability
  • Data Types
  • Functions
  • Control Flow
  • Ownership and Borrowing

Variables and Mutability

In Rust, variables are immutable by default. To make a variable mutable, you need to use the mut keyword.

fn main() {
    let x = 5; // immutable variable
    println!("The value of x is: {}", x);

    let mut y = 10; // mutable variable
    println!("The value of y is: {}", y);
    y = 15;
    println!("The new value of y is: {}", y);
}

Explanation

  • let x = 5; declares an immutable variable x with the value 5.
  • let mut y = 10; declares a mutable variable y with the value 10.
  • y = 15; changes the value of y to 15.

Data Types

Rust is a statically typed language, which means that it must know the types of all variables at compile time. The main data types include:

  • Scalar Types: integers, floating-point numbers, Booleans, and characters.
  • Compound Types: tuples and arrays.

Scalar Types Example

fn main() {
    let integer: i32 = 42;
    let float: f64 = 3.14;
    let boolean: bool = true;
    let character: char = 'R';

    println!("Integer: {}", integer);
    println!("Float: {}", float);
    println!("Boolean: {}", boolean);
    println!("Character: {}", character);
}

Compound Types Example

fn main() {
    let tuple: (i32, f64, u8) = (500, 6.4, 1);
    let array: [i32; 3] = [1, 2, 3];

    println!("Tuple: ({}, {}, {})", tuple.0, tuple.1, tuple.2);
    println!("Array: [{}, {}, {}]", array[0], array[1], array[2]);
}

Functions

Functions in Rust are defined using the fn keyword. They can take parameters and return values.

fn main() {
    let result = add(5, 3);
    println!("The sum is: {}", result);
}

fn add(x: i32, y: i32) -> i32 {
    x + y
}

Explanation

  • fn main() is the entry point of the program.
  • fn add(x: i32, y: i32) -> i32 defines a function named add that takes two i32 parameters and returns an i32.

Control Flow

Rust provides common control flow constructs such as if, else, loop, while, and for.

If-Else Example

fn main() {
    let number = 7;

    if number < 5 {
        println!("The number is less than 5");
    } else {
        println!("The number is greater than or equal to 5");
    }
}

Loop Example

fn main() {
    let mut count = 0;

    loop {
        count += 1;
        if count == 5 {
            break;
        }
    }

    println!("Loop stopped at count: {}", count);
}

Ownership and Borrowing

Ownership is Rust's most unique feature, enabling memory safety without a garbage collector. Each value in Rust has a variable that’s called its owner. There can only be one owner at a time, and when the owner goes out of scope, the value will be dropped.

Ownership Example

fn main() {
    let s1 = String::from("hello");
    let s2 = s1; // s1 is moved to s2

    // println!("{}", s1); // This line would cause a compile-time error
    println!("{}", s2);
}

Borrowing Example

fn main() {
    let s1 = String::from("hello");
    let len = calculate_length(&s1); // s1 is borrowed

    println!("The length of '{}' is {}.", s1, len);
}

fn calculate_length(s: &String) -> usize {
    s.len()
}

Explanation

  • let s2 = s1; moves s1 to s2, making s1 no longer valid.
  • &s1 creates a reference to s1 that can be used without taking ownership.

Conclusion

Understanding Rust's basic syntax and structure is crucial for writing safe and efficient code. By mastering variables, data types, functions, control flow, and ownership, you lay a strong foundation for more advanced Rust programming concepts. Practice these basics thoroughly to become proficient in Rust.
