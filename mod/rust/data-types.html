Introduction

Rust is a statically typed language, which means that the type of a variable must be known at compile time. This ensures safety and performance. In this section, we will explore the various data types available in Rust, from basic scalar types to more complex compound types.

Scalar Types

Scalar types represent a single value. Rust has four primary scalar types: integers, floating-point numbers, Booleans, and characters.

Integers

  • Signed integers: i8, i16, i32, i64, i128, isize
  • Unsigned integers: u8, u16, u32, u64, u128, usize
fn main() {
    let x: i32 = -42; // Signed 32-bit integer
    let y: u32 = 42;  // Unsigned 32-bit integer
    println!("x: {}, y: {}", x, y);
}

Floating-Point Numbers

  • Types: f32, f64
fn main() {
    let x: f32 = 3.14; // 32-bit floating point
    let y: f64 = 2.718; // 64-bit floating point
    println!("x: {}, y: {}", x, y);
}

Booleans

  • Type: bool
fn main() {
    let is_rust_fun: bool = true;
    println!("Is Rust fun? {}", is_rust_fun);
}

Characters

  • Type: char
fn main() {
    let letter: char = 'R';
    let emoji: char = 'ðŸ˜Š';
    println!("letter: {}, emoji: {}", letter, emoji);
}

Compound Types

Compound types can group multiple values into one type. Rust has two primary compound types: tuples and arrays.

Tuples

Tuples can group values of different types.

fn main() {
    let tuple: (i32, f64, char) = (42, 3.14, 'R');
    let (x, y, z) = tuple; // Destructuring the tuple
    println!("x: {}, y: {}, z: {}", x, y, z);
}

Arrays

Arrays can group values of the same type.

fn main() {
    let array: [i32; 3] = [1, 2, 3];
    println!("First element: {}", array[0]);
}

Slices

Slices are a view into a contiguous sequence of elements in an array.

fn main() {
    let array = [1, 2, 3, 4, 5];
    let slice = &array[1..3]; // Slice of the array
    println!("Slice: {:?}", slice);
}

Strings

Rust has two main string types: String and &str.

String

A growable, heap-allocated data structure.

fn main() {
    let mut s = String::from("Hello");
    s.push_str(", world!");
    println!("{}", s);
}

&str

An immutable reference to a string slice.

fn main() {
    let s = "Hello, world!";
    println!("{}", s);
}

Enums

Enums allow you to define a type by enumerating its possible values.

enum Direction {
    Up,
    Down,
    Left,
    Right,
}

fn main() {
    let dir = Direction::Up;
    match dir {
        Direction::Up => println!("Going up!"),
        Direction::Down => println!("Going down!"),
        Direction::Left => println!("Going left!"),
        Direction::Right => println!("Going right!"),
    }
}

Option and Result

Rust provides the Option and Result enums for handling optional and error-prone values.

Option

Used for values that may or may not be present.

fn main() {
    let some_number: Option<i32> = Some(5);
    let absent_number: Option<i32> = None;

    match some_number {
        Some(n) => println!("Number: {}", n),
        None => println!("No number"),
    }
}

Result

Used for functions that can return an error.

fn divide(a: i32, b: i32) -> Result<i32, String> {
    if b == 0 {
        Err(String::from("Cannot divide by zero"))
    } else {
        Ok(a / b)
    }
}

fn main() {
    match divide(10, 2) {
        Ok(result) => println!("Result: {}", result),
        Err(e) => println!("Error: {}", e),
    }
}

Conclusion

Understanding Rust's data types is crucial for writing safe and efficient code. From basic scalar types to more complex compound types, each serves a unique purpose and helps in managing data effectively. Enums, Option, and Result types further enhance Rust's capability to handle various scenarios robustly. Practice using these types to become proficient in Rust programming.
