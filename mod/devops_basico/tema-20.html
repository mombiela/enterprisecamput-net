Introduction to Kubernetes

Kubernetes is an open-source container orchestration platform that automates the deployment, scaling, and operations of containerized applications. It is an essential tool in the DevOps ecosystem for efficiently managing containerized applications.

Key Concepts

  • Cluster: A set of nodes (machines) that run containerized applications managed by Kubernetes.
  • Node: An individual machine within the cluster, which can be physical or virtual.
  • Pod: The smallest and most basic deployment unit in Kubernetes, which can contain one or more containers.
  • Service: A resource that defines a policy for accessing pods, allowing communication between them and with the outside world.
  • Deployment: A resource that manages the creation and scaling of pods, ensuring that the desired state of the application is maintained.

Basic Application Deployment

To deploy an application in Kubernetes, several key steps must be followed:

1. Create a YAML Configuration File

The YAML file defines the resources that Kubernetes should create and manage. Below is a basic example of a configuration file to deploy an Nginx application.

apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.14.2
        ports:
        - containerPort: 80

2. Apply the Configuration File

Use the kubectl apply command to apply the configuration file and create the resources in the cluster.

kubectl apply -f nginx-deployment.yaml

3. Verify the Deployment

Check that the pods have been created and are running.

kubectl get pods

4. Expose the Application

To make the application accessible from outside the cluster, a Service must be created. Here is an example of a YAML file for a LoadBalancer type Service.

apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  selector:
    app: nginx
  ports:
    - protocol: TCP
      port: 80
      targetPort: 80
  type: LoadBalancer

Apply the Service configuration file:

kubectl apply -f nginx-service.yaml

5. Verify the Service

Check that the Service has been created and is assigning an external IP.

kubectl get services

Practical Exercise

  1. Create a YAML configuration file for a simple application of your choice.
  2. Deploy the application in a Kubernetes cluster using kubectl apply.
  3. Create and apply a Service to expose the application.
  4. Verify that the application is running and accessible from outside the cluster.

Conclusion

Deploying applications in Kubernetes involves creating and managing several resources such as Pods, Deployments, and Services. Through YAML configuration files and kubectl commands, IT professionals can automate and efficiently manage their containerized applications. Practicing these basic steps is fundamental to mastering deployment in Kubernetes and advancing in the DevOps journey.
