Introduction

Network configuration in Linux involves setting up and managing network interfaces, IP addresses, routes, and DNS settings. This topic will guide you from basic to advanced network configuration tasks in Linux.

Basic Network Configuration

Viewing Network Interfaces

To view the current network interfaces and their statuses, you can use the ip command.

ip a
  • Explanation: The ip a command lists all network interfaces and their details, such as IP addresses and status (up or down).

Configuring a Static IP Address

To configure a static IP address, you can use the ip command.

sudo ip addr add 192.168.1.100/24 dev eth0
sudo ip link set dev eth0 up
  • Explanation:
    • ip addr add 192.168.1.100/24 dev eth0 assigns the IP address 192.168.1.100 with a subnet mask of 255.255.255.0 to the eth0 interface.
    • ip link set dev eth0 up brings the eth0 interface up.

Configuring DNS

DNS settings are typically configured in the /etc/resolv.conf file.

sudo nano /etc/resolv.conf

Add the following lines to set the DNS servers:

nameserver 8.8.8.8
nameserver 8.8.4.4
  • Explanation: The nameserver lines specify the IP addresses of the DNS servers to be used.

Intermediate Network Configuration

Using NetworkManager

NetworkManager is a tool for managing network settings in Linux.

Checking NetworkManager Status

sudo systemctl status NetworkManager
  • Explanation: This command checks if NetworkManager is running.

Configuring a Network Interface with NetworkManager

nmcli con add type ethernet ifname eth0 con-name my-eth0 ip4 192.168.1.100/24 gw4 192.168.1.1
nmcli con up my-eth0
  • Explanation:
    • nmcli con add adds a new connection named my-eth0 for the eth0 interface with the specified IP address and gateway.
    • nmcli con up my-eth0 activates the my-eth0 connection.

Configuring Routes

Routes determine how packets are forwarded between networks.

Adding a Static Route

sudo ip route add 192.168.2.0/24 via 192.168.1.1
  • Explanation: This command adds a route to the 192.168.2.0/24 network via the gateway 192.168.1.1.

Advanced Network Configuration

Using `netplan` for Network Configuration

netplan is a utility for configuring network settings on Ubuntu and other distributions that use netplan.

Configuring netplan

Edit the configuration file in /etc/netplan/.

sudo nano /etc/netplan/01-netcfg.yaml

Add the following configuration:

network:
  version: 2
  ethernets:
    eth0:
      addresses:
        - 192.168.1.100/24
      gateway4: 192.168.1.1
      nameservers:
        addresses:
          - 8.8.8.8
          - 8.8.4.4

Apply the configuration:

sudo netplan apply
  • Explanation: This YAML configuration sets a static IP address, gateway, and DNS servers for the eth0 interface.

Configuring Firewall with `iptables`

iptables is a powerful tool for configuring network packet filtering rules.

Basic iptables Commands

sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT
sudo iptables -A INPUT -j DROP
  • Explanation:
    • iptables -A INPUT -p tcp --dport 22 -j ACCEPT allows incoming SSH connections.
    • iptables -A INPUT -j DROP drops all other incoming traffic.

Using `firewalld` for Firewall Management

firewalld is a dynamic firewall management tool with iptables as its backend.

Configuring firewalld

sudo firewall-cmd --permanent --add-service=ssh
sudo firewall-cmd --reload
  • Explanation:
    • firewall-cmd --permanent --add-service=ssh allows SSH service permanently.
    • firewall-cmd --reload reloads the firewall configuration.

Conclusion

Network configuration in Linux ranges from basic tasks like setting IP addresses and DNS servers to advanced tasks like configuring firewalls and routing. Mastery of these skills is essential for managing Linux systems effectively. By understanding and practicing these configurations, you can ensure your Linux systems are properly connected and secure.
