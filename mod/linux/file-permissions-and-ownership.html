File Permissions and Ownership
Basic Networking Commands ►

Introduction

Understanding file permissions and ownership is crucial for managing files and directories in Linux. This section will cover the basics of file permissions, how to change them, and how ownership works.

File Permissions

Key Concepts

  • Read (r): Permission to read the contents of the file.
  • Write (w): Permission to modify the contents of the file.
  • Execute (x): Permission to execute the file as a program.

Permission Types

Permissions are divided into three categories:

  1. User (u): The owner of the file.
  2. Group (g): Users who are part of the file's group.
  3. Others (o): All other users.

Viewing Permissions

To view the permissions of a file or directory, use the ls -l command:

$ ls -l filename
-rw-r--r-- 1 user group 1234 Jan 1 12:34 filename

Explanation:

  • -rw-r--r--: Permission string.
    • -: File type (e.g., - for regular file, d for directory).
    • rw-: User permissions (read and write).
    • r--: Group permissions (read only).
    • r--: Others permissions (read only).
  • 1: Number of hard links.
  • user: Owner of the file.
  • group: Group associated with the file.
  • 1234: File size in bytes.
  • Jan 1 12:34: Last modification date and time.
  • filename: Name of the file.

Changing Permissions

Using `chmod`

The chmod command is used to change file permissions.

Symbolic Mode

$ chmod u+x filename

Explanation:

  • u+x: Add execute permission for the user.

Numeric Mode

Permissions can also be set using octal numbers:

  • r = 4
  • w = 2
  • x = 1

Example:

$ chmod 755 filename

Explanation:

  • 7 (user): rwx (4+2+1)
  • 5 (group): r-x (4+1)
  • 5 (others): r-x (4+1)

File Ownership

Key Concepts

  • Owner: The user who owns the file.
  • Group: The group that owns the file.

Viewing Ownership

Ownership can be viewed using the ls -l command (as shown earlier).

Changing Ownership

The chown command is used to change the owner and group of a file.

Changing Owner

$ sudo chown newuser filename

Explanation:

  • newuser: New owner of the file.

Changing Group

$ sudo chown :newgroup filename

Explanation:

  • :newgroup: New group of the file.

Changing Both Owner and Group

$ sudo chown newuser:newgroup filename

Explanation:

  • newuser:newgroup: New owner and group of the file.

Practical Examples

Example 1: Setting Permissions

$ touch example.txt
$ ls -l example.txt
-rw-r--r-- 1 user group 0 Jan 1 12:34 example.txt

$ chmod 744 example.txt
$ ls -l example.txt
-rwxr--r-- 1 user group 0 Jan 1 12:34 example.txt

Explanation:

  • Created a new file example.txt.
  • Changed permissions to 744 (user: rwx, group: r--, others: r--).

Example 2: Changing Ownership

$ sudo chown newuser:newgroup example.txt
$ ls -l example.txt
-rwxr--r-- 1 newuser newgroup 0 Jan 1 12:34 example.txt

Explanation:

  • Changed the owner to newuser and the group to newgroup.

Summary

  • File permissions in Linux are divided into read, write, and execute.
  • Permissions are categorized for the user, group, and others.
  • Use chmod to change permissions and chown to change ownership.
  • Understanding and managing file permissions and ownership is essential for system security and proper file management.

By mastering these concepts, you will be able to effectively control access to files and directories in a Linux environment.
