Introduction to INNER JOIN

In SQL, the INNER JOIN clause is used to combine rows from two or more tables based on a related column between them. It returns only the rows that have matching values in both tables.

Key Concepts

  • Join Condition: The condition that specifies how rows from one table are matched to rows in another table.
  • Result Set: The set of rows returned by the INNER JOIN operation.
  • Primary Key: A unique identifier for a row in a table.
  • Foreign Key: A column or a set of columns in one table that refers to the primary key in another table.

Basic Syntax

The basic syntax for an INNER JOIN is as follows:

SELECT columns
FROM table1
INNER JOIN table2
ON table1.column = table2.column;

Example

Consider two tables, Employees and Departments:

Employees Table

| EmployeeID | Name | DepartmentID | |------------|----------|--------------| | 1 | Alice | 101 | | 2 | Bob | 102 | | 3 | Charlie | 103 | | 4 | David | 101 |

Departments Table

| DepartmentID | DepartmentName | |--------------|----------------| | 101 | HR | | 102 | IT | | 103 | Finance | | 104 | Marketing |

INNER JOIN Query

To get a list of employees along with their department names, you can use the following INNER JOIN query:

SELECT Employees.Name, Departments.DepartmentName
FROM Employees
INNER JOIN Departments
ON Employees.DepartmentID = Departments.DepartmentID;

Result Set

| Name | DepartmentName | |---------|----------------| | Alice | HR | | Bob | IT | | Charlie | Finance | | David | HR |

Advanced Usage

Joining Multiple Tables

You can join more than two tables using multiple INNER JOIN clauses. For example:

SELECT Employees.Name, Departments.DepartmentName, Projects.ProjectName
FROM Employees
INNER JOIN Departments ON Employees.DepartmentID = Departments.DepartmentID
INNER JOIN EmployeeProjects ON Employees.EmployeeID = EmployeeProjects.EmployeeID
INNER JOIN Projects ON EmployeeProjects.ProjectID = Projects.ProjectID;

Using Aliases

Aliases can make your SQL queries more readable:

SELECT e.Name, d.DepartmentName
FROM Employees AS e
INNER JOIN Departments AS d
ON e.DepartmentID = d.DepartmentID;

Filtering Results

You can add a WHERE clause to filter the results:

SELECT e.Name, d.DepartmentName
FROM Employees AS e
INNER JOIN Departments AS d
ON e.DepartmentID = d.DepartmentID
WHERE d.DepartmentName = 'IT';

Exercises

  1. Basic INNER JOIN: Write a query to join the Students and Courses tables to list all students along with the courses they are enrolled in.
  2. Multiple INNER JOINs: Write a query to join Orders, Customers, and Products tables to list all orders along with customer names and product details.
  3. Filtering with INNER JOIN: Write a query to join Employees and Departments tables and filter the results to show only employees in the 'Finance' department.

Conclusion

The INNER JOIN clause is a powerful tool in SQL for combining rows from multiple tables based on a related column. By understanding and using INNER JOIN, you can efficiently retrieve and analyze data from relational databases. Practice with different scenarios and datasets to master the use of INNER JOIN in your SQL queries.
