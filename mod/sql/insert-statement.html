INSERT Statement
UPDATE Statement ►

The INSERT statement in SQL is used to add new rows of data to a table. This is a fundamental operation in database management, allowing you to populate tables with the necessary data for your applications.

Key Concepts

  • Syntax: The basic structure of the INSERT statement.
  • Single Row Insertion: How to insert a single row into a table.
  • Multiple Rows Insertion: How to insert multiple rows in a single statement.
  • Inserting Data from Another Table: Using INSERT INTO ... SELECT to copy data from one table to another.
  • Handling NULL Values: How to deal with columns that can have NULL values.
  • Error Handling: Common errors and how to handle them.

Syntax

The basic syntax for the INSERT statement is as follows:

INSERT INTO table_name (column1, column2, column3, ...)
VALUES (value1, value2, value3, ...);
  • table_name: The name of the table where you want to insert the data.
  • column1, column2, column3, ...: The columns in the table where the data will be inserted.
  • value1, value2, value3, ...: The values to be inserted into the specified columns.

Single Row Insertion

To insert a single row into a table, you can use the following example:

INSERT INTO employees (first_name, last_name, email, hire_date)
VALUES ('John', 'Doe', 'john.doe@example.com', '2023-10-01');

In this example:

  • The employees table has columns first_name, last_name, email, and hire_date.
  • The VALUES clause specifies the data to be inserted into these columns.

Multiple Rows Insertion

You can insert multiple rows into a table in a single INSERT statement:

INSERT INTO employees (first_name, last_name, email, hire_date)
VALUES 
    ('Jane', 'Smith', 'jane.smith@example.com', '2023-10-02'),
    ('Alice', 'Johnson', 'alice.johnson@example.com', '2023-10-03'),
    ('Bob', 'Brown', 'bob.brown@example.com', '2023-10-04');

This example inserts three rows into the employees table in one go.

Inserting Data from Another Table

You can also insert data into a table from another table using the INSERT INTO ... SELECT statement:

INSERT INTO employees_archive (first_name, last_name, email, hire_date)
SELECT first_name, last_name, email, hire_date
FROM employees
WHERE hire_date < '2023-01-01';

In this example:

  • Data is copied from the employees table to the employees_archive table.
  • Only rows where hire_date is before January 1, 2023, are inserted.

Handling NULL Values

If a column can accept NULL values, you can insert NULL explicitly or omit the column from the INSERT statement:

INSERT INTO employees (first_name, last_name, email)
VALUES ('Charlie', 'Davis', NULL);

Or:

INSERT INTO employees (first_name, last_name)
VALUES ('Charlie', 'Davis');

In both cases, the email column will have a NULL value for the new row.

Error Handling

Common errors you might encounter when using the INSERT statement include:

  • Duplicate Key Error: Occurs when you try to insert a row with a primary key that already exists.
  • Data Type Mismatch: Happens when the value being inserted does not match the column's data type.
  • Constraint Violation: Occurs when the data being inserted violates a table constraint, such as a foreign key constraint.

Example of handling a duplicate key error:

INSERT INTO employees (id, first_name, last_name, email)
VALUES (1, 'David', 'Wilson', 'david.wilson@example.com')
ON CONFLICT (id) DO NOTHING;

In this example, if a row with id 1 already exists, the ON CONFLICT clause prevents the insertion and does nothing.

Conclusion

The INSERT statement is a powerful tool in SQL for adding new data to your tables. By understanding its syntax and various use cases, you can efficiently manage your database's data. Whether you're inserting a single row, multiple rows, or copying data from another table, mastering the INSERT statement is essential for any SQL programmer.
