Introduction

In SQL, primary keys and foreign keys are fundamental concepts used to establish and enforce relationships between tables. Understanding these keys is crucial for designing robust and efficient databases.

Primary Keys

A primary key is a unique identifier for a record in a table. It ensures that each record can be uniquely identified and retrieved.

Key Concepts

  • Uniqueness: Each value in the primary key column(s) must be unique.
  • Non-nullability: Primary key columns cannot contain NULL values.
  • Single or Composite: A primary key can consist of a single column or multiple columns (composite key).

Example

Consider a table Students where each student has a unique ID:

CREATE TABLE Students (
    StudentID INT PRIMARY KEY,
    FirstName VARCHAR(50),
    LastName VARCHAR(50),
    BirthDate DATE
);

In this example:

  • StudentID is the primary key.
  • It uniquely identifies each student in the Students table.

Foreign Keys

A foreign key is a column or a set of columns in one table that uniquely identifies a row of another table. It establishes a link between the data in the two tables.

Key Concepts

  • Referential Integrity: Ensures that the value in the foreign key column(s) matches a value in the primary key column(s) of the referenced table.
  • Parent-Child Relationship: The table containing the foreign key is the child table, and the table containing the primary key is the parent table.

Example

Consider a table Enrollments that records which students are enrolled in which courses:

CREATE TABLE Enrollments (
    EnrollmentID INT PRIMARY KEY,
    StudentID INT,
    CourseID INT,
    EnrollmentDate DATE,
    FOREIGN KEY (StudentID) REFERENCES Students(StudentID)
);

In this example:

  • StudentID in the Enrollments table is a foreign key.
  • It references StudentID in the Students table, establishing a relationship between the two tables.

Combining Primary and Foreign Keys

To illustrate how primary and foreign keys work together, let's add a Courses table and modify the Enrollments table to include a foreign key reference to Courses.

Example

CREATE TABLE Courses (
    CourseID INT PRIMARY KEY,
    CourseName VARCHAR(100),
    Credits INT
);

CREATE TABLE Enrollments (
    EnrollmentID INT PRIMARY KEY,
    StudentID INT,
    CourseID INT,
    EnrollmentDate DATE,
    FOREIGN KEY (StudentID) REFERENCES Students(StudentID),
    FOREIGN KEY (CourseID) REFERENCES Courses(CourseID)
);

In this example:

  • CourseID in the Courses table is the primary key.
  • CourseID in the Enrollments table is a foreign key referencing CourseID in the Courses table.
  • This setup ensures that each enrollment record is linked to a valid student and a valid course.

Summary

  • Primary Keys: Ensure each record in a table is unique and identifiable.
  • Foreign Keys: Establish relationships between tables, ensuring referential integrity.
  • Usage: Primary keys are used to uniquely identify records, while foreign keys link records across tables.

Understanding and correctly implementing primary and foreign keys are essential for maintaining data integrity and establishing clear relationships within your database.
