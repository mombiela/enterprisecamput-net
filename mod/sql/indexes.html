Introduction to Indexes

Indexes are special lookup tables that the database search engine can use to speed up data retrieval. They are used to quickly locate data without having to search every row in a database table every time a database table is accessed.

Key Concepts

  • Index: A database object that improves the speed of data retrieval operations on a table.
  • Primary Key Index: Automatically created when a primary key is defined.
  • Unique Index: Ensures that all values in the index are unique.
  • Non-Unique Index: Allows duplicate values and is used to improve the performance of queries.
  • Clustered Index: Sorts and stores the data rows in the table based on the index key.
  • Non-Clustered Index: Contains a sorted list of references to the table data.

Creating Indexes

Creating an index involves specifying the table and the column(s) to be indexed.

Syntax

CREATE INDEX index_name
ON table_name (column1, column2, ...);

Example

CREATE INDEX idx_lastname
ON employees (lastname);

This creates an index named idx_lastname on the lastname column of the employees table.

Types of Indexes

Primary Key Index

Automatically created when a primary key is defined.

CREATE TABLE employees (
    id INT PRIMARY KEY,
    firstname VARCHAR(50),
    lastname VARCHAR(50)
);

In this example, an index is automatically created on the id column.

Unique Index

Ensures that all values in the index are unique.

CREATE UNIQUE INDEX idx_unique_email
ON employees (email);

This creates a unique index on the email column, ensuring no duplicate emails.

Non-Unique Index

Allows duplicate values and is used to improve the performance of queries.

CREATE INDEX idx_department
ON employees (department);

This creates a non-unique index on the department column.

Clustered Index

Sorts and stores the data rows in the table based on the index key. A table can have only one clustered index.

CREATE CLUSTERED INDEX idx_employee_id
ON employees (id);

This creates a clustered index on the id column.

Non-Clustered Index

Contains a sorted list of references to the table data. A table can have multiple non-clustered indexes.

CREATE NONCLUSTERED INDEX idx_lastname_firstname
ON employees (lastname, firstname);

This creates a non-clustered index on the lastname and firstname columns.

Managing Indexes

Dropping an Index

To remove an index, use the DROP INDEX statement.

DROP INDEX idx_lastname;

This removes the idx_lastname index from the table.

Rebuilding an Index

To rebuild an index, which can be useful for defragmenting, use the ALTER INDEX statement.

ALTER INDEX idx_lastname
ON employees
REBUILD;

This rebuilds the idx_lastname index on the employees table.

Performance Considerations

  • Read Performance: Indexes significantly improve read performance by allowing the database to find data quickly.
  • Write Performance: Indexes can slow down write operations (INSERT, UPDATE, DELETE) because the index must be updated whenever the data is modified.
  • Storage: Indexes consume additional disk space.

Example: Query Performance

Without Index:

SELECT * FROM employees WHERE lastname = 'Smith';

With Index:

CREATE INDEX idx_lastname ON employees (lastname);

SELECT * FROM employees WHERE lastname = 'Smith';

The second query will execute faster due to the index on the lastname column.

Conclusion

Indexes are powerful tools for optimizing database performance, especially for read-heavy operations. Understanding the different types of indexes and their appropriate use cases is crucial for efficient database management. While indexes can significantly speed up data retrieval, they come with trade-offs in terms of write performance and storage requirements. Properly managing and maintaining indexes is essential for ensuring optimal database performance.
