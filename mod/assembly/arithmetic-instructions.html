Introduction

Arithmetic instructions in assembly language are fundamental operations that allow a programmer to perform basic mathematical calculations. These instructions are essential for manipulating data and performing computations within a program. This section will cover the basic arithmetic instructions, their syntax, and how to use them effectively.

Key Concepts

  • Registers: Small storage locations within the CPU used to hold data temporarily.
  • Operands: Values or data on which the arithmetic operations are performed.
  • Flags: Special indicators that provide information about the result of an arithmetic operation (e.g., zero flag, carry flag).

Basic Arithmetic Instructions

Addition (`ADD`)

The ADD instruction adds the value of the source operand to the destination operand and stores the result in the destination operand.

Syntax:

ADD destination, source

Example:

MOV AX, 5      ; Load 5 into AX register
ADD AX, 3      ; Add 3 to AX register (AX now contains 8)

Subtraction (`SUB`)

The SUB instruction subtracts the value of the source operand from the destination operand and stores the result in the destination operand.

Syntax:

SUB destination, source

Example:

MOV AX, 10     ; Load 10 into AX register
SUB AX, 4      ; Subtract 4 from AX register (AX now contains 6)

Multiplication (`MUL`)

The MUL instruction multiplies the unsigned value of the source operand by the value in the accumulator register (AX for 16-bit, EAX for 32-bit).

Syntax:

MUL source

Example:

MOV AX, 4      ; Load 4 into AX register
MOV BX, 2      ; Load 2 into BX register
MUL BX         ; Multiply AX by BX (AX now contains 8)

Division (`DIV`)

The DIV instruction divides the unsigned value in the accumulator register by the source operand. The quotient is stored in the accumulator register, and the remainder is stored in the DX register.

Syntax:

DIV source

Example:

MOV AX, 10     ; Load 10 into AX register
MOV BX, 3      ; Load 3 into BX register
DIV BX         ; Divide AX by BX (AX now contains quotient 3, DX contains remainder 1)

Advanced Arithmetic Instructions

Increment (`INC`)

The INC instruction increases the value of the operand by 1.

Syntax:

INC operand

Example:

MOV AX, 5      ; Load 5 into AX register
INC AX         ; Increment AX by 1 (AX now contains 6)

Decrement (`DEC`)

The DEC instruction decreases the value of the operand by 1.

Syntax:

DEC operand

Example:

MOV AX, 5      ; Load 5 into AX register
DEC AX         ; Decrement AX by 1 (AX now contains 4)

Negation (`NEG`)

The NEG instruction negates the value of the operand (i.e., changes its sign).

Syntax:

NEG operand

Example:

MOV AX, 5      ; Load 5 into AX register
NEG AX         ; Negate AX (AX now contains -5)

Flags Affected by Arithmetic Instructions

Arithmetic instructions often affect the CPU flags, which are used to determine the outcome of operations.

  • Zero Flag (ZF): Set if the result of the operation is zero.
  • Carry Flag (CF): Set if there is a carry out of the most significant bit (useful for unsigned arithmetic).
  • Overflow Flag (OF): Set if there is a signed overflow.
  • Sign Flag (SF): Set if the result is negative.
  • Parity Flag (PF): Set if the number of set bits in the result is even.

Exercises

  1. Write an assembly program to add two numbers and store the result in a register.
  2. Modify the program to subtract the second number from the first number.
  3. Create a program to multiply two numbers and handle the result correctly.
  4. Write a division program and ensure the quotient and remainder are stored in the correct registers.

Conclusion

Arithmetic instructions are a crucial part of assembly language programming, enabling basic mathematical operations on data. Understanding how to use these instructions and how they affect CPU flags is essential for writing efficient and effective assembly code. Practice these instructions with various operands to gain a deeper understanding and proficiency.
