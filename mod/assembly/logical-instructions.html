Introduction

Logical instructions in Assembly language are used to perform bitwise operations on binary data. These operations are fundamental for tasks such as setting, clearing, toggling, and testing specific bits within a byte or word. Understanding these instructions is crucial for low-level programming and hardware manipulation.

Key Concepts

  • Bitwise AND (AND): Performs a logical AND operation between corresponding bits of two operands.
  • Bitwise OR (OR): Performs a logical OR operation between corresponding bits of two operands.
  • Bitwise XOR (XOR): Performs a logical exclusive OR operation between corresponding bits of two operands.
  • Bitwise NOT (NOT): Inverts all the bits of the operand.
  • Shift Left (SHL): Shifts bits to the left, filling with zeros.
  • Shift Right (SHR): Shifts bits to the right, filling with zeros.
  • Rotate Left (ROL): Rotates bits to the left, wrapping the leftmost bit to the rightmost position.
  • Rotate Right (ROR): Rotates bits to the right, wrapping the rightmost bit to the leftmost position.

Bitwise AND (`AND`)

The AND instruction performs a bitwise AND operation between two operands. Each bit in the result is set to 1 if both corresponding bits in the operands are 1.

Example

MOV AL, 0b11001100 ; AL = 11001100
AND AL, 0b10101010 ; AL = AL AND 10101010
; Result: AL = 10001000

Explanation:

  • 11001100 AND 10101010 = 10001000

Bitwise OR (`OR`)

The OR instruction performs a bitwise OR operation between two operands. Each bit in the result is set to 1 if at least one of the corresponding bits in the operands is 1.

Example

MOV AL, 0b11001100 ; AL = 11001100
OR AL, 0b10101010 ; AL = AL OR 10101010
; Result: AL = 11101110

Explanation:

  • 11001100 OR 10101010 = 11101110

Bitwise XOR (`XOR`)

The XOR instruction performs a bitwise exclusive OR operation between two operands. Each bit in the result is set to 1 if only one of the corresponding bits in the operands is 1.

Example

MOV AL, 0b11001100 ; AL = 11001100
XOR AL, 0b10101010 ; AL = AL XOR 10101010
; Result: AL = 01100110

Explanation:

  • 11001100 XOR 10101010 = 01100110

Bitwise NOT (`NOT`)

The NOT instruction inverts all the bits of the operand.

Example

MOV AL, 0b11001100 ; AL = 11001100
NOT AL ; AL = NOT AL
; Result: AL = 00110011

Explanation:

  • NOT 11001100 = 00110011

Shift Left (`SHL`)

The SHL instruction shifts bits to the left, filling with zeros. Each shift left operation effectively multiplies the number by 2.

Example

MOV AL, 0b00001111 ; AL = 00001111
SHL AL, 1 ; Shift left by 1 bit
; Result: AL = 00011110

Explanation:

  • 00001111 << 1 = 00011110

Shift Right (`SHR`)

The SHR instruction shifts bits to the right, filling with zeros. Each shift right operation effectively divides the number by 2.

Example

MOV AL, 0b00001111 ; AL = 00001111
SHR AL, 1 ; Shift right by 1 bit
; Result: AL = 00000111

Explanation:

  • 00001111 >> 1 = 00000111

Rotate Left (`ROL`)

The ROL instruction rotates bits to the left, wrapping the leftmost bit to the rightmost position.

Example

MOV AL, 0b10000001 ; AL = 10000001
ROL AL, 1 ; Rotate left by 1 bit
; Result: AL = 00000011

Explanation:

  • 10000001 rotated left by 1 = 00000011

Rotate Right (`ROR`)

The ROR instruction rotates bits to the right, wrapping the rightmost bit to the leftmost position.

Example

MOV AL, 0b10000001 ; AL = 10000001
ROR AL, 1 ; Rotate right by 1 bit
; Result: AL = 11000000

Explanation:

  • 10000001 rotated right by 1 = 11000000

Summary

Logical instructions in Assembly are essential for manipulating data at the bit level. They include operations like AND, OR, XOR, NOT, and various shift and rotate instructions. Mastering these instructions allows programmers to perform efficient bitwise operations, which are crucial for tasks such as setting, clearing, and testing bits, as well as performing arithmetic operations and data encoding/decoding. Understanding and practicing these instructions will enhance your ability to write low-level code and interact directly with hardware.
