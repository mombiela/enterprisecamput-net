Introduction to Assembly Language

Assembly language is a low-level programming language that is closely related to machine code. It provides a way to write programs that are directly executed by the CPU. Understanding basic assembly instructions is crucial for writing efficient and effective assembly code.

Key Concepts

  • Instruction Set Architecture (ISA): The set of instructions that a particular CPU can execute.
  • Registers: Small, fast storage locations within the CPU used to hold data and addresses.
  • Operands: The data or addresses that instructions operate on.
  • Mnemonics: Human-readable representations of machine instructions.

Basic Assembly Instructions

Data Movement Instructions

These instructions are used to move data between registers, memory, and I/O ports.

  • MOV: Move data from one location to another.

    MOV AX, BX  ; Move the contents of register BX into register AX
MOV [1234H], AX  ; Move the contents of register AX into memory location 1234H

  • PUSH/POP: Push data onto the stack and pop data from the stack.

    PUSH AX  ; Push the contents of AX onto the stack
POP BX   ; Pop the top of the stack into BX

Arithmetic Instructions

These instructions perform arithmetic operations such as addition, subtraction, multiplication, and division.

  • ADD: Add two operands.

    ADD AX, BX  ; Add the contents of BX to AX and store the result in AX

  • SUB: Subtract one operand from another.

    SUB AX, BX  ; Subtract the contents of BX from AX and store the result in AX

  • MUL: Multiply two operands.

    MUL BX  ; Multiply AX by BX, result stored in AX (for 16-bit operands)

  • DIV: Divide one operand by another.

    DIV BX  ; Divide AX by BX, quotient in AX, remainder in DX (for 16-bit operands)

Logical Instructions

These instructions perform bitwise logical operations.

  • AND: Perform a bitwise AND operation.

    AND AX, BX  ; Perform bitwise AND on AX and BX, store result in AX

  • OR: Perform a bitwise OR operation.

    OR AX, BX  ; Perform bitwise OR on AX and BX, store result in AX

  • XOR: Perform a bitwise XOR operation.

    XOR AX, BX  ; Perform bitwise XOR on AX and BX, store result in AX

  • NOT: Perform a bitwise NOT operation.

    NOT AX  ; Perform bitwise NOT on AX, store result in AX

Control Flow Instructions

These instructions alter the flow of execution in a program.

  • JMP: Unconditional jump to a specified address.

    JMP 1234H  ; Jump to memory location 1234H

  • Conditional Jumps: Jump to a specified address if a condition is met.

    JE 1234H  ; Jump to 1234H if the zero flag (ZF) is set (equal)
JNE 1234H ; Jump to 1234H if the zero flag (ZF) is not set (not equal)

  • CALL/RET: Call a procedure and return from a procedure.

    CALL 1234H  ; Call procedure at memory location 1234H
RET         ; Return from procedure

Example Program

Here is a simple example program that demonstrates some of the basic assembly instructions discussed above:

section .data
    num1 dw 10
    num2 dw 20
    result dw 0

section .text
    global _start

_start:
    ; Load num1 and num2 into registers
    MOV AX, [num1]
    MOV BX, [num2]

    ; Add num1 and num2
    ADD AX, BX

    ; Store the result
    MOV [result], AX

    ; Exit the program
    MOV AX, 1
    INT 0x80

Conclusion

Understanding basic assembly instructions is the foundation for writing effective assembly programs. By mastering data movement, arithmetic, logical, and control flow instructions, you can begin to write more complex and efficient code. Practice these instructions with simple programs to build a solid understanding before moving on to more advanced topics.
