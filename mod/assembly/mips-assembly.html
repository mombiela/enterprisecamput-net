Introduction to MIPS Assembly

MIPS (Microprocessor without Interlocked Pipeline Stages) is a RISC (Reduced Instruction Set Computer) architecture that is widely used in academic settings to teach assembly language and computer architecture concepts.

Key Concepts

  • RISC Architecture: Simplified instructions that can be executed within one clock cycle.
  • Registers: Small, fast storage locations within the CPU.
  • Memory: Organized into bytes and words, accessed via addresses.
  • Instruction Set: The set of commands the CPU can execute.

MIPS Assembly Basics

Registers

MIPS has 32 general-purpose registers, each 32 bits wide. Some important registers include:

  • $zero: Always holds the value 0.
  • $v0-$v1: Used for function return values.
  • $a0-$a3: Used for function arguments.
  • $t0-$t9: Temporary registers.
  • $s0-$s7: Saved registers.
  • $sp: Stack pointer.
  • $ra: Return address.

Basic Instructions

  • Arithmetic Instructions: add, sub, mul, div
  • Logical Instructions: and, or, xor, nor
  • Data Transfer Instructions: lw (load word), sw (store word)
  • Control Flow Instructions: beq (branch if equal), bne (branch if not equal), j (jump)

Example: Adding Two Numbers

.data
num1: .word 5
num2: .word 10
result: .word 0

.text
main:
    lw $t0, num1       # Load num1 into $t0
    lw $t1, num2       # Load num2 into $t1
    add $t2, $t0, $t1  # Add $t0 and $t1, store result in $t2
    sw $t2, result     # Store the result back into memory
    li $v0, 10         # Exit program
    syscall

Intermediate MIPS Assembly

Control Flow

  • Conditional Branching: beq, bne
  • Unconditional Jump: j, jal (jump and link)

Example: Looping

.data
array: .word 1, 2, 3, 4, 5
sum: .word 0

.text
main:
    li $t0, 0          # Initialize index to 0
    li $t1, 5          # Set loop count to 5
    li $t2, 0          # Initialize sum to 0

loop:
    beq $t0, $t1, end  # If index == 5, exit loop
    lw $t3, array($t0) # Load array element into $t3
    add $t2, $t2, $t3  # Add element to sum
    addi $t0, $t0, 4   # Move to next element (word = 4 bytes)
    j loop             # Repeat loop

end:
    sw $t2, sum        # Store the sum
    li $v0, 10         # Exit program
    syscall

Advanced MIPS Assembly

Procedures and Functions

  • Calling Conventions: How arguments are passed and return values are handled.
  • Stack Management: Using the stack to save and restore registers.

Example: Factorial Function

.data
result: .word 0

.text
main:
    li $a0, 5          # Load argument (5) into $a0
    jal factorial      # Call factorial function
    sw $v0, result     # Store the result
    li $v0, 10         # Exit program
    syscall

factorial:
    addi $sp, $sp, -8  # Allocate stack space
    sw $ra, 4($sp)     # Save return address
    sw $a0, 0($sp)     # Save argument

    li $v0, 1          # Base case: factorial(0) = 1
    beq $a0, $zero, end_factorial

    addi $a0, $a0, -1  # Decrement argument
    jal factorial      # Recursive call
    lw $a0, 0($sp)     # Restore argument
    mul $v0, $a0, $v0  # Multiply result with argument

end_factorial:
    lw $ra, 4($sp)     # Restore return address
    addi $sp, $sp, 8   # Deallocate stack space
    jr $ra             # Return to caller

Conclusion

MIPS assembly language provides a clear and structured way to understand low-level programming and computer architecture. By mastering MIPS, you gain insight into how high-level languages are translated into machine code, how the CPU executes instructions, and how memory and registers are managed. This knowledge is fundamental for optimizing code, understanding system performance, and developing efficient software.
