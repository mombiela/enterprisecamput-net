Introduction to Webpack

  • What is Webpack?

    • Webpack is a module bundler for JavaScript applications.
    • It takes modules with dependencies and generates static assets representing those modules.

  • Why use Webpack?

    • Efficiently manages dependencies.
    • Optimizes code for production.
    • Supports modern JavaScript features and tools.

Setting Up Webpack

  • Installing Webpack

    • Install Webpack and Webpack CLI via npm: 
      npm install --save-dev webpack webpack-cli

  • Creating the Project Structure

    • Basic project structure: 
      my-app/
├── src/
│   └── index.js
├── dist/
├── package.json
└── webpack.config.js

Configuring Webpack

  • Basic Configuration

    • Create webpack.config.js in the root directory: 
      const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  },
  mode: 'development'
};

  • Running Webpack

    • Add a script to package.json: 
      "scripts": {
  "build": "webpack"
}
    • Run the build process: 
      npm run build

Loading and Bundling Assets

  • Handling CSS

    • Install loaders: 
      npm install --save-dev style-loader css-loader
    • Update webpack.config.js: 
      module.exports = {
  // ...other configurations
  module: {
    rules: [
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader']
      }
    ]
  }
};

  • Handling Images

    • Install file-loader: 
      npm install --save-dev file-loader
    • Update webpack.config.js: 
      module.exports = {
  // ...other configurations
  module: {
    rules: [
      {
        test: /\.(png|svg|jpg|gif)$/,
        use: ['file-loader']
      }
    ]
  }
};

Advanced Configuration

  • Using Plugins

    • Install HTML Webpack Plugin: 
      npm install --save-dev html-webpack-plugin
    • Update webpack.config.js: 
      const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
  // ...other configurations
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html'
    })
  ]
};

  • Code Splitting

    • Modify webpack.config.js to enable code splitting: 
      module.exports = {
  // ...other configurations
  optimization: {
    splitChunks: {
      chunks: 'all'
    }
  }
};

Development Tools

  • Using Webpack Dev Server
    • Install Webpack Dev Server: 
      npm install --save-dev webpack-dev-server
    • Update webpack.config.js: 
      module.exports = {
  // ...other configurations
  devServer: {
    contentBase: './dist'
  }
};
    • Add a script to package.json: 
      "scripts": {
  "start": "webpack serve --open"
}
    • Start the development server: 
      npm start

Conclusion

  • Summary

    • Webpack is a powerful tool for bundling JavaScript applications.
    • It simplifies dependency management and optimizes code for production.
    • By using loaders and plugins, Webpack can handle various types of assets and enhance the development experience.

  • Next Steps

    • Explore more advanced features of Webpack such as tree shaking and custom plugins.
    • Integrate Webpack with other tools and frameworks like React or Vue.js.
    • Continue practicing by building more complex applications and configurations.
