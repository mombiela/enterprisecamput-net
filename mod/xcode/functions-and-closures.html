Introduction to Functions

Functions are self-contained chunks of code that perform a specific task. In Swift, functions are first-class citizens, meaning they can be passed around and used in your code just like any other value.

Defining and Calling Functions

To define a function in Swift, you use the func keyword followed by the function name, a pair of parentheses, and a set of curly braces containing the function's code.

func greet(person: String) -> String {
    return "Hello, \(person)!"
}

let greeting = greet(person: "Alice")
print(greeting) // Output: Hello, Alice!
  • func: Keyword to define a function.
  • greet(person: String) -> String: Function signature with a parameter and return type.
  • return "Hello, \\(person)!": The function body that returns a greeting message.

Function Parameters and Return Values

Functions can take multiple parameters and return multiple values using tuples.

func add(a: Int, b: Int) -> Int {
    return a + b
}

let sum = add(a: 3, b: 5)
print(sum) // Output: 8
  • add(a: Int, b: Int) -> Int: Function with two parameters and a return type.

Default Parameter Values

You can provide default values for any parameter in a function by assigning a value to the parameter after its type.

func greet(person: String, from hometown: String = "Cupertino") -> String {
    return "Hello \(person)! Glad you could visit from \(hometown)."
}

print(greet(person: "Alice")) // Output: Hello Alice! Glad you could visit from Cupertino.
print(greet(person: "Bob", from: "New York")) // Output: Hello Bob! Glad you could visit from New York.

Advanced Function Concepts

In-Out Parameters

In-out parameters allow a function to modify a variable passed to it from outside the function.

func swapTwoInts(_ a: inout Int, _ b: inout Int) {
    let temporaryA = a
    a = b
    b = temporaryA
}

var x = 3
var y = 5
swapTwoInts(&x, &y)
print("x: \(x), y: \(y)") // Output: x: 5, y: 3
  • inout: Keyword to indicate an in-out parameter.
  • &: Prefix used when passing a variable as an in-out parameter.

Function Types

Every function has a specific function type, made up of the parameter types and the return type.

func multiply(a: Int, b: Int) -> Int {
    return a * b
}

let mathFunction: (Int, Int) -> Int = multiply
print(mathFunction(2, 3)) // Output: 6
  • (Int, Int) -> Int: Function type for a function that takes two Int parameters and returns an Int.

Introduction to Closures

Closures are self-contained blocks of functionality that can be passed around and used in your code. Closures in Swift are similar to blocks in C and Objective-C and to lambdas in other programming languages.

Defining and Using Closures

Closures can capture and store references to any constants and variables from the context in which they are defined.

let names = ["Chris", "Alex", "Ewa", "Barry", "Daniella"]

let sortedNames = names.sorted(by: { (s1: String, s2: String) -> Bool in
    return s1 < s2
})

print(sortedNames) // Output: ["Alex", "Barry", "Chris", "Daniella", "Ewa"]
  • { (s1: String, s2: String) -> Bool in return s1 < s2 }: Closure expression syntax.

Shorthand Argument Names

Swift provides shorthand argument names to make closures more concise.

let sortedNames = names.sorted(by: { $0 < $1 })
print(sortedNames) // Output: ["Alex", "Barry", "Chris", "Daniella", "Ewa"]
  • $0, $1: Shorthand argument names for the first and second parameters.

Trailing Closures

If a closure is the last argument to a function, you can use trailing closure syntax.

let sortedNames = names.sorted { $0 < $1 }
print(sortedNames) // Output: ["Alex", "Barry", "Chris", "Daniella", "Ewa"]

Capturing Values

Closures can capture constants and variables from the surrounding context in which they are defined.

func makeIncrementer(forIncrement amount: Int) -> () -> Int {
    var total = 0
    let incrementer: () -> Int = {
        total += amount
        return total
    }
    return incrementer
}

let incrementByTwo = makeIncrementer(forIncrement: 2)
print(incrementByTwo()) // Output: 2
print(incrementByTwo()) // Output: 4
  • total += amount: Capturing and modifying the total variable from the surrounding context.

Conclusion

Functions and closures are fundamental concepts in Swift programming. Functions allow you to encapsulate code for reuse and clarity, while closures provide a powerful way to capture and manipulate values from their surrounding context. Mastering these concepts will enable you to write more modular, flexible, and concise code in Swift.
