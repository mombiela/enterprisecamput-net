Introduction

In this section, we will delve into advanced data structures in Swift. Understanding these structures will help you write more efficient and optimized code. We will cover:

  • Linked Lists
  • Trees
  • Graphs
  • Hash Tables
  • Heaps

Linked Lists

A linked list is a linear data structure where each element is a separate object, known as a node. Each node contains data and a reference (or link) to the next node in the sequence.

Key Concepts

  • Node: The basic unit of a linked list, containing data and a reference to the next node.
  • Head: The first node in the linked list.
  • Tail: The last node in the linked list, which points to nil.

Code Example

class Node<T> {
    var value: T
    var next: Node?

    init(value: T) {
        self.value = value
    }
}

class LinkedList<T> {
    var head: Node<T>?

    func append(value: T) {
        let newNode = Node(value: value)
        if let tailNode = head {
            var currentNode = tailNode
            while let nextNode = currentNode.next {
                currentNode = nextNode
            }
            currentNode.next = newNode
        } else {
            head = newNode
        }
    }
}

Explanation

  • Node Class: Defines a node with a value and a reference to the next node.
  • LinkedList Class: Manages the linked list, with methods to append new nodes.

Trees

A tree is a hierarchical data structure consisting of nodes, with a single node as the root and sub-nodes as children.

Key Concepts

  • Root: The top node of the tree.
  • Leaf: A node with no children.
  • Binary Tree: A tree where each node has at most two children.

Code Example

class TreeNode<T> {
    var value: T
    var left: TreeNode?
    var right: TreeNode?

    init(value: T) {
        self.value = value
    }
}

class BinaryTree<T> {
    var root: TreeNode<T>?

    func insert(value: T) {
        let newNode = TreeNode(value: value)
        if let rootNode = root {
            self.insertNode(rootNode, newNode)
        } else {
            root = newNode
        }
    }

    private func insertNode(_ node: TreeNode<T>, _ newNode: TreeNode<T>) {
        if newNode.value < node.value {
            if let leftNode = node.left {
                insertNode(leftNode, newNode)
            } else {
                node.left = newNode
            }
        } else {
            if let rightNode = node.right {
                insertNode(rightNode, newNode)
            } else {
                node.right = newNode
            }
        }
    }
}

Explanation

  • TreeNode Class: Defines a node with a value and references to left and right children.
  • BinaryTree Class: Manages the binary tree, with methods to insert new nodes.

Graphs

A graph is a collection of nodes (vertices) and edges connecting them. Graphs can be directed or undirected.

Key Concepts

  • Vertex: A node in the graph.
  • Edge: A connection between two vertices.
  • Directed Graph: Edges have a direction.
  • Undirected Graph: Edges do not have a direction.

Code Example

class Vertex<T> {
    var value: T
    var edges: [Edge<T>] = []

    init(value: T) {
        self.value = value
    }
}

class Edge<T> {
    var from: Vertex<T>
    var to: Vertex<T>

    init(from: Vertex<T>, to: Vertex<T>) {
        self.from = from
        self.to = to
    }
}

class Graph<T> {
    var vertices: [Vertex<T>] = []

    func addVertex(value: T) -> Vertex<T> {
        let vertex = Vertex(value: value)
        vertices.append(vertex)
        return vertex
    }

    func addEdge(from: Vertex<T>, to: Vertex<T>) {
        let edge = Edge(from: from, to: to)
        from.edges.append(edge)
    }
}

Explanation

  • Vertex Class: Defines a vertex with a value and a list of edges.
  • Edge Class: Defines an edge with references to the starting and ending vertices.
  • Graph Class: Manages the graph, with methods to add vertices and edges.

Hash Tables

A hash table is a data structure that maps keys to values for efficient lookup.

Key Concepts

  • Hash Function: Converts a key into an index in the hash table.
  • Collision: When two keys hash to the same index.
  • Chaining: A method to handle collisions by storing multiple elements at the same index.

Code Example

class HashTable<Key: Hashable, Value> {
    private var buckets: [[(key: Key, value: Value)]]
    private let capacity: Int

    init(capacity: Int) {
        self.capacity = capacity
        self.buckets = Array(repeating: [], count: capacity)
    }

    func insert(key: Key, value: Value) {
        let index = self.index(for: key)
        buckets[index].append((key: key, value: value))
    }

    func value(for key: Key) -> Value? {
        let index = self.index(for: key)
        for element in buckets[index] {
            if element.key == key {
                return element.value
            }
        }
        return nil
    }

    private func index(for key: Key) -> Int {
        return abs(key.hashValue) % capacity
    }
}

Explanation

  • HashTable Class: Manages the hash table with methods to insert and retrieve values.
  • index(for:) Method: Computes the index for a given key using a hash function.

Heaps

A heap is a specialized tree-based data structure that satisfies the heap property.

Key Concepts

  • Max-Heap: The parent node is always greater than or equal to the child nodes.
  • Min-Heap: The parent node is always less than or equal to the child nodes.

Code Example

class Heap<T: Comparable> {
    var elements: [T] = []

    func insert(_ value: T) {
        elements.append(value)
        siftUp(from: elements.count - 1)
    }

    private func siftUp(from index: Int) {
        var childIndex = index
        let childValue = elements[childIndex]
        var parentIndex = (childIndex - 1) / 2

        while childIndex > 0 && childValue > elements[parentIndex] {
            elements[childIndex] = elements[parentIndex]
            childIndex = parentIndex
            parentIndex = (childIndex - 1) / 2
        }
        elements[childIndex] = childValue
    }
}

Explanation

  • Heap Class: Manages the heap with methods to insert elements.
  • siftUp(from:) Method: Ensures the heap property is maintained after insertion.

Conclusion

Understanding these structures will enable you to write more efficient and optimized code. Practice implementing these data structures and using them in your projects to solidify your understanding.
