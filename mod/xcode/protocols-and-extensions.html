Introduction to Protocols

Protocols in Swift define a blueprint of methods, properties, and other requirements that suit a particular task or piece of functionality. Any class, structure, or enumeration can adopt protocols to provide an actual implementation of those requirements.

Key Concepts

  • Protocol Declaration: How to declare a protocol.
  • Adopting a Protocol: How to make a class, struct, or enum conform to a protocol.
  • Protocol Inheritance: How protocols can inherit from other protocols.
  • Optional Protocol Requirements: How to define optional requirements in protocols.

Protocol Declaration

To declare a protocol, use the protocol keyword followed by the protocol's name and a set of requirements.

protocol Describable {
    var description: String { get }
    func describe() -> String
}

Adopting a Protocol

A type adopts a protocol by listing the protocol's name after the type's name, separated by a colon.

struct Car: Describable {
    var description: String
    var make: String
    var model: String

    func describe() -> String {
        return "This car is a \(make) \(model)."
    }
}

Protocol Inheritance

Protocols can inherit from one or more other protocols, and a protocol can require that conforming types implement properties or methods from multiple protocols.

protocol Identifiable {
    var id: String { get }
}

protocol Nameable {
    var name: String { get }
}

protocol Person: Identifiable, Nameable {
    var age: Int { get }
}

Optional Protocol Requirements

Protocols can define optional requirements using the @objc attribute and the optional keyword. This is only available for protocols that are marked with the @objc attribute.

@objc protocol OptionalProtocol {
    @objc optional func optionalMethod()
}

Introduction to Extensions

Extensions add new functionality to an existing class, structure, enumeration, or protocol type. This includes the ability to extend types for which you do not have access to the original source code.

Key Concepts

  • Extension Declaration: How to declare an extension.
  • Adding Computed Properties: How to add computed properties using extensions.
  • Adding Methods: How to add new methods using extensions.
  • Protocol Conformance with Extensions: How to use extensions to conform to protocols.

Extension Declaration

To declare an extension, use the extension keyword followed by the name of the type you want to extend.

extension Int {
    var isEven: Bool {
        return self % 2 == 0
    }
}

Adding Computed Properties

Extensions can add computed properties but cannot add stored properties.

extension Double {
    var km: Double { return self * 1_000.0 }
    var m: Double { return self }
    var cm: Double { return self / 100.0 }
    var mm: Double { return self / 1_000.0 }
}

Adding Methods

Extensions can add new methods to existing types.

extension String {
    func reversedString() -> String {
        return String(self.reversed())
    }
}

Protocol Conformance with Extensions

Extensions can be used to make an existing type conform to a protocol.

protocol Greetable {
    func greet()
}

extension String: Greetable {
    func greet() {
        print("Hello, \(self)!")
    }
}

Summary

In this section, we explored the concepts of protocols and extensions in Swift. Protocols define a blueprint of methods and properties that can be adopted by classes, structs, and enums. Extensions allow us to add new functionality to existing types, including the ability to conform to protocols. Understanding these concepts is crucial for writing flexible and reusable code in Swift.
