Introduction to Cryptography

Cryptography is a fundamental discipline in cybersecurity that deals with protecting information through encoding techniques. In this topic, we will explore how cryptography is applied in practice, its main algorithms, and how to implement them in software development.

Key Concepts

  • Encryption and Decryption: Process of converting readable information into an encoded format and vice versa.
  • Cryptographic Key: Information used to encrypt and decrypt data.
  • Symmetric Algorithms: Use the same key for encryption and decryption.
  • Asymmetric Algorithms: Use a pair of keys (public and private) for encryption and decryption.
  • Hashing: Transformation of data into a fixed-length string that represents the original data.

Symmetric Encryption Algorithms

Symmetric algorithms are fast and efficient for encrypting large amounts of data. Some of the most used are:

  • AES (Advanced Encryption Standard): Block cipher algorithm that uses 128, 192, or 256-bit keys.
  • DES (Data Encryption Standard): Older and less secure than AES, uses a 56-bit key.
  • 3DES (Triple DES): Improvement of DES that applies encryption three times for greater security.

Example of Symmetric Encryption with AES in Python

from Crypto.Cipher import AES
from Crypto.Random import get_random_bytes

# Generate a 16-byte (128-bit) key
key = get_random_bytes(16)

# Create an AES cipher object
cipher = AES.new(key, AES.MODE_EAX)

# Data to encrypt
data = b'This is a secret message'

# Encrypt the data
nonce = cipher.nonce
ciphertext, tag = cipher.encrypt_and_digest(data)

print(f'Encrypted: {ciphertext}')

In this example, we use the pycryptodome library to encrypt a message with AES.

Asymmetric Encryption Algorithms

Asymmetric algorithms are slower but allow for secure key distribution. The most common are:

  • RSA (Rivest-Shamir-Adleman): Uses variable-length keys, commonly 2048 or 4096 bits.
  • ECC (Elliptic Curve Cryptography): Offers the same security as RSA but with shorter keys.

Example of Asymmetric Encryption with RSA in Python

from Crypto.PublicKey import RSA
from Crypto.Cipher import PKCS1_OAEP
from Crypto.Random import get_random_bytes

# Generate an RSA key pair
key = RSA.generate(2048)
public_key = key.publickey()

# Create an RSA cipher object
cipher = PKCS1_OAEP.new(public_key)

# Data to encrypt
data = b'This is a secret message'

# Encrypt the data
ciphertext = cipher.encrypt(data)

print(f'Encrypted: {ciphertext}')

In this example, we use the pycryptodome library to encrypt a message with RSA.

Hashing

Hashing is a technique used to verify data integrity. The most common hashing algorithms are:

  • SHA-256 (Secure Hash Algorithm 256 bits): Produces a 256-bit hash.
  • MD5 (Message Digest Algorithm 5): Produces a 128-bit hash, but is less secure than SHA-256.

Example of Hashing with SHA-256 in Python

import hashlib

# Data to hash
data = b'This is a secret message'

# Create a SHA-256 hashing object
hash_object = hashlib.sha256(data)

# Get the hash in hexadecimal format
hash_hex = hash_object.hexdigest()

print(f'Hash: {hash_hex}')

In this example, we use the hashlib library to generate a SHA-256 hash of a message.

Algorithm Comparison

| Algorithm | Type | Key Length | Security | |-----------|------|------------|----------| | AES | Symmetric | 128, 192, 256 bits | High | | DES | Symmetric | 56 bits | Low | | 3DES | Symmetric | 168 bits | Medium | | RSA | Asymmetric | 1024, 2048, 4096 bits | High | | ECC | Asymmetric | 160-521 bits | High | | SHA-256 | Hashing | 256 bits | High | | MD5 | Hashing | 128 bits | Low |

Conclusion

Cryptography is essential for protecting information in the field of cybersecurity. Knowing and correctly applying the different encryption and hashing algorithms ensures the confidentiality, integrity, and authenticity of data. In this topic, we have explored the key concepts, the most used algorithms, and how to implement them in Python, providing a solid foundation for their practical application in secure software development.
