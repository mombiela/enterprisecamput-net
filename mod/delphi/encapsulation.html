Encapsulation in Delphi/Object Pascal
Interfaces ►

Encapsulation is a fundamental concept in Object-Oriented Programming (OOP) that helps in bundling the data (attributes) and methods (functions or procedures) that operate on the data into a single unit, known as a class. It also restricts direct access to some of the object's components, which is a means of preventing accidental interference and misuse of the data.

Key Concepts of Encapsulation

  • Class: A blueprint for creating objects. It encapsulates data for the object and methods to manipulate that data.
  • Object: An instance of a class.
  • Access Specifiers: Keywords that set the accessibility of classes, methods, and other members. Common access specifiers in Delphi/Object Pascal are private, protected, public, and published.

Access Specifiers

  • Private: Members declared as private are accessible only within the same unit.
  • Protected: Members declared as protected are accessible within the same unit and by descendant classes.
  • Public: Members declared as public are accessible from any code that can see the class.
  • Published: Similar to public, but also enables the properties to be streamed and inspected at runtime.

Example of Encapsulation in Delphi/Object Pascal

Let's create a simple class to demonstrate encapsulation:

unit UPerson;

interface

type
  TPerson = class
  private
    FName: string;
    FAge: Integer;
    procedure SetName(const Value: string);
    procedure SetAge(Value: Integer);
  public
    constructor Create(Name: string; Age: Integer);
    function GetDetails: string;
    property Name: string read FName write SetName;
    property Age: Integer read FAge write SetAge;
  end;

implementation

{ TPerson }

constructor TPerson.Create(Name: string; Age: Integer);
begin
  FName := Name;
  FAge := Age;
end;

procedure TPerson.SetName(const Value: string);
begin
  if Value <> '' then
    FName := Value;
end;

procedure TPerson.SetAge(Value: Integer);
begin
  if (Value > 0) and (Value < 150) then
    FAge := Value;
end;

function TPerson.GetDetails: string;
begin
  Result := Format('Name: %s, Age: %d', [FName, FAge]);
end;

end.

Explanation

  • Class Declaration: The TPerson class is declared with private fields FName and FAge to store the name and age of a person.
  • Constructor: The constructor Create initializes the fields.
  • Property Methods: SetName and SetAge are private methods to set the values of FName and FAge with validation.
  • Properties: Name and Age properties provide controlled access to the private fields.
  • Public Method: GetDetails is a public method to retrieve the details of the person.

Exercises

  1. Create a New Class: Create a class TCar with private fields for Make, Model, and Year. Implement properties and methods to encapsulate these fields.
  2. Modify the Class: Add validation to the SetYear method to ensure the year is within a reasonable range (e.g., 1886 to the current year).
  3. Use the Class: Write a program that creates an instance of TCar, sets its properties, and prints the details.

Conclusion

Encapsulation is a powerful feature in Delphi/Object Pascal that helps in organizing code, protecting data, and enhancing maintainability. By using classes and access specifiers, you can control how data is accessed and modified, ensuring that your objects are used correctly and safely.
