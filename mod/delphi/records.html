Introduction to Records

Records in Delphi/Object Pascal are a way to group related data together. They are similar to structures in C/C++ and can contain fields of different types. Records are useful for organizing data that belongs together and can help make your code more readable and maintainable.

Key Concepts

  • Definition: A record is a composite data type that groups together different types of data.
  • Fields: The individual elements within a record.
  • Access: Fields within a record are accessed using the dot (.) operator.

Defining Records

To define a record in Delphi/Object Pascal, you use the record keyword. Here's a basic example:

type
  TPerson = record
    FirstName: string;
    LastName: string;
    Age: Integer;
  end;

In this example:

  • TPerson is the name of the record type.
  • FirstName, LastName, and Age are fields within the record.

Creating and Using Records

Once you have defined a record type, you can create variables of that type and access their fields.

Example

var
  Person: TPerson;
begin
  Person.FirstName := 'John';
  Person.LastName := 'Doe';
  Person.Age := 30;

  WriteLn('Name: ', Person.FirstName, ' ', Person.LastName);
  WriteLn('Age: ', Person.Age);
end;

In this example:

  • A variable Person of type TPerson is created.
  • The fields FirstName, LastName, and Age are assigned values.
  • The values are then printed to the console.

Advanced Record Features

Methods in Records

Records in Delphi/Object Pascal can also contain methods, which are functions or procedures that operate on the record's data.

Example

type
  TPerson = record
    FirstName: string;
    LastName: string;
    Age: Integer;
    procedure Display;
  end;

procedure TPerson.Display;
begin
  WriteLn('Name: ', FirstName, ' ', LastName);
  WriteLn('Age: ', Age);
end;

var
  Person: TPerson;
begin
  Person.FirstName := 'Jane';
  Person.LastName := 'Smith';
  Person.Age := 25;
  Person.Display;
end;

In this example:

  • A Display method is added to the TPerson record.
  • The Display method prints the record's fields to the console.

Records with Constructors

Records can also have constructors, which are special methods used to initialize the record's fields.

Example

type
  TPerson = record
    FirstName: string;
    LastName: string;
    Age: Integer;
    constructor Create(AFirstName, ALastName: string; AAge: Integer);
  end;

constructor TPerson.Create(AFirstName, ALastName: string; AAge: Integer);
begin
  FirstName := AFirstName;
  LastName := ALastName;
  Age := AAge;
end;

var
  Person: TPerson;
begin
  Person := TPerson.Create('Alice', 'Johnson', 28);
  Person.Display;
end;

In this example:

  • A Create constructor is defined to initialize the fields of TPerson.
  • The Create constructor is used to create and initialize a Person record.

Comparison with Classes

While records are useful for grouping related data, they have some limitations compared to classes. Here's a comparison:

| Feature | Records | Classes | |------------------|---------------------------------|----------------------------------| | Memory Allocation| Stack (usually) | Heap | | Inheritance | No | Yes | | Methods | Yes | Yes | | Constructors | Yes | Yes | | Destructors | No | Yes |

Conclusion

Records in Delphi/Object Pascal are a powerful way to group related data together. They can contain fields of different types, methods, and constructors. While they are not as feature-rich as classes, they are simpler and can be more efficient for certain tasks. Understanding when and how to use records will help you write more organized and maintainable code.
