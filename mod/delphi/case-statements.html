Introduction to Case Statements

Case statements in Delphi/Object Pascal are used to simplify complex conditional logic. They provide a way to execute different blocks of code based on the value of a variable or expression.

Key Concepts

  • Case Statement Structure: The basic structure of a case statement.
  • Simple Case Statements: Using case statements with ordinal types.
  • Nested Case Statements: Using case statements within other case statements.
  • Case Statements with Strings: Using case statements with string values.
  • Default Case: Handling unexpected values.

Case Statement Structure

The general syntax of a case statement in Delphi/Object Pascal is as follows:

case expression of
  value1: statement1;
  value2: statement2;
  value3: statement3;
  ...
else
  default_statement;
end;
  • expression: The variable or expression being evaluated.
  • value1, value2, value3, ...: Possible values of the expression.
  • statement1, statement2, statement3, ...: Statements to execute for each corresponding value.
  • else: Optional. Executes if none of the specified values match the expression.

Simple Case Statements

A simple case statement example with an ordinal type (integer):

var
  day: Integer;
begin
  day := 3;
  case day of
    1: WriteLn('Monday');
    2: WriteLn('Tuesday');
    3: WriteLn('Wednesday');
    4: WriteLn('Thursday');
    5: WriteLn('Friday');
    6: WriteLn('Saturday');
    7: WriteLn('Sunday');
  else
    WriteLn('Invalid day');
  end;
end;

Explanation

  • The variable day is evaluated.
  • Depending on the value of day, the corresponding WriteLn statement is executed.
  • If day is not between 1 and 7, the else clause executes, printing 'Invalid day'.

Nested Case Statements

Case statements can be nested within other case statements to handle more complex logic:

var
  category, item: Integer;
begin
  category := 1;
  item := 2;
  case category of
    1: begin
      case item of
        1: WriteLn('Category 1, Item 1');
        2: WriteLn('Category 1, Item 2');
      else
        WriteLn('Unknown item in Category 1');
      end;
    end;
    2: begin
      case item of
        1: WriteLn('Category 2, Item 1');
        2: WriteLn('Category 2, Item 2');
      else
        WriteLn('Unknown item in Category 2');
      end;
    end;
  else
    WriteLn('Unknown category');
  end;
end;

Explanation

  • The outer case statement evaluates category.
  • Depending on the value of category, an inner case statement evaluates item.
  • This structure allows for detailed handling of combinations of category and item.

Case Statements with Strings

Delphi/Object Pascal also supports case statements with string values:

var
  command: string;
begin
  command := 'start';
  case command of
    'start': WriteLn('Starting the process');
    'stop': WriteLn('Stopping the process');
    'pause': WriteLn('Pausing the process');
  else
    WriteLn('Unknown command');
  end;
end;

Explanation

  • The variable command is evaluated.
  • Depending on the value of command, the corresponding WriteLn statement is executed.
  • If command does not match any specified value, the else clause executes, printing 'Unknown command'.

Default Case

The else clause in a case statement acts as a default case, handling any values not explicitly listed:

var
  grade: Char;
begin
  grade := 'B';
  case grade of
    'A': WriteLn('Excellent');
    'B': WriteLn('Good');
    'C': WriteLn('Average');
    'D': WriteLn('Below Average');
    'F': WriteLn('Fail');
  else
    WriteLn('Invalid grade');
  end;
end;

Explanation

  • The variable grade is evaluated.
  • Depending on the value of grade, the corresponding WriteLn statement is executed.
  • If grade does not match any specified value, the else clause executes, printing 'Invalid grade'.

Conclusion

Case statements in Delphi/Object Pascal provide a powerful way to handle multiple conditions with clear and readable code. They can be used with ordinal types, strings, and can be nested for complex logic. The else clause ensures that unexpected values are handled gracefully. By mastering case statements, you can write more efficient and maintainable code.
