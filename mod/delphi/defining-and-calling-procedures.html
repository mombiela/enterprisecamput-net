Defining and Calling Procedures in Delphi/Object Pascal

Introduction

In Delphi/Object Pascal, procedures are a fundamental part of structured programming. They allow you to encapsulate code into reusable blocks, making your programs more modular and easier to maintain. This section will guide you through defining and calling procedures, from basic to advanced concepts.

Defining Procedures

Basic Procedure Definition

A procedure in Delphi is defined using the procedure keyword, followed by the procedure name and a block of code enclosed in begin and end.

procedure SayHello;
begin
  Writeln('Hello, world!');
end;
  • procedure: Keyword to define a procedure.
  • SayHello: Name of the procedure.
  • begin...end: Block of code that the procedure will execute.

Procedure with Parameters

Procedures can accept parameters, allowing you to pass data into them.

procedure GreetUser(Name: string);
begin
  Writeln('Hello, ' + Name + '!');
end;
  • Name: string: Parameter of type string that the procedure accepts.

Procedure with Multiple Parameters

You can define procedures with multiple parameters of different types.

procedure DisplayUserInfo(Name: string; Age: Integer);
begin
  Writeln('Name: ' + Name);
  Writeln('Age: ' + IntToStr(Age));
end;
  • Name: string: First parameter of type string.
  • Age: Integer: Second parameter of type Integer.

Calling Procedures

Calling a Simple Procedure

To call a procedure, simply use its name followed by parentheses.

begin
  SayHello;
end.

Calling a Procedure with Parameters

When calling a procedure with parameters, provide the arguments within the parentheses.

begin
  GreetUser('Alice');
  DisplayUserInfo('Bob', 30);
end.

Advanced Concepts

Passing Parameters by Reference

By default, parameters are passed by value. To pass a parameter by reference, use the var keyword.

procedure Increment(var Number: Integer);
begin
  Number := Number + 1;
end;

begin
  var MyNumber: Integer := 5;
  Increment(MyNumber);
  Writeln(MyNumber); // Output: 6
end.
  • var Number: Integer: Parameter passed by reference, allowing the procedure to modify the original variable.

Procedure Overloading

Delphi allows you to define multiple procedures with the same name but different parameter lists. This is known as overloading.

procedure PrintMessage(Message: string); overload;
begin
  Writeln(Message);
end;

procedure PrintMessage(Number: Integer); overload;
begin
  Writeln(IntToStr(Number));
end;

begin
  PrintMessage('Hello');
  PrintMessage(123);
end.
  • overload: Keyword to indicate that the procedure is overloaded.

Nested Procedures

Procedures can be nested within other procedures, allowing for more localized scope.

procedure OuterProcedure;
  procedure InnerProcedure;
  begin
    Writeln('Inner Procedure');
  end;
begin
  Writeln('Outer Procedure');
  InnerProcedure;
end;

begin
  OuterProcedure;
end.
  • InnerProcedure: Nested within OuterProcedure, only accessible within OuterProcedure.

Conclusion

Procedures in Delphi/Object Pascal are powerful tools for creating modular and maintainable code. By understanding how to define and call procedures, pass parameters by value or reference, and use advanced features like overloading and nesting, you can write more efficient and organized programs. Practice defining and calling procedures with different parameters to become proficient in using them effectively.
