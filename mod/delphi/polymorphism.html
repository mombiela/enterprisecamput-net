Polymorphism in Delphi/Object Pascal
Encapsulation ►

Polymorphism is a fundamental concept in object-oriented programming that allows objects to be treated as instances of their parent class rather than their actual class. This enables a single interface to represent different underlying forms (data types). In Delphi/Object Pascal, polymorphism is achieved through inheritance and interfaces.

Key Concepts of Polymorphism

  • Inheritance: Allows a class to inherit properties and methods from another class.
  • Method Overriding: Enables a subclass to provide a specific implementation of a method that is already defined in its superclass.
  • Interfaces: Define methods that must be implemented by any class that implements the interface, allowing different classes to be treated uniformly.

Inheritance and Method Overriding

Inheritance is the mechanism by which one class (the subclass) can inherit the properties and methods of another class (the superclass). Method overriding allows the subclass to provide a specific implementation of a method that is already defined in its superclass.

Example of Inheritance and Method Overriding

program PolymorphismExample;

{$APPTYPE CONSOLE}

uses
  SysUtils;

type
  TAnimal = class
  public
    procedure Speak; virtual;
  end;

  TDog = class(TAnimal)
  public
    procedure Speak; override;
  end;

  TCat = class(TAnimal)
  public
    procedure Speak; override;
  end;

procedure TAnimal.Speak;
begin
  WriteLn('The animal makes a sound.');
end;

procedure TDog.Speak;
begin
  WriteLn('The dog barks.');
end;

procedure TCat.Speak;
begin
  WriteLn('The cat meows.');
end;

var
  Animal: TAnimal;

begin
  Animal := TDog.Create;
  Animal.Speak; // Outputs: The dog barks.

  Animal := TCat.Create;
  Animal.Speak; // Outputs: The cat meows.

  ReadLn;
end.

In this example:

  • TAnimal is the base class with a virtual method Speak.
  • TDog and TCat are subclasses that override the Speak method.
  • The Animal variable can hold references to TDog and TCat objects, and the correct Speak method is called based on the actual object type.

Interfaces

Interfaces in Delphi/Object Pascal provide a way to define methods that must be implemented by any class that implements the interface. This allows different classes to be treated uniformly.

Example of Interfaces

program InterfaceExample;

{$APPTYPE CONSOLE}

uses
  SysUtils;

type
  ISpeakable = interface
    ['{8D3E4F5A-6B7C-4D8E-9F1A-2B3C4D5E6F7A}']
    procedure Speak;
  end;

  TDog = class(TInterfacedObject, ISpeakable)
  public
    procedure Speak;
  end;

  TCat = class(TInterfacedObject, ISpeakable)
  public
    procedure Speak;
  end;

procedure TDog.Speak;
begin
  WriteLn('The dog barks.');
end;

procedure TCat.Speak;
begin
  WriteLn('The cat meows.');
end;

var
  Speakable: ISpeakable;

begin
  Speakable := TDog.Create;
  Speakable.Speak; // Outputs: The dog barks.

  Speakable := TCat.Create;
  Speakable.Speak; // Outputs: The cat meows.

  ReadLn;
end.

In this example:

  • ISpeakable is an interface with a Speak method.
  • TDog and TCat implement the ISpeakable interface.
  • The Speakable variable can hold references to any object that implements ISpeakable, and the correct Speak method is called based on the actual object type.

Summary

Polymorphism in Delphi/Object Pascal allows for flexible and reusable code by enabling objects to be treated as instances of their parent class or interface. Key concepts include inheritance, method overriding, and interfaces. By understanding and utilizing polymorphism, you can write more maintainable and scalable applications.

  • Inheritance allows classes to inherit properties and methods from other classes.
  • Method Overriding enables subclasses to provide specific implementations of methods defined in their superclasses.
  • Interfaces define methods that must be implemented by any class that implements the interface, allowing for uniform treatment of different classes.

By mastering these concepts, you can leverage the power of polymorphism to create more dynamic and flexible applications in Delphi/Object Pascal.
