Scope and Lifetime of Variables in Delphi/Object Pascal
Recursion ►

Understanding the scope and lifetime of variables is crucial for writing efficient and error-free code in Delphi/Object Pascal. This topic will cover the basic concepts, provide examples, and include exercises to reinforce learning.

Scope of Variables

The scope of a variable refers to the region of the code where the variable is accessible. In Delphi/Object Pascal, there are several types of scope:

  • Local Scope
  • Global Scope
  • Unit Scope
  • Class Scope

Local Scope

Variables declared within a procedure, function, or method have local scope. They are only accessible within that block of code.

procedure ExampleProcedure;
var
  LocalVar: Integer; // Local variable
begin
  LocalVar := 10;
  WriteLn(LocalVar); // Output: 10
end;

Global Scope

Variables declared outside any procedure, function, or method have global scope. They are accessible from any part of the program.

var
  GlobalVar: Integer; // Global variable

procedure ExampleProcedure;
begin
  GlobalVar := 20;
  WriteLn(GlobalVar); // Output: 20
end;

begin
  GlobalVar := 10;
  ExampleProcedure;
  WriteLn(GlobalVar); // Output: 20
end.

Unit Scope

Variables declared in the interface section of a unit are accessible from other units that use this unit. Variables in the implementation section are only accessible within the same unit.

unit ExampleUnit;

interface

var
  UnitVar: Integer; // Unit scope variable

implementation

procedure ExampleProcedure;
begin
  UnitVar := 30;
  WriteLn(UnitVar); // Output: 30
end;

end.

Class Scope

Variables declared within a class are accessible to all methods of that class. These are also known as fields or member variables.

type
  TExampleClass = class
  private
    FClassVar: Integer; // Class scope variable
  public
    procedure SetVar(Value: Integer);
    procedure ShowVar;
  end;

procedure TExampleClass.SetVar(Value: Integer);
begin
  FClassVar := Value;
end;

procedure TExampleClass.ShowVar;
begin
  WriteLn(FClassVar);
end;

var
  Example: TExampleClass;

begin
  Example := TExampleClass.Create;
  Example.SetVar(40);
  Example.ShowVar; // Output: 40
  Example.Free;
end.

Lifetime of Variables

The lifetime of a variable refers to the duration for which the variable exists in memory. In Delphi/Object Pascal, the lifetime of variables depends on their scope.

Local Variables

Local variables are created when the procedure, function, or method is called and destroyed when it exits.

procedure ExampleProcedure;
var
  LocalVar: Integer;
begin
  LocalVar := 50;
  WriteLn(LocalVar); // Output: 50
end;

begin
  ExampleProcedure;
  // LocalVar is no longer accessible here
end.

Global Variables

Global variables exist for the entire duration of the program.

var
  GlobalVar: Integer;

begin
  GlobalVar := 60;
  WriteLn(GlobalVar); // Output: 60
  // GlobalVar exists until the program terminates
end.

Unit Variables

Unit variables have a lifetime that spans the duration of the program, similar to global variables.

unit ExampleUnit;

interface

var
  UnitVar: Integer;

implementation

procedure ExampleProcedure;
begin
  UnitVar := 70;
  WriteLn(UnitVar); // Output: 70
end;

end.

Class Variables

Class variables (fields) exist as long as the instance of the class exists. They are created when the instance is created and destroyed when the instance is freed.

type
  TExampleClass = class
  private
    FClassVar: Integer;
  public
    procedure SetVar(Value: Integer);
    procedure ShowVar;
  end;

procedure TExampleClass.SetVar(Value: Integer);
begin
  FClassVar := Value;
end;

procedure TExampleClass.ShowVar;
begin
  WriteLn(FClassVar);
end;

var
  Example: TExampleClass;

begin
  Example := TExampleClass.Create;
  Example.SetVar(80);
  Example.ShowVar; // Output: 80
  Example.Free;
  // FClassVar is no longer accessible here
end.

Conclusion

Understanding the scope and lifetime of variables is essential for managing memory and ensuring that your code behaves as expected. Here's a quick summary:

  • Local Scope: Variables are accessible only within the block where they are declared and exist only during the execution of that block.
  • Global Scope: Variables are accessible from anywhere in the program and exist for the program's entire duration.
  • Unit Scope: Variables are accessible within the unit and, if declared in the interface section, from other units as well.
  • Class Scope: Variables are accessible to all methods of the class and exist as long as the class instance exists.

By mastering these concepts, you can write more efficient and maintainable Delphi/Object Pascal code.
