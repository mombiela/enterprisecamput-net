Introduction to Interfaces

Interfaces in Delphi/Object Pascal provide a way to define methods that must be implemented by any class that supports the interface. They are similar to abstract classes but are more flexible and can be used to achieve multiple inheritance.

Key Concepts

  • Definition: An interface is a reference type that defines a set of methods without implementing them.
  • Implementation: Classes that implement an interface must provide concrete implementations for all its methods.
  • Multiple Inheritance: A class can implement multiple interfaces, allowing for a form of multiple inheritance.

Defining Interfaces

To define an interface in Delphi/Object Pascal, use the interface keyword followed by the method signatures.

type
  IShape = interface
    ['{8D5B3E1A-2B9D-4B6A-8D3E-1A2B3C4D5E6F}']
    function GetArea: Double;
    function GetPerimeter: Double;
  end;

Explanation

  • GUID: The string in square brackets is a globally unique identifier (GUID) that uniquely identifies the interface.
  • Method Signatures: The interface declares methods GetArea and GetPerimeter without providing implementations.

Implementing Interfaces

A class that implements an interface must provide concrete implementations for all the methods declared in the interface.

type
  TCircle = class(TInterfacedObject, IShape)
  private
    FRadius: Double;
  public
    constructor Create(ARadius: Double);
    function GetArea: Double;
    function GetPerimeter: Double;
  end;

constructor TCircle.Create(ARadius: Double);
begin
  FRadius := ARadius;
end;

function TCircle.GetArea: Double;
begin
  Result := Pi * FRadius * FRadius;
end;

function TCircle.GetPerimeter: Double;
begin
  Result := 2 * Pi * FRadius;
end;

Explanation

  • TInterfacedObject: This is a base class that provides reference counting and other features needed for interface management.
  • Constructor: Initializes the radius of the circle.
  • GetArea and GetPerimeter: Provide concrete implementations for the methods declared in the IShape interface.

Using Interfaces

Once an interface is implemented, you can use it to interact with objects in a polymorphic way.

var
  Shape: IShape;
  Circle: TCircle;
begin
  Circle := TCircle.Create(5.0);
  Shape := Circle;
  Writeln('Area: ', Shape.GetArea:0:2);
  Writeln('Perimeter: ', Shape.GetPerimeter:0:2);
end;

Explanation

  • Polymorphism: The variable Shape of type IShape can hold any object that implements the IShape interface.
  • Method Calls: Methods GetArea and GetPerimeter are called on the Shape interface reference.

Advanced Topics

Interface Inheritance

Interfaces can inherit from other interfaces, allowing for more complex hierarchies.

type
  IColoredShape = interface(IShape)
    function GetColor: TColor;
  end;

Explanation

  • Inheritance: IColoredShape inherits from IShape and adds an additional method GetColor.

Reference Counting and Memory Management

Delphi uses reference counting to manage the lifetime of interface instances automatically.

  • Reference Counting: Each interface reference increases the reference count, and when the count drops to zero, the object is automatically destroyed.
  • TInterfacedObject: Provides the necessary implementation for reference counting.

Conclusion

Interfaces in Delphi/Object Pascal are a powerful feature that allows for flexible and reusable code. They enable polymorphism, multiple inheritance, and clean separation of method definitions from implementations. By understanding and using interfaces, you can create more modular and maintainable applications.
