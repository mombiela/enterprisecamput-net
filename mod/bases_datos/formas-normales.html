Introduction to Normal Forms

Normal forms are a set of rules used in the design of relational databases to minimize redundancy and avoid inconsistency problems. These rules help structure the tables of a database in a way that optimizes storage and facilitates data maintenance.

First Normal Form (1NF)

The First Normal Form (1NF) states that a table is in 1NF if:

  • All attributes contain atomic (indivisible) values.
  • Each value in a column is of a single type.
  • Each column must have a unique name.
  • The order in which data is stored should not affect the integrity of the database.

Example

Suppose we have a table of students with the following data:

Student_ID Name Courses
1 Juan Pérez Mathematics, Physics
2 Ana Gómez Chemistry, Biology

To normalize this table to 1NF, we must ensure that each column contains atomic values:

Student_ID Name Course
1 Juan Pérez Mathematics
1 Juan Pérez Physics
2 Ana Gómez Chemistry
2 Ana Gómez Biology

Second Normal Form (2NF)

The Second Normal Form (2NF) requires that a table be in 1NF and that all non-key attributes are fully dependent on the primary key. This means there should be no partial dependencies of the primary key.

Example

Consider the following table that is already in 1NF:

Student_ID Name Course_ID Course
1 Juan Pérez 101 Mathematics
1 Juan Pérez 102 Physics
2 Ana Gómez 103 Chemistry
2 Ana Gómez 104 Biology

To normalize this table to 2NF, we must eliminate partial dependencies. We can divide the table into two:

Students Table:

Student_ID Name
1 Juan Pérez
2 Ana Gómez

Courses Table:

Course_ID Course
101 Mathematics
102 Physics
103 Chemistry
104 Biology

Students_Courses Table:

Student_ID Course_ID
1 101
1 102
2 103
2 104

Third Normal Form (3NF)

The Third Normal Form (3NF) requires that a table be in 2NF and that there are no transitive dependencies. That is, non-key attributes must depend solely on the primary key.

Example

Consider the following table that is already in 2NF:

Student_ID Name Course_ID Course Professor
1 Juan Pérez 101 Mathematics Dr. Smith
1 Juan Pérez 102 Physics Dr. Johnson
2 Ana Gómez 103 Chemistry Dr. Brown
2 Ana Gómez 104 Biology Dr. White

To normalize this table to 3NF, we must eliminate transitive dependencies. We can divide the table into three:

Students Table:

Student_ID Name
1 Juan Pérez
2 Ana Gómez

Courses Table:

Course_ID Course Professor
101 Mathematics Dr. Smith
102 Physics Dr. Johnson
103 Chemistry Dr. Brown
104 Biology Dr. White

Students_Courses Table:

Student_ID Course_ID
1 101
1 102
2 103
2 104

Conclusion

Normal forms are essential for designing efficient and redundancy-free databases. By following the rules of 1NF, 2NF, and 3NF, we can ensure that our tables are well-structured, which facilitates maintenance and improves database performance. As we progress in the course, we will explore more advanced normal forms and additional techniques to optimize our databases.
