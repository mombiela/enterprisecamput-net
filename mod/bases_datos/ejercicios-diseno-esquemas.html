Introduction

In this section of the Database Fundamentals course, we will focus on schema design exercises. Schema design is a crucial part of creating efficient and well-structured databases. Through these exercises, you will learn to apply the theoretical principles of database design in practical situations.

Key Concepts

  • Database Schema: The structure that defines the organization of data in a database.
  • Entity-Relationship (ER): A model to describe data and their relationships.
  • Normalization: The process of organizing data to reduce redundancy and improve integrity.
  • Primary and Foreign Keys: Unique identifiers for records and relationships between tables.

Practical Exercises

Exercise 1: Creating an ER Diagram

Description: Design an ER diagram for a library database. The database should include information about books, authors, members, and loans.

Requirements:

  • Each book has a title, an ISBN, and can have multiple authors.
  • Each author has a name and a unique ID.
  • Members have a name, a unique ID, and an address.
  • Loans record which book was loaned to which member and the date of the loan.

Solution:

  1. Identify Entities:

    • Book
    • Author
    • Member
    • Loan

  2. Define Relationships:

    • A book can have multiple authors (many-to-many relationship).
    • An author can write multiple books (many-to-many relationship).
    • A member can make multiple loans (one-to-many relationship).
    • A loan is associated with a book and a member (many-to-one relationship).

  3. ER Diagram:

[Book] --< Written by >-- [Author]
[Book] --< Loaned to >-- [Member]
[Loan] --< Related to >-- [Book]
[Loan] --< Made by >-- [Member]

Exercise 2: Normalizing a Table

Description: Normalize the following table to the Third Normal Form (3NF).

Initial Table:

ID Customer Name Customer Address Product Quantity Price
1 Juan Pérez Calle Falsa 123 Laptop 1 1000
2 Ana Gómez Calle Real 456 Mouse 2 20
3 Juan Pérez Calle Falsa 123 Keyboard 1 50

Normalization Process:

  1. First Normal Form (1NF):

    • Ensure each column contains only atomic values and each record is unique.

  2. Second Normal Form (2NF):

    • Eliminate partial dependencies. Create separate tables for distinct entities.

  3. Third Normal Form (3NF):

    • Eliminate transitive dependencies. Ensure there are no non-key dependencies.

Normalized Tables:

  • Customers:
Customer ID Customer Name Customer Address
1 Juan Pérez Calle Falsa 123
2 Ana Gómez Calle Real 456
  • Products:
Product ID Product Price
1 Laptop 1000
2 Mouse 20
3 Keyboard 50
  • Orders:
Order ID Customer ID Product ID Quantity
1 1 1 1
2 2 2 2
3 1 3 1

Conclusion

Schema design is fundamental for creating efficient and well-structured databases. Through these exercises, we have explored how to design ER diagrams and how to normalize tables to improve integrity and reduce redundancy. Practicing these exercises will help you master the key concepts and apply them in real-world situations.
