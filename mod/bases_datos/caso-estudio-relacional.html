Introduction

In this case study, we will explore how to design and manage a relational database for a small e-commerce business. This exercise will help you understand the fundamental concepts of relational databases and how to apply them in a real-world scenario.

System Requirements

Before starting the database design, it is crucial to understand the system requirements. The e-commerce company needs:

  • Product Management: Information about the products they sell, including name, description, price, and stock.
  • Customer Management: Information about customers, including name, address, email, and phone number.
  • Order Management: Information about orders placed by customers, including order date, purchased products, and quantity.
  • Payment Management: Information about payments made by customers, including payment method and payment status.

Database Design

Entities and Relationships

To meet the system requirements, we identify the following entities and their relationships:

  • Product: Represents the items being sold.
  • Customer: Represents the buyers.
  • Order: Represents purchase orders placed by customers.
  • Payment: Represents payments made by customers.

The relationships between these entities are as follows:

  • A customer can place multiple orders.
  • An order can include multiple products.
  • An order can have an associated payment.

Database Schema

Below is the database schema with tables and their relationships:

Table Columns
Product id_product (PK), name, description, price, stock
Customer id_customer (PK), name, address, email, phone
Order id_order (PK), id_customer (FK), order_date
Order_Product id_order (FK), id_product (FK), quantity
Payment id_payment (PK), id_order (FK), payment_method, payment_status

Implementation in SQL

Below is how to create the tables in SQL:

CREATE TABLE Product (
    id_product INT PRIMARY KEY,
    name VARCHAR(100),
    description TEXT,
    price DECIMAL(10, 2),
    stock INT
);

CREATE TABLE Customer (
    id_customer INT PRIMARY KEY,
    name VARCHAR(100),
    address TEXT,
    email VARCHAR(100),
    phone VARCHAR(15)
);

CREATE TABLE Order (
    id_order INT PRIMARY KEY,
    id_customer INT,
    order_date DATE,
    FOREIGN KEY (id_customer) REFERENCES Customer(id_customer)
);

CREATE TABLE Order_Product (
    id_order INT,
    id_product INT,
    quantity INT,
    PRIMARY KEY (id_order, id_product),
    FOREIGN KEY (id_order) REFERENCES Order(id_order),
    FOREIGN KEY (id_product) REFERENCES Product(id_product)
);

CREATE TABLE Payment (
    id_payment INT PRIMARY KEY,
    id_order INT,
    payment_method VARCHAR(50),
    payment_status VARCHAR(50),
    FOREIGN KEY (id_order) REFERENCES Order(id_order)
);

INSERT INTO Order_Product (id_order, id_product, quantity) VALUES
(1, 1, 1),
(1, 2, 2);

INSERT INTO Payment (id_payment, id_order, payment_method, payment_status) VALUES
(1, 1, 'Credit Card', 'Completed');

Query Data

-- Get all products
SELECT * FROM Product;

-- Get all orders of a specific customer
SELECT * FROM Order WHERE id_customer = 1;

-- Get details of a specific order
SELECT p.name, op.quantity
FROM Order_Product op
JOIN Product p ON op.id_product = p.id_product
WHERE op.id_order = 1;
