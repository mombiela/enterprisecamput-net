Introduction

The theoretical exam is a crucial part of the course on Database Fundamentals. This exam will assess your understanding of key concepts, structures, and essential practices in database management. Below are the topics that will be covered in the exam, along with examples and exercises to help you prepare effectively.

Key Topics

1. Basic Database Concepts

  • Definition of Database: A set of data organized in a way that allows easy access, management, and updating.
  • Database Management Systems (DBMS): Software that enables the creation, management, and manipulation of databases.
  • Types of Databases: Relational, NoSQL, hierarchical, network, etc.

2. Data Models

  • Relational Model: Based on tables (relations) that contain rows and columns.
  • Entity-Relationship (ER) Model: Uses entities and relationships to represent data and their interactions.
  • NoSQL Model: Includes document, key-value, column, and graph databases.

3. Structured Query Language (SQL)

  • Basic Queries: SELECT, INSERT, UPDATE, DELETE.
  • Advanced Queries: JOIN, GROUP BY, HAVING, subqueries.
  • Data Definition: CREATE, ALTER, DROP.

SQL Code Example

-- Create a table
CREATE TABLE employees (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    position VARCHAR(50),
    salary DECIMAL(10, 2)
);

-- Insert data into the table
INSERT INTO employees (id, name, position, salary) VALUES
(1, 'John Doe', 'Developer', 50000.00),
(2, 'Anna Smith', 'Analyst', 55000.00);

-- Query data from the table
SELECT * FROM employees WHERE salary > 50000;

4. Normalization

  • First Normal Form (1NF): Eliminate duplicates and ensure each column contains atomic values.
  • Second Normal Form (2NF): Eliminate partial dependencies.
  • Third Normal Form (3NF): Eliminate transitive dependencies.

5. Transactions and Concurrency

  • ACID: Atomicity, Consistency, Isolation, Durability.
  • Concurrency Control: Methods to ensure multiple transactions can occur simultaneously without causing inconsistencies.

6. Database Security

  • Authentication and Authorization: Access control to data.
  • Encryption: Protection of sensitive data.
  • Auditing: Logging activities and changes in the database.

Practical Exercises

  1. Create and Query Tables:

    • Create a table called clients with columns for id, name, email, and registration_date.
    • Insert at least three records into the table.
    • Perform a query to get all clients registered in the last month.

  2. Normalization:

    • Given a table orders with columns order_id, client, product, quantity, total_price, normalize the table up to 3NF.

  3. Transactions:

    • Write a transaction that inserts a new order into the orders table and updates the inventory in the products table. Ensure the transaction is atomic.

Conclusion

The theoretical exam on Database Fundamentals will assess your understanding of essential concepts and your ability to apply this knowledge in practical situations. Make sure to review each of the key topics, practice with code examples, and solve the proposed exercises. This will prepare you to face the exam with confidence and success.
