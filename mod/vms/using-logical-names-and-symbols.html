Introduction

In OpenVMS, logical names and symbols are essential tools for managing system resources and simplifying command procedures. This topic will guide you through the basics of logical names and symbols, progressing to more advanced usage.

Logical Names

What are Logical Names?

  • Logical names are aliases for file specifications, device names, or other strings.
  • They simplify command procedures and scripts by providing a shorthand reference.

Creating Logical Names

  • Use the DEFINE command to create a logical name.
  • Syntax: DEFINE logical_name equivalence_name
$ DEFINE MYDISK DKA0:[USER.DIRECTORY]
  • In this example, MYDISK is a logical name for the directory DKA0:[USER.DIRECTORY].

Using Logical Names

  • Logical names can be used in place of their equivalence names in commands.
$ DIRECTORY MYDISK
  • This command lists the contents of DKA0:[USER.DIRECTORY].

Types of Logical Names

  • Process Logical Names: Valid only within the current process.
  • Job Logical Names: Valid for all processes in the current job.
  • Group Logical Names: Valid for all processes in the current group.
  • System Logical Names: Valid for all processes on the system.

Managing Logical Names

  • Use the SHOW LOGICAL command to display logical names.
$ SHOW LOGICAL MYDISK
  • Use the DEASSIGN command to remove a logical name.
$ DEASSIGN MYDISK

Symbols

What are Symbols?

  • Symbols are user-defined names that represent commands or command sequences.
  • They are similar to variables in other programming languages.

Creating Symbols

  • Use the = operator to define a symbol.
  • Syntax: symbol_name = value
$ LISTDIR = "DIRECTORY"
  • In this example, LISTDIR is a symbol for the DIRECTORY command.

Using Symbols

  • Symbols can be used in place of their defined values in commands.
$ LISTDIR MYDISK
  • This command is equivalent to DIRECTORY MYDISK.

Local vs. Global Symbols

  • Local Symbols: Valid only within the current command procedure.
  • Global Symbols: Valid across all command procedures in the current process.

Managing Symbols

  • Use the SHOW SYMBOL command to display symbols.
$ SHOW SYMBOL LISTDIR
  • Use the DELETE command to remove a symbol.
$ DELETE SYMBOL LISTDIR

Advanced Usage

Logical Name Tables

  • Logical names are stored in logical name tables.
  • Common tables include LNM$PROCESS_TABLE, LNM$JOB_TABLE, LNM$GROUP_TABLE, and LNM$SYSTEM_TABLE.

Logical Name Translation

  • Use the F$TRNLNM lexical function to translate logical names in command procedures.
$ MYDIR = F$TRNLNM("MYDISK")
$ DIRECTORY 'MYDIR'

Symbol Substitution

  • Use single quotes (') to substitute symbols within command procedures.
$ MYCOMMAND = "SHOW TIME"
$ 'MYCOMMAND'

Combining Logical Names and Symbols

  • Logical names and symbols can be combined for more complex command procedures.
$ DEFINE MYDISK DKA0:[USER.DIRECTORY]
$ LISTDIR = "DIRECTORY"
$ 'LISTDIR' MYDISK

Conclusion

Logical names and symbols are powerful tools in OpenVMS that help streamline command procedures and manage system resources efficiently. By understanding and utilizing these features, you can create more readable, maintainable, and efficient scripts. Practice creating and using logical names and symbols to become proficient in their application.
