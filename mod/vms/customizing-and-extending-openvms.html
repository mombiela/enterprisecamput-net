Introduction

OpenVMS is a versatile and powerful operating system used in various enterprise environments. Customizing and extending OpenVMS allows users to tailor the system to their specific needs, enhancing functionality and improving efficiency. This section will guide you through the basics of customization and extension, from beginner to advanced levels.

Basic Customization

System Parameters

  • SYSGEN Utility: Used to modify system parameters.
  • MODPARAMS.DAT: A file where parameter changes are recorded.

Example:

$ RUN SYS$SYSTEM:SYSGEN
SYSGEN> USE CURRENT
SYSGEN> SET MAXPROCESSCNT 256
SYSGEN> WRITE CURRENT
SYSGEN> EXIT

Explanation: This example shows how to use the SYSGEN utility to set the MAXPROCESSCNT parameter to 256.

Command Procedures

  • DCL (Digital Command Language): Used to write command procedures.
  • LOGIN.COM: A script that runs at user login.

Example:

$! LOGIN.COM
$ SET PROMPT="User>"
$ SHOW TIME

Explanation: This script sets a custom prompt and displays the current time when a user logs in.

Intermediate Customization

Logical Names

  • Defining Logical Names: Used to create aliases for files, devices, or directories.
  • System-Wide Logical Names: Defined in SYLOGICALS.COM.

Example:

$ DEFINE MYDISK DKA0:[USERFILES]
$ SHOW LOGICAL MYDISK

Explanation: This example defines a logical name MYDISK for the directory DKA0:[USERFILES] and then displays it.

User Environment Customization

  • SYLOGIN.COM: A system-wide login script.
  • Customizing User Profiles: Modify user-specific settings.

Example:

$! SYLOGIN.COM
$ SET TERMINAL /INQUIRE
$ DEFINE SYS$SCRATCH DKA0:[SCRATCH]

Explanation: This script sets terminal characteristics and defines a scratch directory for temporary files.

Advanced Customization

Writing System Services

  • System Service Calls: Extend OpenVMS functionality by writing custom services.
  • Using LIB$ and SYS$ Libraries: Libraries that provide system service routines.

Example:

#include <lib$routines.h>
#include <ssdef.h>

int main() {
    int status;
    status = lib$signal(SS$_NORMAL);
    return status;
}

Explanation: This C program uses the lib$signal routine to send a normal completion status.

Developing Custom Utilities

  • Writing Custom Utilities: Create utilities to automate tasks.
  • Using VMS System Routines: Leverage built-in routines for utility development.

Example:

$! CUSTOM_UTILITY.COM
$! This script automates backup tasks
$ BACKUP DKA0:[USERFILES]*.*;* MKA500:[BACKUP] /LOG
$ EXIT

Explanation: This command procedure automates the backup of user files to a tape drive.

Extending OpenVMS

Installing Third-Party Software

  • VMSINSTAL Utility: Used to install third-party software.
  • Managing Software Patches: Apply patches to extend functionality.

Example:

$ @SYS$UPDATE:VMSINSTAL MYAPP032 DKA0:[KITS]

Explanation: This command installs the MYAPP software from the directory DKA0:[KITS].

Developing Custom Applications

  • Using DECwindows: Develop graphical applications.
  • Integrating with Other Systems: Use network protocols and APIs.

Example:

#include <decw$include:Xm.h>

int main(int argc, char **argv) {
    XtAppContext app;
    Widget top_wid;

    top_wid = XtVaAppInitialize(&app, "HelloWorld", NULL, 0, &argc, argv, NULL, NULL);
    XtRealizeWidget(top_wid);
    XtAppMainLoop(app);
    return 0;
}

Explanation: This C program initializes a basic DECwindows application.

Conclusion

Customizing and extending OpenVMS can significantly enhance its capabilities and tailor it to specific needs. By understanding system parameters, command procedures, logical names, and developing custom utilities and applications, users can fully leverage the power of OpenVMS. Whether you are a beginner or an advanced user, these skills are essential for optimizing your OpenVMS environment.
