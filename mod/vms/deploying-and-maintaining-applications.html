Introduction

Deploying and maintaining applications on OpenVMS requires a solid understanding of the operating system's unique features and tools. This section will guide you through the process from a beginner to an advanced level, covering key concepts, practical examples, and best practices.

Understanding OpenVMS Environment

Before diving into deployment, it's essential to understand the OpenVMS environment:

  • DCL (Digital Command Language): The command language used in OpenVMS.
  • File System: OpenVMS uses a hierarchical file system.
  • User Accounts and Privileges: Managing user permissions is crucial for secure deployment.

Preparing the Application for Deployment

Compiling and Linking

  • Compiling: Convert source code into object code.
  • Linking: Combine object code with libraries to create an executable.

Example:

$ FORTRAN myprogram.for
$ LINK myprogram

Explanation:

  • FORTRAN myprogram.for compiles the Fortran source file.
  • LINK myprogram links the compiled object file to create an executable.

Creating Command Procedures

  • Command Procedures: Scripts written in DCL to automate tasks.
  • Example Command Procedure:
$! This is a sample command procedure
$ SET DEFAULT SYS$USER:[MYDIR]
$ RUN MYPROGRAM.EXE

Explanation:

  • SET DEFAULT SYS$USER:[MYDIR] changes the working directory.
  • RUN MYPROGRAM.EXE executes the program.

Deploying the Application

Copying Files

  • Using COPY Command: Transfer files to the target directory.
$ COPY MYPROGRAM.EXE SYS$USER:[TARGETDIR]

Explanation:

  • COPY MYPROGRAM.EXE SYS$USER:[TARGETDIR] copies the executable to the target directory.

Setting Up Logical Names

  • Logical Names: Aliases for file specifications or devices.
$ DEFINE MYAPP SYS$USER:[TARGETDIR]MYPROGRAM.EXE

Explanation:

  • DEFINE MYAPP SYS$USER:[TARGETDIR]MYPROGRAM.EXE creates a logical name MYAPP for the executable.

Managing User Access

  • Granting Access: Use AUTHORIZE utility to manage user permissions.
$ RUN SYS$SYSTEM:AUTHORIZE
UAF> ADD USERNAME /PASSWORD=secret /DEVICE=SYS$USER /DIRECTORY=[USERNAME]
UAF> EXIT

Explanation:

  • ADD USERNAME /PASSWORD=secret /DEVICE=SYS$USER /DIRECTORY=[USERNAME] creates a new user with specified attributes.

Maintaining the Application

Monitoring Performance

  • Using MONITOR Utility: Track system performance.
$ MONITOR SYSTEM

Explanation:

  • MONITOR SYSTEM displays system performance metrics.

Updating the Application

  • Replacing Executables: Use RENAME and COPY to update files.
$ RENAME SYS$USER:[TARGETDIR]MYPROGRAM.EXE MYPROGRAM_OLD.EXE
$ COPY NEW_MYPROGRAM.EXE SYS$USER:[TARGETDIR]MYPROGRAM.EXE

Explanation:

  • RENAME renames the old executable.
  • COPY places the new executable in the target directory.

Backup and Recovery

  • Using BACKUP Utility: Create backups of important files.
$ BACKUP SYS$USER:[TARGETDIR]*.*;* SYS$USER:[BACKUPDIR]

Explanation:

  • BACKUP SYS$USER:[TARGETDIR]*.*;* SYS$USER:[BACKUPDIR] backs up all files in the target directory to the backup directory.

Conclusion

Deploying and maintaining applications on OpenVMS involves understanding the environment, preparing the application, deploying it correctly, and maintaining it through monitoring, updating, and backups. By following these steps and utilizing the provided examples, you can ensure a smooth deployment and maintenance process for your applications on OpenVMS.
