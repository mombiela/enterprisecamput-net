Introduction

Backup and recovery are critical components of system administration on OpenVMS. This topic will cover the fundamental concepts, tools, and best practices for performing backups and recovering data on OpenVMS systems. By the end of this section, you will understand how to safeguard your data and ensure system reliability.

Key Concepts

  • Backup: The process of copying data to a separate location to protect it from loss or corruption.
  • Recovery: The process of restoring data from a backup after data loss or corruption.
  • Incremental Backup: A backup that only includes the data that has changed since the last backup.
  • Full Backup: A complete copy of all data at a specific point in time.
  • Differential Backup: A backup that includes all changes made since the last full backup.

Backup Strategies

Full Backup

A full backup involves copying all data from the system. This type of backup is comprehensive but can be time-consuming and requires significant storage space.

Incremental Backup

Incremental backups save only the data that has changed since the last backup (either full or incremental). This method is faster and requires less storage space but can be more complex to restore.

Differential Backup

Differential backups save all changes made since the last full backup. This method strikes a balance between full and incremental backups, offering faster backups than full backups and simpler restores than incremental backups.

Tools for Backup and Recovery

OpenVMS provides several tools for performing backups and recovery. The primary tool is the BACKUP utility.

BACKUP Utility

The BACKUP utility is a powerful tool for creating and managing backups on OpenVMS. It supports various backup strategies and can be used to back up files, directories, and entire disks.

Basic Syntax

$ BACKUP source destination

Example: Full Backup

$ BACKUP DKA0:[000000...] MKA500:FULL_BACKUP.BCK/SAVE_SET
  • DKA0:[000000...]: Specifies the source directory (entire disk).
  • MKA500:FULL_BACKUP.BCK: Specifies the destination save set.
  • /SAVE_SET: Indicates that the backup should be saved as a save set.

Example: Incremental Backup

$ BACKUP/INCREMENTAL DKA0:[000000...] MKA500:INCREMENTAL_BACKUP.BCK/SAVE_SET
  • /INCREMENTAL: Indicates that only changed files since the last backup should be included.

Recovery Procedures

Recovery involves restoring data from a backup save set. The BACKUP utility is also used for this purpose.

Basic Syntax

$ BACKUP save_set_specification destination

Example: Restore Full Backup

$ BACKUP MKA500:FULL_BACKUP.BCK/SAVE_SET DKA0:[000000...]
  • MKA500:FULL_BACKUP.BCK: Specifies the source save set.
  • DKA0:[000000...]: Specifies the destination directory (entire disk).

Example: Restore Incremental Backup

Restoring an incremental backup typically involves first restoring the full backup and then applying the incremental backups in sequence.

$ BACKUP MKA500:FULL_BACKUP.BCK/SAVE_SET DKA0:[000000...]
$ BACKUP MKA500:INCREMENTAL_BACKUP.BCK/SAVE_SET DKA0:[000000...]

Best Practices

  • Regular Backups: Schedule regular backups to minimize data loss.
  • Test Restores: Periodically test your recovery procedures to ensure backups are valid and can be restored.
  • Offsite Storage: Store backups in a separate physical location to protect against site-specific disasters.
  • Documentation: Maintain detailed documentation of your backup and recovery procedures.

Conclusion

Backup and recovery are essential practices for maintaining data integrity and system reliability on OpenVMS. By understanding the different types of backups, using the BACKUP utility effectively, and following best practices, you can ensure that your data is protected and can be restored in the event of a failure. Regularly review and update your backup strategies to adapt to changing data and system requirements.
