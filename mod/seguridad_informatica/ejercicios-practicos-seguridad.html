Introduction

In this section, we will focus on applying computer security concepts through practical exercises. These exercises are designed to reinforce theoretical knowledge and provide practical experience in identifying and mitigating vulnerabilities.

Exercise 1: Vulnerability Analysis

Objective

Identify and analyze vulnerabilities in a web application using security scanning tools.

Recommended Tools

  • OWASP ZAP: A free and open-source security scanning tool.
  • Nmap: Used for network and port scanning.

Steps to Follow

  1. Installation of OWASP ZAP:

    • Download and install OWASP ZAP from OWASP ZAP.

  2. Scanning a Web Application:

    • Open OWASP ZAP.
    • Enter the URL of the web application you want to scan.
    • Start the scan and wait for it to complete.

  3. Analysis of Results:

    • Review the scan results.
    • Identify the vulnerabilities found.
    • Classify the vulnerabilities according to their severity (high, medium, low).

Code Example

# Installation of Nmap on a Debian-based distribution
sudo apt-get install nmap

# Scanning for open ports on an IP address
nmap -sS 192.168.1.1

Tasks

  • Perform a full scan of a test web application.
  • Document the vulnerabilities found and their classification.
  • Propose possible solutions to mitigate the identified vulnerabilities.

Exercise 2: Implementation of Security Policies

Objective

Develop and implement security policies in a network environment.

Recommended Tools

  • Firewall: Configuration of firewall rules to control network traffic.
  • IDS/IPS: Intrusion detection and prevention systems.

Steps to Follow

  1. Configuration of a Firewall:

    • Define the necessary firewall rules to protect the network.
    • Implement the rules using iptables on Linux.

  2. Implementation of IDS/IPS:

    • Install and configure Snort as IDS/IPS.
    • Define detection and prevention rules.

Code Example

# Basic iptables configuration to allow only HTTP and HTTPS traffic
sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT
sudo iptables -A INPUT -j DROP

# Installation of Snort on a Debian-based distribution
sudo apt-get install snort

# Basic Snort configuration
sudo snort -c /etc/snort/snort.conf -i eth0

Tasks

  • Configure a firewall on a Linux server to allow only HTTP and HTTPS traffic.
  • Install and configure Snort to monitor network traffic.
  • Document the implemented rules and their purpose.

Exercise 3: Data Encryption

Objective

Implement encryption techniques to protect sensitive data.

Recommended Tools

  • OpenSSL: Command-line tool for encryption operations.
  • GPG: Tool for data encryption and signing.

Steps to Follow

  1. File Encryption with OpenSSL:

    • Use OpenSSL to encrypt and decrypt files.

  2. Message Encryption with GPG:

    • Generate a pair of GPG keys.
    • Encrypt and decrypt a message using GPG.

Code Example

# Encrypting a file with OpenSSL
openssl enc -aes-256-cbc -salt -in file.txt -out file.enc

# Decrypting a file with OpenSSL
openssl enc -d -aes-256-cbc -in file.enc -out file.txt

# Generating a pair of GPG keys
gpg --gen-key

# Encrypting a message with GPG
echo "Secret message" | gpg --encrypt --recipient 'name@example.com' > message.gpg

# Decrypting a message with GPG
gpg --decrypt message.gpg

Tasks

  • Encrypt and decrypt a text file using OpenSSL.
  • Generate a pair of GPG keys and encrypt a message.
  • Document the process and the commands used.

Conclusion

These practical exercises are designed to provide hands-on experience in identifying and mitigating vulnerabilities, implementing security policies, and encrypting data. By completing these exercises, students will be better prepared to face security challenges in real-world environments.
