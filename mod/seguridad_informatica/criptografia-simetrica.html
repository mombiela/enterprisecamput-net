Introduction

Symmetric cryptography is an encryption technique where the same key is used for both encrypting and decrypting information. It is one of the oldest and most widely used methods for protecting sensitive data.

Key Concepts

  • Secret Key: The same key is used to encrypt and decrypt the data.
  • Encryption Algorithm: Mathematical procedure that transforms plaintext into ciphertext.
  • Plaintext: Original information that needs to be protected.
  • Ciphertext: Encrypted information resulting from the encryption process.
  • Performance: Generally, symmetric algorithms are faster than asymmetric ones.

Common Algorithms

DES (Data Encryption Standard)

  • Key Length: 56 bits.
  • Structure: Block cipher algorithm.
  • Security: Considered insecure due to its short key length.

AES (Advanced Encryption Standard)

  • Key Length: 128, 192, or 256 bits.
  • Structure: Block cipher algorithm.
  • Security: Currently considered secure and widely used.

RC4

  • Key Length: Variable, typically between 40 and 2048 bits.
  • Structure: Stream cipher algorithm.
  • Security: Vulnerable to certain types of attacks, but still in use in some applications.

Code Example

Below is an example of encryption and decryption using the AES algorithm in Python with the pycryptodome library.

from Crypto.Cipher import AES
from Crypto.Random import get_random_bytes
import base64

# Generate a 16-byte (128-bit) key
key = get_random_bytes(16)

# Create an AES cipher object
cipher = AES.new(key, AES.MODE_EAX)

# Plaintext
data = b"This is a secret message"

# Encrypt the plaintext
nonce = cipher.nonce
ciphertext, tag = cipher.encrypt_and_digest(data)

# Display the ciphertext in base64
print("Ciphertext:", base64.b64encode(ciphertext).decode('utf-8'))

# Create a new AES cipher object to decrypt
cipher_dec = AES.new(key, AES.MODE_EAX, nonce=nonce)

# Decrypt the ciphertext
plaintext = cipher_dec.decrypt(ciphertext)

# Display the plaintext
print("Decrypted text:", plaintext.decode('utf-8'))

Code Explanation

  1. Key Generation: A 16-byte key is generated using get_random_bytes.
  2. Cipher Object Creation: An AES cipher object is created in EAX mode.
  3. Plaintext Encryption: The plaintext is encrypted, resulting in ciphertext and an authentication tag.
  4. Ciphertext Decryption: A new AES cipher object is created with the same key and nonce to decrypt the ciphertext.

Algorithm Comparison

| Algorithm | Key Length | Structure | Security | |-----------|-------------------|------------|-----------| | DES | 56 bits | Block | Insecure | | AES | 128, 192, 256 bits| Block | Secure | | RC4 | Variable | Stream | Vulnerable|

Conclusion

Symmetric cryptography is a fundamental technique in information security, used to protect sensitive data through the use of a shared secret key. While some algorithms like DES have been surpassed for security reasons, others like AES remain robust and reliable standards. It is crucial to understand both the strengths and weaknesses of these algorithms to apply them appropriately in different security contexts.
