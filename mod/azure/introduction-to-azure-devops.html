Introduction to Azure DevOps
Continuous Integration and Continuous Deployment (CI/CD) ►

Azure DevOps is a set of development tools provided by Microsoft to support the entire software development lifecycle. It includes services for planning, developing, delivering, and operating applications. This topic will introduce you to the core concepts and services of Azure DevOps, providing a foundation for further exploration and mastery.

Key Concepts of Azure DevOps

  • Azure Boards: Agile planning tools (Kanban boards, backlogs, team dashboards) for tracking work.
  • Azure Repos: Unlimited cloud-hosted private Git repositories for your project.
  • Azure Pipelines: Build, test, and deploy with CI/CD that works with any language, platform, and cloud.
  • Azure Test Plans: All-in-one planned and exploratory testing solution.
  • Azure Artifacts: Create, host, and share packages with your team, and add artifacts to your CI/CD pipelines.

Setting Up Azure DevOps

Creating an Azure DevOps Account

  1. Go to the Azure DevOps website.
  2. Click on "Start free" or "Sign in" if you already have a Microsoft account.
  3. Follow the prompts to create your Azure DevOps organization.

Creating a New Project

  1. Once logged in, click on "New Project".
  2. Enter a project name and description.
  3. Choose visibility (public or private).
  4. Click "Create".

Azure Boards

Azure Boards provides a rich set of capabilities including native support for Scrum and Kanban, customizable dashboards, and integrated reporting.

Creating and Managing Work Items

  • Work Items: Basic units of work, such as user stories, tasks, bugs.
  • Boards: Visualize and manage work with Kanban boards.
  • Backlogs: Prioritize work and plan sprints.

Example of creating a work item:

1. Navigate to Boards > Work Items.
2. Click on "New Work Item" and select the type (e.g., User Story).
3. Fill in the details and save.

Azure Repos

Azure Repos provides Git repositories or Team Foundation Version Control (TFVC) for source control of your code.

Creating a Repository

  1. Navigate to Repos > Files.
  2. Click on "New Repository".
  3. Choose Git or TFVC.
  4. Initialize with a README if desired and click "Create".

Basic Git Commands

# Clone a repository
git clone https://dev.azure.com/yourorganization/yourproject/_git/yourrepo

# Add a file to staging
git add filename

# Commit changes
git commit -m "Your commit message"

# Push changes to Azure Repos
git push origin main

Azure Pipelines

Azure Pipelines enables you to build, test, and deploy code automatically.

Creating a Pipeline

  1. Navigate to Pipelines > Pipelines.
  2. Click on "New Pipeline".
  3. Select your repository source (e.g., GitHub, Azure Repos).
  4. Configure your pipeline using YAML or the classic editor.

Example YAML pipeline configuration:

trigger:
- main

pool:
  vmImage: 'ubuntu-latest'

steps:
- script: echo Hello, world!
  displayName: 'Run a one-line script'

Azure Test Plans

Azure Test Plans provides tools for manual and exploratory testing.

Creating a Test Plan

  1. Navigate to Test Plans > Test Plans.
  2. Click on "New Test Plan".
  3. Enter a name and description.
  4. Add test cases and configure test suites.

Azure Artifacts

Azure Artifacts allows you to create and share Maven, npm, NuGet, and Python package feeds from public and private sources.

Creating a Feed

  1. Navigate to Artifacts > Feeds.
  2. Click on "New Feed".
  3. Enter a name and set permissions.
  4. Click "Create".

Using Azure Artifacts in a Pipeline

Example of adding a NuGet package to a pipeline:

steps:
- task: UseDotNet@2
  inputs:
    packageType: 'sdk'
    version: '5.x'
    installationPath: $(Agent.ToolsDirectory)/dotnet

- task: NuGetCommand@2
  inputs:
    command: 'restore'
    feedsToUse: 'select'
    vstsFeed: 'your-feed-id'

Conclusion

Azure DevOps is a comprehensive suite of tools designed to support the entire software development lifecycle. By understanding and utilizing Azure Boards, Repos, Pipelines, Test Plans, and Artifacts, you can streamline your development process, improve collaboration, and deliver high-quality software more efficiently. This introduction provides a foundation for further exploration and mastery of Azure DevOps.
