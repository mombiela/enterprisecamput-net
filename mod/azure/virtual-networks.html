Introduction to Virtual Networks (VNet)

Virtual Networks (VNets) are fundamental building blocks for your private network in Azure. They enable many types of Azure resources, such as Azure Virtual Machines (VMs), to securely communicate with each other, the internet, and on-premises networks.

Key Concepts

  • Isolation and Segmentation: VNets provide isolation and segmentation within the Azure environment.
  • Subnets: VNets can be divided into subnets to further segment the network.
  • IP Addressing: VNets use private IP addressing.
  • Network Security Groups (NSGs): Control inbound and outbound traffic to network interfaces (NIC), VMs, and subnets.
  • Peering: Connect VNets to each other.
  • VPN Gateway: Connect VNets to on-premises networks.

Creating a Virtual Network

Creating a VNet is the first step in setting up your Azure network infrastructure.

Steps to Create a VNet

  1. Navigate to Azure Portal: Go to the Azure portal at https://portal.azure.com.

  2. Create a Resource: Click on "Create a resource" and search for "Virtual Network".

  3. Configure Basic Settings:

    • Name: Provide a name for your VNet.
    • Address Space: Define the IP address range (e.g., 10.0.0.0/16).
    • Subscription: Select your Azure subscription.
    • Resource Group: Choose an existing resource group or create a new one.
    • Location: Select the Azure region.

  4. Configure Subnets:

    • Subnet Name: Provide a name for your subnet.
    • Subnet Address Range: Define the IP address range for the subnet (e.g., 10.0.1.0/24).

  5. Review and Create: Review your settings and click "Create".

Example Code: Creating a VNet using Azure CLI

# Create a resource group
az group create --name MyResourceGroup --location eastus

# Create a virtual network
az network vnet create \
  --name MyVNet \
  --resource-group MyResourceGroup \
  --address-prefix 10.0.0.0/16 \
  --subnet-name MySubnet \
  --subnet-prefix 10.0.1.0/24

Subnets

Subnets allow you to segment the VNet into smaller, manageable sections.

Key Concepts

  • Subnet Address Range: Each subnet must have a unique address range within the VNet.
  • Network Security Groups (NSGs): Apply NSGs to subnets to control traffic.

Example Code: Adding a Subnet using Azure CLI

# Add a subnet to an existing VNet
az network vnet subnet create \
  --address-prefix 10.0.2.0/24 \
  --name MySecondSubnet \
  --resource-group MyResourceGroup \
  --vnet-name MyVNet

Network Security Groups (NSGs)

NSGs are used to filter network traffic to and from Azure resources in an Azure virtual network.

Key Concepts

  • Inbound and Outbound Rules: Define rules to allow or deny traffic.
  • Associations: NSGs can be associated with subnets or individual network interfaces.

Example Code: Creating an NSG and Adding Rules

# Create an NSG
az network nsg create \
  --resource-group MyResourceGroup \
  --name MyNSG

# Create an inbound security rule
az network nsg rule create \
  --resource-group MyResourceGroup \
  --nsg-name MyNSG \
  --name AllowHTTP \
  --protocol tcp \
  --priority 1000 \
  --destination-port-range 80 \
  --access allow

VNet Peering

VNet peering connects two VNets, enabling resources in each VNet to communicate with each other.

Key Concepts

  • Low Latency: Provides low-latency, high-bandwidth connectivity.
  • Transitive Peering: Not supported; each VNet must be peered directly.

Example Code: Creating a VNet Peering

# Peer VNet1 with VNet2
az network vnet peering create \
  --name VNet1ToVNet2 \
  --resource-group MyResourceGroup \
  --vnet-name VNet1 \
  --remote-vnet /subscriptions/{subscription-id}/resourceGroups/MyResourceGroup/providers/Microsoft.Network/virtualNetworks/VNet2 \
  --allow-vnet-access

VPN Gateway

VPN Gateway is used to send encrypted traffic between an Azure VNet and an on-premises location over the public Internet.

Key Concepts

  • Gateway Types: Policy-based and route-based.
  • Connection Types: Site-to-Site, Point-to-Site, and VNet-to-VNet.

Example Code: Creating a VPN Gateway

# Create a VPN gateway
az network vnet-gateway create \
  --resource-group MyResourceGroup \
  --name MyVPNGateway \
  --public-ip-address MyGatewayPublicIP \
  --vnet MyVNet \
  --gateway-type Vpn \
  --vpn-type RouteBased \
  --sku VpnGw1 \
  --no-wait

Conclusion

Virtual Networks (VNets) are essential for creating a secure and scalable network infrastructure in Azure. By understanding and utilizing VNets, subnets, NSGs, VNet peering, and VPN gateways, you can effectively manage and secure your Azure resources. This foundational knowledge is crucial for both beginners and advanced users to build robust cloud solutions.
