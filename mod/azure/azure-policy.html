Azure Policy
Compliance and Governance ►

Azure Policy is a service in Azure that you use to create, assign, and manage policies. These policies enforce different rules and effects over your resources, so those resources stay compliant with your corporate standards and service level agreements. Azure Policy does this by running evaluations of your resources and scanning for those not compliant with the policies you have created.

Introduction to Azure Policy

  • Definition: Azure Policy is a governance tool that allows you to enforce organizational standards and assess compliance at-scale.
  • Purpose: Ensure resources are compliant with corporate standards and regulatory requirements.
  • Key Features:
    • Policy Definitions
    • Initiatives
    • Assignments
    • Compliance

Policy Definitions

A policy definition expresses what to evaluate and what action to take. These definitions can be created from scratch or by using built-in policies provided by Azure.

  • Structure:
    • Mode: Specifies the scope of the policy (e.g., All, Indexed).
    • Parameters: Define values that can be used in the policy rule.
    • Rule: The logical evaluation that determines if a resource is compliant.

Example Policy Definition

{
  "properties": {
    "displayName": "Allowed locations",
    "description": "This policy enables you to restrict the locations your organization can specify when deploying resources.",
    "parameters": {
      "allowedLocations": {
        "type": "Array",
        "metadata": {
          "description": "The list of allowed locations for resources.",
          "displayName": "Allowed locations",
          "strongType": "location"
        }
      }
    },
    "policyRule": {
      "if": {
        "not": {
          "field": "location",
          "in": "[parameters('allowedLocations')]"
        }
      },
      "then": {
        "effect": "deny"
      }
    }
  }
}
  • Explanation:
    • displayName: Name of the policy.
    • description: Description of what the policy does.
    • parameters: Defines the allowed locations.
    • policyRule: The rule checks if the resource location is not in the allowed locations and denies the deployment if it is not.

Initiatives

An initiative is a collection of policy definitions that are grouped together towards a specific goal or purpose.

  • Purpose: Simplify the management of multiple policies by grouping them.
  • Structure:
    • Initiative Definition: Contains multiple policy definitions.
    • Parameters: Parameters that can be shared across the policies in the initiative.

Example Initiative Definition

{
  "properties": {
    "displayName": "Security Initiative",
    "description": "Ensures resources comply with security standards.",
    "parameters": {},
    "policyDefinitions": [
      {
        "policyDefinitionId": "/providers/Microsoft.Authorization/policyDefinitions/allowedLocations",
        "parameters": {
          "allowedLocations": {
            "value": ["eastus", "westus"]
          }
        }
      },
      {
        "policyDefinitionId": "/providers/Microsoft.Authorization/policyDefinitions/requireTag",
        "parameters": {
          "tagName": {
            "value": "Environment"
          }
        }
      }
    ]
  }
}
  • Explanation:
    • displayName: Name of the initiative.
    • description: Description of the initiative's purpose.
    • policyDefinitions: List of policy definitions included in the initiative.

Assignments

Assignments are used to apply a policy or initiative to a specific scope, such as a subscription, resource group, or individual resource.

  • Scope: Defines where the policy or initiative is applied.
  • Parameters: Values provided for the parameters defined in the policy or initiative.

Example Assignment

{
  "properties": {
    "displayName": "Enforce allowed locations",
    "policyDefinitionId": "/providers/Microsoft.Authorization/policyDefinitions/allowedLocations",
    "scope": "/subscriptions/{subscriptionId}/resourceGroups/{resourceGroupName}",
    "parameters": {
      "allowedLocations": {
        "value": ["eastus", "westus"]
      }
    }
  }
}
  • Explanation:
    • displayName: Name of the assignment.
    • policyDefinitionId: The ID of the policy definition being assigned.
    • scope: The scope where the policy is applied.
    • parameters: Values for the parameters defined in the policy.

Compliance

Compliance in Azure Policy refers to the state of resources in relation to the policies assigned to them.

  • Compliance State: Indicates whether resources are compliant or non-compliant.
  • Compliance Reports: Provide detailed information about the compliance state of resources.

Viewing Compliance

You can view compliance details in the Azure portal:

  1. Navigate to the Azure Policy service.
  2. Select the "Compliance" tab.
  3. Review the compliance state of your resources.

Conclusion

Azure Policy is a powerful tool for ensuring that your Azure resources comply with organizational standards and regulatory requirements. By understanding and utilizing policy definitions, initiatives, assignments, and compliance reports, you can effectively manage and enforce governance across your Azure environment.
