Introduction to Azure Databases

Azure offers a variety of database services that cater to different needs, from relational databases to NoSQL databases. This section will provide an overview of the different types of databases available on Azure and their use cases.

  • Azure SQL Database: A fully managed relational database with built-in intelligence.
  • Azure Cosmos DB: A globally distributed, multi-model database service.
  • Azure Database for MySQL: A managed MySQL database service.
  • Azure Database for PostgreSQL: A managed PostgreSQL database service.
  • Azure Database for MariaDB: A managed MariaDB database service.

Azure SQL Database

Azure SQL Database is a fully managed relational database service that provides high availability, scalability, and security.

Key Features

  • Managed Service: Automated backups, patching, and updates.
  • Scalability: Scale up or down based on your needs.
  • Security: Advanced threat protection and encryption.
  • Performance: Built-in intelligence for performance optimization.

Getting Started with Azure SQL Database

  1. Create an Azure SQL Database:

    az sql db create --resource-group myResourceGroup --server myServer --name myDatabase --service-objective S0

  2. Connect to Azure SQL Database:

    import pyodbc

server = 'myServer.database.windows.net'
database = 'myDatabase'
username = 'myUsername'
password = 'myPassword'
driver= '{ODBC Driver 17 for SQL Server}'

cnxn = pyodbc.connect('DRIVER=' + driver + ';SERVER=' + server + ';PORT=1433;DATABASE=' + database + ';UID=' + username + ';PWD=' + password)
cursor = cnxn.cursor()

  3. Perform CRUD Operations:

    # Create
cursor.execute("INSERT INTO myTable (name, age) VALUES (?, ?)", ('John Doe', 30))
cnxn.commit()

# Read
cursor.execute("SELECT * FROM myTable")
for row in cursor:
    print(row)

# Update
cursor.execute("UPDATE myTable SET age = ? WHERE name = ?", (31, 'John Doe'))
cnxn.commit()

# Delete
cursor.execute("DELETE FROM myTable WHERE name = ?", ('John Doe',))
cnxn.commit()

Azure Cosmos DB

Azure Cosmos DB is a globally distributed, multi-model database service designed for high availability and low latency.

Key Features

  • Global Distribution: Distribute data across multiple regions.
  • Multi-Model: Supports document, key-value, graph, and column-family data models.
  • Scalability: Automatic scaling of throughput and storage.
  • Consistency Levels: Choose from five consistency levels to balance performance and consistency.

Getting Started with Azure Cosmos DB

  1. Create an Azure Cosmos DB Account:

    az cosmosdb create --name myCosmosDBAccount --resource-group myResourceGroup --kind MongoDB

  2. Connect to Azure Cosmos DB:

    from pymongo import MongoClient

client = MongoClient("mongodb://myCosmosDBAccount.documents.azure.com:10255/?ssl=true&replicaSet=globaldb", username="myUsername", password="myPassword")
db = client.myDatabase

  3. Perform CRUD Operations:

    # Create
db.myCollection.insert_one({"name": "John Doe", "age": 30})

# Read
for doc in db.myCollection.find():
    print(doc)

# Update
db.myCollection.update_one({"name": "John Doe"}, {"$set": {"age": 31}})

# Delete
db.myCollection.delete_one({"name": "John Doe"})

Azure Database for MySQL

Azure Database for MySQL is a managed MySQL database service that provides high availability and security.

Key Features

  • Managed Service: Automated backups, patching, and updates.
  • Scalability: Scale compute and storage independently.
  • Security: Advanced threat protection and encryption.
  • Performance: Built-in performance monitoring and tuning.

Getting Started with Azure Database for MySQL

  1. Create an Azure Database for MySQL:

    az mysql server create --resource-group myResourceGroup --name myMySQLServer --admin-user myAdmin --admin-password myPassword --sku-name GP_Gen5_2

  2. Connect to Azure Database for MySQL:

    import mysql.connector

cnx = mysql.connector.connect(user='myAdmin', password='myPassword',
                              host='myMySQLServer.mysql.database.azure.com',
                              database='myDatabase')
cursor = cnx.cursor()

  3. Perform CRUD Operations:

    # Create
cursor.execute("INSERT INTO myTable (name, age) VALUES (%s, %s)", ('John Doe', 30))
cnx.commit()

# Read
cursor.execute("SELECT * FROM myTable")
for row in cursor:
    print(row)

# Update
cursor.execute("UPDATE myTable SET age = %s WHERE name = %s", (31, 'John Doe'))
cnx.commit()

# Delete
cursor.execute("DELETE FROM myTable WHERE name = %s", ('John Doe',))
cnx.commit()

    az postgres server create --resource-group myResourceGroup --name myPostgreSQLServer --admin-user myAdmin --admin-password myPassword --sku-name GP_Gen5_2

    import psycopg2

conn = psycopg2.connect(
    dbname='myDatabase',
    user='myAdmin',
    password='myPassword',
    host='myPostgreSQLServer.postgres.database.azure.com'

Conclusion

Azure offers a wide range of database services that cater to various needs, from relational databases like Azure SQL Database, MySQL, PostgreSQL, and MariaDB to NoSQL databases like Azure Cosmos DB. Each service provides unique features and benefits, making it easier to choose the right database for your application. By understanding the key features and getting started with basic CRUD operations, you can leverage the power of Azure databases to build scalable, secure, and high-performance applications.
