Introduction

Managing virtual machines (VMs) on Azure is a fundamental skill for cloud computing. This topic will guide you through the process from a beginner to an advanced level, covering key concepts, practical examples, and exercises.

Getting Started with Azure Virtual Machines

What is a Virtual Machine?

  • A virtual machine (VM) is an emulation of a computer system.
  • VMs run on physical hardware but are isolated from it.
  • They allow you to run multiple operating systems on a single physical machine.

Creating Your First VM

  1. Sign in to Azure Portal: Navigate to Azure Portal.
  2. Create a Resource: Click on "Create a resource" and select "Virtual Machine".
  3. Configure Basic Settings:
    • Subscription: Choose your Azure subscription.
    • Resource Group: Create a new resource group or select an existing one.
    • VM Name: Give your VM a unique name.
    • Region: Select the region where you want to deploy your VM.
    • Image: Choose an operating system image (e.g., Windows Server, Ubuntu).
    • Size: Select the VM size based on your requirements.
# Example: Azure CLI command to create a VM
az vm create \
  --resource-group myResourceGroup \
  --name myVM \
  --image UbuntuLTS \
  --admin-username azureuser \
  --generate-ssh-keys

Connecting to Your VM

  • SSH for Linux VMs: Use an SSH client to connect.
  • RDP for Windows VMs: Use Remote Desktop Protocol (RDP).
# Example: SSH into a Linux VM
ssh azureuser@<public-ip-address>

Intermediate Management Tasks

Managing VM Sizes and Scaling

  • Resizing a VM: Change the size of your VM to scale up or down based on your needs.
  • Auto-scaling: Use Azure Scale Sets to automatically scale VMs based on demand.
# Example: Resize a VM using Azure CLI
az vm resize \
  --resource-group myResourceGroup \
  --name myVM \
  --size Standard_DS3_v2

Disk Management

  • Adding Data Disks: Attach additional disks to your VM for extra storage.
  • Disk Types: Understand the difference between Standard HDD, Standard SSD, and Premium SSD.
# Example: Attach a new data disk to a VM
az vm disk attach \
  --resource-group myResourceGroup \
  --vm-name myVM \
  --name myDataDisk \
  --size-gb 128

Networking

  • Virtual Networks (VNet): Create and manage VNets to control VM network settings.
  • Network Security Groups (NSG): Configure NSGs to manage inbound and outbound traffic.
# Example: Create a VNet and Subnet
az network vnet create \
  --resource-group myResourceGroup \
  --name myVNet \
  --address-prefix 10.0.0.0/16 \
  --subnet-name mySubnet \
  --subnet-prefix 10.0.1.0/24

Advanced Management Tasks

  • ARM Templates: Define your infrastructure as code using JSON templates.
  • Deploying ARM Templates: Use Azure CLI or Azure Portal to deploy templates.

Monitoring and Diagnostics

  • Azure Monitor: Collect and analyze telemetry data from your VMs.
  • Log Analytics: Query and analyze logs for deeper insights.
# Example: Enable monitoring on a VM
az vm diagnostics set \
  --resource-group myResourceGroup \
  --vm-name myVM \
  --settings '{"storageAccount": "mystorageaccount"}'

Backup and Recovery

  • Azure Backup: Configure backup policies to protect your VMs.
  • Disaster Recovery: Use Azure Site Recovery for VM replication and failover.
# Example: Enable backup for a VM
az backup protection enable-for-vm \
  --resource-group myResourceGroup \
  --vault-name myRecoveryServicesVault \
  --vm myVM \
  --policy-name DefaultPolicy

Conclusion

Managing virtual machines on Azure involves a range of tasks from basic creation and connection to advanced automation and monitoring. By mastering these skills, you can effectively leverage Azure's capabilities to deploy, scale, and maintain your virtual infrastructure. Practice regularly and explore Azure's extensive documentation to deepen your understanding and proficiency.
