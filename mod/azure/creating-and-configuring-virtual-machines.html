Introduction

In this section, we will explore how to create and configure Virtual Machines (VMs) in Microsoft Azure. This topic is essential for anyone looking to leverage cloud computing for scalable and flexible infrastructure.

Prerequisites

Before we begin, ensure you have:

  • An active Azure subscription.
  • Basic understanding of cloud computing concepts.
  • Familiarity with the Azure portal.

Creating a Virtual Machine

Step-by-Step Guide

  1. Navigate to the Azure Portal:

    • Go to Azure Portal.
    • Sign in with your Azure account.

  2. Create a New Virtual Machine:

    • Click on "Create a resource" in the left-hand menu.
    • Select "Compute" and then "Virtual Machine."

  3. Configure Basic Settings:

    • Subscription: Choose your subscription.
    • Resource Group: Create a new resource group or select an existing one.
    • Virtual Machine Name: Enter a unique name for your VM.
    • Region: Select the region where you want to deploy your VM.
    • Availability Options: Choose availability set or availability zone if needed.
    • Image: Select the operating system image (e.g., Windows Server, Ubuntu).
    • Size: Choose the VM size based on your requirements.

  4. Administrator Account:

    • Username: Enter a username for the admin account.
    • Password: Enter a strong password or use an SSH public key for Linux VMs.

  5. Inbound Port Rules:

    • Select the ports you want to open (e.g., RDP for Windows, SSH for Linux).

  6. Review + Create:

    • Review your settings and click "Create."

Example

# Azure CLI example to create a VM
az vm create \
  --resource-group myResourceGroup \
  --name myVM \
  --image UbuntuLTS \
  --admin-username azureuser \
  --generate-ssh-keys

Configuring a Virtual Machine

Accessing the VM

  • Windows VM: Use Remote Desktop Protocol (RDP) to connect.
  • Linux VM: Use Secure Shell (SSH) to connect.

Installing Software

  • Windows VM:

    • Use the RDP connection to install software manually.
    • Use PowerShell scripts for automation.

  • Linux VM:

    • Use SSH to connect and install software using package managers like apt or yum.

Example

# Example of installing Apache on an Ubuntu VM
sudo apt update
sudo apt install apache2 -y

Advanced Configuration

Networking

  • Virtual Network (VNet): Configure VNets to manage network traffic.
  • Network Security Groups (NSG): Define security rules to control inbound and outbound traffic.

Storage

  • Managed Disks: Use managed disks for better performance and reliability.
  • Data Disks: Attach additional data disks for storage needs.

Scaling

  • Scale Sets: Use VM scale sets to automatically scale the number of VMs based on demand.

Example

# Azure CLI example to add a data disk to a VM
az vm disk attach \
  --resource-group myResourceGroup \
  --vm-name myVM \
  --name myDataDisk \
  --size-gb 128

Monitoring and Maintenance

Monitoring

  • Azure Monitor: Use Azure Monitor to track the performance and health of your VMs.
  • Log Analytics: Collect and analyze logs for troubleshooting.

Maintenance

  • Updates: Regularly update your VMs to ensure security and performance.
  • Backups: Use Azure Backup to create regular backups of your VMs.

Conclusion

Creating and configuring VMs in Azure is a fundamental skill for leveraging cloud infrastructure. By following the steps outlined in this guide, you can set up VMs tailored to your specific needs, configure them for optimal performance, and ensure they are secure and well-maintained. As you advance, explore more complex configurations and integrations to fully utilize Azure's capabilities.
