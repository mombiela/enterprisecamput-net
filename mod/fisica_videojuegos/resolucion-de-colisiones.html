Introduction

Collision resolution is a fundamental aspect of video game physics. Once collisions between objects are detected, it is crucial to resolve them so that the objects react realistically. This topic covers the basic and advanced concepts of collision resolution, including mathematical methods and code examples.

Key Concepts

  • Collision: Occurs when two or more objects in a video game occupy the same space at the same time.
  • Collision Normal: Vector perpendicular to the contact surface at the point of collision.
  • Penetration: The distance an object has penetrated into another during a collision.
  • Impulse: Change in the momentum of an object due to a force applied over a short period of time.

Collision Resolution Methods

Separating Axis Theorem (SAT)

The Separating Axis Theorem (SAT) is a technique used to detect and resolve collisions between convex polygons.

  • Step 1: Project the vertices of the polygons onto an axis.
  • Step 2: Check if the projections overlap.
  • Step 3: If there is overlap on all axes, a collision is detected.
  • Step 4: Resolve the collision by moving the objects along the axis of least penetration.

Code Example: SAT in 2D

def project_polygon(vertices, axis):
    min_proj = float('inf')
    max_proj = float('-inf')
    for vertex in vertices:
        projection = vertex.dot(axis)
        min_proj = min(min_proj, projection)
        max_proj = max(max_proj, projection)
    return min_proj, max_proj

def overlap(minA, maxA, minB, maxB):
    return minB <= maxA and minA <= maxB

def sat_collision(polygonA, polygonB):
    axes = get_axes(polygonA) + get_axes(polygonB)
    for axis in axes:
        minA, maxA = project_polygon(polygonA, axis)
        minB, maxB = project_polygon(polygonB, axis)
        if not overlap(minA, maxA, minB, maxB):
            return False
    return True

# Example usage
polygonA = [Vector2(0, 0), Vector2(1, 0), Vector2(1, 1), Vector2(0, 1)]
polygonB = [Vector2(0.5, 0.5), Vector2(1.5, 0.5), Vector2(1.5, 1.5), Vector2(0.5, 1.5)]
print(sat_collision(polygonA, polygonB))  # Output: True

Impulse and Conservation of Momentum

To resolve collisions more realistically, the concept of impulse and conservation of momentum is used.

  • Impulse: \( J = \frac{-(1 + e) \cdot (v \cdot n)}{(1/m1 + 1/m2)} \)
    • \( e \) is the coefficient of restitution.
    • \( v \) is the relative velocity.
    • \( n \) is the collision normal.
    • \( m1 \) and \( m2 \) are the masses of the objects.

Code Example: Impulse Resolution

def resolve_collision(obj1, obj2, normal, restitution):
    relative_velocity = obj2.velocity - obj1.velocity
    velocity_along_normal = relative_velocity.dot(normal)
    
    if velocity_along_normal > 0:
        return
    
    impulse_magnitude = -(1 + restitution) * velocity_along_normal
    impulse_magnitude /= (1 / obj1.mass + 1 / obj2.mass)
    
    impulse = impulse_magnitude * normal
    obj1.velocity -= (1 / obj1.mass) * impulse
    obj2.velocity += (1 / obj2.mass) * impulse

# Example usage
class GameObject:
    def __init__(self, mass, velocity):
        self.mass = mass
        self.velocity = velocity

obj1 = GameObject(1, Vector2(1, 0))
obj2 = GameObject(1, Vector2(-1, 0))
normal = Vector2(1, 0)
restitution = 0.5

resolve_collision(obj1, obj2, normal, restitution)
print(obj1.velocity, obj2.velocity)  # Output: Vector2(-0.5, 0) Vector2(0.5, 0)

Conclusion

Collision resolution is essential for creating realistic and smooth gaming experiences. From geometric methods like SAT to physical techniques like conservation of momentum, each approach has its applications and advantages. Practicing with code examples and understanding the mathematical fundamentals behind these methods will enable developers to implement effective collision systems in their video games.
