Introduction

PowerShell is built on the .NET framework, which means it has access to a vast library of .NET classes. Understanding how to use these classes can significantly enhance your scripting capabilities. This section will guide you through the basics of using .NET classes in PowerShell, progressing to more advanced usage.

Accessing .NET Classes

Basic Usage

  • Loading .NET Classes: You can access .NET classes directly in PowerShell by using the [Namespace.ClassName] syntax.
  • Creating Instances: Use the New-Object cmdlet or the ::new() method to create instances of .NET classes.

Example: Creating a .NET Object

# Using New-Object
$object = New-Object System.DateTime

# Using ::new() method (PowerShell 5.0+)
$object = [System.DateTime]::new()

Static Methods and Properties

  • Static Methods: Called on the class itself rather than an instance.
  • Static Properties: Accessed directly from the class.

Example: Using Static Methods and Properties

# Static Method
$currentDate = [System.DateTime]::Now

# Static Property
$maxValue = [System.Int32]::MaxValue

Commonly Used .NET Classes

System.String

  • Description: Represents text as a series of Unicode characters.
  • Common Methods: Substring(), Replace(), ToUpper(), ToLower()

Example: Manipulating Strings

$string = "Hello, World!"
$substring = $string.Substring(0, 5)  # Output: Hello
$replacedString = $string.Replace("World", "PowerShell")  # Output: Hello, PowerShell!

System.IO.File

  • Description: Provides static methods for the creation, copying, deletion, moving, and opening of files.
  • Common Methods: ReadAllText(), WriteAllText(), Copy(), Delete()

Example: File Operations

$filePath = "C:\example.txt"

# Write text to a file
[System.IO.File]::WriteAllText($filePath, "Hello, PowerShell!")

# Read text from a file
$content = [System.IO.File]::ReadAllText($filePath)
Write-Output $content  # Output: Hello, PowerShell!

System.Net.WebClient

  • Description: Provides common methods for sending data to and receiving data from a resource identified by a URI.
  • Common Methods: DownloadString(), UploadString()

Example: Downloading Content from the Web

$url = "http://example.com"
$webClient = [System.Net.WebClient]::new()
$content = $webClient.DownloadString($url)
Write-Output $content

Advanced Usage

Using .NET Collections

  • System.Collections.Generic.List: A strongly-typed list of objects.
  • System.Collections.Hashtable: Represents a collection of key/value pairs.

Example: Using a Generic List

# Create a new List of integers
$list = [System.Collections.Generic.List[int]]::new()
$list.Add(1)
$list.Add(2)
$list.Add(3)

# Iterate through the list
foreach ($item in $list) {
    Write-Output $item
}

Exception Handling with .NET Exceptions

  • Try/Catch/Finally: PowerShell supports structured exception handling using .NET exceptions.

Example: Handling Exceptions

try {
    # Attempt to open a non-existent file
    [System.IO.File]::Open("C:\nonexistentfile.txt", [System.IO.FileMode]::Open)
} catch [System.IO.FileNotFoundException] {
    Write-Output "File not found."
} finally {
    Write-Output "Exception handling complete."
}

Conclusion

Using .NET classes in PowerShell opens up a wide range of functionalities that can enhance your scripts. From basic object creation to advanced exception handling, understanding how to leverage these classes is crucial for both beginner and advanced PowerShell users. Practice using different .NET classes and explore the .NET documentation to further expand your knowledge and capabilities.
