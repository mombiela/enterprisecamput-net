Introduction to Regular Expressions

Regular expressions (regex) are sequences of characters that define a search pattern. They are commonly used for string matching and manipulation. In PowerShell, regex can be used to search, replace, and validate strings.

Key Concepts

  • Pattern Matching: Identifying whether a string contains a specified pattern.
  • Character Classes: Sets of characters enclosed in square brackets [] that match any one character within the set.
  • Quantifiers: Specify the number of occurrences of a character or group.
  • Anchors: Assert the position within a string (e.g., start or end).
  • Groups and Capturing: Enclose part of the regex in parentheses () to create groups.

Basic Syntax and Usage

Pattern Matching

To check if a string matches a pattern, use the -match operator.

$string = "Hello World"
if ($string -match "World") {
    Write-Output "Pattern found!"
}

Character Classes

Character classes allow you to match any one of several characters.

$string = "Hello World"
if ($string -match "[aeiou]") {
    Write-Output "String contains a vowel!"
}

Quantifiers

Quantifiers specify how many times a character or group should be matched.

  • * : 0 or more times
  • + : 1 or more times
  • ? : 0 or 1 time
  • {n} : Exactly n times
  • {n,} : n or more times
  • {n,m} : Between n and m times
$string = "Helloooo World"
if ($string -match "o{2,}") {
    Write-Output "String contains two or more consecutive 'o's!"
}

Anchors

Anchors are used to specify the position within the string.

  • ^ : Start of the string
  • $ : End of the string
$string = "Hello World"
if ($string -match "^Hello") {
    Write-Output "String starts with 'Hello'!"
}
if ($string -match "World$") {
    Write-Output "String ends with 'World'!"
}

Groups and Capturing

Groups are used to create sub-patterns within a regex.

$string = "Hello123"
if ($string -match "([A-Za-z]+)(\d+)") {
    Write-Output "Letters: $($matches[1]), Numbers: $($matches[2])"
}

Advanced Techniques

Lookahead and Lookbehind

Lookahead and lookbehind assertions allow you to match a pattern only if it is (or isn't) followed or preceded by another pattern.

  • Positive Lookahead: (?=...)
  • Negative Lookahead: (?!...)
  • Positive Lookbehind: (?<=...)
  • Negative Lookbehind: (?<!...)
$string = "foo123bar"
if ($string -match "foo(?=\d{3})") {
    Write-Output "Found 'foo' followed by three digits!"
}

Non-Capturing Groups

Non-capturing groups are used to group part of a regex without capturing it.

$string = "Hello123"
if ($string -match "(?:Hello)(\d+)") {
    Write-Output "Numbers: $($matches[1])"
}

Named Capturing Groups

Named capturing groups allow you to assign a name to a group.

$string = "Hello123"
if ($string -match "(?<letters>[A-Za-z]+)(?<numbers>\d+)") {
    Write-Output "Letters: $($matches['letters']), Numbers: $($matches['numbers'])"
}

Practical Examples

Validating Email Addresses

$email = "example@example.com"
$regex = "^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$"
if ($email -match $regex) {
    Write-Output "Valid email address!"
} else {
    Write-Output "Invalid email address!"
}

Extracting Data from Text

$text = "Order ID: 12345, Date: 2023-10-01"
if ($text -match "Order ID: (\d+), Date: (\d{4}-\d{2}-\d{2})") {
    $orderId = $matches[1]
    $date = $matches[2]
    Write-Output "Order ID: $orderId, Date: $date"
}

Summary

Regular expressions are powerful tools for text processing in PowerShell. By understanding the basic syntax and advanced techniques, you can perform complex string manipulations and validations efficiently. Practice with different patterns and use cases to become proficient in using regex in your PowerShell scripts.
