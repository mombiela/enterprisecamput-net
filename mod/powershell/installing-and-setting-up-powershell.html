Introduction

PowerShell is a task automation and configuration management framework from Microsoft, consisting of a command-line shell and the associated scripting language. This section will guide you through the process of installing and setting up PowerShell on your system, ensuring you have the right environment to start scripting and automating tasks.

Installing PowerShell

Windows

PowerShell comes pre-installed on Windows 10 and later versions. However, you may want to install the latest version of PowerShell (PowerShell Core) for additional features and cross-platform support.

  1. Download PowerShell Core:

    • Visit the PowerShell GitHub releases page.
    • Download the latest .msi package for Windows.

  2. Install PowerShell Core:

    • Run the downloaded .msi file.
    • Follow the installation wizard steps.
    • Ensure the option to add PowerShell to the system PATH is checked.

  3. Verify Installation:

    • Open a command prompt or the existing Windows PowerShell.
    • Type pwsh and press Enter.
    • You should see the PowerShell Core prompt, indicating successful installation.

macOS

PowerShell Core is available for macOS and can be installed using Homebrew.

  1. Install Homebrew (if not already installed):

    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

  2. Install PowerShell Core:

    brew install --cask powershell

  3. Verify Installation:

    • Open Terminal.
    • Type pwsh and press Enter.
    • You should see the PowerShell Core prompt.

Linux

PowerShell Core can be installed on various Linux distributions. Below are the steps for Ubuntu:

  1. Download and Install the Microsoft repository GPG keys:

    wget -q https://packages.microsoft.com/config/ubuntu/20.04/packages-microsoft-prod.deb
sudo dpkg -i packages-microsoft-prod.deb

  2. Update the list of products:

    sudo apt-get update

  3. Install PowerShell:

    sudo apt-get install -y powershell

  4. Verify Installation:

    • Open Terminal.
    • Type pwsh and press Enter.
    • You should see the PowerShell Core prompt.

Setting Up PowerShell

Configuring Execution Policy

The execution policy determines the conditions under which PowerShell loads configuration files and runs scripts. By default, the execution policy is set to Restricted, which prevents any scripts from running.

  1. Check Current Execution Policy:

    Get-ExecutionPolicy

  2. Set Execution Policy:

    Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
    • RemoteSigned allows scripts created locally to run but requires scripts downloaded from the internet to be signed by a trusted publisher.
    • CurrentUser scope applies the policy to the current user only.

Installing PowerShell Modules

Modules are packages of cmdlets, providers, functions, workflows, variables, and aliases. The PowerShell Gallery is the central repository for PowerShell content.

  1. Install a Module:

    Install-Module -Name Az -Scope CurrentUser -Repository PSGallery -Force
    • Az is a module for managing Azure resources.
    • -Scope CurrentUser installs the module for the current user.
    • -Repository PSGallery specifies the source repository.

  2. Verify Module Installation:

    Get-Module -ListAvailable -Name Az

Customizing the PowerShell Profile

The PowerShell profile is a script that runs every time PowerShell starts. You can use it to customize your environment.

  1. Locate the Profile Path:

    $PROFILE

  2. Create or Edit the Profile:

    if (!(Test-Path -Path $PROFILE)) {
    New-Item -ItemType File -Path $PROFILE -Force
}
notepad $PROFILE

  3. Add Customizations:

    • Example: Adding an alias for a frequently used command.
    Set-Alias ll Get-ChildItem

Conclusion

By following the steps outlined in this section, you should now have PowerShell installed and configured on your system. You have also learned how to set the execution policy, install modules, and customize your PowerShell profile. With this setup, you are ready to start exploring and utilizing PowerShell for automation and scripting tasks.
