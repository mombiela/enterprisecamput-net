Introduction

PowerShell is a powerful scripting language and command-line shell designed for system administration and automation. One of its key strengths is the ability to gather and configure system information. This topic will guide you through the basics to advanced techniques for retrieving and managing system information using PowerShell.

Gathering System Information

Basic System Information

PowerShell provides several cmdlets to retrieve basic system information.

  • Get-ComputerInfo: Retrieves a comprehensive set of system properties.
  • Get-WmiObject: Retrieves management information from local and remote computers.
  • Get-Process: Retrieves information about running processes.

Examples:

# Retrieve comprehensive system information
Get-ComputerInfo

# Retrieve specific system information using WMI
Get-WmiObject -Class Win32_OperatingSystem

# Retrieve information about running processes
Get-Process

Hardware Information

You can gather detailed hardware information using specific WMI classes.

  • Win32_Processor: Information about the CPU.
  • Win32_PhysicalMemory: Information about the physical memory.
  • Win32_DiskDrive: Information about disk drives.

Examples:

# Retrieve CPU information
Get-WmiObject -Class Win32_Processor

# Retrieve physical memory information
Get-WmiObject -Class Win32_PhysicalMemory

# Retrieve disk drive information
Get-WmiObject -Class Win32_DiskDrive

Network Information

PowerShell can also be used to gather network-related information.

  • Get-NetIPAddress: Retrieves IP address configuration.
  • Get-NetAdapter: Retrieves network adapter information.
  • Test-Connection: Tests network connectivity.

Examples:

# Retrieve IP address configuration
Get-NetIPAddress

# Retrieve network adapter information
Get-NetAdapter

# Test network connectivity to a remote host
Test-Connection -ComputerName google.com

Configuring System Settings

Managing Services

PowerShell allows you to manage Windows services efficiently.

  • Get-Service: Retrieves the status of services.
  • Start-Service: Starts a service.
  • Stop-Service: Stops a service.
  • Restart-Service: Restarts a service.

Examples:

# Retrieve the status of all services
Get-Service

# Start a specific service
Start-Service -Name "wuauserv"

# Stop a specific service
Stop-Service -Name "wuauserv"

# Restart a specific service
Restart-Service -Name "wuauserv"

Managing Processes

You can also manage processes using PowerShell.

  • Start-Process: Starts a new process.
  • Stop-Process: Stops a running process.
  • Get-Process: Retrieves information about running processes.

Examples:

# Start a new process (e.g., Notepad)
Start-Process -FilePath "notepad.exe"

# Stop a running process by name
Stop-Process -Name "notepad"

# Retrieve information about running processes
Get-Process

Configuring Network Settings

PowerShell provides cmdlets to configure network settings.

  • New-NetIPAddress: Configures a new IP address.
  • Set-DnsClientServerAddress: Configures DNS server addresses.
  • Set-NetIPInterface: Configures IP interface properties.

Examples:

# Configure a new IP address
New-NetIPAddress -InterfaceAlias "Ethernet" -IPAddress "192.168.1.100" -PrefixLength 24 -DefaultGateway "192.168.1.1"

# Configure DNS server addresses
Set-DnsClientServerAddress -InterfaceAlias "Ethernet" -ServerAddresses ("8.8.8.8", "8.8.4.4")

# Configure IP interface properties
Set-NetIPInterface -InterfaceAlias "Ethernet" -Dhcp Disabled

Advanced Techniques

Using CIM Cmdlets

CIM (Common Information Model) cmdlets provide a more modern approach to accessing system information.

  • Get-CimInstance: Retrieves management information.
  • New-CimSession: Creates a new CIM session.
  • Invoke-CimMethod: Invokes a method on a CIM instance.

Examples:

# Retrieve information using CIM
Get-CimInstance -ClassName Win32_OperatingSystem

# Create a new CIM session
$cimSession = New-CimSession -ComputerName "RemoteComputer"

# Invoke a method on a CIM instance
Invoke-CimMethod -CimSession $cimSession -ClassName Win32_Process -MethodName Create -Arguments @{CommandLine="notepad.exe"}

Scripting and Automation

PowerShell scripts can automate the retrieval and configuration of system information.

Example Script:

# Script to gather and log system information
$systemInfo = Get-ComputerInfo
$cpuInfo = Get-WmiObject -Class Win32_Processor
$memoryInfo = Get-WmiObject -Class Win32_PhysicalMemory
$diskInfo = Get-WmiObject -Class Win32_DiskDrive

# Log information to a file
$logFile = "C:\SystemInfoLog.txt"
$systemInfo | Out-File -FilePath $logFile -Append
$cpuInfo | Out-File -FilePath $logFile -Append
$memoryInfo | Out-File -FilePath $logFile -Append
$diskInfo | Out-File -FilePath $logFile -Append

Conclusion

PowerShell is a versatile tool for gathering and configuring system information. From basic system properties to advanced hardware and network configurations, PowerShell cmdlets provide a powerful interface for system administration. By mastering these cmdlets and techniques, you can efficiently manage and automate system tasks, enhancing your productivity and system management capabilities.
