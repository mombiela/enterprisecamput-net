Switch Statements in PowerShell
Error Handling ►

Switch statements in PowerShell provide a way to execute different blocks of code based on the value of a variable or expression. They are particularly useful for handling multiple conditions more efficiently than using multiple if statements.

Key Concepts

  • Switch Statement Syntax: The basic structure of a switch statement.
  • Pattern Matching: Using wildcards and regular expressions within switch statements.
  • Multiple Conditions: Handling multiple cases and default cases.
  • Advanced Features: Using -file, -regex, and -wildcard options.

Switch Statement Syntax

The basic syntax of a switch statement in PowerShell is as follows:

switch ($variable) {
    case 'value1' {
        # Code to execute if $variable is 'value1'
    }
    case 'value2' {
        # Code to execute if $variable is 'value2'
    }
    default {
        # Code to execute if $variable does not match any case
    }
}

Example

$day = 'Monday'

switch ($day) {
    'Monday' { "Start of the work week." }
    'Friday' { "End of the work week." }
    default { "Midweek day." }
}

In this example:

  • If $day is 'Monday', it will output "Start of the work week."
  • If $day is 'Friday', it will output "End of the work week."
  • For any other value of $day, it will output "Midweek day."

Pattern Matching

PowerShell switch statements support pattern matching using wildcards and regular expressions.

Wildcards

$fruit = 'apple'

switch -Wildcard ($fruit) {
    'a*' { "Starts with 'a'" }
    '*e' { "Ends with 'e'" }
    default { "No match" }
}

Regular Expressions

$fruit = 'banana'

switch -Regex ($fruit) {
    '^b.*a$' { "Starts with 'b' and ends with 'a'" }
    'n{2}' { "Contains double 'n'" }
    default { "No match" }
}

Multiple Conditions

You can handle multiple conditions by specifying multiple values in a single case.

$number = 3

switch ($number) {
    1 { "One" }
    2 { "Two" }
    3 { "Three" }
    4 { "Four" }
    default { "Number not in range 1-4" }
}

Advanced Features

Using `-file`

The -file option allows you to read input from a file and process each line.

switch -File "C:\path\to\file.txt" {
    'Error' { "Found an error" }
    'Warning' { "Found a warning" }
    default { "General information" }
}

Using `-regex` and `-wildcard`

You can combine -regex and -wildcard options to handle more complex scenarios.

$input = 'error: file not found'

switch -Regex ($input) {
    'error:.*' { "An error occurred" }
    'warning:.*' { "A warning occurred" }
    default { "No issues found" }
}

Conclusion

Switch statements in PowerShell are a powerful tool for handling multiple conditions and pattern matching. They can simplify your scripts by reducing the need for multiple if statements and making your code more readable. By understanding the basic syntax, pattern matching capabilities, and advanced features, you can effectively use switch statements to control the flow of your PowerShell scripts.
