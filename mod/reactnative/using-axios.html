Introduction to Axios

Axios is a popular JavaScript library used to make HTTP requests from the browser and Node.js. It is widely used in React Native applications for its simplicity and ease of use.

Key Features of Axios

  • Promise-based: Axios uses Promises, making it easy to work with asynchronous requests.
  • Interceptors: Allows you to intercept requests or responses before they are handled.
  • Automatic JSON data transformation: Automatically transforms JSON data.
  • Error handling: Provides a simple way to handle errors.
  • Request cancellation: Supports request cancellation.

Installing Axios in a React Native Project

To use Axios in a React Native project, you need to install it via npm or yarn.

npm install axios
# or
yarn add axios

Making a Basic GET Request

To make a GET request using Axios, you can use the axios.get method. Below is a simple example of how to fetch data from an API.

import axios from 'axios';
import React, { useEffect, useState } from 'react';
import { View, Text } from 'react-native';

const App = () => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    axios.get('https://jsonplaceholder.typicode.com/posts/1')
      .then(response => {
        setData(response.data);
        setLoading(false);
      })
      .catch(error => {
        console.error('Error fetching data:', error);
        setLoading(false);
      });
  }, []);

  if (loading) {
    return <Text>Loading...</Text>;
  }

  return (
    <View>
      <Text>{data.title}</Text>
      <Text>{data.body}</Text>
    </View>
  );
};

export default App;

Explanation

  • Importing Axios: import axios from 'axios';
  • State Management: Using useState to manage the fetched data and loading state.
  • Effect Hook: Using useEffect to fetch data when the component mounts.
  • GET Request: axios.get to fetch data from the API.
  • Handling Response: Updating the state with the fetched data.
  • Error Handling: Logging errors to the console.

Making a POST Request

To send data to an API, you can use the axios.post method. Below is an example of how to send data to an API.

import axios from 'axios';
import React, { useState } from 'react';
import { View, Text, Button, TextInput } from 'react-native';

const App = () => {
  const [title, setTitle] = useState('');
  const [body, setBody] = useState('');
  const [response, setResponse] = useState(null);

  const handleSubmit = () => {
    axios.post('https://jsonplaceholder.typicode.com/posts', {
      title,
      body,
      userId: 1,
    })
    .then(response => {
      setResponse(response.data);
    })
    .catch(error => {
      console.error('Error posting data:', error);
    });
  };

  return (
    <View>
      <TextInput
        placeholder="Title"
        value={title}
        onChangeText={setTitle}
      />
      <TextInput
        placeholder="Body"
        value={body}
        onChangeText={setBody}
      />
      <Button title="Submit" onPress={handleSubmit} />
      {response && (
        <View>
          <Text>Response:</Text>
          <Text>{response.title}</Text>
          <Text>{response.body}</Text>
        </View>
      )}
    </View>
  );
};

export default App;

Explanation

  • State Management: Using useState to manage form inputs and response data.
  • POST Request: axios.post to send data to the API.
  • Handling Response: Updating the state with the response data.
  • Error Handling: Logging errors to the console.

Using Axios Interceptors

Interceptors allow you to run your code or modify the request/response before it is handled by then or catch.

Adding a Request Interceptor

axios.interceptors.request.use(
  config => {
    // Do something before request is sent
    console.log('Request Interceptor:', config);
    return config;
  },
  error => {
    // Do something with request error
    return Promise.reject(error);
  }
);

Adding a Response Interceptor

axios.interceptors.response.use(
  response => {
    // Do something with response data
    console.log('Response Interceptor:', response);
    return response;
  },
  error => {
    // Do something with response error
    return Promise.reject(error);
  }
);

Explanation

  • Request Interceptor: Allows you to modify the request before it is sent.
  • Response Interceptor: Allows you to modify the response before it is handled.

Error Handling in Axios

Error handling is crucial for a robust application. Axios provides a simple way to handle errors.

Example of Error Handling

axios.get('https://jsonplaceholder.typicode.com/posts/1')
  .then(response => {
    console.log('Data:', response.data);
  })
  .catch(error => {
    if (error.response) {
      // Server responded with a status other than 200 range
      console.error('Response Error:', error.response.data);
    } else if (error.request) {
      // Request was made but no response received
      console.error('Request Error:', error.request);
    } else {
      // Something else happened
      console.error('Error:', error.message);
    }
  });

Explanation

  • Error Types:
    • Response Error: Error response from the server.
    • Request Error: No response received from the server.
    • Other Errors: Any other errors.

Conclusion

Axios is a powerful and easy-to-use library for making HTTP requests in React Native applications. It simplifies the process of fetching and sending data, handling errors, and managing request/response interceptors. By understanding and utilizing Axios, you can enhance the network capabilities of your React Native applications, making them more robust and efficient.
