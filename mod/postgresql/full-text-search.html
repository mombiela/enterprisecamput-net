Introduction to Full-Text Search

Full-text search (FTS) in PostgreSQL allows you to efficiently search for documents or text within a database. Unlike simple pattern matching with LIKE, FTS can handle complex queries and rank results based on relevance.

Key Concepts

  • Tokenization: Breaking down text into tokens (words or terms).
  • Normalization: Converting tokens to a standard form (e.g., lowercasing, stemming).
  • Dictionaries: Used to define how tokens are processed.
  • TSVector: A data type that stores preprocessed text for efficient searching.
  • TSQuery: A data type that represents a search query.

Setting Up Full-Text Search

Creating a TSVector Column

To use FTS, you need to create a column that stores the preprocessed text.

ALTER TABLE articles ADD COLUMN tsv tsvector;
UPDATE articles SET tsv = to_tsvector('english', title || ' ' || body);

Indexing the TSVector Column

Creating an index on the tsvector column improves search performance.

CREATE INDEX idx_fts ON articles USING gin(tsv);

Performing Full-Text Searches

Basic Search

You can perform a basic full-text search using the @@ operator.

SELECT title, body
FROM articles
WHERE tsv @@ to_tsquery('english', 'postgresql & search');

Ranking Search Results

To rank search results by relevance, use the ts_rank function.

SELECT title, body, ts_rank(tsv, to_tsquery('english', 'postgresql & search')) AS rank
FROM articles
WHERE tsv @@ to_tsquery('english', 'postgresql & search')
ORDER BY rank DESC;

Advanced Full-Text Search Techniques

Using Multiple Dictionaries

You can use multiple dictionaries to handle different languages or specialized vocabularies.

CREATE TEXT SEARCH CONFIGURATION my_config (COPY = pg_catalog.english);
ALTER TEXT SEARCH CONFIGURATION my_config
    ALTER MAPPING FOR hword, hword_part, word
    WITH simple, english_stem;

Highlighting Search Terms

Highlighting search terms in the results can make it easier to see why a document was matched.

SELECT title, ts_headline('english', body, to_tsquery('english', 'postgresql & search')) AS highlighted_body
FROM articles
WHERE tsv @@ to_tsquery('english', 'postgresql & search');

Conclusion

Full-text search in PostgreSQL is a powerful feature that allows for efficient and complex text searches. By understanding and utilizing tokenization, normalization, and the tsvector and tsquery data types, you can create robust search functionalities in your applications. Remember to index your tsvector columns and consider using advanced techniques like multiple dictionaries and term highlighting to enhance your search capabilities.
