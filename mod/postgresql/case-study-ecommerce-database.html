Introduction

In this case study, we will explore how to design and implement an e-commerce database using PostgreSQL. This will cover everything from setting up the database to advanced querying techniques. By the end of this topic, you should have a comprehensive understanding of how to manage an e-commerce database effectively.

Setting Up the Database

Creating the Database

First, we need to create a new database for our e-commerce application.

CREATE DATABASE ecommerce_db;

Connecting to the Database

Next, connect to the newly created database.

\c ecommerce_db;

Designing the Schema

Tables and Relationships

An e-commerce database typically includes several key tables:

  • Users: Information about customers.
  • Products: Details of items available for sale.
  • Orders: Records of customer purchases.
  • OrderItems: Items included in each order.
  • Categories: Product categories.

Creating Tables

Let's create these tables with appropriate relationships.

Users Table

CREATE TABLE Users (
    user_id SERIAL PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    email VARCHAR(100) NOT NULL UNIQUE,
    password VARCHAR(100) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Products Table

CREATE TABLE Products (
    product_id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    price DECIMAL(10, 2) NOT NULL,
    category_id INT,
    FOREIGN KEY (category_id) REFERENCES Categories(category_id)
);

Orders Table

CREATE TABLE Orders (
    order_id SERIAL PRIMARY KEY,
    user_id INT NOT NULL,
    order_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(50) NOT NULL,
    FOREIGN KEY (user_id) REFERENCES Users(user_id)
);

OrderItems Table

CREATE TABLE OrderItems (
    order_item_id SERIAL PRIMARY KEY,
    order_id INT NOT NULL,
    product_id INT NOT NULL,
    quantity INT NOT NULL,
    price DECIMAL(10, 2) NOT NULL,
    FOREIGN KEY (order_id) REFERENCES Orders(order_id),
    FOREIGN KEY (product_id) REFERENCES Products(product_id)
);

Categories Table

CREATE TABLE Categories (
    category_id SERIAL PRIMARY KEY,
    name VARCHAR(50) NOT NULL
);

Populating the Database

Inserting Data

Let's insert some sample data into our tables.

Insert Categories

INSERT INTO Categories (name) VALUES ('Electronics'), ('Books'), ('Clothing');

Insert Products

INSERT INTO Products (name, description, price, category_id) VALUES
('Laptop', 'A high-performance laptop', 999.99, 1),
('Smartphone', 'A latest model smartphone', 699.99, 1),
('Novel', 'A best-selling novel', 19.99, 2);

Insert Users

INSERT INTO Users (username, email, password) VALUES
('john_doe', 'john@example.com', 'password123'),
('jane_smith', 'jane@example.com', 'password456');

Insert Orders

INSERT INTO Orders (user_id, status) VALUES
(1, 'Pending'),
(2, 'Completed');

Insert OrderItems

INSERT INTO OrderItems (order_id, product_id, quantity, price) VALUES
(1, 1, 1, 999.99),
(2, 3, 2, 39.98);

Querying the Database

Basic Queries

Retrieve all users:

SELECT * FROM Users;

Retrieve all products in the 'Electronics' category:

SELECT * FROM Products WHERE category_id = 1;

Advanced Queries

Retrieve all orders along with user information:

SELECT Orders.order_id, Users.username, Orders.order_date, Orders.status
FROM Orders
JOIN Users ON Orders.user_id = Users.user_id;

Retrieve total sales for each product:

SELECT Products.name, SUM(OrderItems.quantity * OrderItems.price) AS total_sales
FROM OrderItems
JOIN Products ON OrderItems.product_id = Products.product_id
GROUP BY Products.name;

Conclusion

In this case study, we have designed and implemented an e-commerce database using PostgreSQL. We covered the creation of tables, insertion of data, and various querying techniques. This foundational knowledge will help you manage and query complex databases effectively in real-world applications.
