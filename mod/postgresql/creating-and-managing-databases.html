Creating Databases

  • Creating a Database:

    CREATE DATABASE mydatabase;

    Explanation: This command creates a new database named mydatabase.

  • Listing Databases:

    \l

    Explanation: This command lists all databases in the PostgreSQL instance.

  • Connecting to a Database:

    \c mydatabase

    Explanation: This command connects to the mydatabase database.

Managing Databases

  • Renaming a Database:

    ALTER DATABASE mydatabase RENAME TO newdatabase;

    Explanation: This command renames mydatabase to newdatabase.

  • Dropping a Database:

    DROP DATABASE mydatabase;

    Explanation: This command deletes the mydatabase database. Use with caution as this action is irreversible.

Users and Permissions

  • Creating a User:

    CREATE USER myuser WITH PASSWORD 'mypassword';

    Explanation: This command creates a new user myuser with the password mypassword.

  • Granting Permissions:

    GRANT ALL PRIVILEGES ON DATABASE mydatabase TO myuser;

    Explanation: This command grants all privileges on mydatabase to myuser.

  • Revoking Permissions:

    REVOKE ALL PRIVILEGES ON DATABASE mydatabase FROM myuser;

    Explanation: This command revokes all privileges on mydatabase from myuser.

Tables and Schemas

  • Creating a Table:

    CREATE TABLE employees (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100),
    position VARCHAR(50),
    salary NUMERIC
);

    Explanation: This command creates a table named employees with columns for id, name, position, and salary.

  • Listing Tables:

    \dt

    Explanation: This command lists all tables in the current database.

  • Creating a Schema:

    CREATE SCHEMA myschema;

    Explanation: This command creates a new schema named myschema.

  • Using a Schema:

    SET search_path TO myschema;

    Explanation: This command sets the search path to myschema, making it the default schema for subsequent commands.

Advanced Database Management

  • Backup and Restore:

    • Backup a Database:

      pg_dump mydatabase > mydatabase_backup.sql

      Explanation: This command creates a backup of mydatabase and saves it to mydatabase_backup.sql.

    • Restore a Database:

      psql mydatabase < mydatabase_backup.sql

      Explanation: This command restores mydatabase from the mydatabase_backup.sql file.

  • Replication and Clustering:

    • Overview of replication
    • Setting up master-slave replication
    • Introduction to clustering

Performance Tuning

  • Indexing:

    CREATE INDEX idx_name ON employees (name);

    Explanation: This command creates an index on the name column of the employees table to improve query performance.

  • Query Optimization:

    • Using EXPLAIN to analyze query plans: 
      EXPLAIN SELECT * FROM employees WHERE name = 'John Doe';
      Explanation: This command shows the execution plan for the query, helping to identify performance bottlenecks.

  • Vacuuming:

    VACUUM FULL;

    Explanation: This command reclaims storage and optimizes the database by removing dead tuples.

Conclusion

In this topic, we covered the essentials of creating and managing databases in PostgreSQL, from basic database creation to advanced performance tuning techniques. By understanding these concepts, you can effectively manage your PostgreSQL databases, ensuring they are secure, efficient, and scalable.
