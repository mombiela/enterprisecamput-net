Foreign Data Wrappers
Custom Data Types ►

Foreign Data Wrappers (FDWs) are a powerful feature in PostgreSQL that allow you to access and manipulate data stored in external databases as if it were part of your PostgreSQL database. This topic will guide you through the basics to advanced usage of FDWs.

Introduction to Foreign Data Wrappers

  • Definition: Foreign Data Wrappers are a PostgreSQL feature that allows you to query and interact with external data sources.
  • Use Cases:
    • Integrating data from different databases.
    • Accessing data from non-relational sources.
    • Performing federated queries across multiple data sources.

Setting Up Foreign Data Wrappers

Installing the Required Extension

To use FDWs, you need to install the appropriate extension. For example, to connect to another PostgreSQL database, you would use the postgres_fdw extension.

CREATE EXTENSION IF NOT EXISTS postgres_fdw;

Creating a Foreign Server

A foreign server defines the connection details to the external data source.

CREATE SERVER foreign_server
FOREIGN DATA WRAPPER postgres_fdw
OPTIONS (host 'remote_host', dbname 'remote_db', port '5432');

Creating User Mappings

User mappings define the authentication details for accessing the foreign server.

CREATE USER MAPPING FOR local_user
SERVER foreign_server
OPTIONS (user 'remote_user', password 'remote_password');

Creating Foreign Tables

Foreign tables represent the external data within your PostgreSQL database.

CREATE FOREIGN TABLE foreign_table (
    id integer,
    name text,
    value numeric
)
SERVER foreign_server
OPTIONS (schema_name 'public', table_name 'remote_table');

Querying Foreign Tables

Once the foreign table is set up, you can query it just like any other table in your PostgreSQL database.

SELECT * FROM foreign_table;

Advanced Usage

Importing Foreign Schema

You can import an entire schema from the foreign server.

IMPORT FOREIGN SCHEMA public
FROM SERVER foreign_server
INTO local_schema;

Joining Local and Foreign Tables

You can perform joins between local and foreign tables.

SELECT local_table.id, local_table.name, foreign_table.value
FROM local_table
JOIN foreign_table ON local_table.id = foreign_table.id;

Performance Considerations

  • Pushdown Capabilities: FDWs can push down certain operations (like filters and joins) to the foreign server to improve performance.
  • Network Latency: Be mindful of network latency when querying large datasets from remote servers.

Supported Foreign Data Wrappers

PostgreSQL supports a variety of FDWs for different data sources. Here is a comparison of some popular FDWs:

| FDW | Data Source | Use Case | |-----------------|-------------------|-------------------------------------------| | postgres_fdw | PostgreSQL | Accessing other PostgreSQL databases | | mysql_fdw | MySQL | Integrating MySQL databases | | oracle_fdw | Oracle | Accessing Oracle databases | | file_fdw | CSV/Flat Files | Reading data from CSV or flat files | | mongodb_fdw | MongoDB | Integrating MongoDB collections |

Conclusion

Foreign Data Wrappers are a versatile tool in PostgreSQL that enable seamless integration with various external data sources. By understanding how to set up and use FDWs, you can extend the capabilities of your PostgreSQL database and perform complex queries across different data systems. Whether you're joining data from multiple databases or accessing non-relational data, FDWs provide a robust solution for your data integration needs.
