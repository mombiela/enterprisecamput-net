String Functions in PostgreSQL
User-Defined Functions ►

String functions are essential tools in PostgreSQL for manipulating and querying text data. This section will cover various string functions from basic to advanced levels, providing clear explanations, examples, and exercises.

Basic String Functions

`LENGTH()`

  • Description: Returns the number of characters in a string.
  • Syntax: LENGTH(string)
SELECT LENGTH('PostgreSQL'); -- Returns 10

`UPPER()` and `LOWER()`

  • Description: Converts a string to uppercase or lowercase.
  • Syntax: UPPER(string), LOWER(string)
SELECT UPPER('postgresql'); -- Returns 'POSTGRESQL'
SELECT LOWER('PostgreSQL'); -- Returns 'postgresql'

`TRIM()`

  • Description: Removes leading and trailing spaces from a string.
  • Syntax: TRIM(string)
SELECT TRIM('  PostgreSQL  '); -- Returns 'PostgreSQL'

`SUBSTRING()`

  • Description: Extracts a substring from a string.
  • Syntax: SUBSTRING(string FROM start FOR length)
SELECT SUBSTRING('PostgreSQL' FROM 1 FOR 4); -- Returns 'Post'

Intermediate String Functions

`POSITION()`

  • Description: Finds the position of a substring within a string.
  • Syntax: POSITION(substring IN string)
SELECT POSITION('SQL' IN 'PostgreSQL'); -- Returns 8

`CONCAT()`

  • Description: Concatenates two or more strings.
  • Syntax: CONCAT(string1, string2, ...)
SELECT CONCAT('Post', 'gre', 'SQL'); -- Returns 'PostgreSQL'

`REPLACE()`

  • Description: Replaces all occurrences of a substring within a string with another substring.
  • Syntax: REPLACE(string, old_substring, new_substring)
SELECT REPLACE('PostgreSQL', 'Post', 'My'); -- Returns 'MygreSQL'

`LPAD()` and `RPAD()`

  • Description: Pads a string on the left or right side to a specified length.
  • Syntax: LPAD(string, length, pad_string), RPAD(string, length, pad_string)
SELECT LPAD('SQL', 5, '0'); -- Returns '00SQL'
SELECT RPAD('SQL', 5, '0'); -- Returns 'SQL00'

Advanced String Functions

`REGEXP_MATCHES()`

  • Description: Returns all matches of a regular expression pattern in a string.
  • Syntax: REGEXP_MATCHES(string, pattern)
SELECT REGEXP_MATCHES('abc123xyz', '[0-9]+'); -- Returns {123}

`SPLIT_PART()`

  • Description: Splits a string on a specified delimiter and returns the nth substring.
  • Syntax: SPLIT_PART(string, delimiter, field)
SELECT SPLIT_PART('PostgreSQL is awesome', ' ', 2); -- Returns 'is'

`TO_CHAR()`

  • Description: Converts various data types to a string formatted according to a specified format.
  • Syntax: TO_CHAR(value, format)
SELECT TO_CHAR(12345, '99999'); -- Returns '12345'
SELECT TO_CHAR(current_timestamp, 'YYYY-MM-DD'); -- Returns the current date in 'YYYY-MM-DD' format

Exercises

  1. Basic Exercise: Write a query to convert the string 'hello world' to uppercase.

    SELECT UPPER('hello world');

  2. Intermediate Exercise: Write a query to replace 'SQL' with 'Database' in the string 'PostgreSQL'.

    SELECT REPLACE('PostgreSQL', 'SQL', 'Database');

  3. Advanced Exercise: Write a query to extract the year from the current date using TO_CHAR().

    SELECT TO_CHAR(current_date, 'YYYY');

Conclusion

String functions in PostgreSQL are powerful tools for text manipulation and querying. From basic functions like LENGTH() and UPPER() to advanced functions like REGEXP_MATCHES() and TO_CHAR(), mastering these functions will enhance your ability to handle and analyze text data effectively. Practice the provided exercises to solidify your understanding and apply these functions in real-world scenarios.
