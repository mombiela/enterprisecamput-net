User Management and Roles in PostgreSQL
Securing Connections ►

Introduction

User management and roles are essential components of database security and administration in PostgreSQL. This section will guide you through the basics of creating and managing users, assigning roles, and setting permissions from beginner to advanced levels.

Creating Users

Creating users in PostgreSQL involves using the CREATE USER command. Users are entities that can connect to the database and perform operations.

Basic User Creation

To create a new user, you can use the following syntax:

CREATE USER username WITH PASSWORD 'password';

Example:

CREATE USER john_doe WITH PASSWORD 'securepassword123';
  • username: The name of the user you want to create.
  • password: The password for the user.

Viewing Users

To list all users in the PostgreSQL database, you can query the pg_user table:

SELECT * FROM pg_user;

Managing Roles

Roles in PostgreSQL are a way to group users and manage permissions more efficiently. Roles can be assigned to users, and permissions can be granted to roles.

Creating Roles

To create a new role, use the CREATE ROLE command:

CREATE ROLE role_name;

Example:

CREATE ROLE read_only;

Assigning Roles to Users

To assign a role to a user, use the GRANT command:

GRANT role_name TO username;

Example:

GRANT read_only TO john_doe;

Viewing Roles

To list all roles in the PostgreSQL database, you can query the pg_roles table:

SELECT * FROM pg_roles;

Setting Permissions

Permissions determine what actions a user or role can perform on database objects like tables, views, and sequences.

Granting Permissions

To grant permissions, use the GRANT command. Here are some common permissions:

  • SELECT: Allows reading data from a table.
  • INSERT: Allows inserting data into a table.
  • UPDATE: Allows updating data in a table.
  • DELETE: Allows deleting data from a table.

Example:

GRANT SELECT, INSERT ON table_name TO role_name;

Revoking Permissions

To revoke permissions, use the REVOKE command:

REVOKE permission ON table_name FROM role_name;

Example:

REVOKE INSERT ON employees FROM read_only;

Advanced User Management

Advanced user management involves more complex scenarios like creating superusers, managing user attributes, and using schemas for better organization.

Creating Superusers

Superusers have all privileges and can perform any action in the database. Use the CREATE USER command with the SUPERUSER attribute:

CREATE USER admin WITH PASSWORD 'adminpassword' SUPERUSER;

Managing User Attributes

You can set various attributes for users, such as connection limits and expiration dates.

Example:

ALTER USER john_doe WITH CONNECTION LIMIT 5 VALID UNTIL '2023-12-31';

Using Schemas

Schemas help organize database objects and manage permissions at a higher level.

Example:

CREATE SCHEMA marketing;
GRANT USAGE ON SCHEMA marketing TO read_only;

Summary

In this section, we covered the essentials of user management and roles in PostgreSQL, from creating users and roles to setting and revoking permissions. Understanding these concepts is crucial for maintaining a secure and well-organized database environment. By mastering user management and roles, you can ensure that your PostgreSQL database is both secure and efficient.
