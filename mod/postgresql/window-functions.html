Window Functions in PostgreSQL
Common Table Expressions (CTEs) ►

Window functions are a powerful feature in PostgreSQL that allow you to perform calculations across a set of table rows that are somehow related to the current row. They are particularly useful for tasks such as ranking, running totals, moving averages, and more.

Introduction to Window Functions

  • Definition: Window functions perform calculations across a set of table rows that are related to the current row.
  • Use Cases: Ranking, running totals, moving averages, cumulative sums, etc.
  • Syntax: The basic syntax for a window function is: 
    function_name(expression) OVER (
  [PARTITION BY partition_expression]
  [ORDER BY sort_expression]
  [frame_clause]
)

Basic Components of Window Functions

PARTITION BY Clause

  • Purpose: Divides the result set into partitions to which the window function is applied.
  • Example: 
    SELECT
  department,
  employee,
  salary,
  SUM(salary) OVER (PARTITION BY department) AS department_total
FROM
  employees;
    • Explanation: This query calculates the total salary for each department.

ORDER BY Clause

  • Purpose: Defines the order of rows within each partition.
  • Example: 
    SELECT
  department,
  employee,
  salary,
  RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS rank
FROM
  employees;
    • Explanation: This query ranks employees within each department based on their salary in descending order.

Frame Clause

  • Purpose: Defines a subset of rows within the partition for the window function to operate on.
  • Types:
    • ROWS: Specifies a frame based on physical row counts.
    • RANGE: Specifies a frame based on logical ranges of values.
  • Example: 
    SELECT
  employee,
  salary,
  SUM(salary) OVER (ORDER BY salary ROWS BETWEEN 1 PRECEDING AND 1 FOLLOWING) AS moving_sum
FROM
  employees;
    • Explanation: This query calculates a moving sum of salaries, including the current row, the previous row, and the next row.

Common Window Functions

Aggregate Functions

  • SUM(): Calculates the sum of values.
  • AVG(): Calculates the average of values.
  • MIN(): Finds the minimum value.
  • MAX(): Finds the maximum value.
  • COUNT(): Counts the number of values.

Ranking Functions

  • RANK(): Assigns a rank to each row within the partition of a result set.
  • DENSE_RANK(): Similar to RANK(), but without gaps in ranking values.
  • ROW_NUMBER(): Assigns a unique number to each row within the partition.

Value Functions

  • LAG(): Accesses data from a previous row in the same result set.
  • LEAD(): Accesses data from a subsequent row in the same result set.
  • FIRST_VALUE(): Returns the first value in an ordered set of values.
  • LAST_VALUE(): Returns the last value in an ordered set of values.

Advanced Examples

Running Total

SELECT
  employee,
  salary,
  SUM(salary) OVER (ORDER BY employee) AS running_total
FROM
  employees;
  • Explanation: This query calculates a running total of salaries ordered by employee.

Moving Average

SELECT
  employee,
  salary,
  AVG(salary) OVER (ORDER BY employee ROWS BETWEEN 2 PRECEDING AND 2 FOLLOWING) AS moving_avg
FROM
  employees;
  • Explanation: This query calculates a moving average of salaries, considering the current row and two preceding and two following rows.

Cumulative Distribution

SELECT
  employee,
  salary,
  CUME_DIST() OVER (ORDER BY salary) AS cumulative_distribution
FROM
  employees;
  • Explanation: This query calculates the cumulative distribution of salaries.

Conclusion

Window functions in PostgreSQL provide a versatile and powerful way to perform complex calculations across sets of table rows. By understanding and utilizing the PARTITION BY, ORDER BY, and frame clauses, you can unlock new possibilities for data analysis and reporting. Practice using these functions with different datasets to become proficient in their application.
