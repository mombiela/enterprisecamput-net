Introduction to Vuex

Vuex is a state management pattern + library for Vue.js applications. It serves as a centralized store for all the components in an application, with rules ensuring that the state can only be mutated in a predictable fashion.

Key Concepts

  • State: The single source of truth.
  • Getters: The equivalent of computed properties for stores.
  • Mutations: The only way to actually change state in a Vuex store.
  • Actions: Similar to mutations, but instead of mutating the state, actions commit mutations.
  • Modules: Allow you to divide the store into multiple modules, each with its own state, mutations, actions, and getters.

Setting Up Vuex

To integrate Vuex into your Vue.js project, you need to install it first.

npm install vuex --save

Then, create a store and integrate it into your Vue instance.

// store.js
import Vue from 'vue';
import Vuex from 'vuex';

Vue.use(Vuex);

export default new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++;
    }
  },
  actions: {
    increment(context) {
      context.commit('increment');
    }
  },
  getters: {
    count: state => state.count
  }
});
// main.js
import Vue from 'vue';
import App from './App.vue';
import store from './store';

new Vue({
  render: h => h(App),
  store
}).$mount('#app');

State Management

State in Vuex is a single object that contains all the application-level state.

const store = new Vuex.Store({
  state: {
    count: 0
  }
});

Accessing State

You can access the state in your components using this.$store.state.

<template>
  <div>{{ count }}</div>
</template>

<script>
export default {
  computed: {
    count() {
      return this.$store.state.count;
    }
  }
};
</script>

Mutations

Mutations are the only way to actually change state in a Vuex store. Each mutation has a string type and a handler.

const store = new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++;
    }
  }
});

Committing Mutations

You can commit mutations using store.commit.

<template>
  <button @click="increment">Increment</button>
</template>

<script>
export default {
  methods: {
    increment() {
      this.$store.commit('increment');
    }
  }
};
</script>

Actions

Actions are similar to mutations, but instead of mutating the state, actions commit mutations. Actions can contain arbitrary asynchronous operations.

const store = new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++;
    }
  },
  actions: {
    incrementAsync({ commit }) {
      setTimeout(() => {
        commit('increment');
      }, 1000);
    }
  }
});

Dispatching Actions

You can dispatch actions using store.dispatch.

<template>
  <button @click="incrementAsync">Increment Async</button>
</template>

<script>
export default {
  methods: {
    incrementAsync() {
      this.$store.dispatch('incrementAsync');
    }
  }
};
</script>

Getters

Getters are the equivalent of computed properties for stores. They are used to compute derived state based on store state.

const store = new Vuex.Store({
  state: {
    count: 0
  },
  getters: {
    doubleCount: state => state.count * 2
  }
});

Accessing Getters

You can access getters in your components using this.$store.getters.

<template>
  <div>{{ doubleCount }}</div>
</template>

<script>
export default {
  computed: {
    doubleCount() {
      return this.$store.getters.doubleCount;
    }
  }
};
</script>

Modules

Modules allow you to divide your store into multiple modules, each with its own state, mutations, actions, and getters.

const moduleA = {
  state: { count: 0 },
  mutations: {
    increment(state) {
      state.count++;
    }
  },
  actions: {
    increment({ commit }) {
      commit('increment');
    }
  },
  getters: {
    doubleCount: state => state.count * 2
  }
};

const store = new Vuex.Store({
  modules: {
    a: moduleA
  }
});

Accessing Module State

You can access module state in your components using this.$store.state.moduleName.

<template>
  <div>{{ count }}</div>
</template>

<script>
export default {
  computed: {
    count() {
      return this.$store.state.a.count;
    }
  }
};
</script>

Conclusion

Integrating Vuex into your Vue.js application provides a robust solution for managing state across your application. By understanding and utilizing state, mutations, actions, getters, and modules, you can create scalable and maintainable applications. Practice by building small applications and gradually incorporating more complex state management to become proficient in using Vuex.
