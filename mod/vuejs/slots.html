Introduction to Slots

Slots are a powerful feature in Vue.js that allow you to create flexible and reusable components. They enable you to pass content from a parent component to a child component, making it easier to compose complex UIs.

Key Concepts

  • Slot: A placeholder in a child component where content from the parent component will be injected.
  • Default Slot: The simplest form of slot, used when no name is specified.
  • Named Slots: Slots that are identified by a name, allowing for multiple slots in a single component.
  • Scoped Slots: Slots that allow you to pass data from the child component back to the parent component.

Default Slots

Default slots are the most basic type of slots. They allow you to pass content from a parent component to a child component without specifying a name.

Example

<!-- ParentComponent.vue -->
<template>
  <ChildComponent>
    <p>This is some default slot content.</p>
  </ChildComponent>
</template>

<!-- ChildComponent.vue -->
<template>
  <div>
    <slot></slot>
  </div>
</template>

In this example, the <p> tag inside ParentComponent.vue is passed to ChildComponent.vue and rendered inside the <slot> tag.

Named Slots

Named slots allow you to define multiple slots within a single component. Each slot is identified by a unique name.

Example

<!-- ParentComponent.vue -->
<template>
  <ChildComponent>
    <template v-slot:header>
      <h1>This is the header content.</h1>
    </template>
    <template v-slot:footer>
      <p>This is the footer content.</p>
    </template>
  </ChildComponent>
</template>

<!-- ChildComponent.vue -->
<template>
  <div>
    <header>
      <slot name="header"></slot>
    </header>
    <footer>
      <slot name="footer"></slot>
    </footer>
  </div>
</template>

In this example, the header and footer slots in ChildComponent.vue are populated with content from ParentComponent.vue.

Scoped Slots

Scoped slots allow you to pass data from the child component back to the parent component. This is useful when the parent component needs to use data that is only available in the child component.

Example

<!-- ParentComponent.vue -->
<template>
  <ChildComponent>
    <template v-slot:default="slotProps">
      <p>{{ slotProps.message }}</p>
    </template>
  </ChildComponent>
</template>

<!-- ChildComponent.vue -->
<template>
  <div>
    <slot :message="message"></slot>
  </div>
</template>

<script>
export default {
  data() {
    return {
      message: 'Hello from the child component!'
    };
  }
};
</script>

In this example, the message data from ChildComponent.vue is passed to ParentComponent.vue through the scoped slot.

Conclusion

Slots in Vue.js provide a powerful way to create flexible and reusable components. By understanding default slots, named slots, and scoped slots, you can build more dynamic and maintainable applications. Practice using these different types of slots to become proficient in composing complex UIs in Vue.js.
