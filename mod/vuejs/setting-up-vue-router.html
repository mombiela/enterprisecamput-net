Introduction

Vue Router is the official router for Vue.js, enabling developers to create single-page applications with navigation and dynamic routing. This section will guide you through the process of setting up Vue Router in your Vue.js project.

Installing Vue Router

To start using Vue Router, you need to install it in your Vue.js project. You can do this using npm or yarn.

Using npm

npm install vue-router

Using yarn

yarn add vue-router

Basic Configuration

Once Vue Router is installed, you need to configure it in your project. This involves creating a router instance and defining your routes.

Creating a Router Instance

First, create a new file named router.js (or router/index.js if you prefer a nested structure) in your src directory.

// src/router.js
import Vue from 'vue';
import VueRouter from 'vue-router';
import Home from './components/Home.vue';
import About from './components/About.vue';

Vue.use(VueRouter);

const routes = [
  { path: '/', component: Home },
  { path: '/about', component: About }
];

const router = new VueRouter({
  routes
});

export default router;

Integrating the Router with Your Vue Instance

Next, you need to integrate the router with your main Vue instance. Open your main.js file and import the router.

// src/main.js
import Vue from 'vue';
import App from './App.vue';
import router from './router';

new Vue({
  router,
  render: h => h(App)
}).$mount('#app');

Defining Routes

Routes are the core of Vue Router. They map URLs to components.

Basic Route Definition

In the router.js file, you define routes as objects with path and component properties.

const routes = [
  { path: '/', component: Home },
  { path: '/about', component: About }
];

Named Routes

Named routes make it easier to link to routes programmatically.

const routes = [
  { path: '/', component: Home, name: 'home' },
  { path: '/about', component: About, name: 'about' }
];

Dynamic Routes

Dynamic routes allow you to capture parameters from the URL.

const routes = [
  { path: '/user/:id', component: User }
];

In the User component, you can access the route parameter id using this.$route.params.id.

Navigation

Vue Router provides several ways to navigate between routes.

Using ``

The <router-link> component is used to create navigation links.

<template>
  <div>
    <router-link to="/">Home</router-link>
    <router-link to="/about">About</router-link>
  </div>
</template>

Programmatic Navigation

You can navigate programmatically using the this.$router.push method.

methods: {
  goToAbout() {
    this.$router.push('/about');
  }
}

Nested Routes

Nested routes allow you to create complex route structures.

const routes = [
  {
    path: '/user/:id',
    component: User,
    children: [
      {
        path: 'profile',
        component: UserProfile
      },
      {
        path: 'posts',
        component: UserPosts
      }
    ]
  }
];

Route Guards

Route guards are used to control access to routes.

Global Guards

Global guards are applied to every route change.

router.beforeEach((to, from, next) => {
  if (to.path === '/about') {
    // Perform some check
    next();
  } else {
    next();
  }
});

Per-Route Guards

Per-route guards are defined directly in the route configuration.

const routes = [
  {
    path: '/about',
    component: About,
    beforeEnter: (to, from, next) => {
      // Perform some check
      next();
    }
  }
];

Summary

In this section, you learned how to set up Vue Router in a Vue.js project. We covered installation, basic configuration, defining routes, navigation, nested routes, and route guards. With this knowledge, you can now create dynamic, single-page applications with Vue.js and Vue Router.
