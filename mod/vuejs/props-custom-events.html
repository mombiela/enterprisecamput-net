Props and Custom Events in Vue.js
Slots ►

Introduction

In Vue.js, props and custom events are fundamental concepts that allow components to communicate with each other. Props enable parent components to pass data to child components, while custom events allow child components to send messages back to their parents. Understanding these concepts is crucial for building dynamic and interactive applications.

Props

What are Props?

  • Props are custom attributes you can register on a component.
  • They allow data to be passed from a parent component to a child component.
  • Props are read-only, meaning the child component cannot modify the data directly.

Defining Props

To define props in a Vue component, you use the props option. Here's an example:

<template>
  <div>
    <h1>{{ title }}</h1>
    <p>{{ description }}</p>
  </div>
</template>

<script>
export default {
  props: {
    title: {
      type: String,
      required: true
    },
    description: {
      type: String,
      required: false,
      default: 'No description provided'
    }
  }
}
</script>

In this example:

  • title is a required prop of type String.
  • description is an optional prop with a default value.

Passing Props

Props are passed to child components in the parent component's template:

<template>
  <div>
    <ChildComponent title="Hello World" description="This is a description"></ChildComponent>
  </div>
</template>

<script>
import ChildComponent from './ChildComponent.vue';

export default {
  components: {
    ChildComponent
  }
}
</script>

Prop Validation

Vue allows you to specify the type of props and whether they are required. This helps catch errors and makes your components more robust.

props: {
  title: {
    type: String,
    required: true
  },
  count: {
    type: Number,
    required: true
  }
}

Custom Events

What are Custom Events?

  • Custom events allow child components to communicate with their parent components.
  • They are useful for notifying the parent about changes or actions that occur in the child component.

Emitting Custom Events

To emit a custom event from a child component, you use the $emit method:

<template>
  <button @click="notifyParent">Click Me</button>
</template>

<script>
export default {
  methods: {
    notifyParent() {
      this.$emit('custom-event', 'Hello from Child');
    }
  }
}
</script>

In this example, when the button is clicked, the notifyParent method emits a custom event named custom-event with a message.

Listening to Custom Events

The parent component listens for custom events using the v-on directive or its shorthand @:

<template>
  <div>
    <ChildComponent @custom-event="handleCustomEvent"></ChildComponent>
  </div>
</template>

<script>
import ChildComponent from './ChildComponent.vue';

export default {
  components: {
    ChildComponent
  },
  methods: {
    handleCustomEvent(message) {
      console.log(message); // Output: Hello from Child
    }
  }
}
</script>

Passing Data with Custom Events

Custom events can carry data, which can be accessed by the parent component:

this.$emit('custom-event', { id: 1, name: 'Item' });

The parent component can then handle this data:

methods: {
  handleCustomEvent(data) {
    console.log(data.id); // Output: 1
    console.log(data.name); // Output: Item
  }
}

Conclusion

Props and custom events are essential for component communication in Vue.js. Props allow data to flow from parent to child components, while custom events enable child components to send messages back to their parents. By mastering these concepts, you can create more dynamic and interactive Vue applications. Practice using props and custom events in different scenarios to solidify your understanding and improve your Vue.js skills.
