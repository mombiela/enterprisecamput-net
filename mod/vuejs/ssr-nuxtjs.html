Introduction to Server-Side Rendering (SSR)

Server-Side Rendering (SSR) is a technique used to render web pages on the server instead of the client. This can improve performance, SEO, and user experience.

Key Concepts

  • Client-Side Rendering (CSR): The browser downloads a minimal HTML page and renders the content using JavaScript.
  • Server-Side Rendering (SSR): The server generates the full HTML for a page and sends it to the client.
  • Hydration: The process of making a server-rendered page interactive by attaching event listeners and other dynamic behavior.

Why Use SSR with Nuxt.js?

Nuxt.js is a framework built on top of Vue.js that simplifies the implementation of SSR. It provides a powerful set of features to build performant and SEO-friendly applications.

Benefits of SSR with Nuxt.js

  • Improved SEO: Search engines can index the fully rendered HTML.
  • Faster Time-to-Content: Users see the content faster because the server sends a fully rendered page.
  • Better Performance: Initial load times can be reduced, especially on slower devices.

Setting Up a Nuxt.js Project

To get started with SSR in Nuxt.js, you need to set up a Nuxt.js project.

Installation

First, install the Nuxt.js CLI if you haven't already:

npm install -g create-nuxt-app

Create a new Nuxt.js project:

npx create-nuxt-app my-nuxt-app

Follow the prompts to set up your project. Make sure to select "Universal (SSR / Static)" when asked about the rendering mode.

Project Structure

A typical Nuxt.js project has the following structure:

my-nuxt-app/
├── assets/
├── components/
├── layouts/
├── middleware/
├── pages/
├── plugins/
├── static/
├── store/
├── nuxt.config.js
  • assets/: Uncompiled assets such as Sass, images, or fonts.
  • components/: Vue components.
  • layouts/: Layouts for your pages.
  • middleware/: Custom functions to run before rendering a page.
  • pages/: Vue components for each route.
  • plugins/: JavaScript plugins to run before mounting the root Vue.js application.
  • static/: Static files directly accessible via the root URL.
  • store/: Vuex store files.

Creating Pages and Routes

Nuxt.js uses the pages/ directory to automatically generate routes.

Example Page

Create a file pages/index.vue:

<template>
  <div>
    <h1>Welcome to My Nuxt.js App</h1>
    <p>This page is server-side rendered.</p>
  </div>
</template>

<script>
export default {
  asyncData() {
    return {
      message: 'Hello from SSR!'
    }
  }
}
</script>

Explanation

  • Template: Defines the HTML structure.
  • Script: The asyncData method fetches data before rendering the page on the server.

Fetching Data with SSR

Nuxt.js provides several ways to fetch data for SSR, including asyncData and fetch.

Using `asyncData`

The asyncData method is called before rendering the page, allowing you to fetch data and make it available to the component.

<template>
  <div>
    <h1>{{ post.title }}</h1>
    <p>{{ post.body }}</p>
  </div>
</template>

<script>
import axios from 'axios';

export default {
  async asyncData({ params }) {
    const { data } = await axios.get(`https://jsonplaceholder.typicode.com/posts/${params.id}`);
    return { post: data };
  }
}
</script>

Using `fetch`

The fetch method is called on the server-side before rendering and on the client-side when navigating.

<template>
  <div>
    <h1>{{ post.title }}</h1>
    <p>{{ post.body }}</p>
  </div>
</template>

<script>
import axios from 'axios';

export default {
  data() {
    return {
      post: {}
    }
  },
  async fetch() {
    const { data } = await axios.get(`https://jsonplaceholder.typicode.com/posts/${this.$route.params.id}`);
    this.post = data;
  }
}
</script>

Middleware and Authentication

Nuxt.js allows you to define middleware to run custom logic before rendering a page.

Example Middleware

Create a file middleware/auth.js:

export default function ({ store, redirect }) {
  if (!store.state.authenticated) {
    return redirect('/login');
  }
}

Apply the middleware to a page:

<template>
  <div>
    <h1>Protected Page</h1>
  </div>
</template>

<script>
export default {
  middleware: 'auth'
}
</script>

Conclusion

Server-Side Rendering with Nuxt.js provides significant benefits for performance, SEO, and user experience. By leveraging Nuxt.js's powerful features, you can easily create SSR applications with Vue.js. This topic covered the basics of SSR, setting up a Nuxt.js project, creating pages and routes, fetching data, and using middleware for authentication. With these tools, you are well-equipped to build robust and performant SSR applications.
