Introduction to IAM

Identity and Access Management (IAM) is a critical component of Google Cloud Platform (GCP) that allows you to manage access to resources by defining who (identity) has what access (role) to which resources.

Key Concepts

  • Identity: Represents users, groups, or services that need access to GCP resources.
  • Role: A collection of permissions that can be assigned to identities.
  • Permission: Defines what actions can be performed on a resource.
  • Policy: A collection of bindings that associate roles with identities.

IAM Basics

Understanding Identities

  • Google Accounts: Individual users with a Google account.
  • Service Accounts: Special accounts used by applications or VMs to make authorized API calls.
  • Google Groups: Collections of Google accounts.
  • Cloud Identity Domains: Managed domains for organizations.

Understanding Roles

  • Primitive Roles: Basic roles that include Owner, Editor, and Viewer.
  • Predefined Roles: Granular roles created by Google that provide specific permissions.
  • Custom Roles: User-defined roles that provide fine-grained control over permissions.

Assigning Roles to Identities

To assign roles to identities, you create an IAM policy binding. Here’s an example of how to assign the Viewer role to a user:

bindings:
- members:
  - user:example-user@gmail.com
  role: roles/viewer

Intermediate IAM Concepts

Managing Service Accounts

Service accounts are crucial for applications running on GCP. Here’s how to create and manage them:

Creating a Service Account

gcloud iam service-accounts create my-service-account \
    --display-name "My Service Account"

Assigning Roles to a Service Account

gcloud projects add-iam-policy-binding my-project \
    --member "serviceAccount:my-service-account@my-project.iam.gserviceaccount.com" \
    --role "roles/editor"

Using IAM Policies

IAM policies define who has what type of access. Here’s an example of an IAM policy:

{
  "bindings": [
    {
      "role": "roles/viewer",
      "members": [
        "user:example-user@gmail.com"
      ]
    },
    {
      "role": "roles/editor",
      "members": [
        "serviceAccount:my-service-account@my-project.iam.gserviceaccount.com"
      ]
    }
  ]
}

Advanced IAM Concepts

Custom Roles

Custom roles allow you to define specific permissions tailored to your needs.

Creating a Custom Role

gcloud iam roles create myCustomRole \
    --project my-project \
    --title "My Custom Role" \
    --permissions compute.instances.start,compute.instances.stop \
    --stage GA

Auditing and Monitoring IAM

Monitoring IAM activities is essential for security and compliance.

Enabling Audit Logs

gcloud logging sinks create my-audit-logs \
    storage.googleapis.com/my-audit-logs-bucket \
    --log-filter='resource.type="gce_instance"'

Best Practices

  • Principle of Least Privilege: Grant only the permissions necessary for a user to perform their job.
  • Regular Audits: Regularly review IAM policies and roles.
  • Use Groups: Manage access by assigning roles to groups rather than individual users.

Conclusion

IAM in GCP is a powerful tool for managing access to your cloud resources. By understanding and utilizing identities, roles, and policies, you can ensure that your resources are secure and that users have the appropriate level of access. Remember to follow best practices such as the principle of least privilege and regular audits to maintain a secure environment.
