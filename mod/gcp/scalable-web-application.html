Example: Deploying a Flask App on App Engine

# app.py
from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello():
    return 'Hello, World!'

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=8080)
# app.yaml
runtime: python39
entrypoint: gunicorn -b :$PORT app:app
# Deploy the app
gcloud app deploy

5. Managing Databases

    • Cloud SQL vs. Firestore vs. BigQuery

Example: Connecting a Flask App to Cloud SQL

# app.py
import os
import sqlalchemy

db_user = os.environ.get('DB_USER')
db_pass = os.environ.get('DB_PASS')
db_name = os.environ.get('DB_NAME')
db_host = os.environ.get('DB_HOST')

db = sqlalchemy.create_engine(
    sqlalchemy.engine.url.URL(
        drivername='mysql+pymysql',
        username=db_user,
        password=db_pass,
        database=db_name,
        host=db_host
    )
)

@app.route('/users')
def users():
    with db.connect() as conn:
        result = conn.execute('SELECT * FROM users')
        users = [row['name'] for row in result]
    return ', '.join(users)

8. Advanced Topics

  • Using Kubernetes with GKE
  • Implementing CI/CD with Cloud Build

Example: Deploying a Containerized App with GKE

# Build the Docker image
docker build -t gcr.io/[PROJECT_ID]/my-app:v1 .

# Push the image to Container Registry
docker push gcr.io/[PROJECT_ID]/my-app:v1

# Create a GKE cluster
gcloud container clusters create my-cluster

# Deploy the app to GKE
kubectl run my-app --image=gcr.io/[PROJECT_ID]/my-app:v1 --port=8080
kubectl expose deployment my-app --type=LoadBalancer --port 80 --target-port 8080

By following these guidelines and examples, you will be well-equipped to create robust and scalable web applications on GCP.
