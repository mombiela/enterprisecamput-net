Firestore is a flexible, scalable database for mobile, web, and server development from Firebase and Google Cloud Platform (GCP). It is a NoSQL document database that lets you store, sync, and query data for your apps.

Introduction to Firestore

  • What is Firestore?

    • Firestore is a NoSQL document database.
    • It is part of the Firebase platform and integrates seamlessly with other Firebase and GCP services.
    • Firestore is designed to handle large-scale data and real-time updates.

  • Key Features:

    • Real-time synchronization.
    • Offline support.
    • Strong consistency.
    • Flexible data model.
    • Powerful querying capabilities.

Setting Up Firestore

  • Prerequisites:

    • A Google Cloud Platform account.
    • Firebase project setup.

  • Steps to Set Up Firestore:

    1. Create a Firebase Project:
      • Go to the Firebase Console.
      • Click on "Add project" and follow the setup instructions.
    2. Enable Firestore:
      • In the Firebase Console, navigate to the Database section.
      • Select Firestore and click "Create database".
      • Choose a location for your database and set up security rules.

Firestore Data Model

  • Documents and Collections:

    • Firestore stores data in documents, which are organized into collections.
    • A document is a set of key-value pairs.
    • Collections are containers for documents and can contain subcollections.

  • Example Structure:

    - users (collection)
  - userID (document)
    - name: "John Doe"
    - email: "john.doe@example.com"
    - posts (subcollection)
      - postID (document)
        - title: "My First Post"
        - content: "Hello, world!"

Basic Operations

  • Adding Data:

    // Initialize Firestore
const db = firebase.firestore();

// Add a new document in the "users" collection
db.collection("users").add({
    name: "John Doe",
    email: "john.doe@example.com"
})
.then((docRef) => {
    console.log("Document written with ID: ", docRef.id);
})
.catch((error) => {
    console.error("Error adding document: ", error);
});

  • Reading Data:

    // Get a document from the "users" collection
db.collection("users").doc("userID").get().then((doc) => {
    if (doc.exists) {
        console.log("Document data:", doc.data());
    } else {
        console.log("No such document!");
    }
}).catch((error) => {
    console.log("Error getting document:", error);
});

  • Updating Data:

    // Update an existing document in the "users" collection
db.collection("users").doc("userID").update({
    email: "new.email@example.com"
})
.then(() => {
    console.log("Document successfully updated!");
})
.catch((error) => {
    console.error("Error updating document: ", error);
});

  • Deleting Data:

    // Delete a document from the "users" collection
db.collection("users").doc("userID").delete().then(() => {
    console.log("Document successfully deleted!");
}).catch((error) => {
    console.error("Error removing document: ", error);
});

Advanced Features

  • Real-time Updates:

    // Listen for real-time updates to a document
db.collection("users").doc("userID")
  .onSnapshot((doc) => {
      console.log("Current data: ", doc.data());
  });

  • Queries:

    // Query documents in the "users" collection where the name is "John Doe"
db.collection("users").where("name", "==", "John Doe")
  .get()
  .then((querySnapshot) => {
      querySnapshot.forEach((doc) => {
          console.log(doc.id, " => ", doc.data());
      });
  })
  .catch((error) => {
      console.log("Error getting documents: ", error);
  });

  • Transactions:

    // Perform a transaction to update a document
db.runTransaction((transaction) => {
    return transaction.get(db.collection("users").doc("userID")).then((doc) => {
        if (!doc.exists) {
            throw "Document does not exist!";
        }

        // Update the email field
        transaction.update(db.collection("users").doc("userID"), { email: "new.email@example.com" });
    });
}).then(() => {
    console.log("Transaction successfully committed!");
}).catch((error) => {
    console.log("Transaction failed: ", error);
});

Security Rules

  • Setting Up Security Rules:

    • Firestore security rules control access to your database.
    • You can define rules to allow or deny access based on request properties.

  • Example Security Rule:

    service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}

Conclusion

Firestore is a powerful NoSQL database that offers real-time synchronization, offline support, and a flexible data model. By understanding its basic operations, advanced features, and security rules, you can effectively use Firestore to build scalable and responsive applications. Whether you're working on a small project or a large-scale application, Firestore provides the tools you need to manage your data efficiently.
