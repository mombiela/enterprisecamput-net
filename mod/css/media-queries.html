Introduction to Media Queries

Media Queries are a fundamental part of responsive web design, allowing you to apply CSS rules based on the characteristics of the device rendering the content, such as its width, height, or orientation.

Key Concepts

  • Responsive Design: Creating web pages that look good on all devices.
  • Viewport: The visible area of a web page on a device.
  • Breakpoints: Specific points where the layout of the web page changes to accommodate different screen sizes.

Basic Syntax

Media Queries use the @media rule to apply styles conditionally.

@media (condition) {
  /* CSS rules */
}

Example

/* Default styles */
body {
  background-color: white;
  color: black;
}

/* Styles for screens wider than 600px */
@media (min-width: 600px) {
  body {
    background-color: lightblue;
  }
}

In this example:

  • The default background color is white.
  • When the screen width is 600px or wider, the background color changes to lightblue.

Common Media Features

  • Width and Height: min-width, max-width, min-height, max-height
  • Orientation: orientation: portrait or orientation: landscape
  • Resolution: min-resolution, max-resolution

Example with Multiple Conditions

@media (min-width: 600px) and (orientation: landscape) {
  body {
    background-color: lightgreen;
  }
}

This rule applies when the screen is at least 600px wide and in landscape orientation.

Using Media Queries for Responsive Design

Breakpoints

Breakpoints are specific screen widths where the layout changes. Common breakpoints include:

  • 480px (small devices)
  • 768px (tablets)
  • 1024px (desktops)

Example with Multiple Breakpoints

/* Default styles */
body {
  font-size: 16px;
}

/* Small devices */
@media (max-width: 480px) {
  body {
    font-size: 14px;
  }
}

/* Tablets */
@media (min-width: 481px) and (max-width: 768px) {
  body {
    font-size: 15px;
  }
}

/* Desktops */
@media (min-width: 769px) {
  body {
    font-size: 18px;
  }
}

Advanced Media Queries

Combining Media Features

You can combine multiple media features using logical operators like and, not, and only.

Example with Logical Operators

/* Apply styles only if the device is in portrait mode and the width is at least 600px */
@media only screen and (min-width: 600px) and (orientation: portrait) {
  body {
    background-color: lightcoral;
  }
}

Media Queries for High-Resolution Displays

High-resolution displays, like Retina screens, require different styles for better clarity.

@media (min-resolution: 2dppx) {
  body {
    font-size: 20px;
  }
}

Practical Exercises

  1. Basic Media Query: Create a simple web page with a background color that changes when the screen width is 600px or wider.
  2. Responsive Text: Adjust the font size of a paragraph based on different screen widths (480px, 768px, 1024px).
  3. Orientation Change: Change the layout of a web page when the device orientation switches between portrait and landscape.

Conclusion

Media Queries are a powerful tool in CSS for creating responsive designs that adapt to various screen sizes and orientations. By understanding and using media queries effectively, you can ensure that your web pages provide a great user experience across all devices. Practice with different breakpoints and conditions to become proficient in responsive web design.
