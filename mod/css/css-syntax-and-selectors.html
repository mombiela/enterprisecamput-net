Introduction to CSS Syntax

CSS (Cascading Style Sheets) is a stylesheet language used to describe the presentation of a document written in HTML or XML. Understanding the basic syntax of CSS is crucial for styling web pages effectively.

Basic CSS Syntax

  • Selector: Indicates which HTML elements the style applies to.
  • Declaration Block: Contains one or more declarations separated by semicolons.
  • Declaration: Consists of a property and a value, separated by a colon.
selector {
  property: value;
}

Example

p {
  color: blue;
  font-size: 16px;
}

In this example:

  • p is the selector, targeting all <p> elements.
  • color and font-size are properties.
  • blue and 16px are the corresponding values.

Types of CSS Selectors

CSS selectors are used to select the HTML elements you want to style. There are several types of selectors, each with its own use case.

Basic Selectors

  • Universal Selector (*): Selects all elements.
  • Type Selector: Selects all elements of a given type.
  • Class Selector (.classname): Selects all elements with a specific class.
  • ID Selector (#idname): Selects a single element with a specific ID.
  • Attribute Selector: Selects elements based on an attribute or attribute value.

Examples

* {
  margin: 0;
  padding: 0;
}

h1 {
  color: red;
}

.container {
  width: 100%;
}

#header {
  background-color: #f1f1f1;
}

input[type="text"] {
  border: 1px solid #ccc;
}

Combinator Selectors

  • Descendant Selector (ancestor descendant): Selects all descendants of a specified element.
  • Child Selector (parent > child): Selects all direct children of a specified element.
  • Adjacent Sibling Selector (element + adjacent): Selects an element that is immediately preceded by a specified element.
  • General Sibling Selector (element ~ siblings): Selects all siblings of a specified element.

Examples

div p {
  color: green;
}

ul > li {
  list-style-type: none;
}

h1 + p {
  margin-top: 0;
}

h1 ~ p {
  color: gray;
}

Pseudo-classes and Pseudo-elements

  • Pseudo-classes: Used to define a special state of an element.
  • Pseudo-elements: Used to style specified parts of an element.

Examples

/* Pseudo-classes */
a:hover {
  color: orange;
}

input:focus {
  border-color: blue;
}

/* Pseudo-elements */
p::first-line {
  font-weight: bold;
}

div::before {
  content: "Note: ";
  color: red;
}

Grouping and Nesting Selectors

  • Grouping Selectors: Multiple selectors can share the same styles by separating them with commas.
  • Nesting Selectors: CSS does not support true nesting, but you can achieve similar results using combinators.

Examples

/* Grouping Selectors */
h1, h2, h3 {
  font-family: Arial, sans-serif;
}

/* Nesting Selectors */
nav ul {
  list-style-type: none;
}

nav ul li {
  display: inline;
}

Specificity and Inheritance

  • Specificity: Determines which styles are applied when multiple rules could apply to the same element.
  • Inheritance: Some CSS properties are inherited from parent to child elements, while others are not.

Specificity Calculation

  • Inline styles have the highest specificity.
  • ID selectors are more specific than class selectors.
  • Class selectors are more specific than type selectors.

Example

/* Specificity Example */
#unique {
  color: red; /* Highest specificity */
}

.container {
  color: blue; /* Medium specificity */
}

p {
  color: green; /* Lowest specificity */
}

If a <p> element is inside an element with the class container and has the ID unique, it will be red.

Conclusion

Understanding CSS syntax and selectors is fundamental for creating well-styled web pages. By mastering different types of selectors and their specificity, you can apply styles precisely and efficiently. Practice using these selectors in various scenarios to become proficient in CSS.

Summary

  • CSS syntax consists of selectors and declaration blocks.
  • Basic selectors include universal, type, class, ID, and attribute selectors.
  • Combinator selectors help target elements based on their relationships.
  • Pseudo-classes and pseudo-elements provide additional styling capabilities.
  • Grouping and nesting selectors can simplify your CSS.
  • Specificity and inheritance determine how styles are applied.

By grasping these concepts, you will be well on your way to becoming proficient in CSS.
