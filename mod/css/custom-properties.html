Introduction to Custom Properties

Custom properties, also known as CSS variables, allow you to store values that you can reuse throughout your stylesheet. This makes your CSS more maintainable and easier to read.

  • Definition: Custom properties are entities defined by CSS authors that contain specific values to be reused throughout a document.
  • Syntax: Custom properties are defined using the -- prefix and are accessed using the var() function.

Basic Syntax

To define a custom property, you use the -- prefix followed by the property name. You can then use the var() function to access the value of the custom property.

:root {
  --main-bg-color: #3498db;
  --main-text-color: #ffffff;
}

body {
  background-color: var(--main-bg-color);
  color: var(--main-text-color);
}
  • :root: This is a pseudo-class that matches the document's root element. It's often used to define global custom properties.
  • --main-bg-color: This is a custom property for the main background color.
  • var(--main-bg-color): This function retrieves the value of the custom property.

Advantages of Using Custom Properties

  • Reusability: Define a value once and reuse it throughout your stylesheet.
  • Maintainability: Change the value in one place, and it updates everywhere it's used.
  • Dynamic Changes: Custom properties can be updated dynamically using JavaScript.

Example: Reusability and Maintainability

:root {
  --primary-color: #e74c3c;
  --secondary-color: #2ecc71;
}

.header {
  background-color: var(--primary-color);
}

.footer {
  background-color: var(--secondary-color);
}

.button {
  background-color: var(--primary-color);
  color: var(--secondary-color);
}

In this example, changing --primary-color or --secondary-color in the :root will update the color scheme across all elements that use these custom properties.

Advanced Usage

Fallback Values

You can provide a fallback value in case the custom property is not defined.

.button {
  background-color: var(--nonexistent-color, #333);
}

In this case, if --nonexistent-color is not defined, the button's background color will default to #333.

Media Queries and Custom Properties

Custom properties can be used within media queries to create responsive designs.

:root {
  --font-size: 16px;
}

@media (min-width: 768px) {
  :root {
    --font-size: 18px;
  }
}

body {
  font-size: var(--font-size);
}

Here, the font size changes based on the viewport width, demonstrating how custom properties can adapt to different screen sizes.

JavaScript and Custom Properties

You can manipulate custom properties using JavaScript to create dynamic styles.

<!DOCTYPE html>
<html lang="en">
<head>
  <style>
    :root {
      --dynamic-color: #3498db;
    }
    .dynamic {
      color: var(--dynamic-color);
    }
  </style>
</head>
<body>
  <p class="dynamic">This text will change color dynamically.</p>
  <button onclick="changeColor()">Change Color</button>

  <script>
    function changeColor() {
      document.documentElement.style.setProperty('--dynamic-color', '#e74c3c');
    }
  </script>
</body>
</html>

In this example, clicking the button changes the color of the text by updating the custom property --dynamic-color.

Conclusion

Custom properties in CSS provide a powerful way to manage and reuse values across your stylesheets. They enhance maintainability, reusability, and allow for dynamic changes, making them an essential tool for modern web development. By understanding and utilizing custom properties, you can create more efficient and adaptable stylesheets.
