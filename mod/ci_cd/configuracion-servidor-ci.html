Introduction

Continuous Integration (CI) is a software development practice where developers integrate their code into a shared repository several times a day. Each integration is verified by an automated build (including tests) to detect errors as early as possible. In this section, we will learn how to set up a Continuous Integration server.

Key Concepts

  • Continuous Integration (CI): Automation process that allows developers to frequently integrate their code into a shared repository.
  • CI Server: Tool that automates the building and testing of code each time an integration is performed.
  • Pipeline: Sequence of steps that the CI server follows to build, test, and deploy the code.

Popular CI Tools

  • Jenkins
  • Travis CI
  • CircleCI
  • GitLab CI/CD

Setting Up Jenkins as a CI Server

Prerequisites

  • Java Development Kit (JDK) installed.
  • Web server (e.g., Apache Tomcat) or Docker container.
  • Administrative access to the server where Jenkins will be installed.

Installing Jenkins

  1. Download Jenkins:

    • Visit the official Jenkins page and download the .war file or the package for your operating system.

  2. Install Jenkins:

    • If using Docker: 
      docker run -p 8080:8080 -p 50000:50000 jenkins/jenkins:lts
    • If using the .war file: 
      java -jar jenkins.war

  3. Access Jenkins:

    • Open a web browser and navigate to http://localhost:8080.
    • Follow the on-screen instructions to complete the initial setup.

Initial Setup

  1. Unlock Jenkins:

    • During the first run, Jenkins will ask for an administrator password found in the initialAdminPassword file.

  2. Install Recommended Plugins:

    • Jenkins will suggest a list of recommended plugins. Install them to get basic functionalities.

  3. Create an Administrator User:

    • Set up an administrator user to manage Jenkins.

Create a Basic Pipeline

  1. Create a New Project:

    • Navigate to "New Item" and select "Pipeline".

  2. Configure the Pipeline:

    • In the "Pipeline" section, select "Pipeline script" and add the following example script: 
      pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                echo 'Building...'
                // Build commands go here
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
                // Test commands go here
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
                // Deployment commands go here
            }
        }
    }
}

  3. Save and Run:

    • Save the configuration and run the pipeline to verify everything works correctly.

Example Pipeline Configuration in Jenkins

pipeline {
    agent any
    environment {
        // Define environment variables if needed
        NODE_ENV = 'production'
    }
    stages {
        stage('Checkout') {
            steps {
                // Clone the repository
                git 'https://github.com/your-repository.git'
            }
        }
        stage('Build') {
            steps {
                // Build commands
                sh 'npm install'
                sh 'npm run build'
            }
        }
        stage('Test') {
            steps {
                // Test commands
                sh 'npm test'
            }
        }
        stage('Deploy') {
            steps {
                // Deployment commands
                sh 'npm run deploy'
            }
        }
    }
    post {
        success {
            echo 'Pipeline completed successfully!'
        }
        failure {
            echo 'Pipeline failed. Check the logs for more details.'
        }
    }
}

Conclusion

Setting up a Continuous Integration server like Jenkins is a crucial step to automate the process of building, testing, and deploying applications. With proper configuration, you can detect errors quickly and improve software quality. In this topic, we covered the basic concepts, installation and configuration of Jenkins, and how to create a basic pipeline. Practice these steps and adapt the pipeline to your project's specific needs to achieve the best results.
