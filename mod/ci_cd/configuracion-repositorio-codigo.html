Introduction

In this section of the CI/CD course, you will learn how to set up a code repository, a fundamental task for any continuous integration and delivery workflow. A well-configured repository facilitates collaboration, version control, and process automation.

Key Concepts

  • Code Repository: A place where the source code of a project is stored. It can be local or remote.
  • Version Control: A system that records changes to a file or set of files over time.
  • Git: The most widely used distributed version control system.
  • GitHub/GitLab/Bitbucket: Git repository hosting services that offer additional features like continuous integration, project management, etc.

Steps to Set Up a Code Repository

1. Installing Git

Before setting up a repository, you need to have Git installed on your machine.

On Windows:

  1. Download the installer from git-scm.com.
  2. Run the installer and follow the instructions.

On macOS:

brew install git

On Linux:

sudo apt-get install git

2. Initial Git Configuration

Set up your username and email, which will be associated with your commits.

git config --global user.name "Your Name"
git config --global user.email "youremail@example.com"

3. Creating a New Repository

You can create a new repository locally or clone an existing one.

Create a Local Repository

mkdir my-project
cd my-project
git init

Clone an Existing Repository

git clone https://github.com/user/repository.git

4. Adding Files to the Repository

Add files to the staging area and make a commit.

echo "# My Project" >> README.md
git add README.md
git commit -m "First commit"

5. Connecting the Local Repository with a Remote Repository

To connect your local repository with a remote repository, use the following command:

git remote add origin https://github.com/user/repository.git
git push -u origin master

Complete Example

Below is a complete example of setting up a code repository:

# Create a new directory for the project
mkdir my-project
cd my-project

# Initialize a Git repository
git init

# Create a README file
echo "# My Project" >> README.md

# Add the README file to the staging area
git add README.md

# Make the first commit
git commit -m "First commit"

# Connect the local repository with a remote repository
git remote add origin https://github.com/user/repository.git

# Push the changes to the remote repository
git push -u origin master

Best Practices

  • Frequent Commits: Make small and frequent commits to facilitate tracking changes.
  • Descriptive Commit Messages: Use clear and descriptive messages for your commits.
  • Branches: Use branches to develop new features or fix bugs without affecting the main branch.

Conclusion

Setting up a code repository is an essential step in any CI/CD workflow. A well-configured repository not only facilitates collaboration and version control but is also crucial for process automation. By following the steps and best practices mentioned, you will be on the right path to managing your code efficiently and effectively.
