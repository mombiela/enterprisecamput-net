Introduction

Ansible ad-hoc commands allow you to perform quick, one-time tasks without the need to write a playbook. They are useful for simple operations, testing, and troubleshooting. This section will guide you through the basics of running ad-hoc commands in Ansible, from beginner to advanced levels.

Key Concepts

  • Ad-Hoc Command: A one-time command executed directly from the command line.
  • Modules: Reusable, standalone scripts that Ansible runs on your behalf. Examples include ping, shell, copy, and yum.
  • Inventory: A list of hosts that Ansible manages.

Basic Ad-Hoc Commands

Ping Module

The ping module is used to check the connectivity of your hosts.

ansible all -m ping
  • all: Target all hosts in the inventory.
  • -m ping: Use the ping module.

Shell Module

The shell module allows you to run shell commands on your hosts.

ansible all -m shell -a 'uptime'
  • -a 'uptime': The argument to pass to the shell module, which in this case is the uptime command.

Copy Module

The copy module is used to copy files to remote hosts.

ansible all -m copy -a 'src=/path/to/local/file dest=/path/to/remote/file'
  • src: Source file path on the local machine.
  • dest: Destination file path on the remote machine.

Intermediate Ad-Hoc Commands

Package Management

You can manage packages on your hosts using modules like yum for Red Hat-based systems or apt for Debian-based systems.

Yum Module

ansible all -m yum -a 'name=httpd state=present'
  • name: The package name.
  • state: The desired state of the package (present to install, absent to remove).

Apt Module

ansible all -m apt -a 'name=nginx state=latest'
  • state=latest: Ensures the package is the latest version.

User Management

You can manage users on your hosts using the user module.

ansible all -m user -a 'name=johndoe state=present'
  • name: The username.
  • state=present: Ensures the user exists.

Advanced Ad-Hoc Commands

Using Variables

You can pass variables to ad-hoc commands for more dynamic operations.

ansible all -m shell -a 'echo {{ my_var }}' -e 'my_var=HelloWorld'
  • -e 'my_var=HelloWorld': Define the variable my_var.

Running Commands on Specific Hosts

You can target specific hosts or groups in your inventory.

ansible webservers -m shell -a 'systemctl restart nginx'
  • webservers: Target the webservers group defined in your inventory.

Using Tags

Tags can be used to run specific parts of your playbooks, but they can also be applied in ad-hoc commands for more granular control.

ansible all -m shell -a 'systemctl restart nginx' --tags "web"
  • --tags "web": Only run the command on hosts tagged with web.

Summary

Ad-hoc commands in Ansible are powerful tools for performing quick, one-time tasks across multiple hosts. They are useful for testing, troubleshooting, and simple operations without the need to write a full playbook. By understanding the basic, intermediate, and advanced uses of ad-hoc commands, you can efficiently manage your infrastructure with Ansible.

In this section, we covered:

  • Basic ad-hoc commands like ping, shell, and copy.
  • Intermediate commands for package and user management.
  • Advanced commands using variables, targeting specific hosts, and applying tags.

By mastering these commands, you can leverage Ansible's full potential for ad-hoc operations.
