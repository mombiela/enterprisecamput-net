Introduction

Practice exams and sample questions are essential tools for mastering Ansible. They help reinforce learning, identify areas for improvement, and build confidence. This section provides a variety of practice questions and mock exams, ranging from beginner to advanced levels, to help you solidify your understanding of Ansible.

Beginner Level

Key Concepts

  • Basic Ansible commands
  • Inventory files
  • Playbooks and YAML syntax
  • Modules and tasks

Sample Questions

  1. What command is used to check the version of Ansible installed on your system?

    ansible --version

  2. Write a simple inventory file for two hosts: webserver and dbserver.

    [web]
webserver

[db]
dbserver

  3. Create a basic playbook to install nginx on webserver.

    ---
- name: Install nginx on webserver
  hosts: web
  tasks:
    - name: Install nginx
      apt:
        name: nginx
        state: present

  4. Explain the purpose of the ansible.cfg file.

    • The ansible.cfg file is used to configure Ansible settings such as inventory location, remote user, and SSH settings.

Intermediate Level

Key Concepts

  • Roles and role structure
  • Variables and facts
  • Handlers and notifications
  • Conditionals and loops

Sample Questions

  1. What is the purpose of roles in Ansible?

    • Roles help organize playbooks by grouping tasks, variables, files, templates, and handlers into reusable components.

  2. Write a playbook that uses a variable to install a package specified by the user.

    ---
- name: Install a package
  hosts: all
  vars:
    package_name: "default_package"
  tasks:
    - name: Install the specified package
      apt:
        name: "{{ package_name }}"
        state: present

  3. Create a handler to restart nginx and notify it in a task.

    ---
- name: Install and configure nginx
  hosts: web
  tasks:
    - name: Install nginx
      apt:
        name: nginx
        state: present
      notify:
        - Restart nginx

  handlers:
    - name: Restart nginx
      service:
        name: nginx
        state: restarted

  4. Explain how to use conditionals in Ansible tasks. Provide an example.

    • Conditionals in Ansible are used to execute tasks based on certain conditions. They are defined using the when keyword.
    - name: Install nginx only on Ubuntu
  apt:
    name: nginx
    state: present
  when: ansible_os_family == "Debian"

Advanced Level

Key Concepts

  • Custom modules
  • Ansible Vault
  • Dynamic inventory
  • Advanced playbook strategies

Sample Questions

  1. Describe how to create a custom Ansible module.

    • Custom modules can be created using Python or any other scripting language. They must return a JSON object with specific keys like changed and msg.

  2. Write a playbook that uses Ansible Vault to encrypt sensitive data.

    ---
- name: Decrypt and use sensitive data
  hosts: all
  vars_files:
    - vault.yml
  tasks:
    - name: Use sensitive data
      debug:
        msg: "The secret is {{ secret_value }}"
    • To encrypt the vault.yml file: 
      ansible-vault encrypt vault.yml

  3. Explain the purpose of dynamic inventory and provide an example script.

    • Dynamic inventory allows Ansible to query external data sources to generate inventory dynamically.
    # dynamic_inventory.py
import json

inventory = {
    "web": {
        "hosts": ["webserver1", "webserver2"]
    },
    "db": {
        "hosts": ["dbserver1"]
    }
}

print(json.dumps(inventory))
    • To use the script: 
      ansible-playbook -i dynamic_inventory.py playbook.yml

  4. Discuss advanced playbook strategies like serial, max_fail_percentage, and delegate_to.

    • serial: Controls the number of hosts to be managed at a time.
    • max_fail_percentage: Defines the maximum percentage of hosts that can fail before the play is aborted.
    • delegate_to: Executes a task on a different host than the one specified in the hosts directive.

Conclusion

Practice exams and sample questions are invaluable for mastering Ansible. By working through these questions, you can identify your strengths and areas for improvement, ensuring you are well-prepared for real-world scenarios and certification exams. Keep practicing and exploring Ansible's vast capabilities to become proficient in automation and configuration management.
