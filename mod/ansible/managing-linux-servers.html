Introduction to Ansible

  • What is Ansible?
    • Ansible is an open-source automation tool for IT tasks such as configuration management, application deployment, and task automation.
  • Why use Ansible for managing Linux servers?
    • Agentless architecture
    • Simple YAML syntax
    • Idempotency (ensures tasks are only applied when necessary)

Setting Up Ansible

  • Installation
    • On a Linux machine, install Ansible using the package manager: 
      sudo apt update
sudo apt install ansible
  • Configuration
    • Ansible configuration file (ansible.cfg): 
      [defaults]
inventory = ./hosts
  • Inventory File
    • Define your managed nodes in an inventory file (hosts): 
      [webservers]
web1.example.com
web2.example.com

[dbservers]
db1.example.com
db2.example.com

Basic Ansible Commands

  • Ping all hosts 
    ansible all -m ping
  • Run a command on all hosts 
    ansible all -a "/bin/echo hello"

Writing Playbooks

  • Introduction to Playbooks
    • Playbooks are YAML files that define a series of tasks to be executed on managed nodes.
  • Basic Playbook Structure 
    ---
- name: Ensure Apache is installed
  hosts: webservers
  tasks:
    - name: Install Apache
      apt:
        name: apache2
        state: present
  • Running a Playbook 
    ansible-playbook playbook.yml

Advanced Playbook Features

  • Variables 
    vars:
  http_port: 80
tasks:
  - name: Ensure Apache is listening on port 80
    lineinfile:
      path: /etc/apache2/ports.conf
      regexp: '^Listen'
      line: "Listen {{ http_port }}"
  • Handlers 
    tasks:
  - name: Install Apache
    apt:
      name: apache2
      state: present
    notify: Restart Apache

handlers:
  - name: Restart Apache
    service:
      name: apache2
      state: restarted
  • Templates
    • Using Jinja2 templates to manage configuration files: 
      tasks:
  - name: Deploy Apache configuration
    template:
      src: templates/apache2.conf.j2
      dest: /etc/apache2/apache2.conf

Roles and Reusability

  • Introduction to Roles
    • Roles allow you to organize playbooks into reusable components.
  • Creating a Role 
    ansible-galaxy init myrole
  • Using a Role in a Playbook 
    ---
- name: Apply myrole to webservers
  hosts: webservers
  roles:
    - myrole

Managing Multiple Environments

  • Environment-specific Variables
    • Define variables for different environments in separate files: 
      # group_vars/staging.yml
http_port: 8080

# group_vars/production.yml
http_port: 80
  • Using Environment-specific Playbooks 
    ansible-playbook -i staging playbook.yml
ansible-playbook -i production playbook.yml

Ansible Vault

  • Encrypting Sensitive Data
    • Encrypt sensitive data using Ansible Vault: 
      ansible-vault encrypt vars/secrets.yml
  • Using Encrypted Files in Playbooks 
    vars_files:
  - vars/secrets.yml

Conclusion

Managing Linux servers with Ansible simplifies and automates many administrative tasks. By understanding the basics of Ansible, writing playbooks, and using advanced features like roles and Ansible Vault, you can efficiently manage and scale your infrastructure. This course has provided a comprehensive overview from beginner to advanced topics, equipping you with the knowledge to leverage Ansible in your environment.
