Setting Up the Environment

Setting up the environment is the first step in getting started with Ansible. This section will guide you through the necessary steps to install and configure Ansible on your system, ensuring you have a solid foundation to build upon as you progress through the course.

Prerequisites

Before installing Ansible, ensure you have the following prerequisites:

  • A Unix-like operating system (Linux, macOS, or WSL on Windows).
  • Python 3.6 or later installed on your system.
  • Basic knowledge of command-line operations.

Installing Ansible

Using Package Manager

On Ubuntu/Debian

To install Ansible on Ubuntu or Debian, use the following commands:

sudo apt update
sudo apt install ansible -y

On CentOS/RHEL

For CentOS or RHEL, use the following commands:

sudo yum install epel-release -y
sudo yum install ansible -y

On macOS

For macOS users, Homebrew is the easiest way to install Ansible:

brew install ansible

Using Pip

Alternatively, you can install Ansible using Python's package manager, pip:

pip install ansible

Verifying the Installation

After installing Ansible, verify the installation by checking the version:

ansible --version

You should see output similar to:

ansible [core 2.10.7]
  config file = /etc/ansible/ansible.cfg
  configured module search path = ['/home/user/.ansible/plugins/modules', '/usr/share/ansible/plugins/modules']
  ansible python module location = /usr/local/lib/python3.8/dist-packages/ansible
  executable location = /usr/local/bin/ansible
  python version = 3.8.5 (default, Jul 28 2020, 12:59:40) [GCC 9.3.0]

Configuring Ansible

Ansible Configuration File

Ansible's behavior can be customized using the configuration file ansible.cfg. By default, this file is located in /etc/ansible/ansible.cfg. You can also create a custom configuration file in your project directory.

Here is an example of a basic ansible.cfg file:

[defaults]
inventory = ./hosts
remote_user = your_username
host_key_checking = False

Inventory File

The inventory file lists the hosts and groups of hosts that Ansible manages. Here is an example of a simple inventory file named hosts:

[webservers]
webserver1 ansible_host=192.168.1.10 ansible_user=ubuntu
webserver2 ansible_host=192.168.1.11 ansible_user=ubuntu

[databases]
dbserver1 ansible_host=192.168.1.20 ansible_user=ubuntu

Testing the Setup

To ensure everything is set up correctly, run a simple Ansible command to ping all the hosts in your inventory:

ansible all -m ping

You should see output similar to:

webserver1 | SUCCESS => {
    "changed": false,
    "ping": "pong"
}
webserver2 | SUCCESS => {
    "changed": false,
    "ping": "pong"
}
dbserver1 | SUCCESS => {
    "changed": false,
    "ping": "pong"
}

Conclusion

In this section, we covered the essential steps to set up the Ansible environment, including installation, configuration, and verification. With your environment ready, you are now prepared to dive deeper into Ansible and start automating your tasks.
