Introduction to Ansible Galaxy

Ansible Galaxy is a hub for finding, sharing, and reusing Ansible content. It allows users to download roles and collections that can be used to automate various tasks. This section will cover the basics of Ansible Galaxy, how to use it, and how to contribute to it.

Key Concepts

  • Roles: Pre-packaged units of Ansible content that can be reused across multiple playbooks.
  • Collections: A distribution format for Ansible content that can include roles, modules, plugins, and more.
  • Galaxy CLI: Command-line interface for interacting with Ansible Galaxy.

Using Ansible Galaxy

Installing Roles

Roles can be easily installed from Ansible Galaxy using the ansible-galaxy command.

ansible-galaxy install username.role_name
  • username: The Galaxy username of the role's author.
  • role_name: The name of the role you want to install.

Example

To install a role for managing Nginx:

ansible-galaxy install geerlingguy.nginx

Using Installed Roles in Playbooks

Once a role is installed, it can be used in your playbooks.

---
- hosts: webservers
  roles:
    - geerlingguy.nginx

Managing Roles

You can list installed roles using:

ansible-galaxy list

To remove a role:

ansible-galaxy remove username.role_name

Creating and Sharing Roles

Creating a Role

You can create a new role using the ansible-galaxy command:

ansible-galaxy init my_new_role

This command creates a directory structure for your new role.

Directory Structure

The created role will have the following structure:

my_new_role/
├── defaults
│   └── main.yml
├── files
├── handlers
│   └── main.yml
├── meta
│   └── main.yml
├── tasks
│   └── main.yml
├── templates
├── tests
│   ├── inventory
│   └── test.yml
└── vars
    └── main.yml

Publishing a Role

To share your role on Ansible Galaxy, you need to:

  1. Create a Galaxy account.
  2. Link your GitHub account to Galaxy.
  3. Push your role to a GitHub repository.
  4. Import the role into Galaxy using the Galaxy web interface.

Ansible Collections

Installing Collections

Collections can be installed similarly to roles:

ansible-galaxy collection install namespace.collection_name

Example

To install a collection for AWS:

ansible-galaxy collection install amazon.aws

Using Collections in Playbooks

Collections can be used in playbooks by specifying the fully qualified collection name.

---
- hosts: localhost
  tasks:
    - name: Create an S3 bucket
      amazon.aws.s3_bucket:
        name: my-bucket
        state: present

Creating a Collection

You can create a new collection using the ansible-galaxy command:

ansible-galaxy collection init my_namespace.my_collection

This command creates a directory structure for your new collection.

Directory Structure

The created collection will have the following structure:

my_namespace-my_collection/
├── docs
├── plugins
│   ├── modules
│   └── ...
├── roles
├── playbooks
└── tests

Publishing a Collection

To share your collection on Ansible Galaxy, you need to:

  1. Create a Galaxy account.
  2. Link your GitHub account to Galaxy.
  3. Push your collection to a GitHub repository.
  4. Import the collection into Galaxy using the Galaxy web interface.

Conclusion

Ansible Galaxy is a powerful tool for finding, sharing, and reusing Ansible content. By understanding how to use roles and collections, you can significantly streamline your automation tasks. Whether you are installing existing roles and collections or creating and sharing your own, Ansible Galaxy provides a robust platform to enhance your Ansible experience.
