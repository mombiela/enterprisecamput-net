Aggregation operations in MongoDB process data records and return computed results. They are often used to group values from multiple documents, perform operations on the grouped data, and return a single result. This section will cover the most common aggregation stages you will encounter, from beginner to advanced levels.

Introduction to Aggregation Framework

The aggregation framework in MongoDB is a powerful tool for data analysis and transformation. It consists of a pipeline of stages, each performing a specific operation on the input documents.

Common Aggregation Stages

$match

  • Purpose: Filters the documents to pass only those that match the specified condition(s) to the next pipeline stage.
  • Usage: Similar to the find query, but used within the aggregation pipeline.
db.collection.aggregate([
  { $match: { status: "A" } }
])

$group

  • Purpose: Groups input documents by a specified identifier expression and applies the accumulator expressions to each group.
  • Usage: Often used to calculate aggregates like sums, averages, counts, etc.
db.collection.aggregate([
  { $group: { _id: "$status", total: { $sum: "$amount" } } }
])

$project

  • Purpose: Reshapes each document in the stream, such as by adding new fields or removing existing fields.
  • Usage: Useful for including, excluding, or computing new fields.
db.collection.aggregate([
  { $project: { item: 1, total: { $multiply: ["$price", "$quantity"] } } }
])

$sort

  • Purpose: Sorts all input documents and returns them in sorted order.
  • Usage: Similar to the sort method but used within the aggregation pipeline.
db.collection.aggregate([
  { $sort: { total: -1 } }
])

$limit

  • Purpose: Limits the number of documents passed to the next stage in the pipeline.
  • Usage: Useful for pagination or restricting the output size.
db.collection.aggregate([
  { $limit: 5 }
])

$skip

  • Purpose: Skips the first N documents and passes the remaining documents to the next stage.
  • Usage: Often used in conjunction with $limit for pagination.
db.collection.aggregate([
  { $skip: 10 }
])

$unwind

  • Purpose: Deconstructs an array field from the input documents to output a document for each element.
  • Usage: Useful for normalizing data stored in arrays.
db.collection.aggregate([
  { $unwind: "$items" }
])

$lookup

  • Purpose: Performs a left outer join to another collection in the same database to filter in documents from the "joined" collection for processing.
  • Usage: Useful for combining data from multiple collections.
db.collection.aggregate([
  {
    $lookup: {
      from: "otherCollection",
      localField: "foreignKey",
      foreignField: "_id",
      as: "joinedData"
    }
  }
])

$addFields

  • Purpose: Adds new fields to documents.
  • Usage: Useful for adding computed fields.
db.collection.aggregate([
  { $addFields: { totalPrice: { $multiply: ["$price", "$quantity"] } } }
])

$out

  • Purpose: Writes the resulting documents of the aggregation pipeline to a specified collection.
  • Usage: Useful for storing the results of an aggregation for future use.
db.collection.aggregate([
  { $match: { status: "A" } },
  { $out: "newCollection" }
])

Summary

Understanding and utilizing the common aggregation stages in MongoDB is crucial for effective data analysis and transformation. Each stage serves a specific purpose, from filtering and grouping to reshaping and joining data. By mastering these stages, you can perform complex queries and gain deeper insights from your data.

  • $match: Filters documents.
  • $group: Groups documents and performs aggregations.
  • $project: Reshapes documents.
  • $sort: Sorts documents.
  • $limit: Limits the number of documents.
  • $skip: Skips documents.
  • $unwind: Deconstructs arrays.
  • $lookup: Joins collections.
  • $addFields: Adds new fields.
  • $out: Outputs results to a collection.

These stages form the building blocks of the MongoDB aggregation framework, enabling you to create powerful and efficient data processing pipelines.
