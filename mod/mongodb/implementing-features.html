In this section, we will explore how to implement various features in MongoDB, ranging from basic to advanced functionalities. By the end of this topic, you will be able to create, read, update, and delete documents, as well as implement more complex features such as indexing, aggregation, and transactions.

Basic CRUD Operations

CRUD stands for Create, Read, Update, and Delete. These are the basic operations you will perform on your MongoDB collections.

Create

To insert a document into a collection, you use the insertOne or insertMany methods.

// Insert a single document
db.collection('users').insertOne({
  name: "John Doe",
  age: 30,
  email: "john.doe@example.com"
});

// Insert multiple documents
db.collection('users').insertMany([
  { name: "Jane Doe", age: 25, email: "jane.doe@example.com" },
  { name: "Alice Smith", age: 28, email: "alice.smith@example.com" }
]);

Read

To retrieve documents from a collection, you use the find method.

// Find all documents in the collection
db.collection('users').find({});

// Find documents with a specific condition
db.collection('users').find({ age: { $gt: 25 } });

Update

To modify existing documents, you use the updateOne or updateMany methods.

// Update a single document
db.collection('users').updateOne(
  { name: "John Doe" },
  { $set: { age: 31 } }
);

// Update multiple documents
db.collection('users').updateMany(
  { age: { $lt: 30 } },
  { $set: { status: "active" } }
);

Delete

To remove documents from a collection, you use the deleteOne or deleteMany methods.

// Delete a single document
db.collection('users').deleteOne({ name: "John Doe" });

// Delete multiple documents
db.collection('users').deleteMany({ age: { $lt: 25 } });

Indexing

Indexes improve the performance of search queries. MongoDB supports various types of indexes.

  • Single Field Index: Indexes a single field in a document.
  • Compound Index: Indexes multiple fields in a document.
  • Multikey Index: Indexes fields that hold arrays.
  • Text Index: Indexes text content for text search.
  • Geospatial Index: Indexes for location-based queries.

Creating an Index

// Create a single field index
db.collection('users').createIndex({ email: 1 });

// Create a compound index
db.collection('users').createIndex({ name: 1, age: -1 });

Using Indexes

Indexes are automatically used by MongoDB to optimize query performance.

// Query that benefits from an index
db.collection('users').find({ email: "john.doe@example.com" });

Aggregation

Aggregation operations process data records and return computed results. MongoDB provides the aggregation framework, which is a powerful tool for data analysis.

Basic Aggregation Pipeline

db.collection('orders').aggregate([
  { $match: { status: "completed" } },
  { $group: { _id: "$customerId", totalAmount: { $sum: "$amount" } } },
  { $sort: { totalAmount: -1 } }
]);
  • $match: Filters the documents.
  • $group: Groups documents by a specified identifier.
  • $sort: Sorts the documents.

Transactions

Transactions allow you to execute multiple operations in isolation and ensure data consistency.

Using Transactions

const session = client.startSession();

session.startTransaction();

try {
  db.collection('accounts').updateOne(
    { accountId: 1 },
    { $inc: { balance: -100 } },
    { session }
  );

  db.collection('accounts').updateOne(
    { accountId: 2 },
    { $inc: { balance: 100 } },
    { session }
  );

  session.commitTransaction();
} catch (error) {
  session.abortTransaction();
  throw error;
} finally {
  session.endSession();
}

Conclusion

In this topic, we covered the essential features you need to implement in MongoDB, from basic CRUD operations to more advanced functionalities like indexing, aggregation, and transactions. Understanding these features will help you build efficient and scalable applications using MongoDB. Practice these concepts with real-world data to solidify your understanding and become proficient in MongoDB.
