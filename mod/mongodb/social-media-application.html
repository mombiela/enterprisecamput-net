Introduction

In this course, we will explore how to use MongoDB to build a social media application. We will start with the basics of MongoDB and progressively move towards more advanced topics. By the end of this course, you will have a solid understanding of MongoDB and how to use it to manage data for a social media application.

Getting Started with MongoDB

What is MongoDB?

  • MongoDB is a NoSQL database that stores data in JSON-like documents.
  • It is designed for scalability and flexibility, making it ideal for modern web applications.

Installing MongoDB

  • Follow the official MongoDB installation guide for your operating system.
  • Verify the installation by running mongo --version in your terminal.

Basic MongoDB Operations

  • Connecting to MongoDB: Use the MongoDB shell or a client like MongoDB Compass.
  • CRUD Operations:
    • Create: Insert documents into a collection.
    • Read: Query documents from a collection.
    • Update: Modify existing documents.
    • Delete: Remove documents from a collection.
// Connecting to MongoDB
const { MongoClient } = require('mongodb');
const uri = "your_mongodb_connection_string";
const client = new MongoClient(uri, { useNewUrlParser: true, useUnifiedTopology: true });

async function run() {
    try {
        await client.connect();
        console.log("Connected to MongoDB");
    } finally {
        await client.close();
    }
}
run().catch(console.dir);

Designing the Social Media Application Schema

User Collection

  • Fields:
    • username: String, unique identifier for the user.
    • email: String, unique email address.
    • password: String, hashed password.
    • profile: Object containing user profile information.
const userSchema = {
    username: "john_doe",
    email: "john@example.com",
    password: "hashed_password",
    profile: {
        fullName: "John Doe",
        bio: "Software Developer",
        avatar: "url_to_avatar"
    }
};

Post Collection

  • Fields:
    • userId: ObjectId, reference to the user who created the post.
    • content: String, the content of the post.
    • createdAt: Date, timestamp of when the post was created.
    • likes: Array of ObjectIds, references to users who liked the post.
    • comments: Array of comment objects.
const postSchema = {
    userId: ObjectId("user_id"),
    content: "This is a sample post",
    createdAt: new Date(),
    likes: [ObjectId("user_id_1"), ObjectId("user_id_2")],
    comments: [
        {
            userId: ObjectId("commenter_id"),
            comment: "Nice post!",
            createdAt: new Date()
        }
    ]
};

Comment Collection

  • Fields:
    • postId: ObjectId, reference to the post.
    • userId: ObjectId, reference to the user who commented.
    • comment: String, the comment text.
    • createdAt: Date, timestamp of when the comment was created.
const commentSchema = {
    postId: ObjectId("post_id"),
    userId: ObjectId("user_id"),
    comment: "This is a comment",
    createdAt: new Date()
};

Advanced MongoDB Features

Indexing

  • Purpose: Improve query performance.
  • Types:
    • Single Field Index
    • Compound Index
    • Text Index
// Creating an index on the username field
db.users.createIndex({ username: 1 });

// Creating a compound index on email and username
db.users.createIndex({ email: 1, username: 1 });

Aggregation Framework

  • Purpose: Perform complex data processing and analysis.
  • Stages:
    • $match: Filter documents.
    • $group: Group documents by a specified field.
    • $sort: Sort documents.
// Aggregation pipeline to get the number of posts per user
db.posts.aggregate([
    { $group: { _id: "$userId", postCount: { $sum: 1 } } },
    { $sort: { postCount: -1 } }
]);

Transactions

  • Purpose: Ensure data consistency across multiple operations.
  • Usage: Use transactions for operations that need to be atomic.
async function createPostWithTransaction(client, userId, content) {
    const session = client.startSession();
    session.startTransaction();
    try {
        const postsCollection = client.db("social_media").collection("posts");
        const result = await postsCollection.insertOne({ userId, content, createdAt: new Date() }, { session });
        await session.commitTransaction();
        console.log("Post created successfully");
    } catch (error) {
        await session.abortTransaction();
        console.error("Transaction aborted due to an error: ", error);
    } finally {
        session.endSession();
    }
}
