Authentication and Authorization in MongoDB
Role-Based Access Control (RBAC) ►

Authentication and authorization are critical components of database security. This section will guide you through the concepts and implementation of authentication and authorization in MongoDB, from beginner to advanced levels.

Introduction to Authentication and Authorization

  • Authentication: The process of verifying the identity of a user or process.
  • Authorization: The process of determining what an authenticated user or process is allowed to do.

Authentication in MongoDB

Basic Concepts

  • User: An entity that can be authenticated.
  • Role: A set of privileges that can be assigned to a user.
  • Privilege: A specific action that a user can perform on a resource.

Enabling Authentication

By default, MongoDB does not require authentication. To enable authentication, you need to start the MongoDB server with the --auth option.

mongod --auth --dbpath /data/db

Creating Users

To create a user, you need to connect to the MongoDB instance without authentication and use the db.createUser method.

use admin
db.createUser({
  user: "admin",
  pwd: "password123",
  roles: [ { role: "userAdminAnyDatabase", db: "admin" } ]
})

Authenticating Users

Once users are created, they can authenticate using the db.auth method.

use admin
db.auth("admin", "password123")

Authorization in MongoDB

Role-Based Access Control (RBAC)

MongoDB uses Role-Based Access Control (RBAC) to manage authorization. Roles grant users specific privileges on resources.

Built-in Roles

MongoDB provides several built-in roles, such as:

  • read: Grants read-only access to a database.
  • readWrite: Grants read and write access to a database.
  • dbAdmin: Grants administrative rights to a database.
  • userAdmin: Grants the ability to create and modify roles and users on a database.

Custom Roles

You can also create custom roles to meet specific needs.

db.createRole({
  role: "customRole",
  privileges: [
    { resource: { db: "test", collection: "" }, actions: [ "find", "insert" ] }
  ],
  roles: []
})

Assigning Roles to Users

Roles can be assigned to users when they are created or modified.

db.createUser({
  user: "appUser",
  pwd: "appPassword",
  roles: [ { role: "readWrite", db: "test" } ]
})

Practical Examples

Example 1: Creating an Admin User

  1. Start MongoDB without authentication.
  2. Create an admin user.
use admin
db.createUser({
  user: "admin",
  pwd: "adminPass",
  roles: [ { role: "root", db: "admin" } ]
})
  1. Restart MongoDB with authentication enabled.
mongod --auth --dbpath /data/db
  1. Authenticate as the admin user.
use admin
db.auth("admin", "adminPass")

Example 2: Creating and Authenticating a Regular User

  1. Authenticate as the admin user.
use admin
db.auth("admin", "adminPass")
  1. Create a regular user with readWrite access to the test database.
use test
db.createUser({
  user: "testUser",
  pwd: "testPass",
  roles: [ { role: "readWrite", db: "test" } ]
})
  1. Authenticate as the regular user.
use test
db.auth("testUser", "testPass")

Summary

In this section, we covered the basics of authentication and authorization in MongoDB. We learned how to:

  • Enable authentication.
  • Create and authenticate users.
  • Use built-in roles and create custom roles.
  • Assign roles to users.

Understanding these concepts is crucial for securing your MongoDB instances and ensuring that only authorized users can access and manipulate your data.
