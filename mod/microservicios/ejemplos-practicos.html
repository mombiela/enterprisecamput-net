In this section, we will explore practical examples that will help you understand how to implement and manage microservices in a real environment. These examples are designed to be clear and understandable, even if you are a beginner on the subject.

Example 1: Creating a Basic Microservice

Example Description

In this first example, we will create a basic microservice using Spring Boot, a popular framework in the Java ecosystem. This microservice will be responsible for managing user information.

Steps to Create the Microservice

  1. Project Setup:

    • Create a new Spring Boot project using Spring Initializr.
    • Select the necessary dependencies: Spring Web, Spring Data JPA, and H2 Database.

  2. Model Definition:

    • Create a User class that represents the user entity.
@Entity
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String name;
    private String email;

    // Getters and Setters
}
  1. Repository Creation:
    • Create a UserRepository interface that extends JpaRepository.
public interface UserRepository extends JpaRepository<User, Long> {
}
  1. Controller Implementation:
    • Create a UserController controller to handle HTTP requests.
@RestController
@RequestMapping("/users")
public class UserController {
    @Autowired
    private UserRepository userRepository;

    @GetMapping
    public List<User> getAllUsers() {
        return userRepository.findAll();
    }

    @PostMapping
    public User createUser(@RequestBody User user) {
        return userRepository.save(user);
    }
}

Execution and Testing

  • Run the application and use tools like Postman to test the /users endpoints (GET and POST).

Example 2: Communication between Microservices

Example Description

In this example, we will see how two microservices can communicate with each other using REST API. We will have an Order microservice that will communicate with the User microservice to get user information.

Steps to Configure Communication

  1. Creation of the Order Microservice:

    • Set up a new Spring Boot project for the Order microservice.

  2. Order Model Definition:

    • Create an Order class that represents the order entity.
@Entity
public class Order {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private Long userId;
    private String product;
    private int quantity;

    // Getters and Setters
}
  1. REST Client Implementation:
    • Create a UserClient class to communicate with the User microservice.
@Service
public class UserClient {
    @Autowired
    private RestTemplate restTemplate;

    public User getUserById(Long userId) {
        return restTemplate.getForObject("http://localhost:8080/users/" + userId, User.class);
    }
}
  1. Order Controller Configuration:
    • Create an OrderController controller that uses UserClient.
@RestController
@RequestMapping("/orders")
public class OrderController {
    @Autowired
    private OrderRepository orderRepository;

    @Autowired
    private UserClient userClient;

    @PostMapping
    public Order createOrder(@RequestBody Order order) {
        User user = userClient.getUserById(order.getUserId());
        if (user != null) {
            return orderRepository.save(order);
        } else {
            throw new RuntimeException("User not found");
        }
    }
}

Execution and Testing

  • Run both microservices (User and Order).
  • Use Postman to create orders and verify communication between the microservices.

Conclusion

In this section, we have seen two practical examples that illustrate how to create and manage microservices. The first example focused on creating a basic microservice, while the second showed how microservices can communicate with each other. These examples are fundamental to understanding the key concepts of microservices and will serve as a foundation for more complex projects.
