Introduction to Containers

Containers are a technology that allows packaging and running applications in isolation from the underlying operating system. Unlike virtual machines, containers share the same operating system kernel, making them lighter and more efficient.

Key Concepts

  • Isolation: Containers provide an isolated environment to run applications, improving security and stability.
  • Portability: Containers can run on any system that has a compatible container engine, such as Docker.
  • Efficiency: Containers are lighter than virtual machines because they share the operating system kernel.

Introduction to Docker

Docker is a containerization platform that allows developers to package applications and their dependencies into containers. Docker facilitates the creation, deployment, and execution of applications in containers.

Key Concepts

  • Docker Engine: The Docker engine is the main component that allows the creation and execution of containers.
  • Docker Images: Docker images are read-only templates that contain the operating system and the necessary applications to run a container.
  • Docker Containers: Docker containers are running instances of Docker images.
  • Docker Hub: A public registry where users can store and share Docker images.

Installing Docker

To install Docker on different operating systems, follow the steps below:

Installation on Linux

sudo apt-get update
sudo apt-get install \
    ca-certificates \
    curl \
    gnupg \
    lsb-release

curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io

Installation on Windows

  1. Download Docker Desktop from Docker Hub.
  2. Run the installer and follow the on-screen instructions.
  3. Restart your computer if necessary.

Installation on macOS

  1. Download Docker Desktop from Docker Hub.
  2. Open the .dmg file and drag Docker to the Applications folder.
  3. Run Docker from the Applications folder.

Basic Docker Usage

Essential Commands

  • docker run: Runs a container from an image.
docker run hello-world
  • docker ps: Lists running containers.
docker ps
  • docker images: Lists available images on the system.
docker images
  • docker stop: Stops a running container.
docker stop <container_id>

Creating a Docker Image

To create a Docker image, a Dockerfile is used. Below is an example of a simple Dockerfile:

# Use a Python base image
FROM python:3.8-slim

# Set the working directory
WORKDIR /app

# Copy the necessary files
COPY . /app

# Install dependencies
RUN pip install --no-cache-dir -r requirements.txt

# Command to run the application
CMD ["python", "app.py"]

To build the image, use the following command:

docker build -t my-python-app .

Practical Example: Deploying a Web Application

Step 1: Create the `Dockerfile`

FROM node:14

WORKDIR /usr/src/app

COPY package*.json ./

RUN npm install

COPY . .

EXPOSE 8080

CMD ["node", "server.js"]

Step 2: Build the Image

docker build -t my-node-app .

Step 3: Run the Container

docker run -p 8080:8080 -d my-node-app

Conclusion

Docker is a powerful tool for containerizing applications, providing portability, efficiency, and isolation. With Docker, developers can package applications and their dependencies into containers that can run in any compatible environment. In this topic, we have covered the basics of containers and Docker, including installation, essential commands, and a practical example of deploying a web application.
