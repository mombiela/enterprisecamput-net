Deploying the Application

Deploying a Go application involves several steps, from building the executable to configuring the environment where the application will run. This section will guide you through the process of deploying a Go application, covering both basic and advanced deployment strategies.

Building the Go Application

Before deploying, you need to build your Go application into an executable binary.

  • Building for the current system:

    go build -o myapp

    This command compiles the Go source code and produces an executable named myapp for the current operating system and architecture.

  • Cross-compiling for different systems:

    GOOS=linux GOARCH=amd64 go build -o myapp-linux

    This command sets the target operating system to Linux and the architecture to AMD64, producing an executable named myapp-linux.

Preparing the Environment

Ensure the target environment is ready to run your Go application.

  • Setting up the server:

    • Choose a server (e.g., AWS, DigitalOcean, Google Cloud).
    • Install necessary dependencies (e.g., Docker, database systems).

  • Environment variables:

    • Store sensitive information like API keys and database credentials in environment variables.
    • Example in a .env file: 
      DB_HOST=localhost
DB_USER=root
DB_PASS=password

Deploying with Docker

Docker simplifies deployment by packaging your application and its dependencies into a container.

  • Creating a Dockerfile:

    # Use the official Golang image as the base image
FROM golang:1.19-alpine

# Set the Current Working Directory inside the container
WORKDIR /app

# Copy go mod and sum files
COPY go.mod go.sum ./

# Download all dependencies. Dependencies will be cached if the go.mod and go.sum files are not changed
RUN go mod download

# Copy the source from the current directory to the Working Directory inside the container
COPY . .

# Build the Go app
RUN go build -o myapp

# Command to run the executable
CMD ["./myapp"]

  • Building and running the Docker container:

    docker build -t myapp .
docker run -p 8080:8080 myapp

Deploying to a Cloud Service

Deploying to AWS Elastic Beanstalk

  • Create an Elastic Beanstalk application:

    • Use the AWS Management Console to create a new Elastic Beanstalk application.
    • Choose a platform (e.g., Docker if using a Docker container).

  • Deploy the application:

    • Package your application code and configuration files.
    • Use the Elastic Beanstalk CLI to deploy: 
      eb init -p docker myapp
eb create myapp-env
eb deploy

Deploying to Google Cloud Run

  • Containerize the application:

    • Ensure you have a Dockerfile as shown previously.

  • Deploy using the gcloud CLI:

    gcloud builds submit --tag gcr.io/PROJECT-ID/myapp
gcloud run deploy --image gcr.io/PROJECT-ID/myapp --platform managed

Conclusion

Deploying a Go application involves building the executable, preparing the environment, and choosing a deployment strategy. Whether you use Docker, AWS Elastic Beanstalk, or Google Cloud Run, the key is to ensure your application is packaged correctly and the target environment is configured properly. By following these steps, you can deploy your Go applications efficiently and reliably.
