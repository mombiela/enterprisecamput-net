Error Handling Basics

Error handling is a crucial aspect of programming, and Go provides a unique and effective way to manage errors. This section will cover the basics of error handling in Go, including how to create, handle, and propagate errors.

Key Concepts

  • Error Type: In Go, errors are represented by the built-in error type.
  • Creating Errors: Use the errors.New function or the fmt.Errorf function to create new error values.
  • Handling Errors: Check for errors using simple if statements.
  • Propagating Errors: Return errors from functions to propagate them up the call stack.
  • Custom Errors: Implement the error interface to create custom error types.

Creating Errors

In Go, you can create an error using the errors.New function from the errors package or the fmt.Errorf function from the fmt package.

Using `errors.New`

package main

import (
    "errors"
    "fmt"
)

func main() {
    err := errors.New("something went wrong")
    if err != nil {
        fmt.Println(err)
    }
}

Using `fmt.Errorf`

package main

import (
    "fmt"
)

func main() {
    err := fmt.Errorf("an error occurred: %v", "something went wrong")
    if err != nil {
        fmt.Println(err)
    }
}

Handling Errors

Error handling in Go is typically done using if statements. If a function returns an error, you should check if the error is nil to determine if an error occurred.

package main

import (
    "fmt"
    "strconv"
)

func main() {
    _, err := strconv.Atoi("123a")
    if err != nil {
        fmt.Println("Error:", err)
    } else {
        fmt.Println("Conversion successful")
    }
}

Propagating Errors

When writing functions, you should return errors to allow the caller to handle them appropriately. This is known as error propagation.

package main

import (
    "errors"
    "fmt"
)

func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("cannot divide by zero")
    }
    return a / b, nil
}

func main() {
    result, err := divide(4, 0)
    if err != nil {
        fmt.Println("Error:", err)
    } else {
        fmt.Println("Result:", result)
    }
}

Custom Errors

You can create custom error types by implementing the error interface. This allows you to add additional context to your errors.

package main

import (
    "fmt"
)

type MyError struct {
    Code    int
    Message string
}

func (e *MyError) Error() string {
    return fmt.Sprintf("Error %d: %s", e.Code, e.Message)
}

func doSomething(flag bool) error {
    if !flag {
        return &MyError{Code: 123, Message: "something went wrong"}
    }
    return nil
}

func main() {
    err := doSomething(false)
    if err != nil {
        fmt.Println(err)
    }
}

Summary

  • Go uses the built-in error type for error handling.
  • Create errors using errors.New or fmt.Errorf.
  • Handle errors using if statements to check if the error is nil.
  • Propagate errors by returning them from functions.
  • Implement the error interface to create custom error types.

Understanding these basics will help you write more robust and maintainable Go programs. As you progress, you'll learn more advanced techniques for error handling, such as using the errors package for error wrapping and unwrapping.
