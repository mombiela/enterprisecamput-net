Introduction to File I/O

File I/O (Input/Output) is a fundamental aspect of programming that allows a program to read from and write to files. In Go, the os package provides a platform-independent interface to operating system functionality, including file I/O.

Reading Files

Opening a File

To read a file, you first need to open it using the os.Open function. This function returns a file descriptor and an error.

package main

import (
    "fmt"
    "os"
)

func main() {
    file, err := os.Open("example.txt")
    if err != nil {
        fmt.Println("Error opening file:", err)
        return
    }
    defer file.Close()
}

Reading File Content

Once the file is opened, you can read its content using various methods from the bufio and io/ioutil packages.

Using bufio.Scanner

The bufio.Scanner is useful for reading a file line by line.

package main

import (
    "bufio"
    "fmt"
    "os"
)

func main() {
    file, err := os.Open("example.txt")
    if err != nil {
        fmt.Println("Error opening file:", err)
        return
    }
    defer file.Close()

    scanner := bufio.NewScanner(file)
    for scanner.Scan() {
        fmt.Println(scanner.Text())
    }

    if err := scanner.Err(); err != nil {
        fmt.Println("Error reading file:", err)
    }
}

Using ioutil.ReadFile

The ioutil.ReadFile function reads the entire file content into memory.

package main

import (
    "fmt"
    "io/ioutil"
)

func main() {
    content, err := ioutil.ReadFile("example.txt")
    if err != nil {
        fmt.Println("Error reading file:", err)
        return
    }
    fmt.Println(string(content))
}

Writing Files

Creating or Opening a File for Writing

To write to a file, you can use os.Create or os.OpenFile. The os.Create function creates a file if it does not exist or truncates it if it does.

package main

import (
    "fmt"
    "os"
)

func main() {
    file, err := os.Create("example.txt")
    if err != nil {
        fmt.Println("Error creating file:", err)
        return
    }
    defer file.Close()
}

Writing Content to a File

You can write to a file using the Write or WriteString methods.

package main

import (
    "fmt"
    "os"
)

func main() {
    file, err := os.Create("example.txt")
    if err != nil {
        fmt.Println("Error creating file:", err)
        return
    }
    defer file.Close()

    _, err = file.WriteString("Hello, World!\n")
    if err != nil {
        fmt.Println("Error writing to file:", err)
    }
}

Using `bufio.Writer`

For buffered writing, you can use bufio.Writer.

package main

import (
    "bufio"
    "fmt"
    "os"
)

func main() {
    file, err := os.Create("example.txt")
    if err != nil {
        fmt.Println("Error creating file:", err)
        return
    }
    defer file.Close()

    writer := bufio.NewWriter(file)
    _, err = writer.WriteString("Hello, Buffered World!\n")
    if err != nil {
        fmt.Println("Error writing to file:", err)
    }
    writer.Flush()
}

Advanced File Operations

File Permissions

When creating or opening files, you can specify file permissions using os.OpenFile.

package main

import (
    "fmt"
    "os"
)

func main() {
    file, err := os.OpenFile("example.txt", os.O_CREATE|os.O_WRONLY, 0644)
    if err != nil {
        fmt.Println("Error opening file:", err)
        return
    }
    defer file.Close()

    _, err = file.WriteString("Hello with permissions!\n")
    if err != nil {
        fmt.Println("Error writing to file:", err)
    }
}

File Information

You can retrieve file information using the os.Stat function.

package main

import (
    "fmt"
    "os"
)

func main() {
    fileInfo, err := os.Stat("example.txt")
    if err != nil {
        fmt.Println("Error getting file info:", err)
        return
    }
    fmt.Printf("File Name: %s\n", fileInfo.Name())
    fmt.Printf("Size: %d bytes\n", fileInfo.Size())
    fmt.Printf("Permissions: %s\n", fileInfo.Mode())
    fmt.Printf("Last Modified: %s\n", fileInfo.ModTime())
}

Conclusion

File I/O is a crucial skill for any programmer. In Go, the os, bufio, and io/ioutil packages provide powerful tools for working with files. By understanding how to read from and write to files, handle errors, and manage file permissions, you can create robust applications that interact with the file system effectively.
