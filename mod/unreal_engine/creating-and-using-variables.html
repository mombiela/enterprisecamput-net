Introduction

Variables are fundamental in programming as they store data that can be used and manipulated throughout your code. In Unreal Engine, variables are used within Blueprints and C++ to control game logic, store player data, and manage game states. This section will guide you through the basics of creating and using variables in Unreal Engine, progressing from beginner to advanced concepts.

Beginner Level

What is a Variable?

  • A variable is a storage location identified by a name that holds data which can be modified during program execution.
  • Variables have types, which define the kind of data they can hold (e.g., integers, floats, strings).

Creating Variables in Blueprints

  1. Open a Blueprint Class: Start by opening any Blueprint class, such as an Actor or a Character.
  2. Add a Variable:
    • In the "My Blueprint" panel, click the "+" button next to "Variables".
    • Name your variable (e.g., Health).
    • Choose the variable type from the dropdown (e.g., Float for health points).
+-------------------+
| My Blueprint      |
|-------------------|
| Variables         |
| + Health (Float)  |
+-------------------+

Using Variables in Blueprints

  • Get and Set Nodes: Use "Get" and "Set" nodes to retrieve and modify the value of a variable.
  • Example: Setting the player's health to 100 when the game starts.
Event BeginPlay
    |
    V
Set Health (100)

Example Blueprint Code

Event BeginPlay
    |
    V
Set Health (100)

Creating Variables in C++

  1. Open Your C++ Class: Open any C++ class, such as a custom ACharacter class.
  2. Declare a Variable:
    • In the header file (.h), declare your variable.
    • Example: float Health;
// MyCharacter.h
public:
    float Health;
  1. Initialize the Variable:
    • In the source file (.cpp), initialize your variable in the constructor or another appropriate method.
// MyCharacter.cpp
AMyCharacter::AMyCharacter()
{
    Health = 100.0f;
}

Using Variables in C++

  • Access and Modify: Use the variable directly within your class methods.
  • Example: Print the player's health to the console.
UE_LOG(LogTemp, Warning, TEXT("Player Health: %f"), Health);

Intermediate Level

Variable Scope and Lifetime

  • Local Variables: Declared within a function and only accessible within that function.
  • Member Variables: Declared in a class and accessible by all class methods.
  • Global Variables: Declared outside any class or function and accessible from anywhere in the code (use sparingly).

Arrays and Collections

  • Arrays: Store multiple items of the same type.
  • Example in Blueprints: Create an array of enemies.
+-------------------+
| My Blueprint      |
|-------------------|
| Variables         |
| + Enemies (Array) |
+-------------------+
  • Example in C++:
TArray<AEnemy*> Enemies;

Structs

  • Definition: A struct is a custom data type that groups multiple variables.
  • Example in Blueprints: Create a struct for player stats.
+-------------------+
| My Blueprint      |
|-------------------|
| Structs           |
| + PlayerStats     |
+-------------------+
  • Example in C++:
USTRUCT(BlueprintType)
struct FPlayerStats
{
    GENERATED_BODY()

    UPROPERTY(EditAnywhere, BlueprintReadWrite)
    float Health;

    UPROPERTY(EditAnywhere, BlueprintReadWrite)
    float Stamina;
};

Advanced Level

Advanced Data Structures

  • Maps: Store key-value pairs.
  • Sets: Store unique items.
  • Example in C++:
TMap<FString, int32> PlayerScores;
TSet<int32> UniqueIDs;

Enums

  • Definition: Enums are user-defined types consisting of named values.
  • Example in Blueprints: Create an enum for weapon types.
+-------------------+
| My Blueprint      |
|-------------------|
| Enums             |
| + WeaponType      |
+-------------------+
  • Example in C++:
UENUM(BlueprintType)
enum class EWeaponType : uint8
{
    EWT_Sword UMETA(DisplayName = "Sword"),
    EWT_Bow UMETA(DisplayName = "Bow"),
    EWT_Axe UMETA(DisplayName = "Axe")
};

Using Pointers and References

  • Pointers: Variables that store memory addresses.
  • References: Aliases for existing variables.
  • Example in C++:
AEnemy* EnemyPointer;
AEnemy& EnemyReference = *EnemyPointer;

Conclusion

Understanding and effectively using variables is crucial for game development in Unreal Engine. Starting from basic variable creation and manipulation in Blueprints and C++, you can progress to more complex data structures and advanced concepts. Mastery of these topics will enable you to manage game data efficiently and create more dynamic and interactive gameplay experiences.
