Blueprint Scripting
Next ►

Blueprint Scripting is a powerful visual scripting system in Unreal Engine that allows you to create gameplay elements without writing a single line of code. This module will guide you through the basics of Blueprint Scripting, helping you understand how to create complex behaviors and interactions in your game.

Key Concepts

  1. Blueprint Classes: These are templates for creating objects in your game. They can contain components, variables, and functions.
  2. Nodes: The building blocks of Blueprints. They represent actions, events, and data.
  3. Events: Special nodes that trigger actions when certain conditions are met.
  4. Variables: Used to store data that can be accessed and modified by your Blueprints.
  5. Functions: Reusable blocks of code that can be called from other parts of your Blueprint.

Creating a Blueprint Class

  1. Open the Content Browser: Navigate to the Content Browser in the Unreal Engine editor.
  2. Create a New Blueprint Class: Right-click in the Content Browser, select "Blueprint Class," and choose a parent class (e.g., Actor).
  3. Name Your Blueprint: Give your new Blueprint a meaningful name.

Adding Components

  1. Open the Blueprint Editor: Double-click your new Blueprint to open the Blueprint Editor.
  2. Add Components: In the Components panel, click "Add Component" and choose the type of component you want to add (e.g., Static Mesh, Collision).

Creating a Simple Script

Example: Making an Object Rotate

  1. Add a Static Mesh Component: In the Components panel, add a Static Mesh component and assign a mesh to it.
  2. Create a Rotating Movement: In the Event Graph, right-click to add a "Tick" event. This event fires every frame.
  3. Add a Rotation Node: Drag off the "Tick" event and add a "Add Actor Local Rotation" node.
  4. Set Rotation Values: Create a "Make Rotator" node and connect it to the "Delta Rotation" input of the "Add Actor Local Rotation" node. Set the desired rotation values (e.g., 0, 1, 0 for rotation around the Y-axis).
Event Tick
    -> Add Actor Local Rotation
        -> Delta Rotation: Make Rotator (0, 1, 0)

Explanation

  • Event Tick: This event is called every frame.
  • Add Actor Local Rotation: This node rotates the actor locally.
  • Make Rotator: This node creates a rotator with specified values.

Practical Exercise

Exercise: Creating a Door that Opens

  1. Create a Door Blueprint: Follow the steps to create a new Blueprint class and name it "BP_Door."
  2. Add Components: Add a Static Mesh component for the door and a Box Collision component to detect player interaction.
  3. Setup Collision: In the Details panel of the Box Collision, set the collision presets to "OverlapAllDynamic."
  4. Create an Open Door Function:
    • Add a custom event called "OpenDoor."
    • Add a "Timeline" node and create a float track that goes from 0 to 90 over 1 second.
    • Connect the "Update" pin of the Timeline to a "Set Relative Rotation" node.
    • Use a "Make Rotator" node to set the rotation values.
  5. Trigger the Function:
    • Add an "OnComponentBeginOverlap" event for the Box Collision.
    • Connect it to the "OpenDoor" custom event.
Box Collision: OnComponentBeginOverlap
    -> OpenDoor

    -> Timeline (0 to 90 over 1 second)
        -> Set Relative Rotation
            -> New Rotation: Make Rotator (0, 90, 0)

Common Mistakes and Tips

  • Incorrect Node Connections: Ensure all nodes are connected correctly. Misconnections can lead to unexpected behavior.
  • Timeline Setup: Make sure the timeline is set up correctly with the right values and duration.
  • Collision Settings: Ensure the collision settings are correct to detect overlaps properly.

Conclusion
