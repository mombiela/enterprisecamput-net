Introduction

In this section of the course, we will explore how to take automation to the next level through advanced customization. We will learn to use tools and techniques that allow us to adapt automated processes to the specific needs of each project or client.

Key Concepts

  • Automation: Use of technology to perform repetitive tasks without human intervention.
  • Customization: Adaptation of processes and tools to meet specific needs.
  • Scripts: Small programs that automate specific tasks.
  • APIs: Programming interfaces that allow interaction between different systems.
  • Workflows: Sequences of automated tasks that are executed in a specific order.

Automation Tools

Scripts

Scripts are a powerful way to customize automation. They can be written in various programming languages such as Python, Bash, PowerShell, among others.

Example of a Python Script

import os

def backup_files(source_dir, backup_dir):
    if not os.path.exists(backup_dir):
        os.makedirs(backup_dir)
    for filename in os.listdir(source_dir):
        full_file_name = os.path.join(source_dir, filename)
        if os.path.isfile(full_file_name):
            os.path.copy(full_file_name, backup_dir)

source_directory = '/path/to/source'
backup_directory = '/path/to/backup'

backup_files(source_directory, backup_directory)

Explanation:

  • import os: Imports the os module to interact with the operating system.
  • backup_files: Function that copies files from one directory to another.
  • os.makedirs: Creates the backup directory if it does not exist.
  • os.listdir: Lists the files in the source directory.
  • os.path.isfile: Checks if the file is a regular file.
  • os.path.copy: Copies the file to the backup directory.

APIs

APIs allow integration and automation between different systems and applications.

Example of Using an API in Python

import requests

def get_weather(city):
    api_key = 'your_api_key'
    base_url = 'http://api.openweathermap.org/data/2.5/weather'
    complete_url = f"{base_url}?q={city}&appid={api_key}"
    response = requests.get(complete_url)
    data = response.json()
    if data['cod'] == 200:
        main = data['main']
        temperature = main['temp']
        print(f"Temperature in {city}: {temperature}K")
    else:
        print(f"City {city} not found.")

city_name = 'London'
get_weather(city_name)

Explanation:

  • import requests: Imports the requests module to make HTTP requests.
  • get_weather: Function that gets the weather of a city using the OpenWeatherMap API.
  • requests.get: Makes a GET request to the API.
  • response.json(): Converts the response to JSON format.
  • data['main']: Accesses the main section of the weather data.
  • data['cod']: Checks the response code to ensure the city was found.

Workflows

Workflows allow the execution of a series of automated tasks in a specific order. Tools like Jenkins, GitHub Actions, and Zapier are popular for creating workflows.

Example of a Workflow in GitHub Actions

name: CI

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Python
      uses: actions/setup-python@v2
      with:
        python-version: '3.x'

    - name: Install dependencies
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt

    - name: Run tests
      run: |
        pytest

Explanation:

  • name: Name of the workflow.
  • on: Event that triggers the workflow (in this case, a push to the main branch).
  • jobs: Defines the jobs to be performed.
  • runs-on: Specifies the operating system on which the job will run.
  • steps: List of steps to follow in the job.
  • actions/checkout@v2: Action to check out the code.
  • actions/setup-python@v2: Action to set up Python.
  • pip install -r requirements.txt: Installs the project's dependencies.
  • pytest: Runs the project's tests.

Advanced automation and customization allow optimizing and adapting processes to the specific needs of each project. By using scripts, APIs, and workflows, it is possible to create efficient and customized automated solutions. The key is to understand the available tools and how to combine them to achieve the desired objectives.
