5-3: Inheritance

Introduction to Inheritance

Inheritance is a fundamental concept in object-oriented programming (OOP) that allows a class to inherit attributes and methods from another class. This promotes code reusability and establishes a natural hierarchy between classes.

Key Concepts

  1. Base Class (Parent Class): The class whose properties and methods are inherited.
  2. Derived Class (Child Class): The class that inherits properties and methods from the base class.
  3. super() Function: A function used to call methods from the base class within the derived class.

Syntax

class BaseClass:
    # Base class code

class DerivedClass(BaseClass):
    # Derived class code

Example

Let's create a simple example to understand inheritance better.

Base Class

class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        raise NotImplementedError("Subclass must implement abstract method")

Derived Class

class Dog(Animal):
    def speak(self):
        return f"{self.name} says Woof!"

class Cat(Animal):
    def speak(self):
        return f"{self.name} says Meow!"

Using the Classes

dog = Dog("Buddy")
cat = Cat("Whiskers")

print(dog.speak())  # Output: Buddy says Woof!
print(cat.speak())  # Output: Whiskers says Meow!

Detailed Explanation

  1. Base Class (Animal):

    • The Animal class has an __init__ method that initializes the name attribute.
    • It also has a speak method that raises a NotImplementedError. This is a way to enforce that derived classes must implement this method.

  2. Derived Classes (Dog and Cat):

    • Both Dog and Cat classes inherit from the Animal class.
    • They override the speak method to provide specific implementations.

  3. Creating Instances:

    • Instances of Dog and Cat are created with names "Buddy" and "Whiskers", respectively.
    • Calling the speak method on these instances returns the respective sounds.

Practical Exercises

Exercise 1: Creating a Derived Class

Create a base class Vehicle with attributes make and model, and a method info that prints these attributes. Then, create a derived class Car that adds an attribute year and overrides the info method to include the year.

class Vehicle:
    def __init__(self, make, model):
        self.make = make
        self.model = model

    def info(self):
        print(f"Make: {self.make}, Model: {self.model}")

class Car(Vehicle):
    def __init__(self, make, model, year):
        super().__init__(make, model)
        self.year = year

    def info(self):
        print(f"Make: {self.make}, Model: {self.model}, Year: {self.year}")

# Test the classes
car = Car("Toyota", "Corolla", 2020)
car.info()  # Output: Make: Toyota, Model: Corolla, Year: 2020

Solution Explanation

  1. Base Class (Vehicle):

    • The Vehicle class has an __init__ method that initializes make and model.
    • The info method prints the make and model.

  2. Derived Class (Car):

    • The Car class inherits from Vehicle and adds a year attribute.
    • It uses super().__init__(make, model) to call the base class constructor.
    • The info method is overridden to include the year.

Common Mistakes and Tips

  1. Not Using super() Correctly:

    • Ensure you use super() to call the base class constructor when initializing derived class attributes.
    • Example: super().__init__(make, model).

  2. Forgetting to Override Methods:

    • If a base class method is meant to be overridden, ensure derived classes provide their own implementation.
    • Use raise NotImplementedError in the base class to enforce this.

  3. Understanding Method Resolution Order (MRO):

    • Python uses the C3 linearization algorithm to determine the method resolution order. Use ClassName.__mro__ to inspect the MRO.

Summary

  • Inheritance allows a class to inherit properties and methods from another class, promoting code reusability.
  • Use the super() function to call methods from the base class.
  • Override methods in derived classes to provide specific implementations.
  • Practice creating and using base and derived classes to solidify your understanding.
