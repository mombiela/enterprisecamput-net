Topic 5-1: Classes and Objects

Introduction

Object-Oriented Programming (OOP) is a programming paradigm that uses "objects" to design applications and computer programs. It simplifies complex software design by modeling real-world entities as classes and objects.

Key Concepts

  1. Class: A blueprint for creating objects (a particular data structure), containing methods (functions) and attributes (data).
  2. Object: An instance of a class. When a class is defined, no memory is allocated until an object of that class is created.
  3. Attributes: Variables that belong to a class.
  4. Methods: Functions that belong to a class.

Defining a Class

A class is defined using the class keyword. Here’s a simple example:

class Dog:
    # Class attribute
    species = "Canis familiaris"

    # Initializer / Instance attributes
    def __init__(self, name, age):
        self.name = name
        self.age = age

    # Instance method
    def description(self):
        return f"{self.name} is {self.age} years old"

    # Another instance method
    def speak(self, sound):
        return f"{self.name} says {sound}"

Creating Objects

Objects are instances of a class. Here’s how you create an object of the Dog class:

# Instantiate the Dog class
buddy = Dog("Buddy", 9)
miles = Dog("Miles", 4)

# Access the instance attributes
print(buddy.name)  # Output: Buddy
print(buddy.age)   # Output: 9

# Call our instance methods
print(buddy.description())  # Output: Buddy is 9 years old
print(miles.speak("Woof Woof"))  # Output: Miles says Woof Woof

Practical Example

Let's create a simple class Car and instantiate objects from it:

class Car:
    # Class attribute
    wheels = 4

    # Initializer / Instance attributes
    def __init__(self, make, model, year):
        self.make = make
        self.model = model
        self.year = year

    # Instance method
    def description(self):
        return f"{self.year} {self.make} {self.model}"

# Instantiate the Car class
car1 = Car("Toyota", "Corolla", 2020)
car2 = Car("Honda", "Civic", 2019)

# Access the instance attributes
print(car1.make)  # Output: Toyota
print(car1.model)  # Output: Corolla
print(car1.year)  # Output: 2020

# Call the instance method
print(car1.description())  # Output: 2020 Toyota Corolla
print(car2.description())  # Output: 2019 Honda Civic

Exercises

Exercise 1

Create a class Book with the following attributes:

  • title
  • author
  • pages

Include a method description that returns a string in the format: "Title by Author, Pages pages".

Solution:

class Book:
    def __init__(self, title, author, pages):
        self.title = title
        self.author = author
        self.pages = pages

    def description(self):
        return f"{self.title} by {self.author}, {self.pages} pages"

# Instantiate the Book class
book1 = Book("1984", "George Orwell", 328)
book2 = Book("To Kill a Mockingbird", "Harper Lee", 281)

# Call the instance method
print(book1.description())  # Output: 1984 by George Orwell, 328 pages
print(book2.description())  # Output: To Kill a Mockingbird by Harper Lee, 281 pages

Exercise 2

Create a class Student with the following attributes:

  • name
  • student_id
  • grade

Include a method is_passing that returns True if the grade is 60 or above, and False otherwise.

Solution:

class Student:
    def __init__(self, name, student_id, grade):
        self.name = name
        self.student_id = student_id
        self.grade = grade

    def is_passing(self):
        return self.grade >= 60

# Instantiate the Student class
student1 = Student("Alice", "S12345", 85)
student2 = Student("Bob", "S67890", 55)

# Call the instance method
print(student1.is_passing())  # Output: True
print(student2.is_passing())  # Output: False

Common Mistakes and Tips

  • Forgetting self in method definitions: Always include self as the first parameter in instance methods.
  • Confusing class attributes with instance attributes: Class attributes are shared across all instances, while instance attributes are unique to each instance.
  • Not initializing attributes in the __init__ method: Ensure all instance attributes are initialized in the __init__ method.

Conclusion

In this lesson, you learned about the basics of classes and objects in Python. You now know how to define a class, create objects, and use attributes and methods. These concepts form the foundation of Object-Oriented Programming and will be built upon in subsequent lessons.
