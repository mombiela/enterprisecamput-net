Topic 6-3: Handling Exceptions

Introduction

Exception handling is a crucial part of writing robust and error-free code. In Python, exceptions are events that can alter the normal flow of a program. This topic will cover how to handle exceptions using try, except, else, and finally blocks.

Key Concepts

  1. Exceptions: Errors detected during execution.
  2. try block: Code that might throw an exception.
  3. except block: Code that runs if an exception occurs.
  4. else block: Code that runs if no exception occurs.
  5. finally block: Code that runs no matter what, often used for cleanup.

Basic Syntax

try:
    # Code that might raise an exception
    risky_code()
except SomeException as e:
    # Code that runs if an exception occurs
    handle_exception(e)
else:
    # Code that runs if no exception occurs
    no_exception_occurred()
finally:
    # Code that runs no matter what
    cleanup()

Practical Examples

Example 1: Handling a ZeroDivisionError

try:
    result = 10 / 0
except ZeroDivisionError as e:
    print(f"Error: {e}")
else:
    print(f"Result: {result}")
finally:
    print("Execution completed.")

Explanation:

  • The try block contains code that may raise a ZeroDivisionError.
  • The except block handles the exception and prints an error message.
  • The else block would execute if no exception occurs (not in this case).
  • The finally block always executes, useful for cleanup actions.

Example 2: Handling Multiple Exceptions

try:
    value = int(input("Enter a number: "))
    result = 10 / value
except ValueError as e:
    print(f"Invalid input: {e}")
except ZeroDivisionError as e:
    print(f"Cannot divide by zero: {e}")
else:
    print(f"Result: {result}")
finally:
    print("Execution completed.")

Explanation:

  • The try block contains code that may raise either a ValueError or ZeroDivisionError.
  • Multiple except blocks handle different exceptions separately.
  • The else block executes if no exception occurs.
  • The finally block always executes.

Practical Exercises

Exercise 1: Handling File Not Found

Task: Write a program that attempts to open a file and handles the FileNotFoundError.

try:
    with open('non_existent_file.txt', 'r') as file:
        content = file.read()
except FileNotFoundError as e:
    print(f"Error: {e}")
else:
    print(content)
finally:
    print("Execution completed.")

Solution:

try:
    with open('non_existent_file.txt', 'r') as file:
        content = file.read()
except FileNotFoundError as e:
    print(f"Error: {e}")
else:
    print(content)
finally:
    print("Execution completed.")

Explanation:

  • The try block attempts to open a non-existent file.
  • The except block handles the FileNotFoundError.
  • The else block would print the file content if no exception occurs.
  • The finally block always executes.

Exercise 2: Handling Multiple Exceptions

Task: Write a program that takes user input for a filename and a divisor, then handles potential exceptions.

try:
    filename = input("Enter the filename: ")
    divisor = int(input("Enter a divisor: "))
    with open(filename, 'r') as file:
        content = file.read()
    result = len(content) / divisor
except FileNotFoundError as e:
    print(f"Error: {e}")
except ValueError as e:
    print(f"Invalid input: {e}")
except ZeroDivisionError as e:
    print(f"Cannot divide by zero: {e}")
else:
    print(f"Result: {result}")
finally:
    print("Execution completed.")

Solution:

try:
    filename = input("Enter the filename: ")
    divisor = int(input("Enter a divisor: "))
    with open(filename, 'r') as file:
        content = file.read()
    result = len(content) / divisor
except FileNotFoundError as e:
    print(f"Error: {e}")
except ValueError as e:
    print(f"Invalid input: {e}")
except ZeroDivisionError as e:
    print(f"Cannot divide by zero: {e}")
else:
    print(f"Result: {result}")
finally:
    print("Execution completed.")

Explanation:

  • The try block contains code that may raise FileNotFoundError, ValueError, or ZeroDivisionError.
  • Multiple except blocks handle different exceptions separately.
  • The else block executes if no exception occurs.
  • The finally block always executes.

Common Mistakes and Tips

  • Common Mistake: Not specifying the exception type in the except block.

    • Tip: Always specify the exception type to handle specific errors.

  • Common Mistake: Using a broad except block that catches all exceptions.

    • Tip: Use specific exceptions to avoid masking other errors.

  • Common Mistake: Forgetting the finally block for cleanup.

    • Tip: Always use the finally block for cleanup actions like closing files or releasing resources.

Conclusion

In this topic, you learned how to handle exceptions in Python using try, except, else, and finally blocks. Exception handling is essential for writing robust and error-free code. By practicing the provided exercises, you can reinforce your understanding and become proficient in managing exceptions in your Python programs.
