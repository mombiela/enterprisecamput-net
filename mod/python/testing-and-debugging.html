Introduction

Testing and debugging are crucial skills for any programmer. They ensure that your code works as expected and help you identify and fix errors. This section will cover the basics of testing and debugging in Python, progressing from beginner to advanced levels.

Unit Testing with `unittest`

Unit testing involves testing individual components of your code to ensure they work correctly.

Basics of `unittest`

  • What is unittest?
    • unittest is a built-in Python module for creating and running tests.
  • Creating a Test Case
    • A test case is a single unit of testing. It checks for a specific response to a particular set of inputs.
import unittest

def add(a, b):
    return a + b

class TestMathOperations(unittest.TestCase):
    def test_add(self):
        self.assertEqual(add(1, 2), 3)
        self.assertEqual(add(-1, 1), 0)

if __name__ == '__main__':
    unittest.main()

Running Tests

  • Command Line
    • Run your tests from the command line using: 
      python -m unittest test_module.py
  • Test Discovery
    • Automatically discover and run tests: 
      python -m unittest discover

Advanced Testing with `pytest`

pytest is a more powerful testing framework that simplifies writing and running tests.

Basics of `pytest`

  • Installation
    • Install pytest using pip: 
      pip install pytest
  • Writing Tests
    • Tests are written as functions prefixed with test_.
def add(a, b):
    return a + b

def test_add():
    assert add(1, 2) == 3
    assert add(-1, 1) == 0

Running Tests

  • Command Line
    • Run your tests using: 
      pytest

Fixtures

  • What are Fixtures?
    • Fixtures are used to set up a known state before tests run and clean up afterward.
import pytest

@pytest.fixture
def sample_data():
    return [1, 2, 3, 4, 5]

def test_sum(sample_data):
    assert sum(sample_data) == 15

Debugging with `pdb`

pdb is Python's built-in debugger, allowing you to step through code, inspect variables, and evaluate expressions.

Basics of `pdb`

  • Starting the Debugger
    • Insert pdb.set_trace() at the point where you want to start debugging.
import pdb

def add(a, b):
    pdb.set_trace()
    return a + b

print(add(1, 2))

Common Commands

  • n (next)
    • Move to the next line within the same function.
  • s (step)
    • Step into a function call.
  • c (continue)
    • Continue execution until the next breakpoint.
  • p (print)
    • Print the value of an expression.

Advanced Debugging Techniques

Using `logging`

  • Why Use Logging?
    • Logging provides a way to track events that happen when some software runs. It can be more flexible and informative than print statements.
import logging

logging.basicConfig(level=logging.DEBUG)

def add(a, b):
    logging.debug(f'Adding {a} and {b}')
    return a + b

print(add(1, 2))

Profiling Code

  • Why Profile?
    • Profiling helps you understand where your program spends its time and identify bottlenecks.
import cProfile

def slow_function():
    total = 0
    for i in range(10000):
        total += i
    return total

cProfile.run('slow_function()')

Conclusion

Testing and debugging are essential practices for writing reliable and maintainable Python code. By mastering tools like unittest, pytest, pdb, and logging, you can ensure your code is robust and efficient. Remember, the goal is not just to find bugs but to understand why they occur and how to prevent them in the future.
