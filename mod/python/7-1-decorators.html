7-1: Decorators

Introduction to Decorators

Decorators are a powerful and useful tool in Python that allows you to modify the behavior of a function or class. They are often used to add functionality to existing code in a clean and readable way.

Key Concepts

  1. Function as First-Class Objects:

    • In Python, functions are first-class objects, meaning they can be passed around and used as arguments just like any other object (string, int, float, etc.).

  2. Inner Functions:

    • Python supports the definition of functions inside other functions, known as inner functions.

  3. Higher-Order Functions:

    • A function that takes another function as an argument or returns a function as a result.

  4. Decorators:

    • A decorator is a function that takes another function and extends its behavior without explicitly modifying it.

Basic Syntax

def decorator_function(original_function):
    def wrapper_function():
        print("Wrapper executed this before {}".format(original_function.__name__))
        return original_function()
    return wrapper_function

@decorator_function
def display():
    print("Display function ran")

display()

Explanation

  • Decorator Function: decorator_function takes original_function as an argument and defines an inner function wrapper_function that adds some behavior before calling original_function.
  • Wrapper Function: This inner function adds functionality (in this case, a print statement) before calling the original function.
  • @decorator_function: This is a syntactic sugar to apply the decorator to display function.

Practical Example

Let's create a decorator that logs the execution time of a function.

import time

def timer_decorator(func):
    def wrapper(*args, **kwargs):
        start_time = time.time()
        result = func(*args, **kwargs)
        end_time = time.time()
        print(f"{func.__name__} executed in {end_time - start_time} seconds")
        return result
    return wrapper

@timer_decorator
def slow_function():
    time.sleep(2)
    print("Function complete")

slow_function()

Explanation

  • timer_decorator: This decorator measures the time taken by the function to execute.
  • wrapper: This inner function captures the start time, calls the original function, captures the end time, and prints the elapsed time.
  • @timer_decorator: This applies the timer_decorator to slow_function.

Exercises

Exercise 1: Create a Simple Logger Decorator

Create a decorator named logger that logs the name of the function being called.

def logger(func):
    def wrapper(*args, **kwargs):
        print(f"Function {func.__name__} is about to be called")
        result = func(*args, **kwargs)
        print(f"Function {func.__name__} has been called")
        return result
    return wrapper

@logger
def greet(name):
    print(f"Hello, {name}!")

greet("Alice")

Solution

def logger(func):
    def wrapper(*args, **kwargs):
        print(f"Function {func.__name__} is about to be called")
        result = func(*args, **kwargs)
        print(f"Function {func.__name__} has been called")
        return result
    return wrapper

@logger
def greet(name):
    print(f"Hello, {name}!")

greet("Alice")

Common Mistakes and Tips

  • Forgetting to return the wrapper function: Ensure that the decorator returns the wrapper function.
  • Not using *args and **kwargs in the wrapper: This is crucial to ensure that the wrapper can handle any number of positional and keyword arguments.
  • Decorating functions with arguments: Always use *args and **kwargs in the wrapper to pass arguments to the original function.

Conclusion

Decorators are a powerful feature in Python that allows you to add functionality to functions or methods in a clean and readable way. By understanding and using decorators, you can write more modular and maintainable code. In the next section, we will explore generators, another advanced topic in Python.
