Introduction to NoSQL Databases

NoSQL databases are designed to handle large volumes of data, high user loads, and frequent changes to data structures. Unlike traditional SQL databases, NoSQL databases do not use a fixed schema, making them more flexible and scalable.

Key Concepts

  • Schema-less: NoSQL databases do not require a predefined schema.
  • Scalability: Designed to scale out by distributing data across multiple servers.
  • Types of NoSQL Databases:
    • Document Stores: Store data in JSON-like documents (e.g., MongoDB).
    • Key-Value Stores: Store data as key-value pairs (e.g., Redis).
    • Column Stores: Store data in columns rather than rows (e.g., Cassandra).
    • Graph Databases: Store data in graph structures (e.g., Neo4j).

Setting Up the Environment

Before interacting with NoSQL databases in Python, you need to install the necessary libraries. For this course, we will use MongoDB as an example.

Installing MongoDB and pymongo

  1. Install MongoDB: Follow the instructions on the official MongoDB website to install MongoDB on your system.
  2. Install pymongo: Use pip to install the pymongo library, which allows Python to interact with MongoDB. 
    pip install pymongo

Connecting to MongoDB

To interact with MongoDB, you need to establish a connection using the pymongo library.

Example: Connecting to MongoDB

from pymongo import MongoClient

# Create a MongoClient to the running MongoDB instance
client = MongoClient('localhost', 27017)

# Access the 'test' database
db = client.test

print("Connected to MongoDB")
  • MongoClient: This class is used to create a connection to the MongoDB server.
  • localhost: The hostname where MongoDB is running.
  • 27017: The default port number for MongoDB.

CRUD Operations

CRUD stands for Create, Read, Update, and Delete. These are the basic operations you can perform on a database.

Create

To insert a document into a collection, use the insert_one or insert_many methods.

Example: Inserting a Document

# Define a sample document
document = {
    "name": "Alice",
    "age": 30,
    "city": "New York"
}

# Insert the document into the 'users' collection
result = db.users.insert_one(document)

print(f"Inserted document with ID: {result.inserted_id}")

Read

To retrieve documents from a collection, use the find or find_one methods.

Example: Retrieving Documents

# Retrieve a single document
user = db.users.find_one({"name": "Alice"})
print(user)

# Retrieve multiple documents
users = db.users.find({"age": {"$gt": 25}})
for user in users:
    print(user)
  • find_one: Retrieves a single document that matches the query.
  • find: Retrieves multiple documents that match the query.
  • $gt: A MongoDB query operator that stands for "greater than".

Update

To update documents, use the update_one or update_many methods.

Example: Updating a Document

# Update the age of the user named 'Alice'
result = db.users.update_one(
    {"name": "Alice"},  # Query to match the document
    {"$set": {"age": 31}}  # Update operation
)

print(f"Matched {result.matched_count} document(s) and modified {result.modified_count} document(s)")
  • $set: A MongoDB update operator used to set the value of a field.

Delete

To delete documents, use the delete_one or delete_many methods.

Example: Deleting a Document

# Delete the user named 'Alice'
result = db.users.delete_one({"name": "Alice"})

print(f"Deleted {result.deleted_count} document(s)")

Summary

In this topic, we covered the basics of interacting with NoSQL databases using Python, specifically focusing on MongoDB. We discussed:

  • The key concepts of NoSQL databases.
  • Setting up the environment and installing necessary libraries.
  • Establishing a connection to MongoDB.
  • Performing CRUD operations: Create, Read, Update, and Delete.

By understanding these fundamental operations, you can effectively manage and manipulate data in a NoSQL database using Python.
