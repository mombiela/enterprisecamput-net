Topic 10-3: Using pdb for Debugging

Introduction

Debugging is an essential skill for any programmer. Python provides a powerful built-in debugger called pdb (Python Debugger). This tool allows you to set breakpoints, step through code, inspect variables, and evaluate expressions. In this section, we will explore how to use pdb to debug Python programs effectively.

Key Concepts

  1. Breakpoints: Points in the code where the execution will pause, allowing you to inspect the state of the program.
  2. Stepping: Executing the program one line at a time to observe its behavior.
  3. Inspecting Variables: Checking the values of variables at different points in the execution.
  4. Evaluating Expressions: Running Python expressions in the context of the paused program to understand its state.

Setting Up pdb

To start using pdb, you need to import it in your script and set breakpoints where you want the execution to pause.

import pdb

def add(a, b):
    pdb.set_trace()  # Set a breakpoint here
    return a + b

result = add(3, 5)
print(result)

Basic pdb Commands

Here are some basic pdb commands you can use:

Command Description
l or list List the source code around the current line.
n or next Continue execution until the next line in the current function is reached.
s or step Execute the current line and stop at the first possible occasion (either in a function that is called or in the current function).
c or continue Continue execution until the next breakpoint is encountered.
p or print Print the value of an expression.
q or quit Quit the debugger and terminate the program.

Practical Example

Let's walk through a practical example to see how pdb can be used to debug a simple program.

import pdb

def factorial(n):
    if n == 0:
        return 1
    else:
        return n * factorial(n - 1)

def main():
    pdb.set_trace()  # Set a breakpoint here
    num = 5
    result = factorial(num)
    print(f"The factorial of {num} is {result}")

if __name__ == "__main__":
    main()

Step-by-Step Debugging

  1. Run the Script: Execute the script in the terminal. 
    python script.py
  2. Breakpoint Hit: The execution will pause at pdb.set_trace().
  3. Inspect Variables: Use the p command to inspect variables. 
    (Pdb) p num
5
  4. Step Through Code: Use the n command to step through the code. 
    (Pdb) n
> script.py(12)main()
-> result = factorial(num)
  5. Continue Execution: Use the c command to continue execution until the next breakpoint or the end of the program. 
    (Pdb) c
The factorial of 5 is 120

Common Mistakes and Tips

  • Forgetting to Remove Breakpoints: Remember to remove or comment out pdb.set_trace() before deploying your code.
  • Using print Statements: While print statements can be useful, pdb provides a more interactive and powerful way to debug your code.
  • Not Understanding the Flow: Use the list command to see the surrounding code and understand the flow of execution.

Exercises

Exercise 1: Debugging a Function

Given the following code, use pdb to find and fix the bug.

def divide(a, b):
    return a / b

def main():
    result = divide(10, 0)
    print(f"Result: {result}")

if __name__ == "__main__":
    main()

Solution:

  1. Add pdb.set_trace() before the division operation.
  2. Run the script and use pdb commands to inspect the values of a and b.
  3. Identify the division by zero error and modify the code to handle this case.
import pdb

def divide(a, b):
    pdb.set_trace()  # Set a breakpoint here
    if b == 0:
        return "Error: Division by zero"
    return a / b

def main():
    result = divide(10, 0)
    print(f"Result: {result}")

if __name__ == "__main__":
    main()

Conclusion

In this section, we explored the basics of using pdb for debugging Python programs. We learned how to set breakpoints, step through code, inspect variables, and evaluate expressions. By mastering pdb, you can significantly improve your debugging skills and write more reliable code. In the next topic, we will delve into packaging and distributing Python applications.
