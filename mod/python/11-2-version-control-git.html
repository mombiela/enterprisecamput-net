11-2: Version Control with Git

Introduction

Version control is a system that records changes to a file or set of files over time so that you can recall specific versions later. Git is a distributed version control system that allows multiple developers to work on a project simultaneously without interfering with each otherâ€™s work.

Key Concepts

  1. Repository (Repo): A directory which contains your project work, as well as a few files (hidden by default) which are used by Git to keep track of changes.
  2. Commit: A snapshot of your repository at a specific point in time.
  3. Branch: A parallel version of your repository. It allows you to work on different features or bug fixes independently.
  4. Merge: Combining changes from different branches.
  5. Clone: A copy of a repository.
  6. Pull: Fetching changes from a remote repository and merging them into your local repository.
  7. Push: Sending your committed changes to a remote repository.

Setting Up Git

  1. Install Git:

    • On Windows: Download and install from git-scm.com.
    • On macOS: Use Homebrew: brew install git.
    • On Linux: Use your package manager, e.g., sudo apt-get install git on Debian-based systems.

  2. Configure Git:

    git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

Basic Git Commands

  1. Initialize a Repository:

    git init

    This command creates a new Git repository.

  2. Clone a Repository:

    git clone https://github.com/user/repo.git

    This command creates a copy of an existing repository.

  3. Check Repository Status:

    git status

    This command shows the status of changes as untracked, modified, or staged.

  4. Add Files to Staging Area:

    git add filename

    This command stages changes for the next commit.

  5. Commit Changes:

    git commit -m "Commit message"

    This command commits the staged changes to the repository.

  6. View Commit History:

    git log

    This command shows the commit history.

  7. Create a New Branch:

    git branch branch_name

    This command creates a new branch.

  8. Switch to a Branch:

    git checkout branch_name

    This command switches to the specified branch.

  9. Merge Branches:

    git merge branch_name

    This command merges the specified branch into the current branch.

  10. Push Changes to Remote Repository:

    git push origin branch_name

    This command pushes your changes to the remote repository.

  11. Pull Changes from Remote Repository:

    git pull origin branch_name

    This command fetches and merges changes from the remote repository.

Practical Example

Let's walk through a simple example of using Git:

  1. Initialize a Repository:

    mkdir my_project
cd my_project
git init

  2. Create a File and Add Content:

    echo "print('Hello, World!')" > hello.py

  3. Check Repository Status:

    git status

  4. Stage the File:

    git add hello.py

  5. Commit the File:

    git commit -m "Add hello.py with a simple print statement"

  6. Create a New Branch:

    git branch new_feature

  7. Switch to the New Branch:

    git checkout new_feature

  8. Make Changes and Commit:

    echo "print('New feature')" >> hello.py
git add hello.py
git commit -m "Add new feature to hello.py"

  9. Merge Changes Back to Main Branch:

    git checkout main
git merge new_feature

Exercises

  1. Exercise 1: Initialize a new Git repository and create a file named example.txt. Add some text to the file, stage it, and commit it.
  2. Exercise 2: Create a new branch named feature_branch, switch to it, and make some changes to example.txt. Commit the changes and merge them back to the main branch.
  3. Exercise 3: Clone an existing repository from GitHub, make some changes, and push them back to the remote repository.

Solutions

  1. Solution 1:

    mkdir my_repo
cd my_repo
git init
echo "This is an example file." > example.txt
git add example.txt
git commit -m "Add example.txt"

  2. Solution 2:

    git branch feature_branch
git checkout feature_branch
echo "Adding more text." >> example.txt
git add example.txt
git commit -m "Update example.txt"
git checkout main
git merge feature_branch

  3. Solution 3:

    git clone https://github.com/user/repo.git
cd repo
echo "Some changes" >> file.txt
git add file.txt
git commit -m "Update file.txt"
git push origin main

Common Mistakes and Tips

  • Forgetting to Stage Changes: Always remember to use git add to stage your changes before committing.
  • Commit Messages: Use clear and descriptive commit messages to make it easier to understand the history of changes.
  • Branch Management: Regularly merge branches to avoid large merge conflicts.

Conclusion

In this section, we covered the basics of Git, including setting up a repository, making commits, branching, and merging. Understanding these concepts is crucial for effective collaboration and version control in software development.
