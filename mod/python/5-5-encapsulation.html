Topic 5-5: Encapsulation

Introduction to Encapsulation

Encapsulation is one of the fundamental principles of Object-Oriented Programming (OOP). It refers to the bundling of data (attributes) and methods (functions) that operate on the data into a single unit, typically a class. Encapsulation also involves restricting direct access to some of an object's components, which is a means of preventing accidental interference and misuse of the data.

Key Concepts

  1. Private Attributes and Methods:

    • In Python, private attributes and methods are denoted by a leading underscore (_) or double underscore (__).
    • Example: 
      class MyClass:
    def __init__(self):
        self._private_attribute = "I am private"
        self.__very_private_attribute = "I am very private"

    def _private_method(self):
        print("This is a private method")

    def __very_private_method(self):
        print("This is a very private method")

  2. Getter and Setter Methods:

    • These methods provide controlled access to private attributes.
    • Example: 
      class MyClass:
    def __init__(self):
        self.__attribute = None

    def get_attribute(self):
        return self.__attribute

    def set_attribute(self, value):
        self.__attribute = value

  3. Property Decorators:

    • Python provides a more elegant way to define getters and setters using the @property decorator.
    • Example: 
      class MyClass:
    def __init__(self):
        self.__attribute = None

    @property
    def attribute(self):
        return self.__attribute

    @attribute.setter
    def attribute(self, value):
        self.__attribute = value

Practical Example

Let's create a class BankAccount that demonstrates encapsulation by using private attributes and methods, along with getter and setter methods.

class BankAccount:
    def __init__(self, owner, balance=0):
        self.__owner = owner
        self.__balance = balance

    @property
    def balance(self):
        return self.__balance

    @balance.setter
    def balance(self, amount):
        if amount < 0:
            print("Balance cannot be negative")
        else:
            self.__balance = amount

    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount
            print(f"Deposited {amount}. New balance is {self.__balance}")
        else:
            print("Deposit amount must be positive")

    def withdraw(self, amount):
        if 0 < amount <= self.__balance:
            self.__balance -= amount
            print(f"Withdrew {amount}. New balance is {self.__balance}")
        else:
            print("Invalid withdrawal amount")

# Usage
account = BankAccount("Alice", 100)
print(account.balance)  # Output: 100
account.deposit(50)     # Output: Deposited 50. New balance is 150
account.withdraw(30)    # Output: Withdrew 30. New balance is 120
account.balance = -50   # Output: Balance cannot be negative

Exercises

  1. Exercise 1: Create a class Student with private attributes name and grade. Implement getter and setter methods for these attributes using property decorators.

  2. Exercise 2: Modify the BankAccount class to include a private method _log_transaction that logs every deposit and withdrawal. Ensure this method is called within the deposit and withdraw methods.

Solutions

Exercise 1 Solution:

class Student:
    def __init__(self, name, grade):
        self.__name = name
        self.__grade = grade

    @property
    def name(self):
        return self.__name

    @name.setter
    def name(self, value):
        self.__name = value

    @property
    def grade(self):
        return self.__grade

    @grade.setter
    def grade(self, value):
        if 0 <= value <= 100:
            self.__grade = value
        else:
            print("Grade must be between 0 and 100")

# Usage
student = Student("Bob", 85)
print(student.name)  # Output: Bob
print(student.grade) # Output: 85
student.grade = 95   # Valid grade
print(student.grade) # Output: 95
student.grade = 105  # Invalid grade, Output: Grade must be between 0 and 100

Exercise 2 Solution:

class BankAccount:
    def __init__(self, owner, balance=0):
        self.__owner = owner
        self.__balance = balance

    @property
    def balance(self):
        return self.__balance

    @balance.setter
    def balance(self, amount):
        if amount < 0:
            print("Balance cannot be negative")
        else:
            self.__balance = amount

    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount
            self._log_transaction("deposit", amount)
            print(f"Deposited {amount}. New balance is {self.__balance}")
        else:
            print("Deposit amount must be positive")

    def withdraw(self, amount):
        if 0 < amount <= self.__balance:
            self.__balance -= amount
            self._log_transaction("withdraw", amount)
            print(f"Withdrew {amount}. New balance is {self.__balance}")
        else:
            print("Invalid withdrawal amount")

    def _log_transaction(self, transaction_type, amount):
        print(f"Logged {transaction_type} of {amount}")

# Usage
account = BankAccount("Alice", 100)
account.deposit(50)     # Output: Logged deposit of 50
                        #         Deposited 50. New balance is 150
account.withdraw(30)    # Output: Logged withdraw of 30
                        #         Withdrew 30. New balance is 120

Conclusion

Encapsulation is a powerful concept in OOP that helps in protecting the internal state of an object and promotes modularity and maintainability. By using private attributes and methods, along with getter and setter methods, you can control how data is accessed and modified, ensuring that the object remains in a valid state.
