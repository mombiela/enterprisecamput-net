9-4: Form Handling

Introduction

Form handling is a crucial aspect of web development. Forms are used to collect user input and send it to the server for processing. In this section, we will learn how to handle forms in Flask, including how to create forms, validate user input, and process the data on the server.

Key Concepts

  1. HTML Forms: Basic structure and elements of an HTML form.
  2. Flask-WTF: A Flask extension for handling forms.
  3. Form Validation: Ensuring that the user input meets certain criteria.
  4. CSRF Protection: Protecting forms from Cross-Site Request Forgery attacks.

HTML Forms

HTML forms are used to collect user input. Here is a basic example of an HTML form:

<form method="POST" action="/submit">
    <label for="name">Name:</label>
    <input type="text" id="name" name="name">
    <input type="submit" value="Submit">
</form>

Setting Up Flask-WTF

Flask-WTF is an extension that simplifies form handling in Flask. To install Flask-WTF, use the following command:

pip install Flask-WTF

Creating a Form Class

Create a form class using Flask-WTF. This class will define the fields and validation rules for the form.

from flask import Flask, render_template, request, redirect, url_for
from flask_wtf import FlaskForm
from wtforms import StringField, SubmitField
from wtforms.validators import DataRequired

app = Flask(__name__)
app.config['SECRET_KEY'] = 'your_secret_key'

class NameForm(FlaskForm):
    name = StringField('Name', validators=[DataRequired()])
    submit = SubmitField('Submit')

Rendering the Form in a Template

Create an HTML template to render the form. Save this as form.html in your templates directory.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Form Handling</title>
</head>
<body>
    <form method="POST" action="{{ url_for('submit') }}">
        {{ form.hidden_tag() }}
        <p>
            {{ form.name.label }}<br>
            {{ form.name(size=32) }}<br>
            {% for error in form.name.errors %}
                <span style="color: red;">[{{ error }}]</span>
            {% endfor %}
        </p>
        <p>{{ form.submit() }}</p>
    </form>
</body>
</html>

Handling Form Submission

Create a route to handle the form submission. This route will validate the form and process the data.

@app.route('/submit', methods=['GET', 'POST'])
def submit():
    form = NameForm()
    if form.validate_on_submit():
        name = form.name.data
        # Process the data (e.g., save to database)
        return redirect(url_for('success'))
    return render_template('form.html', form=form)

@app.route('/success')
def success():
    return "Form submitted successfully!"

if __name__ == '__main__':
    app.run(debug=True)

Practical Exercise

Exercise 1: Create a Contact Form

  1. Create a new form class ContactForm with the following fields:

    • name: A required text field.
    • email: A required email field.
    • message: A required text area field.
    • submit: A submit button.

  2. Create an HTML template contact.html to render the form.

  3. Create a route /contact to handle the form submission and display a success message.

Solution:

from flask import Flask, render_template, request, redirect, url_for
from flask_wtf import FlaskForm
from wtforms import StringField, TextAreaField, SubmitField
from wtforms.validators import DataRequired, Email

app = Flask(__name__)
app.config['SECRET_KEY'] = 'your_secret_key'

class ContactForm(FlaskForm):
    name = StringField('Name', validators=[DataRequired()])
    email = StringField('Email', validators=[DataRequired(), Email()])
    message = TextAreaField('Message', validators=[DataRequired()])
    submit = SubmitField('Submit')

@app.route('/contact', methods=['GET', 'POST'])
def contact():
    form = ContactForm()
    if form.validate_on_submit():
        name = form.name.data
        email = form.email.data
        message = form.message.data
        # Process the data (e.g., save to database)
        return redirect(url_for('contact_success'))
    return render_template('contact.html', form=form)

@app.route('/contact_success')
def contact_success():
    return "Contact form submitted successfully!"

if __name__ == '__main__':
    app.run(debug=True)

Common Mistakes and Tips

  • Missing CSRF Token: Always include {{ form.hidden_tag() }} in your form to prevent CSRF attacks.
  • Validation Errors: Ensure that you handle and display validation errors to the user.
  • Form Method: Use method="POST" for forms that modify data on the server.

Conclusion

In this section, we learned how to handle forms in Flask using Flask-WTF. We covered creating forms, rendering them in templates, validating user input, and processing the data on the server. Form handling is a fundamental skill in web development, and mastering it will enable you to create interactive and user-friendly web applications.
