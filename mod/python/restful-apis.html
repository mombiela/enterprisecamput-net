Introduction to RESTful APIs

  • Definition: REST (Representational State Transfer) is an architectural style for designing networked applications. It relies on a stateless, client-server, cacheable communications protocol -- the HTTP.
  • Key Concepts:
    • Resources: Objects or representations of data that can be accessed via a URL.
    • HTTP Methods: Common methods include GET, POST, PUT, DELETE.
    • Statelessness: Each request from a client to server must contain all the information needed to understand and process the request.
    • CRUD Operations: Create, Read, Update, Delete operations that map to HTTP methods.

Setting Up Your Environment

  • Python Installation: Ensure Python is installed on your system.
  • Required Libraries: Flask for creating APIs, requests for consuming APIs. 
    pip install Flask requests

Creating a Simple RESTful API with Flask

Setting Up Flask

  • Basic Flask Application: 
    from flask import Flask, jsonify, request

app = Flask(__name__)

@app.route('/')
def home():
    return "Welcome to the RESTful API with Flask!"

if __name__ == '__main__':
    app.run(debug=True)

Defining Endpoints

  • GET Request:

    @app.route('/api/v1/resource', methods=['GET'])
def get_resource():
    data = {"message": "This is a GET request"}
    return jsonify(data)

  • POST Request:

    @app.route('/api/v1/resource', methods=['POST'])
def create_resource():
    new_data = request.get_json()
    response = {"message": "Resource created", "data": new_data}
    return jsonify(response), 201

  • PUT Request:

    @app.route('/api/v1/resource/<int:id>', methods=['PUT'])
def update_resource(id):
    updated_data = request.get_json()
    response = {"message": f"Resource {id} updated", "data": updated_data}
    return jsonify(response)

  • DELETE Request:

    @app.route('/api/v1/resource/<int:id>', methods=['DELETE'])
def delete_resource(id):
    response = {"message": f"Resource {id} deleted"}
    return jsonify(response), 204

Consuming RESTful APIs with `requests`

  • GET Request:

    import requests

response = requests.get('http://localhost:5000/api/v1/resource')
print(response.json())

  • POST Request:

    import requests

data = {"name": "New Resource"}
response = requests.post('http://localhost:5000/api/v1/resource', json=data)
print(response.json())

  • PUT Request:

    import requests

data = {"name": "Updated Resource"}
response = requests.put('http://localhost:5000/api/v1/resource/1', json=data)
print(response.json())

  • DELETE Request:

    import requests

response = requests.delete('http://localhost:5000/api/v1/resource/1')
print(response.status_code)

Advanced Topics

Authentication and Authorization

  • Basic Authentication: 
    from flask import request, jsonify

@app.route('/api/v1/secure-resource', methods=['GET'])
def secure_resource():
    auth = request.authorization
    if auth and auth.username == 'user' and auth.password == 'pass':
        return jsonify({"message": "Authenticated"})
    return jsonify({"message": "Authentication failed"}), 401

Using Flask-RESTful

  • Installation:

    pip install Flask-RESTful

  • Creating a RESTful API with Flask-RESTful:

    from flask import Flask
from flask_restful import Resource, Api

app = Flask(__name__)
api = Api(app)

class HelloWorld(Resource):
    def get(self):
        return {'hello': 'world'}

api.add_resource(HelloWorld, '/')

if __name__ == '__main__':
    app.run(debug=True)

Conclusion

  • Summary:
    • RESTful APIs are a powerful way to enable communication between different parts of a web application or between different applications.
    • Flask is a lightweight and easy-to-use framework for creating RESTful APIs in Python.
    • Understanding how to create and consume RESTful APIs is essential for modern web development.
  • Next Steps:
    • Explore more advanced topics like OAuth, JWT for secure API authentication.
    • Practice by building a full-fledged application that uses RESTful APIs for its backend.
