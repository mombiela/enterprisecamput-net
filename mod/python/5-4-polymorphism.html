5-4: Polymorphism

Introduction to Polymorphism

Polymorphism is a core concept in object-oriented programming (OOP) that allows objects of different classes to be treated as objects of a common superclass. It is a way to perform a single action in different forms. Polymorphism is achieved through method overriding and method overloading.

Key Concepts

  1. Method Overriding:

    • Allows a subclass to provide a specific implementation of a method that is already defined in its superclass.
    • The method in the subclass should have the same name, return type, and parameters as the method in the superclass.

  2. Method Overloading:

    • Allows a class to have more than one method with the same name, as long as their parameter lists are different.
    • Python does not support method overloading directly, but we can achieve similar behavior using default arguments.

Practical Examples

Method Overriding

Let's consider an example where we have a base class Animal and two derived classes Dog and Cat.

class Animal:
    def speak(self):
        print("Animal speaks")

class Dog(Animal):
    def speak(self):
        print("Dog barks")

class Cat(Animal):
    def speak(self):
        print("Cat meows")

# Creating objects
animal = Animal()
dog = Dog()
cat = Cat()

# Calling the speak method
animal.speak()  # Output: Animal speaks
dog.speak()     # Output: Dog barks
cat.speak()     # Output: Cat meows

In this example, the speak method is overridden in the Dog and Cat classes. When we call the speak method on an object of Dog or Cat, the overridden method in the respective class is executed.

Method Overloading (Using Default Arguments)

Python does not support method overloading directly, but we can use default arguments to achieve similar functionality.

class MathOperations:
    def add(self, a, b, c=0):
        return a + b + c

# Creating an object
math_ops = MathOperations()

# Calling the add method
print(math_ops.add(2, 3))       # Output: 5
print(math_ops.add(2, 3, 4))    # Output: 9

In this example, the add method can take either two or three arguments. The third argument c has a default value of 0, allowing the method to be called with two arguments.

Practical Exercises

Exercise 1: Method Overriding

Create a base class Shape with a method area that prints "Calculating area". Then, create two derived classes Rectangle and Circle that override the area method to calculate and print the area of the rectangle and circle, respectively.

class Shape:
    def area(self):
        print("Calculating area")

class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height

    def area(self):
        print(f"Rectangle area: {self.width * self.height}")

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius

    def area(self):
        print(f"Circle area: {3.14 * self.radius * self.radius}")

# Creating objects
rectangle = Rectangle(4, 5)
circle = Circle(3)

# Calling the area method
rectangle.area()  # Output: Rectangle area: 20
circle.area()     # Output: Circle area: 28.26

Exercise 2: Method Overloading (Using Default Arguments)

Create a class Calculator with a method multiply that can take either two or three arguments. If three arguments are provided, the method should return the product of all three numbers; otherwise, it should return the product of the two numbers.

class Calculator:
    def multiply(self, a, b, c=1):
        return a * b * c

# Creating an object
calc = Calculator()

# Calling the multiply method
print(calc.multiply(2, 3))       # Output: 6
print(calc.multiply(2, 3, 4))    # Output: 24

Common Mistakes and Tips

  • Method Overriding: Ensure that the method signature (name, return type, and parameters) in the subclass matches exactly with the method in the superclass.
  • Method Overloading: Remember that Python does not support method overloading directly. Use default arguments to achieve similar functionality.

Conclusion

Polymorphism allows for flexibility and the ability to extend code without modifying existing code. By understanding and utilizing method overriding and method overloading (using default arguments), you can create more dynamic and reusable code. This concept is crucial for designing robust and scalable object-oriented applications.
