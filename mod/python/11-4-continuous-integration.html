11-4 Continuous Integration

Introduction to Continuous Integration (CI)

Continuous Integration (CI) is a software development practice where developers frequently integrate their code changes into a shared repository. Each integration is verified by an automated build and automated tests. CI aims to detect and address bugs quicker, improve software quality, and reduce the time it takes to validate and release new software updates.

Key Concepts of Continuous Integration

  1. Version Control System (VCS):

    • A system that records changes to a file or set of files over time so that you can recall specific versions later.
    • Examples: Git, Subversion (SVN).

  2. Automated Builds:

    • The process of automating the creation of a software build and the associated processes including compiling, packaging, and running automated tests.

  3. Automated Testing:

    • Running tests automatically to ensure that the code changes do not break existing functionality.
    • Types of tests: Unit tests, Integration tests, End-to-End tests.

  4. Build Server/CI Server:

    • A server that automates the process of building and testing code.
    • Examples: Jenkins, Travis CI, CircleCI, GitHub Actions.

Setting Up Continuous Integration with GitHub Actions

GitHub Actions is a CI/CD service provided by GitHub. It allows you to automate your build, test, and deployment pipeline.

Step-by-Step Guide

  1. Create a GitHub Repository:

    • Create a new repository on GitHub or use an existing one.

  2. Add a Workflow File:

    • Create a .github/workflows directory in your repository.
    • Add a new YAML file (e.g., ci.yml) in the .github/workflows directory.

  3. Define the Workflow:

    • Use the following example to create a basic CI workflow for a Python project:
name: Python CI

on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Python
      uses: actions/setup-python@v2
      with:
        python-version: '3.8'

    - name: Install dependencies
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt

    - name: Run tests
      run: |
        pytest

Explanation of the Workflow

  • name: The name of the workflow.
  • on: Specifies the events that trigger the workflow. In this case, it triggers on push and pull_request.
  • jobs: Defines the jobs that will run as part of the workflow.
    • build: The name of the job.
      • runs-on: Specifies the operating system for the job. Here, it uses ubuntu-latest.
      • steps: The sequence of steps to run in the job.
        • Checkout code: Uses the actions/checkout action to check out the repository.
        • Set up Python: Uses the actions/setup-python action to set up Python 3.8.
        • Install dependencies: Installs the dependencies listed in requirements.txt.
        • Run tests: Runs the tests using pytest.

Practical Exercise

Exercise: Set Up CI for a Python Project

  1. Create a GitHub Repository:

    • Create a new repository on GitHub.

  2. Add Python Code and Tests:

    • Add a simple Python script and a test file using pytest.
# script.py
def add(a, b):
    return a + b

# test_script.py
from script import add

def test_add():
    assert add(1, 2) == 3
    assert add(-1, 1) == 0
    assert add(0, 0) == 0
  1. Add a requirements.txt File:
    • List the dependencies in requirements.txt.
pytest

  1. Add the Workflow File:

    • Create a .github/workflows/ci.yml file with the content provided in the example above.

  2. Push Changes to GitHub:

    • Push your changes to GitHub and observe the CI pipeline running.

Common Mistakes and Tips

  • Incorrect YAML Syntax: YAML is sensitive to indentation. Ensure that the indentation is correct.
  • Missing Dependencies: Ensure all required dependencies are listed in requirements.txt.
  • Failing Tests: If tests fail, check the test output to debug and fix the issues.

Conclusion

In this section, you learned about Continuous Integration (CI) and how to set up a CI pipeline using GitHub Actions. CI helps in maintaining code quality and ensures that the codebase is always in a deployable state. By integrating CI into your development workflow, you can catch bugs early and improve the overall efficiency of your software development process.
