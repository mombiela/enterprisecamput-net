9-2: Creating a Simple Web Application

In this section, we will create a simple web application using Flask. Flask is a lightweight WSGI web application framework in Python. It is designed with simplicity and flexibility in mind, making it a popular choice for beginners and experienced developers alike.

Objectives

  • Understand the basic structure of a Flask application.
  • Create a simple web application that responds to HTTP requests.
  • Learn how to run the Flask application on a local server.

Key Concepts

  1. Flask Application Structure: Understand the basic components of a Flask application.
  2. Routing: Learn how to define routes to handle different URLs.
  3. Running the Application: Learn how to start the Flask development server.

Flask Application Structure

A basic Flask application typically consists of a single Python file. Here’s a breakdown of the essential components:

  • Importing Flask: Import the Flask class from the flask module.
  • Creating an Instance: Create an instance of the Flask class.
  • Defining Routes: Use the @app.route decorator to define routes.
  • Running the Application: Use the app.run() method to start the server.

Step-by-Step Guide

Step 1: Setting Up Your Environment

Before we start, ensure you have Flask installed. You can install Flask using pip:

pip install Flask

Step 2: Creating the Flask Application

Create a new Python file named app.py and add the following code:

from flask import Flask

# Create an instance of the Flask class
app = Flask(__name__)

# Define a route for the root URL
@app.route('/')
def home():
    return "Hello, World!"

# Run the application
if __name__ == '__main__':
    app.run(debug=True)

Explanation:

  • Importing Flask: We import the Flask class from the flask module.
  • Creating an Instance: We create an instance of the Flask class and name it app.
  • Defining a Route: We use the @app.route('/') decorator to define a route for the root URL (/). The home function returns a simple "Hello, World!" message.
  • Running the Application: We use app.run(debug=True) to start the Flask development server. The debug=True parameter enables debug mode, which provides helpful error messages and auto-reloads the server when code changes.

Step 3: Running the Application

To run the application, open your terminal, navigate to the directory containing app.py, and execute the following command:

python app.py

You should see output similar to this:

 * Running on http://127.0.0.1:5000/ (Press CTRL+C to quit)
 * Restarting with stat
 * Debugger is active!
 * Debugger PIN: 123-456-789

Open your web browser and navigate to http://127.0.0.1:5000/. You should see the message "Hello, World!".

Practical Exercise

Exercise 1: Create a New Route

  1. Add a new route to your Flask application that responds to the URL /about with the message "About Page".
  2. Test the new route by navigating to http://127.0.0.1:5000/about in your web browser.

Solution:

from flask import Flask

app = Flask(__name__)

@app.route('/')
def home():
    return "Hello, World!"

@app.route('/about')
def about():
    return "About Page"

if __name__ == '__main__':
    app.run(debug=True)

Common Mistakes:

  • Indentation Errors: Ensure your functions are properly indented.
  • Route Definitions: Make sure your route decorators (@app.route) are correctly defined and match the intended URL patterns.

Summary

In this section, you learned how to create a simple Flask web application. You now understand the basic structure of a Flask application, how to define routes, and how to run the application on a local server. This foundational knowledge will be crucial as we move on to more advanced topics in Flask web development.
