12-2: Planning and Design

In this section, we will cover the essential steps for planning and designing your capstone project. Proper planning and design are crucial for the successful implementation of any software project. This module will guide you through the process of defining project requirements, creating a project plan, and designing the architecture of your application.

Objectives

  • Understand the importance of project planning and design.
  • Learn how to define project requirements.
  • Create a project plan with milestones and deadlines.
  • Design the architecture of your application.

Key Concepts

  1. Project Requirements

    • Functional Requirements: What the system should do.
    • Non-Functional Requirements: Performance, security, usability, etc.
    • User Stories: Descriptions of features from the end-user perspective.

  2. Project Planning

    • Milestones: Key points in the project timeline.
    • Deadlines: Specific dates by which tasks should be completed.
    • Task Breakdown: Dividing the project into manageable tasks.

  3. Designing the Architecture

    • System Architecture: High-level structure of the system.
    • Database Design: Schema and relationships.
    • User Interface Design: Layout and user experience.

Step-by-Step Guide

1. Define Project Requirements

Functional Requirements:

  • List all the features your application should have.
  • Example: "The application should allow users to register and log in."

Non-Functional Requirements:

  • Define performance criteria, security measures, and usability standards.
  • Example: "The application should load within 2 seconds."

User Stories:

  • Write user stories to capture the requirements from the user's perspective.
  • Example: "As a user, I want to be able to reset my password so that I can regain access to my account if I forget it."

2. Create a Project Plan

Milestones:

  • Identify key milestones in your project.
  • Example: "Complete user authentication module by Week 2."

Deadlines:

  • Set realistic deadlines for each milestone.
  • Example: "User registration and login feature by the end of Week 1."

Task Breakdown:

  • Break down the project into smaller, manageable tasks.
  • Example:
    • Task 1: Set up the project repository.
    • Task 2: Implement user registration.
    • Task 3: Implement user login.

3. Design the Architecture

System Architecture:

  • Create a high-level diagram of your system architecture.
  • Example: Use a flowchart to show how different components interact.

Database Design:

  • Design the database schema, including tables and relationships.
  • Example: 
    CREATE TABLE users (
    id INT PRIMARY KEY,
    username VARCHAR(50) UNIQUE,
    password_hash VARCHAR(255),
    email VARCHAR(100) UNIQUE
);

User Interface Design:

  • Sketch the layout of your application’s user interface.
  • Example: Use wireframes to visualize the user interface.

Practical Example

Let's consider a simple project: a To-Do List application.

Functional Requirements:

  • Users can add, edit, and delete tasks.
  • Users can mark tasks as completed.

Non-Functional Requirements:

  • The application should be responsive and work on mobile devices.
  • The application should handle up to 1000 tasks per user without performance degradation.

User Stories:

  • "As a user, I want to add a new task so that I can keep track of my to-dos."
  • "As a user, I want to mark a task as completed so that I can see my progress."

Project Plan:

  • Milestone 1: Set up project repository (Deadline: Day 1)
  • Milestone 2: Implement task addition feature (Deadline: Day 3)
  • Milestone 3: Implement task editing feature (Deadline: Day 5)
  • Milestone 4: Implement task deletion feature (Deadline: Day 7)
  • Milestone 5: Implement task completion feature (Deadline: Day 9)

System Architecture:

  • Frontend: HTML/CSS/JavaScript
  • Backend: Flask (Python)
  • Database: SQLite

Database Design:

CREATE TABLE tasks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    description TEXT,
    completed BOOLEAN DEFAULT FALSE
);

User Interface Design:

  • Wireframe showing a list of tasks with options to add, edit, delete, and mark as completed.

Exercises

  1. Define Requirements for Your Project

    • Write down at least five functional requirements and three non-functional requirements for your capstone project.

  2. Create a Project Plan

    • Identify at least three milestones and set deadlines for each.
    • Break down the project into at least five tasks.

  3. Design the Architecture

    • Create a high-level system architecture diagram.
    • Design the database schema for your project.
    • Sketch the user interface layout.

Solutions

Exercise 1: Define Requirements

  • Functional Requirements:
    1. Users can register and log in.
    2. Users can create, edit, and delete posts.
    3. Users can comment on posts.
    4. Users can like and unlike posts.
    5. Users can follow and unfollow other users.
  • Non-Functional Requirements:
    1. The application should be secure and protect user data.
    2. The application should be responsive and work on all devices.
    3. The application should handle up to 10,000 users without performance issues.

Exercise 2: Create a Project Plan

  • Milestones:
    1. Complete user authentication module (Deadline: Week 1)
    2. Complete post creation and management module (Deadline: Week 2)
    3. Complete commenting and liking features (Deadline: Week 3)
  • Task Breakdown:
    1. Set up project repository.
    2. Implement user registration.
    3. Implement user login.
    4. Implement post creation.
    5. Implement post editing.

Exercise 3: Design the Architecture

  • System Architecture Diagram:
    • Frontend: React.js
    • Backend: Django (Python)
    • Database: PostgreSQL
  • Database Schema: 
    CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE,
    password_hash VARCHAR(255),
    email VARCHAR(100) UNIQUE
);

CREATE TABLE posts (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    title VARCHAR(100),
    content TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE comments (
    id SERIAL PRIMARY KEY,
    post_id INTEGER REFERENCES posts(id),
    user_id INTEGER REFERENCES users(id),
    content TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
  • User Interface Layout:
    • Wireframe showing a feed of posts with options to add, edit, delete, comment, and like.

Conclusion

In this section, you learned how to plan and design your capstone project. By defining clear requirements, creating a detailed project plan, and designing a robust architecture, you set a solid foundation for the successful implementation of your project. In the next section, we will move on to the implementation phase, where you will start coding your application based on the plan and design you have created.
