JUnit 5 is a powerful testing framework for Java applications. It introduces several annotations that help in writing and organizing tests. This topic will cover the essential JUnit 5 annotations, their purposes, and how to use them effectively.

Key Annotations in JUnit 5

JUnit 5 provides a variety of annotations to control the lifecycle of tests and to specify test behaviors. Here are the most commonly used annotations:

  • @Test
  • @BeforeEach
  • @AfterEach
  • @BeforeAll
  • @AfterAll
  • @DisplayName
  • @Disabled
  • @RepeatedTest
  • @ParameterizedTest
  • @ExtendWith

@Test

The @Test annotation is used to mark a method as a test method.

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

public class CalculatorTest {

    @Test
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

@BeforeEach

The @BeforeEach annotation is used to specify a method that should be executed before each test method.

import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    private Calculator calculator;

    @BeforeEach
    void setUp() {
        calculator = new Calculator();
    }

    @Test
    void addition() {
        assertEquals(2, calculator.add(1, 1));
    }
}

@AfterEach

The @AfterEach annotation is used to specify a method that should be executed after each test method.

import org.junit.jupiter.api.AfterEach;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    private Calculator calculator;

    @BeforeEach
    void setUp() {
        calculator = new Calculator();
    }

    @AfterEach
    void tearDown() {
        calculator = null;
    }

    @Test
    void addition() {
        assertEquals(2, calculator.add(1, 1));
    }
}

@BeforeAll

The @BeforeAll annotation is used to specify a method that should be executed once before all test methods in the class. This method must be static.

import org.junit.jupiter.api.BeforeAll;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    @BeforeAll
    static void initAll() {
        // Initialization code
    }

    @Test
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

@AfterAll

The @AfterAll annotation is used to specify a method that should be executed once after all test methods in the class. This method must be static.

import org.junit.jupiter.api.AfterAll;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    @BeforeAll
    static void initAll() {
        // Initialization code
    }

    @AfterAll
    static void tearDownAll() {
        // Cleanup code
    }

    @Test
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

@DisplayName

The @DisplayName annotation is used to provide a custom name for the test method.

import org.junit.jupiter.api.DisplayName;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    @Test
    @DisplayName("Addition of two numbers")
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

@Disabled

The @Disabled annotation is used to disable a test method or a test class.

import org.junit.jupiter.api.Disabled;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    @Test
    @Disabled("Disabled until bug #42 is fixed")
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

@RepeatedTest

The @RepeatedTest annotation is used to repeat a test a specified number of times.

import org.junit.jupiter.api.RepeatedTest;

public class CalculatorTest {

    @RepeatedTest(5)
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

@ParameterizedTest

The @ParameterizedTest annotation is used to run the same test multiple times with different arguments.

import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.ValueSource;

public class CalculatorTest {

    @ParameterizedTest
    @ValueSource(ints = {1, 2, 3, 4, 5})
    void isPositive(int number) {
        assertTrue(number > 0);
    }
}

@ExtendWith

The @ExtendWith annotation is used to register extensions for a test class or test method.

import org.junit.jupiter.api.extension.ExtendWith;
import org.junit.jupiter.api.Test;

@ExtendWith(MyExtension.class)
public class CalculatorTest {

    @Test
    void addition() {
        Calculator calculator = new Calculator();
        assertEquals(2, calculator.add(1, 1));
    }
}

Summary

JUnit 5 annotations provide a powerful way to control the lifecycle and behavior of your tests. Understanding and using these annotations effectively can help you write more organized, readable, and maintainable tests. Here is a quick recap of the key annotations covered:

  • @Test: Marks a method as a test method.
  • @BeforeEach: Executes a method before each test method.
  • @AfterEach: Executes a method after each test method.
  • @BeforeAll: Executes a method once before all test methods (must be static).
  • @AfterAll: Executes a method once after all test methods (must be static).
  • @DisplayName: Provides a custom name for the test method.
  • @Disabled: Disables a test method or class.
  • @RepeatedTest: Repeats a test a specified number of times.
  • @ParameterizedTest: Runs the same test multiple times with different arguments.
  • @ExtendWith: Registers extensions for a test class or method.

By mastering these annotations, you can take full advantage of JUnit 5's capabilities to create robust and efficient test suites.
