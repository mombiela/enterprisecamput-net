Continuous Integration with JUnit
Writing Maintainable Tests ►

Continuous Integration (CI) is a development practice where developers integrate code into a shared repository frequently, ideally several times a day. Each integration can then be verified by an automated build and automated tests, allowing teams to detect problems early.

In this section, we will explore how to integrate JUnit tests into a CI pipeline.

Key Concepts of Continuous Integration

  • Automated Builds: Automatically compile and build the project whenever changes are made.
  • Automated Testing: Run tests automatically to ensure new code changes do not break existing functionality.
  • Version Control Systems (VCS): Tools like Git to manage code changes.
  • CI Servers: Tools like Jenkins, Travis CI, or GitHub Actions to automate the build and test process.

Setting Up a CI Pipeline with JUnit

Step 1: Version Control System

Ensure your project is under version control. For this example, we will use Git.

  1. Initialize a Git repository:

    git init

  2. Add your project files and commit:

    git add .
git commit -m "Initial commit"

Step 2: Writing JUnit Tests

Ensure you have JUnit tests in your project. Here is a simple example:

import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.assertEquals;

public class CalculatorTest {

    @Test
    public void testAddition() {
        Calculator calculator = new Calculator();
        assertEquals(5, calculator.add(2, 3));
    }
}

Step 3: Choosing a CI Server

There are several CI servers available. We will use GitHub Actions for this example.

Step 4: Configuring GitHub Actions

  1. Create a .github/workflows directory in your project.

  2. Create a new workflow file, e.g., ci.yml:

    name: Java CI with Maven

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:

    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up JDK 11
      uses: actions/setup-java@v2
      with:
        java-version: '11'

    - name: Build with Maven
      run: mvn clean install

    - name: Run tests
      run: mvn test

Step 5: Pushing Changes to Trigger CI

  1. Add the workflow file to Git:

    git add .github/workflows/ci.yml
git commit -m "Add CI workflow"

  2. Push changes to the repository:

    git push origin main

GitHub Actions will now automatically run the CI pipeline whenever changes are pushed to the main branch or a pull request is made.

Conclusion

Integrating JUnit tests into a CI pipeline ensures that your code is continuously tested, providing immediate feedback on any issues introduced by new changes. This practice helps maintain code quality and reduces the risk of bugs in production.

By following the steps outlined above, you can set up a CI pipeline using GitHub Actions to automatically build and test your Java project with JUnit. This setup can be adapted to other CI servers like Jenkins or Travis CI with minor modifications.
