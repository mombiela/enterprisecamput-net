Introduction to Linked Lists

Linked lists are a fundamental data structure consisting of a sequence of elements, where each element points to the next. Unlike arrays, linked lists do not require a fixed size and allow for more efficient insertion and deletion of elements.

  • Node: The basic unit of a linked list that contains a value and a reference (or link) to the next node.
  • Head: The first node of the linked list.
  • Tail: The last node of the linked list, which points to null or None.

Types of Linked Lists

Singly Linked List

In a singly linked list, each node points to the next node in the sequence.

class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None

    def append(self, data):
        new_node = Node(data)
        if not self.head:
            self.head = new_node
            return
        last_node = self.head
        while last_node.next:
            last_node = last_node.next
        last_node.next = new_node

    def print_list(self):
        current_node = self.head
        while current_node:
            print(current_node.data, end=" -> ")
            current_node = current_node.next
        print("None")

# Usage example
ll = LinkedList()
ll.append(1)
ll.append(2)
ll.append(3)
ll.print_list()  # Output: 1 -> 2 -> 3 -> None

Doubly Linked List

In a doubly linked list, each node has two references: one to the next node and one to the previous node.

class Node:
    def __init__(self, data):
        self.data = data
        self.next = None
        self.prev = None

class DoublyLinkedList:
    def __init__(self):
        self.head = None

    def append(self, data):
        new_node = Node(data)
        if not self.head:
            self.head = new_node
            return
        last_node = self.head
        while last_node.next:
            last_node = last_node.next
        last_node.next = new_node
        new_node.prev = last_node

    def print_list(self):
        current_node = self.head
        while current_node:
            print(current_node.data, end=" <-> ")
            current_node = current_node.next
        print("None")

# Usage example
dll = DoublyLinkedList()
dll.append(1)
dll.append(2)
dll.append(3)
dll.print_list()  # Output: 1 <-> 2 <-> 3 <-> None

Circular Linked List

In a circular linked list, the last node points back to the first node, forming a cycle.

class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class CircularLinkedList:
    def __init__(self):
        self.head = None

    def append(self, data):
        new_node = Node(data)
        if not self.head:
            self.head = new_node
            new_node.next = self.head
            return
        last_node = self.head
        while last_node.next != self.head:
            last_node = last_node.next
        last_node.next = new_node
        new_node.next = self.head

    def print_list(self):
        current_node = self.head
        if not self.head:
            return
        while True:
            print(current_node.data, end=" -> ")
            current_node = current_node.next
            if current_node == self.head:
                break
        print("Head")

# Usage example
cll = CircularLinkedList()
cll.append(1)
cll.append(2)
cll.append(3)
cll.print_list()  # Output: 1 -> 2 -> 3 -> Head

Common Operations

  • Insertion: Add a new node to the list.
  • Deletion: Remove an existing node from the list.
  • Search: Find a node with a specific value.
  • Traversal: Visit each node in the list to perform some operation.

Comparison of Linked Lists

| Type of List | Advantages | Disadvantages | |---------------------|---------------------------------------------|----------------------------------------------| | Singly | Easy to implement, less memory used | Cannot be traversed backwards | | Doubly | Bidirectional traversal | More memory used due to extra references | | Circular | Continuous traversal without end | Additional complexity in handling links |

Conclusion

Linked lists are a versatile and efficient data structure for many applications. Understanding the different types of linked lists and their basic operations is crucial for any IT professional. Practicing with code examples and exercises will help consolidate this knowledge and apply it in real-world situations.
