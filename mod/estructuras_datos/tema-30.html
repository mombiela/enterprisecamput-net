Introduction

In the study of data structures, graphs are a fundamental tool for modeling complex relationships between different entities. Graphs can be classified into two main types: directed and undirected. In this section, we will explore the characteristics, differences, and applications of these two types of graphs.

Undirected Graphs

An undirected graph is a structure composed of a set of nodes (or vertices) and a set of edges (or links) that connect pairs of nodes. In an undirected graph, the edges do not have a specific direction, meaning that the relationship between the nodes is bidirectional.

Characteristics of Undirected Graphs

  • Symmetry: If there is an edge between node A and node B, then there is also an edge between node B and node A.
  • Representation: It can be represented by a symmetric adjacency matrix or an adjacency list.

Example of an Undirected Graph

Consider an undirected graph with nodes A, B, C, and D, and the following edges: (A-B), (A-C), (B-D), (C-D).

Representation in Adjacency List

A: B, C
B: A, D
C: A, D
D: B, C

Representation in Adjacency Matrix

| | A | B | C | D | |---|---|---|---|---| | A | 0 | 1 | 1 | 0 | | B | 1 | 0 | 0 | 1 | | C | 1 | 0 | 0 | 1 | | D | 0 | 1 | 1 | 0 |

Example of Code in Python

# Representation of an undirected graph using a dictionary (adjacency list)
undirected_graph = {
    'A': ['B', 'C'],
    'B': ['A', 'D'],
    'C': ['A', 'D'],
    'D': ['B', 'C']
}

# Print the graph
for node, adjacents in undirected_graph.items():
    print(f"{node}: {', '.join(adjacents)}")

Directed Graphs

A directed graph, also known as a digraph, is a structure in which the edges have a specific direction, meaning that the relationship between the nodes is unidirectional.

Characteristics of Directed Graphs

  • Direction: Each edge has a direction that goes from a source node to a destination node.
  • Representation: It can be represented by a non-symmetric adjacency matrix or an adjacency list.

Example of a Directed Graph

Consider a directed graph with nodes A, B, C, and D, and the following edges: (A->B), (A->C), (B->D), (C->D).

Representation in Adjacency List

A: B, C
B: D
C: D
D:

Representation in Adjacency Matrix

| | A | B | C | D | |---|---|---|---|---| | A | 0 | 1 | 1 | 0 | | B | 0 | 0 | 0 | 1 | | C | 0 | 0 | 0 | 1 | | D | 0 | 0 | 0 | 0 |

Example of Code in Python

# Representation of a directed graph using a dictionary (adjacency list)
directed_graph = {
    'A': ['B', 'C'],
    'B': ['D'],
    'C': ['D'],
    'D': []
}

# Print the graph
for node, adjacents in directed_graph.items():
    print(f"{node}: {', '.join(adjacents)}")

Conclusion

Directed and undirected graphs are powerful tools for modeling different types of relationships and structures in the field of computer science and other disciplines. Understanding the differences and characteristics of each type of graph is essential for selecting the appropriate structure according to the problem to be solved. Through examples and representations, we have explored how these graphs can be implemented and used in practice.
