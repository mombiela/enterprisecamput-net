Introduction to Stacks

Stacks are a fundamental data structure in computer science that follows the LIFO (Last In, First Out) principle. This means that the last element added to the stack will be the first one to be removed. The basic operations of a stack are:

  • push: Add an element to the top of the stack.
  • pop: Remove the element from the top of the stack.
  • peek: View the element at the top of the stack without removing it.
  • isEmpty: Check if the stack is empty.

Exercise 1: Basic Stack Implementation

Objective

Implement a stack using a list in Python and perform basic operations.

Instructions

  1. Create a Stack class.
  2. Implement the push, pop, peek, and isEmpty methods.
  3. Test the stack functionality with some examples.

Code Example

class Stack:
    def __init__(self):
        self.items = []

    def push(self, item):
        self.items.append(item)

    def pop(self):
        if not self.isEmpty():
            return self.items.pop()
        else:
            return "The stack is empty"

    def peek(self):
        if not self.isEmpty():
            return self.items[-1]
        else:
            return "The stack is empty"

    def isEmpty(self):
        return len(self.items) == 0

# Tests
stack = Stack()
stack.push(1)
stack.push(2)
stack.push(3)
print(stack.peek())  # Output: 3
print(stack.pop())   # Output: 3
print(stack.pop())   # Output: 2
print(stack.isEmpty())  # Output: False
print(stack.pop())   # Output: 1
print(stack.isEmpty())  # Output: True

Explanation

  • push: Adds an element to the end of the list.
  • pop: Removes and returns the last element of the list.
  • peek: Returns the last element of the list without removing it.
  • isEmpty: Checks if the list is empty.

Exercise 2: Balanced Parentheses Verification

Objective

Use a stack to verify if the parentheses in an expression are balanced.

Instructions

  1. Create a verify_parentheses function that takes a string as input.
  2. Use a stack to check if the parentheses are balanced.

Code Example

def verify_parentheses(expression):
    stack = Stack()
    for char in expression:
        if char == '(':
            stack.push(char)
        elif char == ')':
            if stack.isEmpty():
                return False
            stack.pop()
    return stack.isEmpty()

# Tests
print(verify_parentheses("(())"))  # Output: True
print(verify_parentheses("(()"))   # Output: False
print(verify_parentheses(")("))    # Output: False

Explanation

  • Iterate through each character of the expression.
  • If the character is '(', it is added to the stack.
  • If the character is ')', an element is removed from the stack.
  • At the end, the stack must be empty for the parentheses to be balanced.

Exercise 3: Infix to Postfix Notation Conversion

Objective

Convert an expression from infix notation (conventional) to postfix notation (reverse Polish notation) using a stack.

Instructions

  1. Create an infix_to_postfix function that takes an infix expression as input.
  2. Use a stack to handle operators and parentheses.

Code Example

def infix_to_postfix(expression):
    precedence = {'+': 1, '-': 1, '*': 2, '/': 2, '^': 3}
    stack = Stack()
    output = []

    for char in expression:
        if char.isalnum():
            output.append(char)
        elif char == '(':
            stack.push(char)
        elif char == ')':
            while not stack.isEmpty() and stack.peek() != '(':
                output.append(stack.pop())
            stack.pop()
        else:
            while (not stack.isEmpty() and stack.peek() != '(' and
                   precedence[char] <= precedence[stack.peek()]):
                output.append(stack.pop())
            stack.push(char)

    while not stack.isEmpty():
        output.append(stack.pop())

    return ''.join(output)

# Tests
print(infix_to_postfix("a+b*(c^d-e)^(f+g*h)-i"))  # Output: abcd^e-fgh*+^*+i-

Explanation

  • Use a stack to store operators and parentheses.
  • Operands are added directly to the output.
  • Operators are handled according to their precedence and added to the output in the correct order.

These exercises not only reinforce the understanding of stacks but also demonstrate their applicability in real-world problems.
