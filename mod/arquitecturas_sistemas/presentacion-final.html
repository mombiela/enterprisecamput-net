Objectives of the Final Presentation

The final presentation aims for students to demonstrate their understanding and application of the concepts learned throughout the Systems Architectures course. Students must:

  • Present a complete system architecture project.
  • Explain the design decisions made.
  • Show how architectural principles are implemented in their project.
  • Answer questions and receive feedback.

Presentation Structure

Each presentation should follow a clear and logical structure to ensure all important aspects are covered. The recommended structure is as follows:

  1. Introduction

    • Brief description of the project.
    • System objectives.
    • Target audience and user needs.

  2. System Architecture

    • General architecture diagram.
    • Main components and their interactions.
    • Justification for the choice of architecture (e.g., microservices, monolithic, etc.).

  3. Technologies Used

    • Programming languages.
    • Frameworks and libraries.
    • Development and deployment tools.

  4. Design Patterns

    • Applied design patterns (e.g., Singleton, Factory, Observer).
    • Code examples demonstrating the implementation of these patterns.

  5. Security and Scalability

    • Implemented security strategies.
    • Scalability plans and how workloads are managed.

  6. Testing and Validation

    • Testing methods used (e.g., unit tests, integration tests).
    • Test results and how the system was validated.

  7. Conclusion

    • Summary of key points.
    • Lessons learned.
    • Possible future improvements.

Project Example

Introduction

The presented project is an inventory management system for an online store. The system's objective is to allow administrators to manage product inventory, while customers can view product availability in real-time.

System Architecture

  • General Architecture Diagram:

  • Main Components:

    • Frontend: Web application developed in React.
    • Backend: RESTful API built with Node.js and Express.
    • Database: MongoDB for data storage.

  • Interactions:

    • The frontend communicates with the backend through API calls.
    • The backend interacts with the database for CRUD (Create, Read, Update, Delete) operations.

Technologies Used

  • Programming Languages:

    • JavaScript (Node.js, React)

  • Frameworks and Libraries:

    • Express for the backend.
    • Mongoose for interaction with MongoDB.
    • Redux for state management in the frontend.

  • Development and Deployment Tools:

    • Docker for containerization.
    • Jenkins for CI/CD.
    • AWS for cloud deployment.

Design Patterns

  • Singleton:

  • Factory:

    class ProductFactory {
  static createProduct(type, attributes) {
    switch (type) {

Security and Scalability

  • Security Strategies:

    • Authentication and authorization with JWT.
    • Encryption of sensitive data.
    • Input validation to prevent SQL injection and XSS.

  • Scalability Plans:

    • Use of load balancers.
    • Implementation of microservices for critical components.
    • Horizontal scaling of the database.

Testing and Validation

  • Testing Methods:

    • Unit tests with Jest.
    • Integration tests with Supertest.
    • Load tests with JMeter.

  • Test Results:

    • 95% code coverage.
    • All integration tests passed.
    • The system supports up to 1000 requests per second without performance degradation.

Conclusion

In the final presentation, students must demonstrate their ability to design and build a robust and scalable system architecture. They should be able to justify their design decisions and show how they have applied the architectural principles and patterns learned during the course. The feedback received during the presentation will be crucial for their professional development and will help them identify areas for improvement for future projects.
