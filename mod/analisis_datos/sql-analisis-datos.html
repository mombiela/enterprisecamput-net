Introduction to SQL

SQL (Structured Query Language) is a programming language used to manage and manipulate relational databases. It is an essential tool for data analysts as it allows for efficient data extraction, filtering, and transformation.

Basic SQL Concepts

  • Relational Databases: A set of tables related to each other.
  • Tables: Structures that store data in rows and columns.
  • Rows: Each row represents a unique record.
  • Columns: Each column represents an attribute of the record.

Basic SQL Operations

SELECT

The SELECT statement is used to query data from one or more tables.

SELECT column1, column2
FROM table_name;

WHERE

The WHERE clause is used to filter records that meet a specific condition.

SELECT column1, column2
FROM table_name
WHERE condition;

Example

SELECT name, age
FROM employees
WHERE age > 30;

This example selects the names and ages of employees who are over 30 years old.

Aggregation Functions

Aggregation functions allow calculations over a set of values and return a single value.

  • COUNT: Counts the number of rows.
  • SUM: Sums the values of a column.
  • AVG: Calculates the average of the values of a column.
  • MAX: Finds the maximum value.
  • MIN: Finds the minimum value.

Example

SELECT COUNT(*), AVG(salary), MAX(salary), MIN(salary)
FROM employees;

This example counts the number of employees and calculates the average, maximum, and minimum salary.

Joins

JOINs are used to combine rows from two or more tables based on a related column between them.

  • INNER JOIN: Returns rows when there is a match in both tables.
  • LEFT JOIN: Returns all rows from the left table and the matching rows from the right table.
  • RIGHT JOIN: Returns all rows from the right table and the matching rows from the left table.
  • FULL JOIN: Returns rows when there is a match in one of the tables.

Example

SELECT employees.name, departments.name
FROM employees
INNER JOIN departments ON employees.department_id = departments.id;

This example selects the names of employees and the names of the departments they work in, showing only the matches.

Subqueries

Subqueries are queries nested within another query.

Example

SELECT name, salary
FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);

This example selects the names and salaries of employees who earn more than the average salary.

Data Grouping

The GROUP BY clause is used to group rows that have the same values in specified columns.

Example

SELECT department_id, COUNT(*)
FROM employees
GROUP BY department_id;

This example counts the number of employees in each department.

Data Sorting

The ORDER BY clause is used to sort the results of a query.

Example

SELECT name, salary
FROM employees
ORDER BY salary DESC;

This example selects the names and salaries of employees, sorting them from highest to lowest salary.

Conclusion

SQL is a powerful tool for data analysis, allowing analysts to efficiently extract, filter, and transform data. Understanding basic operations, aggregation functions, joins, subqueries, data grouping, and sorting is fundamental for any data analysis professional. Practicing with examples and exercises is crucial to mastering these skills.
