Introduction

In this section, we will explore the fundamental concepts of variables and data types in PL/SQL. Understanding these concepts is crucial for writing efficient and effective PL/SQL code.

Variables in PL/SQL

Variables in PL/SQL are used to store data that can be manipulated during the execution of a block of code. They are essential for performing calculations, storing intermediate results, and managing data flow.

Declaring Variables

To declare a variable in PL/SQL, you need to specify its name and data type. Optionally, you can also initialize it with a value.

DECLARE
    variable_name data_type [:= initial_value];
BEGIN
    -- PL/SQL block
END;
  • variable_name: The name of the variable.
  • data_type: The type of data the variable will hold.
  • initial_value: An optional initial value for the variable.

Example

DECLARE
    employee_id NUMBER := 1001;
    employee_name VARCHAR2(50) := 'John Doe';
    hire_date DATE := SYSDATE;
BEGIN
    DBMS_OUTPUT.PUT_LINE('Employee ID: ' || employee_id);
    DBMS_OUTPUT.PUT_LINE('Employee Name: ' || employee_name);
    DBMS_OUTPUT.PUT_LINE('Hire Date: ' || hire_date);
END;

In this example:

  • employee_id is a numeric variable initialized to 1001.
  • employee_name is a string variable initialized to 'John Doe'.
  • hire_date is a date variable initialized to the current date.

Data Types in PL/SQL

PL/SQL supports various data types to handle different kinds of data. These data types can be broadly categorized into scalar, composite, reference, and LOB (Large Object) data types.

Scalar Data Types

Scalar data types hold a single value. They include:

  • NUMBER: For numeric values.
  • VARCHAR2: For variable-length character strings.
  • CHAR: For fixed-length character strings.
  • DATE: For date and time values.
  • BOOLEAN: For logical values (TRUE, FALSE, NULL).

Example

DECLARE
    age NUMBER := 30;
    first_name VARCHAR2(20) := 'Alice';
    is_active BOOLEAN := TRUE;
    birth_date DATE := TO_DATE('1990-01-01', 'YYYY-MM-DD');
BEGIN
    DBMS_OUTPUT.PUT_LINE('Age: ' || age);
    DBMS_OUTPUT.PUT_LINE('First Name: ' || first_name);
    DBMS_OUTPUT.PUT_LINE('Is Active: ' || is_active);
    DBMS_OUTPUT.PUT_LINE('Birth Date: ' || birth_date);
END;

Composite Data Types

Composite data types can hold multiple values. They include:

  • RECORD: A group of related data items.
  • TABLE: A collection of data items of the same type.

Example

DECLARE
    TYPE EmployeeRec IS RECORD (
        id NUMBER,
        name VARCHAR2(50),
        salary NUMBER
    );
    employee EmployeeRec;
BEGIN
    employee.id := 1001;
    employee.name := 'John Doe';
    employee.salary := 50000;
    DBMS_OUTPUT.PUT_LINE('Employee ID: ' || employee.id);
    DBMS_OUTPUT.PUT_LINE('Employee Name: ' || employee.name);
    DBMS_OUTPUT.PUT_LINE('Employee Salary: ' || employee.salary);
END;

Reference Data Types

Reference data types hold pointers to other data items. They include:

  • REF CURSOR: A pointer to a cursor that can be used to fetch rows from a query result set.

Example

DECLARE
    TYPE EmpCurTyp IS REF CURSOR;
    emp_cursor EmpCurTyp;
    emp_record employees%ROWTYPE;
BEGIN
    OPEN emp_cursor FOR SELECT * FROM employees;
    LOOP
        FETCH emp_cursor INTO emp_record;
        EXIT WHEN emp_cursor%NOTFOUND;
        DBMS_OUTPUT.PUT_LINE('Employee ID: ' || emp_record.employee_id);
    END LOOP;
    CLOSE emp_cursor;
END;

LOB Data Types

LOB data types are used to store large objects such as text, images, and binary data. They include:

  • BLOB: Binary Large Object.
  • CLOB: Character Large Object.
  • NCLOB: National Character Large Object.
  • BFILE: Binary File.

Example

DECLARE
    image BLOB;
    text CLOB;
BEGIN
    -- Assuming image and text are already populated
    DBMS_OUTPUT.PUT_LINE('Image size: ' || DBMS_LOB.GETLENGTH(image));
    DBMS_OUTPUT.PUT_LINE('Text size: ' || DBMS_LOB.GETLENGTH(text));
END;

Summary

In this section, we covered the basics of variables and data types in PL/SQL. We learned how to declare and initialize variables, and explored various data types including scalar, composite, reference, and LOB data types. Understanding these concepts is fundamental for writing robust PL/SQL programs.
