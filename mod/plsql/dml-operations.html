Introduction

In PL/SQL, Data Manipulation Language (DML) operations are essential for managing and manipulating data within database tables. The primary DML operations include INSERT, UPDATE, and DELETE. This section will cover these operations in detail, providing clear explanations, examples, and exercises to help you understand and apply these concepts effectively.

INSERT Operation

The INSERT statement is used to add new rows to a table.

Syntax

INSERT INTO table_name (column1, column2, column3, ...)
VALUES (value1, value2, value3, ...);

Example

INSERT INTO employees (employee_id, first_name, last_name, email, hire_date, job_id)
VALUES (101, 'John', 'Doe', 'john.doe@example.com', '2023-01-15', 'IT_PROG');

Explanation

  • employees: The name of the table where the data will be inserted.
  • (employee_id, first_name, last_name, email, hire_date, job_id): The columns in the table where the values will be inserted.
  • VALUES (101, 'John', 'Doe', 'john.doe@example.com', '2023-01-15', 'IT_PROG'): The values to be inserted into the specified columns.

Exercise

  1. Insert a new row into the departments table with the following values:
    • department_id: 10
    • department_name: 'Finance'
    • manager_id: 205
    • location_id: 1700

UPDATE Operation

The UPDATE statement is used to modify existing rows in a table.

Syntax

UPDATE table_name
SET column1 = value1, column2 = value2, ...
WHERE condition;

Example

UPDATE employees
SET email = 'john.doe@newexample.com', job_id = 'SA_REP'
WHERE employee_id = 101;

Explanation

  • employees: The name of the table where the data will be updated.
  • SET email = 'john.doe@newexample.com', job_id = 'SA_REP': The columns and their new values.
  • WHERE employee_id = 101: The condition to identify which row(s) to update.

Exercise

  1. Update the departments table to change the manager_id to 206 for the department with department_id 10.

DELETE Operation

The DELETE statement is used to remove existing rows from a table.

Syntax

DELETE FROM table_name
WHERE condition;

Example

DELETE FROM employees
WHERE employee_id = 101;

Explanation

  • employees: The name of the table from which the data will be deleted.
  • WHERE employee_id = 101: The condition to identify which row(s) to delete.

Exercise

  1. Delete the row from the departments table where the department_id is 10.

Summary

In this section, we covered the basic DML operations in PL/SQL: INSERT, UPDATE, and DELETE. These operations allow you to add, modify, and remove data within your database tables. Understanding and mastering these operations are fundamental to effectively managing data in any database application.

  • INSERT: Adds new rows to a table.
  • UPDATE: Modifies existing rows in a table.
  • DELETE: Removes existing rows from a table.

Practice these operations with the provided exercises to reinforce your understanding and build confidence in using DML statements in PL/SQL.
