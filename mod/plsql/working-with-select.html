Introduction

The SELECT statement is one of the most fundamental and frequently used commands in SQL and PL/SQL. It allows you to retrieve data from one or more tables in a database. This section will cover the basics of the SELECT statement and gradually move towards more advanced usage.

Basic SELECT Statement

The simplest form of a SELECT statement retrieves all columns from a single table.

SELECT * FROM employees;
  • SELECT: Specifies the columns to retrieve.
  • *: A wildcard character that means "all columns."
  • FROM: Specifies the table to retrieve data from.
  • employees: The name of the table.

Selecting Specific Columns

You can also specify which columns you want to retrieve.

SELECT first_name, last_name, salary FROM employees;
  • first_name, last_name, salary: The columns to retrieve.

Filtering Data with WHERE Clause

The WHERE clause allows you to filter the rows that are returned.

SELECT first_name, last_name, salary 
FROM employees 
WHERE department_id = 10;
  • WHERE: Specifies the condition to filter rows.
  • department_id = 10: The condition that must be met for a row to be included in the result.

Using Comparison Operators

You can use various comparison operators in the WHERE clause.

  • =: Equal to
  • <>: Not equal to
  • >: Greater than
  • <: Less than
  • >=: Greater than or equal to
  • <=: Less than or equal to
SELECT first_name, last_name, salary 
FROM employees 
WHERE salary > 50000;

Sorting Data with ORDER BY

The ORDER BY clause allows you to sort the result set.

SELECT first_name, last_name, salary 
FROM employees 
ORDER BY salary DESC;
  • ORDER BY: Specifies the column(s) to sort by.
  • salary: The column to sort by.
  • DESC: Sorts the result in descending order. Use ASC for ascending order (default).

Aggregating Data with GROUP BY

The GROUP BY clause is used to group rows that have the same values in specified columns into summary rows.

SELECT department_id, AVG(salary) AS average_salary 
FROM employees 
GROUP BY department_id;
  • GROUP BY: Specifies the column(s) to group by.
  • AVG(salary): An aggregate function that calculates the average salary.
  • AS average_salary: An alias for the calculated column.

Common Aggregate Functions

  • AVG(): Calculates the average value.
  • COUNT(): Counts the number of rows.
  • SUM(): Calculates the sum of values.
  • MAX(): Finds the maximum value.
  • MIN(): Finds the minimum value.

Filtering Groups with HAVING

The HAVING clause is used to filter groups based on a condition.

SELECT department_id, AVG(salary) AS average_salary 
FROM employees 
GROUP BY department_id 
HAVING AVG(salary) > 60000;
  • HAVING: Specifies the condition to filter groups.
  • AVG(salary) > 60000: The condition that must be met for a group to be included in the result.

Joining Tables

You can retrieve data from multiple tables using joins.

INNER JOIN

An INNER JOIN returns rows when there is a match in both tables.

SELECT e.first_name, e.last_name, d.department_name 
FROM employees e 
INNER JOIN departments d ON e.department_id = d.department_id;
  • INNER JOIN: Specifies the type of join.
  • ON: Specifies the condition for the join.

LEFT JOIN

A LEFT JOIN returns all rows from the left table and the matched rows from the right table.

SELECT e.first_name, e.last_name, d.department_name 
FROM employees e 
LEFT JOIN departments d ON e.department_id = d.department_id;

RIGHT JOIN

A RIGHT JOIN returns all rows from the right table and the matched rows from the left table.

SELECT e.first_name, e.last_name, d.department_name 
FROM employees e 
RIGHT JOIN departments d ON e.department_id = d.department_id;

Subqueries

A subquery is a query nested inside another query.

Simple Subquery

A simple subquery returns a single value.

SELECT first_name, last_name 
FROM employees 
WHERE department_id = (SELECT department_id FROM departments WHERE department_name = 'Sales');

Correlated Subquery

A correlated subquery references columns from the outer query.

SELECT first_name, last_name 
FROM employees e 
WHERE salary > (SELECT AVG(salary) FROM employees WHERE department_id = e.department_id);

Conclusion

The SELECT statement is a powerful tool for querying data in PL/SQL. By mastering its various clauses and options, you can retrieve and manipulate data efficiently. Practice using different types of SELECT statements to become proficient in data retrieval and analysis.
