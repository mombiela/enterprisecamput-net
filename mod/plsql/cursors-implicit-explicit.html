Cursors: Implicit and Explicit
Working with SELECT Statements ►

Introduction

In PL/SQL, cursors are used to handle query results. They allow you to fetch and manipulate data retrieved by SQL queries. There are two types of cursors in PL/SQL: implicit and explicit. Understanding these cursors is crucial for efficient database programming.

Implicit Cursors

Implicit cursors are automatically created by Oracle whenever an SQL statement is executed. They are simpler to use but offer less control compared to explicit cursors.

Key Concepts

  • Automatic Handling: Implicit cursors are managed automatically by PL/SQL.
  • Scope: They are only available for the duration of the SQL statement execution.
  • Usage: Commonly used for single-row queries.

Example

DECLARE
  v_employee_name employees.first_name%TYPE;
BEGIN
  SELECT first_name INTO v_employee_name FROM employees WHERE employee_id = 101;
  DBMS_OUTPUT.PUT_LINE('Employee Name: ' || v_employee_name);
END;
/

In this example:

  • An implicit cursor is used for the SELECT statement.
  • The result is stored in the variable v_employee_name.

Explicit Cursors

Explicit cursors are defined and controlled by the programmer. They provide more flexibility and are used for multi-row queries.

Key Concepts

  • Manual Handling: You need to declare, open, fetch, and close explicit cursors.
  • Control: Offers more control over the query execution and data retrieval.
  • Usage: Suitable for multi-row queries and complex data manipulation.

Steps to Use Explicit Cursors

  1. Declare the Cursor: Define the SQL query.
  2. Open the Cursor: Execute the SQL query.
  3. Fetch Data: Retrieve the result set row by row.
  4. Close the Cursor: Release the resources.

Example

DECLARE
  CURSOR emp_cursor IS
    SELECT first_name, last_name FROM employees WHERE department_id = 10;
  v_first_name employees.first_name%TYPE;
  v_last_name employees.last_name%TYPE;
BEGIN
  OPEN emp_cursor;
  LOOP
    FETCH emp_cursor INTO v_first_name, v_last_name;
    EXIT WHEN emp_cursor%NOTFOUND;
    DBMS_OUTPUT.PUT_LINE('Employee: ' || v_first_name || ' ' || v_last_name);
  END LOOP;
  CLOSE emp_cursor;
END;
/

In this example:

  • The cursor emp_cursor is declared to select employees from department 10.
  • The cursor is opened, and data is fetched in a loop until all rows are processed.
  • The cursor is then closed.

Comparison of Implicit and Explicit Cursors

Feature Implicit Cursor Explicit Cursor
Declaration Automatic Manual
Control Limited Full
Usage Single-row queries Multi-row queries
Resource Management Automatic Manual (open, fetch, close)
Flexibility Less flexible More flexible

Conclusion

Cursors are essential for handling query results in PL/SQL. Implicit cursors are easy to use for simple, single-row queries, while explicit cursors provide greater control and flexibility for complex, multi-row queries. Understanding when and how to use each type of cursor will help you write efficient and effective PL/SQL code.
