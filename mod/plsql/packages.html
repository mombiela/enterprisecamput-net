Introduction to Packages

Packages in PL/SQL are schema objects that group logically related PL/SQL types, variables, and subprograms (procedures and functions). They help in organizing and managing large PL/SQL programs by providing modularity, encapsulation, and reusability.

Key Concepts

  • Modularity: Packages allow you to group related procedures, functions, variables, and other PL/SQL constructs together.
  • Encapsulation: Packages can hide the implementation details of the subprograms and expose only the necessary parts.
  • Reusability: Once a package is created, its components can be reused across multiple applications.

Structure of a Package

A package in PL/SQL consists of two parts:

  1. Package Specification: Declares the public items that can be accessed by other PL/SQL blocks.
  2. Package Body: Defines the implementation of the subprograms declared in the package specification.

Package Specification

The package specification is the interface to the package. It declares the types, variables, constants, exceptions, cursors, and subprograms that are accessible from outside the package.

CREATE OR REPLACE PACKAGE package_name IS
  -- Public type declarations
  TYPE type_name IS RECORD (...);

  -- Public variable declarations
  variable_name datatype;

  -- Public constant declarations
  constant_name CONSTANT datatype := value;

  -- Public exception declarations
  exception_name EXCEPTION;

  -- Public cursor declarations
  CURSOR cursor_name IS select_statement;

  -- Public procedure and function declarations
  PROCEDURE procedure_name(parameter_list);
  FUNCTION function_name(parameter_list) RETURN return_datatype;
END package_name;

Package Body

The package body contains the implementation of the subprograms declared in the package specification. It can also contain private declarations that are not accessible outside the package.

CREATE OR REPLACE PACKAGE BODY package_name IS
  -- Private type declarations
  TYPE private_type_name IS RECORD (...);

  -- Private variable declarations
  private_variable_name datatype;

  -- Private constant declarations
  private_constant_name CONSTANT datatype := value;

  -- Private exception declarations
  private_exception_name EXCEPTION;

  -- Private cursor declarations
  CURSOR private_cursor_name IS select_statement;

  -- Implementation of public procedures and functions
  PROCEDURE procedure_name(parameter_list) IS
  BEGIN
    -- Procedure implementation
  END procedure_name;

  FUNCTION function_name(parameter_list) RETURN return_datatype IS
  BEGIN
    -- Function implementation
    RETURN value;
  END function_name;

  -- Private procedures and functions
  PROCEDURE private_procedure_name IS
  BEGIN
    -- Procedure implementation
  END private_procedure_name;

  FUNCTION private_function_name RETURN return_datatype IS
  BEGIN
    -- Function implementation
    RETURN value;
  END private_function_name;
END package_name;

Example: Creating and Using a Package

Let's create a simple package that manages employee data.

Package Specification

CREATE OR REPLACE PACKAGE emp_pkg IS
  -- Public type declaration
  TYPE emp_rec IS RECORD (
    emp_id NUMBER,
    emp_name VARCHAR2(100),
    emp_salary NUMBER
  );

  -- Public procedure declaration
  PROCEDURE add_employee(p_emp_id NUMBER, p_emp_name VARCHAR2, p_emp_salary NUMBER);

  -- Public function declaration
  FUNCTION get_employee(p_emp_id NUMBER) RETURN emp_rec;
END emp_pkg;

Package Body

CREATE OR REPLACE PACKAGE BODY emp_pkg IS
  -- Private variable to store employee data
  emp_data emp_rec;

  -- Implementation of add_employee procedure
  PROCEDURE add_employee(p_emp_id NUMBER, p_emp_name VARCHAR2, p_emp_salary NUMBER) IS
  BEGIN
    emp_data.emp_id := p_emp_id;
    emp_data.emp_name := p_emp_name;
    emp_data.emp_salary := p_emp_salary;
  END add_employee;

  -- Implementation of get_employee function
  FUNCTION get_employee(p_emp_id NUMBER) RETURN emp_rec IS
  BEGIN
    IF emp_data.emp_id = p_emp_id THEN
      RETURN emp_data;
    ELSE
      RETURN NULL;
    END IF;
  END get_employee;
END emp_pkg;

Using the Package

BEGIN
  -- Add an employee
  emp_pkg.add_employee(1, 'John Doe', 50000);

  -- Retrieve employee data
  DECLARE
    emp emp_pkg.emp_rec;
  BEGIN
    emp := emp_pkg.get_employee(1);
    DBMS_OUTPUT.PUT_LINE('Employee Name: ' || emp.emp_name);
  END;
END;

Conclusion

Packages in PL/SQL provide a powerful way to organize and manage your PL/SQL code. By grouping related procedures, functions, and other elements, packages enhance modularity, encapsulation, and reusability. Understanding how to create and use packages is essential for developing robust and maintainable PL/SQL applications.
