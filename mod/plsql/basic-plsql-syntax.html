Introduction to PL/SQL

PL/SQL (Procedural Language/Structured Query Language) is Oracle Corporation's procedural extension for SQL and the Oracle relational database. It combines the data manipulation power of SQL with the processing power of procedural languages.

Key Concepts

  • Block Structure: PL/SQL is based on a block structure. Each block consists of three sections: Declaration, Execution, and Exception Handling.
  • Variables: Used to store data that can be manipulated during program execution.
  • Control Structures: Include conditional statements and loops to control the flow of execution.
  • Cursors: Used to retrieve multiple rows from a query.
  • Exception Handling: Mechanisms to handle runtime errors.

PL/SQL Block Structure

A PL/SQL block has the following structure:

DECLARE
   -- Declaration section: variables, constants, cursors, etc.
BEGIN
   -- Execution section: SQL and PL/SQL statements
EXCEPTION
   -- Exception handling section: error handling code
END;
/

Example of a Simple PL/SQL Block

DECLARE
   v_message VARCHAR2(50);
BEGIN
   v_message := 'Hello, PL/SQL!';
   DBMS_OUTPUT.PUT_LINE(v_message);
EXCEPTION
   WHEN OTHERS THEN
      DBMS_OUTPUT.PUT_LINE('An error occurred.');
END;
/
  • DECLARE: This section is optional and used to declare variables, constants, and cursors.
  • BEGIN: This section contains the executable statements.
  • EXCEPTION: This section is optional and used to handle exceptions.
  • END: Marks the end of the PL/SQL block.
  • /: Used to execute the block in SQL*Plus or SQL Developer.

Variables

Variables in PL/SQL are used to store data temporarily. They must be declared in the declaration section.

Declaring Variables

DECLARE
   v_name VARCHAR2(30);
   v_age NUMBER;
BEGIN
   v_name := 'John Doe';
   v_age := 25;
   DBMS_OUTPUT.PUT_LINE('Name: ' || v_name);
   DBMS_OUTPUT.PUT_LINE('Age: ' || v_age);
END;
/
  • v_name: A variable of type VARCHAR2.
  • v_age: A variable of type NUMBER.

Control Structures

PL/SQL supports various control structures to manage the flow of execution.

Conditional Statements

IF-THEN-ELSE

DECLARE
   v_score NUMBER := 85;
BEGIN
   IF v_score >= 90 THEN
      DBMS_OUTPUT.PUT_LINE('Grade: A');
   ELSIF v_score >= 80 THEN
      DBMS_OUTPUT.PUT_LINE('Grade: B');
   ELSE
      DBMS_OUTPUT.PUT_LINE('Grade: C');
   END IF;
END;
/

Loops

FOR Loop

BEGIN
   FOR i IN 1..5 LOOP
      DBMS_OUTPUT.PUT_LINE('Iteration: ' || i);
   END LOOP;
END;
/

WHILE Loop

DECLARE
   v_counter NUMBER := 1;
BEGIN
   WHILE v_counter <= 5 LOOP
      DBMS_OUTPUT.PUT_LINE('Counter: ' || v_counter);
      v_counter := v_counter + 1;
   END LOOP;
END;
/

Cursors

Cursors are used to handle multiple rows returned by a query.

Implicit Cursor

BEGIN
   FOR rec IN (SELECT first_name, last_name FROM employees WHERE department_id = 10) LOOP
      DBMS_OUTPUT.PUT_LINE('Name: ' || rec.first_name || ' ' || rec.last_name);
   END LOOP;
END;
/

Explicit Cursor

DECLARE
   CURSOR emp_cursor IS SELECT first_name, last_name FROM employees WHERE department_id = 10;
   v_first_name employees.first_name%TYPE;
   v_last_name employees.last_name%TYPE;
BEGIN
   OPEN emp_cursor;
   LOOP
      FETCH emp_cursor INTO v_first_name, v_last_name;
      EXIT WHEN emp_cursor%NOTFOUND;
      DBMS_OUTPUT.PUT_LINE('Name: ' || v_first_name || ' ' || v_last_name);
   END LOOP;
   CLOSE emp_cursor;
END;
/

Exception Handling

Exceptions are used to handle errors that occur during the execution of a PL/SQL block.

Example of Exception Handling

DECLARE
   v_num NUMBER;
BEGIN
   v_num := 10 / 0; -- This will cause a division by zero error
EXCEPTION
   WHEN ZERO_DIVIDE THEN
      DBMS_OUTPUT.PUT_LINE('Error: Division by zero');
   WHEN OTHERS THEN
      DBMS_OUTPUT.PUT_LINE('An unexpected error occurred');
END;
/

Conclusion

Understanding the basic syntax of PL/SQL is crucial for writing efficient and error-free code. This topic covered the fundamental building blocks of PL/SQL, including block structure, variable declaration, control structures, cursors, and exception handling. Mastery of these basics will provide a strong foundation for more advanced PL/SQL programming.
