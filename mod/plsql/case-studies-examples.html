Introduction

In this section, we will explore various case studies and examples to understand how PL/SQL can be applied in real-world scenarios. These examples will range from beginner to advanced levels, helping you to grasp the practical applications of PL/SQL.

Beginner Level

Example 1: Simple PL/SQL Block

A simple PL/SQL block that prints a message.

BEGIN
    DBMS_OUTPUT.PUT_LINE('Hello, PL/SQL!');
END;
/
  • Explanation:
    • BEGIN and END define the start and end of the PL/SQL block.
    • DBMS_OUTPUT.PUT_LINE is a built-in procedure to print messages to the console.

Example 2: Variable Declaration and Assignment

A PL/SQL block that declares a variable and assigns a value to it.

DECLARE
    v_message VARCHAR2(50);
BEGIN
    v_message := 'Welcome to PL/SQL!';
    DBMS_OUTPUT.PUT_LINE(v_message);
END;
/
  • Explanation:
    • DECLARE section is used to declare variables.
    • v_message is a variable of type VARCHAR2 with a maximum length of 50 characters.
    • The variable is assigned a value and then printed.

Intermediate Level

Example 3: Using Cursors

A PL/SQL block that uses an explicit cursor to fetch and display employee names.

DECLARE
    CURSOR emp_cursor IS
        SELECT first_name, last_name FROM employees;
    v_first_name employees.first_name%TYPE;
    v_last_name employees.last_name%TYPE;
BEGIN
    OPEN emp_cursor;
    LOOP
        FETCH emp_cursor INTO v_first_name, v_last_name;
        EXIT WHEN emp_cursor%NOTFOUND;
        DBMS_OUTPUT.PUT_LINE(v_first_name || ' ' || v_last_name);
    END LOOP;
    CLOSE emp_cursor;
END;
/
  • Explanation:
    • CURSOR emp_cursor IS declares a cursor to select first and last names from the employees table.
    • OPEN emp_cursor opens the cursor.
    • FETCH retrieves the data into variables.
    • EXIT WHEN emp_cursor%NOTFOUND exits the loop when no more rows are found.
    • CLOSE emp_cursor closes the cursor.

Example 4: Exception Handling

A PL/SQL block that demonstrates exception handling.

DECLARE
    v_num NUMBER;
BEGIN
    v_num := 10 / 0;
EXCEPTION
    WHEN ZERO_DIVIDE THEN
        DBMS_OUTPUT.PUT_LINE('Error: Division by zero');
END;
/
  • Explanation:
    • The BEGIN block attempts to divide a number by zero.
    • The EXCEPTION block catches the ZERO_DIVIDE exception and prints an error message.

Advanced Level

Example 5: Creating and Using Procedures

A PL/SQL procedure that calculates the total salary of employees in a department.

CREATE OR REPLACE PROCEDURE calculate_total_salary (p_dept_id IN NUMBER, p_total_salary OUT NUMBER) IS
BEGIN
    SELECT SUM(salary) INTO p_total_salary
    FROM employees
    WHERE department_id = p_dept_id;
END;
/
  • Explanation:
    • CREATE OR REPLACE PROCEDURE defines a new procedure.
    • p_dept_id is an input parameter for the department ID.
    • p_total_salary is an output parameter for the total salary.
    • The SELECT statement calculates the total salary for the specified department.

Example 6: Using Triggers

A trigger that logs changes to the employees table.

CREATE OR REPLACE TRIGGER log_employee_changes
AFTER INSERT OR UPDATE OR DELETE ON employees
FOR EACH ROW
BEGIN
    IF INSERTING THEN
        INSERT INTO employee_log (log_date, action, emp_id)
        VALUES (SYSDATE, 'INSERT', :NEW.employee_id);
    ELSIF UPDATING THEN
        INSERT INTO employee_log (log_date, action, emp_id)
        VALUES (SYSDATE, 'UPDATE', :NEW.employee_id);
    ELSIF DELETING THEN
        INSERT INTO employee_log (log_date, action, emp_id)
        VALUES (SYSDATE, 'DELETE', :OLD.employee_id);
    END IF;
END;
/
  • Explanation:
    • CREATE OR REPLACE TRIGGER defines a new trigger.
    • The trigger fires AFTER INSERT OR UPDATE OR DELETE on the employees table.
    • FOR EACH ROW specifies that the trigger acts on each row affected.
    • The IF statements check the type of DML operation and log the changes accordingly.

Conclusion

Through these case studies and examples, we've covered a range of PL/SQL applications from simple blocks to advanced procedures and triggers. By practicing these examples, you will gain a deeper understanding of how to use PL/SQL effectively in various scenarios. Continue experimenting with these concepts to solidify your knowledge and enhance your programming skills.
