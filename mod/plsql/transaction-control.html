Transaction Control in PL/SQL
Stored Procedures ►

Transaction control is a crucial aspect of database management, ensuring data integrity and consistency. In PL/SQL, transaction control statements allow you to manage changes made by DML (Data Manipulation Language) statements such as INSERT, UPDATE, and DELETE.

Key Concepts

  • Transaction: A sequence of one or more SQL statements that are executed as a single unit of work.
  • Commit: Saves all changes made during the current transaction.
  • Rollback: Undoes all changes made during the current transaction.
  • Savepoint: Sets a point within a transaction to which you can later roll back.

Transaction Control Statements

COMMIT

The COMMIT statement is used to save all changes made during the current transaction. Once a transaction is committed, the changes are permanent and visible to other users.

BEGIN
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (101, 'John', 'Doe');
    
    COMMIT;
END;

ROLLBACK

The ROLLBACK statement undoes all changes made during the current transaction. This is useful if an error occurs and you need to revert the database to its previous state.

BEGIN
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (102, 'Jane', 'Smith');
    
    ROLLBACK;
END;

SAVEPOINT

The SAVEPOINT statement sets a point within a transaction to which you can later roll back. This allows for partial rollbacks within a transaction.

BEGIN
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (103, 'Alice', 'Johnson');
    
    SAVEPOINT savepoint1;
    
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (104, 'Bob', 'Brown');
    
    ROLLBACK TO savepoint1;
    
    COMMIT;
END;

In this example, the insertion of 'Alice Johnson' is saved, but the insertion of 'Bob Brown' is rolled back.

Practical Examples

Example 1: Basic Transaction Control

BEGIN
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (105, 'Charlie', 'Davis');
    
    UPDATE employees
    SET last_name = 'Davison'
    WHERE employee_id = 105;
    
    COMMIT;
END;

In this example, both the INSERT and UPDATE statements are part of a single transaction. The COMMIT statement ensures that both changes are saved.

Example 2: Using SAVEPOINT

BEGIN
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (106, 'Eve', 'White');
    
    SAVEPOINT savepoint1;
    
    INSERT INTO employees (employee_id, first_name, last_name)
    VALUES (107, 'Frank', 'Black');
    
    SAVEPOINT savepoint2;
    
    DELETE FROM employees
    WHERE employee_id = 106;
    
    ROLLBACK TO savepoint1;
    
    COMMIT;
END;

In this example:

  • The insertion of 'Eve White' and 'Frank Black' are initially part of the transaction.
  • The deletion of 'Eve White' is rolled back to savepoint1, so 'Eve White' remains in the database.
  • The transaction is then committed, saving the insertion of 'Frank Black'.

Conclusion

Transaction control in PL/SQL is essential for maintaining data integrity and consistency. By using COMMIT, ROLLBACK, and SAVEPOINT, you can manage transactions effectively, ensuring that your database operations are reliable and error-free. Understanding these concepts and how to apply them will significantly enhance your ability to write robust PL/SQL code.
