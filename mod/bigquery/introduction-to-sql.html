Overview

In this section, we will introduce SQL (Structured Query Language) and its application in Google BigQuery. This will serve as the foundation for understanding how to interact with and manipulate data within BigQuery.

What is SQL?

SQL is a standard language for accessing and manipulating databases. It is used to perform tasks such as querying data, updating records, and managing database structures.

Key Concepts

  • Database: A structured collection of data.
  • Table: A collection of related data entries consisting of rows and columns.
  • Query: A request for data or information from a database.

Basic SQL Syntax

Let's start with some basic SQL commands that are essential for working with BigQuery.

SELECT Statement

The SELECT statement is used to query data from a database.

SELECT column1, column2, ...
FROM table_name;

Example:

SELECT name, age
FROM employees;

WHERE Clause

The WHERE clause is used to filter records.

SELECT column1, column2, ...
FROM table_name
WHERE condition;

Example:

SELECT name, age
FROM employees
WHERE age > 30;

ORDER BY Clause

The ORDER BY clause is used to sort the result set.

SELECT column1, column2, ...
FROM table_name
ORDER BY column1 [ASC|DESC];

Example:

SELECT name, age
FROM employees
ORDER BY age DESC;

LIMIT Clause

The LIMIT clause is used to specify the number of records to return.

SELECT column1, column2, ...
FROM table_name
LIMIT number;

Example:

SELECT name, age
FROM employees
LIMIT 5;

SQL in BigQuery

BigQuery uses standard SQL (also known as BigQuery SQL) for querying data. Here are some specific features and differences when using SQL in BigQuery.

Key Features

  • Scalability: BigQuery can handle large datasets efficiently.
  • Performance: Optimized for fast query performance.
  • Integration: Easily integrates with other Google Cloud services.

Example Query in BigQuery

SELECT name, age
FROM `project.dataset.employees`
WHERE age > 30
ORDER BY age DESC
LIMIT 5;

Explanation:

  • Project: The Google Cloud project ID.
  • Dataset: The dataset within the project.
  • Table: The table within the dataset.

Common SQL Functions in BigQuery

BigQuery provides a variety of built-in functions to perform operations on data.

Aggregate Functions

  • COUNT(): Returns the number of rows.
  • SUM(): Returns the sum of a numeric column.
  • AVG(): Returns the average value of a numeric column.
  • MAX(): Returns the maximum value.
  • MIN(): Returns the minimum value.

Example:

SELECT COUNT(*), AVG(age), MAX(age), MIN(age)
FROM `project.dataset.employees`;

String Functions

  • UPPER(): Converts a string to uppercase.
  • LOWER(): Converts a string to lowercase.
  • CONCAT(): Concatenates two or more strings.

Example:

SELECT UPPER(name), LOWER(name), CONCAT(first_name, ' ', last_name) AS full_name
FROM `project.dataset.employees`;

Date Functions

  • CURRENT_DATE(): Returns the current date.
  • DATE_ADD(): Adds a specified time interval to a date.
  • FORMAT_DATE(): Formats a date according to a specified format.

Example:

SELECT CURRENT_DATE(), DATE_ADD(CURRENT_DATE(), INTERVAL 7 DAY) AS next_week
FROM `project.dataset.employees`;

Conclusion

In this section, we covered the basics of SQL and its application in BigQuery. We discussed fundamental SQL commands such as SELECT, WHERE, ORDER BY, and LIMIT, and explored how to use these commands in BigQuery. Additionally, we introduced some common SQL functions available in BigQuery. This foundational knowledge will be crucial as we move on to more advanced topics in BigQuery.
