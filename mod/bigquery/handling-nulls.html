Introduction to NULLs

  • Definition: In SQL, NULL represents a missing or undefined value.
  • Importance: Understanding how to handle NULL values is crucial for accurate data analysis and manipulation.

Identifying NULLs

  • IS NULL: Used to check if a value is NULL.
  • IS NOT NULL: Used to check if a value is not NULL.
SELECT *
FROM my_table
WHERE column_name IS NULL;
SELECT *
FROM my_table
WHERE column_name IS NOT NULL;

Handling NULLs in Queries

Using COALESCE

  • Purpose: Replaces NULL with a specified value.
  • Syntax: COALESCE(expression, replacement_value)
SELECT COALESCE(column_name, 'default_value') AS column_name
FROM my_table;

Using IFNULL

  • Purpose: Similar to COALESCE, but only takes two arguments.
  • Syntax: IFNULL(expression, replacement_value)
SELECT IFNULL(column_name, 'default_value') AS column_name
FROM my_table;

Using NULLIF

  • Purpose: Returns NULL if two expressions are equal.
  • Syntax: NULLIF(expression1, expression2)
SELECT NULLIF(column_name, 'value_to_compare') AS column_name
FROM my_table;

Aggregating with NULLs

  • COUNT: COUNT(column_name) ignores NULL values, while COUNT(*) includes them.
  • SUM, AVG, MIN, MAX: These functions ignore NULL values.
SELECT COUNT(column_name) AS non_null_count,
       COUNT(*) AS total_count
FROM my_table;

Advanced Handling of NULLs

Using CASE Statements

  • Purpose: Provides more complex conditional handling of NULL values.
  • Syntax: CASE WHEN condition THEN result ELSE result END
SELECT CASE 
           WHEN column_name IS NULL THEN 'No Value'
           ELSE column_name
       END AS column_name
FROM my_table;

NULLs in Joins

  • LEFT JOIN: Includes all records from the left table and matched records from the right table. If no match, NULL values are returned for columns from the right table.
  • INNER JOIN: Excludes records with NULL values in the join condition.
SELECT a.*, b.*
FROM table_a a
LEFT JOIN table_b b
ON a.id = b.id;

NULLs in Data Types

  • String: NULL is different from an empty string ''.
  • Numeric: NULL is different from zero 0.

| Data Type | NULL Example | Non-NULL Example | |-----------|--------------|------------------| | String | NULL | '' | | Numeric | NULL | 0 |

Conclusion

Handling NULL values effectively is essential for accurate data analysis in BigQuery. By using functions like COALESCE, IFNULL, and NULLIF, as well as understanding how NULL values interact with joins and aggregations, you can ensure your queries return the correct results. Always be mindful of how NULL values are treated in different contexts to avoid unexpected outcomes in your data analysis.
