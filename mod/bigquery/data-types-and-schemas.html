Introduction

In this section, we will explore the fundamental concepts of data types and schemas in BigQuery. Understanding these concepts is crucial for efficiently storing, querying, and analyzing data. We will cover the basic data types, how to define schemas, and advanced topics such as nested and repeated fields.

Basic Data Types

BigQuery supports a variety of data types to accommodate different kinds of data. Here are the primary data types you will encounter:

  • STRING: Used for text data.
  • INT64: Used for integer numbers.
  • FLOAT64: Used for floating-point numbers.
  • BOOL: Used for boolean values (TRUE or FALSE).
  • BYTES: Used for binary data.
  • DATE: Used for calendar dates (without time).
  • DATETIME: Used for date and time (without time zone).
  • TIMESTAMP: Used for date and time with time zone.
  • TIME: Used for time of day (without date).
  • GEOGRAPHY: Used for geographical data.

Example

CREATE TABLE my_dataset.my_table (
  name STRING,
  age INT64,
  height FLOAT64,
  is_student BOOL,
  profile_picture BYTES,
  birth_date DATE,
  last_login TIMESTAMP,
  daily_schedule TIME
);

In this example, we define a table with various columns, each using a different data type.

Defining Schemas

A schema in BigQuery is a blueprint that defines the structure of a table, including the names and data types of columns. Schemas can be defined manually or inferred automatically when loading data.

Manual Schema Definition

When creating a table, you can specify the schema explicitly:

CREATE TABLE my_dataset.user_info (
  user_id INT64,
  user_name STRING,
  email STRING,
  signup_date DATE
);

Auto Schema Inference

BigQuery can also infer the schema from the data when you load it from a file:

bq load --autodetect --source_format=CSV my_dataset.user_info gs://my_bucket/user_info.csv

Nested and Repeated Fields

BigQuery supports nested and repeated fields, which allow you to store complex data structures within a table. This is particularly useful for handling JSON-like data.

Nested Fields

Nested fields are fields that contain other fields. They are defined using the STRUCT data type.

CREATE TABLE my_dataset.orders (
  order_id INT64,
  customer STRUCT<
    customer_id INT64,
    customer_name STRING,
    contact_info STRUCT<
      email STRING,
      phone STRING
    >
  >,
  order_date DATE
);

Repeated Fields

Repeated fields are arrays of values. They are defined using the ARRAY data type.

CREATE TABLE my_dataset.orders (
  order_id INT64,
  items ARRAY<STRUCT<
    item_id INT64,
    item_name STRING,
    quantity INT64
  >>,
  order_date DATE
);

Combining Nested and Repeated Fields

You can combine nested and repeated fields to create even more complex data structures.

CREATE TABLE my_dataset.orders (
  order_id INT64,
  customer STRUCT<
    customer_id INT64,
    customer_name STRING
  >,
  items ARRAY<STRUCT<
    item_id INT64,
    item_name STRING,
    quantity INT64
  >>,
  order_date DATE
);

Conclusion

Understanding data types and schemas is fundamental for working effectively with BigQuery. By mastering these concepts, you can design efficient and scalable data structures that meet your analytical needs. Whether you are dealing with simple tables or complex nested and repeated fields, BigQuery provides the flexibility to handle a wide range of data scenarios.
