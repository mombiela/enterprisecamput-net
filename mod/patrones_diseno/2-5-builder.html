Topic 2-5: Builder

Introduction to the Builder Pattern

The Builder pattern is a creational design pattern that provides a way to construct complex objects step by step. Unlike other creational patterns, the Builder pattern does not require that products have a common interface. This pattern is particularly useful when the construction process involves multiple steps or when the product can have different representations.

Key Concepts

  • Builder: An interface that defines the steps to build the product.
  • Concrete Builder: A class that implements the Builder interface and provides specific implementations of the construction steps.
  • Product: The complex object that is being built.
  • Director: A class that controls the construction process using a Builder object.

Structure

The Builder pattern can be visualized with the following UML diagram:

Component Description
Builder Declares the interface for creating parts of a Product object.
ConcreteBuilder Constructs and assembles parts of the product by implementing the Builder.
Product Represents the complex object under construction.
Director Constructs an object using the Builder interface.

Example

Let's consider an example where we want to build different types of houses. Each house can have different features like walls, doors, and windows.

Step-by-Step Implementation

  1. Define the Product
public class House {
    private String walls;
    private String doors;
    private String windows;

    public void setWalls(String walls) {
        this.walls = walls;
    }

    public void setDoors(String doors) {
        this.doors = doors;
    }

    public void setWindows(String windows) {
        this.windows = windows;
    }

    @Override
    public String toString() {
        return "House [walls=" + walls + ", doors=" + doors + ", windows=" + windows + "]";
    }
}
  1. Create the Builder Interface
public interface HouseBuilder {
    void buildWalls();
    void buildDoors();
    void buildWindows();
    House getHouse();
}
  1. Implement Concrete Builders
public class WoodenHouseBuilder implements HouseBuilder {
    private House house;

    public WoodenHouseBuilder() {
        this.house = new House();
    }

    @Override
    public void buildWalls() {
        house.setWalls("Wooden Walls");
    }

    @Override
    public void buildDoors() {
        house.setDoors("Wooden Doors");
    }

    @Override
    public void buildWindows() {
        house.setWindows("Wooden Windows");
    }

    @Override
    public House getHouse() {
        return this.house;
    }
}

public class BrickHouseBuilder implements HouseBuilder {
    private House house;

    public BrickHouseBuilder() {
        this.house = new House();
    }

    @Override
    public void buildWalls() {
        house.setWalls("Brick Walls");
    }

    @Override
    public void buildDoors() {
        house.setDoors("Brick Doors");
    }

    @Override
    public void buildWindows() {
        house.setWindows("Brick Windows");
    }

    @Override
    public House getHouse() {
        return this.house;
    }
}
  1. Create the Director
public class HouseDirector {
    private HouseBuilder houseBuilder;

    public HouseDirector(HouseBuilder houseBuilder) {
        this.houseBuilder = houseBuilder;
    }

    public void constructHouse() {
        houseBuilder.buildWalls();
        houseBuilder.buildDoors();
        houseBuilder.buildWindows();
    }

    public House getHouse() {
        return this.houseBuilder.getHouse();
    }
}
  1. Client Code
public class BuilderPatternDemo {
    public static void main(String[] args) {
        HouseBuilder woodenHouseBuilder = new WoodenHouseBuilder();
        HouseDirector director = new HouseDirector(woodenHouseBuilder);
        director.constructHouse();
        House woodenHouse = director.getHouse();
        System.out.println("House built:\n" + woodenHouse);

        HouseBuilder brickHouseBuilder = new BrickHouseBuilder();
        director = new HouseDirector(brickHouseBuilder);
        director.constructHouse();
        House brickHouse = director.getHouse();
        System.out.println("House built:\n" + brickHouse);
    }
}

Advantages and Disadvantages

Advantages

  • Separation of Concerns: The construction process is isolated from the representation.
  • Flexibility: Different builders can produce different representations of the product.
  • Reusability: The same construction process can be reused to create different products.

Disadvantages

  • Complexity: The pattern can add unnecessary complexity if the product is simple.
  • Overhead: Requires creating multiple classes for each type of product.

Conclusion

The Builder pattern is a powerful tool for constructing complex objects step by step. It provides flexibility and reusability by separating the construction process from the final product. However, it should be used judiciously to avoid unnecessary complexity. Understanding and implementing the Builder pattern can significantly enhance your ability to manage and construct complex objects in a clean and maintainable way.
