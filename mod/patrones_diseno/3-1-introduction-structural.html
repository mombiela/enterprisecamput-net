3-1: Introduction to Structural Patterns

What are Structural Patterns?

Structural design patterns are concerned with how classes and objects are composed to form larger structures. They help ensure that if one part of a system changes, the entire system doesn't need to change as well. These patterns focus on simplifying the structure by identifying relationships between entities.

Key Concepts

  1. Composition over Inheritance:

    • Structural patterns often emphasize composition over inheritance. This means that instead of creating new functionality by extending existing classes, new functionality is created by composing objects together.

  2. Object and Class Patterns:

    • Structural patterns can be divided into class patterns and object patterns. Class patterns deal with relationships between classes and their subclasses, while object patterns deal with object composition.

  3. Flexibility and Scalability:

    • These patterns provide flexible and scalable solutions by promoting loose coupling between components, making it easier to extend and maintain the system.

Common Structural Patterns

  1. Adapter:

    • Converts the interface of a class into another interface that a client expects.
    • Example: A power adapter that allows a device from one country to be plugged into a different country's power outlet.

  2. Bridge:

    • Decouples an abstraction from its implementation so that the two can vary independently.
    • Example: Separating the interface of a remote control from the implementation of the devices it controls (TV, DVD player).

  3. Composite:

    • Composes objects into tree structures to represent part-whole hierarchies.
    • Example: A file system where directories can contain files and other directories.

  4. Decorator:

    • Adds additional responsibilities to an object dynamically.
    • Example: Adding scrollbars to a window in a graphical user interface.

  5. Facade:

    • Provides a simplified interface to a complex subsystem.
    • Example: A computer system's facade might provide a simple interface to start up the computer, hiding the complexity of the underlying operations.

  6. Flyweight:

    • Reduces the cost of creating and manipulating a large number of similar objects.
    • Example: Using a single object to represent multiple instances of a character in a text editor.

  7. Proxy:

    • Provides a surrogate or placeholder for another object to control access to it.
    • Example: A proxy server that controls access to a website.

Example: Adapter Pattern

Let's look at a simple example of the Adapter pattern in Python:

class EuropeanSocketInterface:
    def voltage(self): pass
    def live(self): pass
    def neutral(self): pass
    def earth(self): pass

class EuropeanSocket(EuropeanSocketInterface):
    def voltage(self):
        return 230

    def live(self):
        return 1

    def neutral(self):
        return -1

    def earth(self):
        return 0

class USASocketInterface:
    def voltage(self): pass
    def live(self): pass
    def neutral(self): pass

class Adapter(USASocketInterface):
    def __init__(self, european_socket):
        self.european_socket = european_socket

    def voltage(self):
        return 110

    def live(self):
        return self.european_socket.live()

    def neutral(self):
        return self.european_socket.neutral()

# Client code
european_socket = EuropeanSocket()
adapter = Adapter(european_socket)

print(f"Voltage: {adapter.voltage()}V")  # Output: Voltage: 110V
print(f"Live: {adapter.live()}")         # Output: Live: 1
print(f"Neutral: {adapter.neutral()}")   # Output: Neutral: -1

Explanation

  • EuropeanSocketInterface: An interface for European sockets.
  • EuropeanSocket: A class that implements the EuropeanSocketInterface.
  • USASocketInterface: An interface for USA sockets.
  • Adapter: A class that adapts the EuropeanSocket to the USASocketInterface.
  • Client code: Uses the Adapter to interact with a EuropeanSocket as if it were a USASocket.

Conclusion

Structural patterns play a crucial role in building flexible and maintainable software systems. By focusing on the composition of classes and objects, these patterns help manage relationships and dependencies, making it easier to extend and modify the system. Understanding and applying structural patterns can significantly improve the design and architecture of your software projects.
