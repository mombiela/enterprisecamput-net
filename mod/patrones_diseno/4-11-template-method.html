Topic: Template Method

What is the Template Method Pattern?

The Template Method pattern defines the skeleton of an algorithm in a method, deferring some steps to subclasses. It allows subclasses to redefine certain steps of an algorithm without changing the algorithm's structure.

Key Concepts

  • Algorithm Skeleton: The overall structure of the algorithm is defined in the base class.
  • Abstract Methods: Steps of the algorithm that can vary are defined as abstract methods in the base class.
  • Concrete Methods: Steps that are common across all implementations are defined as concrete methods in the base class.
  • Hook Methods: Optional methods that can be overridden by subclasses to provide additional behavior.

Structure

The Template Method pattern typically involves the following components:

  1. Abstract Class: Contains the template method and abstract methods.
  2. Concrete Class: Implements the abstract methods defined in the abstract class.

Example

Let's consider an example where we have a base class Game that defines the skeleton of playing a game. The steps to play a game include initializing the game, starting the game, and ending the game. Different games will have different implementations for these steps.

Abstract Class

from abc import ABC, abstractmethod

class Game(ABC):
    def play(self):
        self.initialize()
        self.start_play()
        self.end_play()

    @abstractmethod
    def initialize(self):
        pass

    @abstractmethod
    def start_play(self):
        pass

    @abstractmethod
    def end_play(self):
        pass

Concrete Class 1: Football

class Football(Game):
    def initialize(self):
        print("Football Game Initialized! Start playing.")

    def start_play(self):
        print("Football Game Started. Enjoy the game!")

    def end_play(self):
        print("Football Game Finished!")

Concrete Class 2: Cricket

class Cricket(Game):
    def initialize(self):
        print("Cricket Game Initialized! Start playing.")

    def start_play(self):
        print("Cricket Game Started. Enjoy the game!")

    def end_play(self):
        print("Cricket Game Finished!")

Using the Template Method

if __name__ == "__main__":
    game = Football()
    game.play()
    
    print("\n")
    
    game = Cricket()
    game.play()

Explanation

  • Abstract Class (Game): Defines the play method, which is the template method. It calls three abstract methods: initialize, start_play, and end_play.
  • Concrete Classes (Football and Cricket): Implement the abstract methods defined in the Game class. Each game has its own way of initializing, starting, and ending the game.
  • Execution: When the play method is called on an instance of Football or Cricket, it follows the same sequence of steps defined in the Game class but with the specific implementations provided by the concrete classes.

Advantages

  • Code Reusability: Common code is placed in the base class, reducing duplication.
  • Flexibility: Subclasses can override specific steps of the algorithm without changing its structure.
  • Maintainability: Changes to the algorithm structure are made in one place (the base class).

Disadvantages

  • Inheritance Overuse: Can lead to a deep inheritance hierarchy, making the code harder to understand and maintain.
  • Limited Flexibility: All subclasses must follow the same algorithm structure, which may not always be desirable.

Conclusion

The Template Method pattern is a powerful behavioral design pattern that allows you to define the skeleton of an algorithm in a base class and let subclasses implement specific steps. This promotes code reuse and flexibility while maintaining a consistent algorithm structure. However, it should be used judiciously to avoid overly complex inheritance hierarchies.
