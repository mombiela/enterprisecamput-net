Introduction

The Factory Method is a creational design pattern that provides an interface for creating objects in a superclass but allows subclasses to alter the type of objects that will be created. This pattern promotes loose coupling by eliminating the need to bind application-specific classes into the code.

Key Concepts

  1. Creator Class: Defines the factory method that returns an object of a Product class. The Creator may also define a default implementation of the factory method that returns a default ConcreteProduct object.
  2. Concrete Creator Class: Overrides the factory method to return an instance of a ConcreteProduct.
  3. Product Interface: Declares the interface for the objects the factory method creates.
  4. Concrete Product Class: Implements the Product interface.

Example

Let's consider an example where we need to create different types of documents (e.g., Word, PDF). We'll use the Factory Method pattern to achieve this.

Step-by-Step Implementation

  1. Define the Product Interface:
public interface Document {
    void open();
    void close();
}
  1. Implement Concrete Products:
public class WordDocument implements Document {
    @Override
    public void open() {
        System.out.println("Opening Word document...");
    }

    @Override
    public void close() {
        System.out.println("Closing Word document...");
    }
}

public class PdfDocument implements Document {
    @Override
    public void open() {
        System.out.println("Opening PDF document...");
    }

    @Override
    public void close() {
        System.out.println("Closing PDF document...");
    }
}
  1. Define the Creator Class:
public abstract class DocumentCreator {
    public abstract Document createDocument();

    public void someOperation() {
        Document doc = createDocument();
        doc.open();
        // Perform operations on the document
        doc.close();
    }
}
  1. Implement Concrete Creators:
public class WordDocumentCreator extends DocumentCreator {
    @Override
    public Document createDocument() {
        return new WordDocument();
    }
}

public class PdfDocumentCreator extends DocumentCreator {
    @Override
    public Document createDocument() {
        return new PdfDocument();
    }
}
  1. Client Code:
public class Client {
    public static void main(String[] args) {
        DocumentCreator creator;

        // Create a Word document
        creator = new WordDocumentCreator();
        creator.someOperation();

        // Create a PDF document
        creator = new PdfDocumentCreator();
        creator.someOperation();
    }
}

Explanation

  • Product Interface (Document): Declares the operations that all concrete products must implement.
  • Concrete Products (WordDocument, PdfDocument): Implement the Document interface.
  • Creator (DocumentCreator): Declares the factory method createDocument() that returns an object of type Document. It may also define a default implementation of the factory method.
  • Concrete Creators (WordDocumentCreator, PdfDocumentCreator): Override the factory method to return an instance of a specific Document.

Advantages and Disadvantages

Advantages

  • Flexibility: The Factory Method pattern provides flexibility in terms of object creation. It allows the client code to be independent of the concrete classes.
  • Reusability: Promotes code reusability by decoupling the code that creates objects from the code that uses the objects.
  • Extensibility: New products can be introduced without changing the existing code.

Disadvantages

  • Complexity: The pattern can introduce complexity into the codebase due to the need for additional classes and interfaces.
  • Overhead: May lead to an overhead of creating many small classes.

Conclusion

The Factory Method pattern is a powerful tool for creating objects in a flexible and reusable manner. By defining a common interface for creating objects and allowing subclasses to specify the type of objects to be created, this pattern promotes loose coupling and enhances the maintainability of the code. While it can introduce some complexity, the benefits often outweigh the drawbacks, especially in large and complex systems.
