Introduction

The Command Pattern is a behavioral design pattern that turns a request into an independent object that contains all the information about the request. This pattern allows methods to be parameterized with different requests, delay or queue the execution of a request, and support operations that can be undone.

Key Concepts

  • Command: Interface that declares a method to execute an operation.
  • ConcreteCommand: Implements the Command interface and defines the relationship between the receiver and an action.
  • Client: Creates a ConcreteCommand object and configures it with the receiver.
  • Invoker: Asks the Command object to execute the request.
  • Receiver: Knows how to perform the operations associated with carrying out a request.

Structure

| Component | Description | |----------------|-----------------------------------------------------------------------------| | Command | Interface that declares the execute method. | | ConcreteCommand| Implements the Command interface and defines the action to be performed. | | Client | Creates and configures the Command objects. | | Invoker | Invokes the execute method on the Command object. | | Receiver | Performs the necessary actions to fulfill the Command's request. |

Code Example

Below is a Python example to illustrate the Command Pattern:

# Command Interface
class Command:
    def execute(self):
        pass

# ConcreteCommand
class LightOnCommand(Command):
    def __init__(self, light):
        self.light = light

    def execute(self):
        self.light.on()

class LightOffCommand(Command):
    def __init__(self, light):
        self.light = light

    def execute(self):
        self.light.off()

# Receiver
class Light:
    def on(self):
        print("The light is on")

    def off(self):
        print("The light is off")

# Invoker
class RemoteControl:
    def __init__(self):
        self.command = None

    def set_command(self, command):
        self.command = command

    def press_button(self):
        self.command.execute()

# Client
if __name__ == "__main__":
    light = Light()
    light_on = LightOnCommand(light)
    light_off = LightOffCommand(light)

    remote = RemoteControl()
    remote.set_command(light_on)
    remote.press_button()

    remote.set_command(light_off)
    remote.press_button()

Code Explanation

  1. Command Interface: The Command class defines the interface with the execute method.
  2. ConcreteCommand: The LightOnCommand and LightOffCommand classes implement the Command interface and define the specific actions to turn the light on and off.
  3. Receiver: The Light class contains the on and off methods that perform the actual actions.
  4. Invoker: The RemoteControl class maintains a reference to a Command object and calls its execute method when the button is pressed.
  5. Client: In the if __name__ == "__main__": block, the Light, LightOnCommand, LightOffCommand, and RemoteControl objects are created. The client configures the RemoteControl with the commands and simulates the button press.

Conclusion

The Command Pattern is a powerful tool for decoupling the object that invokes an operation from the object that actually performs it. This allows for greater flexibility and extensibility in software design, facilitating the implementation of operations that can be parameterized, undone, or queued. This pattern is especially useful in applications that require a high degree of flexibility and control over the actions that are executed.
