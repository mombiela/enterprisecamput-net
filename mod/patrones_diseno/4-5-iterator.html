Topic 4-5: Iterator

What is the Iterator Pattern?

The Iterator pattern is a behavioral design pattern that provides a way to access the elements of an aggregate object sequentially without exposing its underlying representation. This pattern is particularly useful when you need to traverse a collection of objects.

Key Concepts

  1. Iterator Interface: Defines the methods for accessing and traversing elements.
  2. Concrete Iterator: Implements the Iterator interface and keeps track of the current position in the traversal.
  3. Aggregate Interface: Defines the method(s) to create an iterator object.
  4. Concrete Aggregate: Implements the Aggregate interface to return an instance of the Concrete Iterator.

Structure

Component Description
Iterator Interface for accessing and traversing elements.
Concrete Iterator Implements the Iterator interface.
Aggregate Interface for creating an Iterator object.
Concrete Aggregate Implements the Aggregate interface to create an Iterator.

Example

Let's consider a simple example where we have a collection of Book objects and we want to iterate over them.

Book Class

public class Book {
    private String title;

    public Book(String title) {
        this.title = title;
    }

    public String getTitle() {
        return title;
    }
}

Iterator Interface

public interface Iterator {
    boolean hasNext();
    Object next();
}

Concrete Iterator

public class BookIterator implements Iterator {
    private Book[] books;
    private int position = 0;

    public BookIterator(Book[] books) {
        this.books = books;
    }

    @Override
    public boolean hasNext() {
        return position < books.length && books[position] != null;
    }

    @Override
    public Object next() {
        return books[position++];
    }
}

Aggregate Interface

public interface Aggregate {
    Iterator createIterator();
}

Concrete Aggregate

public class BookCollection implements Aggregate {
    private Book[] books;
    private int index = 0;

    public BookCollection(int size) {
        books = new Book[size];
    }

    public void addBook(Book book) {
        if (index < books.length) {
            books[index++] = book;
        }
    }

    @Override
    public Iterator createIterator() {
        return new BookIterator(books);
    }
}

Client Code

public class Main {
    public static void main(String[] args) {
        BookCollection bookCollection = new BookCollection(5);
        bookCollection.addBook(new Book("Design Patterns"));
        bookCollection.addBook(new Book("Refactoring"));
        bookCollection.addBook(new Book("Clean Code"));

        Iterator iterator = bookCollection.createIterator();
        while (iterator.hasNext()) {
            Book book = (Book) iterator.next();
            System.out.println("Book Title: " + book.getTitle());
        }
    }
}

Explanation

  1. Book Class: Represents the objects we want to iterate over.
  2. Iterator Interface: Defines the methods hasNext() and next().
  3. BookIterator Class: Implements the Iterator interface and keeps track of the current position.
  4. Aggregate Interface: Defines the method createIterator().
  5. BookCollection Class: Implements the Aggregate interface and provides a method to add books and create an iterator.
  6. Client Code: Demonstrates how to use the iterator to traverse the collection of books.

Advantages of the Iterator Pattern

  • Simplifies Access: Provides a standard way to traverse collections.
  • Encapsulation: Hides the internal structure of the collection.
  • Flexibility: Can be used to traverse different types of collections.

Disadvantages of the Iterator Pattern

  • Overhead: May introduce additional complexity and overhead.
  • Single Responsibility: The iterator may need to be updated if the collection structure changes.

Conclusion

The Iterator pattern is a powerful tool for traversing collections without exposing their internal structure. By implementing this pattern, you can simplify the process of accessing elements in a collection, making your code more maintainable and flexible. Understanding and applying the Iterator pattern can greatly enhance your ability to manage collections in a clean and efficient manner.
