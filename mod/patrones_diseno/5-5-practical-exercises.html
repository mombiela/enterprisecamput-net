Practical Exercises

In this section, we will provide a series of practical exercises designed to help you apply the design patterns you've learned throughout the course. These exercises will cover various scenarios and require you to implement different design patterns to solve common development problems. Each exercise will include a problem statement, a set of requirements, and hints or tips to guide you.

Exercise 1: Singleton Pattern

Problem Statement: You are developing a logging system for an application. The system should ensure that only one instance of the logger exists throughout the application to avoid conflicts and ensure consistency.

Requirements:

  1. Implement a Logger class that follows the Singleton pattern.
  2. Ensure that the Logger class has a method to log messages.
  3. Demonstrate that only one instance of the Logger class is created, even if multiple threads try to create an instance simultaneously.

Hints:

  • Use a private constructor to prevent direct instantiation.
  • Use a static method to provide a global point of access to the instance.
  • Consider thread safety when implementing the Singleton pattern.

Solution Example:

import threading

class Logger:
    _instance = None
    _lock = threading.Lock()

    def __new__(cls):
        if cls._instance is None:
            with cls._lock:
                if cls._instance is None:
                    cls._instance = super(Logger, cls).__new__(cls)
        return cls._instance

    def log(self, message):
        print(message)

# Testing the Singleton
logger1 = Logger()
logger2 = Logger()

print(logger1 is logger2)  # Should print True

logger1.log("This is a log message.")

Exercise 2: Factory Method Pattern

Problem Statement: You are developing a notification system that can send different types of notifications (e.g., Email, SMS, Push). Each notification type should be created using a factory method.

Requirements:

  1. Create an abstract Notification class with a method send.
  2. Implement concrete classes EmailNotification, SMSNotification, and PushNotification that inherit from Notification.
  3. Create a NotificationFactory class with a method create_notification that returns an instance of the appropriate notification type based on input.

Hints:

  • Use inheritance to create the concrete notification classes.
  • The factory method should take a parameter to determine the type of notification to create.

Solution Example:

class Notification:
    def send(self, message):
        raise NotImplementedError

class EmailNotification(Notification):
    def send(self, message):
        print(f"Sending Email: {message}")

class SMSNotification(Notification):
    def send(self, message):
        print(f"Sending SMS: {message}")

class PushNotification(Notification):
    def send(self, message):
        print(f"Sending Push Notification: {message}")

class NotificationFactory:
    @staticmethod
    def create_notification(notification_type):
        if notification_type == "email":
            return EmailNotification()
        elif notification_type == "sms":
            return SMSNotification()
        elif notification_type == "push":
            return PushNotification()
        else:
            raise ValueError("Unknown notification type")

# Testing the Factory Method
factory = NotificationFactory()
notification = factory.create_notification("email")
notification.send("Hello via Email!")

notification = factory.create_notification("sms")
notification.send("Hello via SMS!")

Exercise 3: Observer Pattern

Problem Statement: You are developing a stock market monitoring application. The application should notify multiple observers (e.g., traders, analysts) whenever the stock price changes.

Requirements:

  1. Create an Observable class that maintains a list of observers and notifies them of changes.
  2. Create an Observer interface with a method update.
  3. Implement concrete observer classes Trader and Analyst that follow the Observer interface.
  4. Demonstrate the observer pattern by changing the stock price and notifying the observers.

Hints:

  • Use a list to keep track of observers in the Observable class.
  • Implement methods to add and remove observers.
  • Call the update method on each observer when the stock price changes.

Solution Example:

class Observer:
    def update(self, price):
        raise NotImplementedError

class Trader(Observer):
    def update(self, price):
        print(f"Trader notified of stock price change: {price}")

class Analyst(Observer):
    def update(self, price):
        print(f"Analyst notified of stock price change: {price}")

class StockMarket(Observable):
    def __init__(self):
        self._observers = []
        self._price = None

    def add_observer(self, observer):
        self._observers.append(observer)

    def remove_observer(self, observer):
        self._observers.remove(observer)

    def set_price(self, price):
        self._price = price
        self._notify_observers()

    def _notify_observers(self):
        for observer in self._observers:
            observer.update(self._price)

# Testing the Observer Pattern
market = StockMarket()
trader = Trader()
analyst = Analyst()

market.add_observer(trader)
market.add_observer(analyst)

market.set_price(100)  # Both trader and analyst should be notified
market.set_price(105)  # Both trader and analyst should be notified again

Conclusion

These practical exercises are designed to help you understand and apply the design patterns covered in this course. By working through these exercises, you will gain hands-on experience and deepen your understanding of how to implement design patterns to solve common development problems. Remember to refer back to the theoretical concepts and examples provided in the earlier modules if you need additional guidance. Happy coding!
