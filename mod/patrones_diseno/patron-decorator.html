Introduction

The Decorator Pattern is a structural design pattern that allows adding functionalities to an object dynamically. This pattern is useful when we want to extend the capabilities of a class without modifying its original code, thus promoting the open/closed principle of SOLID.

Key Concepts

  • Composition over inheritance: The Decorator Pattern favors composition over inheritance to extend the functionalities of objects.
  • Common interface: Both the original object and the decorators implement a common interface, allowing the decorators to be interchangeable.
  • Encapsulation: Decorators encapsulate the original object and can add or modify its behavior.

Structure

The Decorator Pattern is composed of the following elements:

  • Component: Defines the common interface for objects that can have additional responsibilities.
  • Concrete Component: Implements the Component interface. It is the object to which additional responsibilities can be added.
  • Decorator: Maintains a reference to the Component and defines an interface that conforms to the Component's interface.
  • Concrete Decorator: Extends the functionality of the Component by adding additional responsibilities.

Code Example

Below is a Python example to illustrate how the Decorator Pattern works.

from abc import ABC, abstractmethod

# Component
class Beverage(ABC):
    @abstractmethod
    def cost(self) -> float:
        pass

# Concrete Component
class Espresso(Beverage):
    def cost(self) -> float:
        return 1.99

# Decorator
class AddonDecorator(Beverage, ABC):
    def __init__(self, beverage: Beverage):
        self._beverage = beverage

# Concrete Decorator
class Milk(AddonDecorator):
    def cost(self) -> float:
        return self._beverage.cost() + 0.50

class Mocha(AddonDecorator):
    def cost(self) -> float:
        return self._beverage.cost() + 0.75

# Using the Decorator Pattern
beverage = Espresso()
print(f"Cost of Espresso: ${beverage.cost()}")

beverage_with_milk = Milk(beverage)
print(f"Cost of Espresso with Milk: ${beverage_with_milk.cost()}")

beverage_with_milk_and_mocha = Mocha(beverage_with_milk)
print(f"Cost of Espresso with Milk and Mocha: ${beverage_with_milk_and_mocha.cost()}")

Code Explanation

  1. Component: Beverage is an abstract class that defines the cost method.
  2. Concrete Component: Espresso is a concrete implementation of Beverage.
  3. Decorator: AddonDecorator is an abstract class that implements Beverage and contains a reference to a Beverage object.
  4. Concrete Decorators: Milk and Mocha are concrete decorators that add additional functionalities to the Beverage object.

Comparison: Inheritance vs Decorator

| Feature | Inheritance | Decorator | |----------------------|-----------------------------------|------------------------------------| | Extensibility | Limited to the class hierarchy | Very flexible, multiple decorators can be combined | | Complexity | Can become complex with many subclasses | Simpler, as functionalities are added modularly | | Reusability | Less reusable | High reusability of decorators |

Conclusion

The Decorator Pattern is a powerful tool for adding functionalities to objects dynamically and flexibly. By favoring composition over inheritance, this pattern allows creating more modular and maintainable systems. It is especially useful in situations where additional functionalities can be combined in different ways, providing great flexibility in software design.
