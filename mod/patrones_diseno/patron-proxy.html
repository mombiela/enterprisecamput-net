Introduction to the Proxy Pattern

The Proxy pattern is a structural design pattern that provides a substitute or placeholder for another object. A proxy controls access to an object, allowing certain actions to be performed before or after the request to the real object.

Objectives of the Proxy Pattern

  • Access Control: Restrict access to certain methods or data.
  • Optimization: Load heavy objects only when needed.
  • Security: Protect the real object from unauthorized access.
  • Logging and Monitoring: Log requests and responses for auditing or monitoring.

Types of Proxy

  • Virtual Proxy: Manages the creation and access to resource-intensive objects.
  • Protection Proxy: Controls access to an object, allowing different permissions for different users.
  • Remote Proxy: Represents an object that resides in a different address space (e.g., on a remote machine).
  • Cache Proxy: Stores the results of expensive operations to avoid repeating them.

Structure of the Proxy Pattern

The Proxy pattern generally involves the following components:

  • Subject: Defines the common interface for RealSubject and Proxy.
  • RealSubject: The real object that the proxy represents.
  • Proxy: Maintains a reference to the RealSubject and controls access to it.

Code Example

Below is a Java example to illustrate the Proxy pattern:

// Common interface
public interface Image {
    void display();
}

// Real Subject
public class RealImage implements Image {
    private String fileName;

    public RealImage(String fileName) {
        this.fileName = fileName;
        loadFromDisk();
    }

    private void loadFromDisk() {
        System.out.println("Loading " + fileName);
    }

    @Override
    public void display() {
        System.out.println("Displaying " + fileName);
    }
}

// Proxy
public class ProxyImage implements Image {
    private RealImage realImage;
    private String fileName;

    public ProxyImage(String fileName) {
        this.fileName = fileName;
    }

    @Override
    public void display() {
        if (realImage == null) {
            realImage = new RealImage(fileName);
        }
        realImage.display();
    }
}

// Client
public class ProxyPatternDemo {
    public static void main(String[] args) {
        Image image = new ProxyImage("test_image.jpg");

        // Image will be loaded from disk
        image.display();
        System.out.println("");

        // Image will not be loaded from disk
        image.display();
    }
}

Code Explanation

  • Image Interface: Defines the display() method to be implemented by RealImage and ProxyImage.
  • RealImage: Implements the Image interface and represents the real object that is loaded from disk.
  • ProxyImage: Implements the Image interface and controls access to RealImage. Only loads RealImage when necessary.
  • ProxyPatternDemo: Client class that uses ProxyImage to access RealImage.

Advantages and Disadvantages

Advantages

  • Access Control: Allows controlling access to the real object.
  • Resource Optimization: Loads heavy objects only when necessary.
  • Security: Can add additional security layers.
  • Monitoring and Logging: Facilitates logging of requests and responses.

Disadvantages

  • Additional Complexity: Adds complexity to the system due to the introduction of new classes.
  • Performance: May introduce slight overhead due to request delegation.

Conclusion

The Proxy pattern is a powerful tool for controlling access to objects, optimizing resource usage, and adding layers of security and monitoring. However, it should be used carefully to avoid adding unnecessary complexity to the system. By understanding and correctly applying this pattern, developers can create more efficient and secure systems.
