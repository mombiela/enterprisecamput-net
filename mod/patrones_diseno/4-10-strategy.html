Topic: Strategy Pattern

Introduction to the Strategy Pattern

The Strategy Pattern is a behavioral design pattern that enables selecting an algorithm's behavior at runtime. Instead of implementing a single algorithm directly, code receives run-time instructions as to which in a family of algorithms to use.

Key Concepts

  1. Strategy Interface: Defines a common interface for all supported algorithms.
  2. Concrete Strategies: Implement the algorithm defined in the Strategy interface.
  3. Context: Maintains a reference to a Strategy object and is configured with a ConcreteStrategy object.

Structure

The Strategy Pattern involves three main components:

  1. Strategy Interface: Declares operations common to all supported versions of some algorithm.
  2. Concrete Strategies: Implement the algorithm using the Strategy interface.
  3. Context: Uses a reference to a Strategy object to call the algorithm defined by a ConcreteStrategy.

Example

Let's consider a simple example where we have different strategies for sorting a list of numbers.

Step-by-Step Implementation

  1. Define the Strategy Interface
from abc import ABC, abstractmethod

class SortStrategy(ABC):
    @abstractmethod
    def sort(self, data):
        pass
  1. Implement Concrete Strategies
class BubbleSortStrategy(SortStrategy):
    def sort(self, data):
        n = len(data)
        for i in range(n):
            for j in range(0, n-i-1):
                if data[j] > data[j+1]:
                    data[j], data[j+1] = data[j+1], data[j]
        return data

class QuickSortStrategy(SortStrategy):
    def sort(self, data):
        if len(data) <= 1:
            return data
        pivot = data[len(data) // 2]
        left = [x for x in data if x < pivot]
        middle = [x for x in data if x == pivot]
        right = [x for x in data if x > pivot]
        return self.sort(left) + middle + self.sort(right)
  1. Create the Context Class
class SortContext:
    def __init__(self, strategy: SortStrategy):
        self._strategy = strategy

    def set_strategy(self, strategy: SortStrategy):
        self._strategy = strategy

    def sort(self, data):
        return self._strategy.sort(data)
  1. Using the Strategy Pattern
if __name__ == "__main__":
    data = [5, 2, 9, 1, 5, 6]

    context = SortContext(BubbleSortStrategy())
    print("Bubble Sort:", context.sort(data.copy()))

    context.set_strategy(QuickSortStrategy())
    print("Quick Sort:", context.sort(data.copy()))

Explanation

  1. Strategy Interface: SortStrategy is an abstract base class that defines the sort method.
  2. Concrete Strategies: BubbleSortStrategy and QuickSortStrategy implement the sort method in different ways.
  3. Context: SortContext uses a SortStrategy to sort data. It allows changing the sorting strategy at runtime.

Advantages and Disadvantages

Advantages

  • Flexibility: The Strategy Pattern allows changing the algorithm at runtime.
  • Encapsulation: Each algorithm is encapsulated in its own class, making the code easier to understand and maintain.
  • Open/Closed Principle: New strategies can be added without modifying the existing code.

Disadvantages

  • Overhead: Introducing new classes for each algorithm can increase the complexity of the code.
  • Communication Overhead: The context must be aware of the different strategies, which can lead to increased coupling.

Conclusion

The Strategy Pattern is a powerful tool for designing flexible and maintainable software. By encapsulating algorithms in separate classes and using a common interface, it allows for easy swapping of algorithms at runtime. This pattern is particularly useful when you have multiple ways to perform an operation and want to choose the best one dynamically.
