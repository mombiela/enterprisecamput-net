Introduction

In this topic, we will explore two fundamental data structures in programming: stacks and queues. These structures are essential for data management in many algorithms and applications. We will learn what they are, how they work, and how to implement them in code.

Stacks

Definition

A stack is a data structure that follows the LIFO (Last In, First Out) principle, meaning the last element to enter is the first to leave.

Basic Operations

  • Push: Add an element to the top of the stack.
  • Pop: Remove the element from the top of the stack.
  • Peek: Get the value of the element at the top without removing it.
  • isEmpty: Check if the stack is empty.

Python Code Example

class Stack:
    def __init__(self):
        self.items = []

    def push(self, item):
        self.items.append(item)

    def pop(self):
        if not self.is_empty():
            return self.items.pop()
        else:
            return None

    def peek(self):
        if not self.is_empty():
            return self.items[-1]
        else:
            return None

    def is_empty(self):
        return len(self.items) == 0

# Usage example
stack = Stack()
stack.push(1)
stack.push(2)
print(stack.peek())  # Output: 2
print(stack.pop())   # Output: 2
print(stack.pop())   # Output: 1
print(stack.is_empty())  # Output: True

Common Applications

  • Expression evaluation: Used in the evaluation of arithmetic and logical expressions.
  • Recursion: Stacks are fundamental for implementing recursive calls.
  • Undo/redo: Functionalities in text editors and other programs.

Queues

Definition

A queue is a data structure that follows the FIFO (First In, First Out) principle, meaning the first element to enter is the first to leave.

Basic Operations

  • Enqueue: Add an element to the end of the queue.
  • Dequeue: Remove the first element from the queue.
  • Front: Get the value of the first element without removing it.
  • isEmpty: Check if the queue is empty.

Python Code Example

class Queue:
    def __init__(self):
        self.items = []

    def enqueue(self, item):
        self.items.append(item)

    def dequeue(self):
        if not self.is_empty():
            return self.items.pop(0)
        else:
            return None

    def front(self):
        if not self.is_empty():
            return self.items[0]
        else:
            return None

    def is_empty(self):
        return len(self.items) == 0

# Usage example
queue = Queue()
queue.enqueue(1)
queue.enqueue(2)
print(queue.front())  # Output: 1
print(queue.dequeue())  # Output: 1
print(queue.dequeue())  # Output: 2
print(queue.is_empty())  # Output: True

Common Applications

  • Task management: Used in operating systems to manage processes.
  • Printing systems: Handling print jobs in a printer.
  • Networks: Managing data packets in communication networks.

Comparison between Stacks and Queues

| Characteristic | Stack (LIFO) | Queue (FIFO) | |----------------|--------------|--------------| | Access order | Last in, first out | First in, first out | | Main operations| Push, Pop, Peek, isEmpty | Enqueue, Dequeue, Front, isEmpty | | Usage examples | Expression evaluation, Recursion, Undo/redo | Task management, Printing systems, Networks |

Conclusion

Stacks and queues are fundamental data structures that every programmer should know. Understanding how they work and how to implement them is crucial for solving a variety of programming problems. Practicing with examples and exercises will help consolidate these concepts and apply them in real situations.
