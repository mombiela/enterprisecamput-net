Introduction

In this section, you will learn how to create your first Angular application. We will cover the following key concepts:

  • Setting up the Angular environment
  • Creating a new Angular project
  • Understanding the project structure
  • Running the Angular application
  • Basic Angular components

Setting Up the Angular Environment

Before you can create an Angular application, you need to set up your development environment.

Prerequisites

  • Node.js and npm: Angular requires Node.js and npm (Node Package Manager). You can download and install them from nodejs.org.
  • Angular CLI: The Angular Command Line Interface (CLI) is a powerful tool to initialize, develop, scaffold, and maintain Angular applications.

Installing Angular CLI

To install the Angular CLI, open your terminal or command prompt and run:

npm install -g @angular/cli

Creating a New Angular Project

Once the Angular CLI is installed, you can create a new Angular project.

Steps to Create a New Project

  1. Open your terminal and navigate to the directory where you want to create your project.
  2. Run the following command to create a new Angular project: 
    ng new my-first-angular-app
  3. Navigate into the project directory: 
    cd my-first-angular-app

Understanding the Project Structure

When you create a new Angular project, the Angular CLI generates a set of files and folders. Here is a brief overview of the main components:

  • e2e/: End-to-end tests for the application.
  • node_modules/: Contains the npm packages installed for the project.
  • src/: The source code of the application.
    • app/: Contains the main application code.
    • assets/: Contains static assets like images and styles.
    • environments/: Contains environment-specific configuration files.
  • angular.json: Configuration file for Angular CLI.
  • package.json: Lists the npm packages the project depends on.
  • tsconfig.json: TypeScript configuration file.

Running the Angular Application

To run your Angular application, use the Angular CLI:

ng serve

This command will compile the application and start a development server. By default, the application will be available at http://localhost:4200/.

Basic Angular Components

Angular applications are built using components. A component in Angular is a TypeScript class with an HTML template and styles.

Creating a New Component

You can create a new component using the Angular CLI:

ng generate component my-component

This command will generate a new component with the following files:

  • my-component.component.ts: The TypeScript file containing the component logic.
  • my-component.component.html: The HTML template for the component.
  • my-component.component.css: The styles for the component.
  • my-component.component.spec.ts: The test file for the component.

Example Component

Here is an example of a simple Angular component:

app.component.ts

import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  title = 'my-first-angular-app';
}

app.component.html

<div style="text-align:center">
  <h1>
    Welcome to {{ title }}!
  </h1>
</div>

Explanation

  • @Component: This decorator marks the class as an Angular component and provides metadata about the component.
  • selector: The custom HTML tag that represents this component.
  • templateUrl: The path to the HTML template file.
  • styleUrls: The path to the CSS file(s).

Conclusion

Congratulations! You have successfully created your first Angular application. You learned how to set up the Angular environment, create a new project, understand the project structure, run the application, and create a basic Angular component. This foundational knowledge will help you as you continue to build more complex Angular applications.
