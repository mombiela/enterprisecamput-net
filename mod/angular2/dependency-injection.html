Introduction to Dependency Injection

Dependency Injection (DI) is a design pattern used to implement IoC (Inversion of Control), allowing a class to receive its dependencies from an external source rather than creating them itself. Angular's DI system is a core part of the framework, making it easier to manage dependencies and promote code reusability and testability.

Key Concepts

  • Injector: The mechanism that resolves dependencies and provides instances of services.
  • Provider: A way to tell the injector how to create the dependency.
  • Service: A class that provides functionality to be used across the application.
  • Dependency: An object that a class needs to perform its function.

Setting Up a Service

To demonstrate DI, we will create a simple service and inject it into a component.

Creating a Service

First, create a service using Angular CLI:

ng generate service example

This command generates a service file example.service.ts:

import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root',
})
export class ExampleService {
  constructor() {}

  getMessage(): string {
    return 'Hello from ExampleService!';
  }
}

Injecting the Service into a Component

Next, inject the ExampleService into a component.

  1. Open the component file, e.g., app.component.ts.
  2. Import the ExampleService.
  3. Inject the service via the constructor.
import { Component, OnInit } from '@angular/core';
import { ExampleService } from './example.service';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent implements OnInit {
  message: string;

  constructor(private exampleService: ExampleService) {}

  ngOnInit() {
    this.message = this.exampleService.getMessage();
  }
}

Displaying the Message in the Template

Update the template file app.component.html to display the message:

<div>
  {{ message }}
</div>

Advanced Dependency Injection

Hierarchical Injectors

Angular's DI system is hierarchical, meaning injectors can be nested. This allows for different instances of a service to be provided at different levels of the application.

  • Root Injector: Provides services application-wide.
  • Component Injector: Provides services specific to a component and its children.

Providing Services in a Module

You can provide services at the module level to control their scope.

import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AppComponent } from './app.component';
import { ExampleService } from './example.service';

@NgModule({
  declarations: [AppComponent],
  imports: [BrowserModule],
  providers: [ExampleService],
  bootstrap: [AppComponent]
})
export class AppModule {}

Using `@Inject` and `@Optional`

Sometimes you need more control over how dependencies are injected.

  • @Inject: Explicitly specify the dependency to inject.
  • @Optional: Allow a dependency to be optional.
import { Component, Inject, Optional } from '@angular/core';
import { ExampleService } from './example.service';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  constructor(@Optional() @Inject(ExampleService) private exampleService: ExampleService) {
    if (this.exampleService) {
      console.log(this.exampleService.getMessage());
    } else {
      console.log('ExampleService is not available');
    }
  }
}

Summary

Dependency Injection is a powerful pattern that Angular leverages to manage dependencies efficiently. By understanding and utilizing Angular's DI system, you can create more modular, testable, and maintainable applications. Key takeaways include:

  • Injector: Manages the creation and provision of dependencies.
  • Provider: Configures how dependencies are created.
  • Service: A reusable class that provides functionality.
  • Hierarchical Injectors: Allow different instances of services at different levels.

Mastering DI in Angular will significantly enhance your ability to build robust applications.
