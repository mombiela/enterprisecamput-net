Introduction to Angular Routing

Routing in Angular allows you to navigate between different views or components within your application. It is a crucial aspect of single-page applications (SPAs) as it enables the user to move between different parts of the app without reloading the entire page.

Key Concepts of Angular Routing

  • Router Module: The Angular module that provides the necessary services and directives for routing.
  • Routes: Configuration objects that define the mapping between URL paths and components.
  • Router Outlet: A directive that acts as a placeholder for the routed component.
  • Router Link: A directive for linking to different routes.

Setting Up Basic Routes

To set up routing in an Angular application, follow these steps:

  1. Install Angular Router: If not already included, install the Angular Router package.

    npm install @angular/router

  2. Import RouterModule: Import RouterModule and Routes in your main application module.

    import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { RouterModule, Routes } from '@angular/router';

import { AppComponent } from './app.component';
import { HomeComponent } from './home/home.component';
import { AboutComponent } from './about/about.component';

const routes: Routes = [
  { path: '', component: HomeComponent },
  { path: 'about', component: AboutComponent }
];

@NgModule({
  declarations: [
    AppComponent,
    HomeComponent,
    AboutComponent
  ],
  imports: [
    BrowserModule,
    RouterModule.forRoot(routes)
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

  3. Add Router Outlet: Add the <router-outlet> directive in your main component template.

    <!-- app.component.html -->
<nav>
  <a routerLink="/">Home</a>
  <a routerLink="/about">About</a>
</nav>
<router-outlet></router-outlet>

  4. Create Components: Ensure that the components you want to route to are created.

    // home.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-home',
  template: `<h1>Home</h1>`
})
export class HomeComponent { }

// about.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-about',
  template: `<h1>About</h1>`
})
export class AboutComponent { }

Advanced Routing Techniques

Nested Routes

Nested routes allow you to define routes within other routes, creating a hierarchy.

const routes: Routes = [
  { path: '', component: HomeComponent },
  { 
    path: 'about', 
    component: AboutComponent,
    children: [
      { path: 'team', component: TeamComponent },
      { path: 'history', component: HistoryComponent }
    ]
  }
];

Route Guards

Route guards are used to control access to certain routes based on conditions.

import { Injectable } from '@angular/core';
import { CanActivate, ActivatedRouteSnapshot, RouterStateSnapshot, Router } from '@angular/router';

@Injectable({
  providedIn: 'root'
})
export class AuthGuard implements CanActivate {
  constructor(private router: Router) {}

  canActivate(
    next: ActivatedRouteSnapshot,
    state: RouterStateSnapshot): boolean {
    const isAuthenticated = // logic to check if user is authenticated
    if (!isAuthenticated) {
      this.router.navigate(['/login']);
      return false;
    }
    return true;
  }
}

// Applying the guard to a route
const routes: Routes = [
  { path: '', component: HomeComponent },
  { path: 'protected', component: ProtectedComponent, canActivate: [AuthGuard] }
];

Lazy Loading

Lazy loading helps to load feature modules only when they are needed, improving the performance of the application.

const routes: Routes = [
  { path: '', component: HomeComponent },
  { path: 'feature', loadChildren: () => import('./feature/feature.module').then(m => m.FeatureModule) }
];

Summary

Setting up routes in Angular involves configuring the RouterModule with route definitions, using the <router-outlet> directive to display routed components, and employing advanced techniques like nested routes, route guards, and lazy loading to enhance the application's routing capabilities. By mastering these concepts, you can create robust and efficient navigation within your Angular applications.
