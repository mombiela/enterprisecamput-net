In Angular, a component is a fundamental building block of the application. Each component consists of an HTML template that defines the view, a TypeScript class that defines the behavior, and metadata that ties the view and behavior together. This section focuses on Component Templates, which are essential for defining the user interface of an Angular application.

Key Concepts

  • Template: The HTML that defines the view for a component.
  • Interpolation: Embedding expressions in the template to display dynamic data.
  • Directives: Special tokens in the template that change the appearance or behavior of an element.
  • Binding: Connecting the component's data and the template.

Creating a Basic Component Template

A component template is defined using the template or templateUrl property in the component's metadata. Here's an example of a simple component:

import { Component } from '@angular/core';

@Component({
  selector: 'app-hello-world',
  template: `<h1>Hello, {{name}}!</h1>`,
})
export class HelloWorldComponent {
  name: string = 'Angular';
}

In this example:

  • The selector defines the custom HTML tag (<app-hello-world>) for this component.
  • The template property contains the HTML template.
  • The name property is interpolated in the template using {{name}}.

Interpolation

Interpolation allows you to embed expressions in the template. These expressions are evaluated and the result is inserted into the DOM.

Example:

@Component({
  selector: 'app-user',
  template: `<p>Welcome, {{user.name}}!</p>`,
})
export class UserComponent {
  user = { name: 'John Doe' };
}

In this example, {{user.name}} is replaced with John Doe in the rendered HTML.

Directives

Directives are special tokens in the template that can change the appearance or behavior of an element. There are three types of directives in Angular:

  • Component Directives: Directives with a template.
  • Structural Directives: Change the DOM layout by adding or removing elements (e.g., *ngIf, *ngFor).
  • Attribute Directives: Change the appearance or behavior of an element (e.g., ngClass, ngStyle).

Structural Directives

*ngIf

The *ngIf directive conditionally includes or excludes an element in the DOM.

Example:

@Component({
  selector: 'app-conditional',
  template: `
    <div *ngIf="isVisible">This is visible</div>
    <button (click)="toggleVisibility()">Toggle Visibility</button>
  `,
})
export class ConditionalComponent {
  isVisible: boolean = true;

  toggleVisibility() {
    this.isVisible = !this.isVisible;
  }
}

*ngFor

The *ngFor directive repeats an element for each item in a list.

Example:

@Component({
  selector: 'app-item-list',
  template: `
    <ul>
      <li *ngFor="let item of items">{{item}}</li>
    </ul>
  `,
})
export class ItemListComponent {
  items: string[] = ['Item 1', 'Item 2', 'Item 3'];
}

Attribute Directives

ngClass

The ngClass directive adds and removes CSS classes.

Example:

@Component({
  selector: 'app-highlight',
  template: `
    <p [ngClass]="{'highlight': isHighlighted}">Highlight me!</p>
    <button (click)="toggleHighlight()">Toggle Highlight</button>
  `,
  styles: [`
    .highlight {
      background-color: yellow;
    }
  `]
})
export class HighlightComponent {
  isHighlighted: boolean = false;

  toggleHighlight() {
    this.isHighlighted = !this.isHighlighted;
  }
}

Binding

Binding connects the component's data and the template. There are four types of binding in Angular:

  • Interpolation: {{expression}}
  • Property Binding: [property]="expression"
  • Event Binding: (event)="handler"
  • Two-way Binding: [(ngModel)]="property"

Property Binding

Example:

@Component({
  selector: 'app-image',
  template: `<img [src]="imageUrl" alt="Image">`,
})
export class ImageComponent {
  imageUrl: string = 'https://example.com/image.jpg';
}

Event Binding

Example:

@Component({
  selector: 'app-click',
  template: `<button (click)="handleClick()">Click me</button>`,
})
export class ClickComponent {
  handleClick() {
    alert('Button clicked!');
  }
}

Two-way Binding

Two-way binding combines property and event binding using the ngModel directive.

Example:

import { Component } from '@angular/core';

@Component({
  selector: 'app-input',
  template: `
    <input [(ngModel)]="name">
    <p>Hello, {{name}}!</p>
  `,
})
export class InputComponent {
  name: string = '';
}

Conclusion

Component templates in Angular are powerful tools for defining the user interface of your application. By understanding and utilizing interpolation, directives, and binding, you can create dynamic and interactive web applications. Practice these concepts with the provided examples and exercises to solidify your understanding and build more complex components.
