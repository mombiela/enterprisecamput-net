Introduction to Form Validation

Form validation is a crucial aspect of web development, ensuring that user inputs are correct and safe before processing. Angular 2+ provides powerful tools to handle form validation efficiently.

Types of Forms in Angular

Angular supports two types of forms:

  • Template-driven forms: Easier to use, suitable for simple forms.
  • Reactive forms: More flexible and scalable, suitable for complex forms.

Template-driven Forms

Template-driven forms rely heavily on Angular's directives and are defined in the template.

Creating a Basic Template-driven Form

<form #userForm="ngForm" (ngSubmit)="onSubmit(userForm)">
  <div>
    <label for="name">Name:</label>
    <input type="text" id="name" name="name" ngModel required>
  </div>
  <div>
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" ngModel required email>
  </div>
  <button type="submit" [disabled]="userForm.invalid">Submit</button>
</form>

Validating Template-driven Forms

  • Required: Ensures the field is not empty.
  • Email: Ensures the field contains a valid email address.

Displaying Validation Errors

<div *ngIf="userForm.controls.name?.invalid && userForm.controls.name?.touched">
  <small *ngIf="userForm.controls.name?.errors?.required">Name is required.</small>
</div>
<div *ngIf="userForm.controls.email?.invalid && userForm.controls.email?.touched">
  <small *ngIf="userForm.controls.email?.errors?.required">Email is required.</small>
  <small *ngIf="userForm.controls.email?.errors?.email">Invalid email address.</small>
</div>

Reactive Forms

Reactive forms are more robust and provide greater control over form validation.

Creating a Basic Reactive Form

import { Component } from '@angular/core';
import { FormBuilder, FormGroup, Validators } from '@angular/forms';

@Component({
  selector: 'app-user-form',
  templateUrl: './user-form.component.html'
})
export class UserFormComponent {
  userForm: FormGroup;

  constructor(private fb: FormBuilder) {
    this.userForm = this.fb.group({
      name: ['', Validators.required],
      email: ['', [Validators.required, Validators.email]]
    });
  }

  onSubmit() {
    if (this.userForm.valid) {
      console.log(this.userForm.value);
    }
  }
}

Template for Reactive Form

<form [formGroup]="userForm" (ngSubmit)="onSubmit()">
  <div>
    <label for="name">Name:</label>
    <input type="text" id="name" formControlName="name">
    <div *ngIf="userForm.controls.name.invalid && userForm.controls.name.touched">
      <small *ngIf="userForm.controls.name.errors.required">Name is required.</small>
    </div>
  </div>
  <div>
    <label for="email">Email:</label>
    <input type="email" id="email" formControlName="email">
    <div *ngIf="userForm.controls.email.invalid && userForm.controls.email.touched">
      <small *ngIf="userForm.controls.email.errors.required">Email is required.</small>
      <small *ngIf="userForm.controls.email.errors.email">Invalid email address.</small>
    </div>
  </div>
  <button type="submit" [disabled]="userForm.invalid">Submit</button>
</form>

Custom Validators

Custom validators allow you to create specific validation logic.

Creating a Custom Validator

import { AbstractControl, ValidationErrors } from '@angular/forms';

export function forbiddenNameValidator(control: AbstractControl): ValidationErrors | null {
  const forbidden = /admin/.test(control.value);
  return forbidden ? { forbiddenName: { value: control.value } } : null;
}

Using Custom Validators

this.userForm = this.fb.group({
  name: ['', [Validators.required, forbiddenNameValidator]],
  email: ['', [Validators.required, Validators.email]]
});

Displaying Custom Validation Errors

<div *ngIf="userForm.controls.name.invalid && userForm.controls.name.touched">
  <small *ngIf="userForm.controls.name.errors.required">Name is required.</small>
  <small *ngIf="userForm.controls.name.errors.forbiddenName">Name cannot be 'admin'.</small>
</div>

Summary

Form validation in Angular 2+ can be handled using template-driven or reactive forms. Template-driven forms are simpler and suitable for basic use cases, while reactive forms offer more control and scalability for complex scenarios. Custom validators provide flexibility to implement specific validation logic. Understanding these concepts will help you create robust and user-friendly forms in your Angular applications.
