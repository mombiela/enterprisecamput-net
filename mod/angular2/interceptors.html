Interceptors in Angular 2+

Interceptors in Angular are a powerful feature that allows you to intercept and modify HTTP requests and responses. They are useful for tasks such as adding authentication tokens, logging, error handling, and more.

What are Interceptors?

  • Interceptors are part of Angular's HttpClient module.
  • They allow you to intercept and modify HTTP requests and responses globally.
  • They are implemented as services that implement the HttpInterceptor interface.

Why Use Interceptors?

  • Authentication: Automatically add authentication tokens to requests.
  • Logging: Log HTTP requests and responses for debugging purposes.
  • Error Handling: Handle errors globally and provide user-friendly messages.
  • Modify Requests/Responses: Modify request headers, body, or response data.

Creating an Interceptor

To create an interceptor, you need to:

  1. Implement the HttpInterceptor interface.
  2. Provide the interceptor in your Angular module.

Step-by-Step Example

  1. Create the Interceptor Service
import { Injectable } from '@angular/core';
import { HttpInterceptor, HttpRequest, HttpHandler, HttpEvent } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable()
export class AuthInterceptor implements HttpInterceptor {
  intercept(req: HttpRequest<any>, next: HttpHandler): Observable<HttpEvent<any>> {
    // Clone the request to add the new header
    const authReq = req.clone({
      headers: req.headers.set('Authorization', 'Bearer YOUR_TOKEN_HERE')
    });

    // Pass on the cloned request instead of the original request
    return next.handle(authReq);
  }
}
  1. Provide the Interceptor in Your Module
import { HTTP_INTERCEPTORS } from '@angular/common/http';
import { AuthInterceptor } from './auth.interceptor';

@NgModule({
  providers: [
    { provide: HTTP_INTERCEPTORS, useClass: AuthInterceptor, multi: true }
  ]
})
export class AppModule { }

Common Use Cases

Adding Authentication Tokens

  • Automatically add JWT or other tokens to every HTTP request.
intercept(req: HttpRequest<any>, next: HttpHandler): Observable<HttpEvent<any>> {
  const authToken = this.authService.getToken();
  const authReq = req.clone({
    headers: req.headers.set('Authorization', `Bearer ${authToken}`)
  });
  return next.handle(authReq);
}

Logging Requests and Responses

  • Log details of HTTP requests and responses for debugging.
intercept(req: HttpRequest<any>, next: HttpHandler): Observable<HttpEvent<any>> {
  console.log('Request:', req);
  return next.handle(req).pipe(
    tap(event => {
      if (event instanceof HttpResponse) {
        console.log('Response:', event);
      }
    })
  );
}

Global Error Handling

  • Handle HTTP errors globally and show user-friendly messages.
import { catchError } from 'rxjs/operators';
import { throwError } from 'rxjs';

intercept(req: HttpRequest<any>, next: HttpHandler): Observable<HttpEvent<any>> {
  return next.handle(req).pipe(
    catchError((error: HttpErrorResponse) => {
      let errorMessage = '';
      if (error.error instanceof ErrorEvent) {
        // Client-side error
        errorMessage = `Error: ${error.error.message}`;
      } else {
        // Server-side error
        errorMessage = `Error Code: ${error.status}\nMessage: ${error.message}`;
      }
      window.alert(errorMessage);
      return throwError(errorMessage);
    })
  );
}

Conclusion

Interceptors in Angular provide a powerful way to handle HTTP requests and responses globally. They can be used for a variety of tasks such as adding authentication tokens, logging, and error handling. By implementing the HttpInterceptor interface and providing the interceptor in your Angular module, you can easily extend the functionality of your HTTP requests and responses.

Understanding and utilizing interceptors can greatly enhance the robustness and maintainability of your Angular applications.
