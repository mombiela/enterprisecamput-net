Introduction

TypeScript is a statically typed superset of JavaScript that brings optional static typing and other features to the language. Understanding variables and data types in TypeScript is crucial for developing robust Angular applications. This section will cover the basics of declaring variables and the various data types available in TypeScript.

Declaring Variables

In TypeScript, variables can be declared using let, const, and var. Each has its own scope and usage considerations.

`let`

  • Block-scoped.
  • Can be reassigned.
let message: string = "Hello, TypeScript!";
message = "Hello, Angular!";
console.log(message); // Output: Hello, Angular!

`const`

  • Block-scoped.
  • Cannot be reassigned.
const pi: number = 3.14;
// pi = 3.14159; // Error: Assignment to constant variable.
console.log(pi); // Output: 3.14

`var`

  • Function-scoped.
  • Can be reassigned.
var greeting: string = "Hi!";
greeting = "Hello!";
console.log(greeting); // Output: Hello!

Data Types

TypeScript supports various data types, including primitive types and more complex types.

Primitive Types

  • Number: Represents both integer and floating-point numbers.
  • String: Represents text data.
  • Boolean: Represents true/false values.
  • Null: Represents an intentional absence of any object value.
  • Undefined: Represents an uninitialized variable.
  • Symbol: Represents a unique and immutable primitive value.
let age: number = 25;
let name: string = "John Doe";
let isStudent: boolean = true;
let notAssigned: undefined = undefined;
let empty: null = null;

Array

Arrays can be defined in two ways: using the type followed by [] or using the Array<type> syntax.

let numbers: number[] = [1, 2, 3, 4, 5];
let fruits: Array<string> = ["Apple", "Banana", "Cherry"];

Tuple

Tuples allow you to express an array with a fixed number of elements whose types are known.

let person: [string, number];
person = ["Alice", 30]; // Correct
// person = [30, "Alice"]; // Error: Type 'number' is not assignable to type 'string'.

Enum

Enums allow you to define a set of named constants.

enum Color {
    Red,
    Green,
    Blue
}
let c: Color = Color.Green;
console.log(c); // Output: 1

Any

The any type allows you to opt-out of type-checking.

let randomValue: any = 10;
randomValue = "Hello";
randomValue = true;

Void

The void type is used to indicate the absence of a value, commonly used as the return type of functions that do not return a value.

function logMessage(message: string): void {
    console.log(message);
}

Object

The object type represents non-primitive types.

let user: { name: string; age: number } = {
    name: "John",
    age: 30
};

Type Inference

TypeScript can infer the type of a variable based on the value assigned to it.

let inferredString = "This is a string"; // inferredString is of type string
let inferredNumber = 42; // inferredNumber is of type number

Type Assertions

Type assertions are used to tell the compiler to treat a variable as a specific type.

let someValue: any = "Hello, TypeScript!";
let strLength: number = (someValue as string).length;
console.log(strLength); // Output: 17

Conclusion

Understanding variables and data types in TypeScript is fundamental for building Angular applications. By leveraging TypeScript's static typing, you can write more predictable and maintainable code. This section covered the basics of declaring variables, the various data types available, and how to use type inference and type assertions. With this knowledge, you are well-equipped to handle data effectively in your Angular projects.
