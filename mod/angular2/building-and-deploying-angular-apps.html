Building and Deploying Angular Apps

Introduction

In this section, we will cover the essential steps and best practices for building and deploying Angular applications. This includes setting up the development environment, building the application for production, and deploying it to various platforms.

Setting Up the Development Environment

Before we start building and deploying Angular apps, we need to set up our development environment.

  • Node.js and npm: Angular requires Node.js and npm (Node Package Manager) to manage dependencies.
  • Angular CLI: The Angular Command Line Interface (CLI) is a powerful tool that helps in creating, building, and managing Angular applications.

Installing Node.js and npm

  1. Download and install Node.js from the official website.
  2. Verify the installation by running the following commands in your terminal: 
    node -v
npm -v

Installing Angular CLI

  1. Install Angular CLI globally using npm: 
    npm install -g @angular/cli
  2. Verify the installation by running: 
    ng version

Creating an Angular Application

Let's create a new Angular application using Angular CLI.

  1. Open your terminal and run the following command: 
    ng new my-angular-app
  2. Navigate to the project directory: 
    cd my-angular-app
  3. Serve the application locally: 
    ng serve
  4. Open your browser and navigate to http://localhost:4200 to see your running application.

Building the Application for Production

To deploy an Angular application, we need to build it for production. This process optimizes the application for performance and creates the necessary files for deployment.

  1. Run the following command to build the application: 
    ng build --prod
  2. The build artifacts will be stored in the dist/ directory.

Understanding the Build Output

The dist/ directory contains the following files:

  • index.html: The main HTML file.
  • main.js: The main JavaScript bundle.
  • polyfills.js: Polyfills for browser compatibility.
  • styles.css: The global styles for the application.
  • vendor.js: Third-party libraries and dependencies.

Deploying the Application

There are several ways to deploy an Angular application. We will cover deployment to popular platforms such as GitHub Pages, Firebase Hosting, and a custom server.

Deploying to GitHub Pages

  1. Install the Angular CLI GitHub Pages deployer: 
    npm install -g angular-cli-ghpages
  2. Build the application: 
    ng build --prod --base-href "https://<username>.github.io/<repository-name>/"
  3. Deploy the application: 
    ngh --dir=dist/<project-name>

Deploying to Firebase Hosting

  1. Install the Firebase CLI: 
    npm install -g firebase-tools
  2. Log in to Firebase: 
    firebase login
  3. Initialize Firebase in your project: 
    firebase init
  4. Build the application: 
    ng build --prod
  5. Deploy the application: 
    firebase deploy

Deploying to a Custom Server

  1. Build the application: 
    ng build --prod
  2. Copy the contents of the dist/ directory to your server's public directory.
  3. Configure your server to serve the index.html file for all routes.

Conclusion

In this topic, we covered the essential steps for building and deploying Angular applications. We started by setting up the development environment, creating a new Angular application, and building it for production. Finally, we explored different deployment options, including GitHub Pages, Firebase Hosting, and custom servers. By following these steps, you can ensure that your Angular applications are optimized and ready for production deployment.
