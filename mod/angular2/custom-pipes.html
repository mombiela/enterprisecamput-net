Introduction to Pipes

Pipes in Angular are a way to transform data in templates. They are used to format data displayed to the user. Angular provides several built-in pipes, such as DatePipe, UpperCasePipe, LowerCasePipe, CurrencyPipe, and PercentPipe.

Why Custom Pipes?

While built-in pipes cover many common scenarios, there are times when you need to create custom pipes to handle specific data transformations that are unique to your application.

Creating a Custom Pipe

Creating a custom pipe involves the following steps:

  1. Generate the Pipe: Use Angular CLI to generate a new pipe.
  2. Implement the Pipe Logic: Define the transformation logic in the pipe class.
  3. Register the Pipe: Ensure the pipe is declared in an Angular module.

Step 1: Generate the Pipe

Use Angular CLI to generate a new pipe:

ng generate pipe custom

This command creates a new file custom.pipe.ts with a basic structure.

Step 2: Implement the Pipe Logic

Edit the generated pipe file to implement the transformation logic. For example, let's create a pipe that converts a string to title case (the first letter of each word is capitalized).

import { Pipe, PipeTransform } from '@angular/core';

@Pipe({
  name: 'titleCase'
})
export class TitleCasePipe implements PipeTransform {
  transform(value: string): string {
    if (!value) return value;
    return value.replace(/\w\S*/g, (txt) => {
      return txt.charAt(0).toUpperCase() + txt.substr(1).toLowerCase();
    });
  }
}

Step 3: Register the Pipe

Ensure the pipe is declared in an Angular module, typically in app.module.ts.

import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AppComponent } from './app.component';
import { TitleCasePipe } from './title-case.pipe';

@NgModule({
  declarations: [
    AppComponent,
    TitleCasePipe
  ],
  imports: [
    BrowserModule
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

Using the Custom Pipe

Once the custom pipe is created and registered, you can use it in your templates just like any other pipe.

<p>{{ 'hello world' | titleCase }}</p>

This will output:

Hello World

Parameterized Pipes

Custom pipes can also accept parameters. For example, let's create a pipe that formats a number with a specified number of decimal places.

import { Pipe, PipeTransform } from '@angular/core';

@Pipe({
  name: 'decimalFormat'
})
export class DecimalFormatPipe implements PipeTransform {
  transform(value: number, decimalPlaces: number): string {
    return value.toFixed(decimalPlaces);
  }
}

Usage in a template:

<p>{{ 1234.5678 | decimalFormat:2 }}</p>

This will output:

1234.57

Pure vs Impure Pipes

  • Pure Pipes: These are stateless and only depend on their input arguments. They are only called when Angular detects a change in the input value.
  • Impure Pipes: These can have side effects and are called on every change detection cycle.

To create an impure pipe, set the pure property to false in the @Pipe decorator.

@Pipe({
  name: 'impurePipe',
  pure: false
})
export class ImpurePipe implements PipeTransform {
  // Implementation
}

Summary

  • Custom Pipes: Allow you to create reusable data transformation logic.
  • Steps to Create: Generate, implement, and register the pipe.
  • Usage: Use custom pipes in templates like built-in pipes.
  • Parameterized Pipes: Accept additional arguments for more flexible transformations.
  • Pure vs Impure Pipes: Pure pipes are stateless, while impure pipes can have side effects and are called more frequently.

By mastering custom pipes, you can significantly enhance the data presentation capabilities of your Angular applications.
