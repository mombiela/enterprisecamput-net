Error Handling in Angular 2+
Introduction to State Management ►

Error handling is a crucial aspect of any application, ensuring that your app can gracefully handle unexpected situations and provide meaningful feedback to users. In this section, we'll cover error handling in Angular 2+ from beginner to advanced levels.

Introduction to Error Handling

Error handling in Angular involves catching and managing errors that occur during the execution of your application. This can include errors in HTTP requests, component logic, and more.

Key Concepts

  • Error Types: Understand the different types of errors (e.g., runtime errors, HTTP errors).
  • Error Handling Strategies: Learn various strategies to handle errors effectively.
  • Angular Error Handling Mechanisms: Explore built-in Angular mechanisms for error handling.

Basic Error Handling

Try-Catch Blocks

The simplest form of error handling in JavaScript is using try-catch blocks. This can be used in Angular components and services.

try {
  // Code that may throw an error
} catch (error) {
  console.error('An error occurred:', error);
}

Handling HTTP Errors

When making HTTP requests using Angular's HttpClient, you can handle errors using the catchError operator from RxJS.

import { HttpClient } from '@angular/common/http';
import { catchError } from 'rxjs/operators';
import { throwError } from 'rxjs';

constructor(private http: HttpClient) {}

getData() {
  this.http.get('https://api.example.com/data')
    .pipe(
      catchError(error => {
        console.error('HTTP Error:', error);
        return throwError(error);
      })
    )
    .subscribe(
      data => console.log('Data:', data),
      error => console.error('Subscription Error:', error)
    );
}

Intermediate Error Handling

Global Error Handling

Angular provides a way to handle errors globally using the ErrorHandler class. You can create a custom error handler by extending this class.

import { ErrorHandler, Injectable } from '@angular/core';

@Injectable()
export class GlobalErrorHandler implements ErrorHandler {
  handleError(error: any): void {
    // Handle the error here
    console.error('Global Error:', error);
  }
}

To use this global error handler, provide it in your app module.

import { NgModule, ErrorHandler } from '@angular/core';
import { GlobalErrorHandler } from './global-error-handler';

@NgModule({
  providers: [{ provide: ErrorHandler, useClass: GlobalErrorHandler }]
})
export class AppModule {}

Custom Error Pages

You can create custom error pages to provide a better user experience when an error occurs.

// error-page.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-error-page',
  template: `<h1>An error occurred</h1><p>Please try again later.</p>`
})
export class ErrorPageComponent {}

Then, configure your routes to display the error page.

import { RouterModule, Routes } from '@angular/router';
import { ErrorPageComponent } from './error-page/error-page.component';

const routes: Routes = [
  // Other routes
  { path: 'error', component: ErrorPageComponent },
  { path: '**', redirectTo: '/error' }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule {}

Advanced Error Handling

Error Interceptors

HTTP interceptors can be used to handle errors globally for all HTTP requests.

import { Injectable } from '@angular/core';
import { HttpInterceptor, HttpRequest, HttpHandler, HttpEvent, HttpErrorResponse } from '@angular/common/http';
import { Observable, throwError } from 'rxjs';
import { catchError } from 'rxjs/operators';

@Injectable()
export class ErrorInterceptor implements HttpInterceptor {
  intercept(req: HttpRequest<any>, next: HttpHandler): Observable<HttpEvent<any>> {
    return next.handle(req).pipe(
      catchError((error: HttpErrorResponse) => {
        console.error('HTTP Interceptor Error:', error);
        return throwError(error);
      })
    );
  }
}

To use the interceptor, provide it in your app module.

import { HTTP_INTERCEPTORS } from '@angular/common/http';
import { ErrorInterceptor } from './error-interceptor';

@NgModule({
  providers: [
    { provide: HTTP_INTERCEPTORS, useClass: ErrorInterceptor, multi: true }
  ]
})
export class AppModule {}

Error Logging Services

For advanced error handling, you might want to log errors to an external service.

import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';

@Injectable()
export class ErrorLoggingService {
  constructor(private http: HttpClient) {}

  logError(error: any): void {
    // Send the error to an external logging service
    this.http.post('https://logging-service.example.com/log', { error })
      .subscribe();
  }
}

You can then use this service in your global error handler.

import { ErrorHandler, Injectable } from '@angular/core';
import { ErrorLoggingService } from './error-logging.service';

@Injectable()
export class GlobalErrorHandler implements ErrorHandler {
  constructor(private errorLoggingService: ErrorLoggingService) {}

  handleError(error: any): void {
    console.error('Global Error:', error);
    this.errorLoggingService.logError(error);
  }
}

Conclusion

Error handling is an essential part of building robust Angular applications. By understanding and implementing various error handling strategies, you can ensure that your application can gracefully handle unexpected situations and provide a better user experience. From basic try-catch blocks to advanced error interceptors and logging services, Angular provides a comprehensive set of tools to manage errors effectively.
