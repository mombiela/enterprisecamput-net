Reactive Forms in Angular provide a model-driven approach to handling form inputs whose values change over time. This approach is more powerful and scalable compared to Template-driven forms, especially for complex forms.

Introduction to Reactive Forms

  • Model-driven approach: Reactive forms are built around observable streams, making it easier to manage complex form interactions.
  • FormControl, FormGroup, and FormArray: Core building blocks for creating reactive forms.
  • Validation: Built-in and custom validators for form controls.

Setting Up Reactive Forms

To use Reactive Forms, you need to import the ReactiveFormsModule in your Angular module.

import { ReactiveFormsModule } from '@angular/forms';

@NgModule({
  imports: [
    ReactiveFormsModule,
    // other imports
  ],
  // other configurations
})
export class AppModule { }

Creating a Basic Reactive Form

Step 1: Import Required Classes

import { Component } from '@angular/core';
import { FormControl, FormGroup } from '@angular/forms';

Step 2: Define the Form Model

@Component({
  selector: 'app-basic-form',
  template: `
    <form [formGroup]="profileForm" (ngSubmit)="onSubmit()">
      <label>
        First Name:
        <input formControlName="firstName">
      </label>
      <label>
        Last Name:
        <input formControlName="lastName">
      </label>
      <button type="submit">Submit</button>
    </form>
  `
})
export class BasicFormComponent {
  profileForm = new FormGroup({
    firstName: new FormControl(''),
    lastName: new FormControl('')
  });

  onSubmit() {
    console.log(this.profileForm.value);
  }
}

Explanation

  • FormGroup: Represents the form as a whole.
  • FormControl: Represents individual form fields.
  • formControlName: Binds the form control to the input element.

Advanced Form Features

Nested FormGroups

You can nest FormGroup instances to create complex forms.

this.profileForm = new FormGroup({
  firstName: new FormControl(''),
  lastName: new FormControl(''),
  address: new FormGroup({
    street: new FormControl(''),
    city: new FormControl(''),
    state: new FormControl(''),
    zip: new FormControl('')
  })
});

FormArray

FormArray is used to manage an array of form controls.

this.profileForm = new FormGroup({
  firstName: new FormControl(''),
  lastName: new FormControl(''),
  aliases: new FormArray([
    new FormControl('')
  ])
});

get aliases() {
  return this.profileForm.get('aliases') as FormArray;
}

addAlias() {
  this.aliases.push(new FormControl(''));
}

Dynamic Form Controls

You can dynamically add or remove form controls.

addAlias() {
  this.aliases.push(new FormControl(''));
}

removeAlias(index: number) {
  this.aliases.removeAt(index);
}

Validation

Built-in Validators

Angular provides several built-in validators like Validators.required, Validators.minLength, etc.

import { Validators } from '@angular/forms';

this.profileForm = new FormGroup({
  firstName: new FormControl('', Validators.required),
  lastName: new FormControl('', Validators.minLength(2))
});

Custom Validators

You can create custom validators to suit your specific needs.

import { AbstractControl, ValidatorFn } from '@angular/forms';

export function forbiddenNameValidator(nameRe: RegExp): ValidatorFn {
  return (control: AbstractControl): {[key: string]: any} | null => {
    const forbidden = nameRe.test(control.value);
    return forbidden ? {'forbiddenName': {value: control.value}} : null;
  };
}

this.profileForm = new FormGroup({
  firstName: new FormControl('', [Validators.required, forbiddenNameValidator(/bob/i)]),
  lastName: new FormControl('')
});

Reactive Forms vs Template-driven Forms

Feature Reactive Forms Template-driven Forms
Setup More complex, requires more setup Simpler, less setup
Form Model Defined in component class Defined in the template
Validation Defined in component class Defined in the template
Scalability More scalable for complex forms Less scalable for complex forms
Testing Easier to test Harder to test

Conclusion

Reactive Forms in Angular provide a robust and scalable way to handle form inputs and validations. By leveraging the power of observables, Reactive Forms offer a more flexible and testable approach compared to Template-driven forms. Whether you're dealing with simple forms or complex nested forms, Reactive Forms can handle it all efficiently.
