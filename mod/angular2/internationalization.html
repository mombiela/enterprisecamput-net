Introduction to Internationalization (i18n)

Internationalization (i18n) is the process of designing and preparing your application to be adaptable to various languages and regions without requiring engineering changes to the source code. Angular provides robust support for i18n, making it easier to create applications that can be translated into different languages.

Key Concepts of i18n

  • Localization (l10n): The process of translating your internationalized application to a specific language and region.
  • Translation Files: Files that contain the translated text for different languages.
  • Angular i18n Tools: Tools provided by Angular to extract, manage, and apply translations.

Setting Up i18n in Angular

Step 1: Mark Text for Translation

Angular provides the i18n attribute to mark text for translation in your templates.

<h1 i18n="@@homeTitle">Welcome to our application!</h1>
<p i18n="@@homeDescription">This is a sample application to demonstrate i18n in Angular.</p>
  • i18n="@@homeTitle": The i18n attribute marks the text for translation.
  • @@homeTitle: An optional custom ID for the translation.

Step 2: Extract Translation Messages

Use Angular CLI to extract the marked text into a translation file.

ng extract-i18n --output-path src/locale

This command generates a messages.xlf file in the specified output path.

Step 3: Translate Messages

Translate the extracted messages into the desired languages. For example, create messages.fr.xlf for French translations.

<trans-unit id="homeTitle" datatype="html">
  <source>Welcome to our application!</source>
  <target>Bienvenue dans notre application!</target>
</trans-unit>
<trans-unit id="homeDescription" datatype="html">
  <source>This is a sample application to demonstrate i18n in Angular.</source>
  <target>Ceci est une application d'exemple pour démontrer i18n dans Angular.</target>
</trans-unit>

Step 4: Configure Angular to Use Translations

Modify angular.json to include the translation files and configure the build options.

"projects": {
  "your-app-name": {
    "i18n": {
      "sourceLocale": "en",
      "locales": {
        "fr": "src/locale/messages.fr.xlf"
      }
    },
    ...
  }
}

Step 5: Build and Serve the Application

Build the application for a specific locale.

ng build --localize

Serve the application for a specific locale.

ng serve --configuration=fr

Advanced i18n Features

Pluralization

Handle plural forms in translations using Angular's ICU message syntax.

<p i18n="@@itemCount">
  {itemCount, plural, 
    =0 {No items}
    =1 {One item}
    other {# items}
  }
</p>

Dynamic Parameters

Include dynamic values in translations.

<p i18n="@@welcomeUser">Welcome, {{ userName }}!</p>

Custom Locale Data

Add custom locale data for specific languages or regions.

import { registerLocaleData } from '@angular/common';
import localeFr from '@angular/common/locales/fr';

registerLocaleData(localeFr, 'fr');

Conclusion

Internationalization in Angular 2+ is a powerful feature that allows you to create applications that can be easily adapted to different languages and regions. By following the steps outlined in this topic, you can mark text for translation, extract and translate messages, and configure your application to use these translations. Advanced features like pluralization and dynamic parameters further enhance the flexibility of your internationalized application.
