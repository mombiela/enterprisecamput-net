What is Angular?
Setting Up the Development Environment ►

Angular is a platform and framework for building single-page client applications using HTML and TypeScript. Developed and maintained by Google, Angular is a complete rewrite from the same team that built AngularJS. Angular 2+ refers to all versions of Angular starting from version 2 and beyond.

Key Features of Angular

  • Component-Based Architecture: Angular applications are built using components, which are the fundamental building blocks of the user interface.
  • TypeScript: Angular is written in TypeScript, which is a superset of JavaScript that adds static types.
  • Dependency Injection: Angular has a powerful dependency injection system that makes it easy to manage and inject dependencies.
  • Reactive Programming with RxJS: Angular uses RxJS for handling asynchronous operations and event-based programs.
  • Modular Development: Angular applications are modular, allowing developers to divide the application into smaller, reusable pieces.
  • Two-Way Data Binding: Angular supports two-way data binding, which means that any changes in the model are reflected in the view and vice versa.
  • Routing: Angular provides a robust routing mechanism to navigate between different views or components.

Basic Concepts in Angular

Components

Components are the building blocks of an Angular application. Each component consists of:

  • Template: Defines the view for the component using HTML.
  • Class: Contains the logic and data for the component, written in TypeScript.
  • Styles: Defines the CSS styles for the component.

Example of a simple Angular component:

import { Component } from '@angular/core';

@Component({
  selector: 'app-hello-world',
  template: `<h1>Hello, World!</h1>`,
  styles: [`h1 { font-family: Lato; }`]
})
export class HelloWorldComponent {
}

Modules

Modules are used to group related components, directives, pipes, and services. Every Angular application has at least one module, the root module.

Example of a basic Angular module:

import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AppComponent } from './app.component';
import { HelloWorldComponent } from './hello-world.component';

@NgModule({
  declarations: [
    AppComponent,
    HelloWorldComponent
  ],
  imports: [
    BrowserModule
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

Services and Dependency Injection

Services are used to share data and logic across multiple components. Angular's dependency injection system makes it easy to inject services into components.

Example of a simple service:

import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root',
})
export class DataService {
  private data: string[] = [];

  addData(item: string) {
    this.data.push(item);
  }

  getData(): string[] {
    return this.data;
  }
}

Injecting the service into a component:

import { Component } from '@angular/core';
import { DataService } from './data.service';

@Component({
  selector: 'app-data',
  template: `
    <div>
      <input #newItem>
      <button (click)="addItem(newItem.value)">Add</button>
      <ul>
        <li *ngFor="let item of items">{{ item }}</li>
      </ul>
    </div>
  `,
})
export class DataComponent {
  items: string[] = [];

  constructor(private dataService: DataService) {}

  addItem(item: string) {
    this.dataService.addData(item);
    this.items = this.dataService.getData();
  }
}

Conclusion

Angular is a powerful and versatile framework for building modern web applications. Its component-based architecture, use of TypeScript, and robust features like dependency injection and reactive programming make it a popular choice among developers. Understanding the basic concepts of Angular, such as components, modules, and services, provides a solid foundation for building scalable and maintainable applications. As you progress through this course, you will gain a deeper understanding of Angular's advanced features and best practices.
