Lazy Loading Modules in Angular 2+
Angular Universal (Server-side Rendering) ►

Lazy loading is a design pattern commonly used in Angular applications to load JavaScript components asynchronously when a specific route is activated. This can significantly improve the performance of your application by reducing the initial load time.

Key Concepts

  • Lazy Loading: Loading modules only when they are needed, rather than at the start of the application.
  • Modules: Angular modules (NgModule) are used to group related components, directives, pipes, and services.
  • Routes: Define the navigation paths in your application. Lazy loading is often implemented using Angular's router.

Why Use Lazy Loading?

  • Performance: Reduces the initial load time of the application by loading only the necessary modules.
  • Scalability: Helps manage large applications by breaking them into smaller, more manageable pieces.
  • Resource Optimization: Loads resources only when needed, which can save bandwidth and improve user experience.

Setting Up Lazy Loading

Step 1: Create a Feature Module

First, create a new feature module that you want to load lazily.

ng generate module feature --route feature --module app.module

This command will create a new module and configure the router to lazy load it.

Step 2: Configure Routes for Lazy Loading

In your app-routing.module.ts, configure the routes to lazy load the feature module.

import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';

const routes: Routes = [
  {
    path: 'feature',
    loadChildren: () => import('./feature/feature.module').then(m => m.FeatureModule)
  }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

Step 3: Define Routes in the Feature Module

In your feature-routing.module.ts, define the routes for the feature module.

import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { FeatureComponent } from './feature.component';

const routes: Routes = [
  { path: '', component: FeatureComponent }
];

@NgModule({
  imports: [RouterModule.forChild(routes)],
  exports: [RouterModule]
})
export class FeatureRoutingModule { }

Step 4: Create the Feature Component

Create the component that will be part of the feature module.

ng generate component feature/feature

Step 5: Update the Feature Module

Update the feature.module.ts to include the new component and routing module.

import { NgModule } from '@angular/core';
import { CommonModule } from '@angular/common';
import { FeatureComponent } from './feature.component';
import { FeatureRoutingModule } from './feature-routing.module';

@NgModule({
  declarations: [FeatureComponent],
  imports: [
    CommonModule,
    FeatureRoutingModule
  ]
})
export class FeatureModule { }

Example: Lazy Loading a Dashboard Module

Let's create a more concrete example by lazy loading a DashboardModule.

Step 1: Generate the Dashboard Module

ng generate module dashboard --route dashboard --module app.module

Step 2: Configure Routes for Lazy Loading

Update app-routing.module.ts:

const routes: Routes = [
  {
    path: 'dashboard',
    loadChildren: () => import('./dashboard/dashboard.module').then(m => m.DashboardModule)
  }
];

Step 3: Define Routes in the Dashboard Module

Update dashboard-routing.module.ts:

const routes: Routes = [
  { path: '', component: DashboardComponent }
];

Step 4: Create the Dashboard Component

ng generate component dashboard/dashboard

Step 5: Update the Dashboard Module

Update dashboard.module.ts:

@NgModule({
  declarations: [DashboardComponent],
  imports: [
    CommonModule,
    DashboardRoutingModule
  ]
})
export class DashboardModule { }

Conclusion

Lazy loading is a powerful feature in Angular that can help improve the performance and scalability of your application. By loading modules only when they are needed, you can reduce the initial load time and optimize resource usage. This guide provided a step-by-step approach to implementing lazy loading in Angular, from creating a feature module to configuring routes and updating modules. With these techniques, you can build more efficient and responsive Angular applications.
