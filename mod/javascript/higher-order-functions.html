Higher-Order Functions
Closures ►

Higher-order functions are a fundamental concept in JavaScript and functional programming. They are functions that can take other functions as arguments or return functions as their result. This topic will guide you from the basics to more advanced uses of higher-order functions.

What are Higher-Order Functions?

  • Definition: A higher-order function is a function that either takes one or more functions as arguments or returns a function as its result.
  • Importance: They allow for more abstract and flexible code, enabling powerful patterns like callbacks, function composition, and more.

Examples of Higher-Order Functions

Basic Example

function higherOrderFunction(callback) {
    return callback();
}

function sayHello() {
    return "Hello, World!";
}

console.log(higherOrderFunction(sayHello)); // Output: Hello, World!

In this example:

  • higherOrderFunction is a higher-order function because it takes another function (callback) as an argument.
  • sayHello is a simple function that returns a string.
  • When higherOrderFunction is called with sayHello as an argument, it executes sayHello and returns its result.

Common Higher-Order Functions in JavaScript

Array Methods

JavaScript provides several built-in higher-order functions for arrays, such as map, filter, and reduce.

map

  • Purpose: Creates a new array populated with the results of calling a provided function on every element in the calling array.
const numbers = [1, 2, 3, 4];
const doubled = numbers.map(num => num * 2);
console.log(doubled); // Output: [2, 4, 6, 8]

filter

  • Purpose: Creates a new array with all elements that pass the test implemented by the provided function.
const numbers = [1, 2, 3, 4];
const evenNumbers = numbers.filter(num => num % 2 === 0);
console.log(evenNumbers); // Output: [2, 4]

reduce

  • Purpose: Executes a reducer function (that you provide) on each element of the array, resulting in a single output value.
const numbers = [1, 2, 3, 4];
const sum = numbers.reduce((accumulator, currentValue) => accumulator + currentValue, 0);
console.log(sum); // Output: 10

Advanced Usage of Higher-Order Functions

Function Composition

  • Definition: Combining two or more functions to produce a new function.
const add = x => y => x + y;
const multiply = x => y => x * y;

const addAndMultiply = (a, b, c) => multiply(add(a)(b))(c);

console.log(addAndMultiply(1, 2, 3)); // Output: 9

Currying

  • Definition: The process of transforming a function that takes multiple arguments into a sequence of functions that each take a single argument.
function curry(f) {
    return function(a) {
        return function(b) {
            return f(a, b);
        };
    };
}

function sum(a, b) {
    return a + b;
}

const curriedSum = curry(sum);
console.log(curriedSum(1)(2)); // Output: 3

Exercises

  1. Basic Exercise: Write a higher-order function that takes a function and a number, and applies the function to the number three times.
function applyThrice(func, num) {
    return func(func(func(num)));
}

function increment(x) {
    return x + 1;
}

console.log(applyThrice(increment, 5)); // Output: 8
  1. Intermediate Exercise: Use map to create a new array that contains the lengths of each string in an array of strings.
const strings = ["hello", "world", "JavaScript"];
const lengths = strings.map(str => str.length);
console.log(lengths); // Output: [5, 5, 10]
  1. Advanced Exercise: Write a function compose that takes two functions and returns a new function that is their composition.
function compose(f, g) {
    return function(x) {
        return f(g(x));
    };
}

const add1 = x => x + 1;
const double = x => x * 2;

const add1ThenDouble = compose(double, add1);
console.log(add1ThenDouble(5)); // Output: 12

Conclusion

Higher-order functions are a powerful feature in JavaScript that enable more abstract and flexible code. By understanding and utilizing higher-order functions, you can write more concise and expressive code. Practice using built-in higher-order functions like map, filter, and reduce, and try creating your own higher-order functions to deepen your understanding.
