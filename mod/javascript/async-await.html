Introduction to Asynchronous Programming

  • Asynchronous Programming: Allows the program to start a potentially time-consuming task and move on to other tasks before the first one finishes.
  • Callbacks: Functions passed as arguments to other functions to be executed later.
  • Promises: Objects representing the eventual completion (or failure) of an asynchronous operation and its resulting value.

Understanding Promises

  • States of a Promise:
    • Pending: Initial state, neither fulfilled nor rejected.
    • Fulfilled: Operation completed successfully.
    • Rejected: Operation failed.
  • Creating a Promise: 
    const myPromise = new Promise((resolve, reject) => {
  // Asynchronous operation
  if (/* operation successful */) {
    resolve('Success!');
  } else {
    reject('Failure!');
  }
});

Introduction to Async/Await

  • Async Functions: Functions declared with the async keyword, which return a Promise.
  • Await Keyword: Used inside an async function to pause execution until a Promise is resolved or rejected.

Using Async/Await

  • Basic Syntax: 
    async function fetchData() {
  try {
    let response = await fetch('https://api.example.com/data');
    let data = await response.json();
    console.log(data);
  } catch (error) {
    console.error('Error:', error);
  }
}
fetchData();
    • async function fetchData() {}: Declares an async function.
    • await fetch('https://api.example.com/data'): Pauses execution until the fetch Promise is resolved.
    • try...catch: Handles any errors that occur during the async operations.

Error Handling with Async/Await

  • Using try...catch: 
    async function getData() {
  try {
    let response = await fetch('https://api.example.com/data');
    if (!response.ok) {
      throw new Error('Network response was not ok');
    }
    let data = await response.json();
    return data;
  } catch (error) {
    console.error('Fetch error:', error);
  }
}

Chaining Async Functions

  • Sequential Execution: 
    async function processData() {
  let data1 = await fetchData1();
  let data2 = await fetchData2(data1);
  let result = await processData(data2);
  return result;
}

Parallel Execution with Async/Await

  • Using Promise.all: 
    async function fetchMultipleData() {
  try {
    let [data1, data2] = await Promise.all([
      fetch('https://api.example.com/data1').then(response => response.json()),
      fetch('https://api.example.com/data2').then(response => response.json())
    ]);
    console.log(data1, data2);
  } catch (error) {
    console.error('Error fetching data:', error);
  }
}
fetchMultipleData();

Advanced Patterns

  • Retrying Failed Requests: 
    async function fetchWithRetry(url, retries = 3) {
  for (let i = 0; i < retries; i++) {
    try {
      let response = await fetch(url);
      if (!response.ok) throw new Error('Network response was not ok');
      return await response.json();
    } catch (error) {
      if (i === retries - 1) throw error;
    }
  }
}

Conclusion

  • Async/Await simplifies working with Promises, making asynchronous code easier to read and write.
  • Error Handling is straightforward with try...catch blocks.
  • Sequential and Parallel Execution can be managed effectively using async/await and Promise.all.
  • Advanced Patterns like retrying failed requests can be implemented to handle more complex scenarios.

By mastering async/await, you can write cleaner, more maintainable asynchronous JavaScript code.
