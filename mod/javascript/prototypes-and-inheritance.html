Prototypes and Inheritance

Introduction

In JavaScript, understanding prototypes and inheritance is crucial for mastering object-oriented programming. This section will cover the basics of prototypes, how inheritance works in JavaScript, and how to use these concepts to create more efficient and organized code.

Prototypes

What is a Prototype?

  • Every JavaScript object has a prototype.
  • A prototype is also an object.
  • Prototypes are the mechanism by which JavaScript objects inherit features from one another.

Prototype Chain

  • Objects can have a prototype, which in turn can have its own prototype, forming a chain.
  • The chain ends when an object has null as its prototype.

Example of Prototype Chain

let animal = {
    eats: true
};

let rabbit = {
    jumps: true
};

rabbit.__proto__ = animal; // setting animal as the prototype of rabbit

console.log(rabbit.eats); // true
console.log(rabbit.jumps); // true

Accessing Prototypes

  • Use Object.getPrototypeOf(obj) to get the prototype of an object.
  • Use Object.setPrototypeOf(obj, prototype) to set the prototype of an object.

Example

let animal = {
    eats: true
};

let rabbit = Object.create(animal); // rabbit inherits from animal

console.log(Object.getPrototypeOf(rabbit) === animal); // true

Inheritance

What is Inheritance?

  • Inheritance is a feature that allows one object to inherit properties and methods from another object.
  • This helps in reusing code and creating a hierarchical structure.

Classical vs Prototypal Inheritance

  • Classical inheritance involves classes and is common in languages like Java and C++.
  • Prototypal inheritance involves objects and is more flexible.

Creating an Inheritance Chain

  • Use Object.create() to create a new object with a specified prototype.

Example

let animal = {
    eats: true,
    walk() {
        console.log("Animal walks");
    }
};

let rabbit = Object.create(animal);
rabbit.jumps = true;

rabbit.walk(); // Animal walks
console.log(rabbit.eats); // true
console.log(rabbit.jumps); // true

Constructor Functions and Prototypes

Constructor Functions

  • Constructor functions are used to create multiple similar objects.
  • Use the new keyword to create an instance of an object.

Example

function Animal(name) {
    this.name = name;
    this.eats = true;
}

Animal.prototype.walk = function() {
    console.log(this.name + " walks");
};

let rabbit = new Animal("Rabbit");
rabbit.walk(); // Rabbit walks

Prototype Property

  • Every function in JavaScript has a prototype property.
  • This property is used to attach methods and properties that should be inherited by instances of the constructor function.

Example

function Animal(name) {
    this.name = name;
}

Animal.prototype.eats = true;

let rabbit = new Animal("Rabbit");
console.log(rabbit.eats); // true

ES6 Classes

Introduction to Classes

  • ES6 introduced classes to make it easier to write and understand inheritance in JavaScript.
  • Classes are syntactical sugar over JavaScript's existing prototype-based inheritance.

Example

class Animal {
    constructor(name) {
        this.name = name;
        this.eats = true;
    }

    walk() {
        console.log(this.name + " walks");
    }
}

let rabbit = new Animal("Rabbit");
rabbit.walk(); // Rabbit walks

Inheritance with Classes

  • Use the extends keyword to create a class that inherits from another class.

Example

class Animal {
    constructor(name) {
        this.name = name;
    }

    walk() {
        console.log(this.name + " walks");
    }
}

class Rabbit extends Animal {
    constructor(name, color) {
        super(name); // call the parent constructor
        this.color = color;
    }

    jump() {
        console.log(this.name + " jumps");
    }
}

let rabbit = new Rabbit("Rabbit", "white");
rabbit.walk(); // Rabbit walks
rabbit.jump(); // Rabbit jumps

Summary

  • Prototypes are fundamental to understanding JavaScript's inheritance model.
  • Objects inherit properties and methods from their prototypes.
  • Constructor functions and the new keyword are used to create instances with shared properties and methods.
  • ES6 classes provide a more intuitive syntax for creating and managing inheritance in JavaScript.
  • Understanding these concepts allows for more efficient and organized code, leveraging the power of object-oriented programming in JavaScript.
