Handling HTTP Methods (GET, POST, PUT, DELETE) in Spring Boot

Introduction

In this section, we will cover how to handle the primary HTTP methods (GET, POST, PUT, DELETE) using Spring Boot. Understanding these methods is crucial for building RESTful web services. We will start with the basics and gradually move to more advanced concepts.

GET Method

The GET method is used to retrieve information from the server. It is idempotent, meaning multiple identical requests should have the same effect as a single request.

Key Concepts

  • Used to fetch data.
  • Should not alter the state of the server.
  • Can be cached.

Example

@RestController
@RequestMapping("/api")
public class ExampleController {

    @GetMapping("/items")
    public List<Item> getAllItems() {
        // Logic to retrieve items
        return itemService.getAllItems();
    }
}

In this example, the @GetMapping annotation maps HTTP GET requests to the getAllItems method.

POST Method

The POST method is used to submit data to be processed to a specified resource. It often results in a change in server state or side effects.

Key Concepts

  • Used to create new resources.
  • Can alter the state of the server.
  • Not idempotent.

Example

@RestController
@RequestMapping("/api")
public class ExampleController {

    @PostMapping("/items")
    public Item createItem(@RequestBody Item item) {
        // Logic to create an item
        return itemService.createItem(item);
    }
}

Here, the @PostMapping annotation maps HTTP POST requests to the createItem method. The @RequestBody annotation is used to bind the request body to the method parameter.

PUT Method

The PUT method is used to update an existing resource or create a new resource if it does not exist.

Key Concepts

  • Used to update existing resources.
  • Can also create resources if they do not exist.
  • Idempotent.

Example

@RestController
@RequestMapping("/api")
public class ExampleController {

    @PutMapping("/items/{id}")
    public Item updateItem(@PathVariable Long id, @RequestBody Item item) {
        // Logic to update an item
        return itemService.updateItem(id, item);
    }
}

In this example, the @PutMapping annotation maps HTTP PUT requests to the updateItem method. The @PathVariable annotation is used to extract the id from the URL.

DELETE Method

The DELETE method is used to delete a specified resource.

Key Concepts

  • Used to delete resources.
  • Idempotent.

Example

@RestController
@RequestMapping("/api")
public class ExampleController {

    @DeleteMapping("/items/{id}")
    public void deleteItem(@PathVariable Long id) {
        // Logic to delete an item
        itemService.deleteItem(id);
    }
}

Here, the @DeleteMapping annotation maps HTTP DELETE requests to the deleteItem method.

Summary

In this section, we have covered the four primary HTTP methods (GET, POST, PUT, DELETE) and how to handle them using Spring Boot. Each method serves a specific purpose in RESTful web services:

  • GET: Retrieve data.
  • POST: Create new resources.
  • PUT: Update existing resources or create new ones.
  • DELETE: Remove resources.

Understanding these methods and their appropriate use cases is essential for building robust and efficient web services.
