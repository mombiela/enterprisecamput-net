Introduction

Spring Boot simplifies the configuration of Spring applications by providing a wide range of properties and configuration options. This section will cover how to use these properties and configurations to customize your Spring Boot application.

Key Concepts

  • Application Properties: Centralized configuration using application.properties or application.yml.
  • External Configuration: Loading properties from external sources.
  • Profiles: Environment-specific configurations.
  • Configuration Properties: Binding properties to Java objects.
  • Custom Properties: Defining and using custom properties.

Application Properties

Spring Boot allows you to define properties in a centralized file, typically application.properties or application.yml.

Using `application.properties`

The application.properties file is a simple key-value pair configuration file.

server.port=8081
spring.datasource.url=jdbc:mysql://localhost:3306/mydb
spring.datasource.username=root
spring.datasource.password=secret

Using `application.yml`

The application.yml file uses YAML format, which is more readable for hierarchical data.

server:
  port: 8081

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mydb
    username: root
    password: secret

External Configuration

Spring Boot can load properties from external sources such as environment variables, command-line arguments, and external configuration files.

Environment Variables

You can set environment variables to override properties defined in application.properties.

export SERVER_PORT=8082

Command-Line Arguments

You can pass properties as command-line arguments when starting your Spring Boot application.

java -jar myapp.jar --server.port=8083

Profiles

Profiles allow you to define different configurations for different environments (e.g., development, testing, production).

Defining Profiles

You can define profile-specific properties files, such as application-dev.properties and application-prod.properties.

# application-dev.properties
server.port=8081
spring.datasource.url=jdbc:mysql://localhost:3306/devdb

# application-prod.properties
server.port=8080
spring.datasource.url=jdbc:mysql://localhost:3306/proddb

Activating Profiles

You can activate profiles using the spring.profiles.active property.

# application.properties
spring.profiles.active=dev

Configuration Properties

Spring Boot allows you to bind properties to Java objects using the @ConfigurationProperties annotation.

Example

Define a configuration properties class:

import org.springframework.boot.context.properties.ConfigurationProperties;
import org.springframework.stereotype.Component;

@Component
@ConfigurationProperties(prefix = "app")
public class AppProperties {
    private String name;
    private String description;

    // getters and setters
}

Define properties in application.properties:

app.name=MyApp
app.description=This is my application.

Use the properties in your application:

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class AppController {

    @Autowired
    private AppProperties appProperties;

    @GetMapping("/info")
    public String getAppInfo() {
        return appProperties.getName() + ": " + appProperties.getDescription();
    }
}

Custom Properties

You can define and use custom properties in your Spring Boot application.

Defining Custom Properties

Add custom properties to application.properties:

custom.property1=value1
custom.property2=value2

Accessing Custom Properties

Use the @Value annotation to inject custom properties:

import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;

@Component
public class CustomProperties {

    @Value("${custom.property1}")
    private String property1;

    @Value("${custom.property2}")
    private String property2;

    // getters and setters
}

Conclusion

Spring Boot provides a flexible and powerful way to manage application properties and configurations. By understanding how to use application.properties or application.yml, external configurations, profiles, configuration properties, and custom properties, you can effectively customize and manage your Spring Boot applications for different environments and requirements.
