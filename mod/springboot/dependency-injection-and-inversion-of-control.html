Introduction to Dependency Injection (DI)

Dependency Injection (DI) is a design pattern used to implement IoC, allowing the creation of dependent objects outside of a class and providing those objects to a class in various ways.

  • Definition: Dependency Injection is a technique where an object receives other objects that it depends on.
  • Purpose: To decouple the creation of a dependency from the class that uses it, making the code more modular and testable.

Types of Dependency Injection

  1. Constructor Injection: Dependencies are provided through a class constructor.
  2. Setter Injection: Dependencies are provided through setter methods.
  3. Field Injection: Dependencies are injected directly into fields.

Example: Constructor Injection

public class Service {
    private final Repository repository;

    // Constructor Injection
    public Service(Repository repository) {
        this.repository = repository;
    }

    public void performAction() {
        repository.action();
    }
}

Inversion of Control (IoC)

Inversion of Control is a principle in software engineering where the control of objects or portions of a program is transferred to a container or framework.

  • Definition: IoC is a design principle in which the control flow of a program is inverted compared to traditional programming.
  • Purpose: To increase modularity and make the code easier to manage and test.

IoC in Spring Boot

Spring Boot uses the IoC principle to manage the lifecycle and dependencies of beans.

  • Bean: An object that is instantiated, assembled, and managed by a Spring IoC container.
  • IoC Container: The core of the Spring Framework, responsible for instantiating, configuring, and assembling beans.

Example: IoC Container in Spring Boot

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.ApplicationContext;

@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        ApplicationContext context = SpringApplication.run(Application.class, args);
        Service service = context.getBean(Service.class);
        service.performAction();
    }
}

Combining DI and IoC in Spring Boot

Spring Boot leverages both DI and IoC to manage dependencies and the lifecycle of beans.

Example: Full Implementation

  1. Define a Repository Interface and Implementation:

    public interface Repository {
    void action();
}

@Service
public class RepositoryImpl implements Repository {
    @Override
    public void action() {
        System.out.println("Action performed!");
    }
}

  2. Define a Service Class with Constructor Injection:

    @Service
public class Service {
    private final Repository repository;

    @Autowired
    public Service(Repository repository) {
        this.repository = repository;
    }

    public void performAction() {
        repository.action();
    }
}

  3. Run the Application:

    @SpringBootApplication
public class Application {
    public static void main(String[] args) {
        ApplicationContext context = SpringApplication.run(Application.class, args);
        Service service = context.getBean(Service.class);
        service.performAction();
    }
}

Advanced Topics

Qualifiers and Primary Beans

When multiple beans of the same type exist, you can use @Qualifier or @Primary to specify which bean should be injected.

  • @Qualifier: Used to specify which bean to inject when multiple beans of the same type exist.
  • @Primary: Used to designate a bean as the primary candidate for injection.

Example: Using @Qualifier

@Service
@Qualifier("repository1")
public class RepositoryImpl1 implements Repository {
    @Override
    public void action() {
        System.out.println("Repository1 action performed!");
    }
}

@Service
@Qualifier("repository2")
public class RepositoryImpl2 implements Repository {
    @Override
    public void action() {
        System.out.println("Repository2 action performed!");
    }
}

@Service
public class Service {
    private final Repository repository;

    @Autowired
    public Service(@Qualifier("repository1") Repository repository) {
        this.repository = repository;
    }

    public void performAction() {
        repository.action();
    }
}

Example: Using @Primary

@Service
@Primary
public class PrimaryRepositoryImpl implements Repository {
    @Override
    public void action() {
        System.out.println("Primary repository action performed!");
    }
}

@Service
public class Service {
    private final Repository repository;

    @Autowired
    public Service(Repository repository) {
        this.repository = repository;
    }

    public void performAction() {
        repository.action();
    }
}

Conclusion

Dependency Injection and Inversion of Control are fundamental concepts in Spring Boot that help in creating loosely coupled, modular, and testable code. By understanding and applying these principles, you can leverage the full power of the Spring Framework to build robust and maintainable applications.
