Pagination and Sorting in Spring Boot
Introduction to Spring Security ►

Pagination and sorting are essential features for managing large datasets in web applications. They help in breaking down large data into manageable chunks and ordering the data in a meaningful way. In this section, we will cover how to implement pagination and sorting in a Spring Boot application.

Introduction to Pagination and Sorting

  • Pagination: The process of dividing a dataset into discrete pages.
  • Sorting: Arranging data in a particular order (ascending or descending) based on one or more fields.

Setting Up the Project

Before diving into pagination and sorting, ensure you have a Spring Boot project set up. You can create a new Spring Boot project using Spring Initializr or your preferred IDE.

Dependencies

Add the following dependencies to your pom.xml file:

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>runtime</scope>
</dependency>

Configuring the Data Model

Let's create a simple entity class Product to work with.

@Entity
public class Product {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String name;
    private Double price;

    // Getters and Setters
}

Creating the Repository

Spring Data JPA provides built-in support for pagination and sorting. Create a repository interface for the Product entity.

public interface ProductRepository extends JpaRepository<Product, Long> {
}

Implementing Pagination

To implement pagination, you need to use the Pageable interface provided by Spring Data JPA.

Example: Fetching a Page of Products

@RestController
@RequestMapping("/products")
public class ProductController {

    @Autowired
    private ProductRepository productRepository;

    @GetMapping
    public Page<Product> getProducts(@RequestParam int page, @RequestParam int size) {
        Pageable pageable = PageRequest.of(page, size);
        return productRepository.findAll(pageable);
    }
}

Explanation

  • @RequestParam int page: The page number to fetch.
  • @RequestParam int size: The size of the page.
  • PageRequest.of(page, size): Creates a Pageable object with the specified page number and size.
  • productRepository.findAll(pageable): Fetches the page of products.

Implementing Sorting

Sorting can be easily added to the pagination logic using the Sort class.

Example: Fetching Sorted Products

@GetMapping("/sorted")
public Page<Product> getSortedProducts(@RequestParam int page, @RequestParam int size, @RequestParam String sortBy) {
    Pageable pageable = PageRequest.of(page, size, Sort.by(sortBy));
    return productRepository.findAll(pageable);
}

Explanation

  • @RequestParam String sortBy: The field to sort by.
  • Sort.by(sortBy): Creates a Sort object to specify the sorting field.

Combining Pagination and Sorting

You can combine pagination and sorting in a single request.

Example: Fetching a Page of Sorted Products

@GetMapping("/paged-sorted")
public Page<Product> getPagedSortedProducts(@RequestParam int page, @RequestParam int size, @RequestParam String sortBy, @RequestParam String direction) {
    Sort.Direction sortDirection = Sort.Direction.fromString(direction);
    Pageable pageable = PageRequest.of(page, size, Sort.by(sortDirection, sortBy));
    return productRepository.findAll(pageable);
}

Explanation

  • @RequestParam String direction: The direction of sorting (ASC or DESC).
  • Sort.Direction.fromString(direction): Converts the direction string to a Sort.Direction object.
  • Sort.by(sortDirection, sortBy): Creates a Sort object with the specified direction and field.

Conclusion

In this section, we covered the basics of implementing pagination and sorting in a Spring Boot application using Spring Data JPA. We learned how to:

  • Set up the project and dependencies.
  • Create a data model and repository.
  • Implement pagination and sorting.
  • Combine pagination and sorting in a single request.

These features are crucial for building efficient and user-friendly web applications that handle large datasets. By mastering pagination and sorting, you can significantly improve the performance and usability of your applications.
