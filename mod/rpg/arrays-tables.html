Introduction to Arrays and Tables in RPG

Arrays and tables are fundamental data structures in RPG (Report Program Generator) that allow you to store and manipulate collections of data efficiently. Understanding how to use these structures is crucial for writing effective and optimized RPG programs.

Arrays in RPG

Arrays are used to store multiple values of the same type in a single variable. Each value in an array is accessed using an index.

Defining Arrays

To define an array in RPG, you use the D specification.

Dcl-S myArray Char(10) Dim(5);
  • Dcl-S: Declares a standalone variable.
  • myArray: The name of the array.
  • Char(10): Specifies that each element in the array is a character string of length 10.
  • Dim(5): Specifies that the array has 5 elements.

Accessing Array Elements

You can access and modify elements in an array using their index.

myArray(1) = 'First';
myArray(2) = 'Second';
myArray(3) = 'Third';
myArray(4) = 'Fourth';
myArray(5) = 'Fifth';

Iterating Through Arrays

You can use a loop to iterate through the elements of an array.

For i = 1 to %Elem(myArray);
    Dsply myArray(i);
EndFor;
  • For i = 1 to %Elem(myArray);: Loops from 1 to the number of elements in the array.
  • Dsply myArray(i);: Displays each element of the array.

Tables in RPG

Tables, also known as data structures, are used to store complex data types and can include multiple fields of different types.

Defining Tables

To define a table, you use the D specification with a data structure.

Dcl-DS myTable Dim(5);
    name Char(10);
    age  Int(3);
End-DS;
  • Dcl-DS: Declares a data structure.
  • myTable: The name of the table.
  • Dim(5): Specifies that the table has 5 elements.
  • name Char(10);: A field in the table that stores a character string of length 10.
  • age Int(3);: A field in the table that stores an integer of length 3.

Accessing Table Elements

You can access and modify elements in a table using their index and field name.

myTable(1).name = 'Alice';
myTable(1).age = 30;
myTable(2).name = 'Bob';
myTable(2).age = 25;

Iterating Through Tables

You can use a loop to iterate through the elements of a table.

For i = 1 to %Elem(myTable);
    Dsply myTable(i).name + ' is ' + %Char(myTable(i).age) + ' years old.';
EndFor;

Comparing Arrays and Tables

| Feature | Arrays | Tables | |---------------|---------------------------------|---------------------------------| | Definition | Single data type elements | Multiple fields of different types | | Access | By index | By index and field name | | Use Case | Simple lists | Complex data structures |

Conclusion

Arrays and tables are powerful tools in RPG programming. Arrays are ideal for storing lists of simple data types, while tables are better suited for complex data structures with multiple fields. Mastering these data structures will enhance your ability to write efficient and organized RPG programs.

By understanding how to define, access, and iterate through arrays and tables, you can handle a wide range of programming tasks more effectively. Practice using these structures in different scenarios to become proficient in their application.
