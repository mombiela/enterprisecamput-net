Introduction

Binding directories are a crucial concept in RPG (Report Program Generator) programming, especially when dealing with modular programming and service programs. They help manage and organize the various modules and service programs that an RPG application might use.

What is a Binding Directory?

A binding directory is a collection of references to modules and service programs. When you create or update a program or service program, the binding directory helps the binder (the component that links modules together) locate the necessary modules and service programs.

Key Concepts

  • Modules: Individual pieces of compiled code that can be linked together to form a program.
  • Service Programs: Collections of modules that can be used by multiple programs.
  • Binder: The component that links modules and service programs together to create a runnable program.

Why Use Binding Directories?

Binding directories simplify the process of managing dependencies between different parts of an application. They allow you to:

  • Reuse code efficiently.
  • Simplify the maintenance of large applications.
  • Improve the modularity and organization of your codebase.

Creating and Using Binding Directories

Creating a Binding Directory

You can create a binding directory using the CRTBNDDIR (Create Binding Directory) command in the IBM i operating system.

CRTBNDDIR BNDDIR(MYLIB/MYBNDDIR)

This command creates a binding directory named MYBNDDIR in the library MYLIB.

Adding Entries to a Binding Directory

Once you have created a binding directory, you can add entries to it using the ADDBNDDIRE (Add Binding Directory Entry) command.

ADDBNDDIRE BNDDIR(MYLIB/MYBNDDIR) OBJ((MYLIB/MYMODULE *MODULE) (MYLIB/MYSRVPGM *SRVPGM))

This command adds a module named MYMODULE and a service program named MYSRVPGM to the binding directory MYBNDDIR.

Using a Binding Directory in a Program

When compiling an RPG program, you can specify the binding directory to use with the BNDDIR keyword in the H specification or via the CRTRPGMOD and CRTPGM commands.

Using H Specification

H BNDDIR('MYLIB/MYBNDDIR')

Using CRTRPGMOD and CRTPGM Commands

CRTRPGMOD MODULE(MYLIB/MYMODULE) SRCFILE(MYLIB/QRPGLESRC)
CRTPGM PGM(MYLIB/MYPROGRAM) BNDDIR(MYLIB/MYBNDDIR)

Example: Modular RPG Program with Binding Directory

Step 1: Create Modules

Create two simple modules, ModuleA and ModuleB.

ModuleA (ModuleA.rpgle)

**FREE
Dcl-Proc ModuleA;
  Dcl-Pi *N;
    InputValue Int(10);
  End-Pi;

  Dsply ('ModuleA: ' + %Char(InputValue));
End-Proc;

ModuleB (ModuleB.rpgle)

**FREE
Dcl-Proc ModuleB;
  Dcl-Pi *N;
    InputValue Int(10);
  End-Pi;

  Dsply ('ModuleB: ' + %Char(InputValue));
End-Proc;

Step 2: Create a Binding Directory

Create a binding directory and add the modules to it.

CRTBNDDIR BNDDIR(MYLIB/MYBNDDIR)
ADDBNDDIRE BNDDIR(MYLIB/MYBNDDIR) OBJ((MYLIB/MODULEA *MODULE) (MYLIB/MODULEB *MODULE))

Step 3: Create a Main Program

Create a main program that uses the modules.

MainProgram (MainProgram.rpgle)

**FREE
H BNDDIR('MYLIB/MYBNDDIR')

Dcl-Proc Main;
  Dcl-Pi *N;
  End-Pi;

  ModuleA(10);
  ModuleB(20);
End-Proc;

Step 4: Compile and Run

Compile the modules and the main program.

CRTRPGMOD MODULE(MYLIB/MODULEA) SRCFILE(MYLIB/QRPGLESRC)
CRTRPGMOD MODULE(MYLIB/MODULEB) SRCFILE(MYLIB/QRPGLESRC)
CRTRPGMOD MODULE(MYLIB/MAINPROGRAM) SRCFILE(MYLIB/QRPGLESRC)
CRTPGM PGM(MYLIB/MAINPROGRAM) MODULE(MYLIB/MAINPROGRAM) BNDDIR(MYLIB/MYBNDDIR)

Run the main program to see the output.

CALL PGM(MYLIB/MAINPROGRAM)

Conclusion

Binding directories are a powerful tool in RPG programming that help manage and organize modules and service programs. By using binding directories, you can improve the modularity, maintainability, and reusability of your code. This topic covered the basics of creating, managing, and using binding directories, along with a practical example to illustrate their use.
