Introduction

In this section, we will explore file handling and data structures in RPG (Report Program Generator). Understanding these concepts is crucial for managing data efficiently and writing robust programs. We will start with the basics of file handling and gradually move to more advanced data structures.

File Handling

Basics of File Handling

File handling in RPG involves reading from and writing to files. RPG supports various file types, including physical files, logical files, and display files.

  • Physical Files: These are the actual files that store data.
  • Logical Files: These provide a different view or order of the data stored in physical files.
  • Display Files: Used for creating user interfaces.

Reading from a File

To read data from a file, you need to define the file in the File Specification (F-spec) and use the READ operation.

FMYFILE    IF   E           K DISK

DName            S             50A

C     READ      MYFILE
C     IF        %EOF(MYFILE)
C     ELSE
C       EVAL    Name = MYFILE.Name
C     ENDIF
C     *INLR     = *ON
  • F-spec: Defines the file and its attributes.
  • READ: Reads a record from the file.
  • %EOF: Checks if the end of the file has been reached.

Writing to a File

Writing data to a file involves using the WRITE operation.

FMYFILE    O   E           K DISK

DName            S             50A

C     EVAL      MYFILE.Name = 'John Doe'
C     WRITE     MYFILE
C     *INLR     = *ON
  • WRITE: Writes a record to the file.

Updating a File

To update a record in a file, use the CHAIN and UPDATE operations.

FMYFILE    UF   E           K DISK

DName            S             50A

C     CHAIN     'John Doe' MYFILE
C     IF        %FOUND(MYFILE)
C       EVAL    MYFILE.Name = 'Jane Doe'
C       UPDATE  MYFILE
C     ENDIF
C     *INLR     = *ON
  • CHAIN: Retrieves a record by key.
  • UPDATE: Updates the retrieved record.

Data Structures

Introduction to Data Structures

Data structures in RPG are used to group related data together. They can be simple or complex, depending on the requirements.

Defining Data Structures

A data structure is defined using the D-spec.

DNameDS          DS
D  FirstName                   25A
D  LastName                    25A
D  Age                         3P 0
  • DS: Indicates a data structure.
  • A: Alphanumeric data type.
  • P: Packed decimal data type.

Using Data Structures

Data structures can be used to simplify data manipulation.

DNameDS          DS
D  FirstName                   25A
D  LastName                    25A
D  Age                         3P 0

C     EVAL      NameDS.FirstName = 'John'
C     EVAL      NameDS.LastName = 'Doe'
C     EVAL      NameDS.Age = 30

C     DSPLY     NameDS.FirstName + ' ' + NameDS.LastName
C     *INLR     = *ON

Nested Data Structures

Data structures can be nested to represent more complex data.

DAddressDS       DS
D  Street                     50A
D  City                       25A
D  ZipCode                    10A

DPersonDS        DS
D  FirstName                   25A
D  LastName                    25A
D  Age                         3P 0
D  Address                     LIKE(AddressDS)
  • LIKE: Used to define a subfield with the same attributes as another field.

Arrays in Data Structures

Arrays can be used within data structures to handle multiple values.

DGradesDS        DS
D  StudentName                25A
D  Grades                     5P 0 DIM(5)

C     EVAL      GradesDS.StudentName = 'John Doe'
C     EVAL      GradesDS.Grades(1) = 85
C     EVAL      GradesDS.Grades(2) = 90
C     EVAL      GradesDS.Grades(3) = 88

C     DSPLY     GradesDS.StudentName
C     DSPLY     GradesDS.Grades(1)
C     *INLR     = *ON

Conclusion

In this section, we covered the essentials of file handling and data structures in RPG. We learned how to read, write, and update files, as well as how to define and use data structures. Mastering these concepts will enable you to handle data more efficiently and write more organized and maintainable code. As you progress, try to implement these concepts in your projects to solidify your understanding.
