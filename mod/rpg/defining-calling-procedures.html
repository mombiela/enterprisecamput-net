In this section, we will explore how to define and call procedures in RPG (Report Program Generator). Procedures are essential for structuring your code, making it more readable, and reusing code efficiently.

What is a Procedure?

A procedure in RPG is a block of code that performs a specific task. Procedures help in:

  • Organizing code into manageable sections.
  • Reusing code across different parts of the program.
  • Enhancing readability and maintainability.

Defining a Procedure

To define a procedure in RPG, you use the DCL-PROC and END-PROC keywords. Here is a basic structure:

DCL-PROC MyProcedure;
   // Procedure logic goes here
END-PROC;

Example: Simple Procedure

Let's define a simple procedure that prints "Hello, World!".

DCL-PROC PrintHello;
   DCL-S msg CHAR(20) INZ('Hello, World!');
   DSPLY msg;
END-PROC;

In this example:

  • DCL-PROC PrintHello; starts the procedure definition.
  • DCL-S msg CHAR(20) INZ('Hello, World!'); declares a variable msg initialized with "Hello, World!".
  • DSPLY msg; displays the message.
  • END-PROC; ends the procedure.

Calling a Procedure

To call a procedure, simply use its name followed by parentheses.

Example: Calling the Procedure

Here is how you call the PrintHello procedure:

PrintHello();

This will execute the code within the PrintHello procedure and display "Hello, World!".

Procedures with Parameters

Procedures can also accept parameters, allowing you to pass data into them.

Example: Procedure with Parameters

Let's create a procedure that takes a name as a parameter and prints a greeting.

DCL-PROC Greet;
   DCL-PI Greet;
      name CHAR(20);
   END-PI;

   DCL-S msg CHAR(40);
   msg = 'Hello, ' + %TRIM(name) + '!';
   DSPLY msg;
END-PROC;

In this example:

  • DCL-PI Greet; starts the procedure interface definition.
  • name CHAR(20); declares a parameter name of type CHAR(20).
  • END-PI; ends the procedure interface definition.
  • DCL-S msg CHAR(40); declares a variable msg.
  • msg = 'Hello, ' + %TRIM(name) + '!'; constructs the greeting message.
  • DSPLY msg; displays the message.

Example: Calling the Procedure with Parameters

Here is how you call the Greet procedure with a parameter:

Greet('Alice');

This will display "Hello, Alice!".

Procedures with Return Values

Procedures can also return values. This is useful when you need to perform calculations or retrieve data.

Example: Procedure with Return Value

Let's create a procedure that adds two numbers and returns the result.

DCL-PROC AddNumbers;
   DCL-PI AddNumbers INT(10);
      num1 INT(10);
      num2 INT(10);
   END-PI;

   RETURN num1 + num2;
END-PROC;

In this example:

  • DCL-PI AddNumbers INT(10); defines the procedure interface with a return type of INT(10).
  • num1 INT(10); and num2 INT(10); are parameters.
  • RETURN num1 + num2; returns the sum of num1 and num2.

Example: Calling the Procedure with Return Value

Here is how you call the AddNumbers procedure and use its return value:

DCL-S result INT(10);
result = AddNumbers(5, 10);
DSPLY result;

This will display 15.

Conclusion

Procedures are a powerful feature in RPG that help you organize, reuse, and maintain your code efficiently. In this section, we covered:

  • Basic procedure definition and calling.
  • Procedures with parameters.
  • Procedures with return values.

By mastering procedures, you can write more modular and maintainable RPG programs. Practice defining and calling procedures with different parameters and return types to solidify your understanding.
