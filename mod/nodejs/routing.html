Routing is a fundamental concept in web development, allowing you to define how your application responds to different HTTP requests. In Node.js, routing is typically handled using frameworks like Express.js. This section will guide you from the basics of routing to more advanced techniques.

Introduction to Routing

Routing is the process of determining how an application responds to a client request to a particular endpoint, which is a URI (or path) and a specific HTTP request method (GET, POST, etc.).

Key Concepts

  • Endpoint: A specific URL path and HTTP method combination.
  • HTTP Methods: Common methods include GET, POST, PUT, DELETE, etc.
  • Route Handler: A function that executes when a specific route is matched.

Setting Up Express.js

To handle routing in Node.js, we will use the Express.js framework. First, let's set up a basic Express application.

Installation

npm install express

Basic Setup

const express = require('express');
const app = express();
const port = 3000;

app.listen(port, () => {
  console.log(`Server is running on http://localhost:${port}`);
});

Basic Routing

Defining Routes

Routes are defined using methods of the app object that correspond to HTTP methods. For example:

app.get('/', (req, res) => {
  res.send('Hello, World!');
});

app.post('/submit', (req, res) => {
  res.send('Form submitted!');
});

Route Parameters

Route parameters are named URL segments used to capture values at specific positions in the URL.

app.get('/user/:id', (req, res) => {
  const userId = req.params.id;
  res.send(`User ID: ${userId}`);
});

Query Parameters

Query parameters are used to pass data to the server via the URL.

app.get('/search', (req, res) => {
  const query = req.query.q;
  res.send(`Search query: ${query}`);
});

Advanced Routing

Middleware

Middleware functions are functions that have access to the request object (req), the response object (res), and the next middleware function in the applicationâ€™s request-response cycle.

const logger = (req, res, next) => {
  console.log(`${req.method} ${req.url}`);
  next();
};

app.use(logger);

Route Groups

Route groups allow you to group routes that share a common path prefix.

const router = express.Router();

router.get('/profile', (req, res) => {
  res.send('User profile');
});

router.get('/settings', (req, res) => {
  res.send('User settings');
});

app.use('/user', router);

Error Handling

Error-handling middleware functions are defined in the same way as other middleware functions, except they have four arguments instead of three: err, req, res, and next.

app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send('Something broke!');
});

Summary

Routing is a crucial part of any web application, defining how your application responds to various HTTP requests. In Node.js, Express.js provides a simple and powerful way to handle routing. We covered:

  • Basic routing with different HTTP methods.
  • Using route and query parameters.
  • Advanced techniques like middleware, route groups, and error handling.

By mastering these concepts, you can build robust and scalable web applications with Node.js and Express.js.
