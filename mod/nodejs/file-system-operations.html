Introduction

File system operations are a crucial part of many applications, allowing you to read, write, and manipulate files and directories. Node.js provides a built-in module called fs to handle these operations efficiently.

Basic File System Operations

Reading Files

Reading files is one of the most common operations. Node.js provides both synchronous and asynchronous methods to read files.

Asynchronous Reading

Asynchronous methods are non-blocking, meaning the program can continue executing while the file is being read.

const fs = require('fs');

fs.readFile('example.txt', 'utf8', (err, data) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log(data);
});

Synchronous Reading

Synchronous methods block the execution until the operation is complete.

const fs = require('fs');

try {
  const data = fs.readFileSync('example.txt', 'utf8');
  console.log(data);
} catch (err) {
  console.error(err);
}

Writing Files

Writing files can also be done asynchronously or synchronously.

Asynchronous Writing

const fs = require('fs');

const content = 'This is some content to write into the file.';

fs.writeFile('example.txt', content, 'utf8', (err) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log('File has been written');
});

Synchronous Writing

const fs = require('fs');

const content = 'This is some content to write into the file.';

try {
  fs.writeFileSync('example.txt', content, 'utf8');
  console.log('File has been written');
} catch (err) {
  console.error(err);
}

Appending to Files

Appending data to an existing file can be done using fs.appendFile and fs.appendFileSync.

Asynchronous Appending

const fs = require('fs');

const content = '\nThis is additional content.';

fs.appendFile('example.txt', content, 'utf8', (err) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log('Content has been appended');
});

Synchronous Appending

const fs = require('fs');

const content = '\nThis is additional content.';

try {
  fs.appendFileSync('example.txt', content, 'utf8');
  console.log('Content has been appended');
} catch (err) {
  console.error(err);
}

Deleting Files

Deleting files can be done using fs.unlink and fs.unlinkSync.

Asynchronous Deleting

const fs = require('fs');

fs.unlink('example.txt', (err) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log('File has been deleted');
});

Synchronous Deleting

const fs = require('fs');

try {
  fs.unlinkSync('example.txt');
  console.log('File has been deleted');
} catch (err) {
  console.error(err);
}

Advanced File System Operations

Working with Directories

Node.js allows you to create, read, and delete directories.

Creating Directories

const fs = require('fs');

// Asynchronous
fs.mkdir('newDir', (err) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log('Directory created');
});

// Synchronous
try {
  fs.mkdirSync('newDir');
  console.log('Directory created');
} catch (err) {
  console.error(err);
}

Reading Directories

const fs = require('fs');

// Asynchronous
fs.readdir('.', (err, files) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log(files);
});

// Synchronous
try {
  const files = fs.readdirSync('.');
  console.log(files);
} catch (err) {
  console.error(err);
}

Deleting Directories

const fs = require('fs');

// Asynchronous
fs.rmdir('newDir', (err) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log('Directory deleted');
});

// Synchronous
try {
  fs.rmdirSync('newDir');
  console.log('Directory deleted');
} catch (err) {
  console.error(err);
}

Watching Files and Directories

Node.js allows you to watch for changes in files and directories using fs.watch.

const fs = require('fs');

fs.watch('example.txt', (eventType, filename) => {
  if (filename) {
    console.log(`${filename} file Changed`);
  }
});

Conclusion

Understanding file system operations in Node.js is essential for developing robust applications. Whether you are reading, writing, appending, or deleting files, Node.js provides both synchronous and asynchronous methods to suit your needs. Additionally, working with directories and watching for changes can help you build more dynamic and responsive applications.
