Introduction

In this section, we will learn how to create a simple web server using Node.js. This will cover the basics of setting up a server, handling requests and responses, and understanding the core modules involved.

Prerequisites

Before starting, ensure you have the following:

  • Node.js installed on your machine.
  • A basic understanding of JavaScript.

Setting Up Your Environment

  1. Install Node.js: Download and install Node.js from the official website.
  2. Create a Project Directory: Open your terminal and create a new directory for your project. 
    mkdir simple-web-server
cd simple-web-server
  3. Initialize a Node.js Project: Run the following command to create a package.json file. 
    npm init -y

Creating the Server

  1. Create the Server File: Create a file named server.js in your project directory.

  2. Import the HTTP Module: Node.js has a built-in module called http that allows you to create a server.

    const http = require('http');

  3. Create the Server: Use the http.createServer method to create a server.

    const server = http.createServer((req, res) => {
    res.statusCode = 200;
    res.setHeader('Content-Type', 'text/plain');
    res.end('Hello, World!\n');
});

  4. Listen on a Port: Make the server listen on a specific port.

    const PORT = 3000;
server.listen(PORT, () => {
    console.log(`Server running at http://localhost:${PORT}/`);
});

Full Code Example

const http = require('http');

const server = http.createServer((req, res) => {
    res.statusCode = 200;
    res.setHeader('Content-Type', 'text/plain');
    res.end('Hello, World!\n');
});

const PORT = 3000;
server.listen(PORT, () => {
    console.log(`Server running at http://localhost:${PORT}/`);
});

Running the Server

  1. Start the Server: Run the following command in your terminal. 
    node server.js
  2. Access the Server: Open your web browser and navigate to http://localhost:3000. You should see "Hello, World!" displayed.

Handling Different Routes

To handle different routes, you can inspect the req.url property.

Example: Handling Multiple Routes

const http = require('http');

const server = http.createServer((req, res) => {
    if (req.url === '/') {
        res.statusCode = 200;
        res.setHeader('Content-Type', 'text/plain');
        res.end('Welcome to the Home Page!\n');
    } else if (req.url === '/about') {
        res.statusCode = 200;
        res.setHeader('Content-Type', 'text/plain');
        res.end('About Us\n');
    } else {
        res.statusCode = 404;
        res.setHeader('Content-Type', 'text/plain');
        res.end('Page Not Found\n');
    }
});

const PORT = 3000;
server.listen(PORT, () => {
    console.log(`Server running at http://localhost:${PORT}/`);
});

Advanced Concepts

Using Express.js

For more advanced routing and middleware, you can use the Express.js framework.

  1. Install Express:

    npm install express

  2. Create an Express Server:

    const express = require('express');
const app = express();
const PORT = 3000;

app.get('/', (req, res) => {
    res.send('Welcome to the Home Page!');
});

app.get('/about', (req, res) => {
    res.send('About Us');
});

app.use((req, res) => {
    res.status(404).send('Page Not Found');
});

app.listen(PORT, () => {
    console.log(`Server running at http://localhost:${PORT}/`);
});

Comparison: HTTP Module vs. Express.js

| Feature | HTTP Module | Express.js | |-----------------------|--------------------------------------|------------------------------------------| | Routing | Manual handling with req.url | Built-in routing with app.get, etc. | | Middleware | Manual implementation | Built-in support | | Simplicity | Basic and minimal | Higher-level abstractions | | Performance | Slightly faster for simple tasks | Slightly slower due to additional features|

Conclusion

In this topic, we covered the basics of creating a simple web server using Node.js. We started with the built-in http module and then explored more advanced routing with Express.js. By understanding these fundamentals, you can build more complex and feature-rich web servers.
