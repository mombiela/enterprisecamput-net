Authentication is a crucial aspect of web development, ensuring that users are who they claim to be. In this module, we will explore how to implement authentication in a Node.js application, covering both basic and advanced concepts.

What is Authentication?

Authentication is the process of verifying the identity of a user or system. It is a fundamental part of securing web applications and services.

Key Concepts:

  • User Credentials: Typically a combination of a username and password.
  • Tokens: Used to maintain user sessions without storing sensitive information on the client side.
  • Sessions: Server-side storage of user data to maintain state across multiple requests.
  • OAuth: An open standard for access delegation, commonly used for token-based authentication.

Setting Up a Node.js Project

Before diving into authentication, let's set up a basic Node.js project.

  1. Initialize a Node.js Project:

    mkdir auth-demo
cd auth-demo
npm init -y

  2. Install Required Packages:

    npm install express body-parser bcryptjs jsonwebtoken

  3. Create Basic Server:

    const express = require('express');
const bodyParser = require('body-parser');

const app = express();
app.use(bodyParser.json());

app.get('/', (req, res) => {
    res.send('Welcome to the Authentication Demo!');
});

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
    console.log(`Server is running on port ${PORT}`);
});

Basic Authentication

User Registration

  1. Hashing Passwords: Use bcryptjs to hash passwords before storing them in the database.

    const bcrypt = require('bcryptjs');

const hashPassword = async (password) => {
    const salt = await bcrypt.genSalt(10);
    const hashedPassword = await bcrypt.hash(password, salt);
    return hashedPassword;
};

  2. Register Endpoint:

    app.post('/register', async (req, res) => {
    const { username, password } = req.body;
    const hashedPassword = await hashPassword(password);
    // Save the user to the database (mocked here)
    const user = { id: Date.now(), username, password: hashedPassword };
    res.status(201).send(user);
});

User Login

  1. Verify Passwords: Compare the hashed password with the stored hash.

    const verifyPassword = async (password, hashedPassword) => {
    return await bcrypt.compare(password, hashedPassword);
};

  2. Login Endpoint:

    app.post('/login', async (req, res) => {
    const { username, password } = req.body;
    // Retrieve the user from the database (mocked here)
    const user = { id: 1, username, password: '$2a$10$...' }; // hashed password
    const isMatch = await verifyPassword(password, user.password);
    if (isMatch) {
        res.status(200).send('Login successful');
    } else {
        res.status(400).send('Invalid credentials');
    }
});

Token-Based Authentication

Generating Tokens

  1. Install jsonwebtoken:

    npm install jsonwebtoken

  2. Generate JWT:

    const jwt = require('jsonwebtoken');

const generateToken = (user) => {
    const token = jwt.sign({ id: user.id, username: user.username }, 'secretKey', { expiresIn: '1h' });
    return token;
};

  3. Login Endpoint with Token:

    app.post('/login', async (req, res) => {
    const { username, password } = req.body;
    const user = { id: 1, username, password: '$2a$10$...' };
    const isMatch = await verifyPassword(password, user.password);
    if (isMatch) {
        const token = generateToken(user);
        res.status(200).send({ token });
    } else {
        res.status(400).send('Invalid credentials');
    }
});

Protecting Routes

  1. Middleware to Verify Token:

    const authenticateToken = (req, res, next) => {
    const token = req.header('Authorization').replace('Bearer ', '');
    if (!token) {
        return res.status(401).send('Access Denied');
    }
    try {
        const verified = jwt.verify(token, 'secretKey');
        req.user = verified;
        next();
    } catch (err) {
        res.status(400).send('Invalid Token');
    }
};

  2. Protected Route:

    app.get('/protected', authenticateToken, (req, res) => {
    res.send('This is a protected route');
});

Conclusion

In this module, we covered the basics of authentication in a Node.js application. We started with setting up a Node.js project, then moved on to implementing basic authentication with hashed passwords. We also explored token-based authentication using JSON Web Tokens (JWT) and how to protect routes using middleware.

By understanding these concepts, you can build more secure Node.js applications and ensure that only authenticated users can access certain resources.
