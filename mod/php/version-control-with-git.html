Introduction to Version Control

Version control is a system that records changes to a file or set of files over time so that you can recall specific versions later. Git is one of the most popular version control systems used today.

  • Why use version control?
    • Track changes over time.
    • Collaborate with others.
    • Revert to previous versions.
    • Backup your work.

Getting Started with Git

Before diving into Git, ensure you have it installed on your system. You can download it from git-scm.com.

Basic Git Commands

Here are some fundamental Git commands to get you started:

  • Initialize a repository:

    git init

    This command initializes a new Git repository in your current directory.

  • Clone a repository:

    git clone <repository_url>

    This command creates a copy of an existing repository.

  • Check the status of your repository:

    git status

    This command shows the status of changes as untracked, modified, or staged.

  • Add files to staging area:

    git add <file_name>

    This command stages specific files for the next commit. Use git add . to stage all changes.

  • Commit changes:

    git commit -m "Your commit message"

    This command commits the staged changes to the repository with a message.

  • Push changes to a remote repository:

    git push origin <branch_name>

    This command pushes your local commits to the remote repository.

  • Pull changes from a remote repository:

    git pull origin <branch_name>

    This command fetches and merges changes from the remote repository to your local repository.

Branching and Merging

Branching is a powerful feature in Git that allows you to create separate lines of development.

Creating and Managing Branches

  • Create a new branch:

    git branch <branch_name>

    This command creates a new branch.

  • Switch to a branch:

    git checkout <branch_name>

    This command switches to the specified branch.

  • Create and switch to a new branch:

    git checkout -b <branch_name>

    This command creates a new branch and switches to it immediately.

Merging Branches

  • Merge a branch into the current branch: 
    git merge <branch_name>
    This command merges the specified branch into the current branch.

Handling Merge Conflicts

Sometimes, you may encounter conflicts during a merge. Git will mark the conflicts in the affected files, and you will need to resolve them manually.

  • View conflicts:

    git status

    This command will show you which files have conflicts.

  • After resolving conflicts, add the resolved files:

    git add <file_name>

  • Commit the merge:

    git commit

Working with Remote Repositories

Remote repositories are versions of your project that are hosted on the internet or network.

Adding a Remote Repository

  • Add a remote repository: 
    git remote add origin <repository_url>
    This command adds a remote repository.

Fetching and Pulling from Remote Repositories

  • Fetch changes from a remote repository:

    git fetch origin

    This command fetches changes from the remote repository but does not merge them.

  • Pull changes from a remote repository:

    git pull origin <branch_name>

    This command fetches and merges changes from the remote repository.

Pushing to Remote Repositories

  • Push changes to a remote repository: 
    git push origin <branch_name>
    This command pushes your local commits to the remote repository.

Advanced Git Features

Stashing Changes

Stashing allows you to save your changes temporarily without committing them.

  • Stash changes:

    git stash

    This command stashes your changes.

  • Apply stashed changes:

    git stash apply

    This command applies the stashed changes.

Rebasing

Rebasing is a way to integrate changes from one branch into another.

  • Rebase a branch: 
    git rebase <branch_name>
    This command rebases the current branch onto the specified branch.

Tagging

Tags are used to mark specific points in history as important.

  • Create a tag:

    git tag <tag_name>

    This command creates a tag.

  • Push tags to a remote repository:

    git push origin --tags

    This command pushes all tags to the remote repository.

Conclusion

Git is an essential tool for any developer, providing powerful features for version control and collaboration. By mastering Git, you can manage your PHP projects more effectively, collaborate with others, and maintain a history of your work. Practice the commands and concepts discussed in this topic to become proficient in using Git for your PHP development.
