Sessions and Cookies in PHP
Working with APIs ►

Introduction

In web development, managing user state and data across multiple pages is crucial. PHP offers two primary mechanisms for this: Sessions and Cookies. This section will cover the basics of both, how to use them, and their differences.

Cookies

Cookies are small pieces of data stored on the client's browser. They are used to remember information about the user between requests.

Setting a Cookie

To set a cookie in PHP, use the setcookie() function. Here's a basic example:

<?php
// Set a cookie named "user" with the value "John Doe" that expires in 30 days
setcookie("user", "John Doe", time() + (86400 * 30), "/"); // 86400 = 1 day
?>

Retrieving a Cookie

To retrieve a cookie, simply access the $_COOKIE superglobal array:

<?php
if(isset($_COOKIE["user"])) {
    echo "User is " . $_COOKIE["user"];
} else {
    echo "User is not set";
}
?>

Deleting a Cookie

To delete a cookie, set its expiration date to a past time:

<?php
// Delete the cookie named "user"
setcookie("user", "", time() - 3600, "/");
?>

Key Concepts

  • Lifetime: Cookies can have an expiration date, after which they are automatically deleted.
  • Scope: Cookies can be scoped to a specific path or domain.
  • Security: Cookies can be made secure by setting the Secure and HttpOnly flags.

Sessions

Sessions store data on the server and use a unique identifier (session ID) to link the client to the stored data.

Starting a Session

To start a session, use the session_start() function. This must be called at the beginning of the script:

<?php
session_start();
?>

Setting Session Variables

To set session variables, use the $_SESSION superglobal array:

<?php
session_start();
$_SESSION["username"] = "JohnDoe";
$_SESSION["email"] = "john@example.com";
?>

Retrieving Session Variables

To retrieve session variables, access the $_SESSION array:

<?php
session_start();
echo "Username: " . $_SESSION["username"];
echo "Email: " . $_SESSION["email"];
?>

Destroying a Session

To destroy a session and delete all session data, use session_destroy():

<?php
session_start();
session_destroy();
?>

Key Concepts

  • Lifetime: Sessions last until the browser is closed or the session is explicitly destroyed.
  • Storage: Session data is stored on the server, making it more secure than cookies.
  • Session ID: A unique identifier that links the client to the session data.

Comparison of Sessions and Cookies

Feature Cookies Sessions
Storage Location Client-side (browser) Server-side
Security Less secure (can be tampered with) More secure (data stored on server)
Lifetime Can be set to expire at a specific time Lasts until the browser is closed or destroyed
Data Size Limited to 4KB No size limit (depends on server configuration)
Use Cases Remembering user preferences, tracking Storing user login state, shopping cart data

Conclusion

Sessions and cookies are essential tools for managing user state and data in PHP applications. Cookies are useful for storing small amounts of data on the client side, while sessions are better suited for storing sensitive data securely on the server side. Understanding when and how to use each will help you build more robust and secure web applications.
