Introduction

The config.xml file is a crucial component in Apache Cordova projects. It serves as the central configuration file where you define various settings and preferences for your application. This file allows you to customize your app's behavior, appearance, and permissions across different platforms.

Key Concepts

  • Global Preferences: Settings that apply to all platforms.
  • Platform-Specific Preferences: Settings that apply to specific platforms like Android or iOS.
  • Plugins: External libraries that add functionality to your app.
  • Access Control: Configuring which external resources your app can access.

Structure of config.xml

The config.xml file is an XML document that follows a specific structure. Here is a basic example:

<?xml version='1.0' encoding='utf-8'?>
<widget id="com.example.hello" version="1.0.0" xmlns="http://www.w3.org/ns/widgets" xmlns:cdv="http://cordova.apache.org/ns/1.0">
    <name>HelloWorld</name>
    <description>
        A sample Apache Cordova application.
    </description>
    <author email="dev@example.com" href="http://example.com">
        Apache Cordova Team
    </author>
    <content src="index.html" />
    <access origin="*" />
    <preference name="Fullscreen" value="true" />
    <platform name="android">
        <preference name="AndroidLaunchMode" value="singleTop" />
    </platform>
    <platform name="ios">
        <preference name="BackupWebStorage" value="none" />
    </platform>
</widget>

Global Preferences

Global preferences are settings that apply to all platforms. They are defined directly under the <widget> element.

Example

<preference name="Fullscreen" value="true" />
  • Fullscreen: This preference makes the app run in fullscreen mode.

Common Global Preferences

  • Fullscreen: true or false
  • Orientation: portrait or landscape
  • BackgroundColor: Hex color code (e.g., #FFFFFF for white)

Platform-Specific Preferences

Platform-specific preferences are settings that apply only to a particular platform. They are defined within a <platform> element.

Example

<platform name="android">
    <preference name="AndroidLaunchMode" value="singleTop" />
</platform>
  • AndroidLaunchMode: This preference sets the launch mode for Android.

Common Platform-Specific Preferences

  • Android:
    • AndroidLaunchMode: singleTop, singleTask, etc.
    • AndroidPersistentFileLocation: internal or compatibility
  • iOS:
    • BackupWebStorage: none, local, or cloud
    • DisallowOverscroll: true or false

Plugins

Plugins are external libraries that add functionality to your Cordova app. They are defined using the <plugin> element.

Example

<plugin name="cordova-plugin-camera" spec="^4.1.0" />
  • cordova-plugin-camera: This plugin allows the app to access the device's camera.

Adding Plugins

To add a plugin, you can use the Cordova CLI:

cordova plugin add cordova-plugin-camera

Access Control

Access control settings define which external resources your app can access. They are defined using the <access> element.

Example

<access origin="*" />
  • origin="*": This allows the app to access any external resource.

Restricting Access

You can restrict access to specific domains:

<access origin="https://example.com" />

Conclusion

The config.xml file is essential for configuring your Apache Cordova app. It allows you to set global and platform-specific preferences, add plugins, and control access to external resources. Understanding how to configure this file effectively will enable you to customize your app's behavior and appearance across different platforms.

By mastering the config.xml file, you can ensure that your Cordova app meets your specific requirements and provides a consistent user experience across all supported platforms.
