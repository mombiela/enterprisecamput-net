Creating Your First Cordova Project
Understanding the Cordova Project Structure ►

In this section, we will guide you through the process of creating your first Apache Cordova project. Apache Cordova is a popular mobile application development framework that allows you to build mobile apps using HTML, CSS, and JavaScript. By the end of this section, you will have a basic Cordova project set up and ready for further development.

Prerequisites

Before you start, make sure you have the following installed on your system:

  • Node.js and npm (Node Package Manager)
  • Cordova CLI (Command Line Interface)

Installing Node.js and npm

If you haven't installed Node.js and npm yet, you can download and install them from the official Node.js website.

Installing Cordova CLI

Once Node.js and npm are installed, you can install the Cordova CLI by running the following command in your terminal or command prompt:

npm install -g cordova

The -g flag installs Cordova globally, making it accessible from any directory on your system.

Creating a New Cordova Project

To create a new Cordova project, follow these steps:

  1. Open your terminal or command prompt.
  2. Navigate to the directory where you want to create your project.
  3. Run the following command:
cordova create MyApp com.example.myapp MyApp

This command creates a new Cordova project with the following parameters:

  • MyApp: The directory name for your project.
  • com.example.myapp: The reverse domain-style identifier for your project.
  • MyApp: The display name of your app.

Project Structure

After running the cordova create command, you will have a new directory named MyApp with the following structure:

MyApp/
├── config.xml
├── hooks/
├── platforms/
├── plugins/
└── www/
  • config.xml: The configuration file for your Cordova project.
  • hooks/: Scripts that can be triggered at different stages of the Cordova build process.
  • platforms/: Contains platform-specific code (e.g., Android, iOS).
  • plugins/: Contains Cordova plugins used in your project.
  • www/: Contains your web assets (HTML, CSS, JavaScript).

Adding Platforms

To build your Cordova project for different platforms, you need to add the desired platforms. For example, to add the Android platform, run the following command:

cordova platform add android

Similarly, to add the iOS platform, run:

cordova platform add ios

Listing Installed Platforms

You can list the platforms added to your project by running:

cordova platform ls

Building the Project

Once you have added the necessary platforms, you can build your project using the following command:

cordova build

This command compiles your project for all the added platforms. You can also build for a specific platform by specifying the platform name:

cordova build android

Running the Project

To run your Cordova project on an emulator or a connected device, use the following command:

cordova run android

Replace android with ios if you want to run the project on an iOS device or emulator.

Conclusion

In this section, you have learned how to create your first Cordova project, add platforms, build the project, and run it on an emulator or device. This is the foundation for developing mobile applications using Apache Cordova. In the next sections, we will dive deeper into customizing your project, using plugins, and optimizing your app for different platforms.
