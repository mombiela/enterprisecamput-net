Introduction

Apache Cordova provides a simple way to access the geolocation capabilities of a device. This allows you to retrieve the geographic location of the device, which can be useful for a variety of applications, such as mapping, navigation, and location-based services.

Setting Up Geolocation in Cordova

Installing the Geolocation Plugin

To use geolocation in your Cordova project, you need to install the Cordova Geolocation plugin. This can be done using the following command:

cordova plugin add cordova-plugin-geolocation

Adding Permissions

Ensure that your application has the necessary permissions to access the device's location. For Android, this involves modifying the AndroidManifest.xml file, and for iOS, updating the Info.plist file.

Android

Add the following permissions to your AndroidManifest.xml:

<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />

iOS

Add the following keys to your Info.plist:

<key>NSLocationWhenInUseUsageDescription</key>
<string>We need your location to provide better services.</string>
<key>NSLocationAlwaysUsageDescription</key>
<string>We need your location to provide better services.</string>

Basic Geolocation Usage

Getting the Current Position

To get the current position of the device, use the navigator.geolocation.getCurrentPosition method. Here is a basic example:

document.addEventListener("deviceready", onDeviceReady, false);

function onDeviceReady() {
    navigator.geolocation.getCurrentPosition(onSuccess, onError);
}

function onSuccess(position) {
    console.log('Latitude: ' + position.coords.latitude);
    console.log('Longitude: ' + position.coords.longitude);
}

function onError(error) {
    console.error('Error code: ' + error.code + '\nMessage: ' + error.message);
}

Handling Errors

The onError callback function handles any errors that occur while trying to retrieve the geolocation. The error object contains a code and a message.

  • error.code can be:
    • 1: Permission denied
    • 2: Position unavailable
    • 3: Timeout

Advanced Geolocation Usage

Watching the Position

To continuously monitor the device's position, use the navigator.geolocation.watchPosition method. This is useful for applications that need to track the user's movement.

var watchID;

document.addEventListener("deviceready", onDeviceReady, false);

function onDeviceReady() {
    watchID = navigator.geolocation.watchPosition(onSuccess, onError, { timeout: 30000 });
}

function onSuccess(position) {
    console.log('Latitude: ' + position.coords.latitude);
    console.log('Longitude: ' + position.coords.longitude);
}

function onError(error) {
    console.error('Error code: ' + error.code + '\nMessage: ' + error.message);
}

function stopWatching() {
    if (watchID) {
        navigator.geolocation.clearWatch(watchID);
        watchID = null;
    }
}

Geolocation Options

You can customize the behavior of the geolocation methods by passing an options object. The options include:

  • enableHighAccuracy: Boolean, indicates if the application wants the best possible results.
  • timeout: Maximum time (in milliseconds) the device is allowed to take to return a position.
  • maximumAge: Maximum age (in milliseconds) of a cached position that is acceptable to return.

Example with options:

var options = {
    enableHighAccuracy: true,
    timeout: 5000,
    maximumAge: 0
};

navigator.geolocation.getCurrentPosition(onSuccess, onError, options);

Conclusion

Accessing geolocation in Apache Cordova is straightforward with the help of the cordova-plugin-geolocation. By understanding the basic and advanced usage of geolocation methods, you can create powerful location-based applications. Always remember to handle errors gracefully and respect user privacy by requesting permissions appropriately.
