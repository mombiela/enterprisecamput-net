Inheritance in Objective-C
Polymorphism ►

Inheritance is a fundamental concept in object-oriented programming that allows a class to inherit properties and methods from another class. This helps in code reusability and establishing a hierarchical relationship between classes.

What is Inheritance?

  • Definition: Inheritance allows a new class (subclass) to inherit the properties and methods of an existing class (superclass).
  • Purpose: It promotes code reuse and establishes a natural hierarchy between classes.

Basic Concepts

  • Superclass: The class whose properties and methods are inherited.
  • Subclass: The class that inherits properties and methods from the superclass.
  • @interface and @implementation: Keywords used to define classes in Objective-C.

Creating a Superclass

Let's start by creating a simple superclass called Person.

// Person.h
#import <Foundation/Foundation.h>

@interface Person : NSObject

@property NSString *name;
@property NSInteger age;

- (void)displayInfo;

@end

// Person.m
#import "Person.h"

@implementation Person

- (void)displayInfo {
    NSLog(@"Name: %@, Age: %ld", self.name, (long)self.age);
}

@end
  • Explanation:
    • @interface Person : NSObject: Declares a new class Person that inherits from NSObject.
    • @property: Declares properties name and age.
    • - (void)displayInfo;: Declares a method to display the person's information.
    • @implementation Person: Implements the Person class.
    • NSLog: Outputs the name and age to the console.

Creating a Subclass

Now, let's create a subclass called Student that inherits from Person.

// Student.h
#import "Person.h"

@interface Student : Person

@property NSString *schoolName;

- (void)displayStudentInfo;

@end

// Student.m
#import "Student.h"

@implementation Student

- (void)displayStudentInfo {
    [self displayInfo];
    NSLog(@"School: %@", self.schoolName);
}

@end
  • Explanation:
    • @interface Student : Person: Declares a new class Student that inherits from Person.
    • @property: Declares a new property schoolName.
    • - (void)displayStudentInfo;: Declares a method to display the student's information.
    • @implementation Student: Implements the Student class.
    • [self displayInfo]: Calls the displayInfo method from the superclass Person.

Using the Subclass

Let's see how to use the Student subclass in a program.

#import <Foundation/Foundation.h>
#import "Student.h"

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        Student *student = [[Student alloc] init];
        student.name = @"John Doe";
        student.age = 20;
        student.schoolName = @"XYZ University";
        
        [student displayStudentInfo];
    }
    return 0;
}
  • Explanation:
    • Student *student = [[Student alloc] init];: Creates an instance of the Student class.
    • student.name = @"John Doe";: Sets the name property inherited from Person.
    • student.age = 20;: Sets the age property inherited from Person.
    • student.schoolName = @"XYZ University";: Sets the schoolName property.
    • [student displayStudentInfo];: Calls the displayStudentInfo method to display all information.

Overriding Methods

A subclass can override methods of its superclass to provide specific implementations.

// Student.m
#import "Student.h"

@implementation Student

- (void)displayInfo {
    NSLog(@"Student Name: %@, Age: %ld", self.name, (long)self.age);
}

- (void)displayStudentInfo {
    [self displayInfo];
    NSLog(@"School: %@", self.schoolName);
}

@end
  • Explanation:
    • - (void)displayInfo: Overrides the displayInfo method from Person to provide a specific implementation for Student.

Summary

  • Inheritance allows a class to inherit properties and methods from another class.
  • The superclass provides the base properties and methods.
  • The subclass can add new properties and methods or override existing ones.
  • This promotes code reuse and helps in creating a hierarchical structure.

By understanding and utilizing inheritance, you can create more organized and maintainable code in Objective-C.
