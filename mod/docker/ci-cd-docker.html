Introduction to CI/CD

  • Continuous Integration (CI): The practice of automatically integrating code changes from multiple contributors into a shared repository several times a day.
  • Continuous Deployment (CD): The practice of automatically deploying every change that passes all stages of your production pipeline to production.

Benefits of Using Docker in CI/CD

  • Consistency: Docker ensures that the application runs the same way regardless of where it is deployed.
  • Isolation: Each application runs in its own container, avoiding conflicts.
  • Scalability: Docker containers can be easily scaled up or down.

Setting Up a CI/CD Pipeline with Docker

Prerequisites

  • Basic knowledge of Docker.
  • A version control system (e.g., Git).
  • A CI/CD tool (e.g., Jenkins, GitLab CI, GitHub Actions).

Step-by-Step Guide

1. Dockerizing Your Application

  • Create a Dockerfile to define your application environment.
# Use an official Python runtime as a parent image
FROM python:3.8-slim

# Set the working directory in the container
WORKDIR /app

# Copy the current directory contents into the container at /app
COPY . /app

# Install any needed packages specified in requirements.txt
RUN pip install --no-cache-dir -r requirements.txt

# Make port 80 available to the world outside this container
EXPOSE 80

# Define environment variable
ENV NAME World

# Run app.py when the container launches
CMD ["python", "app.py"]

2. Building and Testing the Docker Image

  • Build the Docker image locally to ensure it works as expected.
docker build -t my-python-app .
  • Run the Docker container to test it.
docker run -p 4000:80 my-python-app

3. Setting Up a CI/CD Tool

  • Choose a CI/CD tool and configure it to use your Dockerfile.
Example: GitHub Actions
  • Create a .github/workflows/main.yml file in your repository.
name: CI/CD Pipeline

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Docker Buildx
      uses: docker/setup-buildx-action@v1

    - name: Build Docker image
      run: docker build -t my-python-app .

    - name: Run tests
      run: docker run my-python-app pytest

4. Deploying the Docker Container

  • Push the Docker image to a container registry (e.g., Docker Hub).
docker tag my-python-app mydockerhubusername/my-python-app
docker push mydockerhubusername/my-python-app
  • Configure your CI/CD tool to deploy the image to your production environment.
Example: GitHub Actions (continued)
    - name: Log in to Docker Hub
      run: echo "${{ secrets.DOCKER_PASSWORD }}" | docker login -u "${{ secrets.DOCKER_USERNAME }}" --password-stdin

    - name: Push Docker image
      run: docker push mydockerhubusername/my-python-app

    - name: Deploy to production
      run: |
        ssh user@yourserver "docker pull mydockerhubusername/my-python-app && docker run -d -p 80:80 mydockerhubusername/my-python-app"

Conclusion

In this topic, we covered the essentials of setting up a CI/CD pipeline using Docker. We started with an introduction to CI/CD concepts and the benefits of using Docker. Then, we walked through the steps to Dockerize an application, build and test the Docker image, set up a CI/CD tool, and deploy the Docker container. By following these steps, you can ensure a consistent, isolated, and scalable deployment process for your applications.
