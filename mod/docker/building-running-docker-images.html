Introduction

In this section, we will cover the fundamental concepts of building and running Docker images. Docker images are the foundation of containers, and understanding how to create and manage them is crucial for any Docker user.

Key Concepts

  • Dockerfile: A text document that contains all the commands to assemble an image.
  • Image: A lightweight, standalone, executable package that includes everything needed to run a piece of software.
  • Container: A runtime instance of an image.

Building Docker Images

Writing a Dockerfile

A Dockerfile is a script that contains a series of instructions on how to build a Docker image. Here is a simple example:

# Use an official Python runtime as a parent image
FROM python:3.8-slim

# Set the working directory in the container
WORKDIR /app

# Copy the current directory contents into the container at /app
COPY . /app

# Install any needed packages specified in requirements.txt
RUN pip install --no-cache-dir -r requirements.txt

# Make port 80 available to the world outside this container
EXPOSE 80

# Define environment variable
ENV NAME World

# Run app.py when the container launches
CMD ["python", "app.py"]

Building the Image

To build an image from a Dockerfile, use the docker build command. Here is the syntax:

docker build -t my-python-app .
  • -t my-python-app: Tags the image with the name my-python-app.
  • .: Specifies the build context, which is the current directory.

Understanding the Build Process

When you run docker build, Docker performs the following steps:

  1. Reads the Dockerfile.
  2. Executes each instruction in the Dockerfile.
  3. Creates a new image layer for each instruction.
  4. Tags the final image with the specified name.

Running Docker Images

Running a Container

To run a container from an image, use the docker run command. Here is an example:

docker run -p 4000:80 my-python-app
  • -p 4000:80: Maps port 4000 on the host to port 80 in the container.
  • my-python-app: The name of the image to run.

Managing Containers

  • List running containers: docker ps
  • Stop a container: docker stop <container_id>
  • Remove a container: docker rm <container_id>

Example: Running a Web Application

Let's assume you have a simple Flask web application. Here is how you can build and run it using Docker:

  1. Create a Dockerfile:

    FROM python:3.8-slim
WORKDIR /app
COPY . /app
RUN pip install --no-cache-dir -r requirements.txt
EXPOSE 5000
CMD ["python", "app.py"]

  2. Build the image:

    docker build -t flask-app .

  3. Run the container:

    docker run -p 5000:5000 flask-app

Conclusion

In this section, we covered the basics of building and running Docker images. We learned how to write a Dockerfile, build an image, and run a container. These skills are fundamental for working with Docker and deploying applications in a containerized environment. As you progress, you will learn more advanced techniques for optimizing and managing Docker images and containers.
