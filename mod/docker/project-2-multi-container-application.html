Introduction

In this project, we will build a multi-container application using Docker. This project will help you understand how to orchestrate multiple services, manage dependencies, and ensure seamless communication between containers.

Key Concepts

  • Docker Compose: A tool for defining and running multi-container Docker applications.
  • Service: A container in a Docker Compose file.
  • Network: A Docker network that allows containers to communicate with each other.
  • Volume: A persistent storage mechanism for Docker containers.

Setting Up the Project

To start, we will create a simple multi-container application consisting of a web server and a database.

Step 1: Create the Project Directory

First, create a directory for your project:

mkdir multi-container-app
cd multi-container-app

Step 2: Define the Docker Compose File

Create a docker-compose.yml file to define the services:

version: '3.8'

services:
  web:
    image: nginx:latest
    ports:
      - "80:80"
    networks:
      - app-network

  db:
    image: postgres:latest
    environment:
      POSTGRES_USER: example
      POSTGRES_PASSWORD: example
    volumes:
      - db-data:/var/lib/postgresql/data
    networks:
      - app-network

networks:
  app-network:
    driver: bridge

volumes:
  db-data:

Explanation

  • version: Specifies the version of Docker Compose.
  • services: Defines the containers to be run.
    • web: The web server service using the Nginx image.
      • ports: Maps port 80 on the host to port 80 in the container.
      • networks: Connects the container to the app-network.
    • db: The database service using the PostgreSQL image.
      • environment: Sets environment variables for the database user and password.
      • volumes: Mounts a volume for persistent storage.
      • networks: Connects the container to the app-network.
  • networks: Defines a custom network named app-network.
  • volumes: Defines a volume named db-data for persistent storage.

Running the Application

To run the application, use the following command:

docker-compose up -d
  • The -d flag runs the containers in detached mode.

Checking the Services

You can check the running services using:

docker-compose ps

Interacting with the Containers

Accessing the Web Server

Open your browser and navigate to http://localhost. You should see the default Nginx welcome page.

Accessing the Database

You can interact with the PostgreSQL database using a PostgreSQL client or the psql command-line tool.

Stopping the Application

To stop the application, use:

docker-compose down
  • This command stops and removes the containers, networks, and volumes defined in the docker-compose.yml file.

Conclusion

In this project, you have learned how to:

  • Define and run a multi-container application using Docker Compose.
  • Set up services with specific configurations.
  • Manage networks and volumes for container communication and persistent storage.

By mastering these concepts, you can now build and manage more complex multi-container applications, enhancing your Docker skills from beginner to advanced levels.
