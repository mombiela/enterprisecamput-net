Running Containers

In this section, we will explore how to run containers using Docker. This is a fundamental skill for anyone working with Docker, as containers are the core unit of software deployment in Docker.

Key Concepts

  • Container: A lightweight, standalone, executable package that includes everything needed to run a piece of software, including the code, runtime, libraries, and system tools.
  • Image: A read-only template used to create containers. Images are often based on other images, with some additional customization.
  • Docker Hub: A cloud-based repository where Docker users can create, test, store, and distribute container images.

Running Your First Container

To run a container, you need a Docker image. Docker Hub provides a vast repository of pre-built images that you can use.

Example: Running a Hello World Container

  1. Pull the Image: First, you need to pull the image from Docker Hub.

    docker pull hello-world

  2. Run the Container: Use the docker run command to create and start a container from the image.

    docker run hello-world

    When you run this command, Docker will:

    • Check if the hello-world image is available locally. If not, it will download it from Docker Hub.
    • Create a new container from the image.
    • Start the container.
    • Execute the command specified in the image (in this case, printing "Hello from Docker!").

Understanding the `docker run` Command

The docker run command is versatile and can be used with various options to customize the container's behavior.

  • Basic Syntax:

    docker run [OPTIONS] IMAGE [COMMAND] [ARG...]

  • Common Options:

    • -d: Run the container in detached mode (in the background).
    • -p: Publish a container's port(s) to the host.
    • -v: Bind mount a volume.
    • --name: Assign a name to the container.
    • -e: Set environment variables.

Example: Running a Web Server

Let's run a simple web server using the nginx image.

  1. Pull the Image:

    docker pull nginx

  2. Run the Container:

    docker run -d -p 8080:80 --name mynginx nginx

    Explanation:

    • -d: Runs the container in detached mode.
    • -p 8080:80: Maps port 8080 on the host to port 80 in the container.
    • --name mynginx: Names the container mynginx.
    • nginx: The image to use.

  3. Access the Web Server: Open a web browser and navigate to http://localhost:8080. You should see the default Nginx welcome page.

Managing Containers

Once you have containers running, you need to manage them. Docker provides several commands to help with this.

Listing Containers

  • List Running Containers:

    docker ps

  • List All Containers (including stopped ones):

    docker ps -a

Stopping and Removing Containers

  • Stop a Running Container:

    docker stop <container_id_or_name>

  • Remove a Stopped Container:

    docker rm <container_id_or_name>

Example: Stopping and Removing the Nginx Container

  1. Stop the Container:

    docker stop mynginx

  2. Remove the Container:

    docker rm mynginx

Conclusion

Running containers is a fundamental skill in Docker. By understanding how to pull images, run containers, and manage them, you can start leveraging Docker's power to deploy and manage applications efficiently. Practice running different containers and using various options with the docker run command to become more comfortable with Docker.
