Introduction

In this section, we will explore the lifecycle of a Docker container. Understanding the container lifecycle is crucial for managing and orchestrating containers effectively. We will cover the various stages a container goes through from creation to deletion, along with practical examples and exercises.

Key Concepts

  • Container Image: A lightweight, standalone, executable package that includes everything needed to run a piece of software.
  • Container: A runtime instance of an image.
  • Docker Daemon: The background service running on the host that manages Docker containers.
  • Docker CLI: Command Line Interface used to interact with Docker Daemon.

Container Lifecycle Stages

The lifecycle of a Docker container can be broken down into several stages:

  1. Create
  2. Start
  3. Run
  4. Pause/Unpause
  5. Stop
  6. Restart
  7. Kill
  8. Remove

Create

  • Description: The container is created from a Docker image but not started.
  • Command: docker create [OPTIONS] IMAGE [COMMAND] [ARG...]
  • Example: 
    docker create --name my_container ubuntu
  • Explanation: This command creates a container named my_container from the ubuntu image.

Start

  • Description: The container is started and begins executing.
  • Command: docker start [OPTIONS] CONTAINER
  • Example: 
    docker start my_container
  • Explanation: This command starts the previously created my_container.

Run

  • Description: Combines create and start in one step.
  • Command: docker run [OPTIONS] IMAGE [COMMAND] [ARG...]
  • Example: 
    docker run --name my_container -d ubuntu
  • Explanation: This command creates and starts a container named my_container from the ubuntu image in detached mode (-d).

Pause/Unpause

  • Description: Temporarily suspends all processes in the container.
  • Commands:
    • Pause: docker pause CONTAINER
    • Unpause: docker unpause CONTAINER
  • Example: 
    docker pause my_container
docker unpause my_container
  • Explanation: These commands pause and unpause the my_container, respectively.

Stop

  • Description: Gracefully stops the container.
  • Command: docker stop [OPTIONS] CONTAINER
  • Example: 
    docker stop my_container
  • Explanation: This command stops the my_container gracefully.

Restart

  • Description: Stops and then starts the container.
  • Command: docker restart [OPTIONS] CONTAINER
  • Example: 
    docker restart my_container
  • Explanation: This command restarts the my_container.

Kill

  • Description: Forcibly stops the container.
  • Command: docker kill [OPTIONS] CONTAINER
  • Example: 
    docker kill my_container
  • Explanation: This command forcefully stops the my_container.

Remove

  • Description: Deletes the container.
  • Command: docker rm [OPTIONS] CONTAINER
  • Example: 
    docker rm my_container
  • Explanation: This command removes the my_container from the Docker host.

Summary

Understanding the lifecycle of a Docker container is essential for effective container management. The lifecycle stages include creating, starting, running, pausing, stopping, restarting, killing, and removing containers. Each stage has specific Docker commands associated with it, which allow you to control the state of your containers. By mastering these commands, you can efficiently manage your containerized applications.

Exercises

  1. Create and Start a Container:

    • Create a container from the nginx image and start it.
    docker create --name web_server nginx
docker start web_server

  2. Pause and Unpause a Container:

    • Pause and then unpause the web_server container.
    docker pause web_server
docker unpause web_server

  3. Stop and Remove a Container:

    • Stop and then remove the web_server container.
    docker stop web_server
docker rm web_server

By practicing these exercises, you will gain hands-on experience with the Docker container lifecycle, reinforcing your understanding of each stage and its associated commands.
