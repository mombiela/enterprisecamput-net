In this project, we will create a simple web application and containerize it using Docker. This project will help you understand the basics of Docker, including creating Dockerfiles, building images, and running containers.

Objectives

By the end of this project, you will:

  • Understand how to create a Dockerfile.
  • Learn how to build a Docker image.
  • Run a Docker container.
  • Expose a containerized web application to the outside world.

Prerequisites

Before starting this project, ensure you have:

  • Basic knowledge of web development (HTML, CSS, JavaScript).
  • Basic understanding of Docker concepts.
  • Docker installed on your machine.

Step 1: Create a Simple Web Application

First, let's create a simple web application using Node.js.

  1. Initialize a Node.js project:

    mkdir simple-web-app
cd simple-web-app
npm init -y

  2. Install Express.js:

    npm install express

  3. Create app.js:

    const express = require('express');
const app = express();
const port = 3000;

app.get('/', (req, res) => {
    res.send('Hello, World!');
});

app.listen(port, () => {
    console.log(`App running at http://localhost:${port}`);
});

  4. Update package.json to include a start script:

    "scripts": {
    "start": "node app.js"
}

Step 2: Create a Dockerfile

A Dockerfile is a text document that contains all the commands to assemble an image.

  1. Create a Dockerfile in the root of your project:

    # Use the official Node.js image as the base image
FROM node:14

# Set the working directory
WORKDIR /usr/src/app

# Copy package.json and package-lock.json
COPY package*.json ./

# Install dependencies
RUN npm install

# Copy the rest of the application code
COPY . .

# Expose the port the app runs on
EXPOSE 3000

# Command to run the application
CMD ["npm", "start"]

Step 3: Build the Docker Image

To build the Docker image, use the docker build command.

  1. Build the image:

    docker build -t simple-web-app .
    • -t simple-web-app: Tags the image with the name simple-web-app.
    • .: Specifies the current directory as the build context.

Step 4: Run the Docker Container

Once the image is built, you can run it using the docker run command.

  1. Run the container:

    docker run -p 3000:3000 simple-web-app
    • -p 3000:3000: Maps port 3000 on your host to port 3000 in the container.

  2. Access the application:

    Open your browser and navigate to http://localhost:3000. You should see "Hello, World!".

Conclusion

In this project, you learned how to:

  • Create a simple web application using Node.js and Express.
  • Write a Dockerfile to containerize the application.
  • Build a Docker image from the Dockerfile.
  • Run a Docker container from the image and expose it to the outside world.

This foundational knowledge will serve as a stepping stone for more advanced Docker topics, such as multi-stage builds, Docker Compose, and orchestrating containers with Kubernetes.
