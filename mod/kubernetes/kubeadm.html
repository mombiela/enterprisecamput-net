Kubeadm is a tool that helps you bootstrap a minimum viable Kubernetes cluster that conforms to best practices. It performs the actions necessary to get a minimum viable, secure cluster up and running in a user-friendly way.

Introduction to Kubeadm

  • What is Kubeadm?

    • Kubeadm is a tool provided by Kubernetes to simplify the process of setting up a Kubernetes cluster.
    • It handles the installation and configuration of the control plane components and worker nodes.

  • Why use Kubeadm?

    • Simplifies the cluster setup process.
    • Ensures best practices and security configurations.
    • Provides a consistent and repeatable way to create clusters.

Installing Kubeadm

Prerequisites

  • Operating System: Linux (Ubuntu, CentOS, etc.)
  • Hardware Requirements:
    • 2 GB or more of RAM per machine (any less will leave little room for your apps).
    • 2 CPUs or more.
    • Full network connectivity between all machines in the cluster (public or private network is fine).
    • Unique hostname, MAC address, and product_uuid for every node.
  • Software Requirements:
    • Docker (or another container runtime).
    • Kubeadm, kubelet, and kubectl installed on all nodes.

Installation Steps

  1. Install Docker:

    sudo apt-get update
sudo apt-get install -y docker.io
sudo systemctl enable docker
sudo systemctl start docker

  2. Install Kubeadm, Kubelet, and Kubectl:

    sudo apt-get update && sudo apt-get install -y apt-transport-https curl
curl -s https://packages.cloud.google.com/apt/doc/apt-key.gpg | sudo apt-key add -
cat <<EOF | sudo tee /etc/apt/sources.list.d/kubernetes.list
deb https://apt.kubernetes.io/ kubernetes-xenial main
EOF
sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl

Bootstrapping a Cluster with Kubeadm

Initializing the Control Plane

  • Command:

    sudo kubeadm init --pod-network-cidr=10.244.0.0/16

  • Explanation:

    • --pod-network-cidr=10.244.0.0/16: This flag specifies the CIDR for the pod network. This example uses Flannel's default CIDR.

  • Post-Initialization Steps:

    mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config

Setting Up a Pod Network

  • Install Flannel: 
    kubectl apply -f https://raw.githubusercontent.com/coreos/flannel/master/Documentation/kube-flannel.yml

Joining Worker Nodes

  • Command:

    • After initializing the control plane, kubeadm will output a kubeadm join command. Run this command on each worker node to join them to the cluster.

  • Example:

    sudo kubeadm join <control-plane-host>:<control-plane-port> --token <token> --discovery-token-ca-cert-hash sha256:<hash>

Managing the Cluster

Common Commands

  • Check Node Status:

    kubectl get nodes

  • Check Pod Status:

    kubectl get pods --all-namespaces

  • Deploy an Application:

    kubectl create deployment nginx --image=nginx

  • Expose the Deployment:

    kubectl expose deployment nginx --port=80 --type=NodePort

Troubleshooting

  • Common Issues:

    • Pods not starting: Check the pod logs using kubectl logs <pod-name>.
    • Node not ready: Ensure that the kubelet service is running and check the node status using kubectl describe node <node-name>.

  • Useful Commands:

    • kubectl describe pod <pod-name>: Provides detailed information about a pod.
    • kubectl logs <pod-name>: Fetches the logs of a pod.

Conclusion

Kubeadm is a powerful tool that simplifies the process of setting up a Kubernetes cluster. By following best practices and providing a consistent setup process, it ensures that your cluster is secure and reliable. Whether you're setting up a small development cluster or a large production environment, Kubeadm provides the tools you need to get started quickly and efficiently.
