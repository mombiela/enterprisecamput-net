Minikube
Case Study 1: Migrating to Kubernetes ►

Minikube is a tool that makes it easy to run Kubernetes locally. It runs a single-node Kubernetes cluster inside a virtual machine (VM) on your laptop. This is particularly useful for development, testing, and learning purposes.

Key Concepts

  • Single-node Kubernetes Cluster: Minikube runs a single-node Kubernetes cluster, which means it includes both the master and worker node functionalities in one VM.
  • Virtual Machine: Minikube uses a VM to run the Kubernetes cluster. It supports various VM drivers like VirtualBox, VMware, Hyper-V, and Docker.
  • Local Development: Minikube is designed for local development and testing, making it easier to experiment with Kubernetes without needing a full-scale cluster.

Installation

Prerequisites

Before installing Minikube, ensure you have the following:

  • A hypervisor installed (e.g., VirtualBox, VMware, Hyper-V, or Docker).
  • kubectl installed. kubectl is the command-line tool for interacting with Kubernetes clusters.

Installing Minikube

  1. Download Minikube:

    • For macOS: 
      brew install minikube
    • For Linux: 
      curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
sudo install minikube-linux-amd64 /usr/local/bin/minikube
    • For Windows: Download the installer from the Minikube releases page.

  2. Start Minikube:

    minikube start

    This command starts a local Kubernetes cluster. You can specify the VM driver using the --driver flag, for example:

    minikube start --driver=virtualbox

Basic Usage

Interacting with Minikube

  • Check Cluster Status:

    minikube status

  • Access Kubernetes Dashboard: Minikube provides a built-in Kubernetes dashboard for managing the cluster.

    minikube dashboard

Deploying Applications

  1. Create a Deployment:

    kubectl create deployment hello-minikube --image=k8s.gcr.io/echoserver:1.4

  2. Expose the Deployment:

    kubectl expose deployment hello-minikube --type=NodePort --port=8080

  3. Access the Application: Get the URL to access the application:

    minikube service hello-minikube --url

Managing Minikube

  • Stop Minikube:

    minikube stop

  • Delete Minikube Cluster:

    minikube delete

Advanced Features

Configuring Resources

You can configure the amount of CPU and memory allocated to the Minikube VM:

minikube start --cpus=4 --memory=8192

Add-ons

Minikube supports various add-ons to extend its functionality. Some popular add-ons include:

  • Ingress:

    minikube addons enable ingress

  • Metrics Server:

    minikube addons enable metrics-server

Persistent Storage

Minikube supports persistent storage, allowing you to create PersistentVolume and PersistentVolumeClaim resources.

  1. Create a PersistentVolume:

    apiVersion: v1
kind: PersistentVolume
metadata:
  name: pv0001
spec:
  capacity:
    storage: 1Gi
  accessModes:
    - ReadWriteOnce
  hostPath:
    path: "/mnt/data"

  2. Create a PersistentVolumeClaim:

    apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: pvc0001
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 1Gi

  3. Apply the configurations:

    kubectl apply -f pv.yaml
kubectl apply -f pvc.yaml

Conclusion

Minikube is an excellent tool for developers and learners to get hands-on experience with Kubernetes. It simplifies the process of setting up a local Kubernetes environment, making it easier to develop, test, and experiment with Kubernetes features. By understanding how to install, configure, and use Minikube, you can effectively simulate a Kubernetes cluster on your local machine and gain valuable insights into Kubernetes operations.
