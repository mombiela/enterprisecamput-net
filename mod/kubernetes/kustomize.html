Kustomize is a tool for customizing Kubernetes configurations. It allows you to manage and customize Kubernetes manifests without the need to use templates. This topic will guide you from the basics of Kustomize to advanced usage, providing clear explanations, examples, and exercises.

Introduction to Kustomize

Kustomize is built into kubectl and provides a way to manage Kubernetes resources declaratively. It allows you to define a base set of resources and then create overlays to customize these resources for different environments.

Key Concepts

  • Base: A set of Kubernetes manifests that define the common resources.
  • Overlay: Customizations applied on top of the base manifests for different environments (e.g., development, staging, production).
  • kustomization.yaml: The configuration file used by Kustomize to define how resources should be customized.

Getting Started with Kustomize

Installing Kustomize

Kustomize is integrated into kubectl, but you can also install it as a standalone tool.

Using kubectl

Kustomize is included in kubectl starting from version 1.14. You can use it directly with the kubectl kustomize command.

kubectl kustomize <path-to-kustomization-directory>

Standalone Installation

To install Kustomize as a standalone tool, follow the instructions on the Kustomize GitHub page.

Creating a Base

A base is a set of Kubernetes manifests that define the common resources. Let's create a simple base for a Deployment and a Service.

# deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 2
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app
        image: my-app:1.0.0
        ports:
        - containerPort: 80
# service.yaml
apiVersion: v1
kind: Service
metadata:
  name: my-app
spec:
  selector:
    app: my-app
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80

Creating a kustomization.yaml

Create a kustomization.yaml file to define the base.

# kustomization.yaml
resources:
- deployment.yaml
- service.yaml

Applying the Base

To apply the base, use the following command:

kubectl apply -k <path-to-base-directory>

Creating Overlays

Overlays allow you to customize the base resources for different environments.

Example: Development Overlay

Create a directory for the development overlay and a kustomization.yaml file within it.

# overlays/development/kustomization.yaml
resources:
- ../../base

patchesStrategicMerge:
- deployment-patch.yaml

Create a patch file to customize the Deployment for the development environment.

# overlays/development/deployment-patch.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 1
  template:
    spec:
      containers:
      - name: my-app
        image: my-app:1.0.1

Applying the Overlay

To apply the development overlay, use the following command:

kubectl apply -k overlays/development

Advanced Kustomize Features

Using ConfigMaps and Secrets

Kustomize allows you to generate ConfigMaps and Secrets from files or literals.

# kustomization.yaml
configMapGenerator:
- name: my-config
  files:
  - config.properties

secretGenerator:
- name: my-secret
  literals:
  - username=admin
  - password=secret

Common Labels and Annotations

You can add common labels and annotations to all resources.

# kustomization.yaml
commonLabels:
  app: my-app
commonAnnotations:
  environment: development

Transformations

Kustomize supports various transformations, such as namePrefix, nameSuffix, and image transformations.

# kustomization.yaml
namePrefix: dev-
nameSuffix: -v1

images:
- name: my-app
  newName: my-app
  newTag: 1.0.1

Conclusion

Kustomize is a powerful tool for managing and customizing Kubernetes configurations. It allows you to define a base set of resources and create overlays for different environments, making it easier to manage complex deployments. By understanding and utilizing Kustomize, you can streamline your Kubernetes workflows and improve the maintainability of your configurations.

Summary

  • Base: Common set of Kubernetes manifests.
  • Overlay: Customizations for different environments.
  • kustomization.yaml: Configuration file for Kustomize.
  • Key Features: ConfigMap and Secret generation, common labels and annotations, transformations.

Practice using Kustomize by creating your own bases and overlays, and explore its advanced features to fully leverage its capabilities in your Kubernetes deployments.
