Introduction

Kubernetes, often abbreviated as K8s, is an open-source platform designed to automate deploying, scaling, and operating application containers. Understanding the key concepts and terminology is crucial for mastering Kubernetes from beginner to advanced levels.

Core Concepts

Cluster

  • Definition: A set of nodes (machines) that run containerized applications managed by Kubernetes.
  • Components:
    • Master Node: Manages the cluster and coordinates all activities.
    • Worker Nodes: Execute the containerized applications.

Node

  • Definition: A single machine in the Kubernetes cluster, which can be either a physical or virtual machine.
  • Types:
    • Master Node: Handles the control plane.
    • Worker Node: Runs the application workloads.

Pod

  • Definition: The smallest and simplest Kubernetes object. A pod represents a single instance of a running process in your cluster.
  • Characteristics:
    • Can contain one or more containers.
    • Shares storage and network resources.
apiVersion: v1
kind: Pod
metadata:
  name: example-pod
spec:
  containers:
  - name: example-container
    image: nginx

Service

  • Definition: An abstraction that defines a logical set of Pods and a policy by which to access them.
  • Types:
    • ClusterIP: Exposes the service on a cluster-internal IP.
    • NodePort: Exposes the service on each Node’s IP at a static port.
    • LoadBalancer: Exposes the service externally using a cloud provider’s load balancer.
apiVersion: v1
kind: Service
metadata:
  name: example-service
spec:
  selector:
    app: example
  ports:
    - protocol: TCP
      port: 80
      targetPort: 9376
  type: ClusterIP

Deployment

  • Definition: Provides declarative updates to applications, ensuring that the desired number of Pods are running.
  • Features:
    • Rolling updates.
    • Rollbacks.
apiVersion: apps/v1
kind: Deployment
metadata:
  name: example-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: example
  template:
    metadata:
      labels:
        app: example
    spec:
      containers:
      - name: example-container
        image: nginx

Namespace

  • Definition: A way to divide cluster resources between multiple users.
  • Use Cases:
    • Environment separation (e.g., dev, staging, production).
    • Resource quota management.
apiVersion: v1
kind: Namespace
metadata:
  name: example-namespace

Advanced Concepts

StatefulSet

  • Definition: Manages the deployment and scaling of a set of Pods, and provides guarantees about the ordering and uniqueness of these Pods.
  • Use Cases:
    • Stateful applications like databases.
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: example-statefulset
spec:
  serviceName: "example"
  replicas: 3
  selector:
    matchLabels:
      app: example
  template:
    metadata:
      labels:
        app: example
    spec:
      containers:
      - name: example-container
        image: nginx

DaemonSet

  • Definition: Ensures that all (or some) Nodes run a copy of a Pod.
  • Use Cases:
    • Node monitoring.
    • Log collection.
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: example-daemonset
spec:
  selector:
    matchLabels:
      app: example
  template:
    metadata:
      labels:
        app: example
    spec:
      containers:
      - name: example-container
        image: nginx

ConfigMap and Secret

  • ConfigMap:
    • Definition: Used to store non-confidential data in key-value pairs.
    • Use Cases: Configuration settings.
apiVersion: v1
kind: ConfigMap
metadata:
  name: example-configmap
data:
  example.property: "exampleValue"
  • Secret:
    • Definition: Used to store sensitive information, such as passwords, OAuth tokens, and ssh keys.
    • Use Cases: Secure storage of sensitive data.
apiVersion: v1
kind: Secret
metadata:
  name: example-secret
type: Opaque
data:
  password: cGFzc3dvcmQ=

Conclusion

Understanding these key concepts and terminology is essential for effectively using Kubernetes. From the basic building blocks like Pods and Services to more advanced constructs like StatefulSets and DaemonSets, each component plays a critical role in the Kubernetes ecosystem. Mastery of these concepts will provide a strong foundation for further exploration and advanced usage of Kubernetes.
