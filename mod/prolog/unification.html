Unification in Prolog
Backtracking ►

Unification is a fundamental concept in Prolog, essential for understanding how the language processes queries and matches patterns. This topic will guide you from the basics to advanced aspects of unification, providing clear explanations, examples, and exercises.

What is Unification?

Unification is the process of making two terms identical by finding a suitable substitution for variables. It is the core mechanism that Prolog uses to match goals with facts and rules in the database.

Key Concepts

  • Term: The basic building block in Prolog, which can be an atom, number, variable, or compound term.
  • Variable: A placeholder that can be unified with different terms.
  • Substitution: The process of replacing a variable with a term.
  • Matching: The process of comparing two terms to see if they can be made identical through substitution.

Basic Unification

In Prolog, unification is used to determine if two terms can be made identical. Let's start with some simple examples.

Example 1: Unifying Atoms

?- apple = apple.
true.

In this example, the two atoms apple and apple are identical, so the unification succeeds.

Example 2: Unifying Variables with Atoms

?- X = apple.
X = apple.

Here, the variable X is unified with the atom apple. The variable X now holds the value apple.

Example 3: Unifying Variables with Variables

?- X = Y.
X = Y.

In this case, both X and Y are variables. They can be unified with each other, meaning they can take the same value.

Unification with Compound Terms

Unification also works with compound terms, which are terms that have a functor and arguments.

Example 4: Unifying Compound Terms

?- point(X, Y) = point(1, 2).
X = 1,
Y = 2.

Here, point(X, Y) is unified with point(1, 2). The variables X and Y are unified with 1 and 2, respectively.

Example 5: Unification Failure

?- point(X, Y) = point(1, 2, 3).
false.

In this example, the unification fails because the terms have different structures. point(X, Y) has two arguments, while point(1, 2, 3) has three.

Advanced Unification

Let's explore some more complex scenarios involving lists and nested structures.

Example 6: Unifying Lists

?- [H|T] = [1, 2, 3].
H = 1,
T = [2, 3].

In this example, the list [H|T] is unified with [1, 2, 3]. The head H is unified with 1, and the tail T is unified with [2, 3].

Example 7: Nested Structures

?- person(name(First, Last), age(Age)) = person(name(john, doe), age(30)).
First = john,
Last = doe,
Age = 30.

Here, the nested structure person(name(First, Last), age(Age)) is unified with person(name(john, doe), age(30)). The variables First, Last, and Age are unified with john, doe, and 30, respectively.

Exercises

  1. Unify the following terms and provide the resulting substitutions:

    • ?- X = Y, Y = 3.
    • ?- f(a, X) = f(Y, b).
    • ?- [A, B | T] = [1, 2, 3, 4].

  2. Determine if the following unifications succeed or fail:

    • ?- g(X, Y) = g(a, b).
    • ?- h(X, X) = h(a, b).
    • ?- [H|T] = [1].

Conclusion

Unification is a powerful and essential concept in Prolog, enabling the language to match patterns and solve queries. By understanding and practicing unification, you can harness the full potential of Prolog's logical programming capabilities. Remember to experiment with different terms and structures to deepen your understanding of how unification works in various scenarios.
