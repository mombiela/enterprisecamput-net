Introduction

In Ada, generic procedures and functions allow you to write flexible and reusable code. Generics enable you to define algorithms and data structures that can operate with any data type. This section will guide you through the basics of generics in Ada, from defining generic procedures and functions to using them in your programs.

Key Concepts

  • Generics: Templates that allow you to write procedures and functions that can operate on different data types.
  • Generic Parameters: Placeholders for types, subprograms, or values that are specified when the generic unit is instantiated.
  • Instantiation: The process of creating a specific instance of a generic unit by providing actual parameters.

Defining Generic Procedures and Functions

To define a generic procedure or function, you use the generic keyword followed by the list of generic parameters. Here is the basic syntax:

generic
   -- List of generic parameters
procedure Generic_Procedure (Parameter : in Type);

Example: Generic Swap Procedure

Let's start with a simple example: a generic procedure to swap two values of any type.

generic
   type Element_Type is private;
procedure Swap (X, Y : in out Element_Type);

procedure Swap (X, Y : in out Element_Type) is
   Temp : Element_Type;
begin
   Temp := X;
   X := Y;
   Y := Temp;
end Swap;

In this example:

  • Element_Type is a generic parameter representing any type.
  • Swap is a procedure that takes two parameters of type Element_Type and swaps their values.

Instantiating the Generic Procedure

To use the generic Swap procedure, you need to instantiate it with a specific type.

procedure Swap_Integer is new Swap (Integer);

procedure Test_Swap is
   A, B : Integer := 1;
begin
   Swap_Integer (A, B);
   -- Now A = 2 and B = 1
end Test_Swap;

Generic Functions

Generic functions are defined similarly to generic procedures. Here is the basic syntax:

generic
   -- List of generic parameters
function Generic_Function (Parameter : in Type) return Return_Type;

Example: Generic Max Function

Let's create a generic function to find the maximum of two values.

generic
   type Element_Type is private;
   with function "<" (Left, Right : Element_Type) return Boolean;
function Max (X, Y : Element_Type) return Element_Type;

function Max (X, Y : Element_Type) return Element_Type is
begin
   if X < Y then
      return Y;
   else
      return X;
   end if;
end Max;

In this example:

  • Element_Type is a generic parameter representing any type.
  • The function "<" is a generic parameter representing a comparison function for Element_Type.
  • Max is a function that returns the maximum of two values of type Element_Type.

Instantiating the Generic Function

To use the generic Max function, you need to instantiate it with a specific type and comparison function.

function Integer_Less (Left, Right : Integer) return Boolean is
begin
   return Left < Right;
end Integer_Less;

function Max_Integer is new Max (Integer, Integer_Less);

procedure Test_Max is
   A, B : Integer := 1;
   C : Integer;
begin
   C := Max_Integer (A, B);
   -- Now C = 2
end Test_Max;

Summary

Generic procedures and functions in Ada provide a powerful way to write reusable and type-safe code. By defining generic units, you can create algorithms and data structures that work with any data type, enhancing the flexibility and maintainability of your programs. Remember to:

  • Define generic parameters using the generic keyword.
  • Instantiate generic units with specific types and subprograms.
  • Use generics to create flexible and reusable code components.

With these tools, you can write more general and adaptable Ada programs, making your codebase more robust and easier to maintain.
