Polymorphism in Ada
Introduction to Generics ►

Polymorphism is a fundamental concept in object-oriented programming that allows objects of different types to be treated as objects of a common super type. In Ada, polymorphism is achieved through the use of tagged types and dynamic dispatching. This topic will guide you from the basics to advanced usage of polymorphism in Ada.

Introduction to Polymorphism

  • Definition: Polymorphism allows methods to operate on objects of different types through a common interface.
  • Key Concepts:
    • Tagged Types: Special types in Ada that support inheritance and polymorphism.
    • Dynamic Dispatching: The mechanism by which the correct method is called for an object, based on its actual type at runtime.

Tagged Types

Tagged types are the foundation of polymorphism in Ada. They allow the creation of a hierarchy of types where derived types inherit properties and methods from their parent types.

Defining a Tagged Type

package Shapes is
   type Shape is tagged record
      X, Y : Float;
   end record;

   procedure Draw(S : in Shape) is abstract;
end Shapes;
  • Explanation:
    • Shape is a tagged type with two fields, X and Y.
    • The Draw procedure is declared as abstract, meaning it must be overridden in derived types.

Deriving from a Tagged Type

package Shapes.Circles is
   type Circle is new Shape with record
      Radius : Float;
   end record;

   procedure Draw(C : in Circle);
end Shapes.Circles;
  • Explanation:
    • Circle is a new type derived from Shape, adding a Radius field.
    • The Draw procedure is overridden to provide specific behavior for Circle.

Dynamic Dispatching

Dynamic dispatching allows the correct method to be called based on the actual type of the object at runtime.

Example of Dynamic Dispatching

with Shapes; use Shapes;
with Shapes.Circles; use Shapes.Circles;

procedure Main is
   S : Shape'Class := Circle'(X => 0.0, Y => 0.0, Radius => 5.0);
begin
   Draw(S); -- Calls Circle's Draw procedure
end Main;
  • Explanation:
    • S is declared as Shape'Class, which is a class-wide type that can hold any type derived from Shape.
    • The Draw procedure called on S will dynamically dispatch to the Draw procedure for Circle.

Advanced Polymorphism

Class-Wide Operations

Class-wide operations allow you to write procedures that can operate on any type within a class hierarchy.

procedure Move(S : in out Shape'Class; DX, DY : Float) is
begin
   S.X := S.X + DX;
   S.Y := S.Y + DY;
end Move;
  • Explanation:
    • The Move procedure can operate on any object of type Shape or any of its derivatives.

Abstract Types and Interfaces

Abstract types and interfaces provide a way to define common behavior without providing a full implementation.

package Shapes is
   type Shape is abstract tagged record
      X, Y : Float;
   end record;

   procedure Draw(S : in Shape) is abstract;
end Shapes;
  • Explanation:
    • Shape is declared as an abstract type, meaning it cannot be instantiated directly.
    • The Draw procedure is also abstract, requiring derived types to provide an implementation.

Conclusion

Polymorphism in Ada is a powerful feature that allows for flexible and reusable code. By using tagged types and dynamic dispatching, you can create a hierarchy of types that share common behavior while allowing for specific implementations in derived types. Understanding and utilizing polymorphism will enable you to write more modular and maintainable Ada programs.
