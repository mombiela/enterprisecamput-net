Introduction to Records

Records in Ada are a way to group related data together. They are similar to structs in C or objects in other programming languages. Records allow you to define a composite data type that can contain multiple elements, each with its own type.

Key Concepts

  • Composite Data Type: A record is a composite data type that can hold multiple elements.
  • Fields: The individual elements within a record are called fields.
  • Type Declaration: You need to declare a record type before you can use it.

Defining a Record

To define a record in Ada, you use the type keyword followed by the record name and the record keyword. Each field within the record is defined with its name and type.

Example

type Person is record
    Name  : String(1..50);
    Age   : Integer;
    Email : String(1..100);
end record;

In this example:

  • Person is the name of the record type.
  • Name, Age, and Email are fields within the record.
  • Each field has a specific type (String or Integer).

Creating and Using Records

Once you have defined a record type, you can create variables of that type and access their fields.

Example

procedure Main is
    John : Person;
begin
    John.Name  := "John Doe";
    John.Age   := 30;
    John.Email := "john.doe@example.com";
    
    Ada.Text_IO.Put_Line("Name: " & John.Name);
    Ada.Text_IO.Put_Line("Age: " & Integer'Image(John.Age));
    Ada.Text_IO.Put_Line("Email: " & John.Email);
end Main;

In this example:

  • A variable John of type Person is created.
  • The fields of John are assigned values.
  • The values of the fields are printed to the console.

Nested Records

Records can contain other records as fields, allowing you to create complex data structures.

Example

type Address is record
    Street : String(1..100);
    City   : String(1..50);
    Zip    : String(1..10);
end record;

type Employee is record
    Name    : String(1..50);
    Age     : Integer;
    Contact : Address;
end record;

procedure Main is
    Jane : Employee;
begin
    Jane.Name := "Jane Smith";
    Jane.Age  := 28;
    Jane.Contact.Street := "123 Main St";
    Jane.Contact.City   := "Anytown";
    Jane.Contact.Zip    := "12345";
    
    Ada.Text_IO.Put_Line("Name: " & Jane.Name);
    Ada.Text_IO.Put_Line("Age: " & Integer'Image(Jane.Age));
    Ada.Text_IO.Put_Line("Address: " & Jane.Contact.Street & ", " & Jane.Contact.City & " " & Jane.Contact.Zip);
end Main;

In this example:

  • Address is a record type that contains fields for street, city, and zip code.
  • Employee is a record type that contains a field of type Address.
  • The Jane variable of type Employee is created and its fields are assigned values, including the nested Contact record.

Record Operations

Assigning Records

You can assign one record to another if they are of the same type.

John := Jane;

Comparing Records

You can compare records for equality or inequality.

if John = Jane then
    Ada.Text_IO.Put_Line("John and Jane are the same.");
else
    Ada.Text_IO.Put_Line("John and Jane are different.");
end if;

Updating Fields

You can update individual fields of a record.

John.Age := 31;

Conclusion

Records in Ada provide a powerful way to group related data together. They are flexible and can be nested to create complex data structures. Understanding how to define, create, and manipulate records is essential for effective Ada programming. Practice creating and using records to become comfortable with this important feature of the language.
