Introduction

Parameter passing is a fundamental concept in programming that allows you to pass data to subprograms (procedures and functions). In Ada, parameter passing is versatile and provides several modes to control how data is passed and manipulated.

Parameter Modes

Ada supports three primary parameter modes:

  • In: Passes data into the subprogram.
  • Out: Passes data out of the subprogram.
  • In Out: Passes data both into and out of the subprogram.

In Mode

The in mode is used when you want to pass data to a subprogram without allowing the subprogram to modify the original data.

procedure Print_Value(Value : in Integer) is
begin
   Put_Line("The value is: " & Integer'Image(Value));
end Print_Value;
  • Key Concept: The in parameter is read-only within the subprogram.

Out Mode

The out mode is used when you want the subprogram to modify the parameter and return the result.

procedure Increment(Value : out Integer) is
begin
   Value := Value + 1;
end Increment;
  • Key Concept: The out parameter is write-only within the subprogram. It is used to return a value to the caller.

In Out Mode

The in out mode allows the subprogram to read and modify the parameter.

procedure Double_Value(Value : in out Integer) is
begin
   Value := Value * 2;
end Double_Value;
  • Key Concept: The in out parameter can be read and modified within the subprogram.

Parameter Passing Mechanisms

Ada uses different mechanisms to implement parameter passing:

  • By Copy: The actual value is copied to the subprogram.
  • By Reference: A reference to the actual data is passed to the subprogram.

By Copy

Typically used for in parameters. The subprogram works with a copy of the actual parameter.

procedure Show_Copy(Value : in Integer) is
   Temp : Integer := Value;
begin
   Temp := Temp + 1;
   Put_Line("Temp value: " & Integer'Image(Temp));
   Put_Line("Original value: " & Integer'Image(Value));
end Show_Copy;

By Reference

Typically used for out and in out parameters. The subprogram works directly with the actual parameter.

procedure Modify_Reference(Value : in out Integer) is
begin
   Value := Value + 10;
end Modify_Reference;

Examples and Exercises

Example 1: Using `in` Mode

procedure Display_Sum(A, B : in Integer) is
begin
   Put_Line("Sum: " & Integer'Image(A + B));
end Display_Sum;

-- Usage
begin
   Display_Sum(5, 10);  -- Output: Sum: 15
end;

Example 2: Using `out` Mode

procedure Initialize(Value : out Integer) is
begin
   Value := 0;
end Initialize;

-- Usage
declare
   Num : Integer;
begin
   Initialize(Num);
   Put_Line("Initialized value: " & Integer'Image(Num));  -- Output: Initialized value: 0
end;

Example 3: Using `in out` Mode

procedure Square(Value : in out Integer) is
begin
   Value := Value * Value;
end Square;

-- Usage
declare
   Num : Integer := 4;
begin
   Square(Num);
   Put_Line("Squared value: " & Integer'Image(Num));  -- Output: Squared value: 16
end;

Exercise 1

Write a procedure Swap that takes two in out parameters and swaps their values.

procedure Swap(A, B : in out Integer) is
   Temp : Integer;
begin
   Temp := A;
   A := B;
   B := Temp;
end Swap;

-- Test your procedure
declare
   X : Integer := 5;
   Y : Integer := 10;
begin
   Swap(X, Y);
   Put_Line("X: " & Integer'Image(X));  -- Output: X: 10
   Put_Line("Y: " & Integer'Image(Y));  -- Output: Y: 5
end;

Conclusion

Understanding parameter passing in Ada is crucial for writing efficient and effective subprograms. By mastering the in, out, and in out modes, you can control how data is passed and manipulated within your programs. Practice with the provided examples and exercises to solidify your understanding of these concepts.
