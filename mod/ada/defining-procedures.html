Introduction

In Ada, procedures are a fundamental way to encapsulate and organize code. They allow you to define reusable blocks of code that can be called with different parameters. This section will cover the basics of defining procedures, including syntax, parameter passing, and examples.

Basic Syntax of Procedures

A procedure in Ada is defined using the procedure keyword followed by the procedure name and an optional list of parameters. The body of the procedure is enclosed between begin and end.

Example

procedure Greet is
begin
   Put_Line("Hello, World!");
end Greet;

In this example, Greet is a simple procedure that prints "Hello, World!" to the console.

Parameters in Procedures

Procedures can take parameters to make them more flexible and reusable. Parameters can be of different types and can be passed in different modes: in, out, and in out.

Parameter Modes

  • in: The parameter is read-only within the procedure.
  • out: The parameter is write-only within the procedure.
  • in out: The parameter can be read and modified within the procedure.

Example with Parameters

procedure Add_Numbers(X : in Integer; Y : in Integer; Sum : out Integer) is
begin
   Sum := X + Y;
end Add_Numbers;

In this example, Add_Numbers takes two input parameters X and Y, and an output parameter Sum. The procedure calculates the sum of X and Y and assigns it to Sum.

Calling Procedures

To call a procedure, you simply use its name followed by the actual parameters in parentheses.

Example

declare
   A, B, Result : Integer := 0;
begin
   A := 5;
   B := 10;
   Add_Numbers(A, B, Result);
   Put_Line("The sum is: " & Integer'Image(Result));
end;

In this example, the Add_Numbers procedure is called with A and B as input parameters, and Result as the output parameter. The result is then printed to the console.

Advanced Topics

Nested Procedures

Procedures can be nested within other procedures, allowing for more complex and organized code structures.

Example

procedure Outer is
   procedure Inner is
   begin
      Put_Line("This is the inner procedure.");
   end Inner;
begin
   Put_Line("This is the outer procedure.");
   Inner;
end Outer;

In this example, Inner is a nested procedure within Outer. When Outer is called, it first prints a message and then calls Inner.

Recursion

Ada supports recursive procedures, where a procedure calls itself.

Example

procedure Factorial(N : in Integer; Result : out Integer) is
begin
   if N = 0 then
      Result := 1;
   else
      declare
         Temp : Integer;
      begin
         Factorial(N - 1, Temp);
         Result := N * Temp;
      end;
   end if;
end Factorial;

In this example, Factorial is a recursive procedure that calculates the factorial of a number N.

Conclusion

Procedures in Ada are a powerful tool for organizing and reusing code. By understanding the basics of defining procedures, parameter passing, and advanced topics like nested procedures and recursion, you can write more efficient and maintainable Ada programs. Practice defining and calling procedures to become more comfortable with these concepts.
