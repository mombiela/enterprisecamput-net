Introduction to Elastic Load Balancing (ELB)

Elastic Load Balancing (ELB) is a service provided by AWS that automatically distributes incoming application traffic across multiple targets, such as EC2 instances, containers, and IP addresses. ELB helps ensure a high level of availability and fault tolerance for your applications.

Key Concepts

  • Load Balancer: A device that distributes network or application traffic across a number of servers.
  • Target: The destination for the traffic, such as EC2 instances, containers, or IP addresses.
  • Listener: A process that checks for connection requests using the protocol and port you configure.
  • Target Group: A group of targets that a load balancer routes requests to.

Types of Load Balancers

AWS offers three types of load balancers:

  1. Application Load Balancer (ALB)

    • Best suited for HTTP and HTTPS traffic.
    • Operates at the application layer (Layer 7).
    • Supports advanced routing, such as path-based and host-based routing.

  2. Network Load Balancer (NLB)

    • Best suited for TCP, UDP, and TLS traffic.
    • Operates at the transport layer (Layer 4).
    • Capable of handling millions of requests per second while maintaining ultra-low latencies.

  3. Classic Load Balancer (CLB)

    • Operates at both the application layer (Layer 7) and the transport layer (Layer 4).
    • Suitable for applications built within the EC2-Classic network.

Comparison Table

| Feature | Application Load Balancer (ALB) | Network Load Balancer (NLB) | Classic Load Balancer (CLB) | |-----------------------------|---------------------------------|-----------------------------|-----------------------------| | Protocols Supported | HTTP, HTTPS | TCP, UDP, TLS | HTTP, HTTPS, TCP, SSL | | Layer | 7 | 4 | 4 and 7 | | Advanced Routing | Yes | No | No | | WebSocket Support | Yes | No | No | | Static IP Support | No | Yes | No | | Health Checks | HTTP, HTTPS | TCP, HTTP, HTTPS | TCP, HTTP, HTTPS |

Setting Up an Application Load Balancer (ALB)

Step-by-Step Guide

  1. Create a Load Balancer

    • Navigate to the EC2 Dashboard.
    • Select "Load Balancers" from the left-hand menu.
    • Click on "Create Load Balancer" and choose "Application Load Balancer".

  2. Configure Load Balancer

    • Provide a name for your load balancer.
    • Select the scheme (Internet-facing or Internal).
    • Choose the IP address type (IPv4 or Dualstack).
    • Configure listeners (default is HTTP on port 80).

  3. Configure Security Settings

    • If using HTTPS, configure SSL certificates.

  4. Configure Security Groups

    • Select or create a security group that allows inbound traffic on the listener ports.

  5. Configure Routing

    • Create a target group.
    • Choose the target type (Instances, IP addresses, or Lambda functions).
    • Configure health checks (protocol, path, and port).

  6. Register Targets

    • Add the instances or IP addresses that you want to route traffic to.

  7. Review and Create

    • Review your settings and click "Create".

Example Code: Creating an ALB using AWS CLI

aws elbv2 create-load-balancer \
    --name my-load-balancer \
    --subnets subnet-12345678 subnet-87654321 \
    --security-groups sg-12345678 \
    --scheme internet-facing \
    --type application

Example Code: Registering Targets using AWS CLI

aws elbv2 register-targets \
    --target-group-arn arn:aws:elasticloadbalancing:region:account-id:targetgroup/my-targets/1234567890123456 \
    --targets Id=i-1234567890abcdef0 Id=i-0abcdef1234567890

Advanced Features of ELB

Sticky Sessions

Sticky sessions (session affinity) allow you to bind a user's session to a specific target within the target group. This ensures that all requests from a user during a session are sent to the same target.

Cross-Zone Load Balancing

Cross-zone load balancing distributes incoming requests evenly across all targets in all enabled Availability Zones. This improves the fault tolerance and availability of your application.

SSL Termination

SSL termination decrypts incoming SSL requests at the load balancer before sending them to the targets. This reduces the load on your targets and simplifies SSL certificate management.

Auto Scaling Integration

ELB integrates seamlessly with Auto Scaling, allowing you to automatically add or remove targets based on demand. This ensures that your application can handle varying levels of traffic without manual intervention.

Monitoring and Logging

CloudWatch Metrics

ELB provides several CloudWatch metrics to monitor the performance and health of your load balancer, such as:

  • RequestCount
  • HealthyHostCount
  • UnHealthyHostCount
  • Latency

Access Logs

Access logs capture detailed information about requests sent to your load balancer, including the time the request was received, the client's IP address, and the request path. These logs can be stored in S3 for analysis.

Example Code: Enabling Access Logs using AWS CLI

aws elbv2 modify-load-balancer-attributes \
    --load-balancer-arn arn:aws:elasticloadbalancing:region:account-id:loadbalancer/app/my-load-balancer/50dc6c495c0c9188 \
    --attributes Key=access_logs.s3.enabled,Value=true Key=access_logs.s3.bucket,Value=my-logs-bucket Key=access_logs.s3.prefix,Value=my-app

Conclusion

Elastic Load Balancing (ELB) is a powerful service that enhances the availability, fault tolerance, and scalability of your applications. By understanding the different types of load balancers, setting up and configuring an ALB, and leveraging advanced features, you can ensure that your applications are robust and can handle varying levels of traffic efficiently. Monitoring and logging capabilities further help in maintaining the health and performance of your applications.
