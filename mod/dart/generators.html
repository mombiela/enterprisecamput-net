Generators in Dart
Exception Handling ►

Generators are a powerful feature in Dart that allow you to produce a sequence of values lazily, meaning values are generated on demand rather than all at once. This can be particularly useful when working with large datasets or streams of data.

Key Concepts

  • Generators: Functions that produce a sequence of values.
  • Synchronous Generators: Use sync* and yield to produce values.
  • Asynchronous Generators: Use async* and yield to produce values asynchronously.
  • yield: Keyword used to produce a value in a generator function.
  • *yield`: Keyword used to delegate part of the generation process to another generator.

Synchronous Generators

Synchronous generators are used to produce values in a sequence that can be iterated over synchronously.

Example

Iterable<int> syncGenerator(int n) sync* {
  for (int i = 0; i < n; i++) {
    yield i;
  }
}

void main() {
  var numbers = syncGenerator(5);
  for (var number in numbers) {
    print(number);
  }
}

Explanation

  • The syncGenerator function is a synchronous generator that produces integers from 0 to n-1.
  • The sync* keyword indicates that this function is a synchronous generator.
  • The yield keyword is used to produce each value in the sequence.
  • In the main function, we call syncGenerator and iterate over the produced values using a for loop.

Asynchronous Generators

Asynchronous generators are used to produce values in a sequence that can be iterated over asynchronously, which is useful for handling streams of data or performing asynchronous operations.

Example

Stream<int> asyncGenerator(int n) async* {
  for (int i = 0; i < n; i++) {
    await Future.delayed(Duration(seconds: 1)); // Simulate async operation
    yield i;
  }
}

void main() async {
  var numbers = asyncGenerator(5);
  await for (var number in numbers) {
    print(number);
  }
}

Explanation

  • The asyncGenerator function is an asynchronous generator that produces integers from 0 to n-1.
  • The async* keyword indicates that this function is an asynchronous generator.
  • The yield keyword is used to produce each value in the sequence.
  • The await keyword is used to simulate an asynchronous operation (e.g., a delay).
  • In the main function, we call asyncGenerator and iterate over the produced values using an await for loop.

yield* Keyword

The yield* keyword is used to delegate part of the generation process to another generator.

Example

Iterable<int> subGenerator(int n) sync* {
  for (int i = 0; i < n; i++) {
    yield i;
  }
}

Iterable<int> mainGenerator(int n) sync* {
  yield* subGenerator(n);
  yield n;
}

void main() {
  var numbers = mainGenerator(5);
  for (var number in numbers) {
    print(number);
  }
}

Explanation

  • The subGenerator function produces integers from 0 to n-1.
  • The mainGenerator function uses yield* to delegate the generation of values to subGenerator and then yields an additional value n.
  • In the main function, we call mainGenerator and iterate over the produced values using a for loop.

Conclusion

Generators in Dart provide a flexible and efficient way to produce sequences of values, either synchronously or asynchronously. By using sync* and yield for synchronous generators, and async* and yield for asynchronous generators, you can handle a wide range of data generation scenarios. The yield* keyword further enhances this capability by allowing delegation to other generators, making your code more modular and easier to manage. Understanding and utilizing generators can greatly improve the performance and readability of your Dart programs, especially when dealing with large datasets or asynchronous data streams.
