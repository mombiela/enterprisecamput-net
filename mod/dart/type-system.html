Introduction to Dart's Type System

Dart is a statically typed language, which means that the type of a variable is known at compile time. This helps catch errors early and improves code quality and performance.

Key Concepts

  • Static Typing: Types are checked at compile time.
  • Type Inference: Dart can infer types, so you don't always need to explicitly declare them.
  • Dynamic Type: Dart also supports dynamic typing using the dynamic keyword.

Basic Types

Dart provides several built-in types that are commonly used in programming.

Common Types

  • int: Represents integer values.
  • double: Represents floating-point values.
  • String: Represents a sequence of characters.
  • bool: Represents boolean values (true or false).
  • List: Represents a collection of ordered objects.
  • Map: Represents a collection of key-value pairs.

Code Examples

void main() {
  int age = 25;
  double height = 5.9;
  String name = 'Alice';
  bool isStudent = true;
  List<int> numbers = [1, 2, 3, 4, 5];
  Map<String, String> capitals = {
    'USA': 'Washington, D.C.',
    'France': 'Paris',
    'Japan': 'Tokyo'
  };

  print('Name: $name, Age: $age, Height: $height, Student: $isStudent');
  print('Numbers: $numbers');
  print('Capitals: $capitals');
}

Type Inference

Dart can infer the type of a variable based on the assigned value, which makes the code cleaner and easier to read.

Code Examples

void main() {
  var age = 25; // Dart infers that age is of type int
  var height = 5.9; // Dart infers that height is of type double
  var name = 'Alice'; // Dart infers that name is of type String
  var isStudent = true; // Dart infers that isStudent is of type bool

  print('Name: $name, Age: $age, Height: $height, Student: $isStudent');
}

Dynamic Type

The dynamic keyword allows a variable to hold values of any type. This is useful when the type of the variable is not known at compile time.

Code Examples

void main() {
  dynamic variable = 25;
  print('Variable: $variable, Type: ${variable.runtimeType}');

  variable = 'Hello';
  print('Variable: $variable, Type: ${variable.runtimeType}');

  variable = true;
  print('Variable: $variable, Type: ${variable.runtimeType}');
}

Type Safety

Dart ensures type safety by preventing operations that are not allowed on certain types.

Code Examples

void main() {
  int age = 25;
  // age = 'twenty-five'; // This will cause a compile-time error

  String name = 'Alice';
  // name = 123; // This will cause a compile-time error

  print('Name: $name, Age: $age');
}

Type Casting

Sometimes, you may need to convert a variable from one type to another. Dart provides several ways to perform type casting.

Code Examples

void main() {
  // String to int
  String ageStr = '25';
  int age = int.parse(ageStr);
  print('Age: $age');

  // int to String
  int year = 2023;
  String yearStr = year.toString();
  print('Year: $yearStr');

  // double to int
  double height = 5.9;
  int roundedHeight = height.toInt();
  print('Rounded Height: $roundedHeight');
}

Null Safety

Dart has a sound null safety feature that helps prevent null reference errors. By default, variables cannot be null unless explicitly declared as nullable.

Code Examples

void main() {
  int? nullableAge;
  nullableAge = null; // This is allowed

  int nonNullableAge = 25;
  // nonNullableAge = null; // This will cause a compile-time error

  print('Nullable Age: $nullableAge, Non-Nullable Age: $nonNullableAge');
}

Summary

In this section, we covered the essentials of Dart's type system, including:

  • Basic types and their usage.
  • Type inference for cleaner code.
  • The dynamic type for flexibility.
  • Type safety to prevent errors.
  • Type casting for converting between types.
  • Null safety to avoid null reference errors.

Understanding Dart's type system is crucial for writing robust and error-free code. As you progress, you'll find that these concepts form the foundation for more advanced Dart programming techniques.
