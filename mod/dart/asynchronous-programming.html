Asynchronous Programming in Dart
Streams ►

Asynchronous programming is a key concept in modern software development, allowing applications to handle multiple tasks simultaneously without blocking the main execution thread. In Dart, asynchronous programming is primarily handled using Future, async, and await.

Introduction to Asynchronous Programming

  • Synchronous vs Asynchronous:

    • Synchronous: Tasks are executed one after another. Each task waits for the previous one to complete.
    • Asynchronous: Tasks can be executed concurrently, allowing the program to continue running other tasks while waiting for some operations to complete.

  • Why Asynchronous Programming?:

    • Improves performance by not blocking the main thread.
    • Essential for tasks like network requests, file I/O, and timers.

Futures in Dart

A Future represents a potential value or error that will be available at some time in the future.

  • Creating a Future:

    Future<String> fetchUserOrder() {
  // Imagine that this function is fetching user order from a database or an API
  return Future.delayed(Duration(seconds: 2), () => 'Large Latte');
}

  • Handling a Future:

    void main() {
  fetchUserOrder().then((order) {
    print('Your order is: $order');
  }).catchError((error) {
    print('Something went wrong: $error');
  });
}

Async and Await

The async and await keywords make asynchronous code look more like synchronous code, improving readability.

  • Using async and await:

    Future<void> printOrderMessage() async {
  var order = await fetchUserOrder();
  print('Your order is: $order');
}

void main() {
  printOrderMessage();
}

  • Error Handling with async and await:

    Future<void> printOrderMessage() async {
  try {
    var order = await fetchUserOrder();
    print('Your order is: $order');
  } catch (error) {
    print('Something went wrong: $error');
  }
}

void main() {
  printOrderMessage();
}

Streams in Dart

Streams provide a way to handle a sequence of asynchronous events.

  • Creating a Stream:

    Stream<int> countStream(int max) async* {
  for (int i = 1; i <= max; i++) {
    await Future.delayed(Duration(seconds: 1));
    yield i;
  }
}

  • Listening to a Stream:

    void main() {
  var stream = countStream(5);
  stream.listen((count) {
    print(count);
  });
}

  • Handling Errors in Streams:

    void main() {
  var stream = countStream(5);
  stream.listen(
    (count) {
      print(count);
    },
    onError: (error) {
      print('Error: $error');
    },
    onDone: () {
      print('Stream is done');
    },
  );
}

Comparing Futures and Streams

Feature Future Stream
Single Value Returns a single value or error Can emit multiple values or errors over time
Completion Completes with a value or error Can continue emitting values until done
Usage Ideal for single async operations Ideal for handling sequences of async events

Conclusion

Asynchronous programming in Dart is a powerful tool that allows developers to write efficient, non-blocking code. By understanding and utilizing Future, async, await, and Stream, you can handle asynchronous tasks effectively, making your applications more responsive and performant. Practice with the provided examples and exercises to solidify your understanding of these concepts.
