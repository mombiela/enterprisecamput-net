Introduction

In this topic, we will learn about handling requests and responses in the context of a REST API. This knowledge is fundamental for any IT professional working with web services, as RESTful APIs are one of the most common forms of communication between systems.

Key Concepts

  • HTTP Requests: Requests sent by the client to the server to perform some action.
  • HTTP Responses: Responses sent by the server to the client after processing a request.
  • HTTP Methods: GET, POST, PUT, DELETE, PATCH, etc., which define the action to be performed.
  • HTTP Status Codes: Indicate the result of the request (200 OK, 404 Not Found, 500 Internal Server Error, etc.).
  • HTTP Headers: Additional information sent in requests and responses (Content-Type, Authorization, etc.).
  • Request/Response Body: Data sent in the body of the request or response, generally in JSON format.

HTTP Requests

Structure of an HTTP Request

A typical HTTP request includes:

  • Request Line: Contains the HTTP method, the URL, and the protocol version.
  • Headers: Additional information about the request.
  • Body: Data sent with the request (optional, depends on the HTTP method).

Example of an HTTP GET request:

GET /api/users HTTP/1.1
Host: example.com
Authorization: Bearer token123
Accept: application/json

Example of an HTTP POST request:

POST /api/users HTTP/1.1
Host: example.com
Content-Type: application/json
Authorization: Bearer token123

{
  "name": "John Doe",
  "email": "john.doe@example.com"
}

HTTP Responses

Structure of an HTTP Response

A typical HTTP response includes:

  • Status Line: Contains the protocol version, the status code, and the status message.
  • Headers: Additional information about the response.
  • Body: Data sent with the response (optional, depends on the status code).

Example of an HTTP 200 OK response:

HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": 1,
  "name": "John Doe",
  "email": "john.doe@example.com"
}

Example of an HTTP 404 Not Found response:

HTTP/1.1 404 Not Found
Content-Type: application/json

{
  "error": "User not found"
}

HTTP Methods

Main HTTP Methods

  • GET: Retrieves information from the server.
  • POST: Sends data to the server to create a new resource.
  • PUT: Updates an existing resource on the server.
  • DELETE: Deletes a resource from the server.
  • PATCH: Applies partial modifications to a resource.
Method Description Idempotent Safe
GET Retrieves information Yes Yes
POST Creates a new resource No No
PUT Updates an existing resource Yes No
DELETE Deletes a resource Yes No
PATCH Partially modifies a resource No No

HTTP Status Codes

Main Status Codes

  • 2xx (Success): The request was received, understood, and accepted.
    • 200 OK: The request was successful.
    • 201 Created: A new resource was created.
  • 4xx (Client Errors): The request contains incorrect syntax or cannot be processed.
    • 400 Bad Request: The request is invalid.
    • 401 Unauthorized: Authentication is required.
    • 404 Not Found: The resource was not found.
  • 5xx (Server Errors): The server failed to fulfill a seemingly valid request.
    • 500 Internal Server Error: Generic server error.
    • 503 Service Unavailable: The service is unavailable.

HTTP Headers

Main Headers

  • Content-Type: Indicates the type of content of the request/response body.
  • Authorization: Contains credentials to authenticate the client.
  • Accept: Indicates what types of content the client can process.

Example of header usage:

GET /api/users HTTP/1.1
Host: example.com
Authorization: Bearer token123
Accept: application/json

Conclusion

Handling requests and responses is a crucial aspect of developing and consuming RESTful APIs. Understanding the structure of requests and responses, HTTP methods, status codes, and headers is essential for effectively interacting with any API. With this foundation, you will be better prepared to develop and consume web services efficiently and effectively.
