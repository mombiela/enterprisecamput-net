Introduction

In the context of REST APIs, HTTP methods are fundamental for defining the operations that can be performed on resources. Each HTTP method has a specific purpose and is used to perform different types of actions on an API's resources. The most common methods are GET, POST, PUT, DELETE, PATCH, OPTIONS, and HEAD.

Common HTTP Methods

GET

  • Purpose: Retrieve data from a server.
  • Idempotent: Yes (multiple requests do not change the state of the resource).
  • Safe: Yes (does not modify the resource).

Example of use:

GET /api/users/123 HTTP/1.1
Host: example.com

Explanation: This example requests information about the user with ID 123.

POST

  • Purpose: Send data to the server to create a new resource.
  • Idempotent: No (multiple requests can create multiple resources).
  • Safe: No (modifies the state of the server).

Example of use:

POST /api/users HTTP/1.1
Host: example.com
Content-Type: application/json

{
  "name": "John Doe",
  "email": "john.doe@example.com"
}

Explanation: This example creates a new user with the name "John Doe" and the email "john.doe@example.com".

PUT

  • Purpose: Update an existing resource or create one if it does not exist.
  • Idempotent: Yes (multiple requests have the same effect as a single one).
  • Safe: No (modifies the state of the server).

Example of use:

PUT /api/users/123 HTTP/1.1
Host: example.com
Content-Type: application/json

{
  "name": "John Doe",
  "email": "john.doe@example.com"
}

Explanation: This example updates the information of the user with ID 123.

DELETE

  • Purpose: Delete an existing resource.
  • Idempotent: Yes (multiple requests have the same effect as a single one).
  • Safe: No (modifies the state of the server).

Example of use:

DELETE /api/users/123 HTTP/1.1
Host: example.com

Explanation: This example deletes the user with ID 123.

PATCH

  • Purpose: Apply partial modifications to a resource.
  • Idempotent: Not necessarily (depends on the implementation).
  • Safe: No (modifies the state of the server).

Example of use:

PATCH /api/users/123 HTTP/1.1
Host: example.com
Content-Type: application/json

{
  "email": "new.email@example.com"
}

Explanation: This example updates only the email of the user with ID 123.

OPTIONS

  • Purpose: Describe the communication options for the target resource.
  • Idempotent: Yes.
  • Safe: Yes.

Example of use:

OPTIONS /api/users HTTP/1.1
Host: example.com

Explanation: This example requests the available communication options for the /api/users resource.

HEAD

  • Purpose: Retrieve response headers identical to those of a GET request, but without the response body.
  • Idempotent: Yes.
  • Safe: Yes.

Example of use:

HEAD /api/users/123 HTTP/1.1
Host: example.com

Explanation: This example requests the response headers for the user with ID 123 without the response body.

Comparison of HTTP Methods

| Method | Purpose | Idempotent | Safe | |---------|----------------------------------|------------|------| | GET | Retrieve data | Yes | Yes | | POST | Create a new resource | No | No | | PUT | Update or create a resource | Yes | No | | DELETE | Delete a resource | Yes | No | | PATCH | Partially modify a resource | No | No | | OPTIONS | Describe communication options | Yes | Yes | | HEAD | Retrieve headers | Yes | Yes |

Conclusion

Understanding HTTP methods is essential for working effectively with REST APIs. Each method has a specific purpose and is used in different situations to interact with an API's resources. Make sure to use the appropriate method for each operation to maintain the integrity and efficiency of your API.
