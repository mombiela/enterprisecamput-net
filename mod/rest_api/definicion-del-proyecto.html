Introduction

In this section, we will define the project of a REST API that we will develop throughout the course. This project will serve as a practical example to understand the concepts and techniques necessary to build a robust and efficient RESTful API.

Project Objectives

  • Understand the fundamentals of REST APIs: We will learn what a REST API is, its principles, and how it differs from other types of APIs.
  • Design a RESTful API: We will cover best practices for designing an API that is easy to use and maintain.
  • Implement the API: We will use a specific programming language and framework to build our API.
  • Test and Document the API: We will learn how to test our API to ensure it works correctly and document it so that other developers can easily use it.

Project Description

The project consists of developing a RESTful API for a task management application. The API will allow users to:

  • Create tasks: Users will be able to add new tasks to their list.
  • Read tasks: Users will be able to view all their tasks or a specific task.
  • Update tasks: Users will be able to modify the details of an existing task.
  • Delete tasks: Users will be able to delete tasks they no longer need.

Project Requirements

Functional

  • Task CRUD: The API must support Create, Read, Update, and Delete (CRUD) operations for tasks.
  • Authentication and Authorization: The API must ensure that only authenticated users can access their tasks.
  • Data Validation: The API must validate input data to ensure it is correct and complete.

Non-Functional

  • Scalability: The API must be able to handle a large number of concurrent requests.
  • Maintainability: The code must be easy to understand and maintain.
  • Documentation: The API must be well-documented to facilitate its use by other developers.

API Structure

Endpoints

| HTTP Method | Endpoint | Description | |-------------|-----------------|---------------------------------------| | POST | /tasks | Create a new task | | GET | /tasks | Get all tasks | | GET | /tasks/{id} | Get a specific task by ID | | PUT | /tasks/{id} | Update a specific task by ID | | DELETE | /tasks/{id} | Delete a specific task by ID |

Code Example

Below is an example of how the endpoint to create a new task might look using Node.js and Express:

const express = require('express');
const app = express();
app.use(express.json());

let tasks = [];

app.post('/tasks', (req, res) => {
    const task = {
        id: tasks.length + 1,
        title: req.body.title,
        description: req.body.description,
        completed: false
    };
    tasks.push(task);
    res.status(201).send(task);
});

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
    console.log(`Server is running on port ${PORT}`);
});

In this example, we have defined an endpoint /tasks that allows users to create a new task. The task is added to a tasks array and returned to the client with a 201 (Created) status code.
