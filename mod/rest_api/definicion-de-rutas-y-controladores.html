Introduction

In the development of a REST API, the definition of routes and controllers is fundamental to organize and manage HTTP requests efficiently. Routes determine how resources are accessed, while controllers handle the logic associated with those routes.

Routes

Routes are the endpoints of the API that respond to HTTP requests. Each route is associated with a specific URL and an HTTP method (GET, POST, PUT, DELETE, etc.).

Key Concepts of Routes

  • Endpoint: The specific URL used to access a resource.
  • HTTP Method: Defines the action to be performed on the resource (GET, POST, PUT, DELETE).
  • Route Parameters: Variables included in the URL to identify specific resources.

Example of Routes

GET /users          # Get a list of users
GET /users/{id}     # Get a specific user by ID
POST /users         # Create a new user
PUT /users/{id}     # Update an existing user by ID
DELETE /users/{id}  # Delete a user by ID

Controllers

Controllers are responsible for handling the business logic associated with each route. Each controller generally contains methods that correspond to CRUD actions (Create, Read, Update, Delete).

Key Concepts of Controllers

  • Controller Method: A function that handles a specific HTTP request.
  • Separation of Responsibilities: Keeping business logic separate from route definitions to improve code maintainability.

Example of Controller in Node.js with Express

const express = require('express');
const router = express.Router();

// Controller to handle user requests
const userController = {
    getAllUsers: (req, res) => {
        // Logic to get all users
        res.send('List of users');
    },
    getUserById: (req, res) => {
        const userId = req.params.id;
        // Logic to get a user by ID
        res.send(`User with ID: ${userId}`);
    },
    createUser: (req, res) => {
        const newUser = req.body;
        // Logic to create a new user
        res.send('User created');
    },
    updateUser: (req, res) => {
        const userId = req.params.id;
        const updatedUser = req.body;
        // Logic to update an existing user
        res.send(`User with ID: ${userId} updated`);
    },
    deleteUser: (req, res) => {
        const userId = req.params.id;
        // Logic to delete a user
        res.send(`User with ID: ${userId} deleted`);
    }
};

// Definition of routes and assignment of controllers
router.get('/users', userController.getAllUsers);
router.get('/users/:id', userController.getUserById);
router.post('/users', userController.createUser);
router.put('/users/:id', userController.updateUser);
router.delete('/users/:id', userController.deleteUser);

module.exports = router;

Comparison of HTTP Methods

| HTTP Method | Description | Example of Use | |-------------|--------------------------------------|------------------------------| | GET | Get data from a server | GET /users | | POST | Send data to the server to create | POST /users | | PUT | Update existing data on the server | PUT /users/{id} | | DELETE | Delete data from the server | DELETE /users/{id} |

Conclusion

The definition of routes and controllers is essential for the development of a well-structured and maintainable REST API. Routes determine how resources are accessed, while controllers handle the business logic associated with those routes. Understanding these concepts and their practical implementation is crucial for any API developer.
