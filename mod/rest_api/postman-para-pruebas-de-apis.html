Introduction to Postman

Postman is a popular tool for API development and testing. It allows developers to send HTTP requests, save responses, and automate tests. It is especially useful for working with RESTful APIs.

Main Features of Postman

  • User-friendly Interface: Allows creating and managing HTTP requests intuitively.
  • Collections: Groups multiple requests in one place for easy organization.
  • Variables: Allows the use of variables to reuse values in different requests.
  • Scripts: Supports pre-request and test scripts to automate tasks.
  • Documentation: Automatically generates documentation from collections.

Installation and Configuration

To start using Postman, you first need to install and configure it.

Installation Steps

  1. Download Postman: Visit Postman and download the appropriate version for your operating system.
  2. Install: Follow the installation instructions specific to your operating system.
  3. Create an Account: Sign up or log in to sync your collections and settings.

Creating HTTP Requests

Postman allows creating different types of HTTP requests, such as GET, POST, PUT, DELETE, among others.

Example of a GET Request

To make a GET request to an API, follow these steps:

  1. Open Postman.
  2. Create a new request: Click on "New" and select "Request".
  3. Configure the request:
    • Select the HTTP method (GET).
    • Enter the API URL (e.g., https://jsonplaceholder.typicode.com/posts).
  4. Send the request: Click on "Send".
GET /posts HTTP/1.1
Host: jsonplaceholder.typicode.com

Example of a POST Request

To make a POST request, follow these steps:

  1. Create a new request: Click on "New" and select "Request".
  2. Configure the request:
    • Select the HTTP method (POST).
    • Enter the API URL (e.g., https://jsonplaceholder.typicode.com/posts).
    • In the "Body" tab, select "raw" and choose "JSON" as the content type.
    • Enter the request body in JSON format.
{
  "title": "foo",
  "body": "bar",
  "userId": 1
}
  1. Send the request: Click on "Send".
POST /posts HTTP/1.1
Host: jsonplaceholder.typicode.com
Content-Type: application/json

{
  "title": "foo",
  "body": "bar",
  "userId": 1
}

Using Collections and Variables

Collections and variables in Postman are powerful tools for organizing and reusing requests.

Creating Collections

  1. Create a new collection: Click on "New" and select "Collection".
  2. Add requests to the collection: Drag and drop existing requests or create new requests within the collection.

Using Variables

  1. Define variables: In the "Variables" tab of a collection, define the necessary variables.
  2. Use variables in requests: Use the syntax {{variableName}} to insert variables in URLs, headers, request bodies, etc.
{
  "title": "{{postTitle}}",
  "body": "{{postBody}}",
  "userId": {{userId}}
}

Automating Tests with Scripts

Postman allows writing scripts to automate tests and validate responses.

Pre-request Scripts

These scripts run before sending the request. You can use them to set up variables or modify the request.

pm.environment.set("currentTime", new Date().toISOString());

Test Scripts

These scripts run after receiving the response. You can use them to validate the response content.

pm.test("Status code is 200", function () {
    pm.response.to.have.status(200);
});

pm.test("Response time is less than 200ms", function () {
    pm.expect(pm.response.responseTime).to.be.below(200);
});

Conclusion

Postman is an essential tool for any developer working with APIs. Its intuitive interface, along with advanced features like collections, variables, and scripts, greatly facilitates the API development and testing process. By mastering Postman, you can improve the efficiency and quality of your API tests, ensuring that your applications work correctly and meet the expected requirements.
