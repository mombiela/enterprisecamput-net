Variables and Outputs in Terraform
Terraform Expressions ►

Introduction

In this section, we will explore how to use variables and outputs in Terraform. Variables allow you to parameterize your Terraform configurations, making them more flexible and reusable. Outputs enable you to extract and display information from your Terraform state.

Variables

What are Variables?

Variables in Terraform are used to parameterize your configurations. They allow you to define values that can be reused throughout your configuration files.

Defining Variables

Variables are defined in a .tf file using the variable keyword. Here is an example:

variable "instance_type" {
  description = "Type of EC2 instance"
  type        = string
  default     = "t2.micro"
}
  • description: A brief explanation of what the variable is for.
  • type: The type of the variable (e.g., string, number, list, map).
  • default: A default value for the variable if none is provided.

Using Variables

Once defined, variables can be used in your Terraform configuration files. Here is an example of how to use the instance_type variable:

resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = var.instance_type
}

Variable Types

Terraform supports several variable types:

  • string: A single string value.
  • number: A numeric value.
  • bool: A boolean value (true or false).
  • list: A list of values.
  • map: A map of key-value pairs.

Providing Values for Variables

Values for variables can be provided in several ways:

  1. Command-line flags:

    terraform apply -var="instance_type=t2.large"

  2. Variable definitions file (.tfvars):

    instance_type = "t2.large"

  3. Environment variables:

    export TF_VAR_instance_type="t2.large"

Example: Using a List Variable

Here is an example of defining and using a list variable:

variable "availability_zones" {
  description = "List of availability zones"
  type        = list(string)
  default     = ["us-west-1a", "us-west-1b"]
}

resource "aws_instance" "example" {
  ami               = "ami-0c55b159cbfafe1f0"
  instance_type     = "t2.micro"
  availability_zone = var.availability_zones[0]
}

Outputs

What are Outputs?

Outputs in Terraform are used to display information from your Terraform state. They can be useful for debugging, or for providing information to other configurations or users.

Defining Outputs

Outputs are defined in a .tf file using the output keyword. Here is an example:

output "instance_id" {
  description = "The ID of the EC2 instance"
  value       = aws_instance.example.id
}
  • description: A brief explanation of what the output is for.
  • value: The value to be output, which can reference resources and variables.

Using Outputs

Outputs can be viewed after running terraform apply by using the terraform output command:

terraform output instance_id

Example: Multiple Outputs

Here is an example of defining multiple outputs:

output "instance_id" {
  description = "The ID of the EC2 instance"
  value       = aws_instance.example.id
}

output "instance_public_ip" {
  description = "The public IP of the EC2 instance"
  value       = aws_instance.example.public_ip
}

Summary

In this section, we covered the basics of variables and outputs in Terraform. Variables allow you to parameterize your configurations, making them more flexible and reusable. Outputs enable you to extract and display information from your Terraform state. By understanding and utilizing these features, you can create more dynamic and informative Terraform configurations.

Key Takeaways

  • Variables can be defined, used, and provided values in multiple ways.
  • Outputs help in extracting and displaying useful information from the Terraform state.
  • Proper use of variables and outputs can significantly enhance the flexibility and readability of your Terraform configurations.
