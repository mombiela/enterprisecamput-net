Introduction

In this section, we will cover the essential steps and best practices for building and deploying Angular applications. This includes setting up the environment, building the application, and deploying it to various platforms.

Setting Up the Environment

Before we can build and deploy an Angular application, we need to set up our development environment.

Prerequisites

  • Node.js and npm: Ensure you have Node.js and npm installed. You can download them from nodejs.org.
  • Angular CLI: Install the Angular CLI globally using npm. 
    npm install -g @angular/cli

Creating a New Angular Project

Use the Angular CLI to create a new project.

ng new my-angular-app
  • Follow the prompts to set up your project. You can choose routing and the style format (CSS, SCSS, etc.).

Running the Development Server

Navigate to your project directory and start the development server.

cd my-angular-app
ng serve
  • Open your browser and go to http://localhost:4200 to see your application running.

Building the Application

Once your application is ready, you need to build it for production.

Production Build

Use the Angular CLI to create a production build.

ng build --prod
  • This command will create an optimized build of your application in the dist/ directory.

Build Configuration

You can customize the build process using the angular.json file. Here are some common configurations:

  • Output Path: Change the output directory.
  • File Replacements: Replace environment files for different configurations (e.g., development vs. production).
  • Optimization: Enable or disable optimizations like minification and tree-shaking.

Deploying the Application

There are several ways to deploy an Angular application. We will cover deployment to popular platforms like GitHub Pages, Firebase Hosting, and AWS S3.

Deploying to GitHub Pages

  1. Install the Angular CLI GitHub Pages deployer. 
    npm install -g angular-cli-ghpages
  2. Build your project. 
    ng build --prod --base-href "https://<username>.github.io/<repository>/"
  3. Deploy to GitHub Pages. 
    ngh --dir=dist/<project-name>

Deploying to Firebase Hosting

  1. Install the Firebase CLI. 
    npm install -g firebase-tools
  2. Login to Firebase. 
    firebase login
  3. Initialize Firebase in your project. 
    firebase init
    • Choose Hosting and follow the prompts.
  4. Build your project. 
    ng build --prod
  5. Deploy to Firebase. 
    firebase deploy

Deploying to AWS S3

  1. Install the AWS CLI and configure it with your credentials.
  2. Build your project. 
    ng build --prod
  3. Sync the build output to your S3 bucket. 
    aws s3 sync dist/<project-name> s3://<your-bucket-name> --acl public-read

Conclusion

Building and deploying Angular applications involves setting up the development environment, creating a production build, and deploying it to a hosting platform. By following the steps outlined in this section, you can ensure that your Angular application is optimized and accessible to users. Whether you choose to deploy on GitHub Pages, Firebase Hosting, or AWS S3, the Angular CLI provides the tools you need to streamline the process.
