Introduction to Dependency Injection

Dependency Injection (DI) is a design pattern used to implement IoC (Inversion of Control), allowing a class to receive its dependencies from an external source rather than creating them itself. Angular uses DI to provide components with the services they need.

Key Concepts

  • Dependency: An object that a class needs to function.
  • Injection: The process of providing a class with its dependencies.
  • Provider: An object that tells the DI system how to obtain a dependency.

Why Use Dependency Injection?

  • Decoupling: Reduces the dependency between classes, making the code more modular and easier to test.
  • Reusability: Services can be reused across different components.
  • Maintainability: Easier to manage and update dependencies.

How Dependency Injection Works in Angular

Angular's DI system is hierarchical, meaning that you can define providers at different levels (module, component, etc.), and Angular will resolve dependencies based on the hierarchy.

Example: Basic Dependency Injection

Let's create a simple service and inject it into a component.

Step 1: Create a Service

// src/app/services/logger.service.ts
import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root',
})
export class LoggerService {
  log(message: string) {
    console.log(message);
  }
}
  • @Injectable({ providedIn: 'root' }): This decorator marks the class as a service that can be injected. The providedIn: 'root' syntax means that the service is available application-wide.

Step 2: Inject the Service into a Component

// src/app/components/example/example.component.ts
import { Component, OnInit } from '@angular/core';
import { LoggerService } from '../../services/logger.service';

@Component({
  selector: 'app-example',
  templateUrl: './example.component.html',
  styleUrls: ['./example.component.css']
})
export class ExampleComponent implements OnInit {
  constructor(private logger: LoggerService) {}

  ngOnInit() {
    this.logger.log('ExampleComponent initialized');
  }
}
  • constructor(private logger: LoggerService) {}: This syntax injects the LoggerService into the component.

Advanced Dependency Injection Techniques

Hierarchical Injectors

Angular's DI system allows you to define providers at different levels, such as in modules or components. This can be useful for creating singleton services or for providing different instances of a service.

Module-Level Providers

// src/app/app.module.ts
import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AppComponent } from './app.component';
import { LoggerService } from './services/logger.service';

@NgModule({
  declarations: [AppComponent],
  imports: [BrowserModule],
  providers: [LoggerService],
  bootstrap: [AppComponent]
})
export class AppModule {}
  • providers: [LoggerService]: This makes the LoggerService available to all components in the module.

Component-Level Providers

// src/app/components/example/example.component.ts
import { Component, OnInit } from '@angular/core';
import { LoggerService } from '../../services/logger.service';

@Component({
  selector: 'app-example',
  templateUrl: './example.component.html',
  styleUrls: ['./example.component.css'],
  providers: [LoggerService]
})
export class ExampleComponent implements OnInit {
  constructor(private logger: LoggerService) {}

  ngOnInit() {
    this.logger.log('ExampleComponent initialized');
  }
}
  • providers: [LoggerService]: This makes the LoggerService available only to this component and its children.

Injection Tokens

Injection tokens are used to create non-class-based providers. This is useful for providing configuration objects or other non-class dependencies.

Example: Using Injection Tokens

// src/app/app.module.ts
import { NgModule, InjectionToken } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AppComponent } from './app.component';

export const API_URL = new InjectionToken<string>('apiUrl');

@NgModule({
  declarations: [AppComponent],
  imports: [BrowserModule],
  providers: [{ provide: API_URL, useValue: 'https://api.example.com' }],
  bootstrap: [AppComponent]
})
export class AppModule {}
  • new InjectionToken<string>('apiUrl'): Creates a new injection token.
  • { provide: API_URL, useValue: 'https://api.example.com' }: Provides a value for the injection token.

Injecting the Token

// src/app/components/example/example.component.ts
import { Component, OnInit, Inject } from '@angular/core';
import { API_URL } from '../../app.module';

@Component({
  selector: 'app-example',
  templateUrl: './example.component.html',
  styleUrls: ['./example.component.css']
})
export class ExampleComponent implements OnInit {
  constructor(@Inject(API_URL) private apiUrl: string) {}

  ngOnInit() {
    console.log(`API URL: ${this.apiUrl}`);
  }
}
  • @Inject(API_URL) private apiUrl: string: Injects the value associated with the API_URL token.

Conclusion

Dependency Injection is a powerful feature in Angular that promotes modularity, reusability, and maintainability. By understanding how to create and inject services, as well as how to use hierarchical injectors and injection tokens, you can build more robust and scalable Angular applications.
