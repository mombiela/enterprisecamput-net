Introduction to Child Routes

In Angular, child routes are used to create nested routes within a parent route. This allows for more complex routing scenarios and helps in organizing the application into smaller, manageable parts. Child routes are particularly useful when creating multi-view applications where different views are displayed within a parent component.

Key Concepts

  • Parent Route: The main route that contains one or more child routes.
  • Child Route: A route that is nested within a parent route.
  • Router Module: Angular module that provides the necessary services and directives for routing.

Setting Up Child Routes

To set up child routes, you need to define them within the children property of a parent route in your routing module.

Example

Let's create a simple Angular application with a parent route and two child routes.

  1. Create Components: First, create the necessary components for the parent and child routes.

    ng generate component parent
ng generate component child1
ng generate component child2

  2. Define Routes: Next, define the routes in your app-routing.module.ts file.

    import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { ParentComponent } from './parent/parent.component';
import { Child1Component } from './child1/child1.component';
import { Child2Component } from './child2/child2.component';

const routes: Routes = [
  {
    path: 'parent',
    component: ParentComponent,
    children: [
      { path: 'child1', component: Child1Component },
      { path: 'child2', component: Child2Component }
    ]
  }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

  3. Update Parent Component Template: In the parent component template, add a <router-outlet> to display the child components.

    <!-- parent.component.html -->
<h2>Parent Component</h2>
<nav>
  <a routerLink="child1">Child 1</a>
  <a routerLink="child2">Child 2</a>
</nav>
<router-outlet></router-outlet>

  4. Run the Application: Start your Angular application and navigate to /parent. You should see the parent component with links to the child components. Clicking on the links will display the respective child components within the parent component.

Lazy Loading Child Routes

Lazy loading is a technique to load child routes only when they are needed, which can improve the performance of your application.

Example

  1. Create a Module for Child Routes:

    ng generate module child --route child --module app.module

  2. Define Child Routes in the Module:

    import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { Child1Component } from './child1/child1.component';
import { Child2Component } from './child2/child2.component';

const routes: Routes = [
  {
    path: '',
    component: Child1Component,
    children: [
      { path: 'child2', component: Child2Component }
    ]
  }
];

@NgModule({
  imports: [RouterModule.forChild(routes)],
  exports: [RouterModule]
})
export class ChildModule { }

  3. Update Main Routing Module:

    import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';

const routes: Routes = [
  {
    path: 'parent',
    loadChildren: () => import('./child/child.module').then(m => m.ChildModule)
  }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

Conclusion

Child routes in Angular allow for a more organized and modular approach to routing within an application. By nesting routes, you can create complex routing structures that are easier to manage and maintain. Additionally, leveraging lazy loading for child routes can significantly enhance the performance of your application by loading only the necessary parts when needed. Understanding and implementing child routes is a crucial skill for building scalable and efficient Angular applications.
