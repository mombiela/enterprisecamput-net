Introduction to Pipes

Pipes in Angular are a way to transform data in templates. They are simple functions that accept an input value and return a transformed value. Angular provides several built-in pipes, such as DatePipe, UpperCasePipe, LowerCasePipe, and CurrencyPipe.

Why Use Custom Pipes?

While built-in pipes cover many common use cases, sometimes you need to transform data in a way that is specific to your application. Custom pipes allow you to create reusable data transformation logic that can be applied directly in your templates.

Creating a Custom Pipe

Creating a custom pipe in Angular involves the following steps:

  1. Generate a new pipe using Angular CLI.
  2. Implement the PipeTransform interface.
  3. Define the transformation logic.
  4. Use the custom pipe in your templates.

Step 1: Generate a New Pipe

Use the Angular CLI to generate a new pipe:

ng generate pipe custom

This command will create two files: custom.pipe.ts and custom.pipe.spec.ts.

Step 2: Implement the `PipeTransform` Interface

Open the custom.pipe.ts file and implement the PipeTransform interface:

import { Pipe, PipeTransform } from '@angular/core';

@Pipe({
  name: 'custom'
})
export class CustomPipe implements PipeTransform {
  transform(value: any, ...args: any[]): any {
    // Transformation logic goes here
    return value;
  }
}

Step 3: Define the Transformation Logic

Add your custom transformation logic inside the transform method. For example, let's create a pipe that converts a string to title case:

export class CustomPipe implements PipeTransform {
  transform(value: string): string {
    if (!value) return value;
    return value.replace(/\w\S*/g, (txt) => {
      return txt.charAt(0).toUpperCase() + txt.substr(1).toLowerCase();
    });
  }
}

Step 4: Use the Custom Pipe in Templates

Once your custom pipe is defined, you can use it in your templates just like any other pipe:

<p>{{ 'hello world' | custom }}</p>

This will output:

Hello World

Passing Parameters to Custom Pipes

Custom pipes can also accept parameters. For example, let's create a pipe that formats a number to a specified decimal place:

@Pipe({
  name: 'decimal'
})
export class DecimalPipe implements PipeTransform {
  transform(value: number, decimalPlaces: number): string {
    return value.toFixed(decimalPlaces);
  }
}

Usage in template:

<p>{{ 123.4567 | decimal:2 }}</p>

This will output:

123.46

Pure vs Impure Pipes

  • Pure Pipes: These pipes are only called when Angular detects a change in the input value. They are more performant and should be used in most cases.
  • Impure Pipes: These pipes are called on every change detection cycle, which can impact performance. Use them only when necessary.

To create an impure pipe, set the pure property to false in the @Pipe decorator:

@Pipe({
  name: 'impurePipe',
  pure: false
})
export class ImpurePipe implements PipeTransform {
  transform(value: any): any {
    // Transformation logic
    return value;
  }
}

Conclusion

Custom pipes in Angular are a powerful way to encapsulate and reuse data transformation logic in your templates. By following the steps outlined in this topic, you can create your own custom pipes to meet the specific needs of your application. Remember to consider the performance implications when deciding between pure and impure pipes.
