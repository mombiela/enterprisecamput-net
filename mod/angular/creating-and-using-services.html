Creating and Using Services
Dependency Injection ►

In Angular, services are a fundamental concept used to share data, logic, and functions across different components of an application. This topic will guide you through the creation and usage of services, from basic to advanced levels.

What is a Service?

  • Definition: A service is a class with a focused purpose. It is typically used to perform tasks such as fetching data from a server, validating user input, or logging.
  • Purpose: Services help keep your components lean and focused on their primary tasks, such as displaying data and handling user interaction.

Creating a Basic Service

Step-by-Step Guide

  1. Generate a Service: Use Angular CLI to generate a service.

    ng generate service my-service

    This command creates two files: my-service.service.ts and my-service.service.spec.ts.

  2. Define the Service: Open my-service.service.ts and define the service.

    import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class MyService {
  constructor() { }

  getData() {
    return 'Hello from MyService!';
  }
}

Explanation

  • @Injectable({ providedIn: 'root' }): This decorator makes the service available throughout the application.
  • getData(): A simple method that returns a string.

Using a Service in a Component

Step-by-Step Guide

  1. Inject the Service: Open a component file (e.g., app.component.ts) and inject the service.

    import { Component, OnInit } from '@angular/core';
import { MyService } from './my-service.service';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent implements OnInit {
  message: string;

  constructor(private myService: MyService) {}

  ngOnInit() {
    this.message = this.myService.getData();
  }
}

  2. Display the Data: Bind the data in the component's template (e.g., app.component.html).

    <div>
  {{ message }}
</div>

Explanation

  • constructor(private myService: MyService): The service is injected into the component.
  • ngOnInit(): The getData() method of the service is called when the component initializes.

Advanced Service Features

Dependency Injection (DI)

  • Hierarchical Injectors: Angular's DI system allows services to be provided at different levels (root, module, component).
  • Example: Providing a service at the component level. 
    @Component({
  selector: 'app-child',
  templateUrl: './child.component.html',
  styleUrls: ['./child.component.css'],
  providers: [MyService]
})
export class ChildComponent {
  constructor(private myService: MyService) {}
}

HTTP Services

  • Fetching Data from an API: Use Angular's HttpClient to fetch data from a server. 
    import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class DataService {
  private apiUrl = 'https://api.example.com/data';

  constructor(private http: HttpClient) {}

  fetchData(): Observable<any> {
    return this.http.get<any>(this.apiUrl);
  }
}

Explanation

  • HttpClient: Angular's built-in service for making HTTP requests.
  • fetchData(): Method to fetch data from the specified API URL.

Using HTTP Service in a Component

  1. Inject the Service: Inject the DataService into a component.

    import { Component, OnInit } from '@angular/core';
import { DataService } from './data.service';

@Component({
  selector: 'app-data',
  templateUrl: './data.component.html',
  styleUrls: ['./data.component.css']
})
export class DataComponent implements OnInit {
  data: any;

  constructor(private dataService: DataService) {}

  ngOnInit() {
    this.dataService.fetchData().subscribe(response => {
      this.data = response;
    });
  }
}

  2. Display the Data: Bind the fetched data in the component's template.

    <div *ngIf="data">
  <pre>{{ data | json }}</pre>
</div>

Explanation

  • fetchData().subscribe(response => { this.data = response; }): Subscribes to the observable returned by fetchData() and assigns the response to the data property.

Summary

  • Services: Essential for sharing data and logic across components.
  • Creating Services: Use Angular CLI and @Injectable decorator.
  • Using Services: Inject services into components and call their methods.
  • Advanced Features: Dependency injection at different levels, using HttpClient for API calls.

By mastering services in Angular, you can create more modular, maintainable, and testable applications.
