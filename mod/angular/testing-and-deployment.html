Testing and Deployment

Introduction

In this section, we will cover the essential aspects of testing and deploying Angular applications. Testing ensures that your application works as expected, while deployment involves making your application available to users. We will start with the basics of testing and gradually move to more advanced topics, followed by deployment strategies.

Testing in Angular

Unit Testing

Unit testing involves testing individual components or services in isolation.

  • Tools: Angular uses Jasmine for writing tests and Karma as the test runner.
  • Setup: Angular CLI sets up Jasmine and Karma by default.

Example: Testing a Component

import { ComponentFixture, TestBed } from '@angular/core/testing';
import { MyComponent } from './my-component.component';

describe('MyComponent', () => {
  let component: MyComponent;
  let fixture: ComponentFixture<MyComponent>;

  beforeEach(async () => {
    await TestBed.configureTestingModule({
      declarations: [ MyComponent ]
    })
    .compileComponents();
  });

  beforeEach(() => {
    fixture = TestBed.createComponent(MyComponent);
    component = fixture.componentInstance;
    fixture.detectChanges();
  });

  it('should create', () => {
    expect(component).toBeTruthy();
  });
});
  • Explanation:
    • TestBed.configureTestingModule: Configures the testing module.
    • beforeEach: Sets up the component before each test.
    • it: Defines an individual test case.

Integration Testing

Integration testing involves testing multiple components or services together to ensure they work as expected.

  • Tools: Jasmine and Karma are also used for integration testing.

Example: Testing a Service with a Component

import { TestBed } from '@angular/core/testing';
import { MyService } from './my-service.service';
import { MyComponent } from './my-component.component';

describe('MyComponent with MyService', () => {
  let component: MyComponent;
  let service: MyService;

  beforeEach(() => {
    TestBed.configureTestingModule({
      providers: [MyService],
      declarations: [MyComponent]
    });

    service = TestBed.inject(MyService);
    component = TestBed.createComponent(MyComponent).componentInstance;
  });

  it('should use the service', () => {
    expect(service).toBeTruthy();
    expect(component.myService).toBe(service);
  });
});
  • Explanation:
    • TestBed.inject: Injects the service into the test environment.
    • component.myService: Ensures the component uses the injected service.

End-to-End (E2E) Testing

E2E testing involves testing the entire application flow from start to finish.

  • Tools: Protractor is commonly used for E2E testing in Angular.

Example: Basic E2E Test

import { browser, by, element } from 'protractor';

describe('My App', () => {
  it('should display the welcome message', () => {
    browser.get('/');
    expect(element(by.css('h1')).getText()).toEqual('Welcome to My App!');
  });
});
  • Explanation:
    • browser.get('/'): Navigates to the root URL.
    • element(by.css('h1')).getText(): Selects the h1 element and gets its text content.

Deployment

Preparing for Deployment

Before deploying your Angular application, you need to build it for production.

  • Build Command: ng build --prod
    • This command optimizes the application for production by minifying the code and removing unnecessary parts.

Deployment Options

There are several ways to deploy an Angular application:

  1. Static File Hosting: Deploying the built files to a static file hosting service like GitHub Pages, Netlify, or Firebase Hosting.
  2. Server Deployment: Deploying to a server like Apache, Nginx, or a cloud service like AWS, Azure, or Google Cloud.

Example: Deploying to GitHub Pages

  1. Install Angular CLI GitHub Pages Deploy: 
    npm install -g angular-cli-ghpages
  2. Build the Project: 
    ng build --prod --base-href "https://<username>.github.io/<repository>/"
  3. Deploy: 
    ngh --dir=dist/<project-name>

Continuous Integration and Continuous Deployment (CI/CD)

CI/CD automates the process of testing and deploying your application.

  • Tools: Jenkins, Travis CI, CircleCI, GitHub Actions.

Example: GitHub Actions Workflow

name: CI/CD Pipeline

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2
    - name: Set up Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '14'
    - name: Install dependencies
      run: npm install
    - name: Run tests
      run: npm test
    - name: Build project
      run: ng build --prod
    - name: Deploy to GitHub Pages
      run: |
        npm install -g angular-cli-ghpages
        ngh --dir=dist/<project-name>
  • Explanation:
    • actions/checkout@v2: Checks out the repository.
    • actions/setup-node@v2: Sets up Node.js.
    • npm install: Installs dependencies.
    • npm test: Runs tests.
    • ng build --prod: Builds the project.
    • ngh --dir=dist/<project-name>: Deploys to GitHub Pages.

Conclusion

Testing and deployment are crucial aspects of Angular development. Proper testing ensures that your application is reliable and free of bugs, while effective deployment strategies make your application accessible to users. By mastering these topics, you can ensure that your Angular applications are both robust and efficiently delivered to your audience.
