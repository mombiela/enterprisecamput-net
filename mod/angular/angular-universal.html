Introduction to Angular Universal

Angular Universal is a technology that allows Angular applications to be rendered on the server side. This can improve performance, SEO, and user experience by delivering fully rendered pages to the client.

Key Concepts

  • Server-side Rendering (SSR): The process of rendering web pages on the server instead of the client.
  • Angular Universal: A tool that enables SSR for Angular applications.
  • SEO Benefits: Search engines can crawl fully rendered pages, improving search engine rankings.
  • Performance: Faster initial load times as the server sends a fully rendered page to the client.

Setting Up Angular Universal

To get started with Angular Universal, you need to add it to your existing Angular project.

Steps to Add Angular Universal

  1. Install Angular Universal:

    ng add @nguniversal/express-engine

    This command will configure your Angular application to use Angular Universal with an Express server.

  2. Update angular.json: Ensure that the build and server configurations are correctly set up in your angular.json file.

  3. Modify server.ts: The server.ts file is the entry point for your server-side application. It should look something like this:

    import 'zone.js/dist/zone-node';
import { ngExpressEngine } from '@nguniversal/express-engine';
import * as express from 'express';
import { join } from 'path';

import { AppServerModule } from './src/main.server';
import { APP_BASE_HREF } from '@angular/common';
import { existsSync } from 'fs';

// The Express app is exported so that it can be used by serverless Functions.
export function app() {
  const server = express();
  const distFolder = join(process.cwd(), 'dist/your-project-name/browser');
  const indexHtml = existsSync(join(distFolder, 'index.original.html')) ? 'index.original.html' : 'index';

  server.engine('html', ngExpressEngine({
    bootstrap: AppServerModule,
  }));

  server.set('view engine', 'html');
  server.set('views', distFolder);

  server.get('*.*', express.static(distFolder, {
    maxAge: '1y'
  }));

  server.get('*', (req, res) => {
    res.render(indexHtml, { req, providers: [{ provide: APP_BASE_HREF, useValue: req.baseUrl }] });
  });

  return server;
}

function run() {
  const port = process.env.PORT || 4000;

  const server = app();
  server.listen(port, () => {
    console.log(`Node Express server listening on http://localhost:${port}`);
  });
}

declare const __non_webpack_require__: NodeRequire;

if (require.main === module) {
  run();
}

export * from './src/main.server';

  4. Update main.server.ts: Ensure that your main.server.ts file bootstraps the server module:

    import { enableProdMode } from '@angular/core';
import { environment } from './environments/environment';

if (environment.production) {
  enableProdMode();
}

export { AppServerModule } from './app/app.server.module';
export { renderModule, renderModuleFactory } from '@angular/platform-server';

Running the Application

To run your Angular Universal application, you need to build both the client and server-side code and then serve it.

Build and Serve

  1. Build the Client and Server:

    npm run build:ssr

  2. Serve the Application:

    npm run serve:ssr

Benefits of Angular Universal

  • Improved SEO: Search engines can index your site more effectively.
  • Faster Initial Load: Users receive a fully rendered page from the server, reducing the time to first meaningful paint.
  • Better User Experience: Users can see content faster, even on slow networks.

Conclusion

Angular Universal is a powerful tool for enhancing the performance and SEO of Angular applications through server-side rendering. By following the setup steps and understanding the key concepts, you can leverage Angular Universal to deliver a better user experience and improve your application's visibility on search engines.
