Introduction to Directives

Directives are a core feature of Angular that allow you to attach behavior to elements in the DOM. They can change the appearance or behavior of an element, component, or another directive. Angular provides several built-in directives that you can use to manage forms, lists, styles, and more.

Types of Built-in Directives

Angular's built-in directives can be categorized into three main types:

  • Attribute Directives: Change the appearance or behavior of an element, component, or another directive.
  • Structural Directives: Change the DOM layout by adding or removing elements.
  • Component Directives: These are directives with a template.

Attribute Directives

Attribute directives change the appearance or behavior of an element. Here are some commonly used attribute directives:

`ngClass`

The ngClass directive allows you to dynamically set CSS classes on an element.

Example:

<div [ngClass]="{'active': isActive, 'disabled': isDisabled}">
  This div's classes change based on the component's state.
</div>

In the above example, the active class will be applied if isActive is true, and the disabled class will be applied if isDisabled is true.

`ngStyle`

The ngStyle directive allows you to set multiple inline styles dynamically.

Example:

<div [ngStyle]="{'color': textColor, 'font-size': fontSize + 'px'}">
  This div's styles change based on the component's state.
</div>

In this example, the color and font-size styles are dynamically set based on the component's textColor and fontSize properties.

Structural Directives

Structural directives change the DOM layout by adding or removing elements. Here are some commonly used structural directives:

`ngIf`

The ngIf directive conditionally includes a template based on the value of an expression.

Example:

<div *ngIf="isVisible">
  This div is only visible if isVisible is true.
</div>

In this example, the div will only be included in the DOM if isVisible is true.

`ngFor`

The ngFor directive is used to repeat a template for each item in a list.

Example:

<ul>
  <li *ngFor="let item of items">
    {{ item }}
  </li>
</ul>

In this example, an <li> element is created for each item in the items array.

`ngSwitch`

The ngSwitch directive is used to conditionally include one of several possible templates, based on the value of an expression.

Example:

<div [ngSwitch]="viewMode">
  <div *ngSwitchCase="'map'">Map View</div>
  <div *ngSwitchCase="'list'">List View</div>
  <div *ngSwitchDefault>Default View</div>
</div>

In this example, the template that matches the value of viewMode is included in the DOM.

Component Directives

Component directives are essentially Angular components. They have a template, styles, and encapsulated logic.

Example:

@Component({
  selector: 'app-hello',
  template: `<h1>Hello, {{name}}!</h1>`,
  styles: [`h1 { font-family: Lato; }`]
})
export class HelloComponent  {
  @Input() name: string;
}

In this example, HelloComponent is a component directive that displays a greeting message.

Conclusion

Built-in directives in Angular are powerful tools that allow you to manipulate the DOM in a declarative way. Understanding how to use attribute directives like ngClass and ngStyle, structural directives like ngIf, ngFor, and ngSwitch, and component directives will enable you to create dynamic and responsive web applications. Practice using these directives in your projects to get a better grasp of their capabilities and nuances.
