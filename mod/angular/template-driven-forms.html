Introduction

Template-driven forms are a simple and powerful way to handle user input in Angular applications. They rely on Angular's two-way data binding and directives to create and manage forms. This approach is suitable for simpler forms and is easier to set up compared to reactive forms.

Key Concepts

  • NgModel Directive: Binds form controls to model properties.
  • FormsModule: Angular module required to use template-driven forms.
  • Form Validation: Built-in validators and custom validators.
  • Form Submission: Handling form data upon submission.

Setting Up Template-driven Forms

Importing FormsModule

To use template-driven forms, you need to import the FormsModule in your Angular module.

import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { FormsModule } from '@angular/forms';
import { AppComponent } from './app.component';

@NgModule({
  declarations: [AppComponent],
  imports: [BrowserModule, FormsModule],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

Creating a Basic Form

Here's a simple example of a template-driven form:

<!-- app.component.html -->
<form #userForm="ngForm" (ngSubmit)="onSubmit(userForm)">
  <div>
    <label for="name">Name:</label>
    <input type="text" id="name" name="name" ngModel required>
  </div>
  <div>
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" ngModel required email>
  </div>
  <button type="submit" [disabled]="userForm.invalid">Submit</button>
</form>

Handling Form Submission

In your component, you can handle the form submission and access the form data:

// app.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  onSubmit(form: any) {
    console.log('Form Data:', form.value);
  }
}

Form Validation

Built-in Validators

Angular provides several built-in validators such as required, minlength, maxlength, email, etc.

<div>
  <label for="password">Password:</label>
  <input type="password" id="password" name="password" ngModel required minlength="8">
  <div *ngIf="userForm.controls.password?.invalid && userForm.controls.password?.touched">
    <small *ngIf="userForm.controls.password?.errors?.required">Password is required.</small>
    <small *ngIf="userForm.controls.password?.errors?.minlength">Password must be at least 8 characters long.</small>
  </div>
</div>

Custom Validators

You can also create custom validators for more complex validation logic.

// app.component.ts
import { Component } from '@angular/core';
import { NgForm } from '@angular/forms';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  onSubmit(form: NgForm) {
    console.log('Form Data:', form.value);
  }

  validateUsername(control: any): { [key: string]: boolean } | null {
    const forbiddenUsernames = ['admin', 'user'];
    if (forbiddenUsernames.includes(control.value)) {
      return { 'forbiddenUsername': true };
    }
    return null;
  }
}
<div>
  <label for="username">Username:</label>
  <input type="text" id="username" name="username" ngModel #username="ngModel" [ngModelOptions]="{ updateOn: 'blur' }" [ngModel]="validateUsername">
  <div *ngIf="username.invalid && username.touched">
    <small *ngIf="username.errors?.forbiddenUsername">This username is not allowed.</small>
  </div>
</div>

Advanced Topics

Two-way Data Binding

Template-driven forms leverage Angular's two-way data binding to sync form controls with model properties.

<!-- app.component.html -->
<form #userForm="ngForm" (ngSubmit)="onSubmit(userForm)">
  <div>
    <label for="name">Name:</label>
    <input type="text" id="name" name="name" [(ngModel)]="user.name" required>
  </div>
  <div>
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" [(ngModel)]="user.email" required email>
  </div>
  <button type="submit" [disabled]="userForm.invalid">Submit</button>
</form>
// app.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  user = {
    name: '',
    email: ''
  };

  onSubmit(form: any) {
    console.log('Form Data:', form.value);
  }
}

Form State and CSS Classes

Angular automatically adds CSS classes to form controls based on their state (e.g., ng-touched, ng-dirty, ng-valid, ng-invalid).

<div>
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" ngModel required>
  <div *ngIf="userForm.controls.name?.invalid && userForm.controls.name?.touched">
    <small *ngIf="userForm.controls.name?.errors?.required">Name is required.</small>
  </div>
</div>

Conclusion

Template-driven forms in Angular provide a straightforward way to create and manage forms using Angular's powerful directives and two-way data binding. They are ideal for simpler forms and offer built-in validation, custom validation, and easy form state management. By understanding and utilizing these concepts, you can efficiently handle user input and form validation in your Angular applications.
