Internationalization (i18n) in Angular
Angular and Web Workers ►

Internationalization (i18n) is the process of designing and preparing your application to be usable in different languages and regions without requiring engineering changes to the source code. Angular provides robust support for i18n, making it easier to create multilingual applications.

Introduction to Internationalization (i18n)

  • Definition: Internationalization is the process of designing an application so that it can be adapted to various languages and regions without requiring changes to the code.
  • Importance: It allows your application to reach a broader audience by supporting multiple languages and regional formats.

Setting Up Angular for i18n

To get started with i18n in Angular, you need to set up your project to support translations.

Step 1: Install Angular CLI

Ensure you have the Angular CLI installed:

npm install -g @angular/cli

Step 2: Create a New Angular Project

Create a new Angular project if you don't already have one:

ng new my-i18n-app
cd my-i18n-app

Step 3: Add i18n Markers in Templates

Use Angular's i18n attributes to mark text for translation in your templates:

<h1 i18n="@@homeTitle">Welcome to My App</h1>
<p i18n="@@homeDescription">This is a sample application for demonstrating i18n in Angular.</p>
  • i18n attribute: Marks the text for translation.
  • @@id: Optional custom identifier for the translation unit.

Extracting Translation Strings

After marking the text for translation, you need to extract these strings into a translation file.

Step 1: Extract Strings

Use Angular CLI to extract the marked strings:

ng extract-i18n

This command generates a messages.xlf file in the root directory of your project.

Step 2: Translate Strings

Translate the extracted strings in the messages.xlf file:

<trans-unit id="homeTitle" datatype="html">
  <source>Welcome to My App</source>
  <target>Bienvenido a Mi Aplicación</target>
</trans-unit>
<trans-unit id="homeDescription" datatype="html">
  <source>This is a sample application for demonstrating i18n in Angular.</source>
  <target>Esta es una aplicación de muestra para demostrar i18n en Angular.</target>
</trans-unit>

Configuring Angular for Multiple Locales

To support multiple locales, you need to configure Angular to use different translation files based on the user's locale.

Step 1: Update `angular.json`

Modify the angular.json file to include different locales:

"projects": {
  "my-i18n-app": {
    ...
    "i18n": {
      "sourceLocale": "en",
      "locales": {
        "es": "src/locale/messages.es.xlf"
      }
    }
  }
}

Step 2: Build for Different Locales

Build your application for different locales:

ng build --localize

This command generates separate builds for each locale specified in the angular.json file.

Dynamic Locale Switching

For more advanced applications, you might want to switch locales dynamically at runtime.

Step 1: Install Dependencies

Install the necessary Angular packages:

npm install @ngx-translate/core @ngx-translate/http-loader

Step 2: Configure `app.module.ts`

Configure the translation loader in your AppModule:

import { HttpClient } from '@angular/common/http';
import { TranslateLoader, TranslateModule } from '@ngx-translate/core';
import { TranslateHttpLoader } from '@ngx-translate/http-loader';

export function HttpLoaderFactory(http: HttpClient) {
  return new TranslateHttpLoader(http);
}

@NgModule({
  declarations: [AppComponent],
  imports: [
    BrowserModule,
    HttpClientModule,
    TranslateModule.forRoot({
      loader: {
        provide: TranslateLoader,
        useFactory: HttpLoaderFactory,
        deps: [HttpClient]
      }
    })
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

Step 3: Load Translations in Components

Load and use translations in your components:

import { Component } from '@angular/core';
import { TranslateService } from '@ngx-translate/core';

@Component({
  selector: 'app-root',
  template: `
    <div>
      <h1>{{ 'homeTitle' | translate }}</h1>
      <p>{{ 'homeDescription' | translate }}</p>
      <button (click)="switchLanguage('en')">English</button>
      <button (click)="switchLanguage('es')">Español</button>
    </div>
  `
})
export class AppComponent {
  constructor(private translate: TranslateService) {
    this.translate.setDefaultLang('en');
  }

  switchLanguage(language: string) {
    this.translate.use(language);
  }
}

Conclusion

Internationalization (i18n) in Angular is a powerful feature that allows you to create applications that can be easily adapted to different languages and regions. By following the steps outlined in this guide, you can set up your Angular application for i18n, extract and translate strings, configure multiple locales, and even switch locales dynamically at runtime. This ensures your application can reach a global audience with minimal effort.
