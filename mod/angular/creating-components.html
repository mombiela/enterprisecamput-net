Introduction to Components

Components are the building blocks of any Angular application. They encapsulate the view, logic, and styles, making it easier to manage and reuse code.

Key Concepts

  • Component: A class that interacts with the view (HTML) and defines the behavior of a part of the UI.
  • Template: The HTML part of the component that defines the view.
  • Styles: CSS or SCSS that defines the look and feel of the component.
  • Metadata: Information about the component, such as its selector, template, and styles.

Creating a Basic Component

To create a new component in Angular, you can use the Angular CLI.

Using Angular CLI

Run the following command in your terminal:

ng generate component my-component

This command will create a new folder named my-component with the following files:

  • my-component.component.ts: The TypeScript file containing the component class.
  • my-component.component.html: The HTML template for the component.
  • my-component.component.css: The CSS styles for the component.
  • my-component.component.spec.ts: The test file for the component.

Component Structure

Let's break down the generated my-component.component.ts file:

import { Component } from '@angular/core';

@Component({
  selector: 'app-my-component',
  templateUrl: './my-component.component.html',
  styleUrls: ['./my-component.component.css']
})
export class MyComponent {
  // Component logic goes here
}
  • @Component Decorator: This decorator provides metadata about the component.
    • selector: The custom HTML tag that represents the component.
    • templateUrl: The path to the HTML template file.
    • styleUrls: The path to the CSS file(s).

Using the Component

To use the newly created component, add its selector to the template of another component, such as app.component.html:

<app-my-component></app-my-component>

Component Interaction

Components often need to interact with each other. This can be done using:

  • Input: Pass data from a parent component to a child component.
  • Output: Emit events from a child component to a parent component.

Input Example

In the child component (my-component):

import { Component, Input } from '@angular/core';

@Component({
  selector: 'app-my-component',
  templateUrl: './my-component.component.html',
  styleUrls: ['./my-component.component.css']
})
export class MyComponent {
  @Input() data: string;
}

In the parent component (app.component.html):

<app-my-component [data]="parentData"></app-my-component>

In the parent component class (app.component.ts):

export class AppComponent {
  parentData = 'Hello from Parent';
}

Output Example

In the child component (my-component):

import { Component, Output, EventEmitter } from '@angular/core';

@Component({
  selector: 'app-my-component',
  templateUrl: './my-component.component.html',
  styleUrls: ['./my-component.component.css']
})
export class MyComponent {
  @Output() notify: EventEmitter<string> = new EventEmitter<string>();

  sendNotification() {
    this.notify.emit('Notification from Child');
  }
}

In the child component template (my-component.component.html):

<button (click)="sendNotification()">Notify Parent</button>

In the parent component (app.component.html):

<app-my-component (notify)="onNotify($event)"></app-my-component>

In the parent component class (app.component.ts):

export class AppComponent {
  onNotify(message: string) {
    console.log(message);
  }
}

Advanced Component Features

Lifecycle Hooks

Angular provides lifecycle hooks that allow you to tap into key events in a component's lifecycle:

  • ngOnInit(): Called once the component is initialized.
  • ngOnChanges(): Called when an input property changes.
  • ngOnDestroy(): Called just before the component is destroyed.

Example:

import { Component, OnInit, OnDestroy } from '@angular/core';

@Component({
  selector: 'app-my-component',
  templateUrl: './my-component.component.html',
  styleUrls: ['./my-component.component.css']
})
export class MyComponent implements OnInit, OnDestroy {
  ngOnInit() {
    console.log('Component Initialized');
  }

  ngOnDestroy() {
    console.log('Component Destroyed');
  }
}

ViewEncapsulation

Angular provides three encapsulation strategies:

  • Emulated: Default, emulates native shadow DOM behavior.
  • None: No encapsulation, styles are global.
  • ShadowDom: Uses native shadow DOM.

Example:

import { Component, ViewEncapsulation } from '@angular/core';

@Component({
  selector: 'app-my-component',
  templateUrl: './my-component.component.html',
  styleUrls: ['./my-component.component.css'],
  encapsulation: ViewEncapsulation.Emulated // Default
})
export class MyComponent {}

Conclusion

Components are fundamental to Angular applications. They encapsulate the view, logic, and styles, making it easier to manage and reuse code. By understanding how to create and interact with components, you can build complex and scalable Angular applications. Remember to leverage Angular CLI for efficient component creation and use lifecycle hooks to manage component behavior effectively.
