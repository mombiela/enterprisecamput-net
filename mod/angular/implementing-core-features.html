Introduction

In this section, we will delve into the core features of Angular, a powerful framework for building dynamic web applications. By the end of this topic, you will have a solid understanding of how to implement essential features such as components, services, routing, and forms.

Components

Components are the building blocks of Angular applications. They encapsulate the HTML, CSS, and logic for a part of the user interface.

Creating a Component

To create a new component, use the Angular CLI command:

ng generate component my-component

This command generates four files:

  • my-component.component.ts: The TypeScript file containing the component logic.
  • my-component.component.html: The HTML template for the component.
  • my-component.component.css: The CSS styles for the component.
  • my-component.component.spec.ts: The test file for the component.

Component Example

Here's a simple example of a component:

// my-component.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-my-component',
  templateUrl: './my-component.component.html',
  styleUrls: ['./my-component.component.css']
})
export class MyComponent {
  title = 'Hello, Angular!';
}
<!-- my-component.component.html -->
<h1>{{ title }}</h1>

In this example, the title property is bound to the template using Angular's interpolation syntax ({{ }}).

Services

Services in Angular are used to encapsulate business logic and data access. They can be injected into components and other services.

Creating a Service

To create a new service, use the Angular CLI command:

ng generate service my-service

This command generates two files:

  • my-service.service.ts: The TypeScript file containing the service logic.
  • my-service.service.spec.ts: The test file for the service.

Service Example

Here's a simple example of a service:

// my-service.service.ts
import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class MyService {
  getValue(): string {
    return 'Service Value';
  }
}

To use this service in a component:

// my-component.component.ts
import { Component } from '@angular/core';
import { MyService } from './my-service.service';

@Component({
  selector: 'app-my-component',
  templateUrl: './my-component.component.html',
  styleUrls: ['./my-component.component.css']
})
export class MyComponent {
  value: string;

  constructor(private myService: MyService) {
    this.value = this.myService.getValue();
  }
}
<!-- my-component.component.html -->
<p>{{ value }}</p>

Routing

Routing in Angular allows you to navigate between different views or components.

Setting Up Routing

First, define your routes in the app-routing.module.ts file:

// app-routing.module.ts
import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { MyComponent } from './my-component/my-component.component';

const routes: Routes = [
  { path: 'my-component', component: MyComponent }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

Then, add the <router-outlet> directive in your main template:

<!-- app.component.html -->
<router-outlet></router-outlet>

To navigate to a route programmatically:

// some-component.component.ts
import { Component } from '@angular/core';
import { Router } from '@angular/router';

@Component({
  selector: 'app-some-component',
  templateUrl: './some-component.component.html',
  styleUrls: ['./some-component.component.css']
})
export class SomeComponent {
  constructor(private router: Router) {}

  navigateToMyComponent() {
    this.router.navigate(['/my-component']);
  }
}
<!-- some-component.component.html -->
<button (click)="navigateToMyComponent()">Go to My Component</button>

Forms

Angular provides two approaches for handling forms: Template-driven and Reactive forms.

Template-driven Forms

Template-driven forms are easy to use and suitable for simple forms.

<!-- template-driven-form.component.html -->
<form #myForm="ngForm" (ngSubmit)="onSubmit(myForm)">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" ngModel>
  <button type="submit">Submit</button>
</form>
// template-driven-form.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-template-driven-form',
  templateUrl: './template-driven-form.component.html',
  styleUrls: ['./template-driven-form.component.css']
})
export class TemplateDrivenFormComponent {
  onSubmit(form: any) {
    console.log('Form Submitted!', form);
  }
}

Reactive Forms

Reactive forms are more powerful and suitable for complex forms.

// reactive-form.component.ts
import { Component, OnInit } from '@angular/core';
import { FormBuilder, FormGroup } from '@angular/forms';

@Component({
  selector: 'app-reactive-form',
  templateUrl: './reactive-form.component.html',
  styleUrls: ['./reactive-form.component.css']
})
export class ReactiveFormComponent implements OnInit {
  myForm: FormGroup;

  constructor(private fb: FormBuilder) {}

  ngOnInit() {
    this.myForm = this.fb.group({
      name: ['']
    });
  }

  onSubmit() {
    console.log('Form Submitted!', this.myForm.value);
  }
}
<!-- reactive-form.component.html -->
<form [formGroup]="myForm" (ngSubmit)="onSubmit()">
  <label for="name">Name:</label>
  <input type="text" id="name" formControlName="name">
  <button type="submit">Submit</button>
</form>

Conclusion

In this topic, we covered the core features of Angular, including components, services, routing, and forms. These features are fundamental to building robust and maintainable Angular applications. By mastering these concepts, you will be well-equipped to tackle more advanced topics and build sophisticated web applications.
