Component Interaction
Built-in Directives ►

Component interaction is a fundamental concept in Angular that allows different components to communicate and share data. This topic will cover various methods of component interaction, ranging from simple parent-child communication to more advanced techniques like using services and state management.

Parent to Child Communication

Parent to child communication is one of the most common forms of component interaction in Angular. This is typically achieved using input properties.

Input Properties

  • Definition: Input properties allow a parent component to pass data to a child component.
  • Usage: Use the @Input decorator to define an input property in the child component.

Example

Parent Component Template (parent.component.html):

<app-child [childProperty]="parentData"></app-child>

Parent Component Class (parent.component.ts):

import { Component } from '@angular/core';

@Component({
  selector: 'app-parent',
  templateUrl: './parent.component.html',
})
export class ParentComponent {
  parentData = 'Data from parent';
}

Child Component Class (child.component.ts):

import { Component, Input } from '@angular/core';

@Component({
  selector: 'app-child',
  template: '<p>{{ childProperty }}</p>',
})
export class ChildComponent {
  @Input() childProperty: string;
}

In this example, the parent component passes the parentData to the child component using the childProperty input property.

Child to Parent Communication

Child to parent communication is often achieved using output properties and event emitters.

Output Properties and Event Emitters

  • Definition: Output properties allow a child component to emit events that a parent component can listen to.
  • Usage: Use the @Output decorator along with an EventEmitter to define an output property in the child component.

Example

Child Component Class (child.component.ts):

import { Component, Output, EventEmitter } from '@angular/core';

@Component({
  selector: 'app-child',
  template: '<button (click)="sendData()">Send Data</button>',
})
export class ChildComponent {
  @Output() dataEmitter = new EventEmitter<string>();

  sendData() {
    this.dataEmitter.emit('Data from child');
  }
}

Parent Component Template (parent.component.html):

<app-child (dataEmitter)="receiveData($event)"></app-child>

Parent Component Class (parent.component.ts):

import { Component } from '@angular/core';

@Component({
  selector: 'app-parent',
  templateUrl: './parent.component.html',
})
export class ParentComponent {
  receiveData(data: string) {
    console.log(data);
  }
}

In this example, the child component emits an event with data, which the parent component listens to and handles.

Sibling Communication

Sibling components can communicate through a shared parent component or a common service.

Using a Shared Parent Component

  • Definition: Sibling components can communicate by passing data through their shared parent component.
  • Usage: The parent component acts as a mediator, receiving data from one child and passing it to another.

Example

Parent Component Template (parent.component.html):

<app-sibling-one (dataEmitter)="receiveData($event)"></app-sibling-one>
<app-sibling-two [siblingData]="sharedData"></app-sibling-two>

Parent Component Class (parent.component.ts):

import { Component } from '@angular/core';

@Component({
  selector: 'app-parent',
  templateUrl: './parent.component.html',
})
export class ParentComponent {
  sharedData: string;

  receiveData(data: string) {
    this.sharedData = data;
  }
}

Sibling One Component Class (sibling-one.component.ts):

import { Component, Output, EventEmitter } from '@angular/core';

@Component({
  selector: 'app-sibling-one',
  template: '<button (click)="sendData()">Send Data</button>',
})
export class SiblingOneComponent {
  @Output() dataEmitter = new EventEmitter<string>();

  sendData() {
    this.dataEmitter.emit('Data from sibling one');
  }
}

Sibling Two Component Class (sibling-two.component.ts):

import { Component, Input } from '@angular/core';

@Component({
  selector: 'app-sibling-two',
  template: '<p>{{ siblingData }}</p>',
})
export class SiblingTwoComponent {
  @Input() siblingData: string;
}

Using a Service

  • Definition: A service can be used to share data and facilitate communication between sibling components.
  • Usage: Create a service with shared data and methods to update and retrieve that data.

Example

Shared Service (shared.service.ts):

import { Injectable } from '@angular/core';
import { Subject } from 'rxjs';

@Injectable({
  providedIn: 'root',
})
export class SharedService {
  private dataSubject = new Subject<string>();
  data$ = this.dataSubject.asObservable();

  sendData(data: string) {
    this.dataSubject.next(data);
  }
}

Sibling One Component Class (sibling-one.component.ts):

import { Component } from '@angular/core';
import { SharedService } from './shared.service';

@Component({
  selector: 'app-sibling-one',
  template: '<button (click)="sendData()">Send Data</button>',
})
export class SiblingOneComponent {
  constructor(private sharedService: SharedService) {}

  sendData() {
    this.sharedService.sendData('Data from sibling one');
  }
}

Sibling Two Component Class (sibling-two.component.ts):

import { Component, OnInit } from '@angular/core';
import { SharedService } from './shared.service';

@Component({
  selector: 'app-sibling-two',
  template: '<p>{{ siblingData }}</p>',
})
export class SiblingTwoComponent implements OnInit {
  siblingData: string;

  constructor(private sharedService: SharedService) {}

  ngOnInit() {
    this.sharedService.data$.subscribe(data => {
      this.siblingData = data;
    });
  }
}

In this example, the SharedService facilitates communication between SiblingOneComponent and SiblingTwoComponent.

Conclusion

Component interaction is a crucial aspect of building Angular applications. Understanding how to effectively communicate between components using input and output properties, shared parent components, and services will enable you to create more dynamic and responsive applications. As you progress from beginner to advanced levels, mastering these techniques will significantly enhance your ability to manage and structure your Angular projects.
