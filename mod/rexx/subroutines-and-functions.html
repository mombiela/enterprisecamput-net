Subroutines and Functions in REXX
Reading from Files ►

Introduction

Subroutines and functions are fundamental concepts in REXX programming that allow you to modularize your code, making it more readable, maintainable, and reusable. This section will guide you through the basics of subroutines and functions, and gradually introduce more advanced concepts.

Key Concepts

  • Subroutines: Blocks of code that perform a specific task and can be called from different parts of a program.
  • Functions: Similar to subroutines but designed to return a value.
  • CALL Statement: Used to invoke a subroutine.
  • RETURN Statement: Used to return control to the calling code, optionally with a value.

Subroutines

Defining a Subroutine

A subroutine in REXX is defined using a label followed by a colon. The subroutine can be called using the CALL statement.

/* Main Program */
say "Starting the program"
call mySubroutine
say "Back in the main program"

/* Subroutine Definition */
mySubroutine:
    say "Inside the subroutine"
    return

Calling a Subroutine

You can call a subroutine from anywhere in your program using the CALL statement.

call mySubroutine

Passing Arguments to Subroutines

Arguments can be passed to subroutines and accessed using the ARG instruction.

/* Main Program */
call mySubroutine "Hello, World!"

/* Subroutine Definition */
mySubroutine:
    parse arg message
    say message
    return

Functions

Defining a Function

Functions are similar to subroutines but are designed to return a value. They are defined using the same label and colon syntax.

/* Main Program */
result = myFunction(5, 3)
say "The result is" result

/* Function Definition */
myFunction:
    parse arg a, b
    return a + b

Calling a Function

Functions are called by using their name followed by parentheses containing any arguments.

result = myFunction(5, 3)

Returning Values from Functions

The RETURN statement is used to return a value from a function.

return a + b

Advanced Concepts

Local Variables

Variables in subroutines and functions are global by default. To create local variables, use the PROCEDURE instruction.

/* Main Program */
x = 10
call mySubroutine
say "Value of x in main program:" x

/* Subroutine Definition */
mySubroutine:
    procedure
    x = 5
    say "Value of x in subroutine:" x
    return

Recursive Functions

REXX supports recursive functions, where a function calls itself.

/* Main Program */
result = factorial(5)
say "Factorial of 5 is" result

/* Recursive Function Definition */
factorial:
    parse arg n
    if n = 1 then return 1
    return n * factorial(n - 1)

Summary

Subroutines and functions are essential tools in REXX for creating modular and reusable code. Subroutines perform tasks without returning values, while functions are designed to return values. Understanding how to define, call, and manage these constructs, including handling arguments and local variables, is crucial for effective REXX programming. By mastering these concepts, you can write more organized and efficient code.
