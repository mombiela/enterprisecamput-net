Calling External Programs in REXX
Interfacing with Databases ►

In this section, we will explore how to call external programs from within a REXX script. This is a powerful feature that allows REXX scripts to interact with other software and perform a wide range of tasks beyond the capabilities of REXX itself.

Introduction

Calling external programs can be useful for:

  • Automating system tasks.
  • Integrating REXX scripts with other software.
  • Extending the functionality of REXX by leveraging other tools.

Basic Syntax

In REXX, you can call an external program using the ADDRESS instruction. The ADDRESS instruction allows you to specify the environment in which the command should be executed.

Example: Calling a Shell Command

Here is a simple example of calling a shell command from a REXX script:

/* REXX script to call an external shell command */
ADDRESS SYSTEM 'echo Hello, World!'

In this example:

  • ADDRESS SYSTEM specifies that the command should be executed in the system shell.
  • 'echo Hello, World!' is the command being executed.

Capturing Output

You can capture the output of an external command by using the LINEIN function. This is useful when you need to process the output of the command within your REXX script.

Example: Capturing Command Output

/* REXX script to capture the output of a shell command */
ADDRESS SYSTEM 'dir > temp.txt'
output = LINEIN('temp.txt')
SAY output

In this example:

  • ADDRESS SYSTEM 'dir > temp.txt' executes the dir command and redirects the output to a file named temp.txt.
  • output = LINEIN('temp.txt') reads the first line of the output from the file.
  • SAY output prints the captured output.

Error Handling

When calling external programs, it is important to handle errors gracefully. You can check the return code of the command to determine if it executed successfully.

Example: Checking Return Code

/* REXX script to check the return code of a shell command */
ADDRESS SYSTEM 'nonexistent_command'
if RC \= 0 then
  SAY 'Command failed with return code:' RC
else
  SAY 'Command executed successfully'

In this example:

  • ADDRESS SYSTEM 'nonexistent_command' attempts to execute a command that does not exist.
  • if RC \= 0 then checks if the return code (RC) is not equal to 0, indicating an error.
  • SAY 'Command failed with return code:' RC prints an error message if the command failed.

Advanced Techniques

For more advanced use cases, you might need to interact with the external program in a more complex manner, such as passing arguments or handling multiple lines of output.

Example: Passing Arguments

/* REXX script to pass arguments to an external program */
arg1 = 'Hello'
arg2 = 'World'
ADDRESS SYSTEM 'echo' arg1 arg2

In this example:

  • arg1 and arg2 are variables containing the arguments to be passed.
  • ADDRESS SYSTEM 'echo' arg1 arg2 executes the echo command with the arguments.

Example: Handling Multiple Lines of Output

/* REXX script to handle multiple lines of output */
ADDRESS SYSTEM 'dir > temp.txt'
do while LINES('temp.txt') > 0
  line = LINEIN('temp.txt')
  SAY line
end

In this example:

  • do while LINES('temp.txt') > 0 loops while there are lines to read in the file.
  • line = LINEIN('temp.txt') reads each line of the output.
  • SAY line prints each line.

Conclusion

Calling external programs from REXX scripts expands the capabilities of your scripts by allowing them to interact with the system and other software. By understanding the basic syntax, capturing output, handling errors, and using advanced techniques, you can effectively integrate external commands into your REXX scripts.

Practice these concepts with various external programs and commands to become proficient in leveraging this powerful feature of REXX.
