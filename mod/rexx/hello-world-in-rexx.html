Introduction

REXX (Restructured Extended Executor) is a versatile scripting language developed by IBM. It is known for its simplicity and ease of use, making it an excellent choice for beginners. In this section, we will cover the basics of writing a simple "Hello World" program in REXX.

Writing Your First REXX Program

Key Concepts

  • REXX Syntax: REXX syntax is straightforward and easy to read.
  • Comments: Lines starting with /* and ending with */ are comments.
  • Statements: Each line in REXX is a statement, and they do not require a semicolon at the end.

Code Example

Let's start with a simple "Hello World" program in REXX.

/* This is a simple REXX program */
say "Hello, World!"

Explanation

  • Comments: The first line is a comment. Comments in REXX are enclosed within /* and */.
  • say Statement: The say statement is used to display output to the screen. In this case, it prints "Hello, World!".

Running Your REXX Program

Steps to Execute

  1. Save the Program: Save the above code in a file with a .rexx or .rex extension, for example, hello.rexx.
  2. Run the Program: Use a REXX interpreter to run the program. The command might vary depending on your environment. For example, in a Unix-like system, you can use: 
    rexx hello.rexx
    On Windows, you might use: 
    rexx.exe hello.rexx

Expected Output

When you run the program, you should see the following output:

Hello, World!

Common Errors and Troubleshooting

Syntax Errors

  • Missing Quotes: Ensure that the string "Hello, World!" is enclosed in double quotes.
  • Incorrect File Extension: Make sure the file is saved with the correct .rexx or .rex extension.

Runtime Errors

  • Interpreter Not Found: Ensure that the REXX interpreter is installed and properly configured in your system's PATH.

Summary

In this section, we covered the basics of writing and running a simple "Hello World" program in REXX. We discussed the syntax, how to save and execute the program, and common errors you might encounter. This foundational knowledge will help you as you progress to more complex REXX programming tasks.
