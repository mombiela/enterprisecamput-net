Introduction

REXX (Restructured Extended Executor) is a versatile scripting language that can be used in conjunction with other programming languages to enhance functionality and streamline processes. This section will explore how REXX can interact with other languages, providing examples and exercises to solidify your understanding.

Key Concepts

  • Interoperability: The ability of REXX to work with other programming languages.
  • APIs and Libraries: Utilizing APIs and libraries to facilitate communication between REXX and other languages.
  • Scripting and Automation: Using REXX scripts to automate tasks in environments where other languages are used.

Interoperability with Other Languages

Calling External Programs

REXX can call external programs written in other languages, such as C, Python, or Java. This is often done using the ADDRESS instruction.

Example:

/* REXX script to call a Python script */
ADDRESS SYSTEM 'python myscript.py'

Explanation:

  • ADDRESS SYSTEM tells REXX to execute a command in the system's command interpreter.
  • 'python myscript.py' is the command to run the Python script myscript.py.

Using APIs and Libraries

REXX can interact with APIs and libraries written in other languages. This is particularly useful for extending the capabilities of REXX scripts.

Example:

/* REXX script to call a C function via a shared library */
CALL RxFuncAdd 'SysLoadFuncs', 'RexxUtil', 'SysLoadFuncs'
CALL SysLoadFuncs
CALL MyCFunction

Explanation:

  • RxFuncAdd registers an external function.
  • SysLoadFuncs loads the functions from the RexxUtil library.
  • MyCFunction is a function defined in a C library that has been loaded.

Embedding REXX in Other Languages

REXX can be embedded in other languages to provide scripting capabilities within those languages.

Example in C:

#include <rexxsaa.h>

int main() {
    const char *script = "say 'Hello from REXX'";
    RexxStart(0, NULL, script, NULL, "COMMAND", RXCOMMAND, NULL, NULL, NULL);
    return 0;
}

Explanation:

  • RexxStart is a function from the REXX API that runs a REXX script.
  • The script say 'Hello from REXX' is executed within the C program.

Scripting and Automation

Automating Tasks

REXX is often used to automate tasks in environments where other languages are used. This can include file manipulation, data processing, and system administration tasks.

Example:

/* REXX script to automate a backup process */
ADDRESS SYSTEM 'tar -czf backup.tar.gz /path/to/directory'

Explanation:

  • The tar command is used to create a compressed archive of a directory.
  • This REXX script automates the backup process by running the tar command.

Integrating with Databases

REXX can interact with databases using SQL. This allows REXX scripts to perform database operations such as queries and updates.

Example:

/* REXX script to query a database */
ADDRESS SQL 'CONNECT TO mydatabase USER myuser USING mypassword'
ADDRESS SQL 'SELECT * FROM mytable'
DO WHILE SQLCODE = 0
    ADDRESS SQL 'FETCH NEXT ROW INTO :myvar'
    SAY myvar
END
ADDRESS SQL 'DISCONNECT'

Explanation:

  • ADDRESS SQL is used to send SQL commands to the database.
  • The script connects to a database, runs a query, fetches the results, and prints them.

Conclusion

Using REXX with other languages expands its capabilities and allows for more complex and powerful scripts. Whether calling external programs, using APIs, embedding REXX in other languages, or automating tasks, REXX's interoperability makes it a valuable tool for any programmer. Practice the examples provided and experiment with integrating REXX into your own projects to fully grasp its potential.
