Control Structures: IF, DO, and SELECT
String Manipulation ►

Introduction

Control structures are fundamental in any programming language as they allow you to control the flow of execution based on certain conditions. In REXX, the primary control structures include IF, DO, and SELECT. This section will guide you through these structures from a beginner to an advanced level.

IF Statements

The IF statement in REXX is used to execute a block of code conditionally.

Basic IF Statement

The simplest form of an IF statement checks a single condition.

/* Basic IF statement */
age = 20
if age >= 18 then
  say "You are an adult."

IF-ELSE Statement

You can extend the IF statement with an ELSE clause to handle alternative conditions.

/* IF-ELSE statement */
age = 16
if age >= 18 then
  say "You are an adult."
else
  say "You are a minor."

Nested IF Statements

IF statements can be nested to check multiple conditions.

/* Nested IF statements */
age = 16
if age >= 18 then
  say "You are an adult."
else
  if age >= 13 then
    say "You are a teenager."
  else
    say "You are a child."

DO Loops

The DO loop in REXX is used to repeat a block of code multiple times.

Basic DO Loop

A simple DO loop runs a fixed number of times.

/* Basic DO loop */
do i = 1 to 5
  say "Iteration" i
end

DO WHILE Loop

A DO WHILE loop runs as long as a condition is true.

/* DO WHILE loop */
i = 1
do while i <= 5
  say "Iteration" i
  i = i + 1
end

DO UNTIL Loop

A DO UNTIL loop runs until a condition becomes true.

/* DO UNTIL loop */
i = 1
do until i > 5
  say "Iteration" i
  i = i + 1
end

DO FOREVER Loop

A DO FOREVER loop runs indefinitely until explicitly terminated.

/* DO FOREVER loop */
i = 1
do forever
  say "Iteration" i
  i = i + 1
  if i > 5 then leave
end

SELECT Case

The SELECT statement in REXX is used to handle multiple conditions more efficiently than nested IF statements.

Basic SELECT Statement

A SELECT statement evaluates multiple conditions and executes the corresponding block of code.

/* Basic SELECT statement */
day = "Monday"
select
  when day = "Monday" then
    say "Start of the work week."
  when day = "Friday" then
    say "End of the work week."
  otherwise
    say "Midweek day."
end

Nested SELECT Statements

SELECT statements can also be nested for more complex decision-making.

/* Nested SELECT statements */
day = "Monday"
time = "Morning"
select
  when day = "Monday" then
    select
      when time = "Morning" then
        say "Start of the work week in the morning."
      otherwise
        say "Start of the work week in the afternoon."
    end
  when day = "Friday" then
    say "End of the work week."
  otherwise
    say "Midweek day."
end

Conclusion

Control structures like IF, DO, and SELECT are essential for managing the flow of your REXX programs. Understanding these constructs allows you to write more efficient and readable code. Practice these examples and try creating your own scenarios to deepen your understanding.
