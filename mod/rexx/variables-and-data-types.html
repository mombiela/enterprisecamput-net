Variables and Data Types in REXX

Introduction

In this section, we will explore the fundamental concepts of variables and data types in REXX. Understanding these concepts is crucial for writing effective and efficient REXX programs. We will start with the basics and gradually move to more advanced topics.

Variables in REXX

What is a Variable?

  • A variable in REXX is a symbolic name associated with a value.
  • Variables are used to store data that can be manipulated and retrieved throughout the program.

Declaring Variables

  • In REXX, variables do not need to be explicitly declared before use.
  • A variable is created the first time it is assigned a value.
/* Example of variable assignment */
name = "John Doe"
age = 30

Variable Naming Rules

  • Variable names must start with a letter (A-Z, a-z) or a special character (such as . or _).
  • Subsequent characters can be letters, digits (0-9), or special characters.
  • Variable names are case-insensitive.
/* Valid variable names */
firstName = "Alice"
_lastName = "Smith"
age2 = 25

/* Invalid variable names */
// 2ndName = "Bob"  // Starts with a digit
// @age = 30        // Starts with an invalid character

Assigning Values to Variables

  • Values are assigned to variables using the = operator.
  • REXX automatically determines the type of the value assigned.
/* Assigning different types of values */
name = "Jane Doe"
age = 28
height = 5.7
isStudent = "Yes"

Data Types in REXX

Overview of Data Types

  • REXX is a dynamically typed language, meaning that the type of a variable is determined at runtime.
  • The primary data types in REXX are:
    • Strings
    • Numbers (integers and floating-point)
    • Boolean values (represented as strings "0" or "1")

Strings

  • Strings are sequences of characters enclosed in double quotes (").
/* Example of string variables */
greeting = "Hello, World!"
name = "Alice"

Numbers

  • Numbers in REXX can be integers or floating-point.
  • REXX handles numeric operations automatically.
/* Example of numeric variables */
integerNumber = 42
floatingNumber = 3.14

/* Performing arithmetic operations */
sum = integerNumber + floatingNumber

Boolean Values

  • REXX does not have a distinct boolean type. Instead, it uses strings "0" (false) and "1" (true).
/* Example of boolean values */
isTrue = "1"
isFalse = "0"

Type Conversion

  • REXX automatically converts between strings and numbers as needed.
  • Explicit conversion can be done using built-in functions.
/* Automatic type conversion */
numString = "123"
number = numString + 1  /* Converts "123" to 123 and adds 1 */

/* Explicit type conversion */
stringNumber = "456"
convertedNumber = VALUE(stringNumber)

Advanced Topics

Compound Variables

  • Compound variables (also known as arrays or associative arrays) allow for more complex data structures.
  • They are indexed by strings or numbers.
/* Example of compound variables */
person.1 = "Alice"
person.2 = "Bob"
person.3 = "Charlie"

say person.1  /* Outputs: Alice */

Variable Scope

  • Variables in REXX can be local or global.
  • Local variables are defined within a procedure and are not accessible outside of it.
  • Global variables are accessible throughout the entire program.
/* Example of variable scope */
globalVar = "I am global"

procedure
  localVar = "I am local"
  say localVar
return

say globalVar  /* Outputs: I am global */
// say localVar  /* Error: localVar is not defined outside the procedure */

Conclusion

In this section, we covered the basics of variables and data types in REXX. We learned how to declare and assign values to variables, the different data types available, and how to perform type conversions. We also touched on more advanced topics like compound variables and variable scope. Understanding these concepts is essential for writing effective REXX programs.
