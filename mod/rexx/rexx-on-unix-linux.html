Introduction to REXX

  • What is REXX?
    • REXX (REstructured eXtended eXecutor) is a scripting language developed by IBM.
    • Known for its ease of use and readability.
  • History and Evolution
    • Originated in the 1970s for IBM mainframes.
    • Evolved to support multiple platforms including Unix/Linux.

Setting Up REXX on Unix/Linux

  • Installation
    • Most Unix/Linux distributions include REXX interpreters like Regina REXX.
    • Installation command example for Regina REXX: 
      sudo apt-get install regina-rexx
  • Verifying Installation
    • Check the installation by running: 
      rexx -v

Basic Syntax and Structure

  • Hello World Program

    • Example: 
      /* Hello World in REXX */
say "Hello, World!"
    • Explanation:
      • /* ... */ is used for comments.
      • say is a built-in function to print text to the screen.

  • Variables and Data Types

    • Variables are dynamically typed.
    • Example: 
      name = "Alice"
age = 30
say "Name:" name
say "Age:" age

Control Structures

  • Conditional Statements

    • if, else, and select statements.
    • Example: 
      age = 20
if age >= 18 then
  say "You are an adult."
else
  say "You are a minor."

  • Loops

    • do loops.
    • Example: 
      do i = 1 to 5
  say "Number:" i
end

Functions and Procedures

  • Built-in Functions

    • Common functions like length, substr, left, right.
    • Example: 
      text = "Hello, World!"
say "Length:" length(text)
say "Substring:" substr(text, 1, 5)

  • User-defined Functions

    • Example: 
      say "Sum of 5 and 3 is:" sum(5, 3)

sum: procedure
  parse arg a, b
  return a + b

File Handling

  • Reading from a File

    • Example: 
      file = "example.txt"
call linein file
do while lines(file) > 0
  line = linein(file)
  say line
end
call lineout file

  • Writing to a File

    • Example: 
      file = "output.txt"
call lineout file, "This is a line of text."
call lineout file

Advanced Topics

  • Interfacing with Unix/Linux Commands

    • Using address to run shell commands.
    • Example: 
      address system "ls -l"

  • Error Handling

    • Using signal to handle errors.
    • Example: 
      signal on error
call nonexistent_function
exit

error:
  say "An error occurred."

  • Regular Expressions

    • Using regular expressions for pattern matching.
    • Example: 
      text = "The quick brown fox"
if text \= ".*quick.*" then
  say "Match found!"

Conclusion

  • Summary
    • REXX is a versatile scripting language suitable for various tasks on Unix/Linux.
    • Key concepts include basic syntax, control structures, file handling, and advanced interfacing with system commands.
  • Next Steps
    • Practice writing REXX scripts to automate tasks.
    • Explore more advanced features and libraries available for REXX.

By following this structured approach, you will gain a solid understanding of REXX on Unix/Linux, from basic scripting to advanced functionalities.
