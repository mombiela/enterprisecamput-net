Case Statements in Bash
Functions in Bash ►

Introduction

Case statements in Bash are a powerful tool for handling multiple conditions. They allow you to execute different blocks of code based on the value of a variable, making your scripts more readable and maintainable.

Key Concepts

  • Syntax: The basic structure of a case statement.
  • Patterns: How to match different patterns.
  • Fall-through: Understanding how case statements handle multiple matches.
  • Default Case: Using the default case to handle unmatched patterns.

Syntax

The basic syntax of a case statement in Bash is as follows:

case expression in
    pattern1)
        # commands
        ;;
    pattern2)
        # commands
        ;;
    *)
        # default commands
        ;;
esac
  • expression: The variable or value you are testing.
  • pattern1, pattern2, etc.: Patterns to match against the expression.
  • ;;: Ends each block of commands.
  • *): The default case, executed if no patterns match.
  • esac: Ends the case statement.

Examples

Basic Example

Here's a simple example to illustrate how case statements work:

#!/bin/bash

echo "Enter a number between 1 and 3:"
read number

case $number in
    1)
        echo "You entered one."
        ;;
    2)
        echo "You entered two."
        ;;
    3)
        echo "You entered three."
        ;;
    *)
        echo "Invalid number."
        ;;
esac

Using Patterns

You can use wildcards and ranges in patterns:

#!/bin/bash

echo "Enter a character:"
read char

case $char in
    [a-z])
        echo "You entered a lowercase letter."
        ;;
    [A-Z])
        echo "You entered an uppercase letter."
        ;;
    [0-9])
        echo "You entered a digit."
        ;;
    *)
        echo "You entered a special character."
        ;;
esac

Multiple Patterns

You can match multiple patterns with a single block of commands:

#!/bin/bash

echo "Enter a day of the week:"
read day

case $day in
    "Monday"|"Tuesday"|"Wednesday"|"Thursday"|"Friday")
        echo "It's a weekday."
        ;;
    "Saturday"|"Sunday")
        echo "It's the weekend."
        ;;
    *)
        echo "Invalid day."
        ;;
esac

Advanced Usage

Nested Case Statements

You can nest case statements within each other for more complex logic:

#!/bin/bash

echo "Enter a fruit (apple, banana, cherry):"
read fruit

case $fruit in
    "apple")
        echo "You chose apple."
        ;;
    "banana")
        echo "You chose banana."
        ;;
    "cherry")
        echo "You chose cherry."
        ;;
    *)
        echo "Unknown fruit."
        ;;
esac

Using Variables in Patterns

You can use variables within patterns to make your case statements more dynamic:

#!/bin/bash

echo "Enter a file extension (e.g., .txt, .sh):"
read extension

case $extension in
    *.txt)
        echo "This is a text file."
        ;;
    *.sh)
        echo "This is a shell script."
        ;;
    *)
        echo "Unknown file type."
        ;;
esac

Conclusion

Case statements in Bash provide a flexible and readable way to handle multiple conditions. By understanding the syntax, using patterns effectively, and leveraging advanced features like nested case statements and variable patterns, you can write more efficient and maintainable scripts. Practice with the examples provided and try creating your own case statements to solidify your understanding.
