Introduction to Process Signals

In Unix-like operating systems, signals are a form of inter-process communication used to notify processes that a specific event has occurred. Signals can be sent by the operating system, other processes, or the process itself.

  • Signals: Notifications sent to a process to trigger predefined actions.
  • Common Signals: SIGINT, SIGTERM, SIGKILL, SIGHUP, SIGSTOP, SIGCONT.

Common Signals and Their Uses

Here is a table of some common signals and their typical uses:

| Signal | Number | Description | Default Action | |---------|--------|---------------------------------------|----------------------| | SIGINT| 2 | Interrupt from keyboard (Ctrl+C) | Terminate process | | SIGTERM| 15 | Termination signal | Terminate process | | SIGKILL| 9 | Kill signal | Forcefully terminate | | SIGHUP| 1 | Hangup detected on controlling terminal | Terminate process | | SIGSTOP| 19 | Stop the process | Stop process | | SIGCONT| 18 | Continue if stopped | Continue process |

Sending Signals

You can send signals to processes using the kill command. Despite its name, kill can send any signal, not just terminate processes.

Examples:

  • Sending SIGTERM to a process:

    kill -SIGTERM <pid>

    or

    kill -15 <pid>

  • Sending SIGKILL to a process:

    kill -SIGKILL <pid>

    or

    kill -9 <pid>

  • Sending SIGINT to a process:

    kill -SIGINT <pid>

    or

    kill -2 <pid>

Handling Signals in Bash Scripts

You can write Bash scripts to handle signals using the trap command. This allows you to define custom actions when your script receives specific signals.

Example:

#!/bin/bash

# Define a handler for SIGINT
trap 'echo "Caught SIGINT signal"; exit' SIGINT

echo "Script is running. Press Ctrl+C to trigger SIGINT."

# Infinite loop to keep the script running
while true; do
  sleep 1
done

In this script:

  • The trap command catches the SIGINT signal (usually sent by pressing Ctrl+C).
  • When SIGINT is received, the script prints a message and exits.

Practical Exercises

  1. Send a SIGTERM signal to a process:

    • Start a long-running process (e.g., sleep 1000).
    • Use the ps command to find the process ID (PID) of the sleep command.
    • Send a SIGTERM signal to the process using the kill command.

  2. Handle SIGINT in a Bash script:

    • Write a script that runs indefinitely.
    • Use the trap command to catch the SIGINT signal.
    • Print a message and exit when SIGINT is received.

  3. Experiment with other signals:

    • Modify the script to handle SIGTERM and SIGHUP.
    • Observe the behavior when these signals are sent to the script.

Conclusion

Understanding process signals is crucial for managing and controlling processes in Unix-like operating systems. Signals allow you to gracefully handle interruptions, terminate processes, and perform cleanup tasks. By mastering the use of signals and the trap command in Bash scripts, you can create more robust and responsive scripts.
