System Monitoring
User and Group Management ►

System monitoring is a crucial aspect of system administration and scripting. In this topic, we will explore how to use Bash to monitor system performance and resource usage. This includes checking CPU usage, memory usage, disk space, and network activity.

Key Concepts

  • CPU Usage: Monitoring how much of the CPU is being used by processes.
  • Memory Usage: Checking the amount of RAM being used and available.
  • Disk Space: Keeping track of disk usage and available space.
  • Network Activity: Monitoring network traffic and connections.

CPU Usage

To monitor CPU usage, we can use the top and ps commands.

Using `top`

The top command provides a dynamic, real-time view of the system's processes.

top
  • Explanation: When you run top, it displays a list of processes sorted by CPU usage. You can see the percentage of CPU each process is using, along with other details like memory usage and process ID (PID).

Using `ps`

The ps command can be used to display information about active processes.

ps aux --sort=-%cpu | head -n 10
  • Explanation: This command lists all processes (ps aux), sorts them by CPU usage in descending order (--sort=-%cpu), and then displays the top 10 processes (head -n 10).

Memory Usage

To monitor memory usage, we can use the free and vmstat commands.

Using `free`

The free command displays the amount of free and used memory in the system.

free -h
  • Explanation: The -h option makes the output human-readable, showing memory sizes in KB, MB, or GB.

Using `vmstat`

The vmstat command reports information about processes, memory, paging, block IO, traps, and CPU activity.

vmstat 1 5
  • Explanation: This command displays system performance statistics every second for 5 seconds.

Disk Space

To monitor disk space usage, we can use the df and du commands.

Using `df`

The df command reports the amount of disk space used and available on filesystems.

df -h
  • Explanation: The -h option makes the output human-readable.

Using `du`

The du command estimates file space usage.

du -sh /path/to/directory
  • Explanation: The -s option provides a summary of the directory, and the -h option makes the output human-readable.

Network Activity

To monitor network activity, we can use the netstat and iftop commands.

Using `netstat`

The netstat command displays network connections, routing tables, interface statistics, masquerade connections, and multicast memberships.

netstat -tuln
  • Explanation: The -tuln options display TCP and UDP connections in a listening state, without resolving hostnames.

Using `iftop`

The iftop command displays bandwidth usage on an interface by host.

sudo iftop
  • Explanation: This command requires root privileges and provides a real-time view of network traffic.

Summary

In this topic, we covered the basics of system monitoring using Bash. We learned how to monitor CPU usage with top and ps, check memory usage with free and vmstat, track disk space with df and du, and observe network activity with netstat and iftop. These tools are essential for maintaining the health and performance of a system. By regularly monitoring these metrics, you can identify and address potential issues before they become critical.
