Managing Processes
Using Pipes and Redirection ►

Managing processes is a crucial skill when working with Bash. This topic will cover the basics of process management, including how to start, stop, and monitor processes. By the end of this section, you will have a solid understanding of how to manage processes effectively in a Unix-like environment.

Starting Processes

In Bash, starting a process is as simple as typing a command and pressing Enter. However, there are different ways to start processes depending on your needs.

Foreground Processes

  • Definition: A foreground process is one that takes over the terminal until it completes.

  • Example:

    ls -l

    This command lists the contents of the current directory in long format and runs in the foreground.

Background Processes

  • Definition: A background process runs independently of the terminal, allowing you to continue using the terminal for other commands.

  • Example:

    sleep 30 &

    The & at the end of the command sends the sleep command to the background, allowing you to use the terminal while the command runs.

Monitoring Processes

Monitoring processes is essential for understanding system performance and managing resources.

`ps` Command

  • Definition: The ps command displays information about active processes.

  • Example:

    ps aux

    This command shows a detailed list of all running processes.

    | Column | Description | |--------|-------------| | USER | The user who owns the process | | PID | Process ID | | %CPU | CPU usage percentage | | %MEM | Memory usage percentage | | VSZ | Virtual memory size | | RSS | Resident Set Size (physical memory) | | TTY | Terminal associated with the process | | STAT | Process state | | START | Start time of the process | | TIME | Cumulative CPU time | | COMMAND| Command that started the process |

`top` Command

  • Definition: The top command provides a dynamic, real-time view of running processes.

  • Example:

    top

    This command opens an interactive interface showing the most resource-intensive processes.

Controlling Processes

Sometimes, you need to stop or pause processes. Bash provides several commands for this purpose.

`kill` Command

  • Definition: The kill command sends a signal to a process, usually to terminate it.

  • Example:

    kill 1234

    This command sends the default TERM signal to the process with PID 1234, asking it to terminate gracefully.

`killall` Command

  • Definition: The killall command sends a signal to all processes running a specified command.

  • Example:

    killall firefox

    This command sends the default TERM signal to all instances of the firefox process.

`bg` and `fg` Commands

  • Definition: The bg command resumes a suspended job in the background, while the fg command brings a background job to the foreground.

  • Example:

    sleep 100
^Z
bg
fg

    In this example, ^Z (Ctrl+Z) suspends the sleep command, bg resumes it in the background, and fg brings it back to the foreground.

Summary

In this section, we covered the basics of managing processes in Bash. You learned how to start processes in the foreground and background, monitor them using commands like ps and top, and control them using commands like kill, killall, bg, and fg. Mastering these commands will help you manage system resources effectively and ensure smooth operation of your Unix-like environment.
