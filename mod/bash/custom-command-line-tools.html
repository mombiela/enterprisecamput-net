Introduction

Creating custom command line tools can significantly enhance your productivity and streamline repetitive tasks. In this section, we will explore how to create, manage, and utilize custom command line tools using Bash.

Key Concepts

  • Shebang (#!): The first line in a script that specifies the interpreter.
  • Script Permissions: Making scripts executable.
  • Command Line Arguments: Passing arguments to scripts.
  • Environment Variables: Using and managing environment variables.
  • Functions: Creating reusable code blocks within scripts.
  • Error Handling: Managing errors and exit statuses.

Creating a Simple Script

Example: Hello World Script

#!/bin/bash
# This is a simple Hello World script
echo "Hello, World!"
  • Explanation:
    • #!/bin/bash: Shebang line specifying the script should be run with Bash.
    • echo "Hello, World!": Prints "Hello, World!" to the terminal.

Making the Script Executable

chmod +x hello_world.sh
  • Explanation:
    • chmod +x hello_world.sh: Changes the file permissions to make the script executable.

Command Line Arguments

Example: Greeting Script with Arguments

#!/bin/bash
# This script greets the user with their name
echo "Hello, $1!"
  • Explanation:
    • $1: Represents the first argument passed to the script.

Running the Script with Arguments

./greeting.sh Alice
  • Output:
    • Hello, Alice!

Environment Variables

Example: Using Environment Variables

#!/bin/bash
# This script demonstrates the use of environment variables
echo "The current user is $USER"
echo "The home directory is $HOME"
  • Explanation:
    • $USER: Environment variable for the current user.
    • $HOME: Environment variable for the home directory.

Functions in Scripts

Example: Function to Add Two Numbers

#!/bin/bash
# Function to add two numbers
add_numbers() {
  local sum=$(( $1 + $2 ))
  echo "The sum is: $sum"
}

# Calling the function
add_numbers 3 5
  • Explanation:
    • add_numbers() { ... }: Defines a function named add_numbers.
    • local sum=$(( $1 + $2 )): Calculates the sum of two arguments.
    • echo "The sum is: $sum": Prints the sum.

Error Handling

Example: Checking for File Existence

#!/bin/bash
# This script checks if a file exists
if [ -f "$1" ]; then
  echo "File $1 exists."
else
  echo "File $1 does not exist."
  exit 1
fi
  • Explanation:
    • [ -f "$1" ]: Checks if the file specified by the first argument exists.
    • exit 1: Exits the script with a status code of 1 if the file does not exist.

Conclusion

Creating custom command line tools in Bash can greatly enhance your workflow by automating repetitive tasks and providing quick solutions to common problems. By understanding the basics of scripting, handling command line arguments, using environment variables, defining functions, and implementing error handling, you can build powerful and efficient tools tailored to your needs. Practice these concepts with various examples to become proficient in creating your own custom command line tools.
