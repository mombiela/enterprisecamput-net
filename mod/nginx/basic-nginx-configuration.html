Introduction to NGINX

NGINX is a high-performance HTTP server, reverse proxy, and an IMAP/POP3 proxy server. It is known for its stability, rich feature set, simple configuration, and low resource consumption.

Key Concepts

  • Server Block: Defines a virtual server used to handle requests.
  • Location Block: Defines how to process specific sets of requests.
  • Directives: Instructions that control NGINX behavior.

Installing NGINX

Before configuring NGINX, you need to install it. Here’s how to install NGINX on a Debian-based system:

sudo apt update
sudo apt install nginx

Basic Configuration File Structure

The main configuration file for NGINX is typically located at /etc/nginx/nginx.conf. Here’s a basic structure:

user www-data;
worker_processes auto;
pid /run/nginx.pid;

events {
    worker_connections 1024;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    types_hash_max_size 2048;

    include /etc/nginx/conf.d/*.conf;
    include /etc/nginx/sites-enabled/*;
}

Explanation:

  • user: Defines the user and group that the NGINX worker processes will run as.
  • worker_processes: Specifies the number of worker processes.
  • events: Contains directives that affect the connection processing.
  • http: Main context for HTTP configuration.

Configuring a Server Block

A server block defines a virtual server. Here’s an example configuration for a basic server block:

server {
    listen 80;
    server_name example.com www.example.com;

    location / {
        root /var/www/html;
        index index.html index.htm;
    }

    error_page 404 /404.html;
    location = /404.html {
        internal;
    }
}

Explanation:

  • listen: Specifies the port NGINX will listen on.
  • server_name: Defines the domain names that this server block will respond to.
  • location: Defines how to process requests for the root URL (/).
  • root: Specifies the root directory for the server.
  • index: Defines the index files to look for when a directory is requested.
  • error_page: Customizes the error page for a specific HTTP status code.

Configuring a Location Block

Location blocks allow you to define how NGINX should process specific sets of requests. Here’s an example:

server {
    listen 80;
    server_name example.com;

    location / {
        root /var/www/html;
        index index.html index.htm;
    }

    location /images/ {
        root /var/www/media;
    }

    location /api/ {
        proxy_pass http://localhost:3000;
    }
}

Explanation:

  • location /images/: Requests to /images/ will be served from /var/www/media.
  • location /api/: Requests to /api/ will be proxied to http://localhost:3000.

Common Directives

Here are some common directives you might use in your NGINX configuration:

| Directive | Description | |-----------------|--------------------------------------------------| | listen | Specifies the port and address to listen on. | | server_name | Defines the domain names for the server block. | | root | Sets the root directory for requests. | | index | Specifies the index files for directories. | | error_page | Customizes error pages. | | proxy_pass | Forwards requests to another server. | | location | Defines how to process specific request URIs. |

Conclusion

Understanding the basic configuration of NGINX is crucial for setting up a stable and efficient web server. By mastering server blocks, location blocks, and common directives, you can effectively manage and serve web content.
