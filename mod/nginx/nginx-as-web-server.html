Introduction to NGINX

NGINX is a powerful, high-performance web server that can also be used as a reverse proxy, load balancer, and HTTP cache. It is known for its stability, rich feature set, simple configuration, and low resource consumption.

Installing NGINX

To start using NGINX, you first need to install it on your system. The installation process varies depending on your operating system.

On Ubuntu/Debian

sudo apt update
sudo apt install nginx

On CentOS/RHEL

sudo yum install epel-release
sudo yum install nginx

On Windows

For Windows, you can download the NGINX binaries from the official NGINX website and follow the installation instructions provided.

Basic Configuration

Once installed, you need to configure NGINX to serve your web content.

Default Configuration File

The main configuration file for NGINX is usually located at /etc/nginx/nginx.conf.

Basic Configuration Example

Below is a simple example of an NGINX configuration file to serve static content from /var/www/html:

server {
    listen 80;
    server_name example.com;

    location / {
        root /var/www/html;
        index index.html index.htm;
    }
}
  • listen 80; - Tells NGINX to listen on port 80.
  • server_name example.com; - Specifies the domain name.
  • location / { ... } - Defines how to handle requests to the root URL.

Starting and Stopping NGINX

You can control the NGINX service using the following commands:

On Ubuntu/Debian

sudo systemctl start nginx
sudo systemctl stop nginx
sudo systemctl restart nginx

On CentOS/RHEL

sudo systemctl start nginx
sudo systemctl stop nginx
sudo systemctl restart nginx

Serving Static Content

NGINX is often used to serve static content such as HTML, CSS, and JavaScript files.

Example Configuration

server {
    listen 80;
    server_name example.com;

    location / {
        root /var/www/html;
        index index.html;
    }

    location /images/ {
        root /var/www/html;
    }
}
  • The location /images/ block specifies that requests to /images/ should be served from /var/www/html/images/.

Handling Dynamic Content

NGINX can also be configured to handle dynamic content by passing requests to an application server such as PHP-FPM, Node.js, or Python.

Example with PHP-FPM

server {
    listen 80;
    server_name example.com;

    root /var/www/html;
    index index.php index.html;

    location / {
        try_files $uri $uri/ =404;
    }

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/var/run/php/php7.4-fpm.sock;
    }
}
  • try_files $uri $uri/ =404; - Tries to serve the requested file, if not found, returns a 404 error.
  • location ~ \.php$ { ... } - Handles PHP files by passing them to the PHP-FPM service.

Security Best Practices

To ensure your NGINX server is secure, follow these best practices:

  • Disable Server Tokens: Prevent NGINX from displaying its version number.

    server_tokens off;

  • Limit Request Size: Prevent large requests that could potentially harm your server.

    client_max_body_size 1M;

  • Use HTTPS: Always use HTTPS to encrypt data between the client and server.

    server {
    listen 443 ssl;
    server_name example.com;

    ssl_certificate /etc/nginx/ssl/example.com.crt;
    ssl_certificate_key /etc/nginx/ssl/example.com.key;

    location / {
        root /var/www/html;
        index index.html;
    }
}

Conclusion

Using NGINX as a web server involves installing the software, configuring it to serve static and dynamic content, and applying security best practices. By mastering these basics, you can leverage NGINX's powerful features to build robust and efficient web applications.
