Setting Up SSL/TLS

Introduction to SSL/TLS

  • SSL (Secure Sockets Layer) and TLS (Transport Layer Security) are protocols for establishing authenticated and encrypted links between networked computers.
  • SSL is the predecessor to TLS. TLS is more secure and is the modern standard.
  • Using SSL/TLS with NGINX ensures that data transmitted between the server and clients is encrypted and secure.

Prerequisites

  • Basic understanding of NGINX configuration.
  • A domain name pointing to your server.
  • Administrative access to your server.
  • OpenSSL installed on your server.

Obtaining an SSL Certificate

  • Self-Signed Certificate: Useful for testing and development.
  • Certificate Authority (CA) Signed Certificate: Required for production environments to ensure trust.

Generating a Self-Signed Certificate

  1. Generate a Private Key: 
    openssl genpkey -algorithm RSA -out /etc/nginx/ssl/private.key
  2. Generate a Certificate Signing Request (CSR): 
    openssl req -new -key /etc/nginx/ssl/private.key -out /etc/nginx/ssl/certificate.csr
  3. Generate the Self-Signed Certificate: 
    openssl x509 -req -days 365 -in /etc/nginx/ssl/certificate.csr -signkey /etc/nginx/ssl/private.key -out /etc/nginx/ssl/certificate.crt

Obtaining a CA-Signed Certificate

  1. Generate a Private Key: 
    openssl genpkey -algorithm RSA -out /etc/nginx/ssl/private.key
  2. Generate a Certificate Signing Request (CSR): 
    openssl req -new -key /etc/nginx/ssl/private.key -out /etc/nginx/ssl/certificate.csr
  3. Submit the CSR to a Certificate Authority (CA): Follow the CA's instructions to obtain your signed certificate.

Configuring NGINX for SSL/TLS

  1. Edit the NGINX Configuration File:

    • Open the NGINX configuration file (usually located at /etc/nginx/nginx.conf or /etc/nginx/sites-available/default).

  2. Add the SSL Configuration:

    server {
    listen 443 ssl;
    server_name your_domain.com;

    ssl_certificate /etc/nginx/ssl/certificate.crt;
    ssl_certificate_key /etc/nginx/ssl/private.key;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    location / {
        root /var/www/html;
        index index.html index.htm;
    }
}

  3. Redirect HTTP to HTTPS:

    server {
    listen 80;
    server_name your_domain.com;

    location / {
        return 301 https://$host$request_uri;
    }
}

  4. Test the Configuration:

    sudo nginx -t

  5. Reload NGINX:

    sudo systemctl reload nginx

Advanced SSL/TLS Configuration

  • HTTP/2 Support: Improves performance by allowing multiple requests and responses to be multiplexed over a single connection. 
    server {
    listen 443 ssl http2;
    ...
}
  • OCSP Stapling: Improves SSL/TLS performance by allowing the server to provide the client with a cached OCSP response. 
    ssl_stapling on;
ssl_stapling_verify on;
ssl_trusted_certificate /etc/nginx/ssl/ca-certs.pem;
resolver 8.8.8.8 8.8.4.4 valid=300s;
resolver_timeout 5s;
  • HSTS (HTTP Strict Transport Security): Instructs browsers to only interact with your site using HTTPS. 
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

Conclusion

Setting up SSL/TLS with NGINX is crucial for securing web traffic. By following the steps outlined, you can ensure that your server communicates securely with clients. Remember to keep your SSL/TLS configuration up to date to protect against vulnerabilities and to provide the best performance and security for your users.
