Overview

Kafka Connect is a framework for connecting Kafka with external systems such as databases, key-value stores, search indexes, and file systems. It simplifies the process of integrating Kafka with other systems, allowing for scalable and reliable data pipelines.

Key Concepts

  • Connector: A connector is a reusable component that captures data from a source system or sends data to a target system.
    • Source Connector: Captures data from an external system and writes it to Kafka topics.
    • Sink Connector: Reads data from Kafka topics and writes it to an external system.
  • Task: A task is a unit of work that a connector performs. Connectors can be configured to run multiple tasks in parallel for scalability.
  • Worker: A worker is a JVM process that executes connectors and tasks. Workers can be run in standalone or distributed mode.
  • Standalone Mode: Suitable for simple, single-node setups.
  • Distributed Mode: Suitable for large-scale, fault-tolerant setups.

Setting Up Kafka Connect

Prerequisites

  • Apache Kafka installed and running.
  • Java Development Kit (JDK) installed.

Standalone Mode Example

  1. Create a properties file for the connector:

    name=local-file-source
connector.class=FileStreamSource
tasks.max=1
file=/path/to/input/file.txt
topic=connect-test

  2. Run Kafka Connect in standalone mode:

    bin/connect-standalone.sh config/connect-standalone.properties config/connect-file-source.properties

Distributed Mode Example

  1. Create a properties file for the worker:

    bootstrap.servers=localhost:9092
group.id=connect-cluster
key.converter=org.apache.kafka.connect.json.JsonConverter
value.converter=org.apache.kafka.connect.json.JsonConverter
config.storage.topic=connect-configs
offset.storage.topic=connect-offsets
status.storage.topic=connect-status

  2. Start the Kafka Connect worker:

    bin/connect-distributed.sh config/connect-distributed.properties

  3. Submit a connector configuration:

    curl -X POST -H "Content-Type: application/json" --data '{
    "name": "local-file-source",
    "config": {
        "connector.class": "FileStreamSource",
        "tasks.max": "1",
        "file": "/path/to/input/file.txt",
        "topic": "connect-test"
    }
}' http://localhost:8083/connectors

Common Connectors

Source Connectors

  • JDBC Source Connector: Captures data from relational databases.
  • File Source Connector: Captures data from files.

Sink Connectors

  • JDBC Sink Connector: Writes data to relational databases.
  • Elasticsearch Sink Connector: Writes data to Elasticsearch.

Example: JDBC Source Connector

Configuration

  1. Create a properties file for the JDBC Source Connector:

    name=my-jdbc-source
connector.class=io.confluent.connect.jdbc.JdbcSourceConnector
tasks.max=1
connection.url=jdbc:mysql://localhost:3306/mydb
connection.user=myuser
connection.password=mypassword
table.whitelist=mytable
mode=incrementing
incrementing.column.name=id
topic.prefix=mydb-

  2. Start the connector:

    bin/connect-standalone.sh config/connect-standalone.properties config/my-jdbc-source.properties

Monitoring and Management

  • REST API: Kafka Connect provides a REST API for managing connectors, tasks, and workers.
  • Metrics: Kafka Connect exposes metrics via JMX for monitoring performance and health.

Conclusion

Kafka Connect is a powerful tool for integrating Kafka with various external systems. By understanding its key concepts and configurations, you can build scalable and reliable data pipelines. Whether you are working in standalone or distributed mode, Kafka Connect simplifies the process of data ingestion and export, making it an essential component in the Kafka ecosystem.
