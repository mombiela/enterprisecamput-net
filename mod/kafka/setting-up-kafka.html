Setting Up Kafka
Kafka Topics and Partitions ►

Setting up Kafka involves several steps, from downloading and installing Kafka to configuring it for your specific needs. This section will guide you through the process, ensuring you have a working Kafka environment by the end.

Prerequisites

Before setting up Kafka, ensure you have the following:

  • Java Development Kit (JDK): Kafka requires Java to run. Ensure you have JDK 8 or later installed.
  • Zookeeper: Kafka uses Zookeeper to manage its cluster. Zookeeper must be installed and running before starting Kafka.

Downloading Kafka

  1. Visit the Kafka Download Page: Go to the Apache Kafka download page.
  2. Select a Version: Choose the Kafka version you want to download. It's recommended to use the latest stable version.
  3. Download the Binary: Download the binary files for your operating system.

Installing Kafka

  1. Extract the Archive: Once downloaded, extract the Kafka archive to your desired directory.

    tar -xzf kafka_2.13-2.8.0.tgz
cd kafka_2.13-2.8.0

  2. Directory Structure: After extraction, you should see the following directory structure:

    kafka_2.13-2.8.0/
├── bin
├── config
├── libs
├── logs
└── site-docs

Configuring Zookeeper

Kafka requires Zookeeper to manage its cluster. Zookeeper is included in the Kafka distribution.

  1. Start Zookeeper: Use the provided script to start Zookeeper.

    bin/zookeeper-server-start.sh config/zookeeper.properties

  2. Verify Zookeeper: Ensure Zookeeper is running by checking the logs or using a Zookeeper client.

Configuring Kafka

  1. Edit Kafka Configuration: Open the config/server.properties file to configure Kafka.

    • Broker ID: Unique identifier for each Kafka broker. 
      broker.id=0
    • Log Directory: Directory where Kafka will store its logs. 
      log.dirs=/tmp/kafka-logs
    • Zookeeper Connection: Address of the Zookeeper instance. 
      zookeeper.connect=localhost:2181

  2. Start Kafka: Use the provided script to start Kafka.

    bin/kafka-server-start.sh config/server.properties

Verifying the Installation

  1. Create a Topic: Use the Kafka script to create a new topic.

    bin/kafka-topics.sh --create --topic test --bootstrap-server localhost:9092 --partitions 1 --replication-factor 1

  2. List Topics: Verify the topic was created.

    bin/kafka-topics.sh --list --bootstrap-server localhost:9092

  3. Send Messages: Use the Kafka console producer to send messages to the topic.

    bin/kafka-console-producer.sh --topic test --bootstrap-server localhost:9092
>Hello Kafka
>This is a test message

  4. Consume Messages: Use the Kafka console consumer to read messages from the topic.

    bin/kafka-console-consumer.sh --topic test --from-beginning --bootstrap-server localhost:9092

Summary

Setting up Kafka involves downloading the Kafka binaries, configuring Zookeeper, and starting both Zookeeper and Kafka servers. By following the steps outlined above, you should have a working Kafka environment. This setup allows you to create topics, produce messages, and consume messages, forming the foundation for more advanced Kafka operations.
