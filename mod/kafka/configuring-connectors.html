Apache Kafka Connect is a tool for scalably and reliably streaming data between Apache Kafka and other systems. Configuring connectors is a crucial part of setting up Kafka Connect. This topic will guide you through the process from beginner to advanced levels.

Introduction to Kafka Connectors

Kafka Connectors are plugins that help you move large collections of data into and out of Kafka. They come in two types:

  • Source Connectors: Import data from external systems into Kafka topics.
  • Sink Connectors: Export data from Kafka topics to external systems.

Basic Configuration

Setting Up Kafka Connect

Before configuring connectors, you need to set up Kafka Connect. Here’s a basic configuration for a standalone mode:

# connect-standalone.properties
bootstrap.servers=localhost:9092
key.converter=org.apache.kafka.connect.json.JsonConverter
value.converter=org.apache.kafka.connect.json.JsonConverter
offset.storage.file.filename=/tmp/connect.offsets

Configuring a Source Connector

A source connector configuration typically includes the following properties:

  • name: Unique name for the connector.
  • connector.class: The Java class for the connector.
  • tasks.max: Maximum number of tasks to use for this connector.
  • topics: List of Kafka topics to write data to.

Example configuration for a FileStreamSourceConnector:

# file-source.properties
name=file-source-connector
connector.class=org.apache.kafka.connect.file.FileStreamSourceConnector
tasks.max=1
file=/path/to/input/file.txt
topic=example-topic

Configuring a Sink Connector

A sink connector configuration includes similar properties but focuses on reading from Kafka topics and writing to external systems.

Example configuration for a FileStreamSinkConnector:

# file-sink.properties
name=file-sink-connector
connector.class=org.apache.kafka.connect.file.FileStreamSinkConnector
tasks.max=1
file=/path/to/output/file.txt
topics=example-topic

Advanced Configuration

Distributed Mode

For production environments, you should use Kafka Connect in distributed mode. This mode provides scalability and fault tolerance.

Example configuration for distributed mode:

# connect-distributed.properties
bootstrap.servers=localhost:9092
group.id=connect-cluster
key.converter=org.apache.kafka.connect.json.JsonConverter
value.converter=org.apache.kafka.connect.json.JsonConverter
config.storage.topic=connect-configs
offset.storage.topic=connect-offsets
status.storage.topic=connect-status

Custom Converters

You can use custom converters to handle different data formats. For example, using AvroConverter:

key.converter=io.confluent.connect.avro.AvroConverter
key.converter.schema.registry.url=http://localhost:8081
value.converter=io.confluent.connect.avro.AvroConverter
value.converter.schema.registry.url=http://localhost:8081

Transformations

Single Message Transforms (SMTs) allow you to modify messages as they flow through Kafka Connect. Here’s an example of adding a timestamp to each message:

transforms=InsertTimestamp
transforms.InsertTimestamp.type=org.apache.kafka.connect.transforms.InsertField$Value
transforms.InsertTimestamp.timestamp.field=timestamp

Monitoring and Management

REST API

Kafka Connect provides a REST API for managing connectors. You can use it to create, configure, and monitor connectors.

Example to create a connector using curl:

curl -X POST -H "Content-Type: application/json" --data '{
  "name": "file-source-connector",
  "config": {
    "connector.class": "org.apache.kafka.connect.file.FileStreamSourceConnector",
    "tasks.max": "1",
    "file": "/path/to/input/file.txt",
    "topic": "example-topic"
  }
}' http://localhost:8083/connectors

Metrics

Kafka Connect exposes metrics via JMX. You can monitor these metrics to ensure your connectors are running smoothly.

Conclusion

Configuring connectors in Kafka Connect involves setting up the environment, defining connector properties, and optionally using advanced features like custom converters and transformations. Whether you're running in standalone or distributed mode, understanding these configurations will help you effectively stream data between Kafka and other systems. Monitoring and managing your connectors via the REST API and metrics is also crucial for maintaining a healthy data pipeline.
