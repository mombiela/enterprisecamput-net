Introduction to Exception Handling

Exception handling is a crucial aspect of robust software development. It allows a program to deal with unexpected situations (exceptions) gracefully, rather than crashing or producing incorrect results.

Key Concepts

  • Exception: An event that disrupts the normal flow of the program.
  • Try Block: A block of code that is tested for exceptions.
  • Catch Block: A block of code that handles the exception.
  • Throw Statement: Used to signal the occurrence of an exception.

Basic Exception Handling

Try and Catch Blocks

In C++, exceptions are handled using try and catch blocks. The try block contains the code that might throw an exception, and the catch block contains the code to handle the exception.

#include <iostream>
using namespace std;

int main() {
    try {
        int num1 = 10;
        int num2 = 0;
        if (num2 == 0) {
            throw "Division by zero error!";
        }
        cout << num1 / num2 << endl;
    } catch (const char* e) {
        cout << "Exception: " << e << endl;
    }
    return 0;
}

In this example:

  • The try block contains code that might throw an exception.
  • The throw statement throws an exception if num2 is zero.
  • The catch block catches the exception and handles it by printing an error message.

Throwing Exceptions

You can throw exceptions of any type, including built-in types and user-defined types.

Throwing Built-in Types

#include <iostream>
using namespace std;

int main() {
    try {
        throw 404;
    } catch (int e) {
        cout << "Exception: Error code " << e << endl;
    }
    return 0;
}

Throwing User-defined Types

#include <iostream>
using namespace std;

class MyException {
public:
    const char* what() const {
        return "Custom exception occurred!";
    }
};

int main() {
    try {
        throw MyException();
    } catch (MyException& e) {
        cout << "Exception: " << e.what() << endl;
    }
    return 0;
}

Multiple Catch Blocks

You can have multiple catch blocks to handle different types of exceptions.

#include <iostream>
using namespace std;

int main() {
    try {
        throw 3.14;
    } catch (int e) {
        cout << "Caught an integer: " << e << endl;
    } catch (double e) {
        cout << "Caught a double: " << e << endl;
    } catch (...) {
        cout << "Caught an unknown exception." << endl;
    }
    return 0;
}

In this example:

  • The catch blocks handle different types of exceptions.
  • The catch(...) block is a catch-all handler for any type of exception not caught by previous handlers.

Standard Exception Classes

C++ provides a set of standard exception classes defined in the <stdexcept> header.

Common Standard Exceptions

| Exception Class | Description | |-----------------|-------------| | std::exception | Base class for all standard exceptions | | std::runtime_error | Represents runtime errors | | std::logic_error | Represents logical errors | | std::out_of_range | Thrown when an out-of-range error occurs | | std::invalid_argument | Thrown when an invalid argument is passed |

Using Standard Exceptions

#include <iostream>
#include <stdexcept>
using namespace std;

int main() {
    try {
        throw runtime_error("Runtime error occurred");
    } catch (runtime_error& e) {
        cout << "Exception: " << e.what() << endl;
    }
    return 0;
}

Advanced Exception Handling

Custom Exception Classes

You can create your own exception classes by inheriting from std::exception.

#include <iostream>
#include <exception>
using namespace std;

class MyException : public exception {
public:
    const char* what() const noexcept override {
        return "My custom exception";
    }
};

int main() {
    try {
        throw MyException();
    } catch (MyException& e) {
        cout << "Exception: " << e.what() << endl;
    }
    return 0;
}

Exception Specifications (Deprecated)

Exception specifications were used to specify which exceptions a function might throw, but they are deprecated in C++11 and removed in C++17.

noexcept Specifier

The noexcept specifier indicates that a function does not throw any exceptions.

#include <iostream>
using namespace std;

void safeFunction() noexcept {
    cout << "This function is guaranteed not to throw exceptions." << endl;
}

int main() {
    safeFunction();
    return 0;
}

Conclusion

Exception handling in C++ is a powerful feature that helps create robust and error-resistant programs. By using try, catch, and throw, you can manage errors gracefully. Understanding standard exception classes and creating custom exceptions further enhances your ability to handle errors effectively. Remember to use the noexcept specifier for functions that are guaranteed not to throw exceptions, improving code reliability and performance.
