Introduction

In this section, we will explore how to implement new features and collaborate effectively using Git. This includes creating branches, merging changes, resolving conflicts, and using pull requests. By the end of this section, you will be able to manage your codebase collaboratively and efficiently.

Branching in Git

What is a Branch?

A branch in Git is a lightweight movable pointer to a commit. Branching allows you to diverge from the main line of development and continue to work without affecting the main line.

Creating a Branch

To create a new branch, use the git branch command followed by the name of the branch.

git branch feature-branch

Switching to a Branch

To switch to the newly created branch, use the git checkout command.

git checkout feature-branch

Alternatively, you can create and switch to a new branch in one command:

git checkout -b feature-branch

Listing Branches

To list all branches in your repository, use:

git branch

Merging Branches

What is Merging?

Merging is the process of combining the changes from one branch into another. This is commonly used to integrate feature branches back into the main branch.

Merging a Branch

To merge a branch into your current branch, use the git merge command followed by the branch name.

git checkout main
git merge feature-branch

Handling Merge Conflicts

Sometimes, changes in different branches conflict with each other. Git will prompt you to resolve these conflicts manually.

Example of a Merge Conflict

Suppose you have the following file in both branches:

main branch (file.txt):

Hello, World!

feature-branch (file.txt):

Hello, Git!

When you try to merge feature-branch into main, Git will produce a conflict:

file.txt:

<<<<<<< HEAD
Hello, World!
=======
Hello, Git!
>>>>>>> feature-branch

To resolve the conflict, edit the file to your desired state and then add and commit the changes:

git add file.txt
git commit -m "Resolved merge conflict in file.txt"

Collaboration with Pull Requests

What is a Pull Request?

A pull request (PR) is a way to propose changes to a repository. It allows team members to review and discuss the changes before they are merged.

Creating a Pull Request

  1. Push your branch to the remote repository:

    git push origin feature-branch

  2. Go to the repository on GitHub (or another Git hosting service) and navigate to the "Pull requests" tab.

  3. Click "New pull request" and select your branch.

  4. Add a title and description for your pull request, then click "Create pull request."

Reviewing and Merging a Pull Request

Team members can review the pull request, leave comments, and suggest changes. Once the pull request is approved, it can be merged into the main branch.

Summary

In this section, we covered:

  • Branching: Creating, switching, and listing branches.
  • Merging: Combining changes from different branches and resolving conflicts.
  • Pull Requests: Proposing, reviewing, and merging changes collaboratively.

By mastering these concepts, you can effectively manage and collaborate on code in a Git repository, ensuring a smooth and efficient development workflow.
