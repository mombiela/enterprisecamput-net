Cherry-Picking in Git
Stashing Changes ►

Cherry-picking in Git allows you to apply the changes introduced by some existing commits onto your current branch. This can be particularly useful when you want to incorporate specific changes from one branch into another without merging the entire branch.

What is Cherry-Picking?

  • Definition: Cherry-picking is the process of selecting specific commits from one branch and applying them to another.
  • Use Cases:
    • Applying bug fixes from one branch to another.
    • Incorporating specific features or changes without merging the entire branch.
    • Backporting changes to older versions of the codebase.

How to Cherry-Pick

Basic Cherry-Picking

To cherry-pick a commit, you need the commit hash (SHA) of the commit you want to apply.

  1. Find the Commit Hash:

    git log

    This command will display the commit history. Copy the hash of the commit you want to cherry-pick.

  2. Cherry-Pick the Commit:

    git cherry-pick <commit-hash>

    Replace <commit-hash> with the actual hash of the commit.

Example

Let's say you have the following commit history:

commit 1a2b3c4d5e6f7g8h9i0j
Author: John Doe <john@example.com>
Date:   Mon Oct 4 12:34:56 2021 -0700

    Fix bug in user authentication

commit 0a9b8c7d6e5f4g3h2i1j
Author: Jane Smith <jane@example.com>
Date:   Sun Oct 3 11:23:45 2021 -0700

    Add new feature to dashboard

If you want to apply the bug fix from commit 1a2b3c4d5e6f7g8h9i0j to your current branch, you would run:

git cherry-pick 1a2b3c4d5e6f7g8h9i0j

Resolving Conflicts

Sometimes, cherry-picking can result in conflicts if the changes in the commit you're cherry-picking conflict with changes in your current branch.

  1. Identify Conflicts: Git will notify you of conflicts and mark the conflicting files.

  2. Resolve Conflicts: Open the conflicting files and manually resolve the conflicts. Look for conflict markers (<<<<<<<, =======, >>>>>>>).

  3. Mark Conflicts as Resolved: After resolving the conflicts, mark them as resolved:

    git add <resolved-file>

  4. Continue Cherry-Picking:

    git cherry-pick --continue

Cherry-Picking Multiple Commits

You can cherry-pick a range of commits by specifying the commit hashes:

git cherry-pick <start-commit-hash>^..<end-commit-hash>

For example, to cherry-pick commits from 0a9b8c7d6e5f4g3h2i1j to 1a2b3c4d5e6f7g8h9i0j, you would run:

git cherry-pick 0a9b8c7d6e5f4g3h2i1j^..1a2b3c4d5e6f7g8h9i0j

Best Practices

  • Avoid Cherry-Picking Large Changes: Cherry-picking is best suited for small, isolated changes. For larger changes, consider merging or rebasing.
  • Document Cherry-Picks: Keep track of cherry-picked commits to avoid confusion and potential conflicts in the future.
  • Test Thoroughly: Always test the changes after cherry-picking to ensure they integrate well with the current branch.

Conclusion

Cherry-picking is a powerful tool in Git that allows you to selectively apply changes from one branch to another. By understanding how to cherry-pick commits, resolve conflicts, and follow best practices, you can effectively manage and incorporate specific changes in your codebase. This technique is particularly useful for applying bug fixes, backporting changes, and integrating specific features without merging entire branches.
