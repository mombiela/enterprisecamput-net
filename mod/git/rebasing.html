Rebasing is a powerful feature in Git that allows you to rewrite commit history. It can be used to streamline a series of commits, update branches, and maintain a cleaner project history. This topic will cover the basics of rebasing, how to perform a rebase, and advanced rebasing techniques.

What is Rebasing?

Rebasing is the process of moving or combining a sequence of commits to a new base commit. It is an alternative to merging and can be used to keep a linear project history.

Key Concepts

  • Base Commit: The commit at which a branch diverges from another branch.
  • Upstream Branch: The branch onto which you are rebasing your changes.
  • Rebase vs. Merge: While merging creates a new commit that combines the changes from two branches, rebasing moves the entire branch to start from a new base commit.

Basic Rebasing

How to Rebase

To rebase a branch, you use the git rebase command. Here’s a simple example:

# Switch to the branch you want to rebase
git checkout feature-branch

# Rebase onto the main branch
git rebase main

Example Explained

  1. Switch to the branch: You first switch to the branch that you want to rebase.
  2. Rebase onto the main branch: This command will move the commits from feature-branch to start from the latest commit on the main branch.

Handling Conflicts

During a rebase, you might encounter conflicts. Git will pause the rebase process and allow you to resolve these conflicts.

# Resolve conflicts in your editor
# After resolving conflicts, stage the changes
git add .

# Continue the rebase process
git rebase --continue

If you want to abort the rebase process, you can use:

git rebase --abort

Interactive Rebasing

Interactive rebasing allows you to edit, reorder, squash, or drop commits. This is useful for cleaning up your commit history before merging it into the main branch.

How to Perform an Interactive Rebase

# Start an interactive rebase
git rebase -i HEAD~n

Replace n with the number of commits you want to rebase. This will open an editor with a list of commits.

Example Explained

  1. Pick: Use this to keep a commit as is.
  2. Reword: Use this to change the commit message.
  3. Edit: Use this to stop and edit the commit.
  4. Squash: Use this to combine this commit with the previous one.
  5. Fixup: Like squash, but discards the commit message.
  6. Drop: Use this to remove the commit.

Example

pick 1234567 Commit message 1
squash 89abcde Commit message 2
reword fghijkl Commit message 3

In this example:

  • The second commit will be squashed into the first.
  • The third commit's message will be edited.

Advanced Rebasing Techniques

Rebasing Onto Another Branch

You can rebase your branch onto any other branch, not just the main branch.

git rebase another-branch

Using `--onto`

The --onto option allows you to rebase a range of commits onto another branch.

# Rebase commits from feature-branch onto another-branch
git rebase --onto another-branch main feature-branch

Example Explained

  1. main: The branch from which the commits are taken.
  2. feature-branch: The branch that contains the commits to be rebased.
  3. another-branch: The branch onto which the commits will be rebased.

Summary

Rebasing is a powerful tool in Git that helps maintain a clean and linear project history. It can be used for:

  • Moving a branch to a new base commit.
  • Cleaning up commit history with interactive rebasing.
  • Resolving conflicts in a controlled manner.

While rebasing can be complex, understanding its basics and practicing with simple examples will help you master this essential Git feature. Remember to always communicate with your team when performing rebases on shared branches to avoid disrupting the workflow.
