Introduction to Stashing

Stashing in Git allows you to temporarily save changes in your working directory without committing them. This is particularly useful when you need to switch branches or work on something else but want to come back to your current changes later.

Key Concepts

  • Stash: A stash is a saved state of your working directory and the index.
  • Stash List: A list of all stashes you have saved.
  • Applying a Stash: Bringing back the changes from a stash to your working directory.
  • Dropping a Stash: Deleting a stash from the stash list.

Basic Stashing Commands

Saving Changes

To save your current changes, use the git stash command:

git stash

This command will save your changes and revert your working directory to the state of the last commit.

Listing Stashes

To see all the stashes you have saved, use:

git stash list

This will display a list of stashes with their identifiers.

Applying a Stash

To apply the most recent stash, use:

git stash apply

To apply a specific stash, use:

git stash apply stash@{index}

Replace index with the number of the stash you want to apply.

Dropping a Stash

To delete a specific stash, use:

git stash drop stash@{index}

To delete all stashes, use:

git stash clear

Advanced Stashing Techniques

Stashing Untracked Files

By default, git stash does not include untracked files. To stash untracked files, use:

git stash -u

or

git stash --include-untracked

Creating a Named Stash

You can name your stashes for better identification:

git stash save "meaningful message"

Stashing Specific Files

To stash specific files, use the git stash push command:

git stash push -m "message" path/to/file

Pop vs. Apply

  • Apply: Applies the stash but keeps it in the stash list.
  • Pop: Applies the stash and removes it from the stash list.
git stash pop

Examples

Example 1: Basic Stashing and Applying

  1. Make some changes to your files.
  2. Save the changes: 
    git stash
  3. List the stashes: 
    git stash list
  4. Apply the stash: 
    git stash apply

Example 2: Stashing with Untracked Files

  1. Make changes and add new untracked files.
  2. Save all changes, including untracked files: 
    git stash -u
  3. List and apply the stash: 
    git stash list
git stash apply

Example 3: Named Stash and Pop

  1. Make changes and save with a name: 
    git stash save "work in progress"
  2. List stashes: 
    git stash list
  3. Pop the stash: 
    git stash pop

Summary

Stashing is a powerful feature in Git that allows you to save your work temporarily and come back to it later. It is especially useful for managing work in progress when you need to switch contexts. Understanding how to stash, list, apply, and drop stashes will enhance your workflow and make you more efficient in managing your code changes.
