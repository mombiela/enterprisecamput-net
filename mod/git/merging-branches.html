Merging Branches
Resolving Merge Conflicts ►

Merging branches is a fundamental concept in Git that allows you to combine changes from different branches into a single branch. This is essential for collaborative development and managing different features or versions of a project.

Key Concepts

  • Branch: A parallel version of a repository. Branches allow you to work on different features or fixes independently.
  • Merge: The process of integrating changes from one branch into another.
  • Conflict: Occurs when changes in different branches interfere with each other and Git cannot automatically resolve them.

Types of Merges

  1. Fast-Forward Merge:

    • Occurs when the branch being merged has not diverged from the target branch.
    • Git simply moves the pointer forward.

  2. Three-Way Merge:

    • Used when branches have diverged.
    • Git uses the common ancestor and the two branch tips to create a new merge commit.

Fast-Forward Merge

Example

Suppose you have a branch feature that you want to merge into main.

  1. Checkout the main branch:

    git checkout main

  2. Merge the feature branch:

    git merge feature

If main has not diverged from feature, Git will perform a fast-forward merge.

Explanation

In a fast-forward merge, Git simply moves the main branch pointer forward to the latest commit in the feature branch. This is the simplest type of merge and does not create a new commit.

Three-Way Merge

Example

Suppose you have a branch feature that has diverged from main.

  1. Checkout the main branch:

    git checkout main

  2. Merge the feature branch:

    git merge feature

If main has diverged from feature, Git will perform a three-way merge.

Explanation

In a three-way merge, Git uses the common ancestor of the two branches and the tips of both branches to create a new merge commit. This new commit represents the combined changes from both branches.

Handling Merge Conflicts

Example

Suppose you have a conflict when merging feature into main.

  1. Git will notify you of the conflict:

    Auto-merging file.txt
CONFLICT (content): Merge conflict in file.txt
Automatic merge failed; fix conflicts and then commit the result.

  2. Open the conflicted file and resolve the conflict:

    <<<<<<< HEAD
This is some content from the main branch.
=======
This is some content from the feature branch.
>>>>>>> feature

  3. After resolving the conflict, add the resolved file:

    git add file.txt

  4. Complete the merge by committing:

    git commit

Explanation

When a conflict occurs, Git marks the conflicting areas in the file. You need to manually edit the file to resolve the conflict, then add and commit the resolved file to complete the merge.

Summary

Merging branches in Git is a crucial skill for managing changes in a collaborative environment. Understanding the difference between fast-forward and three-way merges, as well as how to handle merge conflicts, will help you maintain a clean and functional codebase. Practice merging branches and resolving conflicts to become proficient in these essential Git operations.
