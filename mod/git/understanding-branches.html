Introduction to Branches

Branches in Git are a powerful feature that allows you to diverge from the main line of development and continue to work without affecting that main line. This is particularly useful for working on new features, bug fixes, or experiments in isolation.

Key Concepts

  • Branch: A branch in Git is simply a lightweight movable pointer to one of these commits.
  • Master/Main: The default branch in most repositories. It is often the main line of development.
  • HEAD: A pointer that represents your current working directory. It usually points to the latest commit in your current branch.

Creating and Managing Branches

Creating a Branch

To create a new branch, you use the git branch command followed by the name of the branch you want to create.

git branch new-feature

This command creates a new branch called new-feature but does not switch to it. To switch to the new branch, you use the git checkout command:

git checkout new-feature

Alternatively, you can create and switch to a new branch in one command:

git checkout -b new-feature

Listing Branches

To list all the branches in your repository, use the git branch command without any arguments:

git branch

This will show all the branches and highlight the current branch with an asterisk (*).

Deleting a Branch

Once you have finished working on a branch, you might want to delete it. To delete a branch, use the -d option with the git branch command:

git branch -d new-feature

If the branch has not been merged, Git will prevent you from deleting it. To force delete a branch, use the -D option:

git branch -D new-feature

Merging Branches

Fast-Forward Merge

A fast-forward merge occurs when there is a direct path from the current branch to the target branch. Git simply moves the pointer forward.

git checkout main
git merge new-feature

Three-Way Merge

When the branches have diverged, Git performs a three-way merge. This involves creating a new commit that combines the changes from both branches.

git checkout main
git merge new-feature

If there are conflicts, Git will prompt you to resolve them manually.

Branching Strategies

Feature Branch Workflow

  • Create a new branch for each feature or bug fix.
  • Merge the feature branch back into the main branch once the work is complete.

Gitflow Workflow

  • Uses multiple branches for different stages of development (e.g., feature, develop, release, hotfix).
  • Provides a robust framework for managing larger projects.

Forking Workflow

  • Each developer forks the main repository and works on their own copy.
  • Changes are integrated back into the main repository via pull requests.

Practical Exercises

Exercise 1: Creating and Switching Branches

  1. Create a new branch called experiment.
  2. Switch to the experiment branch.
  3. Make some changes and commit them.
  4. Switch back to the main branch.

Exercise 2: Merging Branches

  1. Create a new branch called feature-1.
  2. Make some changes and commit them.
  3. Switch back to the main branch.
  4. Merge feature-1 into main.

Exercise 3: Resolving Merge Conflicts

  1. Create a new branch called conflict-branch.
  2. Make conflicting changes in main and conflict-branch.
  3. Attempt to merge conflict-branch into main and resolve the conflicts.

Summary

Understanding branches in Git is crucial for effective version control. Branches allow you to work on different features, bug fixes, or experiments in isolation. You can create, switch, list, and delete branches easily. Merging branches can be straightforward or complex, depending on the changes made. By mastering branching strategies and workflows, you can manage your projects more efficiently and collaborate with others seamlessly.
