Introduction to Classes and Objects

In Groovy, as in many object-oriented programming languages, classes and objects are fundamental concepts. A class is a blueprint for creating objects, providing initial values for state (member variables or properties) and implementations of behavior (member functions or methods). An object is an instance of a class.

Defining a Class

To define a class in Groovy, you use the class keyword followed by the class name. Here is a simple example:

class Person {
    String name
    int age

    void displayInfo() {
        println "Name: $name, Age: $age"
    }
}
  • class Person: Defines a new class named Person.
  • String name: Declares a property name of type String.
  • int age: Declares a property age of type int.
  • void displayInfo(): Declares a method that prints the person's information.

Creating Objects

To create an object (or instance) of a class, you use the new keyword followed by the class name and parentheses. Here is how you can create an object of the Person class:

def person = new Person()
person.name = "John Doe"
person.age = 30
person.displayInfo()
  • new Person(): Creates a new instance of the Person class.
  • person.name = "John Doe": Sets the name property of the person object.
  • person.age = 30: Sets the age property of the person object.
  • person.displayInfo(): Calls the displayInfo method on the person object.

Constructors

Constructors are special methods used to initialize objects. In Groovy, you can define constructors to set initial values for properties. Here is an example:

class Person {
    String name
    int age

    Person(String name, int age) {
        this.name = name
        this.age = age
    }

    void displayInfo() {
        println "Name: $name, Age: $age"
    }
}

def person = new Person("Jane Doe", 25)
person.displayInfo()
  • Person(String name, int age): Constructor that initializes the name and age properties.
  • this.name = name: Assigns the constructor parameter name to the property name.
  • this.age = age: Assigns the constructor parameter age to the property age.

Methods

Methods define the behavior of objects. They are functions defined within a class. Here is an example of a class with multiple methods:

class Calculator {
    int add(int a, int b) {
        return a + b
    }

    int subtract(int a, int b) {
        return a - b
    }
}

def calc = new Calculator()
println calc.add(5, 3)       // Output: 8
println calc.subtract(5, 3)  // Output: 2
  • int add(int a, int b): Method that adds two integers.
  • int subtract(int a, int b): Method that subtracts one integer from another.

Inheritance

Inheritance allows a class to inherit properties and methods from another class. In Groovy, you use the extends keyword to achieve inheritance. Here is an example:

class Animal {
    String name

    void eat() {
        println "$name is eating"
    }
}

class Dog extends Animal {
    void bark() {
        println "$name is barking"
    }
}

def dog = new Dog()
dog.name = "Buddy"
dog.eat()   // Output: Buddy is eating
dog.bark()  // Output: Buddy is barking
  • class Dog extends Animal: Dog class inherits from Animal class.
  • dog.eat(): Calls the inherited eat method from Animal class.
  • dog.bark(): Calls the bark method defined in Dog class.

Polymorphism

Polymorphism allows objects to be treated as instances of their parent class rather than their actual class. This is useful for writing more generic and reusable code. Here is an example:

class Animal {
    void makeSound() {
        println "Some generic animal sound"
    }
}

class Dog extends Animal {
    void makeSound() {
        println "Bark"
    }
}

class Cat extends Animal {
    void makeSound() {
        println "Meow"
    }
}

def animals = [new Dog(), new Cat()]
animals.each { it.makeSound() }
// Output:
// Bark
// Meow
  • void makeSound(): Method overridden in Dog and Cat classes.
  • animals.each { it.makeSound() }: Calls the makeSound method on each animal, demonstrating polymorphism.

Summary

In this topic, we covered the basics of classes and objects in Groovy, including:

  • Defining classes and creating objects.
  • Using constructors to initialize objects.
  • Defining and using methods.
  • Implementing inheritance to create a class hierarchy.
  • Utilizing polymorphism to write flexible and reusable code.

Understanding these concepts is crucial for mastering object-oriented programming in Groovy. Practice creating your own classes and objects to reinforce these concepts.
