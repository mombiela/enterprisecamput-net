File Handling in COBOL
Sequential File Processing ►

File handling is a crucial aspect of COBOL programming, as it is often used for business applications that require extensive data processing. This topic will guide you through the basics of file handling in COBOL, from understanding file types to advanced file operations.

Introduction to File Handling

In COBOL, files are used to store data that can be read and written by the program. Understanding how to handle files is essential for any COBOL programmer.

Key Concepts

  • File Types: Sequential, Indexed, and Relative.
  • File Organization: How data is structured within the file.
  • File Access Modes: Input, Output, I-O, and Extend.
  • File Control: Using the SELECT and ASSIGN statements.
  • File Description: Using the FD (File Description) entry in the Data Division.

File Types

COBOL supports three main types of files:

  1. Sequential Files: Records are stored one after another.
  2. Indexed Files: Records are stored with an index, allowing for faster access.
  3. Relative Files: Records are stored at relative positions, allowing for direct access.

Sequential Files

Sequential files are the simplest type of file in COBOL. Records are read and written in a sequential manner.

Example: Reading a Sequential File

IDENTIFICATION DIVISION.
PROGRAM-ID. ReadSeqFile.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT InputFile ASSIGN TO 'input.dat'
    ORGANIZATION IS SEQUENTIAL.

DATA DIVISION.
FILE SECTION.
FD InputFile.
01 InputRecord PIC X(80).

WORKING-STORAGE SECTION.
01 WS-EOF PIC X VALUE 'N'.

PROCEDURE DIVISION.
    OPEN INPUT InputFile
    PERFORM UNTIL WS-EOF = 'Y'
        READ InputFile INTO InputRecord
            AT END
                MOVE 'Y' TO WS-EOF
            NOT AT END
                DISPLAY InputRecord
        END-READ
    END-PERFORM
    CLOSE InputFile
    STOP RUN.

Indexed Files

Indexed files allow for faster access to records using an index.

Example: Reading an Indexed File

IDENTIFICATION DIVISION.
PROGRAM-ID. ReadIdxFile.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT IndexedFile ASSIGN TO 'indexed.dat'
    ORGANIZATION IS INDEXED
    ACCESS MODE IS RANDOM
    RECORD KEY IS KeyField.

DATA DIVISION.
FILE SECTION.
FD IndexedFile.
01 IndexedRecord.
   05 KeyField PIC X(10).
   05 DataField PIC X(70).

WORKING-STORAGE SECTION.
01 WS-Key PIC X(10).

PROCEDURE DIVISION.
    OPEN I-O IndexedFile
    MOVE 'somekey' TO WS-Key
    READ IndexedFile
        INVALID KEY
            DISPLAY 'Record not found'
        NOT INVALID KEY
            DISPLAY IndexedRecord
    END-READ
    CLOSE IndexedFile
    STOP RUN.

Relative Files

Relative files store records at specific relative positions.

Example: Reading a Relative File

IDENTIFICATION DIVISION.
PROGRAM-ID. ReadRelFile.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT RelativeFile ASSIGN TO 'relative.dat'
    ORGANIZATION IS RELATIVE
    ACCESS MODE IS RANDOM
    RELATIVE KEY IS RelKey.

DATA DIVISION.
FILE SECTION.
FD RelativeFile.
01 RelativeRecord.
   05 RelKey PIC 9(4).
   05 DataField PIC X(76).

WORKING-STORAGE SECTION.
01 WS-RelKey PIC 9(4).

PROCEDURE DIVISION.
    OPEN I-O RelativeFile
    MOVE 1234 TO WS-RelKey
    READ RelativeFile
        INVALID KEY
            DISPLAY 'Record not found'
        NOT INVALID KEY
            DISPLAY RelativeRecord
    END-READ
    CLOSE RelativeFile
    STOP RUN.

File Access Modes

COBOL provides different modes to access files:

  • INPUT: Read-only access.
  • OUTPUT: Write-only access, creates a new file.
  • I-O: Read and write access.
  • EXTEND: Append records to the end of the file.

Example: Writing to a Sequential File

IDENTIFICATION DIVISION.
PROGRAM-ID. WriteSeqFile.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT OutputFile ASSIGN TO 'output.dat'
    ORGANIZATION IS SEQUENTIAL.

DATA DIVISION.
FILE SECTION.
FD OutputFile.
01 OutputRecord PIC X(80).

WORKING-STORAGE SECTION.
01 WS-Data PIC X(80) VALUE 'This is a test record'.

PROCEDURE DIVISION.
    OPEN OUTPUT OutputFile
    MOVE WS-Data TO OutputRecord
    WRITE OutputRecord
    CLOSE OutputFile
    STOP RUN.

File Control and Description

File Control

The SELECT and ASSIGN statements in the File Control section of the Environment Division are used to associate a file with a physical storage medium.

File Description

The FD (File Description) entry in the Data Division describes the structure of the file's records.

Example: File Control and Description

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT MyFile ASSIGN TO 'myfile.dat'
    ORGANIZATION IS SEQUENTIAL.

DATA DIVISION.
FILE SECTION.
FD MyFile.
01 MyRecord.
   05 Field1 PIC X(10).
   05 Field2 PIC 9(5).

Conclusion

File handling in COBOL is a fundamental skill for any COBOL programmer. By understanding the different file types, access modes, and how to control and describe files, you can efficiently manage data within your COBOL programs. Practice with the provided examples and experiment with different file operations to deepen your understanding.
