Introduction

COBOL (Common Business-Oriented Language) is one of the oldest programming languages, primarily used in business, finance, and administrative systems for companies and governments. In this section, we will cover how to write a simple "Hello World" program in COBOL, which is a great starting point for beginners.

Key Concepts

  • COBOL Program Structure: Understanding the basic structure of a COBOL program.
  • Divisions: COBOL programs are divided into four main divisions: Identification, Environment, Data, and Procedure.
  • Syntax: Learning the syntax and conventions used in COBOL.

COBOL Program Structure

A COBOL program is divided into four main divisions:

  1. Identification Division: Provides the program name and other identification details.
  2. Environment Division: Specifies the environment in which the program will run.
  3. Data Division: Defines the variables and data structures used in the program.
  4. Procedure Division: Contains the actual code and logic of the program.

Writing "Hello World" in COBOL

Identification Division

The Identification Division is where you name your program. This is mandatory in every COBOL program.

IDENTIFICATION DIVISION.
PROGRAM-ID. HelloWorld.

Environment Division

The Environment Division specifies the configuration and environment settings. For a simple "Hello World" program, this can be minimal or even omitted.

ENVIRONMENT DIVISION.

Data Division

The Data Division is where you declare variables. For "Hello World," we don't need any variables, so this can be minimal as well.

DATA DIVISION.

Procedure Division

The Procedure Division contains the executable instructions. This is where we will write the code to display "Hello World."

PROCEDURE DIVISION.
    DISPLAY 'Hello, World!'.
    STOP RUN.

Explanation of the Code

  • IDENTIFICATION DIVISION: This section names the program as HelloWorld.
  • ENVIRONMENT DIVISION: This section is included but empty, as no specific environment settings are needed.
  • DATA DIVISION: This section is also included but empty, as no variables are required.
  • PROCEDURE DIVISION:

    • STOP RUN: This command ends the program.

Exercises

  1. Modify the Program: Change the displayed message to "Hello, COBOL!" and run the program.
  2. Add a Variable: Declare a variable in the Data Division to store the message and then display it in the Procedure Division.
  3. Environment Settings: Research and add a simple environment setting in the Environment Division.

Conclusion

In this section, we covered the basics of writing a "Hello World" program in COBOL. We learned about the main divisions of a COBOL program and how to use the DISPLAY and STOP RUN commands. This foundational knowledge will help you as you progress to more complex COBOL programming tasks.
