Introduction to Tables and Arrays

Tables and arrays in COBOL are used to store collections of data elements that are of the same type. They are essential for handling repetitive data structures efficiently.

  • Tables: Often used interchangeably with arrays in COBOL, tables are a way to store multiple occurrences of data items.
  • Arrays: A structured way to store elements in a sequential manner.

Defining Tables and Arrays

Basic Definition

In COBOL, tables (arrays) are defined using the OCCURS clause in the DATA DIVISION.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 STUDENT-GRADES.
   05 STUDENT-GRADE PIC 99 OCCURS 10 TIMES.
  • STUDENT-GRADES: The name of the table.
  • STUDENT-GRADE: The data item that repeats.
  • OCCURS 10 TIMES: Specifies that there are 10 occurrences of STUDENT-GRADE.

Multi-Dimensional Arrays

COBOL also supports multi-dimensional arrays by using nested OCCURS clauses.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 CLASS-GRADES.
   05 STUDENT-GRADES OCCURS 10 TIMES.
      10 STUDENT-GRADE PIC 99 OCCURS 5 TIMES.
  • CLASS-GRADES: The name of the multi-dimensional array.
  • STUDENT-GRADES: The first dimension with 10 occurrences.
  • STUDENT-GRADE: The second dimension with 5 occurrences.

Accessing Table Elements

Using Subscripts

Subscripts are used to access specific elements in a table.

PROCEDURE DIVISION.
MOVE 85 TO STUDENT-GRADE (3).
DISPLAY STUDENT-GRADE (3).
  • STUDENT-GRADE (3): Accesses the third element of the STUDENT-GRADE array.
  • MOVE 85 TO STUDENT-GRADE (3): Assigns the value 85 to the third element.
  • DISPLAY STUDENT-GRADE (3): Displays the value of the third element.

Using Indexes

Indexes provide a more efficient way to access table elements, especially in large tables.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 STUDENT-GRADES.
   05 STUDENT-GRADE PIC 99 OCCURS 10 TIMES INDEXED BY IDX.

PROCEDURE DIVISION.
SET IDX TO 1.
MOVE 90 TO STUDENT-GRADE (IDX).
DISPLAY STUDENT-GRADE (IDX).
  • INDEXED BY IDX: Declares an index named IDX for the table.
  • SET IDX TO 1: Initializes the index to 1.
  • MOVE 90 TO STUDENT-GRADE (IDX): Assigns the value 90 to the element at index 1.
  • DISPLAY STUDENT-GRADE (IDX): Displays the value of the element at index 1.

Performing Operations on Tables

Iterating Over Tables

Loops are used to iterate over table elements.

PROCEDURE DIVISION.
PERFORM VARYING IDX FROM 1 BY 1 UNTIL IDX > 10
   DISPLAY STUDENT-GRADE (IDX)
END-PERFORM.
  • PERFORM VARYING IDX FROM 1 BY 1 UNTIL IDX > 10: Loops from 1 to 10.
  • DISPLAY STUDENT-GRADE (IDX): Displays each element of the STUDENT-GRADE array.

Searching Tables

COBOL provides the SEARCH and SEARCH ALL statements for searching tables.

PROCEDURE DIVISION.
SEARCH STUDENT-GRADES
   AT END DISPLAY 'NOT FOUND'
   WHEN STUDENT-GRADE (IDX) = 85
      DISPLAY 'FOUND AT INDEX ' IDX
END-SEARCH.
  • SEARCH STUDENT-GRADES: Searches the STUDENT-GRADES table.
  • AT END: Executes if the search completes without finding the element.
  • WHEN STUDENT-GRADE (IDX) = 85: Condition to check each element.
  • DISPLAY 'FOUND AT INDEX ' IDX: Displays the index if the element is found.

Advanced Topics

Dynamic Arrays

COBOL allows for dynamic allocation of arrays using the ALLOCATE and FREE statements.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 DYNAMIC-TABLE.
   05 DYNAMIC-ELEMENT PIC 99 OCCURS 0 TO 100 TIMES DEPENDING ON TABLE-SIZE.

PROCEDURE DIVISION.
MOVE 50 TO TABLE-SIZE.
ALLOCATE DYNAMIC-TABLE.
  • OCCURS 0 TO 100 TIMES DEPENDING ON TABLE-SIZE: Defines a dynamic array.
  • MOVE 50 TO TABLE-SIZE: Sets the size of the array to 50.
  • ALLOCATE DYNAMIC-TABLE: Allocates memory for the array.

Sorting Tables

COBOL provides the SORT statement to sort table elements.

PROCEDURE DIVISION.
SORT STUDENT-GRADES ON ASCENDING KEY STUDENT-GRADE.
  • SORT STUDENT-GRADES ON ASCENDING KEY STUDENT-GRADE: Sorts the STUDENT-GRADES table in ascending order based on STUDENT-GRADE.

Conclusion

Tables and arrays are powerful tools in COBOL for managing collections of data. Understanding how to define, access, and manipulate these structures is crucial for efficient COBOL programming. With practice, you can leverage these constructs to handle complex data processing tasks effectively.
