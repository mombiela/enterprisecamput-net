Subroutines and Functions in COBOL

Introduction

Subroutines and functions are essential components in COBOL programming, allowing for modular, reusable, and organized code. This section will cover the basics of subroutines and functions, their syntax, usage, and best practices from beginner to advanced levels.

Key Concepts

  • Subroutines: Also known as paragraphs or sections, these are blocks of code that perform specific tasks and can be called from different parts of the program.
  • Functions: Predefined or user-defined operations that return a value and can be used within expressions.

Subroutines in COBOL

Basic Structure

Subroutines in COBOL are defined in the PROCEDURE DIVISION. They can be organized into paragraphs or sections.

Paragraphs

A paragraph is a named block of code that can be executed by using the PERFORM statement.

IDENTIFICATION DIVISION.
PROGRAM-ID. SubroutineExample.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 NUM1 PIC 9(2) VALUE 10.
01 NUM2 PIC 9(2) VALUE 20.
01 RESULT PIC 9(3).

PROCEDURE DIVISION.
    PERFORM ADD-NUMBERS
    DISPLAY "The result is: " RESULT
    STOP RUN.

ADD-NUMBERS.
    ADD NUM1 TO NUM2 GIVING RESULT.

Sections

Sections are larger blocks that can contain multiple paragraphs.

IDENTIFICATION DIVISION.
PROGRAM-ID. SectionExample.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 NUM1 PIC 9(2) VALUE 10.
01 NUM2 PIC 9(2) VALUE 20.
01 RESULT PIC 9(3).

PROCEDURE DIVISION.
    PERFORM CALCULATION-SECTION
    DISPLAY "The result is: " RESULT
    STOP RUN.

CALCULATION-SECTION.
ADD-NUMBERS.
    ADD NUM1 TO NUM2 GIVING RESULT.

Calling Subroutines

  • PERFORM: Executes a paragraph or section.
  • THRU: Executes a range of paragraphs.
PERFORM ADD-NUMBERS
PERFORM CALCULATION-SECTION THRU END-CALCULATION

Functions in COBOL

Intrinsic Functions

COBOL provides a set of built-in functions for common operations.

Example: NUMVAL Function

Converts a numeric string to a numeric value.

IDENTIFICATION DIVISION.
PROGRAM-ID. FunctionExample.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 NUM-STRING PIC X(5) VALUE '12345'.
01 NUM-VALUE PIC 9(5).

PROCEDURE DIVISION.
    COMPUTE NUM-VALUE = FUNCTION NUMVAL(NUM-STRING)
    DISPLAY "The numeric value is: " NUM-VALUE
    STOP RUN.

User-Defined Functions

User-defined functions can be created using the FUNCTION-ID and END FUNCTION statements.

IDENTIFICATION DIVISION.
FUNCTION-ID. MyFunction.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 INPUT-NUMBER PIC 9(2).
01 RESULT PIC 9(3).

PROCEDURE DIVISION.
    ENTRY "MyFunction" USING INPUT-NUMBER
    ADD INPUT-NUMBER TO INPUT-NUMBER GIVING RESULT
    EXIT FUNCTION RESULT.
END FUNCTION MyFunction.

Calling User-Defined Functions

User-defined functions can be called using the FUNCTION keyword.

IDENTIFICATION DIVISION.
PROGRAM-ID. CallFunctionExample.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 INPUT-NUMBER PIC 9(2) VALUE 10.
01 RESULT PIC 9(3).

PROCEDURE DIVISION.
    COMPUTE RESULT = FUNCTION MyFunction(INPUT-NUMBER)
    DISPLAY "The result is: " RESULT
    STOP RUN.

Best Practices

  • Modularity: Break down complex tasks into smaller subroutines or functions.
  • Reusability: Write subroutines and functions that can be reused across different programs.
  • Clarity: Use meaningful names for subroutines and functions to enhance code readability.
  • Documentation: Comment your code to explain the purpose and usage of subroutines and functions.

Summary

Subroutines and functions are powerful tools in COBOL that help in creating organized, modular, and reusable code. Understanding their structure, usage, and best practices is crucial for efficient COBOL programming. By mastering these concepts, you can write more maintainable and scalable COBOL programs.
