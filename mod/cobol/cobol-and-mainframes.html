Introduction to COBOL

COBOL (Common Business-Oriented Language) is a high-level programming language designed for business applications. It has been widely used in various industries, particularly in financial institutions and government agencies.

  • History of COBOL: Developed in 1959 by CODASYL (Conference on Data Systems Languages).
  • Key Features:
    • English-like syntax
    • Strong data processing capabilities
    • Portability across different systems

Setting Up Your COBOL Environment

Before diving into COBOL programming, you need to set up your development environment.

  • COBOL Compilers:

    • GnuCOBOL (open-source)
    • Micro Focus COBOL
    • IBM COBOL for z/OS

  • Installation:

    • Follow the specific instructions for your chosen compiler.
    • Ensure you have access to a mainframe or a mainframe emulator if necessary.

Basic COBOL Syntax and Structure

COBOL programs are divided into four main divisions:

  1. Identification Division: Contains metadata about the program.
  2. Environment Division: Specifies the environment in which the program will run.
  3. Data Division: Defines the variables and data structures used.
  4. Procedure Division: Contains the executable code.

Example Program

IDENTIFICATION DIVISION.
PROGRAM-ID. HelloWorld.

ENVIRONMENT DIVISION.

DATA DIVISION.

PROCEDURE DIVISION.
    DISPLAY 'Hello, World!'.
    STOP RUN.
  • Explanation:
    • IDENTIFICATION DIVISION: Declares the program's name.
    • PROCEDURE DIVISION: Contains the DISPLAY statement to print "Hello, World!" and STOP RUN to end the program.

Data Types and Variables

COBOL supports various data types, primarily focused on numeric and alphanumeric data.

  • Numeric Data Types:

    • PIC 9(n): Numeric data with n digits.
    • PIC S9(n): Signed numeric data.

  • Alphanumeric Data Types:

    • PIC X(n): Alphanumeric data with n characters.

Example

DATA DIVISION.
WORKING-STORAGE SECTION.
01 NUM-VAR PIC 9(5).
01 ALPHA-VAR PIC X(10).

PROCEDURE DIVISION.
    MOVE 12345 TO NUM-VAR.
    MOVE 'Hello' TO ALPHA-VAR.
    DISPLAY NUM-VAR.
    DISPLAY ALPHA-VAR.
    STOP RUN.
  • Explanation:
    • NUM-VAR is a numeric variable with 5 digits.
    • ALPHA-VAR is an alphanumeric variable with 10 characters.
    • MOVE statements assign values to the variables.
    • DISPLAY statements print the values.

Control Structures

COBOL provides various control structures for flow control, including conditional statements and loops.

  • IF Statements:

    IF condition
    THEN
        statements
    ELSE
        statements
END-IF.

  • PERFORM Loops:

    PERFORM UNTIL condition
    statements
END-PERFORM.

Example

DATA DIVISION.
WORKING-STORAGE SECTION.
01 COUNTER PIC 9(2) VALUE 0.

PROCEDURE DIVISION.
    PERFORM UNTIL COUNTER > 10
        ADD 1 TO COUNTER
        DISPLAY 'Counter: ' COUNTER
    END-PERFORM.
    STOP RUN.
  • Explanation:
    • COUNTER is a numeric variable initialized to 0.
    • PERFORM UNTIL loop increments COUNTER until it exceeds 10.
    • DISPLAY prints the value of COUNTER.

File Handling

COBOL is renowned for its robust file handling capabilities, essential for business applications.

  • File Control:

    • SELECT statement to associate a file with a file descriptor.
    • ASSIGN clause to specify the physical file.

  • File Operations:

    • OPEN, READ, WRITE, CLOSE.

Example

IDENTIFICATION DIVISION.
PROGRAM-ID. FileExample.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT IN-FILE ASSIGN TO 'input.txt'
    ORGANIZATION IS LINE SEQUENTIAL.

DATA DIVISION.
FILE SECTION.
FD IN-FILE.
01 IN-RECORD PIC X(100).

WORKING-STORAGE SECTION.

PROCEDURE DIVISION.
    OPEN INPUT IN-FILE.
    READ IN-FILE INTO IN-RECORD
        AT END
            DISPLAY 'End of file reached.'
    END-READ.
    DISPLAY IN-RECORD.
    CLOSE IN-FILE.
    STOP RUN.
  • Explanation:
    • FILE-CONTROL section associates IN-FILE with input.txt.
    • FD (File Description) defines the structure of the file.
    • OPEN, READ, and CLOSE operations manage file access.

Advanced Topics

As you advance, you will encounter more complex concepts and features in COBOL.

  • Subprograms: Modularize code using CALL and PERFORM statements.
  • Error Handling: Use INVALID KEY and AT END clauses.
  • Database Access: Integrate with databases using embedded SQL (EXEC SQL).

Example of a Subprogram

IDENTIFICATION DIVISION.
PROGRAM-ID. MainProgram.

DATA DIVISION.
WORKING-STORAGE SECTION.
01 RESULT PIC 9(2).

PROCEDURE DIVISION.
    CALL 'SubProgram' USING RESULT.
    DISPLAY 'Result from SubProgram: ' RESULT.
    STOP RUN.

IDENTIFICATION DIVISION.
PROGRAM-ID. SubProgram.

DATA DIVISION.
LINKAGE SECTION.
01 RESULT PIC 9(2).

PROCEDURE DIVISION USING RESULT.
    MOVE 42 TO RESULT.
    EXIT PROGRAM.
  • Explanation:
    • MainProgram calls SubProgram and passes RESULT.
    • SubProgram sets RESULT to 42 and returns control to MainProgram.

Conclusion

COBOL remains a vital language in the business world, particularly for legacy systems on mainframes. Understanding its syntax, control structures, and file handling capabilities is crucial for maintaining and developing business applications. As you progress, mastering advanced topics like subprograms and database access will enable you to handle more complex tasks efficiently.
