Sequential File Processing in COBOL
Indexed File Processing ►

Sequential file processing is a fundamental concept in COBOL programming, essential for handling data stored in files. This topic will guide you from the basics to advanced techniques, ensuring you understand how to read from and write to sequential files effectively.

Introduction to Sequential Files

Sequential files are files where data is stored in a specific order, and records are accessed sequentially, one after the other. This is the most common file organization in COBOL.

Key Concepts

  • File Control: Specifies the file's organization and access mode.
  • File Description (FD): Describes the structure of the file's records.
  • OPEN Statement: Opens the file for reading, writing, or both.
  • READ Statement: Reads records from the file.
  • WRITE Statement: Writes records to the file.
  • CLOSE Statement: Closes the file after processing.

File Control and File Description

File Control

The SELECT and ASSIGN clauses in the FILE-CONTROL paragraph specify the file's organization and the external file name.

PROGRAM-ID. SeqFileExample.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT InputFile ASSIGN TO 'input.dat'
        ORGANIZATION IS SEQUENTIAL.
    SELECT OutputFile ASSIGN TO 'output.dat'
        ORGANIZATION IS SEQUENTIAL.

File Description

The FD entry in the DATA DIVISION describes the structure of the file's records.

FD  OutputFile.
01  OutputRecord.
    05  CustomerID    PIC X(10).

Opening and Closing Files

OPEN Statement

The OPEN statement is used to open files for processing.

PROCEDURE DIVISION.
    OPEN INPUT InputFile.
    OPEN OUTPUT OutputFile.

CLOSE Statement

The CLOSE statement is used to close files after processing.

    CLOSE InputFile.
    CLOSE OutputFile.

Reading and Writing Records

READ Statement

The READ statement reads the next record from the file.

    READ InputFile INTO InputRecord
        AT END
            MOVE 'YES' TO EndOfFile
    END-READ.

WRITE Statement

The WRITE statement writes a record to the file.

    MOVE InputRecord TO OutputRecord.
    WRITE OutputRecord.

Example Program

Here is a complete example that reads records from an input file and writes them to an output file.

IDENTIFICATION DIVISION.
PROGRAM-ID. SeqFileExample.

ENVIRONMENT DIVISION.
INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT InputFile ASSIGN TO 'input.dat'
        ORGANIZATION IS SEQUENTIAL.
    SELECT OutputFile ASSIGN TO 'output.dat'
        ORGANIZATION IS SEQUENTIAL.

DATA DIVISION.
FILE SECTION.
FD  InputFile.
01  InputRecord.
    05  CustomerID    PIC X(10).
    05  CustomerName  PIC X(30).
    05  CustomerBalance PIC 9(7)V99.

FD  OutputFile.
01  OutputRecord.
    05  CustomerID    PIC X(10).
    05  CustomerName  PIC X(30).
    05  CustomerBalance PIC 9(7)V99.

WORKING-STORAGE SECTION.
01  EndOfFile        PIC X VALUE 'NO'.

PROCEDURE DIVISION.
    OPEN INPUT InputFile.
    OPEN OUTPUT OutputFile.

    PERFORM UNTIL EndOfFile = 'YES'
        READ InputFile INTO InputRecord
            AT END
                MOVE 'YES' TO EndOfFile
        END-READ

        IF EndOfFile = 'NO'
            MOVE InputRecord TO OutputRecord
            WRITE OutputRecord
        END-IF
    END-PERFORM.

    CLOSE InputFile.
    CLOSE OutputFile.

    STOP RUN.

Advanced Techniques

Handling File Status

Using the FILE STATUS clause to handle errors during file operations.

FILE-CONTROL.
    SELECT InputFile ASSIGN TO 'input.dat'
        ORGANIZATION IS SEQUENTIAL
        FILE STATUS IS WS-FileStatus.

WORKING-STORAGE SECTION.
01  WS-FileStatus    PIC XX.

Checking File Status

Checking the file status after each operation to ensure successful execution.

    READ InputFile INTO InputRecord
        AT END
            MOVE 'YES' TO EndOfFile
    END-READ.

    IF WS-FileStatus NOT = '00'
        DISPLAY 'Error reading file: ' WS-FileStatus
        STOP RUN
    END-IF.

Conclusion

Sequential file processing in COBOL involves understanding how to define, open, read, write, and close files. By mastering these basic and advanced techniques, you can efficiently handle data stored in sequential files, a common requirement in many COBOL applications. Practice with different file structures and error-handling mechanisms to become proficient in sequential file processing.
