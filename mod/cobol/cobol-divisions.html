COBOL Divisions
COBOL Sections and Paragraphs ►

COBOL (Common Business-Oriented Language) is structured into four main divisions. Each division serves a specific purpose and helps in organizing the code in a logical and readable manner. Understanding these divisions is crucial for writing and maintaining COBOL programs.

1. Identification Division

The Identification Division is the first division in a COBOL program. It provides metadata about the program, such as the program name and author details.

Key Concepts:

  • PROGRAM-ID: Specifies the name of the program.
  • AUTHOR: Provides the name of the programmer.
  • INSTALLATION: Describes the installation site.
  • DATE-WRITTEN: Indicates when the program was written.
  • DATE-COMPILED: Shows when the program was compiled.

Example:

IDENTIFICATION DIVISION.
PROGRAM-ID. HelloWorld.
AUTHOR. John Doe.
INSTALLATION. XYZ Corporation.
DATE-WRITTEN. 2023-10-01.
DATE-COMPILED. 2023-10-02.

2. Environment Division

The Environment Division specifies the computer environment in which the program will run. It includes information about the configuration and the files used by the program.

Key Concepts:

  • CONFIGURATION SECTION: Describes the system configuration.
  • INPUT-OUTPUT SECTION: Defines the files and devices used by the program.

Example:

ENVIRONMENT DIVISION.
CONFIGURATION SECTION.
SOURCE-COMPUTER. IBM-370.
OBJECT-COMPUTER. IBM-370.

INPUT-OUTPUT SECTION.
FILE-CONTROL.
    SELECT InputFile ASSIGN TO 'input.dat'.
    SELECT OutputFile ASSIGN TO 'output.dat'.

3. Data Division

The Data Division is where all the variables and data structures used in the program are defined. It is divided into several sections, but the most commonly used are the Working-Storage Section and the File Section.

Key Concepts:

  • FILE SECTION: Defines the structure of files.
  • WORKING-STORAGE SECTION: Declares variables and constants.
  • LINKAGE SECTION: Used for parameters passed between programs.

Example:

DATA DIVISION.
FILE SECTION.
FD  InputFile.
01  InputRecord.
    05  InputField1  PIC X(10).
    05  InputField2  PIC 9(5).

FD  OutputFile.
01  OutputRecord.
    05  OutputField1  PIC X(10).
    05  OutputField2  PIC 9(5).

WORKING-STORAGE SECTION.
01  WS-Variable1  PIC X(20) VALUE 'Hello, COBOL!'.
01  WS-Variable2  PIC 9(4) VALUE 2023.

4. Procedure Division

The Procedure Division contains the executable instructions of the program. This is where the logic of the program is implemented through various COBOL statements.

Key Concepts:

  • Paragraphs: Logical blocks of code.
  • Sections: Group of paragraphs.
  • Statements: Individual instructions.

Example:

PROCEDURE DIVISION.
Main-Procedure.
    OPEN INPUT InputFile.
    OPEN OUTPUT OutputFile.
    PERFORM Read-Process-Write UNTIL End-Of-File.
    CLOSE InputFile.
    CLOSE OutputFile.
    STOP RUN.

Read-Process-Write.
    READ InputFile INTO InputRecord
        AT END
            SET End-Of-File TO TRUE
        NOT AT END
            MOVE InputField1 TO OutputField1
            MOVE InputField2 TO OutputField2
            WRITE OutputRecord.

Summary

Understanding the four main divisions in COBOL is essential for writing structured and maintainable programs. Each division has a specific role:

  • Identification Division: Provides metadata about the program.
  • Environment Division: Specifies the system environment and file usage.
  • Data Division: Defines the data structures and variables.
  • Procedure Division: Contains the executable logic of the program.

By mastering these divisions, you will be able to create well-organized COBOL programs that are easier to read, debug, and maintain.
