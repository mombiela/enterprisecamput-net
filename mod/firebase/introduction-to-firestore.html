Introduction to Firestore
Setting Up Firestore ►

Firestore is a flexible, scalable database for mobile, web, and server development from Firebase and Google Cloud Platform. It is a NoSQL document database that lets you easily store, sync, and query data for your apps.

Key Concepts

  • NoSQL Database: Firestore is a NoSQL database, which means it stores data in a non-tabular format.
  • Documents and Collections: Data is stored in documents, which are organized into collections.
  • Real-time Updates: Firestore can sync data in real-time across all clients.
  • Offline Support: Firestore supports offline data persistence, allowing apps to work even when there is no network connection.
  • Security Rules: Firestore provides a powerful rules language to control access to your data.

Setting Up Firestore

  1. Create a Firebase Project: Go to the Firebase Console and create a new project.
  2. Add Firestore to Your Project: Navigate to the Firestore section in the Firebase Console and enable Firestore.
  3. Install Firebase SDK: Add the Firebase SDK to your project.

Example: Adding Firebase SDK to a Web Project

<!-- Add Firebase SDKs -->
<script src="https://www.gstatic.com/firebasejs/9.0.0/firebase-app.js"></script>
<script src="https://www.gstatic.com/firebasejs/9.0.0/firebase-firestore.js"></script>

<script>
  // Your web app's Firebase configuration
  var firebaseConfig = {
    apiKey: "YOUR_API_KEY",
    authDomain: "YOUR_PROJECT_ID.firebaseapp.com",
    projectId: "YOUR_PROJECT_ID",
    storageBucket: "YOUR_PROJECT_ID.appspot.com",
    messagingSenderId: "YOUR_MESSAGING_SENDER_ID",
    appId: "YOUR_APP_ID"
  };
  // Initialize Firebase
  firebase.initializeApp(firebaseConfig);
  var db = firebase.firestore();
</script>

Working with Firestore

Adding Data

To add data to Firestore, you create a document within a collection.

// Add a new document with a generated ID
db.collection("users").add({
    first: "Ada",
    last: "Lovelace",
    born: 1815
})
.then((docRef) => {
    console.log("Document written with ID: ", docRef.id);
})
.catch((error) => {
    console.error("Error adding document: ", error);
});

Reading Data

You can read data from Firestore by querying documents.

// Get a list of users
db.collection("users").get().then((querySnapshot) => {
    querySnapshot.forEach((doc) => {
        console.log(`${doc.id} => ${doc.data()}`);
    });
});

Updating Data

To update a document, you can use the update method.

var userRef = db.collection("users").doc("USER_ID");

// Set the "capital" field of the city 'DC'
return userRef.update({
    capital: true
})
.then(() => {
    console.log("Document successfully updated!");
})
.catch((error) => {
    console.error("Error updating document: ", error);
});

Deleting Data

To delete a document, use the delete method.

db.collection("users").doc("USER_ID").delete().then(() => {
    console.log("Document successfully deleted!");
}).catch((error) => {
    console.error("Error removing document: ", error);
});

Real-time Updates

Firestore allows you to listen to real-time updates.

db.collection("users").onSnapshot((querySnapshot) => {
    var users = [];
    querySnapshot.forEach((doc) => {
        users.push(doc.data());
    });
    console.log("Current users in collection: ", users.join(", "));
});

Security Rules

Firestore security rules allow you to control access to your data.

service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}

Conclusion

Firestore is a powerful NoSQL database that offers real-time data synchronization, offline support, and robust security features. By understanding the basics of setting up Firestore, adding, reading, updating, and deleting data, as well as implementing real-time updates and security rules, you can effectively manage your app's data needs.
