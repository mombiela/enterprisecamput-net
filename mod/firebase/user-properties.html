Introduction

User properties in Firebase are attributes you can define to describe segments of your user base. These properties help you understand your users better and can be used to target specific audiences for analytics, notifications, and more.

Key Concepts

  • User Properties: Custom attributes you define to describe segments of your user base.
  • Predefined User Properties: Firebase provides some predefined user properties like first_open_time, last_sign_in_time, etc.
  • Custom User Properties: You can define up to 25 custom user properties per project.

Setting User Properties

Using Firebase Console

  1. Navigate to the Firebase Console.
  2. Select your project.
  3. Go to the "Analytics" section.
  4. Click on "User Properties".
  5. Add a new user property by defining its name and description.

Using Firebase SDK

Android (Java)

// Import Firebase Analytics
import com.google.firebase.analytics.FirebaseAnalytics;

// Get Firebase Analytics instance
FirebaseAnalytics mFirebaseAnalytics = FirebaseAnalytics.getInstance(this);

// Set user property
mFirebaseAnalytics.setUserProperty("favorite_food", "pizza");

iOS (Swift)

// Import Firebase
import Firebase

// Get Firebase Analytics instance
let analytics = Analytics.self

// Set user property
analytics.setUserProperty("favorite_food", forName: "pizza")

Web (JavaScript)

// Import Firebase
import firebase from "firebase/app";
import "firebase/analytics";

// Initialize Firebase
firebase.initializeApp({
  apiKey: "YOUR_API_KEY",
  authDomain: "YOUR_AUTH_DOMAIN",
  projectId: "YOUR_PROJECT_ID",
  storageBucket: "YOUR_STORAGE_BUCKET",
  messagingSenderId: "YOUR_MESSAGING_SENDER_ID",
  appId: "YOUR_APP_ID",
  measurementId: "YOUR_MEASUREMENT_ID"
});

// Get Firebase Analytics instance
const analytics = firebase.analytics();

// Set user property
analytics.setUserProperties({ favorite_food: "pizza" });

Using User Properties in Analytics

Creating Audiences

  1. Go to the Firebase Console.
  2. Select your project.
  3. Navigate to the "Analytics" section.
  4. Click on "Audiences".
  5. Create a new audience by defining conditions based on user properties.

Filtering Reports

  • You can filter your analytics reports based on user properties to gain insights into specific segments of your user base.

Best Practices

  • Keep it Simple: Use clear and concise names for your user properties.
  • Consistency: Ensure consistency in naming conventions across different platforms.
  • Privacy: Avoid using sensitive information as user properties.

Example Use Cases

Targeting Notifications

You can use user properties to target notifications to specific user segments. For example, if you have a user property favorite_food, you can send a notification about a pizza discount to users who have set favorite_food to "pizza".

Customizing User Experience

Use user properties to customize the user experience. For instance, if you have a user property preferred_language, you can display content in the user's preferred language.

Conclusion

User properties in Firebase are powerful tools for understanding and segmenting your user base. By setting and using user properties effectively, you can gain valuable insights and create more personalized experiences for your users. Remember to follow best practices to ensure consistency and privacy.
