Custom Domains in Firebase
Hosting and Cloud Functions ►

Firebase Hosting allows you to deploy web apps quickly and securely. One of the powerful features it offers is the ability to use custom domains for your web apps. This section will guide you through the process of setting up and managing custom domains in Firebase, from beginner to advanced levels.

Introduction to Custom Domains

Custom domains allow you to use your own domain name (e.g., www.yourdomain.com) instead of the default Firebase subdomain (e.g., yourapp.firebaseapp.com). This can enhance your brand's identity and make your app more accessible.

Key Concepts

  • Custom Domain: Your own domain name that you want to use with your Firebase-hosted site.
  • DNS Records: Settings that tell the internet where to find your domain.
  • SSL Certificates: Security certificates that encrypt data between your users and your site.

Setting Up a Custom Domain

Step 1: Purchase a Domain

Before you can set up a custom domain, you need to purchase one from a domain registrar like Google Domains, GoDaddy, or Namecheap.

Step 2: Add Your Domain to Firebase

  1. Go to the Firebase Console.
  2. Select your project.
  3. In the left-hand menu, click on Hosting.
  4. Click on Add Custom Domain.
  5. Enter your domain name and click Continue.

Step 3: Verify Domain Ownership

Firebase will provide you with a TXT record that you need to add to your DNS settings to verify that you own the domain.

Example:

Type: TXT
Name: @
Value: firebase=your-verification-code

Step 4: Update DNS Records

After verifying your domain, you need to update your DNS records to point to Firebase Hosting.

Example:

Type: A
Name: @
Value: 151.101.1.195

Step 5: Deploy Your Site

Deploy your site using the Firebase CLI:

firebase deploy

Managing SSL Certificates

Firebase automatically provisions SSL certificates for your custom domain, ensuring that your site is served over HTTPS.

Key Points

  • Automatic SSL: Firebase handles SSL certificates for you.
  • Renewal: SSL certificates are automatically renewed by Firebase.
  • Custom SSL: Advanced users can upload their own SSL certificates if needed.

Advanced Configuration

Redirects and Rewrites

You can configure redirects and rewrites in your firebase.json file to manage how URLs are handled.

Example:

{
  "hosting": {
    "redirects": [
      {
        "source": "/old-page",
        "destination": "/new-page",
        "type": 301
      }
    ],
    "rewrites": [
      {
        "source": "/api/**",
        "function": "api"
      }
    ]
  }
}

Multiple Sites

Firebase Hosting supports multiple sites within a single Firebase project. This is useful for managing different environments (e.g., staging and production) or different parts of a large application.

Example:

{
  "hosting": [
    {
      "target": "project1",
      "public": "public1",
      "ignore": ["firebase.json", "**/.*", "**/node_modules/**"]
    },
    {
      "target": "project2",
      "public": "public2",
      "ignore": ["firebase.json", "**/.*", "**/node_modules/**"]
    }
  ]
}

Troubleshooting

Common Issues

  • DNS Propagation: Changes to DNS records can take up to 48 hours to propagate.
  • SSL Errors: Ensure that your domain is correctly verified and that DNS records are properly configured.
  • Deployment Issues: Check the Firebase CLI output for errors and ensure that your firebase.json is correctly configured.

Conclusion

Setting up a custom domain in Firebase Hosting enhances your web app's professionalism and accessibility. By following the steps outlined in this section, you can easily configure and manage custom domains, ensuring a secure and branded experience for your users. Whether you're a beginner or an advanced user, Firebase provides the tools and flexibility needed to manage your web app's hosting effectively.
