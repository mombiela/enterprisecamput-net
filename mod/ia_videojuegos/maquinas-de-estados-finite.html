Introduction to Finite State Machines

Finite State Machines (FSM) are a mathematical model used to design logical and control systems. In the context of video games, FSMs are a powerful tool for managing the behavior of non-playable characters (NPCs), allowing them to react coherently to different game situations.

Key Concepts

  • State: Represents a specific condition or situation in which the system can be.
  • Transition: The change from one state to another, usually triggered by an event or condition.
  • Event: An action or condition that causes a transition between states.
  • Action: An operation that is executed when entering, exiting, or remaining in a state.

Structure of an FSM

An FSM is composed of:

  1. Set of States: All possible states in which the system can be.
  2. Initial State: The state in which the system begins.
  3. Events/Conditions: Factors that cause transitions between states.
  4. Transitions: The rules that define how and when to change from one state to another.
  5. Actions: The operations that are executed during transitions or while in a state.

Example of FSM in Video Games

Consider an NPC in a video game that can be in one of the following states: Patrolling, Chasing, Attacking, Fleeing.

Definition of States and Transitions

| State | Event/Condition | Next State | Action | |-------------|-----------------------------|------------------|--------------------------------| | Patrolling | Sees the player | Chasing | Start following the player | | Chasing | Player out of sight | Patrolling | Return to patrolling | | Chasing | Near the player | Attacking | Start attacking | | Attacking | Player out of reach | Chasing | Return to following the player | | Attacking | Low health | Fleeing | Escape from the player | | Fleeing | Health recovered | Patrolling | Return to patrolling |

Example Code in Python

Below is a basic example of how to implement an FSM for an NPC in Python:

class NPC:
    def __init__(self):
        self.state = 'Patrolling'
    
    def see_player(self):
        if self.state == 'Patrolling':
            self.state = 'Chasing'
            print("Start following the player")
    
    def lose_sight_of_player(self):
        if self.state == 'Chasing':
            self.state = 'Patrolling'
            print("Return to patrolling")
    
    def reach_player(self):
        if self.state == 'Chasing':
            self.state = 'Attacking'
            print("Start attacking")
    
    def player_out_of_reach(self):
        if self.state == 'Attacking':
            self.state = 'Chasing'
            print("Return to following the player")
    
    def low_health(self):
        if self.state == 'Attacking':
            self.state = 'Fleeing'
            print("Escape from the player")
    
    def health_recovered(self):
        if self.state == 'Fleeing':
            self.state = 'Patrolling'
            print("Return to patrolling")

# Example usage
npc = NPC()
npc.see_player()  # Start following the player
npc.reach_player()  # Start attacking
npc.low_health()  # Escape from the player
npc.health_recovered()  # Return to patrolling

Advantages and Disadvantages of FSM

Advantages

  • Simplicity: Easy to understand and implement.
  • Clarity: States and transitions are explicit, making maintenance and debugging easier.
  • Determinism: The system's behavior is predictable and reproducible.

Disadvantages

  • Scalability: Can become complex and difficult to manage with a large number of states and transitions.
  • Flexibility: Less flexible for very dynamic or adaptive behaviors.

Conclusion

Finite State Machines are a fundamental tool in the development of AI for video games, providing a clear and manageable structure to define NPC behavior. Although they have their limitations, their simplicity and clarity make them ideal for many common scenarios in video game development. With a solid understanding of FSMs, developers can create more realistic and reactive characters, enhancing the gaming experience for users.
