Introduction

Decision making is a crucial part of artificial intelligence (AI) in video games. It allows non-playable characters (NPCs) and other game elements to act autonomously and respond to player actions and the game environment. In this section, we will explore different techniques and algorithms for implementing decision making in video games.

Decision Making Techniques

Finite State Machines (FSM)

Finite State Machines are one of the most common and simple techniques for decision making in video games.

  • Definition: An FSM is a mathematical model consisting of a finite number of states, transitions between those states, and actions.
  • Advantages:
    • Easy to implement and understand.
    • Suitable for simple and predictable behaviors.
  • Disadvantages:
    • Can become complex and difficult to maintain with more advanced behaviors.

Decision Trees

Decision trees are another popular technique that allows for more complex and structured decision making.

  • Definition: A decision tree is a tree structure where each node represents a decision based on a condition.
  • Advantages:
    • More flexible than FSMs.
    • Easy to visualize and understand.
  • Disadvantages:
    • Can become very large and complex.

Code Example

class DecisionTreeNode:
    def __init__(self, condition, true_branch=None, false_branch=None):
        self.condition = condition
        self.true_branch = true_branch
        self.false_branch = false_branch

    def evaluate(self, context):
        if self.condition(context):
            if self.true_branch:
                return self.true_branch.evaluate(context)
            else:
                return True
        else:
            if self.false_branch:
                return self.false_branch.evaluate(context)
            else:
                return False

# Example usage
def is_enemy_near(context):
    return context["enemy_distance"] < 10

root = DecisionTreeNode(is_enemy_near, true_branch=DecisionTreeNode(lambda x: True), false_branch=DecisionTreeNode(lambda x: False))
context = {"enemy_distance": 5}
decision = root.evaluate(context)

Behavior Trees

Behavior trees are an evolution of decision trees and are used to model more complex and hierarchical behaviors.

  • Definition: A behavior tree is a hierarchical structure of nodes that represent tasks and conditions.
  • Advantages:
    • Modular and reusable.
    • Facilitates the creation of complex behaviors.
  • Disadvantages:
    • Requires more time to learn and master.

Code Example

class BehaviorTreeNode:
    def run(self):
        raise NotImplementedError

class Selector(BehaviorTreeNode):
    def __init__(self, children):
        self.children = children

    def run(self):
        for child in self.children:
            if child.run():
                return True
        return False

class Sequence(BehaviorTreeNode):
    def __init__(self, children):
        self.children = children

    def run(self):
        for child in self.children:
            if not child.run():
                return False
        return True

class Condition(BehaviorTreeNode):
    def __init__(self, condition):
        self.condition = condition

    def run(self):
        return self.condition()

class Action(BehaviorTreeNode):
    def __init__(self, action):
        self.action = action

    def run(self):
        self.action()
        return True

Conclusion

Implementing decision making in video games is essential for creating immersive and challenging gaming experiences. Techniques such as Finite State Machines, Decision Trees, and Behavior Trees offer different levels of complexity and flexibility, allowing developers to choose the best tool for their specific needs. By mastering these techniques, developers can create NPCs and other game elements that respond intelligently and dynamically to player actions and the game environment.
