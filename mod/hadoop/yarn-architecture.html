Introduction to YARN

YARN (Yet Another Resource Negotiator) is a core component of Hadoop that manages resources and schedules jobs across the cluster. It was introduced in Hadoop 2.x to overcome the limitations of the original MapReduce framework.

Key Components of YARN

YARN consists of several key components that work together to manage resources and execute jobs efficiently.

ResourceManager

  • Role: The ResourceManager is the master daemon responsible for managing resources across the cluster.
  • Functions:
    • Resource Allocation: Allocates resources to various applications running in the cluster.
    • Job Scheduling: Schedules jobs based on resource availability and policies.

NodeManager

  • Role: The NodeManager is a per-node daemon responsible for managing resources on individual nodes.
  • Functions:
    • Resource Monitoring: Monitors resource usage (CPU, memory, disk) on the node.
    • Container Management: Manages the lifecycle of containers (units of computation).

ApplicationMaster

  • Role: The ApplicationMaster is a per-application daemon responsible for negotiating resources from the ResourceManager and working with the NodeManager to execute and monitor tasks.
  • Functions:
    • Resource Negotiation: Requests resources from the ResourceManager.
    • Task Execution: Manages the execution of tasks within the allocated resources.

Containers

  • Role: Containers are the basic units of resource allocation in YARN.
  • Functions:
    • Resource Isolation: Each container is allocated a specific amount of CPU and memory.
    • Task Execution: Containers run tasks as specified by the ApplicationMaster.

YARN Workflow

Understanding the workflow of YARN helps in grasping how it manages resources and executes jobs.

  1. Job Submission: A client submits a job to the ResourceManager.
  2. ApplicationMaster Launch: The ResourceManager allocates a container for the ApplicationMaster.
  3. Resource Negotiation: The ApplicationMaster negotiates resources with the ResourceManager.
  4. Task Execution: The ApplicationMaster requests containers from NodeManagers to execute tasks.
  5. Monitoring and Completion: The ApplicationMaster monitors task execution and reports status back to the client.

Example: Submitting a YARN Job

Here's a simple example to illustrate how a job is submitted and executed in YARN.

# Submit a YARN job using the Hadoop command line
hadoop jar /path/to/your/hadoop-job.jar com.example.YourMainClass input output

Explanation:

  • hadoop jar: Command to run a Hadoop job.
  • /path/to/your/hadoop-job.jar: Path to your compiled Hadoop job JAR file.
  • com.example.YourMainClass: The main class of your Hadoop job.
  • input: Input directory in HDFS.
  • output: Output directory in HDFS.

Resource Allocation Policies

YARN supports various resource allocation policies to manage resources effectively.

  • FIFO (First In, First Out): Jobs are scheduled in the order they are submitted.
  • Capacity Scheduler: Divides cluster resources into queues, each with a guaranteed capacity.
  • Fair Scheduler: Allocates resources such that all jobs get a fair share of resources over time.

Conclusion

YARN is a powerful resource management and job scheduling framework in Hadoop. It decouples resource management from job execution, allowing for more flexible and efficient resource utilization. Understanding YARN's architecture and workflow is crucial for optimizing Hadoop jobs and managing large-scale data processing tasks effectively.
