Introduction to HDFS

HDFS, or Hadoop Distributed File System, is the primary storage system used by Hadoop applications. It is designed to store large datasets reliably and to stream those data sets at high bandwidth to user applications.

Key Concepts

  • Scalability: HDFS can scale up to thousands of nodes and petabytes of data.
  • Fault Tolerance: Data is replicated across multiple nodes to ensure reliability.
  • High Throughput: Designed for high data throughput, not low latency.
  • Streaming Data Access: Optimized for large data sets and batch processing.
  • Commodity Hardware: Runs on inexpensive, off-the-shelf hardware.

Architecture of HDFS

HDFS follows a master-slave architecture where the master is called the NameNode and the slaves are called DataNodes.

NameNode

  • Role: Manages the metadata and namespace of the file system.
  • Responsibilities:
    • Keeps track of the file system tree and the metadata for all the files and directories.
    • Manages the DataNodes and the blocks they store.
    • Handles client requests for file operations.

DataNode

  • Role: Stores the actual data blocks.
  • Responsibilities:
    • Serves read and write requests from the file system's clients.
    • Performs block creation, deletion, and replication upon instruction from the NameNode.

Block Storage

  • Block Size: Default block size is 128 MB.
  • Replication: Default replication factor is 3.
  • Advantages:
    • Large block size minimizes the cost of seeks.
    • Replication ensures data reliability and availability.

HDFS Operations

File Read and Write

  • Write Operation:

    1. Client requests the NameNode to create a new file.
    2. NameNode checks for permissions and creates metadata.
    3. Client writes data to DataNodes in blocks.
    4. DataNodes replicate the blocks to other DataNodes.

  • Read Operation:

    1. Client requests the NameNode for file metadata.
    2. NameNode provides the list of DataNodes containing the blocks.
    3. Client reads the blocks directly from the DataNodes.

Example Code: Writing and Reading Files

// Writing a file to HDFS
import org.apache.hadoop.conf.Configuration;
import org.apache.hadoop.fs.FileSystem;
import org.apache.hadoop.fs.Path;
import java.io.OutputStream;

public class HDFSWriteExample {
    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        FileSystem fs = FileSystem.get(conf);
        Path path = new Path("/user/hadoop/example.txt");
        OutputStream os = fs.create(path);
        os.write("Hello HDFS!".getBytes());
        os.close();
    }
}

// Reading a file from HDFS
import org.apache.hadoop.conf.Configuration;
import org.apache.hadoop.fs.FileSystem;
import org.apache.hadoop.fs.Path;
import java.io.InputStream;
import java.io.BufferedReader;
import java.io.InputStreamReader;

public class HDFSReadExample {
    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        FileSystem fs = FileSystem.get(conf);
        Path path = new Path("/user/hadoop/example.txt");
        InputStream is = fs.open(path);
        BufferedReader br = new BufferedReader(new InputStreamReader(is));
        String line;
        while ((line = br.readLine()) != null) {
            System.out.println(line);
        }
        br.close();
    }
}

Advanced HDFS Features

Federation

  • Purpose: To scale the NameNode horizontally.
  • How it works: Multiple NameNodes manage different namespaces, and DataNodes store blocks for multiple NameNodes.

High Availability (HA)

  • Purpose: To ensure the NameNode is not a single point of failure.
  • How it works: Uses a pair of NameNodes in an active-standby configuration.

Snapshots

  • Purpose: To capture the state of the file system at a specific point in time.
  • How it works: Allows for read-only or writable snapshots of the file system.

Conclusion

HDFS is a robust and scalable file system designed to handle large datasets with high throughput. Its architecture ensures data reliability and availability through replication and fault tolerance. Understanding HDFS operations and features is crucial for leveraging Hadoop's full potential in big data processing.
