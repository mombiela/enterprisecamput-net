Introduction

Hadoop is an open-source framework that allows for the distributed processing of large data sets across clusters of computers using simple programming models. It is designed to scale up from single servers to thousands of machines, each offering local computation and storage. This section will cover the core components of Hadoop, which are essential for understanding how Hadoop operates and how to use it effectively.

Core Components of Hadoop

1. Hadoop Distributed File System (HDFS)

HDFS is the primary storage system used by Hadoop applications. It provides high-throughput access to application data and is designed to be highly fault-tolerant.

  • Key Concepts:

    • Blocks: HDFS stores each file as a series of blocks. Each block is typically 128 MB by default.
    • Namenode: The master server that manages the file system namespace and controls access to files by clients.
    • Datanodes: These are the worker nodes that store and retrieve blocks when they are told to (by the Namenode).

  • Example:

    # Listing files in HDFS
hdfs dfs -ls /

# Uploading a file to HDFS
hdfs dfs -put localfile.txt /hdfs/directory/

# Reading a file from HDFS
hdfs dfs -cat /hdfs/directory/localfile.txt

2. MapReduce

MapReduce is a programming model and an associated implementation for processing and generating large data sets with a parallel, distributed algorithm on a cluster.

  • Key Concepts:

    • Map Function: Processes input key/value pairs to generate a set of intermediate key/value pairs.
    • Reduce Function: Merges all intermediate values associated with the same intermediate key.

  • Example:

    public class WordCount {
    public static class TokenizerMapper
         extends Mapper<Object, Text, Text, IntWritable>{
        private final static IntWritable one = new IntWritable(1);
        private Text word = new Text();

        public void map(Object key, Text value, Context context
                        ) throws IOException, InterruptedException {
            StringTokenizer itr = new StringTokenizer(value.toString());
            while (itr.hasMoreTokens()) {
                word.set(itr.nextToken());
                context.write(word, one);
            }
        }
    }

    public static class IntSumReducer
         extends Reducer<Text,IntWritable,Text,IntWritable> {
        private IntWritable result = new IntWritable();

        public void reduce(Text key, Iterable<IntWritable> values,
                           Context context
                           ) throws IOException, InterruptedException {
            int sum = 0;
            for (IntWritable val : values) {
                sum += val.get();
            }
            result.set(sum);
            context.write(key, result);
        }
    }
}

3. YARN (Yet Another Resource Negotiator)

YARN is Hadoop's cluster resource management system. It allows multiple data processing engines such as interactive SQL, real-time streaming, data science, and batch processing to run and process data stored in HDFS.

  • Key Concepts:

    • ResourceManager: The master that arbitrates all the available cluster resources.
    • NodeManager: The per-machine framework agent responsible for containers, monitoring their resource usage (CPU, memory, disk, network) and reporting the same to the ResourceManager.
    • ApplicationMaster: Manages the lifecycle of applications running on YARN.

  • Example:

    # Checking the status of YARN
yarn application -list

# Killing a YARN application
yarn application -kill application_1234567890123_0001

4. Hadoop Common

Hadoop Common contains libraries and utilities needed by other Hadoop modules. These libraries provide file system and OS level abstractions and contain the necessary Java files and scripts required to start Hadoop.

  • Key Concepts:

    • Configuration: Provides a way to configure the various parameters of Hadoop.
    • Serialization: Mechanism for converting data structures or object state into a format that can be stored or transmitted and reconstructed later.

  • Example:

    Configuration conf = new Configuration();
conf.set("fs.defaultFS", "hdfs://namenode:8020");
FileSystem fs = FileSystem.get(conf);

Conclusion

Understanding the core components of Hadoop is crucial for effectively utilizing the framework for big data processing. HDFS provides the storage layer, MapReduce offers the processing model, YARN manages resources, and Hadoop Common includes essential utilities. Together, these components form the backbone of the Hadoop ecosystem, enabling scalable and efficient data processing across distributed environments.
