Introduction to YARN

YARN, which stands for Yet Another Resource Negotiator, is a core component of Hadoop that manages resources and schedules jobs in a Hadoop cluster. It was introduced in Hadoop 2.0 to overcome the limitations of the original MapReduce framework.

Key Concepts

  • Resource Management: YARN manages the resources of the Hadoop cluster, ensuring that applications have the necessary resources to run efficiently.
  • Job Scheduling: YARN schedules jobs and allocates resources dynamically based on the needs of the applications.
  • Scalability: YARN enhances the scalability of Hadoop by allowing multiple applications to run simultaneously.

Architecture of YARN

YARN's architecture consists of several key components that work together to manage resources and schedule jobs.

Components

  • ResourceManager (RM): The master daemon that manages resources and schedules applications.
  • NodeManager (NM): The per-node daemon that monitors resource usage and reports to the ResourceManager.
  • ApplicationMaster (AM): A framework-specific library that negotiates resources from the ResourceManager and works with the NodeManager to execute and monitor tasks.
  • Container: A logical bundle of resources (memory, CPU) allocated by the ResourceManager to an application.

How YARN Works

YARN operates by separating resource management and job scheduling/monitoring into separate daemons.

Workflow

  1. Client Submission: A client submits an application to the ResourceManager.
  2. Resource Allocation: The ResourceManager allocates a container for the ApplicationMaster.
  3. ApplicationMaster Initialization: The ApplicationMaster initializes and negotiates resources for the application.
  4. Task Execution: The ApplicationMaster requests containers from the ResourceManager, and the NodeManager launches the tasks.
  5. Monitoring and Completion: The ApplicationMaster monitors the progress of the tasks and reports back to the ResourceManager upon completion.

YARN Configuration

To configure YARN, you need to modify several configuration files in the Hadoop setup.

Key Configuration Files

  • yarn-site.xml: Contains YARN-specific configurations.
  • mapred-site.xml: Contains MapReduce-specific configurations.

Example Configuration

<!-- yarn-site.xml -->
<configuration>
  <property>
    <name>yarn.resourcemanager.hostname</name>
    <value>resourcemanager.example.com</value>
  </property>
  <property>
    <name>yarn.nodemanager.resource.memory-mb</name>
    <value>8192</value>
  </property>
  <property>
    <name>yarn.nodemanager.resource.cpu-vcores</name>
    <value>4</value>
  </property>
</configuration>
<!-- mapred-site.xml -->
<configuration>
  <property>
    <name>mapreduce.framework.name</name>
    <value>yarn</value>
  </property>
  <property>
    <name>mapreduce.jobhistory.address</name>
    <value>jobhistory.example.com:10020</value>
  </property>
</configuration>

Advanced YARN Features

YARN provides several advanced features to enhance resource management and job scheduling.

Features

  • Resource Scheduling Policies: YARN supports different scheduling policies like FIFO, Capacity Scheduler, and Fair Scheduler.
  • Resource Preemption: Allows preemption of resources from lower-priority applications to higher-priority ones.
  • High Availability: YARN supports high availability for the ResourceManager to ensure continuous operation.

Conclusion

YARN is a powerful resource management and job scheduling framework that enhances the scalability and efficiency of Hadoop clusters. By understanding its architecture, workflow, and configuration, you can effectively manage and optimize your Hadoop applications. Advanced features like resource scheduling policies and high availability further extend YARN's capabilities, making it a critical component of modern big data processing.
