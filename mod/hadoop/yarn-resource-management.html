Introduction to YARN

YARN (Yet Another Resource Negotiator) is a core component of Hadoop that manages resources and schedules jobs. It allows multiple data processing engines such as batch processing, interactive processing, and real-time processing to run and process data stored in HDFS.

Key Concepts of YARN

  • ResourceManager (RM): The master daemon that manages resources and schedules applications.
  • NodeManager (NM): The per-node agent responsible for managing containers, monitoring their resource usage (CPU, memory, disk, network), and reporting to the ResourceManager.
  • ApplicationMaster (AM): A framework-specific library that negotiates resources from the ResourceManager and works with the NodeManager(s) to execute and monitor tasks.
  • Container: A collection of physical resources (CPU, memory, etc.) allocated by the ResourceManager for a specific application.

YARN Architecture

YARN follows a master-slave architecture with the ResourceManager as the master and NodeManagers as the slaves.

ResourceManager Components

  • Scheduler: Allocates resources to various running applications based on resource availability and configured policies.
  • ApplicationManager: Manages the lifecycle of applications, including accepting job submissions, negotiating the first container for executing the ApplicationMaster, and restarting the ApplicationMaster on failure.

NodeManager Components

  • ContainerManager: Manages the lifecycle of containers on a node, including starting, stopping, and monitoring resource usage.
  • Resource Monitoring: Monitors the resource usage of containers and reports to the ResourceManager.

Resource Allocation in YARN

YARN uses a pluggable scheduler to allocate resources. The most common schedulers are:

  • FIFO Scheduler: Simple and straightforward, processes jobs in the order they arrive.
  • Capacity Scheduler: Allows sharing of resources among multiple organizations, ensuring a minimum capacity guarantee.
  • Fair Scheduler: Allocates resources fairly among all running jobs, ensuring no job is starved.

Example Configuration for Capacity Scheduler

<property>
  <name>yarn.scheduler.capacity.root.queues</name>
  <value>default,production</value>
</property>
<property>
  <name>yarn.scheduler.capacity.root.default.capacity</name>
  <value>50</value>
</property>
<property>
  <name>yarn.scheduler.capacity.root.production.capacity</name>
  <value>50</value>
</property>

In this example, two queues (default and production) are configured, each with 50% of the total cluster capacity.

Running Applications on YARN

When an application is submitted to YARN, the following steps occur:

  1. Submission: The client submits the application to the ResourceManager.
  2. ApplicationMaster Launch: The ResourceManager allocates a container for the ApplicationMaster and starts it.
  3. Resource Request: The ApplicationMaster requests resources for the application's tasks.
  4. Task Execution: The NodeManagers launch containers to execute the tasks.
  5. Monitoring and Completion: The ApplicationMaster monitors the tasks and reports the status back to the client. Once all tasks are completed, the ApplicationMaster shuts down.

Example of Submitting a YARN Application

yarn jar my-app.jar com.example.MyApp

This command submits the application my-app.jar to YARN, where com.example.MyApp is the main class.

Monitoring and Managing YARN Applications

YARN provides several tools for monitoring and managing applications:

  • ResourceManager Web UI: Accessible via http://<ResourceManager-host>:8088, provides information about running applications, cluster metrics, and resource usage.
  • Command-Line Tools: Commands like yarn application -list, yarn application -status <appId>, and yarn application -kill <appId> allow users to manage applications from the command line.

Example of Checking Application Status

yarn application -status application_1234567890123_0001

This command retrieves the status of the application with the ID application_1234567890123_0001.

Conclusion

YARN is a powerful resource management layer in Hadoop that enables efficient resource allocation and job scheduling. Understanding its architecture, components, and resource allocation mechanisms is crucial for effectively managing and running applications on a Hadoop cluster. By leveraging YARN's capabilities, users can optimize resource utilization and ensure fair and efficient processing of their data workloads.
