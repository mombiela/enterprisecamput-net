Creating and Configuring Jobs
Understanding Jenkins Pipelines ►

In this section, we will cover how to create and configure jobs in Jenkins. Jobs are the fundamental building blocks in Jenkins, allowing you to define tasks to be executed, such as building software, running tests, or deploying applications. By the end of this section, you will be able to create and configure jobs to automate your development workflow.

Creating a New Job

Creating a new job in Jenkins is straightforward. Follow these steps to create your first job:

  1. Open Jenkins Dashboard: Navigate to your Jenkins instance and log in.
  2. Create a New Item: Click on the "New Item" link on the left-hand side of the dashboard.
  3. Enter Job Name: Provide a unique name for your job.
  4. Select Job Type: Choose the type of job you want to create. For beginners, "Freestyle project" is a good starting point.
  5. Click OK: Click the "OK" button to proceed to the job configuration page.

Configuring a Job

Once you have created a job, you need to configure it to define what it should do. Here are the key sections you will encounter in the job configuration page:

General

  • Description: Provide a meaningful description of what the job does.
  • Discard Old Builds: Configure Jenkins to discard old builds to save disk space.
  • GitHub Project: If your project is hosted on GitHub, you can link it here.

Source Code Management

  • None: Select this if your job does not involve source code.
  • Git: Configure Git repository details if your job involves source code from a Git repository. 
    Repository URL: https://github.com/your-repo.git
Credentials: (Add credentials if required)
Branches to build: */main

Build Triggers

  • Build Periodically: Schedule the job to run at specific intervals using cron syntax. 
    H/15 * * * *
  • Poll SCM: Configure Jenkins to poll the source code repository for changes. 
    H/5 * * * *

Build Environment

  • Delete workspace before build starts: Ensure a clean workspace for each build.
  • Use secret text(s) or file(s): Manage sensitive information securely.

Build Steps

  • Execute Shell: Run shell commands as part of the build process. 
    echo "Building the project..."
./build.sh
  • Invoke Gradle Script: If you are using Gradle, you can invoke Gradle tasks. 
    Tasks: clean build

Post-build Actions

  • Archive the artifacts: Save build artifacts for future reference. 
    Files to archive: **/target/*.jar
  • Publish JUnit test result report: Publish test results if your build includes tests. 
    Test report XMLs: **/target/test-*.xml

Example: Creating a Simple Freestyle Job

Let's create a simple Freestyle job that prints "Hello, Jenkins!" to the console.

  1. Create a New Item: Name it "HelloJenkins" and select "Freestyle project".
  2. General: Add a description like "A simple job to print Hello, Jenkins!".
  3. Build Steps: Add an "Execute Shell" build step with the following command: 
    echo "Hello, Jenkins!"
  4. Save: Click "Save" to create the job.

Running the Job

  • Build Now: Go to the job page and click "Build Now".
  • Console Output: Click on the build number in the "Build History" and then "Console Output" to see the result.

Summary

In this section, we covered the basics of creating and configuring jobs in Jenkins. We learned how to create a new job, configure various aspects of the job, and run a simple Freestyle job. Understanding how to create and configure jobs is essential for automating your development workflow with Jenkins. As you progress, you can explore more advanced job types and configurations to suit your project's needs.
