Continuous Integration and Deployment

Continuous Integration (CI) and Continuous Deployment (CD) are essential practices in modern software development. They help ensure that your code is always in a deployable state and that new features and bug fixes can be delivered quickly and reliably. In this section, we'll explore how to set up CI/CD for a Flutter project, from beginner to advanced levels.

Introduction to CI/CD

  • Continuous Integration (CI): The practice of automatically integrating code changes from multiple contributors into a shared repository several times a day.
  • Continuous Deployment (CD): The practice of automatically deploying every change that passes all stages of your production pipeline to your users.

Setting Up CI for Flutter

Choosing a CI Service

There are several CI services available that support Flutter, including:

  • GitHub Actions
  • Travis CI
  • CircleCI
  • Bitrise

For this course, we'll use GitHub Actions as it is free for public repositories and integrates seamlessly with GitHub.

Basic CI Workflow with GitHub Actions

  1. Create a .github/workflows directory in your Flutter project.
  2. Add a new workflow file, e.g., ci.yml.
name: Flutter CI

on:
  push:
    branches:
      - main
  pull_request:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2
    - name: Set up Flutter
      uses: subosito/flutter-action@v1
      with:
        flutter-version: '2.5.0'
    - name: Install dependencies
      run: flutter pub get
    - name: Run tests
      run: flutter test

Explanation

  • name: The name of the workflow.
  • on: Specifies the events that trigger the workflow. Here, it's triggered on pushes and pull requests to the main branch.
  • jobs: Defines the jobs that will run as part of the workflow.
  • steps: The individual steps to be executed in the job.
    • actions/checkout@v2: Checks out the repository.
    • subosito/flutter-action@v1: Sets up Flutter.
    • flutter pub get: Installs dependencies.
    • flutter test: Runs the tests.

Setting Up CD for Flutter

Deploying to Firebase App Distribution

Firebase App Distribution is a great tool for distributing your Flutter apps to testers.

  1. Add Firebase to your Flutter project.
  2. Create a new workflow file, e.g., cd.yml.
name: Flutter CD

on:
  push:
    tags:
      - 'v*.*.*'

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2
    - name: Set up Flutter
      uses: subosito/flutter-action@v1
      with:
        flutter-version: '2.5.0'
    - name: Install dependencies
      run: flutter pub get
    - name: Build APK
      run: flutter build apk --release
    - name: Upload to Firebase App Distribution
      uses: wzieba/Firebase-Distribution-Github-Action@v1
      with:
        appId: ${{ secrets.FIREBASE_APP_ID }}
        token: ${{ secrets.FIREBASE_TOKEN }}
        groups: testers
        file: build/app/outputs/flutter-apk/app-release.apk

Explanation

  • on: The workflow is triggered on pushes to tags that follow the v*.*.* pattern (e.g., v1.0.0).
  • steps:
    • flutter build apk --release: Builds the release APK.
    • wzieba/Firebase-Distribution-Github-Action@v1: Uploads the APK to Firebase App Distribution.

Secrets Management

  • FIREBASE_APP_ID: The App ID for your Firebase project.
  • FIREBASE_TOKEN: The token for Firebase CLI authentication.

You can add these secrets in your GitHub repository settings under Settings > Secrets.

Advanced CI/CD Practices

Running Integration Tests

Integration tests can be run on real devices using services like Firebase Test Lab.

- name: Run integration tests
  run: flutter drive --target=test_driver/app.dart

Code Coverage

You can generate code coverage reports and upload them to services like Codecov.

- name: Generate code coverage
  run: flutter test --coverage
- name: Upload coverage to Codecov
  uses: codecov/codecov-action@v1
  with:
    file: coverage/lcov.info

Linting and Formatting

Ensure code quality by adding linting and formatting steps.

- name: Analyze code
  run: flutter analyze
- name: Format code
  run: flutter format --set-exit-if-changed .

Conclusion

Setting up CI/CD for your Flutter project can significantly improve your development workflow by automating repetitive tasks and ensuring that your code is always in a deployable state. By following the steps outlined in this section, you can set up a robust CI/CD pipeline that includes building, testing, and deploying your Flutter applications.

By mastering these practices, you'll be able to deliver high-quality apps faster and more efficiently, making you a more effective and productive developer.
