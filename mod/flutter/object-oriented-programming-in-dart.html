Introduction to Object-Oriented Programming (OOP)

Object-Oriented Programming (OOP) is a programming paradigm based on the concept of "objects", which can contain data and code: data in the form of fields (often known as attributes or properties), and code in the form of procedures (often known as methods).

Key Concepts of OOP

  • Classes and Objects
  • Inheritance
  • Polymorphism
  • Encapsulation
  • Abstraction

Classes and Objects in Dart

A class is a blueprint for creating objects. An object is an instance of a class.

Defining a Class

class Animal {
  String name;
  int age;

  // Constructor
  Animal(this.name, this.age);

  // Method
  void makeSound() {
    print('$name makes a sound.');
  }
}

Creating an Object

void main() {
  Animal cat = Animal('Whiskers', 2);
  cat.makeSound(); // Output: Whiskers makes a sound.
}

Inheritance

Inheritance allows a class to inherit properties and methods from another class.

Example of Inheritance

class Animal {
  String name;
  int age;

  Animal(this.name, this.age);

  void makeSound() {
    print('$name makes a sound.');
  }
}

class Dog extends Animal {
  String breed;

  Dog(String name, int age, this.breed) : super(name, age);

  void bark() {
    print('$name barks.');
  }
}

void main() {
  Dog dog = Dog('Buddy', 3, 'Golden Retriever');
  dog.makeSound(); // Output: Buddy makes a sound.
  dog.bark();      // Output: Buddy barks.
}

Polymorphism

Polymorphism allows methods to do different things based on the object it is acting upon.

Example of Polymorphism

class Animal {
  void makeSound() {
    print('Some generic animal sound');
  }
}

class Dog extends Animal {
  @override
  void makeSound() {
    print('Bark');
  }
}

class Cat extends Animal {
  @override
  void makeSound() {
    print('Meow');
  }
}

void main() {
  Animal myDog = Dog();
  Animal myCat = Cat();

  myDog.makeSound(); // Output: Bark
  myCat.makeSound(); // Output: Meow
}

Encapsulation

Encapsulation is the concept of wrapping data and methods that work on the data within one unit, e.g., a class in Dart.

Example of Encapsulation

class Person {
  String _name; // Private property
  int _age;    // Private property

  Person(this._name, this._age);

  // Getter for name
  String get name => _name;

  // Setter for name
  set name(String name) {
    _name = name;
  }

  // Getter for age
  int get age => _age;

  // Setter for age
  set age(int age) {
    if (age > 0) {
      _age = age;
    }
  }
}

void main() {
  Person person = Person('John', 30);
  print(person.name); // Output: John
  person.age = 35;
  print(person.age);  // Output: 35
}

Abstraction

Abstraction is the concept of hiding the complex implementation details and showing only the necessary features of an object.

Example of Abstraction

abstract class Shape {
  void draw(); // Abstract method
}

class Circle extends Shape {
  @override
  void draw() {
    print('Drawing a Circle');
  }
}

class Square extends Shape {
  @override
  void draw() {
    print('Drawing a Square');
  }
}

void main() {
  Shape circle = Circle();
  Shape square = Square();

  circle.draw(); // Output: Drawing a Circle
  square.draw(); // Output: Drawing a Square
}

Conclusion

Object-Oriented Programming in Dart is a powerful paradigm that helps in structuring code in a more modular, reusable, and maintainable way. By understanding and applying the principles of classes and objects, inheritance, polymorphism, encapsulation, and abstraction, you can create robust and scalable applications in Flutter.
