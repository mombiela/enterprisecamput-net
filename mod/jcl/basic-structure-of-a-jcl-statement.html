Introduction

Job Control Language (JCL) is a scripting language used on IBM mainframe operating systems to instruct the system on how to run a batch job or start a subsystem. Understanding the basic structure of a JCL statement is crucial for anyone looking to work with mainframe systems.

Key Concepts

  • JCL Statements: Instructions written in JCL to perform specific tasks.
  • Job: A unit of work that the system executes.
  • Steps: Individual tasks within a job.
  • Parameters: Instructions that define how the job or step should be executed.

Basic Structure of a JCL Statement

A JCL statement typically consists of three parts:

  1. Identifier: Specifies the type of statement.
  2. Name Field: An optional field that names the statement.
  3. Operation Field: Specifies the operation to be performed.
  4. Parameter Field: Contains parameters that control the operation.

Example of a JCL Statement

//JOBNAME  JOB  (ACCT),'PROGRAMMER',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID

Breakdown of the Example

  • Identifier: // indicates the start of a JCL statement.
  • Name Field: JOBNAME is the name of the job.
  • Operation Field: JOB specifies that this is a job statement.
  • Parameter Field: Contains parameters like (ACCT), 'PROGRAMMER', CLASS=A, MSGCLASS=X, and NOTIFY=&SYSUID.

Common JCL Statements

JOB Statement

Defines the start of a job and provides job-level parameters.

//MYJOB  JOB  (ACCT),'MY JOB',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID

EXEC Statement

Specifies the execution of a program or procedure.

//STEP1  EXEC  PGM=MYPROG

DD Statement

Defines the data sets to be used in the job step.

//INPUT  DD  DSN=MY.DATA.SET,DISP=SHR

Parameters in JCL Statements

JOB Statement Parameters

  • ACCT: Accounting information.
  • 'MY JOB': Job description.
  • CLASS: Job class.
  • MSGCLASS: Output class.
  • NOTIFY: User to be notified upon job completion.

EXEC Statement Parameters

  • PGM: Program name to execute.
  • PROC: Procedure name if using a cataloged or in-stream procedure.

DD Statement Parameters

  • DSN: Data set name.
  • DISP: Disposition of the data set (e.g., NEW, OLD, SHR).
  • UNIT: Device type.
  • SPACE: Space allocation.

Example Job with Multiple Steps

//MYJOB  JOB  (ACCT),'MY JOB',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID
//STEP1  EXEC PGM=MYPROG
//INPUT  DD  DSN=MY.DATA.SET,DISP=SHR
//STEP2  EXEC PGM=ANOTHERPGM
//OUTPUT DD  DSN=ANOTHER.DATA.SET,DISP=(NEW,CATLG,DELETE),
//            UNIT=SYSDA,SPACE=(CYL,(5,5),RLSE)

Explanation

  • STEP1 runs MYPROG using MY.DATA.SET as input.
  • STEP2 runs ANOTHERPGM and creates ANOTHER.DATA.SET with specific space allocation and disposition parameters.

Conclusion

Understanding the basic structure of JCL statements is essential for writing effective job scripts on IBM mainframe systems. Each statement serves a specific purpose, and knowing how to configure parameters correctly can significantly impact the execution and management of jobs. Practice writing simple JCL scripts to become more familiar with these concepts.
