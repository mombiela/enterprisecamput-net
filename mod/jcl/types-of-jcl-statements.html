Types of JCL Statements
JOB Statement ►

Job Control Language (JCL) is a scripting language used on IBM mainframe operating systems to instruct the system on how to run a batch job or start a subsystem. Understanding the different types of JCL statements is crucial for effectively managing and executing jobs. This section will cover the primary types of JCL statements, providing clear explanations, examples, and exercises to help you grasp their usage.

1. Job Statement

The JOB statement is the first statement in a JCL and it defines the start of a job. It provides information about the job, such as the job name, accounting information, and job processing options.

Key Concepts:

  • Job Name: Identifies the job.
  • Accounting Information: Used for billing and tracking.
  • Job Processing Options: Specifies parameters like priority, class, and notification.

Example:

//MYJOB JOB (ACCT),'MY JOB',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID
  • MYJOB: Job name.
  • (ACCT): Accounting information.
  • 'MY JOB': Job description.
  • CLASS=A: Job class.
  • MSGCLASS=X: Output message class.
  • NOTIFY=&SYSUID: Notify the user who submitted the job.

2. Exec Statement

The EXEC statement specifies the program or procedure to be executed. It can also pass parameters to the program.

Key Concepts:

  • Program Name: The name of the program to execute.
  • Procedure Name: The name of the procedure to execute.
  • Parameters: Optional parameters for the program or procedure.

Example:

//STEP1 EXEC PGM=MYPROG,PARM='INPUT1'
  • STEP1: Step name.
  • PGM=MYPROG: Program name.
  • PARM='INPUT1': Parameter passed to the program.

3. DD Statement

The Data Definition (DD) statement describes the data sets used in the job. It specifies the input and output files, their attributes, and how they should be handled.

Key Concepts:

  • Data Set Name: The name of the data set.
  • Disposition: Specifies the status of the data set (e.g., new, old, delete).
  • Device Type: Indicates the type of device (e.g., disk, tape).

Example:

//INPUT1 DD DSN=MY.DATA.SET,DISP=SHR
  • INPUT1: DD name.
  • DSN=MY.DATA.SET: Data set name.
  • DISP=SHR: Disposition (shared).

4. Comment Statement

Comment statements are used to include comments in the JCL. They are ignored by the system and are meant for documentation purposes.

Key Concepts:

  • Comment Indicator: Starts with //*.

Example:

//* This is a comment

5. PROC Statement

The PROC statement is used to define a procedure, which is a set of JCL statements that can be reused. Procedures help in reducing redundancy and simplifying job management.

Key Concepts:

  • Procedure Name: The name of the procedure.
  • Parameters: Optional parameters for the procedure.

Example:

//MYPROC PROC
//STEP1 EXEC PGM=MYPROG
//INPUT1 DD DSN=MY.DATA.SET,DISP=SHR
  • MYPROC: Procedure name.
  • STEP1: Step within the procedure.
  • PGM=MYPROG: Program name.
  • DSN=MY.DATA.SET: Data set name.
  • DISP=SHR: Disposition.

6. INCLUDE Statement

The INCLUDE statement is used to include a member of a partitioned data set (PDS) into the JCL. This is useful for including common JCL code segments.

Key Concepts:

  • Member Name: The name of the PDS member to include.

Example:

//INCLUDE MEMBER=MYCOMMON
  • MEMBER=MYCOMMON: Member name to include.

Summary

In this section, we covered the primary types of JCL statements:

  • JOB Statement: Defines the start of a job.
  • EXEC Statement: Specifies the program or procedure to execute.
  • DD Statement: Describes the data sets used in the job.
  • Comment Statement: Includes comments for documentation.
  • PROC Statement: Defines reusable procedures.
  • INCLUDE Statement: Includes common JCL code segments.

Understanding these statements is fundamental to writing and managing JCL scripts effectively. Practice writing and executing these statements to become proficient in JCL.
