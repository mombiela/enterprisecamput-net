Introduction

In Job Control Language (JCL), managing and organizing your job streams efficiently is crucial. Two important statements that help in this regard are JCLLIB and INCLUDE. This section will cover their purposes, usage, and provide examples to help you understand how to implement them in your JCL scripts.

JCLLIB Statement

The JCLLIB statement is used to specify one or more libraries that contain JCL procedures (PROCs) that you want to include in your job. This helps in modularizing your JCL scripts and reusing common procedures across multiple jobs.

Key Concepts

  • Purpose: To specify the libraries containing JCL procedures.
  • Syntax: //JCLLIB ORDER=(library1,library2,...)
  • Placement: Typically placed at the beginning of the JCL script, before any job steps that use the procedures.

Example

//MYJOB    JOB  (ACCT),'Example Job'
//JCLLIB   ORDER=(MY.PROCLIB, SYS1.PROCLIB)
//STEP1    EXEC  PROC=MYPROC

In this example:

  • MY.PROCLIB and SYS1.PROCLIB are the libraries specified in the JCLLIB statement.
  • MYPROC is a procedure located in one of these libraries.

INCLUDE Statement

The INCLUDE statement allows you to include a set of JCL statements from a member of a partitioned data set (PDS). This is useful for including common JCL code segments without duplicating them in multiple jobs.

Key Concepts

  • Purpose: To include predefined JCL code segments.
  • Syntax: //INCLUDE MEMBER=membername
  • Placement: Can be placed anywhere in the JCL script where you want to include the JCL code segment.

Example

//MYJOB    JOB  (ACCT),'Example Job'
//INCLUDE  MEMBER=COMMONJCL
//STEP1    EXEC  PGM=MYPROG

In this example:

  • COMMONJCL is a member of a PDS that contains common JCL statements.
  • The INCLUDE statement will insert the contents of COMMONJCL at that point in the job.

Combining JCLLIB and INCLUDE

You can use both JCLLIB and INCLUDE statements in the same JCL script to manage procedures and common JCL segments efficiently.

Example

//MYJOB    JOB  (ACCT),'Example Job'
//JCLLIB   ORDER=(MY.PROCLIB, SYS1.PROCLIB)
//INCLUDE  MEMBER=COMMONJCL
//STEP1    EXEC  PROC=MYPROC

In this example:

  • The JCLLIB statement specifies the libraries containing procedures.
  • The INCLUDE statement includes common JCL code from the COMMONJCL member.
  • STEP1 executes the MYPROC procedure from the specified libraries.

Conclusion

Understanding and using the JCLLIB and INCLUDE statements can greatly enhance the modularity and maintainability of your JCL scripts. By specifying libraries for procedures and including common JCL segments, you can avoid redundancy and make your job streams more organized and easier to manage. Practice using these statements in your JCL scripts to become proficient in managing complex job streams efficiently.
