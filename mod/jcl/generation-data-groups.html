Introduction to GDGs

Generation Data Groups (GDGs) are a method in JCL (Job Control Language) to manage datasets that are logically related and need to be processed in a specific sequence. GDGs are particularly useful for handling datasets that are created on a regular basis, such as daily logs or backups.

Key Concepts

  • Generation Data Group (GDG): A collection of datasets that are related and can be referenced by a common name.
  • Generation: Each dataset within a GDG is referred to as a generation and is identified by a generation number.
  • Base: The common name used to reference the entire GDG.
  • Relative Generation Number: A way to reference datasets within a GDG relative to the current generation.

Creating and Managing GDGs

Defining a GDG Base

To create a GDG, you first need to define a GDG base. This is done using the IDCAMS utility.

//DEFINEGDG EXEC PGM=IDCAMS
//SYSPRINT DD SYSOUT=*
//SYSIN    DD *
  DEFINE GDG (NAME(MY.GDG.BASE) LIMIT(5) SCRATCH)
/*
  • NAME: Specifies the name of the GDG base.
  • LIMIT: Specifies the maximum number of generations that can be maintained.
  • SCRATCH: Indicates that old generations should be deleted when the limit is exceeded.

Creating a New Generation

To create a new generation within a GDG, you use the GDG base name followed by (0).

//NEWGEN   EXEC PGM=IEFBR14
//NEWFILE  DD DSN=MY.GDG.BASE(+1),
//            DISP=(NEW,CATLG,DELETE),
//            SPACE=(TRK,(1,1)),
//            DCB=(LRECL=80,BLKSIZE=800,RECFM=FB)
  • MY.GDG.BASE(+1): Creates a new generation.
  • DISP: Specifies the dataset disposition.
  • SPACE: Allocates space for the dataset.
  • DCB: Defines the dataset control block parameters.

Accessing GDG Generations

You can access specific generations using relative generation numbers.

//ACCESSGEN EXEC PGM=IEFBR14
//OLDGEN   DD DSN=MY.GDG.BASE(-1),DISP=SHR
//CURGEN   DD DSN=MY.GDG.BASE(0),DISP=SHR
  • MY.GDG.BASE(-1): Refers to the previous generation.
  • MY.GDG.BASE(0): Refers to the current generation.

Advanced GDG Management

Rolling Off Generations

When the limit of generations is reached, the oldest generation is automatically deleted if the SCRATCH option is used. If NOSCRATCH is specified, the oldest generation is uncataloged but not deleted.

Using GDGs in Procedures

GDGs can be used in JCL procedures to automate the creation and processing of datasets.

//PROC1    PROC
//STEP1    EXEC PGM=MYPROG
//INPUT    DD DSN=MY.GDG.BASE(0),DISP=SHR
//OUTPUT   DD DSN=MY.GDG.BASE(+1),DISP=(NEW,CATLG,DELETE),
//            SPACE=(TRK,(1,1)),
//            DCB=(LRECL=80,BLKSIZE=800,RECFM=FB)
/*
//PROC1    PEND
  • PROC: Defines the start of a procedure.
  • PEND: Marks the end of the procedure.

Comparing GDG Structures

| Feature | Description | |------------------|-----------------------------------------------------------------------------| | Base Name | Common name for all generations. | | Generation | Individual dataset within the GDG. | | Relative Gen | Number used to reference datasets relative to the current generation. | | Limit | Maximum number of generations maintained. | | SCRATCH | Option to delete old generations when the limit is exceeded. | | NOSCRATCH | Option to uncatalog but not delete old generations when the limit is exceeded. |

Conclusion

Generation Data Groups (GDGs) in JCL provide a powerful mechanism for managing datasets that are created and processed in a sequence. By understanding how to define, create, and access GDGs, you can effectively manage datasets that need to be handled in a logical order. Advanced features like rolling off generations and using GDGs in procedures further enhance their utility in complex JCL workflows.
