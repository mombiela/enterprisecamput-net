Introduction

The JOB statement is the first statement in a JCL (Job Control Language) script. It provides the system with essential information about the job, such as the job name, accounting information, and various job-level parameters. Understanding the JOB statement is crucial for anyone working with JCL, as it sets the stage for the execution of the job.

Key Concepts

  • Job Name: A unique identifier for the job.
  • Accounting Information: Used for billing and tracking purposes.
  • Job Parameters: Control various aspects of job execution, such as priority, class, and notification.

Basic Structure of a JOB Statement

A basic JOB statement has the following syntax:

//JOBNAME JOB (accounting_info), 'programmer_name', CLASS=class, MSGCLASS=msgclass, MSGLEVEL=msglevel, NOTIFY=userid

Example

//MYJOB01 JOB (12345), 'John Doe', CLASS=A, MSGCLASS=X, MSGLEVEL=(1,1), NOTIFY=&SYSUID

Explanation

  • MYJOB01: The job name, which must be unique within the system.
  • (12345): Accounting information for billing purposes.
  • 'John Doe': The name of the programmer or job owner.
  • CLASS=A: Specifies the job class, which determines the job's priority and resource allocation.
  • MSGCLASS=X: Specifies the output class for job-related messages.
  • MSGLEVEL=(1,1): Controls the level of detail in the job's output messages.
  • NOTIFY=&SYSUID: Specifies the user to be notified upon job completion.

Detailed Parameters

CLASS

  • Purpose: Determines the priority and resource allocation for the job.
  • Values: Typically, single letters (A, B, C, etc.) or numbers (1, 2, 3, etc.).

MSGCLASS

  • Purpose: Specifies the output class for job-related messages.
  • Values: Single letters (A, B, C, etc.) or numbers (1, 2, 3, etc.).

MSGLEVEL

  • Purpose: Controls the level of detail in the job's output messages.
  • Values: A pair of numbers (x,y) where:
    • x: Controls the output of JCL statements.
    • y: Controls the output of system messages.

NOTIFY

  • Purpose: Specifies the user to be notified upon job completion.
  • Values: User ID or &SYSUID for the submitting user.

Advanced Usage

Conditional Execution

You can use conditional parameters to control the execution of jobs based on specific conditions.

Example

//MYJOB02 JOB (12345), 'Jane Doe', CLASS=B, MSGCLASS=Y, MSGLEVEL=(2,0), NOTIFY=&SYSUID, COND=(4,LT)

Explanation

  • COND=(4,LT): The job will only execute if the return code of the previous job step is less than 4.

Using JCLLIB

You can include a JCLLIB statement to specify a library of JCL procedures.

Example

//MYJOB03 JOB (12345), 'Alice Smith', CLASS=C, MSGCLASS=Z, MSGLEVEL=(1,1), NOTIFY=&SYSUID
//JCLLIB ORDER=(MY.PROCLIB)

Explanation

  • JCLLIB ORDER=(MY.PROCLIB): Specifies the library MY.PROCLIB to search for procedures.

Conclusion

The JOB statement is a fundamental component of JCL that provides essential information about the job to the system. By understanding its structure and parameters, you can effectively control job execution, prioritize tasks, and manage job outputs. Mastering the JOB statement is a crucial step in becoming proficient in JCL.
