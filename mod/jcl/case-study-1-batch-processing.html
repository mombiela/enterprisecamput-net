Introduction to Batch Processing

Batch processing is a method of executing a series of jobs in a program on a computer without manual intervention. In the context of JCL (Job Control Language), batch processing is a fundamental concept that allows for the automation of repetitive tasks, efficient resource management, and streamlined operations.

Key Concepts

  • Job: A unit of work that is submitted to the system.
  • Job Step: A single task within a job.
  • Dataset: A collection of data that is stored and managed by the system.
  • Procedure (PROC): A reusable set of JCL statements.

Understanding JCL for Batch Processing

JCL is used to instruct the mainframe operating system on how to execute batch jobs. It specifies the resources required, the sequence of job steps, and the datasets to be used.

Basic Structure of a JCL Job

A JCL job typically consists of the following components:

  1. JOB Statement: Defines the job and its attributes.
  2. EXEC Statement: Specifies the program or procedure to be executed.
  3. DD (Data Definition) Statements: Describe the datasets to be used.

Example of a Simple JCL Job

//MYJOB    JOB  (ACCT),'Example Job',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID
//STEP1    EXEC PGM=MYPROGRAM
//INPUT    DD   DSN=MY.INPUT.DATASET,DISP=SHR
//OUTPUT   DD   DSN=MY.OUTPUT.DATASET,DISP=(NEW,CATLG,DELETE),
//              SPACE=(CYL,(5,5),RLSE),UNIT=SYSDA

Explanation

  • JOB Statement: //MYJOB JOB (ACCT),'Example Job',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID
    • MYJOB: Job name.
    • (ACCT): Accounting information.
    • 'Example Job': Job description.
    • CLASS=A: Job class.
    • MSGCLASS=X: Message class.
    • NOTIFY=&SYSUID: Notify the user who submitted the job.
  • EXEC Statement: //STEP1 EXEC PGM=MYPROGRAM
    • STEP1: Step name.
    • PGM=MYPROGRAM: Program to be executed.
  • DD Statements:
    • //INPUT DD DSN=MY.INPUT.DATASET,DISP=SHR
      • INPUT: DD name.
      • DSN=MY.INPUT.DATASET: Dataset name.
      • DISP=SHR: Dataset disposition (shared).
    • //OUTPUT DD DSN=MY.OUTPUT.DATASET,DISP=(NEW,CATLG,DELETE),SPACE=(CYL,(5,5),RLSE),UNIT=SYSDA
      • OUTPUT: DD name.
      • DSN=MY.OUTPUT.DATASET: Dataset name.
      • DISP=(NEW,CATLG,DELETE): Dataset disposition (new, catalog, delete if unsuccessful).
      • SPACE=(CYL,(5,5),RLSE): Space allocation (5 cylinders primary, 5 cylinders secondary, release unused space).
      • UNIT=SYSDA: Device type.

Advanced Batch Processing Techniques

As you progress, you will encounter more complex scenarios that require advanced JCL techniques.

Using Procedures (PROCs)

Procedures allow you to define a set of JCL statements that can be reused across multiple jobs. This promotes code reuse and simplifies job management.

Example of a Procedure

//MYPROC   PROC
//STEP1    EXEC PGM=MYPROGRAM
//INPUT    DD   DSN=&INDSN,DISP=SHR
//OUTPUT   DD   DSN=&OUTDSN,DISP=(NEW,CATLG,DELETE),
//              SPACE=(CYL,(5,5),RLSE),UNIT=SYSDA

Invoking the Procedure

//MYJOB    JOB  (ACCT),'Job Using PROC',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID
//STEP1    EXEC MYPROC,INDSN=MY.INPUT.DATASET,OUTDSN=MY.OUTPUT.DATASET
  • Procedure Definition: //MYPROC PROC
    • Defines a procedure named MYPROC.
  • Procedure Invocation: //STEP1 EXEC MYPROC,INDSN=MY.INPUT.DATASET,OUTDSN=MY.OUTPUT.DATASET
    • Invokes the procedure MYPROC with parameters INDSN and OUTDSN.

Conditional Processing

JCL allows for conditional execution of job steps based on the return codes of previous steps.

Example of Conditional Processing

//MYJOB    JOB  (ACCT),'Conditional Job',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID
//STEP1    EXEC PGM=MYPROGRAM1
//STEP2    EXEC PGM=MYPROGRAM2,COND=(4,LT,STEP1)
//STEP3    EXEC PGM=MYPROGRAM3,COND=(0,EQ,STEP2)
  • COND Parameter: Used to specify conditions under which a step should be bypassed.
    • COND=(4,LT,STEP1): Bypass STEP2 if the return code of STEP1 is less than 4.
    • COND=(0,EQ,STEP2): Bypass STEP3 if the return code of STEP2 is equal to 0.

Conclusion

Batch processing in JCL is a powerful technique for automating and managing large-scale operations on mainframe systems. By understanding the basic structure of JCL jobs, using procedures for code reuse, and implementing conditional processing, you can create efficient and maintainable batch jobs. As you advance, these skills will enable you to handle more complex scenarios and optimize your batch processing workflows.
