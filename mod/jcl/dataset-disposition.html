Dataset Disposition
Dataset Parameters ►

In this section, we will cover the concept of Dataset Disposition in JCL (Job Control Language). Understanding how to manage datasets is crucial for efficient job execution and data handling in mainframe environments.

Key Concepts

  • Dataset: A dataset is a collection of data records stored on a disk or tape.
  • Disposition: Disposition refers to the status or state of a dataset before and after a job step executes.

DISP Parameter

The DISP parameter in JCL specifies the status of a dataset at the beginning and end of a job step. It is used in the DD (Data Definition) statement.

DISP Syntax

The DISP parameter has the following syntax:

DISP=(status, normal-termination, abnormal-termination)
  • status: Specifies the status of the dataset before the job step executes.
  • normal-termination: Specifies the action to be taken if the job step ends normally.
  • abnormal-termination: Specifies the action to be taken if the job step ends abnormally.

Status Values

  • NEW: Indicates that the dataset is to be created.
  • OLD: Indicates that the dataset already exists and is to be used.
  • MOD: Indicates that the dataset already exists and is to be appended.
  • SHR: Indicates that the dataset already exists and can be shared.

Termination Values

  • DELETE: Deletes the dataset.
  • KEEP: Keeps the dataset.
  • CATLG: Catalogs the dataset.
  • UNCATLG: Uncatalogs the dataset.

Example DISP Parameter

//MYDATA  DD  DSN=MY.DATASET, DISP=(NEW,CATLG,DELETE)

In this example:

  • The dataset MY.DATASET is created if it does not exist.
  • If the job step ends normally, the dataset is cataloged.
  • If the job step ends abnormally, the dataset is deleted.

Common DISP Scenarios

Creating a New Dataset

//NEWDS   DD  DSN=NEW.DATASET, DISP=(NEW,CATLG,DELETE), 
//            SPACE=(CYL,(5,1)), UNIT=SYSDA
  • NEW: Indicates the dataset is to be created.
  • CATLG: Catalogs the dataset upon normal termination.
  • DELETE: Deletes the dataset upon abnormal termination.

Using an Existing Dataset

//OLDDATA DD  DSN=EXISTING.DATASET, DISP=(OLD,KEEP,KEEP)
  • OLD: Indicates the dataset already exists.
  • KEEP: Keeps the dataset regardless of job step termination status.

Modifying an Existing Dataset

//MODDATA DD  DSN=EXISTING.DATASET, DISP=(MOD,CATLG,DELETE)
  • MOD: Indicates the dataset is to be appended.
  • CATLG: Catalogs the dataset upon normal termination.
  • DELETE: Deletes the dataset upon abnormal termination.

Sharing an Existing Dataset

//SHRDATA DD  DSN=EXISTING.DATASET, DISP=SHR
  • SHR: Indicates the dataset can be shared.

Summary

Understanding the DISP parameter is essential for managing datasets effectively in JCL. It allows you to control the status of datasets before and after job execution, ensuring that data is handled appropriately based on the job's success or failure. By mastering the DISP parameter, you can create, use, modify, and share datasets efficiently, which is a fundamental skill for any JCL programmer.
