Introduction

In this section, we will explore how to use system utilities in JCL (Job Control Language). System utilities are essential tools that help manage and manipulate datasets, perform system maintenance, and automate various tasks. Understanding these utilities is crucial for both beginner and advanced JCL users.

Key System Utilities

IEBGENER

IEBGENER is a utility used for copying datasets, creating backups, and printing datasets.

  • Purpose: Copying data from one dataset to another.
  • Common Use Cases:
    • Copying sequential datasets.
    • Creating backups of datasets.
    • Printing datasets.

Example

//COPYJOB  JOB (ACCT),'COPY DATASET',CLASS=A,MSGCLASS=A
//STEP1    EXEC PGM=IEBGENER
//SYSPRINT DD  SYSOUT=A
//SYSUT1   DD  DSN=INPUT.DATASET,DISP=SHR
//SYSUT2   DD  DSN=OUTPUT.DATASET,DISP=(NEW,CATLG,DELETE),
//             UNIT=SYSDA,SPACE=(TRK,(1,1)),DCB=*.SYSUT1
//SYSIN    DD  DUMMY
  • Explanation:
    • SYSUT1 specifies the input dataset.
    • SYSUT2 specifies the output dataset.
    • SYSIN is set to DUMMY because no control statements are needed.

IEFBR14

IEFBR14 is a simple utility program that does nothing but return control to the operating system. It is often used to allocate or delete datasets.

  • Purpose: Allocate or delete datasets.
  • Common Use Cases:
    • Allocating new datasets.
    • Deleting existing datasets.

Example

//ALLOCJOB JOB (ACCT),'ALLOCATE DATASET',CLASS=A,MSGCLASS=A
//STEP1    EXEC PGM=IEFBR14
//NEWDS    DD  DSN=NEW.DATASET,DISP=(NEW,CATLG,DELETE),
//             UNIT=SYSDA,SPACE=(TRK,(1,1)),DCB=(RECFM=FB,LRECL=80,BLKSIZE=800)
  • Explanation:
    • NEWDS specifies the new dataset to be allocated.
    • DISP=(NEW,CATLG,DELETE) indicates the dataset should be cataloged if the job completes successfully and deleted if it fails.

IDCAMS

IDCAMS (Integrated Data Cluster Access Method Services) is a versatile utility used for managing VSAM datasets and performing catalog-related tasks.

  • Purpose: Manage VSAM datasets and catalogs.
  • Common Use Cases:
    • Defining VSAM datasets.
    • Deleting VSAM datasets.
    • Listing catalog entries.

Example

//IDCAMSJOB JOB (ACCT),'IDCAMS EXAMPLE',CLASS=A,MSGCLASS=A
//STEP1    EXEC PGM=IDCAMS
//SYSPRINT DD  SYSOUT=A
//SYSIN    DD  *
  DELETE MY.VSAM.DATASET CLUSTER
  DEFINE CLUSTER (NAME(MY.VSAM.DATASET) -
                  INDEXED -
                  RECORDSIZE(80 80) -
                  KEYS(8 0) -
                  TRACKS(1 1) -
                  FREESPACE(3 3))
/*
  • Explanation:
    • DELETE command removes the specified VSAM dataset.
    • DEFINE CLUSTER command creates a new VSAM dataset with specified attributes.

SORT

SORT is a utility used to sort, merge, and copy datasets. It is highly flexible and can handle complex data manipulation tasks.

  • Purpose: Sort, merge, and copy datasets.
  • Common Use Cases:
    • Sorting datasets based on specified fields.
    • Merging multiple datasets.
    • Filtering and copying datasets.

Example

//SORTJOB  JOB (ACCT),'SORT EXAMPLE',CLASS=A,MSGCLASS=A
//STEP1    EXEC PGM=SORT
//SORTIN   DD  DSN=INPUT.DATASET,DISP=SHR
//SORTOUT  DD  DSN=OUTPUT.DATASET,DISP=(NEW,CATLG,DELETE),
//             UNIT=SYSDA,SPACE=(TRK,(1,1)),DCB=*.SORTIN
//SYSIN    DD  *
  SORT FIELDS=(1,8,CH,A)
/*
  • Explanation:
    • SORTIN specifies the input dataset.
    • SORTOUT specifies the output dataset.
    • SORT FIELDS=(1,8,CH,A) sorts the dataset based on the first 8 characters in ascending order.

Conclusion

Understanding and utilizing system utilities in JCL is essential for efficient dataset management and job automation. By mastering utilities like IEBGENER, IEFBR14, IDCAMS, and SORT, you can perform a wide range of tasks, from simple dataset copying to complex data manipulation. Practice using these utilities with different datasets and scenarios to become proficient in JCL.
