Introduction

In Job Control Language (JCL), procedures (PROCs) are reusable sets of JCL statements that can be included in multiple jobs. One of the powerful features of JCL is the ability to override parameters in these procedures, allowing for flexibility and customization without modifying the original PROC.

Key Concepts

  • Procedure (PROC): A set of JCL statements that can be reused across multiple jobs.
  • Overriding Parameters: The process of changing the values of parameters defined in a PROC when it is invoked in a job.

Defining a Procedure

A procedure is defined in a JCL library and can be invoked in a job. Here is an example of a simple PROC:

//MYPROC PROC
//STEP1  EXEC PGM=MYPROG
//DD1    DD   DSN=&DATASET,DISP=SHR
//DD2    DD   DSN=OUTPUT.FILE,DISP=(NEW,CATLG,DELETE)

In this example:

  • MYPROC is the name of the procedure.
  • STEP1 is a step within the procedure that executes the program MYPROG.
  • DD1 and DD2 are data definition (DD) statements. DD1 uses a symbolic parameter &DATASET.

Invoking a Procedure

To invoke the procedure in a job, you use the EXEC statement with the procedure name:

//MYJOB  JOB  (ACCT),'MY JOB'
//STEP1  EXEC MYPROC,DATASET=INPUT.FILE

In this example:

  • MYJOB is the job name.
  • STEP1 is the step that invokes MYPROC.
  • The symbolic parameter &DATASET is overridden with the value INPUT.FILE.

Overriding Parameters

You can override parameters within a procedure by specifying new values in the job that invokes the procedure. This allows you to customize the behavior of the procedure without modifying its definition.

Overriding DD Statements

You can override entire DD statements or specific parameters within a DD statement. Here are some examples:

Overriding Entire DD Statement

//MYJOB  JOB  (ACCT),'MY JOB'
//STEP1  EXEC MYPROC,DATASET=INPUT.FILE
//STEP1.DD2 DD DSN=NEW.OUTPUT.FILE,DISP=(NEW,CATLG,DELETE)

In this example:

  • The DD2 statement in MYPROC is completely overridden with a new dataset name NEW.OUTPUT.FILE.

Overriding Specific Parameters

//MYJOB  JOB  (ACCT),'MY JOB'
//STEP1  EXEC MYPROC,DATASET=INPUT.FILE
//STEP1.DD2 DISP=(MOD,CATLG,DELETE)

In this example:

  • Only the DISP parameter of DD2 is overridden, changing it to MOD.

Overriding EXEC Parameters

You can also override parameters in the EXEC statement of a procedure:

//MYJOB  JOB  (ACCT),'MY JOB'
//STEP1  EXEC MYPROC,DATASET=INPUT.FILE
//STEP1  EXEC PGM=NEWPROG

In this example:

  • The PGM parameter in the EXEC statement of STEP1 is overridden to execute NEWPROG instead of MYPROG.

Examples and Exercises

Example 1: Basic Parameter Override

//MYPROC PROC
//STEP1  EXEC PGM=MYPROG
//DD1    DD   DSN=&DATASET,DISP=SHR
//DD2    DD   DSN=OUTPUT.FILE,DISP=(NEW,CATLG,DELETE)

//MYJOB  JOB  (ACCT),'MY JOB'
//STEP1  EXEC MYPROC,DATASET=INPUT.FILE

Exercise 1: Override the DISP Parameter

Modify the job to override the DISP parameter of DD2 to MOD.

//MYJOB  JOB  (ACCT),'MY JOB'
//STEP1  EXEC MYPROC,DATASET=INPUT.FILE
//STEP1.DD2 DISP=(MOD,CATLG,DELETE)

Example 2: Overriding EXEC Parameters

//MYPROC PROC
//STEP1  EXEC PGM=MYPROG
//DD1    DD   DSN=&DATASET,DISP=SHR

//MYJOB  JOB  (ACCT),'MY JOB'
//STEP1  EXEC MYPROC,DATASET=INPUT.FILE
//STEP1  EXEC PGM=NEWPROG

Exercise 2: Override the Dataset Name

Modify the job to override the dataset name in DD1 to NEW.INPUT.FILE.

//MYJOB  JOB  (ACCT),'MY JOB'
//STEP1  EXEC MYPROC,DATASET=NEW.INPUT.FILE

Conclusion

Overriding parameters in procedures is a powerful feature in JCL that allows for flexibility and reusability. By understanding how to override DD statements and EXEC parameters, you can customize the behavior of procedures to meet the specific needs of your jobs without modifying the original procedure definitions. Practice these concepts with the provided examples and exercises to gain a deeper understanding of parameter overriding in JCL.
