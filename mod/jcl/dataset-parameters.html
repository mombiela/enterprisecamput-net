Introduction

In Job Control Language (JCL), datasets are crucial for managing and processing data. Understanding how to define and manipulate dataset parameters is essential for efficient job execution. This section will cover the basics of dataset parameters, their types, and how to use them in JCL.

Key Concepts

  • Dataset: A collection of data records stored on a disk or tape.
  • Dataset Parameters: Attributes that define the properties and behavior of datasets in JCL.

Types of Dataset Parameters

1. Dataset Name (DSN)

  • Definition: The unique identifier for a dataset.
  • Syntax: DSN=dataset.name
  • Example: 
    //MYJOB   JOB  ...
//STEP1   EXEC PGM=MYPROG
//DD1     DD   DSN=MY.DATASET,DISP=SHR

2. Disposition (DISP)

  • Definition: Specifies the status of the dataset and what to do with it after the job step completes.
  • Syntax: DISP=(status,normal-termination,abnormal-termination)
  • Status Values:
    • NEW: Create a new dataset.
    • OLD: Use an existing dataset.
    • SHR: Share an existing dataset.
    • MOD: Modify an existing dataset.
  • Example: 
    //MYJOB   JOB  ...
//STEP1   EXEC PGM=MYPROG
//DD1     DD   DSN=MY.DATASET,DISP=(NEW,CATLG,DELETE)

3. Space Allocation (SPACE)

  • Definition: Defines the amount of space to allocate for a dataset.
  • Syntax: SPACE=(unit,(primary,secondary,dir),RLSE)
  • Parameters:
    • unit: The unit of allocation (e.g., CYL, TRK, BLK).
    • primary: Primary space allocation.
    • secondary: Secondary space allocation.
    • dir: Number of directory blocks (for partitioned datasets).
    • RLSE: Release unused space.
  • Example: 
    //MYJOB   JOB  ...
//STEP1   EXEC PGM=MYPROG
//DD1     DD   DSN=MY.DATASET,DISP=(NEW,CATLG,DELETE),
//             SPACE=(CYL,(10,5),RLSE)

4. Data Control Block (DCB)

  • Definition: Specifies the physical characteristics of the dataset.
  • Syntax: DCB=(parameter1,parameter2,...)
  • Common Parameters:
    • RECFM: Record format (e.g., FB, VB).
    • LRECL: Logical record length.
    • BLKSIZE: Block size.
  • Example: 
    //MYJOB   JOB  ...
//STEP1   EXEC PGM=MYPROG
//DD1     DD   DSN=MY.DATASET,DISP=(NEW,CATLG,DELETE),
//             DCB=(RECFM=FB,LRECL=80,BLKSIZE=800)

5. Volume Serial (VOL)

  • Definition: Specifies the volume serial number where the dataset resides.
  • Syntax: VOL=SER=volume
  • Example: 
    //MYJOB   JOB  ...
//STEP1   EXEC PGM=MYPROG
//DD1     DD   DSN=MY.DATASET,DISP=SHR,
//             VOL=SER=123456

6. Unit (UNIT)

  • Definition: Specifies the device type or group where the dataset is stored.
  • Syntax: UNIT=device
  • Example: 
    //MYJOB   JOB  ...
//STEP1   EXEC PGM=MYPROG
//DD1     DD   DSN=MY.DATASET,DISP=SHR,
//             UNIT=SYSDA

Comparison of Dataset Parameters

| Parameter | Description | Syntax Example | |-----------|-------------|----------------| | DSN | Dataset Name | DSN=MY.DATASET | | DISP | Disposition | DISP=(NEW,CATLG,DELETE) | | SPACE | Space Allocation | SPACE=(CYL,(10,5),RLSE) | | DCB | Data Control Block | DCB=(RECFM=FB,LRECL=80,BLKSIZE=800) | | VOL | Volume Serial | VOL=SER=123456 | | UNIT | Unit Type | UNIT=SYSDA |

Conclusion

Understanding dataset parameters in JCL is fundamental for managing datasets effectively. By mastering these parameters, you can control how datasets are created, accessed, and managed throughout job execution. Practice using these parameters in various combinations to become proficient in JCL dataset management.
