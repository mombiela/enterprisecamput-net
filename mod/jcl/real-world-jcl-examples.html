Introduction

In this section, we will explore real-world examples of Job Control Language (JCL) to help you understand how JCL is used in practical scenarios. We will cover a range of examples from basic to advanced levels, providing clear explanations and code snippets to illustrate each concept.

Basic Examples

Example 1: Simple Job Submission

This example demonstrates how to submit a simple job that executes a program.

//SIMPLEJOB JOB (ACCT),'SIMPLE JOB',CLASS=A,MSGCLASS=A
//STEP1    EXEC PGM=IEFBR14
  • Explanation:
    • //SIMPLEJOB JOB (ACCT),'SIMPLE JOB',CLASS=A,MSGCLASS=A: This line defines the job card with job name SIMPLEJOB, accounting information (ACCT), a description 'SIMPLE JOB', and specifies the job class and message class.
    • //STEP1 EXEC PGM=IEFBR14: This line defines a step named STEP1 that executes the program IEFBR14, a utility program that does nothing but is often used for testing.

Example 2: Copying a Dataset

This example shows how to copy a dataset using the IEBGENER utility.

//COPYJOB  JOB (ACCT),'COPY DATASET',CLASS=A,MSGCLASS=A
//STEP1    EXEC PGM=IEBGENER
//SYSPRINT DD SYSOUT=A
//SYSUT1   DD DSN=SOURCE.DATASET,DISP=SHR
//SYSUT2   DD DSN=TARGET.DATASET,DISP=(NEW,CATLG,DELETE),
//            UNIT=SYSDA,SPACE=(TRK,(1,1)),DCB=(RECFM=FB,LRECL=80,BLKSIZE=800)
//SYSIN    DD DUMMY
  • Explanation:
    • //COPYJOB JOB (ACCT),'COPY DATASET',CLASS=A,MSGCLASS=A: Defines the job card.
    • //STEP1 EXEC PGM=IEBGENER: Executes the IEBGENER utility.
    • //SYSPRINT DD SYSOUT=A: Directs the utility's output to the system output.
    • //SYSUT1 DD DSN=SOURCE.DATASET,DISP=SHR: Specifies the source dataset.
    • //SYSUT2 DD DSN=TARGET.DATASET,DISP=(NEW,CATLG,DELETE),...: Specifies the target dataset with its attributes.
    • //SYSIN DD DUMMY: Indicates that no control statements are needed for IEBGENER.

Intermediate Examples

Example 3: Conditional Job Steps

This example demonstrates how to execute steps conditionally based on the return code of a previous step.

//CONDJOB  JOB (ACCT),'CONDITIONAL STEPS',CLASS=A,MSGCLASS=A
//STEP1    EXEC PGM=PROG1
//STEP2    EXEC PGM=PROG2,COND=(4,LT,STEP1)
//STEP3    EXEC PGM=PROG3,COND=(0,EQ,STEP2)
  • Explanation:
    • //CONDJOB JOB (ACCT),'CONDITIONAL STEPS',CLASS=A,MSGCLASS=A: Defines the job card.
    • //STEP1 EXEC PGM=PROG1: Executes PROG1.
    • //STEP2 EXEC PGM=PROG2,COND=(4,LT,STEP1): Executes PROG2 only if the return code from STEP1 is less than 4.
    • //STEP3 EXEC PGM=PROG3,COND=(0,EQ,STEP2): Executes PROG3 only if the return code from STEP2 is 0.

Example 4: Using Symbolic Parameters

This example shows how to use symbolic parameters to make JCL more flexible.

//SYMBOLIC JOB (ACCT),'SYMBOLIC PARAMS',CLASS=A,MSGCLASS=A
//SET VAR1='MYDATA'
//STEP1    EXEC PGM=IEFBR14
//DD1      DD DSN=&VAR1..DATASET,DISP=(NEW,CATLG,DELETE),
//            UNIT=SYSDA,SPACE=(TRK,(1,1)),DCB=(RECFM=FB,LRECL=80,BLKSIZE=800)
  • Explanation:
    • //SYMBOLIC JOB (ACCT),'SYMBOLIC PARAMS',CLASS=A,MSGCLASS=A: Defines the job card.
    • //SET VAR1='MYDATA': Defines a symbolic parameter VAR1.
    • //STEP1 EXEC PGM=IEFBR14: Executes the IEFBR14 program.
    • //DD1 DD DSN=&VAR1..DATASET,...: Uses the symbolic parameter VAR1 to define the dataset name.

Advanced Examples

Example 5: Using Procedures

This example demonstrates how to use procedures to simplify JCL.

//PROCJOB  JOB (ACCT),'USING PROCEDURES',CLASS=A,MSGCLASS=A
//         EXEC MYPROC
  • Explanation:
    • //PROCJOB JOB (ACCT),'USING PROCEDURES',CLASS=A,MSGCLASS=A: Defines the job card.
    • // EXEC MYPROC: Executes a predefined procedure MYPROC.

Example 6: Restarting a Job

This example shows how to restart a job from a specific step.

//RESTART JOB (ACCT),'RESTART JOB',CLASS=A,MSGCLASS=A,RESTART=STEP2
//STEP1    EXEC PGM=PROG1
//STEP2    EXEC PGM=PROG2
//STEP3    EXEC PGM=PROG3
  • Explanation:
    • //RESTART JOB (ACCT),'RESTART JOB',CLASS=A,MSGCLASS=A,RESTART=STEP2: Defines the job card with a restart parameter to start from STEP2.
    • //STEP1 EXEC PGM=PROG1: Executes PROG1.
    • //STEP2 EXEC PGM=PROG2: Executes PROG2.
    • //STEP3 EXEC PGM=PROG3: Executes PROG3.

Conclusion

In this section, we have covered a variety of real-world JCL examples ranging from basic to advanced levels. These examples illustrate how JCL can be used to submit jobs, copy datasets, execute conditional steps, use symbolic parameters, utilize procedures, and restart jobs. By understanding these examples, you will be better equipped to write and manage JCL in practical scenarios.
