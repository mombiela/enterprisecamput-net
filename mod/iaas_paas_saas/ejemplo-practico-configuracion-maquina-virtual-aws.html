Introduction

In this section, we will learn how to configure a virtual machine in Amazon Web Services (AWS), one of the most popular Infrastructure as a Service (IaaS) platforms. This practical exercise will allow you to understand the necessary steps to deploy and manage an EC2 (Elastic Compute Cloud) instance in AWS.

Prerequisites

Before starting, make sure you have the following:

  • An active AWS account.
  • Access to the AWS management console.
  • Basic knowledge of networks and operating systems.

Steps to Configure a Virtual Machine in AWS

1. Log in to the AWS Console

  1. Go to AWS Management Console.
  2. Log in with your AWS credentials.

2. Navigate to EC2

  1. In the AWS console, search for "EC2" in the search bar and select "EC2" under the "Compute" category.

3. Launch a New Instance

  1. Click on the "Launch Instance" button.
  2. Select an Amazon Machine Image (AMI):
    • Example: Choose "Amazon Linux 2 AMI (HVM), SSD Volume Type".
  3. Select the instance type:
    • Example: Choose "t2.micro" (free tier eligible).
  4. Configure the instance details:
    • Example: Use the default configuration for this exercise.
  5. Add storage:
    • Example: Use the default storage of 8 GB.
  6. Add tags (optional):
    • Example: Add a tag with the key "Name" and the value "MyFirstInstance".
  7. Configure the security group:
    • Example: Create a new security group with a rule that allows SSH traffic from your IP.

4. Review and Launch

  1. Review all configurations.
  2. Click on "Launch".
  3. Select an existing key pair or create a new one to access your instance.
    • Example: If you create a new one, download the .pem file and save it in a secure place.

5. Connect to the Instance

  1. Once the instance is in the "running" state, select the instance and click on "Connect".
  2. Follow the instructions to connect to your instance using SSH:
    • Example of SSH command: 
      ssh -i /path/to/your-key-pair.pem ec2-user@your-instance-public-dns

Code Example

Below is an example of how to connect to your EC2 instance using SSH from a terminal:

# Navigate to the directory where you saved your .pem file
cd /path/to/your-key-pair.pem

# Change the permissions of the .pem file so that only the owner can read it
chmod 400 your-key-pair.pem

# Connect to the EC2 instance
ssh -i your-key-pair.pem ec2-user@your-instance-public-dns

Conclusion

In this exercise, we have configured and launched a virtual machine in AWS using EC2. This process included selecting an AMI, configuring the instance, configuring storage and security, and finally, connecting to the instance via SSH. This knowledge is fundamental for any IT professional working with cloud services and wishing to leverage the IaaS capabilities provided by AWS.
