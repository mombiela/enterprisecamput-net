Practical Exercise: Configuring a Virtual Machine
Next ►

In this exercise, you will learn how to configure a virtual machine (VM) using a popular IaaS provider. We will use Amazon Web Services (AWS) for this example, but the steps are similar across other providers like Microsoft Azure and Google Cloud Platform.

Objectives

  • Understand the process of creating and configuring a virtual machine.
  • Learn how to select appropriate VM specifications.
  • Gain hands-on experience with AWS Management Console.

Steps to Configure a Virtual Machine

Step 1: Sign in to AWS Management Console

  1. Go to the AWS Management Console.
  2. Sign in with your AWS account credentials. If you do not have an account, you will need to create one.

Step 2: Navigate to EC2 Dashboard

  1. In the AWS Management Console, search for "EC2" in the search bar and select "EC2" from the dropdown.
  2. This will take you to the EC2 Dashboard.

Step 3: Launch an Instance

  1. Click on the "Launch Instance" button.
  2. You will be taken to the "Choose an Amazon Machine Image (AMI)" page.

Step 4: Choose an Amazon Machine Image (AMI)

  1. Select an AMI that suits your needs. For this exercise, we will use the "Amazon Linux 2 AMI (HVM), SSD Volume Type".
  2. Click "Select" next to the chosen AMI.

Step 5: Choose an Instance Type

  1. On the "Choose an Instance Type" page, select the instance type. For this exercise, we will use the "t2.micro" instance type, which is eligible for the free tier.
  2. Click "Next: Configure Instance Details".

Step 6: Configure Instance Details

  1. Leave the default settings as they are for this exercise.
  2. Click "Next: Add Storage".

Step 7: Add Storage

  1. The default storage configuration should be sufficient for this exercise.
  2. Click "Next: Add Tags".

Step 8: Add Tags

  1. Add a tag to your instance to help identify it. For example:
    • Key: Name
    • Value: MyFirstVM
  2. Click "Next: Configure Security Group".

Step 9: Configure Security Group

  1. Create a new security group with the following settings:
    • Security group name: MyFirstSecurityGroup
    • Description: Security group for my first VM
  2. Add a rule to allow SSH access:
    • Type: SSH
    • Protocol: TCP
    • Port Range: 22
    • Source: Anywhere (0.0.0.0/0)
  3. Click "Review and Launch".

Step 10: Review and Launch

  1. Review your instance configuration.
  2. Click "Launch".
  3. You will be prompted to select an existing key pair or create a new one. Select "Create a new key pair".
  4. Enter a key pair name (e.g., MyFirstKeyPair) and click "Download Key Pair". Save the key pair file (.pem) securely.
  5. Click "Launch Instances".

Step 11: Connect to Your Instance

  1. Once your instance is running, go back to the EC2 Dashboard and click on "Instances" in the left-hand menu.
  2. Select your instance and click "Connect".
  3. Follow the instructions to connect to your instance using SSH. You will need the .pem file you downloaded earlier.
ssh -i "MyFirstKeyPair.pem" ec2-user@<your-instance-public-dns>

Replace <your-instance-public-dns> with the public DNS of your instance, which you can find in the instance details.

Summary

In this exercise, you have successfully configured and launched a virtual machine using AWS EC2. You learned how to:

  • Sign in to the AWS Management Console.
  • Navigate to the EC2 Dashboard.
  • Launch an instance and configure its details.
  • Connect to the instance using SSH.

Common Mistakes and Tips

  • Incorrect Key Pair Permissions: Ensure your .pem file has the correct permissions. Use chmod 400 MyFirstKeyPair.pem to set the correct permissions.
  • Security Group Configuration: Make sure your security group allows SSH access from your IP address.
  • Instance Type Selection: Choose an instance type that fits your needs and budget. The free tier is a good starting point for learning.

By completing this exercise, you have gained practical experience with IaaS and are now ready to explore more advanced configurations and use cases.
