Reflection in C#
Dynamic Programming ►

Reflection is a powerful feature in C# that allows you to inspect and interact with the metadata of types at runtime. This can be particularly useful for tasks such as dynamic type creation, method invocation, and accessing attributes.

Key Concepts of Reflection

  • Metadata: Information about the types defined in a program, such as classes, interfaces, and methods.
  • Assembly: A compiled code library used by .NET applications. Reflection can be used to inspect the contents of an assembly.
  • Type: Represents type declarations: class types, interface types, array types, value types, enumeration types, type parameters, and pointer types.
  • MemberInfo: Provides information about the members of a class, such as methods, properties, fields, and events.

Using Reflection

Loading an Assembly

To use reflection, you often start by loading an assembly. This can be done using the Assembly class.

using System;
using System.Reflection;

class Program
{
    static void Main()
    {
        // Load the current assembly
        Assembly assembly = Assembly.GetExecutingAssembly();
        Console.WriteLine("Assembly Full Name: " + assembly.FullName);
    }
}

Getting Types from an Assembly

Once you have an assembly, you can get the types defined in it.

using System;
using System.Reflection;

class Program
{
    static void Main()
    {
        Assembly assembly = Assembly.GetExecutingAssembly();
        Type[] types = assembly.GetTypes();

        foreach (Type type in types)
        {
            Console.WriteLine("Type: " + type.Name);
        }
    }
}

Inspecting Type Members

You can inspect the members of a type, such as its methods, properties, and fields.

using System;
using System.Reflection;

class SampleClass
{
    public int SampleProperty { get; set; }
    public void SampleMethod() { }
}

class Program
{
    static void Main()
    {
        Type type = typeof(SampleClass);

        // Get properties
        PropertyInfo[] properties = type.GetProperties();
        foreach (PropertyInfo property in properties)
        {
            Console.WriteLine("Property: " + property.Name);
        }

        // Get methods
        MethodInfo[] methods = type.GetMethods();
        foreach (MethodInfo method in methods)
        {
            Console.WriteLine("Method: " + method.Name);
        }
    }
}

Invoking Methods Dynamically

Reflection allows you to invoke methods at runtime without knowing their names at compile time.

using System;
using System.Reflection;

class SampleClass
{
    public void SampleMethod()
    {
        Console.WriteLine("SampleMethod invoked!");
    }
}

class Program
{
    static void Main()
    {
        Type type = typeof(SampleClass);
        MethodInfo method = type.GetMethod("SampleMethod");

        if (method != null)
        {
            object instance = Activator.CreateInstance(type);
            method.Invoke(instance, null);
        }
    }
}

Working with Attributes

Reflection can be used to read custom attributes applied to types and members.

using System;
using System.Reflection;

[AttributeUsage(AttributeTargets.Class | AttributeTargets.Method)]
public class SampleAttribute : Attribute
{
    public string Description { get; set; }
}

[Sample(Description = "This is a sample class.")]
class SampleClass
{
    [Sample(Description = "This is a sample method.")]
    public void SampleMethod() { }
}

class Program
{
    static void Main()
    {
        Type type = typeof(SampleClass);
        object[] classAttributes = type.GetCustomAttributes(typeof(SampleAttribute), false);

        foreach (SampleAttribute attr in classAttributes)
        {
            Console.WriteLine("Class Attribute Description: " + attr.Description);
        }

        MethodInfo method = type.GetMethod("SampleMethod");
        object[] methodAttributes = method.GetCustomAttributes(typeof(SampleAttribute), false);

        foreach (SampleAttribute attr in methodAttributes)
        {
            Console.WriteLine("Method Attribute Description: " + attr.Description);
        }
    }
}

Conclusion

Reflection in C# is a versatile tool that allows you to inspect and manipulate the metadata of types at runtime. It can be used for a variety of tasks, including dynamic type creation, method invocation, and attribute inspection. While powerful, reflection should be used judiciously due to its potential impact on performance and security. Understanding the basics of reflection can greatly enhance your ability to write flexible and dynamic C# applications.
