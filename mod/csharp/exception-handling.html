Exception Handling in C#
Defining and Calling Methods ►

Exception handling is a crucial aspect of robust software development. It allows developers to manage and respond to runtime errors gracefully, ensuring that the application can handle unexpected situations without crashing.

Introduction to Exception Handling

  • Definition: Exception handling is a mechanism to handle runtime errors, allowing the program to continue its execution or terminate gracefully.
  • Importance: Helps in maintaining the normal flow of the application even when unexpected errors occur.

Basic Concepts

What is an Exception?

  • Exception: An event that occurs during the execution of a program that disrupts the normal flow of instructions.
  • Common Exceptions: NullReferenceException, IndexOutOfRangeException, InvalidOperationException, etc.

The try-catch Block

  • try Block: Contains the code that might throw an exception.
  • catch Block: Contains the code that handles the exception.
try
{
    // Code that may throw an exception
    int[] numbers = { 1, 2, 3 };
    Console.WriteLine(numbers[5]);
}
catch (IndexOutOfRangeException ex)
{
    // Code to handle the exception
    Console.WriteLine("An error occurred: " + ex.Message);
}

The finally Block

  • finally Block: Contains code that is always executed, regardless of whether an exception was thrown or not. It is typically used for cleanup activities.
try
{
    // Code that may throw an exception
    int[] numbers = { 1, 2, 3 };
    Console.WriteLine(numbers[5]);
}
catch (IndexOutOfRangeException ex)
{
    // Code to handle the exception
    Console.WriteLine("An error occurred: " + ex.Message);
}
finally
{
    // Code that will always execute
    Console.WriteLine("Execution completed.");
}

Advanced Concepts

Throwing Exceptions

  • throw Statement: Used to explicitly throw an exception.
public void CheckAge(int age)
{
    if (age < 18)
    {
        throw new ArgumentException("Age must be 18 or older.");
    }
}

Custom Exceptions

  • Custom Exception: Creating user-defined exceptions by extending the Exception class.
public class InvalidAgeException : Exception
{
    public InvalidAgeException(string message) : base(message)
    {
    }
}

public void CheckAge(int age)
{
    if (age < 18)
    {
        throw new InvalidAgeException("Age must be 18 or older.");
    }
}

Exception Propagation

  • Propagation: If an exception is not caught in the current method, it propagates to the calling method.
public void Method1()
{
    Method2();
}

public void Method2()
{
    throw new InvalidOperationException("An error occurred in Method2.");
}

try
{
    Method1();
}
catch (InvalidOperationException ex)
{
    Console.WriteLine("Caught an exception: " + ex.Message);
}

Exception Filtering

  • Filtering: Using multiple catch blocks to handle different types of exceptions.
try
{
    // Code that may throw multiple types of exceptions
    int[] numbers = { 1, 2, 3 };
    Console.WriteLine(numbers[5]);
}
catch (IndexOutOfRangeException ex)
{
    Console.WriteLine("Index out of range: " + ex.Message);
}
catch (Exception ex)
{
    Console.WriteLine("An error occurred: " + ex.Message);
}

Best Practices

  • Specific Catch Blocks: Catch specific exceptions rather than using a general catch block.
  • Avoid Empty Catch Blocks: Always handle exceptions or log them; do not leave catch blocks empty.
  • Use finally for Cleanup: Use the finally block to release resources like file handles, database connections, etc.
  • Custom Exceptions: Create custom exceptions for specific error scenarios to make the code more readable and maintainable.

Conclusion

Exception handling is a fundamental part of writing robust and reliable C# applications. By understanding and implementing proper exception handling techniques, developers can ensure their applications can gracefully handle unexpected errors and maintain a smooth user experience.

  • Key Takeaways:
    • Use try-catch blocks to handle exceptions.
    • Use finally blocks for cleanup.
    • Create and throw custom exceptions for specific error scenarios.
    • Follow best practices to write clean and maintainable exception handling code.

By mastering these concepts, you will be well-equipped to handle exceptions effectively in your C# applications.
