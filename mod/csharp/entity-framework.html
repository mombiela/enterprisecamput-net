Entity Framework (EF) is an open-source object-relational mapper (ORM) for ADO.NET, which allows developers to work with a database using .NET objects. It eliminates the need for most of the data-access code that developers usually need to write.

Introduction to Entity Framework

  • What is Entity Framework?

    • An ORM framework for .NET applications.
    • Simplifies data access by allowing developers to work with data as objects.
    • Supports multiple database providers (SQL Server, SQLite, MySQL, etc.).

  • Why use Entity Framework?

    • Reduces the amount of boilerplate code.
    • Provides a higher level of abstraction for database operations.
    • Supports LINQ queries, making data manipulation more intuitive.

Setting Up Entity Framework

  • Installing Entity Framework

    • Use NuGet Package Manager in Visual Studio.
    • Command: Install-Package EntityFramework

  • Creating a Model

    • Define your data model using classes.
    • Example: 
      public class Product
{
    public int ProductId { get; set; }
    public string Name { get; set; }
    public decimal Price { get; set; }
}

  • DbContext Class

    • Acts as a bridge between your model and the database.
    • Example: 
      public class ApplicationDbContext : DbContext
{
    public DbSet<Product> Products { get; set; }
}

Basic CRUD Operations

  • Create

    • Adding a new entity to the database.
    • Example: 
      using (var context = new ApplicationDbContext())
{
    var product = new Product { Name = "Laptop", Price = 999.99M };
    context.Products.Add(product);
    context.SaveChanges();
}

  • Read

    • Retrieving entities from the database.
    • Example: 
      using (var context = new ApplicationDbContext())
{
    var products = context.Products.ToList();
    foreach (var product in products)
    {
        Console.WriteLine($"Product: {product.Name}, Price: {product.Price}");
    }
}

  • Update

    • Modifying an existing entity.
    • Example: 
      using (var context = new ApplicationDbContext())
{
    var product = context.Products.First(p => p.ProductId == 1);
    product.Price = 899.99M;
    context.SaveChanges();
}

  • Delete

    • Removing an entity from the database.
    • Example: 
      using (var context = new ApplicationDbContext())
{
    var product = context.Products.First(p => p.ProductId == 1);
    context.Products.Remove(product);
    context.SaveChanges();
}

Advanced Features

  • Migrations

    • Manage database schema changes.
    • Commands:
      • Add Migration: Add-Migration MigrationName
      • Update Database: Update-Database

  • Relationships

    • Define relationships between entities (one-to-many, many-to-many).
    • Example: 
      public class Category
{
    public int CategoryId { get; set; }
    public string Name { get; set; }
    public ICollection<Product> Products { get; set; }
}

public class Product
{
    public int ProductId { get; set; }
    public string Name { get; set; }
    public decimal Price { get; set; }
    public int CategoryId { get; set; }
    public Category Category { get; set; }
}

  • LINQ Queries

    • Use LINQ to query the database.
    • Example: 
      using (var context = new ApplicationDbContext())
{
    var expensiveProducts = context.Products
                                   .Where(p => p.Price > 500)
                                   .ToList();
}

  • Asynchronous Operations

    • Perform database operations asynchronously.
    • Example: 
      using (var context = new ApplicationDbContext())
{
    var products = await context.Products.ToListAsync();
}

Performance Considerations

  • Lazy Loading vs. Eager Loading

    • Lazy Loading: Loads related data on demand.
    • Eager Loading: Loads related data as part of the initial query.
    • Example of Eager Loading: 
      using (var context = new ApplicationDbContext())
{
    var products = context.Products.Include(p => p.Category).ToList();
}

  • Tracking vs. No-Tracking Queries

    • Tracking: Keeps track of changes to entities.
    • No-Tracking: Better performance for read-only operations.
    • Example of No-Tracking Query: 
      using (var context = new ApplicationDbContext())
{
    var products = context.Products.AsNoTracking().ToList();
}

Conclusion

Entity Framework is a powerful ORM that simplifies data access in .NET applications. By understanding its basic and advanced features, you can efficiently manage your database operations with minimal code. Whether you're performing basic CRUD operations or optimizing performance with advanced techniques, Entity Framework provides the tools you need to build robust and scalable applications.
