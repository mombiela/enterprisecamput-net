Generics in C#

Generics are a powerful feature in C# that allows you to define classes, methods, and data structures with a placeholder for the type of data they store or use. This enables code reusability, type safety, and performance optimization.

Introduction to Generics

Generics allow you to write a class or method that can work with any data type. Instead of specifying a specific data type, you use a placeholder, which is replaced with the actual data type when the class or method is instantiated or called.

Key Concepts

  • Type Parameter: A placeholder for a data type, specified using angle brackets (<T>).
  • Generic Class: A class that can operate on any data type.
  • Generic Method: A method that can operate on any data type.
  • Type Safety: Ensures that the data type is consistent, reducing runtime errors.
  • Code Reusability: Allows the same code to work with different data types.

Generic Classes

A generic class is defined with a type parameter. Here’s an example of a simple generic class:

public class GenericClass<T>
{
    private T data;

    public GenericClass(T value)
    {
        data = value;
    }

    public T GetData()
    {
        return data;
    }
}

Example Usage

GenericClass<int> intInstance = new GenericClass<int>(10);
Console.WriteLine(intInstance.GetData()); // Output: 10

GenericClass<string> stringInstance = new GenericClass<string>("Hello");
Console.WriteLine(stringInstance.GetData()); // Output: Hello

Generic Methods

A generic method is defined with a type parameter. Here’s an example:

public class GenericMethods
{
    public void PrintData<T>(T data)
    {
        Console.WriteLine(data);
    }
}

Example Usage

GenericMethods gm = new GenericMethods();
gm.PrintData<int>(100); // Output: 100
gm.PrintData<string>("Hello World"); // Output: Hello World

Constraints on Type Parameters

You can impose constraints on the type parameters to restrict the types that can be used. Here are some common constraints:

  • where T : struct: T must be a value type.
  • where T : class: T must be a reference type.
  • where T : new(): T must have a parameterless constructor.
  • where T : BaseClass: T must inherit from BaseClass.
  • where T : Interface: T must implement the specified interface.

Example with Constraints

public class GenericClassWithConstraints<T> where T : class, new()
{
    private T data;

    public GenericClassWithConstraints()
    {
        data = new T();
    }

    public T GetData()
    {
        return data;
    }
}

Generic Collections

C# provides several generic collections in the System.Collections.Generic namespace, such as List<T>, Dictionary<TKey, TValue>, and Queue<T>.

Example with List

List<int> numbers = new List<int>();
numbers.Add(1);
numbers.Add(2);
numbers.Add(3);

foreach (int number in numbers)
{
    Console.WriteLine(number); // Output: 1 2 3
}

Example with Dictionary

Dictionary<int, string> dict = new Dictionary<int, string>();
dict.Add(1, "One");
dict.Add(2, "Two");

foreach (var kvp in dict)
{
    Console.WriteLine($"Key: {kvp.Key}, Value: {kvp.Value}");
    // Output: Key: 1, Value: One
    //         Key: 2, Value: Two
}

Advanced Generics

Covariance and Contravariance

Covariance and contravariance allow you to use a more derived type than originally specified (covariance) or a more generic type (contravariance).

  • Covariance: Enables you to use a derived type where a base type is expected.
  • Contravariance: Enables you to use a base type where a derived type is expected.

Example of Covariance

IEnumerable<string> strings = new List<string>();
IEnumerable<object> objects = strings; // Covariance

Example of Contravariance

Action<object> actObject = (obj) => Console.WriteLine(obj);
Action<string> actString = actObject; // Contravariance

Summary

Generics in C# provide a way to create flexible and reusable code components that can work with any data type while maintaining type safety. By understanding and utilizing generic classes, methods, and collections, you can write more efficient and maintainable code. Constraints on type parameters and advanced concepts like covariance and contravariance further enhance the power of generics, making them an essential tool in a C# programmer's toolkit.
