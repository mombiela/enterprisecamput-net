Introduction to Redux

Redux is a predictable state container for JavaScript applications, often used with React. It helps manage the state of your application in a more structured and predictable way.

Key Concepts

  • State: The single source of truth for your application.
  • Actions: Plain JavaScript objects that describe what happened.
  • Reducers: Functions that determine how the state changes in response to actions.
  • Store: The object that brings actions and reducers together, holding the application state.

Setting Up Redux

To start using Redux in a React application, you need to install the necessary packages.

npm install redux react-redux

Creating a Redux Store

The store is the central piece of a Redux application. It holds the state and allows it to be updated via actions.

import { createStore } from 'redux';

// A simple reducer
const initialState = { count: 0 };
const counterReducer = (state = initialState, action) => {
  switch (action.type) {
    case 'INCREMENT':
      return { count: state.count + 1 };
    case 'DECREMENT':
      return { count: state.count - 1 };
    default:
      return state;
  }
};

// Create the store
const store = createStore(counterReducer);

Connecting Redux to React

To connect Redux with React, you use the Provider component from react-redux.

import React from 'react';
import ReactDOM from 'react-dom';
import { Provider } from 'react-redux';
import App from './App'; // Your main App component

ReactDOM.render(
  <Provider store={store}>
    <App />
  </Provider>,
  document.getElementById('root')
);

Dispatching Actions

Actions are dispatched to the store to update the state.

// Action creators
const increment = () => ({ type: 'INCREMENT' });
const decrement = () => ({ type: 'DECREMENT' });

// Dispatching actions
store.dispatch(increment());
store.dispatch(decrement());

Using `connect` to Access State and Dispatch in Components

The connect function from react-redux is used to connect React components to the Redux store.

import React from 'react';
import { connect } from 'react-redux';

// A simple component
const Counter = ({ count, increment, decrement }) => (
  <div>
    <h1>{count}</h1>
    <button onClick={increment}>Increment</button>
    <button onClick={decrement}>Decrement</button>
  </div>
);

// Map state and dispatch to props
const mapStateToProps = state => ({ count: state.count });
const mapDispatchToProps = dispatch => ({
  increment: () => dispatch(increment()),
  decrement: () => dispatch(decrement())
});

// Connect the component to the Redux store
export default connect(mapStateToProps, mapDispatchToProps)(Counter);

Summary

Redux helps manage the state of your application in a predictable way. By understanding the core concepts—state, actions, reducers, and store—you can effectively use Redux in your React applications. Setting up Redux involves creating a store, connecting it to your React app using the Provider, dispatching actions to update the state, and using connect to access state and dispatch actions in your components. With these basics, you can start building more complex and scalable applications.
