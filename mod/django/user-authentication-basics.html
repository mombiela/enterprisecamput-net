Introduction

User authentication is a crucial aspect of web development, ensuring that users are who they claim to be. In Django, the framework provides robust tools to handle authentication efficiently. This section will guide you through the basics of user authentication in Django, from setting up to implementing advanced features.

Setting Up Django Authentication

Installing Django

Before diving into authentication, ensure you have Django installed. You can install it using pip:

pip install django

Creating a Django Project

Create a new Django project and navigate into it:

django-admin startproject myproject
cd myproject

Creating a Django App

Create a new app within your project to handle user authentication:

python manage.py startapp accounts

Adding the App to INSTALLED_APPS

Add the newly created app to the INSTALLED_APPS list in settings.py:

# myproject/settings.py

INSTALLED_APPS = [
    ...
    'accounts',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    ...
]

User Model and Authentication Views

Using Django's Built-in User Model

Django provides a built-in user model that you can use for authentication. You can import it as follows:

from django.contrib.auth.models import User

Creating User Registration View

Create a view to handle user registration in views.py:

# accounts/views.py

from django.shortcuts import render, redirect
from django.contrib.auth.forms import UserCreationForm

def register(request):
    if request.method == 'POST':
        form = UserCreationForm(request.POST)
        if form.is_valid():
            form.save()
            return redirect('login')
    else:
        form = UserCreationForm()
    return render(request, 'accounts/register.html', {'form': form})

Creating User Registration Template

Create a template for user registration in templates/accounts/register.html:

<!-- templates/accounts/register.html -->

<!DOCTYPE html>
<html>
<head>
    <title>Register</title>
</head>
<body>
    <h2>Register</h2>
    <form method="post">
        {% csrf_token %}
        {{ form.as_p }}
        <button type="submit">Register</button>
    </form>
</body>
</html>

Adding URL Patterns

Add URL patterns to route requests to the registration view:

# accounts/urls.py

from django.urls import path
from . import views

urlpatterns = [
    path('register/', views.register, name='register'),
]

Include the accounts URLs in the main project urls.py:

# myproject/urls.py

from django.contrib import admin
from django.urls import include, path

urlpatterns = [
    path('admin/', admin.site.urls),
    path('accounts/', include('accounts.urls')),
]

User Login and Logout

Creating Login View

Django provides built-in views for login and logout. You can use the LoginView and LogoutView from django.contrib.auth.views:

# accounts/urls.py

from django.contrib.auth import views as auth_views

urlpatterns = [
    ...
    path('login/', auth_views.LoginView.as_view(template_name='accounts/login.html'), name='login'),
    path('logout/', auth_views.LogoutView.as_view(), name='logout'),
]

Creating Login Template

Create a template for user login in templates/accounts/login.html:

<!-- templates/accounts/login.html -->

<!DOCTYPE html>
<html>
<head>
    <title>Login</title>
</head>
<body>
    <h2>Login</h2>
    <form method="post">
        {% csrf_token %}
        {{ form.as_p }}
        <button type="submit">Login</button>
    </form>
</body>
</html>

Protecting Views with Login Required

Using the Login Required Decorator

To restrict access to certain views, use the login_required decorator:

# accounts/views.py

from django.contrib.auth.decorators import login_required

@login_required
def profile(request):
    return render(request, 'accounts/profile.html')

Creating Profile Template

Create a template for the user profile in templates/accounts/profile.html:

<!-- templates/accounts/profile.html -->

<!DOCTYPE html>
<html>
<head>
    <title>Profile</title>
</head>
<body>
    <h2>Profile</h2>
    <p>Welcome, {{ user.username }}!</p>
</body>
</html>

Adding Profile URL Pattern

Add the URL pattern for the profile view:

# accounts/urls.py

urlpatterns = [
    ...
    path('profile/', views.profile, name='profile'),
]

Summary

In this section, you learned the basics of user authentication in Django. We covered:

  • Setting up a Django project and app.
  • Using Django's built-in user model.
  • Creating registration, login, and profile views.
  • Protecting views with the login_required decorator.

By understanding these fundamentals, you can build more secure and user-friendly web applications. In the next sections, we will delve into more advanced authentication topics, such as custom user models and third-party authentication.
