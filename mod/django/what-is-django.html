Introduction

Django is a high-level Python web framework that encourages rapid development and clean, pragmatic design. It was designed to help developers take applications from concept to completion as swiftly as possible. Django emphasizes reusability and "pluggability" of components, less code, low coupling, and the principle of "don't repeat yourself" (DRY).

Key Concepts

  • Web Framework: A collection of modules and tools that help developers build web applications.
  • High-Level Framework: Provides abstractions and tools to handle common web development tasks, allowing developers to focus on writing their application logic.
  • Python-Based: Written in Python, leveraging its simplicity and readability.

Features of Django

  • ORM (Object-Relational Mapping): Allows developers to interact with databases using Python code instead of SQL.
  • Admin Interface: Automatically generated admin panel for managing application data.
  • URL Routing: Maps URLs to views using a clean and readable syntax.
  • Template Engine: Facilitates the separation of design (HTML) from Python code.
  • Form Handling: Simplifies form creation, validation, and processing.
  • Security: Includes built-in protection against common web attacks like SQL injection, cross-site scripting (XSS), and cross-site request forgery (CSRF).
  • Scalability: Designed to handle high-traffic websites and applications.

Code Example: A Simple Django Application

Step 1: Setting Up Django

First, you need to install Django. You can do this using pip:

pip install django

Step 2: Creating a Django Project

Create a new Django project using the django-admin command:

django-admin startproject myproject

Step 3: Creating a Django App

Navigate into your project directory and create a new app:

cd myproject
python manage.py startapp myapp

Step 4: Defining a Model

In myapp/models.py, define a simple model:

from django.db import models

class Post(models.Model):
    title = models.CharField(max_length=100)
    content = models.TextField()
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.title

Step 5: Registering the Model in Admin

In myapp/admin.py, register the model to make it available in the admin interface:

from django.contrib import admin
from .models import Post

admin.site.register(Post)

Step 6: Creating a View

In myapp/views.py, create a simple view to display posts:

from django.shortcuts import render
from .models import Post

def post_list(request):
    posts = Post.objects.all()
    return render(request, 'post_list.html', {'posts': posts})

Step 7: Mapping the URL

In myapp/urls.py, map the URL to the view:

from django.urls import path
from .views import post_list

urlpatterns = [
    path('', post_list, name='post_list'),
]

Step 8: Creating a Template

In myapp/templates/post_list.html, create a simple template to display the posts:

<!DOCTYPE html>
<html>
<head>
    <title>Post List</title>
</head>
<body>
    <h1>Posts</h1>
    <ul>
        {% for post in posts %}
            <li>{{ post.title }} - {{ post.created_at }}</li>
        {% endfor %}
    </ul>
</body>
</html>

Step 9: Running the Server

Run the development server to see your application in action:

python manage.py runserver

Visit http://127.0.0.1:8000/ in your browser to see the list of posts.

Conclusion

Django is a powerful and flexible framework that simplifies the process of building web applications. Its built-in features like ORM, admin interface, and security mechanisms make it a great choice for both beginners and advanced developers. By following the steps outlined above, you can quickly set up a basic Django application and start exploring its capabilities. As you progress through this course, you'll learn more advanced features and best practices to build robust and scalable web applications with Django.
