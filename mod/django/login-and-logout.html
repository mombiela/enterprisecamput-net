Login and Logout in Django

Introduction

In this section, we will cover how to implement user login and logout functionality in a Django application. This includes setting up the necessary views, forms, and templates, as well as configuring Django's built-in authentication system.

Setting Up Authentication

Installing Django

Before we start, ensure you have Django installed. If not, you can install it using pip:

pip install django

Creating a Django Project and App

First, create a new Django project and an app within it:

django-admin startproject myproject
cd myproject
django-admin startapp myapp

Configuring the Project

Add the new app to the INSTALLED_APPS list in myproject/settings.py:

INSTALLED_APPS = [
    ...
    'myapp',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    ...
]

Implementing Login

Creating the Login View

Create a view for handling user login in myapp/views.py:

from django.shortcuts import render, redirect
from django.contrib.auth import authenticate, login
from django.contrib import messages

def login_view(request):
    if request.method == 'POST':
        username = request.POST['username']
        password = request.POST['password']
        user = authenticate(request, username=username, password=password)
        if user is not None:
            login(request, user)
            return redirect('home')
        else:
            messages.error(request, 'Invalid username or password')
    return render(request, 'myapp/login.html')

Creating the Login Template

Create a template for the login page in myapp/templates/myapp/login.html:

<!DOCTYPE html>
<html>
<head>
    <title>Login</title>
</head>
<body>
    <h2>Login</h2>
    {% if messages %}
        <ul>
            {% for message in messages %}
                <li>{{ message }}</li>
            {% endfor %}
        </ul>
    {% endif %}
    <form method="post">
        {% csrf_token %}
        <label for="username">Username:</label>
        <input type="text" name="username" id="username" required><br>
        <label for="password">Password:</label>
        <input type="password" name="password" id="password" required><br>
        <button type="submit">Login</button>
    </form>
</body>
</html>

Updating URLs

Add a URL pattern for the login view in myapp/urls.py:

from django.urls import path
from .views import login_view

urlpatterns = [
    path('login/', login_view, name='login'),
]

Implementing Logout

Creating the Logout View

Create a view for handling user logout in myapp/views.py:

from django.contrib.auth import logout

def logout_view(request):
    logout(request)
    return redirect('login')

Updating URLs

Add a URL pattern for the logout view in myapp/urls.py:

from .views import logout_view

urlpatterns = [
    ...
    path('logout/', logout_view, name='logout'),
]

Protecting Views with Login Required

To ensure certain views are only accessible to logged-in users, use the login_required decorator. For example, in myapp/views.py:

from django.contrib.auth.decorators import login_required

@login_required
def home_view(request):
    return render(request, 'myapp/home.html')

Update the URL pattern for the home view in myapp/urls.py:

from .views import home_view

urlpatterns = [
    ...
    path('', home_view, name='home'),
]

Conclusion

In this section, we covered the basics of implementing login and logout functionality in a Django application. We created views, templates, and URL patterns to handle user authentication. Additionally, we learned how to protect views using the login_required decorator. With these fundamentals, you can now manage user sessions and secure parts of your application effectively.
