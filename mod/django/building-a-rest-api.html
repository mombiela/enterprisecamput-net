Building a REST API with Django
Code Organization and Best Practices ►

Introduction

In this module, we will learn how to build a REST API using Django. We will cover the basics of RESTful principles, setting up Django, and creating endpoints. By the end of this module, you will be able to create a fully functional REST API.

Prerequisites

Before starting, ensure you have:

  • Basic understanding of Python and Django.
  • Python installed on your machine.
  • Django installed (pip install django).

Setting Up the Project

  1. Create a Django Project:

    django-admin startproject myapi
cd myapi

  2. Create a Django App:

    python manage.py startapp api

  3. Add the App to INSTALLED_APPS: Edit myapi/settings.py:

    INSTALLED_APPS = [
    ...
    'api',
    'rest_framework',  # Add Django REST framework
]

Creating Models

Define the data structure for your API using Django models.

  1. Create a Model: Edit api/models.py:

    from django.db import models

class Item(models.Model):
    name = models.CharField(max_length=100)
    description = models.TextField()
    price = models.DecimalField(max_digits=10, decimal_places=2)
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.name

  2. Apply Migrations:

    python manage.py makemigrations
python manage.py migrate

Serializers

Serializers convert complex data types, such as querysets and model instances, into native Python datatypes that can then be easily rendered into JSON or XML.

  1. Create a Serializer: Edit api/serializers.py: 
    from rest_framework import serializers
from .models import Item

class ItemSerializer(serializers.ModelSerializer):
    class Meta:
        model = Item
        fields = '__all__'

Views

Views handle the logic of your API endpoints.

  1. Create Views: Edit api/views.py: 
    from rest_framework import generics
from .models import Item
from .serializers import ItemSerializer

class ItemListCreate(generics.ListCreateAPIView):
    queryset = Item.objects.all()
    serializer_class = ItemSerializer

class ItemDetail(generics.RetrieveUpdateDestroyAPIView):
    queryset = Item.objects.all()
    serializer_class = ItemSerializer

URLs

Define the URL patterns for your API endpoints.

  1. Create URL Patterns: Edit api/urls.py:

    from django.urls import path
from .views import ItemListCreate, ItemDetail

urlpatterns = [
    path('items/', ItemListCreate.as_view(), name='item-list-create'),
    path('items/<int:pk>/', ItemDetail.as_view(), name='item-detail'),
]

  2. Include API URLs in Project URLs: Edit myapi/urls.py:

    from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('api/', include('api.urls')),
]

Testing the API

Use tools like Postman or curl to test your API endpoints.

  1. List Items:

    curl -X GET http://127.0.0.1:8000/api/items/

  2. Create an Item:

    curl -X POST http://127.0.0.1:8000/api/items/ -d '{"name": "Item1", "description": "Description1", "price": "10.00"}' -H "Content-Type: application/json"

  3. Retrieve an Item:

    curl -X GET http://127.0.0.1:8000/api/items/1/

  4. Update an Item:

    curl -X PUT http://127.0.0.1:8000/api/items/1/ -d '{"name": "Updated Item", "description": "Updated Description", "price": "20.00"}' -H "Content-Type: application/json"

  5. Delete an Item:

    curl -X DELETE http://127.0.0.1:8000/api/items/1/

Conclusion

In this module, we covered the basics of building a REST API using Django. We went through setting up a Django project, creating models, serializers, views, and defining URL patterns. We also tested our API using curl commands. With these fundamentals, you can now expand your API to include more complex features and handle various types of data.
