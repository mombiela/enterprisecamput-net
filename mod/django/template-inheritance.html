Template Inheritance in Django
Context and Template Tags ►

Template inheritance is a powerful feature in Django that allows you to create a base template and extend it to create more specific templates. This helps in maintaining a consistent look and feel across your website and makes it easier to manage and update your templates.

Key Concepts

  • Base Template: A general template that contains common elements like headers, footers, and navigation bars.
  • Child Template: A more specific template that extends the base template and overrides certain sections.
  • Blocks: Sections in the base template that can be overridden by child templates.

Creating a Base Template

The base template is the foundation of your template inheritance structure. It contains the common layout and elements that will be shared across multiple pages.

Example: `base.html`

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>{% block title %}My Website{% endblock %}</title>
</head>
<body>
    <header>
        <h1>Welcome to My Website</h1>
        <nav>
            <ul>
                <li><a href="/">Home</a></li>
                <li><a href="/about/">About</a></li>
                <li><a href="/contact/">Contact</a></li>
            </ul>
        </nav>
    </header>
    <main>
        {% block content %}
        <!-- Default content goes here -->
        {% endblock %}
    </main>
    <footer>
        <p>&copy; 2023 My Website</p>
    </footer>
</body>
</html>

In this example:

  • {% block title %} and {% block content %} are placeholders that can be overridden by child templates.

Creating a Child Template

A child template extends the base template and provides specific content for the placeholders defined in the base template.

Example: `home.html`

{% extends "base.html" %}

{% block title %}Home - My Website{% endblock %}

{% block content %}
    <h2>Home Page</h2>
    <p>Welcome to the home page of my website.</p>
{% endblock %}

In this example:

  • {% extends "base.html" %} tells Django that this template should use base.html as its base.
  • {% block title %} and {% block content %} override the corresponding blocks in the base template.

Using Template Inheritance in Views

To render a template that uses inheritance, you simply specify the child template in your view.

Example: `views.py`

from django.shortcuts import render

def home(request):
    return render(request, 'home.html')

In this example:

  • The home view renders the home.html template, which in turn extends base.html.

Advanced Usage

Nested Blocks

You can nest blocks within other blocks to create more complex templates.

Example: Nested Blocks

{% extends "base.html" %}

{% block content %}
    <h2>Home Page</h2>
    <p>Welcome to the home page of my website.</p>
    {% block extra_content %}
    <!-- Additional content can be added here by child templates -->
    {% endblock %}
{% endblock %}

Example: Overriding Nested Blocks

{% extends "home.html" %}

{% block extra_content %}
    <p>This is some extra content added to the home page.</p>
{% endblock %}

Conclusion

Template inheritance in Django is a powerful tool for creating maintainable and scalable web applications. By using base templates and child templates, you can ensure a consistent look and feel across your site while making it easy to update and manage your templates. Understanding and utilizing template inheritance will significantly improve your efficiency and the quality of your Django projects.
