Introduction

In Django, templates are used to generate dynamic HTML content. Templates can include variables and logic to display data from your views. This section will cover how to pass context to templates and how to use template tags to add logic to your templates.

Context in Django Templates

Context is a dictionary that maps variable names to Python objects. This dictionary is passed to the template engine to render the template with dynamic data.

Passing Context from Views

To pass context to a template, you need to include it in the render function in your view.

from django.shortcuts import render

def my_view(request):
    context = {
        'name': 'John Doe',
        'age': 30,
        'hobbies': ['Reading', 'Cycling', 'Hiking']
    }
    return render(request, 'my_template.html', context)

In the example above, the context dictionary contains three keys: name, age, and hobbies. These can be accessed in the template.

Accessing Context in Templates

In your template, you can access the context variables using double curly braces {{ }}.

<!DOCTYPE html>
<html>
<head>
    <title>My Template</title>
</head>
<body>
    <h1>Hello, {{ name }}!</h1>
    <p>You are {{ age }} years old.</p>
    <h2>Your Hobbies:</h2>
    <ul>
        {% for hobby in hobbies %}
            <li>{{ hobby }}</li>
        {% endfor %}
    </ul>
</body>
</html>

Template Tags

Template tags provide logic in the template. They are enclosed in {% %} and can be used for loops, conditionals, and more.

Common Template Tags

Here are some commonly used template tags:

  • {% for %}: Loop over a sequence.
  • {% if %}: Conditional statement.
  • {% block %}: Define a block that can be overridden in child templates.
  • {% extends %}: Extend a base template.
  • {% include %}: Include another template.

Examples of Template Tags

Looping with {% for %}

<ul>
    {% for item in item_list %}
        <li>{{ item }}</li>
    {% endfor %}
</ul>

Conditional Statements with {% if %}

{% if user.is_authenticated %}
    <p>Welcome, {{ user.username }}!</p>
{% else %}
    <p>Please log in.</p>
{% endif %}

Extending Templates with {% extends %} and {% block %}

<!-- base.html -->
<!DOCTYPE html>
<html>
<head>
    <title>{% block title %}My Site{% endblock %}</title>
</head>
<body>
    <header>
        <h1>My Site</h1>
    </header>
    <main>
        {% block content %}{% endblock %}
    </main>
</body>
</html>
<!-- child.html -->
{% extends "base.html" %}

{% block title %}Home{% endblock %}

{% block content %}
    <p>Welcome to the homepage!</p>
{% endblock %}

Including Templates with {% include %}

<!-- sidebar.html -->
<div class="sidebar">
    <h2>Sidebar</h2>
    <p>This is the sidebar content.</p>
</div>
<!-- main.html -->
<!DOCTYPE html>
<html>
<head>
    <title>Main Page</title>
</head>
<body>
    <h1>Main Content</h1>
    {% include "sidebar.html" %}
</body>
</html>

Conclusion

Understanding how to pass context and use template tags is crucial for creating dynamic and reusable templates in Django. By mastering these concepts, you can build more efficient and maintainable web applications. Practice by creating different views and templates, experimenting with various context data, and using template tags to control the rendering logic.
