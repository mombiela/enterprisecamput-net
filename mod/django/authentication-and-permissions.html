Introduction

In this section, we will explore how Django handles authentication and permissions. Authentication is the process of verifying the identity of a user, while permissions determine what an authenticated user is allowed to do. We will cover the following topics:

  • Django's built-in authentication system
  • User model and user management
  • Permissions and authorization
  • Customizing authentication and permissions

Django's Built-in Authentication System

Django provides a robust and flexible authentication system out of the box. This includes user login, logout, password management, and more.

Key Concepts

  • User Model: Represents a user in the system.
  • Authentication: Verifying the identity of a user.
  • Authorization: Determining what an authenticated user is allowed to do.
  • Sessions: Mechanism to persist user state across requests.

Basic Setup

To use Django's authentication system, ensure django.contrib.auth and django.contrib.contenttypes are included in your INSTALLED_APPS setting.

# settings.py
INSTALLED_APPS = [
    ...
    'django.contrib.auth',
    'django.contrib.contenttypes',
    ...
]

User Model and User Management

Creating and Managing Users

Django provides a default User model which can be used to create and manage users.

Creating a User

You can create a user using the Django shell or within your views.

# Using Django shell
from django.contrib.auth.models import User

user = User.objects.create_user('john', 'john@example.com', 'password123')
user.save()

Authenticating a User

To authenticate a user, use the authenticate function and then log them in using the login function.

from django.contrib.auth import authenticate, login

def my_view(request):
    username = request.POST['username']
    password = request.POST['password']
    user = authenticate(request, username=username, password=password)
    if user is not None:
        login(request, user)
        # Redirect to a success page.
    else:
        # Return an 'invalid login' error message.

User Management in Admin

Django's admin interface provides a convenient way to manage users. Ensure you have created a superuser to access the admin interface.

python manage.py createsuperuser

Permissions and Authorization

Built-in Permissions

Django provides a simple way to manage permissions using the User and Group models.

Default Permissions

Each model in Django automatically gets three permissions: add, change, and delete.

# Checking permissions
user.has_perm('app_label.add_modelname')
user.has_perm('app_label.change_modelname')
user.has_perm('app_label.delete_modelname')

Custom Permissions

You can define custom permissions in your models.

# models.py
from django.db import models

class MyModel(models.Model):
    ...
    class Meta:
        permissions = [
            ("can_publish", "Can publish posts"),
        ]

Using Permissions in Views

You can use the permission_required decorator to restrict access to views based on permissions.

from django.contrib.auth.decorators import permission_required

@permission_required('app_label.can_publish')
def my_view(request):
    ...

Customizing Authentication and Permissions

Custom User Model

For more complex user requirements, you can create a custom user model.

# models.py
from django.contrib.auth.models import AbstractUser

class CustomUser(AbstractUser):
    # Add additional fields here
    bio = models.TextField(blank=True, null=True)

Custom Authentication Backends

You can create custom authentication backends to implement custom authentication logic.

# backends.py
from django.contrib.auth.backends import BaseBackend
from django.contrib.auth.models import User

class EmailBackend(BaseBackend):
    def authenticate(self, request, username=None, password=None, **kwargs):
        try:
            user = User.objects.get(email=username)
            if user.check_password(password):
                return user
        except User.DoesNotExist:
            return None

    def get_user(self, user_id):
        try:
            return User.objects.get(pk=user_id)
        except User.DoesNotExist:
            return None

Using Custom Backends

Add your custom backend to the AUTHENTICATION_BACKENDS setting.

# settings.py
AUTHENTICATION_BACKENDS = [
    'django.contrib.auth.backends.ModelBackend',  # default
    'path.to.EmailBackend',
]

Conclusion

Django's authentication and permissions system is powerful and flexible, allowing you to manage user authentication and authorization with ease. By understanding the built-in features and learning how to customize them, you can implement robust security measures in your Django applications.
