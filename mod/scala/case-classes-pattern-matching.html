Introduction

In this section, we will explore two powerful features of Scala: Case Classes and Pattern Matching. These features are essential for writing concise and readable code, especially when dealing with immutable data structures and complex data manipulations.

Case Classes

What are Case Classes?

Case Classes are a special type of class in Scala that are optimized for use in pattern matching. They are immutable by default and come with several built-in features that make them very convenient to use.

Key Features of Case Classes

  • Immutability: Fields are immutable by default.
  • Automatic toString, equals, and hashCode: These methods are automatically generated.
  • Pattern Matching: Case classes can be decomposed using pattern matching.
  • Copy Method: Provides a convenient way to create modified copies of instances.
  • Default apply Method: Allows instantiation without the new keyword.

Defining a Case Class

case class Person(name: String, age: Int)

In this example, Person is a case class with two fields: name and age.

Using Case Classes

val alice = Person("Alice", 25)
val bob = Person("Bob", 30)

// Accessing fields
println(alice.name)  // Output: Alice
println(bob.age)     // Output: 30

// Copying with modifications
val olderAlice = alice.copy(age = 26)
println(olderAlice)  // Output: Person(Alice,26)

Pattern Matching

What is Pattern Matching?

Pattern Matching is a mechanism for checking a value against a pattern. It is a more powerful version of the switch statement found in many other languages. It allows for decomposing data structures and executing code based on their shape and content.

Basic Syntax

value match {
  case pattern1 => result1
  case pattern2 => result2
  // more cases
  case _ => defaultResult
}

Example with Case Classes

def greet(person: Person): String = person match {
  case Person("Alice", _) => "Hello, Alice!"
  case Person(_, age) if age < 18 => "Hi there, young one!"
  case Person(name, age) => s"Hello, $name! You are $age years old."
}

val alice = Person("Alice", 25)
val bob = Person("Bob", 17)
val charlie = Person("Charlie", 30)

println(greet(alice))    // Output: Hello, Alice!
println(greet(bob))      // Output: Hi there, young one!
println(greet(charlie))  // Output: Hello, Charlie! You are 30 years old.

Pattern Matching with Sealed Traits

Sealed traits ensure that all implementations are known at compile time, which makes pattern matching exhaustive and safe.

sealed trait Shape
case class Circle(radius: Double) extends Shape
case class Rectangle(width: Double, height: Double) extends Shape

def area(shape: Shape): Double = shape match {
  case Circle(radius) => Math.PI * radius * radius
  case Rectangle(width, height) => width * height
}

val circle = Circle(5.0)
val rectangle = Rectangle(4.0, 6.0)

println(area(circle))      // Output: 78.53981633974483
println(area(rectangle))   // Output: 24.0

Conclusion

Case Classes and Pattern Matching are fundamental features in Scala that enable concise and expressive code. Case Classes provide a convenient way to define immutable data structures, while Pattern Matching allows for elegant data decomposition and control flow. By mastering these features, you can write more readable and maintainable Scala code.
