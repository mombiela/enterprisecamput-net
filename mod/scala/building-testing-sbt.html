Introduction to SBT

SBT (Simple Build Tool) is a popular build tool for Scala projects. It is used to compile, run, and test Scala code, as well as manage dependencies and automate tasks.

Key Concepts

  • Build Definition: The build definition is written in a file named build.sbt.
  • Tasks: Actions that SBT can perform, such as compiling code or running tests.
  • Settings: Configuration options for your build, such as the Scala version or library dependencies.
  • Plugins: Extensions that add functionality to SBT.

Setting Up SBT

To get started with SBT, you need to install it and set up a basic project structure.

Installation

  1. Download and Install SBT: Follow the instructions on the official SBT website.
  2. Verify Installation: Open a terminal and run: 
    sbt sbtVersion
    This should print the installed SBT version.

Creating a New Project

  1. Create Project Directory: 
    mkdir my-scala-project
cd my-scala-project
  2. Create Build Definition File: Create a file named build.sbt with the following content: 
    name := "MyScalaProject"

version := "0.1"

scalaVersion := "2.13.6"
  3. Create Source Directory: 
    mkdir -p src/main/scala
mkdir -p src/test/scala

Managing Dependencies

Dependencies are external libraries that your project needs. You can add them to your build.sbt file.

Adding Dependencies

  1. Find the Dependency: Search for the library on Maven Central.
  2. Add to Build Definition: Add the dependency to your build.sbt file: 
    libraryDependencies += "org.scalatest" %% "scalatest" % "3.2.9" % Test

Compiling and Running Code

SBT provides commands to compile and run your Scala code.

Compiling Code

  1. Compile Command: In the terminal, run: 
    sbt compile
    This compiles the code in the src/main/scala directory.

Running Code

  1. Create Main Object: Create a file src/main/scala/Main.scala: 
    object Main extends App {
  println("Hello, Scala!")
}
  2. Run Command: In the terminal, run: 
    sbt run

Testing with SBT

Testing is an essential part of software development. SBT integrates well with testing frameworks like ScalaTest.

Writing Tests

  1. Create Test File: Create a file src/test/scala/MainSpec.scala: 
    import org.scalatest.flatspec.AnyFlatSpec

class MainSpec extends AnyFlatSpec {
  "The Main object" should "print Hello, Scala!" in {
    assert(Main.main(Array()) == "Hello, Scala!")
  }
}

Running Tests

  1. Test Command: In the terminal, run: 
    sbt test

Automating Tasks

SBT allows you to automate repetitive tasks using custom tasks and plugins.

Custom Tasks

  1. Define Custom Task: Add the following to your build.sbt: 
    lazy val hello = taskKey[Unit]("Prints 'Hello, World!'")

hello := {
  println("Hello, World!")
}
  2. Run Custom Task: In the terminal, run: 
    sbt hello

Conclusion

SBT is a powerful tool for managing Scala projects. It simplifies the process of compiling, running, and testing your code, as well as managing dependencies and automating tasks. By mastering SBT, you can significantly improve your productivity and streamline your development workflow.
