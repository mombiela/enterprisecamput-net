Introduction

In this section, we will outline the project you will be working on throughout this Scala course. This project will help you apply the concepts you learn in a practical, hands-on manner. By the end of the course, you will have a fully functional application that showcases your Scala programming skills.

Project Description

The project you will be developing is a Task Management Application. This application will allow users to create, update, delete, and view tasks. It will also include features such as task categorization, due dates, and priority levels.

Key Features

  • User Authentication: Users can sign up, log in, and log out.
  • Task CRUD Operations: Users can create, read, update, and delete tasks.
  • Task Categorization: Tasks can be categorized into different groups.
  • Due Dates and Priority Levels: Tasks can have due dates and priority levels assigned to them.
  • Search and Filter: Users can search and filter tasks based on various criteria.

Requirements

To successfully complete this project, you will need to meet the following requirements:

Technical Requirements

  • Scala Version: Use Scala 2.13 or later.
  • Build Tool: Use SBT (Simple Build Tool) for managing dependencies and building the project.
  • Database: Use an in-memory database like H2 for development and testing purposes.
  • Frameworks and Libraries:
    • Play Framework: For building the web application.
    • Slick: For database interaction.
    • ScalaTest: For writing and running tests.

Functional Requirements

  • User Authentication:
    • Users should be able to sign up with a username and password.
    • Users should be able to log in and log out.
  • Task Management:
    • Users should be able to create new tasks with a title, description, due date, and priority level.
    • Users should be able to view a list of all their tasks.
    • Users should be able to update the details of a task.
    • Users should be able to delete a task.
  • Categorization and Filtering:
    • Users should be able to categorize tasks into different groups.
    • Users should be able to filter tasks based on category, due date, and priority level.

Example Code

Here is a simple example of how you might define a Task case class in Scala:

case class Task(
  id: Long,
  title: String,
  description: String,
  dueDate: Option[java.time.LocalDate],
  priority: PriorityLevel,
  category: Option[String]
)

sealed trait PriorityLevel
case object High extends PriorityLevel
case object Medium extends PriorityLevel
case object Low extends PriorityLevel

Explanation

  • Case Class: Task is a case class that represents a task with various attributes.
  • Option Type: dueDate and category are optional fields, represented by the Option type.
  • Sealed Trait: PriorityLevel is a sealed trait with three possible values: High, Medium, and Low.

Conclusion

In this section, we have outlined the project you will be working on throughout the course. By understanding the project requirements and key features, you will be better prepared to apply the Scala concepts you learn in a practical manner. As you progress through the course, you will gradually build out this Task Management Application, enhancing your skills from beginner to advanced levels.
