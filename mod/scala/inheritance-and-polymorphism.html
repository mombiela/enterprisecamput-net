Introduction

Inheritance and polymorphism are fundamental concepts in object-oriented programming (OOP). They allow for the creation of flexible and reusable code. In Scala, these concepts are implemented in a way that leverages both the functional and object-oriented paradigms.

Inheritance

Inheritance allows a class to inherit properties and methods from another class. The class that inherits is called the subclass, and the class being inherited from is called the superclass.

Key Concepts

  • Superclass: The class whose properties and methods are inherited.
  • Subclass: The class that inherits properties and methods from the superclass.
  • extends keyword: Used to denote inheritance.

Example

// Superclass
class Animal {
  def eat(): Unit = {
    println("This animal is eating")
  }
}

// Subclass
class Dog extends Animal {
  def bark(): Unit = {
    println("The dog is barking")
  }
}

// Usage
val dog = new Dog
dog.eat()  // Inherited method
dog.bark() // Subclass-specific method

In this example:

  • Animal is the superclass.
  • Dog is the subclass that inherits from Animal.
  • The Dog class can use the eat method from the Animal class.

Polymorphism

Polymorphism allows methods to do different things based on the object it is acting upon, even though they share the same name. In Scala, polymorphism is achieved through method overriding and the use of traits.

Key Concepts

  • Method Overriding: A subclass provides a specific implementation of a method that is already defined in its superclass.
  • Traits: Traits are similar to interfaces in Java and can be used to achieve polymorphism.

Example of Method Overriding

class Animal {
  def sound(): Unit = {
    println("Some generic animal sound")
  }
}

class Dog extends Animal {
  override def sound(): Unit = {
    println("Bark")
  }
}

class Cat extends Animal {
  override def sound(): Unit = {
    println("Meow")
  }
}

// Usage
val animals: List[Animal] = List(new Dog, new Cat)
animals.foreach(_.sound())

In this example:

  • The sound method is overridden in both Dog and Cat classes.
  • When sound is called on each animal, the specific implementation for that animal is executed.

Example of Traits

trait Speaker {
  def speak(): Unit
}

class Human extends Speaker {
  def speak(): Unit = {
    println("Hello!")
  }
}

class Robot extends Speaker {
  def speak(): Unit = {
    println("Beep boop")
  }
}

// Usage
val speakers: List[Speaker] = List(new Human, new Robot)
speakers.foreach(_.speak())

In this example:

  • Speaker is a trait with an abstract method speak.
  • Both Human and Robot classes implement the speak method.
  • The speak method is called polymorphically on each instance in the list.

Conclusion

Inheritance and polymorphism are powerful features in Scala that enable code reuse and flexibility. By understanding how to use these concepts, you can create more maintainable and scalable applications. Remember:

  • Use inheritance to share common code among classes.
  • Use polymorphism to allow methods to operate differently based on the object they are acting upon.
  • Leverage traits to achieve polymorphism and multiple inheritance in Scala.
