Introduction

Anonymous functions, also known as lambdas, are a powerful feature in Scala that allow you to define functions without giving them a name. They are particularly useful for short, throwaway functions that are used only once or a few times. In this section, we will explore how to create and use anonymous functions in Scala, from basic to advanced usage.

Basic Syntax

  • Anonymous functions in Scala are defined using the => symbol.
  • The general syntax is (parameters) => expression.

Example

val add = (x: Int, y: Int) => x + y
println(add(3, 4))  // Output: 7

In this example, add is an anonymous function that takes two integers and returns their sum.

Using Anonymous Functions with Collections

Anonymous functions are often used with collection methods like map, filter, and reduce.

Example with `map`

val numbers = List(1, 2, 3, 4, 5)
val doubled = numbers.map(x => x * 2)
println(doubled)  // Output: List(2, 4, 6, 8, 10)

Here, the anonymous function x => x * 2 is applied to each element of the list numbers.

Example with `filter`

val numbers = List(1, 2, 3, 4, 5)
val evenNumbers = numbers.filter(x => x % 2 == 0)
println(evenNumbers)  // Output: List(2, 4)

In this example, the anonymous function x => x % 2 == 0 filters out the even numbers from the list.

Simplifying Anonymous Functions

Scala provides several ways to simplify anonymous functions, making them more concise.

Placeholder Syntax

  • You can use underscores (_) as placeholders for parameters.

Example

val numbers = List(1, 2, 3, 4, 5)
val doubled = numbers.map(_ * 2)
println(doubled)  // Output: List(2, 4, 6, 8, 10)

Here, _ * 2 is a shorthand for x => x * 2.

Type Inference

  • Scala can often infer the types of parameters, allowing you to omit them.

Example

val add = (_: Int) + (_: Int)
println(add(3, 4))  // Output: 7

In this case, the types of the parameters are inferred from the context.

Advanced Usage

Anonymous functions can be used in more complex scenarios, such as higher-order functions and currying.

Higher-Order Functions

  • Functions that take other functions as parameters or return functions as results.

Example

def applyOperation(x: Int, y: Int, operation: (Int, Int) => Int): Int = {
  operation(x, y)
}

val sum = applyOperation(3, 4, (a, b) => a + b)
println(sum)  // Output: 7

Here, applyOperation is a higher-order function that takes an anonymous function as a parameter.

Currying

  • Transforming a function that takes multiple arguments into a series of functions that each take a single argument.

Example

def curriedAdd(x: Int)(y: Int): Int = x + y
val addThree = curriedAdd(3) _
println(addThree(4))  // Output: 7

In this example, curriedAdd is a curried function, and addThree is a partially applied function.

Conclusion

Anonymous functions (lambdas) in Scala are a versatile tool that can simplify your code and make it more expressive. They are particularly useful for short, throwaway functions and are commonly used with collection methods. By understanding the basic syntax, simplifying techniques, and advanced usage, you can leverage the full power of anonymous functions in your Scala programs.
