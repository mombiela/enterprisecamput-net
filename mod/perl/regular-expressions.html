Introduction to Regular Expressions

Regular expressions (regex) are powerful tools for pattern matching and text manipulation. In Perl, regex is integrated directly into the language, making it a versatile tool for string processing.

Key Concepts

  • Pattern Matching: Finding specific patterns within strings.
  • Substitution: Replacing parts of a string that match a pattern.
  • Modifiers: Changing the behavior of regex operations.

Basic Syntax

Perl uses the =~ operator to apply a regex to a string.

Example

my $string = "Hello, world!";
if ($string =~ /world/) {
    print "Match found!\n";
}
  • /world/ is the regex pattern.
  • =~ applies the pattern to $string.

Common Metacharacters

  • .: Matches any single character except newline.
  • *: Matches 0 or more of the preceding element.
  • +: Matches 1 or more of the preceding element.
  • ?: Matches 0 or 1 of the preceding element.
  • \d: Matches any digit.
  • \w: Matches any word character (alphanumeric plus underscore).
  • \s: Matches any whitespace character.

Character Classes and Quantifiers

Character classes allow you to match any one of several characters.

Example

my $string = "abc123";
if ($string =~ /[a-z]/) {
    print "Contains a lowercase letter.\n";
}
  • [a-z] matches any lowercase letter.

Quantifiers

  • {n}: Matches exactly n occurrences.
  • {n,}: Matches n or more occurrences.
  • {n,m}: Matches between n and m occurrences.

Example

my $string = "aaaab";
if ($string =~ /a{3,5}/) {
    print "Matches between 3 and 5 'a's.\n";
}

Grouping and Capturing

Parentheses () are used for grouping parts of a regex and capturing matched substrings.

Example

my $string = "Hello, world!";
if ($string =~ /(Hello), (world)/) {
    print "First part: $1\n";
    print "Second part: $2\n";
}
  • $1 and $2 store the matched substrings.

Substitution

The s/// operator is used for substitution.

Example

my $string = "Hello, world!";
$string =~ s/world/Perl/;
print "$string\n";  # Output: Hello, Perl!
  • s/world/Perl/ replaces "world" with "Perl".

Advanced Topics

Lookahead and Lookbehind

Lookahead and lookbehind assertions allow you to match a pattern only if it is (or is not) followed or preceded by another pattern.

Example

my $string = "abc123";
if ($string =~ /abc(?=\d)/) {
    print "abc is followed by a digit.\n";
}
  • (?=\d) is a positive lookahead.

Non-capturing Groups

Non-capturing groups (?:...) are used to group parts of a regex without capturing the matched substrings.

Example

my $string = "Hello, world!";
if ($string =~ /(?:Hello), (world)/) {
    print "Matched: $1\n";  # Output: Matched: world
}

Modifiers

Modifiers change the behavior of regex operations.

Common Modifiers

  • i: Case-insensitive matching.
  • m: Treats the string as multiple lines.
  • s: Treats the string as a single line.
  • g: Global matching.

Example

my $string = "Hello, world!";
if ($string =~ /HELLO/i) {
    print "Case-insensitive match found!\n";
}
  • /HELLO/i matches "HELLO" case-insensitively.

Practical Exercises

Exercise 1: Basic Matching

Write a Perl script to check if a string contains the word "Perl".

Exercise 2: Substitution

Write a Perl script to replace all occurrences of "foo" with "bar" in a given string.

Exercise 3: Capturing Groups

Write a Perl script to extract the domain name from an email address.

Conclusion

Regular expressions in Perl are a powerful tool for text processing. By understanding the basic syntax, character classes, quantifiers, grouping, and advanced features like lookahead and lookbehind, you can perform complex pattern matching and text manipulation tasks efficiently. Practice with the provided exercises to solidify your understanding and become proficient in using regex in Perl.
