Introduction to GraphQL Types

GraphQL is a query language for APIs that allows clients to request exactly the data they need. One of the core concepts in GraphQL is its type system, which defines the structure of the data that can be queried. Understanding GraphQL types is essential for both defining your schema and writing effective queries.

Basic Types

GraphQL comes with a set of basic scalar types that represent the most common data types.

  • Int: A signed 32-bit integer.
  • Float: A signed double-precision floating-point value.
  • String: A UTF-8 character sequence.
  • Boolean: A true or false value.
  • ID: A unique identifier, often used to refetch an object or as a key for a cache.

Example

type Query {
  id: ID
  name: String
  age: Int
  isActive: Boolean
  score: Float
}

Object Types

Object types are the most common type in a GraphQL schema. They represent a collection of fields, each of which can be another type.

Example

type User {
  id: ID!
  name: String!
  age: Int
  email: String
}

In this example:

  • id and name are non-nullable fields (denoted by !).
  • age and email are nullable fields.

Query and Mutation Types

In GraphQL, the schema defines two special types: Query and Mutation.

  • Query: Defines the entry points for read operations.
  • Mutation: Defines the entry points for write operations.

Example

type Query {
  user(id: ID!): User
  users: [User]
}

type Mutation {
  createUser(name: String!, email: String!): User
}

In this example:

  • The Query type has two fields: user and users.
  • The Mutation type has one field: createUser.

Enum Types

Enum types are a special kind of scalar that is restricted to a particular set of allowed values.

Example

enum Role {
  ADMIN
  USER
  GUEST
}

type User {
  id: ID!
  name: String!
  role: Role!
}

In this example:

  • The Role enum can have one of three values: ADMIN, USER, or GUEST.

List and Non-Null Types

GraphQL also supports list and non-null types to define more complex structures.

  • List: Represents an array of a certain type.
  • Non-Null: Ensures that a field cannot be null.

Example

type Query {
  users: [User!]!
}

In this example:

  • users is a non-nullable list of non-nullable User objects.

Input Types

Input types are used as arguments in queries and mutations. They allow you to pass complex objects as arguments.

Example

input UserInput {
  name: String!
  email: String!
  age: Int
}

type Mutation {
  createUser(input: UserInput!): User
}

In this example:

  • UserInput is an input type used in the createUser mutation.

Interface Types

Interface types are abstract types that define a set of fields that multiple types must include.

Example

interface Character {
  id: ID!
  name: String!
}

type Human implements Character {
  id: ID!
  name: String!
  homePlanet: String
}

type Droid implements Character {
  id: ID!
  name: String!
  primaryFunction: String
}

type Query {
  characters: [Character]
}

In this example:

  • Character is an interface implemented by Human and Droid.

Union Types

Union types are similar to interfaces but do not require the types to share any fields.

Example

union SearchResult = Human | Droid

type Query {
  search(text: String!): [SearchResult]
}

In this example:

  • SearchResult is a union type that can be either Human or Droid.

Conclusion

Understanding GraphQL types is fundamental to designing a robust and efficient GraphQL schema. By leveraging the various types available, you can create a schema that accurately represents your data and supports complex queries and mutations. As you progress, you'll find that mastering these types will enable you to build more powerful and flexible APIs.
