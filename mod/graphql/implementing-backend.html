In this section, we will delve into the process of implementing the backend for a GraphQL API. This includes setting up the server, defining the schema, and creating resolvers. By the end of this section, you will have a solid understanding of how to build a functional GraphQL backend.

Setting Up the Server

To start, we need to set up a server that can handle GraphQL requests. We'll use Node.js with the Express framework and the Apollo Server library.

Installing Dependencies

First, let's install the necessary packages:

npm init -y
npm install express apollo-server-express graphql

Creating the Server

Next, create a file named server.js and add the following code to set up the server:

const express = require('express');
const { ApolloServer, gql } = require('apollo-server-express');

const app = express();

// Define a simple schema
const typeDefs = gql`
  type Query {
    hello: String
  }
`;

// Define resolvers
const resolvers = {
  Query: {
    hello: () => 'Hello, world!',
  },
};

// Create an Apollo Server instance
const server = new ApolloServer({ typeDefs, resolvers });

// Apply the Apollo GraphQL middleware and set the path to /graphql
server.applyMiddleware({ app, path: '/graphql' });

// Start the server
app.listen({ port: 4000 }, () =>
  console.log(`Server ready at http://localhost:4000${server.graphqlPath}`)
);

Explanation

  • Dependencies: We import express, ApolloServer, and gql from their respective packages.
  • Schema Definition: We define a simple schema with a single Query type that returns a string.
  • Resolvers: We create a resolver for the hello query that returns a static string.
  • Apollo Server: We create an instance of ApolloServer with the schema and resolvers, then apply it as middleware to the Express app.
  • Server Start: Finally, we start the server on port 4000.

Defining the Schema

The schema is a crucial part of any GraphQL API. It defines the types, queries, and mutations that clients can execute.

Basic Schema

Let's expand our schema to include more complex types and relationships. Update the typeDefs in server.js:

const typeDefs = gql`
  type Book {
    title: String
    author: Author
  }

  type Author {
    name: String
    books: [Book]
  }

  type Query {
    books: [Book]
    authors: [Author]
  }
`;

Explanation

  • Types: We define Book and Author types. Each Book has a title and an author, and each Author has a name and a list of books.
  • Queries: We add queries to fetch lists of books and authors.

Creating Resolvers

Resolvers provide the logic for fetching the data defined in the schema.

Basic Resolvers

Update the resolvers in server.js:

const books = [
  { title: 'The Awakening', author: 'Kate Chopin' },
  { title: 'City of Glass', author: 'Paul Auster' },
];

const authors = [
  { name: 'Kate Chopin', books: ['The Awakening'] },
  { name: 'Paul Auster', books: ['City of Glass'] },
];

const resolvers = {
  Query: {
    books: () => books,
    authors: () => authors,
  },
  Book: {
    author: (book) => authors.find((author) => author.name === book.author),
  },
  Author: {
    books: (author) => books.filter((book) => book.author === author.name),
  },
};

Explanation

  • Static Data: We use static arrays for books and authors for simplicity.
  • Query Resolvers: We provide resolvers for the books and authors queries that return the static data.
  • Field Resolvers: We add field resolvers for Book.author and Author.books to resolve the relationships between books and authors.

Conclusion

In this section, we covered the basics of setting up a GraphQL backend using Node.js, Express, and Apollo Server. We defined a simple schema and created resolvers to fetch data. This foundational knowledge will enable you to build more complex GraphQL APIs as you progress through the course.

By understanding how to set up the server, define the schema, and create resolvers, you are now equipped to handle the core components of a GraphQL backend. In the next sections, we will explore more advanced topics such as authentication, error handling, and optimizing performance.
