Enums and Unions in GraphQL

Introduction

In this section, we will explore two important GraphQL features: Enums and Unions. These features help in defining more precise and flexible schemas, making your GraphQL API more robust and easier to use.

Enums in GraphQL

What are Enums?

  • Enums (short for Enumerations) are a special kind of scalar that restricts the value to a predefined set of constants.
  • They are useful for representing a fixed set of possible values, such as days of the week, user roles, or status codes.

Defining Enums

To define an Enum in GraphQL, you use the enum keyword followed by the name of the Enum and the set of possible values.

enum Role {
  ADMIN
  USER
  GUEST
}

Using Enums in Schema

Enums can be used as field types in your GraphQL schema. Here’s an example of how to use the Role enum in a User type:

type User {
  id: ID!
  name: String!
  role: Role!
}

Example Query with Enums

Here’s how you might query a user and get their role:

query {
  user(id: "1") {
    name
    role
  }
}

Example Mutation with Enums

You can also use Enums in mutations. For example, setting a user's role:

mutation {
  updateUserRole(id: "1", role: ADMIN) {
    id
    name
    role
  }
}

Unions in GraphQL

What are Unions?

  • Unions allow a field to return one of several different types.
  • Unlike interfaces, unions do not share any fields in common.

Defining Unions

To define a Union, you use the union keyword followed by the name of the Union and the possible types it can return.

union SearchResult = User | Post

Using Unions in Schema

Unions can be used as field types in your GraphQL schema. Here’s an example of how to use the SearchResult union in a query:

type Query {
  search(text: String!): [SearchResult!]!
}

Example Query with Unions

Here’s how you might query a search and get different types of results:

query {
  search(text: "GraphQL") {
    ... on User {
      id
      name
    }
    ... on Post {
      id
      title
      content
    }
  }
}

Resolving Unions

When using unions, you need to define how to resolve the union type in your server-side code. Here’s an example in JavaScript using Apollo Server:

const resolvers = {
  SearchResult: {
    __resolveType(obj, context, info) {
      if (obj.name) {
        return 'User';
      }
      if (obj.title) {
        return 'Post';
      }
      return null;
    },
  },
  Query: {
    search: (parent, args, context, info) => {
      // Implement search logic here
      return [
        { id: '1', name: 'Alice' },
        { id: '2', title: 'GraphQL Introduction', content: '...' },
      ];
    },
  },
};

Conclusion

Enums and Unions are powerful features in GraphQL that allow you to create more precise and flexible schemas. Enums restrict a field to a set of predefined values, making your API more predictable and easier to use. Unions allow a field to return different types, providing flexibility in how you structure your data.

By understanding and utilizing Enums and Unions, you can enhance the robustness and usability of your GraphQL APIs, making them more intuitive for clients to interact with.
