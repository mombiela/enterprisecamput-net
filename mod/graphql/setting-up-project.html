In this section, we will cover how to set up a GraphQL project from scratch. This includes installing necessary tools, initializing the project, and setting up the basic structure. By the end of this section, you will have a working GraphQL server that you can build upon in subsequent lessons.

Prerequisites

Before we start, ensure you have the following installed on your machine:

  • Node.js (version 12.x or later)
  • npm (Node Package Manager) or yarn

You can check if Node.js and npm are installed by running the following commands in your terminal:

node -v
npm -v

Initializing the Project

  1. Create a new directory for your project:

    mkdir graphql-project
cd graphql-project

  2. Initialize a new Node.js project:

    npm init -y

    This command will create a package.json file with default settings.

Installing Dependencies

We will need several packages to set up our GraphQL server. The primary packages are express for our server, graphql for GraphQL functionality, and express-graphql to integrate GraphQL with Express.

  1. Install the necessary packages: 
    npm install express graphql express-graphql

Setting Up the Server

  1. Create a new file named server.js in the root directory:

    touch server.js

  2. Open server.js and add the following code:

    const express = require('express');
const { graphqlHTTP } = require('express-graphql');
const { buildSchema } = require('graphql');

// Construct a schema, using GraphQL schema language
const schema = buildSchema(`
  type Query {
    hello: String
  }
`);

// The root provides a resolver function for each API endpoint
const root = {
  hello: () => {
    return 'Hello world!';
  },
};

const app = express();
app.use('/graphql', graphqlHTTP({
  schema: schema,
  rootValue: root,
  graphiql: true,
}));

app.listen(4000, () => console.log('Now browse to localhost:4000/graphql'));

Explanation of the Code

  • Importing Modules:

    const express = require('express');
const { graphqlHTTP } = require('express-graphql');
const { buildSchema } = require('graphql');

    We import the necessary modules to set up our server and GraphQL integration.

  • Defining the Schema:

    const schema = buildSchema(`
  type Query {
    hello: String
  }
`);

    We define a simple schema with a single query hello that returns a String.

  • Creating the Root Resolver:

    const root = {
  hello: () => {
    return 'Hello world!';
  },
};

    The resolver function for the hello query returns the string "Hello world!".

  • Setting Up the Express Server:

    const app = express();
app.use('/graphql', graphqlHTTP({
  schema: schema,
  rootValue: root,
  graphiql: true,
}));

app.listen(4000, () => console.log('Now browse to localhost:4000/graphql'));

    We set up an Express server and configure the /graphql endpoint to use express-graphql with our schema and root resolver. We also enable the GraphiQL interface for testing our GraphQL queries.

Running the Server

  1. Start the server by running:

    node server.js

  2. Open your browser and navigate to:

    http://localhost:4000/graphql

    You should see the GraphiQL interface where you can test your GraphQL queries.

Conclusion

In this section, we have successfully set up a basic GraphQL server using Node.js, Express, and the express-graphql middleware. We created a simple schema and resolver, and we configured our server to use GraphiQL for testing. This foundational setup will allow us to build more complex GraphQL APIs in the following sections.
