Introduction to Interfaces

Interfaces in GraphQL are a powerful way to define a contract that multiple types can implement. They allow you to create a set of fields that different types must include, ensuring consistency across your schema.

  • Definition: An interface is an abstract type that includes a certain set of fields that a type must implement.
  • Purpose: Interfaces are used to define a common structure for different types, enabling polymorphism in your GraphQL schema.

Defining Interfaces

To define an interface in GraphQL, you use the interface keyword followed by the name of the interface and the fields it includes.

Example

interface Character {
  id: ID!
  name: String!
  appearsIn: [Episode!]!
}

In this example:

  • Character is the interface.
  • It includes three fields: id, name, and appearsIn.

Implementing Interfaces

Types that implement an interface must include all the fields defined in the interface.

Example

type Human implements Character {
  id: ID!
  name: String!
  appearsIn: [Episode!]!
  totalCredits: Int
}

type Droid implements Character {
  id: ID!
  name: String!
  appearsIn: [Episode!]!
  primaryFunction: String
}

In this example:

  • Human and Droid both implement the Character interface.
  • Both types include the fields id, name, and appearsIn as required by the Character interface.
  • They also include additional fields specific to each type (totalCredits for Human and primaryFunction for Droid).

Querying Interfaces

When querying an interface, you can request fields that are part of the interface, and you can also use inline fragments to request fields specific to the implementing types.

Example

{
  characters {
    id
    name
    appearsIn
    ... on Human {
      totalCredits
    }
    ... on Droid {
      primaryFunction
    }
  }
}

In this query:

  • characters is a field that returns a list of Character interface types.
  • The query requests the id, name, and appearsIn fields, which are part of the Character interface.
  • Inline fragments (... on Human and ... on Droid) are used to request fields specific to the Human and Droid types.

Interfaces vs. Unions

While both interfaces and unions allow for polymorphism, they serve different purposes.

| Feature | Interface | Union | |------------------|------------------------------------------------|------------------------------------------------| | Common Fields | Yes | No | | Type Checking | Can check for specific fields | Can only check for type | | Use Case | When types share common fields | When types do not share common fields | | Syntax | interface keyword | union keyword |

Example of Union

union SearchResult = Human | Droid | Starship

{
  search(text: "R2") {
    ... on Human {
      name
    }
    ... on Droid {
      primaryFunction
    }
    ... on Starship {
      model
    }
  }
}

In this example:

  • SearchResult is a union of Human, Droid, and Starship.
  • The query uses inline fragments to request fields specific to each type in the union.

Conclusion

Interfaces in GraphQL provide a robust way to enforce a consistent structure across different types. By defining common fields in an interface and implementing them in various types, you can ensure that your schema remains organized and predictable. Understanding how to define, implement, and query interfaces is essential for creating flexible and maintainable GraphQL APIs.
