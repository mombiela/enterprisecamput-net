Object-Oriented Programming in R
Functional Programming in R ►

Object-Oriented Programming (OOP) is a programming paradigm that uses "objects" to design applications and computer programs. In R, OOP is implemented through different systems, primarily S3, S4, and Reference Classes (R5). This section will guide you through these systems, providing clear explanations, examples, and exercises.

Introduction to Object-Oriented Programming

  • Definition: OOP is a programming paradigm based on the concept of "objects", which can contain data and code to manipulate that data.
  • Benefits:
    • Encapsulation: Bundling data with methods that operate on that data.
    • Inheritance: Mechanism to create new classes from existing ones.
    • Polymorphism: Ability to process objects differently based on their class.

S3 System

The S3 system is the simplest and most commonly used OOP system in R. It is informal and relies on naming conventions.

Creating S3 Objects

  • Creating an S3 object: An S3 object is usually a list with a class attribute.
# Create a simple S3 object
person <- list(name = "John Doe", age = 30)
class(person) <- "Person"
  • Defining S3 methods: Methods are functions that operate on objects of a specific class.
# Define a print method for the Person class
print.Person <- function(x) {
  cat("Name:", x$name, "\nAge:", x$age, "\n")
}

# Test the print method
print(person)

Exercises

  1. Create an S3 object for a "Car" with attributes like make, model, and year.
  2. Define a print method for the "Car" class.

S4 System

The S4 system is more formal and rigorous than S3, providing better data validation and encapsulation.

Creating S4 Classes and Objects

  • Defining an S4 class: Use the setClass function.
# Define an S4 class
setClass("Person",
         slots = list(name = "character", age = "numeric"))

# Create an S4 object
john <- new("Person", name = "John Doe", age = 30)
  • Defining S4 methods: Use the setMethod function.
# Define a show method for the Person class
setMethod("show", "Person", function(object) {
  cat("Name:", object@name, "\nAge:", object@age, "\n")
})

# Test the show method
john

Exercises

  1. Define an S4 class for a "Car" with slots for make, model, and year.
  2. Create an S4 object for a car and define a show method for it.

Reference Classes (R5)

Reference Classes (R5) provide a more traditional OOP approach, similar to other programming languages like Java or C++.

Creating Reference Classes

  • Defining a Reference Class: Use the setRefClass function.
# Define a Reference Class
Person <- setRefClass("Person",
                      fields = list(name = "character", age = "numeric"),
                      methods = list(
                        show = function() {
                          cat("Name:", name, "\nAge:", age, "\n")
                        }
                      ))

# Create an instance of the Reference Class
john <- Person$new(name = "John Doe", age = 30)
john$show()

Exercises

  1. Define a Reference Class for a "Car" with fields for make, model, and year.
  2. Create an instance of the Car class and define a method to display its details.

Comparison of OOP Systems in R

| Feature | S3 | S4 | Reference Classes (R5) | |---------------|---------------|---------------|------------------------| | Formality | Informal | Formal | Formal | | Data Validation | No | Yes | Yes | | Encapsulation | Partial | Full | Full | | Inheritance | Yes | Yes | Yes | | Polymorphism | Yes | Yes | Yes |

Conclusion

Object-Oriented Programming in R offers multiple systems to suit different needs and levels of formality. The S3 system is simple and flexible, making it suitable for quick and informal tasks. The S4 system provides more rigorous data validation and encapsulation, ideal for complex applications. Reference Classes (R5) offer a more traditional OOP approach, similar to other programming languages.

By understanding and practicing these systems, you can leverage the power of OOP to write more organized, reusable, and maintainable code in R.
