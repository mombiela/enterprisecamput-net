Introduction to Supervised Learning

Supervised learning is a type of machine learning where the model is trained on labeled data. The goal is to learn a mapping from inputs to outputs based on the provided labels.

  • Labeled Data: Data that includes both input features and the corresponding output labels.
  • Training Phase: The model learns from the labeled data.
  • Prediction Phase: The model predicts labels for new, unseen data.

What is Classification?

Classification is a type of supervised learning where the output variable is categorical. The goal is to predict the category or class of new observations based on past observations.

  • Binary Classification: Two possible classes (e.g., spam or not spam).
  • Multi-class Classification: More than two classes (e.g., classifying types of animals).

Key Concepts in Classification

  • Features: The input variables used to make predictions.
  • Labels: The output categories we want to predict.
  • Training Set: The subset of data used to train the model.
  • Test Set: The subset of data used to evaluate the model's performance.

Common Classification Algorithms

  • Logistic Regression
  • k-Nearest Neighbors (k-NN)
  • Decision Trees
  • Random Forests
  • Support Vector Machines (SVM)
  • Naive Bayes

Implementing Classification in R

Logistic Regression

Logistic regression is used for binary classification problems. It models the probability that a given input belongs to a particular class.

# Load necessary library
library(caret)

# Load dataset
data(iris)
# Use only two classes for binary classification
iris_binary <- subset(iris, Species != "setosa")

# Split the data into training and test sets
set.seed(123)
trainIndex <- createDataPartition(iris_binary$Species, p = .8, 
                                  list = FALSE, 
                                  times = 1)
irisTrain <- iris_binary[ trainIndex,]
irisTest  <- iris_binary[-trainIndex,]

# Train the logistic regression model
model <- train(Species ~ ., data = irisTrain, method = "glm", family = "binomial")

# Make predictions
predictions <- predict(model, irisTest)

# Evaluate the model
confusionMatrix(predictions, irisTest$Species)

k-Nearest Neighbors (k-NN)

k-NN is a simple, instance-based learning algorithm that classifies a new sample based on the majority class among its k-nearest neighbors.

# Load necessary library
library(class)

# Prepare the data
trainData <- irisTrain[, -5]
trainLabels <- irisTrain$Species
testData <- irisTest[, -5]
testLabels <- irisTest$Species

# Train and predict using k-NN
k <- 3
predictions <- knn(trainData, testData, trainLabels, k = k)

# Evaluate the model
confusionMatrix(predictions, testLabels)

Decision Trees

Decision trees classify data by splitting it into subsets based on the value of input features.

# Load necessary library
library(rpart)

# Train the decision tree model
model <- rpart(Species ~ ., data = irisTrain, method = "class")

# Make predictions
predictions <- predict(model, irisTest, type = "class")

# Evaluate the model
confusionMatrix(predictions, irisTest$Species)

Model Evaluation Metrics

  • Accuracy: The proportion of correctly classified instances.
  • Precision: The proportion of true positive instances among the instances classified as positive.
  • Recall (Sensitivity): The proportion of true positive instances among all actual positive instances.
  • F1 Score: The harmonic mean of precision and recall.

| Metric | Formula | Interpretation | |------------|----------------------------------------------|-----------------------------------------------------| | Accuracy | (TP + TN) / (TP + TN + FP + FN) | Overall correctness of the model | | Precision | TP / (TP + FP) | Correctness of positive predictions | | Recall | TP / (TP + FN) | Ability to find all positive instances | | F1 Score | 2 * (Precision * Recall) / (Precision + Recall) | Balance between precision and recall |

Conclusion

Supervised learning, specifically classification, is a powerful tool for predictive modeling. By understanding and implementing various classification algorithms in R, you can build models to predict categorical outcomes. Remember to evaluate your models using appropriate metrics to ensure their effectiveness.
