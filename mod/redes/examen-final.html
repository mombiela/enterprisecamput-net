Introduction

The final exam for the Networking course aims to assess your understanding and skills in the key concepts we have covered throughout the course. This exam will include theoretical questions, practical exercises, and analysis problems to ensure you can apply what you've learned in real-world situations.

Exam Structure

The exam will be divided into the following sections:

  1. Theoretical Questions
  2. Network Configuration
  3. Network Traffic Analysis
  4. Network Troubleshooting

Theoretical Questions

In this section, your knowledge of fundamental networking concepts will be assessed. Below are some examples of questions you might encounter:

  • Define the following terms:
    • IP Address
    • Subnet Mask
    • Gateway
    • DNS
  • Explain the difference between TCP and UDP.
  • Describe the OSI model and its layers.

Network Configuration

This section will assess your ability to configure and manage networks. You will be provided with scenarios where you must configure network devices such as routers and switches. Here is an example exercise:

Example Exercise

Configure a simple network with two subnets using a router and two switches. Ensure that the two subnets can communicate with each other.

Steps:

  1. Assign the following IP addresses to the devices:
    • Subnet 1: 192.168.1.0/24
    • Subnet 2: 192.168.2.0/24
  2. Configure the router to route traffic between the two subnets.
  3. Verify connectivity between devices on different subnets.
Router(config)# interface g0/0
Router(config-if)# ip address 192.168.1.1 255.255.255.0
Router(config-if)# no shutdown

Router(config)# interface g0/1
Router(config-if)# ip address 192.168.2.1 255.255.255.0
Router(config-if)# no shutdown

Router(config)# ip route 192.168.2.0 255.255.255.0 g0/1
Router(config)# ip route 192.168.1.0 255.255.255.0 g0/0

Network Traffic Analysis

In this section, you will be asked to analyze network traffic captures using tools like Wireshark. You will need to identify problems and propose solutions.

Example Exercise

Analyze the following traffic capture and answer the questions:

  • What is the source and destination IP address of packet number 10?
  • What protocol is being used?
  • Is there any indication of network issues in the capture?
No.     Time           Source                Destination           Protocol Length Info
10      0.000123       192.168.1.2           192.168.1.1           TCP      60     443 → 12345 [SYN] Seq=0 Win=65535 Len=0 MSS=1460

Network Troubleshooting

This section will assess your ability to diagnose and resolve network problems. You will be presented with scenarios with specific issues that you must solve.

Example Exercise

A user reports that they cannot access the internet. Their IP address is 192.168.1.10 and their gateway is 192.168.1.1. Describe the steps you would take to diagnose and resolve the problem.

Steps:

  1. Verify the user's connectivity with the gateway. 
    ping 192.168.1.1
  2. Check the DNS configuration. 
    nslookup www.google.com
  3. Review the router configuration and ensure there are no firewall rules blocking the traffic.

Conclusion

The final exam is an opportunity to demonstrate your mastery of the networking concepts and skills you have acquired during the course. Be sure to review all the topics covered and practice with exercises similar to those provided. Good luck!
