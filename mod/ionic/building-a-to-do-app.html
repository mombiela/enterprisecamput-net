Introduction

In this topic, we will guide you through building a To-Do app using Ionic. This project will help you understand the core concepts of Ionic, including UI components, navigation, state management, and data storage. By the end of this topic, you will have a functional To-Do app and a solid understanding of how to build mobile applications with Ionic.

Setting Up the Environment

Before we start building the app, we need to set up our development environment.

  • Install Node.js and npm: Ensure you have Node.js and npm installed on your system.
  • Install Ionic CLI: Run the following command to install the Ionic CLI globally: 
    npm install -g @ionic/cli
  • Create a New Ionic Project: Use the Ionic CLI to create a new project: 
    ionic start todo-app blank
  • Navigate to Project Directory: Move into the project directory: 
    cd todo-app

Building the User Interface

We will start by building the user interface for our To-Do app.

Home Page

The home page will display the list of to-do items and a button to add new items.

  • Create Home Page: Generate a new page using the Ionic CLI:

    ionic generate page home

  • Update app-routing.module.ts: Set the home page as the default route.

    const routes: Routes = [
  { path: '', redirectTo: 'home', pathMatch: 'full' },
  { path: 'home', loadChildren: () => import('./home/home.module').then(m => m.HomePageModule) }
];

  • Design the Home Page: Modify home.page.html to include a list and a button.

    <ion-header>
  <ion-toolbar>
    <ion-title>To-Do List</ion-title>
  </ion-toolbar>
</ion-header>

<ion-content>
  <ion-list>
    <ion-item *ngFor="let item of todoItems">
      {{ item }}
    </ion-item>
  </ion-list>
  <ion-fab vertical="bottom" horizontal="end" slot="fixed">
    <ion-fab-button (click)="addItem()">
      <ion-icon name="add"></ion-icon>
    </ion-fab-button>
  </ion-fab>
</ion-content>

Adding Items

We need to implement the functionality to add new items to the list.

  • Update home.page.ts: Add a method to handle adding new items. 
    import { Component } from '@angular/core';
import { AlertController } from '@ionic/angular';

@Component({
  selector: 'app-home',
  templateUrl: 'home.page.html',
  styleUrls: ['home.page.scss'],
})
export class HomePage {
  todoItems: string[] = [];

  constructor(private alertController: AlertController) {}

  async addItem() {
    const alert = await this.alertController.create({
      header: 'New Item',
      inputs: [
        {
          name: 'item',
          type: 'text',
          placeholder: 'Enter your to-do item'
        }
      ],
      buttons: [
        {
          text: 'Cancel',
          role: 'cancel',
          cssClass: 'secondary'
        },
        {
          text: 'Add',
          handler: (data) => {
            if (data.item) {
              this.todoItems.push(data.item);
            }
          }
        }
      ]
    });

    await alert.present();
  }
}

State Management

To manage the state of our application, we will use a service.

  • Create a To-Do Service: Generate a new service using the Ionic CLI:

    ionic generate service todo

  • Implement the Service: Update todo.service.ts to manage the list of to-do items.

    import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class TodoService {
  private todoItems: string[] = [];

  constructor() {}

  getItems(): string[] {
    return this.todoItems;
  }

  addItem(item: string) {
    this.todoItems.push(item);
  }
}

  • Update Home Page to Use Service: Modify home.page.ts to use the TodoService.

    import { Component } from '@angular/core';
import { AlertController } from '@ionic/angular';
import { TodoService } from '../services/todo.service';

@Component({
  selector: 'app-home',
  templateUrl: 'home.page.html',
  styleUrls: ['home.page.scss'],
})
export class HomePage {
  todoItems: string[];

  constructor(private alertController: AlertController, private todoService: TodoService) {
    this.todoItems = this.todoService.getItems();
  }

  async addItem() {
    const alert = await this.alertController.create({
      header: 'New Item',
      inputs: [
        {
          name: 'item',
          type: 'text',
          placeholder: 'Enter your to-do item'
        }
      ],
      buttons: [
        {
          text: 'Cancel',
          role: 'cancel',
          cssClass: 'secondary'
        },
        {
          text: 'Add',
          handler: (data) => {
            if (data.item) {
              this.todoService.addItem(data.item);
              this.todoItems = this.todoService.getItems();
            }
          }
        }
      ]
    });

    await alert.present();
  }
}

Data Storage

To persist the to-do items, we will use Ionic Storage.

  • Install Ionic Storage: Add the necessary packages.

    npm install @ionic/storage-angular
npm install @ionic/storage

  • Configure Storage: Update app.module.ts to configure Ionic Storage.

    import { IonicStorageModule } from '@ionic/storage-angular';

@NgModule({
  ...
  imports: [
    ...
    IonicStorageModule.forRoot()
  ],
  ...
})
export class AppModule {}

  • Update To-Do Service: Modify todo.service.ts to use Ionic Storage.

    import { Injectable } from '@angular/core';
import { Storage } from '@ionic/storage-angular';

@Injectable({
  providedIn: 'root'
})
export class TodoService {
  private todoItems: string[] = [];

  constructor(private storage: Storage) {
    this.init();
  }

  async init() {
    await this.storage.create();
    this.todoItems = (await this.storage.get('todoItems')) || [];
  }

  getItems(): string[] {
    return this.todoItems;
  }

  async addItem(item: string) {
    this.todoItems.push(item);
    await this.storage.set('todoItems', this.todoItems);
  }
}

Conclusion

In this topic, we have built a simple To-Do app using Ionic. We covered setting up the environment, building the user interface, managing state with a service, and persisting data using Ionic Storage. This project has provided you with a foundational understanding of Ionic, which you can build upon to create more complex applications.
