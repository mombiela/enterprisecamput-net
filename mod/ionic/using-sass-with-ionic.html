Introduction to Sass

Sass (Syntactically Awesome Stylesheets) is a CSS preprocessor that adds power and elegance to the basic language. It allows you to use variables, nested rules, mixins, functions, and more, all with a fully CSS-compatible syntax.

Key Concepts of Sass

  • Variables: Store values that you can reuse throughout your stylesheet.
  • Nesting: Nest your CSS selectors in a way that follows the same visual hierarchy of your HTML.
  • Partials and Import: Break your CSS into smaller, reusable pieces.
  • Mixins: Create reusable chunks of code.
  • Inheritance: Share a set of CSS properties from one selector to another.

Setting Up Sass in an Ionic Project

Installing Sass

To use Sass with Ionic, you need to install the necessary tools. Ionic CLI comes with built-in support for Sass.

  1. Create a new Ionic project:

    ionic start myApp tabs
cd myApp

  2. Install Sass:

    npm install sass --save-dev

Configuring Sass

Ionic uses a scss folder for Sass files by default. You can configure your project to use Sass by creating .scss files.

  1. Create a scss folder:

    mkdir src/theme/scss

  2. Create a main Sass file:

    touch src/theme/scss/variables.scss

  3. Import the main Sass file in your global styles:

    // src/global.scss
@import 'theme/scss/variables';

Using Sass Features in Ionic

Variables

Variables in Sass allow you to store values that you can reuse throughout your stylesheet.

// src/theme/scss/variables.scss
$primary-color: #3880ff;
$secondary-color: #3dc2ff;

body {
  background-color: $primary-color;
  color: $secondary-color;
}

Nesting

Nesting allows you to nest your CSS selectors in a way that follows the same visual hierarchy of your HTML.

// src/theme/scss/variables.scss
nav {
  ul {
    margin: 0;
    padding: 0;
    list-style: none;

    li {
      display: inline-block;
      margin-right: 5px;

      a {
        text-decoration: none;
        color: $primary-color;

        &:hover {
          color: $secondary-color;
        }
      }
    }
  }
}

Partials and Import

Partials allow you to break your CSS into smaller, reusable pieces. You can then import these partials into your main stylesheet.

// src/theme/scss/_buttons.scss
.button {
  background-color: $primary-color;
  border: none;
  color: white;
  padding: 10px 20px;
  text-align: center;
  text-decoration: none;
  display: inline-block;
  font-size: 16px;
  margin: 4px 2px;
  cursor: pointer;
}

// src/theme/scss/variables.scss
@import 'buttons';

Mixins

Mixins allow you to create reusable chunks of code.

// src/theme/scss/_mixins.scss
@mixin border-radius($radius) {
  -webkit-border-radius: $radius;
  -moz-border-radius: $radius;
  -ms-border-radius: $radius;
  -o-border-radius: $radius;
  border-radius: $radius;
}

// src/theme/scss/variables.scss
@import 'mixins';

.box { 
  @include border-radius(10px); 
}

Inheritance

Inheritance allows you to share a set of CSS properties from one selector to another.

// src/theme/scss/_base.scss
.message {
  padding: 10px;
  border: 1px solid #ccc;
  color: #333;
}

// src/theme/scss/variables.scss
@import 'base';

.success {
  @extend .message;
  border-color: green;
}

.error {
  @extend .message;
  border-color: red;
}

Conclusion

Using Sass with Ionic enhances your ability to manage and maintain your stylesheets efficiently. By leveraging Sass features like variables, nesting, partials, mixins, and inheritance, you can create more organized and reusable CSS. This not only improves your development workflow but also makes your codebase more scalable and easier to maintain.
