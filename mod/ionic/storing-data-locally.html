Storing Data Locally
Using Firebase with Ionic ►

In this section, we will explore how to store data locally in an Ionic application. This is crucial for creating apps that can function offline or need to save user preferences, settings, or other data.

Key Concepts

  • Local Storage: A simple key-value storage mechanism.
  • SQLite: A full-featured SQL database engine.
  • Ionic Storage: A unified API for different storage engines.

Local Storage

Local Storage is a web storage mechanism that allows you to store data in the browser. It is simple to use but has limitations in terms of storage size and data complexity.

Using Local Storage

To use Local Storage in an Ionic application, you can directly interact with the window.localStorage object.

Example

// Save data to Local Storage
window.localStorage.setItem('username', 'john_doe');

// Retrieve data from Local Storage
const username = window.localStorage.getItem('username');
console.log(username); // Output: john_doe

// Remove data from Local Storage
window.localStorage.removeItem('username');

Limitations

  • Limited storage capacity (usually around 5MB).
  • Only supports string data types.
  • Synchronous API, which might block the main thread.

SQLite

SQLite is a powerful, full-featured SQL database engine that can be embedded into your Ionic application. It is suitable for more complex data storage needs.

Setting Up SQLite

To use SQLite in an Ionic application, you need to install the Cordova SQLite plugin.

ionic cordova plugin add cordova-sqlite-storage
npm install @ionic-native/sqlite

Using SQLite

Example

import { SQLite, SQLiteObject } from '@ionic-native/sqlite/ngx';

constructor(private sqlite: SQLite) {}

this.sqlite.create({
  name: 'data.db',
  location: 'default'
})
.then((db: SQLiteObject) => {

  db.executeSql('create table danceMoves(name VARCHAR(32))', [])
    .then(() => console.log('Executed SQL'))
    .catch(e => console.log(e));

})
.catch(e => console.log(e));

Advantages

  • Supports complex queries and transactions.
  • Can handle large amounts of data.
  • Asynchronous API, which is better for performance.

Ionic Storage

Ionic Storage provides a simple, unified API for different storage engines, including Local Storage, IndexedDB, and SQLite. It is highly recommended for most use cases due to its flexibility and ease of use.

Setting Up Ionic Storage

To use Ionic Storage, you need to install the necessary packages.

npm install @ionic/storage-angular
npm install @ionic/storage

Configuring Ionic Storage

You need to configure Ionic Storage in your app module.

import { IonicStorageModule } from '@ionic/storage-angular';

@NgModule({
  declarations: [AppComponent],
  imports: [
    BrowserModule,
    IonicModule.forRoot(),
    AppRoutingModule,
    IonicStorageModule.forRoot()
  ],
  bootstrap: [AppComponent]
})
export class AppModule {}

Using Ionic Storage

Example

import { Storage } from '@ionic/storage-angular';

constructor(private storage: Storage) {
  this.init();
}

async init() {
  await this.storage.create();
}

// Save data
this.storage.set('username', 'john_doe');

// Retrieve data
this.storage.get('username').then((val) => {
  console.log(val); // Output: john_doe
});

// Remove data
this.storage.remove('username');

Advantages

  • Unified API for different storage engines.
  • Asynchronous API.
  • Easy to set up and use.

Summary

In this section, we explored different methods for storing data locally in an Ionic application:

  • Local Storage: Simple and easy to use but limited in capacity and functionality.
  • SQLite: Powerful and suitable for complex data storage needs.
  • Ionic Storage: Provides a unified API for different storage engines, making it flexible and easy to use.

Choosing the right storage mechanism depends on your application's requirements, such as the amount of data, complexity of queries, and need for offline functionality.
