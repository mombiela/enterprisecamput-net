Introduction to Ionic Router

The Ionic Router is a powerful tool that allows you to manage navigation within your Ionic applications. It is built on top of Angular's Router, providing a seamless way to handle navigation and routing in your app.

Key Concepts

  • Routes: Define the paths and components that should be displayed.
  • Router Outlet: Acts as a placeholder for the routed components.
  • Navigation: Programmatically navigating between routes.
  • Guards: Protect routes from unauthorized access.

Setting Up Ionic Router

To use the Ionic Router, you need to ensure that your project is set up correctly.

Installing Dependencies

Ensure you have the necessary dependencies installed:

npm install @ionic/angular @angular/router

Configuring Routes

Define your routes in the app-routing.module.ts file:

import { NgModule } from '@angular/core';
import { PreloadAllModules, RouterModule, Routes } from '@angular/router';

const routes: Routes = [
  {
    path: '',
    redirectTo: 'home',
    pathMatch: 'full'
  },
  {
    path: 'home',
    loadChildren: () => import('./home/home.module').then(m => m.HomePageModule)
  },
  {
    path: 'details',
    loadChildren: () => import('./details/details.module').then(m => m.DetailsPageModule)
  }
];

@NgModule({
  imports: [
    RouterModule.forRoot(routes, { preloadingStrategy: PreloadAllModules })
  ],
  exports: [RouterModule]
})
export class AppRoutingModule { }

Using Router Outlet

The RouterOutlet directive is used to display the routed components:

<ion-app>
  <ion-router-outlet></ion-router-outlet>
</ion-app>

Navigating Between Routes

You can navigate between routes using the Router service or the routerLink directive.

Using Router Service

Inject the Router service and use the navigate method:

import { Component } from '@angular/core';
import { Router } from '@angular/router';

@Component({
  selector: 'app-home',
  templateUrl: 'home.page.html',
  styleUrls: ['home.page.scss'],
})
export class HomePage {
  constructor(private router: Router) {}

  goToDetails() {
    this.router.navigate(['/details']);
  }
}

Using routerLink Directive

Use the routerLink directive in your templates:

<ion-button [routerLink]="['/details']">Go to Details</ion-button>

Route Guards

Route guards can be used to control access to certain routes.

Creating a Guard

Generate a guard using Angular CLI:

ng generate guard auth

Implementing the Guard

Implement the CanActivate interface in your guard:

import { Injectable } from '@angular/core';
import { CanActivate, ActivatedRouteSnapshot, RouterStateSnapshot, UrlTree } from '@angular/router';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class AuthGuard implements CanActivate {
  canActivate(
    next: ActivatedRouteSnapshot,
    state: RouterStateSnapshot): Observable<boolean | UrlTree> | Promise<boolean | UrlTree> | boolean | UrlTree {
    // Add your authentication logic here
    return true; // or false based on authentication status
  }
}

Applying the Guard to Routes

Apply the guard to your routes in app-routing.module.ts:

const routes: Routes = [
  {
    path: 'details',
    loadChildren: () => import('./details/details.module').then(m => m.DetailsPageModule),
    canActivate: [AuthGuard]
  }
];

Conclusion

The Ionic Router is an essential part of building Ionic applications, providing a robust way to manage navigation and routing. By understanding routes, router outlets, navigation methods, and route guards, you can create dynamic and secure applications. Practice implementing these concepts in your projects to gain a deeper understanding and proficiency in using the Ionic Router.
