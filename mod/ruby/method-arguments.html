Introduction

In Ruby, methods can take arguments to provide input data for their operations. Understanding how to use method arguments effectively is crucial for writing flexible and reusable code. This section will cover the basics of method arguments, including different types of arguments and how to use them.

Basic Method Arguments

Defining Methods with Arguments

To define a method that takes arguments, you specify the argument names in parentheses after the method name.

def greet(name)
  puts "Hello, #{name}!"
end

greet("Alice")  # Output: Hello, Alice!
  • Key Concept: The name inside the parentheses is a parameter, and "Alice" is an argument passed to the method.

Multiple Arguments

You can define methods that take multiple arguments by separating them with commas.

def add(a, b)
  a + b
end

puts add(3, 4)  # Output: 7
  • Key Concept: a and b are parameters, and 3 and 4 are arguments passed to the method.

Default Arguments

Ruby allows you to set default values for method arguments. If an argument is not provided, the default value is used.

def greet(name = "World")
  puts "Hello, #{name}!"
end

greet()         # Output: Hello, World!
greet("Alice")  # Output: Hello, Alice!
  • Key Concept: name = "World" sets a default value for the name parameter.

Keyword Arguments

Keyword arguments are a way to define methods with named parameters, making the code more readable and flexible.

def print_address(street:, city:, zip:)
  puts "#{street}, #{city}, #{zip}"
end

print_address(street: "123 Main St", city: "Springfield", zip: "12345")
  • Key Concept: street:, city:, and zip: are keyword arguments, and they must be provided when calling the method.

Default Keyword Arguments

You can also set default values for keyword arguments.

def print_address(street:, city:, zip: "00000")
  puts "#{street}, #{city}, #{zip}"
end

print_address(street: "123 Main St", city: "Springfield")  # Output: 123 Main St, Springfield, 00000
  • Key Concept: zip: "00000" sets a default value for the zip keyword argument.

Variable-Length Arguments

Ruby allows methods to accept a variable number of arguments using the splat operator (*).

def sum(*numbers)
  numbers.reduce(0) { |total, num| total + num }
end

puts sum(1, 2, 3)  # Output: 6
puts sum(4, 5)     # Output: 9
  • Key Concept: *numbers collects all arguments into an array called numbers.

Block Arguments

Methods in Ruby can also take blocks as arguments, which are chunks of code that can be executed within the method.

def repeat(n)
  n.times { yield }
end

repeat(3) { puts "Hello!" }
# Output:
# Hello!
# Hello!
# Hello!
  • Key Concept: yield is used to call the block passed to the method.

Conclusion

Understanding method arguments in Ruby is essential for writing clear and flexible code. This section covered:

  • Basic method arguments and multiple arguments.
  • Default arguments and keyword arguments.
  • Variable-length arguments using the splat operator.
  • Block arguments for passing chunks of code.

By mastering these concepts, you can create more dynamic and reusable methods in your Ruby programs. Practice using these different types of arguments to become proficient in writing versatile Ruby methods.
