Introduction to Methods

Methods in Ruby are a way to group reusable code into a single unit. They help in making the code more modular, readable, and maintainable. A method in Ruby is defined using the def keyword followed by the method name and optional parameters.

Key Concepts

  • Method Definition: The syntax for defining a method.
  • Method Invocation: How to call or invoke a method.
  • Parameters and Arguments: Passing data to methods.
  • Return Values: What a method returns after execution.
  • Scope: Understanding variable scope within methods.

Method Definition

To define a method in Ruby, you use the def keyword followed by the method name. Here is the basic syntax:

def method_name
  # code to be executed
end

Example

def say_hello
  puts "Hello, World!"
end

In this example, say_hello is a method that prints "Hello, World!" to the console.

Method Invocation

To invoke or call a method, you simply use the method name followed by parentheses (optional if no arguments are passed).

Example

say_hello

This will output:

Hello, World!

Parameters and Arguments

Methods can take parameters, which are variables that hold the values passed to the method when it is called.

Example

def greet(name)
  puts "Hello, #{name}!"
end

greet("Alice")

This will output:

Hello, Alice!

Multiple Parameters

You can define methods with multiple parameters.

def add(a, b)
  a + b
end

puts add(2, 3)

This will output:

5

Return Values

By default, a Ruby method returns the value of the last evaluated expression. You can also use the return keyword to explicitly return a value.

Example

def multiply(a, b)
  return a * b
end

puts multiply(4, 5)

This will output:

20

Scope

Variables defined inside a method are not accessible outside of it. This is known as variable scope.

Example

def example_method
  local_var = "I'm inside the method"
  puts local_var
end

example_method
# puts local_var # This will raise an error

This will output:

I'm inside the method

Attempting to access local_var outside the method will result in an error.

Advanced Method Concepts

Default Parameters

You can assign default values to parameters. If no argument is passed, the default value is used.

def greet(name="World")
  puts "Hello, #{name}!"
end

greet
greet("Alice")

This will output:

Hello, World!
Hello, Alice!

Variable Number of Arguments

Ruby allows methods to accept a variable number of arguments using the splat operator (*).

def sum(*numbers)
  numbers.inject(0) { |sum, number| sum + number }
end

puts sum(1, 2, 3, 4)

This will output:

10

Keyword Arguments

Ruby also supports keyword arguments, which are a way to pass arguments using key-value pairs.

def print_address(city:, state:)
  puts "City: #{city}, State: #{state}"
end

print_address(city: "San Francisco", state: "CA")

This will output:

City: San Francisco, State: CA

Conclusion

Defining methods in Ruby is a fundamental skill that helps in writing clean, reusable, and maintainable code. By understanding how to define methods, pass parameters, handle return values, and manage scope, you can create more complex and efficient Ruby programs. As you advance, you will also encounter more sophisticated features like default parameters, variable arguments, and keyword arguments, which provide additional flexibility and power in method definitions.
