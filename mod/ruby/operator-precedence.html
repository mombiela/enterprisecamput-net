Operator Precedence in Ruby
Conditional Statements ►

Understanding operator precedence is crucial for writing clear and correct Ruby code. Operator precedence determines the order in which parts of an expression are evaluated. In this section, we will cover the basics of operator precedence, provide examples, and offer exercises to solidify your understanding.

Key Concepts

  • Operator Precedence: The rules that determine the order in which different operators in an expression are evaluated.
  • Associativity: The direction (left-to-right or right-to-left) in which operators of the same precedence level are evaluated.

Basic Operator Precedence

In Ruby, operators have different levels of precedence. Here is a simplified list of some common operators in order of precedence from highest to lowest:

  1. ** (Exponentiation)
  2. !, ~, +, - (Unary operators)
  3. *, /, % (Multiplicative operators)
  4. +, - (Additive operators)
  5. <<, >> (Bitwise shift operators)
  6. & (Bitwise AND)
  7. ^, | (Bitwise OR and XOR)
  8. >, >=, <, <= (Comparison operators)
  9. ==, !=, ===, =~, !~ (Equality operators)
  10. && (Logical AND)
  11. || (Logical OR)
  12. .., ... (Range operators)
  13. ? : (Ternary conditional operator)
  14. =, +=, -=, *=, /=, %= (Assignment operators)
  15. not (Logical NOT)
  16. or, and (Logical OR and AND)

Examples

Example 1: Basic Arithmetic

Consider the following expression:

result = 2 + 3 * 4
  • Explanation: According to operator precedence, the multiplication (*) operator has higher precedence than the addition (+) operator. Therefore, 3 * 4 is evaluated first, resulting in 12. Then, 2 + 12 is evaluated, resulting in 14.

Example 2: Using Parentheses

You can use parentheses to change the order of evaluation:

result = (2 + 3) * 4
  • Explanation: Here, the expression inside the parentheses (2 + 3) is evaluated first, resulting in 5. Then, 5 * 4 is evaluated, resulting in 20.

Example 3: Logical Operators

Consider the following logical expression:

result = true || false && false
  • Explanation: The logical AND (&&) operator has higher precedence than the logical OR (||) operator. Therefore, false && false is evaluated first, resulting in false. Then, true || false is evaluated, resulting in true.

Exercises

Exercise 1: Evaluate the Expression

Evaluate the following expression and determine the result:

result = 5 + 3 * 2 - 8 / 4

Exercise 2: Parentheses Usage

Rewrite the following expression using parentheses to change the order of evaluation so that addition is performed before multiplication:

result = 4 + 2 * 3

Exercise 3: Logical Operators

Evaluate the following logical expression:

result = (true && false) || (false || true)

Conclusion

Understanding operator precedence is essential for writing correct and efficient Ruby code. By knowing the order in which operators are evaluated, you can avoid common pitfalls and ensure that your expressions produce the expected results. Remember to use parentheses to explicitly define the order of evaluation when necessary. Practice with the provided exercises to reinforce your understanding of operator precedence in Ruby.
