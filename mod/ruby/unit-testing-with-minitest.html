Introduction to Unit Testing

  • Definition: Unit testing involves testing individual units or components of a software to ensure they work as intended.
  • Importance: Helps in identifying bugs early, ensures code reliability, and facilitates refactoring.

Getting Started with Minitest

  • What is Minitest?: A testing library for Ruby that provides a complete suite of testing facilities supporting TDD, BDD, mocking, and benchmarking.
  • Installation: 
    gem install minitest

Writing Your First Test

  • Creating a Test File: Typically, test files are placed in a test directory.
  • Basic Structure: 
    require 'minitest/autorun'

class ExampleTest < Minitest::Test
  def test_example
    assert_equal 4, 2 + 2
  end
end
    • require 'minitest/autorun': Loads Minitest and runs the tests automatically.
    • class ExampleTest < Minitest::Test: Defines a test class inheriting from Minitest::Test.
    • def test_example: Defines a test method. Test methods should start with test_.
    • assert_equal 4, 2 + 2: An assertion that checks if the expression 2 + 2 equals 4.

Common Assertions

  • assert_equal: Checks if two values are equal. 
    assert_equal expected, actual
  • assert: Checks if a condition is true. 
    assert condition
  • assert_nil: Checks if a value is nil. 
    assert_nil value
  • assert_raises: Checks if a specific exception is raised. 
    assert_raises(ExpectedException) { code }

Advanced Testing Techniques

  • Setup and Teardown: Methods to run code before and after each test. 
    class AdvancedTest < Minitest::Test
  def setup
    @value = 2
  end

  def teardown
    # Clean up code
  end

  def test_value
    assert_equal 2, @value
  end
end
  • Mocking and Stubbing: Simulating objects or methods to isolate tests. 
    require 'minitest/mock'

def test_mock
  mock = Minitest::Mock.new
  mock.expect(:foo, true)

  assert mock.foo
  mock.verify
end

Organizing Tests

  • Test Directory Structure: 
    project_root/
├── lib/
│   └── example.rb
└── test/
    ├── example_test.rb
    └── test_helper.rb
  • Test Helper: A file to require common dependencies and setup configurations. 
    # test/test_helper.rb
require 'minitest/autorun'

Running Tests

  • Command Line: Run all tests with: 
    ruby -Ilib:test test/example_test.rb
  • Rake: Using Rake to automate running tests. 
    # Rakefile
require 'rake/testtask'

Rake::TestTask.new do |t|
  t.libs << 'test'
  t.test_files = FileList['test/**/*_test.rb']
end

task default: :test

Conclusion

Unit testing with Minitest is a powerful way to ensure your Ruby code is reliable and maintainable. By writing tests, you can catch bugs early, refactor with confidence, and build a robust codebase. Start with simple assertions and gradually incorporate more advanced techniques like mocking and setup/teardown methods to enhance your testing suite.
