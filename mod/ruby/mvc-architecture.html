Introduction to MVC Architecture

Model-View-Controller (MVC) is a design pattern used to separate concerns in software applications. It divides an application into three interconnected components:

  • Model: Manages the data and business logic.
  • View: Handles the display and user interface.
  • Controller: Manages user input and interacts with the Model and View.

Understanding the Components

Model

The Model represents the data and the rules that govern access to and updates of this data. In Ruby, this often involves classes that interact with the database.

# app/models/article.rb
class Article < ApplicationRecord
  validates :title, presence: true
  validates :body, presence: true
end
  • Explanation: This Article class inherits from ApplicationRecord, which means it is connected to the database. The validates methods ensure that both title and body must be present for an Article object to be valid.

View

The View is responsible for rendering the user interface. In Ruby on Rails, views are typically written in Embedded Ruby (ERB) or other templating languages.

<!-- app/views/articles/show.html.erb -->
<h1><%= @article.title %></h1>
<p><%= @article.body %></p>
  • Explanation: This ERB template displays the title and body of an article. The <%= %> tags are used to embed Ruby code within the HTML.

Controller

The Controller handles the user input, interacts with the Model, and renders the appropriate View.

# app/controllers/articles_controller.rb
class ArticlesController < ApplicationController
  def show
    @article = Article.find(params[:id])
  end
end
  • Explanation: The show action in the ArticlesController finds an article by its ID (passed in the URL parameters) and assigns it to an instance variable @article, which is then accessible in the view.

How MVC Components Interact

  1. User Interaction: The user interacts with the View (e.g., clicking a link).
  2. Controller Action: The Controller receives the input, processes it (e.g., fetching data from the Model), and determines the appropriate response.
  3. Model Interaction: The Controller may interact with the Model to retrieve or update data.
  4. View Rendering: The Controller passes data to the View, which renders the final output to the user.

Example Flow

  • User: Clicks on a link to view an article.
  • Controller: ArticlesController's show action is triggered.
  • Model: The Article model fetches the article from the database.
  • View: The show.html.erb template is rendered with the article data.

Advanced Concepts

Routing

Routing in Rails maps URLs to controller actions.

# config/routes.rb
Rails.application.routes.draw do
  resources :articles, only: [:show]
end
  • Explanation: This route declaration creates a route for the show action of the ArticlesController.

Partial Views

Partial views allow you to reuse view code.

<!-- app/views/articles/_article.html.erb -->
<div class="article">
  <h2><%= article.title %></h2>
  <p><%= article.body %></p>
</div>
  • Explanation: This partial can be rendered in other views to display an article.
<!-- app/views/articles/index.html.erb -->
<% @articles.each do |article| %>
  <%= render partial: "article", locals: { article: article } %>
<% end %>
  • Explanation: The index view renders the _article partial for each article in the @articles collection.

Strong Parameters

Strong parameters provide a way to whitelist attributes that can be mass-assigned.

# app/controllers/articles_controller.rb
class ArticlesController < ApplicationController
  def create
    @article = Article.new(article_params)
    if @article.save
      redirect_to @article
    else
      render 'new'
    end
  end

  private

  def article_params
    params.require(:article).permit(:title, :body)
  end
end
  • Explanation: The article_params method ensures only the title and body attributes can be mass-assigned.

Conclusion

The MVC architecture is a powerful design pattern that helps in organizing code and separating concerns in a Ruby on Rails application. By understanding and implementing MVC, you can create scalable and maintainable applications. This topic covered the basics of MVC, how the components interact, and some advanced concepts to deepen your understanding.
