Creating a Simple Rails Application
Project Overview ►

Introduction

In this section, we'll walk through the process of creating a simple Rails application. Rails is a powerful web application framework written in Ruby, and it follows the Model-View-Controller (MVC) architecture. By the end of this section, you will have a basic understanding of how to set up a Rails project, create models, views, and controllers, and run your application.

Prerequisites

Before we begin, ensure you have the following installed:

  • Ruby (version 2.5 or higher)
  • Rails (version 6 or higher)
  • SQLite3 (or another database system)

Setting Up a New Rails Application

  1. Install Rails: If you haven't installed Rails yet, you can do so by running:

    gem install rails

  2. Create a New Rails Application: Use the Rails command-line tool to create a new application:

    rails new my_first_app

    This command will generate a new Rails project in a directory named my_first_app.

  3. Navigate to Your Application Directory:

    cd my_first_app

  4. Start the Rails Server: To see your application in action, start the Rails server:

    rails server

    Open your web browser and navigate to http://localhost:3000. You should see the Rails welcome page.

Creating a Model

Models in Rails are used to interact with the database. Let's create a simple model for a Post.

  1. Generate the Model:

    rails generate model Post title:string body:text

    This command creates a migration file, a model file, and test files for the Post model.

  2. Run the Migration:

    rails db:migrate

    This command applies the migration, creating the posts table in the database.

Creating a Controller

Controllers handle the web requests and responses. Let's create a controller for our Post model.

  1. Generate the Controller:

    rails generate controller Posts

    This command creates a controller file, view templates, and test files for the Posts controller.

  2. Define Actions in the Controller: Open app/controllers/posts_controller.rb and define actions for index, show, new, and create:

    class PostsController < ApplicationController
  def index
    @posts = Post.all
  end

  def show
    @post = Post.find(params[:id])
  end

  def new
    @post = Post.new
  end

  def create
    @post = Post.new(post_params)
    if @post.save
      redirect_to @post
    else
      render 'new'
    end
  end

  private

  def post_params
    params.require(:post).permit(:title, :body)
  end
end

Creating Views

Views are used to display the data to the user. Let's create views for our Post actions.

  1. Create the index View: Create a file app/views/posts/index.html.erb:

    <h1>All Posts</h1>
<ul>
  <% @posts.each do |post| %>
    <li>
      <%= link_to post.title, post %>
    </li>
  <% end %>
</ul>
<%= link_to 'New Post', new_post_path %>

  2. Create the show View: Create a file app/views/posts/show.html.erb:

    <h1><%= @post.title %></h1>
<p><%= @post.body %></p>
<%= link_to 'Back', posts_path %>

  3. Create the new View: Create a file app/views/posts/new.html.erb:

    <h1>New Post</h1>
<%= form_with model: @post, local: true do |form| %>
  <div>
    <%= form.label :title %>
    <%= form.text_field :title %>
  </div>
  <div>
    <%= form.label :body %>
    <%= form.text_area :body %>
  </div>
  <div>
    <%= form.submit %>
  </div>
<% end %>
<%= link_to 'Back', posts_path %>

Routing

To connect the controller actions to URLs, we need to define routes.

  1. Define Resourceful Routes: Open config/routes.rb and add: 
    Rails.application.routes.draw do
  resources :posts
  root 'posts#index'
end

Running the Application

  1. Start the Rails Server: 
    rails server
  2. Access the Application: Open your web browser and navigate to http://localhost:3000. You should see the list of posts and be able to create new posts.

Conclusion

In this section, we covered the basics of creating a simple Rails application. We set up a new Rails project, created a model, a controller, and views, and defined routes to connect everything. This is just the beginning; Rails offers many more features and tools to build complex web applications. Continue exploring Rails to deepen your understanding and build more sophisticated applications.
