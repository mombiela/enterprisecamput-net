Introduction to Modules

Modules in Ruby are a way to group related methods, constants, and class variables. They serve two main purposes: namespacing and mixins.

  • Namespacing: Prevents name clashes by grouping related classes, methods, and constants.
  • Mixins: Allows sharing of methods across multiple classes.

Creating a Module

To define a module, use the module keyword followed by the module name. Module names are constants and should start with an uppercase letter.

module Greetings
  def say_hello
    "Hello!"
  end

  def say_goodbye
    "Goodbye!"
  end
end

Using Modules for Namespacing

Namespacing helps organize code and avoid conflicts with method and class names.

module MathTools
  PI = 3.14159

  def self.square(x)
    x * x
  end
end

puts MathTools::PI          # => 3.14159
puts MathTools.square(3)    # => 9

Mixins with Modules

Mixins allow you to include module methods into classes. This is done using the include or extend keywords.

Including Modules

When you include a module, its methods become instance methods of the class.

module Greetings
  def say_hello
    "Hello!"
  end
end

class Person
  include Greetings
end

person = Person.new
puts person.say_hello  # => "Hello!"

Extending Modules

When you extend a module, its methods become class methods.

module Greetings
  def say_hello
    "Hello!"
  end
end

class Person
  extend Greetings
end

puts Person.say_hello  # => "Hello!"

Advanced Module Features

Module Constants

Modules can contain constants which can be accessed using the scope resolution operator ::.

module Config
  MAX_USERS = 100
end

puts Config::MAX_USERS  # => 100

Module Methods

Modules can have methods defined with self. to make them module methods.

module MathTools
  def self.square(x)
    x * x
  end
end

puts MathTools.square(4)  # => 16

Including Multiple Modules

A class can include multiple modules, allowing it to inherit methods from all included modules.

module Greetings
  def say_hello
    "Hello!"
  end
end

module Farewells
  def say_goodbye
    "Goodbye!"
  end
end

class Person
  include Greetings
  include Farewells
end

person = Person.new
puts person.say_hello   # => "Hello!"
puts person.say_goodbye # => "Goodbye!"

Practical Exercises

  1. Create a module named MathOperations with methods for addition, subtraction, multiplication, and division. Include this module in a class and demonstrate its usage.
module MathOperations
  def add(a, b)
    a + b
  end

  def subtract(a, b)
    a - b
  end

  def multiply(a, b)
    a * b
  end

  def divide(a, b)
    a / b
  end
end

class Calculator
  include MathOperations
end

calc = Calculator.new
puts calc.add(5, 3)        # => 8
puts calc.subtract(5, 3)   # => 2
puts calc.multiply(5, 3)   # => 15
puts calc.divide(6, 3)     # => 2
  1. Define a module Trackable with a method track that prints "Tracking...". Include this module in a class Vehicle and call the track method.
module Trackable
  def track
    "Tracking..."
  end
end

class Vehicle
  include Trackable
end

car = Vehicle.new
puts car.track  # => "Tracking..."

Conclusion

Modules and mixins in Ruby provide a powerful way to organize and reuse code. By using modules for namespacing, you can avoid name clashes and keep your codebase clean. Mixins allow you to share methods across multiple classes, promoting code reuse and reducing duplication. Understanding and effectively using modules and mixins is essential for writing maintainable and scalable Ruby applications.
