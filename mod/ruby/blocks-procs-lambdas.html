Introduction

In Ruby, blocks, procs, and lambdas are powerful tools for handling chunks of code. They allow you to pass code around like variables, making your programs more flexible and concise. This section will guide you through understanding and using these features effectively.

Blocks

What is a Block?

  • A block is an anonymous piece of code that can be passed to methods.
  • Blocks are not objects, but they can be converted into objects using Proc.

Syntax

Blocks can be defined using either do...end or curly braces {...}.

# Using do...end
[1, 2, 3].each do |number|
  puts number
end

# Using curly braces
[1, 2, 3].each { |number| puts number }

Yielding to a Block

  • Methods can yield control to a block using the yield keyword.
def greet
  yield "Hello"
end

greet { |greeting| puts greeting }  # Output: Hello

Block Parameters

  • Blocks can take parameters, which are defined between vertical bars | |.
def greet
  yield "Hello", "World"
end

greet { |greeting, name| puts "#{greeting}, #{name}!" }  # Output: Hello, World!

Procs

What is a Proc?

  • A Proc is an object that encapsulates a block of code, which can be stored in a variable, passed to methods, or called.

Creating a Proc

  • Use Proc.new or the proc method to create a Proc.
# Using Proc.new
my_proc = Proc.new { |name| puts "Hello, #{name}!" }

# Using proc
my_proc = proc { |name| puts "Hello, #{name}!" }

Calling a Proc

  • Call a Proc using the call method.
my_proc.call("Alice")  # Output: Hello, Alice!

Proc vs Block

| Feature | Block | Proc | |-------------------|--------------------------------|--------------------------------| | Definition | Anonymous, not an object | Object, can be stored in a variable | | Passing to Methods| Implicitly | Explicitly | | Multiple Blocks | No | Yes |

Lambdas

What is a Lambda?

  • A lambda is a special kind of Proc with stricter argument checking and different behavior for return.

Creating a Lambda

  • Use the lambda method or the -> syntax.
# Using lambda method
my_lambda = lambda { |name| puts "Hello, #{name}!" }

# Using -> syntax
my_lambda = ->(name) { puts "Hello, #{name}!" }

Calling a Lambda

  • Call a lambda using the call method.
my_lambda.call("Bob")  # Output: Hello, Bob!

Lambda vs Proc

| Feature | Proc | Lambda | |-------------------|--------------------------------|--------------------------------| | Argument Checking | Lenient | Strict | | Return Behavior | Returns from the enclosing method | Returns from the lambda itself |

Conclusion

Understanding blocks, procs, and lambdas is essential for writing flexible and concise Ruby code. Blocks are useful for passing chunks of code to methods, while procs and lambdas provide more control and flexibility by allowing you to store and reuse code. Lambdas offer stricter argument checking and different return behavior compared to procs. By mastering these concepts, you can write more powerful and maintainable Ruby programs.
