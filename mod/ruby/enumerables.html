Enumerables in Ruby
Classes and Objects ►

Enumerables are a fundamental part of Ruby, providing a powerful set of methods to traverse, search, sort, and manipulate collections. This topic will cover the essentials of Enumerables, from basic concepts to advanced usage.

Introduction to Enumerables

  • Definition: An Enumerable is a module that provides a set of methods for traversing, searching, sorting, and manipulating collections.
  • Common Classes: Arrays, Hashes, and Ranges are common classes that include the Enumerable module.

Basic Methods

each

The each method is the most basic way to iterate over a collection.

[1, 2, 3].each do |number|
  puts number
end
  • Explanation: This code iterates over the array [1, 2, 3] and prints each number.

map

The map method creates a new array containing the values returned by the block.

squares = [1, 2, 3].map do |number|
  number * 2
end
puts squares.inspect
  • Explanation: This code multiplies each number in the array by 2 and returns a new array [2, 4, 6].

select

The select method returns a new array containing all elements for which the given block returns a true value.

evens = [1, 2, 3, 4].select do |number|
  number.even?
end
puts evens.inspect
  • Explanation: This code selects even numbers from the array, resulting in [2, 4].

Intermediate Methods

`reduce`

The reduce method combines all elements of an enumerable by applying a binary operation, specified by a block or a symbol.

sum = [1, 2, 3, 4].reduce(0) do |accumulator, number|
  accumulator + number
end
puts sum
  • Explanation: This code calculates the sum of the array elements, resulting in 10.

`find`

The find method returns the first element for which the block returns a true value.

first_even = [1, 2, 3, 4].find do |number|
  number.even?
end
puts first_even
  • Explanation: This code finds the first even number in the array, which is 2.

`all?` and `any?`

  • all?: Returns true if the block never returns false or nil for all elements.
  • any?: Returns true if the block ever returns a value other than false or nil for any element.
all_even = [2, 4, 6].all? { |number| number.even? }
any_even = [1, 3, 4].any? { |number| number.even? }

puts all_even  # true
puts any_even  # true

Advanced Methods

`group_by`

The group_by method groups the collection by the result of the block.

grouped = [1, 2, 3, 4, 5, 6].group_by do |number|
  number.even?
end
puts grouped.inspect
  • Explanation: This code groups numbers into even and odd, resulting in {false=>[1, 3, 5], true=>[2, 4, 6]}.

`partition`

The partition method divides the collection into two arrays, the first containing elements for which the block evaluates to true, the second containing the rest.

evens, odds = [1, 2, 3, 4, 5, 6].partition do |number|
  number.even?
end
puts evens.inspect  # [2, 4, 6]
puts odds.inspect   # [1, 3, 5]

`lazy`

The lazy method allows for lazy enumeration, which can be useful for handling large or infinite collections.

infinite = (1..Float::INFINITY).lazy
first_ten = infinite.first(10)
puts first_ten.inspect
  • Explanation: This code creates a lazy enumerator for an infinite range and retrieves the first 10 elements.

Conclusion

Enumerables in Ruby provide a rich set of methods for working with collections. From basic iteration with each to advanced grouping with group_by, understanding these methods can greatly enhance your ability to write efficient and readable Ruby code. Practice using these methods to become proficient in manipulating collections in Ruby.
