Introduction to Variable Scope

Variable scope refers to the context within a program where a variable is accessible. Understanding variable scope is crucial for writing efficient and bug-free code. In Ruby, variable scope is determined by where the variable is declared and the type of variable.

Types of Variables in Ruby

Ruby has several types of variables, each with its own scope:

  • Local Variables: Accessible only within the block of code where they are declared.
  • Instance Variables: Accessible within the instance of a class.
  • Class Variables: Shared among all instances of a class.
  • Global Variables: Accessible from anywhere in the Ruby program.
  • Constants: Accessible throughout the application but should not be changed once assigned.

Local Variables

Local variables are defined within a method, block, or loop and are only accessible within that context.

def example_method
  local_var = "I'm a local variable"
  puts local_var
end

example_method
# Output: I'm a local variable

puts local_var
# Error: undefined local variable or method `local_var'

Key Points:

  • Local variables start with a lowercase letter or an underscore.
  • They are not accessible outside the method or block where they are defined.

Instance Variables

Instance variables are prefixed with @ and are accessible within the instance of a class.

class ExampleClass
  def initialize(name)
    @name = name
  end

  def display_name
    puts @name
  end
end

example = ExampleClass.new("Ruby")
example.display_name
# Output: Ruby

Key Points:

  • Instance variables are used to store object-specific data.
  • They are accessible across methods within the same instance.

Class Variables

Class variables are prefixed with @@ and are shared among all instances of a class.

class ExampleClass
  @@class_var = 0

  def initialize
    @@class_var += 1
  end

  def self.class_var
    @@class_var
  end
end

example1 = ExampleClass.new
example2 = ExampleClass.new

puts ExampleClass.class_var
# Output: 2

Key Points:

  • Class variables store data that is shared among all instances of a class.
  • They can be accessed and modified by any instance of the class.

Global Variables

Global variables are prefixed with $ and are accessible from anywhere in the Ruby program.

$global_var = "I'm a global variable"

def display_global_var
  puts $global_var
end

display_global_var
# Output: I'm a global variable

$global_var = "I've been changed"
display_global_var
# Output: I've been changed

Key Points:

  • Global variables can be accessed from any part of the program.
  • They should be used sparingly as they can make debugging difficult.

Constants

Constants are defined with an uppercase letter and are meant to remain unchanged.

CONSTANT_VAR = "I'm a constant"

def display_constant
  puts CONSTANT_VAR
end

display_constant
# Output: I'm a constant

# Attempting to change a constant will generate a warning
CONSTANT_VAR = "I've been changed"
# Warning: already initialized constant CONSTANT_VAR

Key Points:

  • Constants are accessible throughout the application.
  • They should not be reassigned once set.

Summary

Understanding variable scope is essential for managing data and avoiding bugs in Ruby programs. Here's a quick recap:

  • Local Variables: Limited to the method or block where they are defined.
  • Instance Variables: Accessible within the instance of a class.
  • Class Variables: Shared among all instances of a class.
  • Global Variables: Accessible from anywhere in the program.
  • Constants: Should remain unchanged and are accessible throughout the application.

By mastering variable scope, you can write more efficient and maintainable Ruby code.
