Access Control in Ruby
Reading Files ►

Access control is a fundamental concept in object-oriented programming that helps manage the visibility and accessibility of methods and attributes in a class. In Ruby, access control is managed using three main keywords: public, protected, and private.

Public Methods

  • Definition: Public methods can be called by anyone—there are no restrictions on their visibility.
  • Default: By default, all methods in a Ruby class are public unless specified otherwise.
class Greeter
  def greet
    puts "Hello!"
  end
end

greeter = Greeter.new
greeter.greet  # Output: Hello!

Protected Methods

  • Definition: Protected methods can be called by any instance of the defining class or its subclasses. However, they cannot be called with an explicit receiver outside the class hierarchy.
  • Use Case: Typically used for methods that should be accessible within the class and its subclasses but not from outside.
class Person
  def initialize(name)
    @name = name
  end

  def compare_name(other_person)
    self.name == other_person.name
  end

  protected

  def name
    @name
  end
end

alice = Person.new("Alice")
bob = Person.new("Bob")
puts alice.compare_name(bob)  # Output: false

Private Methods

  • Definition: Private methods can only be called within the context of the current object. They cannot be called with an explicit receiver, even if that receiver is self.
  • Use Case: Used for internal utility methods that should not be exposed outside the class.
class BankAccount
  def initialize(balance)
    @balance = balance
  end

  def display_balance
    puts "Your balance is $#{balance}"
  end

  private

  def balance
    @balance
  end
end

account = BankAccount.new(1000)
account.display_balance  # Output: Your balance is $1000
# account.balance  # This will raise an error: private method `balance' called

Access Control Keywords

  • public: Methods following this keyword are accessible from anywhere.
  • protected: Methods following this keyword are accessible within the class and its subclasses.
  • private: Methods following this keyword are only accessible within the context of the current object.

Example of Using Access Control Keywords

class Example
  def public_method
    puts "This is a public method"
  end

  protected

  def protected_method
    puts "This is a protected method"
  end

  private

  def private_method
    puts "This is a private method"
  end
end

example = Example.new
example.public_method  # Output: This is a public method
# example.protected_method  # Raises an error: protected method `protected_method' called
# example.private_method  # Raises an error: private method `private_method' called

Summary

Access control in Ruby is a crucial concept for managing the visibility and accessibility of methods within a class. By using public, protected, and private keywords, you can control how methods are accessed and ensure that your class's internal workings are appropriately encapsulated. Understanding and applying these access control mechanisms will help you write more secure and maintainable code.
