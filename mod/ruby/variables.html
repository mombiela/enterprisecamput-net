Introduction to Variables

Variables are fundamental in any programming language, and Ruby is no exception. They allow you to store and manipulate data throughout your program. In this section, we'll cover the basics of variables in Ruby, including how to create them, different types of variables, and best practices.

Creating Variables

In Ruby, you create a variable by simply assigning a value to a name. Ruby is dynamically typed, which means you don't need to declare the type of the variable.

name = "Alice"
age = 30
is_student = true
  • name is a variable that stores a string.
  • age is a variable that stores an integer.
  • is_student is a variable that stores a boolean.

Types of Variables

Ruby has several types of variables, each with its own scope and naming conventions.

Local Variables

Local variables are the most common type of variable. They are only accessible within the context where they are defined, such as within a method or a block.

def greet
  greeting = "Hello, World!"
  puts greeting
end

greet
# Output: Hello, World!
  • Local variables start with a lowercase letter or an underscore (_).

Instance Variables

Instance variables are used in object-oriented programming. They are accessible within the instance of a class.

class Person
  def initialize(name, age)
    @name = name
    @age = age
  end

  def display_info
    puts "Name: #{@name}, Age: #{@age}"
  end
end

person = Person.new("Alice", 30)
person.display_info
# Output: Name: Alice, Age: 30
  • Instance variables start with an @ symbol.

Class Variables

Class variables are shared among all instances of a class. They are used when you want to store information that is common to all objects of a class.

class Person
  @@species = "Homo sapiens"

  def self.species
    @@species
  end
end

puts Person.species
# Output: Homo sapiens
  • Class variables start with @@.

Global Variables

Global variables are accessible from anywhere in the Ruby program. They should be used sparingly as they can make debugging difficult.

$global_var = "I am a global variable"

def display_global
  puts $global_var
end

display_global
# Output: I am a global variable
  • Global variables start with a $ symbol.

Constants

Constants are variables that, once assigned, should not be changed. They are used to store values that remain constant throughout the program.

PI = 3.14159
puts PI
# Output: 3.14159
  • Constants start with an uppercase letter.

Variable Scope

The scope of a variable determines where it can be accessed within the program. Understanding scope is crucial for writing clean and maintainable code.

  • Local Scope: Variables defined within a method or block.
  • Instance Scope: Variables accessible within an instance of a class.
  • Class Scope: Variables shared across all instances of a class.
  • Global Scope: Variables accessible from anywhere in the program.

Best Practices

  • Use meaningful variable names to make your code more readable.
  • Avoid using global variables unless absolutely necessary.
  • Follow Ruby's naming conventions for different types of variables.
  • Keep the scope of variables as limited as possible to avoid unintended side effects.

Conclusion

Variables are a core concept in Ruby, allowing you to store and manipulate data efficiently. By understanding the different types of variables and their scopes, you can write more effective and maintainable code. Remember to follow best practices to ensure your code is clean and easy to understand.
