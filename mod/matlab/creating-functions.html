Introduction to Functions

Functions are a fundamental building block in MATLAB programming. They allow you to encapsulate code into reusable blocks, making your programs more modular, readable, and maintainable.

Key Concepts

  • Function Definition: How to define a function in MATLAB.
  • Input and Output Arguments: Handling inputs and outputs in functions.
  • Scope and Lifetime: Understanding variable scope within functions.
  • Anonymous Functions: Creating simple, unnamed functions.

Defining a Function

In MATLAB, a function is defined in a separate file with the same name as the function. The file should have a .m extension.

Basic Structure

function [output1, output2, ...] = functionName(input1, input2, ...)
    % Function description
    % Detailed explanation goes here
    
    % Function code
    output1 = ...;
    output2 = ...;
end

Example

Let's create a simple function that adds two numbers.

function result = addNumbers(a, b)
    % addNumbers adds two numbers a and b
    result = a + b;
end

To use this function, save it in a file named addNumbers.m and call it from the MATLAB command window or another script:

sum = addNumbers(3, 5);
disp(sum);  % Output: 8

Input and Output Arguments

Functions can have multiple input and output arguments. The number of inputs and outputs should match the function's definition when calling it.

Example with Multiple Outputs

function [sum, difference] = addAndSubtract(a, b)
    % addAndSubtract returns the sum and difference of a and b
    sum = a + b;
    difference = a - b;
end

Calling the function:

[sum, diff] = addAndSubtract(10, 4);
disp(sum);  % Output: 14
disp(diff); % Output: 6

Scope and Lifetime

Variables defined within a function are local to that function and cannot be accessed from outside. This is known as the variable's scope.

Example

function result = multiplyNumbers(a, b)
    % multiplyNumbers multiplies two numbers a and b
    product = a * b;  % 'product' is local to this function
    result = product;
end

Attempting to access product outside the function will result in an error.

Anonymous Functions

Anonymous functions are simple, one-line functions that do not require a separate file. They are useful for short operations.

Example

square = @(x) x^2;
result = square(5);
disp(result);  % Output: 25

Conclusion

Creating functions in MATLAB is a powerful way to organize and reuse code. By understanding how to define functions, handle input and output arguments, and manage variable scope, you can write more efficient and readable MATLAB programs. Additionally, anonymous functions provide a quick way to define simple operations without the need for separate files.
