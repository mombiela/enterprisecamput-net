Introduction

OWASP Dependency-Check is a software composition analysis (SCA) tool that identifies known vulnerabilities in a project's dependencies. It is crucial for maintaining application security, as many vulnerabilities can reside in third-party libraries and components.

Topic Objectives

  • Understand what OWASP Dependency-Check is.
  • Learn how to install and configure OWASP Dependency-Check.
  • Run dependency scans and understand the results.
  • Integrate OWASP Dependency-Check into a CI/CD workflow.

What is OWASP Dependency-Check?

OWASP Dependency-Check is an open-source tool that helps identify known vulnerabilities in a project's dependencies. It works by analyzing dependency configuration files (such as pom.xml for Maven or package.json for Node.js) and comparing library versions with a database of known vulnerabilities.

Key Features

  • Dependency Analysis: Scans configuration files to identify libraries and versions.
  • Vulnerability Database: Uses databases like NVD (National Vulnerability Database) to identify known issues.
  • Detailed Reports: Generates reports detailing the vulnerabilities found and their severity.
  • CI/CD Integration: Can be easily integrated into CI/CD pipelines for automated scans.

Installation and Configuration

Prerequisites

  • Java JDK 8 or higher.
  • Build tool like Maven, Gradle, or Node.js.

Installation

Using Maven

For Maven-based projects, add the following plugin to the pom.xml file:

<build>
    <plugins>
        <plugin>
            <groupId>org.owasp</groupId>
            <artifactId>dependency-check-maven</artifactId>
            <version>6.5.0</version>
            <executions>
                <execution>
                    <goals>
                        <goal>check</goal>
                    </goals>
                </execution>
            </executions>
        </plugin>
    </plugins>
</build>

Using Gradle

For Gradle-based projects, add the following plugin to the build.gradle file:

plugins {
    id "org.owasp.dependencycheck" version "6.5.0"
}

dependencyCheck {
    failBuildOnCVSS = 7
}

Using Node.js

For Node.js-based projects, install the dependency-check package:

npm install -g dependency-check

Running Dependency Scans

Maven

To run the scan on a Maven project, use the following command:

mvn dependency-check:check

Gradle

To run the scan on a Gradle project, use the following command:

./gradlew dependencyCheckAnalyze

Node.js

To run the scan on a Node.js project, use the following command:

dependency-check --project <project-name> --scan <path-to-project>

Understanding the Results

After running the scan, OWASP Dependency-Check will generate a detailed report. This report typically includes:

  • List of Dependencies: All dependencies identified in the project.
  • Found Vulnerabilities: Details of the vulnerabilities, including CVE (Common Vulnerabilities and Exposures) ID, description, and severity.
  • Recommendations: Suggestions to mitigate the vulnerabilities, such as updating to a more secure version of the library.

Sample Report

Dependency: com.fasterxml.jackson.core:jackson-databind:2.9.8
Vulnerability: CVE-2018-19360
Severity: HIGH
Description: FasterXML jackson-databind 2.x before 2.9.9 might allow remote attackers to conduct server-side request forgery (SSRF) attacks.
Recommendation: Update to version 2.9.9 or later.

CI/CD Integration

Integrating OWASP Dependency-Check into a CI/CD pipeline allows for automated dependency scans and ensures that vulnerabilities are continuously detected and addressed.

Jenkins

To integrate into Jenkins, add a shell execution step in your pipeline:

mvn dependency-check:check

GitLab CI

To integrate into GitLab CI, add the following script to your .gitlab-ci.yml file:

dependency-check:
  image: maven:3.6.3-jdk-8
  script:
    - mvn dependency-check:check
  artifacts:
    paths:
      - target/dependency-check-report.html

Conclusion

OWASP Dependency-Check is an essential tool for any development team using third-party dependencies. By integrating this tool into the development workflow, known vulnerabilities can be proactively identified and mitigated, thereby improving the overall security of applications.

Always remember to keep your dependencies updated and perform regular scans to ensure your project is protected against known vulnerabilities.
