Introduction to Beats

Beats are lightweight data shippers that you install on your servers to send operational data to Elasticsearch. They are designed to be simple and efficient, making it easy to collect and ship data from various sources.

  • Filebeat: Ships log files.
  • Metricbeat: Collects metrics from the operating system and services.
  • Packetbeat: Monitors network traffic.
  • Winlogbeat: Ships Windows event logs.
  • Auditbeat: Collects audit data from the Linux audit framework.
  • Heartbeat: Monitors the availability of services.

Installing and Configuring Beats

Installing Filebeat

  1. Download and install Filebeat:

    curl -L -O https://artifacts.elastic.co/downloads/beats/filebeat/filebeat-7.10.0-amd64.deb
sudo dpkg -i filebeat-7.10.0-amd64.deb

  2. Configure Filebeat: Edit the filebeat.yml configuration file to specify the log files to monitor and the Elasticsearch instance to send data to:

    filebeat.inputs:
- type: log
  enabled: true
  paths:
    - /var/log/*.log

output.elasticsearch:
  hosts: ["localhost:9200"]

  3. Start Filebeat:

    sudo service filebeat start

Installing Metricbeat

  1. Download and install Metricbeat:

    curl -L -O https://artifacts.elastic.co/downloads/beats/metricbeat/metricbeat-7.10.0-amd64.deb
sudo dpkg -i metricbeat-7.10.0-amd64.deb

  2. Configure Metricbeat: Edit the metricbeat.yml configuration file to specify the modules to enable and the Elasticsearch instance to send data to:

    metricbeat.modules:
- module: system
  metricsets:
    - cpu
    - memory
    - network
  enabled: true
  period: 10s
  processes: ['.*']

output.elasticsearch:
  hosts: ["localhost:9200"]

  3. Start Metricbeat:

    sudo service metricbeat start

Using Beats with Elasticsearch

Sending Data to Elasticsearch

Beats can be configured to send data directly to Elasticsearch or through Logstash for additional processing.

  • Direct to Elasticsearch: This is the simplest setup where Beats send data directly to Elasticsearch.
  • Through Logstash: This setup allows for more complex data processing and transformation before the data is indexed in Elasticsearch.

Example: Filebeat to Elasticsearch

  1. Configure Filebeat:

    filebeat.inputs:
- type: log
  enabled: true
  paths:
    - /var/log/*.log

  hosts: ["localhost:9200"]

  2. Start Filebeat:

    sudo service filebeat start

    - network
  enabled: true
  period: 10s
  processes: ['.*']

output.elasticsearch:
  hosts: ["localhost:9200"]

  2. Start Metricbeat:

Monitoring and Visualizing Data

Using Kibana

Kibana is the visualization tool for Elasticsearch. It allows you to create dashboards and visualizations based on the data collected by Beats.

  1. Access Kibana: Open your browser and navigate to http://localhost:5601.

  2. Create Index Patterns:

    • Go to the "Management" section.
    • Create an index pattern for the data collected by Beats (e.g., filebeat-*, metricbeat-*).

  3. Create Visualizations:

    • Use the "Visualize" section to create visualizations based on the data.
    • Combine visualizations into dashboards for comprehensive monitoring.

Example: Creating a Dashboard

  1. Create Visualizations:

    • Create visualizations for CPU usage, memory usage, and network traffic using Metricbeat data.

  2. Combine Visualizations into a Dashboard:

    • Go to the "Dashboard" section.
    • Add the visualizations created in the previous step to a new dashboard.

Conclusion

Beats are powerful tools for collecting and shipping data to Elasticsearch. They are lightweight and easy to configure, making them ideal for monitoring various aspects of your infrastructure. By integrating Beats with Elasticsearch and Kibana, you can create comprehensive monitoring solutions that provide valuable insights into your systems and applications.
