Introduction

In this section of the IT Infrastructure course, we will explore storage configuration and management. Storage is a critical component in any IT infrastructure, as it directly affects the performance, availability, and security of data. We will learn about the different types of storage, how to configure them, and manage them efficiently.

Types of Storage

Direct Attached Storage (DAS)

  • Definition: Direct Attached Storage (DAS) connects directly to a server or workstation without passing through a network.
  • Advantages:
    • Low latency.
    • Easy to configure.
  • Disadvantages:
    • Not scalable.
    • Difficult to share among multiple servers.

Network Attached Storage (NAS)

  • Definition: Network Attached Storage (NAS) is a storage device that connects to a network and allows access to multiple users and devices.
  • Advantages:
    • Easy to share.
    • Scalable.
  • Disadvantages:
    • May have latency due to the network.
    • Requires network configuration.

Storage Area Network (SAN)

  • Definition: A Storage Area Network (SAN) is a dedicated network of storage devices that provides access to consolidated storage.
  • Advantages:
    • High scalability.
    • High availability and performance.
  • Disadvantages:
    • Expensive.
    • Complex to configure and manage.

| Type of Storage | Advantages | Disadvantages | |-----------------|-------------------------------------|--------------------------------------| | DAS | Low latency, easy to configure | Not scalable, difficult to share | | NAS | Easy to share, scalable | Network latency, requires network configuration | | SAN | High scalability, high performance | Expensive, complex to manage |

Storage Configuration

DAS Configuration

  1. Physical Connection: Connect the storage device directly to the server using cables (SATA, SAS).
  2. BIOS/UEFI Configuration: Ensure the device is recognized in the BIOS/UEFI.
  3. Partitioning and Formatting: 
    # List available disks
sudo fdisk -l

# Create a new partition
sudo fdisk /dev/sdX

# Format the partition
sudo mkfs.ext4 /dev/sdX1

NAS Configuration

  1. Network Connection: Connect the NAS to the network using an Ethernet cable.
  2. Network Configuration: Assign a static or dynamic IP address.
  3. Share Configuration:
    • Create shared folders.
    • Assign access permissions.
    • Example configuration on a Synology NAS: 
      # Create a shared folder
synoshare --create MyShare --path /volume1/MyShare

# Assign permissions
synoshare --set MyShare --user admin --read-write

SAN Configuration

  1. SAN Network Configuration: Configure switches and fiber optic cables.
  2. LUN (Logical Unit Numbers) Configuration:
    • Create LUNs on the storage device.
    • Assign LUNs to specific servers.
  3. Server Configuration:
    • Install and configure the SAN client software.
    • Map LUNs to block devices on the server.
    # Example of LUN mapping in Linux
sudo iscsiadm --mode discovery --type sendtargets --portal 192.168.1.100
sudo iscsiadm --mode node --targetname iqn.2023-10.com.example:storage --login

Storage Management

Monitoring

  • Monitoring Tools:
    • Nagios
    • Zabbix
    • Prometheus
  • Key Metrics:
    • Space utilization.
    • Performance (IOPS, latency).
    • Disk status.

Backup and Recovery

  • Backup Strategies:
    • Full.
    • Incremental.
    • Differential.
  • Backup Tools:
    • Bacula
    • Veeam
    • Acronis
  • Example Backup Script in Linux: 
    #!/bin/bash
tar -czvf /backup/full_backup_$(date +%F).tar.gz /data

Security

  • Data Encryption:
    • Encryption at rest.
    • Encryption in transit.
  • Access Control:
    • Implement role-based access control (RBAC).
    • Use multi-factor authentication (MFA).

Conclusion

Storage configuration and management are essential to ensure the availability, performance, and security of data in an IT infrastructure. Knowing the different types of storage and how to configure them properly allows IT professionals to design efficient and scalable solutions. Additionally, continuous management through monitoring, backup, and security measures ensures the integrity and availability of data over time.
