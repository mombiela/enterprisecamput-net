In this section, we will explore examples of applications that use distributed architectures. These applications span a variety of industries and use cases, demonstrating the versatility and power of distributed architectures.

Scalable Web Applications

Scalable web applications are a classic example of distributed architectures. These applications must handle large volumes of traffic and data, requiring efficient resource distribution.

Example: Social Networks

Social networks like Facebook, Twitter, and LinkedIn use distributed architectures to handle millions of users simultaneously. These architectures allow:

  • Load balancing: Distributing traffic among multiple servers to avoid overloads.
  • Data replication: Keeping copies of data in different locations to improve availability and access speed.
  • Microservices: Dividing the application into smaller, manageable services that can be developed, deployed, and scaled independently.

Example Code: Load Balancing with Nginx

http {
    upstream backend {
        server backend1.example.com;
        server backend2.example.com;
        server backend3.example.com;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://backend;
        }
    }
}

In this example, Nginx is configured to distribute traffic among three backend servers.

Distributed Storage Systems

Distributed storage systems are essential for handling large volumes of data and ensuring their availability and durability.

Example: Hadoop

Hadoop is a framework that allows the distributed processing of large data sets across clusters of computers. It uses the HDFS (Hadoop Distributed File System) to store data redundantly.

  • MapReduce: A programming model for processing large data sets.
  • HDFS: Distributed file system that stores data in blocks distributed across multiple nodes.

Example Code: MapReduce Job in Hadoop

public class WordCount {
    public static class TokenizerMapper extends Mapper<Object, Text, Text, IntWritable> {
        private final static IntWritable one = new IntWritable(1);
        private Text word = new Text();

        public void map(Object key, Text value, Context context) throws IOException, InterruptedException {
            StringTokenizer itr = new StringTokenizer(value.toString());
            while (itr.hasMoreTokens()) {
                word.set(itr.nextToken());
                context.write(word, one);
            }
        }
    }

    public static class IntSumReducer extends Reducer<Text, IntWritable, Text, IntWritable> {
        public void reduce(Text key, Iterable<IntWritable> values, Context context) throws IOException, InterruptedException {
            int sum = 0;
            for (IntWritable val : values) {
                sum += val.get();
            }
            context.write(key, new IntWritable(sum));
        }
    }

    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        Job job = Job.getInstance(conf, "word count");
        job.setJarByClass(WordCount.class);
        job.setMapperClass(TokenizerMapper.class);
        job.setCombinerClass(IntSumReducer.class);
        job.setReducerClass(IntSumReducer.class);
        job.setOutputKeyClass(Text.class);
        job.setOutputValueClass(IntWritable.class);
        FileInputFormat.addInputPath(job, new Path(args[0]));
        FileOutputFormat.setOutputPath(job, new Path(args[1]));
        System.exit(job.waitForCompletion(true) ? 0 : 1);
    }
}

This example shows a MapReduce job for word counting in Hadoop.

Messaging and Queue Services

Messaging and queue services are fundamental in distributed architectures for asynchronous communication between different system components.

Example: RabbitMQ

RabbitMQ is a messaging system that implements the AMQP (Advanced Message Queuing Protocol). It is used to send and receive messages between different parts of a distributed application.

  • Producers: Send messages to the queue.
  • Consumers: Receive messages from the queue.
  • Exchanges: Direct messages to the appropriate queues.

Example Code: Sending and Receiving Messages with RabbitMQ

# Sending messages (Producer)
import pika

connection = pika.BlockingConnection(pika.ConnectionParameters('localhost'))
channel = connection.channel()

channel.queue_declare(queue='hello')

channel.basic_publish(exchange='',
                      routing_key='hello',
                      body='Hello World!')
print(" [x] Sent 'Hello World!'")
connection.close()

# Receiving messages (Consumer)
import pika

connection = pika.BlockingConnection(pika.ConnectionParameters('localhost'))
channel = connection.channel()

channel.queue_declare(queue='hello')

def callback(ch, method, properties, body):
    print(" [x] Received %r" % body)

channel.basic_consume(queue='hello',
                      on_message_callback=callback,
                      auto_ack=True)

print(' [*] Waiting for messages. To exit press CTRL+C')
channel.start_consuming()

In this example, a producer sends a "Hello World!" message to a queue, and a consumer receives the message from the queue.

Conclusion

Distributed architectures are essential for building scalable, robust, and efficient applications. Through examples such as scalable web applications, distributed storage systems, and messaging services, we have seen how these architectures are applied in the real world. Understanding and properly using these architectures allow developers to create systems that can handle large volumes of data and traffic while ensuring availability and resilience.
