Introduction

Distributed systems are a collection of independent computers that appear as a single coherent system to users. In this topic, we will explore the different models of distributed systems, their characteristics, advantages and disadvantages, and how they are applied in practice.

Distributed Systems Models

1. Client-Server Model

  • Description: In this model, clients request services and servers provide those services.
  • Characteristics:
    • Centralization of resources on the server.
    • Lightweight clients that request services.
    • Communication through network protocols.
  • Advantages:
    • Easy to manage and maintain.
    • Vertical scalability (server improvement).
  • Disadvantages:
    • Single point of failure.
    • Scalability limited by the server.

Code Example

# Example of a simple server in Python using sockets
import socket

def start_server():
    server_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    server_socket.bind(('localhost', 8080))
    server_socket.listen(5)
    print("Server started and listening on port 8080")

    while True:
        client_socket, address = server_socket.accept()
        print(f"Connection established with {address}")
        client_socket.send(b'Hello from the server!')
        client_socket.close()

start_server()

2. Peer-to-Peer (P2P) Model

  • Description: In this model, each node acts both as a client and a server, sharing resources directly among them.
  • Characteristics:
    • Decentralization.
    • Each node has equivalent capabilities.
    • Direct communication between nodes.
  • Advantages:
    • High availability and redundancy.
    • Horizontal scalability (adding more nodes).
  • Disadvantages:
    • Complex management and coordination.
    • Security and reliability can be an issue.

Code Example

# Example of a simple P2P node in Python using sockets
import socket
import threading

def handle_client(client_socket):
    request = client_socket.recv(1024)
    print(f"Received: {request.decode('utf-8')}")
    client_socket.send(b'Hello from the P2P node!')
    client_socket.close()

def start_node():
    node_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    node_socket.bind(('localhost', 8081))
    node_socket.listen(5)
    print("P2P node started and listening on port 8081")

    while True:
        client_socket, address = node_socket.accept()
        print(f"Connection established with {address}")
        client_handler = threading.Thread(target=handle_client, args=(client_socket,))
        client_handler.start()

start_node()

3. Cloud Computing Model

  • Description: This model provides computing and storage resources over the internet.
  • Characteristics:
    • On-demand services.
    • Elastic scalability.
    • Pay-per-use.
  • Advantages:
    • Reduction of infrastructure costs.
    • Flexibility and scalability.
  • Disadvantages:
    • Dependence on the service provider.
    • Security and privacy concerns.

Usage Example

# Example of using AWS S3 for cloud storage
import boto3

def upload_to_s3(file_name, bucket, object_name=None):
    s3_client = boto3.client('s3')
    try:
        s3_client.upload_file(file_name, bucket, object_name or file_name)
        print(f"File {file_name} uploaded to {bucket}")
    except Exception as e:
        print(f"Error uploading file: {e}")

upload_to_s3('my_file.txt', 'my-bucket')

Conclusion

Distributed systems models offer different approaches for managing and distributing resources. The choice of the appropriate model depends on the specific needs of the system, including scalability, management, security, and costs. Understanding these models is crucial for designing efficient and effective distributed architectures.
