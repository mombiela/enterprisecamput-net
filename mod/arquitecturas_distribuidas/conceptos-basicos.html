Introduction

Distributed systems are a collection of independent computers that appear as a single coherent system to users. These systems allow the distribution of tasks and resources across multiple nodes, improving efficiency, scalability, and fault tolerance.

Characteristics of Distributed Systems

  • Transparency: Users and applications do not perceive the distribution of resources.
  • Scalability: Ability to grow and handle a larger workload.
  • Fault Tolerance: Ability to continue functioning despite failures in some of its components.
  • Concurrency: Multiple processes can run simultaneously.
  • Heterogeneity: Integration of different types of hardware and software.

Models of Distributed Systems

Client-Server Model

In this model, clients request services and servers provide them. It is one of the most common models in distributed systems.

  • Client: Requests services.
  • Server: Provides services.

Code Example: Client-Server in Python

Server:

import socket

def start_server():
    server_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    server_socket.bind(('localhost', 8080))
    server_socket.listen(5)
    print("Server waiting for connections...")

    while True:
        client_socket, address = server_socket.accept()
        print(f"Connection from {address} established.")
        client_socket.send(b'Hello from the server!')
        client_socket.close()

if __name__ == "__main__":
    start_server()

Client:

import socket

def start_client():
    client_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    client_socket.connect(('localhost', 8080))
    message = client_socket.recv(1024)
    print(f"Message from the server: {message.decode()}")
    client_socket.close()

if __name__ == "__main__":
    start_client()

Peer-to-Peer (P2P) Model

In this model, all nodes have equivalent capabilities and can act both as clients and servers.

  • Nodes: Act as clients and servers simultaneously.
  • Decentralization: There is no central server.

Comparison: Client-Server vs Peer-to-Peer

| Characteristic | Client-Server | Peer-to-Peer | |----------------------|---------------------------------|--------------------------------| | Structure | Centralized | Decentralized | | Scalability | Limited by the server | High, each node adds resources | | Fault Tolerance | Low, depends on the server | High, does not depend on a single node | | Examples | Web, Email | BitTorrent, Blockchain |

Communication in Distributed Systems

  • RPC (Remote Procedure Call): Allows a program to execute procedures in another address space.
  • Messaging: Exchange of messages between processes.
  • Sockets: Endpoint for communication between two machines.

Conclusion

Distributed systems are fundamental in modern software architecture, allowing the distribution of tasks and resources across multiple nodes. Understanding the basic concepts, such as client-server and peer-to-peer models, as well as communication techniques, is essential for designing and maintaining efficient and scalable systems.
