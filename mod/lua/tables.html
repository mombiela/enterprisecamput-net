Tables in Lua
Arrays ►

Tables are a fundamental part of Lua, serving as the primary data structure. They can be used as arrays, dictionaries, or even objects. This section will guide you through the basics to advanced usage of tables in Lua.

Introduction to Tables

Tables in Lua are versatile and can store different types of values, including numbers, strings, functions, and even other tables.

Creating a Table

To create a table, you use the {} syntax.

local myTable = {}

Adding Elements to a Table

You can add elements to a table using either numeric or string keys.

-- Using numeric keys
myTable[1] = "Lua"
myTable[2] = "Programming"

-- Using string keys
myTable["language"] = "Lua"
myTable["type"] = "Scripting"

Accessing Table Elements

Access elements using their keys.

print(myTable[1])          -- Output: Lua
print(myTable["language"]) -- Output: Lua

Tables as Arrays

In Lua, tables can be used as arrays, where numeric indices are used to store and access elements.

Example: Simple Array

local fruits = {"Apple", "Banana", "Cherry"}

for i = 1, #fruits do
    print(fruits[i])
end

Explanation

  • #fruits gives the length of the array.
  • The loop iterates through each element, printing it.

Tables as Dictionaries

Tables can also function as dictionaries (or maps), where string keys are used.

Example: Simple Dictionary

local capitals = {
    France = "Paris",
    Germany = "Berlin",
    Italy = "Rome"
}

print(capitals["France"]) -- Output: Paris

Explanation

  • Keys are country names, and values are their capitals.
  • Accessing the value using capitals["France"] returns "Paris".

Nested Tables

Tables can contain other tables, allowing for complex data structures.

Example: Nested Table

local person = {
    name = "John",
    age = 30,
    address = {
        street = "123 Lua St",
        city = "Lua City"
    }
}

print(person.name)          -- Output: John
print(person.address.city)  -- Output: Lua City

Explanation

  • person table contains another table address.
  • Access nested elements using dot notation.

Table Functions

Lua provides several built-in functions to manipulate tables.

Common Table Functions

  • table.insert(table, [pos,] value): Inserts a value into the table at the specified position.
  • table.remove(table, [pos]): Removes the element at the specified position.
  • table.sort(table, [comp]): Sorts the elements of the table.

Example: Using Table Functions

local numbers = {5, 3, 8, 1}

table.insert(numbers, 2, 7) -- Insert 7 at position 2
table.sort(numbers)         -- Sort the table

for i, v in ipairs(numbers) do
    print(v)
end

Explanation

  • table.insert(numbers, 2, 7) inserts 7 at the second position.
  • table.sort(numbers) sorts the table in ascending order.
  • ipairs is used to iterate over the sorted table.

Metatables and Metamethods

Metatables allow you to change the behavior of tables. Metamethods are special functions that can be defined in a metatable to override default behaviors.

Example: Metatable with __index

local default = {x = 0, y = 0}
local point = setmetatable({}, {__index = default})

print(point.x) -- Output: 0

Explanation

  • default table provides default values.
  • setmetatable sets the metatable for point with __index pointing to default.
  • Accessing point.x returns 0 because point inherits from default.

Conclusion

Tables in Lua are powerful and flexible, serving as arrays, dictionaries, and even objects. Understanding how to create, manipulate, and extend tables is crucial for effective Lua programming. Practice using tables in different scenarios to become proficient in their usage.
