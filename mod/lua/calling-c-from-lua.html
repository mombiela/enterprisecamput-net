Calling C from Lua
Using Lua with Other Languages ►

In this section, we will explore how to extend Lua by calling C functions from Lua scripts. This is a powerful feature that allows you to leverage existing C libraries or optimize performance-critical parts of your Lua application.

Why Call C from Lua?

  • Performance: C is a compiled language and can execute certain tasks faster than Lua.
  • Existing Libraries: Many libraries are available in C that you might want to use in your Lua application.
  • System-Level Access: C provides more direct access to system resources and hardware.

Prerequisites

Before diving into calling C from Lua, ensure you have:

  • Basic understanding of Lua scripting.
  • Basic knowledge of C programming.
  • A working development environment for both Lua and C.

Setting Up the Environment

To call C from Lua, you need to:

  1. Install Lua: Ensure Lua is installed on your system.
  2. Install a C Compiler: GCC or Clang are commonly used.
  3. Lua C API: Familiarize yourself with the Lua C API, which allows C code to interact with Lua.

Writing a Simple C Function

Let's start by writing a simple C function that we will call from Lua.

// mylib.c
#include <lua.h>
#include <lauxlib.h>
#include <lualib.h>

// A simple C function that adds two numbers
static int add(lua_State *L) {
    double a = luaL_checknumber(L, 1);
    double b = luaL_checknumber(L, 2);
    lua_pushnumber(L, a + b);
    return 1; // Number of return values
}

// Registering the function
static const struct luaL_Reg mylib [] = {
    {"add", add},
    {NULL, NULL} // Sentinel
};

int luaopen_mylib(lua_State *L) {
    luaL_newlib(L, mylib);
    return 1;
}

Explanation

  • luaL_checknumber: Checks and retrieves the function arguments from the Lua stack.
  • lua_pushnumber: Pushes the result back onto the Lua stack.
  • luaL_Reg: Structure used to register the C functions.
  • luaopen_mylib: Function called by Lua to load the library.

Compiling the C Code

Compile the C code into a shared library that Lua can load.

gcc -shared -o mylib.so -fPIC mylib.c -I/usr/include/lua5.3
  • -shared: Creates a shared library.
  • -fPIC: Generates position-independent code.
  • -I/usr/include/lua5.3: Includes the Lua headers.

Calling the C Function from Lua

Now, let's write a Lua script to call our C function.

-- test.lua
local mylib = require("mylib")

local result = mylib.add(10, 20)
print("The result is: " .. result)

Explanation

  • require("mylib"): Loads the shared library.
  • mylib.add(10, 20): Calls the C function.

Running the Lua Script

Run the Lua script to see the result.

lua test.lua

You should see the output:

The result is: 30

Error Handling

When calling C functions from Lua, it's important to handle errors gracefully.

  • luaL_error: Generates a Lua error from C.
  • luaL_checktype: Ensures the arguments are of the expected type.

Example:

static int safe_add(lua_State *L) {
    if (!lua_isnumber(L, 1) || !lua_isnumber(L, 2)) {
        return luaL_error(L, "Expected two numbers");
    }
    double a = lua_tonumber(L, 1);
    double b = lua_tonumber(L, 2);
    lua_pushnumber(L, a + b);
    return 1;
}

Conclusion

Calling C from Lua allows you to extend Lua's capabilities by leveraging the power and performance of C. By following the steps outlined in this section, you can integrate C functions into your Lua scripts, providing a seamless way to enhance your applications.

  • Performance Boost: Optimize critical parts of your application.
  • Library Integration: Use existing C libraries.
  • System Access: Gain more control over system resources.

Experiment with more complex C functions and explore the Lua C API documentation to fully harness the power of calling C from Lua.
