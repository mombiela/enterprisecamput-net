Introduction

In this final project, students will apply the agile methodologies learned during the course to a real project. The objective is to consolidate theoretical knowledge through practice and the resolution of real problems that could be encountered in a professional environment.

Project Objectives

  • Apply the principles and practices of agile methodologies.
  • Develop teamwork and communication skills.
  • Manage a project from planning to delivery.
  • Continuously evaluate and improve the development process.

Project Description

Students will work in teams to develop a web application using an agile methodology, such as Scrum or Kanban. The project will include all phases of the software development lifecycle, from requirements gathering to final delivery.

Project Phases

  1. Initial Planning

    • Define the scope of the project.
    • Create a product backlog.
    • Establish roles and responsibilities within the team.

  2. Iterative Development

    • Divide the work into sprints (if using Scrum) or tasks (if using Kanban).
    • Conduct daily stand-up meetings.
    • Implement and test the planned functionalities.

  3. Review and Retrospective

    • Conduct sprint reviews to demonstrate progress.
    • Conduct retrospectives to identify areas for improvement.

  4. Final Delivery

    • Prepare project documentation.
    • Conduct a final presentation of the developed product.

Recommended Tools

  • Project Management: Jira, Trello
  • Version Control: Git, GitHub
  • Communication: Slack, Microsoft Teams
  • Development and Testing: Visual Studio Code, Postman

Application Example

To illustrate how an agile methodology can be applied in a real project, consider the development of a task management application.

Initial Planning

  • Product Backlog:

    • Create tasks.
    • Edit tasks.
    • Delete tasks.
    • Assign tasks to users.
    • Mark tasks as completed.

  • Roles and Responsibilities:

    • Product Owner: Responsible for defining and prioritizing the backlog.
    • Scrum Master: Facilitates meetings and removes impediments.
    • Development Team: Implements the functionalities.

Iterative Development

  • Sprint 1:

    • Create tasks.
    • Edit tasks.

  • Sprint 2:

    • Delete tasks.
    • Assign tasks to users.

  • Sprint 3:

    • Mark tasks as completed.
    • Testing and bug fixing.

Review and Retrospective

  • Sprint Review:

    • Demonstration of completed functionalities.
    • Feedback from the Product Owner.

  • Retrospective:

    • What went well?
    • What can be improved?
    • Action plan for the next sprint.

Conclusion

The final project is an opportunity for students to apply the knowledge acquired about agile methodologies in a practical environment. Through planning, iterative development, and continuous reviews, students will be able to experience firsthand the benefits of agile methodologies and develop key skills for their professional career in IT.
