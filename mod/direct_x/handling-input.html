Handling Input in DirectX

Handling input is a crucial aspect of developing interactive applications and games using DirectX. This topic will guide you through the basics of capturing and processing user input, progressing to more advanced techniques.

Introduction to Input Handling

In DirectX, handling input typically involves capturing data from various input devices such as the keyboard, mouse, and game controllers. We'll start with the basics and gradually move to more complex scenarios.

Key Concepts

  • DirectInput: A legacy API for handling input devices.
  • Raw Input: A more modern and flexible way to handle input.
  • XInput: Specifically designed for Xbox controllers but also works with other gamepads.

Using DirectInput

DirectInput is part of the DirectX API and provides a way to interact with input devices. Although it's considered somewhat outdated, it's still useful for certain applications.

Initializing DirectInput

To use DirectInput, you need to initialize it and create device objects for the input devices you want to handle.

#include <dinput.h>

// Global DirectInput variables
LPDIRECTINPUT8 g_pDI = NULL;
LPDIRECTINPUTDEVICE8 g_pKeyboard = NULL;

// Initialize DirectInput
HRESULT InitDirectInput(HINSTANCE hInstance, HWND hWnd) {
    HRESULT hr;

    // Create the DirectInput interface
    hr = DirectInput8Create(hInstance, DIRECTINPUT_VERSION, IID_IDirectInput8, (void**)&g_pDI, NULL);
    if (FAILED(hr)) return hr;

    // Create a device for the keyboard
    hr = g_pDI->CreateDevice(GUID_SysKeyboard, &g_pKeyboard, NULL);
    if (FAILED(hr)) return hr;

    // Set the data format for the keyboard
    hr = g_pKeyboard->SetDataFormat(&c_dfDIKeyboard);
    if (FAILED(hr)) return hr;

    // Set the cooperative level
    hr = g_pKeyboard->SetCooperativeLevel(hWnd, DISCL_FOREGROUND | DISCL_NONEXCLUSIVE);
    if (FAILED(hr)) return hr;

    return S_OK;
}

Reading Keyboard Input

Once initialized, you can read the state of the keyboard.

BYTE g_KeyboardState[256];

// Read the keyboard state
void ReadKeyboard() {
    HRESULT hr = g_pKeyboard->GetDeviceState(sizeof(g_KeyboardState), (LPVOID)&g_KeyboardState);
    if (FAILED(hr)) {
        // Reacquire the device if necessary
        if (hr == DIERR_INPUTLOST || hr == DIERR_NOTACQUIRED) {
            g_pKeyboard->Acquire();
        }
    }
}

// Check if a key is pressed
bool IsKeyPressed(BYTE key) {
    return (g_KeyboardState[key] & 0x80) != 0;
}

Using Raw Input

Raw Input provides more flexibility and is the recommended way to handle input in modern applications.

Registering for Raw Input

First, you need to register your application to receive raw input data.

RAWINPUTDEVICE Rid[1];

// Register for raw input
void RegisterRawInput(HWND hWnd) {
    Rid[0].usUsagePage = HID_USAGE_PAGE_GENERIC;
    Rid[0].usUsage = HID_USAGE_GENERIC_MOUSE;
    Rid[0].dwFlags = RIDEV_INPUTSINK;
    Rid[0].hwndTarget = hWnd;

    if (RegisterRawInputDevices(Rid, 1, sizeof(Rid[0])) == FALSE) {
        // Registration failed
    }
}

Processing Raw Input

You can process raw input data in your window procedure.

LRESULT CALLBACK WindowProc(HWND hWnd, UINT message, WPARAM wParam, LPARAM lParam) {
    switch (message) {
        case WM_INPUT: {
            UINT dwSize;
            GetRawInputData((HRAWINPUT)lParam, RID_INPUT, NULL, &dwSize, sizeof(RAWINPUTHEADER));
            LPBYTE lpb = new BYTE[dwSize];
            if (lpb == NULL) return 0;

            if (GetRawInputData((HRAWINPUT)lParam, RID_INPUT, lpb, &dwSize, sizeof(RAWINPUTHEADER)) != dwSize) {
                OutputDebugString(TEXT("GetRawInputData does not return correct size !\n"));
            }

            RAWINPUT* raw = (RAWINPUT*)lpb;

            if (raw->header.dwType == RIM_TYPEMOUSE) {
                // Process mouse input
                int xPosRelative = raw->data.mouse.lLastX;
                int yPosRelative = raw->data.mouse.lLastY;
                // Handle mouse movement
            }

            delete[] lpb;
            return 0;
        }
        // Handle other messages
    }
    return DefWindowProc(hWnd, message, wParam, lParam);
}

Using XInput

XInput is designed for Xbox controllers but can also be used for other gamepads. It provides a simple interface for handling controller input.

Initializing XInput

XInput does not require explicit initialization. You can start using it directly.

Reading Controller Input

You can read the state of the controller using XInputGetState.

#include <XInput.h>

// Read the state of the controller
void ReadController() {
    XINPUT_STATE state;
    ZeroMemory(&state, sizeof(XINPUT_STATE));

    DWORD dwResult = XInputGetState(0, &state);
    if (dwResult == ERROR_SUCCESS) {
        // Controller is connected
        XINPUT_GAMEPAD* pad = &state.Gamepad;

        // Check if the A button is pressed
        if (pad->wButtons & XINPUT_GAMEPAD_A) {
            // Handle A button press
        }

        // Check the thumbstick positions
        float LX = pad->sThumbLX / 32767.0f;
        float LY = pad->sThumbLY / 32767.0f;
        // Handle thumbstick movement
    } else {
        // Controller is not connected
    }
}

Summary

Handling input in DirectX involves understanding and using different APIs depending on the type of input device and the requirements of your application.

  • DirectInput is useful for legacy applications and provides a straightforward way to handle keyboard and mouse input.
  • Raw Input offers more flexibility and is suitable for modern applications that need detailed input data.
  • XInput is ideal for handling game controllers, especially Xbox controllers, with a simple and efficient interface.

By mastering these techniques, you can create responsive and interactive DirectX applications that provide a great user experience.
