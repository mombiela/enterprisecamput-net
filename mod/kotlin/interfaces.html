Introduction to Interfaces

Interfaces in Kotlin are a powerful way to define contracts that classes can implement. They allow you to specify methods and properties that must be implemented by any class that implements the interface.

  • Definition: An interface is a blueprint of a class that contains abstract methods and properties.
  • Purpose: To provide a way to achieve abstraction and multiple inheritance in Kotlin.

Key Concepts

  • Abstract Methods: Methods without a body that must be implemented by the class.
  • Default Methods: Methods with a body that can be optionally overridden by the class.
  • Properties in Interfaces: Interfaces can also contain properties, which can be abstract or have default implementations.

Defining an Interface

To define an interface in Kotlin, use the interface keyword.

interface Drivable {
    fun drive() // Abstract method
    val maxSpeed: Int // Abstract property
}

Implementing an Interface

A class that implements an interface must provide implementations for all abstract methods and properties.

class Car : Drivable {
    override fun drive() {
        println("Car is driving")
    }

    override val maxSpeed: Int = 200
}

Default Methods and Properties

Interfaces can also contain methods and properties with default implementations.

interface Flyable {
    fun fly() {
        println("Flying")
    }

    val altitude: Int
        get() = 1000
}

class Airplane : Flyable {
    // No need to override fly() or altitude unless custom behavior is needed
}

Multiple Interfaces

A class in Kotlin can implement multiple interfaces, allowing for more flexible and reusable code.

interface Swimmable {
    fun swim()
}

class AmphibiousCar : Drivable, Swimmable {
    override fun drive() {
        println("Amphibious car is driving")
    }

    override val maxSpeed: Int = 150

    override fun swim() {
        println("Amphibious car is swimming")
    }
}

Interface Inheritance

Interfaces can inherit from other interfaces, allowing for more complex and hierarchical designs.

interface Vehicle {
    fun start()
}

interface ElectricVehicle : Vehicle {
    fun chargeBattery()
}

class Tesla : ElectricVehicle {
    override fun start() {
        println("Tesla is starting")
    }

    override fun chargeBattery() {
        println("Charging Tesla's battery")
    }
}

Practical Examples

Example 1: Simple Interface Implementation

interface Animal {
    fun sound()
}

class Dog : Animal {
    override fun sound() {
        println("Bark")
    }
}

fun main() {
    val dog = Dog()
    dog.sound() // Output: Bark
}

Example 2: Interface with Default Methods

interface Printer {
    fun print() {
        println("Printing document")
    }
}

class LaserPrinter : Printer

fun main() {
    val printer = LaserPrinter()
    printer.print() // Output: Printing document
}

Example 3: Multiple Interfaces

interface Scanner {
    fun scan()
}

class MultiFunctionPrinter : Printer, Scanner {
    override fun scan() {
        println("Scanning document")
    }
}

fun main() {
    val mfp = MultiFunctionPrinter()
    mfp.print() // Output: Printing document
    mfp.scan()  // Output: Scanning document
}

Conclusion

Interfaces in Kotlin provide a robust mechanism for defining contracts that classes can implement. They support abstraction, multiple inheritance, and can contain both abstract and default methods and properties. By understanding and utilizing interfaces, you can create more modular, reusable, and maintainable code.
