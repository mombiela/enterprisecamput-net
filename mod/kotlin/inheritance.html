Inheritance in Kotlin
Interfaces ►

Inheritance is a fundamental concept in object-oriented programming that allows a class to inherit properties and functions from another class. In Kotlin, inheritance is straightforward and provides a powerful way to reuse code and create a hierarchical class structure.

Key Concepts

  • Superclass (Parent Class): The class whose properties and functions are inherited.
  • Subclass (Child Class): The class that inherits properties and functions from the superclass.
  • open Keyword: By default, classes in Kotlin are final and cannot be inherited. The open keyword is used to make a class inheritable.
  • override Keyword: Used to override a function or property in the subclass.

Creating a Superclass

In Kotlin, you need to mark a class as open to allow other classes to inherit from it.

open class Animal {
    open fun sound() {
        println("Animal makes a sound")
    }
}

Creating a Subclass

A subclass inherits from a superclass using the : symbol.

class Dog : Animal() {
    override fun sound() {
        println("Dog barks")
    }
}

Example: Basic Inheritance

Let's see a complete example of inheritance in Kotlin.

// Superclass
open class Animal {
    open fun sound() {
        println("Animal makes a sound")
    }
}

// Subclass
class Dog : Animal() {
    override fun sound() {
        println("Dog barks")
    }
}

fun main() {
    val myDog = Dog()
    myDog.sound()  // Output: Dog barks
}

In this example:

  • Animal is the superclass with an open function sound.
  • Dog is the subclass that overrides the sound function.

Properties in Inheritance

You can also inherit properties from the superclass.

open class Animal {
    open val type: String = "Unknown"
}

class Dog : Animal() {
    override val type: String = "Dog"
}

fun main() {
    val myDog = Dog()
    println(myDog.type)  // Output: Dog
}

Calling Superclass Functions

You can call the superclass function using the super keyword.

open class Animal {
    open fun sound() {
        println("Animal makes a sound")
    }
}

class Dog : Animal() {
    override fun sound() {
        super.sound()  // Calls the superclass function
        println("Dog barks")
    }
}

fun main() {
    val myDog = Dog()
    myDog.sound()
    // Output:
    // Animal makes a sound
    // Dog barks
}

Abstract Classes

Sometimes, you want to create a superclass that should not be instantiated directly. For this, you can use abstract classes.

abstract class Animal {
    abstract fun sound()
}

class Dog : Animal() {
    override fun sound() {
        println("Dog barks")
    }
}

fun main() {
    val myDog = Dog()
    myDog.sound()  // Output: Dog barks
}

In this example:

  • Animal is an abstract class with an abstract function sound.
  • Dog is a subclass that provides an implementation for the sound function.

Summary

  • Inheritance allows a class to inherit properties and functions from another class.
  • Use the open keyword to make a class inheritable.
  • Use the override keyword to override functions and properties in the subclass.
  • Abstract classes can be used to define classes that should not be instantiated directly but can be inherited.

By understanding and utilizing inheritance, you can create more modular, reusable, and maintainable code in Kotlin.
