ViewModel and LiveData in Kotlin
Kotlin Multiplatform ►

Introduction

In this section, we will explore the concepts of ViewModel and LiveData in Kotlin, which are essential components of Android's architecture components. These tools help manage UI-related data in a lifecycle-conscious way, ensuring that data survives configuration changes such as screen rotations.

ViewModel

The ViewModel class is designed to store and manage UI-related data in a lifecycle-conscious way. It allows data to survive configuration changes such as screen rotations.

Key Concepts

  • Lifecycle Awareness: ViewModel objects are automatically retained during configuration changes.
  • Separation of Concerns: Helps in separating UI data from UI controllers like Activities and Fragments.

Example

Let's create a simple ViewModel to manage a counter.

// CounterViewModel.kt
import androidx.lifecycle.ViewModel

class CounterViewModel : ViewModel() {
    var counter: Int = 0

    fun incrementCounter() {
        counter++
    }
}

In your Activity or Fragment, you can use this ViewModel as follows:

// MainActivity.kt
import android.os.Bundle
import androidx.activity.viewModels
import androidx.appcompat.app.AppCompatActivity
import androidx.lifecycle.ViewModelProvider
import kotlinx.android.synthetic.main.activity_main.*

class MainActivity : AppCompatActivity() {
    private val viewModel: CounterViewModel by viewModels()

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        button_increment.setOnClickListener {
            viewModel.incrementCounter()
            textView_counter.text = viewModel.counter.toString()
        }
    }
}

LiveData

LiveData is an observable data holder class. Unlike a regular observable, LiveData is lifecycle-aware, meaning it respects the lifecycle of other app components, such as activities, fragments, or services.

Key Concepts

  • Lifecycle Awareness: LiveData only updates app component observers that are in an active lifecycle state.
  • Automatic Cleanup: Observers are automatically removed when their associated lifecycle is destroyed.
  • No Memory Leaks: Observers are bound to lifecycle objects and are cleaned up when the lifecycle is destroyed.

Example

Let's modify our previous example to use LiveData.

// CounterViewModel.kt
import androidx.lifecycle.LiveData
import androidx.lifecycle.MutableLiveData
import androidx.lifecycle.ViewModel

class CounterViewModel : ViewModel() {
    private val _counter = MutableLiveData<Int>()
    val counter: LiveData<Int> get() = _counter

    init {
        _counter.value = 0
    }

    fun incrementCounter() {
        _counter.value = (_counter.value ?: 0) + 1
    }
}

In your Activity or Fragment, you can observe the LiveData as follows:

// MainActivity.kt
import android.os.Bundle
import androidx.activity.viewModels
import androidx.appcompat.app.AppCompatActivity
import androidx.lifecycle.Observer
import kotlinx.android.synthetic.main.activity_main.*

class MainActivity : AppCompatActivity() {
    private val viewModel: CounterViewModel by viewModels()

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        viewModel.counter.observe(this, Observer { newCounterValue ->
            textView_counter.text = newCounterValue.toString()
        })

        button_increment.setOnClickListener {
            viewModel.incrementCounter()
        }
    }
}

Combining ViewModel and LiveData

Combining ViewModel and LiveData provides a robust way to manage UI-related data in a lifecycle-aware manner. The ViewModel holds the data, and LiveData ensures that the UI is updated whenever the data changes.

Example

Let's create a more complex example where we fetch data from a repository.

// UserRepository.kt
class UserRepository {
    fun getUser(): LiveData<User> {
        val user = MutableLiveData<User>()
        // Simulate a network call or database fetch
        user.value = User("John Doe", 30)
        return user
    }
}

// UserViewModel.kt
import androidx.lifecycle.LiveData
import androidx.lifecycle.ViewModel

class UserViewModel : ViewModel() {
    private val userRepository = UserRepository()
    val user: LiveData<User> = userRepository.getUser()
}

// User.kt
data class User(val name: String, val age: Int)

In your Activity or Fragment, observe the user data:

// MainActivity.kt
import android.os.Bundle
import androidx.activity.viewModels
import androidx.appcompat.app.AppCompatActivity
import androidx.lifecycle.Observer
import kotlinx.android.synthetic.main.activity_main.*

class MainActivity : AppCompatActivity() {
    private val viewModel: UserViewModel by viewModels()

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        viewModel.user.observe(this, Observer { user ->
            textView_name.text = user.name
            textView_age.text = user.age.toString()
        })
    }
}

Conclusion

ViewModel and LiveData are powerful tools for managing UI-related data in a lifecycle-aware manner in Android applications. By using these components, you can ensure that your app's data survives configuration changes and that your UI is always up-to-date with the latest data. This separation of concerns also makes your code more modular and easier to test.
