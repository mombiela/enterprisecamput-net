Introduction to Data Classes

Data classes in Kotlin are a concise way to create classes that are primarily used to hold data. They automatically provide several useful methods, such as equals(), hashCode(), toString(), and copy(), which makes them very convenient for handling data.

Key Concepts

  • Primary Constructor: Data classes must have at least one primary constructor parameter.
  • Properties: All primary constructor parameters need to be marked as val or var.
  • Generated Methods: Kotlin automatically generates equals(), hashCode(), toString(), copy(), and componentN() methods for data classes.
  • Immutability: By default, data classes are immutable if you use val for properties.

Creating a Data Class

To define a data class, use the data keyword before the class declaration.

data class User(val name: String, val age: Int)

Example Explained

  • data: Keyword indicating this is a data class.
  • User: Name of the data class.
  • val name: String: Immutable property name of type String.
  • val age: Int: Immutable property age of type Int.

Generated Methods

`toString()`

Provides a string representation of the object.

val user = User("Alice", 30)
println(user) // Output: User(name=Alice, age=30)

`equals()` and `hashCode()`

Used to compare objects and for hashing.

val user1 = User("Alice", 30)
val user2 = User("Alice", 30)
println(user1 == user2) // Output: true
println(user1.hashCode() == user2.hashCode()) // Output: true

`copy()`

Creates a copy of the object, optionally modifying some properties.

val user1 = User("Alice", 30)
val user2 = user1.copy(age = 31)
println(user2) // Output: User(name=Alice, age=31)

`componentN()`

Allows destructuring declarations.

val user = User("Alice", 30)
val (name, age) = user
println("Name: $name, Age: $age") // Output: Name: Alice, Age: 30

Advanced Features

Default Values

You can provide default values for properties in the primary constructor.

data class User(val name: String = "Unknown", val age: Int = 0)
val user = User()
println(user) // Output: User(name=Unknown, age=0)

Data Class Inheritance

Data classes cannot be abstract, open, sealed, or inner. However, they can implement interfaces.

interface Printable {
    fun print()
}

data class User(val name: String, val age: Int) : Printable {
    override fun print() {
        println("User(name=$name, age=$age)")
    }
}

Copying with Nested Data Classes

When copying nested data classes, only the top-level object is copied. Nested objects are not deeply copied.

data class Address(val city: String, val country: String)
data class User(val name: String, val age: Int, val address: Address)

val address = Address("New York", "USA")
val user1 = User("Alice", 30, address)
val user2 = user1.copy()

println(user1.address === user2.address) // Output: true (same reference)

Conclusion

Data classes in Kotlin provide a powerful and concise way to handle data-centric classes. They automatically generate useful methods, reducing boilerplate code and enhancing readability. Understanding and utilizing data classes effectively can significantly improve your Kotlin programming skills, especially when dealing with data-heavy applications.
