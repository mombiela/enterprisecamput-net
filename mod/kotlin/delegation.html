Delegation in Kotlin
Lambda Expressions ►

Delegation is a design pattern in Kotlin that allows an object to hand over some of its responsibilities to another object. This can be particularly useful for code reuse and composition. Kotlin provides built-in support for delegation through the by keyword, making it easier to implement delegation patterns.

What is Delegation?

  • Definition: Delegation is a design pattern where an object delegates some of its responsibilities to another object.
  • Purpose: It helps in code reuse and composition, making the code more modular and easier to maintain.

Types of Delegation in Kotlin

  1. Class Delegation
  2. Property Delegation

Class Delegation

Class delegation allows one class to delegate its implementation to another class. This is done using the by keyword.

Example

interface Printer {
    fun print()
}

class SimplePrinter : Printer {
    override fun print() {
        println("Printing from SimplePrinter")
    }
}

class AdvancedPrinter(printer: Printer) : Printer by printer

fun main() {
    val simplePrinter = SimplePrinter()
    val advancedPrinter = AdvancedPrinter(simplePrinter)
    advancedPrinter.print()  // Output: Printing from SimplePrinter
}
  • Explanation:
    • Printer is an interface with a print method.
    • SimplePrinter is a class that implements the Printer interface.
    • AdvancedPrinter delegates the print method to an instance of Printer using the by keyword.

Property Delegation

Property delegation allows you to delegate the getter and setter of a property to another object. Kotlin provides several built-in delegates, such as lazy, observable, and vetoable.

Lazy Initialization

Lazy initialization is a common use case for property delegation. The lazy delegate initializes the property only when it is accessed for the first time.

Example

class LazyExample {
    val lazyValue: String by lazy {
        println("Computed!")
        "Hello, Kotlin!"
    }
}

fun main() {
    val example = LazyExample()
    println(example.lazyValue)  // Output: Computed! Hello, Kotlin!
    println(example.lazyValue)  // Output: Hello, Kotlin!
}
  • Explanation:
    • lazyValue is a property that uses the lazy delegate.
    • The first time lazyValue is accessed, the lambda expression is executed, and "Computed!" is printed.
    • Subsequent accesses to lazyValue do not re-execute the lambda expression.

Observable Properties

The observable delegate allows you to track changes to a property.

Example

import kotlin.properties.Delegates

class User {
    var name: String by Delegates.observable("<no name>") { prop, old, new ->
        println("Property ${prop.name} changed from $old to $new")
    }
}

fun main() {
    val user = User()
    user.name = "John"  // Output: Property name changed from <no name> to John
    user.name = "Doe"   // Output: Property name changed from John to Doe
}
  • Explanation:
    • name is a property that uses the observable delegate.
    • The lambda expression is called whenever the property is changed, printing the old and new values.

Custom Property Delegates

You can also create your own custom property delegates by implementing the ReadOnlyProperty or ReadWriteProperty interfaces.

Example

import kotlin.reflect.KProperty

class CustomDelegate {
    operator fun getValue(thisRef: Any?, property: KProperty<*>): String {
        return "Custom Delegate Value"
    }

    operator fun setValue(thisRef: Any?, property: KProperty<*>, value: String) {
        println("Setting value '$value' to '${property.name}'")
    }
}

class Example {
    var customProperty: String by CustomDelegate()
}

fun main() {
    val example = Example()
    println(example.customProperty)  // Output: Custom Delegate Value
    example.customProperty = "New Value"  // Output: Setting value 'New Value' to 'customProperty'
}
  • Explanation:
    • CustomDelegate class implements the getValue and setValue operators.
    • customProperty in the Example class uses the CustomDelegate.

Conclusion

Delegation in Kotlin is a powerful feature that allows for cleaner, more modular code. By using class and property delegation, you can easily delegate responsibilities and behaviors to other objects, promoting code reuse and reducing boilerplate. Whether using built-in delegates like lazy and observable or creating custom delegates, Kotlin's delegation mechanisms provide a flexible and efficient way to manage object responsibilities.
