RedisTimeSeries is a Redis module designed to handle time-series data efficiently. It extends Redis with a set of commands and data structures specifically optimized for time-series data, making it easier to store, query, and analyze time-series data.

Introduction to RedisTimeSeries

RedisTimeSeries provides a specialized data structure for time-series data, which is data that is indexed by time. This module is particularly useful for applications that require high-throughput ingestion and querying of time-series data, such as monitoring systems, IoT applications, and financial market data analysis.

Key Features

  • Efficient Data Storage: Optimized for storing large volumes of time-series data.
  • High Throughput: Capable of handling high write and read throughput.
  • Aggregation and Downsampling: Built-in support for data aggregation and downsampling.
  • Compaction: Automatic data compaction to reduce storage requirements.
  • Querying: Advanced querying capabilities, including range queries and filtering.

Installing RedisTimeSeries

To use RedisTimeSeries, you need to install the module. You can do this using the Redis module CLI or by including it in your Redis configuration.

Installation Example

# Download and install RedisTimeSeries
git clone https://github.com/RedisTimeSeries/RedisTimeSeries.git
cd RedisTimeSeries
make

# Load the module into Redis
redis-server --loadmodule ./redistimeseries.so

Basic Commands

Creating a Time-Series

To create a time-series, use the TS.CREATE command. This command initializes a new time-series key.

TS.CREATE temperature:room1

Adding Data Points

To add data points to a time-series, use the TS.ADD command. This command requires a timestamp and a value.

TS.ADD temperature:room1 1633024800 22.5
TS.ADD temperature:room1 1633028400 23.0

Querying Data

To query data from a time-series, use the TS.RANGE command. This command retrieves data points within a specified time range.

TS.RANGE temperature:room1 1633024800 1633028400

Aggregation and Downsampling

RedisTimeSeries supports aggregation and downsampling to summarize data over specified intervals.

TS.RANGE temperature:room1 1633024800 1633028400 AGGREGATION avg 3600

Advanced Features

Compaction Rules

Compaction rules allow you to automatically downsample data as it is ingested. This is useful for reducing storage requirements while retaining important information.

TS.CREATERULE temperature:room1 avg_temperature:room1 AVG 3600

Multi-Range Queries

Multi-range queries allow you to query multiple time-series at once. This is useful for comparing data across different series.

TS.MRANGE 1633024800 1633028400 FILTER room=room1

Labeling and Filtering

You can add labels to time-series to categorize and filter them more easily.

TS.CREATE temperature:room1 LABELS room room1
TS.CREATE temperature:room2 LABELS room room2

# Query all time-series with the label 'room=room1'
TS.MRANGE 1633024800 1633028400 FILTER room=room1

Practical Example

Let's walk through a practical example of using RedisTimeSeries to monitor temperature data from multiple rooms.

Step-by-Step Example

  1. Create Time-Series for Each Room

    TS.CREATE temperature:room1 LABELS room room1
TS.CREATE temperature:room2 LABELS room room2

  2. Add Data Points

    TS.ADD temperature:room1 1633024800 22.5
TS.ADD temperature:room1 1633028400 23.0
TS.ADD temperature:room2 1633024800 21.0
TS.ADD temperature:room2 1633028400 21.5

  3. Query Data for a Specific Room

    TS.RANGE temperature:room1 1633024800 1633028400

  4. Aggregate Data

    TS.RANGE temperature:room1 1633024800 1633028400 AGGREGATION avg 3600

  5. Filter Data by Labels

    TS.MRANGE 1633024800 1633028400 FILTER room=room1

Conclusion

RedisTimeSeries is a powerful module for managing time-series data in Redis. It provides efficient storage, high throughput, and advanced querying capabilities, making it ideal for applications that require real-time analysis of time-series data. By leveraging features like aggregation, compaction, and labeling, you can build robust and scalable time-series data solutions with RedisTimeSeries.
