Basic Redis Data Types
Redis CLI Basics ►

Redis is an in-memory data structure store used as a database, cache, and message broker. Understanding the basic data types in Redis is crucial for leveraging its full potential. This section will cover the fundamental data types in Redis, including Strings, Lists, Sets, Hashes, and Sorted Sets.

Strings

Strings are the most basic type of value you can store in Redis. They are binary-safe, meaning they can contain any data, such as a JPEG image or a serialized Ruby object.

  • Key Concepts:

    • Strings can store up to 512 MB of data.
    • Common operations include setting, getting, and appending values.

  • Examples:

SET mykey "Hello, Redis!"
GET mykey

Explanation:

  • SET mykey "Hello, Redis!" stores the string "Hello, Redis!" under the key mykey.
  • GET mykey retrieves the value stored at mykey.

Lists

Lists are collections of ordered values. They are implemented as linked lists, which means that inserting an element at the head or the tail is very efficient.

  • Key Concepts:

    • Lists can hold up to 2^32 - 1 elements (over 4 billion).
    • Common operations include pushing and popping elements from both ends.

  • Examples:

LPUSH mylist "World"
LPUSH mylist "Hello"
LRANGE mylist 0 -1

Explanation:

  • LPUSH mylist "World" pushes "World" onto the list mylist.
  • LPUSH mylist "Hello" pushes "Hello" onto the list mylist.
  • LRANGE mylist 0 -1 retrieves all elements in the list mylist.

Sets

Sets are collections of unique, unordered values. They are useful for storing unique items and performing set operations like unions and intersections.

  • Key Concepts:

    • Sets do not allow duplicate values.
    • Common operations include adding, removing, and checking for membership.

  • Examples:

SADD myset "Hello"
SADD myset "World"
SADD myset "Hello"  # This will not add "Hello" again
SMEMBERS myset

Explanation:

  • SADD myset "Hello" adds "Hello" to the set myset.
  • SADD myset "World" adds "World" to the set myset.
  • SMEMBERS myset retrieves all members of the set myset.

Hashes

Hashes are maps between string fields and string values. They are perfect for representing objects.

  • Key Concepts:

    • Hashes are ideal for storing small pieces of data.
    • Common operations include setting, getting, and deleting fields.

  • Examples:

HSET myhash field1 "Hello"
HSET myhash field2 "World"
HGET myhash field1
HGETALL myhash

Explanation:

  • HSET myhash field1 "Hello" sets the field field1 in the hash myhash to "Hello".
  • HSET myhash field2 "World" sets the field field2 in the hash myhash to "World".
  • HGET myhash field1 retrieves the value of field1 in the hash myhash.
  • HGETALL myhash retrieves all fields and values in the hash myhash.

Sorted Sets

Sorted Sets are similar to Sets but with an associated score for each member. Members are ordered by their scores.

  • Key Concepts:

    • Sorted Sets are useful for ranking systems.
    • Common operations include adding elements with scores, retrieving elements by score, and removing elements.

  • Examples:

ZADD myzset 1 "one"
ZADD myzset 2 "two"
ZRANGE myzset 0 -1 WITHSCORES

Explanation:

  • ZADD myzset 1 "one" adds the member "one" with a score of 1 to the sorted set myzset.
  • ZADD myzset 2 "two" adds the member "two" with a score of 2 to the sorted set myzset.
  • ZRANGE myzset 0 -1 WITHSCORES retrieves all members in the sorted set myzset along with their scores.

Conclusion

Understanding the basic data types in Redis is essential for effectively using this powerful in-memory data structure store. Each data type serves different use cases, from simple key-value storage with Strings to more complex data structures like Lists, Sets, Hashes, and Sorted Sets. Mastering these basics will provide a solid foundation for more advanced Redis features and operations.
