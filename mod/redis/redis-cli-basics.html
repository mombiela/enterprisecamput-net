Introduction

Redis CLI (Command Line Interface) is a powerful tool that allows you to interact with your Redis server directly from the command line. This section will cover the basics of using Redis CLI, including connecting to a Redis server, executing commands, and understanding the output.

Connecting to Redis Server

To start using Redis CLI, you first need to connect to your Redis server. This is typically done using the redis-cli command followed by the server's address and port.

Example

redis-cli -h 127.0.0.1 -p 6379
  • -h: Specifies the host (default is 127.0.0.1).
  • -p: Specifies the port (default is 6379).

Basic Commands

Once connected, you can start executing Redis commands. Here are some of the most commonly used commands:

PING

The PING command checks if the server is running.

PING
  • Expected output: PONG

SET and GET

The SET command stores a key-value pair, and the GET command retrieves the value of a key.

SET mykey "Hello, Redis!"
GET mykey
  • Expected output for GET mykey: "Hello, Redis!"

DEL

The DEL command deletes a key.

DEL mykey
  • Expected output: (integer) 1 (if the key existed and was deleted)

Working with Data Types

Redis supports various data types, including strings, lists, sets, hashes, and sorted sets. Here are some basic commands for each type:

Strings

SET key1 "value1"
GET key1

Lists

LPUSH mylist "item1"
LPUSH mylist "item2"
LRANGE mylist 0 -1
  • LPUSH: Adds an item to the list.
  • LRANGE: Retrieves items from the list.

Sets

SADD myset "member1"
SADD myset "member2"
SMEMBERS myset
  • SADD: Adds a member to the set.
  • SMEMBERS: Retrieves all members of the set.

Hashes

HSET myhash field1 "value1"
HGET myhash field1
  • HSET: Sets the value of a field in a hash.
  • HGET: Retrieves the value of a field in a hash.

Sorted Sets

ZADD myzset 1 "member1"
ZADD myzset 2 "member2"
ZRANGE myzset 0 -1 WITHSCORES
  • ZADD: Adds a member with a score to the sorted set.
  • ZRANGE: Retrieves members in a specified range.

Useful CLI Options

Redis CLI offers several options to enhance your experience:

Interactive Mode

Start Redis CLI in interactive mode by simply running:

redis-cli
  • This mode allows you to enter commands one at a time and see the results immediately.

Non-Interactive Mode

You can also run commands directly from the shell without entering interactive mode:

redis-cli SET key "value"
  • This is useful for scripting and automation.

Pipelining

To execute multiple commands in a single request, use pipelining:

redis-cli --pipe
  • This can significantly improve performance for batch operations.

Conclusion

Redis CLI is an essential tool for managing and interacting with your Redis server. By understanding the basic commands and options, you can efficiently perform various operations and troubleshoot issues. Practice using these commands to become more comfortable with Redis CLI, and you'll be well-prepared to tackle more advanced topics in Redis.
