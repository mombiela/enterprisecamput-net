Introduction to Redis Commands

Redis is an in-memory data structure store, used as a database, cache, and message broker. To interact with Redis, you use a set of commands that perform various operations on the data stored in Redis. This section will cover the basic commands you need to get started with Redis.

Connecting to Redis

Before you can run any commands, you need to connect to your Redis server. You can do this using the Redis CLI (Command Line Interface).

redis-cli

Once connected, you can start issuing commands.

Basic Data Types and Commands

Strings

Strings are the most basic type of value you can store in Redis. They are binary safe, meaning they can contain any kind of data, such as a JPEG image or a serialized Ruby object.

  • SET: Set the value of a key.

    SET key "value"

    Example:

    SET name "John"

  • GET: Get the value of a key.

    GET key

    Example:

    GET name

  • DEL: Delete a key.

    DEL key

    Example:

    DEL name

Hashes

Hashes are maps between string fields and string values, making them perfect for representing objects.

  • HSET: Set the value of a field in a hash.

    HSET key field value

    Example:

    HSET user:1000 name "John"

  • HGET: Get the value of a field in a hash.

    HGET key field

    Example:

    HGET user:1000 name

  • HDEL: Delete a field in a hash.

    HDEL key field

    Example:

    HDEL user:1000 name

Lists

Lists are ordered collections of strings. You can add elements to the head or the tail of the list.

  • LPUSH: Add one or more elements to the head of a list.

    LPUSH key value [value ...]

    Example:

    LPUSH mylist "world"
LPUSH mylist "hello"

  • LRANGE: Get a range of elements from a list.

    LRANGE key start stop

    Example:

    LRANGE mylist 0 -1

  • LPOP: Remove and get the first element in a list.

    LPOP key

    Example:

    LPOP mylist

Sets

Sets are unordered collections of unique strings.

  • SADD: Add one or more members to a set.

    SADD key member [member ...]

    Example:

    SADD myset "hello"
SADD myset "world"

  • SMEMBERS: Get all the members in a set.

    SMEMBERS key

    Example:

    SMEMBERS myset

  • SREM: Remove one or more members from a set.

    SREM key member [member ...]

    Example:

    SREM myset "hello"

Sorted Sets

Sorted sets are similar to sets but where every member is associated with a score. The elements are ordered by their scores.

  • ZADD: Add one or more members to a sorted set, or update its score if it already exists.

    ZADD key score member [score member ...]

    Example:

    ZADD myzset 1 "one"
ZADD myzset 2 "two"

  • ZRANGE: Return a range of members in a sorted set, by index.

    ZRANGE key start stop [WITHSCORES]

    Example:

    ZRANGE myzset 0 -1

  • ZREM: Remove one or more members from a sorted set.

    ZREM key member [member ...]

    Example:

    ZREM myzset "one"

Conclusion

In this section, we covered the basic commands for interacting with Redis. We explored how to connect to Redis and perform operations on different data types such as strings, hashes, lists, sets, and sorted sets. Understanding these basic commands is essential for effectively using Redis in your applications. As you progress, you will learn more advanced commands and techniques to leverage the full power of Redis.
