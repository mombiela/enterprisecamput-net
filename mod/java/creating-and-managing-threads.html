Introduction to Threads

Threads allow a program to operate more efficiently by doing multiple things at the same time. They are particularly useful in Java for performing background tasks, improving performance, and managing asynchronous events.

  • Definition: A thread is a lightweight process that can run concurrently with other threads within the same program.
  • Use Cases: Background tasks, parallel processing, handling multiple client requests in a server, etc.

Creating Threads in Java

There are two main ways to create a thread in Java:

  1. Extending the Thread class
  2. Implementing the Runnable interface

Extending the `Thread` Class

To create a thread by extending the Thread class, you need to follow these steps:

  1. Extend the Thread class.
  2. Override the run() method.
  3. Create an instance of your class and call the start() method.
class MyThread extends Thread {
    public void run() {
        System.out.println("Thread is running.");
    }

    public static void main(String[] args) {
        MyThread thread = new MyThread();
        thread.start(); // This will call the run() method
    }
}

Implementing the `Runnable` Interface

To create a thread by implementing the Runnable interface, follow these steps:

  1. Implement the Runnable interface.
  2. Override the run() method.
  3. Create an instance of Thread class, passing your Runnable object to its constructor.
  4. Call the start() method on the Thread instance.
class MyRunnable implements Runnable {
    public void run() {
        System.out.println("Thread is running.");
    }

    public static void main(String[] args) {
        MyRunnable myRunnable = new MyRunnable();
        Thread thread = new Thread(myRunnable);
        thread.start(); // This will call the run() method
    }
}

Managing Threads

Managing threads involves controlling their execution and ensuring that they run efficiently without conflicts.

Thread Lifecycle

A thread can be in one of the following states:

  • New: When a thread is created but not yet started.
  • Runnable: When the thread is ready to run and waiting for CPU time.
  • Blocked/Waiting: When the thread is waiting for a resource or another thread to perform a task.
  • Timed Waiting: When the thread is waiting for a specified period.
  • Terminated: When the thread has finished executing.

Thread Methods

Some important methods for managing threads include:

  • start(): Starts the execution of the thread.
  • sleep(long millis): Causes the thread to sleep for the specified number of milliseconds.
  • join(): Waits for the thread to die.
  • interrupt(): Interrupts the thread.
  • isAlive(): Checks if the thread is still running.

Example: Using `sleep` and `join`

class MyRunnable implements Runnable {
    public void run() {
        for (int i = 0; i < 5; i++) {
            System.out.println("Thread: " + i);
            try {
                Thread.sleep(1000); // Sleep for 1 second
            } catch (InterruptedException e) {
                System.out.println("Thread interrupted.");
            }
        }
    }

    public static void main(String[] args) {
        Thread thread = new Thread(new MyRunnable());
        thread.start();
        try {
            thread.join(); // Wait for the thread to finish
        } catch (InterruptedException e) {
            System.out.println("Main thread interrupted.");
        }
        System.out.println("Main thread finished.");
    }
}

Synchronization

When multiple threads access shared resources, synchronization is crucial to prevent data inconsistency.

Synchronized Methods

You can use the synchronized keyword to make methods thread-safe.

class Counter {
    private int count = 0;

    public synchronized void increment() {
        count++;
    }

    public int getCount() {
        return count;
    }
}

class MyRunnable implements Runnable {
    private Counter counter;

    public MyRunnable(Counter counter) {
        this.counter = counter;
    }

    public void run() {
        for (int i = 0; i < 1000; i++) {
            counter.increment();
        }
    }

    public static void main(String[] args) throws InterruptedException {
        Counter counter = new Counter();
        Thread t1 = new Thread(new MyRunnable(counter));
        Thread t2 = new Thread(new MyRunnable(counter));
        t1.start();
        t2.start();
        t1.join();
        t2.join();
        System.out.println("Final count: " + counter.getCount());
    }
}

Advanced Thread Management

For more advanced thread management, Java provides the java.util.concurrent package, which includes classes like ExecutorService, Future, and Callable.

Using `ExecutorService`

The ExecutorService framework simplifies thread management by handling thread creation and task scheduling.

import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

class MyRunnable implements Runnable {
    public void run() {
        System.out.println("Thread is running.");
    }

    public static void main(String[] args) {
        ExecutorService executor = Executors.newFixedThreadPool(2);
        executor.submit(new MyRunnable());
        executor.submit(new MyRunnable());
        executor.shutdown();
    }
}

Conclusion

Creating and managing threads in Java is essential for building efficient, high-performance applications. By understanding the basics of thread creation, synchronization, and advanced thread management, you can develop robust multi-threaded programs. Practice with the provided examples and explore the java.util.concurrent package for more advanced features.
