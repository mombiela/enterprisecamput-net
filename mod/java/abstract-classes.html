Introduction to Abstract Classes

Abstract classes in Java are a fundamental concept that allows you to define classes that cannot be instantiated on their own. They are used to provide a base for subclasses to build upon, ensuring a common structure and behavior.

Key Concepts

  • Abstract Class: A class declared with the abstract keyword. It cannot be instantiated directly.
  • Abstract Method: A method declared without an implementation, using the abstract keyword. Subclasses must provide an implementation for these methods.
  • Concrete Method: A method in an abstract class that has an implementation. Subclasses can inherit or override these methods.

Defining Abstract Classes

An abstract class is defined using the abstract keyword. It can contain both abstract methods (without a body) and concrete methods (with a body).

abstract class Animal {
    // Abstract method (does not have a body)
    abstract void makeSound();

    // Concrete method (has a body)
    void sleep() {
        System.out.println("Zzz...");
    }
}

Example Explained

  • Animal is an abstract class.
  • makeSound() is an abstract method, meaning subclasses must implement this method.
  • sleep() is a concrete method, meaning it provides a default implementation that can be inherited by subclasses.

Implementing Abstract Classes

To use an abstract class, you need to create a subclass that extends the abstract class and provides implementations for all abstract methods.

class Dog extends Animal {
    // Providing implementation for the abstract method
    void makeSound() {
        System.out.println("Woof");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog myDog = new Dog();
        myDog.makeSound(); // Outputs: Woof
        myDog.sleep();     // Outputs: Zzz...
    }
}

Example Explained

  • Dog is a subclass of Animal.
  • Dog provides an implementation for the makeSound() method.
  • In the main method, we create an instance of Dog and call its methods.

Abstract Classes vs Interfaces

Both abstract classes and interfaces are used to achieve abstraction in Java, but they have some differences.

| Feature | Abstract Class | Interface | |------------------------|----------------------------------------|----------------------------------------| | Methods | Can have both abstract and concrete methods | Can only have abstract methods (Java 7 and earlier) or default/static methods (Java 8 and later) | | Fields | Can have instance variables | Can only have static final variables | | Multiple Inheritance | Does not support multiple inheritance | Supports multiple inheritance | | Constructor | Can have constructors | Cannot have constructors |

Example of Interface for Comparison

interface AnimalInterface {
    void makeSound();
    default void sleep() {
        System.out.println("Zzz...");
    }
}

class Cat implements AnimalInterface {
    public void makeSound() {
        System.out.println("Meow");
    }
}

public class Main {
    public static void main(String[] args) {
        Cat myCat = new Cat();
        myCat.makeSound(); // Outputs: Meow
        myCat.sleep();     // Outputs: Zzz...
    }
}

When to Use Abstract Classes

  • When you want to share code among several closely related classes.
  • When you expect that classes extending your abstract class will have many common methods or fields.
  • When you want to define non-static or non-final fields.

Conclusion

Abstract classes in Java provide a powerful way to define a common template for a group of related classes. They allow you to enforce a contract for subclasses while also providing shared code. Understanding when and how to use abstract classes is crucial for designing robust and maintainable object-oriented systems.
