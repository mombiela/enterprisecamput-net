Introduction

In Java, a thread is a lightweight process that allows concurrent execution of code. Understanding the lifecycle of a thread is crucial for writing efficient and effective multithreaded applications. This section will cover the different states a thread can be in, how to transition between these states, and provide code examples to illustrate these concepts.

Thread States

A thread in Java can be in one of the following states:

  1. New
  2. Runnable
  3. Blocked
  4. Waiting
  5. Timed Waiting
  6. Terminated

New

  • A thread is in the "New" state when it is created but not yet started.
  • It remains in this state until the start() method is called.
Thread thread = new Thread(() -> {
    System.out.println("Thread is running");
});
// Thread is in New state

Runnable

  • A thread enters the "Runnable" state when the start() method is called.
  • In this state, the thread is ready to run and is waiting for CPU time.
thread.start(); // Thread is now in Runnable state

Blocked

  • A thread enters the "Blocked" state when it is waiting for a monitor lock to enter a synchronized block/method.
  • It transitions back to the "Runnable" state once it acquires the lock.
synchronized (someObject) {
    // Thread is in Blocked state if it cannot acquire the lock
}

Waiting

  • A thread enters the "Waiting" state when it is waiting indefinitely for another thread to perform a particular action.
  • Methods that can cause a thread to enter this state include Object.wait(), Thread.join(), and LockSupport.park().
synchronized (someObject) {
    someObject.wait(); // Thread is in Waiting state
}

Timed Waiting

  • A thread enters the "Timed Waiting" state when it is waiting for another thread to perform an action for a specified waiting time.
  • Methods that can cause a thread to enter this state include Thread.sleep(), Object.wait(long timeout), Thread.join(long millis), LockSupport.parkNanos(), and LockSupport.parkUntil().
Thread.sleep(1000); // Thread is in Timed Waiting state for 1 second

Terminated

  • A thread enters the "Terminated" state when it has completed its execution or has been explicitly terminated.
  • Once a thread is in this state, it cannot be restarted.
// Thread is in Terminated state after run() method completes

Thread Lifecycle Transitions

The following table summarizes the transitions between different thread states:

| Current State | Transition Method | Next State | |---------------|-------------------|------------------| | New | start() | Runnable | | Runnable | Acquires CPU time | Running | | Running | yield() | Runnable | | Running | wait() | Waiting | | Running | sleep() | Timed Waiting | | Running | join() | Waiting/Timed Waiting | | Running | Completes run() | Terminated | | Waiting | notify()/notifyAll() | Runnable | | Timed Waiting | Time expires | Runnable | | Blocked | Acquires lock | Runnable |

Code Example

Below is a complete example demonstrating the different states of a thread:

public class ThreadLifecycleDemo {
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            try {
                System.out.println("Thread is in Runnable state");
                Thread.sleep(1000); // Timed Waiting state
                synchronized (ThreadLifecycleDemo.class) {
                    ThreadLifecycleDemo.class.wait(); // Waiting state
                }
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println("Thread is in Terminated state");
        });

        System.out.println("Thread is in New state");
        thread.start(); // Runnable state

        try {
            Thread.sleep(500); // Main thread sleeps to allow child thread to start
            synchronized (ThreadLifecycleDemo.class) {
                ThreadLifecycleDemo.class.notify(); // Notify the waiting thread
            }
            thread.join(); // Main thread waits for child thread to terminate
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}

Conclusion

Understanding the thread lifecycle is essential for managing thread states and transitions effectively in a multithreaded Java application. By mastering these concepts, you can write more efficient and robust programs that make better use of system resources.
