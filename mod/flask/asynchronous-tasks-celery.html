Introduction to Celery

Celery is an asynchronous task queue/job queue based on distributed message passing. It is focused on real-time operation but supports scheduling as well. Celery is used for handling asynchronous tasks in web applications, allowing you to offload long-running tasks to the background, improving the responsiveness of your application.

Key Concepts

  • Task Queue: A mechanism to distribute work across threads or machines.
  • Broker: A message transport system that sends and receives messages. Common brokers are RabbitMQ and Redis.
  • Worker: A process that executes the tasks.
  • Task: A unit of work that is executed asynchronously.

Setting Up Celery with Flask

To get started with Celery in a Flask application, you need to install Celery and a message broker like Redis.

Installation

pip install celery redis

Configuration

Create a celery.py file to configure Celery with your Flask application.

from celery import Celery

def make_celery(app):
    celery = Celery(
        app.import_name,
        backend=app.config['CELERY_RESULT_BACKEND'],
        broker=app.config['CELERY_BROKER_URL']
    )
    celery.conf.update(app.config)
    return celery

Integrating Celery with Flask

In your Flask application, initialize Celery with the Flask app context.

from flask import Flask
from celery import Celery

def create_app():
    app = Flask(__name__)
    app.config.update(
        CELERY_BROKER_URL='redis://localhost:6379/0',
        CELERY_RESULT_BACKEND='redis://localhost:6379/0'
    )

    celery = make_celery(app)

    @app.route('/process/<name>')
    def process(name):
        reverse_name.delay(name)
        return f'Processing {name}...'

    return app

@celery.task
def reverse_name(name):
    return name[::-1]

if __name__ == '__main__':
    app = create_app()
    app.run(debug=True)

Creating and Running Tasks

Defining Tasks

Tasks are defined using the @celery.task decorator.

@celery.task
def add(x, y):
    return x + y

Calling Tasks

Tasks can be called asynchronously using the delay method.

result = add.delay(4, 6)

Checking Task Status

You can check the status of a task using the AsyncResult object.

from celery.result import AsyncResult

result = AsyncResult(task_id)
if result.state == 'PENDING':
    print('Task is pending...')
elif result.state == 'SUCCESS':
    print('Task completed successfully:', result.result)

Advanced Usage

Task Retries

Celery allows tasks to be retried in case of failure.

@celery.task(bind=True, max_retries=3)
def add(self, x, y):
    try:
        return x + y
    except Exception as exc:
        raise self.retry(exc=exc, countdown=60)

Task Scheduling

Celery can also be used to schedule tasks.

from celery.schedules import crontab

celery.conf.beat_schedule = {
    'add-every-morning': {
        'task': 'add',
        'schedule': crontab(hour=7, minute=30),
        'args': (16, 16),
    },
}

Conclusion

Celery is a powerful tool for managing asynchronous tasks in a Flask application. By offloading long-running tasks to the background, you can improve the responsiveness and performance of your web application. With features like task retries and scheduling, Celery provides a robust solution for handling various asynchronous operations.
