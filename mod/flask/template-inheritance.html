Template Inheritance in Flask
Working with Static Files ►

Template inheritance is a powerful feature in Flask that allows you to create a base template and extend it in other templates. This helps in maintaining a consistent look and feel across your web application and makes your code more modular and easier to manage.

Key Concepts

  • Base Template: A template that contains the common structure and elements (like header, footer, navigation) that are shared across multiple pages.
  • Child Template: A template that extends the base template and overrides specific blocks to provide unique content for different pages.
  • Blocks: Sections within a template that can be overridden by child templates.

Creating a Base Template

A base template typically includes the HTML structure and common elements like the header and footer. Here's an example of a base template named base.html:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>{% block title %}My Flask App{% endblock %}</title>
</head>
<body>
    <header>
        <h1>Welcome to My Flask App</h1>
    </header>
    <nav>
        <ul>
            <li><a href="/">Home</a></li>
            <li><a href="/about">About</a></li>
        </ul>
    </nav>
    <main>
        {% block content %}{% endblock %}
    </main>
    <footer>
        <p>&copy; 2023 My Flask App</p>
    </footer>
</body>
</html>

Explanation

  • {% block title %}My Flask App{% endblock %}: This block allows child templates to set their own title.
  • {% block content %}{% endblock %}: This block will be overridden by child templates to provide unique content for different pages.

Creating Child Templates

Child templates extend the base template and override specific blocks to provide unique content. Here's an example of a child template named home.html:

{% extends "base.html" %}

{% block title %}Home - My Flask App{% endblock %}

{% block content %}
    <h2>Home Page</h2>
    <p>Welcome to the home page of my Flask app!</p>
{% endblock %}

And another child template named about.html:

{% extends "base.html" %}

{% block title %}About - My Flask App{% endblock %}

{% block content %}
    <h2>About Us</h2>
    <p>This is the about page of my Flask app.</p>
{% endblock %}

Explanation

  • {% extends "base.html" %}: This line tells Flask that this template extends the base.html template.
  • {% block title %}Home - My Flask App{% endblock %}: This block overrides the title block in the base template.
  • {% block content %}: This block overrides the content block in the base template to provide unique content for the home and about pages.

Using Templates in Flask Routes

To render these templates in your Flask application, you need to set up routes in your app.py file:

from flask import Flask, render_template

app = Flask(__name__)

@app.route('/')
def home():
    return render_template('home.html')

@app.route('/about')
def about():
    return render_template('about.html')

if __name__ == '__main__':
    app.run(debug=True)

Explanation

  • render_template('home.html'): This function renders the home.html template.
  • render_template('about.html'): This function renders the about.html template.

Conclusion

Template inheritance in Flask allows you to create a consistent and maintainable structure for your web application. By using a base template and extending it in child templates, you can easily manage common elements and unique content across different pages. This approach not only saves time but also makes your code more modular and easier to maintain.
