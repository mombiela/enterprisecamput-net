Rendering Templates
Jinja2 Templating Engine ►

In this section, we'll explore how to render templates in Flask, a crucial aspect of building dynamic web applications. We'll cover the basics of template rendering, how to pass data to templates, and advanced techniques for creating reusable and maintainable templates.

Introduction to Templates

Templates allow you to separate the presentation layer from the business logic of your application. Flask uses Jinja2 as its template engine, which provides a powerful way to generate HTML dynamically.

Key Concepts

  • Template Engine: A software component that combines templates with a data model to produce documents.
  • Jinja2: The default template engine for Flask, which allows for dynamic content generation.
  • Rendering: The process of generating the final HTML from templates and data.

Basic Template Rendering

To render a template in Flask, you use the render_template function. This function takes the name of the template file and any data you want to pass to it.

Example

  1. Create a Template File: Save the following HTML code in a file named index.html inside a folder named templates.

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>{{ title }}</title>
</head>
<body>
    <h1>{{ heading }}</h1>
    <p>{{ message }}</p>
</body>
</html>

  2. Render the Template in Flask: Use the render_template function in your Flask route.

    from flask import Flask, render_template

app = Flask(__name__)

@app.route('/')
def home():
    return render_template('index.html', title='Home Page', heading='Welcome to Flask', message='This is a basic template rendering example.')

if __name__ == '__main__':
    app.run(debug=True)

Explanation

  • Template Variables: {{ title }}, {{ heading }}, and {{ message }} are placeholders in the template that will be replaced with the values passed from the Flask route.
  • render_template Function: This function takes the template file name and keyword arguments for the data to be passed to the template.

Passing Data to Templates

You can pass various types of data to templates, including strings, lists, dictionaries, and more.

Example with List and Dictionary

  1. Template File: Save the following HTML code in a file named data.html.

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Data Example</title>
</head>
<body>
    <h1>Users</h1>
    <ul>
        {% for user in users %}
            <li>{{ user.name }} - {{ user.email }}</li>
        {% endfor %}
    </ul>
</body>
</html>

  2. Render the Template with Data:

    @app.route('/users')
def users():
    users = [
        {'name': 'Alice', 'email': 'alice@example.com'},
        {'name': 'Bob', 'email': 'bob@example.com'},
        {'name': 'Charlie', 'email': 'charlie@example.com'}
    ]
    return render_template('data.html', users=users)

Explanation

  • Looping in Templates: {% for user in users %} is a Jinja2 loop that iterates over the list of users and renders each user's name and email.
  • Template Context: The users variable is passed to the template and can be accessed within the template.

Advanced Template Features

Flask and Jinja2 offer advanced features to make templates more powerful and maintainable.

Template Inheritance

Template inheritance allows you to create a base template that other templates can extend. This is useful for maintaining a consistent layout across multiple pages.

  1. Base Template: Save the following HTML code in a file named base.html.

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>{% block title %}My Website{% endblock %}</title>
</head>
<body>
    <header>
        <h1>My Website</h1>
    </header>
    <main>
        {% block content %}{% endblock %}
    </main>
    <footer>
        <p>&copy; 2023 My Website</p>
    </footer>
</body>
</html>

  2. Child Template: Save the following HTML code in a file named child.html.

    {% extends "base.html" %}

{% block title %}Home Page{% endblock %}

{% block content %}
    <h2>Welcome to the Home Page</h2>
    <p>This is the content of the home page.</p>
{% endblock %}

  3. Render the Child Template:

    @app.route('/home')
def home():
    return render_template('child.html')

Explanation

  • Template Blocks: {% block title %} and {% block content %} define sections of the template that can be overridden by child templates.
  • Extending Templates: {% extends "base.html" %} indicates that child.html extends base.html.

Conclusion

Rendering templates in Flask is a powerful way to create dynamic web pages. By understanding the basics of template rendering, passing data to templates, and using advanced features like template inheritance, you can build maintainable and scalable web applications. Practice these concepts with different types of data and templates to become proficient in Flask template rendering.
