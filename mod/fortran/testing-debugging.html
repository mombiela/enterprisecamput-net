Testing and Debugging in Fortran
Presentation and Review ►

Introduction

Testing and debugging are crucial aspects of software development, ensuring that your Fortran programs run correctly and efficiently. This section will cover the basics of testing and debugging, progressing from beginner to advanced levels.

Basics of Testing

What is Testing?

  • Definition: Testing is the process of executing a program with the intent of finding errors.
  • Purpose: To ensure the program behaves as expected under various conditions.

Types of Testing

  • Unit Testing: Testing individual components or functions.
  • Integration Testing: Testing combined parts of an application to ensure they work together.
  • System Testing: Testing the complete system as a whole.
  • Acceptance Testing: Testing to ensure the system meets the requirements.

Writing Test Cases

  • Identify Test Scenarios: Determine what needs to be tested.
  • Create Test Data: Prepare input data for testing.
  • Define Expected Results: Specify what the correct output should be.

Example of a Simple Test Case

program test_addition
  implicit none
  integer :: result

  result = add(2, 3)
  if (result /= 5) then
    print *, "Test Failed: Expected 5, got ", result
  else
    print *, "Test Passed"
  end if
end program test_addition

function add(a, b)
  integer :: add
  integer, intent(in) :: a, b
  add = a + b
end function add
  • Explanation: This program tests the add function. It checks if adding 2 and 3 results in 5.

Basics of Debugging

What is Debugging?

  • Definition: Debugging is the process of identifying, analyzing, and removing errors from a program.
  • Purpose: To fix bugs and ensure the program runs correctly.

Common Debugging Techniques

  • Print Statements: Insert print statements to display variable values and program flow.
  • Using Debuggers: Utilize tools like GDB (GNU Debugger) for more advanced debugging.
  • Code Review: Reviewing code with peers to find potential issues.

Example of Debugging with Print Statements

program debug_example
  implicit none
  integer :: a, b, result

  a = 5
  b = 0
  print *, "Before division: a =", a, ", b =", b
  result = a / b
  print *, "After division: result =", result
end program debug_example
  • Explanation: This program will crash due to division by zero. The print statements help identify the values of a and b before the error occurs.

Advanced Testing Techniques

Automated Testing

  • Definition: Using scripts and tools to run tests automatically.
  • Tools: Frameworks like pFUnit for Fortran.

Example of Automated Testing with pFUnit

module test_addition
  use pfunit
  implicit none

  @test
  subroutine test_add
    integer :: result
    result = add(2, 3)
    @assertEqual(result, 5)
  end subroutine test_add

  function add(a, b)
    integer :: add
    integer, intent(in) :: a, b
    add = a + b
  end function add
end module test_addition
  • Explanation: This module uses pFUnit to automate the test for the add function.

Advanced Debugging Techniques

Using GDB with Fortran

  • Setting Breakpoints: Pause execution at specific lines.
  • Inspecting Variables: Check the values of variables at different points.
  • Stepping Through Code: Execute code line-by-line to find errors.

Example of Using GDB

  1. Compile with Debug Information: 
    gfortran -g -o debug_example debug_example.f90
  2. Run GDB: 
    gdb ./debug_example
  3. Set Breakpoint and Run: 
    (gdb) break debug_example.f90:6
(gdb) run

Conclusion

Testing and debugging are essential skills for any Fortran programmer. Starting with basic techniques like writing test cases and using print statements, you can progress to more advanced methods such as automated testing with pFUnit and using GDB for debugging. By mastering these techniques, you can ensure your Fortran programs are robust, reliable, and efficient.
