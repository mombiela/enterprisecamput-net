Introduction

Code style and documentation are crucial aspects of programming that ensure code readability, maintainability, and collaboration. In this section, we will cover best practices for writing clean and well-documented Fortran code, suitable for both beginners and advanced programmers.

Code Style

Indentation and Spacing

  • Consistent Indentation: Use consistent indentation throughout your code. The common practice is to use 2 or 4 spaces per indentation level.

  • Spacing Around Operators: Add spaces around operators to improve readability.

    ! Good example
a = b + c * d

! Bad example
a=b+c*d

Line Length

  • Line Length Limit: Keep lines within 72 characters for fixed-form source and 132 characters for free-form source to ensure readability and compatibility with various editors and tools.

Naming Conventions

  • Variables and Functions: Use meaningful names for variables and functions. Use lowercase letters with underscores for variable names and camelCase for function names.

    ! Good example
integer :: total_sum
real :: averageValue

! Bad example
integer :: ts
real :: avgval

Commenting

  • Inline Comments: Use inline comments to explain complex logic or important steps in the code.

    ! Calculate the average value
averageValue = total_sum / number_of_elements

  • Block Comments: Use block comments to describe the purpose of a section of code or a function.

    !--------------------------------------------------
! This function calculates the factorial of a number
! Input: integer n
! Output: integer factorial
!--------------------------------------------------

Documentation

File Headers

  • File Header Comments: Include a header comment at the beginning of each file to describe the file's purpose, author, date, and any relevant notes.

    !--------------------------------------------------
! File: factorial.f90
! Purpose: Calculate the factorial of a number
! Author: Jane Doe
! Date: 2023-10-01
!--------------------------------------------------

Function and Subroutine Documentation

  • Function Headers: Document each function and subroutine with a header comment that describes its purpose, inputs, outputs, and any side effects.

    !--------------------------------------------------
! Function: factorial
! Purpose: Calculate the factorial of a number
! Input: integer n
! Output: integer factorial
!--------------------------------------------------
function factorial(n)
  integer, intent(in) :: n
  integer :: factorial
  ! Function logic here
end function factorial

Inline Documentation

  • Inline Documentation: Use comments within the function or subroutine to explain complex logic or important steps.

    function factorial(n)
  integer, intent(in) :: n
  integer :: factorial
  integer :: i

  factorial = 1
  do i = 1, n
    factorial = factorial * i
    ! Multiply factorial by the current value of i
  end do
end function factorial

Example: Well-Documented Fortran Code

!--------------------------------------------------
! File: example.f90
! Purpose: Demonstrate well-documented Fortran code
! Author: John Smith
! Date: 2023-10-01
!--------------------------------------------------

program example
  implicit none
  integer :: num, result

  ! Prompt user for input
  print *, 'Enter a number:'
  read *, num

  ! Calculate factorial
  result = factorial(num)

  ! Display result
  print *, 'The factorial of', num, 'is', result
end program example

!--------------------------------------------------
! Function: factorial
! Purpose: Calculate the factorial of a number
! Input: integer n
! Output: integer factorial
!--------------------------------------------------
function factorial(n)
  integer, intent(in) :: n
  integer :: factorial
  integer :: i

  factorial = 1
  do i = 1, n
    factorial = factorial * i
    ! Multiply factorial by the current value of i
  end do
end function factorial

Conclusion

Adhering to good code style and documentation practices in Fortran is essential for creating readable, maintainable, and collaborative code. By following consistent indentation, naming conventions, and thorough documentation, you can ensure that your code is easy to understand and maintain, both for yourself and for others who may work with your code in the future.
