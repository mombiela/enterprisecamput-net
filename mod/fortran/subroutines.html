Subroutines in Fortran
Functions ►

Subroutines are a fundamental concept in Fortran, allowing you to encapsulate and reuse code. This section will guide you from the basics to advanced usage of subroutines in Fortran.

Introduction to Subroutines

Subroutines in Fortran are similar to functions in other programming languages. They allow you to break down your program into smaller, manageable pieces. Subroutines can take arguments, perform operations, and return results.

Key Concepts

  • Definition: A subroutine is defined using the SUBROUTINE keyword.
  • Arguments: Subroutines can accept input arguments and return results through these arguments.
  • Call: Subroutines are invoked using the CALL statement.
  • Scope: Variables within a subroutine are local to that subroutine unless explicitly shared.

Basic Subroutine Structure

Here's a simple example to illustrate the basic structure of a subroutine in Fortran.

PROGRAM Main
    CALL Greet
END PROGRAM Main

SUBROUTINE Greet
    PRINT *, "Hello, World!"
END SUBROUTINE Greet

Explanation

  • PROGRAM Main: The main program block.
  • CALL Greet: Invokes the Greet subroutine.
  • SUBROUTINE Greet: Defines the subroutine named Greet.
  • PRINT: Outputs "Hello, World!" to the console.
  • END SUBROUTINE Greet: Marks the end of the subroutine.

Subroutines with Arguments

Subroutines can take arguments to perform operations on input data.

PROGRAM Main
    INTEGER :: num1, num2, result
    num1 = 10
    num2 = 20
    CALL Add(num1, num2, result)
    PRINT *, "The sum is: ", result
END PROGRAM Main

SUBROUTINE Add(a, b, sum)
    INTEGER, INTENT(IN) :: a, b
    INTEGER, INTENT(OUT) :: sum
    sum = a + b
END SUBROUTINE Add

Explanation

  • Arguments: a, b, and sum are arguments of the Add subroutine.
  • INTENT(IN): Specifies that a and b are input arguments.
  • INTENT(OUT): Specifies that sum is an output argument.
  • sum = a + b: Performs the addition and stores the result in sum.

Advanced Subroutine Features

Optional Arguments

Subroutines can have optional arguments, which are not required when the subroutine is called.

PROGRAM Main
    CALL PrintMessage("Hello, World!")
    CALL PrintMessage()
END PROGRAM Main

SUBROUTINE PrintMessage(msg)
    CHARACTER(LEN=*), OPTIONAL :: msg
    IF (PRESENT(msg)) THEN
        PRINT *, msg
    ELSE
        PRINT *, "No message provided."
    END IF
END SUBROUTINE PrintMessage

Explanation

  • OPTIONAL: Indicates that msg is an optional argument.
  • PRESENT: Checks if the optional argument msg is provided.

Recursive Subroutines

Subroutines can call themselves recursively.

PROGRAM Main
    INTEGER :: result
    result = Factorial(5)
    PRINT *, "Factorial of 5 is: ", result
END PROGRAM Main

RECURSIVE SUBROUTINE Factorial(n, result)
    INTEGER, INTENT(IN) :: n
    INTEGER, INTENT(OUT) :: result
    IF (n <= 1) THEN
        result = 1
    ELSE
        CALL Factorial(n - 1, result)
        result = n * result
    END IF
END SUBROUTINE Factorial

Explanation

  • RECURSIVE: Indicates that the subroutine can call itself.
  • Factorial: Computes the factorial of n using recursion.

Summary

Subroutines in Fortran are powerful tools for organizing and reusing code. They can take arguments, return results, and even call themselves recursively. By mastering subroutines, you can write more modular, readable, and maintainable Fortran programs.

Key Takeaways

  • Subroutines help in breaking down complex programs into manageable pieces.
  • They can accept input arguments and return results.
  • Optional arguments and recursion add flexibility to subroutine usage.

Practice writing and using subroutines to become proficient in Fortran programming.
