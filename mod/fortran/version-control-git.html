Introduction to Version Control

  • Definition: Version control is a system that records changes to a file or set of files over time so that you can recall specific versions later.
  • Importance: It allows multiple people to work on a project simultaneously, tracks changes, and helps in managing project history.

Getting Started with Git

  • What is Git?: Git is a distributed version control system designed to handle everything from small to very large projects with speed and efficiency.
  • Installation:
    • Windows: Download from git-scm.com and follow the installation instructions.
    • macOS: Install via Homebrew with brew install git.
    • Linux: Use the package manager, e.g., sudo apt-get install git for Debian-based systems.

Basic Git Commands

  • Configuring Git: 
    git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
  • Initializing a Repository: 
    git init
  • Cloning a Repository: 
    git clone https://github.com/username/repository.git
  • Checking Status: 
    git status
  • Adding Files: 
    git add filename
  • Committing Changes: 
    git commit -m "Commit message"
  • Pushing Changes: 
    git push origin main

Working with Branches

  • Creating a Branch: 
    git branch new-branch
  • Switching Branches: 
    git checkout new-branch
  • Merging Branches: 
    git checkout main
git merge new-branch

Collaborating with Git

  • Forking a Repository: Fork a repository on GitHub to create your own copy.
  • Pull Requests: Submit changes to the original repository via pull requests.
  • Handling Merge Conflicts:
    • Identify Conflict: Git will mark conflicts in the files.
    • Resolve Conflict: Edit the files to resolve conflicts.
    • Commit Resolution: 
      git add resolved-file
git commit -m "Resolved merge conflict"

Advanced Git Features

  • Stashing Changes: 
    git stash
git stash pop
  • Rebasing: 
    git rebase main
  • Tagging: 
    git tag -a v1.0 -m "Version 1.0"
git push origin v1.0

Example: Managing a Fortran Project with Git

Step-by-Step Example

  1. Initialize Repository: 
    git init fortran-project
cd fortran-project
  2. Create a Fortran File: 
    ! hello.f90
PROGRAM HelloWorld
  PRINT *, "Hello, World!"
END PROGRAM HelloWorld
  3. Add and Commit: 
    git add hello.f90
git commit -m "Add HelloWorld program"
  4. Create a New Branch for Feature: 
    git checkout -b feature-addition
  5. Modify the Fortran File: 
    ! hello.f90
PROGRAM HelloWorld
  PRINT *, "Hello, World!"
  PRINT *, "This is a new feature."
END PROGRAM HelloWorld
  6. Add and Commit Changes: 
    git add hello.f90
git commit -m "Add new feature to HelloWorld program"
  7. Merge Feature Branch: 
    git checkout main
git merge feature-addition

Summary

In this topic, we covered the basics of version control with Git, including installation, basic commands, branching, collaboration, and advanced features. We also provided a practical example of managing a Fortran project using Git. By mastering these skills, you can efficiently manage your codebase, collaborate with others, and maintain a robust project history.
