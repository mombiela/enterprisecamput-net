Scope and Lifetime of Variables in Fortran
Recursion ►

Understanding the scope and lifetime of variables is crucial for writing efficient and error-free Fortran programs. This topic will guide you through the concepts of variable scope and lifetime, from basic to advanced levels.

Scope of Variables

The scope of a variable refers to the region of the program where the variable is accessible.

Local Scope

  • Definition: Variables declared within a subroutine, function, or block are local to that scope.
  • Example: 
    program local_scope_example
    implicit none
    call my_subroutine()
contains
    subroutine my_subroutine()
        integer :: local_var
        local_var = 10
        print *, "Local variable value: ", local_var
    end subroutine my_subroutine
end program local_scope_example
    In this example, local_var is only accessible within my_subroutine.

Global Scope

  • Definition: Variables declared in the main program or in a module are accessible throughout the program or module.
  • Example: 
    module global_scope_example
    implicit none
    integer :: global_var
end module global_scope_example

program use_global_scope
    use global_scope_example
    implicit none
    global_var = 20
    call print_global_var()
contains
    subroutine print_global_var()
        print *, "Global variable value: ", global_var
    end subroutine print_global_var
end program use_global_scope
    Here, global_var is accessible in both the main program and the subroutine print_global_var.

Lifetime of Variables

The lifetime of a variable refers to the duration for which the variable retains its value during program execution.

Automatic Variables

  • Definition: Variables that are automatically created and destroyed when entering and exiting a scope.
  • Example: 
    program automatic_variable_example
    implicit none
    call my_subroutine()
    call my_subroutine()
contains
    subroutine my_subroutine()
        integer :: auto_var
        auto_var = auto_var + 1
        print *, "Automatic variable value: ", auto_var
    end subroutine my_subroutine
end program automatic_variable_example
    Each time my_subroutine is called, auto_var is reinitialized, so the output will always be 1.

Static Variables

  • Definition: Variables that retain their value between calls to the subroutine or function.
  • Example: 
    program static_variable_example
    implicit none
    call my_subroutine()
    call my_subroutine()
contains
    subroutine my_subroutine()
        integer, save :: static_var = 0
        static_var = static_var + 1
        print *, "Static variable value: ", static_var
    end subroutine my_subroutine
end program static_variable_example
    Here, static_var retains its value between calls, so the output will be 1 and then 2.

Summary

  • Scope: Determines where a variable can be accessed.
    • Local scope: Variables are accessible only within the block where they are declared.
    • Global scope: Variables are accessible throughout the program or module.
  • Lifetime: Determines how long a variable retains its value.
    • Automatic variables: Created and destroyed with the scope.
    • Static variables: Retain their value between calls.

Understanding these concepts helps in managing memory efficiently and avoiding common programming errors related to variable accessibility and persistence.
