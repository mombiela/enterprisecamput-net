Dynamic Memory Allocation in Fortran
Linked Lists ►

Dynamic memory allocation is a crucial concept in Fortran, especially for handling data structures whose size may not be known at compile time. This topic will guide you through the basics to advanced concepts of dynamic memory allocation in Fortran.

Introduction to Dynamic Memory Allocation

Dynamic memory allocation allows you to allocate memory during the runtime of the program. This is particularly useful when dealing with arrays or other data structures where the size may vary.

Key Concepts

  • Static vs. Dynamic Memory Allocation: Static memory allocation occurs at compile time, while dynamic memory allocation happens at runtime.
  • ALLOCATE and DEALLOCATE Statements: Used to allocate and deallocate memory dynamically.
  • Pointers: Variables that store the address of dynamically allocated memory.

Basic Dynamic Memory Allocation

ALLOCATE and DEALLOCATE Statements

The ALLOCATE statement is used to allocate memory for arrays or other data structures, and the DEALLOCATE statement is used to free that memory.

Example: Allocating and Deallocating an Array

PROGRAM DynamicArray
  IMPLICIT NONE
  INTEGER, ALLOCATABLE :: array(:)
  INTEGER :: n, i

  ! Ask the user for the size of the array
  PRINT *, 'Enter the size of the array:'
  READ *, n

  ! Allocate memory for the array
  ALLOCATE(array(n))

  ! Initialize the array
  DO i = 1, n
    array(i) = i * 2
  END DO

  ! Print the array
  PRINT *, 'Array elements:'
  PRINT *, array

  ! Deallocate the memory
  DEALLOCATE(array)
END PROGRAM DynamicArray

Error Handling in Allocation

It's important to handle errors that may occur during memory allocation. This can be done using the STAT specifier.

Example: Handling Allocation Errors

PROGRAM AllocationErrorHandling
  IMPLICIT NONE
  INTEGER, ALLOCATABLE :: array(:)
  INTEGER :: n, stat

  ! Ask the user for the size of the array
  PRINT *, 'Enter the size of the array:'
  READ *, n

  ! Allocate memory for the array with error handling
  ALLOCATE(array(n), STAT=stat)
  IF (stat /= 0) THEN
    PRINT *, 'Error: Unable to allocate memory.'
    STOP
  END IF

  ! Deallocate the memory
  DEALLOCATE(array)
END PROGRAM AllocationErrorHandling

Advanced Dynamic Memory Allocation

Using Pointers

Pointers in Fortran can be used to dynamically allocate memory and reference it. This is useful for creating complex data structures like linked lists.

Example: Using Pointers for Dynamic Allocation

PROGRAM PointerExample
  IMPLICIT NONE
  INTEGER, POINTER :: pArray(:)
  INTEGER :: n, i

  ! Ask the user for the size of the array
  PRINT *, 'Enter the size of the array:'
  READ *, n

  ! Allocate memory for the array using pointers
  ALLOCATE(pArray(n))

  ! Initialize the array
  DO i = 1, n
    pArray(i) = i * 3
  END DO

  ! Print the array
  PRINT *, 'Pointer array elements:'
  PRINT *, pArray

  ! Deallocate the memory
  DEALLOCATE(pArray)
END PROGRAM PointerExample

Dynamic Arrays in Derived Types

Fortran allows dynamic arrays within derived types, which can be useful for creating more complex data structures.

Example: Dynamic Arrays in Derived Types

MODULE DynamicModule
  TYPE :: DynamicType
    INTEGER, ALLOCATABLE :: array(:)
  END TYPE DynamicType
END MODULE DynamicModule

PROGRAM DerivedTypeExample
  USE DynamicModule
  IMPLICIT NONE
  TYPE(DynamicType) :: myType
  INTEGER :: n, i

  ! Ask the user for the size of the array
  PRINT *, 'Enter the size of the array:'
  READ *, n

  ! Allocate memory for the array within the derived type
  ALLOCATE(myType%array(n))

  ! Initialize the array
  DO i = 1, n
    myType%array(i) = i * 4
  END DO

  ! Print the array
  PRINT *, 'Derived type array elements:'
  PRINT *, myType%array

  ! Deallocate the memory
  DEALLOCATE(myType%array)
END PROGRAM DerivedTypeExample

Summary

Dynamic memory allocation in Fortran is a powerful feature that allows for flexible and efficient memory management. By using ALLOCATE and DEALLOCATE statements, handling errors, and utilizing pointers and derived types, you can create complex and dynamic data structures. Understanding these concepts is essential for advanced Fortran programming and can significantly enhance the performance and flexibility of your programs.
