Unit Testing in Android Studio
UI Testing with Espresso ►

Unit testing is a crucial part of the software development process, ensuring that individual units of code work as expected. In Android development, unit testing helps maintain code quality and reliability. This topic will guide you through the basics to advanced concepts of unit testing in Android Studio.

Introduction to Unit Testing

  • Definition: Unit testing involves testing individual components or functions of an application in isolation.
  • Purpose: To verify that each unit of the software performs as designed.
  • Benefits:
    • Detects bugs early.
    • Simplifies debugging.
    • Facilitates code refactoring.
    • Enhances code quality and reliability.

Setting Up Unit Testing in Android Studio

  1. Create a New Project:

    • Open Android Studio and create a new project.
    • Ensure you select the appropriate SDK and project settings.

  2. Add Testing Dependencies:

    • Open the build.gradle file (Module: app) and add the following dependencies: 
      dependencies {
    testImplementation 'junit:junit:4.13.2'
    testImplementation 'org.mockito:mockito-core:3.9.0'
}

  3. Sync the Project:

    • Click on "Sync Now" to download and integrate the dependencies.

Writing Your First Unit Test

  1. Create a Test Class:

    • Navigate to the src/test/java directory.
    • Right-click and select New > Java Class.
    • Name the class ExampleUnitTest.

  2. Write a Simple Test:

    • Add a simple test method to the class: 
      import org.junit.Test;
import static org.junit.Assert.*;

public class ExampleUnitTest {
    @Test
    public void addition_isCorrect() {
        assertEquals(4, 2 + 2);
    }
}

  3. Run the Test:

    • Right-click on the test class and select Run 'ExampleUnitTest'.
    • Check the results in the Run window.

Advanced Unit Testing Concepts

Mocking with Mockito

  • Definition: Mocking is a technique to create a simulated version of an object.
  • Purpose: To isolate the unit of code being tested and control its dependencies.

Example: Mocking a Dependency

  1. Add Mockito Dependency (already added in the setup step):

    testImplementation 'org.mockito:mockito-core:3.9.0'

  2. Create a Class with Dependencies:

    public class UserService {
    private UserRepository userRepository;

    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    public String getUserFullName(int userId) {
        User user = userRepository.getUserById(userId);
        return user.getFirstName() + " " + user.getLastName();
    }
}

public interface UserRepository {
    User getUserById(int userId);
}

public class User {
    private String firstName;
    private String lastName;

    // Constructor, getters, and setters
}

  3. Write a Test with Mockito:

    import org.junit.Before;
import org.junit.Test;
import org.mockito.Mock;
import org.mockito.MockitoAnnotations;

import static org.mockito.Mockito.*;
import static org.junit.Assert.*;

public class UserServiceTest {
    @Mock
    private UserRepository userRepository;

    private UserService userService;

    @Before
    public void setUp() {
        MockitoAnnotations.initMocks(this);
        userService = new UserService(userRepository);
    }

    @Test
    public void getUserFullName_returnsCorrectFullName() {
        User user = new User("John", "Doe");
        when(userRepository.getUserById(1)).thenReturn(user);

        String fullName = userService.getUserFullName(1);
        assertEquals("John Doe", fullName);
    }
}

Best Practices for Unit Testing

  • Write Independent Tests: Ensure tests do not depend on each other.
  • Use Descriptive Names: Name tests clearly to describe their purpose.
  • Test Edge Cases: Include tests for boundary conditions and edge cases.
  • Keep Tests Fast: Unit tests should be quick to execute.
  • Maintain Tests: Regularly update tests as the code evolves.

Conclusion

Unit testing is an essential practice in Android development, helping to ensure that your code is reliable and maintainable. By setting up unit tests in Android Studio, writing basic and advanced tests, and following best practices, you can significantly improve the quality of your applications. Remember, the key to effective unit testing is consistency and thoroughness. Happy testing!
