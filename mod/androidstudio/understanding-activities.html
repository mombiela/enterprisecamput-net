Introduction to Activities

Activities are a fundamental component of Android applications. They represent a single screen with a user interface. An application can have multiple activities, and each activity is independent of the others.

Key Concepts

  • Activity Lifecycle: The series of states an activity goes through from creation to destruction.
  • Intents: Messaging objects used to request an action from another app component.
  • UI Components: Widgets and layouts that make up the user interface of an activity.

Activity Lifecycle

Understanding the activity lifecycle is crucial for managing the state of your application and ensuring a smooth user experience.

Lifecycle Methods

  • onCreate(): Called when the activity is first created. This is where you initialize your activity.
  • onStart(): Called when the activity becomes visible to the user.
  • onResume(): Called when the activity starts interacting with the user.
  • onPause(): Called when the system is about to start resuming another activity.
  • onStop(): Called when the activity is no longer visible to the user.
  • onDestroy(): Called before the activity is destroyed.

Lifecycle Diagram

Here is a simplified diagram of the activity lifecycle:

onCreate() -> onStart() -> onResume()
   ^            |            |
   |            v            v
onDestroy() <- onStop() <- onPause()

Code Example

Here is a basic implementation of an activity with lifecycle methods:

public class MainActivity extends AppCompatActivity {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);
        Log.d("Lifecycle", "onCreate called");
    }

    @Override
    protected void onStart() {
        super.onStart();
        Log.d("Lifecycle", "onStart called");
    }

    @Override
    protected void onResume() {
        super.onResume();
        Log.d("Lifecycle", "onResume called");
    }

    @Override
    protected void onPause() {
        super.onPause();
        Log.d("Lifecycle", "onPause called");
    }

    @Override
    protected void onStop() {
        super.onStop();
        Log.d("Lifecycle", "onStop called");
    }

    @Override
    protected void onDestroy() {
        super.onDestroy();
        Log.d("Lifecycle", "onDestroy called");
    }
}

Intents

Intents are used to start activities and communicate between different components of an application.

Types of Intents

  • Explicit Intents: Specify the component to start by name.
  • Implicit Intents: Do not specify the component directly; instead, declare a general action to perform.

Code Example

Here is an example of using an explicit intent to start a new activity:

Intent intent = new Intent(this, SecondActivity.class);
startActivity(intent);

And here is an example of using an implicit intent to open a web page:

Intent intent = new Intent(Intent.ACTION_VIEW);
intent.setData(Uri.parse("http://www.example.com"));
startActivity(intent);

UI Components

Activities are composed of various UI components, such as buttons, text views, and layouts.

Common UI Components

  • TextView: Displays text to the user.
  • Button: A clickable button.
  • EditText: A text field that allows user input.
  • LinearLayout: A layout that arranges its children in a single column or row.

Code Example

Here is an example of a simple activity layout using XML:

<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical">

    <TextView
        android:id="@+id/textView"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Hello, World!" />

    <Button
        android:id="@+id/button"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Click Me" />
</LinearLayout>

Conclusion

Understanding activities is essential for developing Android applications. By mastering the activity lifecycle, intents, and UI components, you can create robust and user-friendly applications. Remember to manage the lifecycle states properly to ensure a smooth user experience and use intents to navigate between different parts of your application effectively.
