Introduction to HTTP and REST APIs

In this section, we will cover the basics of HTTP and REST APIs, which are essential for communicating with web services in Android applications.

  • HTTP (HyperText Transfer Protocol): The foundation of any data exchange on the Web and a protocol used for transmitting hypertext requests and information between servers and clients.
  • REST (Representational State Transfer): An architectural style for designing networked applications. RESTful APIs use HTTP requests to perform CRUD (Create, Read, Update, Delete) operations.

Understanding HTTP

HTTP is a protocol used for sending and receiving data over the web. Here are some key concepts:

  • HTTP Methods:

    • GET: Retrieve data from a server.
    • POST: Send data to a server to create a resource.
    • PUT: Update an existing resource on the server.
    • DELETE: Remove a resource from the server.

  • HTTP Status Codes:

    • 200 OK: The request was successful.
    • 201 Created: The request was successful, and a resource was created.
    • 400 Bad Request: The request could not be understood or was missing required parameters.
    • 401 Unauthorized: Authentication failed or user does not have permissions for the desired action.
    • 404 Not Found: The requested resource could not be found.
    • 500 Internal Server Error: An error occurred on the server.

Introduction to REST APIs

REST APIs allow you to interact with web services using HTTP methods. They are stateless and can be used to perform CRUD operations.

  • Resources and URIs: In REST, resources are identified by URIs (Uniform Resource Identifiers).

    • Example: https://api.example.com/users might represent a collection of users.

  • JSON (JavaScript Object Notation): A lightweight data-interchange format that is easy for humans to read and write and easy for machines to parse and generate.

Making HTTP Requests in Android

To make HTTP requests in Android, you can use libraries like HttpURLConnection, OkHttp, or Retrofit. Here, we will focus on Retrofit due to its simplicity and ease of use.

Setting Up Retrofit

First, add the Retrofit dependency to your build.gradle file:

dependencies {
    implementation 'com.squareup.retrofit2:retrofit:2.9.0'
    implementation 'com.squareup.retrofit2:converter-gson:2.9.0'
}

Creating a Retrofit Instance

Create a Retrofit instance to define the base URL and the converter factory:

Retrofit retrofit = new Retrofit.Builder()
    .baseUrl("https://api.example.com/")
    .addConverterFactory(GsonConverterFactory.create())
    .build();

Defining API Endpoints

Define the endpoints using an interface:

public interface ApiService {
    @GET("users")
    Call<List<User>> getUsers();

    @POST("users")
    Call<User> createUser(@Body User user);
}

Making a Request

Create an instance of the API service and make a request:

ApiService apiService = retrofit.create(ApiService.class);

// Making a GET request
Call<List<User>> call = apiService.getUsers();
call.enqueue(new Callback<List<User>>() {
    @Override
    public void onResponse(Call<List<User>> call, Response<List<User>> response) {
        if (response.isSuccessful()) {
            List<User> users = response.body();
            // Handle the list of users
        }
    }

    @Override
    public void onFailure(Call<List<User>> call, Throwable t) {
        // Handle failure
    }
});

Handling JSON Responses

Use the Gson library to parse JSON responses into Java objects. Ensure your model classes match the JSON structure:

public class User {
    private int id;
    private String name;
    private String email;

    // Getters and setters
}

Conclusion

In this topic, we covered the basics of HTTP and REST APIs, including HTTP methods, status codes, and how to make HTTP requests in Android using Retrofit. Understanding these concepts is crucial for integrating web services into your Android applications. By practicing the examples and exercises provided, you will be able to effectively communicate with RESTful APIs and handle JSON data in your apps.
