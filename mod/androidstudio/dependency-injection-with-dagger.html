Introduction to Dependency Injection

  • Definition: Dependency Injection (DI) is a design pattern used to implement IoC (Inversion of Control), allowing a class to receive its dependencies from an external source rather than creating them itself.
  • Benefits:
    • Improved code modularity.
    • Easier testing and maintenance.
    • Decoupling of components.

Introduction to Dagger

  • Definition: Dagger is a fully static, compile-time dependency injection framework for Java, Kotlin, and Android.
  • Why Dagger?:
    • Compile-time verification of dependencies.
    • Performance efficiency.
    • Simplifies dependency management in large projects.

Setting Up Dagger in Android Studio

  1. Add Dagger dependencies: 
    dependencies {
    implementation 'com.google.dagger:dagger:2.x'
    kapt 'com.google.dagger:dagger-compiler:2.x'
}
  2. Enable annotation processing: 
    android {
    ...
    kapt {
        generateStubs = true
    }
}

Basic Concepts of Dagger

  • Modules: Classes annotated with @Module that provide dependencies.
  • Components: Interfaces annotated with @Component that act as bridges between modules and injection targets.
  • Inject: Annotation used to request dependencies.

Example: Basic Dagger Setup

  1. Create a Module: 
    @Module
public class NetworkModule {
    @Provides
    public Retrofit provideRetrofit() {
        return new Retrofit.Builder()
                .baseUrl("https://api.example.com")
                .build();
    }
}
  2. Create a Component: 
    @Component(modules = NetworkModule.class)
public interface AppComponent {
    void inject(MainActivity mainActivity);
}
  3. Inject Dependencies: 
    public class MainActivity extends AppCompatActivity {
    @Inject
    Retrofit retrofit;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        AppComponent appComponent = DaggerAppComponent.create();
        appComponent.inject(this);

        // Use the injected Retrofit instance
    }
}

Advanced Dagger Concepts

  • Scopes: Used to manage the lifecycle of dependencies.

    • Singleton Scope: Ensures a single instance of a dependency. 
      @Module
public class AppModule {
    @Provides
    @Singleton
    public Context provideContext(Application application) {
        return application.getApplicationContext();
    }
}
    • Custom Scopes: Define custom scopes for specific use cases. 
      @Scope
@Retention(RetentionPolicy.RUNTIME)
public @interface ActivityScope {}

  • Subcomponents: Allow for more granular dependency graphs.

    @Subcomponent
public interface ActivityComponent {
    void inject(MyActivity myActivity);
}

  • Multibindings: Inject collections of dependencies.

    @Module
public abstract class MultibindingsModule {
    @Binds
    @IntoMap
    @StringKey("example")
    abstract MyInterface bindExample(ExampleImpl impl);
}

Testing with Dagger

  • Unit Testing: Use Dagger's @Inject to inject mocks.

    @RunWith(MockitoJUnitRunner.class)
public class MyTest {
    @Mock
    Retrofit retrofit;

    @Inject
    MyService myService;

    @Before
    public void setUp() {
        DaggerTestComponent.create().inject(this);
    }

    @Test
    public void testService() {
        // Test using the injected mock Retrofit
    }
}

  • Espresso Testing: Use Dagger to inject dependencies in UI tests.

    @RunWith(AndroidJUnit4.class)
public class MyEspressoTest {
    @Rule
    public ActivityTestRule<MainActivity> activityRule = new ActivityTestRule<>(MainActivity.class);

    @Before
    public void setUp() {
        // Inject dependencies
    }

    @Test
    public void testUI() {
        // Perform UI tests
    }
}

Conclusion

  • Summary:
    • Dagger simplifies dependency management in Android applications.
    • It provides compile-time safety and performance benefits.
    • Understanding modules, components, and scopes is crucial for effective use.
    • Advanced features like subcomponents and multibindings offer powerful tools for complex dependency graphs.
    • Dagger integrates well with testing frameworks, enhancing testability of Android applications.

By mastering Dagger, you can write cleaner, more maintainable, and testable code, making your Android applications more robust and scalable.
