Introduction

Control Language (CL) is a scripting language used on IBM's AS/400 systems (now known as IBM i). It is primarily used for system-level scripting and automation. In this section, we will write our first CL program, understand its structure, and execute it.

Key Concepts

  • CL Program: A sequence of CL commands that perform a specific task.
  • Source Physical File: A file where CL source code is stored.
  • Compiling: The process of converting CL source code into an executable object.
  • Executing: Running the compiled CL program.

Setting Up the Environment

Before writing your first CL program, ensure you have access to an IBM i system and the necessary permissions to create and execute CL programs.

Writing Your First CL Program

  1. Create a Source Physical File:

    • A source physical file is where you will store your CL source code.
    • Use the following command to create a source physical file:
    CRTSRCPF FILE(QGPL/QCLSRC) RCDLEN(112)
    • This command creates a source physical file named QCLSRC in the QGPL library with a record length of 112.

  2. Add a Member to the Source Physical File:

    • A member is a place within the source physical file where you write your CL program.
    • Use the following command to add a member:
    ADDPFM FILE(QGPL/QCLSRC) MBR(MYCLPGM) SRCTYPE(CL)
    • This command adds a member named MYCLPGM to the QCLSRC file with a source type of CL.

  3. Edit the Source Member:

    • Use the Source Entry Utility (SEU) or any other editor available on your system to edit the source member.
    • Use the following command to start editing:
    STRSEU SRCFILE(QGPL/QCLSRC) SRCMBR(MYCLPGM)
    • This command opens the MYCLPGM member in the QCLSRC file for editing.

  4. Write the CL Program:

    • Enter the following code in the editor:
    PGM
DCL VAR(&MSG) TYPE(*CHAR) LEN(50)
CHGVAR VAR(&MSG) VALUE('Hello, World!')
SNDPGMMSG MSG(&MSG)
ENDPGM
    • Explanation of the code:
      • PGM: Indicates the start of the CL program.
      • DCL VAR(&MSG) TYPE(*CHAR) LEN(50): Declares a variable &MSG of type character with a length of 50.
      • CHGVAR VAR(&MSG) VALUE('Hello, World!'): Changes the value of &MSG to 'Hello, World!'.
      • SNDPGMMSG MSG(&MSG): Sends a program message containing the value of &MSG.
      • ENDPGM: Indicates the end of the CL program.

Compiling the CL Program

  • After writing the CL program, you need to compile it to create an executable object.
  • Use the following command to compile the program:
CRTCLPGM PGM(QGPL/MYCLPGM) SRCFILE(QGPL/QCLSRC) SRCMBR(MYCLPGM)
  • This command creates a program object named MYCLPGM in the QGPL library using the source member MYCLPGM from the QCLSRC file.

Executing the CL Program

  • To execute the compiled CL program, use the following command:
CALL PGM(QGPL/MYCLPGM)
  • This command runs the MYCLPGM program from the QGPL library, and you should see the message 'Hello, World!' displayed.

Conclusion

In this section, you have learned how to create, write, compile, and execute a simple CL program on an IBM i system. This foundational knowledge is crucial as you progress to more advanced topics in CL programming. Practice writing and running basic CL programs to become comfortable with the syntax and commands.
