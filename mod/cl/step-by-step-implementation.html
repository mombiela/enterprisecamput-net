Introduction to CL (Control Language)

Control Language (CL) is a scripting language used on IBM's iSeries (AS/400) systems. It is primarily used for system operations, job control, and automation tasks. This section will guide you through the step-by-step implementation of CL programs, from beginner to advanced levels.

Beginner Level

Understanding the Basics

  • What is CL?

    • CL is used to control the execution of programs and system commands on IBM iSeries systems.
    • It is similar to shell scripting in Unix/Linux but specific to IBM's environment.

  • Basic Structure of a CL Program

    • A CL program consists of a series of commands that are executed sequentially.
    • Each command performs a specific task, such as calling another program, manipulating files, or controlling job flow.

Writing Your First CL Program

  • Example: Hello World Program 
    PGM
DCL VAR(&MSG) TYPE(*CHAR) LEN(50)
CHGVAR VAR(&MSG) VALUE('Hello, World!')
SNDPGMMSG MSG(&MSG)
ENDPGM
    • Explanation:
      • PGM and ENDPGM define the start and end of the program.
      • DCL declares a variable &MSG of type character with a length of 50.
      • CHGVAR changes the value of &MSG to 'Hello, World!'.
      • SNDPGMMSG sends the message stored in &MSG to the program message queue.

Running a CL Program

  • Steps to Run:
    1. Save the program source code in a source physical file.
    2. Compile the source code using the CRTCLPGM command.
    3. Execute the compiled program using the CALL command.

Intermediate Level

Variables and Data Types

  • Common Data Types:

    • *CHAR (Character)
    • *DEC (Decimal)
    • *INT (Integer)
    • *LGL (Logical)

  • Example: Using Different Data Types

    PGM
DCL VAR(&NAME) TYPE(*CHAR) LEN(20)
DCL VAR(&AGE) TYPE(*DEC) LEN(3 0)
DCL VAR(&ISADULT) TYPE(*LGL)

CHGVAR VAR(&NAME) VALUE('John Doe')
CHGVAR VAR(&AGE) VALUE(25)
CHGVAR VAR(&ISADULT) VALUE('1')

IF COND(&AGE *GE 18) THEN(DO)
  SNDPGMMSG MSG('Adult: ' *CAT &NAME)
ENDDO
ENDPGM
    • Explanation:
      • Declares variables for name, age, and a logical flag for adulthood.
      • Checks if the age is greater than or equal to 18 and sends a message accordingly.

Control Structures

  • Conditional Statements:

    • IF, ELSE, DO, ENDDO

  • Loops:

    • DOFOR, DOWHILE, DOUNTIL

  • Example: Looping with DOFOR

    PGM
DCL VAR(&I) TYPE(*INT) LEN(3)

DOFOR VAR(&I) FROM(1) TO(5)
  SNDPGMMSG MSG('Iteration: ' *CAT &I)
ENDDO
ENDPGM
    • Explanation:
      • Loops from 1 to 5, sending a message for each iteration.

Advanced Level

Advanced Error Handling

  • Using MONMSG (Monitor Message)

    • MONMSG is used to handle errors and exceptions in CL programs.

  • Example: Error Handling

    PGM
DCL VAR(&FILE) TYPE(*CHAR) LEN(10)

CHGVAR VAR(&FILE) VALUE('MYFILE')
OVRDBF FILE(&FILE) TOFILE(LIBRARY/&FILE)
MONMSG MSGID(CPF0000) EXEC(DO)
  SNDPGMMSG MSG('Error overriding file: ' *CAT &FILE)
ENDDO

ENDPGM
    • Explanation:
      • Attempts to override a database file and monitors for any errors.
      • If an error occurs, it sends an error message.

Advanced File Operations

  • Reading and Writing Files

    • Use commands like RCVF (Receive File) and SNDF (Send File) for file operations.

  • Example: Reading a File

    PGM
DCLF FILE(MYLIB/MYFILE)

RCVF
MONMSG MSGID(CPF0864) EXEC(GOTO CMDLBL(END))

SNDPGMMSG MSG('Record: ' *CAT &FIELD1 *CAT ' ' *CAT &FIELD2)

ENDPGM
    • Explanation:
      • Declares a file and reads a record.
      • If end-of-file is reached, it jumps to the end of the program.
      • Sends a message with the record's fields.

Conclusion

In this topic, we covered the step-by-step implementation of CL programs, starting from basic concepts and progressing to more advanced features. By understanding the structure, variables, control structures, error handling, and file operations, you can create robust and efficient CL programs to automate tasks on IBM iSeries systems. Practice with the provided examples and gradually build more complex scripts to enhance your skills.
