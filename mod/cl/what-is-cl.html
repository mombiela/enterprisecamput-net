What is Control Language (CL)?
Setting Up Your Environment ►

Control Language (CL) is a scripting language used primarily on IBM's midrange systems, such as the AS/400 (now known as IBM i). It is designed to control system operations, manage jobs, and handle system resources. CL is essential for system administrators and developers working in IBM environments.

Key Concepts of Control Language (CL)

  • Purpose: CL is used to automate and control system tasks, such as job scheduling, file management, and system configuration.
  • Environment: CL scripts run in the IBM i operating system environment.
  • Syntax: CL has a unique syntax that includes commands, parameters, and variables.

Basic Structure of CL Programs

A typical CL program consists of a series of commands that perform specific tasks. Each command is written on a new line and can include parameters to customize its behavior.

Example of a Simple CL Program

PGM
    DCL VAR(&MSG) TYPE(*CHAR) LEN(50)
    CHGVAR VAR(&MSG) VALUE('Hello, World!')
    SNDPGMMSG MSG(&MSG)
ENDPGM
  • PGM: Indicates the start of a CL program.
  • DCL: Declares a variable. In this case, &MSG is a character variable with a length of 50.
  • CHGVAR: Changes the value of a variable. Here, &MSG is set to 'Hello, World!'.
  • SNDPGMMSG: Sends a program message. The message sent is the value of &MSG.
  • ENDPGM: Marks the end of the CL program.

Common CL Commands

  • DCL: Declare a variable.
  • CHGVAR: Change the value of a variable.
  • SNDPGMMSG: Send a program message.
  • CALL: Call another program.
  • IF/ELSE: Conditional execution of commands.
  • DO/ENDDO: Looping constructs.

Example of Conditional Execution

PGM
    DCL VAR(&STATUS) TYPE(*CHAR) LEN(10)
    CHGVAR VAR(&STATUS) VALUE('ACTIVE')

    IF COND(&STATUS *EQ 'ACTIVE') THEN(DO)
        SNDPGMMSG MSG('System is active')
    ENDDO
    ELSE DO
        SNDPGMMSG MSG('System is not active')
    ENDDO
ENDPGM
  • IF COND: Checks if the condition is true.
  • THEN(DO): Executes the block of commands if the condition is true.
  • ELSE DO: Executes the block of commands if the condition is false.

Advanced Features

As you progress, you will encounter more advanced features of CL, such as:

  • Subroutines: Encapsulate reusable code blocks.
  • Error Handling: Manage errors and exceptions in CL programs.
  • File Operations: Read from and write to files.
  • Job Control: Manage job queues and job attributes.

Example of a Subroutine

PGM
    DCL VAR(&MSG) TYPE(*CHAR) LEN(50)
    CALLSUBR SUBR(MYSUBR)
    SNDPGMMSG MSG(&MSG)
    ENDPGM

SUBR MYSUBR
    CHGVAR VAR(&MSG) VALUE('Hello from subroutine!')
    RETURN
ENDSUBR
  • CALLSUBR: Calls a subroutine.
  • SUBR: Defines a subroutine.
  • RETURN: Returns control to the calling program.

Summary

Control Language (CL) is a powerful tool for managing IBM i systems. It allows for automation of system tasks, job control, and resource management. Understanding the basic structure, common commands, and advanced features of CL is essential for effective system administration and development in IBM environments.

By mastering CL, you can create efficient, automated processes that enhance the functionality and reliability of your IBM i systems.
