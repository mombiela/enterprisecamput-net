Introduction

Integrating with databases is a crucial aspect of working with Control Language (CL) on IBM i systems. This topic will guide you through the basics of database integration, progressing to more advanced techniques. By the end of this section, you will be able to perform database operations using CL commands effectively.

Key Concepts

  • Database Files: Understand the types of database files (physical and logical) and their roles.
  • CL Commands: Learn the CL commands used for database operations.
  • Error Handling: Implement error handling in database operations.
  • Advanced Techniques: Explore advanced database integration techniques.

Database Files

Physical Files

Physical files are the actual database files that store data.

CRTPF FILE(MYLIB/MYFILE) RCDLEN(100)
  • CRTPF: Command to create a physical file.
  • FILE: Specifies the library and file name.
  • RCDLEN: Defines the record length.

Logical Files

Logical files provide a different view or access path to the data in physical files.

CRTLF FILE(MYLIB/MYLOGICAL) SRCFILE(MYLIB/QDDSSRC)
  • CRTLF: Command to create a logical file.
  • FILE: Specifies the library and logical file name.
  • SRCFILE: Specifies the source file containing the DDS (Data Description Specifications).

CL Commands for Database Operations

Adding Records

Use the ADDPFTRG command to add records to a physical file.

ADDPFTRG FILE(MYLIB/MYFILE) TRGTIME(*AFTER) TRGEVENT(*INSERT) PGM(MYLIB/MYPGM)
  • ADDPFTRG: Command to add a trigger to a physical file.
  • FILE: Specifies the library and file name.
  • TRGTIME: Specifies when the trigger should be activated.
  • TRGEVENT: Specifies the event that activates the trigger.
  • PGM: Specifies the program to run when the trigger is activated.

Updating Records

Use the UPDPF command to update records in a physical file.

UPDPF FILE(MYLIB/MYFILE) MBR(MYMBR) FROMFILE(MYLIB/NEWFILE)
  • UPDPF: Command to update a physical file.
  • FILE: Specifies the library and file name.
  • MBR: Specifies the member to update.
  • FROMFILE: Specifies the source file for the update.

Deleting Records

Use the DLTPF command to delete records from a physical file.

DLTPF FILE(MYLIB/MYFILE)
  • DLTPF: Command to delete a physical file.
  • FILE: Specifies the library and file name.

Error Handling

Implementing error handling ensures your CL programs can manage exceptions gracefully.

MONMSG MSGID(CPF0000) EXEC(DO)
  /* Error handling code here */
ENDDO
  • MONMSG: Command to monitor for messages.
  • MSGID: Specifies the message ID to monitor.
  • EXEC: Specifies the command to execute if the message is received.

Advanced Techniques

Using SQL in CL

You can embed SQL statements within CL programs for more complex database operations.

RUNSQL SQL('INSERT INTO MYLIB.MYTABLE (COL1, COL2) VALUES (''Value1'', ''Value2'')')
  • RUNSQL: Command to run an SQL statement.
  • SQL: Specifies the SQL statement to execute.

Using Data Areas

Data areas can be used to store and retrieve data efficiently.

CRTDTAARA DTAARA(MYLIB/MYDATA) TYPE(*CHAR) LEN(100)
  • CRTDTAARA: Command to create a data area.
  • DTAARA: Specifies the library and data area name.
  • TYPE: Specifies the data type.
  • LEN: Specifies the length of the data area.

Conclusion

Integrating with databases using CL involves understanding the types of database files, mastering the essential CL commands for database operations, implementing error handling, and exploring advanced techniques like using SQL within CL programs. With these skills, you can efficiently manage and manipulate data on IBM i systems.
