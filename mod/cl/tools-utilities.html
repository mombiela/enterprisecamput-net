Tools and Utilities

Introduction

Control Language (CL) is a powerful scripting language used on IBM's iSeries (AS/400) systems. This section will cover various tools and utilities that can enhance your productivity and efficiency when working with CL. We will explore both built-in tools and third-party utilities that can assist in development, debugging, and maintenance of CL programs.

Built-in Tools

SEU (Source Entry Utility)

SEU is a text editor for creating and modifying CL source code.

  • Key Features:

    • Syntax highlighting for CL commands.
    • Line numbering for easy reference.
    • Basic error checking.

  • Example:

    STRSEU SRCFILE(MYLIB/QCLSRC) SRCMBR(MYCLPGM)

PDM (Programming Development Manager)

PDM is a menu-driven interface for managing source code and objects.

  • Key Features:

    • Browse and edit source members.
    • Compile and submit jobs.
    • Manage libraries and objects.

  • Example:

    STRPDM

SDA (Screen Design Aid)

SDA is used for designing display files (DSPF) for interactive CL programs.

  • Key Features:

    • Graphical interface for designing screens.
    • Field-level validation and formatting.
    • Integration with CL programs.

  • Example:

    STRSDA FILE(MYLIB/MYDSPF)

DFU (Data File Utility)

DFU allows for quick and easy data entry and modification in database files.

  • Key Features:

    • Direct access to database records.
    • Simple interface for data manipulation.
    • Supports various data types.

  • Example:

    STRDFU FILE(MYLIB/MYFILE)

Conclusion

Understanding and utilizing the various tools and utilities available for CL can significantly enhance your productivity and efficiency. Built-in tools like SEU, PDM, SDA, and DFU provide essential functionalities for development and maintenance, while third-party utilities like RDi, TurnOver, and Aldon offer advanced features for larger projects and team collaboration. By mastering these tools, you can streamline your workflow and improve the quality of your CL programs.
