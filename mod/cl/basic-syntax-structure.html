Introduction to CL (Control Language)

Control Language (CL) is a scripting language used on IBM's AS/400 systems (now known as IBM i). It is primarily used to control the execution of programs and manage system resources. Understanding the basic syntax and structure of CL is essential for anyone looking to work with IBM i systems.

Key Concepts

  • Commands: The fundamental units of CL, used to perform specific tasks.
  • Statements: Instructions written in CL that can include commands, labels, and control structures.
  • Variables: Used to store data that can be manipulated within a CL program.
  • Control Structures: Constructs like loops and conditionals that control the flow of the program.

Basic Syntax

Commands

Commands in CL are predefined instructions that perform specific tasks. Each command has a specific syntax and set of parameters.

Example:

SNDMSG MSG('Hello, World!') TOUSR(USER1)

Explanation:

  • SNDMSG is the command to send a message.
  • MSG is a parameter specifying the message content.
  • TOUSR is a parameter specifying the recipient.

Statements

A CL program consists of a series of statements. Each statement can be a command, a label, or a control structure.

Example:

PGM
    SNDMSG MSG('Starting the process') TOUSR(USER1)
    /* Additional commands go here */
ENDPGM

Explanation:

  • PGM marks the beginning of a CL program.
  • SNDMSG is a command statement.
  • ENDPGM marks the end of the CL program.

Variables

Variables in CL are used to store data that can be used and manipulated within the program. They are defined using the DCL (Declare) command.

Example:

DCL VAR(&MSG) TYPE(*CHAR) LEN(50)
CHGVAR VAR(&MSG) VALUE('Hello, World!')
SNDMSG MSG(&MSG) TOUSR(USER1)

Explanation:

  • DCL declares a variable named &MSG of type character with a length of 50.
  • CHGVAR changes the value of the variable &MSG to 'Hello, World!'.
  • SNDMSG sends the message stored in &MSG to USER1.

Control Structures

Control structures in CL include loops and conditional statements that control the flow of the program.

IF Statement

The IF statement is used to execute a block of code only if a specified condition is true.

Example:

DCL VAR(&STATUS) TYPE(*CHAR) LEN(10)
CHGVAR VAR(&STATUS) VALUE('SUCCESS')

IF COND(&STATUS *EQ 'SUCCESS') THEN(DO)
    SNDMSG MSG('Process completed successfully') TOUSR(USER1)
ENDDO

Explanation:

  • IF COND checks if the variable &STATUS is equal to 'SUCCESS'.
  • THEN(DO) begins a block of code to execute if the condition is true.
  • ENDDO marks the end of the block.

DO Loop

The DO loop is used to repeat a block of code a specified number of times.

Example:

DCL VAR(&I) TYPE(*DEC) LEN(2 0)
DCL VAR(&MSG) TYPE(*CHAR) LEN(50)

DOFOR VAR(&I) FROM(1) TO(5)
    CHGVAR VAR(&MSG) VALUE('Iteration ' *CAT &I)
    SNDMSG MSG(&MSG) TOUSR(USER1)
ENDDO

Explanation:

  • DOFOR starts a loop with the variable &I ranging from 1 to 5.
  • CHGVAR changes the value of &MSG to 'Iteration ' followed by the current value of &I.
  • SNDMSG sends the message stored in &MSG to USER1.
  • ENDDO marks the end of the loop.

Conclusion

Understanding the basic syntax and structure of CL is crucial for effectively writing and managing scripts on IBM i systems. This includes knowing how to use commands, statements, variables, and control structures. With these fundamentals, you can start creating more complex and functional CL programs.
