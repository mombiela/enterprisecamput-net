Introduction to Generics

Generics provide a way to create reusable components. They allow you to define a component that can work with a variety of data types rather than a single one. This is especially useful in TypeScript for creating flexible and reusable code.

Why Use Generics with Interfaces?

  • Reusability: Create interfaces that can handle multiple data types.
  • Type Safety: Ensure that the types are consistent throughout the code.
  • Flexibility: Allow functions and classes to operate on different data types without losing type information.

Defining a Generic Interface

A generic interface can be defined using the <T> syntax, where T is a placeholder for the type that will be specified later.

Example: Basic Generic Interface

interface Box<T> {
    contents: T;
}

let stringBox: Box<string> = { contents: "Hello, World!" };
let numberBox: Box<number> = { contents: 42 };

console.log(stringBox.contents); // Output: Hello, World!
console.log(numberBox.contents); // Output: 42
  • Explanation:
    • Box<T> is a generic interface where T can be any type.
    • stringBox is an instance of Box with T as string.
    • numberBox is an instance of Box with T as number.

Using Multiple Type Parameters

You can define interfaces with multiple type parameters to handle more complex scenarios.

Example: Generic Interface with Multiple Type Parameters

interface Pair<K, V> {
    key: K;
    value: V;
}

let numberStringPair: Pair<number, string> = { key: 1, value: "One" };
let stringBooleanPair: Pair<string, boolean> = { key: "isActive", value: true };

console.log(numberStringPair.key, numberStringPair.value); // Output: 1 One
console.log(stringBooleanPair.key, stringBooleanPair.value); // Output: isActive true
  • Explanation:
    • Pair<K, V> is a generic interface with two type parameters K and V.
    • numberStringPair is an instance of Pair with K as number and V as string.
    • stringBooleanPair is an instance of Pair with K as string and V as boolean.

Constraints on Generics

Sometimes, you may want to restrict the types that can be used with your generic interface. This can be done using constraints.

Example: Generic Interface with Constraints

interface Lengthwise {
    length: number;
}

interface Collection<T extends Lengthwise> {
    items: T[];
    getLength(): number;
}

let stringCollection: Collection<string> = {
    items: ["Hello", "World"],
    getLength: function() {
        return this.items.length;
    }
};

console.log(stringCollection.getLength()); // Output: 2
  • Explanation:
    • Lengthwise is an interface that requires a length property.
    • Collection<T extends Lengthwise> is a generic interface where T must have a length property.
    • stringCollection is an instance of Collection with T as string.

Practical Use Cases

  • Data Structures: Implementing stacks, queues, and linked lists.
  • API Responses: Handling different types of API responses in a type-safe manner.
  • Utility Functions: Creating functions that can operate on various data types.

Example: Generic Stack Interface

interface Stack<T> {
    push(item: T): void;
    pop(): T | undefined;
    peek(): T | undefined;
    size(): number;
}

class ArrayStack<T> implements Stack<T> {
    private storage: T[] = [];

    push(item: T): void {
        this.storage.push(item);
    }

    pop(): T | undefined {
        return this.storage.pop();
    }

    peek(): T | undefined {
        return this.storage[this.storage.length - 1];
    }

    size(): number {
        return this.storage.length;
    }
}

let numberStack = new ArrayStack<number>();
numberStack.push(10);
numberStack.push(20);
console.log(numberStack.pop()); // Output: 20
console.log(numberStack.size()); // Output: 1
  • Explanation:
    • Stack<T> is a generic interface for a stack data structure.
    • ArrayStack<T> is a class that implements the Stack<T> interface using an array.
    • numberStack is an instance of ArrayStack with T as number.

Conclusion

Using generics with interfaces in TypeScript allows you to create flexible, reusable, and type-safe code. By understanding how to define and use generic interfaces, you can build more robust applications that can handle a variety of data types without sacrificing type safety.
