TypeScript Compiler Options

Introduction

TypeScript provides a powerful compiler (tsc) that comes with a wide range of options to customize the compilation process. Understanding these options is crucial for configuring your TypeScript projects effectively.

Basic Compiler Options

`target`

  • Specifies the ECMAScript target version.
  • Common values: ES5, ES6/ES2015, ES2016, ES2017, ES2018, ES2019, ES2020, ES2021, ES2022, ESNext.
{
  "compilerOptions": {
    "target": "ES6"
  }
}

`module`

  • Determines the module system to be used.
  • Common values: CommonJS, AMD, System, UMD, ES6/ES2015, ESNext.
{
  "compilerOptions": {
    "module": "CommonJS"
  }
}

`lib`

  • Specifies the library files to be included in the compilation.
  • Common values: ES5, ES6, DOM, DOM.Iterable, ScriptHost, WebWorker.
{
  "compilerOptions": {
    "lib": ["ES6", "DOM"]
  }
}

Strict Type-Checking Options

`strict`

  • Enables all strict type-checking options.
  • Includes: strictNullChecks, strictFunctionTypes, strictPropertyInitialization, noImplicitAny, noImplicitThis, alwaysStrict.
{
  "compilerOptions": {
    "strict": true
  }
}

`noImplicitAny`

  • Raises an error on expressions and declarations with an implied any type.
{
  "compilerOptions": {
    "noImplicitAny": true
  }
}

`strictNullChecks`

  • Ensures that null and undefined are only assignable to themselves and any.
{
  "compilerOptions": {
    "strictNullChecks": true
  }
}

Module Resolution Options

`baseUrl`

  • Specifies the base directory to resolve non-relative module names.
{
  "compilerOptions": {
    "baseUrl": "./src"
  }
}

`paths`

  • A series of entries that re-map imports to lookup locations relative to the baseUrl.
{
  "compilerOptions": {
    "baseUrl": "./",
    "paths": {
      "@app/*": ["src/app/*"],
      "@components/*": ["src/components/*"]
    }
  }
}

Output Options

`outDir`

  • Redirects output structure to the directory.
{
  "compilerOptions": {
    "outDir": "./dist"
  }
}

`rootDir`

  • Specifies the root directory of input files.
{
  "compilerOptions": {
    "rootDir": "./src"
  }
}

`sourceMap`

  • Generates corresponding .map files.
{
  "compilerOptions": {
    "sourceMap": true
  }
}

Advanced Options

`incremental`

  • Enables incremental compilation.
{
  "compilerOptions": {
    "incremental": true
  }
}

`composite`

  • Enables project compilation.
{
  "compilerOptions": {
    "composite": true
  }
}

`declaration`

  • Generates corresponding .d.ts files.
{
  "compilerOptions": {
    "declaration": true
  }
}

Example Configuration

Here is an example tsconfig.json file that combines several of the options discussed:

{
  "compilerOptions": {
    "target": "ES6",
    "module": "CommonJS",
    "lib": ["ES6", "DOM"],
    "strict": true,
    "baseUrl": "./",
    "paths": {
      "@app/*": ["src/app/*"],
      "@components/*": ["src/components/*"]
    },
    "outDir": "./dist",
    "rootDir": "./src",
    "sourceMap": true,
    "incremental": true,
    "composite": true,
    "declaration": true
  }
}

Conclusion

Understanding and configuring TypeScript compiler options can significantly enhance your development workflow. By leveraging these options, you can tailor the compilation process to meet the specific needs of your project, ensuring better type safety, optimized output, and improved module resolution.
