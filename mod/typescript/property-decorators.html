Property Decorators in TypeScript
Parameter Decorators ►

Property decorators are a powerful feature in TypeScript that allow you to modify or annotate properties within a class. They provide a way to add metadata or change the behavior of properties in a declarative manner. This topic will guide you from the basics to advanced usage of property decorators.

Introduction to Property Decorators

  • Definition: Property decorators are functions that are applied to the properties of a class.
  • Purpose: They can be used to add metadata, perform validation, or modify the behavior of properties.

Basic Syntax

A property decorator is a function that takes two arguments:

  1. The target object (the class prototype for instance properties or the constructor function for static properties).
  2. The name of the property.
function MyPropertyDecorator(target: any, propertyKey: string) {
    console.log(`PropertyDecorator called on: ${propertyKey}`);
}

class ExampleClass {
    @MyPropertyDecorator
    myProperty: string;
}

In this example:

  • MyPropertyDecorator is a simple decorator that logs the name of the property it is applied to.
  • @MyPropertyDecorator is used to annotate myProperty in ExampleClass.

Practical Use Cases

Adding Metadata

Property decorators can be used to add metadata to properties, which can be useful for various purposes such as validation, serialization, or ORM (Object-Relational Mapping).

function Required(target: any, propertyKey: string) {
    if (!target.constructor.requiredProperties) {
        target.constructor.requiredProperties = [];
    }
    target.constructor.requiredProperties.push(propertyKey);
}

class User {
    @Required
    name: string;

    @Required
    email: string;
}

console.log(User.requiredProperties); // Output: ['name', 'email']

Validation

You can use property decorators to enforce validation rules on properties.

function MinLength(length: number) {
    return function (target: any, propertyKey: string) {
        let value: string;

        const getter = () => value;
        const setter = (newVal: string) => {
            if (newVal.length < length) {
                throw new Error(`The length of ${propertyKey} should be at least ${length}`);
            }
            value = newVal;
        };

        Object.defineProperty(target, propertyKey, {
            get: getter,
            set: setter,
            enumerable: true,
            configurable: true
        });
    };
}

class Person {
    @MinLength(3)
    name: string;
}

const person = new Person();
person.name = "Jo"; // Throws an error: The length of name should be at least 3

Advanced Usage

Combining Multiple Decorators

You can apply multiple decorators to a single property. They are applied in the order they are declared.

function FirstDecorator(target: any, propertyKey: string) {
    console.log(`FirstDecorator called on: ${propertyKey}`);
}

function SecondDecorator(target: any, propertyKey: string) {
    console.log(`SecondDecorator called on: ${propertyKey}`);
}

class Demo {
    @FirstDecorator
    @SecondDecorator
    demoProperty: string;
}

// Output:
// SecondDecorator called on: demoProperty
// FirstDecorator called on: demoProperty

Using Decorators with Metadata Reflection

TypeScript supports metadata reflection through the reflect-metadata library, which can be used to store and retrieve metadata about properties.

import 'reflect-metadata';

function ReflectMetadataExample(target: any, propertyKey: string) {
    Reflect.defineMetadata('exampleMetadata', 'someValue', target, propertyKey);
}

class Sample {
    @ReflectMetadataExample
    sampleProperty: string;
}

const metadataValue = Reflect.getMetadata('exampleMetadata', Sample.prototype, 'sampleProperty');
console.log(metadataValue); // Output: 'someValue'

Conclusion

Property decorators in TypeScript provide a powerful way to add metadata, enforce validation, and modify the behavior of class properties. They can be used in a variety of scenarios, from simple logging to complex validation and metadata management. By understanding and utilizing property decorators, you can write more declarative and maintainable code.

In this topic, we covered:

  • The basic syntax and purpose of property decorators.
  • Practical use cases including adding metadata and validation.
  • Advanced usage such as combining multiple decorators and using metadata reflection.

Experiment with property decorators in your projects to see how they can simplify and enhance your code!
