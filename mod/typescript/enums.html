Introduction to Enums

Enums (short for Enumerations) are a feature in TypeScript that allows you to define a set of named constants. They make it easier to work with sets of related values and improve code readability and maintainability.

Key Concepts

  • Enumeration: A way to define a collection of related values.
  • Named Constants: Each value in an enum is a constant with a name.
  • Type Safety: Enums provide type safety by ensuring that only predefined values can be used.

Defining Enums

In TypeScript, you can define an enum using the enum keyword.

Example

enum Direction {
    Up,
    Down,
    Left,
    Right
}

In this example, Direction is an enum with four possible values: Up, Down, Left, and Right.

Explanation

  • By default, the values of the enum members are auto-incremented starting from 0.
  • Up is 0, Down is 1, Left is 2, and Right is 3.

Using Enums

You can use enums to declare variables and assign them enum values.

Example

let move: Direction = Direction.Up;
console.log(move); // Output: 0

Explanation

  • The variable move is of type Direction.
  • It is assigned the value Direction.Up, which corresponds to 0.

Customizing Enum Values

You can assign custom values to enum members.

Example

enum Status {
    Success = 1,
    Failure = -1,
    Pending = 0
}

Explanation

  • Success is assigned the value 1.
  • Failure is assigned the value -1.
  • Pending is assigned the value 0.

String Enums

Enums can also have string values.

Example

enum Color {
    Red = "RED",
    Green = "GREEN",
    Blue = "BLUE"
}

Explanation

  • Each member of the Color enum is assigned a string value.

Heterogeneous Enums

Enums can have both numeric and string values, though this is less common.

Example

enum Mixed {
    No = 0,
    Yes = "YES"
}

Explanation

  • No is assigned the numeric value 0.
  • Yes is assigned the string value "YES".

Enum Member Types

Enum members can be constant or computed.

Constant Members

These are members with values that can be computed at compile time.

Example

enum ConstantEnum {
    A = 1,
    B = A + 1
}

Explanation

  • A is 1.
  • B is A + 1, which is 2.

Computed Members

These are members with values that are computed at runtime.

Example

enum ComputedEnum {
    X = "X".length,
    Y = 10 * 2
}

Explanation

  • X is the length of the string "X", which is 1.
  • Y is the result of the expression 10 * 2, which is 20.

Enum Reverse Mapping

TypeScript provides reverse mapping for numeric enums.

Example

enum Days {
    Monday,
    Tuesday,
    Wednesday
}

console.log(Days.Monday); // Output: 0
console.log(Days[0]); // Output: "Monday"

Explanation

  • Days.Monday gives the numeric value 0.
  • Days[0] gives the string "Monday".

Enum as Types

Enums can be used as types to ensure that variables can only take on the values defined in the enum.

Example

enum Size {
    Small,
    Medium,
    Large
}

function selectSize(size: Size): void {
    console.log(`Selected size: ${Size[size]}`);
}

selectSize(Size.Medium); // Output: Selected size: Medium

Explanation

  • The function selectSize accepts a parameter of type Size.
  • This ensures that only valid Size values can be passed to the function.

Conclusion

Enums in TypeScript provide a powerful way to define and use sets of related constants. They improve code readability, maintainability, and type safety. By understanding how to define, use, and customize enums, you can leverage their full potential in your TypeScript projects.
