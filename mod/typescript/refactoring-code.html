Refactoring TypeScript Code
Performance Optimization ►

Refactoring is the process of restructuring existing computer code without changing its external behavior. In TypeScript, refactoring can help improve code readability, maintainability, and performance. This section will guide you through the principles and techniques of refactoring TypeScript code, from basic to advanced levels.

Why Refactor?

  • Improve Readability: Makes the code easier to understand.
  • Enhance Maintainability: Simplifies future modifications.
  • Optimize Performance: Improves the efficiency of the code.
  • Reduce Complexity: Breaks down complex code into simpler parts.

Basic Refactoring Techniques

Renaming Variables and Functions

Renaming variables and functions to more descriptive names can significantly improve code readability.

Before:

function calc(a: number, b: number): number {
    return a + b;
}

const x = 5;
const y = 10;
console.log(calc(x, y));

After:

function calculateSum(firstNumber: number, secondNumber: number): number {
    return firstNumber + secondNumber;
}

const firstValue = 5;
const secondValue = 10;
console.log(calculateSum(firstValue, secondValue));

Extracting Functions

Extracting parts of a function into smaller, reusable functions can make the code more modular and easier to test.

Before:

function processNumbers(numbers: number[]): number {
    let total = 0;
    for (let i = 0; i < numbers.length; i++) {
        total += numbers[i];
    }
    return total / numbers.length;
}

After:

function sum(numbers: number[]): number {
    return numbers.reduce((acc, curr) => acc + curr, 0);
}

function average(numbers: number[]): number {
    return sum(numbers) / numbers.length;
}

function processNumbers(numbers: number[]): number {
    return average(numbers);
}

Simplifying Conditional Expressions

Simplifying complex conditional expressions can make the code easier to follow.

Before:

function getDiscount(price: number): number {
    if (price > 100) {
        return price * 0.1;
    } else {
        return 0;
    }
}

After:

function getDiscount(price: number): number {
    return price > 100 ? price * 0.1 : 0;
}

Intermediate Refactoring Techniques

Using Type Aliases and Interfaces

Type aliases and interfaces can make complex types more readable and reusable.

Before:

function printUser(user: { name: string; age: number; email: string }): void {
    console.log(`Name: ${user.name}, Age: ${user.age}, Email: ${user.email}`);
}

After:

interface User {
    name: string;
    age: number;
    email: string;
}

function printUser(user: User): void {
    console.log(`Name: ${user.name}, Age: ${user.age}, Email: ${user.email}`);
}

Removing Magic Numbers

Replacing magic numbers with named constants can make the code more understandable.

Before:

function calculateArea(radius: number): number {
    return 3.14 * radius * radius;
}

After:

const PI = 3.14;

function calculateArea(radius: number): number {
    return PI * radius * radius;
}

Advanced Refactoring Techniques

Applying Design Patterns

Using design patterns can help solve common problems in a more structured way.

Before:

class UserService {
    private users: string[] = [];

    addUser(user: string): void {
        this.users.push(user);
    }

    getUsers(): string[] {
        return this.users;
    }
}

After (Singleton Pattern):

class UserService {
    private static instance: UserService;
    private users: string[] = [];

    private constructor() {}

    static getInstance(): UserService {
        if (!UserService.instance) {
            UserService.instance = new UserService();
        }
        return UserService.instance;
    }

    addUser(user: string): void {
        this.users.push(user);
    }

    getUsers(): string[] {
        return this.users;
    }
}

const userService = UserService.getInstance();
userService.addUser('John Doe');
console.log(userService.getUsers());

Refactoring for Performance

Optimizing code for performance can involve various techniques, such as memoization or lazy loading.

Before:

function fibonacci(n: number): number {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2);
}

After (Memoization):

const memo: { [key: number]: number } = {};

function fibonacci(n: number): number {
    if (n <= 1) return n;
    if (memo[n]) return memo[n];
    memo[n] = fibonacci(n - 1) + fibonacci(n - 2);
    return memo[n];
}

Conclusion

Refactoring is a crucial skill for any developer. It helps improve the quality of your code, making it more readable, maintainable, and efficient. By applying the techniques discussed in this section, you can ensure that your TypeScript code remains clean and robust as your projects grow in complexity. Remember, the goal of refactoring is not just to make the code work but to make it work well.
