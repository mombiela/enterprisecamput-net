Introduction

Variable declarations are a fundamental concept in TypeScript, as they are in any programming language. Understanding how to declare and use variables effectively is crucial for writing clean, efficient, and bug-free code.

Basic Variable Declarations

In TypeScript, you can declare variables using three keywords: var, let, and const.

`var`

  • The var keyword is used to declare a variable that can be re-assigned.
  • Variables declared with var are function-scoped or globally-scoped, not block-scoped.
var message = "Hello, World!";
console.log(message); // Output: Hello, World!

if (true) {
    var count = 10;
}
console.log(count); // Output: 10 (accessible outside the block)

`let`

  • The let keyword is used to declare a block-scoped variable that can be re-assigned.
  • It is preferred over var due to its block-scoping behavior.
let message = "Hello, TypeScript!";
console.log(message); // Output: Hello, TypeScript!

if (true) {
    let count = 10;
    console.log(count); // Output: 10 (accessible inside the block)
}
console.log(count); // Error: count is not defined (not accessible outside the block)

`const`

  • The const keyword is used to declare a block-scoped variable that cannot be re-assigned.
  • It is used for variables that should remain constant throughout the program.
const message = "Hello, TypeScript!";
console.log(message); // Output: Hello, TypeScript!

message = "New Message"; // Error: Assignment to constant variable.

Type Annotations

TypeScript allows you to specify the type of a variable using type annotations. This helps catch errors at compile time and makes the code more readable.

let message: string = "Hello, TypeScript!";
let count: number = 10;
let isActive: boolean = true;

Type Inference

TypeScript can automatically infer the type of a variable based on the value assigned to it. This means you don't always need to explicitly specify the type.

let message = "Hello, TypeScript!"; // TypeScript infers that message is of type string
let count = 10; // TypeScript infers that count is of type number
let isActive = true; // TypeScript infers that isActive is of type boolean

Variable Scope

Understanding the scope of variables is crucial for writing maintainable code. Scope determines the visibility and lifetime of a variable.

Global Scope

  • Variables declared outside any function or block are in the global scope.
  • They can be accessed from anywhere in the code.
var globalMessage = "I am global";

function showMessage() {
    console.log(globalMessage); // Output: I am global
}

Function Scope

  • Variables declared inside a function are in the function scope.
  • They can only be accessed within that function.
function showMessage() {
    var localMessage = "I am local";
    console.log(localMessage); // Output: I am local
}
console.log(localMessage); // Error: localMessage is not defined

Block Scope

  • Variables declared with let or const inside a block (e.g., inside an if statement or loop) are in the block scope.
  • They can only be accessed within that block.
if (true) {
    let blockMessage = "I am block-scoped";
    console.log(blockMessage); // Output: I am block-scoped
}
console.log(blockMessage); // Error: blockMessage is not defined

Summary

  • Use var for function-scoped or globally-scoped variables, but prefer let and const for block-scoped variables.
  • Use let for variables that can be re-assigned and const for variables that should remain constant.
  • Type annotations help catch errors and make the code more readable, but TypeScript can also infer types.
  • Understanding variable scope (global, function, and block) is essential for writing maintainable code.

By mastering variable declarations in TypeScript, you lay a strong foundation for writing robust and efficient TypeScript applications.
