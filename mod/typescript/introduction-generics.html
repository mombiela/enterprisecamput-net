Introduction to Generics
Generic Functions ►

Generics are a powerful feature in TypeScript that allow you to create reusable and flexible components. They enable you to write functions, classes, and interfaces that work with a variety of data types while still maintaining type safety. This section will guide you through the basics of generics, their syntax, and practical use cases.

What are Generics?

Generics provide a way to create components that can work with any data type. They help in making the code more flexible and reusable without sacrificing type safety.

  • Type Safety: Ensures that the types of variables are known at compile time.
  • Reusability: Allows the same function, class, or interface to work with different data types.
  • Flexibility: Enables the creation of more general and abstract components.

Basic Syntax

The basic syntax for generics involves using angle brackets (< >) to define a generic type.

Generic Functions

A generic function can accept any data type as an argument and return a value of the same type.

function identity<T>(arg: T): T {
    return arg;
}

let output1 = identity<string>("Hello, TypeScript!");
let output2 = identity<number>(42);
  • T is a type variable that acts as a placeholder for the type that will be passed to the function.
  • identity<string>("Hello, TypeScript!") specifies that T is string.
  • identity<number>(42) specifies that T is number.

Generic Classes

Generic classes work similarly to generic functions. They allow you to create a class that can handle different types.

class GenericNumber<T> {
    zeroValue: T;
    add: (x: T, y: T) => T;
}

let myGenericNumber = new GenericNumber<number>();
myGenericNumber.zeroValue = 0;
myGenericNumber.add = (x, y) => x + y;
  • GenericNumber<T> is a generic class where T is a type variable.
  • myGenericNumber is an instance of GenericNumber with T set to number.

Generic Interfaces

Generic interfaces allow you to define the shape of an object that can work with various types.

interface GenericIdentityFn<T> {
    (arg: T): T;
}

function identity<T>(arg: T): T {
    return arg;
}

let myIdentity: GenericIdentityFn<number> = identity;
  • GenericIdentityFn<T> is a generic interface.
  • myIdentity is a function that adheres to the GenericIdentityFn<number> interface.

Constraints

Sometimes, you need to impose constraints on the types that can be used with generics. This can be done using the extends keyword.

interface Lengthwise {
    length: number;
}

function loggingIdentity<T extends Lengthwise>(arg: T): T {
    console.log(arg.length);
    return arg;
}

loggingIdentity({ length: 10, value: "Hello" });
  • T extends Lengthwise ensures that T has a length property.
  • loggingIdentity can now only accept arguments that have a length property.

Using Multiple Type Variables

You can use multiple type variables in a single generic function, class, or interface.

function swap<T, U>(tuple: [T, U]): [U, T] {
    return [tuple[1], tuple[0]];
}

let swappedTuple = swap([7, "seven"]); // [ "seven", 7 ]
  • T and U are type variables.
  • swap function swaps the elements of the tuple.

Practical Use Cases

Generics are widely used in TypeScript for various purposes:

  • Collections: Arrays, linked lists, stacks, queues.
  • Utility Functions: Sorting, searching, filtering.
  • Data Structures: Trees, graphs, maps.

Example: Generic Stack

class Stack<T> {
    private items: T[] = [];

    push(item: T): void {
        this.items.push(item);
    }

    pop(): T | undefined {
        return this.items.pop();
    }
}

let numberStack = new Stack<number>();
numberStack.push(10);
numberStack.push(20);
console.log(numberStack.pop()); // 20

let stringStack = new Stack<string>();
stringStack.push("Hello");
stringStack.push("World");
console.log(stringStack.pop()); // "World"
  • Stack<T> is a generic class.
  • numberStack and stringStack are instances of Stack with different types.

Conclusion

Generics in TypeScript provide a powerful way to create flexible, reusable, and type-safe components. By understanding and utilizing generics, you can write more abstract and generalized code that works with a variety of data types. This not only enhances code reusability but also ensures type safety, making your TypeScript applications more robust and maintainable.
