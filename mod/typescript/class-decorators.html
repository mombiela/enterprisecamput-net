Introduction to Class Decorators

Class decorators are a powerful feature in TypeScript that allow you to modify or annotate classes. They are functions that take a class constructor as an argument and return a new constructor or modify the existing one.

Key Concepts

  • Decorator Function: A function that takes a class constructor as an argument.
  • Annotation: Adding metadata to a class.
  • Modification: Changing the behavior or properties of a class.

Basic Syntax of Class Decorators

A class decorator is defined with the @ symbol followed by the decorator function name.

function simpleDecorator(constructor: Function) {
    console.log("SimpleDecorator called.");
}

@simpleDecorator
class MyClass {
    constructor() {
        console.log("MyClass instance created.");
    }
}

const myInstance = new MyClass();
// Output:
// SimpleDecorator called.
// MyClass instance created.

Explanation

  • The simpleDecorator function logs a message when it is called.
  • The @simpleDecorator syntax applies the decorator to MyClass.
  • When MyClass is instantiated, the decorator function is executed first.

Creating Useful Class Decorators

Class decorators can be used for various purposes, such as logging, validation, or modifying class properties.

Example: Logging Decorator

function logClass(constructor: Function) {
    console.log(`Class ${constructor.name} is created.`);
}

@logClass
class User {
    constructor(public name: string) {}
}

const user = new User("Alice");
// Output:
// Class User is created.

Explanation

  • The logClass decorator logs the name of the class when it is created.
  • The User class is annotated with @logClass, so when an instance of User is created, the decorator logs the class name.

Advanced Usage: Modifying Class Behavior

Class decorators can also modify the behavior of the class by returning a new constructor.

Example: Adding a Timestamp

function addTimestamp<T extends { new(...args: any[]): {} }>(constructor: T) {
    return class extends constructor {
        timestamp = new Date();
    }
}

@addTimestamp
class Document {
    constructor(public title: string) {}
}

const doc = new Document("My Document");
console.log(doc.timestamp); // Outputs the current date and time

Explanation

  • The addTimestamp decorator adds a timestamp property to the class.
  • It returns a new class that extends the original class and adds the timestamp property.
  • When an instance of Document is created, it includes the timestamp property with the current date and time.

Combining Multiple Decorators

You can apply multiple decorators to a single class. They are executed in the order they are declared, from top to bottom.

Example: Combining Decorators

function firstDecorator(constructor: Function) {
    console.log("FirstDecorator called.");
}

function secondDecorator(constructor: Function) {
    console.log("SecondDecorator called.");
}

@firstDecorator
@secondDecorator
class Product {
    constructor(public name: string) {}
}

const product = new Product("Laptop");
// Output:
// FirstDecorator called.
// SecondDecorator called.

Explanation

  • firstDecorator and secondDecorator are applied to the Product class.
  • Both decorators log a message when the class is created.
  • The decorators are executed in the order they are declared.

Conclusion

Class decorators in TypeScript provide a flexible way to modify and annotate classes. They can be used for logging, validation, adding properties, or even altering class behavior. Understanding how to create and apply class decorators can greatly enhance your TypeScript programming skills.

Summary

  • Class decorators are functions that take a class constructor as an argument.
  • They can be used to annotate or modify classes.
  • Decorators are applied using the @ symbol.
  • Multiple decorators can be combined and are executed in the order they are declared.

By mastering class decorators, you can write more modular, reusable, and maintainable code in TypeScript.
