End-to-End Testing in TypeScript
TypeScript Compiler Options ►

End-to-end (E2E) testing is a crucial part of the software development lifecycle, ensuring that the entire application flow works as expected. This topic will guide you through the basics of E2E testing in TypeScript, from setting up your environment to writing and running your first tests.

What is End-to-End Testing?

End-to-end testing involves testing the complete functionality of an application, from the user interface to the backend and database. It simulates real user scenarios to validate the system's behavior.

  • Purpose: To ensure that the application works as a whole.
  • Scope: Covers the entire application, including UI, APIs, databases, and other integrated services.
  • Tools: Common tools include Cypress, Selenium, and Playwright.

Setting Up Your Environment

Before writing E2E tests, you need to set up your testing environment. For this guide, we'll use Cypress, a popular E2E testing framework.

Installing Cypress

First, install Cypress via npm:

npm install cypress --save-dev

Configuring Cypress

After installation, you need to configure Cypress. Create a cypress.json file in the root of your project:

{
  "baseUrl": "http://localhost:3000",
  "integrationFolder": "cypress/integration"
}
  • baseUrl: The URL where your application is running.
  • integrationFolder: The folder where your test files will be located.

Writing Your First E2E Test

Let's write a simple E2E test to check if the homepage loads correctly.

Creating a Test File

Create a new file named home.spec.ts in the cypress/integration folder:

describe('Homepage', () => {
  it('should load the homepage', () => {
    cy.visit('/');
    cy.contains('Welcome to My App');
  });
});

Explanation

  • describe: Groups related tests.
  • it: Defines a single test case.
  • cy.visit('/'): Navigates to the homepage.
  • cy.contains('Welcome to My App'): Checks if the text "Welcome to My App" is present on the page.

Running Your Tests

To run your tests, use the following command:

npx cypress open

This will open the Cypress Test Runner, where you can select and run your tests.

Advanced E2E Testing Concepts

As you become more comfortable with E2E testing, you can explore more advanced concepts.

Handling Authentication

Many applications require authentication. You can handle this in Cypress by using custom commands.

Example: Custom Command for Login

Create a new file named commands.ts in the cypress/support folder:

Cypress.Commands.add('login', (username, password) => {
  cy.request('POST', '/api/login', {
    username,
    password
  }).then((response) => {
    window.localStorage.setItem('authToken', response.body.token);
  });
});

Use this custom command in your tests:

describe('Dashboard', () => {
  beforeEach(() => {
    cy.login('testuser', 'password123');
  });

  it('should load the dashboard', () => {
    cy.visit('/dashboard');
    cy.contains('Dashboard');
  });
});

Mocking API Requests

To isolate frontend tests from backend dependencies, you can mock API requests.

Example: Mocking an API Request

describe('User Profile', () => {
  beforeEach(() => {
    cy.intercept('GET', '/api/user', {
      statusCode: 200,
      body: { name: 'John Doe', email: 'john.doe@example.com' }
    }).as('getUser');
  });

  it('should display user profile', () => {
    cy.visit('/profile');
    cy.wait('@getUser');
    cy.contains('John Doe');
    cy.contains('john.doe@example.com');
  });
});

Conclusion

End-to-end testing is essential for ensuring the reliability and functionality of your application. By using tools like Cypress and writing comprehensive tests, you can catch bugs early and improve the overall quality of your software. As you advance, explore more complex scenarios and integrate E2E testing into your continuous integration pipeline for automated testing.
