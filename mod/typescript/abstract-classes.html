Introduction to Abstract Classes

Abstract classes are a fundamental concept in object-oriented programming (OOP) that allow you to define a base class with common functionality, while deferring some implementation details to derived classes. In TypeScript, abstract classes provide a way to create blueprints for other classes.

Key Concepts

  • Abstract Class: A class that cannot be instantiated directly. It is meant to be subclassed.
  • Abstract Method: A method that is declared in an abstract class but does not have an implementation. Subclasses are required to provide an implementation for these methods.
  • Inheritance: Abstract classes are used as base classes from which other classes can inherit.

Defining Abstract Classes

To define an abstract class in TypeScript, use the abstract keyword. Abstract classes can contain both abstract methods (without implementation) and concrete methods (with implementation).

Example

abstract class Animal {
    // Abstract method (must be implemented in derived classes)
    abstract makeSound(): void;

    // Concrete method
    move(): void {
        console.log("Moving...");
    }
}

In this example, Animal is an abstract class with an abstract method makeSound and a concrete method move.

Implementing Abstract Classes

When a class inherits from an abstract class, it must implement all abstract methods from the parent class.

Example

class Dog extends Animal {
    makeSound(): void {
        console.log("Bark!");
    }
}

const myDog = new Dog();
myDog.makeSound(); // Output: Bark!
myDog.move();      // Output: Moving...

In this example, Dog is a subclass of Animal and provides an implementation for the makeSound method.

Use Cases for Abstract Classes

Abstract classes are useful when you want to:

  • Define a common interface for a group of related classes.
  • Provide some shared functionality while forcing subclasses to implement specific methods.
  • Ensure that certain methods are implemented in derived classes.

Comparison: Abstract Classes vs Interfaces

Both abstract classes and interfaces are used to define the structure of a class, but they have some differences.

| Feature | Abstract Class | Interface | |---------------------|---------------------------------------|------------------------------------| | Inheritance | Can inherit from one abstract class | Can implement multiple interfaces | | Method Implementation | Can have concrete methods | Cannot have concrete methods | | Properties | Can have properties with implementations | Only declarations, no implementations | | Instantiation | Cannot be instantiated directly | Cannot be instantiated directly |

Example of Interface

interface AnimalInterface {
    makeSound(): void;
    move(): void;
}

class Cat implements AnimalInterface {
    makeSound(): void {
        console.log("Meow!");
    }

    move(): void {
        console.log("Moving silently...");
    }
}

In this example, Cat implements the AnimalInterface and provides implementations for both makeSound and move methods.

Conclusion

Abstract classes in TypeScript are a powerful tool for creating a common blueprint for related classes. They allow you to define methods that must be implemented by subclasses, ensuring a consistent interface while providing shared functionality. Understanding when and how to use abstract classes, as well as the differences between abstract classes and interfaces, is crucial for effective TypeScript programming.

By mastering abstract classes, you can create more organized, maintainable, and scalable codebases.
