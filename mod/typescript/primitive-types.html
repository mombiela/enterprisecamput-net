Introduction

Primitive types are the most basic data types available in TypeScript. Understanding these types is crucial as they form the foundation for more complex data structures and operations. In this section, we will explore the different primitive types available in TypeScript, their usage, and how to work with them effectively.

Key Primitive Types

Number

  • Represents both integer and floating-point numbers.
  • Supports hexadecimal, decimal, binary, and octal literals.
let decimal: number = 6;
let hex: number = 0xf00d;
let binary: number = 0b1010;
let octal: number = 0o744;

console.log(decimal); // Output: 6
console.log(hex); // Output: 61453
console.log(binary); // Output: 10
console.log(octal); // Output: 484

String

  • Represents textual data.
  • Can be enclosed in single quotes ('), double quotes ("), or backticks (`) for template literals.
let color: string = "blue";
let sentence: string = `The color is ${color}`;

console.log(color); // Output: blue
console.log(sentence); // Output: The color is blue

Boolean

  • Represents a logical value: true or false.
let isDone: boolean = false;

console.log(isDone); // Output: false

Null and Undefined

  • null and undefined are types that have only one value each: null and undefined respectively.
  • By default, null and undefined are subtypes of all other types.
let u: undefined = undefined;
let n: null = null;

console.log(u); // Output: undefined
console.log(n); // Output: null

Symbol

  • Represents a unique identifier.
  • Created using the Symbol function.
let sym1 = Symbol();
let sym2 = Symbol("key");

console.log(sym1 === sym2); // Output: false

BigInt

  • Represents whole numbers larger than the largest number JavaScript can reliably represent with the number primitive.
  • Created by appending n to the end of an integer or using the BigInt function.
let big: bigint = 100n;

console.log(big); // Output: 100n

Comparison of Primitive Types

Type Example Description
number let num = 42; Represents numeric values
string let str = "Hi"; Represents textual data
boolean let isTrue = true; Represents logical values
null let n = null; Represents the intentional absence of any object value
undefined let u; Represents an uninitialized variable
symbol let sym = Symbol(); Represents a unique identifier
bigint let big = 100n; Represents large integers

Type Assertions

TypeScript allows you to override the inferred type using type assertions. This is useful when you know more about the type of a value than TypeScript does.

let someValue: any = "this is a string";
let strLength: number = (someValue as string).length;

console.log(strLength); // Output: 16

Conclusion

Understanding primitive types in TypeScript is essential for any programmer. These types form the building blocks of your applications, and knowing how to use them effectively will help you write more robust and error-free code. In this section, we covered the basic primitive types, their usage, and how to perform type assertions. As you progress, you'll see how these fundamental concepts integrate with more advanced TypeScript features.
