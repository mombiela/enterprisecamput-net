Introduction

In this section, we will explore the concepts of Props and State in TypeScript, particularly in the context of React. Understanding these concepts is crucial for building dynamic and interactive web applications.

Props

Props (short for properties) are read-only attributes passed from a parent component to a child component. They are used to pass data and event handlers down the component tree.

Key Concepts

  • Immutability: Props are immutable, meaning they cannot be changed by the child component.
  • Type Safety: TypeScript allows you to define the types of props, providing better type safety and autocompletion.

Example

Let's create a simple React component that uses props in TypeScript.

import React from 'react';

// Define the type for props
interface GreetingProps {
  name: string;
  age: number;
}

// Functional component using props
const Greeting: React.FC<GreetingProps> = ({ name, age }) => {
  return (
    <div>
      <h1>Hello, {name}!</h1>
      <p>You are {age} years old.</p>
    </div>
  );
};

// Usage
const App: React.FC = () => {
  return (
    <div>
      <Greeting name="Alice" age={30} />
    </div>
  );
};

export default App;

In this example:

  • We define an interface GreetingProps to specify the types of name and age.
  • The Greeting component uses these props to render a message.
  • The App component passes the props to Greeting.

State

State is a way to manage data that can change over time within a component. Unlike props, state is mutable and can be updated by the component itself.

Key Concepts

  • Mutability: State can be updated using the setState function.
  • Local to Component: State is local to the component and cannot be accessed by other components unless passed down as props.

Example

Let's create a simple React component that uses state in TypeScript.

import React, { useState } from 'react';

// Functional component using state
const Counter: React.FC = () => {
  // Define state with initial value
  const [count, setCount] = useState<number>(0);

  return (
    <div>
      <h1>Count: {count}</h1>
      <button onClick={() => setCount(count + 1)}>Increment</button>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
    </div>
  );
};

// Usage
const App: React.FC = () => {
  return (
    <div>
      <Counter />
    </div>
  );
};

export default App;

In this example:

  • We use the useState hook to create a state variable count with an initial value of 0.
  • The setCount function is used to update the state.
  • The component renders buttons to increment and decrement the count.

Comparing Props and State

| Feature | Props | State | |---------------|--------------------------------|--------------------------------| | Mutability | Immutable | Mutable | | Scope | Passed from parent to child | Local to the component | | Update Method | Cannot be updated by the child | Updated using setState | | Usage | Pass data and event handlers | Manage dynamic data within the component |

Conclusion

Understanding props and state is fundamental to building React applications with TypeScript. Props allow you to pass data and functions down the component tree, while state enables you to manage dynamic data within a component. By leveraging TypeScript's type safety, you can create more robust and maintainable code. Practice using props and state in your projects to solidify these concepts.
