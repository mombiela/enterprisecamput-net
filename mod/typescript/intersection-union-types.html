Intersection and Union Types in TypeScript
Type Guards ►

Introduction

In TypeScript, understanding intersection and union types is crucial for creating flexible and robust type systems. These types allow you to combine multiple types into one, enabling more precise type definitions and safer code.

Union Types

Union types allow a variable to be one of several types. This is useful when a value can be more than one type.

Key Concepts

  • Definition: A union type is defined using the | (pipe) symbol.
  • Usage: Union types are used when a value can be one of several types.

Examples

let value: string | number;

value = "Hello"; // valid
value = 42;      // valid
value = true;    // invalid, TypeScript will throw an error

Explanation

In the example above, value can be either a string or a number. Assigning a boolean value to value will result in a TypeScript error because boolean is not part of the union type.

Practical Use Cases

  • Function Parameters: When a function can accept multiple types.
  • API Responses: When an API can return different types of data.
function printId(id: string | number) {
    console.log(`ID: ${id}`);
}

printId("abc123"); // valid
printId(123);      // valid

Intersection Types

Intersection types combine multiple types into one. This means that a value must satisfy all the combined types.

Key Concepts

  • Definition: An intersection type is defined using the & (ampersand) symbol.
  • Usage: Intersection types are used when a value must have all the properties of the combined types.

Examples

interface Person {
    name: string;
}

interface Employee {
    employeeId: number;
}

type EmployeePerson = Person & Employee;

const employee: EmployeePerson = {
    name: "John Doe",
    employeeId: 1234
};

Explanation

In the example above, EmployeePerson is an intersection type that combines Person and Employee. The employee object must have both name and employeeId properties.

Practical Use Cases

  • Combining Interfaces: When an object needs to satisfy multiple interfaces.
  • Enhancing Types: When extending types with additional properties.
interface Address {
    street: string;
    city: string;
}

type EmployeeWithAddress = Employee & Address;

const employeeWithAddress: EmployeeWithAddress = {
    employeeId: 1234,
    street: "123 Main St",
    city: "Anytown"
};

Comparison of Union and Intersection Types

| Feature | Union Types (|) | Intersection Types (&) | |------------------------|---------------------------------|----------------------------------| | Definition | type AorB = A \| B | type AandB = A & B | | Usage | Value can be one of the types | Value must satisfy all types | | Example | string \| number | Person & Employee | | Practical Use Cases | Function parameters, API responses | Combining interfaces, enhancing types |

Conclusion

Understanding and utilizing intersection and union types in TypeScript can greatly enhance the flexibility and safety of your code. Union types allow for more versatile variable definitions, while intersection types ensure that objects meet multiple type requirements. Mastering these concepts will help you write more robust and maintainable TypeScript code.
