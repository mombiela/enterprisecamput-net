Mocking in Tests
End-to-End Testing ►

Mocking is a crucial concept in testing, especially when dealing with external dependencies or complex systems. In this section, we will explore how to use mocking in TypeScript tests, from basic to advanced levels.

Introduction to Mocking

Mocking allows you to simulate the behavior of complex objects or external systems in your tests. This is particularly useful for:

  • Isolating the unit of code being tested.
  • Simulating different scenarios and edge cases.
  • Improving test performance by avoiding slow or unreliable external systems.

Basic Mocking with Jest

Jest is a popular testing framework that provides built-in support for mocking. Let's start with a simple example.

Example: Mocking a Function

Suppose we have a function that fetches user data from an API:

// userService.ts
export const fetchUserData = async (userId: string): Promise<User> => {
  const response = await fetch(`https://api.example.com/users/${userId}`);
  return response.json();
};

We want to test a function that uses fetchUserData without making actual API calls:

// userController.ts
import { fetchUserData } from './userService';

export const getUserProfile = async (userId: string): Promise<UserProfile> => {
  const user = await fetchUserData(userId);
  return {
    id: user.id,
    name: user.name,
    email: user.email,
  };
};

Here's how you can mock fetchUserData in your test:

// userController.test.ts
import { getUserProfile } from './userController';
import { fetchUserData } from './userService';

jest.mock('./userService');

test('should return user profile', async () => {
  const mockUser = { id: '1', name: 'John Doe', email: 'john@example.com' };
  (fetchUserData as jest.Mock).mockResolvedValue(mockUser);

  const userProfile = await getUserProfile('1');

  expect(userProfile).toEqual(mockUser);
});

Intermediate Mocking Techniques

Mocking Modules

Sometimes, you need to mock entire modules. Jest makes this easy with the jest.mock function.

// math.ts
export const add = (a: number, b: number): number => a + b;
export const subtract = (a: number, b: number): number => a - b;
// math.test.ts
import * as math from './math';

jest.mock('./math');

test('should mock add function', () => {
  (math.add as jest.Mock).mockReturnValue(10);

  expect(math.add(1, 2)).toBe(10);
});

Mocking Classes

You can also mock classes and their methods:

// database.ts
export class Database {
  connect() {
    // Connect to the database
  }

  query(sql: string) {
    // Execute a query
  }
}
// database.test.ts
import { Database } from './database';

jest.mock('./database');

test('should mock Database class', () => {
  const mockConnect = jest.fn();
  Database.prototype.connect = mockConnect;

  const db = new Database();
  db.connect();

  expect(mockConnect).toHaveBeenCalled();
});

Advanced Mocking Techniques

Mocking Asynchronous Functions

When dealing with asynchronous functions, you can use mockResolvedValue and mockRejectedValue to simulate resolved and rejected promises.

// api.ts
export const fetchData = async (): Promise<string> => {
  const response = await fetch('https://api.example.com/data');
  return response.json();
};
// api.test.ts
import { fetchData } from './api';

jest.mock('./api');

test('should mock fetchData with resolved value', async () => {
  (fetchData as jest.Mock).mockResolvedValue('mock data');

  const data = await fetchData();

  expect(data).toBe('mock data');
});

test('should mock fetchData with rejected value', async () => {
  (fetchData as jest.Mock).mockRejectedValue(new Error('Network error'));

  await expect(fetchData()).rejects.toThrow('Network error');
});

Mocking with TypeScript Interfaces

When working with TypeScript interfaces, you can use libraries like ts-mockito to create mocks.

// userService.ts
export interface UserService {
  getUser(id: string): Promise<User>;
}
// userService.test.ts
import { mock, instance, when, verify } from 'ts-mockito';
import { UserService } from './userService';

test('should mock UserService interface', async () => {
  const userServiceMock = mock<UserService>();
  when(userServiceMock.getUser('1')).thenResolve({ id: '1', name: 'John Doe' });

  const userService = instance(userServiceMock);
  const user = await userService.getUser('1');

  expect(user.name).toBe('John Doe');
  verify(userServiceMock.getUser('1')).called();
});

Conclusion

Mocking is a powerful tool in your testing arsenal. It allows you to isolate the code under test, simulate various scenarios, and improve test performance. By mastering basic, intermediate, and advanced mocking techniques, you can write more robust and reliable tests for your TypeScript applications.
