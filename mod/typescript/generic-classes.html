In this section, we will explore the concept of Generic Classes in TypeScript. Generics allow you to create flexible and reusable components that can work with a variety of data types. This is particularly useful for creating data structures like lists, stacks, and queues.

What are Generics?

Generics provide a way to create classes, functions, and interfaces that can work with different data types without sacrificing type safety. They allow you to define a placeholder for a type that can be specified later.

Key Concepts

  • Type Parameter: A placeholder for a type that is specified when the generic class is instantiated.
  • Type Safety: Ensures that the types of variables are known at compile time, reducing runtime errors.
  • Reusability: Generic classes can be reused with different data types, making your code more flexible and maintainable.

Defining a Generic Class

To define a generic class, you use angle brackets (<>) to specify the type parameter. Here's a simple example of a generic class in TypeScript:

class Box<T> {
    private content: T;

    constructor(content: T) {
        this.content = content;
    }

    getContent(): T {
        return this.content;
    }

    setContent(content: T): void {
        this.content = content;
    }
}

In this example:

  • T is the type parameter.
  • The class Box can now hold any type of content, specified when the class is instantiated.

Using a Generic Class

You can create instances of the generic class with different types:

let numberBox = new Box<number>(123);
console.log(numberBox.getContent()); // Output: 123

let stringBox = new Box<string>("Hello, TypeScript!");
console.log(stringBox.getContent()); // Output: Hello, TypeScript!

Multiple Type Parameters

You can also define a generic class with multiple type parameters:

class Pair<K, V> {
    private key: K;
    private value: V;

    constructor(key: K, value: V) {
        this.key = key;
        this.value = value;
    }

    getKey(): K {
        return this.key;
    }

    getValue(): V {
        return this.value;
    }
}

Using the Pair class:

let pair = new Pair<string, number>("age", 30);
console.log(pair.getKey()); // Output: age
console.log(pair.getValue()); // Output: 30

Constraints on Type Parameters

Sometimes you might want to restrict the types that can be used as type parameters. You can achieve this using constraints:

class Container<T extends { length: number }> {
    private content: T;

    constructor(content: T) {
        this.content = content;
    }

    getContentLength(): number {
        return this.content.length;
    }
}

In this example:

  • The type parameter T is constrained to types that have a length property.

Using the Container class:

let stringContainer = new Container<string>("Hello");
console.log(stringContainer.getContentLength()); // Output: 5

let arrayContainer = new Container<number[]>([1, 2, 3, 4]);
console.log(arrayContainer.getContentLength()); // Output: 4

Summary

Generic classes in TypeScript provide a powerful way to create flexible and reusable components. By using type parameters, you can write classes that work with any data type while maintaining type safety. This makes your code more robust and easier to maintain.

Key Takeaways

  • Generics allow you to create classes that can work with any data type.
  • Type parameters are placeholders for types that are specified when the class is instantiated.
  • You can define multiple type parameters and add constraints to them.
  • Using generics enhances code reusability and type safety.

By mastering generics, you can write more flexible and maintainable TypeScript code, making your applications more robust and easier to extend.
