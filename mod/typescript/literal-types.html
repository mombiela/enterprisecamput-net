Introduction to Literal Types

Literal types in TypeScript allow you to specify exact values a variable can hold. This feature enhances type safety by ensuring that variables only accept predefined values.

Key Concepts

  • Literal Types: These are types that represent specific values rather than general types.
  • String Literals: Types that can only be one of a set of string values.
  • Numeric Literals: Types that can only be one of a set of numeric values.
  • Boolean Literals: Types that can only be true or false.
  • Union Types: Combining multiple literal types to allow a variable to hold any one of several values.

String Literal Types

String literal types allow you to specify that a variable can only be one of a set of string values.

type Direction = 'North' | 'South' | 'East' | 'West';

let direction: Direction;

direction = 'North'; // Valid
direction = 'South'; // Valid
direction = 'Up';    // Error: Type '"Up"' is not assignable to type 'Direction'.

Example Explained

  • type Direction = 'North' | 'South' | 'East' | 'West';: Defines a type Direction that can only be one of the four specified strings.
  • let direction: Direction;: Declares a variable direction of type Direction.
  • Assigning direction to 'North' or 'South' is valid, but assigning it to 'Up' results in a type error.

Numeric Literal Types

Numeric literal types restrict a variable to specific numeric values.

type DiceRoll = 1 | 2 | 3 | 4 | 5 | 6;

let roll: DiceRoll;

roll = 1; // Valid
roll = 7; // Error: Type '7' is not assignable to type 'DiceRoll'.

Example Explained

  • type DiceRoll = 1 | 2 | 3 | 4 | 5 | 6;: Defines a type DiceRoll that can only be one of the six specified numbers.
  • let roll: DiceRoll;: Declares a variable roll of type DiceRoll.
  • Assigning roll to 1 is valid, but assigning it to 7 results in a type error.

Boolean Literal Types

Boolean literal types restrict a variable to either true or false.

type IsEnabled = true | false;

let enabled: IsEnabled;

enabled = true;  // Valid
enabled = false; // Valid
enabled = 'yes'; // Error: Type '"yes"' is not assignable to type 'IsEnabled'.

Example Explained

  • type IsEnabled = true | false;: Defines a type IsEnabled that can only be true or false.
  • let enabled: IsEnabled;: Declares a variable enabled of type IsEnabled.
  • Assigning enabled to true or false is valid, but assigning it to 'yes' results in a type error.

Union Types

Union types combine multiple literal types, allowing a variable to hold any one of several values.

type Status = 'success' | 'error' | 'loading';

let currentStatus: Status;

currentStatus = 'success'; // Valid
currentStatus = 'error';   // Valid
currentStatus = 'pending'; // Error: Type '"pending"' is not assignable to type 'Status'.

Example Explained

  • type Status = 'success' | 'error' | 'loading';: Defines a type Status that can only be one of the three specified strings.
  • let currentStatus: Status;: Declares a variable currentStatus of type Status.
  • Assigning currentStatus to 'success' or 'error' is valid, but assigning it to 'pending' results in a type error.

Conclusion

Literal types in TypeScript provide a way to specify exact values a variable can hold, enhancing type safety and reducing errors. By using string, numeric, and boolean literal types, you can create more precise and predictable code. Union types further extend this capability by allowing variables to accept multiple literal values, making your TypeScript code both flexible and robust.

Summary

  • Literal types restrict variables to specific values.
  • String, numeric, and boolean literals are common types.
  • Union types combine multiple literals for greater flexibility.
  • Using literal types improves type safety and reduces runtime errors.
