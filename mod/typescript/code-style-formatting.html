Introduction

Code style and formatting are crucial for maintaining readability, consistency, and collaboration in TypeScript projects. This section will cover best practices, tools, and guidelines to help you write clean and maintainable TypeScript code.

Key Concepts

  • Consistency: Ensuring that your code follows a consistent style throughout the project.
  • Readability: Writing code that is easy to read and understand.
  • Tooling: Using tools to automate code formatting and enforce style guidelines.

Best Practices

Naming Conventions

  • Variables and Functions: Use camelCase. 
    let userName: string;
function getUserInfo() {}
  • Classes and Interfaces: Use PascalCase. 
    class UserProfile {}
interface User {
  name: string;
  age: number;
}
  • Constants: Use UPPER_CASE with underscores. 
    const MAX_USERS = 100;

Indentation and Spacing

  • Use 2 or 4 spaces for indentation (choose one and stick to it).
  • Add spaces around operators and after commas. 
    let sum = a + b;
let numbers = [1, 2, 3];

Semicolons

  • Always use semicolons to terminate statements. 
    let user = 'John';
console.log(user);

Line Length

  • Keep lines under 80 or 100 characters to improve readability.

Braces and Blocks

  • Use braces for all control structures, even single-line blocks. 
    if (isValid) {
  console.log('Valid');
} else {
  console.log('Invalid');
}

Tooling

Prettier

Prettier is an opinionated code formatter that supports TypeScript. It enforces a consistent style by parsing your code and re-printing it with its own rules.

  • Installation: 
    npm install --save-dev prettier
  • Configuration: Create a .prettierrc file in your project root. 
    {
  "semi": true,
  "singleQuote": true,
  "trailingComma": "all",
  "tabWidth": 2
}
  • Usage: 
    npx prettier --write src/

ESLint

ESLint is a static code analysis tool for identifying problematic patterns in JavaScript and TypeScript code.

  • Installation: 
    npm install --save-dev eslint @typescript-eslint/parser @typescript-eslint/eslint-plugin
  • Configuration: Create a .eslintrc.json file. 
    {
  "parser": "@typescript-eslint/parser",
  "plugins": ["@typescript-eslint"],
  "extends": ["eslint:recommended", "plugin:@typescript-eslint/recommended"],
  "rules": {
    "semi": ["error", "always"],
    "quotes": ["error", "single"]
  }
}
  • Usage: 
    npx eslint src/**/*.ts

Example

Here is an example of a well-formatted TypeScript file:

// Importing necessary modules
import { User } from './models/User';

// Constants
const MAX_USERS = 100;

// Class definition
class UserProfile {
  private user: User;

  constructor(user: User) {
    this.user = user;
  }

  public getUserInfo(): string {
    return `Name: ${this.user.name}, Age: ${this.user.age}`;
  }
}

// Function to create a new user profile
function createUserProfile(name: string, age: number): UserProfile {
  const user: User = { name, age };
  return new UserProfile(user);
}

// Example usage
const userProfile = createUserProfile('John Doe', 30);
console.log(userProfile.getUserInfo());

Conclusion

Adhering to a consistent code style and formatting guidelines is essential for writing clean and maintainable TypeScript code. By following best practices and leveraging tools like Prettier and ESLint, you can ensure that your codebase remains readable and easy to work with, both for yourself and for others.
