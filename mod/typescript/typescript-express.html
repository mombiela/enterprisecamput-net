Introduction to TypeScript with Express

TypeScript is a powerful language that builds on JavaScript by adding static type definitions. Express is a minimal and flexible Node.js web application framework that provides a robust set of features for web and mobile applications. Combining TypeScript with Express allows developers to write more robust and maintainable code.

Setting Up TypeScript with Express

Before diving into coding, we need to set up our development environment.

Prerequisites

  • Node.js installed
  • npm (Node Package Manager) installed

Steps to Set Up

  1. Initialize a new Node.js project:

    mkdir typescript-express-app
cd typescript-express-app
npm init -y

  2. Install TypeScript and Express:

    npm install typescript ts-node @types/node @types/express express

  3. Create a tsconfig.json file:

    {
  "compilerOptions": {
    "target": "ES6",
    "module": "commonjs",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "outDir": "./dist"
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules"]
}

  4. Create the project structure:

    mkdir src
touch src/index.ts

Building a Basic Express Server with TypeScript

Let's create a simple Express server using TypeScript.

Writing the Server Code

  1. Import necessary modules:

    import express, { Request, Response } from 'express';

  2. Initialize the Express application:

    const app = express();
const port = 3000;

  3. Define a basic route:

    app.get('/', (req: Request, res: Response) => {
  res.send('Hello, TypeScript with Express!');
});

  4. Start the server:

    app.listen(port, () => {
  console.log(`Server is running at http://localhost:${port}`);
});

  5. Complete src/index.ts file:

    import express, { Request, Response } from 'express';

const app = express();
const port = 3000;

app.get('/', (req: Request, res: Response) => {
  res.send('Hello, TypeScript with Express!');
});

app.listen(port, () => {
  console.log(`Server is running at http://localhost:${port}`);
});

  6. Run the server:

    npx ts-node src/index.ts

Adding Middleware

Middleware functions are functions that have access to the request object (req), the response object (res), and the next middleware function in the applicationâ€™s request-response cycle.

Example Middleware

  1. Create a logging middleware:

    const loggerMiddleware = (req: Request, res: Response, next: Function) => {
  console.log(`${req.method} ${req.url}`);
  next();
};

  2. Use the middleware in the application:

    app.use(loggerMiddleware);

  3. Complete src/index.ts with middleware:

    import express, { Request, Response } from 'express';

const app = express();
const port = 3000;

const loggerMiddleware = (req: Request, res: Response, next: Function) => {
  console.log(`${req.method} ${req.url}`);
  next();
};

app.use(loggerMiddleware);

app.get('/', (req: Request, res: Response) => {
  res.send('Hello, TypeScript with Express!');
});

app.listen(port, () => {
  console.log(`Server is running at http://localhost:${port}`);
});

Advanced Topics

Type Definitions for Request and Response

TypeScript allows us to define custom types for request and response objects to ensure type safety.

  1. Define a custom type for request body:

    interface UserRequest extends Request {
  body: {
    name: string;
    age: number;
  };
}

  2. Use the custom type in a route:

    app.post('/user', (req: UserRequest, res: Response) => {
  const { name, age } = req.body;
  res.send(`User ${name} is ${age} years old`);
});

Error Handling Middleware

Error handling middleware is defined in the same way as other middleware, except it has four arguments instead of three: (err, req, res, next).

  1. Create an error handling middleware:

    const errorHandler = (err: Error, req: Request, res: Response, next: Function) => {
  console.error(err.message);
  res.status(500).send('Internal Server Error');
};

  2. Use the error handling middleware:

    app.use(errorHandler);

Using Async/Await with Express

Handling asynchronous operations is crucial in modern web applications.

  1. Create an async route handler:

    app.get('/data', async (req: Request, res: Response) => {
  try {
    const data = await fetchDataFromDatabase();
    res.json(data);
  } catch (error) {
    res.status(500).send('Error fetching data');
  }
});

  2. Mock fetchDataFromDatabase function:

    const fetchDataFromDatabase = async () => {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve({ message: 'Data fetched successfully' });
    }, 1000);
  });
};

Conclusion

Combining TypeScript with Express enhances the development experience by providing type safety and better tooling. This course covered setting up a TypeScript project with Express, creating a basic server, adding middleware, defining custom types, handling errors, and using async/await. With these skills, you can build robust and maintainable web applications.
