Introduction to Declaration Files

Declaration files in TypeScript are used to describe the shape of JavaScript code to the TypeScript compiler. They allow TypeScript to understand the types of objects, functions, and other entities in JavaScript libraries, enabling type checking and IntelliSense support.

  • Purpose: Provide type information about JavaScript code.
  • File Extension: .d.ts

When to Use Declaration Files

  • Using JavaScript Libraries: When using third-party JavaScript libraries that do not have built-in TypeScript support.
  • Interoperability: To ensure smooth interoperability between TypeScript and JavaScript codebases.
  • Type Safety: To enhance type safety and developer experience with better tooling support.

Creating Declaration Files

Basic Structure

A declaration file typically contains type declarations for variables, functions, classes, and modules. Here’s a basic example:

// myLibrary.d.ts
declare module 'myLibrary' {
  export function greet(name: string): void;
  export const version: string;
}

Declaring Variables and Functions

  • Variables: Use declare keyword to declare variables.
  • Functions: Use declare keyword to declare functions with their parameter and return types.
// variables.d.ts
declare const API_URL: string;
declare let isProduction: boolean;

// functions.d.ts
declare function add(a: number, b: number): number;
declare function log(message: string): void;

Declaring Classes

Classes can be declared with their properties and methods:

// classes.d.ts
declare class Person {
  name: string;
  age: number;
  constructor(name: string, age: number);
  greet(): void;
}

Declaring Modules

Modules can be declared to encapsulate types and values:

// modules.d.ts
declare module 'myModule' {
  export interface User {
    id: number;
    name: string;
  }
  export function getUser(id: number): User;
}

Advanced Declaration Files

Merging Declarations

TypeScript allows merging multiple declarations to extend existing types or modules:

// merging.d.ts
interface Window {
  customProperty: string;
}

declare module 'myLibrary' {
  export function newFunction(): void;
}

Global Declarations

Global declarations are used to define types that are available globally across the project:

// globals.d.ts
declare global {
  interface String {
    toTitleCase(): string;
  }
}

Using `@types` Packages

Many popular JavaScript libraries have type definitions available via the @types scope on npm:

npm install @types/lodash

These can be used directly without needing to write your own declaration files.

Practical Examples

Example: Using a Declaration File for a JavaScript Library

Suppose you are using a JavaScript library called awesomeLib:

// awesomeLib.d.ts
declare module 'awesomeLib' {
  export function awesomeFunction(param: string): number;
  export interface AwesomeInterface {
    id: number;
    name: string;
  }
}

In your TypeScript code, you can now use awesomeLib with type safety:

import { awesomeFunction, AwesomeInterface } from 'awesomeLib';

const result: number = awesomeFunction('test');
const obj: AwesomeInterface = { id: 1, name: 'Awesome' };

Example: Extending a Global Object

Extending the Window object to include a custom property:

// globalExtensions.d.ts
declare global {
  interface Window {
    customProperty: string;
  }
}

// usage.ts
window.customProperty = 'Hello, World!';
console.log(window.customProperty);

Summary

Declaration files are a powerful feature in TypeScript that provide type information for JavaScript code, enhancing type safety and developer experience. They are essential when working with JavaScript libraries and for ensuring smooth interoperability between TypeScript and JavaScript. By understanding how to create and use declaration files, you can leverage the full potential of TypeScript in your projects.
