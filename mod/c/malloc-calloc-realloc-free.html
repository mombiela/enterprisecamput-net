Dynamic Memory Management in C: malloc, calloc, realloc, and free
Dynamic Arrays ►

Dynamic memory management is a crucial aspect of programming in C. It allows you to allocate memory at runtime, which is essential for creating flexible and efficient programs. In this section, we will cover the following functions: malloc, calloc, realloc, and free.

malloc

malloc stands for "memory allocation." It allocates a specified number of bytes and returns a pointer to the first byte of the allocated memory. The memory is not initialized, meaning it contains garbage values.

Syntax

void* malloc(size_t size);

Example

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr;
    int n = 5;

    // Allocating memory for 5 integers
    ptr = (int*) malloc(n * sizeof(int));

    if (ptr == NULL) {
        printf("Memory allocation failed\n");
        return 1;
    }

    // Initializing and displaying the allocated memory
    for (int i = 0; i < n; i++) {
        ptr[i] = i + 1;
        printf("%d ", ptr[i]);
    }

    // Freeing the allocated memory
    free(ptr);

    return 0;
}

calloc

calloc stands for "contiguous allocation." It allocates memory for an array of elements, initializes the memory to zero, and then returns a pointer to the memory.

Syntax

void* calloc(size_t num, size_t size);

Example

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr;
    int n = 5;

    // Allocating memory for 5 integers and initializing to zero
    ptr = (int*) calloc(n, sizeof(int));

    if (ptr == NULL) {
        printf("Memory allocation failed\n");
        return 1;
    }

    // Displaying the allocated memory
    for (int i = 0; i < n; i++) {
        printf("%d ", ptr[i]);
    }

    // Freeing the allocated memory
    free(ptr);

    return 0;
}

realloc

realloc stands for "reallocation." It changes the size of previously allocated memory. If the new size is larger, the new memory is uninitialized. If the new size is smaller, the excess memory is freed.

Syntax

void* realloc(void* ptr, size_t size);

Example

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr;
    int n = 5;

    // Allocating memory for 5 integers
    ptr = (int*) malloc(n * sizeof(int));

    if (ptr == NULL) {
        printf("Memory allocation failed\n");
        return 1;
    }

    // Initializing the allocated memory
    for (int i = 0; i < n; i++) {
        ptr[i] = i + 1;
    }

    // Reallocating memory for 10 integers
    ptr = (int*) realloc(ptr, 10 * sizeof(int));

    if (ptr == NULL) {
        printf("Memory reallocation failed\n");
        return 1;
    }

    // Initializing the new memory
    for (int i = n; i < 10; i++) {
        ptr[i] = i + 1;
    }

    // Displaying the reallocated memory
    for (int i = 0; i < 10; i++) {
        printf("%d ", ptr[i]);
    }

    // Freeing the allocated memory
    free(ptr);

    return 0;
}

free

free deallocates the memory previously allocated by malloc, calloc, or realloc. It does not change the value of the pointer itself, but the memory it points to is released.

Syntax

void free(void* ptr);

Example

#include <stdio.h>
#include <stdlib.h>

int main() {
    int *ptr;
    int n = 5;

    // Allocating memory for 5 integers
    ptr = (int*) malloc(n * sizeof(int));

    if (ptr == NULL) {
        printf("Memory allocation failed\n");
        return 1;
    }

    // Initializing and displaying the allocated memory
    for (int i = 0; i < n; i++) {
        ptr[i] = i + 1;
        printf("%d ", ptr[i]);
    }

    // Freeing the allocated memory
    free(ptr);

    return 0;
}

Summary

  • malloc: Allocates uninitialized memory.
  • calloc: Allocates zero-initialized memory.
  • realloc: Changes the size of previously allocated memory.
  • free: Deallocates previously allocated memory.

Understanding these functions is essential for effective memory management in C. Proper use of dynamic memory allocation can lead to more flexible and efficient programs, but it also requires careful handling to avoid memory leaks and other issues.
