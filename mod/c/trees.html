Introduction to Trees

Trees are a fundamental data structure in computer science, used to represent hierarchical relationships. They are particularly useful for organizing data that has a natural hierarchy, such as file systems, organizational structures, and more.

Key Concepts

  • Node: The basic unit of a tree, containing data and references to other nodes.
  • Root: The topmost node in a tree.
  • Child: A node directly connected to another node when moving away from the root.
  • Parent: The converse notion of a child.
  • Leaf: A node with no children.
  • Subtree: A tree formed by a node and its descendants.
  • Binary Tree: A tree where each node has at most two children.
  • Binary Search Tree (BST): A binary tree where the left child contains only nodes with values less than the parent node, and the right child only nodes with values greater than the parent node.

Basic Tree Structure in C

Defining a Node

In C, a tree node can be defined using a struct. Here is an example of a basic node structure for a binary tree:

#include <stdio.h>
#include <stdlib.h>

// Define the structure for a tree node
struct Node {
    int data;
    struct Node* left;
    struct Node* right;
};

// Function to create a new node
struct Node* createNode(int data) {
    struct Node* newNode = (struct Node*)malloc(sizeof(struct Node));
    newNode->data = data;
    newNode->left = NULL;
    newNode->right = NULL;
    return newNode;
}

Creating a Simple Tree

Let's create a simple binary tree with three nodes:

int main() {
    // Create the root node
    struct Node* root = createNode(1);
    
    // Create left and right children
    root->left = createNode(2);
    root->right = createNode(3);
    
    // Print the root and its children
    printf("Root: %d\n", root->data);
    printf("Left Child: %d\n", root->left->data);
    printf("Right Child: %d\n", root->right->data);
    
    return 0;
}

Binary Search Tree (BST)

Insertion in BST

Inserting a node in a BST involves placing the new node in the correct position to maintain the BST properties.

struct Node* insert(struct Node* node, int data) {
    // If the tree is empty, return a new node
    if (node == NULL) return createNode(data);

    // Otherwise, recur down the tree
    if (data < node->data)
        node->left = insert(node->left, data);
    else if (data > node->data)
        node->right = insert(node->right, data);

    // Return the (unchanged) node pointer
    return node;
}

Searching in BST

Searching for a value in a BST involves comparing the target value with the current node's value and deciding whether to go left or right.

struct Node* search(struct Node* root, int data) {
    // Base Cases: root is null or data is present at root
    if (root == NULL || root->data == data)
       return root;
    
    // Data is greater than root's data
    if (root->data < data)
       return search(root->right, data);

    // Data is smaller than root's data
    return search(root->left, data);
}

Tree Traversals

Tree traversal refers to the process of visiting all the nodes in a tree in a specific order.

Inorder Traversal

Inorder traversal visits nodes in the following order: left subtree, root, right subtree.

void inorderTraversal(struct Node* root) {
    if (root != NULL) {
        inorderTraversal(root->left);
        printf("%d ", root->data);
        inorderTraversal(root->right);
    }
}

Preorder Traversal

Preorder traversal visits nodes in the following order: root, left subtree, right subtree.

void preorderTraversal(struct Node* root) {
    if (root != NULL) {
        printf("%d ", root->data);
        preorderTraversal(root->left);
        preorderTraversal(root->right);
    }
}

Postorder Traversal

Postorder traversal visits nodes in the following order: left subtree, right subtree, root.

void postorderTraversal(struct Node* root) {
    if (root != NULL) {
        postorderTraversal(root->left);
        postorderTraversal(root->right);
        printf("%d ", root->data);
    }
}

Advanced Topics

Balancing Trees

Balancing a tree ensures that the tree remains efficient for operations like insertion, deletion, and search. Common balanced trees include AVL trees and Red-Black trees.

AVL Trees

An AVL tree is a self-balancing binary search tree where the difference between the heights of left and right subtrees cannot be more than one for all nodes.

Red-Black Trees

A Red-Black tree is a kind of self-balancing binary search tree where each node contains an extra bit for denoting the color of the node, either red or black.

Conclusion

Trees are a versatile and essential data structure in computer science. Understanding the basics of tree structures, binary search trees, and tree traversals is crucial for solving many complex problems efficiently. As you advance, exploring balanced trees like AVL and Red-Black trees will further enhance your ability to manage hierarchical data effectively.
