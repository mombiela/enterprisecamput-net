Preprocessor Directives in C
Macros and Inline Functions ►

Preprocessor directives are commands that give instructions to the compiler to preprocess the source code before actual compilation starts. They are a crucial part of the C programming language, enabling conditional compilation, macro definitions, file inclusion, and more.

Key Concepts

  • Preprocessor Directives: Instructions that are executed before the actual compilation of code.
  • Syntax: Preprocessor directives start with a # symbol and do not end with a semicolon.
  • Common Directives: #include, #define, #undef, #ifdef, #ifndef, #if, #else, #elif, #endif, #error, #pragma.

Common Preprocessor Directives

`#include`

The #include directive is used to include the contents of a file or library in the program.

#include <stdio.h>  // Includes the standard input-output library
#include "myheader.h"  // Includes a user-defined header file
  • Angle Brackets < >: Used for standard library files.
  • Double Quotes " ": Used for user-defined files.

`#define`

The #define directive is used to create macro definitions. It can define constants or macros that can take arguments.

#define PI 3.14159  // Defines a constant
#define SQUARE(x) ((x) * (x))  // Defines a macro that calculates the square of a number

`#undef`

The #undef directive is used to undefine a macro.

#undef PI  // Undefines the macro PI

Conditional Compilation

Conditional compilation allows parts of the code to be compiled or ignored based on certain conditions.

#ifdef and #ifndef

#ifdef DEBUG
    printf("Debug mode is on.\n");
#endif

#ifndef RELEASE
    printf("Release mode is off.\n");
#endif
  • #ifdef: Compiles the code if the macro is defined.
  • #ifndef: Compiles the code if the macro is not defined.

#if, #else, #elif, and #endif

#define VERSION 2

#if VERSION == 1
    printf("Version 1\n");
#elif VERSION == 2
    printf("Version 2\n");
#else
    printf("Unknown version\n");
#endif
  • #if: Starts a conditional block.
  • #else: Provides an alternative if the #if condition is false.
  • #elif: Else if, provides another condition to check.
  • #endif: Ends the conditional block.

`#error`

The #error directive generates a compilation error with a specified message.

#if VERSION > 3
    #error "Version number too high!"
#endif

`#pragma`

The #pragma directive is used to provide additional information to the compiler. Its behavior is compiler-specific.

#pragma message("Compiling the code...")

Examples and Exercises

Example 1: Using `#include` and `#define`

#include <stdio.h>
#define GREETING "Hello, World!"

int main() {
    printf("%s\n", GREETING);
    return 0;
}

Example 2: Conditional Compilation

#include <stdio.h>
#define DEBUG

int main() {
#ifdef DEBUG
    printf("Debugging is enabled.\n");
#endif
    printf("Program is running.\n");
    return 0;
}

Exercise 1

Write a program that uses #define to create a macro for calculating the area of a circle. Use the macro in the main function to calculate and print the area of a circle with a radius of 5.

Exercise 2

Modify the previous program to include conditional compilation. Define a macro DEBUG and use #ifdef to print additional debugging information if DEBUG is defined.

Summary

Preprocessor directives in C are powerful tools that allow you to control the compilation process. They enable you to include files, define constants and macros, and conditionally compile code. Understanding and using these directives effectively can greatly enhance the flexibility and maintainability of your C programs.
