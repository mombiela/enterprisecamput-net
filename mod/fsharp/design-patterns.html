Introduction to Design Patterns

Design patterns are typical solutions to common problems in software design. They are like blueprints that you can customize to solve a particular design problem in your code. In this section, we'll explore how to implement some of the most common design patterns in F#.

Creational Patterns

Singleton Pattern

The Singleton pattern ensures a class has only one instance and provides a global point of access to it.

Example:

type Singleton private () =
    static let instance = Singleton()
    static member Instance = instance

// Usage
let singletonInstance = Singleton.Instance

Factory Method Pattern

The Factory Method pattern defines an interface for creating an object but lets subclasses alter the type of objects that will be created.

Example:

type IProduct =
    abstract member Operation: unit -> string

type ConcreteProductA() =
    interface IProduct with
        member this.Operation() = "Result of ConcreteProductA"

type ConcreteProductB() =
    interface IProduct with
        member this.Operation() = "Result of ConcreteProductB"

type Creator() =
    member this.FactoryMethod(productType: string) : IProduct =
        match productType with
        | "A" -> ConcreteProductA() :> IProduct
        | "B" -> ConcreteProductB() :> IProduct
        | _ -> failwith "Unknown product type"

// Usage
let creator = Creator()
let productA = creator.FactoryMethod("A")
printfn "%s" (productA.Operation())

Structural Patterns

Adapter Pattern

The Adapter pattern allows incompatible interfaces to work together. It acts as a bridge between two incompatible interfaces.

Example:

type ITarget =
    abstract member Request: unit -> string

type Adaptee() =
    member this.SpecificRequest() = "Specific request."

type Adapter(adaptee: Adaptee) =
    interface ITarget with
        member this.Request() = adaptee.SpecificRequest()

// Usage
let adaptee = Adaptee()
let adapter = Adapter(adaptee)
printfn "%s" (adapter :> ITarget).Request()

Decorator Pattern

The Decorator pattern allows behavior to be added to an individual object, dynamically, without affecting the behavior of other objects from the same class.

Example:

type IComponent =
    abstract member Operation: unit -> string

type ConcreteComponent() =
    interface IComponent with
        member this.Operation() = "ConcreteComponent"

type Decorator(component: IComponent) =
    interface IComponent with
        member this.Operation() = sprintf "Decorator(%s)" (component.Operation())

// Usage
let component = ConcreteComponent()
let decorator = Decorator(component)
printfn "%s" (decorator :> IComponent).Operation()

Behavioral Patterns

Observer Pattern

The Observer pattern defines a one-to-many dependency between objects so that when one object changes state, all its dependents are notified and updated automatically.

Example:

type IObserver =
    abstract member Update: string -> unit

type Subject() =
    let observers = System.Collections.Generic.List<IObserver>()
    member this.Attach(observer: IObserver) = observers.Add(observer)
    member this.Detach(observer: IObserver) = observers.Remove(observer)
    member this.Notify(message: string) =
        for observer in observers do
            observer.Update(message)

type ConcreteObserver(name: string) =
    interface IObserver with
        member this.Update(message: string) =
            printfn "Observer %s received message: %s" name message

// Usage
let subject = Subject()
let observer1 = ConcreteObserver("A")
let observer2 = ConcreteObserver("B")

subject.Attach(observer1)
subject.Attach(observer2)

subject.Notify("Hello Observers!")

Strategy Pattern

The Strategy pattern defines a family of algorithms, encapsulates each one, and makes them interchangeable. Strategy lets the algorithm vary independently from clients that use it.

Example:

type IStrategy =
    abstract member Execute: int -> int -> int

type ConcreteStrategyAdd() =
    interface IStrategy with
        member this.Execute(a: int, b: int) = a + b

type ConcreteStrategySubtract() =
    interface IStrategy with
        member this.Execute(a: int, b: int) = a - b

type Context(strategy: IStrategy) =
    member this.ExecuteStrategy(a: int, b: int) = strategy.Execute(a, b)

// Usage
let contextAdd = Context(ConcreteStrategyAdd())
let contextSubtract = Context(ConcreteStrategySubtract())

printfn "Add: %d" (contextAdd.ExecuteStrategy(3, 4))
printfn "Subtract: %d" (contextSubtract.ExecuteStrategy(3, 4))

Conclusion

Design patterns are powerful tools in a programmer's toolkit. They provide proven solutions to common problems and can significantly improve the structure and maintainability of your code. By understanding and applying these patterns in F#, you can write more robust and scalable applications.
