Higher-Order Functions in F#
Currying and Partial Application ►

Higher-order functions are a fundamental concept in functional programming, and F# leverages them to create powerful and expressive code. This section will guide you from understanding the basics to mastering advanced techniques involving higher-order functions.

What are Higher-Order Functions?

Higher-order functions are functions that can take other functions as arguments and/or return functions as their result. This allows for a high degree of flexibility and reusability in your code.

Key Concepts

  • Function as Argument: A higher-order function can accept other functions as parameters.
  • Function as Return Value: A higher-order function can return a function as its result.
  • Function Composition: Combining simple functions to build more complex ones.

Basic Examples

Let's start with some simple examples to illustrate these concepts.

Function as Argument

Here is a basic example where a function is passed as an argument to another function:

let applyFunction f x = f x

let square n = n * n

let result = applyFunction square 5
printfn "%d" result  // Output: 25

In this example:

  • applyFunction is a higher-order function that takes a function f and a value x, then applies f to x.
  • square is a simple function that squares its input.
  • applyFunction square 5 applies the square function to 5, resulting in 25.

Function as Return Value

Now, let's see an example where a function returns another function:

let add x = 
    let innerAdd y = x + y
    innerAdd

let addFive = add 5
let result = addFive 10
printfn "%d" result  // Output: 15

In this example:

  • add is a higher-order function that takes a value x and returns a new function innerAdd.
  • innerAdd takes another value y and returns the sum of x and y.
  • add 5 returns a function that adds 5 to its input.
  • addFive 10 applies this returned function to 10, resulting in 15.

Advanced Techniques

Function Composition

Function composition is a powerful technique where you combine multiple functions to create a new function. F# provides the >> and << operators for this purpose.

let addOne x = x + 1
let multiplyByTwo x = x * 2

let addOneThenMultiplyByTwo = addOne >> multiplyByTwo
let result = addOneThenMultiplyByTwo 3
printfn "%d" result  // Output: 8

In this example:

  • addOne adds 1 to its input.
  • multiplyByTwo multiplies its input by 2.
  • addOne >> multiplyByTwo creates a new function that first adds 1 and then multiplies the result by 2.
  • Applying this composed function to 3 results in 8.

Currying and Partial Application

Currying is the process of transforming a function that takes multiple arguments into a series of functions that each take a single argument. Partial application is the process of fixing a few arguments of a function and producing another function.

let add x y = x + y

let addFive = add 5
let result = addFive 10
printfn "%d" result  // Output: 15

In this example:

  • add is a curried function that takes two arguments x and y.
  • add 5 partially applies the function by fixing the first argument to 5, resulting in a new function that adds 5 to its input.
  • addFive 10 applies this new function to 10, resulting in 15.

Summary

Higher-order functions are a powerful feature in F# that allow you to:

  • Pass functions as arguments.
  • Return functions as results.
  • Compose simple functions to create more complex ones.
  • Use currying and partial application to create specialized functions.

By mastering higher-order functions, you can write more flexible, reusable, and expressive code in F#. Practice these concepts with various examples to deepen your understanding and improve your functional programming skills.
