Pattern Matching in F#
Higher-Order Functions ►

Pattern matching is a powerful feature in F# that allows you to check a value against a pattern and deconstruct data in a concise and readable way. This topic will guide you from the basics to advanced usage of pattern matching in F#.

Introduction to Pattern Matching

Pattern matching in F# is used to match values against patterns and execute code based on the match. It is similar to switch-case statements in other languages but more powerful and expressive.

Basic Pattern Matching

In its simplest form, pattern matching can be used to match literal values.

let describeNumber x =
    match x with
    | 1 -> "One"
    | 2 -> "Two"
    | 3 -> "Three"
    | _ -> "Unknown"

printfn "%s" (describeNumber 2)  // Output: Two
  • match x with starts the pattern matching.
  • | separates different patterns.
  • _ is a wildcard pattern that matches any value not matched by previous patterns.

Pattern Matching with Tuples

Pattern matching can also be used with tuples to deconstruct values.

let addTuples (a, b) (c, d) =
    match (a, b, c, d) with
    | (0, 0, 0, 0) -> "All zeros"
    | (x, y, 0, 0) -> sprintf "First tuple: (%d, %d)" x y
    | (0, 0, x, y) -> sprintf "Second tuple: (%d, %d)" x y
    | _ -> "Mixed values"

printfn "%s" (addTuples (1, 2) (0, 0))  // Output: First tuple: (1, 2)
  • Tuples are deconstructed in the pattern matching.
  • Specific patterns can be matched to handle different cases.

Pattern Matching with Lists

Lists can also be pattern matched to handle different structures.

let rec sumList lst =
    match lst with
    | [] -> 0
    | head :: tail -> head + sumList tail

printfn "%d" (sumList [1; 2; 3; 4])  // Output: 10
  • [] matches an empty list.
  • head :: tail matches a non-empty list, where head is the first element and tail is the rest of the list.

Pattern Matching with Records and Discriminated Unions

Pattern matching is particularly powerful when used with records and discriminated unions.

Records

type Person = { Name: string; Age: int }

let describePerson person =
    match person with
    | { Name = "Alice"; Age = 30 } -> "Alice is 30 years old"
    | { Name = name; Age = age } -> sprintf "%s is %d years old" name age

let alice = { Name = "Alice"; Age = 30 }
printfn "%s" (describePerson alice)  // Output: Alice is 30 years old

Discriminated Unions

type Shape =
    | Circle of radius: float
    | Rectangle of width: float * height: float

let area shape =
    match shape with
    | Circle radius -> System.Math.PI * radius * radius
    | Rectangle (width, height) -> width * height

let myCircle = Circle 5.0
printfn "%f" (area myCircle)  // Output: 78.539816

Advanced Pattern Matching

Active Patterns

Active patterns allow you to define custom pattern matching logic.

let (|Even|Odd|) x =
    if x % 2 = 0 then Even else Odd

let describeNumber x =
    match x with
    | Even -> "Even"
    | Odd -> "Odd"

printfn "%s" (describeNumber 4)  // Output: Even

Partial Active Patterns

Partial active patterns can return an option type, allowing for more complex matching.

let (|Positive|_|) x =
    if x > 0 then Some Positive else None

let describeNumber x =
    match x with
    | Positive -> "Positive"
    | _ -> "Non-positive"

printfn "%s" (describeNumber 5)  // Output: Positive

Conclusion

Pattern matching in F# is a versatile and powerful feature that simplifies the handling of complex data structures. From matching simple values to deconstructing tuples, lists, records, and discriminated unions, pattern matching enhances code readability and maintainability. Advanced techniques like active patterns further extend its capabilities, making F# a robust language for functional programming.

By mastering pattern matching, you can write more expressive and concise code, making your F# programming more effective and enjoyable.
