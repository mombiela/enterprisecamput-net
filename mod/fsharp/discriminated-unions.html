Discriminated Unions in F#
Option Types and Error Handling ►

Discriminated Unions (DUs) are a powerful feature in F# that allow you to define a type that can hold one of several different, but fixed, types of values. They are particularly useful for modeling data that can take on multiple forms. In this topic, we will explore what Discriminated Unions are, how to define and use them, and some advanced techniques.

What are Discriminated Unions?

Discriminated Unions are a way to define a type that can be one of several named cases, each potentially with different types and values. They are similar to enums in other languages but are more powerful because each case can carry additional data.

Key Concepts

  • Case: Each possible value of the union type.
  • Tag: The name of the case.
  • Payload: The data associated with a case (optional).

Defining Discriminated Unions

To define a Discriminated Union in F#, you use the type keyword followed by the union name and the cases.

type Shape =
    | Circle of float
    | Rectangle of float * float
    | Triangle of float * float * float

In this example:

  • Shape is the union type.
  • Circle, Rectangle, and Triangle are the cases.
  • Each case has associated data (e.g., Circle has a float representing the radius).

Using Discriminated Unions

Once defined, you can create instances of a Discriminated Union and use pattern matching to work with them.

Creating Instances

let myCircle = Circle(5.0)
let myRectangle = Rectangle(4.0, 6.0)
let myTriangle = Triangle(3.0, 4.0, 5.0)

Pattern Matching

Pattern matching allows you to deconstruct and work with the values of a Discriminated Union.

let describeShape shape =
    match shape with
    | Circle(radius) -> sprintf "A circle with radius %f" radius
    | Rectangle(width, height) -> sprintf "A rectangle with width %f and height %f" width height
    | Triangle(a, b, c) -> sprintf "A triangle with sides %f, %f, %f" a b c

let description = describeShape myCircle
printfn "%s" description

Advanced Techniques

Nested Discriminated Unions

You can nest Discriminated Unions within other Discriminated Unions to model more complex data structures.

type Point = { X: float; Y: float }

type Shape =
    | Circle of float
    | Rectangle of float * float
    | Triangle of float * float * float
    | Polygon of Point list

Using Option Types

Discriminated Unions are often used with the Option type to represent values that may or may not be present.

type Result =
    | Success of string
    | Error of string

let processResult result =
    match result with
    | Success(message) -> printfn "Success: %s" message
    | Error(errorMessage) -> printfn "Error: %s" errorMessage

let result = Success("Operation completed successfully")
processResult result

Recursive Discriminated Unions

Discriminated Unions can be recursive, which is useful for representing tree-like structures.

type BinaryTree =
    | Leaf of int
    | Node of BinaryTree * int * BinaryTree

let rec sumTree tree =
    match tree with
    | Leaf(value) -> value
    | Node(left, value, right) -> sumTree(left) + value + sumTree(right)

let myTree = Node(Node(Leaf(1), 2, Leaf(3)), 4, Leaf(5))
let total = sumTree myTree
printfn "Sum of tree: %d" total

Conclusion

Discriminated Unions are a versatile and powerful feature in F# that allow you to define types with multiple, distinct cases. They are useful for modeling data that can take on different forms and for implementing pattern matching. By understanding and utilizing Discriminated Unions, you can write more expressive and robust F# code.
