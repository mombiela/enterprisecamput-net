Redis Streams
RedisGraph ►

Redis Streams is a powerful data structure introduced in Redis 5.0, designed for managing and processing streams of data. This section will cover the basics to advanced usage of Redis Streams, including key concepts, commands, and practical examples.

Introduction to Redis Streams

Redis Streams is a log data structure that allows you to store multiple fields and string values with an automatic, time-based sequence. It is particularly useful for real-time data processing, event sourcing, and messaging systems.

Key Concepts

  • Stream: A sequence of entries, each with a unique ID.
  • Entry: A single record in the stream, consisting of a unique ID and one or more field-value pairs.
  • Consumer Group: A group of consumers that can read from the stream in a coordinated way.
  • Consumer: A single client that reads entries from the stream.

Basic Commands

Adding Entries to a Stream

To add an entry to a stream, use the XADD command. The basic syntax is:

XADD stream_name * field1 value1 field2 value2 ...

Example:

XADD mystream * sensor-id 1234 temperature 19.8

Reading Entries from a Stream

To read entries from a stream, use the XRANGE command. The basic syntax is:

XRANGE stream_name start end [COUNT count]

Example:

XRANGE mystream - +

This command reads all entries from the stream mystream.

Reading New Entries

To read new entries added to the stream, use the XREAD command. The basic syntax is:

XREAD COUNT count STREAMS stream_name last_id

Example:

XREAD COUNT 2 STREAMS mystream $

This command reads the next two new entries from mystream.

Advanced Usage

Consumer Groups

Consumer groups allow multiple consumers to read from the same stream without duplicating work. Use the XGROUP command to manage consumer groups.

Creating a Consumer Group

XGROUP CREATE stream_name group_name $ MKSTREAM

Example:

XGROUP CREATE mystream mygroup $ MKSTREAM

Reading Entries with Consumer Groups

Use the XREADGROUP command to read entries as part of a consumer group.

XREADGROUP GROUP group_name consumer_name COUNT count STREAMS stream_name last_id

Example:

XREADGROUP GROUP mygroup consumer1 COUNT 2 STREAMS mystream >

Acknowledging Entries

Consumers must acknowledge entries they have processed using the XACK command.

XACK stream_name group_name entry_id

Example:

XACK mystream mygroup 1526569495631-0

Managing Pending Entries

To view pending entries, use the XPENDING command.

XPENDING stream_name group_name [start end count [consumer]]

Example:

XPENDING mystream mygroup

Practical Examples

Example: Simple Producer-Consumer

Producer:

XADD mystream * sensor-id 1234 temperature 19.8
XADD mystream * sensor-id 1235 temperature 20.1

Consumer:

XREAD COUNT 2 STREAMS mystream 0

Example: Using Consumer Groups

Create a consumer group:

XGROUP CREATE mystream mygroup $ MKSTREAM

Producer:

XADD mystream * sensor-id 1234 temperature 19.8
XADD mystream * sensor-id 1235 temperature 20.1

Consumer 1:

XREADGROUP GROUP mygroup consumer1 COUNT 2 STREAMS mystream >

Consumer 2:

XREADGROUP GROUP mygroup consumer2 COUNT 2 STREAMS mystream >

Acknowledge processed entries:

XACK mystream mygroup 1526569495631-0

Summary

Redis Streams is a versatile data structure for handling real-time data streams. It supports adding and reading entries, managing consumer groups, and acknowledging processed entries. By understanding and utilizing these commands, you can efficiently manage and process streams of data in your applications.
