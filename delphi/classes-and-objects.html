Introduction to Classes and Objects

Classes and objects are fundamental concepts in Object-Oriented Programming (OOP). In Delphi/Object Pascal, these concepts allow you to create complex data structures and functionalities in a modular and reusable way.

Key Concepts

  • Class: A blueprint for creating objects. It defines a type of object according to the attributes (fields) and behaviors (methods) it should have.
  • Object: An instance of a class. It represents a specific implementation of the class with actual values and behaviors.

Defining a Class

To define a class in Delphi/Object Pascal, you use the class keyword. Here is a basic example:

type
  TPerson = class
  private
    FName: string;
    FAge: Integer;
  public
    procedure SetName(AName: string);
    function GetName: string;
    procedure SetAge(AAge: Integer);
    function GetAge: Integer;
  end;

Explanation

  • type: Declares a new type.
  • TPerson: The name of the class.
  • private: Section where private fields are declared. These fields are only accessible within the class itself.
  • public: Section where public methods are declared. These methods can be accessed from outside the class.

Creating and Using Objects

To create an object, you use the Create method of the class. Here is an example of how to create and use an object of the TPerson class:

var
  Person: TPerson;
begin
  Person := TPerson.Create;
  Person.SetName('John Doe');
  Person.SetAge(30);
  WriteLn('Name: ', Person.GetName);
  WriteLn('Age: ', Person.GetAge);
  Person.Free;
end;

Explanation

  • Person: A variable of type TPerson.
  • TPerson.Create: Creates an instance of the TPerson class.
  • SetName and SetAge: Methods to set the name and age of the person.
  • GetName and GetAge: Methods to get the name and age of the person.
  • Free: Frees the memory allocated for the object.

Constructors and Destructors

Constructors and destructors are special methods used to initialize and clean up objects.

Constructor Example

type
  TPerson = class
  private
    FName: string;
    FAge: Integer;
  public
    constructor Create(AName: string; AAge: Integer);
    destructor Destroy; override;
    procedure DisplayInfo;
  end;

constructor TPerson.Create(AName: string; AAge: Integer);
begin
  FName := AName;
  FAge := AAge;
end;

destructor TPerson.Destroy;
begin
  // Clean up code here
  inherited Destroy;
end;

procedure TPerson.DisplayInfo;
begin
  WriteLn('Name: ', FName);
  WriteLn('Age: ', FAge);
end;

Using the Constructor

var
  Person: TPerson;
begin
  Person := TPerson.Create('Jane Doe', 25);
  Person.DisplayInfo;
  Person.Free;
end;

Explanation

  • constructor Create: Initializes the object with specific values.
  • destructor Destroy: Cleans up the object before it is destroyed.
  • override: Indicates that the method overrides a method in the base class.

Inheritance

Inheritance allows a class to inherit fields and methods from another class.

Example of Inheritance

type
  TEmployee = class(TPerson)
  private
    FEmployeeID: Integer;
  public
    constructor Create(AName: string; AAge: Integer; AEmployeeID: Integer);
    procedure DisplayInfo; override;
  end;

constructor TEmployee.Create(AName: string; AAge: Integer; AEmployeeID: Integer);
begin
  inherited Create(AName, AAge);
  FEmployeeID := AEmployeeID;
end;

procedure TEmployee.DisplayInfo;
begin
  inherited DisplayInfo;
  WriteLn('Employee ID: ', FEmployeeID);
end;

Using the Inherited Class

var
  Employee: TEmployee;
begin
  Employee := TEmployee.Create('John Smith', 40, 12345);
  Employee.DisplayInfo;
  Employee.Free;
end;

Explanation

  • TEmployee = class(TPerson): TEmployee inherits from TPerson.
  • inherited Create: Calls the constructor of the base class.
  • override: Indicates that the method overrides a method in the base class.

Polymorphism

Polymorphism allows methods to be called on objects of different classes through a common interface.

Example of Polymorphism

type
  TPerson = class
  public
    procedure Speak; virtual;
  end;

  TEmployee = class(TPerson)
  public
    procedure Speak; override;
  end;

procedure TPerson.Speak;
begin
  WriteLn('Person speaking');
end;

procedure TEmployee.Speak;
begin
  WriteLn('Employee speaking');
end;

var
  Person: TPerson;
  Employee: TEmployee;
begin
  Person := TPerson.Create;
  Employee := TEmployee.Create;
  Person.Speak; // Outputs: Person speaking
  Employee.Speak; // Outputs: Employee speaking
  Person := Employee;
  Person.Speak; // Outputs: Employee speaking
  Person.Free;
  Employee.Free;
end;

Explanation

  • virtual: Indicates that the method can be overridden in a derived class.
  • override: Indicates that the method overrides a method in the base class.
  • Person := Employee: Demonstrates polymorphism by assigning an Employee object to a Person variable.

Conclusion

Classes and objects are powerful tools in Delphi/Object Pascal that enable you to create modular, reusable, and maintainable code. By understanding and utilizing concepts such as constructors, destructors, inheritance, and polymorphism, you can build complex applications with ease. Practice these concepts with various examples to solidify your understanding and become proficient in Object-Oriented Programming with Delphi/Object Pascal.
