Overview

Object-Oriented Programming (OOP) is a paradigm that uses "objects" to design applications and computer programs. Delphi, which uses the Object Pascal language, is a powerful tool for implementing OOP concepts. This section will introduce you to the basics of OOP in Delphi, covering key concepts, syntax, and practical examples.

Key Concepts of OOP

  • Class: A blueprint for creating objects (a particular data structure), containing methods and properties.
  • Object: An instance of a class.
  • Encapsulation: The bundling of data with the methods that operate on that data.
  • Inheritance: The mechanism by which one class can inherit traits from another class.
  • Polymorphism: The ability to present the same interface for different underlying forms (data types).

Classes and Objects

Defining a Class

In Delphi, a class is defined using the class keyword. Here is a simple example:

type
  TPerson = class
  private
    FName: string;
    FAge: Integer;
  public
    procedure SetName(AName: string);
    function GetName: string;
    procedure SetAge(AAge: Integer);
    function GetAge: Integer;
  end;

Creating an Object

To create an object, you use the Create method of the class:

var
  Person: TPerson;
begin
  Person := TPerson.Create;
  Person.SetName('John Doe');
  Person.SetAge(30);
  ShowMessage(Person.GetName + ' is ' + IntToStr(Person.GetAge) + ' years old.');
  Person.Free;
end;

Encapsulation

Encapsulation is achieved by using access specifiers like private, protected, and public. In the above example, FName and FAge are private, meaning they can only be accessed within the TPerson class.

Inheritance

Inheritance allows a class to inherit properties and methods from another class. Here is an example:

type
  TEmployee = class(TPerson)
  private
    FEmployeeID: string;
  public
    procedure SetEmployeeID(AEmployeeID: string);
    function GetEmployeeID: string;
  end;

Polymorphism

Polymorphism in Delphi is typically achieved through method overriding and interfaces. Here is an example of method overriding:

type
  TPerson = class
  public
    procedure Speak; virtual;
  end;

  TEmployee = class(TPerson)
  public
    procedure Speak; override;
  end;

procedure TPerson.Speak;
begin
  ShowMessage('Hello, I am a person.');
end;

procedure TEmployee.Speak;
begin
  ShowMessage('Hello, I am an employee.');
end;

Practical Example

Let's create a simple application that demonstrates these concepts:

type
  TPerson = class
  private
    FName: string;
    FAge: Integer;
  public
    procedure SetName(AName: string);
    function GetName: string;
    procedure SetAge(AAge: Integer);
    function GetAge: Integer;
    procedure Speak; virtual;
  end;

  TEmployee = class(TPerson)
  private
    FEmployeeID: string;
  public
    procedure SetEmployeeID(AEmployeeID: string);
    function GetEmployeeID: string;
    procedure Speak; override;
  end;

procedure TPerson.SetName(AName: string);
begin
  FName := AName;
end;

function TPerson.GetName: string;
begin
  Result := FName;
end;

procedure TPerson.SetAge(AAge: Integer);
begin
  FAge := AAge;
end;

function TPerson.GetAge: Integer;
begin
  Result := FAge;
end;

procedure TPerson.Speak;
begin
  ShowMessage('Hello, I am a person.');
end;

procedure TEmployee.SetEmployeeID(AEmployeeID: string);
begin
  FEmployeeID := AEmployeeID;
end;

function TEmployee.GetEmployeeID: string;
begin
  Result := FEmployeeID;
end;

procedure TEmployee.Speak;
begin
  ShowMessage('Hello, I am an employee.');
end;

var
  Person: TPerson;
  Employee: TEmployee;
begin
  Person := TPerson.Create;
  Person.SetName('John Doe');
  Person.SetAge(30);
  Person.Speak;
  Person.Free;

  Employee := TEmployee.Create;
  Employee.SetName('Jane Smith');
  Employee.SetAge(28);
  Employee.SetEmployeeID('E12345');
  Employee.Speak;
  Employee.Free;
end;

Conclusion

In this section, we introduced the fundamental concepts of Object-Oriented Programming in Delphi/Object Pascal. We covered:

  • The definition and creation of classes and objects.
  • Encapsulation, inheritance, and polymorphism.
  • Practical examples to illustrate these concepts.

Understanding these basics will provide a strong foundation for more advanced topics in Delphi programming. As you progress, you will see how these principles are applied in real-world applications, making your code more modular, reusable, and easier to maintain.
