Introduction to Unit Testing

  • Definition: Unit testing involves testing individual units or components of a software to ensure they work as intended.
  • Importance: Helps in identifying bugs early, simplifies integration, and ensures code reliability.

Setting Up Mocha and Chai

  • Mocha: A feature-rich JavaScript test framework running on Node.js, making asynchronous testing simple and fun.
  • Chai: A BDD / TDD assertion library for Node.js that can be paired with any JavaScript testing framework.

Installing Mocha and Chai

To get started, you need to install Mocha and Chai using npm:

npm install --save-dev mocha chai

Configuring Mocha

Create a test directory in your project root and add a configuration in your package.json:

{
  "scripts": {
    "test": "mocha"
  }
}

Writing Your First Test

Basic Test Structure

Create a file named test.js inside the test directory:

const assert = require('chai').assert;

describe('Array', function() {
  it('should start empty', function() {
    let arr = [];
    assert.equal(arr.length, 0);
  });
});
  • describe(): Groups related tests.
  • it(): Defines a single test case.
  • assert: Provides assertion methods to verify test results.

Running Tests

Run your tests using the following command:

npm test

Advanced Testing with Mocha and Chai

Asynchronous Testing

Mocha supports asynchronous tests using callbacks, promises, or async/await.

Using Callbacks

describe('Async Test', function() {
  it('should complete asynchronously', function(done) {
    setTimeout(function() {
      assert.isTrue(true);
      done();
    }, 1000);
  });
});

Using Promises

describe('Async Test', function() {
  it('should complete with a promise', function() {
    return new Promise((resolve) => {
      setTimeout(function() {
        assert.isTrue(true);
        resolve();
      }, 1000);
    });
  });
});

Using Async/Await

describe('Async Test', function() {
  it('should complete with async/await', async function() {
    await new Promise((resolve) => setTimeout(resolve, 1000));
    assert.isTrue(true);
  });
});

Using Chai Assertions

Chai provides three styles of assertions: Assert, Expect, and Should.

Assert Style

assert.equal(3, 3, '3 equals 3');

Expect Style

const expect = require('chai').expect;
expect(3).to.equal(3);

Should Style

const should = require('chai').should();
(3).should.equal(3);

Testing HTTP APIs with Chai HTTP

Install chai-http to test HTTP endpoints:

npm install --save-dev chai-http

Example Test

const chai = require('chai');
const chaiHttp = require('chai-http');
const app = require('../app'); // Your Express app
chai.use(chaiHttp);

describe('GET /api', function() {
  it('should return status 200', function(done) {
    chai.request(app)
      .get('/api')
      .end(function(err, res) {
        chai.assert.equal(res.status, 200);
        done();
      });
  });
});

Mocking and Stubbing with Sinon

Introduction to Sinon

Sinon is a library used for creating spies, mocks, and stubs.

Installing Sinon

npm install --save-dev sinon

Example of Stubbing

const sinon = require('sinon');
const myModule = require('../myModule');

describe('myModule', function() {
  it('should call the callback', function() {
    const callback = sinon.stub();
    myModule.doSomething(callback);
    sinon.assert.calledOnce(callback);
  });
});

Conclusion

Unit testing is a crucial part of the development process, ensuring that individual components of your application work as expected. Mocha and Chai provide a powerful combination for writing and running tests in Node.js. By mastering these tools, you can write reliable, maintainable, and robust code. Remember to:

  • Start with simple tests and gradually move to more complex scenarios.
  • Use asynchronous testing techniques as needed.
  • Leverage Chai's various assertion styles to make your tests more readable.
  • Utilize Sinon for advanced mocking and stubbing.

Happy Testing!
