Introduction to Routing

Routing in Express involves defining how your application responds to various HTTP requests. Each route can have one or more handler functions, which are executed when the route is matched.

Basic Routing

  • Route Definition: A route is defined by an HTTP method, a path, and a handler function.
  • HTTP Methods: Common methods include GET, POST, PUT, DELETE, etc.

Example: Basic GET Route

const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('Hello World!');
});

app.listen(3000, () => {
  console.log('Server is running on port 3000');
});

In this example:

  • app.get('/') defines a route that listens for GET requests on the root URL (/).
  • The handler function sends a "Hello World!" response.

Route Parameters

Route parameters are named URL segments used to capture values at specific positions in the URL.

Example: Route Parameters

app.get('/users/:userId', (req, res) => {
  const userId = req.params.userId;
  res.send(`User ID: ${userId}`);
});

In this example:

  • :userId is a route parameter.
  • req.params.userId captures the value of userId from the URL.

Query Parameters

Query parameters are used to pass data to the server as part of the URL.

Example: Query Parameters

app.get('/search', (req, res) => {
  const query = req.query.q;
  res.send(`Search query: ${query}`);
});

In this example:

  • req.query.q captures the value of the query parameter q from the URL.

Handling Different HTTP Methods

Express allows you to handle different HTTP methods for the same route.

Example: Handling Different Methods

app.route('/book')
  .get((req, res) => {
    res.send('Get a random book');
  })
  .post((req, res) => {
    res.send('Add a book');
  })
  .put((req, res) => {
    res.send('Update the book');
  });

In this example:

  • app.route('/book') chains multiple handlers for different HTTP methods on the same route.

Middleware in Routes

Middleware functions can be used to handle requests before they reach the route handler.

Example: Middleware in Routes

const logRequest = (req, res, next) => {
  console.log(`${req.method} ${req.url}`);
  next();
};

app.use(logRequest);

app.get('/', (req, res) => {
  res.send('Hello World!');
});

In this example:

  • logRequest is a middleware function that logs the request method and URL.
  • app.use(logRequest) applies this middleware to all routes.

Advanced Routing

Route Groups

Route groups allow you to apply middleware to a group of routes.

Example: Route Groups

const express = require('express');
const app = express();
const router = express.Router();

router.use((req, res, next) => {
  console.log('Request URL:', req.originalUrl);
  next();
});

router.get('/user/:id', (req, res) => {
  res.send(`User ID: ${req.params.id}`);
});

router.get('/user/:id/edit', (req, res) => {
  res.send(`Edit User ID: ${req.params.id}`);
});

app.use('/admin', router);

app.listen(3000, () => {
  console.log('Server is running on port 3000');
});

In this example:

  • router.use applies middleware to all routes within the router.
  • app.use('/admin', router) mounts the router on the /admin path.

Summary

Routing in Express is a powerful feature that allows you to define how your application responds to various HTTP requests. Key concepts include:

  • Basic routing with HTTP methods.
  • Using route parameters and query parameters.
  • Handling different HTTP methods on the same route.
  • Applying middleware to routes.
  • Grouping routes with routers.

By mastering these concepts, you can create robust and scalable web applications using Express.
