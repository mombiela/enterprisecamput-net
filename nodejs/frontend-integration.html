Introduction

Integrating Node.js with frontend technologies is a crucial skill for full-stack developers. This section will guide you through the process of connecting your Node.js backend with various frontend frameworks and libraries. By the end of this topic, you will understand how to serve static files, use templating engines, and create APIs that can be consumed by frontend applications.

Serving Static Files

Node.js can serve static files such as HTML, CSS, and JavaScript. This is often the first step in integrating your backend with the frontend.

Example: Serving Static Files with Express

const express = require('express');
const path = require('path');
const app = express();

// Serve static files from the 'public' directory
app.use(express.static(path.join(__dirname, 'public')));

app.get('/', (req, res) => {
  res.sendFile(path.join(__dirname, 'public', 'index.html'));
});

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
  console.log(`Server is running on port ${PORT}`);
});
  • Explanation:
    • express.static middleware is used to serve static files.
    • path.join helps in constructing the path to the static files.
    • res.sendFile sends the HTML file to the client.

Using Templating Engines

Templating engines allow you to generate HTML dynamically. Popular templating engines include EJS, Pug, and Handlebars.

Example: Using EJS with Express

const express = require('express');
const app = express();

// Set EJS as the templating engine
app.set('view engine', 'ejs');

app.get('/', (req, res) => {
  res.render('index', { title: 'Home Page', message: 'Welcome to Node.js with EJS!' });
});

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
  console.log(`Server is running on port ${PORT}`);
});
  • Explanation:
    • app.set('view engine', 'ejs') sets EJS as the templating engine.
    • res.render renders the EJS template and sends it to the client.

Creating APIs for Frontend Consumption

APIs are essential for frontend-backend communication. You can create RESTful APIs using Node.js and Express.

Example: Creating a Simple REST API

const express = require('express');
const app = express();

app.use(express.json());

let items = [
  { id: 1, name: 'Item 1' },
  { id: 2, name: 'Item 2' }
];

// Get all items
app.get('/api/items', (req, res) => {
  res.json(items);
});

// Get a single item by ID
app.get('/api/items/:id', (req, res) => {
  const item = items.find(i => i.id === parseInt(req.params.id));
  if (!item) return res.status(404).send('Item not found');
  res.json(item);
});

// Create a new item
app.post('/api/items', (req, res) => {
  const newItem = {
    id: items.length + 1,
    name: req.body.name
  };
  items.push(newItem);
  res.status(201).json(newItem);
});

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
  console.log(`Server is running on port ${PORT}`);
});
  • Explanation:
    • express.json middleware is used to parse JSON bodies.
    • app.get defines routes for fetching data.
    • app.post defines a route for creating new data.

Integrating with Frontend Frameworks

Modern frontend frameworks like React, Angular, and Vue.js can consume APIs created with Node.js.

Example: Fetching Data in React

import React, { useEffect, useState } from 'react';

function App() {
  const [items, setItems] = useState([]);

  useEffect(() => {
    fetch('/api/items')
      .then(response => response.json())
      .then(data => setItems(data));
  }, []);

  return (
    <div>
      <h1>Items</h1>
      <ul>
        {items.map(item => (
          <li key={item.id}>{item.name}</li>
        ))}
      </ul>
    </div>
  );
}

export default App;
  • Explanation:
    • useEffect is used to fetch data from the API when the component mounts.
    • fetch makes an HTTP request to the Node.js API.
    • setItems updates the state with the fetched data.

Conclusion

Integrating Node.js with frontend technologies involves serving static files, using templating engines, creating APIs, and consuming those APIs in frontend applications. By mastering these skills, you can build robust full-stack applications that provide a seamless user experience.
