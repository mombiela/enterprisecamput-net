Introduction to MongoDB and Mongoose

  • MongoDB: A NoSQL database that stores data in JSON-like documents.
  • Mongoose: An Object Data Modeling (ODM) library for MongoDB and Node.js, providing a schema-based solution to model your application data.

Why Use Mongoose?

  • Simplifies data validation.
  • Provides a schema to structure your data.
  • Offers built-in type casting, query building, and business logic hooks.

Setting Up MongoDB and Mongoose

Installing MongoDB

  1. Download and install MongoDB from the official website.
  2. Start the MongoDB server: 
    mongod

Installing Mongoose

  1. Initialize a Node.js project if you haven't already: 
    npm init -y
  2. Install Mongoose: 
    npm install mongoose

Connecting to MongoDB with Mongoose

Basic Connection

const mongoose = require('mongoose');

mongoose.connect('mongodb://localhost:27017/mydatabase', {
  useNewUrlParser: true,
  useUnifiedTopology: true
}).then(() => {
  console.log('Connected to MongoDB');
}).catch(err => {
  console.error('Connection error', err);
});
  • Explanation:
    • mongoose.connect: Connects to the MongoDB server.
    • useNewUrlParser and useUnifiedTopology: Options to avoid deprecation warnings.

Defining Schemas and Models

Creating a Schema

const { Schema } = mongoose;

const userSchema = new Schema({
  name: String,
  age: Number,
  email: String
});
  • Explanation:
    • Schema: Defines the structure of the document.
    • name, age, email: Fields of the document with their respective data types.

Creating a Model

const User = mongoose.model('User', userSchema);
  • Explanation:
    • model: Compiles the schema into a model, a class with which we construct documents.

CRUD Operations

Create

const newUser = new User({
  name: 'John Doe',
  age: 30,
  email: 'john.doe@example.com'
});

newUser.save().then(() => console.log('User saved'));
  • Explanation:
    • new User: Creates a new instance of the User model.
    • save: Saves the document to the database.

Read

User.find({ age: { $gte: 18 } }).then(users => {
  console.log(users);
});
  • Explanation:
    • find: Retrieves documents matching the query.

Update

User.updateOne({ name: 'John Doe' }, { age: 31 }).then(() => {
  console.log('User updated');
});
  • Explanation:
    • updateOne: Updates a single document matching the query.

Delete

User.deleteOne({ name: 'John Doe' }).then(() => {
  console.log('User deleted');
});
  • Explanation:
    • deleteOne: Deletes a single document matching the query.

Advanced Topics

Middleware

  • Pre Middleware: Runs before a certain event.

    userSchema.pre('save', function(next) {
  console.log('Before saving');
  next();
});

  • Post Middleware: Runs after a certain event.

    userSchema.post('save', function(doc) {
  console.log('After saving', doc);
});

Virtuals

  • Virtual Properties: Fields that are not stored in MongoDB but can be used like any other field. 
    userSchema.virtual('fullName').get(function() {
  return `${this.firstName} ${this.lastName}`;
});

Population

  • Populating References: Replaces the specified paths in the document with document(s) from other collection(s). 
    const postSchema = new Schema({
  title: String,
  author: { type: Schema.Types.ObjectId, ref: 'User' }
});

const Post = mongoose.model('Post', postSchema);

Post.find().populate('author').then(posts => {
  console.log(posts);
});

Conclusion

  • Summary:
    • Mongoose simplifies interactions with MongoDB by providing a schema-based solution.
    • Basic CRUD operations are straightforward with Mongoose.
    • Advanced features like middleware, virtuals, and population enhance the functionality and flexibility of your application.

By mastering Mongoose, you can efficiently manage your MongoDB data and build robust Node.js applications.
