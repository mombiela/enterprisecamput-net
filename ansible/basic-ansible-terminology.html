Basic Ansible Terminology
First Steps with Ansible ►

In this section, we will cover the fundamental terminology used in Ansible. Understanding these terms is crucial for effectively using Ansible for automation and configuration management.

Key Concepts

  • Control Node: The machine where Ansible is installed and from which commands are run. This is the central point from which all automation tasks are managed.

  • Managed Nodes: The machines that are being managed by Ansible. These are also known as hosts. Ansible communicates with these nodes to execute tasks.

  • Inventory: A file that lists all the managed nodes. It can be a simple text file with a list of IP addresses or hostnames, or it can be a more complex file with groups and variables.

    [webservers]
web1.example.com
web2.example.com

[dbservers]
db1.example.com
db2.example.com

  • Playbook: A YAML file that contains a series of tasks to be executed on the managed nodes. Playbooks are the core of Ansible's configuration, deployment, and orchestration language.

    - name: Install and start Apache
  hosts: webservers
  tasks:
    - name: Install Apache
      apt:
        name: apache2
        state: present

    - name: Start Apache
      service:
        name: apache2
        state: started

  • Task: A single unit of work in a playbook. Each task calls an Ansible module to perform a specific action.

  • Module: The units of work that Ansible executes. Modules can manage system resources, install software, and perform various other tasks. Examples include apt, yum, service, and copy.

  • Role: A way to group multiple tasks, variables, files, templates, and handlers into a single unit. Roles help in organizing playbooks and reusing code.

  • Handler: A special type of task that runs only when notified by another task. Handlers are typically used to restart services.

    - name: Install Apache
  apt:
    name: apache2
    state: present
  notify:
    - Restart Apache

handlers:
  - name: Restart Apache
    service:
      name: apache2
      state: restarted

  • Variable: Used to store values that can be reused throughout playbooks and roles. Variables can be defined in many places, including inventory files, playbooks, and roles.

    - name: Install Apache
  hosts: webservers
  vars:
    apache_package: apache2
  tasks:
    - name: Install Apache
      apt:
        name: "{{ apache_package }}"
        state: present

  • Fact: Information about the managed nodes collected by Ansible. Facts are gathered automatically and can be used in playbooks.

  • Template: A file that contains variables and is processed by the Jinja2 templating engine to produce a final configuration file.

    # template.j2
ServerName {{ ansible_hostname }}

    - name: Configure Apache
  hosts: webservers
  tasks:
    - name: Deploy Apache config
      template:
        src: template.j2
        dest: /etc/apache2/sites-available/000-default.conf

Summary

Understanding these basic Ansible terms is essential for anyone starting with Ansible. These concepts form the foundation upon which more advanced topics and practices are built. By familiarizing yourself with these terms, you will be better prepared to write and understand Ansible playbooks, manage configurations, and automate tasks effectively.
