Introduction

Ansible is an open-source automation tool used for configuration management, application deployment, and task automation. This section will guide you through the process of installing Ansible on various operating systems. By the end of this section, you will have Ansible installed and ready to use.

Prerequisites

Before installing Ansible, ensure you have the following:

  • A supported operating system (Linux, macOS, or Windows Subsystem for Linux).
  • Python 3.8 or later installed.
  • Administrative privileges to install software.

Installing Ansible on Linux

Using Package Manager (Recommended)

Ansible can be installed using the package manager specific to your Linux distribution.

Ubuntu/Debian

  1. Update your package index: 
    sudo apt update
  2. Install the software-properties-common package: 
    sudo apt install software-properties-common
  3. Add the Ansible PPA (Personal Package Archive): 
    sudo add-apt-repository --yes --update ppa:ansible/ansible
  4. Install Ansible: 
    sudo apt install ansible

CentOS/RHEL

  1. Install the EPEL repository: 
    sudo yum install epel-release
  2. Install Ansible: 
    sudo yum install ansible

Using Pip (Python Package Installer)

If you prefer to use pip, follow these steps:

  1. Ensure pip is installed: 
    sudo apt install python3-pip  # For Ubuntu/Debian
sudo yum install python3-pip  # For CentOS/RHEL
  2. Install Ansible using pip: 
    pip3 install ansible

Installing Ansible on macOS

You can install Ansible on macOS using Homebrew, a popular package manager for macOS.

  1. Install Homebrew if you haven't already: 
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  2. Install Ansible: 
    brew install ansible

Installing Ansible on Windows

Ansible is not natively supported on Windows, but you can use the Windows Subsystem for Linux (WSL) to run a Linux distribution and install Ansible.

  1. Enable WSL and install a Linux distribution from the Microsoft Store (e.g., Ubuntu).
  2. Open the installed Linux distribution and update the package index: 
    sudo apt update
  3. Install Ansible using the same steps as for Ubuntu/Debian: 
    sudo apt install software-properties-common
sudo add-apt-repository --yes --update ppa:ansible/ansible
sudo apt install ansible

Verifying the Installation

After installing Ansible, verify the installation by checking the version:

ansible --version

You should see output similar to:

ansible [core 2.10.7]
  config file = /etc/ansible/ansible.cfg
  configured module search path = ['/home/user/.ansible/plugins/modules', '/usr/share/ansible/plugins/modules']
  ansible python module location = /usr/local/lib/python3.8/dist-packages/ansible
  executable location = /usr/local/bin/ansible
  python version = 3.8.5 (default, Jul 28 2020, 12:59:40) [GCC 9.3.0]

Conclusion

In this section, you learned how to install Ansible on various operating systems including Linux, macOS, and Windows (via WSL). Ensuring that Ansible is correctly installed is the first step towards leveraging its powerful automation capabilities. In the next sections, we will explore how to configure and use Ansible for different automation tasks.
