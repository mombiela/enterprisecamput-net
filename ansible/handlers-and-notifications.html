In this section, we will explore Handlers and Notifications in Ansible. These are essential concepts for managing state changes and ensuring that tasks are executed only when necessary.

What are Handlers?

Handlers are special tasks in Ansible that are triggered by notifications. They are typically used to restart services, reload configurations, or perform other actions that should only occur when a change has been made.

Key Concepts

  • Handlers: Tasks that are executed only when notified.
  • Notifications: Triggers that tell handlers to run.

Example of a Handler

Let's look at a basic example of a handler:

# playbook.yml
- name: Install and configure Apache
  hosts: webservers
  tasks:
    - name: Install Apache
      apt:
        name: apache2
        state: present
      notify:
        - Restart Apache

  handlers:
    - name: Restart Apache
      service:
        name: apache2
        state: restarted

In this example:

  • The apt task installs Apache and notifies the handler Restart Apache.
  • The handler Restart Apache will restart the Apache service, but only if the apt task makes a change (i.e., installs Apache).

Notifications

Notifications are used to trigger handlers. They are defined within tasks and specify which handlers should be run if the task results in a change.

Example of Notifications

Continuing from the previous example, the notify keyword is used to specify the handler:

- name: Install Apache
  apt:
    name: apache2
    state: present
  notify:
    - Restart Apache

Multiple Notifications

A task can notify multiple handlers. Here’s an example:

- name: Install Apache
  apt:
    name: apache2
    state: present
  notify:
    - Restart Apache
    - Reload Firewall

handlers:
  - name: Restart Apache
    service:
      name: apache2
      state: restarted

  - name: Reload Firewall
    command: ufw reload

In this example:

  • The apt task notifies both Restart Apache and Reload Firewall handlers.
  • Both handlers will be executed if the apt task makes a change.

Practical Use Cases

Restarting Services

One of the most common use cases for handlers is restarting services when configuration files are changed.

- name: Copy Apache config file
  copy:
    src: /path/to/httpd.conf
    dest: /etc/httpd/conf/httpd.conf
  notify:
    - Restart Apache

handlers:
  - name: Restart Apache
    service:
      name: httpd
      state: restarted

Reloading Configurations

Handlers can also be used to reload configurations without restarting services.

- name: Update Nginx configuration
  template:
    src: /path/to/nginx.conf.j2
    dest: /etc/nginx/nginx.conf
  notify:
    - Reload Nginx

handlers:
  - name: Reload Nginx
    service:
      name: nginx
      state: reloaded

Best Practices

  • Idempotency: Ensure that tasks and handlers are idempotent, meaning they can be run multiple times without changing the system after the first run.
  • Descriptive Names: Use descriptive names for handlers to make playbooks easier to understand.
  • Minimal Changes: Only notify handlers when necessary to avoid unnecessary service restarts or reloads.

Conclusion

Handlers and notifications are powerful features in Ansible that help manage state changes efficiently. By using handlers, you can ensure that certain tasks are only executed when necessary, which can help maintain the stability and performance of your systems. Remember to follow best practices to create clear, maintainable playbooks.
