Overview

Active Directory (AD) is a directory service developed by Microsoft for Windows domain networks. PowerShell provides a set of cmdlets specifically designed to interact with Active Directory. This section will introduce you to these cmdlets, explain their usage, and provide practical examples to help you understand how to manage AD objects using PowerShell.

Key Concepts

  • Active Directory: A directory service for Windows domain networks.
  • Cmdlets: Specialized commands in PowerShell that perform specific operations.
  • AD Module: A PowerShell module that contains cmdlets for managing Active Directory.

Prerequisites

Before diving into Active Directory cmdlets, ensure you have:

  • Basic understanding of PowerShell syntax and commands.
  • Access to a Windows Server with Active Directory installed.
  • Administrative privileges to perform AD operations.

Installing the Active Directory Module

To use AD cmdlets, you need to install the Active Directory module. This module is available in the Remote Server Administration Tools (RSAT) for Windows.

Steps to Install RSAT

  1. Open Settings.
  2. Go to Apps > Optional features.
  3. Click on Add a feature.
  4. Search for RSAT: Active Directory Domain Services and Lightweight Directory Tools.
  5. Install the feature.

Common Active Directory Cmdlets

Here are some of the most commonly used Active Directory cmdlets:

| Cmdlet | Description | |-----------------------|--------------------------------------------------| | Get-ADUser | Retrieves user objects from AD. | | New-ADUser | Creates a new user in AD. | | Set-ADUser | Modifies properties of an existing user. | | Remove-ADUser | Deletes a user from AD. | | Get-ADGroup | Retrieves group objects from AD. | | New-ADGroup | Creates a new group in AD. | | Add-ADGroupMember | Adds a user to a group. | | Remove-ADGroupMember| Removes a user from a group. |

Using Active Directory Cmdlets

Retrieving User Information

To retrieve information about a user, you can use the Get-ADUser cmdlet.

Get-ADUser -Identity "jdoe"

Explanation:

  • Get-ADUser: The cmdlet to retrieve user information.
  • -Identity "jdoe": Specifies the user to retrieve by their username.

Creating a New User

To create a new user in Active Directory, use the New-ADUser cmdlet.

New-ADUser -Name "John Doe" -GivenName "John" -Surname "Doe" -SamAccountName "jdoe" -UserPrincipalName "jdoe@example.com" -Path "OU=Users,DC=example,DC=com" -AccountPassword (ConvertTo-SecureString "P@ssw0rd" -AsPlainText -Force) -Enabled $true

Explanation:

  • New-ADUser: The cmdlet to create a new user.
  • -Name "John Doe": The full name of the user.
  • -GivenName "John": The given name of the user.
  • -Surname "Doe": The surname of the user.
  • -SamAccountName "jdoe": The SAM account name.
  • -UserPrincipalName "jdoe@example.com": The UPN of the user.
  • -Path "OU=Users,DC=example,DC=com": The organizational unit where the user will be created.
  • -AccountPassword (ConvertTo-SecureString "P@ssw0rd" -AsPlainText -Force): Sets the account password.
  • -Enabled $true: Enables the user account.

Modifying User Properties

To modify properties of an existing user, use the Set-ADUser cmdlet.

Set-ADUser -Identity "jdoe" -Title "Senior Developer" -Department "IT"

Explanation:

  • Set-ADUser: The cmdlet to modify user properties.
  • -Identity "jdoe": Specifies the user to modify.
  • -Title "Senior Developer": Sets the user's title.
  • -Department "IT": Sets the user's department.

Deleting a User

To delete a user from Active Directory, use the Remove-ADUser cmdlet.

Remove-ADUser -Identity "jdoe"

Explanation:

  • Remove-ADUser: The cmdlet to delete a user.
  • -Identity "jdoe": Specifies the user to delete.

Conclusion

In this section, you learned about the basics of Active Directory cmdlets in PowerShell. You now know how to install the AD module, retrieve user information, create new users, modify user properties, and delete users. These cmdlets are powerful tools for managing Active Directory objects efficiently. Practice using these cmdlets to become more comfortable with managing AD through PowerShell.
