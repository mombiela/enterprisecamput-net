Introduction to Azure PowerShell

Azure PowerShell is a set of cmdlets designed to manage Azure resources directly from the PowerShell command line. This section will cover the basics of Azure PowerShell and how to get started.

Key Concepts

  • Cmdlets: Specialized commands in the PowerShell environment.
  • Azure Resource Manager (ARM): The deployment and management service for Azure.
  • Modules: Packages that contain cmdlets, providers, functions, workflows, variables, and aliases.

Installing Azure PowerShell

To use Azure PowerShell, you need to install the Az module.

# Install the Az module
Install-Module -Name Az -AllowClobber -Scope CurrentUser

Authenticating with Azure

Before managing Azure resources, you need to authenticate.

# Sign in to your Azure account
Connect-AzAccount

Managing Azure Resources

This section will cover how to manage various Azure resources using PowerShell.

Creating a Resource Group

Resource groups are containers that hold related resources for an Azure solution.

# Create a new resource group
New-AzResourceGroup -Name "MyResourceGroup" -Location "EastUS"

Creating a Virtual Machine

Virtual machines (VMs) are one of the most common resources in Azure.

# Create a new virtual machine
New-AzVM -ResourceGroupName "MyResourceGroup" -Name "MyVM" -Location "EastUS" -VirtualNetworkName "MyVNet" -SubnetName "MySubnet" -SecurityGroupName "MyNSG" -PublicIpAddressName "MyPublicIP" -OpenPorts 80,3389

Managing Storage Accounts

Storage accounts provide a unique namespace to store and access Azure Storage data objects.

# Create a new storage account
New-AzStorageAccount -ResourceGroupName "MyResourceGroup" -Name "mystorageaccount" -Location "EastUS" -SkuName "Standard_LRS"

Advanced Azure PowerShell

This section will cover more advanced topics and best practices for using Azure PowerShell.

Automating Tasks with Scripts

Automate repetitive tasks by writing PowerShell scripts.

# Example script to automate VM creation
$resourceGroupName = "MyResourceGroup"
$location = "EastUS"
$vmName = "MyVM"

New-AzResourceGroup -Name $resourceGroupName -Location $location
New-AzVM -ResourceGroupName $resourceGroupName -Name $vmName -Location $location -VirtualNetworkName "MyVNet" -SubnetName "MySubnet" -SecurityGroupName "MyNSG" -PublicIpAddressName "MyPublicIP" -OpenPorts 80,3389

Using Azure PowerShell with CI/CD Pipelines

Integrate Azure PowerShell scripts into CI/CD pipelines for automated deployments.

# Example Azure DevOps pipeline script
trigger:
- main

pool:
  vmImage: 'windows-latest'

steps:
- task: UseDotNet@2
  inputs:
    packageType: 'sdk'
    version: '5.x'
    installationPath: $(Agent.ToolsDirectory)/dotnet

- powershell: |
    Install-Module -Name Az -AllowClobber -Scope CurrentUser
    Connect-AzAccount -ServicePrincipal -Tenant $env:tenantId -Credential (New-Object System.Management.Automation.PSCredential($env:servicePrincipalId, (ConvertTo-SecureString $env:servicePrincipalKey -AsPlainText -Force)))
    New-AzResourceGroup -Name "MyResourceGroup" -Location "EastUS"
  env:
    tenantId: $(tenantId)
    servicePrincipalId: $(servicePrincipalId)
    servicePrincipalKey: $(servicePrincipalKey)

Summary

In this topic, we covered the basics of using PowerShell with Azure, including:

  • Installing and authenticating with Azure PowerShell.
  • Managing Azure resources like resource groups, virtual machines, and storage accounts.
  • Automating tasks with PowerShell scripts.
  • Integrating Azure PowerShell into CI/CD pipelines.

By mastering these skills, you can efficiently manage and automate your Azure resources using PowerShell.
