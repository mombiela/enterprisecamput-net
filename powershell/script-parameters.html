In this section, we'll explore how to use script parameters in PowerShell. Script parameters allow you to pass data into your scripts, making them more flexible and reusable. We'll cover the basics for beginners and move on to more advanced concepts.

Introduction to Script Parameters

Script parameters are a way to make your scripts more dynamic by allowing you to pass different values each time you run the script. This is particularly useful for automation and scripting tasks that require varying inputs.

Basic Syntax

To define parameters in a PowerShell script, you use the param keyword followed by a list of parameters enclosed in parentheses.

param (
    [string]$Name,
    [int]$Age
)

Write-Output "Name: $Name"
Write-Output "Age: $Age"
  • [string]$Name: This defines a parameter named Name of type string.
  • [int]$Age: This defines a parameter named Age of type int.

Running the Script with Parameters

To run the script with parameters, you simply provide the values after the script name.

.\MyScript.ps1 -Name "Alice" -Age 30

Default Values

You can also set default values for parameters, which will be used if no value is provided.

param (
    [string]$Name = "John Doe",
    [int]$Age = 25
)

Write-Output "Name: $Name"
Write-Output "Age: $Age"

Mandatory Parameters

To make a parameter mandatory, you can use the Mandatory attribute.

param (
    [Parameter(Mandatory=$true)]
    [string]$Name,
    [int]$Age
)

Write-Output "Name: $Name"
Write-Output "Age: $Age"

If you run the script without providing the Name parameter, PowerShell will prompt you to enter it.

Advanced Script Parameters

Parameter Validation

PowerShell provides several attributes to validate parameter values.

  • ValidateSet: Ensures the parameter value is one of a predefined set of values.
  • ValidateRange: Ensures the parameter value falls within a specified range.
  • ValidatePattern: Ensures the parameter value matches a regular expression pattern.
param (
    [ValidateSet("Red", "Green", "Blue")]
    [string]$Color,

    [ValidateRange(1, 100)]
    [int]$Percentage,

    [ValidatePattern("^[a-zA-Z]+$")]
    [string]$Name
)

Write-Output "Color: $Color"
Write-Output "Percentage: $Percentage"
Write-Output "Name: $Name"

Parameter Aliases

You can define aliases for parameters to provide alternative names.

param (
    [Alias("FN")]
    [string]$FirstName,

    [Alias("LN")]
    [string]$LastName
)

Write-Output "First Name: $FirstName"
Write-Output "Last Name: $LastName"

Using `CmdletBinding`

The CmdletBinding attribute turns your script into an advanced function, allowing you to use common parameters and other advanced features.

[CmdletBinding()]
param (
    [string]$Name,
    [int]$Age
)

Write-Output "Name: $Name"
Write-Output "Age: $Age"

Parameter Sets

Parameter sets allow you to define different sets of parameters that can be used in different scenarios.

[CmdletBinding()]
param (
    [Parameter(Mandatory=$true, ParameterSetName="Set1")]
    [string]$Name,

    [Parameter(Mandatory=$true, ParameterSetName="Set2")]
    [int]$Age
)

if ($PSCmdlet.ParameterSetName -eq "Set1") {
    Write-Output "Name: $Name"
} elseif ($PSCmdlet.ParameterSetName -eq "Set2") {
    Write-Output "Age: $Age"
}

Conclusion

Script parameters are a powerful feature in PowerShell that can greatly enhance the flexibility and reusability of your scripts. By understanding and utilizing basic and advanced parameter features, you can create more dynamic and robust scripts. Whether you are setting default values, making parameters mandatory, or using advanced validation and parameter sets, mastering script parameters is essential for effective PowerShell scripting.
