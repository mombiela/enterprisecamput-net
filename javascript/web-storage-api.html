Web Storage API
Fetch API ►

The Web Storage API provides mechanisms by which browsers can store key/value pairs, in a much more intuitive fashion than using cookies. This API is essential for building modern web applications that require persistent data storage on the client side.

Key Concepts

  • Local Storage: Stores data with no expiration date. Data is not deleted when the browser is closed and can be accessed later.
  • Session Storage: Stores data for the duration of the page session. Data is deleted when the page session ends (i.e., when the browser is closed).

Local Storage

Local Storage is designed for storing data that needs to persist even after the user closes the browser. It is ideal for storing user preferences, settings, and other data that should be available across sessions.

Basic Operations

Setting Data

To store data in Local Storage, use the setItem method.

localStorage.setItem('username', 'JohnDoe');

Getting Data

To retrieve data from Local Storage, use the getItem method.

let username = localStorage.getItem('username');
console.log(username); // Outputs: JohnDoe

Removing Data

To remove a specific item from Local Storage, use the removeItem method.

localStorage.removeItem('username');

Clearing All Data

To clear all data from Local Storage, use the clear method.

localStorage.clear();

Example: Storing User Preferences

// Storing user preferences
localStorage.setItem('theme', 'dark');
localStorage.setItem('fontSize', '16px');

// Retrieving user preferences
let theme = localStorage.getItem('theme');
let fontSize = localStorage.getItem('fontSize');

console.log(`Theme: ${theme}, Font Size: ${fontSize}`); // Outputs: Theme: dark, Font Size: 16px

Session Storage

Session Storage is designed for storing data that should only be available for the duration of the page session. It is useful for storing temporary data that does not need to persist across sessions.

Basic Operations

Setting Data

To store data in Session Storage, use the setItem method.

sessionStorage.setItem('sessionID', '123456');

Getting Data

To retrieve data from Session Storage, use the getItem method.

let sessionID = sessionStorage.getItem('sessionID');
console.log(sessionID); // Outputs: 123456

Removing Data

To remove a specific item from Session Storage, use the removeItem method.

sessionStorage.removeItem('sessionID');

Clearing All Data

To clear all data from Session Storage, use the clear method.

sessionStorage.clear();

Example: Storing Temporary Data

// Storing temporary data
sessionStorage.setItem('currentPage', 'home');
sessionStorage.setItem('cartItems', JSON.stringify(['item1', 'item2', 'item3']));

// Retrieving temporary data
let currentPage = sessionStorage.getItem('currentPage');
let cartItems = JSON.parse(sessionStorage.getItem('cartItems'));

console.log(`Current Page: ${currentPage}, Cart Items: ${cartItems}`); // Outputs: Current Page: home, Cart Items: item1,item2,item3

Comparison of Local Storage and Session Storage

| Feature | Local Storage | Session Storage | |---------------------|---------------------------------------|---------------------------------------| | Data Persistence | Persistent across sessions | Only for the duration of the session | | Storage Limit | Typically 5-10 MB per origin | Typically 5-10 MB per origin | | Use Cases | User preferences, settings | Temporary data, session-specific info | | Access Scope | Any window/tab from the same origin | Only the window/tab that created it |

Conclusion

The Web Storage API is a powerful tool for client-side data storage in web applications. By understanding the differences between Local Storage and Session Storage, developers can make informed decisions about where to store data based on its persistence requirements. Local Storage is ideal for data that needs to persist across sessions, while Session Storage is suitable for temporary data that should be cleared when the session ends.
