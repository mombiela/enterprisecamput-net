Introduction to Regular Expressions

Regular expressions (regex) are patterns used to match character combinations in strings. In JavaScript, regular expressions are objects that can be used to perform pattern-matching and "search-and-replace" functions on text.

  • Basic Syntax: Regular expressions are created using two ways:
    • Literal notation: Enclosed between slashes (/pattern/).
    • Constructor function: Created with the RegExp constructor (new RegExp('pattern')).

Example:

// Literal notation
const regex1 = /hello/;

// Constructor function
const regex2 = new RegExp('hello');

Basic Patterns

  • Literal Characters: Matches the exact characters in the pattern.
  • Metacharacters: Special characters that have a unique meaning in regex.

Common Metacharacters:

  • .: Matches any single character except newline.
  • \d: Matches any digit (equivalent to [0-9]).
  • \w: Matches any word character (alphanumeric + underscore).
  • \s: Matches any whitespace character (spaces, tabs, line breaks).
  • \b: Matches a word boundary.
  • ^: Matches the start of a string.
  • $: Matches the end of a string.

Example:

const regex = /\d\w\s/;
const str = "3a ";
console.log(regex.test(str)); // true

Character Classes and Ranges

  • Character Classes: Defined using square brackets [] to match any one of the enclosed characters.
  • Ranges: Defined using a hyphen - within square brackets to specify a range of characters.

Example:

const regex = /[a-z]/; // Matches any lowercase letter
const str = "hello";
console.log(regex.test(str)); // true

Quantifiers

Quantifiers specify how many times a character or group should be matched.

Common Quantifiers:

  • *: Matches 0 or more times.
  • +: Matches 1 or more times.
  • ?: Matches 0 or 1 time.
  • {n}: Matches exactly n times.
  • {n,}: Matches n or more times.
  • {n,m}: Matches between n and m times.

Example:

const regex = /a{2,4}/; // Matches 'a' between 2 and 4 times
const str = "aaa";
console.log(regex.test(str)); // true

Groups and Alternation

  • Groups: Parentheses () are used to create groups.
  • Alternation: The pipe | is used to match one thing or another.

Example:

const regex = /(cat|dog)/; // Matches 'cat' or 'dog'
const str = "I have a cat";
console.log(regex.test(str)); // true

Flags

Flags are optional parameters that modify the behavior of the regex.

Common Flags:

  • g: Global search.
  • i: Case-insensitive search.
  • m: Multi-line search.

Example:

const regex = /hello/i; // Case-insensitive search
const str = "Hello";
console.log(regex.test(str)); // true

Methods for Regular Expressions

  • test(): Tests for a match in a string. Returns true or false.
  • exec(): Executes a search for a match in a string. Returns an array of information or null.

Example:

const regex = /hello/;
const str = "hello world";
console.log(regex.test(str)); // true
console.log(regex.exec(str)); // ["hello"]

String Methods with Regular Expressions

  • match(): Retrieves the matches when matching a string against a regex.
  • replace(): Replaces matched substrings with a new substring.
  • search(): Tests for a match in a string and returns the index of the match.
  • split(): Splits a string into an array of substrings using a regex.

Example:

const str = "hello world";
const regex = /o/;

console.log(str.match(regex)); // ["o"]
console.log(str.replace(regex, 'a')); // "hella world"
console.log(str.search(regex)); // 4
console.log(str.split(regex)); // ["hell", " w", "rld"]

Advanced Patterns

  • Lookaheads and Lookbehinds: Assertions that match a group before or after a main pattern without including it in the result.
    • Positive Lookahead: (?=...)
    • Negative Lookahead: (?!...)
    • Positive Lookbehind: (?<=...)
    • Negative Lookbehind: (?<!...)

Example:

const regex = /foo(?=bar)/; // Matches 'foo' only if followed by 'bar'
const str = "foobar";
console.log(regex.test(str)); // true

Conclusion

Regular expressions are a powerful tool for pattern matching and text manipulation in JavaScript. By understanding the basic syntax, character classes, quantifiers, groups, alternation, flags, and methods, you can perform complex text processing tasks efficiently. Practice using regular expressions with different examples to become proficient in their usage.
