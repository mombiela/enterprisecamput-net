Project Description

The final project of the Database Fundamentals course aims to apply the knowledge acquired throughout the course in a practical project. Students must design, implement, and query a relational database that solves a specific problem. This project will allow students to demonstrate their understanding of key database concepts, as well as their skills in database design and querying.

Project Requirements

  • Project Topic: Students must choose a topic for their database. It can be any topic of interest, such as a library database, an inventory management system, a social network, etc.
  • Database Design:
    • Create an entity-relationship (ER) diagram that represents the entities, relationships, and attributes of the database.
    • Normalize the tables up to the third normal form (3NF).
  • Implementation:
    • Create the tables in a database management system (DBMS) such as MySQL, PostgreSQL, or SQLite.
    • Insert test data into the tables.
  • Queries:
    • Write and execute SQL queries to retrieve information from the database.
    • Include queries that use SELECT, JOIN, GROUP BY, HAVING, and ORDER BY.

Deliverables

  1. Design Document:
    • ER Diagram.
    • Description of entities and relationships.
    • Justification of normalization.
  2. SQL Script:
    • Code to create the tables.
    • Code to insert test data.
    • SQL queries.
  3. Final Report:
    • Description of the chosen topic.
    • Explanation of the database design.
    • Results of the SQL queries.
    • Conclusions and learnings.

Project Example

Topic: Library Management System

ER Diagram

Description of Entities and Relationships

  • Entity Book:
    • Attributes: ISBN, Title, Author, Publication Year, Genre
  • Entity User:
    • Attributes: User ID, Name, Email, Registration Date
  • Entity Loan:
    • Attributes: Loan ID, User ID, ISBN, Loan Date, Return Date
  • Relationships:
    • A User can borrow multiple Books.
    • A Book can be borrowed by multiple Users at different times.

SQL Script

-- Table creation
CREATE TABLE Book (
    ISBN VARCHAR(13) PRIMARY KEY,
    Title VARCHAR(255) NOT NULL,
    Author VARCHAR(255) NOT NULL,
    Publication_Year INT,
    Genre VARCHAR(100)
);

CREATE TABLE User (
    User_ID INT PRIMARY KEY AUTO_INCREMENT,
    Name VARCHAR(255) NOT NULL,
    Email VARCHAR(255) UNIQUE NOT NULL,
    Registration_Date DATE NOT NULL
);

CREATE TABLE Loan (
    Loan_ID INT PRIMARY KEY AUTO_INCREMENT,
    User_ID INT,
    ISBN VARCHAR(13),
    Loan_Date DATE NOT NULL,
    Return_Date DATE,
    FOREIGN KEY (User_ID) REFERENCES User(User_ID),
    FOREIGN KEY (ISBN) REFERENCES Book(ISBN)
);

-- SQL Queries
-- 1. Get all currently borrowed books
SELECT B.Title, U.Name, L.Loan_Date
FROM Loan L
JOIN Book B ON L.ISBN = B.ISBN
JOIN User U ON L.User_ID = U.User_ID
WHERE L.Return_Date IS NULL;

-- 2. Count the number of loans per user
SELECT U.Name, COUNT(L.Loan_ID) AS Number_of_Loans
FROM User U
JOIN Loan L ON U.User_ID = L.User_ID
GROUP BY U.Name
ORDER BY Number_of_Loans DESC;
