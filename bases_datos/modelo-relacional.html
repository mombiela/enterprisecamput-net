Introduction

The relational model is one of the fundamental pillars in the design and management of databases. Proposed by Edgar F. Codd in 1970, this model organizes data into tables (or relations) that allow efficient manipulation and a clear structure.

Key Concepts

  • Relation: A table in a relational database.
  • Tuple: A row in a table, representing a unique record.
  • Attribute: A column in a table, representing a characteristic of the record.
  • Primary Key: An attribute or set of attributes that uniquely identify a tuple in a table.
  • Foreign Key: An attribute in a table that is a primary key in another table, used to establish relationships between tables.
  • Relation Schema: The structure of a table, defined by its attributes and their data types.

Table Structure

A table in the relational model can be visualized as a matrix of rows and columns. Each row represents a unique record and each column represents an attribute of the record.

Example of a Table

Suppose we have a table called Customers:

| CustomerID | FirstName | LastName | Email | |------------|-----------|----------|----------------------| | 1 | John | Perez | john.perez@mail.com | | 2 | Mary | Garcia | mary.garcia@mail.com | | 3 | Charles | Lopez | charles.lopez@mail.com|

  • CustomerID: Primary key.
  • FirstName: Attribute that stores the customer's first name.
  • LastName: Attribute that stores the customer's last name.
  • Email: Attribute that stores the customer's email.

Relationships between Tables

Relationships between tables are established through foreign keys. This allows data normalization and avoids redundancy.

Example of Relationships

Suppose we have another table called Orders:

| OrderID | CustomerID | Date | Total | |---------|------------|------------|--------| | 101 | 1 | 2023-01-15 | 150.00 | | 102 | 2 | 2023-01-16 | 200.00 | | 103 | 1 | 2023-01-17 | 100.00 |

  • OrderID: Primary key.
  • CustomerID: Foreign key that references CustomerID in the Customers table.
  • Date: Attribute that stores the order date.
  • Total: Attribute that stores the order total.

SQL Code Examples

Table Creation

CREATE TABLE Customers (
    CustomerID INT PRIMARY KEY,
    FirstName VARCHAR(50),
    LastName VARCHAR(50),
    Email VARCHAR(100)
);

CREATE TABLE Orders (
    OrderID INT PRIMARY KEY,
    CustomerID INT,
    Date DATE,
    Total DECIMAL(10, 2),
    FOREIGN KEY (CustomerID) REFERENCES Customers(CustomerID)
);

Data Insertion

INSERT INTO Customers (CustomerID, FirstName, LastName, Email) VALUES
(1, 'John', 'Perez', 'john.perez@mail.com'),
(2, 'Mary', 'Garcia', 'mary.garcia@mail.com'),
(3, 'Charles', 'Lopez', 'charles.lopez@mail.com');

INSERT INTO Orders (OrderID, CustomerID, Date, Total) VALUES
(101, 1, '2023-01-15', 150.00),
(102, 2, '2023-01-16', 200.00),
(103, 1, '2023-01-17', 100.00);

Basic Queries

  • Select all customers:

    SELECT * FROM Customers;

  • Select all orders of a specific customer:

    SELECT * FROM Orders WHERE CustomerID = 1;

Conclusion

The relational model is a powerful tool for organizing and managing data in databases. Understanding its basic concepts, such as relations, primary and foreign keys, and how to structure and manipulate data using SQL, is essential for any IT professional. This knowledge allows the design of efficient and scalable database systems, facilitating data integrity and consistency.
