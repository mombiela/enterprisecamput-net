Introduction to AWS Config

AWS Config is a service that enables you to assess, audit, and evaluate the configurations of your AWS resources. It helps you monitor and record your AWS resource configurations and automate the evaluation of recorded configurations against desired configurations.

Key Concepts

  • Configuration Item (CI): A record of the configuration of a resource at a given point in time.
  • Configuration Recorder: Records the configurations of the supported resources in your account.
  • Delivery Channel: Specifies where AWS Config delivers the configuration snapshots and configuration history files.
  • Configuration Snapshot: A collection of configuration items for the supported resources in your account.
  • Configuration History: A collection of configuration items for a resource over a period of time.
  • Rules: AWS Config rules represent your desired configurations for your AWS resources.

Setting Up AWS Config

To start using AWS Config, you need to set up the configuration recorder and delivery channel.

Step-by-Step Guide

  1. Open the AWS Management Console.
  2. Navigate to AWS Config.
  3. Set up the Configuration Recorder:
    • Choose the resources you want to record.
    • Specify the recording frequency (e.g., every 1 hour).
  4. Set up the Delivery Channel:
    • Choose an S3 bucket to store configuration snapshots and history.
    • Optionally, set up an SNS topic for notifications.

Example

import boto3

# Create a Config client
config = boto3.client('config')

# Set up the configuration recorder
config.put_configuration_recorder(
    ConfigurationRecorder={
        'name': 'default',
        'roleARN': 'arn:aws:iam::123456789012:role/config-role',
        'recordingGroup': {
            'allSupported': True,
            'includeGlobalResourceTypes': True
        }
    }
)

# Set up the delivery channel
config.put_delivery_channel(
    DeliveryChannel={
        'name': 'default',
        's3BucketName': 'my-config-bucket',
        'snsTopicARN': 'arn:aws:sns:us-east-1:123456789012:config-topic'
    }
)

# Start the configuration recorder
config.start_configuration_recorder(
    ConfigurationRecorderName='default'
)

AWS Config Rules

AWS Config rules allow you to define compliance guidelines for your resources. You can use AWS managed rules or create custom rules.

Managed Rules

AWS provides a set of predefined rules that you can use to evaluate whether your AWS resources comply with common best practices.

Custom Rules

You can create custom rules using AWS Lambda functions to evaluate your resources based on specific criteria.

Example: Managed Rule

config.put_config_rule(
    ConfigRule={
        'ConfigRuleName': 's3-bucket-public-read-prohibited',
        'Description': 'Checks that your S3 buckets do not allow public read access.',
        'Scope': {
            'ComplianceResourceTypes': [
                'AWS::S3::Bucket',
            ]
        },
        'Source': {
            'Owner': 'AWS',
            'SourceIdentifier': 'S3_BUCKET_PUBLIC_READ_PROHIBITED'
        }
    }
)

Example: Custom Rule

config.put_config_rule(
    ConfigRule={
        'ConfigRuleName': 'custom-rule',
        'Description': 'Custom rule to check specific compliance.',
        'Scope': {
            'ComplianceResourceTypes': [
                'AWS::EC2::Instance',
            ]
        },
        'Source': {
            'Owner': 'CUSTOM_LAMBDA',
            'SourceIdentifier': 'arn:aws:lambda:us-east-1:123456789012:function:custom-rule-function'
        }
    }
)

Monitoring and Notifications

AWS Config provides several ways to monitor and receive notifications about the compliance status of your resources.

Monitoring

  • AWS Management Console: View the compliance status of your resources.
  • AWS CLI/SDK: Use commands and API calls to retrieve compliance information.

Notifications

  • SNS Notifications: Set up SNS topics to receive notifications about configuration changes and compliance status.

Example: Retrieving Compliance Status

response = config.get_compliance_details_by_config_rule(
    ConfigRuleName='s3-bucket-public-read-prohibited'
)

for result in response['EvaluationResults']:
    print(result['ComplianceType'], result['EvaluationResultIdentifier']['EvaluationResultQualifier']['ResourceId'])

Conclusion

AWS Config is a powerful tool for managing and auditing the configurations of your AWS resources. By setting up configuration recorders, delivery channels, and rules, you can ensure that your resources comply with your organization's policies and best practices. Regular monitoring and notifications help you stay informed about the compliance status of your resources, enabling you to take corrective actions when necessary.
