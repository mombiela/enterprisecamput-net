Introduction to Kubernetes Engine

Google Kubernetes Engine (GKE) is a managed, production-ready environment for deploying containerized applications. It provides a powerful and flexible way to manage, deploy, and scale applications using Kubernetes.

Key Concepts

  • Kubernetes: An open-source system for automating deployment, scaling, and management of containerized applications.
  • GKE: A managed Kubernetes service provided by Google Cloud Platform.
  • Containers: Lightweight, standalone, executable software packages that include everything needed to run a piece of software, including the code, runtime, libraries, and dependencies.

Setting Up GKE

Prerequisites

  • A Google Cloud Platform account.
  • Basic understanding of Docker and Kubernetes.
  • Google Cloud SDK installed on your local machine.

Steps to Set Up GKE

  1. Create a GCP Project:

    gcloud projects create my-gke-project
gcloud config set project my-gke-project

  2. Enable the Kubernetes Engine API:

    gcloud services enable container.googleapis.com

  3. Create a GKE Cluster:

    gcloud container clusters create my-cluster --zone us-central1-a

  4. Get Authentication Credentials for the Cluster:

    gcloud container clusters get-credentials my-cluster --zone us-central1-a

  5. Verify the Cluster:

    kubectl get nodes

Deploying Applications on GKE

Creating a Deployment

  • Deployment YAML File:

    apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app
        image: gcr.io/my-gke-project/my-app:latest
        ports:
        - containerPort: 80

  • Apply the Deployment:

    kubectl apply -f deployment.yaml

Exposing the Deployment

  • Service YAML File:

    apiVersion: v1
kind: Service
metadata:
  name: my-app-service
spec:
  type: LoadBalancer
  selector:
    app: my-app
  ports:
  - protocol: TCP
    port: 80
    targetPort: 80

  • Apply the Service:

    kubectl apply -f service.yaml

Scaling Applications

Manual Scaling

  • Scale the Deployment: 
    kubectl scale deployment my-app --replicas=5

Autoscaling

  • Enable Autoscaling: 
    kubectl autoscale deployment my-app --cpu-percent=80 --min=1 --max=10

Monitoring and Logging

Stackdriver Integration

  • Enable Stackdriver Logging and Monitoring:

    gcloud container clusters create my-cluster --zone us-central1-a --enable-stackdriver-kubernetes

  • View Logs:

    gcloud logging read "resource.type=k8s_container AND resource.labels.cluster_name=my-cluster"

Advanced Topics

Custom Resource Definitions (CRDs)

  • Define a CRD:

    apiVersion: apiextensions.k8s.io/v1
kind: CustomResourceDefinition
metadata:
  name: crontabs.stable.example.com
spec:
  group: stable.example.com
  versions:
  - name: v1
    served: true
    storage: true
  scope: Namespaced
  names:
    plural: crontabs
    singular: crontab
    kind: CronTab
    shortNames:
    - ct

  • Apply the CRD:

    kubectl apply -f crd.yaml

Network Policies

  • Define a Network Policy:

    apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-same-namespace
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  ingress:
  - from:
    - podSelector: {}

  • Apply the Network Policy:

    kubectl apply -f network-policy.yaml

Conclusion

Google Kubernetes Engine (GKE) simplifies the deployment, management, and scaling of containerized applications using Kubernetes. By leveraging GKE, developers can focus on writing code and deploying applications without worrying about the underlying infrastructure. This course covered the basics of setting up GKE, deploying applications, scaling, monitoring, and advanced topics like CRDs and network policies. With this knowledge, you are well-equipped to manage and deploy applications on GKE effectively.
