Setting Up the Development Environment
Angular Architecture ►

Setting up the development environment is the first step to start working with Angular. This section will guide you through the necessary tools and configurations to get your Angular projects up and running.

Prerequisites

Before setting up Angular, ensure you have the following prerequisites installed on your machine:

  • Node.js and npm: Angular requires Node.js and npm (Node Package Manager) to manage dependencies and run scripts.
  • Text Editor or IDE: A good text editor or Integrated Development Environment (IDE) like Visual Studio Code, WebStorm, or Sublime Text.

Installing Node.js and npm

  1. Download and Install Node.js:

    • Visit the Node.js official website.
    • Download the LTS (Long Term Support) version for your operating system.
    • Follow the installation instructions.

  2. Verify Installation:

    • Open your terminal or command prompt.
    • Run the following commands to check the installed versions: 
      node -v
npm -v
    • You should see the version numbers of Node.js and npm.

Installing Angular CLI

The Angular CLI (Command Line Interface) is a powerful tool that helps you initialize, develop, scaffold, and maintain Angular applications.

  1. Install Angular CLI Globally:

    • Open your terminal or command prompt.
    • Run the following command to install Angular CLI globally: 
      npm install -g @angular/cli
    • This command installs the Angular CLI globally, making it accessible from any directory on your machine.

  2. Verify Installation:

    • Run the following command to check the installed version of Angular CLI: 
      ng version
    • You should see the version details of Angular CLI and its dependencies.

Creating a New Angular Project

Once the Angular CLI is installed, you can create a new Angular project using a simple command.

  1. Create a New Project:

    • Navigate to the directory where you want to create your project.
    • Run the following command: 
      ng new my-angular-app
    • Replace my-angular-app with your desired project name.
    • The CLI will prompt you with some configuration options. You can choose the default options by pressing Enter.

  2. Navigate to the Project Directory:

    • After the project is created, navigate to the project directory: 
      cd my-angular-app

  3. Serve the Application:

    • Run the following command to serve the application: 
      ng serve
    • Open your browser and navigate to http://localhost:4200/. You should see the default Angular welcome page.

Configuring the Development Environment

To enhance your development experience, consider configuring your text editor or IDE with the following tools and extensions:

  • Visual Studio Code Extensions:

    • Angular Language Service: Provides rich editing and navigation capabilities.
    • TSLint: Helps in identifying and fixing TypeScript code issues.
    • Prettier: Automatically formats your code for consistency.

  • WebStorm Plugins:

    • Angular and AngularJS: Provides support for Angular projects.
    • TypeScript: Enhances TypeScript development.

Example Configuration for Visual Studio Code

  1. Install Extensions:

    • Open Visual Studio Code.
    • Go to the Extensions view by clicking on the Extensions icon in the Activity Bar on the side of the window.
    • Search for and install the following extensions:
      • Angular Language Service
      • TSLint
      • Prettier

  2. Configure Prettier:

    • Create a .prettierrc file in the root of your project with the following content: 
      {
  "singleQuote": true,
  "trailingComma": "all"
}
    • This configuration ensures that single quotes are used and trailing commas are added where valid in ES5 (objects, arrays, etc.).

Conclusion

Setting up the development environment is a crucial step in starting your journey with Angular. By ensuring you have Node.js, npm, and Angular CLI installed, and configuring your text editor or IDE with useful extensions, you create a productive and efficient workspace. With your environment ready, you can now move on to building and developing Angular applications with ease.
