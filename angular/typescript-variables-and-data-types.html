Introduction

TypeScript is a statically typed superset of JavaScript that adds optional types, interfaces, and other features to the language. Understanding variables and data types in TypeScript is crucial for developing robust Angular applications. This section will cover the basics of declaring variables and using different data types in TypeScript.

Variables in TypeScript

Declaring Variables

In TypeScript, variables can be declared using let, const, and var. Each has its own scope and usage.

  • let: Block-scoped variable declaration.
  • const: Block-scoped constant declaration.
  • var: Function-scoped variable declaration (not recommended for modern TypeScript).

Examples:

let message: string = "Hello, TypeScript!";
const pi: number = 3.14;
var globalVar: boolean = true;

Variable Scope

  • Block Scope: Variables declared with let and const are limited to the block in which they are defined.
  • Function Scope: Variables declared with var are limited to the function in which they are defined.

Example:

function exampleFunction() {
    if (true) {
        let blockScoped = "I'm block scoped";
        var functionScoped = "I'm function scoped";
    }
    console.log(functionScoped); // Works
    // console.log(blockScoped); // Error: blockScoped is not defined
}

Data Types in TypeScript

Basic Data Types

TypeScript supports several basic data types, including:

  • string
  • number
  • boolean
  • any
  • void
  • null
  • undefined

Examples:

let name: string = "John Doe";
let age: number = 30;
let isStudent: boolean = true;
let anything: any = "Could be anything";
let noReturnValue: void = undefined;
let empty: null = null;
let notAssigned: undefined = undefined;

Arrays and Tuples

  • Arrays: Can be declared using the type[] syntax or Array<type> syntax.
  • Tuples: Allow you to express an array with a fixed number of elements whose types are known.

Examples:

let numbers: number[] = [1, 2, 3];
let strings: Array<string> = ["one", "two", "three"];
let tuple: [string, number] = ["John Doe", 30];

Enums

Enums allow you to define a set of named constants. TypeScript provides both numeric and string-based enums.

Example:

enum Color {
    Red,
    Green,
    Blue
}
let color: Color = Color.Green;

Interfaces

Interfaces define the structure of an object. They are used for type-checking and ensuring that objects adhere to a specific shape.

Example:

interface Person {
    name: string;
    age: number;
    isStudent: boolean;
}

let person: Person = {
    name: "Jane Doe",
    age: 25,
    isStudent: false
};

Type Assertions

Type assertions are a way to tell the compiler what the type of a variable is when you know more about the type than TypeScript does.

Example:

let someValue: any = "This is a string";
let strLength: number = (someValue as string).length;

Conclusion

Understanding variables and data types in TypeScript is fundamental for developing Angular applications. By mastering these concepts, you can write more robust and type-safe code. This section covered variable declarations, scopes, basic data types, arrays, tuples, enums, interfaces, and type assertions. Practice these concepts with the provided examples to solidify your understanding.
