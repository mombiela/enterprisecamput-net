Setting Up Routes
Route Parameters ►

Routing is a core feature in Angular that allows you to navigate between different views or components in your application. This section will guide you through setting up routes from a beginner to an advanced level.

Introduction to Angular Routing

Angular routing enables navigation from one view to another as users perform tasks in your application. It helps in creating Single Page Applications (SPAs) where the page does not reload but the view changes.

Key Concepts

  • Router Module: Angular's module for routing.
  • Routes: Configuration objects that define the path and the component to display.
  • Router Outlet: A directive that acts as a placeholder for the routed component.
  • Router Link: A directive to link to routes.

Setting Up Basic Routes

To get started with routing, you need to import the RouterModule and define your routes.

Step-by-Step Guide

  1. Install Angular Router (if not already installed):

    npm install @angular/router

  2. Import RouterModule in your main module (app.module.ts):

    import { RouterModule, Routes } from '@angular/router';

const routes: Routes = [
  { path: 'home', component: HomeComponent },
  { path: 'about', component: AboutComponent }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppModule { }

  3. Add Router Outlet in your main template (app.component.html):

    <nav>
  <a routerLink="/home">Home</a>
  <a routerLink="/about">About</a>
</nav>
<router-outlet></router-outlet>

Explanation

  • RouterModule.forRoot(routes): Configures the router at the application's root level.
  • routerLink: Directive to bind the anchor tag to a route.
  • router-outlet: Placeholder for the routed component.

Advanced Routing Techniques

As your application grows, you may need more advanced routing features such as nested routes, route guards, and lazy loading.

Nested Routes

Nested routes allow you to define routes within other routes.

const routes: Routes = [
  {
    path: 'dashboard',
    component: DashboardComponent,
    children: [
      { path: 'stats', component: StatsComponent },
      { path: 'reports', component: ReportsComponent }
    ]
  }
];

Route Guards

Route guards are used to control access to routes based on certain conditions.

  1. Create a Guard:

    import { Injectable } from '@angular/core';
import { CanActivate, ActivatedRouteSnapshot, RouterStateSnapshot } from '@angular/router';

@Injectable({
  providedIn: 'root'
})
export class AuthGuard implements CanActivate {
  canActivate(
    next: ActivatedRouteSnapshot,
    state: RouterStateSnapshot): boolean {
    // Add your authentication logic here
    return true; // or false based on the condition
  }
}

  2. Apply the Guard to a Route:

    const routes: Routes = [
  { path: 'admin', component: AdminComponent, canActivate: [AuthGuard] }
];

Lazy Loading

Lazy loading helps in loading feature modules only when they are needed, improving the application's performance.

  1. Create a Feature Module:

    @NgModule({
  declarations: [FeatureComponent],
  imports: [CommonModule, RouterModule.forChild([{ path: '', component: FeatureComponent }])]
})
export class FeatureModule { }

  2. Configure Lazy Loading in Routes:

    const routes: Routes = [
  { path: 'feature', loadChildren: () => import('./feature/feature.module').then(m => m.FeatureModule) }
];

Summary

Setting up routes in Angular involves configuring the RouterModule with route definitions and using directives like routerLink and router-outlet to navigate and display components. As you advance, you can implement nested routes, route guards, and lazy loading to optimize and secure your application. Understanding these concepts will help you build robust and scalable Angular applications.
