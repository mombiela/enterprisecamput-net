Introduction

The Open Web Application Security Project (OWASP) is a non-profit organization dedicated to improving software security. OWASP provides a series of resources, including documentation, tools, and standards, that are essential for IT professionals. This course is designed to offer a deep understanding of the security concepts and practices promoted by OWASP.

Course Objectives

  • Understand the fundamental principles of web application security.
  • Familiarize yourself with the tools and resources provided by OWASP.
  • Learn to identify and mitigate common vulnerabilities in web applications.
  • Apply good security practices in software development and maintenance.

Course Content

1. Introduction to OWASP

  • History and Mission of OWASP
    • Founded in 2001.
    • Mission: Improve software security.
  • Main OWASP Projects
    • OWASP Top Ten.
    • OWASP ASVS (Application Security Verification Standard).
    • OWASP ZAP (Zed Attack Proxy).

2. OWASP Top Ten

  • Overview
    • List of the ten most critical vulnerabilities in web applications.
  • Common Vulnerabilities
    • Injection (SQL, NoSQL, OS, etc.).
    • Broken Authentication.
    • Sensitive Data Exposure.
    • XML External Entities (XXE).
    • Broken Access Control.
    • Security Misconfiguration.
    • Cross-Site Scripting (XSS).
    • Insecure Deserialization.
    • Using Components with Known Vulnerabilities.
    • Insufficient Logging and Monitoring.

3. OWASP Tools

  • OWASP ZAP (Zed Attack Proxy)
    • Penetration testing tool.
    • Usage example: 
      zap.sh -daemon -port 8080 -config api.key=12345
  • OWASP Dependency-Check
    • Tool to identify vulnerabilities in dependencies.
    • Usage example: 
      dependency-check --project "My Project" --scan /path/to/project
  • OWASP ASVS (Application Security Verification Standard)
    • Standard for verifying application security.
    • Verification levels: Level 1, Level 2, Level 3.

4. Practical Workshops

  • Vulnerability Identification Workshop
    • Using OWASP ZAP to identify and exploit vulnerabilities.
  • Vulnerability Mitigation Workshop
    • Applying patches and security configurations.
    • Example of SQL Injection mitigation: 
      # Using prepared statements in Python
cursor.execute("SELECT * FROM users WHERE username = %s", (username,))
  • Best Practices Implementation Workshop
    • Implementing access controls.
    • Example of access control in a web application: 
      @PreAuthorize("hasRole('ADMIN')")
public String adminPage() {
    return "admin";
}

Conclusion

This course provides a solid foundation in the security principles and practices promoted by OWASP. Upon completion, participants will be equipped with the knowledge and skills necessary to identify and mitigate vulnerabilities in web applications, using the tools and resources provided by OWASP. Security is an ongoing process, and this course is an important step towards creating more secure and robust applications.
