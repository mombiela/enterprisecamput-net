Introduction to Intents

Intents are a fundamental concept in Android development, used for communication between components such as activities, services, and broadcast receivers. They can be used to start a new activity, send data to another activity, or even communicate with other applications.

Key Concepts

  • Intent: A messaging object you can use to request an action from another app component.
  • Explicit Intent: Specifies the component to start by name.
  • Implicit Intent: Does not name a specific component but declares a general action to perform.

Explicit Intents

Explicit intents are used to start a specific component within your application. This is useful when you know the exact activity or service you want to start.

Example

Intent intent = new Intent(this, SecondActivity.class);
intent.putExtra("EXTRA_MESSAGE", "Hello, Second Activity!");
startActivity(intent);
  • Intent(this, SecondActivity.class): Creates an explicit intent to start SecondActivity.
  • putExtra("EXTRA_MESSAGE", "Hello, Second Activity!"): Adds extra data to the intent.
  • startActivity(intent): Starts the activity specified in the intent.

Implicit Intents

Implicit intents do not specify a component; instead, they declare an action to perform. The Android system matches the intent with the appropriate component.

Example

Intent intent = new Intent(Intent.ACTION_VIEW);
intent.setData(Uri.parse("http://www.example.com"));
startActivity(intent);
  • Intent(Intent.ACTION_VIEW): Creates an implicit intent with the action ACTION_VIEW.
  • setData(Uri.parse("http://www.example.com")): Sets the data for the intent.
  • startActivity(intent): Starts an activity that can handle the ACTION_VIEW action for the given URI.

Intent Filters

Intent filters are declarations in the AndroidManifest.xml file that specify the types of intents a component can respond to. They are used to register activities, services, and broadcast receivers for specific actions, data types, and categories.

Key Concepts

  • Action: The general action to perform (e.g., ACTION_VIEW, ACTION_SEND).
  • Data: The data type and URI the intent can handle.
  • Category: Additional information about the action (e.g., CATEGORY_DEFAULT, CATEGORY_BROWSABLE).

Example of an Intent Filter

<activity android:name=".SecondActivity">
    <intent-filter>
        <action android:name="android.intent.action.VIEW" />
        <category android:name="android.intent.category.DEFAULT" />
        <data android:scheme="http" android:host="www.example.com" />
    </intent-filter>
</activity>
  • <action android:name="android.intent.action.VIEW" />: Specifies the action VIEW.
  • <category android:name="android.intent.category.DEFAULT" />: Specifies the default category.
  • <data android:scheme="http" android:host="www.example.com" />: Specifies the data scheme and host.

Practical Exercises

Exercise 1: Create an Explicit Intent

  1. Create a new activity named DetailActivity.
  2. In your main activity, create an explicit intent to start DetailActivity.
  3. Pass a string message to DetailActivity using putExtra.
  4. Display the message in DetailActivity.

Exercise 2: Create an Implicit Intent

  1. Create an implicit intent to open a web page.
  2. Use Intent.ACTION_VIEW and set the data to a URL.
  3. Test the intent to ensure it opens the web browser with the specified URL.

Exercise 3: Define an Intent Filter

  1. Modify your AndroidManifest.xml to add an intent filter for DetailActivity.
  2. Configure the intent filter to handle the action ACTION_VIEW and a custom scheme myapp.
  3. Test the intent filter by creating an implicit intent with the custom scheme.

Conclusion

Intents and intent filters are powerful tools in Android development, enabling seamless communication between components and applications. By understanding and utilizing explicit and implicit intents, as well as properly defining intent filters, you can create more dynamic and interactive Android applications. Practice with the provided exercises to solidify your understanding and enhance your skills.
