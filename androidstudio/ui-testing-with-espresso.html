Introduction to UI Testing

  • What is UI Testing?

    • UI Testing ensures that the user interface of an application works as expected.
    • It involves testing the visual elements and user interactions.

  • Why Use Espresso for UI Testing?

    • Espresso is a testing framework for Android that makes it easy to write reliable UI tests.
    • It is part of the Android Testing Support Library.
    • Espresso tests run on actual devices and emulators.

Setting Up Espresso in Android Studio

  • Adding Dependencies

    • Open your build.gradle file and add the following dependencies: 
      androidTestImplementation 'androidx.test.espresso:espresso-core:3.4.0'
androidTestImplementation 'androidx.test.ext:junit:1.1.3'
androidTestImplementation 'androidx.test:rules:1.4.0'

  • Syncing the Project

    • After adding the dependencies, sync your project to download the necessary libraries.

Writing Your First Espresso Test

  • Creating a Test Class

    • Create a new test class in the androidTest directory: 
      package com.example.myapp;

import androidx.test.ext.junit.runners.AndroidJUnit4;
import androidx.test.rule.ActivityTestRule;

import org.junit.Rule;
import org.junit.Test;
import org.junit.runner.RunWith;

import static androidx.test.espresso.Espresso.onView;
import static androidx.test.espresso.action.ViewActions.click;
import static androidx.test.espresso.matcher.ViewMatchers.withId;

@RunWith(AndroidJUnit4.class)
public class MainActivityTest {

    @Rule
    public ActivityTestRule<MainActivity> activityRule =
            new ActivityTestRule<>(MainActivity.class);

    @Test
    public void testButtonClick() {
        onView(withId(R.id.my_button)).perform(click());
    }
}

  • Explanation

    • @RunWith(AndroidJUnit4.class): Specifies that the test should run using the AndroidJUnit4 test runner.
    • ActivityTestRule: Launches the specified activity before each test and closes it after each test.
    • onView(withId(R.id.my_button)): Finds the view with the specified ID.
    • perform(click()): Simulates a click action on the found view.

Advanced Espresso Concepts

  • Matchers

    • Matchers are used to find views in the UI.
    • Common matchers include:
      • withId(int id): Finds a view by its ID.
      • withText(String text): Finds a view by its text.
      • isDisplayed(): Finds views that are currently displayed.

  • View Actions

    • Actions are operations that can be performed on views.
    • Common actions include:
      • click(): Clicks on a view.
      • typeText(String text): Types text into a view.
      • scrollTo(): Scrolls to a view.

  • View Assertions

    • Assertions are used to verify the state of views.
    • Common assertions include:
      • matches(Matcher<View> matcher): Checks if a view matches a given condition.
      • doesNotExist(): Checks if a view does not exist.

Example: Testing a Login Screen

  • Test Class

    package com.example.myapp;

import androidx.test.ext.junit.runners.AndroidJUnit4;
import androidx.test.rule.ActivityTestRule;

import org.junit.Rule;
import org.junit.Test;
import org.junit.runner.RunWith;

import static androidx.test.espresso.Espresso.onView;
import static androidx.test.espresso.action.ViewActions.click;
import static androidx.test.espresso.action.ViewActions.typeText;
import static androidx.test.espresso.matcher.ViewMatchers.withId;
import static androidx.test.espresso.matcher.ViewMatchers.withText;
import static androidx.test.espresso.assertion.ViewAssertions.matches;

@RunWith(AndroidJUnit4.class)
public class LoginActivityTest {

    @Rule
    public ActivityTestRule<LoginActivity> activityRule =
            new ActivityTestRule<>(LoginActivity.class);

    @Test
    public void testLoginSuccess() {
        onView(withId(R.id.username)).perform(typeText("user"));
        onView(withId(R.id.password)).perform(typeText("password"));
        onView(withId(R.id.login_button)).perform(click());
        onView(withText("Login Successful")).check(matches(isDisplayed()));
    }

    @Test
    public void testLoginFailure() {
        onView(withId(R.id.username)).perform(typeText("user"));
        onView(withId(R.id.password)).perform(typeText("wrongpassword"));
        onView(withId(R.id.login_button)).perform(click());
        onView(withText("Login Failed")).check(matches(isDisplayed()));
    }
}

  • Explanation

    • testLoginSuccess(): Tests a successful login scenario.
    • testLoginFailure(): Tests a failed login scenario.
    • typeText("text"): Types text into the specified view.
    • check(matches(isDisplayed())): Asserts that a view is displayed.

Conclusion

  • Summary

    • Espresso is a powerful tool for UI testing in Android.
    • It provides a simple API for interacting with and verifying the state of UI components.
    • By mastering Espresso, you can ensure that your application's UI behaves as expected, providing a better user experience.

  • Next Steps

    • Practice writing more complex tests.
    • Explore additional Espresso features such as Idling Resources and custom matchers.
    • Integrate Espresso tests into your CI/CD pipeline to automate UI testing.
