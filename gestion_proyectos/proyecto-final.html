Introduction

The Final Project is an integral part of the Project Management course. This project aims to consolidate the knowledge acquired throughout the course by practically applying project management methodologies and tools. Students will work in teams to plan, execute, and close a simulated project, following industry best practices.

Final Project Objectives

  • Apply the project management concepts and techniques learned during the course.
  • Develop teamwork and effective communication skills.
  • Manage a project from start to finish, including planning, execution, monitoring, and closing.
  • Use project management tools to document and control project progress.

Project Description

The project involves creating a web application for task management. Students must follow all phases of a project lifecycle, from initiation to closure. The phases and expected deliverables are detailed below:

Initiation Phase

  • Scope Definition: Describe the project's objective, main deliverables, and success criteria.
  • Stakeholder Identification: List all stakeholders and their roles in the project.
  • Project Charter: Create a formal document that authorizes the project and describes its objectives and constraints.

Planning Phase

  • Project Management Plan: Develop a detailed plan that includes the schedule, budget, resources, and risks.
  • Work Breakdown Structure (WBS): Create a WBS to break down the project into more manageable tasks.
  • Project Schedule: Use tools like Gantt or PERT to plan the time for each task.

Execution Phase

  • Application Development: Implement the web application following the project specifications.
  • Team Management: Assign tasks and responsibilities to team members.
  • Communication: Maintain effective communication among team members and stakeholders.

Monitoring and Control Phase

  • Progress Tracking: Use project management tools to monitor project progress.
  • Risk Management: Identify and mitigate risks that may affect the project.
  • Quality Control: Ensure that deliverables meet established quality standards.

Closing Phase

  • Application Delivery: Present the completed web application to stakeholders.
  • Final Documentation: Compile all project documentation, including the final report and lessons learned.
  • Project Evaluation: Conduct a retrospective to evaluate the team's and project's overall performance.

Code Example

Below is a code example for a part of the task management web application. This example shows how to create a task using an HTML form and process it with JavaScript.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Task Management</title>
</head>
<body>
    <h1>Create New Task</h1>
    <form id="taskForm">
        <label for="taskName">Task Name:</label>
        <input type="text" id="taskName" name="taskName" required>
        <br>
        <label for="taskDescription">Description:</label>
        <textarea id="taskDescription" name="taskDescription" required></textarea>
        <br>
        <button type="submit">Add Task</button>
    </form>

    <script>
        document.getElementById('taskForm').addEventListener('submit', function(event) {
            event.preventDefault();
            const taskName = document.getElementById('taskName').value;
            const taskDescription = document.getElementById('taskDescription').value;
            console.log(`Task created: ${taskName} - ${taskDescription}`);
            // Here you could add logic to save the task to a database
        });
    </script>
</body>
</html>

Conclusion

Through this project, students will not only reinforce their technical and management skills but also develop key competencies such as teamwork and effective communication. Upon completing the project, students will be better prepared to face real-world challenges in project management.
